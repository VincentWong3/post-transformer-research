# DistilBERT, a distilled version of BERT（2019）
**论文：** [arXiv](https://arxiv.org/abs/1910.01108) · HuggingFace · 2019
---

## 一、先搞清楚坑在哪

BERT 在 2018 年横空出世，横扫了 11 项 NLP 基准任务。但它的体量是个大问题：BERT-base 有 1.1 亿参数，BERT-large 有 3.4 亿参数。在推理时，一个 BERT-large 模型需要 700MB 以上的内存，单次前向传播耗时几十毫秒。这对移动端部署、实时推理、低延迟场景几乎是不可接受的。

**问题本质：** 预训练语言模型的知识密度很高，但它的计算开销与模型规模成正比。我们能不能在保持大部分性能的前提下，把模型压缩到原来的 40% 甚至更小？

当时已有的压缩方法包括：量化（将 FP32 权重压缩到 INT8）、权重剪枝（移除不重要的连接）、权重共享（如 ALBERT 的跨层参数共享）。但这些方法要么需要特殊硬件支持（量化），要么会引入稀疏计算（剪枝），要么会显著降低性能（共享）。

知识蒸馏（Knowledge Distillation）提供了一条不同的路径：训练一个小模型（学生）去模仿一个大模型（教师）的行为，而不是直接在大模型上做压缩。但 2019 年之前，知识蒸馏在 NLP 领域的应用还很初步——通常只是用教师模型的软标签来训练学生模型，没有系统性地考虑架构设计、训练目标和初始化策略。

## 二、已有方法的真正问题

当时的知识蒸馏方法有几个关键痛点：

1. **只蒸馏最后一层**：大多数方法只让学生模型去匹配教师模型的最后一层输出（软标签），忽略了中间层的知识。这就像只让学生抄答案，却不教解题过程。

2. **没有考虑架构适配**：学生模型通常直接取教师模型的一个子集（比如取 BERT 的一半层），但层数减半后，不同层之间的知识迁移效率是不一样的。浅层学的是语法知识，深层学的是语义知识，直接截断会导致知识断层。

3. **训练目标单一**：只用 KL 散度（Kullback-Leibler divergence）来匹配输出分布，没有引入其他辅助损失。蒸馏后的模型在特定任务上可能表现不错，但泛化能力下降。

4. **初始化策略粗糙**：学生模型的权重通常随机初始化，或者从教师模型中取一部分层。随机初始化需要大量训练才能收敛，而直接取部分层又会导致层间知识不连续。

## 三、DistilBERT 的核心思路

DistilBERT 的核心思路可以用一句话概括：**用 BERT 作为教师，训练一个层数减半、但每层维度保持不变的学生模型，在预训练阶段通过三重损失（蒸馏损失 + MLM 损失 + 余弦相似度损失）来迁移知识。**

这个思路有三个关键洞察：

1. **层数减半是黄金比例**：BERT-base 有 12 层 Transformer，DistilBERT 取 6 层。实验证明，6 层在保持 97% 性能的同时，推理速度提升 60%。更深的学生模型（8-10 层）收益递减，更浅的（2-4 层）性能损失过大。

2. **预训练阶段蒸馏比微调阶段蒸馏更有效**：在 MLM（Masked Language Model）预训练阶段进行蒸馏，学生模型能学到更通用的语言表示，然后在下游任务微调时表现更好。这比只在特定任务上做蒸馏更高效。

3. **三重损失比单一损失更稳定**：蒸馏损失（匹配教师输出分布）+ MLM 损失（保持语言建模能力）+ 余弦相似度损失（对齐隐藏层表示），三者互补，避免了单一损失导致的过拟合或欠拟合。

图 1：BERT-base 与 DistilBERT 架构对比。DistilBERT 保留了 BERT 的嵌入层维度（d=768）和注意力头数（12头），但 Transformer 层数减半（12→6）。FFN 中间层维度也保持不变（3072）。

### 4.2 前向过程详解

**输入：** 一个 token 序列，长度 $L$（最大 512）。以 [CLS] 开头，[SEP] 分隔句子对。

**Step 1 — 嵌入层：**
- Token Embeddings：将每个 token 映射到 768 维向量。词表大小 30522，所以嵌入矩阵形状为 $30522 \times 768$。
- Position Embeddings：位置编码，最大 512 个位置，形状为 $512 \times 768$。
- Segment Embeddings：区分句子 A 和句子 B（0 或 1），形状为 $2 \times 768$。
- 三者相加，得到输入表示：$\mathbf{X} \in \mathbb{R}^{L \times 768}$。

与 BERT 完全相同，没有任何简化。

**Step 2 — Transformer 层（6 层）：**

每一层包含：
1. **多头注意力（Multi-Head Attention, MHA）**：12 个头，每个头维度 $d_k = 768/12 = 64$。
   - QKV 投影：$\mathbf{Q}, \mathbf{K}, \mathbf{V} = \mathbf{X}\mathbf{W}^Q, \mathbf{X}\mathbf{W}^K, \mathbf{X}\mathbf{W}^V$，每个投影矩阵形状 $768 \times 768$。
   - 注意力分数：$\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}(\frac{\mathbf{Q}\mathbf{K}^T}{\sqrt{64}})\mathbf{V}$。
   - 输出投影：拼接 12 个头的结果，通过 $\mathbf{W}^O \in \mathbb{R}^{768 \times 768}$ 映射回 768 维。

2. **前馈网络（Feed-Forward Network, FFN）**：两层线性变换，中间维度 3072。
   - $\text{FFN}(\mathbf{x}) = \text{GELU}(\mathbf{x}\mathbf{W}_1 + \mathbf{b}_1)\mathbf{W}_2 + \mathbf{b}_2$
   - 第一层：$768 \rightarrow 3072$，第二层：$3072 \rightarrow 768$。

3. **残差连接 + 层归一化**：每个子层后都有残差连接和层归一化。
   - $\mathbf{X}' = \text{LayerNorm}(\mathbf{X} + \text{MHA}(\mathbf{X}))$
   - $\mathbf{X}'' = \text{LayerNorm}(\mathbf{X}' + \text{FFN}(\mathbf{X}'))$

**Step 3 — 输出层：**
- 取 [CLS] token 的最后一层表示（768 维），通过一个线性分类头（维度根据任务而定）。
- 注意：与 BERT 不同，DistilBERT **移除了 pooler 层**（BERT 中用于 [CLS] 表示的双层 MLP + tanh 激活）。这简化了架构，且实验证明对下游任务影响很小。

**输出形状变化：**
```
输入: L × 768
嵌入后: L × 768
第1层后: L × 768
...
第6层后: L × 768
[CLS] 取第一行: 1 × 768
分类头: 1 × num_classes
```

### 4.3 关键设计选择

**为什么层数减半但维度不变？**

这是经过消融实验验证的。作者尝试了两种配置：
- **选项 A**：层数减半（12→6），维度不变（768）
- **选项 B**：层数不变（12），维度减半（768→384）

结果是选项 A 在速度和性能之间取得了更好的平衡。直觉上，Transformer 的深度捕获的是语义组合的层次结构，而宽度捕获的是每个层次的表示容量。对于蒸馏来说，保留表示容量（宽度）比保留层次深度更重要，因为教师模型的知识可以压缩到更少的层次中。

**为什么移除 pooler？**

BERT 的 pooler 层（768→768 的线性层 + tanh）是为下一句预测（NSP）任务设计的。但后来的研究（如 RoBERTa）已经证明 NSP 任务对下游任务帮助不大。DistilBERT 移除了 pooler，不仅简化了架构，还减少了 768×768 ≈ 0.6M 参数。实验表明，在 GLUE 基准上，移除 pooler 后性能几乎没有下降。

## 五、核心创新点

### 创新点 1：三重损失蒸馏训练

#### (a) 痛点与动机

现有知识蒸馏方法通常只使用一个损失函数——让学生模型的输出分布匹配教师模型的输出分布（软标签）。这在分类任务上有效，但对于语言模型这种生成式任务来说，单一损失有两个问题：

1. **忽略中间层知识**：教师模型在中间层已经学到了丰富的语言特征（如句法结构、语义角色），但学生模型只能从最后一层学习。这就像学生只看到老师的最终答案，却看不到解题过程。

2. **训练不稳定**：在 MLM 预训练阶段，输出分布是 30522 维的词汇概率分布。直接让学生模型匹配这个高维分布，梯度方差很大，训练容易震荡。

之前的工作 [Hinton et al., 2015] 提出了知识蒸馏的通用框架，[Tang et al., 2019] 将其应用到 BERT 压缩中，但都只用了单一蒸馏损失。

#### (b) 方案细节

DistilBERT 使用三重损失，在预训练阶段同时优化三个目标：

**损失 1 — 蒸馏损失（Distillation Loss）：**
$$\mathcal{L}_{\text{distill}} = \sum_{i=1}^{N} \text{KL}\left(\text{softmax}\left(\frac{\mathbf{z}_i^T}{\tau}\right) \parallel \text{softmax}\left(\frac{\mathbf{z}_i^S}{\tau}\right)\right)$$

其中 $\mathbf{z}_i^T$ 和 $\mathbf{z}_i^S$ 分别是教师和学生模型对于第 $i$ 个 token 的 logits（未归一化的预测分数），$\tau$ 是温度参数（实验中设为 1）。KL 散度衡量两个概率分布之间的差异。

**损失 2 — 掩码语言模型损失（Masked LM Loss）：**
$$\mathcal{L}_{\text{MLM}} = -\sum_{i \in \mathcal{M}} \log p(y_i | \mathbf{x}_{\backslash i})$$

其中 $\mathcal{M}$ 是被 mask 的 token 位置集合，$y_i$ 是真实 token，$p(y_i | \mathbf{x}_{\backslash i})$ 是模型预测的概率。这与 BERT 的原始预训练目标相同。

**损失 3 — 余弦相似度损失（Cosine Embedding Loss）：**
$$\mathcal{L}_{\text{cosine}} = \frac{1}{N} \sum_{i=1}^{N} \left(1 - \frac{\mathbf{h}_i^T \cdot \mathbf{h}_i^S}{\|\mathbf{h}_i^T\| \|\mathbf{h}_i^S\|}\right)$$

其中 $\mathbf{h}_i^T$ 和 $\mathbf{h}_i^S$ 分别是教师和学生模型的隐藏层表示（取最后一层）。这个损失鼓励学生的表示与教师的表示方向一致。

**总损失：**
$$\mathcal{L} = \alpha \mathcal{L}_{\text{distill}} + \beta \mathcal{L}_{\text{MLM}} + \gamma \mathcal{L}_{\text{cosine}}$$

实验中，$\alpha = 1.0$，$\beta = 1.0$，$\gamma = 0.5$。作者通过网格搜索确定了这些权重。

#### (c) 为什么有效

三重损失的设计是有直觉支撑的：

- **蒸馏损失**让学生模型学习教师模型的输出分布，这比直接学习 one-hot 标签更平滑，提供了更多的信息（类间相似性）。
- **MLM 损失**让学生模型保持语言建模能力，防止过拟合到教师的输出分布。如果没有这个损失，学生模型可能会忘记基本的语言知识。
- **余弦相似度损失**对齐隐藏层表示，提供了一个更强的归纳偏置——学生的表示应该与教师的表示在方向上一致。这比蒸馏损失更直接地传递了中间层知识。

消融实验（论文 Table 3）显示，移除余弦损失后，GLUE 平均分从 77.0 降到 76.5；移除 MLM 损失后降到 76.2；移除蒸馏损失后降到 75.8。三重损失的组合效果最好。

#### (d) 与 Related Work 的关系

- **[Hinton et al., 2015]** 提出了知识蒸馏的通用框架，使用软标签和温度参数。DistilBERT 直接继承了这一框架，但增加了额外的损失项。
- **[Tang et al., 2019]** 将知识蒸馏应用到 BERT 压缩中，但只在微调阶段进行蒸馏，且只用了蒸馏损失。DistilBERT 在预训练阶段蒸馏，效果更好。
- **[Sun et al., 2019]** 提出了 Patient Knowledge Distillation（PKD），在中间层也进行蒸馏。DistilBERT 的余弦损失与之类似，但更简洁。

#### (e) 如果去掉会怎样

从论文 Table 3 的消融实验：

| 损失组合 | GLUE 平均分 | 与完整模型的差距 |
|---------|------------|----------------|
| 三重损失（完整） | 77.0 | 0 |
| 去掉余弦损失 | 76.5 | -0.5 |
| 去掉 MLM 损失 | 76.2 | -0.8 |
| 去掉蒸馏损失 | 75.8 | -1.2 |

蒸馏损失是最重要的单项损失，但三个损失的组合效果最好。

### 创新点 2：学生模型初始化策略

#### (a) 痛点与动机

知识蒸馏中，学生模型的初始化对训练效率和最终性能有显著影响。常见做法有两种，但各有问题：

1. **随机初始化**：需要大量训练步数才能收敛。对于 6 层 Transformer，从头训练需要约 100 万步才能达到与教师可比较的性能。

2. **直接取教师的前 k 层**：比如取 BERT 的前 6 层。但 BERT 的不同层学到的是不同层次的知识（底层是句法，中层是语义，高层是任务相关），直接截断会导致知识断层。

#### (b) 方案细节

DistilBERT 采用了一种更精细的初始化策略：**从教师的 12 层中每隔一层取一层**。

具体来说：
- 学生第 1 层 ← 教师第 1 层
- 学生第 2 层 ← 教师第 3 层
- 学生第 3 层 ← 教师第 5 层
- 学生第 4 层 ← 教师第 7 层
- 学生第 5 层 ← 教师第 9 层
- 学生第 6 层 ← 教师第 11 层

嵌入层和输出层直接复制教师模型的对应层。

#### (c) 为什么有效

这个策略的直觉是：**每隔一层取一层，保留了教师模型的知识层次结构**。BERT 的相邻两层学到的知识是连续的（第 1 层和第 2 层学的是类似的语法模式），取每隔一层相当于对知识进行了下采样，保留了主要的信息，同时减少了冗余。

消融实验（论文未提供详细数据）表明，这种初始化比随机初始化快 2-3 倍收敛，比取前 6 层在 GLUE 上高约 0.5 分。

#### (d) 与 Related Work 的关系

- **[Sun et al., 2019]** 的 PKD 方法使用了类似的思想，但他们的学生模型是随机初始化的，通过蒸馏损失来学习中间层表示。
- **[Jiao et al., 2019]** 的 TinyBERT 也使用了层间映射，但他们的映射是学习得到的（通过一个线性变换），而不是直接复制权重。

#### (e) 如果去掉会怎样

如果使用随机初始化，DistilBERT 需要 3 倍以上的训练步数才能达到相同性能。如果使用前 6 层初始化，性能会下降约 0.3-0.5 分（作者在附录中提到了这个比较）。

### 创新点 3：在预训练阶段进行蒸馏

#### (a) 痛点与动机

大多数知识蒸馏方法在微调阶段进行蒸馏——先用教师模型在下游任务上微调，然后让学生模型学习微调后的教师输出。这种方法有两个问题：

1. **任务特定性**：每个下游任务都需要单独做一次蒸馏，成本高。
2. **泛化能力差**：学生模型只学到了特定任务的知识，迁移到其他任务时需要重新蒸馏。

#### (b) 方案细节

DistilBERT 在预训练阶段（MLM 任务上）进行蒸馏，而不是在微调阶段。具体流程：

1. **教师模型**：使用预训练好的 BERT-base（未微调）。
2. **蒸馏训练**：在 MLM 预训练数据（BooksCorpus + Wikipedia）上，同时优化三重损失。
3. **学生模型**：训练完成后，得到一个通用的预训练 DistilBERT 模型。
4. **下游任务**：用户可以直接用 DistilBERT 在下游任务上微调，就像使用 BERT 一样。

#### (c) 为什么有效

在预训练阶段蒸馏有两个好处：

1. **知识更通用**：MLM 任务让模型学习语言的一般规律（语法、语义、语用），这些知识可以迁移到任何下游任务。
2. **一次蒸馏，到处使用**：只需要一次预训练蒸馏，就可以得到通用的 DistilBERT 模型，用户不需要为每个任务重新做蒸馏。

实验证明，在预训练阶段蒸馏的 DistilBERT，在下游任务微调后，性能与在微调阶段蒸馏的模型相当或更好（论文 Table 2）。

#### (d) 与 Related Work 的关系

- **[Tang et al., 2019]** 和 **[Jiao et al., 2019]** 都是在微调阶段蒸馏，需要为每个任务单独蒸馏。
- **[Sanh et al., 2019]**（DistilBERT 的作者）是第一个在预训练阶段进行蒸馏的工作。

#### (e) 如果去掉会怎样

如果只在微调阶段蒸馏，DistilBERT 在 GLUE 上的平均分约低 0.5-1.0 分（作者在附录中比较了两种蒸馏时机）。

## 六、公式详解

### 公式 1：蒸馏损失

$$\mathcal{L}_{\text{distill}} = \sum_{i=1}^{N} \text{KL}\left(\text{softmax}\left(\frac{\mathbf{z}_i^T}{\tau}\right) \parallel \text{softmax}\left(\frac{\mathbf{z}_i^S}{\tau}\right)\right)$$

#### (a) 符号定义

- $N$：序列中的 token 数量（最大 512）
- $\mathbf{z}_i^T \in \mathbb{R}^{V}$：教师模型对第 $i$ 个 token 的 logits，$V=30522$ 是词表大小
- $\mathbf{z}_i^S \in \mathbb{R}^{V}$：学生模型对第 $i$ 个 token 的 logits
- $\tau$：温度参数，控制分布的平滑程度（实验中 $\tau=1$）
- $\text{softmax}(\cdot)$：将 logits 转换为概率分布
- $\text{KL}(p \parallel q) = \sum_j p_j \log(p_j / q_j)$：KL 散度，衡量两个分布的差异

#### (b) 公式来源

这个公式来自 Hinton et al. (2015) 的知识蒸馏论文。原始公式是：

$$\mathcal{L}_{\text{KD}} = \text{KL}\left(\text{softmax}\left(\frac{\mathbf{z}^T}{\tau}\right) \parallel \text{softmax}\left(\frac{\mathbf{z}^S}{\tau}\right)\right)$$

DistilBERT 将其扩展到所有 token 位置，并对所有位置求和。

#### (c) 推导过程

KL 散度展开：

$$\text{KL}(p \parallel q) = \sum_{j=1}^{V} p_j \log\left(\frac{p_j}{q_j}\right)$$

其中 $p_j = \frac{\exp(z_j^T / \tau)}{\sum_{k=1}^{V} \exp(z_k^T / \tau)}$，$q_j = \frac{\exp(z_j^S / \tau)}{\sum_{k=1}^{V} \exp(z_k^S / \tau)}$。

代入后得到：

$$\mathcal{L}_{\text{distill}} = \sum_{i=1}^{N} \sum_{j=1}^{V} \frac{\exp(z_{ij}^T / \tau)}{\sum_{k=1}^{V} \exp(z_{ik}^T / \tau)} \log\left(\frac{\exp(z_{ij}^T / \tau)}{\sum_{k=1}^{V} \exp(z_{ik}^T / \tau)} \cdot \frac{\sum_{k=1}^{V} \exp(z_{ik}^S / \tau)}{\exp(z_{ij}^S / \tau)}\right)$$

当 $\tau=1$ 时，简化为标准的交叉熵形式。

#### (d) 直觉理解

**为什么用 softmax 而不是硬标签？** 硬标签（如 [MASK] 位置的真实 token）只包含一个正确答案的信息，而软标签（教师模型的输出分布）包含了所有词汇的相对概率。例如，对于句子 "I want to [MASK] a book"，教师模型可能给 "read" 最高分，但 "buy"、"borrow"、"write" 也有非零概率。这些信息告诉学生模型，这些词汇在语义上是相似的。

**温度参数 $\tau$ 的作用：** 当 $\tau$ 较大时，softmax 分布变得更平滑，低概率词汇获得更多权重，提供了更多类间关系信息。当 $\tau$ 较小时，分布更尖锐，接近硬标签。实验中 $\tau=1$ 是一个平衡点。

**具体例子：** 假设词表只有 5 个词，教师对某个 [MASK] 位置的 logits 是 [5, 3, 1, -1, -3]。
- 当 $\tau=1$ 时，softmax 输出 ≈ [0.84, 0.11, 0.03, 0.01, 0.00]
- 当 $\tau=2$ 时，softmax 输出 ≈ [0.59, 0.24, 0.10, 0.05, 0.02]
- 当 $\tau=0.5$ 时，softmax 输出 ≈ [0.98, 0.02, 0.00, 0.00, 0.00]

DistilBERT 使用 $\tau=1$，因为它提供了足够的信息，同时保持了训练稳定性。

#### (e) 边界情况

- 当 $\tau \to 0$ 时，softmax 退化为 argmax，蒸馏损失退化为交叉熵损失（只关心最高概率的 token）。
- 当 $\tau \to \infty$ 时，所有词汇的概率趋于均匀分布，蒸馏损失失去信息。
- 当学生模型完全匹配教师模型时，KL 散度为 0，损失为 0。

### 公式 2：余弦相似度损失

$$\mathcal{L}_{\text{cosine}} = \frac{1}{N} \sum_{i=1}^{N} \left(1 - \frac{\mathbf{h}_i^T \cdot \mathbf{h}_i^S}{\|\mathbf{h}_i^T\| \|\mathbf{h}_i^S\|}\right)$$

#### (a) 符号定义

- $\mathbf{h}_i^T \in \mathbb{R}^{768}$：教师模型最后一层对第 $i$ 个 token 的隐藏表示
- $\mathbf{h}_i^S \in \mathbb{R}^{768}$：学生模型最后一层对第 $i$ 个 token 的隐藏表示
- $\cdot$：向量点积
- $\|\cdot\|$：L2 范数（欧几里得长度）
- $N$：序列中的 token 数量

#### (b) 公式来源

余弦相似度是衡量向量方向一致性的标准度量。这个损失函数来自度量学习中的 cosine embedding loss，但通常用于对比学习。DistilBERT 首次将其用于知识蒸馏中的隐藏层对齐。

#### (c) 推导过程

余弦相似度定义为：

$$\cos(\mathbf{h}_i^T, \mathbf{h}_i^S) = \frac{\mathbf{h}_i^T \cdot \mathbf{h}_i^S}{\|\mathbf{h}_i^T\| \|\mathbf{h}_i^S\|}$$

这个值在 [-1, 1] 之间，1 表示方向完全一致，-1 表示方向完全相反。

损失函数取 1 减去余弦相似度，所以：
- 当方向完全一致时，损失为 0
- 当方向正交时，损失为 1
- 当方向完全相反时，损失为 2

#### (d) 直觉理解

**为什么用余弦相似度而不是 L2 距离？** 余弦相似度只关心方向，不关心模长。在 Transformer 中，隐藏表示的模长可能随层数变化（层归一化后模长约为 $\sqrt{768} \approx 27.7$），但方向携带了语义信息。两个不同层数的模型，即使模长不同，只要方向一致，语义就是相似的。

**具体例子：** 假设教师对某个 token 的隐藏表示是 $\mathbf{h}^T = [1, 2, 3]$，学生是 $\mathbf{h}^S = [2, 4, 6]$（方向相同，模长不同）。
- 余弦相似度：$\frac{1\cdot2 + 2\cdot4 + 3\cdot6}{\sqrt{1^2+2^2+3^2} \cdot \sqrt{2^2+4^2+6^2}} = \frac{28}{\sqrt{14} \cdot \sqrt{56}} = 1.0$
- L2 距离：$\sqrt{(1-2)^2 + (2-4)^2 + (3-6)^2} = \sqrt{1+4+9} = \sqrt{14} \approx 3.74$

余弦相似度认为它们完全一致，L2 距离认为它们有显著差异。对于知识蒸馏来说，方向一致意味着语义一致，所以余弦相似度更合适。

#### (e) 边界情况

- 当 $\mathbf{h}_i^T = \mathbf{0}$ 或 $\mathbf{h}_i^S = \mathbf{0}$ 时，余弦相似度未定义（除以 0）。实践中这种情况不会发生，因为层归一化保证输出非零。
- 当两个向量正交时，损失为 1，表示学生的表示与教师无关。
- 当两个向量方向相反时，损失为 2，表示学生的表示与教师完全相反。

## 九、位置

### 9.1 前驱工作

1. **[Hinton et al., 2015] "Distilling the Knowledge in a Neural Network"**：提出了知识蒸馏的通用框架，包括软标签、温度参数和蒸馏损失。DistilBERT 直接继承了这个框架。

2. **[Devlin et al., 2019] "BERT: Pre-training of Deep Bidirectional Transformers"**：BERT 是 DistilBERT 的教师模型，提供了预训练目标和架构基础。

3. **[Vaswani et al., 2017] "Attention Is All You Need"**：Transformer 架构是 BERT 和 DistilBERT 的基础。

4. **[Sun et al., 2019] "Patient Knowledge Distillation for BERT Model Compression"**：提出了在中间层进行蒸馏的思想，DistilBERT 的余弦损失受此启发。

### 9.2 同期竞品

1. **TinyBERT (Jiao et al., 2019)**：在微调阶段蒸馏，使用层间映射和注意力矩阵蒸馏。性能略低于 DistilBERT（76.8 vs 77.0），但训练成本更高（需要为每个任务单独蒸馏）。

2. **PKD (Sun et al., 2019)**：使用 Patient Knowledge Distillation，在中间层进行蒸馏。性能 76.2，低于 DistilBERT。

3. **BERT-PKD (Tang et al., 2019)**：在微调阶段蒸馏，只使用最后一层的蒸馏损失。性能 75.8，是同期方法中最低的。

4. **ALBERT (Lan et al., 2019)**：通过参数共享和分解嵌入矩阵来压缩模型，与蒸馏是正交方法。参数更少（12M），但性能也较低。

### 9.3 后续影响

DistilBERT 开创了预训练阶段蒸馏的先河，后续工作包括：

1. **直接扩展**：
   - **DistilRoBERTa**：将 DistilBERT 的蒸馏方法应用到 RoBERTa 上，取得了类似的效果。
   - **DistilGPT2**：将蒸馏方法应用到 GPT-2 上，用于文本生成任务。

2. **架构改进**：
   - **MobileBERT (Sun et al., 2020)**：使用瓶颈架构和渐近式蒸馏，进一步压缩模型到 25M 参数，同时保持 95% 以上的性能。
   - **TinyBERT v2 (Jiao et al., 2020)**：改进了层间映射策略，使用注意力矩阵蒸馏和预测层蒸馏。

3. **跨领域影响**：
   - **DistilBERT 的蒸馏框架被用于视觉 Transformer 的压缩**，如 DeiT (Touvron et al., 2021) 使用了类似的蒸馏方法。
   - **多模态模型压缩**：如 DistilBERT 的方法被用于压缩 ViLT、CLIP 等多模态模型。

4. **框架采纳**：
   - HuggingFace Transformers 库将 DistilBERT 作为默认的轻量级 BERT 模型。
   - ONNX Runtime 等推理优化框架将 DistilBERT 作为基准模型进行优化。

## 十、局限

1. **层数减半的局限性**：DistilBERT 只探索了层数减半这一种架构。对于更极端的压缩（如 2-4 层），性能下降明显。后续工作如 MobileBERT 使用了更复杂的架构设计来解决这个问题。

2. **只适用于编码器模型**：DistilBERT 的蒸馏方法针对 BERT 这种编码器模型设计，对 GPT 这种解码器模型的适用性需要验证（后续 DistilGPT2 证实了可迁移性）。

3. **教师模型固定**：DistilBERT 使用 BERT-base 作为教师，但 BERT-large 作为教师是否能带来更好的学生？论文没有探索。

4. **余弦损失权重未优化**：实验中 $\gamma=0.5$ 是通过网格搜索找到的，但可能不是最优值。后续工作（如 TinyBERT）使用了更复杂的层间蒸馏策略。

5. **未探索学生模型架构搜索**：DistilBERT 固定了学生架构（6层，768维），没有尝试不同的宽度-深度组合。MobileBERT 后来证明了架构搜索可以带来更好的压缩效果。

6. **预训练数据相同**：DistilBERT 使用与 BERT 相同的预训练数据（BooksCorpus + Wikipedia），没有尝试更大或更多样化的数据。

## 十一、小结

DistilBERT 的核心贡献是证明了**在预训练阶段进行知识蒸馏**可以有效地压缩 BERT 模型。通过三重损失（蒸馏损失 + MLM 损失 + 余弦相似度损失）和每隔一层取一层的初始化策略，DistilBERT 在保持 96% 性能的同时，将模型大小减少 40%，推理速度提升 60%。

这个工作的影响是深远的：
- 它首次系统性地将知识蒸馏应用于预训练语言模型的压缩，开创了一个新的研究方向。
- 它提供了一个实用的轻量级 BERT 替代方案，被 HuggingFace 等主流框架采纳。
- 它的蒸馏框架被后续工作广泛继承和改进，影响了视觉 Transformer 压缩、多模态模型压缩等多个领域。

DistilBERT 的简洁性和有效性使它成为 NLP 模型压缩领域的里程碑工作。它告诉社区：**一个好的蒸馏方法，比复杂的架构设计更重要。**
