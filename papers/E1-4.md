## 一、先搞清楚"问题在哪里"

2017 年 Transformer 横空出世之后，整个 NLP 领域几乎被它"统治"了。BERT、GPT、XLNet 这些模型一个接一个刷新榜单，而且拼模型大小的趋势越来越夸张。到 2019 年左右，最大的 Transformer 层参数已经超过 5 亿（Shazeer et al., 2018），层数也堆到了 64 层（Al-Rfou et al., 2018）。

但与此同时，一个实际问题开始卡脖子：**显存不够用**。

并不是说单个模型参数量太大——0.5B 的参数大约占 2GB，64K tokens × 1024 维度 × batch 8 的激活值也才 2GB，按理说单卡应该扛得住。那问题出在哪？

出在三个"放大系数"上：

1. **层数 N 倍增**：标准 Transformer 训练时，每一层的中间激活值都要保存在显存里，反向传播才能用。N 层就有 N 份激活值，显存消耗随层数线性增长。
2. **FFN 中间维度 d_ff 膨胀**：Transformer 的前馈层中间维度 d_ff 通常是 d_model 的 4 倍（比如 1024→4096），这部分的激活值占比远高于 attention 部分。
3. **Attention 的 O(L²) 复杂度**：QK^T 矩阵的大小是 L×L，序列长度 L=64K 时，光这个矩阵就占 16GB（32-bit float），单卡根本放不下。

这三个因素叠加，导致大模型长序列训练成了一个"只有大厂才玩得起"的事。甚至有人开始质疑这种趋势是否在破坏 NLP 研究的可复现性。Reformer 这篇论文要回答的问题就是：**大 Transformer 是真的需要这么多资源，还是仅仅因为实现不够高效？**

## 二、旧路线的真正问题

要理解 Reformer 的价值，得先搞清楚标准 Transformer 的显存消耗到底是怎么爆掉的。

### 问题 1：层数 N 带来的显存线性倍增

反向传播需要中间激活值来计算梯度。标准做法是前向传播时把所有层的激活值都存下来，反向传播时逐层读取。N 层模型就要存 N 份激活值。如果每层需要 M 字节，总显存就是 N×M。

这其实很浪费——在标准残差网络 y = x + F(x) 中，恢复 x 需要知道 F(x)，而 F(x) 的计算又需要 x，形成循环依赖。这正是 RevNet（可逆残差网络）要解决的问题。

### 问题 2：FFN 的中间激活膨胀

Transformer 的 FFN（前馈网络）是一个两层 MLP，中间维度 d_ff 通常是 d_model 的 4 倍：

$$\text{FFN}(x) = \text{ReLU}(xW_1 + b_1)W_2 + b_2$$

这个中间的 ReLU 激活值维度是 batch × length × d_ff，远超 attention 层的激活值。在 d_ff=4096 时，这部分占的显存比 attention 部分还大。

### 问题 3：Attention 的 O(L²) 黑洞

这是最致命的问题。标准 attention 计算 softmax(QK^T/√d_k)V，其中 QK^T 是一个 L×L 的矩阵。

L 稍微大一点就扛不住：
- L=1024: QK^T 占用 4MB（1K×1K×4 bytes），还好
- L=8192: QK^T 占用 256MB，开始吃紧
- L=65536: QK^T 占用 16GB（64K×64K×4 bytes），batch=1 都扛不住

即使有 memory-efficient attention 的实现方式（每个 query 单独算，不完整物化 QK^T 矩阵），时间复杂度依然是 O(L²)，序列一长就跑不动了。

### 这三个问题叠加的结果

一个 16 层 Transformer，d_model=1024，d_ff=4096，L=64K，batch=8——训练时的显存需求轻松超过 32GB。这还没算参数本身的存储。

所以当时大家的做法要么是缩短序列（比如 BERT 限制 512 tokens），要么是用模型并行把不同层分到不同 GPU 上。但这都只是回避问题，不是解决问题。

## 三、这篇论文的核心思路

Reformer 的核心思路是一句话：**从 attention 计算和激活存储两个维度同时下手，把 Transformer 从"显存饥渴"变成"显存友好"**。

具体来说，作者做了两件事和一个辅助优化：

1. **LSH Attention（局部敏感哈希注意力）**：用哈希把 query 和 key 分到同一个桶里，每个 query 只和同桶的 key 算 attention。复杂度从 O(L²) 降到 O(L log L)。
2. **Reversible Layers（可逆残差层）**：借鉴 RevNet 的思路，让每一层的激活值可以在反向传播时从下一层恢复出来，从而只需存储一份激活值，N 倍因子消失。
3. **Chunked FFN（分块前馈层）**：把 FFN 的中间计算按序列位置分块处理，每次只处理一部分 token，降低显存峰值。

这三个技术合起来就是 Reformer 的全貌。注意它们解决的是不同维度的问题：LSH 砍 attention 的 O(L²)；Reversible 砍 N 倍激活存储；Chunking 砍 d_ff 的中间膨胀。

作者的洞察在于：**这三个问题是正交的，可以独立解决**。标准 Transformer 把这些问题绑在一起，Reformer 把它们拆开，各自找一个高效的替代方案。

把这篇论文放在"E1. 降低 attention 复杂度"的技术树里，它属于"高效 attention 范式"这条分支：它没有改动 attention 的数学定义本身，而是用近似方法降低 attention 的计算范围。

## 四、整体结构与 forward 流程

Reformer 的整体 forward 流程如下：

```text
输入序列（长度 L）
  ↓
Token embedding → 位置编码
  ↓
对每个可逆块（共 N 个块）：
    ↓
    x1, x2 ← 拆分为两部分（各 d_model/2）
    ↓
    y1 = x1 + Attention(x2)       ← LSH Attention
    ↓
    y2 = x2 + FeedForward(y1)     ← Chunked FFN
    ↓
    输出 (y1, y2) 作为下一块的输入
  ↓
输出层（LayerNorm + Linear）
  ↓
Softmax → 预测下一个 token
```

### 几个关键的设计点：

1. **可逆块替代标准残差块**：输入是一个 pair (x1, x2)，输出也是一个 pair (y1, y2)。而不是标准 Transformer 的单路输入输出。

2. **LSH Attention 替代标准 Dot-Product Attention**：不再计算完整的 QK^T 矩阵，而是先哈希排序，再在局部范围内做 attention。

3. **FFN 分块计算**：输入 y1 先按序列维度切分成 c 个 chunk，每个 chunk 独立过 FFN，再拼回去。数学上等价于一次性计算，但显存峰值大幅降低。

4. **Q=K 共享**：为了让 LSH 能正常工作，作者强制让 query 和 key 来自同一个线性投影，并对 key 做长度归一化。

5. **自注意力禁止**：由于 Q=K，query 和自己的 dot product 几乎总是最大，所以要禁止 token 注意自己。

### Tensor shape 变化（以单层为例，batch=1）：

```text
输入: [L, d_model]
  ↓ 拆分为 x1, x2
x1: [L, d_model/2], x2: [L, d_model/2]
  ↓
Attention(x2): [L, d_model/2]
  ↓
y1 = x1 + Attention(x2): [L, d_model/2]
  ↓
FeedForward(y1) → chunked into c chunks: [L/c, d_ff] → [L/c, d_model/2]
  ↓ 收集所有 chunk 并拼接
FFN output: [L, d_model/2]
  ↓
y2 = x2 + FFN(y1): [L, d_model/2]
  ↓
输出 (y1, y2) 拼接: [L, d_model]
```

## 五、核心模块逐个解释

### 5.1 LSH Attention——用哈希替代暴力搜索

#### 问题
标准 attention 要算每个 query 对所有 key 的 attention 权重。但实际上 softmax 的输出主要由最大的几个值主导——离 query 远的 key 贡献近乎为零。

所以理论上，每个 query 只需要关注距离最近的几十个 key 就够了。问题是：**怎么高效找到每个 query 最近的那些 key？**

#### 解决方案：局部敏感哈希（Locality Sensitive Hashing, LSH）

LSH 是一类哈希函数，满足一个性质：**距离近的向量以高概率落入同一个桶，距离远的向量以高概率落入不同桶**。这样只需要在同一个桶里做 attention 即可。

作者使用的是 angular LSH 方案（Andoni et al., 2015）：

1. 先固定一个随机投影矩阵 R，尺寸 [d_k, b/2]（b 是期望的桶数）
2. 对每个向量 x，计算 h(x) = argmax([xR; -xR])
3. 这相当于把向量投影到 b/2 个随机方向上，然后看它在哪个方向上的分量最大

这个过程可以理解为：在高维空间里随机放一些"切割面"，每个向量的哈希值就是它落在哪个"扇区"里。

#### 让 LSH 能用在 Attention 上

要把 LSH 嵌入 attention，需要解决几个工程问题：

**问题一：query 和 key 必须落在同一个桶里才有 attention**

但标准的 LSH 是对 query 和 key 分别哈希的，没有保证它们会落入同一桶。作者的解决方法是：**强制 Q=K**。即 query 和 key 来自同一个线性投影，并且对 key 做长度归一化（k_j = q_j / ||q_j||）。这样 query 和 key 的哈希值天然相等。

**问题二：桶大小不均匀**

有的桶可能有几百个元素，有的只有一两个。这给 batch 计算带来困难。作者的解决方法是：按桶编号排序，然后以固定大小（m=2L/n_buckets）的 chunk 为单位做 attention，每个 chunk 内的 token 互相 attention，再加一个 chunk 的回溯。

```text
排序前: [q1, q2, q3, q4, q5, q6, q7, q8]
          桶A  桶B  桶A  桶C  桶B  桶A  桶C  桶B
排序后: [q1, q3, q6, q2, q5, q8, q4, q7]
          桶A  桶A  桶A  桶B  桶B  桶B  桶C  桶C
分块:  [chunk1]   [chunk2]   [chunk3]  chunk...
```

每个 chunk 里的 token 互相 attention，同时也 attention 前一个 chunk 的 token。这样保证了同桶内的 token 都能互相看到。

#### 完整 LSH Attention 流程

1. 输入 Q=K 和 V
2. 对每个 query/key 计算 LSH 哈希值 h(q_i)
3. 按哈希值排序（同桶内按原始位置排序）
4. 将排序后的序列分成大小为 m 的 chunk
5. 每个 query 只关注同 chunk 和上一个 chunk 的 key
6. 计算 attention（带 mask，防止看到 future 和自己）

### 5.2 Reversible Layers——不存激活值也能反向传播

#### 问题

标准残差网络：y = x + F(x)。前向传播时要把 x 存下来，反向传播时才知道怎么算梯度。N 层就要存 N 份 x。

#### 解决方案：RevNet（Gomez et al., 2017）

可逆残差网络的核心思想是让每一层的输出足以恢复出输入。具体来说：

标准 RevNet 对 pair 输入 (x1, x2) 做变换：

$$y_1 = x_1 + F(x_2)$$
$$y_2 = x_2 + G(y_1)$$

反向传播时不需要存储中间激活值，直接从输出恢复输入：

$$x_2 = y_2 - G(y_1)$$
$$x_1 = y_1 - F(x_2)$$

#### Reformer 中的可逆 Transformer

作者把 RevNet 应用到 Transformer 上：

- F 替换为 Attention
- G 替换为 FeedForward
- LayerNorm 移入残差块内部

具体：
$$Y_1 = X_1 + \text{Attention}(X_2)$$
$$Y_2 = X_2 + \text{FeedForward}(Y_1)$$

这意味着在反向传播时，只需要保存最后一层的激活值 (Y_1, Y_2)，之前的每一层都可以从后往前逐层恢复出来。

**显存收益**：标准 Transformer 的激活存储和层数 N 成正比，可逆 Transformer 的激活存储和层数无关——不管多少层，只存一份。

### 5.3 Chunked FFN——分块处理降低峰值

FFN 中间层 d_ff 通常远大于 d_model，比如 d_model=1024 时 d_ff=4096。这意味着 FFN 的中间激活维度是 [batch, L, 4096]，远高于 attention 层的 [batch, L, 1024]。

但 FFN 有一个很好的性质：**每个位置的 FFN 计算是独立的，完全不依赖其他位置**。这意味着可以按序列位置分块计算：

1. 把序列分成 c 个 chunk，每个 chunk 包含 L/c 个位置
2. 对每个 chunk 独立过 FFN
3. 把所有 chunk 的结果拼回去

数学上等价于一次性计算所有位置，但显存峰值从 O(L × d_ff) 降到了 O(L/c × d_ff)。

作者在实验中使用了固定分块大小（如每块约 128 个位置），将 FFN 中间层分块处理以降低显存峰值。

## 六、核心公式与数学直觉

### 6.1 LSH Attention 的数学形式

标准 attention（单 query 位置 i）：

$$o_i = \sum_{j \in P_i} \exp(q_i \cdot k_j - z(i, P_i)) \, v_j$$

其中 $P_i = \{j: i \ge j\}$ 是 query i 可以 attend 到的所有 key 的位置集合（因为是 causal LM，只能看过去），$z(i, P_i)$ 是 softmax 的归一化项。

实际上为了 batch 计算，通常是对一个更大的集合 $\tilde{P}_i$ 做 attention，然后用 mask 把不在 $P_i$ 内的位置遮掉：

$$o_i = \sum_{j \in \tilde{P}_i} \exp(q_i \cdot k_j - m(j, P_i) - z(i, P_i)) \, v_j$$

其中 mask m 对不允许的位置取 $\infty$，使得 exp(-∞)=0。

LSH Attention 的核心变化是**重新定义了 $P_i$**：

$$P_i = \{j : h(q_i) = h(k_j)\}$$

即 query i 只 attend 到和它哈希值相同的 key。由于强制 Q=K 且 k_j = q_j / ||q_j||，所以 h(q_i) = h(k_j) 简化为 h(q_i) = h(q_j)。

### 6.2 排序后的分块 attention

直接按哈希桶做 attention 的问题是桶大小不均匀，不好 batch。所以作者先排序再分块：

排序后的位置映射：$i \mapsto s_i$（按桶编号排序后的新位置索引）

然后定义 attention 范围：

$$\tilde{P}_i = \left\{ j : \left\lfloor\frac{s_i}{m}\right\rfloor - 1 \le \left\lfloor\frac{s_j}{m}\right\rfloor \le \left\lfloor\frac{s_i}{m}\right\rfloor \right\}$$

这个式子的意思是：排序后，query i 在 chunk $\lfloor s_i/m \rfloor$ 里，它可以 attention 到同一个 chunk 和前一个 chunk 的所有 key。

**关键直觉**：如果 $m$ 设置得合适（$m = 2L / n_{\text{buckets}}$），那么每个桶平均大小就是 $m/2$，所以每个 query 的"真实邻居"大概率在 ±m 范围内。$m$ 不能太小，否则会漏掉同桶的 key。

### 6.3 多轮哈希（Multi-round LSH）

单轮哈希有一个风险：距离近的点也可能因为"运气不好"落入不同桶。为了降低这种概率，作者引入了多轮哈希：

$$P_i = \bigcup_{r=1}^{n_{\text{rounds}}} P_i^{(r)}$$

其中 $P_i^{(r)} = \{j : h^{(r)}(q_i) = h^{(r)}(q_j)\}$

即用 n_rounds 个不同的随机投影矩阵分别做 LSH，取所有桶的并集。每轮哈希都产生一组 candidate，query 可以 attend 到任意一轮中和自己同桶的 key。

这里要注意去重：如果一个 key 在多轮哈希中都属于 query 的有效 attention 集合，它不应该被重复计算。作者通过一个 $N_{i,j}$ 因子来处理——具体来说，mask 中加入了一个 $\log N_{i,j}$ 项来控制重复计数：

$$\text{softmax}(q_i \cdot k_j - \log N_{i,j} - m_{i,j})$$

其中 $N_{i,j} = |\{r : j \in P_i^{(r)}\}|$ 是 key j 在几轮哈希中被选中的次数。

### 6.4 可逆层的反向传播还原

标准残差层：

$$y = x + F(x)$$
$$\frac{\partial L}{\partial x} = \frac{\partial L}{\partial y} \cdot \left(I + \frac{\partial F}{\partial x}\right)$$

反向传播时需要 x（已存储）和 F 的 Jacobian（可以重新计算）。

可逆层（只写出其中一路为例）：

$$y_1 = x_1 + F(x_2)$$
$$y_2 = x_2 + G(y_1)$$

需要 $\partial L / \partial x_1$ 和 $\partial L / \partial x_2$ 时，先恢复 x：

$$x_2 = y_2 - G(y_1)$$
$$x_1 = y_1 - F(x_2)$$

然后照常算梯度。注意这里 $G(y_1)$ 的计算量和前向完全一样，所以反向传播时多了一个 $G$ 和 $F$ 的重新计算开销。

**这就是显存和时间的 trade-off**：可逆层用额外的重计算换取不用存储中间激活值。

## 七、重要图表解释

### 7.1 图 1：Angular LSH 的二维可视化

论文 Figure 1 展示了 angular LSH 的核心机制。在一个简化的 2D 空间中，每个点（向量）表示一个 query/key。随机投影用几条穿过原点的直线表示，直线的方向由随机矩阵 R 的行决定。

```text
        x2 ↑          LSH 切分示意
           │    ●        ← 每个点代表一个 query/key
           │  ●   ●
           │ ●     ●
     ──────┼──────────→ x1
           │    ●
           │  ●   ●
           │●     ●
           │
    三个随机方向 = 
    六角形扇区划分
```

图中展示的核心信息是：**两个距离近的点（x 和 y），在多次随机投影下有高概率落入同一扇区；距离远的点则低概率落入同一扇区**。这正是 LSH 能用于近似最近邻搜索的理论基础。

### 7.2 图 2：LSH Attention 的完整流程

论文 Figure 2 是 LSH Attention 最核心的图示，分四个子图展示了从标准 attention 到 LSH attention 的演变：

**(a) 标准 Full Attention**

一个 L×L 的 attention 矩阵，每个 query 对每个 key 计算权重。虽然实际 attention 权重是稀疏的（大部分接近 0），但计算时没有利用这种稀疏性。

```text
 K
  → [▮▮▮▮▮▮▮▮]   ← 所有 query × 所有 key
Q    ▮▮▮▮▮▮▮▮    O(L²) 计算
  → [▮▮▮▮▮▮▮▮]   显存瓶颈
```

**(b) 按哈希桶排序后的 Attention**

query 和 key 按哈希值排序后，同桶的 token 聚在 attention 矩阵的对角线附近。此时 attention 矩阵变成稀疏块对角形式。

**(c) 分块 Attention**

排序后按固定大小 m 分块，每个 query 只 attend 到同块和前一块：

```text
K (排序后)
  → [■■■□□□□□]   ← chunk1 内
     [■■■□□□□□]
     [■■■□□□□□]
     [□□□■■■□□]   ← chunk2 内
Q    [□□□■■■□□]
(排序) [□□□■■■□□]
  → [□□□□□□■■■]  ← chunk3 内
```

**(d) 最终 LSH Attention 的计算模式**

每个 chunk（长度为 m）内的 query 互相 attention，再加一个 chunk 的回溯。这确保了同桶中的 token 虽然可能分布在相邻的 chunk 中，但仍然能互相 attention。

### 7.3 Table 1：各 attention 变体的复杂度对比

| Attention 类型 | 显存复杂度 | 时间复杂度 |
|---|---|---|
| Standard Dot-Product | O(b n_h l d_k, b n_h l²) | O(b n_h l d_k, b n_h l²) |
| Memory-Efficient | O(b n_h l d_k, b n_h l²) | O(b n_h l d_k, b n_h l²) |
| LSH Attention | O(b n_h l d_k, b n_h l n_r (4l/n_c)²) | O(b n_h l d_k, b n_h n_r l (4l/n_c)²) |

其中 b=batch，n_h=head 数，l=序列长度，n_r=hash 轮数，n_c=chunk 数。

关键差异：Standard 的 L² 项在 LSH 中被替换成了 n_r × (4l/n_c)²，当 n_r=8、n_c=l/32 时，(4l/n_c)² = 128²，远小于 l²。

### 7.4 Table 3：完整 Reformer 的复杂度

| 模型类型 | 显存复杂度 | 时间复杂度 |
|---|---|---|
| Transformer | O(b l d_ff, b n_h l²) × n_l | O(b l d_ff + b n_h l²) × n_l |
| Reversible Transformer | O(b l d_ff, b n_h l²) | O(b l d_ff + b n_h l²) × n_l |
| + Chunked | O(b l d_model, b n_h l²) | 同上 |
| + LSH | O(b l d_ff, b n_h l n_r c) × n_l | O(b l d_ff + b n_h n_r l c) × n_l |
| **Reformer** | **O(b l d_model, b n_h l n_r c)** | **O(b l d_ff + b n_h n_r l c)** |

注意几个关键变化：
- Standard Transformer：显存和层数 n_l 成正比（乘在括号外面）
- Reversible Transformer：去掉了 n_l 因子
- Chunked FFN：d_ff 降为 d_model
- LSH attention：l² 降为 n_r × c（c 是常数）
- **Reformer 完全体**：显存完全不依赖层数和序列长度的平方

## 八、复杂度、效率与可扩展性分析

### 8.1 Attention 复杂度对比

标准 attention 和 LSH attention 的核心差异：

- 标准：$O(L^2)$ 时间和显存
- Memory-efficient：$O(L^2)$ 时间，$O(L)$ 显存（逐 query 计算）
- LSH：$O(L \log L)$ 时间，$O(L)$ 显存

注意 memory-efficient attention 虽然降了显存，但没有降时间——它只是不在内存里完整物化 QK^T 矩阵，但每个 query 仍然要扫描所有 key。

### 8.2 显存节省的具体来源

**来源 1：Reversible Layers → 砍掉 N 倍因子**

标准 Transformer 16 层需要 16 份激活值。Reformer 只需要 1 份（最后一份）。如果每层激活值占 1GB，就从 16GB 降到 1GB。

**来源 2：Chunked FFN → 砍掉 d_ff 膨胀**

d_ff=4096，分 32 个 chunk 后，每个 chunk 的序列长度从 L 降到 L/32，中间激活形状从 [batch, L, d_ff] 降到 [batch, L/32, d_ff]（d_ff 保持不变）。但这需要在时间上多做一次 loop（32 次循环），属于时间换空间的典型 trade-off。

**来源 3：LSH Attention → 砍掉 O(L²)**

L=64K 时，标准 attention 的 QK^T 矩阵是 64K×64K=4B 个元素≈16GB。LSH attention 的 chunk size m=128，每个 chunk 内的 attention 是 128×128=16K≈64KB，整个序列是 64K/128=512 个 chunk，总显存约 32MB。差三个数量级。

### 8.3 时间效率

LSH attention 的时间复杂度 $O(L \log L)$ 来自哈希的计算（O(Ldkb)）和分块 attention（O(L × m²)），其中 m 是常数。

论文 Figure 5（右图）显示：在固定总 token 数的条件下，标准 attention 的耗时随序列长度增加而增长，LSH attention 的耗时基本保持平直。这是因为固定 token 数意味着 L 增大时 batch 减小，但 attention 的 O(L²) 仍然导致标准 attention 变慢，而 LSH 不受影响。

### 8.4 新引入的瓶颈

Reformer 并非没有代价：

1. **多轮哈希的计算开销**：n_rounds 倍的计算量。论文表格显示，n_rounds=8 时接近标准 attention 的精度，但也意味着 8 倍的哈希计算。
2. **可逆层的重计算**：反向传播时多算一次 Attention 和 FFN，可逆层通过重计算节省显存，但需要额外的反向传播计算，是一种时间-显存的权衡。
3. **Q=K 带来的约束**：某些任务可能需要分开的 query 和 key 空间，共享 QK 可能不是普遍适用的。

## 九、实验结果

### 9.1 合成验证：序列复制任务

作者设计了一个合成任务来验证 LSH attention 的有效性：重复序列 0w0w，其中 w 是 1 到 127 的符号序列。

例子：`0 19 113 72 0 19 113 72`

这个任务要完美解决必须做非局部的 attention lookup（第二个 0 要 attention 到第一个 0 后面的第一个位置才能正确预测），所以任何依赖局部稀疏 attention 的方法都解决不了。

结果（1 层 Transformer，d_model=d_ff=256，4 heads，150K steps）：

| 评估/训练 | Full Attn | LSH-8 | LSH-4 | LSH-2 | LSH-1 |
|---|---|---|---|---|---|
| Full Attn | 100% | — | — | — | — |
| LSH-8 | 94.8% | 100% | 99.9% | 99.9% | 99.6% |
| LSH-4 | 92.5% | 99.9% | 99.9% | 99.6% | 76.9% |
| LSH-2 | — | 99.4% | 98.1% | 94.8% | 52.5% |
| LSH-1 | — | 91.9% | 86.8% | 77.9% | — |

几个关键观察：
- 用 full attention 训练的模型可以直接切换到 LSH attention（eval），但精度有损失（94.8%→52.5% 取决于 n_rounds）
- 用 LSH-4 从头训练的模型几乎达到完美精度（99.9%），甚至用 LSH-8 评估可达 100%
- 用 LSH-1 训练的模型只有 77.9%，但用 LSH-8 评估时提升到 91.9%
- **说明多轮哈希在训练时帮助模型学到更好的表示，在评估时增加 n_rounds 也能提升精度**

### 9.2 真实数据：enwik8 字符级语言模型

- 数据集：enwik8（Wikipedia 文本的字符级建模）
- 序列长度：$2^{16} = 65536$ tokens
- 模型配置：d_model=1024, d_ff=4096, n_heads=8, batch=8
- 层数：3 层（消融实验），12-20 层（大规模实验）

**Shared-QK 消融**（Figure 3 左图）：共享 query-key 空间的标准 Transformer 和正常 Transformer 在学习曲线上几乎重叠，在 enwik8 上甚至略快一些。结论是共享 QK 对精度没有负面影响。

**Reversible layers 消融**（Figure 3 右图）：可逆 Transformer 和标准 Transformer 的训练曲线几乎完全一致。结论是可逆层不影响模型精度。

**LSH attention 精度 vs n_rounds**（imagenet-64 实验，Figure 4）：n_rounds=8 时 LSH attention 已经几乎匹配 full attention。n_rounds 越多越接近 full attention。

**大规模 Reformer**：12 层 Reformer，训练 20K steps，dropout 0.1，在 enwik8 测试集上达到 1.19 bits/dim。经过更长时间训练和更多优化后达到 1.05 bits/dim。

### 9.3 图像生成：imagenet-64

- 序列长度：12K（将 64×64 图像按像素序列处理）
- 结果和 enwik8 类似，LSH attention 在 n_rounds=8 时基本匹配 full attention

### 9.4 机器翻译：WMT14 英德翻译

| 模型 | BLEU |
|---|---|
| Vaswani et al. (2017) base | 27.3 |
| Vaswani et al. (2017) big | 28.4 |

这里只用了可逆层（没有用 LSH attention），因为翻译的句子通常很短（WMT14 测试集所有句子都短于 128 个 token）。结果说明可逆层在标准翻译任务上也没有精度损失。

```python
def chunked_ffn(x, chunk_size=128):
    # x: [batch, length, d_model]
    # 分块
    length = x.shape[1]
    n_chunks = length // chunk_size
    
    outputs = []
    for i in range(n_chunks):
        chunk = x[:, i*chunk_size:(i+1)*chunk_size, :]
        # 对每一小块过 FFN
        hidden = relu(dense(chunk, d_ff))
        out = dense(hidden, d_model)
        outputs.append(out)
    
    return concat(outputs, axis=1)
```

## 十一、Related Work 与技术谱系位置

### 11.1 技术谱系

Reformer 属于"高效 Transformer"这条技术路线，具体来说它受以下工作影响：

```text
Transformer (Vaswani et al., 2017)
    │
    ├──→ Sparse Transformer (Child et al., 2019)
    │     结构化稀疏 attention
    │
    ├──→ RevNet (Gomez et al., 2017)
    │     可逆残差网络
    │
    └──→ LSH (Andoni et al., 2015)
          局部敏感哈希 · 近似最近邻搜索
              │
              ↓
          Reformer (Kitaev, Kaiser, Levskaya, 2020)
              │
              ├──→ Longformer (Beltagy et al., 2020)
              │    滑动窗口 + 全局 attention
              │
              └──→ BigBird (Zaheer et al., 2020)
                    随机 + 滑动 + 全局 attention
```

### 11.2 和 Sparse Transformer 的关系

Sparse Transformer（Child et al., 2019）是先驱，它用固定的稀疏 attention 模式（步长模式 + 局部窗口）来降低 O(L²)。但 Sparse Transformer 的稀疏模式是固定的，不依赖数据内容。Reformer 的 LSH attention 是**数据依赖的稀疏**——用哈希动态决定每个 query 应该关注哪些 key。

这是两者最本质的区别：Sparse Transformer 是"固定的稀疏"，Reformer 是"内容感知的稀疏"。

### 11.3 和 RevNet 的关系

可逆残差不是 Reformer 的原创——Gomez et al.（2017）在图像分类上就提出了 RevNet。Reformer 的贡献在于把 RevNet 的思路适配到了 Transformer 上。适配本身并不 trivial：LayerNorm 的位置要调整，attention 和 FFN 要分别分配到 F 和 G 中。

### 11.4 和 LSH 的关系

LSH 也不是 Reformer 的原创，它在近似最近邻搜索领域有几十年的研究历史。Reformer 的贡献在于把 LSH 用到了 attention 里——**这不是单纯的"拿来就用"**，需要解决 Q=K 共享、排序分块、因果掩码、多轮哈希去重等一系列工程问题。

### 11.5 Reformer 之后的注意力稀疏化工作

Reformer 之后，Longformer（Beltagy et al., 2020）和 BigBird（Zaheer et al., 2020）也提出了各自的稀疏 attention 方案。这些工作在 Reformer 的基础上进一步发展了"如何让 attention 变稀疏"这一命题。

## 十三、范式意义

Reformer 的贡献可以分为两个层级：

### Attention 层面：近似 attention 范式

Reformer 不是第一个做高效 attention 的（Sparse Transformer 更早），但它是第一个把 LSH 引入 attention 的。这开创了"基于哈希的稀疏 attention"这个子方向。

本质上，LSH attention 提供了一个框架：

1. 用一个可学习的或固定的函数把 query-key 空间分桶
2. 只做桶内的 attention

这个框架后来被进一步扩展：一些工作用聚类代替哈希（Clusterformer），一些工作用学习到的路由代替哈希（Routing Transformer），还有一些工作用可微哈希代替随机投影。

### 架构层面：可逆 Transformers

Reformer 把 RevNet 引入 Transformer，证明了在大语言模型上使用可逆层不会损失精度。这个发现的影响远远超出 Reformer 本身——后来的很多大模型训练都在使用类似的技术（gradient checkpointing 是更通用的形式，它的设计哲学和可逆层一脉相承：用时间换显存）。

### 范式级别定位

从"范式变化还是工程增强"的角度看，Reformer 更接近**工程增强+架构改进**：

- LSH Attention 不是 attention 的数学重定义（那是 kernel attention 做的事），而是在工程上改变了 attention 的计算方式
- Reversible Layers 也不是创新性的架构变化，而是现有技术的适配
- 但整套方案的组合效果——让 64K 序列的单卡训练成为可能——是实际且有影响的

在技术树中，Reformer 是一个**桥梁节点**：它连接了"Transformer 在短序列上效果好"和"在长序列上也能跑"这两个状态。它不是终点，但让后续的长序列模型研究有了一个可靠基线和可行的观察窗口。

## 十四、局限性与后续方向

### 14.1 LSH Attention 的局限性

**局限性 1：Q=K 共享不一定总是好选择**

强制 query 和 key 来自同一线性空间，对某些任务可能不是最优的。论文在 enwik8 上证明了 QK 共享不会降低精度，甚至略有提升，但这个结论不一定在所有任务上都成立。

**局限性 2：LSH 的近似误差**

无论多少轮哈希，LSH attention 始终是 full attention 的近似。论文实验显示 n_rounds=8 时基本匹配，但 8 轮哈希意味着 8 倍的计算量。如果任务对 attention 质量极其敏感（比如某些需要精确匹配的任务），LSH attention 可能不够用。

**局限性 3：多轮哈希的去重机制增加了复杂性**

多轮哈希中，同一个 key 可能被多个哈希函数选中，需要额外的去重机制。虽然论文给出了一个优雅的 $\log N_{i,j}$ 解法，但实现起来比标准 attention 复杂得多。

**局限性 4：短序列不划算**

LSH 的哈希计算和排序本身也有开销。对于短序列（比如 WMT 翻译任务，大多小于 128 tokens），这些开销可能超过节省。所以 Reformer 在翻译任务上只用了可逆层，没用 LSH attention。

### 14.2 Reversible Layers 的局限性

**局限性 1：时间换空间**

可逆层在反向传播时需要重新计算 Attention 和 FFN，比标准层多一次前向计算。在显存不是瓶颈的情况下，这反而更慢。

**局限性 2：硬件亲和性**

可逆层需要逐层恢复和计算梯度，这个流程不容易在模型并行或多 GPU 环境下高效实现。后来的很多大模型训练框架选择用 gradient checkpointing 代替可逆层，因为 checkpointing 更灵活。

### 14.3 Chunked FFN 的局限性

分块计算 FFN 本质上是"在循环中计算"，虽然降低了峰值显存，但增加了计算时间（因为分批计算无法充分利用矩阵运算的并行性）。

### 14.4 后续方向

Reformer 之后的发展方向主要有：

1. **更好的哈希/路由机制**：Routing Transformer 用可学习的聚类替代随机 LSH，Clusterformer 用 k-means 聚类
2. **其他高效 attention**：Performer 用随机傅里叶特征做 kernel attention，Linformer 用低秩近似
3. **可逆层的进一步优化**：gradient checkpointing 成为主流做法
4. **长序列 Transformer 的工业化**：BigBird、Longformer 在 Reformer 之后进一步推动了长序列 Transformer 的实用化

## 十五、小结

Reformer 这篇论文在高效 Transformer 的发展史上占据一个特殊位置。它没有提出新的数学框架，也没有改变 attention 的定义，而是**把两个已有的技术（LSH + RevNet）系统地适配到了 Transformer 上，使得单卡训练 64K 序列的 Transformer 成为可能**。

它的核心洞察在于：Transformer 的内存瓶颈可以拆解为三个正交的问题（attention 的 O(L²)、层数的 N 倍、FFN 的 d_ff 膨胀），并且每个问题都可以独立解决。这个拆解思路本身可能比具体方案更重要。

直白说，Reformer 的价值不只是提出了 LSH attention 这个具体方法，而是**把"高效 Transformer"这个方向从"有没有可能"推进到了"可以用什么具体方案做"的阶段**。它为此后层出不穷的高效 attention 工作——Longformer、BigBird、Performer、Routing Transformer——铺平了道路。

在技术树里，Reformer 是"E1. 降低 attention 复杂度"路径上的早期主干节点，属于**桥接性贡献**：它不完美，但它让后续的探索变得可能。
