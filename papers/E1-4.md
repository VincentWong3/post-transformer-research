# # Reformer: The Efficient Transformer（2020）

**论文：** [arXiv](https://arxiv.org/abs/2001.04451) · Nikita Kitaev, Łukasz Kaiser, Anselm Levskaya · 2020

---

## 一、先搞清楚坑在哪

Transformer 自从 2017 年横空出世，在 NLP 领域几乎成了默认架构。但有个老大难问题一直没解决——**计算复杂度**。

标准 Transformer 的核心是 self-attention，每个 token 要和所有其他 token 算注意力。如果有 $L$ 个 token，那复杂度就是 $O(L^2)$。当序列长度从几百涨到几千、几万，这个平方项直接炸了。

更麻烦的是，Transformer 的 feed-forward 层（FFN）虽然单次计算不贵，但模型做深了之后，中间激活值的存储也成了问题。16 层 Transformer 训练时，光中间激活就能吃掉几十 GB 显存。

所以 Reformer 的目标很明确：**让 Transformer 能处理超长序列（比如 64K tokens），同时把显存占用降下来**。

## 二、标准 Transformer 的真正问题

具体来说，标准 Transformer 有三个瓶颈：

**1. Attention 的 $O(L^2)$ 复杂度**

假设序列长度 $L=64K$，注意力矩阵就是 $64K \times 64K$，约 40 亿个元素。就算用半精度，也要 8GB 显存。这还只是一层，一个 head。12 层 8 head 的话，显存直接爆炸。

**2. FFN 层的中间激活**

标准 Transformer 的 FFN 是两层线性变换：$d_\text{model} \to d_\text{ff} \to d_\text{model}$，其中 $d_\text{ff}$ 通常是 $d_\text{model}$ 的 4 倍。这意味着 FFN 的中间层比 attention 层大得多。对于 $L=64K$，$d_\text{model}=1024$，单层 FFN 的中间激活就是 $64K \times 4096 \times 2$ bytes ≈ 0.5GB。12 层就是 6GB。

**3. 多层叠加的显存压力**

训练时，反向传播需要保存每一层的中间激活值。层数一多，显存就线性增长。对于长序列，这个增长是 $O(L \times \text{层数})$，很快就超出 GPU 显存。

## 三、Reformer 的核心思路

Reformer 的核心策略是 **"能省就省，能近似就近似"**，用三个关键创新把 Transformer 从平方复杂度里解放出来：

1. **LSH Attention（局部敏感哈希注意力）**：不跟所有 token 算注意力，只跟"相似"的 token 算。复杂度从 $O(L^2)$ 降到 $O(L \log L)$。
2. **Reversible FFN（可逆 FFN）**：训练时不保存中间激活，反向传播时重新计算。把 $O(L \times \text{层数})$ 的显存降到 $O(L)$。
3. **分块处理（Chunking）**：把长序列切成块，逐块计算，进一步降低显存峰值。

这三个技术组合起来，让 Reformer 可以在 64K 长度的序列上训练，而标准 Transformer 连 4K 都跑不动。

## 四、核心创新点详解

### 创新点 1：LSH Attention（局部敏感哈希注意力）

#### (a) 痛点与动机

标准 attention 的问题是：**每个 query 要和所有 key 算相似度**。但直觉上，一个 token 只跟少数 token 有强相关性。比如在文本里，"猫"这个词只跟附近的"猫粮"、"睡觉"有关，跟远处的"宪法"几乎没关系。

但标准 attention 不管这些，它把所有 key 都算一遍，然后 softmax 把不相关的压到接近 0。这就浪费了大量计算。

之前有人试过 sparse attention（稀疏注意力），只算局部窗口内的注意力。但问题是，有些长距离依赖很重要（比如代词指代），局部窗口会漏掉。

#### (b) 方案细节

LSH Attention 的核心想法：**先找到跟 query 相似的 key，只跟这些 key 算注意力**。

怎么找相似的 key？用**局部敏感哈希**（Locality Sensitive Hashing, LSH）。

LSH 的思想很简单：把高维空间里的点投影到低维，然后做哈希。如果两个点在原始空间里接近，它们被哈希到同一个桶的概率就高。如果不接近，概率就低。

具体实现：

**1. 哈希函数**

Reformer 用的是**随机投影哈希**：

$$h(x) = \arg\max([xR; -xR])$$

其中 $R$ 是一个随机矩阵，$[;]$ 表示拼接。这个哈希函数把向量 $x$ 映射到 $b$ 个桶中的一个。

**2. 哈希分桶**

把 query 和 key 都用同一个哈希函数哈希，然后只让同一个桶内的 query 和 key 算注意力。

**3. 多轮哈希（Multi-round Hashing）**

单轮哈希可能因为随机性把相似的 token 分到不同桶。所以 Reformer 做 $n_\text{rounds}$ 轮哈希，用不同的随机矩阵，然后取并集。论文里通常用 4 轮。

**4. 分块注意力（Chunked Attention）**

每个桶内，query 和 key 的数量可能不均匀。为了高效计算，Reformer 把桶内的 token 按排序后的顺序切成固定大小的块，逐块算注意力。块大小 $m = 2 \times \text{chunk\_size}$，这样每个 query 能看到块内所有 key 和上一个块的部分 key。

#### (c) 为什么有效

LSH Attention 把 attention 从 $O(L^2)$ 降到了 $O(L \log L)$。直觉上，每个 token 只跟 $O(\log L)$ 个 token 算注意力（因为哈希桶的大小是 $O(\log L)$）。

实验表明，在 enwik8 和 imagenet 64x64 上，LSH Attention 的表现跟全注意力几乎一样。Table 1 显示，LSH Attention 的 bpc（bits per character）只比全注意力高 0.02-0.05。

#### (d) 与 Related Work 的关系

LSH Attention 借鉴了 [Charikar, 2002] 的 LSH 思想，但首次把 LSH 用在 Transformer attention 上。之前有人用过 sparse attention [Child et al., 2019] 和 adaptive attention [Correia et al., 2019]，但都没有用哈希来做近似。

#### (e) 如果去掉会怎样

去掉 LSH Attention，Reformer 就变回标准 Transformer，复杂度回到 $O(L^2)$，根本跑不动 64K 序列。Ablation 实验（Table 2）显示，减少哈希轮数会让 perplexity 变差，说明多轮哈希确实有帮助。

### 创新点 2：Reversible FFN（可逆 FFN）

#### (a) 痛点与动机

标准 Transformer 训练时，每一层都要保存中间激活值用于反向传播。对于 $L=64K$ 的序列，12 层 Transformer 的中间激活值就需要几十 GB 显存。

之前有人用 gradient checkpointing [Chen et al., 2016]，但那是细粒度的，需要手动指定哪些层要重新计算，而且会引入计算开销。

#### (b) 方案细节

Reformer 采用 **Reversible Transformer** 架构，灵感来自 RevNet [Gomez et al., 2017]。

标准 Transformer 层：

$$y = x + \text{Attention}(x)$$
$$z = y + \text{FFN}(y)$$

Reversible Transformer 层：

$$y_1 = x_1 + \text{Attention}(x_2)$$
$$y_2 = x_2 + \text{FFN}(y_1)$$

这里输入被分成两半 $x_1$ 和 $x_2$。前向传播时，两半交叉计算。反向传播时，可以从输出 $y_1, y_2$ 反推出输入 $x_1, x_2$：

$$x_2 = y_2 - \text{FFN}(y_1)$$
$$x_1 = y_1 - \text{Attention}(x_2)$$

这样就不用保存中间激活值了，反向传播时重新计算就行。

#### (c) 为什么有效

Reversible FFN 把每层的显存从 $O(L \times d_\text{model})$ 降到了 $O(L \times d_\text{model}/2)$（因为输入被分成两半）。更重要的是，它不需要保存每一层的中间激活，所以总显存从 $O(L \times \text{层数})$ 降到了 $O(L)$。

实验（Table 3）显示，Reversible Transformer 的 perplexity 跟标准 Transformer 几乎一样（相差 0.03 bpc），但显存节省了 10-15 倍。

#### (d) 与 Related Work 的关系

Reversible Transformer 直接借鉴了 RevNet [Gomez et al., 2017] 的可逆架构。RevNet 用在 CNN 上，Reformer 把它移植到 Transformer。之前也有人尝试过 gradient checkpointing [Chen et al., 2016]，但 reversible 架构更优雅，不需要手动指定 checkpoint 点。

#### (e) 如果去掉会怎样

去掉 Reversible FFN，Reformer 的显存占用会回到 $O(L \times \text{层数})$，对于 64K 序列和 12 层，需要约 60GB 显存，远超单 GPU 的容量。实际上，没有 reversible 层，Reformer 根本跑不了长序列。

### 创新点 3：分块处理（Chunking）

#### (a) 痛点与动机

即使有了 LSH Attention 和 Reversible FFN，FFN 层的计算仍然需要大量显存。对于 $d_\text{model}=1024$，$d_\text{ff}=4096$，单层 FFN 的中间激活是 $L \times d_\text{ff} \times 2$ bytes。对于 $L=64K$，就是 0.5GB。12 层就是 6GB。

#### (b) 方案细节

Reformer 把 FFN 的输入序列分成固定大小的块（比如 1024 个 token），逐块计算 FFN。块与块之间独立计算，不需要相互依赖。

对于 attention 层，LSH Attention 本身就已经是分块的了（桶内分块）。

#### (c) 为什么有效

分块把显存峰值从 $O(L \times d_\text{ff})$ 降到了 $O(\text{chunk\_size} \times d_\text{ff})$。对于 $L=64K$，chunk_size=1024，显存峰值从 0.5GB 降到 8MB。

分块不影响计算结果，因为 FFN 是逐 token 独立计算的。Attention 的分块因为有 LSH 的桶结构，也是合理的近似。

#### (d) 与 Related Work 的关系

分块处理在很多 Transformer 优化里都有，比如 [Child et al., 2019] 的 Sparse Transformer 也用分块。Reformer 把它跟 LSH 和 reversible 架构结合起来。

#### (e) 如果去掉会怎样

去掉分块，FFN 的显存占用会回到 $O(L \times d_\text{ff})$，对于 64K 序列，单层就需要 0.5GB，12 层 6GB。虽然比 reversible 之前的 60GB 好，但仍然是显存压力。

## 五、公式详解

### 公式 1：LSH Attention 复杂度

**标准 Attention：**

$$O(L^2 \cdot d)$$

其中 $L$ 是序列长度，$d$ 是每个 head 的维度。

**LSH Attention：**

$$O(L \cdot n_\text{rounds} \cdot (m \cdot \log L + m^2))$$

其中 $n_\text{rounds}$ 是哈希轮数（通常 4），$m$ 是块大小（通常 $2 \times \text{chunk\_size}$）。

**推导：**

1. 哈希：$O(L \cdot d)$，每轮一次
2. 排序：$O(L \log L)$，每轮一次
3. 块内 attention：每个块大小 $m$，有 $L/m$ 个块，每个块内 attention 是 $O(m^2)$，所以总 $O(L \cdot m)$

**数值例子：**

$L=64K=65536$，$d=64$，$n_\text{rounds}=4$，$m=128$

标准 Attention：$65536^2 \times 64 \approx 2.7 \times 10^{11}$ 次操作

LSH Attention：$65536 \times 4 \times (128 \times 16 + 128^2) \approx 65536 \times 4 \times 16384 \approx 4.3 \times 10^9$ 次操作

LSH Attention 比标准 Attention 快约 63 倍。

### 公式 2：Reversible Transformer 前向/反向

**前向（Forward）：**

$$y_1 = x_1 + \text{Attention}(x_2)$$
$$y_2 = x_2 + \text{FFN}(y_1)$$

**反向（Backward）：**

$$x_2 = y_2 - \text{FFN}(y_1)$$
$$x_1 = y_1 - \text{Attention}(x_2)$$

**符号定义：**
- $x_1, x_2$：输入的两半，各 $L \times d_\text{model}/2$
- $y_1, y_2$：输出的两半，各 $L \times d_\text{model}/2$
- $\text{Attention}$：多头注意力函数
- $\text{FFN}$：前馈网络函数

**推导：**

从 $y_2 = x_2 + \text{FFN}(y_1)$ 可得：
$$x_2 = y_2 - \text{FFN}(y_1)$$

从 $y_1 = x_1 + \text{Attention}(x_2)$ 可得：
$$x_1 = y_1 - \text{Attention}(x_2)$$

**直觉理解：**

可逆架构就像两个方程解两个未知数。只要知道输出 $y_1, y_2$，就能反推出输入 $x_1, x_2$。这样训练时只需要保存输出，反向传播时重新计算中间值。

**数值例子：**

设 $L=1024$，$d_\text{model}=512$，则每层输入 $x_1, x_2$ 各 $1024 \times 256$。

标准 Transformer 需要保存每层输入 $x$（$1024 \times 512$）和中间值（attention 输出 $1024 \times 512$，FFN 中间 $1024 \times 2048$），共 $1024 \times (512 + 512 + 2048) \approx 3.1$M 个值。

Reversible Transformer 只需要保存输出 $y_1, y_2$（$1024 \times 512$），共 $0.5$M 个值。节省约 6 倍。

### 公式 3：LSH 哈希函数

$$h(x) = \arg\max([xR_1; xR_2; ...; xR_b; -xR_1; -xR_2; ...; -xR_b])$$

其中 $R_i$ 是随机投影矩阵，$b$ 是桶数。

**符号定义：**
- $x$：输入向量，维度 $d$
- $R_i$：随机矩阵，维度 $d \times 1$（每个投影是一个随机向量）
- $[;]$：向量拼接
- $h(x)$：哈希值，范围 $0$ 到 $2b-1$

**推导：**

这个哈希函数等价于：把 $x$ 投影到 $b$ 个随机方向上，每个方向取正负两个桶。所以总桶数是 $2b$。

**直觉理解：**

如果两个向量 $x$ 和 $y$ 在某个方向上的投影接近，它们就会被分到同一个桶。因为随机投影保留了余弦相似度（Johnson-Lindenstrauss 引理），所以相似的向量大概率在多个投影上都接近，从而被分到同一个桶。

**边界情况：**

- 当 $b$ 很小时（如 1），桶数只有 2，hash 很粗糙，很多不相似的 token 也会被分到一起
- 当 $b$ 很大时（如 256），桶数 512，hash 很细，相似的 token 可能被分到不同桶，需要多轮哈希来补偿

### 主要结果（Table 1）

## 七、代码对照

Reformer 的官方代码在 [https://github.com/google/trax](https://github.com/google/trax) 里，作为 trax 库的一部分。

关键代码文件：
- `trax/layers/reformer.py`：Reformer 的核心层实现
- `trax/models/reformer.py`：Reformer 模型定义

**实现细节：**

1. **LSH Attention 实现**：
   - 哈希函数用 `jax.random.randint` 生成随机投影
   - 排序后按桶内顺序分块
   - 每个块内用标准 attention

2. **Reversible 实现**：
   - 用 `ReversibleLayer` 包装 attention 和 FFN
   - 前向传播时保存输出，不保存中间值
   - 反向传播时调用 `recompute` 重新计算中间值

3. **分块实现**：
   - FFN 用 `ChunkedLayer` 包装
   - 输入按 `chunk_size` 切成块
   - 逐块计算后拼接

**代码技巧：**
- 使用 `jax.vmap` 对块进行批处理
- 用 `jax.lax.scan` 实现循环中的分块计算
- 哈希轮数用 `jax.lax.fori_loop` 实现

1. **[Choromanski et al., 2020]** Performer，用核方法近似 attention，比 LSH 更高效

4. **[Jaegle et al., 2021]** Perceiver，用交叉注意力处理长序列，借鉴了 Reformer 的分块思想
5. **[Dao et al., 2022]** FlashAttention，用 IO-aware 算法优化 attention 计算，在硬件层面解决了显存问题

## 九、局限

1. **LSH 的随机性**：LSH 是随机的，不同运行可能得到不同结果。虽然多轮哈希缓解了这个问题，但理论上不如确定性方法稳定。

2. **近似误差**：LSH Attention 是全注意力的近似，在需要精确注意力的任务上（如代码理解、数学推理）可能不够好。

3. **哈希开销**：哈希本身有计算开销，对于短序列（<1K），LSH Attention 可能比标准 attention 还慢。

4. **桶大小不均**：哈希桶的大小可能不均匀，导致某些桶的计算量特别大，影响并行效率。

5. **可逆架构的数值稳定性**：可逆架构在反向传播时需要做减法，可能引入数值误差，尤其是在低精度训练时。

## 十、小结

Reformer 是一个优雅的工程解决方案，它用三个聪明的技巧——LSH Attention、Reversible FFN、分块处理——把 Transformer 从 $O(L^2)$ 的复杂度陷阱里解放出来。

它的核心贡献不是改动 Transformer 的基本结构，而是**让 Transformer 能处理之前完全无法处理的序列长度**。64K tokens 的训练，在 Reformer 之前是不可想象的。

Reformer 的影响是深远的。它开启了"高效 Transformer"这个研究方向，后续的 Performer、Linformer、FlashAttention 等都是在解决同一个问题。虽然 Reformer 本身因为 LSH 的随机性和实现复杂度，没有被广泛采用，但它证明了"近似 attention + 可逆架构 + 分块"这个组合的有效性，为后续工作铺平了道路。

现在回头看，Reformer 最大的价值可能不是它提出的具体技术，而是它展示了一个思路：**与其改进 Transformer 的核心机制，不如在工程上找到巧妙的近似和优化**。这个思路影响了整个高效 Transformer 领域的发展。
