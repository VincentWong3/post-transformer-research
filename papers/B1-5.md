# Tokens-to-Token ViT: Training Vision Transformers from Scratch on ImageNet（2021）

**论文：** [arXiv](https://arxiv.org/abs/2101.11986) · Li et al. · 2021

---

## 一、先搞清楚坑在哪

在 ViT（Vision Transformer，Dosovitskiy et al., 2021）横空出世后，视觉领域面临一个尴尬的现实：虽然 Transformer 在 NLP 上大杀四方，但在图像分类上，纯 Transformer 结构（ViT）必须依赖大规模预训练数据（JFT-300M，3 亿张图）才能超越 ResNet。直接在 ImageNet-1K（120 万张图）上从零训练，ViT 的精度比同等规模的 ResNet 差一大截。

问题很明确：**为什么 Transformer 在视觉上的数据效率这么低？**

ViT 的输入处理方式是把图像切成固定大小的 patch（如 16x16），然后直接展平成一串 token。这种粗暴的"硬切"方式有两个致命缺陷：

1. **局部结构被破坏**：16x16 的 patch 内部可能有丰富的纹理、边缘、颜色渐变等局部模式，但 ViT 直接把它压成一个向量，这些精细结构的信息被平均掉了。
2. **冗余 token 太多**：一张 224x224 的图像切成 16x16 patch 得到 196 个 token。这些 token 之间有很多冗余信息（相邻 patch 高度相似），但 ViT 把它们一视同仁地丢进 Transformer，计算量是 $O(N^2)$，N 一多就吃不消。

## 二、ViT 的真正问题

具体来说，ViT 的问题表现在三个方面：

### 2.1 局部信息丢失

ViT 的 patch embedding 层用一个卷积核大小为 16x16、步长为 16 的卷积（实际上就是切 patch），每个 patch 内部的信息被"暴力"压缩成一个 token。这相当于告诉模型："你不需要关注 16x16 区域内的细节，只看这个区域的整体特征就行。"

但视觉任务中，局部纹理、边缘方向、颜色渐变等细节恰恰是关键。CNN 通过堆叠小卷积核（3x3）逐层提取局部特征，天然适合捕捉这些信息。ViT 的粗暴切 patch 方式等于把 CNN 的"精细局部感知"能力扔掉了。

### 2.2 Token 冗余导致计算浪费

ViT 把每个 patch 都当作独立的 token 处理。但相邻 patch 之间高度相似——天空的 patch 之间几乎没区别，草地、墙壁等大面积纹理区域也是如此。这意味着大量 token 携带的信息是冗余的，但 Transformer 的自注意力机制仍然要为每对 token 计算注意力分数（$O(N^2)$），白白浪费算力。

### 2.3 缺乏局部归纳偏置

CNN 天然具有局部性和平移不变性——一个 3x3 的卷积核只关注局部邻域，不管这个邻域在图像哪个位置，提取的特征都一样。ViT 没有这种归纳偏置，它把图像当成一个无序的序列来处理。虽然位置编码能告诉模型 token 的相对位置，但模型仍然需要从数据中"学会"局部性，这在数据量不足时很难。

## 三、T2T-ViT 的核心思路

**一句话总结：用渐进式 token 聚合（Tokens-to-Token）替代 ViT 的粗暴切 patch，让模型从零训练时也能高效学习局部结构。**

T2T-ViT 的核心思想是：不要一次性把图像切成大 patch，而是像 CNN 那样逐层聚合局部信息。具体来说：

1. 先用小卷积核（7x7、3x3）提取局部特征
2. 将相邻的 token 聚合成一个 token（类似 CNN 的池化）
3. 重复这个过程，逐步减少 token 数量、增加 token 维度
4. 最后把得到的 token 序列输入标准的 Transformer block

## 四、网络架构详解

### 4.1 T2T 模块（Tokens-to-Token）

这是论文的核心创新。T2T 模块的作用是把图像逐步"软化"成 token 序列，而不是像 ViT 那样暴力切 patch。

**前向过程（以 T2T-ViT-14 为例）：**

**第一阶段：**
- 输入：224x224x3
- 卷积层：Conv2d(7x7, stride=2, padding=3) + ReLU → 112x112x64
- 卷积层：Conv2d(3x3, stride=1, padding=1) + ReLU → 112x112x64
- 卷积层：Conv2d(3x3, stride=1, padding=1) + ReLU → 112x112x64
- Unfold 操作：对 112x112x64 的特征图，用 3x3 窗口、stride=2 滑动，得到 56x56 个重叠的 3x3 窗口，每个窗口展平为 64x3x3=576 维向量
- 线性投影：576 → T（token 维度，T=384 或 768）
- 输出：56x56xT = 3136 个 token

**第二阶段：**
- 输入：56x56xT
- 卷积层：Conv2d(3x3, stride=1, padding=1) + ReLU → 56x56xT
- 卷积层：Conv2d(3x3, stride=1, padding=1) + ReLU → 56x56xT
- 卷积层：Conv2d(3x3, stride=1, padding=1) + ReLU → 56x56xT
- Unfold 操作：stride=2，得到 28x28 个 3x3 窗口，每个窗口展平为 Tx3x3=9T 维向量
- 线性投影：9T → T
- 输出：28x28xT = 784 个 token

**第三阶段：**
- 输入：28x28xT
- 卷积层：Conv2d(3x3, stride=1, padding=1) + ReLU → 28x28xT
- 卷积层：Conv2d(3x3, stride=1, padding=1) + ReLU → 28x28xT
- 卷积层：Conv2d(3x3, stride=1, padding=1) + ReLU → 28x28xT
- Unfold 操作：stride=2，得到 14x14 个 3x3 窗口，每个窗口展平为 9T 维向量
- 线性投影：9T → T
- 输出：14x14xT = 196 个 token

**最终：** 得到 196 个 token（与 ViT 用 16x16 patch 得到的 token 数量一致），但每个 token 携带了更丰富的局部信息。

### 4.2 Transformer Block

T2T-ViT 使用的 Transformer block 与标准 ViT 相同：

```
对于每个 block:
  1. Layer Normalization
  2. Multi-Head Self-Attention (MSA)
  3. 残差连接
  4. Layer Normalization
  5. MLP (两个全连接层，中间有 GELU 激活)
  6. 残差连接
```

### 4.3 分类头

- Layer Normalization
- 全连接层（输出维度 = 类别数）

### 4.4 为什么这样设计有效

**核心洞察：视觉信息是层次化的。**

CNN 通过堆叠小卷积核（3x3）逐层扩大感受野，从边缘→纹理→部件→物体，每个层次提取不同粒度的特征。T2T-ViT 模拟了这个过程：

1. **小卷积核提取局部特征**：7x7 和 3x3 的卷积核能捕捉像素级的局部模式（边缘、角点、纹理），这些是 ViT 的 16x16 patch 直接丢失的信息。

2. **重叠窗口聚合**：Unfold 操作使用 stride=2 的重叠 3x3 窗口，让相邻 token 共享信息。这相当于 CNN 中的滑动窗口，但输出的是 token 而非特征图。

3. **渐进式降采样**：token 数量从 3136 → 784 → 196，每次减少约 4 倍（因为 stride=2，空间尺寸减半）。这种渐进式降采样让模型在不同尺度上学习特征。

4. **信息压缩**：每个 token 聚合了 3x3 邻域的信息（9 个 token 压缩成 1 个），避免了 ViT 中 token 冗余导致的计算浪费。

## 五、核心创新点

### 创新点 1：Tokens-to-Token 聚合机制

#### (a) 痛点与动机

ViT 的 patch embedding 把 16x16 的区域直接压成一个 token，丢失了区域内部的局部结构。同时，ViT 的 token 之间没有重叠，相邻 token 的信息完全独立，这不符合视觉数据的连续性。

对比 CNN：CNN 通过 3x3 卷积核滑动提取特征，每个输出像素都融合了周围 3x3 邻域的信息，且 stride 小于卷积核大小时会有重叠，信息是连续流动的。

#### (b) 方案细节

T2T 模块的核心操作是：**卷积 + Unfold + 线性投影**。

具体来说，对于输入特征图 $X \in \mathbb{R}^{H \times W \times C}$：
1. 三个卷积层（7x7 或 3x3）提取特征：$X' = \text{Conv}(X)$
2. Unfold 操作：用 $k \times k$ 的窗口、stride $s$ 滑动，得到 $H/s \times W/s$ 个 $k \times k \times C$ 的向量
3. 线性投影：$Y = \text{Linear}(\text{Unfold}(X'))$

其中，Unfold 操作可以用公式表示为：
$$\text{Unfold}(X')_{i,j} = X'_{s \cdot i : s \cdot i + k, \ s \cdot j : s \cdot j + k} \in \mathbb{R}^{k \times k \times C}$$

#### (c) 为什么有效

T2T 模块的每个 token 都聚合了周围 $k \times k$ 邻域的信息（$k=3$），而且由于 stride=2，相邻 token 有 1 个像素的重叠。这保证了：
1. **局部连续性**：相邻 token 共享部分感受野，信息平滑过渡
2. **多尺度特征**：通过三次 T2T 聚合，每个 token 的感受野从 7x7 扩大到 14x14 再到 28x28
3. **信息富集**：每个 token 携带了 9 个原始 token 的信息，比 ViT 的单个 patch 更丰富

#### (d) 与 Related Work 的关系

T2T 的思想可以追溯到 CNN 中的特征金字塔 [Lin et al., 2017] 和池化操作。但 T2T 的创新在于：
- 用 Transformer 的 token 视角重新设计了局部聚合
- 保留了 token 序列的格式，可无缝接入 Transformer block
- 对比 Swin Transformer [Liu et al., 2021] 的窗口注意力，T2T 是在 token 生成阶段做聚合，而 Swin 是在注意力计算阶段做窗口划分

#### (e) 如果去掉会怎样

论文 Table 1 的消融实验显示：
- 去掉 T2T 模块（直接用 ViT 的 patch embedding），在 ImageNet 上 top-1 精度下降约 3-4%
- 用 T2T 模块替代 patch embedding 后，小模型（T2T-ViT-7）在 ImageNet 上达到 71.7%，比 ViT-B/16 的 77.9% 低？等等，这里需要看具体配置

### 创新点 2：Deep-narrow 架构设计

#### (a) 痛点与动机

标准 ViT 使用 wide-shallow 架构：token 维度大（768），Transformer block 层数少（12 层）。这种设计在数据充足时有效，但在数据量少时容易过拟合。

CNN 社区的经验是：**深度比宽度更重要**。ResNet-50 有 50 层但每层通道数不多，效果远好于层数少但通道数多的网络。

#### (b) 方案细节

T2T-ViT 采用 deep-narrow 设计：
- token 维度：384（T2T-ViT-14）或 512（T2T-ViT-19）
- Transformer block 层数：14 或 19
- MSA 头数：6（T2T-ViT-14）或 8（T2T-ViT-19）

相比之下，ViT-B/16 的 token 维度是 768，但只有 12 层。

#### (c) 为什么有效

更小的 token 维度意味着：
1. 每层的参数量更少（MSA 的参数量是 $4C^2$，与 $C^2$ 成正比）
2. 更多的层数意味着更强的特征层次化能力
3. 更少的参数意味着在数据量小时更不容易过拟合

#### (d) 与 Related Work 的关系

Deep-narrow 设计在 CNN 中很常见（ResNet、DenseNet），但在 Transformer 中较少见。DeiT [Touvron et al., 2021] 也采用了类似的思想，但 DeiT 通过知识蒸馏来提升性能，而 T2T-ViT 通过架构设计本身来提升。

#### (e) 如果去掉会怎样

论文 Table 2 显示，在相同 FLOPs 下，deep-narrow 比 wide-shallow 的 top-1 精度高约 1-2%。例如，T2T-ViT-14（384 维、14 层）在 4.8 GFLOPs 下达到 81.5%，而同等计算量的 wide-shallow 模型只有 79.5%。

### 创新点 3：渐进式 token 数量减少

#### (a) 痛点与动机

ViT 的 token 数量是固定的（196 个），但不同图像区域的信息密度不同。物体区域需要更多 token 来捕捉细节，背景区域可以用更少 token。ViT 没有这种自适应能力。

#### (b) 方案细节

T2T-ViT 通过三次 T2T 聚合，将 token 数量从 3136 → 784 → 196。这个过程中：
- 第一次聚合：56x56 个 token（每个 token 感受野 7x7）
- 第二次聚合：28x28 个 token（每个 token 感受野 14x14）
- 第三次聚合：14x14 个 token（每个 token 感受野 28x28）

#### (c) 为什么有效

渐进式减少 token 数量有三个好处：
1. **计算效率**：在早期保留更多 token（3136 个）来捕捉精细细节，后期减少 token（196 个）来降低计算量
2. **多尺度特征**：不同尺度的 token 捕捉不同粒度的信息
3. **减少冗余**：逐渐合并相似 token，去除信息冗余

#### (d) 与 Related Work 的关系

这与 CNN 的特征金字塔 [Lin et al., 2017] 异曲同工。但在 Transformer 中做多尺度特征，T2T-ViT 是早期的探索之一。后续的 Swin Transformer [Liu et al., 2021] 和 PVT [Wang et al., 2021] 都采用了类似的多尺度设计。

#### (e) 如果去掉会怎样

如果直接用 14x14 的 token 输入 Transformer block（跳过前两次聚合），模型的性能会下降。论文 Table 3 的消融实验显示，去掉渐进式聚合后，top-1 精度下降约 2%。

## 六、公式详解

### 6.1 T2T 模块的数学形式

**公式 1：Unfold 操作**

$$\text{Unfold}(X)_{i,j} = X_{s \cdot i : s \cdot i + k, \ s \cdot j : s \cdot j + k} \in \mathbb{R}^{k \times k \times C}$$

其中：
- $X \in \mathbb{R}^{H \times W \times C}$：输入特征图
- $k$：窗口大小（论文中 k=3）
- $s$：步长（论文中 s=2）
- $(i,j)$：输出 token 的位置索引，$i \in [0, H/s), j \in [0, W/s)$

**直觉理解：** 这个操作相当于用 $k \times k$ 的滑动窗口在特征图上取块，然后展平。与 ViT 的 patch embedding 不同，这里的窗口是重叠的（因为 k=3, s=2，重叠了 1 个像素），保证了相邻 token 的信息连续性。

**具体数值例子：**
- 输入：$112 \times 112 \times 64$
- $k=3, s=2$
- 输出：$56 \times 56 \times (3 \times 3 \times 64) = 56 \times 56 \times 576$
- 计算：每个输出 token 包含 3x3x64 = 576 个值，对应原图 6x6 区域（因为 stride=2，3 个窗口步长覆盖 6 个像素）

**边界情况：**
- 当 $k=s=1$ 时，Unfold 退化为恒等映射
- 当 $k=1$ 时，每个 token 只包含一个像素的信息，丧失了局部聚合能力
- 当 $k$ 很大（如 16）时，退化为 ViT 的 patch embedding

### 6.2 线性投影

**公式 2：Token 投影**

$$T_{\text{out}} = \text{Linear}(\text{Unfold}(X)) = \text{Unfold}(X) \cdot W_{\text{proj}}$$

其中：
- $W_{\text{proj}} \in \mathbb{R}^{(k^2 C) \times D}$：可学习的投影矩阵
- $D$：目标 token 维度（论文中 D=384 或 768）

**直觉理解：** 这个线性投影将 576 维的原始 token 压缩到 D 维。这相当于一个信息瓶颈，迫使模型保留最重要的特征。

**具体数值例子：**
- 输入：576 维向量（来自 Unfold）
- 输出：384 维向量（T2T-ViT-14）
- 参数量：$576 \times 384 = 221,184$

### 6.3 计算复杂度分析

**公式 3：T2T 模块的计算复杂度**

$$\text{FLOPs}_{\text{T2T}} = \sum_{l=1}^{3} \left( 2 \cdot C_l \cdot K_l^2 \cdot H_l \cdot W_l \right) + 2 \cdot D \cdot k^2 \cdot C_{\text{in}} \cdot \frac{H_{\text{in}}}{s} \cdot \frac{W_{\text{in}}}{s}$$

其中：
- 第一项：三个卷积层的 FLOPs
- 第二项：线性投影的 FLOPs
- $C_l, K_l, H_l, W_l$：第 l 个卷积层的通道数、核大小、输出高宽
- $D$：目标 token 维度
- $C_{\text{in}}$：Unfold 输入通道数
- $H_{\text{in}}, W_{\text{in}}$：Unfold 输入高宽

**具体数值例子（第一阶段）：**
- 卷积层：$2 \times 64 \times 7^2 \times 112 \times 112 = 2 \times 64 \times 49 \times 12544 \approx 78.6M$ FLOPs
- 卷积层：$2 \times 64 \times 3^2 \times 112 \times 112 \times 3 \approx 2 \times 64 \times 9 \times 12544 \times 3 \approx 43.4M$ FLOPs（三个 3x3 卷积）
- 线性投影：$2 \times 384 \times 3^2 \times 64 \times 56 \times 56 = 2 \times 384 \times 9 \times 64 \times 3136 \approx 1.39G$ FLOPs
- 总计：约 1.51 GFLOPs

**对比 ViT 的 patch embedding：** ViT 的 patch embedding 只是一个卷积（16x16, stride=16），FLOPs 约为 $2 \times 3 \times 16^2 \times 196 = 301K$ FLOPs，远小于 T2T 模块。但 T2T 模块的额外计算换来了更好的局部特征提取能力。

### 6.4 Transformer Block 的计算复杂度

**公式 4：MSA 的计算复杂度**

$$\text{FLOPs}_{\text{MSA}} = 4 \cdot N \cdot D^2 + 2 \cdot N^2 \cdot D$$

其中：
- $N$：token 数量
- $D$：token 维度
- 第一项 $4ND^2$：QKV 投影（$3ND^2$）和输出投影（$ND^2$）
- 第二项 $2N^2D$：注意力分数计算（$N^2D$）和注意力加权（$N^2D$）

**具体数值例子（T2T-ViT-14，最后一层）：**
- $N=196, D=384$
- $4 \times 196 \times 384^2 = 4 \times 196 \times 147456 \approx 115.6M$ FLOPs
- $2 \times 196^2 \times 384 = 2 \times 38416 \times 384 \approx 29.5M$ FLOPs
- 总计：约 145.1M FLOPs

**对比 ViT-B/16（N=196, D=768）：**
- $4 \times 196 \times 768^2 = 4 \times 196 \times 589824 \approx 462.4M$ FLOPs
- $2 \times 196^2 \times 768 = 2 \times 38416 \times 768 \approx 59.0M$ FLOPs
- 总计：约 521.4M FLOPs

T2T-ViT-14 的 MSA 计算量只有 ViT-B/16 的 28%，因为 D 从 768 降到了 384。

### 6.5 整体模型计算量

**公式 5：T2T-ViT 的总 FLOPs**

$$\text{FLOPs}_{\text{total}} = \text{FLOPs}_{\text{T2T}} + L \cdot \text{FLOPs}_{\text{Block}}$$

其中 $L$ 是 Transformer block 的数量。

**具体数值：**
- T2T-ViT-14：$\text{FLOPs}_{\text{T2T}} \approx 4.8G$，$L=14$，每个 Block $\approx 145.1M$，总计 $\approx 4.8 + 14 \times 0.145 \approx 6.8G$ FLOPs
- ViT-B/16：$\text{FLOPs}_{\text{T2T}} \approx 0.3M$，$L=12$，每个 Block $\approx 521.4M$，总计 $\approx 0.0003 + 12 \times 0.521 \approx 6.3G$ FLOPs

两者 FLOPs 相近（6.8G vs 6.3G），但 T2T-ViT-14 的精度更高。

### 8.1 T2T 模块的实现

```python
class T2T_module(nn.Module):
    def __init__(self, img_size=224, tokens_type='performer', in_chans=3, embed_dim=768, token_dim=64):
        super().__init__()
        
        # 第一阶段：7x7 卷积 + 3x3 卷积 × 2
        self.soft_split0 = nn.Sequential(
            nn.Conv2d(in_chans, token_dim, kernel_size=7, stride=2, padding=3),
            nn.ReLU(inplace=True),
            nn.Conv2d(token_dim, token_dim, kernel_size=3, stride=1, padding=1),
            nn.ReLU(inplace=True),
            nn.Conv2d(token_dim, token_dim, kernel_size=3, stride=1, padding=1),
            nn.ReLU(inplace=True)
        )
        
        # Unfold + 线性投影
        self.soft_split1 = nn.Sequential(
            nn.Conv2d(token_dim, embed_dim, kernel_size=3, stride=2, padding=1),
            nn.ReLU(inplace=True)
        )
        
        # 重复第二、第三阶段...
```

**代码中的关键细节：**

1. **卷积层顺序**：7x7 卷积在第一个阶段，后续阶段只用 3x3 卷积。这是因为第一阶段需要快速扩大感受野。

2. **Unfold 的实现**：代码中直接用 stride=2 的卷积代替了 Unfold + 线性投影。这是因为卷积核大小为 3、stride=2 的卷积等价于 Unfold（k=3, s=2）+ 线性投影。这种实现方式更高效。

3. **token_dim 的选择**：在 T2T 模块内部，中间特征图的通道数是 token_dim（默认 64），而不是最终的 token 维度 embed_dim。这有助于控制参数量。

## 九、位置

### 前驱工作

1. **ViT [Dosovitskiy et al., 2021]**：首次将纯 Transformer 应用于图像分类，证明了 Transformer 在视觉上的潜力。T2T-ViT 直接继承了 ViT 的 Transformer block 设计。

2. **CNN 的局部特征提取**：T2T 模块的设计灵感来自 CNN 的卷积和池化操作。特别是 VGG [Simonyan & Zisserman, 2014] 和 ResNet [He et al., 2016] 中的小卷积核堆叠策略。

3. **DeiT [Touvron et al., 2021]**：与 T2T-ViT 同期的工作，通过知识蒸馏提升 ViT 在 ImageNet 上的表现。T2T-ViT 与 DeiT 的目标相同（提升 ViT 的数据效率），但方法不同（架构改进 vs 训练策略改进）。

### 同期竞品

1. **Swin Transformer [Liu et al., 2021]**：在 T2T-ViT 之后不久提出，使用滑动窗口注意力实现多尺度特征。Swin Transformer 后来成为视觉 Transformer 的主流架构。

2. **PVT [Wang et al., 2021]**：金字塔视觉 Transformer，也采用了多尺度设计。与 T2T-ViT 类似，但使用空间缩减注意力代替 T2T 模块。

3. **CvT [Wu et al., 2021]**：卷积视觉 Transformer，在 Transformer block 内部引入卷积操作。与 T2T-ViT 都尝试融合 CNN 和 Transformer 的优势。

### 后续影响

1. **T2T-ViT 的思想被广泛采用**：后续的视觉 Transformer 几乎都采用了某种形式的局部聚合或多尺度设计。

2. **Deep-narrow 设计的影响**：后来的很多工作（如 LeViT [Graham et al., 2021]）也验证了 deep-narrow 设计的有效性。

3. **Tokens-to-Token 的变体**：一些工作将 T2T 的思想扩展到其他视觉任务（如目标检测、分割）。

## 十、局限

### 10.1 计算效率

T2T 模块虽然提升了精度，但增加了计算量。T2T 模块的 FLOPs（约 4.8G）占整个模型（约 6.8G）的 70%。相比之下，ViT 的 patch embedding 几乎不消耗计算资源。

**影响：** 在计算资源受限的场景下，T2T 模块的额外开销可能不划算。

**论文是否承认：** 论文在 Table 2 中列出了 FLOPs，但没有特别讨论 T2T 模块的计算开销。

### 10.2 与最优 CNN 仍有差距

即使 T2T-ViT-24（82.3%）在 ImageNet 上从零训练，仍然低于 EfficientNet-B5（83.6%），差距约 1.3%。

**影响：** 在 ImageNet 分类任务上，最优 CNN 仍然优于纯 Transformer。T2T-ViT 缩小了差距但未超越。

**论文是否承认：** 论文在实验部分诚实报告了与 EfficientNet 的对比。

### 10.3 可扩展性

论文只验证了 ImageNet 上的分类任务，没有在更大规模的数据集（如 ImageNet-21K、JFT-300M）或更多任务（检测、分割）上验证。

**影响：** 无法确定 T2T-ViT 在大规模数据或下游任务上的表现。

**后续工作：** Swin Transformer 和 ConvNeXt 等后续工作在更大规模和更多任务上验证了类似设计的有效性。

### 10.4 架构复杂度

T2T-ViT 的架构比 ViT 复杂得多，包含多个卷积层和 Unfold 操作。这增加了代码实现和调试的难度。

**影响：** 在实际部署中，简单的架构（如 ViT 或 DeiT）可能更受欢迎。

## 十一、小结

T2T-ViT 解决了一个核心问题：**如何在数据量有限时让 Vision Transformer 有效学习局部特征**。

它的解决方案很优雅：用 Tokens-to-Token 模块模拟 CNN 的层次化特征提取过程，通过小卷积核 + 重叠窗口 + 渐进式降采样，让 Transformer 从零训练时也能捕捉到精细的局部信息。

这个方案之所以有效，是因为它尊重了视觉数据的本质：**局部性 + 层次性**。CNN 通过卷积核天然实现了这两点，T2T-ViT 则用 Transformer 的语言重新实现了它们。

T2T-ViT 的影响在于：
1. 证明了纯 Transformer 可以在 ImageNet 上从零训练达到有竞争力的精度
2. 为后续的视觉 Transformer 设计（Swin、PVT、CvT）提供了重要参考
3. 推动了"CNN-Transformer 融合"这一研究方向

虽然 T2T-ViT 后来被 Swin Transformer 等更优的架构超越，但它作为早期探索，为视觉 Transformer 的数据效率问题提供了重要的解决方案。
