## 一、先搞清楚"问题在哪里"

2020 年底，Vision Transformer（ViT）第一次证明了 Transformer 可以取代卷积做图像分类。这个结果对 CV 界是一个极为强烈的信号——NLP 的 Transformer 在视觉上也能 work。

但社区很快发现了一个尴尬的事实：**ViT 在中等规模数据集（如 ImageNet）上从零训练时，性能远不如同等规模的 CNN（如 ResNet50）。** ViT 需要先在 JFT-300M（300M 图片）这样的超大私有数据集上预训练，然后在 ImageNet 上微调，才能超过 ResNet。对于绝大多数没有 JFT-300M 访问权限的研究组来说，ViT 实际上不可用。

为什么会这样？本文作者通过特征可视化发现了两条根本性原因。

**第一条原因：简单的 patch 切分丢失了局部结构信息。** ViT 把 224×224 图片直接切成 16×16 的 patch（共 196 个 token），每个 patch 内部的像素被暴力展平为一个向量。这就意味着：相邻像素之间的局部结构——边缘、线条、纹理——在 token 化这一步就丢失了。CNN 靠堆叠 3×3 卷积天然地捕捉这些局部模式，但 ViT 的硬切分（hard split）完全不做这件事。结果就是 ViT 需要多得多的数据来"重新学习"这些局部模式。

**第二条原因：ViT 的 backbone 设计冗余且低效。** 作者可视化 ViT 的特征图时发现，大量 channel 的输出是零值（论文 Figure 2 中红色方框标注）。这说明骨干网络中的 channel 存在严重冗余——很多 channel 在给定计算预算和数据量下根本没有学到有效特征。ViT-S/16 用了 768 维的 hidden dimension 和 8 层，但大量维度是浪费的，而层数又不够深导致特征丰富度不足。

所以到 2021 年初，社区面临的问题是：**能不能设计一个纯 Transformer 架构，不依赖 JFT-300M 这样的大数据，从零在 ImageNet 上训练就能超过 ResNet？** T2T-ViT 就是在这个节点上出现的。

## 二、旧路线的真正问题

要理解 T2T-ViT 为什么重要，得先搞清楚 ViT 的两个短板为什么是"结构性的"。

### 2.1 Tokenization 问题：NLP 和 CV 的根本差异

NLP 中，"词"本身就是有语义的基本单位。Transformer 处理的 token 序列中，每个 token 内部已经包含了完整的语义信息。"apple"这个 token 本身就是"苹果"的意思，不需要再通过 token 内部的结构来理解。

但在 CV 中，**图像的基本语义单元不是 16×16 像素的 patch。** 一个 16×16 的 patch 内部可能包含边缘、纹理、颜色过渡等多种局部结构。ViT 的 tokenization 把这些结构全部展平成 768 维（16×16×3）的向量，然后丢给 Transformer。这样做的结果是：Transformer 的 self-attention 虽然可以建模 token 之间的全局关系，但 token **内部**的结构信息完全丢失了。

作者在 Figure 2 中直观地展示了这个差距。ResNet50 从底层（conv1）到中层（conv25）的特征图中，清晰地展示了边缘、线条、纹理等低级局部结构的渐进式学习。而 ViT-L/16 的特征图完全不是这样——所有的 attention block 都在建模全局关系（比如整只狗），局部结构信息没有被显式地建模。

这就是为什么 ViT 需要海量数据（JFT-300M）来弥补这个缺陷：它必须通过训练来"重新学习"如何从展平的 patch 中重建局部结构信息。CNN 通过 Conv1 的 7×7 卷积 + 后续 3×3 卷积层，天生就把这件事做了。

### 2.2 Backbone 设计问题：Wide 结构不是最优

ViT 的 backbone 设计直接搬用了 NLP Transformer 中常用的配置（hidden dimension 768，MLP size 3072，12 layers）。这种配置的特征是：宽（大 hidden dim）且浅（层数不多）。

但作者通过对 ViT 特征图的可视化发现：大量 channel 的输出为零或数值过大（Figure 2 红色框）。这暗示着 high-dimensional 的设置中存在大量冗余——在有限的数据和计算预算下，模型没有足够的训练样本来填满这么多维度。

CNN 领域的设计经验给出了一个反方向的经验：Wide-ResNet（宽而浅）的性能不如深而窄的结构。Zagoruyko 和 Komodakis 在 Wide Residual Networks 中讨论过 channel 宽度和深度之间的权衡。但 ViT 沿用了 NLP 的 wide 风格，没有考虑 CV 场景下数据量对 channel 利用率的影响。

### 2.3 效率问题：ViT 的参数和计算量浪费严重

从数字上看问题更加直观。ViT-S/16 在小配置下（8 layers, 768 hidden dim, 2358 MLP size）也有 48.6M 参数和 10.1G MACs。而对比的 ResNet50 只有 25.5M 参数和 4.3G MACs，但性能却更好（79.1% vs 78.1%）。

换句话说，**ViT 用了将近两倍于 ResNet 的参数和计算量，效果反而更差。** 这说明 ViT 的架构效率远低于精心设计过的 CNN。

### 2.4 小结：需要一个什么样的设计？

综合以上三点，一个能在 ImageNet 上从零训练超过 CNN 的纯 Transformer 设计必须解决：

1. **在 tokenization 阶段保留局部结构信息** — 不能再用 ViT 的 hard split
2. **高效的 backbone** — 减少 channel 冗余，增加特征丰富度
3. **参数和计算效率** — 至少要和 ResNet 相当，而不是远超

T2T-ViT 就是围绕这三个目标设计的。

## 三、这篇论文的核心思路

**这篇论文的核心思路是：在 ViT 之前加一个渐进式 tokenization 模块（T2T module），把图像逐步"结构化"成 token，在这个过程中同时建模局部结构信息并减少 token 数量；然后使用深窄结构（deep-narrow）的 backbone 来代替 ViT 的浅宽结构，减少冗余、提升特征丰富度。**

具体来说，两个主要设计组合在一起：

```text
T2T 模块（渐进式 tokenization） → 建模局部结构 + 逐步减少 token 数
       ↓
深窄 backbone                  → 减少 channel 冗余 + 更多层提升特征丰富度
```

这条技术路线的谱系位置是：

```text
ViT（hard split + 全局 attention, 2020）
   ↓
DeiT（数据增强 + 蒸馏训练, 2021, 不改变架构）
   ↓
T2T-ViT（渐进式 tokenization + 深窄 backbone, 2021） ← 本篇
```

T2T-ViT 和同期工作 DeiT（2021.01）的区别在于：DeiT 主要改进训练策略（数据增强、知识蒸馏），模型架构仍然是 ViT 的全局 attention + 固定 patch size。T2T-ViT 则从架构设计入手，在 tokenization 阶段做文章。两者是正交的——事实上论文实验部分也演示了 T2T-ViT 加上蒸馏可以进一步提升。

## 四、整体结构与 forward 流程

### 4.1 整体架构

T2T-ViT 由两个主要部分组成：T2T 模块和 T2T-ViT backbone（Figure 4）。

```text
输入图像: 224×224×3
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│ T2T 模块（渐进式 tokenization）                          │
│                                                         │
│  Step 1: Soft Split #1（7×7, stride=4, padding=2）       │
│          224×224×3 → 56×56 tokens (每个 token 147维)      │
│          输出: 56×56×147                                  │
│          线性投影到 64                                    │
│    │                                                     │
│    ▼                                                     │
│  Step 2: T2T Transformer #1 + Re-structurization         │
│          Transformer 层处理（MSA + MLP）                   │
│          重塑为图像 56×56×64                               │
│    │                                                     │
│    ▼                                                     │
│  Step 3: Soft Split #2（3×3, stride=2, padding=1）       │
│          56×56×64 → 28×28 tokens (每个 token 576维)       │
│          线性投影到 64                                    │
│    │                                                     │
│    ▼                                                     │
│  Step 4: T2T Transformer #2 + Re-structurization         │
│          Transformer 层处理                                │
│          重塑为图像 28×28×64                               │
│    │                                                     │
│    ▼                                                     │
│  Step 5: Soft Split #3（3×3, stride=2, padding=1）       │
│          28×28×64 → 14×14 tokens (每个 token 576维)       │
│          线性投影到 backbone hidden dimension              │
│          输出: 196 × hidden_dim                           │
└─────────────────┬───────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────┐
│ T2T-ViT Backbone（深窄结构）                              │
│  输入: 196 token + 1 class token = 197 token             │
│  + Sinusoidal Position Embedding                         │
│                                                         │
│  Layer 1...N: 标准 Transformer Block (Pre-LN)            │
│    LayerNorm → MSA → 残差 → LayerNorm → MLP → 残差       │
│                                                         │
│  输出: class token 经 LayerNorm → FC 分类                 │
└─────────────────────────────────────────────────────────┘
```

### 4.2 前向过程逐步跟踪（T2T-ViT-14，224×224 输入）

**Step 1 — 输入：** 224×224×3 RGB 图像。

**Step 2 — Soft Split #1：** patch_size=7×7，stride=4，padding=2。输出 56×56=3136 个 patch，每个 patch 尺寸 7×7×3=147 维。等价于卷积中的 unfold 操作，但**有重叠**（因为 stride=4 < patch_size=7）。

线性投影到 64 维：3136×147 → 3136×64。

**Step 3 — T2T Transformer #1：** 标准的 Pre-LN Transformer block，MSA + MLP。输入 3136 个 token，hidden_dim=64，MLP_size=64。这里的 channel 非常小（64），因为 token 数量很大（3136），需要在 MACs 和性能之间取得平衡。

**Step 4 — Re-structurization + Soft Split #2：** 将 3136 个 64 维的 token 重塑为 56×56×64 的图像空间排列。然后用 Soft Split（3×3，stride=2，padding=1）切分，输出 28×28=784 个 patch，每个 patch 3×3×64=576 维。

线性投影到 64 维：784×576 → 784×64。

**Step 5 — T2T Transformer #2：** 同上，784 个 token，hidden_dim=64。

**Step 6 — Re-structurization + Soft Split #3：** 重塑为 28×28×64，Soft Split（3×3，stride=2，padding=1），输出 14×14=196 个 patch，每个 576 维。

线性投影到 backbone 的 hidden dimension（T2T-ViT-14 为 384）。

**Step 7 — T2T-ViT Backbone：** 196 个 token + 1 个 class token（共 197 token），加上 Sinusoidal Position Embedding。经过 14 层 Transformer block（hidden_dim=384，MLP_size=1152）。class token 经 LayerNorm 后接全连接层分类。

T2T-ViT backbone 输出的是一个 class token（类似 BERT 的 [CLS] token），接一个全连接层做分类。和 ViT 的输出格式完全一致。

## 五、核心模块逐个解释

### 5.1 Soft Split：有重叠的 patch 切分

**问题：** ViT 的 hard split（patch size = stride，无重叠）把图像切成独立 patch，相邻 patch 之间的关联在 tokenization 阶段完全丢失。

**解决方案：** Soft Split（有重叠的 patch 切分）。patch_size=7×7，stride=4（相当于 3 个像素的重叠），padding=2。相邻的 patch 之间存在重叠区域，使得一个像素可以被多个 patch 覆盖。

**为什么这个设计是合理的：**

有重叠的切分在视觉中天然有意义——一个物体轮廓附近的像素，既属于"物体"也属于"背景"。用硬切分一刀切，会把它们分配到不相邻的 patch 中。用软切分，相邻 patch 共享了重叠区域的信息，为后续 T2T 模块中的 Tokens-to-Token 聚合建立了"周边 token 之间应该存在强关联"的先验（prior）。

**工作方式：**

```text
Soft Split 参数:
  patch_size = k (7), stride = k - s = 4 (s=3, 重叠量)
  padding = p (2)

输出 token 数量 = ⌊(224 + 2×3 - 7) / 4 + 1⌋ × ⌊(224 + 2×3 - 7) / 4 + 1⌋
                = ⌊55.75 + 1⌋ × ⌊55.75 + 1⌋ = 56 × 56 = 3136

每个输出 token 维度 = k × k × c = 7 × 7 × 3 = 147
```

**和卷积的关系：** Soft Split 在数学上等价于 `torch.nn.Unfold`（im2col 操作），再加上一个线性投影。但它**没有卷积中的共享权重**——卷积是对每个窗口做加权和（kernel learned weights），而 Soft Split 是直接拼接窗口内所有像素值再投影。论文在消融实验中比较了用 3 层卷积替换 T2T 模块（T2T-ViT_c-14），发现卷积版本不如原始 T2T 模块（80.8% vs 81.5%）。

### 5.2 Re-structurization：把 token 序列"还原"成图像

**问题：** T2T module 中的 Transformer 层处理的是 token 序列（spatial 维度被展平了）。但 Soft Split 需要输入的是**图像空间排列**（H×W×C）。

**解决方案：** 在每个 T2T Transformer 层之后，把输出的 token 序列 `T' ∈ ℝ^{l×c}` 重塑为图像形式 `I ∈ ℝ^{h×w×c}`，其中 `l = h × w`。

```python
def re_structurize(tokens, h, w):
    """tokens: [B, l, c], output: [B, h, w, c] where l = h*w"""
    return tokens.reshape(B, h, w, c)
```

这个操作极其简单——只是一个 reshape。但它的作用很重要：**它把 Transformer 学习到的"全局关系"和 Soft Split 需要的"空间局部性"连接起来了。** Transformer 层可以跨所有 token 做全局 attention，输出被重塑回图像后，Soft Split 在空间局部性上做切分聚合。

**为什么叫"Re-structurization"？** 因为 ViT 的 tokenization 是把图像结构（spatial arrangement）暴力展平为序列，丢失了结构信息。而 Re-structurization 是在 Transformer 处理后"恢复"空间结构，让后续的 Soft Split 可以在空间上感知局部性。整个 T2T 过程就是在 空间结构 → 序列 → 空间结构 → 序列 之间交替。

### 5.3 Tokens-to-Token：局部信息聚合

**核心操作：** 在 Soft Split 中，相邻的 token（来自图像中相邻的 patch）被拼接成一个 token。

假设 Soft Split 输入的图像 I 有 56×56 的空间网格。对于输出 token 矩阵中的位置 (i, j)，输入图像中有 7×7=49 个像素（在通道维度上全部展平）被拼成一个 token。但由于 stride=4，相邻输出 token 的 7×7 区域有重叠。

这等价于：**每个输出 token 包含了来自它局部邻域的信息，并且这个邻域和相邻 token 的邻域有重叠。** 所以"Tokens-to-Token"这个名称很准确——多个相邻 token 的信息被聚合（concatenation）成一个 token。

从第一层 Soft Split（7×7/stride4）后，3136 个 147 维 token，到第二层（3×3/stride2）后，784 个 576 维 token，再到第三层后，196 个 token。每次聚合都在增加每个 token 的"感受野"——最后输出的 196 个 token（14×14 网格），每个 token 的感受野已经覆盖了原始图像的大部分区域。

### 5.4 深窄 Backbone：减少冗余，增加深度的设计

**问题：** ViT 的 backbone 使用宽而浅的结构（ViT-S/16: 8 层，768 hidden dim），大量 channel 无效（Figure 2 红色框），且层数不够导致特征丰富度不足。

**解决方案：** 从 Wide-ResNet 的 deep-narrow vs shallow-wide 对比中得到启发，采用**深窄结构**：减少 hidden dimension（从 768 降到 384），增加层数（从 8 层增加到 T2T-ViT-14 的 14 层），同时减小 MLP 比例。

实验验证了这个方向：
- **ViT-DN（Deep-Narrow）：** 384 hidden dim, 16 层 → 24.5M 参数, 5.5G MACs, **79.0%**（比基线 +0.9%）
- **ViT-SW（Shallow-Wide）：** 1024 hidden dim, 4 层 → 47.9M 参数, 9.9G MACs, **69.9%**（比基线 -8.2%）

同样的 channel 预算下，把维度做窄但层数加多，参数和 MACs 减半，精度反而提升。如果反向去做宽而浅，精度暴跌 8.2%。

**为什么深窄对 ViT 有效？**

直觉上，这是因为视觉信号的局部连续性导致的 channel 冗余。在自然图像中，相邻像素之间的变化是平滑的，因此用 768 维的通道去表示这些局部模式是一种浪费——大量的维度实际上在学相同的模式（或者什么都不学，变为零值）。把维度压到 384，迫使模型在有限的通道中学习更紧凑的表征。同时更多的层数（16 vs 8）给了模型更多的非线性变换机会，提升了特征的丰富度和层次性。

### 5.5 Performer 层 vs Transformer 层

**问题：** T2T 模块中 token 数量很大（第一阶段 3136 个），如果使用标准 Transformer 的全局 attention（复杂度 O(N²)），内存和计算都会爆炸。3136² ≈ 9.8M 的 attention 矩阵，每个 head 一个。

**解决方案：** 有两个选项：
1. **标准 Transformer 层：** 直接做，但把 channel 设得很小（32 或 64）来减少 MACs
2. **Performer 层：** 使用 FAVOR+ 机制（Choromanski et al., 2020），用随机特征映射把 attention 复杂度从 O(N²) 降到 O(N)

论文在 T2T-ViT_t-14 中使用 Transformer 层（MACs: 6.1G），在 T2T-ViT-14 中使用 Performer 层（MACs: 4.8G）。Performer 版本 MACs 减少 21.3%，精度从 81.7% 降到 81.5%（-0.2%）。这是用少量精度换取计算效率的权衡。

## 六、核心公式与数学直觉

### 6.1 T2T 迭代过程

整个 T2T 模块的迭代过程由公式 (4) 描述：

$$T'_i = \text{MLP}(\text{MSA}(T_i))$$
$$I_i = \text{Reshape}(T'_i)$$
$$T_{i+1} = \text{SS}(I_i)$$
$$i = 1, ..., n-1$$

对于输入图像 I₀，先做一次 Soft Split：T₁ = SS(I₀)。经过 n-1 次迭代后，输出固定长度的 token 序列 T_f。

公式背后的直觉：
- **MLP(MSA(T_i))**：标准 Transformer 处理，在所有 token 之间做全局 attention（注意是全局！这一步让 T2T 模块不同于简单的卷积堆叠）
- **Reshape(T'_i)**：恢复空间排列
- **SS(I_i)**：在空间上做重叠切分 + 局部聚合

关键洞察：T2T 模块**同时做了全局 attention（Transformer 层）和局部聚合（Soft Split）**。这和 Conv+Transformer 混合模型不同——那些模型是先用卷积提取局部特征，再交给 Transformer。T2T 是在**同一个过程中交替做全局和局部处理**。

### 6.2 Soft Split 输出长度公式

公式 (3) 给出了 Soft Split 后的 token 长度：

$$l_o = \lfloor \frac{h + 2p - k}{k - s} + 1 \rfloor \times \lfloor \frac{w + 2p - k}{k - s} + 1 \rfloor$$

这和卷积的输出尺寸公式完全一致，其中 k 是 patch 大小，k-s 等效于 stride。

### 6.3 Backbone 工作公式

公式 (5) 描述了 backbone 的工作方式：

$$T_{f_0} = [t_{cls}; T_f] + E, \quad E \in \mathbb{R}^{(l+1) \times d}$$
$$T_{f_i} = \text{MLP}(\text{MSA}(T_{f_{i-1}})), \quad i = 1, ..., b$$
$$y = \text{fc}(\text{LN}(T_{f_b}))$$

这和 ViT 的公式**完全一样**——说明 T2T-ViT 的 backbone 就是 ViT 的 backbone，只是 hidden dimension更小、层数更多。

### 6.4 参数效率对比

T2T-ViT-14 vs ViT-S/16：
- 参数：21.5M vs 48.6M（减少 55.8%）
- MACs：4.8G vs 10.1G（减少 52.5%）
- Top-1：81.5% vs 78.1%（提升 3.4%）

这意味着 T2T-ViT 用不到 ViT 一半的参数和计算量，达到了更高的精度。

## 八、消融实验深度分析

### 8.1 T2T 模块的贡献：+2.0~2.2%

对比 T2T-ViT-14（81.5%）和 T2T-ViT-14 w/o T2T（79.5%），**T2T 模块本身贡献了 2.0% 的提升**。如果用 Transformer 层的版本（81.7%）则贡献 2.2%。

这验证了渐进式 tokenization 确实比 ViT 的 hard split 更有效。而且这个提升不是来自参数量的增加——两模型参数量相近（21.5 vs 21.1M）。

### 8.2 卷积替换实验：T2T 模块优于卷积堆叠

T2T-ViT_c-14 用 3 层卷积（kernel size 7/3/3，stride 4/2/2）替换 T2T 模块，精度 80.8%——比无 T2T 的基线（79.5%）好（因为卷积也能建模局部结构），但比标准 T2T 模块（81.5%）差 0.5~1.0%。

原因：卷积只有局部感受野，而 T2T 模块中的 Transformer 层做了**全局 attention**，让 Soft Split 中的局部聚合受到了全局上下文的影响。这比纯卷积的局部处理更适合建模图像中的远程依赖。

### 8.3 深窄结构的贡献：+2.7%

把 T2T-ViT-14 的 backbone 从深窄（384 dim, 14 layers）改为浅宽（768 dim, 4 layers），T2T-ViT-d768-4 的精度从 81.5% 掉到 78.8%（-2.7%）。两模型参数量和 MACs 相近（25.0M, 5.4G vs 21.5M, 4.8G）。

这验证了论文的核心假设：**ViT 的 shallow-wide 结构确实存在 channel 冗余，深窄结构能在相同甚至更少的预算下提供更好的性能。**

### 8.4 SE 模块的有效性

在 ViT 上加入 SE 模块得到 +0.3%（78.1→78.4%），在 T2T-ViT 上得到 +0.1%（81.5→81.6%）。SE 在 CNN 上效果显著（ResNet50: 79.1→80.3%，+1.2%），但在 Transformer 上增益小得多。这可能是因为 Transformer 的 MSA 已经在通道维度上做了 attention（multi-head 机制），SE 的通道 attention 与之重复。

### 8.5 DenseNet 连接迁移失败

在 ResNet 上有效的 DenseNet 结构（DenseNet201 用更少参数达到更高精度），迁移到 ViT 反而有害（-1.3%），迁移到 T2T-ViT 也有害（-1.1%）。作者推测原因：Transformer 的 residual connection 已经提供了很强的梯度流动路径，再加 dense connection 引入了不必要的冗余和计算开销。

### 10.3 token_transformer.py 核心

```python
class Token_transformer(nn.Module):
    """T2T Transformer block: Pre-LN standard Transformer"""
    def __init__(self, dim, num_heads=1, mlp_ratio=1.0):
        self.norm1 = nn.LayerNorm(dim)
        self.attn = Attention(dim, num_heads=num_heads, qkv_bias=False)
        self.norm2 = nn.LayerNorm(dim)
        self.mlp = Mlp(dim, int(dim * mlp_ratio))
    
    def forward(self, x):
        x = x + self.attn(self.norm1(x))
        x = x + self.mlp(self.norm2(x))
        return x
```

注意 `num_heads=1` 是 T2T-14 的默认设置，因为 channel 只有 64，分成多个 head 后每个 head 的维度太小。

### 10.4 t2t_vit.py 核心架构

```python
class T2T_ViT(nn.Module):
    def __init__(self, ...):
        # T2T 模块
        self.soft_split0 = nn.Unfold(kernel_size=7, stride=4, padding=2)
        self.soft_split1 = nn.Unfold(kernel_size=3, stride=2, padding=1)
        self.soft_split2 = nn.Unfold(kernel_size=3, stride=2, padding=1)
        
        # T2T Transformer 层（channel 很小: 32 或 64）
        self.transformer1 = Token_transformer(dim=64, ...)
        self.transformer2 = Token_transformer(dim=64, ...)
        
        # 线性投影层
        self.project1 = nn.Linear(7*7*3, 64)   # 147→64
        self.project2 = nn.Linear(3*3*64, 64)  # 576→64
        self.project3 = nn.Linear(3*3*64, ...) # 576→backbone_dim
        
        # Backbone (deep-narrow ViT)
        self.blocks = nn.ModuleList([
            Block(dim=backbone_dim, num_heads=...)
            for _ in range(depth)
        ])
```

### 10.5 实现细节

**1. 使用 `nn.Unfold` 实现 Soft Split。** 这和 ViT 代码保持一致，但 ViT 只用一次 Unfold（patch_size=16, stride=16，无重叠），而 T2T-ViT 用了三次 Unfold 且都有重叠。

**2. Re-structurization 的实现。** 代码中通过 `x = x.transpose(1, 2).reshape(B, C, H, W)` 实现，其中 x 是 token 序列 `[B, L, C]`，reshape 到 `[B, C, H, W]`。

**3. 使用 Performer 层。** Performer 由 FAVOR+ 机制实现，通过随机特征映射（random feature map）将 attention 复杂度从 O(N²) 降到 O(N)。代码中使用 `PerformerAttention` 类，继承自标准 Attention。

**4. T2T Transformer 层使用 Pre-LN。** LayerNorm 在 MSA 和 MLP 之前，和标准 ViT（Post-LN）不同，训练稳定性更好。

**5. Sinusoidal Position Embedding。** 和 ViT 的可学习 position embedding 不同，T2T-ViT 使用固定的三角函数位置编码（Sinusoidal PE）。这可能是可学习 PE 在 ImageNet 上从头训练时容易过拟合。

## 十一、Related Work 与技术谱系位置

### 11.1 技术树中的位置

```text
[Transformer, Vaswani et al., 2017]
   │
   ├── [ViT, Dosovitskiy et al., 2020]  ← 第一个视觉 Transformer
   │      │
   │      ├── [DeiT, Touvron et al., 2021]  ← 训练策略改进
   │      │      │
   │      │      └── [T2T-ViT, Yuan et al., 2021]  ← 本篇
   │      │
   │      └── [CNN 架构迁移研究]
   │             │
   │             ├── [Deep-Narrow ViT]
   │             └── [T2T-ViT 的深窄 backbone]
   │
   ├── [Swin Transformer, Liu et al., 2021]  ← 层级 + 局部窗口
   │
   └── [PVT, Wang et al., 2021]  ← 金字塔 Transformer
```

T2T-ViT 在技术树中处于 B1-5（patch tokenization 范式），它的直接前驱是 ViT（B1-3）和 DeiT（B1-4）。

### 11.2 前驱工作

**Transformer（Vaswani et al., 2017）：** T2T-ViT 保留了标准 Transformer 的 MSA + MLP + residual + LayerNorm。T2T-ViT backbone 就是标准 Transformer 的堆叠。

**ViT（Dosovitskiy et al., 2020）：** T2T-ViT 的核心贡献是对 ViT 的 tokenization 进行了改进。保留 ViT 的 backbone 结构（class token + position embedding + Transformer blocks + 分类头），只在 tokenization 阶段加了一个 T2T 模块。

**DeiT（Touvron et al., 2021）：** 同期工作。DeiT 用数据增强和知识蒸馏提升 ViT 训练效果。论文在介绍中明确区分："T2T-ViT 专注于架构设计，DeiT 专注于训练策略，两者正交"。实验也展示了 T2T-ViT + 蒸馏可以进一步提高性能。

**CNN 架构设计（Wide-ResNet, DenseNet, SE-Net, ResNeXt, GhostNet）：** 论文系统地迁移了 5 种 CNN 架构设计到 ViT 中，这是论文的一个重要贡献——它提供了"CNN 设计经验可以指导 ViT backbone 设计"的实证证据。

### 11.3 同期竞品

**DeiT（2021.01）：** T2T-ViT 在 224×224 输入上（81.5%）超过了 DeiT-small（79.9%）和 DeiT-small-Distilled（81.2%），且不需要 CNN teacher 模型。但 DeiT 的蒸馏方案是正交的，可以叠加。

**Swin Transformer（2021.03）：** 同期但稍晚。Swin 也改进了 ViT 的 tokenization，但走的是完全不同的路线——局部窗口 attention + 层级特征图。T2T-ViT 在 ImageNet 分类上（T2T-ViT_t-14: 81.7%）和 Swin-T（81.3%）相比有竞争力，但 Swin 在检测和分割上有更大的优势和影响力。

**PVT（2021.03）：** 同期。PVT 用 Spatial Reduction Attention（空间降维）实现金字塔 Transformer。T2T-ViT 更专注于分类任务，而 PVT 一开始就瞄准了密集预测任务。

### 11.4 后续影响

T2T-ViT 的渐进式 tokenization 思路影响了后续一些工作，但整体影响力不如 Swin Transformer。主要原因：
- T2T-ViT 只做了 ImageNet 分类，没有扩展到检测/分割等密集预测任务
- T2T-ViT backbone 仍然是 ViT 的单分辨率全局 attention，在密集预测上效率不高
- Swin 的层级局部 attention 方案更通用、影响更大

但 T2T-ViT 的两个核心洞察——**渐进式 tokenization** 和 **深窄 backbone**——在后续工作中被广泛采纳：
- 渐进式 tokenization 的思路在后来的 CrossFormer、RegionViT 等工作中有继承
- 深窄 backbone 的设计已经成为视觉 Transformer backbone 的常见选择

## 十三、范式意义

**T2T-ViT 的范式意义：它是"ViT 架构改进"路线的早期关键节点，证明了 ViT 可以通过精心设计的 tokenization 和 backbone 能在中等规模数据上超越 CNN。**

具体来说，T2T-ViT 完成了三件事：

### 13.1 打破了 "Transformer 需要海量数据" 的认知

2020 年 ViT 的结果给人的印象是：Transformer 做视觉需要 JFT-300M 级别的数据，否则打不过 CNN。T2T-ViT 证明这不是 Transformer 架构的根本缺陷，而是 tokenization 和 backbone 设计的问题。**用更好的 tokenization（渐进式、有重叠）+ 更好的 backbone（深窄结构），Transformer 可以在 ImageNet 上从零训练超越 CNN。**

### 13.2 揭示了 tokenization 在视觉 Transformer 中的核心地位

T2T-ViT 最重要的洞察是：**tokenization 不是可以忽视的预处理步骤——它对 Transformer 在视觉任务上的表现有决定性影响。** ViT 简单地把图像切成固定 patch，丢失了局部结构。T2T-ViT 提出的渐进式 tokenization 从粗到细地逐步结构化图像，把局部结构嵌入到 token 中。

这个洞察对后续工作产生了深远影响：Swin 用不同尺度的窗口 attention，PVT 用空间降维金字塔，CrossFormer 用多尺度 patch embedding——都是在 tokenization 阶段做文章。

### 13.3 系统性地验证了 CNN 设计经验对 ViT backbone 的指导作用

论文对 5 种 CNN 架构设计进行了系统性的迁移实验。结果清晰地表明：
- 深窄结构最有效（参数减半精度提升）
- 密集连接对 Transformer 有害
- 通道注意力（SE）有微弱提升
- 多头（ResNeXt）几乎没有影响

这些实验为后续视觉 Transformer 的 backbone 设计提供了实证基础。

### 13.4 分类：refinement node

在技术树中，T2T-ViT 是一个 **refinement node（精化节点）**——它在 ViT 的基本范式下做了系统性改进，提升了性能和效率。它不是 foundation node（建立新范式），因为它仍然使用 ViT 的全局 attention backbone 和 class token 设计。它的贡献在于指出了 ViT 的瓶颈所在，并给出了有效的改进方案。

## 十四、局限性与后续方向

### 14.1 没有扩展到密集预测任务

T2T-ViT 只在 ImageNet 分类上做了实验。它没有检测、分割等密集预测任务的评估。这使得它的通用性受到限制——同期的工作（Swin、PVT）从一开始就考虑了密集预测场景。

T2T-ViT 的 backbone 仍然是 ViT 的单分辨率全局 attention，在检测/分割的高分辨率输入下，token 数量激增会导致 O(N²) 的 attention 复杂度不可接受。

### 14.2 T2T 模块的计算开销

虽然 T2T 模块只用了很小的 channel（32 或 64），但第一阶段有 3136 个 token，即使 Performer 的线性 attention 也有可观的 MACs（T2T-ViT-14 的 T2T 模块部分贡献了 0.6G MACs 中的大部分）。在一些资源受限的场景下，第一阶段的 3136 个 token 仍然太大。

### 14.3 渐进式 tokenization 的三步设计缺乏系统性调优

论文选择 P=[7,3,3], S=[3,1,1] 是启发式的，没有验证不同的 patch size 组合、不同的迭代次数（n=2）是否最优。更大迭代次数（n=3 或 4）可能带来更好的性能，但也会增加计算成本。

### 14.4 在 Lite 模型上 MACs 仍然过高

T2T-ViT-12（6.9M, 76.5%）精度比 MobileNetV2 1.4x（6.9M, 75.6%）高，但 MACs 是 1.8G vs 0.6G——高出 3 倍。这是因为 Transformer 的密集矩阵乘法在理论上比深度可分离卷积（depthwise separable convolution）的计算密度更高。对于移动端部署来说，MACs 是关键指标，T2T-ViT 在这方面仍有差距。

### 14.5 没有在更大规模上验证

论文没有在 ImageNet-21k 或 JFT-300M 上预训练 T2T-ViT，因此不清楚 T2T 模块在大规模训练下是否仍然有效。后续的 ViT-G/14（Zhai et al., 2022）证明了超大 ViT 在 JFT-3B 上的有效性，但 T2T 这样的渐进式 tokenization 没有被采纳。

### 14.6 Performer 的精度损失

使用 Performer 层（T2T-ViT-14: 81.5%）比标准 Transformer 层（T2T-ViT_t-14: 81.7%）低 0.2%。虽然不大，但对追求极致精度的场景需要权衡。

### 14.7 和 DeiT 蒸馏的未充分探索

论文提到了而且展示了蒸馏可以进一步提升 T2T-ViT（T2T-ViT-7-Distilled: 73.1%, T2T-ViT-12-Distilled: 77.4%），但没有深入探索。考虑到 DeiT 证明了蒸馏对 ViT 很有帮助，T2T-ViT + 更强大的 teacher 可能还有提升空间。

## 十五、小结

直白说，T2T-ViT 的价值不在于提出了一个划时代的架构（它用了 ViT 的 backbone，没有造新轮子），而在于**它精准地诊断出了 ViT 的两大问题，并给出了简单有效的修补方案**。

在它之前，社区对 ViT 的认知是："Transformer 做视觉需要海量数据，否则打不过 CNN。"在它之后，社区对 ViT 的认知更新为："ViT 的瓶颈在 tokenization 和 backbone 设计上，修好这两个短板，Transformer 可以在 ImageNet 上从零训练超越 CNN。"

这个认知转变靠的是两条关键洞察：

1. **渐进式 tokenization 比一次性 hard split 好得多。** 把图像逐步结构化成 token（粗→细），在每个阶段既做全局 attention（Transformer）又做局部聚合（Soft Split），可以用较少的参数和计算量实现更好的性能。

2. **ViT 的 shallow-wide backbone 存在严重冗余。** 大量 channel 输出为零，暴露出 hidden dimension 过大而层数不足的问题。采用深窄结构——减少 channel、增加深度——可以在参数减半的情况下提升性能。

T2T-ViT 的这个诊断方法和 Swin 不同：Swin 从根本上重构了 attention 的计算方式（局部窗口 + 偏移），而 T2T-ViT 保留了 ViT 的全局 attention backbone，只是在 ViT 之前加了一个更聪明的 tokenization 模块，同时把 backbone 的宽高比调深了。

这两种方案各有优劣：T2T-ViT 的方案更简单、更接近 ViT 的原始设计，但只做了分类任务；Swin 的方案改动更大但更通用，在密集预测上全面超越了 CNN。

在技术树中，T2T-ViT 是一个重要的 refinement node。它没有开辟全新的方向，但在 ViT 范式内部做了有效的系统优化，并提供了"CNN 架构设计经验可以指导 ViT backbone 改进"的实证证据。
