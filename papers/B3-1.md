# ConvNeXt: A ConvNet for the 2020s（2022）

**论文：** [arXiv:2201.03545](https://arxiv.org/abs/2201.03545) · Zhuang Liu, Hanzi Mao, Chao-Yuan Wu, Christoph Feichtenhofer, Trevor Darrell, Saining Xie · FAIR + UC Berkeley · CVPR 2022

**技术分支：** B3 — 视觉 backbone 局部归纳偏置 / B3-1 — local inductive bias 回归范式

**代码：** [facebookresearch/ConvNeXt](https://github.com/facebookresearch/ConvNeXt)

---

## 一、先搞清楚"问题在哪里"

2021 年，视觉 Transformer 的势头已经不可阻挡。Swin Transformer 在 COCO 检测上把 ResNet 甩开了 +4.2 box AP，在 ADE20K 分割上超出 4+ mIoU。社区的主流叙事是：**Transformer 的 self-attention 机制本质上优于卷积**，CNN 的时代结束了。

但 ConvNeXt 的作者们注意到一个微妙的矛盾：

**Swin Transformer 之所以能在检测和分割上做得这么好，很大程度上是因为它重新引入了卷积的归纳偏置**——局部窗口 attention（等价于卷积的局部连接）、层级下采样（等价于 CNN 的 stage 设计）、4×4 的 patchify stem。换句话说，Transformer 之所以能超越 CNN，恰恰是因为它变得"更像 CNN"了。但社区却把这些成功归因于 attention 的"内在优越性"。

这个矛盾引出了一个非常直接的问题：

> **如果把 ResNet 逐步改造成 Swin Transformer 的架构风格——但完全不引入 attention 模块——性能差距还会有多大？**

这就是 ConvNeXt 的核心问题。它不是在问"Transformer 和 CNN 谁更好"，而是在问：**如果把 Swin 的设计理念（训练配置、宏观结构、宽度配置、kernel 大小、微观模块设计）全部移植到纯卷积网络上，结果会怎样？**

## 二、旧路线的真正问题

在 ConvNeXt 之前，CNN 和 Transformer 的对比存在一个方法论上的问题：

### 2.1 不公平的比较 baseline

标准的 ResNet-50 训练只有 90 epochs，用的 SGD + 基础数据增强（RandomCrop + RandomFlip），没有 Mixup、Cutmix、RandAugment、Stochastic Depth 这些"现代"训练技巧。

而 Swin Transformer 的训练用了 300 epochs、AdamW 优化器、全套数据增强和正则化。拿一个用 90 epochs SGD 练出来的 ResNet-50（76.1%）和一个用 300 epochs AdamW 练出来的 Swin-T（81.3%），说"Transformer 更好"，这显然不是 apples-to-apples 的比较。

### 2.2 "黑箱"比较，缺乏消融

当时的主流论文直接比较 Swin-T vs ResNet-50，没有系统地控制变量——哪些提升来自训练技巧、哪些来自架构设计、哪些来自 attention 本身，没有人知道。

ConvNeXt 的核心贡献之一，就是**系统性地拆解了这个差距**：先统一训练配置，然后一个变量一个变量地改架构，每步都记录精度变化。这不是一个典型的新架构论文——它更像一篇"实验报告"，目标不是发明新模块，而是理解 CNN 和 Transformer 之间到底差在哪里。

### 2.3 为什么这种"往回看"的研究在当时有紧迫性

2021 年底到 2022 年初，越来越多的视觉任务默认使用 Swin 作为 backbone。CNN 看起来像上一个时代的产物。但 ConvNeXt 的作者（很多是 ResNeXt、ResNet 家族的作者本人）想知道：CNN 是真的不行了，还是只是没有用上现代的训练和设计技巧？

如果答案是后者，那么**纯 CNN 架构依然可以是一条可行的路线**——它比 Swin 简单（没有 shifted window 的 mask 逻辑，没有 attention 的 softmax 和 QKV 投影），部署更方便（任何支持 Conv2d 的硬件都能跑），而且或许可以做得一样好甚至更好。

## 三、这篇论文的核心思路

**核心思路：以 ResNet-50 为起点，沿着 Swin Transformer 的设计方向，做 7 步递进式改良——每一步只改一个变量，记录精度变化。最终得到的纯卷积网络（ConvNeXt）在三个主流 benchmark 上全面赶上甚至超过 Swin Transformer。**

这 7 步构成一条清晰的"现代化"路径：

```text
ResNet-50 (76.1%)                              ← 起点（标准训练）
   ↓ ① 训练配方升级
改进训练的 ResNet-50 (78.8%)                    ← 新基线
   ↓ ② 宏观设计 (stage 比例 + patchify stem)
(79.5%)
   ↓ ③ ResNeXt-ify (depthwise conv + 加宽)
(80.5%)
   ↓ ④ Inverted Bottleneck
(80.6%)
   ↓ ⑤ 大 kernel (3×3 → 7×7 depthwise)
(80.6%)
   ↓ ⑥ 微观设计 (GELU, fewer norm, LayerNorm, Layer Scale)
(81.9%) ← 首次超过 Swin-T (81.3%)
   ↓ ⑦ stage 比例调整 (3:3:9:3 → 3:3:27:3 等)
ConvNeXt (82.1%)                               ← 最终模型（Swin-T 81.2%）
```

这里每一步的精度数字来自论文中Figure 2和Section 2的报告。注意步骤③→④→⑤的精度变化并不大（80.5% → 80.6% → 80.6%），但它们的价值在于**对准 FLOPs**——为后续步骤创造合理的 FLOPs 预算。

这条技术路径的谱系位置非常独特：它不是从 Transformer 出发改进 Transformer，而是**从 CNN 出发，把 Transformer 的设计理念"翻译"回卷积语言**。它不是 CNN vs Transformer 的辩论，而是两者的融合。

## 四、7 步现代化：逐步骤详解

### 第一步：训练配方升级（Training Recipe）

**改动：** 把 ResNet-50 的训练从标准的 90 epochs SGD 替换为接近 DeiT/Swin 的 300 epochs 配方。

**具体配置：**
- 优化器：AdamW（lr=4e-3, weight decay=0.05）
- 学习率：Cosine decay schedule, 20 epochs warmup
- 数据增强：Mixup, Cutmix, RandAugment, Random Erasing
- 正则化：Stochastic Depth（rate=0.1）, Label Smoothing
- Batch size：4096
- Epochs：300

**结果：** 76.1% → **78.8%**（+2.7%）

这个提升幅度本身就很有信息量：**一个技巧都没有改架构，仅仅是换了训练配方，就能追回近 3 个点**。这意味着原始 ResNet-50 和 Swin-T 之间至少三分之一的差距来自训练，而非架构。

这个实验颠覆了一个常见的认知误区："ResNet 的精度上限就是 76-78%，Transformer 天生更好。"

### 第二步：宏观设计（Macro Design）

这一步做了两个改动：

#### 2a. Stage compute ratio

ResNet-50 的 block 分布是 (3, 4, 6, 3)。Swin-T 的 block 分布是 (2, 2, 6, 2)，对应更大模型的 (2, 2, 18, 2)。

作者把 ResNet-50 改成 (3, 3, 9, 3)，和 Swin-T 的 1:1:3:1 比例对齐（但保持总 block 数近似）。

**结果：** 78.8% → **79.4%**（+0.6%）

这个调整对齐了 Swin Transformer 的 FLOPs 分布。注意：stage 3（1/16 分辨率）拿到最多的计算资源 + block 数，这对后续的 dense prediction 任务至关重要。

#### 2b. Patchify stem

ResNet 的 stem 是 7×7 stride=2 conv + max pool（4× 下采样）。Swin 用的是 4×4 stride=4 的 patchify layer（非重叠卷积）。

**结果：** 79.4% → **79.5%**（+0.1%）

stem 的影响很小。这说明 ResNet 的经典 stem 设计已经足够好。Patchify stem 的优势在于更简洁（少一层，少一个非线性层），但精度提升微乎其微。

### 第三步：ResNeXt-ify（深度可分离卷积 + 加宽）

这一步把标准 3×3 conv 换成 **depthwise conv**（groups=channels）。Depthwise conv 是 grouped convolution 的特例——每个 channel 独立和自己的空间邻域做卷积，不做 channel 间的信息混合。

为什么需要同时做两件事？

**Depthwise conv 大幅降低 FLOPs**——因为每个 filter 只处理 1 个 input channel 而不是所有 input channel。为了补偿容量损失，**增加 width** 到和 Swin-T 一致（64 → 96）。

这种"空间混合和通道混合分离"的设计，和 Transformer 的 self-attention（空间混合）+ MLP（通道混合）有很强的对应关系。在 Transformer 中，attention 做 token 之间的信息交换（空间维），MLP 做每个 token 内部 channel 的信息交换（通道维）。在 ConvNeXt 中，depthwise conv 做空间混合，1×1 conv 做通道混合——分工完全一致。

**结果：** 79.5% → **80.5%**（+1.0%）

注意 FLOPs 从 ~4.5G 增加到 5.3G——加宽带来了更多的计算量。精度提升 1 个点，说明 depthwise conv + 加宽是高效的。

### 第四步：Inverted Bottleneck

标准的 ResNet bottleneck 是"宽→窄→宽"：1×1 降维（宽→窄）→ 3×3（窄）→ 1×1 升维（窄→宽）。Inverted bottleneck 反过来：1×1 升维（窄→宽）→ depthwise conv（宽）→ 1×1 降维（宽→窄）。

这和 Transformer MLP 中的"expansion ratio=4"完全一致——MLP 把输入维度 expand 4 倍再投影回来。

**为什么用 inverted bottleneck？**

虽然 inverted bottleneck 中的 depthwise conv 在更宽的维度上运行（FLOPs 更高），但 shortcut 路径上的 1×1 conv 从 C→C 变成了 C→4C→C 的路径，其中 shortcut 的 1×1 降维层 FLOPs 大幅降低。整体网络 FLOPs 反而从 5.3G 降到了 4.6G。

**结果：** 80.5% → **80.6%**（+0.1%，FLOPs 降低）

在 ResNet-200/Swin-B regime 中，这个步骤的增益更大：81.9% → 82.6%（+0.7%），同时 FLOPs 也降低了。这说明 inverted bottleneck 在更大容量模型上优势更明显。

### 第五步：Large Kernel Size

Swin Transformer 在每个 block 中用 7×7 的局部窗口做 attention。ConvNeXt 把 depthwise conv 的 kernel size 从 3×3 换成了 7×7。

论文中系统性地比较了 kernel size：3×3、5×5、7×7、9×9、11×11。7×7 达到了最好的精度/FLOPs 平衡点。更大的 kernel（9×9、11×11）继续提升精度但收益递减。

**结果：** 80.6% → **80.6%**（持平，但 FLOPs 从 4.6G 增加到 ~4.6G+）

这里精度虽然没提升（在 ResNet-50/Swin-T regime），但 **建立了一个更大的 kernel baseline**，为后续微观设计的增益做了铺垫。

> 为什么 7×7 depthwise conv 不比 3×3 差？在常规 ConvNet 中，7×7 的参数量和计算量远大于 3×3（49 vs 9），但因为用了 depthwise conv，7×7 的 #params = 7×7×C（没有 depthwise 时的 #params = 7×7×C×C）。而且在 C=96 时，7×7 depthwise 的 49×96 ≈ 4.7K 参数，而 1×1 pointwise conv 有 96×384 ≈ 37K 参数——depthwise conv 占总 FLOPs 的比例很小，所以增加 kernel size 的成本相对有限。

### 第六步：微观设计（Micro Design）

这一步包含多个微调，每个的影响不大，但加起来贡献显著：

#### 6a. GELU 替代 ReLU
Swin 使用 GELU，ConvNeXt 同样换用 GELU。原因是 GELU 在 Transformer 中被验证效果更好，且和深度可分离卷积配合得当。

#### 6b. 减少激活层数量
在 ResNet bottleneck 中有 3 个激活层（每个 conv 后一个 ReLU）。ConvNeXt 改为只在 pointwise conv 之间保留一个激活层（depthwise conv 后不加激活）。

思路：depthwise conv 只是一个空间混合操作，每个 channel 独立做加权平均——在加权平均后立即加非线性激活会破坏空间信息的传播。这其实借鉴了 Transformer：self-attention（对应 depthwise conv）之后也没有激活——只做 weighted sum，然后接 LayerNorm + MLP。

#### 6c. 减少 Normalization 层
同理，只在两个 pointwise conv 之间放一个 LayerNorm。不在 depthwise conv 后独立加 norm。

#### 6d. LayerNorm 替换 BatchNorm
这是一个重要而微妙的选择。BatchNorm 在 CNN 中几乎是标配，但有若干问题：和 batch size 耦合、训练和推理行为不一致、小 batch 不稳定。LayerNorm 在 Transformer 中是标配，但在 convolution 中使用需要注意参数格式。

ConvNeXt 使用 **channels_first 的 LayerNorm**——在 channel 维上计算 mean/std，保持卷积层的数据排列格式（N, C, H, W）。

**为什么 LayerNorm 在 ConvNeXt 中有效？**

在标准 ResNet 中，BN 的统计信息依赖 batch 中的其他样本，这引入了 batch 之间的依赖。而 LN 只依赖当前样本的 channel 统计量，和 batch 无关。虽然 LN 在 ConvNet 中通常不如 BN（因为卷积的滑动窗口特性意味着不同空间位置的统计量应该共享），但 inverted bottleneck 的设计改变了这一点——在 inverted bottleneck 中，宽→窄的 bottleneck 结构导致 channel 维的信息更"密集"，LN 反而比 BN 更适合。

#### 6e. 独立的 downsampling 层
ResNet 的 downsampling 是通过 residual shortcut 中的 stride-2 conv 实现的，但这个 downsampling 在 residual path 内部完成。

Swin Transformer 使用独立的 Patch Merging 层在 stage 之间做下采样。ConvNeXt 同样：在所有残差块之前，加一个 stride-2 的 2×2 conv 做空间下采样 + channel 翻倍，放在一个单独的 downsample layer 中。

具体实现：在每个 stage（除第一个）之前，先做 LayerNorm + 2×2 stride=2 conv，然后进入该 stage 的残差块序列。

这样做的优势是：downsampling 和 feature transformation 解耦，梯度流更清晰。

#### 6f. Layer Scale 初始化
在残差连接之前，每个 block 的输出乘以一个可学习的缩放因子（初始化为 1e-6）。这个技巧来自 CaiT（Touvron et al., 2021），用于稳定深层 Transformer 的训练。

**Layer Scale 的效果**：初始时残差贡献几乎为 0（因子极小），block 相当于恒等映射；随着训练进行，因子逐渐增大，block 逐渐发挥作用。这相当于在训练早期"跳过"未稳定的 block，避免随机初始化带来的梯度问题。

**结果：** 80.6% → **81.9%**（所有微观设计合计，+1.3%）

这是 7 步中单一最大的精度提升（除了训练配方本身）。ConvNeXt 在这个点上首次超越 Swin-T（81.3%），尽管架构中完全没有 attention。

### 第七步：Stage Ratio 调整

在上一步完成后，ConvNeXt-T 的精度达到 81.9%（Swin-T 为 81.3%）。但对应更大模型（ConvNeXt-S/B），初始配置的精度还有差距。

作者发现，当把 stage 3 的 block 数大幅增加（从 9 到 27）后，大模型的精度显著提升：

| 变体 | Depths | Dims | ImageNet Top-1 |
|------|--------|------|---------------|
| ConvNeXt-T | (3, 3, 9, 3) | (96, 192, 384, 768) | 82.0 |
| ConvNeXt-S | (3, 3, **27**, 3) | (96, 192, 384, 768) | 83.1 |
| ConvNeXt-B | (3, 3, **27**, 3) | (128, 256, 512, 1024) | 83.8 |
| ConvNeXt-L | (3, 3, **27**, 3) | (192, 384, 768, 1536) | 84.3 |
| ConvNeXt-XL | (3, 3, **27**, 3) | (256, 512, 1024, 2048) | 85.6 (22K→1K) |

Swin-B 使用 1:1:9:1 的比例 (2, 2, 18, 2)，而 ConvNeXt-B 发现 3:3:27:3 的比例更好——即 stage 3（1/16 分辨率）分配更多的 block。这个调整使 stage 3 的 block 数从 18 增加到 27，stage 4 从 2 增加到 3。

**最终 ConvNeXt-T**：82.1%（比 Swin-T 81.2% 高 0.9%）

论文还尝试了 isotropic（非金字塔）的 ConvNeXt 变体，其中所有特征图保持相同分辨率（类似 ViT），但准确率显著低于金字塔版本（ConvNeXt-S isotropic 78.7% vs pyramid 83.1%），验证了层级结构对视觉任务的重要性。

## 五、前向过程逐步跟踪（ConvNeXt-T, 224×224 输入）

ConvNeXt 的前向路径非常简洁，没有 Swin 那样的窗口划分/移位/还原操作：

```
输入: [B, 3, 224, 224]
   │
   ▼
[Downsample Layer 0: Stem]          4×4 stride=4 conv + LayerNorm(channels_first)
   │                                 [B, 96, 56, 56]
   ▼
[Stage 1: 3× ConvNeXt Block]        dim=96, kernel=7, expansion=4
   │                                 56×56
   ▼
[Downsample Layer 1]                LayerNorm + 2×2 stride=2 conv
   │                                 [B, 192, 28, 28]
   ▼
[Stage 2: 3× ConvNeXt Block]        dim=192, kernel=7, expansion=4
   │                                 28×28
   ▼
[Downsample Layer 2]                LayerNorm + 2×2 stride=2 conv
   │                                 [B, 384, 14, 14]
   ▼
[Stage 3: 9× ConvNeXt Block]        dim=384, kernel=7, expansion=4
   │                                 14×14
   ▼
[Downsample Layer 3]                LayerNorm + 2×2 stride=2 conv
   │                                 [B, 768, 7, 7]
   ▼
[Stage 4: 3× ConvNeXt Block]        dim=768, kernel=7, expansion=4
   │                                 7×7
   ▼
[Global Avg Pooling]                [B, 768]
   ▼
[LayerNorm + Linear Classifier]     [B, 1000]
```

### 5.1 ConvNeXt Block 内部

每个 ConvNeXt Block 的 forward 路径：

```text
输入: [B, C, H, W]
   │
   ├── Residual Path ──────────────────┐
   │                                    │
   ▼                                    │
[7×7 Depthwise Conv, groups=C]          │
   │ (N, C, H, W)                       │
   ▼                                    │
[Permute: N, C, H, W → N, H, W, C]     │
   │                                    │
   ▼                                    │
[LayerNorm (channels_last)]             │
   │                                    │
   ▼                                    │
[Linear: C → 4C (pointwise conv)]       │
   │                                    │
   ▼                                    │
[GELU activation]                       │
   │                                    │
   ▼                                    │
[Linear: 4C → C (pointwise conv)]       │
   │                                    │
   ▼                                    │
[Layer Scale: × γ (init=1e-6)]          │
   │                                    │
   ▼                                    │
[Permute: N, H, W, C → N, C, H, W]     │
   │                                    │
   ▼                                    │
[DropPath (stochastic depth)]           │
   │                                    │
   └────────→ (+) ←─────────────────────┘
                │
                ▼
          输出: [B, C, H, W]
```

### 5.2 几个设计细节

**Permute 操作的目的：** PyTorch 的 `nn.Linear` 要求输入为 (N, *, C) 格式。Depthwise conv 使用 channels_first（N, C, H, W）格式，所以需要在 DWConv 和 LayerNorm 之间 permute。LayerNorm 可以选择 channels_first 实现，但论文指出 channels_last + nn.Linear 的组合在 PyTorch 中略微更快。

**DropPath vs Dropout：** DropPath 以概率 p 跳过整个 block（输出 = 输入），不同于 Dropout 随机丢弃单个神经元。在深层 Transformer 和 ConvNeXt 中，DropPath 被证明比 Dropout 更有效。

**关于 bias 的说明：** 代码中默认使用 `nn.Conv2d(..., bias=True)`，但 LayerNorm 后续会做归一化，bias 项的效果会被吸收。论文本身没有刻意讨论 bias 的去除，这个设计中 conv 带 bias 是最直接和 PyTorch 默认一致的做法。

## 六、核心模块与公式

### 6.1 ConvNeXt Block 的模块化设计

整个 block 可以拆解为三个操作的组合：

- **Depthwise Conv (7×7)** — 空间混合：每个 channel 在自己的空间邻域内做加权平均
- **Pointwise Conv (1×1, expansion=4)** — 通道混合：在 expanded channel 维度做非线性变换
- **Layer Scale + DropPath** — 训练稳定性：控制残差贡献和随机深度

### 6.2 复杂度分析

对于单个 ConvNeXt Block，输入为 [B, C, H, W]：

| 操作 | FLOPs |
|------|-------|
| 7×7 Depthwise Conv | $49 \times HWC$ |
| Linear: C→4C | $4HC^2$ |
| Linear: 4C→C | $4HC^2$ |
| 合计 | $HWC \times (49 + 8C/HW)$ |

和标准 ResNet bottleneck 对比：

| 操作 | ResNet Bottleneck | ConvNeXt Block |
|------|------------------|---------------|
| 1×1 conv | $2HC^2$ | — |
| 3×3 conv | $9HC^2$ | — |
| 7×7 DWConv | — | $49HC$ |
| Linear C→4C | — | $4HC^2$ |
| Linear 4C→C | — | $4HC^2$ |
| 激活数 | 3 (ReLU) | 1 (GELU) |
| Norm 数 | 2 (BN) | 1 (LN) |
| 合计 (主导项) | $11HC^2$ | $8HC^2$ |

ConvNeXt Block 在 FLOPs 上低于 ResNet bottleneck，因为 depthwise conv 的 $49HC$ 远小于 $9HC^2$ （当 C 较大时，如 C=384, 49HC \ll 9×384×HC）。

整个网络的总 FLOPs（ConvNeXt-T, 224²）：

| 组件 | FLOPs（估算） |
|------|-------------|
| Stem (4×4 conv) | ~0.05G |
| Stage 1 (3 blocks, C=96, 56²) | ~0.78G |
| Downsample 1 (2×2 conv) | ~0.02G |
| Stage 2 (3 blocks, C=192, 28²) | ~1.01G |
| Downsample 2 | ~0.04G |
| Stage 3 (9 blocks, C=384, 14²) | ~2.02G |
| Downsample 3 | ~0.06G |
| Stage 4 (3 blocks, C=768, 7²) | ~0.52G |

（注：以上 FLOPs 分解为基于架构参数的近似估算，非论文原始数据。）
| **总计** | **~4.5G** |

### 8.2 核心实现（convnext.py）

```python
class Block(nn.Module):
    def __init__(self, dim, drop_path=0., layer_scale_init_value=1e-6):
        super().__init__()
        self.dwconv = nn.Conv2d(dim, dim, kernel_size=7, padding=3, groups=dim)
        self.norm = LayerNorm(dim, eps=1e-6)
        self.pwconv1 = nn.Linear(dim, 4 * dim)
        self.act = nn.GELU()
        self.pwconv2 = nn.Linear(4 * dim, dim)
        self.gamma = nn.Parameter(
            layer_scale_init_value * torch.ones((dim)),
            requires_grad=True) if layer_scale_init_value > 0 else None
        self.drop_path = DropPath(drop_path) if drop_path > 0. else nn.Identity()

    def forward(self, x):
        input = x
        x = self.dwconv(x)
        x = x.permute(0, 2, 3, 1)  # N,C,H,W → N,H,W,C
        x = self.norm(x)
        x = self.pwconv1(x)
        x = self.act(x)
        x = self.pwconv2(x)
        if self.gamma is not None:
            x = self.gamma * x
        x = x.permute(0, 3, 1, 2)  # N,H,W,C → N,C,H,W
        x = input + self.drop_path(x)
        return x
```

**代码细节：**

- `nn.Conv2d(dim, dim, kernel_size=7, padding=3, groups=dim)` — 这是 depthwise convolution 的实现方式。`groups=dim` 使得每个 input channel 独立和 7×7 kernel 做卷积。

- `LayerNorm` — 自定义实现，支持 channels_first 和 channels_last 两种格式。在 block 中使用 channels_last（N,H,W,C 格式）以配合 nn.Linear。

- `self.gamma` — Layer Scale。初始化为 `layer_scale_init_value * ones(dim)`（默认 1e-6），确保初始残差贡献极小。

- `DropPath` — 来自 timm 库。随机跳过整个 block 的残差路径。

### 8.3 ConvNeXt 顶层结构

```python
class ConvNeXt(nn.Module):
    def __init__(self, in_chans=3, num_classes=1000,
                 depths=[3, 3, 9, 3], dims=[96, 192, 384, 768],
                 drop_path_rate=0., layer_scale_init_value=1e-6):
        super().__init__()

        # Stem: 4×4 stride=4 conv + LayerNorm
        self.downsample_layers = nn.ModuleList()
        stem = nn.Sequential(
            nn.Conv2d(in_chans, dims[0], kernel_size=4, stride=4),
            LayerNorm(dims[0], eps=1e-6, data_format="channels_first"))
        self.downsample_layers.append(stem)

        # Intermediate downsampling: LN + 2×2 stride=2 conv
        for i in range(3):
            downsample_layer = nn.Sequential(
                LayerNorm(dims[i], eps=1e-6, data_format="channels_first"),
                nn.Conv2d(dims[i], dims[i+1], kernel_size=2, stride=2))
            self.downsample_layers.append(downsample_layer)

        # 4 stages, each with Blocks
        self.stages = nn.ModuleList()
        dp_rates = [x.item() for x in torch.linspace(0, drop_path_rate, sum(depths))]
        cur = 0
        for i in range(4):
            stage = nn.Sequential(*[
                Block(dim=dims[i], drop_path=dp_rates[cur + j],
                      layer_scale_init_value=layer_scale_init_value)
                for j in range(depths[i])])
            self.stages.append(stage)
            cur += depths[i]

        # Final norm + classifier
        self.norm = nn.LayerNorm(dims[-1], eps=1e-6)
        self.head = nn.Linear(dims[-1], num_classes)
```

### 8.4 架构变体

| 变体 | depths | dims | 参数 |
|------|--------|------|------|
| ConvNeXt-T | (3, 3, 9, 3) | (96, 192, 384, 768) | 29M |
| ConvNeXt-S | (3, 3, 27, 3) | (96, 192, 384, 768) | 50M |
| ConvNeXt-B | (3, 3, 27, 3) | (128, 256, 512, 1024) | 89M |
| ConvNeXt-L | (3, 3, 27, 3) | (192, 384, 768, 1536) | 198M |
| ConvNeXt-XL | (3, 3, 27, 3) | (256, 512, 1024, 2048) | 350M |

ConvNeXt-S/B/L/XL 都把 stage 3 的 block 数设为 27（相比 ConvNeXt-T 的 9），这是第七步"stage ratio 调整"的结果。Swin-B 的 stage 3 只有 18 个 block，ConvNeXt 在这个关键 stage 上用了更多计算资源。

### 8.5 论文 vs 代码对照

1. **LayerNorm 的实现**：论文中的 LayerNorm 有两种模式——channels_first 在 downsampling 层使用（stem 和中间下采样），channels_last 在 Block 内部使用（配合 nn.Linear）。代码中的 `LayerNorm` 类通过 `data_format` 参数切换两种模式。

2. **权重初始化**：使用 `trunc_normal_(std=0.02)` 初始化 conv 和 linear 层的权重，bias 初始化为 0。分类头的权重和 bias 再乘以 `head_init_scale`（默认 1.0），但在 ImageNet-22K 预训练完后微调时设为 0.1。

3. **DropPath rate 分配**：`dp_rates` 从 0 线性增长到 `drop_path_rate`，按 block 总数平均分配。ConvNeXt-T 的 drop_path_rate=0.0（在 1K 训练中），但 ConvNeXt-B/L/XL 在 1K 训练中使用 drop_path_rate=0.2~0.5。

4. **Conv bias**: 代码使用默认 `nn.Conv2d(..., bias=True)`。LayerNorm 后续做归一化，bias 项会被归一化抵消。

## 九、技术谱系

### 9.1 技术树中的位置

```text
ResNet (He et al., 2016) — CNN backbone 参照系
  │
  ├── ResNeXt (Xie et al., 2017) — grouped conv
  ├── MobileNetV2 (Sandler et al., 2018) — inverted bottleneck, depthwise conv
  │
  ├── Swin Transformer (Liu et al., 2021) — 层级 Transformer backbone
  │     │
  │     └── ConvNeXt (Liu et al., 2022) ← ★ 本篇 ★
  │           │
  │           ├── ConvNeXt V2 (Woo et al., 2023) — FCMAE self-supervised + GRN
  │           └── InternImage (Wang et al., 2022) — deformable conv 代替 attention
  │
  └── DeiT (Touvron et al., 2021) — 训练配方
  └── CaiT (Touvron et al., 2021) — Layer Scale
```

ConvNeXt 的特殊之处：它是这一大堆技术树中**唯一从 CNN 出发，向 Transformer 方向改良的工作**。其他工作（Swin、CSWin、Focal）都是"Transformer 加入 CNN 的归纳偏置"，而 ConvNeXt 是"CNN 引入 Transformer 的设计理念"。

### 9.2 前驱工作

- **ResNet (2016)：** 残差连接 + 4 级金字塔结构 — ConvNeXt 保留了完整的残差框架和 stage 设计
- **ResNeXt (2017)：** Grouped convolution — ConvNeXt 进一步推到 depthwise conv 的极端情况
- **MobileNetV2 (2018)：** Inverted bottleneck + depthwise separable conv — ConvNeXt 的核心模块直接继承自 MobileNetV2
- **Swin Transformer (2021)：** 7×7 局部窗口、stage ratio 1:1:3:1、训练配方 — ConvNeXt 的设计目标就是"用纯卷积复现 Swin 的设计"
- **DeiT (2021)：** 数据增强、训练技巧 — ConvNeXt 的训练配方基本来自 DeiT
- **CaiT (2021)：** Layer Scale — ConvNeXt 的初始化技巧直接借用

### 9.3 后续影响

- **ConvNeXt V2 (Woo et al., 2023)：** 完全继承 ConvNeXt 架构，加入 FCMAE 自监督预训练和 Global Response Normalization（GRN）解决 block collapse 问题。ConvNeXt V2-L 在 ImageNet-1K 上达到 86.5%（1K 训练，无需 22K 数据）。

- **InternImage (Wang et al., 2023)：** 用 Deformable Convolution（DCNv3）替代 ConvNeXt 中的 depthwise conv，实现了大规模可变形卷积 backbone。InternImage-H 在 ImageNet 上达到 90.1%。这也间接证明了 ConvNeXt 的架构设计（inverted bottleneck + LN + Layer Scale）是通用的——替换空间混合算子后依然有效。

- **EfficientViT / FastViT 等轻量级架构：** 借鉴 ConvNeXt 的"通道混合+空间混合分离"设计，混合卷积和 attention 来构建高效 backbone。

- **2023-2024 年的 ViT 改进工作：** 很多 ViT 的工作（如 EVA-02、DeiT III）开始引入 ConvNeXt 式的 stem（4×4 patchify）、Layer Scale 等设计，实际上是"CV 骨干架构的趋同"。

## 十一、范式意义

**ConvNeXt 的范式意义：它证明了 CNN 没有死，Transformer 的成功部分来自训练配方和架构细节而非 attention 本身的魔法。**

具体来说，ConvNeXt 完成了三件事：

### 11.1 拆解了 CNN vs Transformer 的"黑箱差距"

在 ConvNeXt 之前，社区默认认为 Transformer 的 self-attention 是性能提升的核心。ConvNeXt 通过系统消融证明：如果控制训练配方、宏观设计、宽度配置、kernel 大小、微观模块设计，**纯 CNN 可以做得一样好**。

作者在论文中强调："We find that a pure ConvNet can achieve a similar level of scalability as hierarchical vision Transformers." 这是一个强有力的结论——它用实验而不是辩论回答了"CNN vs Transformer"这个当时最热门的问题。

### 11.2 设计了一个极简的"现代 CNN 模板"

ConvNeXt 的核心贡献不只是一个模型系列，更是一个设计原理的集合：

- 7×7 depthwise conv 作为空间混合算子
- Inverted bottleneck 结构
- LayerNorm（channels_first/ channels_last）
- GELU + 单个激活层
- Layer Scale 初始化
- 独立的 downsampling 层

这个"配方"后来被广泛应用于各种视觉架构——不仅是 CNN（InternImage），也包括混合架构（EfficientViT）和纯 Transformer（DeiT III 中的某些设计）。ConvNeXt 证明了这些设计选择是**架构无关的**。

### 11.3 推动了视觉骨干架构的"趋同"

ConvNeXt 的出现让社区意识到，CNN 和 Transformer 之间的差距不在于 attention vs convolution 的"本质差异"，而更多在于工程细节。这促进了后来的"设计融合"趋势：

- Transformer 越来越多地使用卷积式设计（patchify stem、局部 attention、层级结构）
- CNN 越来越多地使用 Transformer 式设计（LayerNorm、GELU、Layer Scale、inverted bottleneck）

两者在 2023 年以后的架构设计中已经越来越难以区分——ConvNeXt block 和 Swin Transformer block 的核心结构几乎一样（DWConv ≈ Window Attention 做空间混合，Linear→GELU→Linear ≈ MLP 做通道混合）。

### 11.4 分类：bridge node

在技术树中，ConvNeXt 是一个 **bridge node**——它连接了 CNN 和 Transformer 两个设计范式，证明了它们的核心差异不在"有没有 attention"，而在 architecture engineering 的细节。它不是 foundation node（没有开创全新范式），也不是 refinement node（不是对已有范式的微调），而是跨越两个领域做"翻译"的桥梁。

## 十二、局限性与后续方向

### 12.1 仍依赖 ImageNet-22K 预训练达到最高性能

ConvNeXt-XL 的 87.8% 是在 ImageNet-22K 上预训练后微调 384² 的结果。在 224² 下，ConvNeXt-XL（22K 预训练）达到 87.0%。ConvNeXt 对大规模预训练数据的依赖和 ViT 类似。

### 12.2 LayerNorm 在 channel 维上的实现开销

ConvNeXt 的 LayerNorm（channels_first 模式）需要通过 `x.mean(1)` 在 channel 维手动计算 mean/std。这和 PyTorch 原生的 `nn.LayerNorm`（只支持 channels_last）不同。在推理时，permute 操作的额外开销在大 batch 和大分辨率下可以测量到（约 5-10% 的延迟）。

论文中的 Table 6 给出了延迟数据：
- 在 V100 上，ConvNeXt-T 的 throughput 为 774.7 img/s，Swin-T 为 757.9 img/s
- ConvNeXt 和 Swin-T 的推理速度基本持平（V100 上约 2% 差异）

### 12.3 Depthwise conv 的硬件效率问题

Depthwise conv 的 FLOPs 效率不如密集卷积。在 GPU 上，depthwise conv 的 arithmetic intensity 低（计算/内存比小），导致 GPU 利用率不如密集 conv 或矩阵乘法。这在移动端或嵌入式设备上影响更明显。

论文在 Table 1 的 caption 中提到了这个问题：虽然 FLOPs 相近，但实际 latency 可能因为不同算子的硬件实现而有所不同。MobileNetV2 当初的主要动机就是优化这个。

### 12.4 没有太大"原创"模块

ConvNeXt 的所有组件都是已有的（DWConv、inverted bottleneck、LN、GELU、Layer Scale、DropPath）。论文的价值不在于发明新模块，而在于**系统性地证明这些 Transformer 风格的设计选择在纯卷积架构中同样有效**。它的贡献是实验性的，而非方法性的。

这不是缺陷，而是一个有意的选择——作者明确说他们的目标是"reexamine the design spaces"而非"propose novel modules"。

### 12.5 对下游任务的具体优势不明确

论文在 COCO 和 ADE20K 上的结果和 Swin 在同一水平线（有时略高，有时持平），但差异不大。对于想要选择 backbone 的 practitioner 来说，ConvNeXt 的最大优势是"简单"而非"更好"——它不需要 shifted window 的 mask 逻辑，没有 cyclic shift 操作，不需要调整窗口大小来适配不同分辨率。但对于已经在使用 Swin 的系统，迁移到 ConvNeXt 的收益可能不足以覆盖迁移成本。

### 12.6 自监督预训练下的表现

ConvNeXt 在自监督场景下的表现最初未知。后来 ConvNeXt V2（2023）专门解决了这个问题：发现 ConvNeXt 在 MAE-style 的 masked image modeling 下遇到"表示坍缩"问题（block 输出退化为简单的高频信号），通过引入 Global Response Normalization 解决。这也是 ConvNeXt V2 的核心贡献。

## 十三、小结

ConvNeXt 是后 Transformer 时代最有意思的一篇论文之一。在 2022 年初——Transformer 全面接管 CV 的顶峰期——它做了一件"逆流而上"的事：

**不发明任何新东西，只是把已有的设计用系统消融的方式重新组合一次。**

它说的不是"CNN 更好"或"Transformer 不好"，而是：
- "如果你给 ResNet 相同的训练配方，它不差"
- "如果你把 Swin 的 macro/micro 设计全部翻译成卷积语言，纯 CNN 也可以做到一样好"

这个信息在当时的语境下是反直觉的——因为 2021 年的主流叙事是 attention 的"magic"。但 ConvNeXt 用实验证明，这个 magic 很大一部分来自工程细节，而不是 attention 本身。

从技术树角度看，ConvNeXt 同时扮演了两个角色：
1. **对过去的解释** — 拆解了 CNN 和 Transformer 之间的差距，让社区理解了"差距在哪"
2. **对未来的模板** — 提供了一个"现代化 CNN"的简洁配方，被后来的大量工作借鉴

在架构深度上，ConvNeXt 不如 Swin 复杂（没有 shifted window、attention mask、relative position bias、cyclie shift），但在性能和简单性之间达到了更好的平衡。这种"少即是多"的设计哲学，使 ConvNeXt 成为 2022 年被引用最多的视觉架构论文之一。

最终，ConvNeXt 回答了一个在 2022 年初很重要的问题：

> **CNN 在 Transformer 时代还有未来吗？**

答案是：有的。而且不需要 attention。
