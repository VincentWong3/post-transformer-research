## 一、先搞清楚"问题在哪里"

2018 年 BERT 发布之后，NLP 界发生了翻天覆地的变化。BERT 通过 masked language modeling（MLM）和 next sentence prediction（NSP）在大规模无标注文本上预训练，然后微调到下游任务，刷新了 11 项 NLP 基准的成绩。这个"预训练→微调"的范式在 NLP 和 CV 领域各自都取得了巨大成功——ImageNet 预训练模型+微调已经是 CV 的标准做法。

但到了视觉-语言（Vision-and-Language）任务上，情况变得很尴尬。

**每条 tech 路线各走各的，没有统一基座。** VQA 模型用一种架构，指代表达式（referring expression）用另一种架构，图文检索又是一种架构。虽然这些任务有一个共同的需求——**visual grounding（视觉接地）**，即把自然语言和图像内容对齐——但没有一个统一的模型能同时支持它们。

**视觉接地只在任务训练中"顺带"学习。** 典型的做法是：拿一个 ImageNet 预训练的 CNN 提取图像特征，拿一个 BERT 做文本编码，然后在特定任务的数据上从头学习视觉和语言之间的关联。这种做法的结果是学到的是任务特定的、片面的接地能力——当训练数据有限或者有偏时，泛化能力很差。

所以 2019 年初的情况是：**视觉-语言任务的解决范式是"先分别预训练好视觉和语言模型，然后在任务中硬学接地"——这既低效又缺乏泛化性。** 有没有一种方式，先在大量图文配对数据上预训练好一个模型，让它学会通用的视觉接地能力，然后只做少量修改就能应用到各种下游任务？

ViLBERT 就是在这个问题上给出的答案。

## 二、旧路线的真正问题

### 2.1 "任务特定模型"的架构碎片化

在 ViLBERT 之前，视觉-语言任务的主流做法是**为每个任务单独设计模型结构**：

- **VQA（视觉问答）：** 主流模型是 Bottom-Up Top-Down Attention（Anderson et al., 2018）——用 Faster R-CNN 提取图像区域特征，用 LSTM/GloVe 处理文本，再通过 attention 融合。每个问题类型可能需要不同的融合策略。
- **指代表达式（Referring Expressions）：** MAttNet（Yu et al., 2018）用三个独立的模块（subject、location、relationship）分别处理，然后组合分数。
- **图文检索（Image-Text Retrieval）：** SCAN（Lee et al., 2018）用 stacked cross-attention，需要单独训练对齐模型。
- **视觉常识推理（VCR）：** R2C（Zellers et al., 2019）用更复杂的推理链路。

**这些模型之间的架构差异如此之大，以至于一个 VQA 的技术进步几乎无法迁移到指代表达式任务上。** 每个任务都从零开始设计融合机制、attention 结构和目标函数。

### 2.2 "分步预训练"的割裂问题

当时的典型 pipeline 是：

1. 拿 ImageNet 预训练的 CNN（或 Faster R-CNN）提取图像特征——这是**视觉特征预训练**，但目标是最佳分类/检测性能，不是图文对齐
2. 拿 BERT/GloVe 处理文本——这是**语言特征预训练**，但目标是语言建模，不涉及图像
3. 在目标任务的少量标注数据上从头学习图文关联——**这个步骤才真正做视觉接地**

这里的根本问题是：**视觉接地能力——即"什么文本描述对应什么图像内容"——在预训练阶段完全被忽略了。** 两个预训练模型各自在自己的模态里学得很好，但在"跨模态对齐"这个关键能力上，它们从零开始。

这产生了两个后果：

- **小样本接地困难。** 当任务数据量小时（比如 VCR 只有 290K 个 QA），模型很难学到鲁棒的图文对齐——它只能看到数据中的表面关联，容易过拟合到数据集偏差（如 VQA 的语言偏见问题）。
- **接地能力无法迁移。** 即使 VQA 模型学会了对齐"狗"和狗的图像区域，这个知识也没法直接用到指代表达式任务上——因为两个模型的架构不同、表示空间不同。

### 2.3 为什么"简单扩展 BERT"不够

一个最朴素的想法是把 BERT 直接扩展到图像：把图像区域特征离散化为 token（通过聚类），然后当作额外的文本 token 喂进 BERT。这就是 "单流（single-stream）" 架构的思路。

但论文明确指出了三条根本性的缺陷：

1. **离散化误差。** 视觉特征是通过聚类离散化的，这个过程会丢失重要的视觉细节。图像中细微的纹理、形状、颜色差异可能对后续的语义理解很关键，但聚类后这些信息被压缩了。
2. **对两个模态的处理需求不同。** 文本和图像的处理深度应该不一样。图像区域特征本身已经是深度网络（Faster R-CNN）的输出，属于高层语义特征，不需要太多的 context aggregation。但单词在句子中的含义高度依赖上下文——"bank" 是"河岸"还是"银行"需要看前后的词。强行让两者通过同一套 Transformer 层做相同深度的处理不合理。
3. **预训练权重被破坏。** BERT 的预训练权重是针对语言 token 学习的。强行插入大量视觉 token（每个图像区域对应一个"视觉词"）会破坏 BERT 学到的语言表示。

这就是为什么需要一个**双流（two-stream）** 架构——视觉和语言各自走自己的处理流，在特定层通过 co-attention 交互。

## 三、这篇论文的核心思路

**这篇论文的核心思路是：扩展 BERT 到多模态领域，但用双流架构（vision stream + language stream）分别处理图像和文本，通过 co-attentional transformer layer 实现跨模态信息交换；然后在大型图文配对数据集（Conceptual Captions, 约 310 万对）上做预训练，让模型学会通用的视觉接地能力。**

具体来说，核心创新点有三个：

```
BERT 语言模型（单模态）    +      Faster R-CNN 视觉特征（单模态）
         ↓                           ↓
         └─────→ 双流架构 ←──────┘
                      ↓
           co-attention 跨模态交互
                      ↓
         预训练任务：MLM + ITM + Masked ROI
                      ↓
         微调到 4 个下游任务 → 全部 SOTA
```

这条技术路线在技术树中的位置是：

```text
BERT（单模态语言模型, 2018）
   ↓
VideoBERT（单流视频+文本, 2019, 同期工作）
   ↓
ViLBERT（双流图文 co-attention, 2019）← 本篇
   ↓
LXMERT（双流图文, 2019）
UNITER（单流图文, 2019）
```

和 VideoBERT（Sun et al., 2019）的最大区别是：VideoBERT 用单流架构，把视频帧聚类为"视觉词"然后和文本 token 拼在一起喂进 BERT。ViLBERT 则坚持双流架构，视觉和语言先各自单独处理，然后通过 co-attention 交互。论文实验证明双流架构在所有下游任务上一致优于单流。

## 四、整体架构与 forward 流程

### 4.1 整体架构

ViLBERT 由两条并行的 BERT-style 流组成：

```text
文本输入:  "Man shopping for fruit"
    │
    ▼
Language BERT Stream (BERTBASE 初始化)
    │  Token Embedding (WordPiece)
    │  + Position Encoding
    │  + Segment Encoding
    │  ┌─────────────────────────────────┐
    │  │ TRM  (transformer block) × 6    │  ← 前 6 层纯文本处理
    │  │ TRM  (transformer block) × 6    │  ← 后 6 层带 co-attention
    │  │ Co-TRM (co-attention block)     │
    │  └─────────────────────────────────┘
    │
    ▼
文本表示: h_w0, h_w1, ..., h_wT

图像输入: 区域特征 v1, v2, ..., vK (Faster R-CNN)
    │
    ▼
Visual Stream
    │  Region Feature (2048-d)
    │  + Spatial Encoding (5-d → 投影)
    │  ┌─────────────────────────────────┐
    │  │ Co-TRM (co-attention block) × 6 │
    │  │ TRM (transformer block) × 6     │
    │  └─────────────────────────────────┘
    │
    ▼
图像表示: h_v0, h_v1, ..., h_vK
```

关键设计决策：
- **文本流更深。** 文本流的前 6 层是纯 Transformer block（TRM），之后再通过 6 个 Co-TRM → TRM 的配对。图像流则反过来：先经过 Co-TRM 与文本交互，再做 TRM 处理。这反映了论文的设计直觉：**图像特征已经是深度网络（Faster R-CNN）的高层输出，不需要太多额外的 context aggregation；而文本词需要更多层的处理来建立上下文依赖。**
- **视觉流隐藏层大小为 1024，8 个 attention head。** 文本流使用 BERTBASE 的配置：768 隐藏层大小，12 个 attention head。两者维度不同（1024 vs 768），正好对应各自的复杂度需求——图像区域特征维度更高（2048-d Faster R-CNN 特征），需要更大的隐藏层来处理。

### 4.2 前向过程逐步跟踪

**Step 1 — 图像区域特征提取**

使用 Faster R-CNN（ResNet-101 backbone，Visual Genome 预训练）提取图像区域。选取类检测置信度超过阈值的区域，每张图像保留 10 到 36 个高得分框。

对每个选中的区域 i，视觉特征 vi 定义为该区域的 mean-pooled convolutional feature（2048 维）。同时构造一个 5 维的空间编码向量：归一化的左上角和右下角坐标（4 维）+ 区域面积占图像比例（1 维）。空间编码通过线性层投影到和视觉特征相同的维度后相加。

特殊 token：用一个特殊的 IMG token 表示整个图像——它使用 mean-pooled 的全局视觉特征，空间编码对应整个图像。

**Step 2 — 文本 token 化**

使用 BERT 的 WordPiece tokenizer。输入的文本序列遵循 BERT 的标准格式：
```
[CLS] w1 w2 ... wT [SEP]
```
每个 token 的输入表示 = token embedding + position encoding + segment encoding（三者和）。

**Step 3 — 前向传播**

文本流和视觉流并行处理。前 6 个 transformer block 中，两条流各自独立运行，没有交叉。从第 7 层开始，每条流在每个 block 内部通过 co-attention 交互。

**Step 4 — 全局表示**

输出的两个关键表示：
- hIMG（视觉流中对应 IMG token 的输出）—— 图像的全局表示
- hCLS（文本流中对应 CLS token 的输出）—— 文本的全局表示

用于对齐判断时，计算 hIMG 和 hCLS 的 element-wise product 作为联合表示。

## 五、核心模块逐个解释

### 5.1 Co-Attentional Transformer Layer

这是 ViLBERT 的核心创新。

**标准 Transformer block 的 attention：**

```
Q = H * W_Q
K = H * W_K
V = H * W_V
Attention(Q, K, V) = SoftMax(QK^T / √d) V
```
其中 Q、K、V 都来自同一个输入的 H。

**Co-attention block 的 attention：**

视觉流（下标 V）和文本流（下标 L）之间交换 key 和 value：

```
# 视觉流的 attention（用文本的 K、V 来 attender）
Q_V = H_V * W_Q                  # query 来自视觉
K_L = H_L * W_K                  # key 来自文本
V_L = H_L * W_V                  # value 来自文本
Visual→Text: Attention(Q_V, K_L, V_L)

# 文本流的 attention（用视觉的 K、V 来 attender）
Q_L = H_L * W_Q                  # query 来自文本
K_V = H_V * W_K                  # key 来自视觉
V_V = H_V * W_V                  # value 来自视觉
Text→Visual: Attention(Q_L, K_V, V_V)
```

这个设计传达的直觉非常清晰：
- **视觉流中的 co-attention：** 每个图像区域"看到"文本中哪些词和它相关，然后从文本中聚合信息来丰富自己的表示。例如，图像中的一块棕色区域可以从文本中的"dog"获得语义标签。
- **文本流中的 co-attention：** 每个文本词"看到"图像中哪些区域和它相关，然后从图像中聚合视觉信息。例如，单词"dog"可以从图像中的棕色区域获得视觉特征。

**为什么比全局融合好？** 传统的视觉-语言模型通常只做单向 attention（文本 attend 到图像），而 ViLBERT 做双向 co-attention。这样做的效果是：视觉表示和文本表示相互增强，在各自的语境中变得更"多模态"。

### 5.2 图像表示：从检测到语义

ViLBERT 的图像表示不同于后来 CLIP/ViT 的 patch-based 做法。它使用了 **Faster R-CNN + Visual Genome 预训练** 的检测模型来提取区域特征。

这个选择在当时是直观的：检测模型能提供"语义上有意义"的区域——它们对应具体的物体或物体部分。相比之下，把图像均匀切 patch 的方法（后来的 ViT 做法）在 2019 年还比较少见。

但这也带来了一个明显的短板：**区域的数量和位置是输入相关的，不同的图像有不同数量的区域。** ViLBERT 通过保持 10-36 个区域来解决这个问题，用稀疏的序列来近似密集的图像内容。

### 5.3 预训练任务

ViLBERT 设计了两个预训练任务，对应 BERT 中的 MLM 和 NSP。

**任务一：Masked Multi-Modal Modeling（掩码多模态建模）**

对应 BERT 的 masked language modeling，但扩展到跨模态。

约 15% 的文本 token 和 15% 的图像区域被随机选中作为"掩码"输入：
- **掩码文本 token：** 和 BERT 一样——80% 替换为 [MASK]，10% 替换为随机词，10% 保持不变
- **掩码图像区域：** 90% 将视觉特征置零，10% 保持不变

模型需要根据完整输入（其他未被掩码的 token/区域）来预测被掩码的内容。

**关键设计选择：** 对于掩码文本 token，预测目标是一个词汇上的分布（交叉熵损失，和标准 BERT 一样）。但对于掩码**图像区域**，模型不是直接回归特征值，而是**预测该区域的语义类别分布**。具体来说，使用同一个 Faster R-CNN 检测模型对该区域输出的类别分布作为监督信号，用 KL 散度来训练。

为什么这样设计？论文的解释很合理：**语言通常只标识视觉内容的"高层语义"** ——一句话说"一只狗坐在沙发上"，但没有描述狗的毛色纹理或沙发的面料。直接用 L2 损失来重建精确的特征向量既不可能（信息量不对称），也不合理（会使两个模态的损失难以平衡）。

**任务二：Multi-Modal Alignment Prediction（多模态对齐预测）**

对应 BERT 的 next sentence prediction，但用于判断图文是否匹配。

输入格式：`[IMG] v1 ... vK [CLS] w1 ... wT [SEP]`

模型使用 hIMG（视觉流中 IMG token 的最终表示）和 hCLS（文本流中 CLS token 的最终表示），计算它们的 element-wise product，然后通过一个线性层做二分类：图像和文本是否对齐。

由于 Conceptual Captions 数据集只包含对齐的图文对（正样本），负样本的构造方式是：随机替换图像或文本。即——从数据集中随机选另一个图像替代当前的图像，或随机选另一个文本替代当前的文本。

### 5.4 Conceptual Captions 数据集

ViLBERT 在 Conceptual Captions（Sharma et al., 2018）上预训练。这个数据集有约 330 万对图文对，自动从网页上带 alt-text 的图像中爬取并清洗。

论文提到，由于部分链接在下载时已经失效，实际训练时用了约 310 万对。

CC 的数据质量和人工标注的 COCO Captions 不同——部分"描述"是编辑化的（如"actors attend the premiere at festival"），有时不太像自然的人类描述。但它的规模和多样性（覆盖广泛的主题）是人工标注数据集无法比拟的。

## 六、实验结果

### 6.1 训练细节

- **8 块 TitanX GPU，总 batch size 512，训练 10 个 epoch**
- **优化器：** Adam，初始学习率 1e-4，linear decay with warmup
- **文本流初始化：** BERTBASE（12 层 Transformer，隐藏层 768，12 attention head），BookCorpus + English Wikipedia 预训练
- **视觉流：** Faster R-CNN（ResNet-101 backbone），Visual Genome 预训练。隐藏层 1024，8 attention head
- **两个预训练任务损失等权重**

### 6.2 四个下游任务结果

#### VQA 2.0

| 方法 | test-dev | test-std |
|------|----------|----------|
| DFAF（前 SOTA） | 70.22 | 70.34 |
| Single-Stream†（无预训练） | 65.90 | — |
| Single-Stream（有预训练） | 68.85 | — |
| ViLBERT†（无预训练） | 68.93 | — |
| **ViLBERT（完整）** | **70.55** | **70.92** |

ViLBERT 在 VQA test-std 上达到 70.92%，比前 SOTA（DFAF 的 70.34%）高 0.58 个百分点。

**关键对比：** ViLBERT†（无预训练）68.93% vs ViLBERT（有预训练）70.55%，预训练贡献了 +1.62 个点。

#### VCR（Visual Commonsense Reasoning）

| 方法 | Q→A | QA→R | Q→AR |
|------|------|-------|-------|
| R2C（前 SOTA） | 63.8 | 67.2 | 43.1 |
| Single-Stream† | 68.15 | 68.89 | 47.27 |
| Single-Stream | 71.09 | 73.93 | 52.73 |
| ViLBERT† | 69.26 | 71.01 | 49.48 |
| **ViLBERT** | **72.42** | **74.47** | **54.04** |

ViLBERT 在所有指标上大幅超越前 SOTA R2C。Q→AR（同时选对答案和理由）比 R2C 高出了惊人的 **+10.9 个点**。

#### RefCOCO+（指代表达式）

| 方法 | val | testA | testB |
|------|-----|-------|-------|
| MAttNet（前 SOTA） | 65.33 | 71.62 | 56.02 |
| Single-Stream† | 47.27 | — | — |
| Single-Stream | 65.64 | — | — |
| ViLBERT† | 68.61 | 75.97 | 58.44 |
| **ViLBERT** | **72.34** | **78.52** | **62.61** |

ViLBERT 在 val 上比 MAttNet 高 +7.0 个点。这是一个巨大的差距——指代表达式是一个非常精细的任务，需要精确对齐短语和小图像区域。

#### Caption-Based Image Retrieval（Flickr30k）

| 方法 | R@1 | R@5 | R@10 |
|------|-----|-----|------|
| SCAN（前 SOTA） | 56.02 | 77.70 | 85.20 |
| ViLBERT† | 68.61 | 75.97 | 85.02 |
| **ViLBERT** | **58.20** | **84.90** | **91.52** |

ViLBERT 的 R@1 为 58.20%，比 SCAN 的 48.60% 高 +9.6 个点。这是一个显著的提升。

#### Zero-Shot Caption-Based Image Retrieval

直接使用预训练后的对齐预测机制（不做微调）做 Flickr30k 检索：

| 方法 | R@1 | R@5 | R@10 |
|------|-----|-----|------|
| ViLBERT† | 0.00 | 0.00 | 0.00 |
| **ViLBERT（zero-shot）** | **31.86** | **61.12** | **72.80** |

**ViLBERT†（无预训练）：R@1 为 0.00。** 完全无法完成 zero-shot 检索。
**ViLBERT（有预训练）：R@1 31.86%。** 虽然远低于微调后的 72.34%，但在完全没见过 Flickr30k 的情况下能做到这个程度——说明预训练确实学会了语义上有意义的图文对齐。

### 6.3 消融实验

**网络深度的影响（Table 2）：**

不同任务对深度的需求不同：
- VQA 和 Image Retrieval 随着深度（Co-TRM→TRM 层数）增加到 6 层持续提升
- Zero-shot Image Retrieval 从 8 层继续受益（R@1 从 26.14% 到 32.80%）
- VCR 和 RefCOCO+ 在中等深度（4-6 层）最优

这说明了论文的一个关键设计直觉：**不同任务需要不同量的 context aggregation。** 双流架构让这种灵活性成为可能——可以只改变 co-attention 层的数量来适配任务，而不需要重写整个模型。

**预训练数据量的影响（Table 3）：**

| 数据量 | VQA test-dev | VCR Q→AR | RefCOCO+ val | Image Retrieval R@1 | ZS Retrieval R@1 |
|--------|-------------|----------|--------------|-----------------|-----------------|
| 0% | 68.93 | 49.48 | 68.61 | 45.50 | 0.00 |
| 25% | 69.82 | 52.66 | 69.90 | 53.08 | 20.40 |
| 50% | 70.30 | 53.03 | 71.16 | 54.84 | 26.76 |
| 100% | 70.55 | 54.04 | 72.34 | 58.20 | 31.86 |

所有指标随着数据量增加单调提升，特别是 zero-shot 检索从 0.00→20.40→26.76→31.86，提升斜率没有饱和的迹象。论文因此推测**更多预训练数据会进一步提升 ViLBERT 的性能**。

**双流 vs 单流（分析性结论）：**

论文直接比较了 ViLBERT（双流）和 Single-Stream（单流）两种架构——两个模型用同样的预训练数据和方法训练。在所有任务上，双流架构一致优于单流：
- VQA test-dev：70.55 vs 68.85（+1.70）
- VCR Q→AR：54.04 vs 52.73（+1.31）
- RefCOCO+ val：72.34 vs 65.64（+6.70）

指代表达式任务上的差距特别大（+6.70），可能是因为双流架构允许视觉和文本保持各自独立的表示空间，然后在 co-attention 中进行精确的交叉定位——这正是指代表达式任务所需要的。

ViLBERT 处于"视觉-语言预训练"这个路线的起始节点。它和几个同期工作是 2019 年 V+L 预训练的开创性研究的代表。

### 7.2 同期工作对比

**VideoBERT（Sun et al., 2019, arXiv:1904.01766）：** 第一个将 BERT 扩展到视频-语言领域的工作。但 VideoBERT 使用**单流架构**——把视频帧通过聚类离散化为"视觉词"，然后和文本 token 拼接在一起送入 BERT。论文认为这种方法有上述三个缺陷（离散化误差、不等处理深度、破坏预训练权重）。ViLBERT 的直接比较实验证实了双流架构的优越性。

**LXMERT（Tan & Bansal, 2019, arXiv:1908.01977）：** 几乎同期（2019 年 8 月）提出的一个非常相似的双流架构。LXMERT 和 ViLBERT 的核心区别在于：LXMERT 在预训练中包含了目标检测对齐任务（object relationship prediction），并且使用了更大的语言模型（BERTLARGE 配置）。LXMERT 在 VQA 上的表现略优于 ViLBERT（72.54% vs 70.92% test-std），但 ViLBERT 在指代表达式等任务上保持了优势。

**UNITER（Chen et al., 2020, arXiv:1909.11740）：** 回归到单流架构，但通过更精细的 masking 策略和更大的预训练数据（COCO + Visual Genome + Conceptual Captions + SBU Captions共 960 万张图像）取得更好的效果。UNITER 没有使用 co-attention，而是将所有模态的 token 送入同一套 Transformer。这说明单流/双流的选择不是绝对的——预训练数据的规模和多样性可能同样重要。

**VisualBERT（Li et al., 2019, arXiv:1908.03557）：** 另一个单流方案，但 visual embeddings 被直接作为 segment embedding 加到 BERT 中的最小修改版本。VisualBERT 更侧重于简洁性——证明了一个几乎不改动 BERT 架构的方案也能在 V+L 任务上取得不错的成绩。

### 7.3 后续影响

ViLBERT 的影响力可以归结为两个层面：

**第一，它开创了"V+L 预训练"这个研究范式。** ViLBERT 发表前，视觉-语言任务的主流范式是"task-specific 模型 + 分别预训练的视觉/语言特征"。ViLBERT 发表后（以及同期几个工作），这个范式转变为"在图文配对数据上预训练通用 V+L 模型，然后微调到下游任务"。

**第二，co-attention 架构影响了后续大量工作。** 虽然后来的研究（UNITER、Oscar、VinVL）倾向于使用更简单的单流架构，但 co-attention 的设计思路——让两个模态在特定层通过 attention 交互——影响了很多其他领域的 cross-modal 建模工作。

## 九、范式意义

**ViLBERT 的范式意义是：它证明了"视觉-语言预训练"是一个可行的、有效的、可迁移的通用范式。**

具体来说，ViLBERT 完成了三件重要的事情：

### 9.1 从"任务特化"到"预训练+微调"的范式转换

在 ViLBERT 之前，每个 V+L 任务（VQA、指代表达式、图文检索、VCR）都有自己的专用模型架构。研究者为每个任务设计不同的 attention 结构、不同的融合策略、不同的目标函数。

ViLBERT 证明了一个新范式：**用同一个预训练模型做基础，每个任务只需要加一个分类层，就能达到甚至超越专门的 task-specific 模型。** 这个"预训练+微调"范式在 NLP 中已经是标准做法（BERT），但在 V+L 领域是 ViLBERT 率先系统性地验证了它的可行性。

### 9.2 双流架构的实用价值

ViLBERT 的双流架构在当时是一个合理的设计选择。它避免了单流架构的三个核心问题，并证明了双流架构在所有下游任务上一致优于等效的单流架构。

但后来的历史发展显示，**单流架构（UNITER、Oscar）在更大的数据规模下也能取得很好的结果**——意味着单流的"信息共享不足"问题可以通过更多的数据来补足。而更后面的发展（CLIP、BLIP、LLaVA）又走向了不同的方向——双编码器或冻结 encoder + 适配器，而不是早期的工作那样把文本和图像信号混在一起 train end-to-end。

从今天的视角来看，ViLBERT 的双流 + co-attention 设计是 V+L 领域在"探索期"的一个重要里程碑，它探索了一条后来证明不是最优但当时最有道理的路径。

### 9.3 数据驱动的方法

ViLBERT 使用 Conceptual Captions（310 万对）作为预训练数据，而不是人工标注的数据集（如 COCO Captions 只有 12 万对）。这是关键的一步——它证明了**自动采集的、有噪声的、大规模的图文配对数据也足以学到有意义的视觉接地能力**。

Table 3 的消融实验非常说明问题：预训练数据量从 0 到 25% 到 50% 到 100%，所有下游任务的性能单调提升，没有出现饱和。这暗示了更大的预训练数据会进一步提升性能——这个预测后来被 CLIP 用 4 亿数据完全验证了。

### 9.4 分类：pioneer node（开创节点）

在技术树中，ViLBERT 是一个 **pioneer node（开创节点）**——作为"早期跨模态 encoder 范式"的第一个节点，它开辟了整个"V+L 预训练"这个研究方向。

它不是 foundation node（基础节点），因为后来 V+L 预训练的路线并没有完全沿用它的双流架构。但它不可否认地影响了后续所有工作——无论是单流还是双流，大家都在验证、改进、扩展它提出的框架。

## 十、局限性与后续方向

### 10.1 对检测模型的依赖

ViLBERT 使用的视觉特征来自 Faster R-CNN（Visual Genome 预训练）。这意味着：

- **推理时也需要跑检测模型**——增加了计算开销。一张图像需要先过 Faster R-CNN 提取区域特征，再进入 ViLBERT。这个双阶段在部署时效率不高。
- **检测模型的类别集是固定的**——Faster R-CNN 在 Visual Genome 上训练，包含 1600 个物体类别和 400 个属性。如果下游任务需要识别检测模型没见过的物体，ViLBERT 的视觉表示就会有盲区。
- **检测质量直接影响下游**——如果检测器漏掉了关键物体或产生了大量噪声框，ViLBERT 的表现会受牵连。

后来的工作（尤其是 CLIP 和 ViT 路线）用 patch-based 方法替代了检测区域，消除了这个依赖。

### 10.2 预训练数据规模不够大

Conceptual Captions 的 310 万对图文数据在 2019 年算大规模，但和后来的 CLIP（4 亿对）、ALIGN（18 亿对）相比只是零头。Table 3 的性能增长曲线没有饱和，说明 ViLBERT 可以从更多数据中受益。

这也是双流 + end-to-end 架构的一个内在限制：**co-attention 的计算复杂度高，难以在大数据上扩展。** 相比之下，CLIP 的双编码器架构（图像 encoder + 文本 encoder，只在对比学习中交互）可以更高效地扩展到海量数据。

### 10.3 无法生成文本

ViLBERT 是一个 encoder-only 模型——它只能理解文本，不能生成文本。这限制了它在一类任务上的应用：图像描述（image captioning）、对话等需要文本生成的任务。

论文在相关工作部分提到了这个局限，并指出"如何从双向模型解码输出文本是一个开放问题，因为 beam search 等传统解码方法不直接适用"。

后来的工作（如 VL-T5、BLIP、OFA）使用 encoder-decoder 架构来解决这个问题。

### 10.4 双流的额外复杂度

双流架构比单流多出一整套 visual Transformer 块，参数量更大、计算量更高。此外，由于两个流在 co-attention 层需要同步（一个流的 K,V 要传给另一个流），并行化不如单流直接。

在 ViLBERT 的实验中，由于计算资源限制，他们只能在 8 张 TitanX GPU 上以 batch size 512 训练 10 个 epoch。如果再增加模型深度或数据量，训练成本会快速增长。

### 10.5 co-attention 的单向性

有趣的是，ViLBERT 的 co-attention 机制虽然在概念上是双向的（图像 attend 到文本，文本 attend 到图像），但实际上**只在特定层做有限次数的交互**——论文的配置是 6 次 co-attention 交互。相比后来在单流架构中每层都做 attention 的做法（UNITER），ViLBERT 的交互频率更低。

这个设计选择在当时是合理的（避免过早融合两个模态），但也意味着**跨模态信息的流动受到限制**——两个模态在大部分层中各自独立运行，只在少数层通过 co-attention 交换信息。

## 十一、小结

直白说，ViLBERT 的价值不只是一个"把 BERT 扩展到图像"的技术成果，而是开创了 **"视觉-语言预训练"这个完整的研究范式**。

在它之前，视觉-语言领域没有"预训练"的概念——每个任务用自己的专用模型，从零学习图文对齐。在它之后，大家开始问：**能不能用一个通用模型在大量图文对数据上预训练视觉接地能力，然后迁移到各种下游任务？**

ViLBERT 用实验给出了明确的肯定答案，并且通过三个核心创新实现了这个目标：

1. **双流架构**：视觉和语言各自走独立的 Transformer 流，避免了单流架构的离散化误差和处理深度不匹配问题。在所有下游任务上一致优于等效的单流架构。
2. **Co-attention 机制**：两个流之间通过交换 key-value 实现双向跨模态交互——视觉流从文本中获取语义标签，文本流从图像中获取视觉上下文。这个设计简洁而有效。
3. **预训练任务设计**：将 BERT 的 MLM 和 NSP 扩展到多模态，设计了 masked multi-modal modeling（掩码文本+掩码区域预测语义类别）和 multi-modal alignment prediction（图文对齐判断）。

实验结果非常清晰地验证了这个方法的有效性：
- **VQA test-std 70.92%**（比前 SOTA 高 0.58 点，预训练本身贡献了 1.62 点）
- **VCR Q→AR 54.04%**（比前 SOTA 高 10.9 点）
- **RefCOCO+ val 72.34%**（比前 SOTA 高 7.0 点）
- **Image Retrieval R@1 72.34%**（比前 SOTA 高 16.3 点）

在技术树中，ViLBERT 是一个 pioneer node——它开辟的方向影响了后续 LXMERT、UNITER、CLIP 等一系列重要工作。虽然最终双流+co-attention 的路线没有被大规模采纳（后来的主流走向了单流或对比学习双编码器），但 ViLBERT 提出的问题和解决方案框架——在图文配对数据上预训练通用视觉接地能力——彻底改变了 V+L 领域的研究范式。
