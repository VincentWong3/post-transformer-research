## 一、先搞清楚"问题在哪里"

2020 年是自监督视觉表征学习的爆发之年。SimCLR（Chen et al., 2020）和 MoCo（He et al., 2019）用对比学习把 ImageNet 线性评估的 top-1 准确率推到了接近 70%（MoCo v2 达到 71.1%），MoCo v2 在检测任务上第一次让自监督学习逼近了有监督基线。

但对比学习的核心机制有一个绕不开的依赖：**负样本**。

对比学习的基本范式是：把同一张图片的不同增强视图拉近（正样本对），把不同图片的增强视图推远（负样本对）。为了让这个"推远"任务足够有挑战性，你需要大量的负样本——SimCLR 靠大 batch（4096 或更大），MoCo 靠 memory bank + momentum encoder 维护一个大的负样本队列。

这带来了三个实际痛点：

1. **大 batch 依赖。** SimCLR 的 batch 掉到 256 时，性能会显著下降。大 batch 意味着更大的显存需求，限制了更多研究者的参与。
2. **数据增强敏感。** SimCLR 发现，只用 random crop 做增强时效果很差——因为裁剪后的图颜色直方图高度相似，模型可以只靠颜色信息就解出对比任务，不需要学到真正的语义特征。所以他们必须加 color distortion 来破坏颜色直方图的相关性。
3. **负样本管理复杂。** 需要调温度参数、需要处理假负样本（False negatives，即不同 view 中实际属于同一类别的两个样本被错误当作负样本）、需要精心设计 mining 策略。

所以 2020 年年中，社区面对的问题是：**负样本到底是不是必须的？** 有没有办法只用正样本对（同一张图片的不同增强视图互相预测），就学到好表征，同时不会坍缩（collapse）——即所有图片都编码成同一个向量？

BYOL 的答案是：**不需要负样本。用 bootstrapping。** 而且性能不仅不差，反而更好。

## 二、旧路线的真正问题

在进入 BYOL 之前，需要先理解为什么"只有正样本"这条路在直觉上似乎是走不通的。

### 2.1 坍缩解（Collapsed Representations）

自监督学习的核心问题可以表述为：在没有标签的情况下，如何让模型学到有用的特征表示？

一个直观的想法是：对同一张图片做两个不同的增强（比如两张随机裁剪），让模型把这两张裁剪图的表征映射到同一个地方。这个目标函数可以写为最小化两个表征之间的距离：

$$\mathcal{L} = \|f(t_1(x)) - f(t_2(x))\|^2$$

问题是这个目标函数有一个平凡的全局极小值：**让 $f$ 输出常数向量**。对所有图片都输出相同的向量，距离当然是 0——这完全满足了目标，但学到的表征完全没有信息量。这就是"坍缩解"。

### 2.2 对比学习如何避免坍缩

对比学习的思路是把"拉近正样本"和"推远负样本"结合起来。只要负样本够多够多样化，模型就不能坍缩——因为如果所有图片都输出相同的向量，它就没有办法把不同图片的表征推远。这就相当于加了一个"隐式的正则项"，迫使模型保留图片间的差异性信息。

这个思路有效，但依赖于负样本的数目和质量。

### 2.3 负样本的隐性成本

除了前面提到的工程问题，负样本还有一个更微妙的问题：**它改变了学习目标的性质。**

对比学习实际上做的是一个判别任务——区分正样本和负样本。在学习过程中，模型被激励去捕捉那些"最能区分不同图片"的特征。这可能不一定是语义上最重要的特征。例如，ImageNet 中"猫"和"狗"的图片，对比学习可能学到的是背景特征的差异（猫照片的背景是沙发，狗的背景是草地），而不是猫和狗本身形状的区别。

BYOL 的作者们提出的问题是：如果我们不做判别，而是做纯粹的**预测**——用一个 view 的表征去预测另一个 view 的表征——会发生什么？预测任务不会鼓励模型区分图片，而是鼓励保留目标表征中的所有信息。这意味着模型可能学到更丰富、更通用的特征。

但问题马上来了：预测任务怎么避免坍缩？

## 三、这篇论文的核心思路

**BYOL 的核心思路是：用两个网络（online 和 target）构建一个自举（bootstrapping）循环——online 网络预测 target 网络的输出，而 target 网络的参数是 online 参数的一个慢速指数移动平均。同时，online 网络上的 predictor 和 target 网络上的 stop-gradient 共同防止了坍缩解的出现。**

这个思路的精妙之处在于三点：

1. **自举不需要负样本。** Online 网络只需要把它的预测做得更接近 target 网络的输出，不需要推开任何其他图片的表征。这彻底改变了学习目标的性质——从判别变成了纯粹的回归/预测。
2. **Target 网络的缓慢变化提供了稳定目标。** 对比学习的目标（来自 memory bank 或 batch 中的负样本）每一步都在剧烈变化，因为所有的表征都在同时更新。BYOL 的 target 网络由于 EMA 更新，变化非常缓慢，为 online 提供了一个稳定、一致的学习目标。
3. **Predictor 的存在打破了对称性。** 如果没有 predictor，online 和 target 网络的结构完全对称，预测任务可以通过双方达成一致（都输出常数）来最小化 loss。但 predictor 的加入使架构变成非对称的——online 分支多了一层可学习的映射，这个映射需要去拟合 target 的输出，从而迫使表征中包含足够的信息来完成这个预测。

在技术树中的位置：

```
对比学习范式 (SimCLR/MoCo) ×
   ↓
BYOL（自举，无负样本, 2020） ← 本篇
   ↓
SimSiam（进一步简化，去掉 target EMA, 2020）
   ↓
DINO（扩展到 ViT, 2021）
   ↓
iBOT（结合 masked image modeling, 2021）
```text

这条技术路线的谱系非常清晰：BYOL 证明了"不需要负样本的自监督学习是可行的"，后续的 SimSiam 进一步去掉了 EMA 参数更新，证明 predictor + stop-gradient 才是核心。DINO 把 BYOL 的思想带到了 Vision Transformer 上，iBOT 进一步结合了 masked image modeling。

## 四、整体结构与 forward 流程

### 4.1 整体架构

BYOL 使用两个神经网络：online 网络（参数 $\theta$ ）和 target 网络（参数 $\xi$ ）。

```
输入图像 x
    │
    ├── 数据增强 t (~𝒯) ──→ v = t(x)
    │       │
    │       ▼
    │   Online 网络 (θ)
    │       │
    │       ├── 编码器 f_θ ──→ 表征 y_θ = f_θ(v)
    │       │
    │       ├── 投影器 g_θ ──→ 投影 z_θ = g_θ(y_θ)
    │       │
    │       └── 预测器 q_θ ──→ 预测 q_θ(z_θ)
    │                               │
    │                               │ ℓ2 normalize
    │                               ▼
    │                          q_θ(z_θ)   ←── MSE loss ──→ z'_ξ (ℓ2 normalized)
    │                                                         ▲
    │                                                         │
    └── 数据增强 t' (~𝒯') ──→ v' = t'(x)
            │
            ▼
        Target 网络 (ξ, 不更新梯度)
            │
            ├── 编码器 f_ξ ──→ 表征 y'_ξ = f_ξ(v')
            │
            └── 投影器 g_ξ ──→ 投影 z'_ξ = g_ξ(y'_ξ)
    
参数更新:
    θ ← optimizer(θ, ∇_θ ℒ, η)       # online 网络通过梯度下降更新
    ξ ← τξ + (1-τ)θ                   # target 网络通过 EMA 更新
```

### 4.2 前向过程逐步跟踪

**Step 1 — 数据增强**

输入图片 $x \sim \mathcal{D}$ ，从两个增强分布 $\mathcal{T}$ 和 $\mathcal{T}'$ 中采样 $t$ 和 $t'$ ，生成两个增强视图：

$$v = t(x), \quad v' = t'(x)$$

BYOL 使用的增强包括：随机裁剪（random crop，面积占原图 8%-100%）、颜色抖动（color jitter）、色相偏移（hue shift）、灰度化（grayscale conversion）、高斯模糊（Gaussian blur）和水平翻转（horizontal flip）。

**Step 2 — Online 网络前向**

Online 网络由三部分组成：

1. **编码器 $f_\theta$ ：** 将增强视图 $v$ 映射到表征 $y_\theta = f_\theta(v)$ 。使用 ResNet-50 或更深的 ResNet 变体。最终使用 GAP（全局平均池化）得到 2048 维向量。
2. **投影器 $g_\theta$ ：** 将 $y_\theta$ 映射到投影 $z_\theta = g_\theta(y_\theta)$ 。是一个 MLP：线性层(2048) → BN → ReLU → 线性层(4096) → BN → ReLU → 线性层(256)。注意这里用的是 batch normalization，后续消融实验表明 BN 对 BYOL 的性能有重要影响。
3. **预测器 $q_\theta$ ：** 将 $z_\theta$ 映射到预测 $q_\theta(z_\theta)$ 。结构和投影器一样（隐藏层 4096）：线性层(256) → BN → ReLU → 线性层(4096) → BN → ReLU → 线性层(256)。**这是 online 网络特有的模块，target 网络没有。**

**Step 3 — Target 网络前向**

Target 网络由编码器 $f_\xi$ 和投影器 $g_\xi$ 组成（没有预测器），对第二个增强视图 $v'$ 进行处理：

$$y'_\xi = f_\xi(v'), \quad z'_\xi = g_\xi(y'_\xi)$$

注意 target 网络上使用了 ***stop-gradient**——梯度不会通过 target 网络回传。

**Step 4 — Loss 计算**

对 $q_\theta(z_\theta)$ 和 $z'_\xi$ 分别做 $\ell_2$ 归一化：

$$\overline{q_\theta}(z_\theta) = \frac{q_\theta(z_\theta)}{\|q_\theta(z_\theta)\|_2}, \quad \overline{z}'_\xi = \frac{z'_\xi}{\|z'_\xi\|_2}$$

然后计算 MSE loss：

$$\mathcal{L}_{\theta,\xi} = \|\overline{q_\theta}(z_\theta) - \overline{z}'_\xi\|_2^2 = 2 - 2 \cdot \frac{\langle q_\theta(z_\theta), z'_\xi \rangle}{\|q_\theta(z_\theta)\|_2 \cdot \|z'_\xi\|_2}$$

这个 loss 等价于**负余弦相似度**（加常数偏移 2，取值范围 [0, 4]）。

**Step 5 — 对称化（Symmetrization）**

为了更加充分地利用数据，BYOL 把 loss 做对称化处理：同样把 $v'$ 喂给 online 网络、 $v$ 喂给 target 网络，计算 $\tilde{\mathcal{L}}_{\theta,\xi}$ 。最终的 loss 是两个方向的和：

$$\mathcal{L}^{\text{BYOL}}_{\theta,\xi} = \mathcal{L}_{\theta,\xi} + \tilde{\mathcal{L}}_{\theta,\xi}$$

对称化的直觉：两张增强视图没有主次之分，两张都应该既能作为"预测的来源"也能作为"预测的目标"。

**Step 6 — 参数更新**

每一步训练：

1. 对 $\theta$ 做梯度下降（只对 $\mathcal{L}$ 关于 $\theta$ 求梯度；对 $\xi$ 使用 stop-gradient，即 $\xi$ 不参与梯度计算）：

$$\theta \leftarrow \text{optimizer}(\theta, \nabla_\theta \mathcal{L}^{\text{BYOL}}_{\theta,\xi}, \eta)$$

2. 用 EMA 更新 $\xi$ ：

$$\xi \leftarrow \tau\xi + (1-\tau)\theta$$

其中 $\tau \in [0, 1]$ 是目标衰减率（target decay rate）。论文默认 $\tau_{\text{base}} = 0.996$ （1000 epoch 训练），消融实验中用 $\tau = 0.99$ （300 epoch，见 Section 5）。

**Step 7 — 推理时**

训练结束后，只保留 online 网络的**编码器** $f_\theta$ 。投影器 $g_\theta$ 、预测器 $q_\theta$ 和整个 target 网络都被丢弃。 $y_\theta = f_\theta(x)$ 被用作下游任务的图像表征。

## 五、核心模块逐个解释

### 5.1 Online 和 Target 的双分支架构

**为什么需要两个网络？**

如果只有一个网络（self-supervised 中最简单的作法：同一张图的两个增强视图送入同一个网络，最小化它们的表征距离），模型会迅速坍缩到常数解。因为梯度更新直接推动两个视图的表征互相靠近，模型学到的最佳策略就是"输出相同的向量"。

BYOL 的方案是引入**两个不同步更新的网络**：

- **Online 网络**（ $\theta$ ）：负责学习表征，通过梯度下降更新。它有预测器 $q_\theta$ ，需要去猜测 target 网络的输出。
- **Target 网络**（ $\xi$ ）：提供回归目标，通过 EMA 从 online 网络缓慢更新。它没有预测器。

这两个网络的**异步更新**是关键：target 网络永远"滞后"于 online 网络。这意味着 online 网络总是试图去追赶一个变化缓慢但更稳定的目标。这种"追赶"的动态过程迫使 online 网络持续改进它的表征——如果 online 和 target 同步更新，两者可以互相妥协达成坍缩解；但 target 的滞后性使得这种妥协无法成立。

**为什么 target 网络的一致性很重要？**

EMA 更新方式：

$$\xi \leftarrow \tau\xi + (1-\tau)\theta$$

当 $\tau$ 接近 1（如 0.99 或 0.996）时， $\xi$ 的变化非常缓慢，近似于"在很长的时间窗口内对 $\theta$ 取平均"。

这种一致性为 online 网络提供了一个稳定目标。对比学习的目标函数中，负样本的表征每步都在变化（因为所有样本的表征都在更新），这造成了目标的"移动靶"问题。BYOL 的 target 网络近似于一个静止靶，让 online 可以更稳定地学习。

### 5.2 Predictor 和 Stop-Gradient

**Predictor 的角色：防止坍缩**

在论文的 Section 5 中，作者做了一个关键的消融实验：移除 predictor（即 online 和 target 网络完全对称，只有编码器 + 投影器），目标函数是最小化 $z_\theta$ 和 $z'_\xi$ 的距离。结果：**模型坍缩**（Row 7, Table 5(b)）。

为什么 predictor 能防止坍缩？

直觉上，predictor $q_\theta$ 的存在打破了 online 和 target 的对称性。如果没有 predictor，模型可以简单地让 $z_\theta$ 和 $z'_\xi$ 都收敛到相同的常数向量——这对两者来说都是最优解。但有了 predictor 后，online 分支多了一层可以学习的非线性变换，它需要去**预测** target 的输出。

关键洞察：如果表征空间发生了坍缩（所有图片都映射到同一点），predictor 可以非常轻松地学会输出这个常数——但这样一来，online 的编码器和投影器就没必要保留任何图片间的差异信息了。然而，由于 target 是 EMA（缓慢变化），它的输出也在缓慢演变，predictor 需要持续不断地调整。**这个持续的调整压力迫使编码器保留足够的信息来使预测成为可能。**

论文提供了进一步的证据：他们发现，即使去掉 target 网络，只要让 predictor 保持近似最优（通过线性回归或提高 predictor 的学习率），模型也不会坍缩（52.5% 和 66.5% top-1 准确率）。这强烈表明 predictor 的"最优性保持"是防止坍缩的关键，而 target 网络的 EMA 机制起到了维持 predictor 最优性的作用。

**Stop-Gradient 的角色：防止 trivial 解**

Stop-gradient 的含义是对 target 网络不计算梯度、不更新参数（梯度流在 target 分支上被截断）。这是必要的，因为如果 target 网络也通过梯度更新，online 和 target 可以"合谋"达成坍缩解——例如，两者都输出常数，梯度为 0，两边都满意。

Stop-gradient 迫使 target 网络只能通过 $f_\theta$ 和 $g_\theta$ 的 EMA 间接更新。这保持了 target 作为"固定目标"的一致性，是稳定自举循环的关键。

### 5.3 EMA 参数更新（Momentum Encoder）

EMA 的参数 $\tau$ 控制 target 网络的更新速度：

- $\tau = 0$ ：target 网络立刻同步到 online（ $\xi = \theta$ ），训练不稳定，性能很差
- $\tau = 1$ ：target 网络从不更新（保持初始化），训练稳定但无法迭代改进，最终表征质量低
- $\tau \in [0.9, 0.999]$ ：合适的中间值

论文在 300 epoch 消融中默认 $\tau = 0.99$ （论文 Section 5）。1000 epoch 训练使用更大的 $\tau_{\text{base}} = 0.996$ ，并在训练过程中使用 cosine schedule 逐渐增大到 1。

论文 Table 5(a) 展示了 $\tau$ 的消融（300 epoch，默认 $\tau=0.99$ 时 top-1 为 72.5%）：

$$\begin{array}{ll}
\tau & \text{Top-1} \\
1.0\ (\text{固定随机网络}) & 18.8\% \\
0.999 & 69.8\% \\
0.99 & \textbf{72.5\%} \\
0.9 & 68.4\% \\
0\ (\text{同步更新}) & \text{坍缩/很差}
\end{array}$$

### 5.4 为什么不需要负样本也不会坍缩

这是 BYOL 最核心也最反直觉的问题。论文没有给出严格的数学证明，而是提供了如下经验和推理支持：

**1. 固定随机目标实验（Motivation）**

论文在 Introduction 和 Section 3 中描述了一个关键实验：用一个固定随机初始化的网络作为 target，训练 online 去预测它。在不使用任何负样本的情况下，**模型没有坍缩**，学到了 18.8% 的 top-1 准确率（线性评估）。相比之下，随机初始化网络本身的准确率只有 1.4%。

这个实验表明：**预测一个固定的、非坍缩的目标，本身就足以防止坍缩。** 即使这个目标的表征质量很低（随机权重），online 网络也能从中提炼出超过随机水平的表征。

**2. Predictor + EMA Target 的组合**

在线性评估的 72.5% 准确率下，模型显然没有坍缩。论文假设组合效应如下：

(a) **Predictor 要求在线表征包含足够信息。** 如果 $z_\theta$ 坍缩到了常数，predictor 只需要学到 $\text{常数} \to \text{常数}$ 的映射。但由于 target 表征 $z'_\xi$ 在不断演变（虽然缓慢），predictor 无法维持这个简单映射——它必须持续调整，从而迫使 $z_\theta$ 保留信息。

(b) **EMA Target 提供了稳定但演化的目标。** 稳定意味着 online 可以收敛到有意义的预测；演化意味着目标"越来越难"，迫使表征持续提升。

(c) **Batch Normalization 也起到了次要作用。** BN 统计量的计算依赖于 batch 中的样本多样性——如果所有样本的表征都相同，BN 的统计量会退化。

**3. 对称化的影响**

对称化 loss（两个方向都算）对性能有显著影响。去掉对称化后，性能从 72.5% 下降到 [see paper Table 5(b)]。对称化迫使 online 网络在两种增强视图配置下都能工作，增加了学习的丰富性。

## 六、核心公式

### 6.1 EMA 参数更新

$$\xi \leftarrow \tau\xi + (1-\tau)\theta$$

这是从 deep RL 中借鉴的技术。在 DQN 中，target network 用于提供稳定的 Q 值目标。在 BYOL 中，target network 提供稳定的表征目标。

### 6.2 MSE Loss（负余弦相似度）

$$\mathcal{L}_{\theta,\xi} = \|\overline{q_\theta}(z_\theta) - \overline{z}'_\xi\|_2^2 = 2 - 2 \cdot \frac{\langle q_\theta(z_\theta), z'_\xi \rangle}{\|q_\theta(z_\theta)\|_2 \cdot \|z'_\xi\|_2}$$

展开后的形式清晰地表明：最小化 MSE 等价于**最大化余弦相似度**。 $\ell_2$ 归一化是为了防止投影向量的范数任意增长或缩小。

### 6.3 统一框架

论文在 ablation 中提出了一个统一的对比/非对比学习框架。定义 similarity function：

$$S_\theta(u_1, u_2) = \frac{\langle \phi(u_1), \psi(u_2) \rangle}{\|\phi(u_1)\|_2 \cdot \|\psi(u_2)\|_2}$$

扩展的损失函数：

$$\text{InfoNCE}^{\alpha,\beta}_\theta = \frac{2}{B}\sum_{i=1}^B S_\theta(v_i, v'_i) - \beta \cdot \frac{2\alpha}{B}\sum_{i=1}^B \ln\left(\sum_{j \neq i} \exp\frac{S_\theta(v_i, v_j)}{\alpha} + \sum_j \exp\frac{S_\theta(v_i, v'_j)}{\alpha}\right)$$

- $\beta = 1$ ， $\phi = \psi = z_\theta$ （无 predictor，无 target）：恢复 SimCLR loss
- $\beta = 0$ ， $\phi = q_\theta(z_\theta)$ ， $\psi = z_\xi$ （有 predictor，有 target）：恢复 BYOL loss
- $\beta = 0$ ，去掉 predictor：坍缩

这个框架清晰地展示了 SimCLR 和 BYOL 之间的关系——区别仅在于是否使用 predictor + target network，以及是否包含负项（ $\beta = 0$ vs $\beta = 1$ ）。

## 八、实验结果

### 8.1 ImageNet 线性评估（Table 1）

BYOL 在所有架构下均超过当时的 SOTA：

- ResNet-50：74.3%（超过 InfoMin Aug 的 73.0%）
- ResNet-200 (2×)：79.6%（超过 SimCLR ResNet-50 (4×) 的 76.5%，同时参数少 30%）

BYOL 在 ResNet-50 (4×) 上达到 78.6%，与有监督 ResNet-50 (4×) 的 78.9% 非常接近——这在当时是自监督学习最接近有监督的成果。

### 8.2 ImageNet 半监督学习（Table 2）

使用 ImageNet 的 1% 和 10% 标签微调的半监督实验：

- ResNet-50：1% 标签 53.2%，10% 标签 78.4%
- ResNet-200 (2×)：1% 标签 Top-1=71.2%, Top-5=89.5%

在 ResNet-50 上，BYOL 的 1% 半监督结果（53.2%）超过了 SimCLR 在 1% 标签下的结果（48.3%）。PIRL 在 10% 标签下 Top-1 为 57.2%，其半监督性能受增强策略影响较大。

### 8.3 迁移学习（Table 3）

BYOL 在 12 个下游分类数据集上进行线性评估和微调。结果：

- 在 Food101、CIFAR100、Birdsnap、SUN397、Cars、Flowers 上，BYOL 的线性评估表现超过 SimCLR 和有监督 ImageNet 基线
- 在 CIFAR10、Pets、Caltech-101 上接近但略低于有监督基线
- 微调后的 BYOL 在几乎所有数据集上都达到或接近 SOTA

### 8.4 鲁棒性（Figure 3）

BYOL 对 batch size 和数据增强的鲁棒性远超 SimCLR。这是 BYOL 最实用的优势之一——它不需要大 batch（减少负样本的隐性需求），也不需要依赖精心调参的增强组合就能工作得很好。

## 九、消融实验深度分析

### 9.1 Batch Size 的影响

SimCLR 在 batch size 下降时性能迅速恶化，因为负样本数量减少。BYOL 在 batch size 4096 到 256 之间性能基本不变，唯一的下降出现在 batch size 极小时，原因是 batch normalization 统计量不稳定，而不是缺少负样本。

论文使用梯度累积（gradient averaging over consecutive steps）来隔离 BN 的影响——在减小 batch size 时，先累积多个 step 的梯度再更新，保持有效的"大 batch"统计。即便如此，BYOL 仍在很小 batch 时受限于 BN 层的 batch 统计量噪声。

### 9.2 数据增强的影响

SimCLR 论文已经强调了 color distortion 对对比学习的重要性。BYOL 的实验证明它更鲁棒：

- 移除 color distortion：SimCLR 下降 22.2 点，BYOL 仅下降 9.1 点
- 仅保留 random crop：SimCLR 降到 40.3%（丢失超过 1/3 性能），BYOL 仍有 59.4%

BYOL 的鲁棒性来自其学习目标的性质：没有判别任务，不需要依赖增强来压制"捷径"特征。BYOL 只关心保留目标表征中的信息。

### 9.3 Predictor 和 Target 的相互作用

Table 5(b) 的消融实验最有洞察力：
- 同时去掉 predictor 和 target（β=0）导致坍缩（Row 7）
- 去其一（但有负样本 β=1），性能从 72.5% 下降到 70.9%/70.7%——性能下降但未坍缩
- 说明 predictor 和 target 网络在一定程度上可以**互相补偿**，但两者的协同作用才是 BYOL 无负样本成功的关键

论文进一步做了"近最优 predictor"实验：
- 用线性回归（在当前的 batch 上求解最优 predictor），去掉 target 网络：52.5%
- 提高 predictor 的学习率，去掉 target 网络：66.5%
- 同时提高 predictor 和投影器的学习率，去掉 target 网络：~25%

这解释了 target 网络的核心作用：**通过 EMA 提供稳定的目标，使 predictor 保持在近似最优状态**。当 predictor 调整速度跟得上目标变化时，target 网络可以被替代。

### 9.4 Mean Teacher 对比

Mean Teacher（MT）是半监督方法，使用 teacher 网络（student 的 EMA）为 student 的 softmax 输出提供 $\ell_2$ 一致性 loss，叠加有监督分类 loss。

在 BYOL 的设定下移除 predictor 后，BYOL 变成了一个无监督版的 Mean Teacher（没有分类 loss，仅使用图像增强作为噪声）。这个变体坍缩了（Table 5, Row 7），说明 predictor 是无监督 bootstrapping 能够工作的关键组件。

## 十、Related Work 与技术谱系位置

### 10.1 技术树中的位置

```text
[对比学习，2018-2020]
   │
   ├── SimCLR (2020.02)        ← 大 batch + strong aug
   ├── MoCo (2019.12)          ← memory bank + momentum encoder
   │
   └── [BYOL (2020.06)]        ← ★ 本篇：无负样本的 bootstrapping ★
          │
          ├── SimSiam (2020.11)   ← 去掉 EMA，仅靠 predictor + stop-gradient
          ├── DINO (2021.04)      ← ViT + self-distillation
          ├── iBOT (2021.12)      ← MIM + online tokenizer
          ├── DINOv2 (2023)       ← 大规模训练
          └── VICReg (2022)       ← 方差正则化解释 collapse
```

### 10.2 前驱工作

**对比学习（SimCLR, MoCo, CPC）：** 证明了自监督视觉表征学习的强大能力，但依赖负样本。BYOL 的问题意识直接继承自这个路线——"负样本到底是不是必须的？"

**DeepCluster（Caron et al., 2018）：** 使用 bootstrapping 来生成伪标签，避免负样本。但需要聚类步骤，且需要特殊预防坍缩。BYOL 直接 bootstrap 表征本身，不需要聚类。

**Mean Teacher（Tarvainen & Valpola, 2017）：** 半监督学习中使用的 teacher-student 结构，EMA 更新 teacher。BYOL 的 target 网络设计直接受此启发，但去掉了有监督 loss 并增加了 predictor。

**Predictions of Bootstrapped Latents (PBL, 2019)：** RL 中的自监督方法，双向预测表征。BYOL 简化了 PBL 的设计，只使用单向预测 + EMA。

**RL 中的 Target Network（DQN, 2013）：** target network 稳定 bootstrapping 的想法直接来自深度强化学习——在 Q-learning 中使用固定或缓慢更新的 target network 来稳定 TD 更新。

### 10.3 关键区别：BYOL vs SimCLR

| 维度 | SimCLR | BYOL |
|------|--------|------|
| 负样本 | 必须 | 不需要 |
| Loss | InfoNCE (对比) | MSE (回归) |
| 架构 | 单一编码器 + 投影器 | Online + Target + Predictor |
| 目标网络 | 无 | EMA 更新 |
| Batch 敏感性 | 高（依赖大 batch 负样本） | 低 |
| 增强敏感性 | 高（需 color distortion） | 低 |
| ImageNet top-1 (R50) | 69.3% | 74.3% |
| 参数数量 | 少 | 多一个 predictor（微不足道） |

### 10.4 关键区别：BYOL vs MoCo

| 维度 | MoCo | BYOL |
|------|------|------|
| 负样本 | 必须（memory bank） | 不需要 |
| Momentum encoder | 用于维护负样本的一致性 | 用于提供稳定的预测目标 |
| Loss | InfoNCE (对比) | MSE (回归) |
| Predictor | 无 | 有 |
| 核心机制 | 对比 + 队列 | 自举 + 预测 |

### 10.5 后续影响

**SimSiam（Chen & He, 2020）：** 去掉了 EMA，只保留 predictor + stop-gradient。证明 stop-gradient 本身足以防止坍缩，甚至不需要缓慢变化的 target 网络。SimSiam 可以被看作 BYOL 的超简化版。

**DINO（Caron et al., 2021）：** 把 BYOL/SimSiam 的思想迁移到 Vision Transformer 上，使用 multi-crop 和 centering + sharpening 来防止坍缩。DINO 的表征具有惊人的语义分割特性（self-attention maps 能自动分割物体）。

**VICReg（Bardes et al., 2022）：** 提出 variance-invariance-covariance 正则化框架来理论上解释为什么 BYOL/SimSiam 不会坍缩。核心思想是：保持表征的方差非零、减少增广对之间的差异、降低表征各维度间的协方差。

**对 CLIP/SigLIP 的影响：** 虽然 CLIP 使用对比学习（需要负样本），但 BYOL 提出的 predictor + EMA 机制在某些多模态设置中也被采用。

## 十一、局限性与后续方向

### 11.1 缺乏严格的理论解释

BYOL 为什么不会坍缩，论文没有给出严格的数学证明。论文提出的假设（predictor + EMA target 的组合效应）基于实验，但缺乏理论支撑。这是一个理论空白，后来的 VICReg 和 SimSiam 的线性分析尝试填补。

### 11.2 仍然依赖数据增强

虽然 BYOL 比对比方法更鲁棒，但它仍然需要数据增强。论文承认："BYOL remains dependent on existing sets of augmentations that are specific to vision applications." 推广到其他模态（音频、视频、文本）需要为每个模态设计合适的增强——这可能是迁移到 NLP/多模态的主要障碍。

### 11.3 Batch Normalization 的重要但未解释的角色

BYOL 的投影器和预测器中都使用了 BN。消融实验显示去掉 BN 后性能显著下降（[see paper Appendix]）。BN 的作用可能包括：
- 防止表征系数过度增长
- 在正样本对上引入隐式的"对比效应"（因为 BN 统计量来源于 batch 中所有样本的分布）
- 但这使得 BN 成为了 BYOL 的一种隐式的"负样本替代品"

SimSiam 的实验表明即使不用 BN（用 LayerNorm 或没有归一化），predictor + stop-gradient 仍然有效，所以 BN 不是防坍缩的必要条件。但 BN 提供了显著的性能增益。

### 11.4 大 batch 的 BN 依赖

虽然 BYOL 对 batch size 比 SimCLR 更鲁棒，但当 batch size 很小时（<128），由于 BN 统计量的噪声增大，BYOL 的性能仍然下降。论文中 BYOL 使用的是 4096 的 batch。

### 11.5 双网络开销

BYOL 需要维护两个网络（online + target），且 target 网络的 forward pass 也需要计算。相比 SimCLR（只有一个网络），BYOL 的每次训练迭代需要多一次 forward pass（target 网络的）。但梯度只回传到 online 网络，所以反向传播的开销相同。

### 11.6 与有监督基线的差距

在 ResNet-50 上，BYOL 的 74.3% 仍然低于有监督的 76.5%（甚至落后于更强的有监督 baseline 78.9%）。虽然在大架构下（ResNet-200 2×）BYOL 的 79.6% 超过了有监督 ResNet-50，但严格来说在"相同架构"的公平对比下，自监督仍有差距。

## 十二、范式意义

**BYOL 的范式意义：它打破了"自监督学习必须依赖负样本"的共识。**

在 BYOL 之前，对比学习范式的核心假设是：要防止表征坍缩，必须通过排斥不同样本的表征来施加"对比压力"。SimCLR 靠大 batch 提供大量负样本，MoCo 靠 memory bank。社区普遍认为，这个"排斥力"是不可或缺的。

BYOL 用实验证明了：**完全没有排斥力，仅靠预测任务 + 异步更新的网络结构，表征一样可以学到，而且学得更好。** 这意味着"防止坍缩"这个问题的解空间比社区想象的更大——它不需要负样本的显式推远，而是可以通过架构设计（非对称网络、慢速目标更新）来隐式地解决。

这让 BYOL 成为了 non-contrastive（非对比式）自监督学习这个子领域的奠基之作。后续的 SimSiam、DINO、VICReg、Barlow Twins 都在这个方向上进一步探索，形成了一个完整的"无负样本表征学习"的技术路线。

从工程角度看，BYOL 更鲁棒的增强需求和对 batch size 的低敏感性，使得自监督学习的门槛大大降低。不需要精心调参的增强组合、不需要超大的 batch 来凑负样本——这让更多算力有限的研究组也能做自监督学习。

在技术树中，BYOL 是一个 **bridge node（桥梁节点）**——它连接了对比学习和无负样本表征学习这两个研究范式，开辟了一个全新的子领域。没有 BYOL，SimSiam 可能会被认为是一种"取巧的简化"而不是"有意义的简化"；DINO 可能会被认为只是"ViT 版 MoCo"。

## 十三、小结

BYOL 的核心贡献是：**提出了一种不需要负样本的自监督表征学习方法，在 ImageNet 线性评估上达到 SOTA，同时对数据增强和 batch size 的鲁棒性远超对比方法。**

它靠的是一个优雅的设计：online 网络 + predictor 负责预测，target 网络（EMA 更新）负责提供目标，stop-gradient 切断循环中的冗余梯度流。Predictor 的存在打破了对称性，迫使网络保留信息以完成预测；EMA 的缓慢更新提供了稳定的自举目标。这两者的组合，加上对称化的 loss 设计，让"没有负样本也能学"从不可能变为了可能。

BYOL 最大的未解之谜是"为什么不会坍缩"——论文给出了直觉和实验证据，但没有严格证明。直到今天，BYOL/SimSiam 的理论理解仍然是一个活跃的研究方向。

对于实践者来说，BYOL 最直接的启示是：**如果你在做自监督学习，不妨试试去掉负样本。** 预测目标网络这件事，可能比你所想的更有力。
