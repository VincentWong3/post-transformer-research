## 1. 本文要解决什么问题

自监督视觉表示学习的主流方法（SimCLR、MoCo、CPC v2 等）都是**对比学习（contrastive learning）**：通过拉近同一图片不同增广视图（positive pairs）的距离、推开不同图片的增广视图（negative pairs）的距离来学习表示。这类方法有以下问题：

1. **依赖负样本**：需要精心设计机制来获取负样本（大 batch size、memory bank、或自定义的 mining 策略），否则模型容易学到平凡解。
2. **对数据增广敏感**：SimCLR 等方法的性能高度依赖增广类型，特别是颜色失真（color distortion）几乎不可或缺。
3. **batch size 影响大**：对比方法在 batch 减小时负样本数量下降，性能会明显退化。

BYOL（Bootstrap Your Own Latent）的核心问题是：**不用负样本，还能不能学到好的表示？** 答案是**能**，而且效果更好。

## 2. 核心创新

BYOL 的核心思想是**自举（bootstrap）**——让一个网络学会预测另一个网络对同一图像另一增广视图的输出，并让目标网络以 EMA 方式缓慢追踪 online 网络的参数，形成一种迭代改进的闭环。

### 2.1 双分支架构

BYOL 使用两个神经网络：

- **Online network（权重 θ）**：由 encoder fθ + projector gθ + predictor qθ 三部分组成。
- **Target network（权重 ξ）**：与 online 架构相同（encoder + projector），**但没有 predictor**，构成不对称设计。

### 2.2 训练流程

对每张图 x：

1. 采样两种增广 t ~ T, t' ~ T'，得到两个视图 v = t(x), v' = t'(x)
2. Online 网络处理 v：yθ = fθ(v), zθ = gθ(y), 再预测 qθ(zθ)
3. Target 网络处理 v'：y'ξ = fξ(v'), z'ξ = gξ(y')
4. 计算 ℓ2-normalized 预测 qθ(zθ) 与 target projection z'ξ 的 MSE 损失
5. **对称化**：交换两个视图的角色再算一次损失，两次损失加起来
6. **只更新 online 参数 θ**（stop-gradient on ξ）
7. **EMA 更新 target 参数 ξ**：ξ ← τξ + (1 − τ)θ

其中 τ 称为 target decay rate，从 τ_base = 0.996 开始，在训练过程中以余弦方式增加到 1。

训练完成后，只保留 encoder fθ，**丢弃 projector 和 predictor**。

### 2.3 架构细节

- **Encoder fθ（投影前的 backbone）**：ResNet 系列（50/101/152/200 层，1×/2×/4× 宽度），输出为 global average pooling 后的 2048 维特征（1× 时）。
- **Projector gθ（投影头）**：MLP，**第一层线性层 output size 4096** + BatchNorm + ReLU，最后一层线性层输出 256 维。与 SimCLR 不同，**最终输出不做 BN**。
- **Predictor qθ（预测头）**：**与 projector 相同架构**（同样 hidden dim 4096 → output dim 256）。

> ⚠️ **易错提醒**：projector 和 predictor 的 hidden dim 是 **4096**（论文 Section 3.3 明确："linear layer with output size 4096"），不是 256 也不是 2048。

### 2.4 训练配置（满分阶段）

- 优化器：LARS
- 学习率调度：cosine decay，无 restart，warmup 10 epochs
- 总 epoch：**1000**（绝大部分消融实验在 **300** epoch 下运行）
- 基础学习率：0.2，线性缩放 LR = 0.2 × BatchSize / 256
- weight decay：1.5 × 10⁻⁶
- 目标网络 EMA：τ_base = 0.996，余弦增长到 1
- batch size：4096（512 个 Cloud TPU v3 核心）
- 训练时间：~8 小时（ResNet-50 ×1）

> ⚠️ **易错提醒**：消融实验全部在 **300 epoch** 下运行（论文 Section 5："we run ablations over 300 epochs"），不是 100 epoch。

## 3. 为什么没有负样本也不坍缩

这是一个核心问题。BYOL 的目标函数本身确实允许坍缩解（对所有图输出相同的向量），但实际训练中并没有收敛到这种解。论文给出的假设和直觉：

### 3.1 stop-gradient + EMA 造成的非对称动力学

BYOL 的更新是**非对称**的：损失只对 θ 求梯度，ξ 通过 EMA 被动跟随。这意味着**不存在一个统一的损失函数 L(θ, ξ) 使得 BYOL 的动力学是联合梯度下降**（类似 GAN）。因此即使损失函数存在坍缩最小值，参数也不必然会收敛到那里。

### 3.2 条件方差视角

在假设 predictor 达到最优（即 q*(zθ) = E[z'ξ | zθ]）的情况下，BYOL 的梯度更新等价于：

**∇θ E[ ||q*(zθ) − z'ξ||² ] = ∇θ E[ Σᵢ Var(z'ξ,ᵢ | zθ) ]**

即**最小化目标投影在给定 online 投影下的条件方差**。关键性质是：条件方差在给定更多信息时不会增加。因此，**丢弃信息（坍缩）不能降低条件方差**——坍缩解反而是不稳定的。

### 3.3 predictor + target network 两者缺一不可

消融实验（Table 5(b)）清楚表明：

| Predictor | Target | β（负样本权重） | Top-1 |
|-----------|--------|:-:|:-:|
| ✔ | ✔ | 0 | **72.5**（BYOL） |
| ✘ | ✔ | 1 | 70.7（稳定，因为有负样本） |
| ✔ | ✘ | 1 | 69.1（稳定，因为有负样本） |
| ✘ | ✔ | 0 | **0.3**（坍缩！） |
| ✔ | ✘ | 0 | **0.2**（坍缩！） |
| ✘ | ✘ | 0 | **0.1**（完全坍缩） |

> ⚠️ **关键理解**：β=1（有负样本）时去掉 predictor 或 target network 都不会坍缩，是负样本在起稳定作用。β=0 时**必须同时保留 predictor 和 target network**，否则模型立即坍缩到 ~0.3%。

### 3.4 predictor 的最优性

BYOL 的 EMA 机制让 target 网络**缓慢变化**，这使得 predictor 能始终保持在近乎最优的状态。实验表明：

- 移掉 target（τ=0，直接把 θ 硬拷贝到 ξ）但把 predictor 的学习率增大 10 倍 → 66.6%（可工作）
- 直接用 closed-form 最优线性 predictor（不用网络学习）→ 52.5%（可工作，但不如 EMA）
- predictor 学习率 ≤ projector 学习率时 → 坍缩

这说明：**target network 的主要作用之一是保持 predictor 始终近似最优**。

## 5. 消融实验（全部在 300 epoch 下）

### 5.1 对 batch size 的鲁棒性

对比 SimCLR 和 BYOL 在不同 batch size（4096 → 64）下的性能：

| Batch Size | BYOL Top-1 | SimCLR Top-1 |
|:-:|:-:|:-:|
| 4096 | 72.5 | 67.9 |
| 2048 | 72.4 | 67.8 |
| 1024 | 72.2 | 67.4 |
| 512 | 72.2 | 66.5 |
| 256 | **71.8** | **64.3** |
| 128 | 69.6 | 63.6 |
| 64 | 59.7 | 59.2 |

> ⚠️ **SimCLR batch 4096→256 下降约 3.6 点（67.9%→64.3%），不是 10 点**。BYOL 几乎不受影响（72.5%→71.8%），直到 batch 降得很小（<256）才因 BN 的 batch 统计不稳定而下降。

### 5.2 对数据增广的鲁棒性（Figure 3）

移除不同增广时 BYOL 和 SimCLR 的性能下降（% points）：

- **仅移颜色失真**：BYOL −9.1 vs SimCLR −22.2
- **仅用随机裁剪（crop only）**：BYOL 59.4%（−13.1） vs SimCLR 40.3%（−27.6）

BYOL 在**仅用随机裁剪**时仍能保持 59.4% 的 Top-1，而 SimCLR 失去超过三分之一（仅剩 40.3%）。这验证了 BYOL 的核心假设：不依赖负样本意味着模型不能靠"颜色直方图差异"来区分正负对，从而被迫学习更多语义信息。

### 5.3 目标网络的 decay rate（Table 5(a)）

| τ_base | 说明 | Top-1 |
|:-:|------|:-:|
| 1 | 固定随机目标网络 | 18.8 |
| 0.999 | 缓慢 EMA | 69.8 |
| **0.99** | **最佳折中** | **72.5** |
| 0.9 | 快速 EMA | 68.4 |
| 0 | 直接拷贝 online (τ=0 常数) | **0.3**（坍缩） |

τ_base = 1（固定随机目标）得 18.8%，说明预测固定的烂表示也有一定效果——这是 BYOL 的思想种子。τ_base = 0.99 最优。

### 5.4 目标网络对对比学习的增益

仅给 SimCLR 加上 target network（用 EMA 做 target，维持 β=1 即有负样本），SimCLR 就从 69.4% 提高到 70.7%（+1.3 点）。**这意味着 target network 的稳定化效果本身就是一种正增益**，而不仅仅是为 BYOL 服务。

### 5.5 其他消融

- **Projector 和 predictor 深度**：深度 2（hidden+output）最优
- **Projection dimension**：128~512 性能相近，256 为默认
- **Weight decay**：0 会导致不稳定，最优在 5×10⁻⁶ 附近
- **学习率**：0.3~0.4 最优
- **Loss 中的 normalization**：ℓ2-norm 最佳（72.5%），不做 normalization 仍有 67.4%

## 6. 与相关工作的对比

| 方法 | 是否用负样本 | 是否用 EMA | 额外机制 | ResNet-50 线性评估 Top-1 |
|------|:-:|:-:|---------|:-:|
| SimCLR | ✔ | ✘ | 大 batch 做负样本 | 69.3 |
| MoCo v2 | ✔ | ✔（momentum encoder） | memory bank 做负样本 | 71.1 |
| InfoMin Aug | ✔ | ✘ | 学习最优增广 | 73.0 |
| DeepCluster | ✘ | ✘ | 聚类做伪标签 | — |
| **BYOL** | **✘** | **✔** | **predictor + EMA** | **74.3** |

BYOL 的独特之处在于：
- 相比 MoCo：MoCo 的 EMA 是为了给 memory bank 中的负样本提供一致表示；BYOL 的 EMA 是为了产生稳定的预测目标
- 相比 Mean Teacher：MT 在半监督中用 teacher 产生 softmax 目标；BYOL 在**无监督**中用目标网络产生 latent 目标；但去掉 predictor 后 BYOL 会坍缩（MT 没这个问题因为有分类损失）
- 相比 DeepCluster：两者都不需要负样本，但 DeepCluster 需要额外的聚类步骤和防坍缩的 careful engineering

## 7. 局限与后续

### 局限性

1. **仍依赖视觉增广**：BYOL 使用的增广套件（random crop + color jitter + blur + solarization）是专门为视觉设计的。扩展音频、视频、文本等其他模态需要对应模态的合适增广。
2. **没有彻底解决自监督的增广依赖问题**：虽然比对比方法更鲁棒，但不做增广仍无法工作。

### 后续影响

BYOL 开启了"**不用负样本的表示学习**"这条新路线。直接后续工作包括：
- **SimSiam**（Chen & He, 2020）：发现甚至连 EMA 都不是必须的——只用 stop-gradient + predictor 就能工作
- **Barlow Twins**（Zbontar et al., 2021）：通过交叉相关矩阵的冗余减少目标来实现
- **VICReg**（Bardes et al., 2022）：方差-不变性-协方差正则化

## 8. 总结

BYOL 用一个优雅的 bootstrap 机制证明了：**负样本不是自监督表示学习的必要条件**。它的核心设计——online + target 双分支 + predictor + stop-gradient + EMA 更新——形成了一个动态稳定的自举循环，在 ImageNet 线性评估（74.3%）、半监督（1% 标签 53.2%）、迁移学习等多个维度全面超越当时的对比学习方法。

这个工作的关键 insight 是：不是"推开不同样本"在驱动学习，而是"预测缓慢变化的 target 表示"这个任务迫使 online 网络不断编码更多信息，以降低条件方差。这种思路对后来的非对比学习（non-contrastive learning）方向影响深远。
