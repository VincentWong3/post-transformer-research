# Deformable DETR: Deformable Transformers for End-to-End Object Detection（2021）

**论文：** [arXiv](https://arxiv.org/abs/2010.04159) · Zhu et al. · 2021

---

## 一、先搞清楚坑在哪

目标检测在深度学习时代经历了从两阶段（Faster R-CNN）到单阶段（YOLO、SSD）的演变，但都有一个共同的前提：**需要大量手工设计的组件**。这些组件包括：

1. **锚框（Anchor Box）**：预设一组不同大小和长宽比的候选框，模型在此基础上进行回归调整。这需要人工设计锚框的尺寸、比例、数量，并且对数据集的统计特性敏感。
2. **非极大值抑制（NMS）**：后处理阶段，去除重叠的检测框。这个操作不可微，且阈值需要人工调参。
3. **正负样本分配策略**：决定哪些预测框应该匹配到真实目标，哪些应该被忽略。这通常涉及复杂的规则和启发式方法。

这些手工设计的组件使得检测系统变得复杂、脆弱，且难以端到端优化。**核心矛盾在于：我们想要一个简洁的端到端系统，但现有的检测器却充斥着各种手工设计的中间步骤。**

## 二、DETR 的真正问题

2020年，Facebook AI 提出了 DETR（Detection Transformer），首次将 Transformer 架构引入目标检测，实现了真正的端到端检测——不需要锚框、NMS、正负样本分配。DETR 的创新在于：

- 将目标检测视为**集合预测问题**（set prediction），直接输出一组检测结果
- 使用 Transformer 的 encoder-decoder 架构和二分图匹配（Hungarian matching）来建立预测和真实目标之间的对应关系
- 完全抛弃了手工设计的组件

乍看之下，DETR 是一个优雅的解决方案。但它有一个致命缺陷：**收敛速度极慢**。在 COCO 数据集上，DETR 需要 500 个 epoch 才能收敛，而 Faster R-CNN 只需要 12-36 个 epoch。为什么？

**问题出在注意力机制上。** DETR 的 Transformer 使用全局注意力（global attention），每个 query 都要与所有位置的 key 计算注意力权重。对于目标检测任务来说，这存在两个根本性缺陷：

1. **计算复杂度高**：空间复杂度 $O(HW \cdot N_q)$，其中 $HW$ 是特征图大小，$N_q$ 是 query 数量。对于高分辨率特征图，这个复杂度是难以承受的。
2. **收敛困难**：全局注意力要求模型在训练初期就从整张图像中学习哪些区域是相关的。这就像让一个新生婴儿在第一次看到世界时就理解所有物体之间的关系——信息过载导致学习效率极低。

有意思的是，DETR 在训练初期，注意力权重几乎是均匀分布在整张图像上的。这意味着每个 query 都要处理大量无关的背景信息，梯度信号被稀释，模型难以聚焦到真正的目标区域。

## 三、Deformable DETR 的核心思路

Deformable DETR 的核心洞察是：**注意力机制不应该是全局的，而应该是自适应的、稀疏的。** 具体来说，对于每个 query，我们不需要关注图像中的所有位置，只需要关注那些与当前 query 最相关的少数几个位置即可。

这听起来像是废话，但关键在于如何实现这种"自适应稀疏注意力"。Deformable DETR 借鉴了可变形卷积（Deformable Convolution）的思想，将**可变形采样**与**Transformer 注意力**结合起来。

**一句话总结：Deformable DETR 用可变形注意力（Deformable Attention）替代了 DETR 中的全局注意力，每个 query 只关注 K 个采样点（K 远小于特征图大小），这些采样点的位置由 query 特征动态预测。**

这种设计的直接好处是：
- 计算复杂度从 $O(HW \cdot N_q)$ 降低到 $O(K \cdot N_q)$，其中 $K$ 通常取 4 或 8
- 收敛速度大幅提升，只需要 50 个 epoch（DETR 的 1/10）
- 能够自然地处理多尺度特征，不需要额外的特征金字塔网络（FPN）

## 四、网络架构详解

Deformable DETR 的整体架构与 DETR 类似，但核心的注意力模块被替换为可变形注意力。下图展示了完整的网络结构：

```
输入图像 (H x W x 3)
      │
      ▼
┌─────────────────────────────────────────────┐
│              CNN Backbone (ResNet-50)        │
│  输出多尺度特征图:                            │
│    Level 1: H/8 x W/8 x 256                  │
│    Level 2: H/16 x W/16 x 256                │
│    Level 3: H/32 x W/32 x 256                │
│    Level 4: H/64 x W/64 x 256                │
└─────────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────────┐
│          Transformer Encoder (6 layers)       │
│  每层: Deformable Self-Attention + FFN       │
│  输入: 所有尺度特征图展平后的 token 序列       │
│  输出: 增强后的多尺度特征表示                   │
└─────────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────────┐
│          Transformer Decoder (6 layers)       │
│  每层包含两个注意力模块:                       │
│    1. Deformable Cross-Attention (核心创新)    │
│    2. Self-Attention (object queries之间)     │
│  输入: N_q 个 object queries (可学习)          │
│  输出: N_q 个预测结果 (bbox + class)          │
└─────────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────────┐
│          预测头 (FFN)                         │
│  每个 query 输出:                             │
│    - 类别概率 (N_class + 1)                   │
│    - 边界框坐标 (4)                           │
└─────────────────────────────────────────────┘
```

### 前向过程详解

让我们一步步追踪数据流，看每个模块做了什么：

**Step 1: 图像输入**
- 输入：$(3, H, W)$ 的 RGB 图像，$H=W=800$（通常）
- 输出：原始图像，保持不变

**Step 2: CNN Backbone**
- 使用 ResNet-50（或 ResNet-101）作为骨干网络
- 提取多尺度特征图，从 stage 3 到 stage 5（C3, C4, C5）
- 通过 1x1 卷积将所有特征图投影到相同的通道数 $C=256$
- 输出四个尺度的特征图：
  - Level 1: $(256, H/8, W/8)$ → 展平后长度 $L_1 = H/8 \times W/8$
  - Level 2: $(256, H/16, W/16)$ → $L_2 = H/16 \times W/16$
  - Level 3: $(256, H/32, W/32)$ → $L_3 = H/32 \times W/32$
  - Level 4: $(256, H/64, W/64)$ → $L_4 = H/64 \times W/64$
- 总 token 数：$L_{total} = L_1 + L_2 + L_3 + L_4$

**Step 3: Transformer Encoder**
- 输入：所有尺度的特征图被展平并拼接成一个序列，每个 token 的维度是 $C=256$
- 每个 encoder 层包含：
  1. **Deformable Self-Attention**：每个 token 关注其他 token（跨尺度）
  2. **Feed-Forward Network**：两层 MLP，隐藏层维度 1024
- 关键：encoder 中的可变形注意力是**跨尺度**的，每个 query 可以从不同尺度的特征图中采样点
- 输出：与输入相同形状的增强特征序列

**Step 4: Transformer Decoder**
- 输入：
  - $N_q$ 个 object queries（可学习的嵌入向量），维度 $(N_q, 256)$
  - encoder 输出的特征序列作为 memory
- 每个 decoder 层包含：
  1. **Self-Attention**：object queries 之间互相交互，避免预测重复的框
  2. **Deformable Cross-Attention**：每个 query 从 encoder 输出的多尺度特征中采样 K 个点
  3. **FFN**：同 encoder
- 输出：$(N_q, 256)$ 的 decoder 输出

**Step 5: 预测头**
- 两个并行的 FFN：
  - 分类头：$(N_q, 256) \rightarrow (N_q, N_{class}+1)$
  - 回归头：$(N_q, 256) \rightarrow (N_q, 4)$，输出归一化的边界框坐标 $(cx, cy, w, h)$
- 训练时使用二分图匹配（Hungarian matching）将预测与真实目标配对

## 五、核心创新点

### 创新点 1：可变形注意力（Deformable Attention）

#### (a) 痛点与动机

**问题：** DETR 的全局注意力机制有两个致命缺陷：

1. **计算复杂度高**：对于特征图大小 $HW$ 和 query 数量 $N_q$，全局注意力的复杂度是 $O(HW \cdot N_q)$。当 $HW$ 很大（例如 100x100=10000）时，这个复杂度是难以接受的。
2. **收敛困难**：全局注意力要求模型关注所有位置，但目标检测中每个 query 只需要关注少数几个相关位置。这种"一刀切"的方式导致梯度信号被大量无关位置稀释。

**现有方法的局限：**
- [Vaswani et al., 2017] 的标准 Transformer 注意力：复杂度 $O(N^2)$，对长序列不友好
- [Carion et al., 2020] 的 DETR：使用全局注意力，收敛需要 500 epoch
- 局部窗口注意力（如 Swin Transformer）：虽然降低了复杂度，但窗口大小是固定的，无法适应目标的形状和大小变化

#### (b) 方案细节

可变形注意力的核心公式如下：

$$ \text{DeformAttn}(z_q, p_q, x) = \sum_{m=1}^{M} W_m \sum_{k=1}^{K} A_{mqk} \cdot W_m' x(p_q + \Delta p_{mqk}) $$

其中：
- $z_q$：query 的特征向量，维度 $C$
- $p_q$：query 的参考点（reference point），是一个 2D 坐标
- $x$：输入特征图
- $M$：注意力头的数量
- $K$：每个头采样的点数（论文中取 4）
- $W_m$：第 $m$ 个头的输出投影矩阵
- $W_m'$：第 $m$ 个头的值投影矩阵
- $A_{mqk}$：第 $m$ 个头第 $k$ 个采样点的注意力权重，通过 query 特征 $z_q$ 预测得到，且 $\sum_{k=1}^{K} A_{mqk} = 1$
- $\Delta p_{mqk}$：第 $m$ 个头第 $k$ 个采样点的偏移量，通过 query 特征 $z_q$ 预测得到

**关键点：**
1. **采样点位置是动态预测的**：$\Delta p_{mqk}$ 不是固定的，而是从 query 特征 $z_q$ 通过一个子网络（通常是一层线性层）预测得到
2. **注意力权重也是动态预测的**：$A_{mqk}$ 同样从 $z_q$ 预测
3. **采样点数量 K 远小于特征图大小**：K=4 意味着每个 query 只关注 4 个位置，而特征图可能有 10000+ 个位置
4. **多尺度扩展**：对于多尺度特征，$x$ 变为多尺度特征图的集合，采样点可以来自不同尺度

#### (c) 为什么有效

**直觉理解：**

想象你在人群中找一个人。全局注意力相当于你同时观察所有人，然后找出你要找的那个人——这需要处理大量无关信息。可变形注意力相当于你先猜测那个人可能在哪几个位置，然后只看向那几个位置——如果猜测准确，效率会高得多。

**为什么动态预测采样点有效？**

1. **目标检测的先验知识**：每个目标只占图像的一小部分区域，而且位置是局部的。因此，每个 query 只需要关注目标周围的一小片区域。
2. **自适应能力**：不同目标有不同的形状和大小，固定位置的采样点无法覆盖所有情况。动态预测让模型能够根据目标的特征自适应地调整采样位置。
3. **梯度信号集中**：由于只关注少数几个位置，梯度信号不会分散到大量无关位置，训练效率大幅提升。

**实验证据：**
- 论文 Table 1 显示：Deformable DETR 在 50 epoch 达到 43.8 AP，而 DETR 在 500 epoch 才达到 44.0 AP
- 论文 Table 5 显示：使用 K=4 的采样点已经足够，增加到 K=8 只有微小的提升（+0.2 AP）

#### (d) 与 Related Work 的关系

- **可变形卷积 [Dai et al., 2017]**：可变形注意力的灵感来源。可变形卷积通过预测卷积核的偏移量来适应目标的形状，但局限于局部感受野。可变形注意力将其扩展到注意力机制中，实现了更大的感受野和更灵活的特征聚合。
- **DETR [Carion et al., 2020]**：Deformable DETR 直接建立在 DETR 的基础上，保留了端到端的集合预测框架，但用可变形注意力替换了全局注意力。
- **Sparse Transformer [Child et al., 2019]**：也探索了稀疏注意力，但使用的是固定模式（如固定步长），而可变形注意力的稀疏模式是动态学习的。

**对比：** 如果使用可变形卷积代替可变形注意力，会怎样？可变形卷积的采样点局限在卷积核的局部范围内（如 3x3），无法覆盖跨尺度的信息。而可变形注意力可以跨尺度采样，这是其优势所在。

#### (e) 如果去掉会怎样

论文 Table 5 的消融实验直接回答了这个问题：

| 变体 | AP |
|------|-----|
| Deformable DETR (完整) | 43.8 |
| 替换为全局注意力（DETR 风格） | 41.2 (-2.6) |
| 替换为可变形卷积 | 42.1 (-1.7) |
| 固定采样点位置（不预测偏移） | 41.8 (-2.0) |

可见，可变形注意力对性能有显著贡献，且动态预测采样点位置比固定位置好得多。

### 创新点 2：多尺度可变形注意力

#### (a) 痛点与动机

**问题：** 目标检测需要处理不同尺度的目标。小目标需要高分辨率特征图来捕获细节，大目标需要低分辨率特征图来捕获整体语义。DETR 只使用单尺度特征（CNN 最后一层的输出），这限制了它对小目标的检测能力。

**现有方法的局限：**
- [Lin et al., 2017] 的 FPN：需要额外的特征金字塔网络来融合多尺度特征，增加了计算开销
- [Ghiasi et al., 2019] 的 NAS-FPN：搜索空间大，训练成本高
- DETR：单尺度特征，小目标 AP 低（在 COCO 上 small AP 只有 12.0）

#### (b) 方案细节

多尺度可变形注意力的公式扩展为：

$$ \text{MSDeformAttn}(z_q, \hat{p}_q, \{x^l\}_{l=1}^L) = \sum_{m=1}^{M} W_m \sum_{l=1}^{L} \sum_{k=1}^{K} A_{mlqk} \cdot W_m' x^l(\phi_l(\hat{p}_q) + \Delta p_{mlqk}) $$

其中：
- $\{x^l\}_{l=1}^L$：L 个尺度的特征图（论文中 L=4）
- $\hat{p}_q$：归一化的参考点坐标，范围 $[0,1]^2$
- $\phi_l(\hat{p}_q)$：将归一化坐标映射到第 l 个特征图的坐标
- $A_{mlqk}$：现在多了一个维度 l，表示从哪个尺度采样

**关键点：**
1. **参考点 $\hat{p}_q$**：在 decoder 中，每个 object query 都有一个参考点，初始化为可学习的嵌入。随着训练进行，参考点会逐渐移动到目标位置。
2. **跨尺度采样**：每个 query 可以从所有尺度的特征图中采样点，但每个尺度的采样点数量由注意力权重 $A_{mlqk}$ 控制。
3. **坐标映射**：$\phi_l(\hat{p}_q)$ 将归一化坐标映射到第 l 个特征图的实际坐标。例如，如果 $\hat{p}_q = (0.5, 0.5)$，在 level 1 特征图上映射到 $(H/16, W/16)$，在 level 4 特征图上映射到 $(H/128, W/128)$。

#### (c) 为什么有效

**直觉理解：**

多尺度可变形注意力就像让每个 query 同时拥有"望远镜"和"显微镜"——它可以从低分辨率特征图获取全局上下文，从高分辨率特征图获取细节信息。而且，模型可以动态决定每个尺度的重要性权重。

**为什么比 FPN 更好？**

1. **自适应融合**：FPN 使用固定的自顶向下路径来融合多尺度特征，而可变形注意力允许每个 query 动态选择从哪个尺度采样。
2. **计算效率**：FPN 需要计算所有位置的多尺度特征，而可变形注意力只计算被采样到的位置。
3. **端到端优化**：多尺度采样作为注意力机制的一部分，可以与整个网络端到端训练。

**实验证据：**
- 论文 Table 3 显示：使用多尺度特征比单尺度特征提升 2.7 AP（43.8 vs 41.1）
- 对于小目标，多尺度特征的提升尤为显著（small AP 从 18.9 提升到 22.9）

#### (d) 与 Related Work 的关系

- **FPN [Lin et al., 2017]**：开创了多尺度特征融合的先河，但融合方式固定。Deformable DETR 的多尺度注意力可以看作是一种"可学习的 FPN"。
- **TridentNet [Li et al., 2019]**：使用多个分支处理不同尺度，但每个分支独立训练。Deformable DETR 的多尺度注意力是共享参数的。

#### (e) 如果去掉会怎样

论文 Table 3 显示：
- 单尺度 + 可变形注意力：41.1 AP
- 多尺度 + 可变形注意力：43.8 AP
- 提升：+2.7 AP

对于小目标，提升更大：+4.0 AP（18.9 → 22.9）

### 创新点 3：两阶段变形 DETR（Two-Stage Deformable DETR）

#### (a) 痛点与动机

**问题：** DETR 使用固定的 object queries（可学习的嵌入），这些 query 在训练过程中逐渐学会关注图像中的特定区域。但这种固定 query 的方式有两个问题：
1. **初始化困难**：query 的初始位置完全随机，模型需要从头学习每个 query 应该关注哪里。
2. **冗余**：多个 query 可能最终关注到同一个目标，造成计算浪费。

**现有方法的局限：**
- DETR：固定 query，每个 query 的参考点是固定的（论文中初始化为 0,0）
- [Carion et al., 2020] 的尝试：增大 query 数量可以缓解但无法根本解决

#### (b) 方案细节

两阶段 Deformable DETR 将检测过程分为两个阶段：

**第一阶段**：使用 encoder 的输出作为 proposal 生成器
1. 将 encoder 输出的每个特征点视为一个 proposal
2. 通过一个轻量级预测头，为每个特征点预测：
   - 目标性得分（objectness score）：该位置是否存在目标
   - 边界框坐标：该位置对应的目标框
3. 选择得分最高的 $N_q$ 个 proposal 作为第二阶段的 query

**第二阶段**：使用第一阶段的 proposal 作为 decoder 的 query
1. 每个 query 的参考点初始化为第一阶段预测的边界框中心
2. query 特征初始化为对应位置的特征
3. 使用标准的 Deformable DETR decoder 进行精炼

#### (c) 为什么有效

**直觉理解：**

两阶段设计相当于先让模型做一次"快速扫描"，找到可能有目标的位置，然后再对这些位置进行"精细检测"。这比从头开始学习每个 query 应该关注哪里要高效得多。

**关键优势：**
1. **更好的初始化**：query 的参考点不再是随机的，而是基于 encoder 的输出预测得到的，更接近真实目标的位置。
2. **更少的冗余**：由于 proposal 已经筛选过，多个 query 指向同一个目标的概率降低。
3. **更高的收敛效率**：论文显示两阶段版本在 50 epoch 时比一阶段版本高 1.5 AP。

#### (d) 与 Related Work 的关系

- **Faster R-CNN [Ren et al., 2015]**：也使用两阶段设计，但 RPN 需要手工设计的锚框和 NMS。Deformable DETR 的两阶段是端到端的，不需要这些手工组件。
- **DETR [Carion et al., 2020]**：一阶段设计，query 初始化是固定的。Deformable DETR 的两阶段是对 DETR 的自然扩展。

- 两阶段 Deformable DETR：45.3 AP

## 六、公式详解

### 公式 1：可变形注意力

$$ \text{DeformAttn}(z_q, p_q, x) = \sum_{m=1}^{M} W_m \sum_{k=1}^{K} A_{mqk} \cdot W_m' x(p_q + \Delta p_{mqk}) $$

#### (a) 符号定义

| 符号 | 含义 | 维度 |
|------|------|------|
| $z_q$ | query 的特征向量 | $C$ |
| $p_q$ | query 的参考点坐标 | $2$ |
| $x$ | 输入特征图 | $(C, H, W)$ |
| $M$ | 注意力头数 | 标量（论文取 8） |
| $K$ | 每个头的采样点数 | 标量（论文取 4） |
| $W_m$ | 第 m 个头的输出投影 | $C \times C/M$ |
| $W_m'$ | 第 m 个头的值投影 | $C \times C/M$ |
| $A_{mqk}$ | 第 m 头第 k 点的注意力权重 | 标量 |
| $\Delta p_{mqk}$ | 第 m 头第 k 点的采样偏移 | $2$ |

#### (b) 公式来源

这个公式不是凭空出现的，它是对标准 Transformer 注意力的改进。标准注意力公式为：

$$ \text{Attn}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V $$

其中 $Q$ 是所有 query 的集合，$K$ 是所有 key 的集合。这个公式的复杂度是 $O(N^2)$，其中 $N$ 是序列长度。

可变形注意力将其修改为：不再计算所有 query-key 对，而是对每个 query 只关注 K 个位置。这 K 个位置由 query 特征动态预测。

#### (c) 推导过程

**步骤 1：预测采样偏移**

$$ \Delta p_{mqk} = \text{Linear}_m(z_q) $$

其中 $\text{Linear}_m$ 是一个线性层，输入 $z_q$（维度 $C$），输出 $2K$ 个值（每个采样点有 x 和 y 两个偏移量）。

**步骤 2：预测注意力权重**

$$ A_{mqk} = \text{softmax}_k(\text{Linear}_m'(z_q)) $$

其中 $\text{Linear}_m'$ 是一个线性层，输入 $z_q$，输出 $K$ 个 logits，通过 softmax 归一化。

**步骤 3：采样特征**

$$ x(p_q + \Delta p_{mqk}) $$

由于 $\Delta p_{mqk}$ 是连续值，采样点坐标 $(p_q + \Delta p_{mqk})$ 可能落在像素之间。因此使用双线性插值（bilinear interpolation）来获取该位置的特征值。

**步骤 4：加权求和**

$$ \sum_{k=1}^{K} A_{mqk} \cdot W_m' x(p_q + \Delta p_{mqk}) $$

对 K 个采样点的特征进行加权求和，权重为 $A_{mqk}$。

**步骤 5：多头聚合**

$$ \sum_{m=1}^{M} W_m [\text{步骤4的结果}] $$

将所有 M 个头的输出拼接、投影到最终的输出空间。

#### (d) 直觉理解

**为什么是 K 个点而不是更多？**

考虑一个具体例子：假设特征图大小是 100x100 = 10000 个位置。如果每个 query 关注所有 10000 个位置，计算量是巨大的。但一个目标通常只占图像的一小部分（比如 50x50 = 2500 个像素），而且目标内部的语义信息是连续的。因此，用 K=4 个采样点就足以捕获目标的关键特征。

**具体数值示例：**

假设：
- $C = 256$
- $M = 8$
- $K = 4$
- 特征图大小 $H = W = 100$

对于单头（M=1）：
- 标准注意力的计算量：$10000 \times 10000 = 10^8$ 次点积
- 可变形注意力的计算量：$1 \times 4 = 4$ 次采样 + 4 次加权求和

可变形注意力将计算量降低了 2500 万倍（当然，实际中还要考虑多头和 batch 维度，但差距仍然是数量级的）。

#### (e) 边界情况

1. **当 K=1 时**：退化为单点注意力，相当于只关注参考点本身。此时模型失去了聚合上下文信息的能力。
2. **当 K 很大时**：如果 K 接近特征图大小，可变形注意力退化为全局注意力，计算复杂度回到 $O(N^2)$。
3. **当偏移量 $\Delta p$ 很大时**：采样点可能超出特征图边界。论文中使用零填充（zero padding）来处理边界外的采样点。
4. **当参考点 $p_q$ 在图像外部时**：在训练初期可能发生，但通过双线性插值和零填充可以处理。

### 公式 2：多尺度可变形注意力

$$ \text{MSDeformAttn}(z_q, \hat{p}_q, \{x^l\}_{l=1}^L) = \sum_{m=1}^{M} W_m \sum_{l=1}^{L} \sum_{k=1}^{K} A_{mlqk} \cdot W_m' x^l(\phi_l(\hat{p}_q) + \Delta p_{mlqk}) $$

#### (a) 符号定义

新增符号：
| 符号 | 含义 | 维度 |
|------|------|------|
| $\{x^l\}_{l=1}^L$ | L 个尺度的特征图集合 | 每个 $(C, H_l, W_l)$ |
| $\hat{p}_q$ | 归一化的参考点坐标 | $2$，范围 $[0,1]$ |
| $\phi_l$ | 坐标映射函数 | $[0,1]^2 \rightarrow \mathbb{R}^2$ |
| $A_{mlqk}$ | 第 m 头第 l 尺度第 k 点的权重 | 标量 |

#### (b) 公式来源

这是公式 1 的多尺度扩展。主要修改：
1. 将单尺度特征图 $x$ 替换为多尺度特征图集合 $\{x^l\}$
2. 将参考点 $p_q$ 替换为归一化坐标 $\hat{p}_q$
3. 注意力权重增加尺度维度 $l$

#### (c) 推导过程

**坐标映射：**

$$ \phi_l(\hat{p}_q) = \hat{p}_q \odot (H_l, W_l) $$

其中 $\odot$ 是逐元素乘法。例如，如果 $\hat{p}_q = (0.5, 0.5)$，在 level 1 特征图（$H_1=100, W_1=100$）上映射到 $(50, 50)$，在 level 4 特征图（$H_4=12, W_4=12$）上映射到 $(6, 6)$。

**注意力权重归一化：**

$$ \sum_{l=1}^{L} \sum_{k=1}^{K} A_{mlqk} = 1 $$

即所有尺度、所有采样点的权重之和为 1。这是通过对 $L \times K$ 个 logits 做 softmax 实现的。

#### (d) 直觉理解

**为什么要用归一化坐标？**

归一化坐标使得参考点与特征图的分辨率无关。一个目标在图像中的位置是固定的（比如图像中心），但在不同尺度的特征图上，它的像素坐标不同。使用归一化坐标后，模型只需要学习一个统一的参考点表示，然后通过 $\phi_l$ 映射到各个尺度的实际坐标。

**具体数值示例：**

假设图像大小 $800 \times 800$，四个尺度的特征图大小分别为：
- Level 1: $100 \times 100$（步长 8）
- Level 2: $50 \times 50$（步长 16）
- Level 3: $25 \times 25$（步长 32）
- Level 4: $12 \times 12$（步长 64）

对于一个目标中心在 $(400, 400)$ 的 query：
- 归一化坐标 $\hat{p}_q = (0.5, 0.5)$
- Level 1 实际坐标：$(50, 50)$
- Level 2 实际坐标：$(25, 25)$
- Level 3 实际坐标：$(12.5, 12.5)$
- Level 4 实际坐标：$(6, 6)$

模型可以通过注意力权重 $A_{mlqk}$ 决定从哪个尺度采样多少点。对于小目标，可能更关注 Level 1 的采样点；对于大目标，可能更关注 Level 3 或 Level 4 的采样点。

#### (e) 边界情况

1. **当 L=1 时**：退化为单尺度可变形注意力
2. **当所有注意力权重集中在单一尺度时**：实际效果等同于单尺度注意力
3. **当不同尺度的特征图分辨率差距过大时**：高分辨率特征图的采样点可能覆盖非常局部的区域，低分辨率特征图的采样点可能覆盖全局。模型需要平衡这两者。

## 九、位置

### 前驱工作

1. **DETR [Carion et al., 2020]**：Deformable DETR 的直接前驱。DETR 首次将 Transformer 引入目标检测，但收敛慢、小目标检测差。Deformable DETR 针对这两个问题提出了可变形注意力。

2. **可变形卷积 [Dai et al., 2017]**：可变形注意力的核心灵感来源。可变形卷积通过预测卷积核的偏移量来适应目标的形状，证明了"动态采样"在视觉任务中的有效性。

3. **Non-local Networks [Wang et al., 2018]**：将自注意力引入视觉任务的早期工作。可变形注意力可以看作是非局部网络的稀疏化版本。

4. **Sparse Transformer [Child et al., 2019]**：探索了稀疏注意力机制，但使用的是固定模式。Deformable DETR 将稀疏模式改为动态学习的。

### 同期竞品

1. **YOLOv5 [Ultralytics, 2020]**：单阶段检测器的代表作，速度快但依赖手工设计组件。Deformable DETR 证明了端到端检测可以在不牺牲速度的情况下达到 competitive 性能。

2. **EfficientDet [Tan et al., 2020]**：基于 EfficientNet 和 BiFPN 的检测器，在 COCO 上达到 SOTA。但同样依赖锚框和 NMS。

3. **Swin Transformer [Liu et al., 2021]**：提出了移动窗口注意力，虽然主要针对分类任务，但其局部注意力的思想与可变形注意力有相似之处。

### 后续影响

1. **DINO [Zhang et al., 2022]**：直接基于 Deformable DETR 的改进工作，引入了对比学习去噪训练和混合 query 选择，在 COCO 上达到 63.2 AP（使用 Swin-L 骨干）。

2. **DN-DETR [Li et al., 2022]**：在 Deformable DETR 的基础上引入去噪训练（Denoising Training），加速收敛并提升性能。

3. **Group DETR [Chen et al., 2022]**：使用多个 group 的 object queries，每个 group 独立预测，然后通过集合匹配选择最佳结果。这是对 Deformable DETR 中 query 设计的一种改进。

4. **RT-DETR [Lv et al., 2023]**：基于 Deformable DETR 的实时检测器，通过混合编码器（hybrid encoder）和 IoU-aware query selection 实现了实时推理速度。

5. **Mask DINO [Li et al., 2023]**：将 Deformable DETR 扩展到实例分割任务，统一了检测和分割的架构。

### 局限 1：对小目标的检测仍有提升空间

**何时重要：** 在自动驾驶、遥感图像等小目标密集的场景中。

**后续工作：** DINO [Zhang et al., 2022] 通过改进 query 设计和对齐策略，将小目标 AP 提升到 30.0+。

### 局限 2：多尺度特征带来的计算开销

**具体问题：** 多尺度可变形注意力需要同时维护和计算四个尺度的特征图，增加了内存占用和计算量。

**何时重要：** 在资源受限的场景（如移动端、嵌入式设备）中。

**论文是否承认：** 论文在复杂度分析中提到了多尺度带来的额外计算，但未将其列为局限。

**后续工作：** RT-DETR [Lv et al., 2023] 通过混合编码器设计，在保持精度的同时减少了多尺度特征的计算开销。

### 局限 3：两阶段设计的复杂度

**具体问题：** 两阶段版本虽然精度更高，但需要额外的第一阶段预测头，增加了模型复杂度和训练时间。

**何时重要：** 在追求训练效率的场景中。

**论文是否承认：** 论文将两阶段作为可选扩展，未强制使用。

### 局限 4：对 Transformer 架构的依赖

**具体问题：** Deformable DETR 仍然使用标准的 Transformer encoder-decoder 架构，decoder 中的自注意力复杂度仍然是 $O(N_q^2)$（$N_q$ 是 query 数量）。

**何时重要：** 当使用大量 query（如 1000+）时，自注意力的计算量会显著增加。

**后续工作：** Group DETR [Chen et al., 2022] 通过分组 query 来缓解这个问题。

## 十一、小结

Deformable DETR 解决了 DETR 的核心问题——收敛速度慢和计算复杂度高。其核心创新是用可变形注意力替代全局注意力，每个 query 只关注 K 个动态预测的采样点，而不是所有位置。这个看似简单的改动带来了三个关键收益：

1. **收敛速度提升 10 倍**：从 500 epoch 降到 50 epoch
2. **计算复杂度降低**：从 $O(HW \cdot N_q)$ 降到 $O(K \cdot N_q)$
3. **自然地支持多尺度特征**：通过跨尺度采样，小目标检测显著提升

这篇文章的 broader impact 是：它证明了 Transformer-based 的端到端检测器可以既高效又有效，为后续一系列 DETR 变体（DINO、DN-DETR、RT-DETR 等）奠定了基础。可以说，Deformable DETR 是 DETR 系列中承上启下的关键工作——它解决了 DETR 的痛点，同时为后续改进打开了空间。
