**论文：** [arXiv](https://arxiv.org/abs/2202.03052) · 作者：Peng Wang, An Yang, Rui Men, Junyang Lin, Shuai Bai, Zhikang Li, Jianxin Ma, Chang Zhou, Jingren Zhou, Hongxia Yang · 年份：2022

## 一、先搞清楚坑在哪

在 OFA 之前，多模态领域的状态可以用一个词概括：**碎片化**。

想象一下你要做一个 AI 助手，它得能看图说话（image captioning）、根据文字描述找图（text-to-image retrieval）、做视觉问答（VQA）、做视觉推理（visual entailment），还得能理解图表（chart understanding）。在 OFA 之前，你需要为每个任务准备一个独立的模型架构、独立的预训练目标和独立的微调策略。

具体来说，2019-2021 年间的多模态预训练模型虽然都号称"统一"，但它们其实只统一了**视觉语言理解任务**（classification-style tasks），比如 VQA、NLVR2、Retrieval。对于**视觉语言生成任务**（generation tasks），比如 image captioning、visual grounding、text-to-image generation，它们要么需要额外加一个解码器，要么干脆不支持。

更糟糕的是，不同模型使用的架构差异巨大：
- **双塔模型**（如 CLIP）：视觉编码器和文本编码器分开，只在最后做对比学习。适合检索，不适合生成。
- **融合编码器模型**（如 UNITER）：单塔结构，输入是图文拼接。适合理解，生成需要改造。
- **编码器-解码器模型**（如 T5）：原本是纯文本的，扩展成多模态需要加视觉模块。

这个领域缺一个**真正的统一框架**：同一个模型，同一个预训练目标，同一个架构，能处理所有跨模态任务——理解+生成，分类+回归，视觉+语言+多模态。

## 二、现有方案（Task-specific Models）的真正问题

以 2021 年的 SOTA 模型为例：

**UNITER** 用一个 BERT-like 的编码器做图像-文本联合编码。它做预训练时用三个目标：MLM（Masked Language Modeling）、MRM（Masked Region Modeling）、ITM（Image-Text Matching）。微调时，VQA 任务加一个分类头，Retrieval 任务用对比学习，Captioning 任务需要额外加一个解码器。

问题来了：**每个任务都要改模型结构**。

这不是一个理论问题，而是一个实操问题。当你想把模型部署到生产环境时，你需要维护 N 个不同的 checkpoint，N 个不同的推理 pipeline，N 套不同的优化策略。更隐蔽的问题是：**不同任务的预训练目标之间可能存在冲突**。比如 MLM 需要模型做生成式预测（预测 masked token），而 ITM 需要模型做判别式判断（判断图文是否匹配），这两个目标在参数空间里可能互相拉扯。

另一个关键问题是**模态不对齐**。视觉模型用 region features（来自目标检测器），语言模型用 token embeddings（来自 BPE），两种特征的空间不一样，语义粒度也不一样。这导致模型很难在模态间做精确的对应（grounding）。

OFA 的核心洞察是：**把一切任务都变成序列到序列（seq2seq）的生成问题**。这样你只需要一个模型、一个架构、一个预训练目标。

## 三、OFA 的核心思路

OFA 的全称是 **One For All**——一个模型搞定所有任务。

核心思路极其简单：**把所有任务都表示成"序列到序列"的形式**。输入是一个 token 序列（可能是图像 patch、文本 token、或者它们的混合），输出也是一个 token 序列。

这意味着：
- **Image captioning**：输入是图像，输出是文本描述（"a dog playing with a ball"）
- **VQA**：输入是图像+问题，输出是答案（"yes" 或 "no" 或 "blue"）
- **Visual grounding**：输入是图像+文本描述，输出是边界框坐标（"<x1>,<y1>,<x2>,<y2>"）
- **Text-to-image generation**：输入是文本，输出是图像 token 序列
- **Image classification**：输入是图像，输出是类别名称（"cat" 或 "dog"）

注意：**连分类任务都被建模成生成任务**。不是加一个线性分类头，而是让模型生成类别名称的 token 序列。

这听起来可能很反直觉——为什么不直接用分类头？后续的实验会证明，这种统一建模方式虽然在某些分类任务上略低于专用模型，但**跨任务的泛化能力**远强于专用模型，而且**零样本迁移**效果惊人。

## 四、OFA 的网络架构详解

### 4.1 整体架构

OFA 使用的是标准的 **Transformer 编码器-解码器架构**，和 T5 非常相似。但有几个关键区别：

1. 输入可以是图像、文本、或两者的混合
2. 图像经过 ResNet 下采样为 patch 特征，然后投影成 token
3. 所有 token（图像 patch token + 文本 token）共享同一个词表
4. 输出可以是文本 token 序列，也可以是离散化的坐标 token

```text
输入序列: [CLS] 文本 token1 token2 ... [IMG] patch1 patch2 ... [SEP]
                              │
                              ▼
                    ┌─────────────────┐
                    │  Transformer     │
                    │  编码器           │
                    └────────┬────────┘
                             │
                             ▼
                    ┌─────────────────┐
                    │  Transformer     │
                    │  解码器           │
                    └────────┬────────┘
                             │
                             ▼
输出序列: token1 token2 ... [EOS]
```

### 4.2 输入表示

OFA 的输入由三部分组成：

**文本 token**：使用 BPE（Byte Pair Encoding），词表大小 65536。每个 token 被映射为一个 embedding。

**图像 patch token**：图像先经过 ResNet 的前三个 block 下采样为 patch 特征（步长 16，每个 patch 对应 16×16 的原始图像区域），而非 ViT 的线性投影方式。预训练中使用 256×256 分辨率，下采样后得到 16×16 的 patch 网格。

**位置编码**：OFA 的位置编码由两个独立的组件组成。

首先，文本和图像各自有独立的**绝对位置 embedding**，这些 embedding 与 token/patch content embedding 分开存储（即"解耦"）。这使得模型可以在不改变内容表示的情况下独立调节位置信息。

其次，注意力计算中额外加入**相对位置偏置（relative position bias）**——文本使用 1D 偏置（感知 token 之间的一维距离），图像使用 2D 偏置（感知 patch 之间的二维空间关系）。这些偏置帮助模型理解 token/patch 之间的相对位置关系，而无需在输入中插入 marker token。

**特殊 token**：
- `<code_v>`：视觉 code（用于 text-to-image generation）
- `<code_l>`：语言 code（用于 image-to-text generation）
- `<bin_v>`：视觉 bin（用于 grounding 任务中的坐标表示）
- `<obj>`：物体标记（用于 object detection 任务）
- `<region>`：区域标记（用于 region-to-text 任务）

### 4.3 编码器

标准 Transformer 编码器，12 层（base）或 24 层（large），hidden size 768（base）/ 1024（large），12 头注意力（base）/ 16 头注意力（large）。

编码器接收一个**扁平化的 token 序列**。对于图文混合输入，序列的构造方式如下：

```
[CLS] 文本 token1 token2 ... [IMG] patch1 patch2 ... [SEP]
```

其中 `[IMG]` 是一个特殊 token，表示图像序列的开始。所有 patch token 紧跟在 `[IMG]` 之后。

编码器输出的 hidden states 包含了图像和文本的联合表示。

### 4.4 解码器

标准 Transformer 解码器，和编码器层数相同。解码器使用 **causal attention mask**（只能看到当前位置之前的 token），通过自回归方式生成输出序列。

解码器的输入是目标序列（shifted right），即：
```
[BOS] token1 token2 ... tokenN
```

解码器的输出是：
```
token1 token2 ... tokenN [EOS]
```

### 4.5 前向过程

以 VQA 任务为例：

1. **输入处理**：图像缩放到 480×480，分割成 30×30 = 900 个 16×16 patch。文本问题被 BPE 分词成 token 序列。
2. **序列构造**：`[CLS] question_tokens [IMG] patch1 ... patch900 [SEP]`
3. **编码器前向**：序列经过编码器，得到 1+len(question)+1+900+1 = 903+ 个 hidden states
4. **解码器前向**：解码器接收 `[BOS] answer_tokens`，自回归生成答案
5. **输出**：生成 `answer_tokens [EOS]`

### 4.6 为什么用 seq2seq 而不用 encoder-only？

这是一个关键的设计选择。

Encoder-only 模型（如 BERT）做分类任务很自然——在 `[CLS]` token 上加一个线性分类头就行。但做生成任务时，它需要额外加一个解码器，或者用 MLM head 做填空式生成。后者在生成较长序列时效果很差。

Encoder-decoder 模型天然支持生成，而且可以通过 **prefix language modeling** 来支持理解任务——把输入放在编码器，输出放在解码器，用交叉注意力让解码器看到输入。这样，同一个模型既能做理解（通过生成答案 token），也能做生成（通过生成目标 token）。

## 五、核心创新点

### 5.1 统一任务表示（Task-Agnostic Representation）

**痛点与动机**

在 OFA 之前，每个任务都有自己独特的输出格式。VQA 输出一个答案（分类），Image Captioning 输出一个句子（生成），Visual Grounding 输出边界框（回归）。这导致模型需要为每个任务设计不同的输出头。

更根本的问题是：**不同任务的输出空间不一致**。分类任务的输出空间是固定的类别集合，生成任务的输出空间是开放的词表，回归任务的输出空间是连续的坐标值。这让跨任务迁移变得困难。

**方案细节**

OFA 的核心思想是：**把所有任务的输出都离散化成 token 序列**。

具体做法：
1. **分类任务**：输出类别名称的 token 序列。比如 ImageNet 分类，输出"goldfish"而不是第 1 类。
2. **回归任务**：把连续值离散化成 bin。比如边界框坐标 `(x1, y1, x2, y2)` 被离散化成 `<bin_0>` 到 `<bin_999>` 的 token。
3. **生成任务**：直接输出文本 token 序列。

**为什么有效**

这个设计有两个关键好处：

第一，**模型只需要学习一个输出分布**，即词表上的概率分布。无论是分类、回归还是生成，输出都是词表中的一个 token 序列。这让预训练目标可以统一为 **cross-entropy loss**。

第二，**不同任务的知识可以共享**。比如在 VQA 任务中学到的"蓝色"概念，可以直接迁移到 Image Captioning 任务中。在 Visual Grounding 任务中学到的空间关系，可以用于 Region-to-Text 任务。

**与 Related Work 的关系**

之前的工作如 VL-T5 和 UNICORN 也尝试过用 seq2seq 做多模态任务，但它们没有做到**完全统一**。VL-T5 仍然为不同任务设计了不同的输出格式（有些用特殊 token，有些用分类头）。OFA 是第一个**把所有任务（包括分类和回归）都统一成纯文本生成**的模型。

**如果去掉会怎样**

从 ablation study 来看，如果改用分类头做分类任务，在 ImageNet 上准确率会提高约 2%（因为分类头可以更好地利用类别之间的层次关系），但**跨任务迁移能力会大幅下降**。具体来说，在 zero-shot VQA 任务上，用分类头的版本准确率下降约 15%。这说明统一表示虽然牺牲了一点单任务性能，但换来了强大的泛化能力。

### 5.2 统一的预训练目标（Unified Pre-training Objective）

**痛点与动机**

之前的预训练模型通常使用多个目标：MLM + MRM + ITM + ... 每个目标对应一个 loss，需要手动调整权重。这不仅增加了训练复杂度，还可能导致目标冲突（比如 MLM 鼓励生成式预测，ITM 鼓励判别式判断）。

**方案细节**

OFA 使用**单一的预训练目标**：**去噪语言建模（Denoising Language Modeling）**。

具体来说，在预训练阶段：
1. 从图文对中构造一个"损坏"的序列
2. 让模型去预测被损坏的部分

损坏的方式有三种：
- **文本去噪**：随机 mask 掉 15% 的文本 token，让模型预测
- **图像去噪**：随机 mask 掉 15% 的图像 patch token，让模型预测
- **图文去噪**：随机 mask 掉图文对中的一部分，让模型预测

所有损坏方式都使用同一个 cross-entropy loss。

**为什么有效**

这个设计借鉴了 T5 的 denoising objective。关键洞察是：**不同的预训练目标（MLM、MRM、ITM）本质上都是"预测被损坏的内容"**。MLM 预测被 mask 的文本 token，MRM 预测被 mask 的图像 patch，ITM 预测图文是否匹配（可以理解为预测一个特殊的 token）。

把它们统一成一个目标，不仅简化了训练流程，还避免了目标冲突。更重要的是，**模型学会了在所有模态上进行生成式预测**，这为后续的跨模态生成任务打下了基础。

**与 Related Work 的关系**

T5 首次提出了用 denoising objective 统一 NLP 任务。SimVLM 把类似的思想扩展到多模态领域，但它使用 prefix language modeling 而不是 denoising。OFA 的 denoising objective 更接近 T5，但扩展到了多模态输入。

**如果去掉会怎样**

Ablation 实验显示，如果改用多个预训练目标（MLM+MRM+ITM），模型在 VQA 上的准确率下降约 3%，在 Image Captioning 上的 CIDEr 分数下降约 5%（该结果来自论文完整消融实验，详见论文 Table 10）。这说明统一的预训练目标不仅简化了训练，还提高了性能。

### 5.3 模态无关的表示空间（Modality-Agnostic Representation）

**痛点与动机**

之前的模型通常为不同模态使用不同的表示空间：图像用 region features（来自 Faster R-CNN），文本用 BPE token embeddings。这导致两个问题：
1. **特征不对齐**：region features 是 2048 维的向量，token embeddings 是 768 维的，无法直接比较
2. **语义粒度不一致**：一个 region 可能对应多个物体或一个物体的一部分，而一个 token 对应一个子词

**方案细节**

OFA 使用**统一的表示空间**：
1. 图像经过 ResNet 前三个 block 下采样为 patch 特征，每个 patch 被投影成和文本 token 相同维度的 embedding
2. 所有 token（文本 token + 图像 patch token）共享同一个 embedding 矩阵
3. 特殊 token（如 `<bin_v>`、`<code_l>`）也在同一个词表中

**为什么有效**

统一表示空间的关键好处是：**模型可以自由地在不同模态之间进行转换**。比如在 text-to-image generation 任务中，模型需要把文本 token 序列映射到图像 patch token 序列。如果两种 token 不在同一个表示空间，这个映射就几乎不可能学习。

此外，共享 embedding 矩阵大大减少了参数量（相比为每个模态单独使用一个 embedding 矩阵）。

**与 Related Work 的关系**

ViT 首次证明了可以用 patch embedding 替代 region features。BEiT 和 MAE 进一步证明了 patch embedding 可以用于生成式预训练。OFA 把 patch embedding 和文本 token embedding 放在同一个空间，这是之前工作没有做的。

**如果去掉会怎样**

如果为图像和文本使用不同的 embedding 层，参数量会增加约 15%（因为需要额外的 embedding 矩阵），而性能在跨模态生成任务上下降约 8%（因为模型需要额外学习两个空间之间的映射）。

## 六、公式详解

### 6.1 预训练目标：Denoising Language Modeling

**符号定义**

- $x$ ：原始输入序列（包含文本 token 和图像 patch token）
- $\tilde{x}$ ：损坏后的序列（部分 token 被 mask 掉）
- $y$ ：目标序列（被 mask 掉的 token 序列）
- $\theta$ ：模型参数
- $p_\theta(y|\tilde{x})$ ：模型在给定损坏序列 $\tilde{x}$ 时生成目标序列 $y$ 的概率

**公式**

$$
\mathcal{L}(\theta) = -\mathbb{E}_{(x,y) \sim \mathcal{D}} \left[ \log p_\theta(y | \tilde{x}) \right]
$$

**推导过程**

这个公式就是标准的 seq2seq 负对数似然。具体展开：

$$
\log p_\theta(y | \tilde{x}) = \sum_{t=1}^{|y|} \log p_\theta(y_t | y_{<t}, \tilde{x})
$$

其中 $y_t$ 是目标序列的第 $t$ 个 token， $y_{<t}$ 是前 $t-1$ 个 token。

**直觉理解**

这个公式的意思是：给定损坏的序列 $\tilde{x}$ ，模型需要逐个预测被 mask 掉的 token。每个预测都是条件概率——基于之前已经预测出的 token 和输入的损坏序列。

**具体例子**

假设输入是："a [MASK] playing with a ball"（文本）加上一张狗的图像（patch token）。目标序列是："dog"。

模型需要计算：
- $p(\text{"dog"} | \tilde{x})$ ：在给定损坏序列时，"dog" 的概率
- 如果目标序列更长，比如 "a dog playing"，则：
  - $p(\text{"a"} | \tilde{x})$
  - $p(\text{"dog"} | \text{"a"}, \tilde{x})$
  - $p(\text{"playing"} | \text{"a dog"}, \tilde{x})$

### 6.2 跨模态注意力

**符号定义**

- $Q$ ：解码器当前层的 query 矩阵，形状为 $(L_d, d)$
- $K$ ：编码器输出的 key 矩阵，形状为 $(L_e, d)$
- $V$ ：编码器输出的 value 矩阵，形状为 $(L_e, d)$
- $L_d$ ：解码器序列长度
- $L_e$ ：编码器序列长度
- $d$ ：hidden size

**公式**

$$
\text{CrossAttn}(Q, K, V) = \text{softmax}\left(\frac{Q K^T}{\sqrt{d}}\right) V
$$

**推导过程**

标准的 attention 公式。 $Q K^T$ 计算每个解码器位置对所有编码器位置的注意力分数，除以 $\sqrt{d}$ 进行缩放（防止 softmax 进入饱和区），softmax 归一化成概率，最后用概率加权求和得到 context vector。

**直觉理解**

解码器在生成每个 token 时，需要"看"编码器输出的哪些部分。比如生成 "dog" 时，解码器应该关注图像中狗的区域和文本中 "a [MASK] playing" 部分。

**具体例子**

假设编码器输出 100 个 token（10 个文本 token + 90 个图像 patch token），解码器正在生成第 5 个 token。 $Q$ 是第 5 个位置的 query， $K$ 是所有 100 个位置的 key。注意力分数是一个 100 维的向量，softmax 后得到每个编码器位置的重要性权重。如果模型正确学习，狗的图像 patch 的权重应该最高。

### 6.3 坐标离散化

**符号定义**

- $x$ ：归一化的横坐标，范围 $[0, 1]$
- $y$ ：归一化的纵坐标，范围 $[0, 1]$
- $w$ ：归一化的宽度，范围 $[0, 1]$
- $h$ ：归一化的高度，范围 $[0, 1]$
- $B$ ：bin 的数量（OFA 使用 $B=1000$ ）

**公式**

$$
\text{bin}(x) = \lfloor x \cdot (B-1) \rfloor
$$

$$
\text{bin}(y) = \lfloor y \cdot (B-1) \rfloor
$$

$$
\text{bin}(w) = \lfloor w \cdot (B-1) \rfloor
$$

$$
\text{bin}(h) = \lfloor h \cdot (B-1) \rfloor
$$

**推导过程**

这是一个简单的线性量化。将连续值 $[0, 1]$ 映射到离散的 bin 索引 $[0, B-1]$ 。

**直觉理解**

坐标是连续值，但模型只能输出离散的 token。通过量化，我们把连续空间分成 1000 个 bin，每个 bin 对应一个 token（`<bin_0>` 到 `<bin_999>`）。模型的任务是预测正确的 bin 索引。

**具体例子**

如果边界框的左上角在图像的 (0.3, 0.5) 位置：
- $\text{bin}(x) = \lfloor 0.3 \cdot 999 \rfloor = \lfloor 299.7 \rfloor = 299$
- $\text{bin}(y) = \lfloor 0.5 \cdot 999 \rfloor = \lfloor 499.5 \rfloor = 499$

模型需要输出 token `<bin_299>` 和 `<bin_499>`。

**边界情况**

当 $x=1.0$ 时： $\text{bin}(x) = \lfloor 1.0 \cdot 999 \rfloor = 999$ ，这正是最后一个 bin。

当 $x=0.0$ 时： $\text{bin}(x) = 0$ ，第一个 bin。

量化误差最大为 $1/(B-1) \approx 0.001$ ，对于 480×480 的图像，相当于约 0.5 像素的误差，完全可以接受。

## 八、代码对照

OFA 的官方代码在 GitHub 上（https://github.com/OFA-Sys/OFA）。以下是一些关键实现细节：

### 8.1 模型定义

OFA 使用基于 BART 初始化权重的**自定义 Transformer**（对标准 Transformer 做了若干修改），核心修改包括：

1. **输入处理**：文本通过 BPE 编码，图像通过 ResNet 前三个 block 下采样为 patch feature，两者在序列维度拼接。

2. **位置编码**：使用两组解耦的绝对位置 embedding（文本 1D + 图像 2D），加上 1D/2D relative position bias。

3. **注意力 mask**：编码器使用双向注意力（bidirectional attention），解码器使用因果注意力（causal attention）。

### 8.2 训练细节

- **优化器**：AdamW，峰值学习率 2e-4，warmup ratio 0.01
- **batch size**：2048（base）/ 1024（large），使用梯度累积
- **训练数据**：约 2000 万图文对（来自 CC3M、CC12M、SBU Captions、COCO 等）
- **图像尺寸**：256×256（预训练），480×480（某些下游任务）
- **训练步数**：约 30 万步（base）/ 50 万步（large）
- **硬件**：128 块 V100 GPU（large）

### 8.3 推理细节

- **解码策略**：beam search（beam size = 5）用于生成任务，greedy decoding 用于分类任务
- **坐标解码**：输出 token 序列中的 `<bin_X>` 被解码回坐标值：`x = (bin_index + 0.5) / 1000`
- **分类解码**：模型生成类别名称的 token 序列，然后映射回类别 ID

### 8.4 代码中的技巧

1. **图像缩放策略**：不同任务使用不同的图像尺寸。预训练用 256×256，某些下游任务用 480×480。代码中通过 `patch_image_size` 参数控制。

2. **数据增强**：使用 RandAugment 进行图像增强，包括随机裁剪、颜色抖动等。

3. **标签平滑**：使用 0.1 的标签平滑（label smoothing），防止模型过拟合。

4. **梯度裁剪**：最大梯度范数 1.0，防止梯度爆炸。

## 十、局限

### 10.1 分辨率限制

OFA 使用固定的图像尺寸（480×480），这对高分辨率图像（如医疗图像、卫星图像）不友好。如果图像包含小物体，patch 尺寸（16×16）可能导致信息丢失。

**论文是否承认**：论文没有明确讨论这个限制。

**后续工作**：BEiT-3 和 PaLI 使用了多尺度训练策略来缓解这个问题。

### 10.2 坐标离散化精度

使用 1000 个 bin 来离散化坐标，对于 480×480 的图像，量化误差约 0.5 像素。这对于大多数任务已经足够，但对于需要像素级精度的任务（如语义分割），这个误差可能不可接受。

**论文是否承认**：论文提到坐标离散化是一个设计选择，但没有讨论精度问题。

**后续工作**：MDETR 和 DETR 使用连续坐标回归（通过 MLP head），避免了离散化误差。

### 10.3 生成式分类的效率

用生成式方式做分类（生成类别名称）比用分类头慢得多。对于 ImageNet 的 1000 类分类，模型需要生成最多 10 个 token（如 "goldfish" 是 8 个 token），而分类头只需要一次前向传播。

**论文是否承认**：论文承认生成式分类的速度较慢，但认为统一建模的好处大于效率损失。

**后续工作**：一些后续工作（如 PaLI）在分类任务上使用分类头，只在生成任务上使用生成式方式。

OFA 在长文本生成任务上有一定局限。论文除了 COCO Captioning，也在 Gigaword 摘要生成上做了评估且表现良好，但总体而言，OFA 的设计更偏向中等长度的跨模态生成任务（如图文描述、问答），在极长文本生成上仍有改进空间。

**后续工作**：Flamingo 和 BLIP-2 通过使用更大的语言模型（7B-70B 参数）来解决长文本生成问题。

## 十一、小结

OFA 的核心贡献是：**用一个模型、一个架构、一个预训练目标，统一了视觉、语言和多模态领域的几乎所有任务**。

它做到了之前没人做到的事情：
- 同一个模型可以做 VQA、Image Captioning、Visual Grounding、Image Classification、Object Detection、Text-to-Image Generation 等 30+ 个任务
- 不需要为每个任务修改模型结构
- 不需要为每个任务设计不同的预训练目标
- 在几乎所有任务上都达到了 SOTA 或接近 SOTA 的水平

OFA 的影响是深远的。它证明了"统一"不仅是可能的，而且是有益的——统一的模型比分散的专用模型泛化能力更强，尤其是在零样本迁移场景下。

在 OFA 之后，多模态领域进入了一个"统一"的时代。BEiT-3、PaLI、Flamingo 等后续工作都在 OFA 的框架基础上进行了扩展和改进。OFA 的思路——把一切任务都变成序列生成——已经成为多模态模型设计的标准范式。
