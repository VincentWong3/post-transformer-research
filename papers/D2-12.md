# Visual Instruction Tuning（2023）

**论文：** [arXiv](https://arxiv.org/abs/2304.08485) · Haotian Liu, Chunyuan Li, Qingyang Wu, Yong Jae Lee · 2023

---

## 一、先搞清楚坑在哪

大语言模型（LLM）在文本任务上已经强得离谱了——GPT-4、LLaMA、Vicuna 这些模型能写代码、做推理、跟你聊天。但问题在于：**它们看不见东西**。你给它一张图，它只能读图下面的文字描述，完全不知道图上到底有什么。

那能不能把视觉能力塞进 LLM 里？直觉上很简单：搞个视觉编码器（比如 CLIP 的 ViT）把图变成向量，然后拼到 LLM 的输入里不就行了？

但这里有个坑：**LLM 和视觉编码器的表示空间是割裂的**。LLM 理解的是 token 序列（词或子词），视觉编码器输出的是图像 patch 特征。这两者之间没有天然的对应关系。强行拼接的话，LLM 根本"看不懂"视觉特征。

## 二、已有方法的真正问题

在 LLaVA 之前，已经有不少尝试：

1. **Flamingo [Alayrac et al., 2022]**：用交叉注意力层把视觉特征注入 LLM，但需要海量图文对数据（2.1B 样本）和复杂的训练流程。

2. **BLIP-2 [Li et al., 2023]**：引入 Q-Former 作为视觉和语言之间的桥接模块，但 Q-Former 本身是个 Transformer，参数量不小。

3. **Kosmos-1 [Huang et al., 2023]**：从零开始训练多模态模型，计算成本极高。

这些方法的核心问题可以总结为三点：

- **数据成本高**：需要收集海量（图文对）数据，Flamingo 用了 2.1B 样本，这对学术实验室来说不现实。
- **架构复杂**：每个方法都引入大量新参数和复杂模块（交叉注意力、Q-Former 等）。
- **对齐效果差**：即使有桥接模块，LLM 对视觉输入的"理解"仍然很浅——它知道图上有"一只猫"，但不知道猫在做什么、表情什么样、周围环境如何。

## 三、LLaVA 的核心思路

LLaVA 的核心洞察极其简单：**把视觉特征投影到 LLM 的 token 空间，然后就像处理文本 token 一样处理它们。**

```
输入图像 → 视觉编码器(CLIP ViT-L/14) → 视觉特征 → 投影矩阵(W) → 视觉token → LLM(LLaMA/Vicuna)
输入文本 → 分词器 → 文本token ──────────────┘
```

关键创新只有两点：

1. **一个简单的线性投影层**：把视觉编码器的输出特征映射到 LLM 的 embedding 空间。这层只有几百万参数，相比 LLM 的几十亿参数几乎可以忽略不计。

2. **用 GPT-4 自动生成指令跟随数据**：与其人工标注（成本高、规模小），不如让 GPT-4 "看图说话"——给它一个文本描述，让它基于这个描述生成各种问题和答案。

这两个设计选择让 LLaVA 做到了：
- **极低的训练成本**：8 张 A100 约 1 小时完成训练
- **超强的少样本能力**：在 ScienceQA 上达到 90.92% 准确率，比 GPT-3.5 的 75.17% 高出 15 个百分点
- **数据效率极高**：仅用 158K 图文对数据就达到了 SOTA

## 四、方法详解

### 4.1 整体架构

LLaVA 的架构图如下（图 1 原文）：

```
┌─────────────────────────────────────────────────────────────────────┐
│  LLaVA: Large Language and Vision Assistant                         │
│                                                                      │
│  ┌──────────────┐    ┌──────────┐    ┌─────────────────────────┐    │
│  │  Visual       │    │  Project │    │  Language Model          │    │
│  │  Encoder      │    │  Matrix  │    │  (LLaMA/Vicuna)          │    │
│  │  (CLIP ViT)   │    │  W       │    │                          │    │
│  │              │    │          │    │  ┌───────────────────┐  │    │
│  │  Image ──►   │──► │  Z_v ──► │──► │  │  X + Z_v          │  │    │
│  │  Patch       │    │          │    │  │  ──────────► LLM  │  │    │
│  │  Features    │    │  ℝ^{D₁}  │    │  │  + Text Tokens    │  │    │
│  │              │    │  → ℝ^{D₂}│    │  └───────────────────┘  │    │
│  └──────────────┘    └──────────┘    └─────────────────────────┘    │
└─────────────────────────────────────────────────────────────────────┘
```

**前向过程（从输入到输出）：**

1. **图像编码**：输入图像 $X_v$，经过 CLIP ViT-L/14 视觉编码器，输出 $Z_v = \text{CLIP}(X_v) \in \mathbb{R}^{256 \times 1024}$（256 个 patch 特征，每个 1024 维）。

2. **投影**：线性投影矩阵 $W \in \mathbb{R}^{1024 \times 4096}$ 将视觉特征映射到 LLM 的 embedding 空间：$H_v = Z_v W \in \mathbb{R}^{256 \times 4096}$。

3. **序列拼接**：将视觉 token $H_v$ 和文本 token $H_q$ 拼接成单个序列：$[H_v, H_q]$。

4. **LLM 推理**：拼接后的序列送入 LLaMA/Vicuna，自回归生成回答。

### 4.2 核心创新点一：简单投影层

#### (a) 痛点与动机

**问题**：视觉特征和语言特征处于不同的表示空间。CLIP 的输出是 1024 维的视觉特征，而 LLaMA 的 token embedding 是 4096 维的文本特征。直接拼接会导致 LLM 无法理解视觉信息。

**为什么现有方法不行**：
- Flamingo 用交叉注意力层——每次推理都需要计算视觉和语言之间的注意力，计算量大。
- BLIP-2 用 Q-Former——需要额外训练一个 Transformer，参数量达 188M。

**动机**：既然 LLM 本身已经很强了，我们能不能用一个极简的桥接模块，让 LLM "自己学会" 理解视觉特征？

#### (b) 方案细节

```python
class ProjectionLayer(nn.Module):
    def __init__(self, input_dim=1024, output_dim=4096):
        super().__init__()
        self.linear = nn.Linear(input_dim, output_dim, bias=True)
    
    def forward(self, x):
        # x: [batch, num_patches, 1024]
        return self.linear(x)  # [batch, num_patches, 4096]
```

就是这么简单。一个线性层，没有激活函数，没有归一化，没有残差连接。

#### (c) 为什么有效

直觉上，这个投影层的作用是"翻译"——把视觉特征的语言含义提取出来。因为 CLIP 本身就是在图文对上训练的，它的视觉特征已经包含了丰富的语义信息（比如"红色"、"猫"、"坐姿"）。线性投影只需要找到这些语义在 LLM 空间中的对应位置。

实验证明（论文 Table 2）：
- 用线性投影层训练 1 个 epoch：ScienceQA 准确率 89.53%
- 用两层 MLP（隐藏层 4096）训练 1 个 epoch：89.66%
- 差异只有 0.13%，说明线性层足够了

#### (d) 与 Related Work 的关系

- **Flamingo [Alayrac et al., 2022]**：使用交叉注意力层（参数量 ~12M），需要冻结 LLM 训练。LLaVA 的线性层（参数量 ~4M）更轻量且不需要冻结 LLM。
- **BLIP-2 [Li et al., 2023]**：使用 Q-Former（参数量 ~188M），需要两阶段训练。LLaVA 的线性层可以端到端训练。
- **MiniGPT-4 [Zhu et al., 2023]**：同期工作，使用相同的线性投影思路，但训练数据不同。

#### (e) 如果去掉会怎样

论文没有做"去掉投影层"的消融（因为那等于让 LLM 直接看 raw CLIP 特征，维度不匹配）。但可以做"随机投影"的对比：
- 如果投影矩阵初始化为随机（不训练）：LLM 无法理解视觉信息，回答质量极差（作者在论文中提及）。

### 4.3 核心创新点二：GPT-4 自动生成指令数据

#### (a) 痛点与动机

**问题**：现有的视觉指令数据（如 COCO Captions）只有简单的描述（"一个男人在打篮球"），缺乏复杂的推理和多轮对话能力。

**为什么现有方法不行**：
- 人工标注：成本高（每张图需要多个问题和答案），规模小（最多几千条）。
- 模板生成：缺乏多样性，模型容易过拟合。

**动机**：GPT-4 已经能理解图像了（通过文本描述），能不能用它来自动生成高质量的指令数据？

#### (b) 方案细节

**两步数据生成流程**：

1. **收集图文对**：从 COCO 数据集（2014/2017 训练集）中随机选取 158K 张图像，使用 BLIP-2 生成详细描述。

2. **用 GPT-4 生成指令**：给 GPT-4 提供以下上下文：
   ```
   用户：<图像描述> 基于此描述，生成一个问题和对应的答案。
   GPT-4：<生成的问题和答案>
   ```

**生成的数据类型**：
- **对话式**：多轮问答，模拟真实对话
- **详细描述**：要求模型描述图像细节
- **复杂推理**：需要模型进行逻辑推理（"为什么这个人看起来开心？"）

**最终数据集**：158K 张图像，每张图像生成约 10 个问答对，总计约 1.5M 条数据。

#### (c) 为什么有效

这里有个关键洞察：**GPT-4 生成的指令数据比人工标注的更适合训练 LLM**。原因有三：

1. **风格一致**：GPT-4 生成的问答风格与 LLM 的训练数据（ShareGPT、Alpaca 等）一致，减少了风格迁移的困难。

2. **难度可控**：可以通过 prompt 控制问题的难度，从简单的"描述图像"到复杂的"推理分析"。

3. **覆盖全面**：GPT-4 能生成各种类型的指令，包括开放性问题（"你觉得这幅图怎么样？"），这是人工标注很难做到的。

实验证明（论文 Table 3）：
- 使用 158K 数据训练：ScienceQA 90.92%
- 使用 40K 数据训练：89.66%（只下降 1.26%）
- 说明数据效率很高，少量数据就能达到不错的效果

#### (d) 与 Related Work 的关系

- **Flamingo**：使用 2.1B 图文对数据，但数据质量参差不齐。
- **BLIP-2**：使用 129M 图文对数据，但需要复杂的训练流程。
- **LLaVA**：仅用 158K 数据就达到了可比甚至更好的效果，核心在于数据质量而非数量。

#### (e) 如果去掉会怎样

消融实验（论文 Table 3）：
- 使用 CC-595K（Conceptual Captions 595K 条数据）：ScienceQA 89.53%
- 使用 LLaVA-158K：90.92%
- 差异 1.39%，说明 GPT-4 生成的高质量指令数据确实有效

## 五、公式详解

### 5.1 训练损失

**公式**：
$$L = -\sum_{t=1}^{T} \log p(y_t | X_v, X_q, y_{<t})$$

#### (a) 符号定义

- $X_v$：输入图像
- $X_q$：输入文本（问题/指令）
- $y_t$：第 $t$ 个输出 token
- $y_{<t}$：前 $t-1$ 个输出 token
- $T$：输出序列长度
- $p$：LLM 预测的概率分布

#### (b) 公式来源

这是标准的自回归语言模型训练损失，与 LLaMA 的训练目标一致。LLaVA 没有改变 LLM 的训练方式，只是改变了输入格式。

#### (c) 推导过程

1. 输入序列：$[H_v, H_q]$，其中 $H_v$ 是视觉 token（256 个），$H_q$ 是文本 token（长度可变）。
2. LLM 自回归生成：$y_1, y_2, ..., y_T$。
3. 每个时间步 $t$，LLM 基于之前的所有 token 预测下一个 token：$p(y_t | [H_v, H_q], y_1, ..., y_{t-1})$。
4. 损失函数是负对数似然：$-\log p(y_t | ...)$。
5. 对所有时间步求和：$L = -\sum_{t=1}^{T} \log p(y_t | ...)$。

#### (d) 直觉理解

这个损失函数告诉 LLM：**对于给定的图像和问题，尽可能准确地预测回答中的每个词**。

举个例子：
- 图像：一只猫坐在窗台上
- 问题："猫在做什么？"
- 回答："猫在窗台上晒太阳"
- 损失函数会惩罚 LLM 在第一个位置输出"狗"（因为正确答案是"猫"），在第二个位置输出"在"（正确），等等。

#### (e) 边界情况

- **T=1**：单 token 回答（如"是/否"问题），损失退化为二元分类。
- **长序列**：T 很大时，梯度可能会消失/爆炸，但 LLM 的归一化技术（RMSNorm、残差连接）缓解了这个问题。
- **训练/推理不一致**：训练时使用 teacher forcing（用真实 token 作为输入），推理时使用自回归（用预测 token 作为输入）。这是标准做法，不是 LLaVA 特有的问题。

### 5.2 投影层参数量

**公式**：
$$P_{\text{proj}} = D_1 \times D_2 + D_2$$

#### (a) 符号定义

- $D_1 = 1024$：CLIP ViT-L/14 的输出维度
- $D_2 = 4096$：LLaMA 的 token embedding 维度
- $P_{\text{proj}}$：投影层的参数量

#### (b) 公式来源

线性层的标准参数量计算：权重矩阵 $W \in \mathbb{R}^{D_1 \times D_2}$ + 偏置向量 $b \in \mathbb{R}^{D_2}$。

#### (c) 推导过程

$$P_{\text{proj}} = 1024 \times 4096 + 4096 = 4,194,304 + 4,096 = 4,198,400$$

#### (d) 直觉理解

这个参数量（~4.2M）相比于 LLaMA-7B（7,000M 参数）只占 0.06%。换句话说，**投影层的参数量比 LLM 的 0.1% 还小**。

这意味着：
- 训练开销极小：反向传播时，投影层的梯度计算量可以忽略不计。
- 存储开销极小：投影层的权重只有 ~16MB（FP16）。

#### (e) 边界情况

- **D1 变化**：如果使用不同的视觉编码器（如 ViT-H，D1=1280），参数量会变化。
- **D2 变化**：如果使用更大的 LLM（如 LLaMA-13B，D2=5120），参数量会增加到 ~5.2M。
- **线性层 vs MLP**：论文实验了 MLP（隐藏层 4096），参数量为 $1024 \times 4096 + 4096 \times 4096 + 4096 \approx 21M$，是线性层的 5 倍，但效果提升有限（0.13%）。

## 六、实验结果

### 6.1 ScienceQA 结果

| 方法 | 准确率 | 训练数据量 | 参数量(投影层) |
|------|--------|------------|----------------|
| GPT-3.5 (text-only) | 75.17% | - | - |
| GPT-3.5 (text + image) | 78.29% | - | - |
| LLaVA (LLaMA-7B) | 89.53% | 158K | 4.2M |
| LLaVA (Vicuna-7B) | 90.92% | 158K | 4.2M |
| LLaVA (Vicuna-13B) | 92.53% | 158K | 5.2M |

**关键发现**：
- LLaVA 在 ScienceQA 上比 GPT-3.5 高出 12-15 个百分点，证明视觉信息确实对科学推理有帮助。
- 从 LLaMA 换到 Vicuna（经过指令微调的版本），准确率提升 1.39%，说明 LLM 的指令跟随能力很重要。
- 从 7B 换到 13B，准确率提升 1.61%，说明更大的 LLM 能更好理解视觉信息。

### 6.2 数据量消融

| 训练数据 | 数据量 | ScienceQA 准确率 |
|----------|--------|------------------|
| CC-595K (Conceptual Captions) | 595K | 89.53% |
| LLaVA-40K | 40K | 89.66% |
| LLaVA-158K | 158K | 90.92% |

**关键发现**：
- 只用 40K 数据（CC-595K 的 6.7%）就达到了可比效果，说明数据质量比数量重要。
- 从 40K 到 158K，准确率只提升 1.26%，说明数据量不是瓶颈。

### 6.3 多轮对话评估

论文还做了多轮对话评估（定性）：

```
用户：这张图里有什么？
LLaVA：这是一张城市街道的照片。画面中有一栋高层建筑，建筑外墙上有很多窗户。街道上有几辆车停在路边。
用户：建筑是什么颜色的？
LLaVA：建筑的外墙主要是浅色（可能是米白色或浅灰色），窗户的边框是深色（可能是深灰色或黑色）。
```

**关键发现**：LLaVA 能保持多轮对话的上下文一致性，说明视觉 token 和文本 token 的混合序列工作良好。

## 七、位置

### 7.1 前驱工作

1. **LLaMA [Touvron et al., 2023]**：提供基础 LLM 架构。LLaVA 直接使用 LLaMA-7B 作为语言模型。
2. **Vicuna [Chiang et al., 2023]**：对 LLaMA 进行指令微调，提升了对话能力。LLaVA 的第二个版本使用 Vicuna。
3. **CLIP [Radford et al., 2021]**：提供视觉编码器。CLIP ViT-L/14 的视觉特征已经包含了丰富的语义信息。
4. **BLIP-2 [Li et al., 2023]**：提供图像描述生成工具。LLaVA 使用 BLIP-2 生成训练数据的图像描述。

### 7.2 同期竞品

1. **MiniGPT-4 [Zhu et al., 2023]**：几乎同时发布，使用相同的线性投影思路，但训练数据不同（使用更复杂的对齐数据）。
2. **InstructBLIP [Dai et al., 2023]**：使用 Q-Former 架构，但改进了指令跟随能力。
3. **Otter [Li et al., 2023]**：基于 Flamingo 架构，使用更丰富的数据集。

### 7.3 后续影响

LLaVA 的简洁性和高效性影响了大量后续工作：

1. **LLaVA-1.5 [Liu et al., 2024]**：直接扩展，引入 MLP 投影层（从线性层改为两层 MLP），使用更高分辨率的图像，在多个基准上达到 SOTA。
2. **LLaVA-NeXT [Liu et al., 2024]**：进一步改进，支持任意分辨率输入，使用动态高分辨率策略。
3. **LLaVA-Phi [Zhu et al., 2024]**：将 LLM 替换为 Phi-3（3.8B 参数），在移动设备上实现多模态能力。
4. **CogVLM [Wang et al., 2023]**：引入视觉专家模块，但核心思想（投影视觉特征到 LLM 空间）与 LLaVA 一致。
5. **Qwen-VL [Bai et al., 2023]**：使用类似的投影层设计，但结合了 Q-Former 和交叉注意力。

## 八、局限

1. **视觉分辨率限制**：CLIP ViT-L/14 处理 336×336 的图像，细节信息（如小文字、小物体）可能丢失。后续工作（LLaVA-1.5）通过高分辨率输入部分解决了这个问题。

2. **图像描述依赖**：训练数据依赖 BLIP-2 生成的图像描述，如果描述有误，GPT-4 生成的指令也会出错。例如，BLIP-2 可能把"猫"描述成"狗"，导致 GPT-4 生成关于狗的问题。

3. **单图输入**：LLaVA 设计为单图输入，无法处理视频或多图场景。后续工作（Video-LLaVA）扩展到了视频领域。

4. **幻觉问题**：LLaVA 可能会生成与图像不符的内容（"这幅图里有埃菲尔铁塔"但实际上没有）。这是所有多模态 LLM 的共性问题。

5. **评估不全面**：论文主要在 ScienceQA 上评估，缺乏对开放域视觉对话的标准化评估。后续工作（MMBench、MME）提供了更全面的评估基准。

## 九、小结
