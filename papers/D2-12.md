# LLaVA: Visual Instruction Tuning（2023）

**论文：** [arXiv:2304.08485](https://arxiv.org/abs/2304.08485) · Haotian Liu, Chunyuan Li, Qingyang Wu, Yong Jae Lee · UW–Madison / Microsoft Research / Columbia University · 2023

**技术分支：** D2 — 视觉与语言模型结合 / D2-12 — 冻结 LLM + 视觉适配器

**代码/数据：** [llava-vl.github.io](https://llava-vl.github.io)

---

## 一、先搞清楚"问题在哪里"

2022–2023 年，NLP 社区已经清楚了一件事：**instruction tuning 是让 LLM 学会"听话"的关键技术**。用机器生成的高质量指令数据（Alpaca、Vicuna、GPT-4-LLM）对预训练 LLM 做 instruction tuning，可以让开源模型在指令遵循能力上逼近闭源模型。但这项技术的核心前提是——数据是文本的，任务也是文本的。

多模态领域的图景则完全不同。虽然 Flamingo、BLIP-2、KOSMOS-1 等模型已经证明，在图文对上训练多模态模型可以实现 impressive 的 zero-shot 任务迁移，但**它们的训练目标不是"遵循用户的视觉指令"**。给 BLIP-2 一张图片问"这张图片里有什么不寻常的地方？"，它大概率会回答一张描述性句子——因为它被训练成做 captioning，不是 instruction following。

这就形成了一个 gap：NLP 已经有了 instruction tuning 的范式 + 方法论，但多模态领域没有现成的 visual instruction tuning 数据，也没有一个专门为此训练的模型。问题很直白：

**能不能把 instruction tuning 从纯文本扩展到视觉-语言多模态空间？**

答案听上去简单，但做起来面临两条并行的难题：

1. **没有数据。** 多模态 instruction-following 数据的标注流程没有明确定义，人工标注耗时且昂贵。
2. **没有模型。** 没有现成的端到端模型架构可以直接做 visual instruction tuning。

LLaVA 要同时解决这两个问题。

---

## 二、旧路线的真正问题

在 LLaVA 之前，多模态模型的构建方式可以分为两类：

**第一类：端到端训练的专用模型。** 每个任务一个模型——做 captioning 的训练一个、做 VQA 的训练一个、做指代理解的再训练一个。这些模型的接口是固定的（输入是图像 + 特定格式的 prompt），用户没法自由地问问题。

**第二类：LangChain 式的系统组合。** 如 Visual ChatGPT、MM-REACT、ViperGPT，用 LLM 做"调度器"，调用专门的视觉工具（检测器、分割器、caption 模型）。这类系统确实能做到 instruction following，但它不是端到端训练的——每个工具的误差会累积，而且系统复杂度随着工具数量线性增长。

这两类方案都没做到一件事：**在统一的端到端模型中，把图像内容和自然语言指令结合起来作为输入，让模型学会"看"着图像"听"指令。**

而 LLaVA 之前的 instruction tuning 工作（Alpaca、Vicuna、GPT-4-LLM）全部是纯文本的，它们生成指令数据的方式无法直接迁移到多模态——因为图像内容不能直接喂给 text-only 的 GPT-4。

核心矛盾在于：**最会生成高质量指令数据的是 GPT-4，但 GPT-4 看不到图像。**

---

## 三、核心思路：用符号表示把图像"翻译"给 GPT-4

LLaVA 的核心洞察其实非常自然：**既然 GPT-4 看不到图像，那就把图像用它可以理解的方式描述出来。** 具体来说，用两种符号表示来编码图像的视觉内容：

1. **Caption（文本描述）**：从多个角度描述视觉场景
2. **Bounding boxes（边界框）**：定位场景中的物体，每个框编码物体概念及其空间位置

这样，一张图像就被"翻译"成了一串 GPT-4 可以识别的文本序列。GPT-4 虽然没有见过图像本身，但基于 caption 和 bounding box 信息，它可以生成高质量的多轮视觉指令数据。

这条技术路线的谱系位置：

```text
纯文本 instruction tuning（Alpaca, Vicuna, 2023）
   ↓
LLaVA（首次用 text-only GPT-4 生成视觉指令数据, 2023） ← 本篇
   ↓
LLaVA-1.5（改进架构+数据, 2024）
```

这个思路和之前的 BLIP-2（用 Q-Former 连接视觉和语言）不一样：BLIP-2 关注的是**特征对齐**，LLaVA 关注的是**数据生成 + 指令遵循**。这也是为什么 LLaVA 的架构极其简单——它把复杂度放到了数据侧，而不是模型侧。

## 四、系统设计全景

### 4.1 数据生成：GPT-4 充当"视觉讲解员"

LLaVA 使用 COCO 数据集中的图像，为每张图像提供了 caption 和 bounding box 作为上下文，然后引导 GPT-4 生成三种类型的指令数据：

**Conversation（58K samples）：** 设计助手和问问题的人之间的对话。问题涵盖物体类型、计数、动作、位置、物体间相对位置等。只考虑有确定答案的问题。

**Detailed description（23K samples）：** 创建一套丰富全面的图像描述问题列表，对每张图像随机采样一个问题让 GPT-4 生成详细描述。

**Complex reasoning（77K samples）：** 在前两类视觉内容基础上，进一步创建需要深度推理的问题，答案通常需要分步骤的逻辑推理过程。

这三种类型共收集了 **158K 个独特的语言-图像指令遵循样本**。其中唯一的人工标注是每种类型的几个 seed example，用于 in-context learning 引导 GPT-4。

关键发现：**GPT-4 始终比 ChatGPT 提供更高质量的指令遵循数据**，尤其是在空间推理方面。

### 4.2 架构：简单到令人惊讶

LLaVA 的网络架构出奇地简洁——整个多模态模型的"胶水"只是一个单层线性投影：

```text
输入图像
    │
    ▼
CLIP ViT-L/14 视觉编码器（预训练，全程冻结）
    │ 输出视觉特征 Z_v（grid features）
    ▼
单层线性投影矩阵 W
    │ 将视觉特征映射到 LLM 词嵌入空间
    ▼
视觉 tokens H_v（与文本 token 同维度）
    │ 和文本指令拼接
    ▼
Vicuna-13B 语言解码器（预训练，第二阶段微调）
    │
    ▼
文本输出
```

数学描述很简单：对于输入图像 X_v，CLIP visual encoder 输出 Z_v = g(X_v)，然后通过投影矩阵得到语言嵌入空间的视觉 token：H_v = W · Z_v。

论文明确提到，这个简单的投影方案是**轻量级的**，目的是快速迭代数据中心的实验。更复杂的连接方案（Flamingo 的 gated cross-attention、BLIP-2 的 Q-Former）留作未来工作。这个设计选择背后有一个清晰的态度：**架构不是瓶颈，数据才是。**

### 4.3 训练：两阶段 instruction tuning

**Stage 1: Pre-training for Feature Alignment（预训练特征对齐）**

从 CC3M 中过滤出 **595K 图文对**，用简单的扩展方法（问题 + caption）转化为单轮指令数据。训练时只优化投影矩阵 W，视觉编码器和 LLM 均冻结。Stage 1 的目标是让图像特征 H_v 与预训练 LLM 的词嵌入空间对齐——可以理解为为冻结的 LLM 训练一个兼容的视觉 tokenizer。

超参数：8×A100，1 epoch，lr=2e-3，batch size=128。

**Stage 2: Fine-tuning End-to-End（端到端微调）**

视觉编码器始终保持冻结。更新投影层 W 和 LLM 的预训练权重 ϕ（即训练参数为 θ = {W, ϕ}）。两种场景：

- **多模态 Chatbot**：在 158K 指令数据上微调，三种类型均匀采样
- **Science QA**：在 ScienceQA 基准上微调，组织为单轮对话

超参数：8×A100，3 epochs，lr=2e-5，batch size=32。

## 五、实验结果

### 5.1 多模态 Chatbot 表现

LLaVA 在一系列需要深度图像理解的示例上展示了令人惊讶的推理能力。尽管训练数据只覆盖了约 80K 张独特的 COCO 图像（都是训练集内的域），但在**域外图像**上，LLaVA 仍然能理解场景并按照用户指令给出合理回应。

与之对比，BLIP-2 和 OpenFlamingo 更倾向于**描述图像内容**，而不是遵循用户指令以适当的方式作答。例如，当问"这张图片有什么不寻常的地方？"时，BLIP-2 回答"一个男人坐在黄色出租车后面"，OpenFlamingo 回答"这个人在他的汽车引擎盖上晾衣服"——它们都在描述场景，但都没有真正回答"不寻常"这个核心问题。LLaVA 则能准确理解指令的意图。

### 5.2 LLaVA-Bench (COCO) 定量评估

从 COCO-Val-2014 中随机选取 **30 张图像**，每张生成三种类型的问题，共 **90 个问题**。使用 GPT-4 作为评判者，给回答的 helpfulness、relevance、accuracy、detail 打分（1-10 分），以 text-only GPT-4（使用 ground truth 文本描述）为基准报告相对分数。

关键结果（Table 4）：

| 训练数据配置 | Conversation | Detail description | Complex reasoning | All |
|------------|:-----------:|:-----------------:|:----------------:|:--:|
| 全部三种数据（Full） | 83.1 | 75.3 | 96.5 | **85.1** |
| Detail + Complex | 81.5 (-1.6) | 73.3 (-2.0) | 90.8 (-5.7) | 81.9 (-3.2) |
| Conv + 5% Detail + 10% Complex | 81.0 (-2.1) | 68.4 (-7.1) | 91.5 (-5.0) | 80.5 (-4.4) |
| 仅 Conversation | 76.5 (-6.6) | 59.8 (-16.2) | 84.9 (-12.4) | 73.8 (-11.3) |

三个关键发现：

1. Instruction tuning 让模型能力大幅提升（相比无 instruction 版本提升超过 50 分，不过基线极低）
2. 加入少量 detail + complex reasoning 数据就带来了 **7 个点**的全面提升
3. 推理能力的提升也反过来增强了对话能力
4. **全部三种数据取得最佳结果 85.1%**

### 5.3 LLaVA-Bench (In-the-Wild) 域外泛化

收集了 **24 张图像，60 个问题**，涵盖室内外场景、meme、画作、素描等多样性内容。每张图像配有高度详细的、人工精选的描述和问题。

| 模型 | Conversation | Detail description | Complex reasoning | All |
|------|:-----------:|:-----------------:|:----------------:|:--:|
| OpenFlamingo | 19.3 | 19.0 | 19.1 | 19.1 |
| BLIP-2 | 54.6 | 29.1 | 32.9 | 38.1 |
| LLaVA | 57.3 | 52.5 | **81.7** | **67.3** |

得益于是 visual instruction tuning，LLaVA 相比 BLIP-2 提升 **+29%**，相比 OpenFlamingo 提升 **+48%**。在复杂推理问题上，LLaVA 达到了 81.7%——这是最接近 GPT-4 的单项表现。

### 5.4 Science QA

ScienceQA 是第一个大规模多模态科学问答数据集，每个问题附带自然语言或图像的上下文。

LLaVA 零样本（zero-shot）推理达到 **90.92% 准确率**，接近当时的 SOTA 91.68%。作为参考，人类基线为 88.40%，GPT-3.5 仅 73.97%。

当用 GPT-4 做 2-shot in-context learning 时，GPT-4 达到 82.69%（比 GPT-3.5 的 75.17% 高出 7.52 个绝对点）。

最引人注目的结果是：**LLaVA 和 GPT-4 的协同组合在所有问题类别上持续改进，最终达到 92.53% 的新 SOTA 准确率**。

**设计选择消融实验（Table 8）：**

| 变体 | 准确率 |
|------|:-----:|
| 最佳变体（完整两阶段训练） | **90.92%** |
| 使用倒数第二层的 CLIP 特征 | 89.96% (-0.96) |
| 先预测答案（而非推理过程） | 89.77% (-1.15) |
| 跳过预训练直接微调 | 85.81% (-5.11) |
| 使用 7B 模型（而非 13B） | 89.84% (-1.08) |

这些消融实验揭示了几个重要结论：

1. **预训练至关重要。** 跳过 Stage 1 后准确率从 90.92% 掉到 85.81%（-5.11%）——特征对齐是一个不可跳过的步骤。
2. **模型规模很重要。** 从 13B 降到 7B 损失 1.08%，但对于没有充足算力的团队，7B 是一个可行的扩展方案。
3. **CLIP 最后一层的 grid features 最好。** 使用最后一层前的特征损失 0.96%。

### 5.5 实验总结

LLaVA 的核心实验逻辑是清晰的数据中心叙事：**用最简单的架构（单层线性投影），搭配 GPT-4 生成的视觉指令数据，在多个评测上取得了和更复杂架构可比的甚至更好的结果。** 消融实验充分验证了数据的价值——不同数据类型、不同数据量、是否使用预训练，每一个设计选择都被量化评估。

## 六、核心贡献与范式意义

### 6.1 提出了 visual instruction tuning 的完整方法论

LLaVA 不只是提出了一个新的模型，而是建立了一套完整的 pipeline：

1. **数据生成方法**：用 text-only GPT-4 + 符号表示（caption + bounding box）生成多模态指令数据
2. **模型架构**：最简方案——CLIP + 单层投影 + Vicuna
3. **训练流程**：两阶段 instruction tuning（预训练对齐 → 端到端微调）
4. **评测框架**：LLaVA-Bench（COCO + In-the-Wild）用以评估视觉指令遵循能力

这套方法论的可迁移性很强——后来的 LLaVA-1.5 做的就是改进架构（用 MLP 替代线性投影）和数据质量，而不是推翻这套范式。

### 6.2 证明了"数据比架构更重要"

LLaVA 用最轻量的架构（单层线性投影 + 冻结视觉编码器）达到了惊人的效果，这个设计选择本身就是一个声明：**多模态 instruction tuning 的瓶颈不是架构容量，而是数据质量和多样性。** 这个发现在当时是反直觉的——很多人都认为需要一个复杂的跨模态连接模块（如 BLIP-2 的 Q-Former 或 Flamingo 的 gated cross-attention）才能做好多模态理解。LLaVA 证明了一个简单的线性层就够了。

### 6.3 打开了"GPT-4 做数据，开源模型做训练"的范式

LLaVA 的成功验证了一个新的协作范式：**用最好的闭源模型（GPT-4）生成训练数据，用开源框架训练开源模型。** 这个范式后来被广泛采用，特别是在 LLaVA 及其衍生工作中。LLaVA 论文也是最早一批将 GPT-4 的文本能力"蒸馏"到多模态领域的尝试之一。

## 七、时间线位置

```text
2022.04  Flamingo (Alayrac et al.)         ← 多模态 few-shot 学习的里程碑
   ↓
2023.01  BLIP-2 (Li et al.)                ← Q-Former 连接视觉和语言
   ↓
2023.01  Vicuna (Chiang et al.)            ← 开源 instruction-tuned LLM
   ↓
2023.03  GPT-4 (OpenAI)                    ← 最强闭源多模态模型（但不开源）
   ↓
2023.04  LLaVA (Liu et al.)                ← ★ 本篇 ★ 视觉 instruction tuning
   ↓
2023.10  LLaVA-1.5                         ← MLP 投影 + 改进数据，显著提升
   ↓
2023.12  LLaVA-NeXT                        ← 更高分辨率 + 更强 LLM backbone
   ↓
2024+    LLaVA 系列的持续改进和生态扩展
```

LLaVA 出现在 2023 年 4 月。这个时间点很关键：

- GPT-4 在 2023 年 3 月发布，展示了强大的多模态理解能力，但不公开模型权重
- Vicuna 和 Alpaca 展示了用 GPT-4 生成的数据训练开源 LLM 的可行性
- 但多模态领域还没有相应的尝试

LLaVA 的位置就是填补这个空白：**用 GPT-4 生成多模态训练数据，用开源 Vicuna 作为 backbone，构建开源的多模态 instruction-following 模型。**

## 八、后续影响

LLaVA 的后续影响主要体现在三个方向：

**LLaVA-1.5 的改进：** 改进了几个关键设计：用两层 MLP 替代单层线性投影（从线性变成非线性，表达能力增强）；使用更高质量的数据；系统地研究了 scaling 和数据配置。在 ScienceQA 上大幅超越原始 LLaVA。

**开源多模态生态的催化剂：** LLaVA 的代码、数据、模型权重完全开源，成为了后续大量多模态研究的基准框架。LLaVA 的代码库结构清晰，让后续研究者可以快速迭代。

**"GPT-4 生成数据"范式的推广：** 后来的 LLaVA-NeXT、LLaVA-RLHF、ShareGPT4V 等工作都延续了"用闭源模型生成数据 → 训练开源模型"的路径。这个范式至今仍是多模态 instruction tuning 的主流。

---

## 九、LLaVA 与其他多模态模型的对比定位

要理解 LLaVA 的独特价值，需要把它放在 2023 年初的多模态模型版图中看。Flamingo（2022）用了 2.1B 图文对训练，用了复杂的 gated cross-attention，但它不做 instruction tuning，也不开源。BLIP-2（2023）用 Q-Former 连接 ViT-g/14 和冻结 LLM，在 129M 图文对上训练，同样不做视觉 instruction tuning。OpenFlamingo 是 Flamingo 的开源复现，但也没有显式做 instruction tuning。

LLaVA 是这些模型中**唯一明确使用 visual instruction tuning 方法训练**的。其他模型虽然可以做 zero-shot 任务迁移，但它们没有专门针对

### 9.1 架构简单带来的上限

单层线性投影虽然轻量，但表达能力有限。原始的 ViT-L/14 输出 24×24=576 个 visual tokens，每个 token 通过线性变换直接映射到 Vicuna 的词嵌入空间——这种映射没有引入任何多模态交互（cross-attention、Q-Former 等）。后来的 LLaVA-1.5 改用两层 MLP 后效果显著提升，说明简单线性投影确实有上限。

### 9.2 数据多样性受限

所有 158K 指令数据都基于 COCO 图像的 caption 和 bounding box。覆盖的图像内容有限（主要是日常场景），且在训练指令数据时 GPT-4 依赖的标注质量（caption + bbox）直接影响生成数据的质量。真实世界中的长尾视觉概念没有被充分覆盖。

### 9.3 冻结视觉编码器的限制

LLaVA 的视觉编码器（CLIP ViT-L/14）在训练全程冻结。这意味着模型无法在训练过程中校准视觉编码器的特征——如果在训练数据中某些视觉概念没有被 CLIP 很好地编码，LLM 也无法"补救"。后来的工作（如 LLaVA-1.5 的一些变体）尝试了解冻 visual encoder，带来了额外的提升。

### 9.4 评测的局限性

LLaVA-Bench 使用 GPT-4 作为评判者，这引入了两个问题：一是 GPT-4 作为评判者的偏见（它可能更喜欢和自己风格相似的回答）；二是 LLaVA-Bench 的规模较小（COCO 版 90 问，In-the-Wild 版 60 问），统计显著性有限。后来的工作开始使用更大的多模态评测集（MMBench、MME、MM-Vet 等）。

## 十、小结

LLaVA 的贡献不在于架构创新——单层线性投影 + 预训练 CLIP + Vicuna 几乎是最简单的方案了。它真正的贡献在于建立了一个可复现的 visual instruction tuning pipeline，从数据生成到模型训练到基准评测的完整链路。简单到极致，反而最好地分离了"数据的价值"和"架构的价值"——因为架构几乎不贡献复杂度，所以所有的性能提升都可以归因于数据。

在技术树中，LLaVA 是一个 **foundation node**。它开启了一条清晰的研究路线（视觉 instruction tuning），启发了大量的后续工作（LLaVA-1.5、LLaVA-NeXT、ShareGPT4V 等），也成为了开源多模态社区的核心参考框架。
