# Fixup Initialization: Residual Learning Without Normalization（2019）
**论文：** [arXiv](https://arxiv.org/abs/1901.09321) · Hongyi Zhang, Yann N. Dauphin, Tengyu Ma · 2019
---

## 一、先搞清楚坑在哪

深度学习走到 2019 年，残差网络（ResNet）已经是标配架构。但有个问题一直没解决：**为什么残差网络必须配合 Batch Normalization（BN）才能训练得动？**

如果你尝试把 ResNet 里的 BN 去掉，直接用原版残差块（`x + F(x)`），训练几乎必然发散。这导致了一个尴尬的局面——虽然 BN 本身有很多毛病（依赖 batch size、训练测试不一致、增加额外计算），但大家不敢不用它，因为没了 BN，残差网络就训练不了。

这就引出了一个核心问题：**残差连接和 BN 之间到底存在什么耦合关系？** 能不能设计一套初始化方案，让残差网络在没有 BN 的情况下也能稳定训练？

## 二、Batch Normalization 的真正问题

先看看 BN 在残差网络中到底扮演了什么角色。BN 的经典好处大家都很熟悉：

1. **稳定激活分布**：每层输出归一化到均值为 0、方差为 1
2. **允许更大学习率**：梯度的尺度被约束
3. **提供一定正则化**：因为 mini-batch 统计量的随机性

但 BN 在残差网络中有个**隐藏的关键作用**——它解决了残差分支带来的**爆炸性方差累积**问题。

具体来说，假设你有 N 个残差块，每个块的输出是 `x_{l+1} = x_l + F(x_l)`。如果 `F(x_l)` 的方差是 `σ²`，那么经过 L 个块之后，激活值的方差会累积到 `L·σ²`。当 L 很大时（比如 ResNet-152 有 152 层），这个方差会爆炸。

BN 之所以能解决这个问题，是因为它**在残差分支的末端做了缩放**——`BN(F(x_l))` 的输出方差被稳定在 1 左右。这就相当于给每个残差分支加了一个自动的增益控制。

但 BN 的问题也不容忽视：

- **batch size 依赖**：BN 在小 batch 下性能严重下降（比如检测任务中 batch size=2）
- **训练/测试不一致**：训练用 batch 统计量，测试用全局统计量，导致表现不一致
- **额外内存和计算**：BN 需要存储 running mean/variance，增加约 30% 的计算量
- **打破独立性**：BN 让样本间相互依赖，不利于某些分布式训练策略

## 三、Fixup Initialization 的核心思路

Fixup 的核心洞察其实很简单：**BN 在残差网络中起的作用，本质上是对残差分支进行缩放，而这个缩放可以通过精心设计的初始化来代替。**

直白说：既然 BN 让残差分支的输出方差保持在 1，那我能不能在初始化阶段，手动把每个残差分支的权重缩放到一个合适的尺度，使得前向传播的方差既不爆炸也不消失？

Fixup 的方案就四个字：**"下采样"残差分支的权重**。具体来说，在初始化时，把残差分支最后一层（通常是卷积层或线性层）的权重除以 `√(2L)`，其中 L 是残差块的数量。

这个看似简单的操作，背后有严谨的理论支撑——论文证明了在特定的初始化条件下，深度残差网络可以在没有 BN 的情况下，实现与有 BN 版本相当的训练稳定性。

## 四、Fixup 初始化详解

### 4.1 前向传播的方差分析

要理解 Fixup 为什么有效，我们需要追踪激活值在网络中传播时的方差变化。

考虑一个深度为 L 的残差网络。每个残差块的形式为：

$$x_{l+1} = x_l + F(x_l)$$

其中 $F(x_l)$ 是残差分支，包含若干层（通常是两个卷积层）。

**关键假设**：在初始化阶段，所有激活值都是独立同分布（i.i.d.）的，且均值为 0。

那么，在初始化时，$x_l$ 和 $F(x_l)$ 是近似不相关的（因为 $F$ 是随机初始化的）。因此：

$$\text{Var}(x_{l+1}) = \text{Var}(x_l) + \text{Var}(F(x_l))$$

经过 L 个块之后：

$$\text{Var}(x_L) = \text{Var}(x_0) + \sum_{l=0}^{L-1} \text{Var}(F(x_l))$$

如果每个 $\text{Var}(F(x_l))$ 都是常数 $\sigma^2$，那么 $\text{Var}(x_L) = \text{Var}(x_0) + L \cdot \sigma^2$，随着 L 增大线性增长。

**Fixup 的解决方案**：让 $\text{Var}(F(x_l)) = O(1/L)$，这样 $\sum_{l=0}^{L-1} \text{Var}(F(x_l)) = O(1)$，总方差保持常数。

### 4.2 具体初始化方案

Fixup 初始化包含以下几个步骤：

1. **标准初始化**：所有层使用默认初始化（如 He 初始化或 Xavier 初始化）
2. **残差分支最后一层缩放**：将每个残差分支的最后一层（卷积或线性层）的权重除以 $\sqrt{2L}$
3. **偏置和标量初始化**：将每个残差分支的偏置初始化为 0，标量乘子初始化为 1（如果有的话）
4. **分类器层缩放**：将最后一层分类器的权重除以 $\sqrt{L}$（可选）

**为什么是 $\sqrt{2L}$？**

假设每个残差分支的输出方差在标准初始化下是 $\sigma_0^2$。经过缩放后，方差变为 $\sigma_0^2 / (2L)$。那么 L 个残差块的累积方差为：

$$\sum_{l=0}^{L-1} \frac{\sigma_0^2}{2L} = \frac{\sigma_0^2}{2}$$

这恰好是输入方差的一半（假设输入方差为 $\sigma_0^2$）。这样，总方差保持为输入方差加上累积方差，即 $\sigma_0^2 + \sigma_0^2/2 = 1.5\sigma_0^2$，是一个常数。

### 4.3 与 BN 的对比

| 特性 | BN | Fixup |
|------|-----|-------|
| 训练稳定性 | 依赖 batch 统计量 | 依赖初始化缩放 |
| 推理成本 | 额外计算 running stats | 无额外成本 |
| 小 batch | 性能下降 | 不受影响 |
| 实现复杂度 | 需要维护额外状态 | 仅修改初始化 |
| 理论保证 | 经验性 | 有理论分析 |

## 五、核心创新点

### 创新点 1：残差分支的缩放初始化

**（a）痛点与动机**

在深度残差网络中，没有 BN 的情况下，前向传播的方差会随着深度线性增长。这导致两个问题：一是激活值爆炸，使得后续层饱和（如果使用 sigmoid/tanh）或数值不稳定；二是反向传播时梯度也爆炸，导致训练发散。

现有方法如 [He et al., 2015] 的 ResNet 默认依赖 BN 来解决这个问题。[Srivastava et al., 2015] 的 Highway Networks 使用门控机制，但引入了额外参数。[Saxe et al., 2014] 的正交初始化虽然能稳定前向传播，但在残差网络中效果有限。

**（b）方案细节**

Fixup 的解决方案是：在初始化时，将每个残差分支的最后一层权重除以 $\sqrt{2L}$。这个缩放因子是**全局的**——它依赖于整个网络的深度 L，而不是局部层的索引。

具体实现时，需要先统计网络中残差块的总数 L，然后在初始化每个残差块时，找到该块中的最后一层（通常是第二个卷积），将其权重矩阵乘以 $1/\sqrt{2L}$。

**（c）为什么有效**

这个设计有效的核心原因是：它确保了前向传播中每个残差块的贡献是 $O(1/L)$ 的量级。这样，经过 L 个块的累积，总贡献是 $O(1)$，与网络深度无关。

论文中的理论分析（Theorem 1）证明了：在 Fixup 初始化下，前向传播的激活值方差在初始化时保持有界，且与网络深度无关。同时，反向传播的梯度也保持有界。

实验上，论文在 CIFAR-10 和 ImageNet 上验证了：使用 Fixup 初始化的 ResNet-110（无 BN）可以达到与有 BN 版本相当的精度。

**（d）与 Related Work 的关系**

Fixup 与以下几个工作有直接关系：

- **[He et al., 2015] ResNet**：Fixup 的目标就是去掉 ResNet 中的 BN，同时保持其训练稳定性
- **[Ioffe & Szegedy, 2015] Batch Normalization**：Fixup 试图从初始化角度替代 BN 的方差稳定功能
- **[Saxe et al., 2014] 正交初始化**：正交初始化能稳定前向传播，但 Fixup 的缩放策略更简单且专门针对残差架构
- **[Balduzzi et al., 2017] 破碎梯度**：Fixup 通过控制残差分支的尺度，间接缓解了梯度破碎问题

**（e）如果去掉会怎样**

论文中的消融实验（Table 1）显示：如果去掉残差分支的缩放（即使用标准初始化），ResNet-110 在 CIFAR-10 上的训练完全发散，准确率仅为 10%（随机猜测）。加上缩放后，准确率提升到 94.2%，与有 BN 的版本（94.5%）几乎持平。

### 创新点 2：多分支残差结构的统一处理

**（a）痛点与动机**

除了标准残差块（两个卷积），现代架构还有各种变体：瓶颈块（Bottleneck Block，1×1→3×3→1×1）、宽残差块（Wide ResNet）、以及 DenseNet 中的密集连接。每种结构对方差的影响不同，需要不同的缩放策略。

**（b）方案细节**

Fixup 提出了一个通用原则：**对于任何残差结构，只需要缩放其最后一个可学习层的权重**。具体来说：

- **标准块（两个 3×3 卷积）**：缩放第二个卷积
- **瓶颈块（1×1→3×3→1×1）**：缩放第三个卷积（1×1）
- **宽残差块**：缩放最后一个卷积
- **DenseNet 块**：缩放最后的卷积

这个原则基于一个观察：残差分支的方差主要由其最后一层的权重尺度决定，因为前面的层只负责特征变换，不改变方差的量级。

**（c）为什么有效**

从方差传播的角度看，残差分支可以看作一个函数 $F(x) = W_K \cdot \sigma(W_{K-1} \cdot \sigma(...W_1 x))$，其中 $W_K$ 是最后一层。如果前面各层使用标准初始化，那么 $F(x)$ 的方差主要由 $W_K$ 的尺度决定。因此，只需要缩放 $W_K$ 就能控制整个分支的方差。

**（d）与 Related Work 的关系**

这个通用原则借鉴了 **[He et al., 2016] Identity Mappings in Deep Residual Networks** 中对残差块结构的分析。He 等人发现，残差块中层的顺序（BN→ReLU→Conv vs Conv→BN→ReLU）对训练稳定性有显著影响。Fixup 则从初始化角度提供了另一种视角。

**（e）如果去掉会怎样**

如果对不同残差结构使用相同的缩放策略（比如都缩放第一个卷积），实验显示（Table 2）训练会变得不稳定，尤其是对于瓶颈块，因为瓶颈块中 1×1 卷积的通道数变化会导致方差传播的复杂性。

### 创新点 3：理论保证与深度无关性

**（a）痛点与动机**

现有初始化方法（如 Xavier、He 初始化）大多只针对前馈网络设计，没有考虑残差连接对方差传播的影响。这些方法保证了单层输出的方差稳定，但无法处理残差网络中多层的累积效应。

**（b）方案细节**

论文提供了严格的理论分析（Theorem 1-3），证明了：

1. **前向传播稳定性**：在 Fixup 初始化下，任何深度 L 的残差网络，其前向传播的激活值方差在初始化时保持有界，且上界与 L 无关
2. **反向传播稳定性**：梯度范数在初始化时也保持有界，与 L 无关
3. **训练动态**：在梯度下降的早期阶段，参数的更新量也保持有界

**（c）为什么有效**

理论证明的核心思想是：通过缩放残差分支，使得每个残差块的贡献是 $O(1/L)$ 的量级。这样，无论网络多深，累积贡献都是 $O(1)$。这类似于数学中的级数求和——如果每项是 $O(1/L)$，那么 L 项的和是 $O(1)$。

**（d）与 Related Work 的关系**

这个理论分析借鉴了 **[Hardt & Ma, 2017] Identity Matters in Deep Learning** 中的思想。Hardt 和 Ma 证明了残差连接的恒等映射（identity mapping）对训练动态的重要性。Fixup 则进一步分析了在没有 BN 的情况下，如何通过初始化来保持这种恒等映射的稳定性。

**（e）如果去掉会怎样**

论文中的理论分析（Theorem 1）给出了一个关键条件：如果残差分支的缩放因子不是 $O(1/\sqrt{L})$，而是 $O(1)$ 或 $O(1/L)$，那么前向/反向传播的方差要么爆炸（$O(1)$ 的情况），要么消失（$O(1/L)$ 的情况）。实验验证了这一理论预测——使用 $O(1/L)$ 的缩放（过度缩放）会导致训练缓慢，而使用 $O(1)$ 的缩放（不缩放）会导致发散。

## 六、公式详解

### 公式 1：残差块方差传播

$$\text{Var}(x_{l+1}) = \text{Var}(x_l) + \text{Var}(F(x_l))$$

**（a）符号定义**

- $x_l$：第 l 个残差块的输入，维度为 $d_l$
- $x_{l+1}$：第 l 个残差块的输出，维度为 $d_{l+1}$（通常 $d_{l+1} = d_l$）
- $F(x_l)$：残差分支的输出，维度与 $x_l$ 相同
- $\text{Var}(\cdot)$：方差算子（逐元素方差，假设各元素独立同分布）

**（b）公式来源**

这个公式来自 **[He et al., 2016] Identity Mappings in Deep Residual Networks** 中的方差分析。Fixup 论文将其作为理论分析的基础。

**（c）推导过程**

假设 $x_l$ 和 $F(x_l)$ 在初始化时是近似不相关的（因为 $F$ 是随机初始化的），那么：

$$\begin{aligned}
\text{Var}(x_{l+1}) &= \text{Var}(x_l + F(x_l)) \\
&= \text{Var}(x_l) + \text{Var}(F(x_l)) + 2\text{Cov}(x_l, F(x_l)) \\
&\approx \text{Var}(x_l) + \text{Var}(F(x_l))
\end{aligned}$$

其中协方差项 $\text{Cov}(x_l, F(x_l)) = 0$ 是因为 $F$ 的权重是随机初始化的，与 $x_l$ 独立。

**（d）直觉理解**

这个公式直观上很好理解：残差块的输出方差等于输入方差加上残差分支的方差。就像往桶里加水——桶里已有的水（输入方差）加上新倒进去的水（残差分支方差），就是总水量。

**具体数值例子**：假设输入方差 $\text{Var}(x_0) = 1$，每个残差分支的方差 $\text{Var}(F(x_l)) = 0.01$（经过 Fixup 缩放后）。对于 L=100 的残差网络，总方差为：

$$\text{Var}(x_{100}) = 1 + 100 \times 0.01 = 2$$

这个值不大也不小，刚好合适。

**（e）边界情况**

- 如果 $\text{Var}(F(x_l)) = 0$，那么 $\text{Var}(x_{l+1}) = \text{Var}(x_l)$，网络退化为恒等映射
- 如果 $\text{Var}(F(x_l)) = \sigma^2$（常数），那么 $\text{Var}(x_L) = \text{Var}(x_0) + L\sigma^2$，随 L 线性增长
- 如果 $\text{Var}(F(x_l)) = O(1/L)$，那么 $\text{Var}(x_L) = O(1)$，与 L 无关

### 公式 2：Fixup 缩放因子

$$W_{\text{last}} \leftarrow \frac{W_{\text{last}}}{\sqrt{2L}}$$

**（a）符号定义**

- $W_{\text{last}}$：残差分支最后一层的权重矩阵
- $L$：网络中残差块的总数（不是层数）

**（b）公式来源**

这个缩放因子是 Fixup 论文的核心贡献。它来自对公式 1 的分析——要使 $\text{Var}(F(x_l)) = O(1/L)$，需要将最后一层的权重缩放 $1/\sqrt{L}$。额外的 $\sqrt{2}$ 因子是为了使总方差恰好等于输入方差的一半（见下面的推导）。

**（c）推导过程**

假设残差分支 $F(x) = W_K \cdot \sigma(W_{K-1} \cdot \sigma(...W_1 x))$，其中 $W_K$ 是最后一层。

如果前面各层使用 He 初始化（方差为 $2/\text{fan\_in}$），那么经过 K-1 层后，激活值的方差保持不变。因此：

$$\text{Var}(F(x)) \approx \text{Var}(W_K) \cdot \text{Var}(\text{输入到 }W_K)$$

He 初始化下 $\text{Var}(W_K) = 2/d_{\text{in}}$，所以 $\text{Var}(F(x)) \approx 2$。

现在缩放 $W_K$ 为 $W_K / \sqrt{2L}$，则：

$$\text{Var}(F_{\text{scaled}}(x)) \approx \frac{2}{2L} = \frac{1}{L}$$

经过 L 个块累积：

$$\sum_{l=0}^{L-1} \text{Var}(F_{\text{scaled}}(x_l)) \approx L \cdot \frac{1}{L} = 1$$

加上输入方差 $\text{Var}(x_0) \approx 1$（假设），总方差为 2。

**（d）直觉理解**

这里的 $\sqrt{2L}$ 可以理解为：$\sqrt{2}$ 来自 He 初始化（每个残差分支的默认方差是 2），$\sqrt{L}$ 来自残差块的数量（需要将每个分支的贡献缩小到 $1/L$）。两者的乘积 $\sqrt{2L}$ 就是总的缩放因子。

**具体数值例子**：对于 ResNet-110（L=54），缩放因子为 $\sqrt{2 \times 54} \approx \sqrt{108} \approx 10.4$。也就是说，残差分支最后一层的权重被缩小到原来的 1/10.4。

**（e）边界情况**

- 如果 L=1（单残差块），缩放因子为 $\sqrt{2}$，相当于 He 初始化下的标准方差
- 如果 L 非常大（如 ResNet-1001，L≈333），缩放因子约为 $\sqrt{666} \approx 25.8$，权重被大幅缩小
- 如果 L=0（无残差块），这个公式不适用——Fixup 只对有残差块的网络有意义

### 公式 3：梯度传播

$$\frac{\partial \mathcal{L}}{\partial x_l} = \frac{\partial \mathcal{L}}{\partial x_L} \cdot \prod_{k=l}^{L-1} \left(I + \frac{\partial F(x_k)}{\partial x_k}\right)$$

**（a）符号定义**

- $\mathcal{L}$：损失函数
- $\frac{\partial \mathcal{L}}{\partial x_l}$：损失对第 l 层输入的梯度
- $I$：单位矩阵
- $\frac{\partial F(x_k)}{\partial x_k}$：残差分支 $F$ 在 $x_k$ 处的 Jacobian 矩阵

**（b）公式来源**

这是残差网络反向传播的标准公式，来自 **[He et al., 2016] Identity Mappings in Deep Residual Networks**。

**（c）推导过程**

从链式法则出发：

$$\frac{\partial \mathcal{L}}{\partial x_l} = \frac{\partial \mathcal{L}}{\partial x_{l+1}} \cdot \frac{\partial x_{l+1}}{\partial x_l}$$

而 $x_{l+1} = x_l + F(x_l)$，所以：

$$\frac{\partial x_{l+1}}{\partial x_l} = I + \frac{\partial F(x_l)}{\partial x_l}$$

递归展开得到：

$$\frac{\partial \mathcal{L}}{\partial x_l} = \frac{\partial \mathcal{L}}{\partial x_L} \cdot \prod_{k=l}^{L-1} \left(I + \frac{\partial F(x_k)}{\partial x_k}\right)$$

**（d）直觉理解**

这个公式揭示了残差网络梯度传播的两个关键特性：

1. **恒等映射项 I**：梯度可以通过恒等路径直接传播，不受残差分支的影响
2. **累积项**：残差分支的 Jacobian 被累积，但如果每个 $\frac{\partial F}{\partial x}$ 都很小，累积项不会爆炸

Fixup 通过缩小残差分支的权重，使得 $\frac{\partial F}{\partial x}$ 也很小（因为权重小，梯度也小），从而保证梯度不会爆炸。

**具体数值例子**：假设每个 $\frac{\partial F}{\partial x_k}$ 的谱范数为 0.01（经过 Fixup 缩放后），那么对于 L=100 的网络，累积项的谱范数为 $(1+0.01)^{100} \approx e^{1} \approx 2.72$，完全可控。

**（e）边界情况**

- 如果 $\frac{\partial F}{\partial x_k} = 0$，梯度传播退化为 $\frac{\partial \mathcal{L}}{\partial x_l} = \frac{\partial \mathcal{L}}{\partial x_L}$，即完美梯度传播
- 如果 $\frac{\partial F}{\partial x_k}$ 的谱范数大于 1，累积项可能指数增长，导致梯度爆炸
- 如果 $\frac{\partial F}{\partial x_k}$ 的谱范数远小于 1，累积项接近单位矩阵，梯度传播稳定

## 八、代码对照

Fixup 的实现非常简单，主要修改在初始化阶段。以下是 PyTorch 风格的伪代码：

```python
def fixup_init(model, num_residual_blocks):
    """
    Apply Fixup initialization to a residual network.
    
    Args:
        model: The neural network model
        num_residual_blocks: Total number of residual blocks (L)
    """
    scaling_factor = (2 * num_residual_blocks) ** 0.5
    
    for name, module in model.named_modules():
        if isinstance(module, (nn.Conv2d, nn.Linear)):
            # Standard initialization (He or Xavier)
            nn.init.kaiming_normal_(module.weight, mode='fan_out', 
                                    nonlinearity='relu')
            
            # Check if this is the last layer of a residual branch
            if is_last_layer_of_residual_branch(name, model):
                module.weight.data /= scaling_factor
            
            if module.bias is not None:
                nn.init.constant_(module.bias, 0)
```

**关键实现细节**：

1. **检测残差分支最后一层**：这需要知道网络结构。通常可以通过检查模块名称（如包含 "conv2"、"conv3" 等）或通过前向传播的 hook 来确定。

2. **缩放时机**：缩放是在**初始化之后**进行的，而不是替换初始化。先使用标准初始化（He 或 Xavier），然后再缩放最后一层。

3. **与 BN 的关系**：如果网络中同时存在 BN 和 Fixup，Fixup 的缩放会被 BN 抵消（因为 BN 会重新归一化）。因此 Fixup 通常只用于**替换 BN**，而不是与 BN 共存。

4. **学习率调整**：论文建议，使用 Fixup 时学习率可以比有 BN 版本稍低（如 0.05 vs 0.1），因为 Fixup 版本的梯度尺度略大。

## 十、局限

1. **需要手动计算残差块数量 L**：Fixup 的缩放因子依赖于全局的 L，这需要知道网络的总深度。对于动态构建的网络（如 NAS 搜索得到的架构），计算 L 可能不方便。

2. **只适用于残差结构**：Fixup 的核心思想是利用残差连接的恒等映射。对于非残差网络（如 VGG、AlexNet），Fixup 不适用。

3. **与 BN 的兼容性**：Fixup 的设计初衷是替代 BN。如果网络中同时存在 BN 和 Fixup，BN 的归一化会抵消 Fixup 的缩放效果，导致 Fixup 失效。

4. **对超参数敏感**：虽然 Fixup 本身没有引入新的超参数，但论文发现使用 Fixup 时需要调整学习率（通常比有 BN 版本低）。这增加了调参成本。

5. **理论分析的限制**：论文的理论分析假设激活值在初始化时是 i.i.d. 的，且忽略了非线性激活函数的影响。在实际训练中，这些假设并不严格成立。

6. **大规模数据集上的差距**：在 ImageNet 上，Fixup 版本与有 BN 版本仍有约 0.3% 的差距。虽然这个差距很小，但在对精度要求极高的场景中，可能仍然需要 BN。

## 十一、小结

Fixup Initialization 的核心贡献是：**揭示了 BN 在残差网络中起的作用本质上是方差缩放，而通过精心设计的初始化可以替代这个功能**。

具体来说，Fixup 通过将每个残差分支最后一层的权重除以 $\sqrt{2L}$（L 为残差块总数），使得前向传播的方差保持有界，反向传播的梯度保持稳定，从而实现了**无需 BN 的残差网络训练**。

这个工作的影响在于：
1. **理论层面**：提供了对残差网络训练稳定性的深入理解，揭示了初始化与架构之间的耦合关系
2. **实践层面**：为无归一化的深度学习提供了可行的方案，尤其有利于小 batch 场景和分布式训练
3. **后续影响**：启发了大量无归一化训练的工作（如 NFNets、KSAC 等），推动了深度学习基础设施的发展

虽然 Fixup 本身并没有完全取代 BN（在 ImageNet 上仍有微小差距），但它证明了**归一化不是深度残差网络训练的必要条件**，这一认识对后续的架构设计产生了深远影响。
