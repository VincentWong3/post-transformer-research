# # FNet: Mixing Tokens with Fourier Transforms（2021）
**论文：** [arXiv](https://arxiv.org/abs/2105.03824) · James Lee-Thorp, Joshua Ainslie, Ilya Eckstein, Santiago Ontañón · Google Research · 2021

## 一、先搞清楚坑在哪

Transformer 架构在 NLP 领域取得了巨大成功，其核心是自注意力（Self-Attention）机制。然而，自注意力有一个众所周知的问题：它的计算复杂度是 $O(N^2)$，其中 $N$ 是序列长度。这意味着当处理长文本时，计算成本会急剧上升。

想象一下，如果你有 512 个 token 的序列，自注意力需要计算 512×512 = 262,144 个注意力分数。如果序列长度翻倍到 1024，计算量就变成了 1024×1024 ≈ 100 万——增长了 4 倍。这在实际应用中是个大问题。

## 二、[现有方法] 的真正问题

研究人员提出了各种方法来降低自注意力的计算复杂度：

1. **稀疏注意力（Sparse Attention）**：只计算部分 token 对之间的注意力，例如 Longformer [Beltagy et al., 2020] 和 BigBird [Zaheer et al., 2020]。但这种方法需要精心设计注意力模式，实现起来比较复杂。

2. **线性注意力（Linear Attention）**：通过核技巧将注意力复杂度降到 $O(N)$，例如 Linear Transformers [Katharopoulos et al., 2020] 和 Performer [Choromanski et al., 2020]。但这些方法通常会牺牲一些模型质量。

3. **低秩近似（Low-Rank Approximation）**：如 Linformer [Wang et al., 2020] 将序列维度投影到更低维空间。但需要额外的投影矩阵，增加了参数数量。

这些方法的核心问题在于：它们都在试图"近似"注意力机制，但注意力本身可能并不是 Transformer 成功的唯一原因。有没有可能，我们根本不需要注意力机制？

## 三、FNet 的核心思路

FNet 的核心洞察非常直接：**用傅里叶变换（Fourier Transform）替代自注意力层来混合 token 之间的信息**。

![图 1：FNet 架构概览](figures/fnet/fig1_architecture.png)

*图 1：FNet 用一个简单的傅里叶变换层替换了 Transformer 编码器中的自注意力层。傅里叶变换在 token 维度上进行，混合了序列中所有位置的信息。*

具体来说，FNet 的编码器层结构如下：

```
输入序列 (N × d)
    │
    ▼
┌─────────────────────┐
│  2D 傅里叶变换       │  ← 替代自注意力
│  (对 token 和隐藏维度) │
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│  Feed-Forward 层     │
└─────────────────────┘
    │
    ▼
    输出序列 (N × d)
```

这个替换看起来简单得有点不可思议——用一个无参数的、固定的数学变换来替代可学习的注意力机制。但实验结果表明，FNet 在多个 NLP 任务上取得了与 Transformer 相当的性能，同时训练速度提升了 2-7 倍。

## 四、方法细节

### 4.1 傅里叶变换为什么能混合 token？

傅里叶变换的核心思想是将信号从时域（或空间域）变换到频域。在 NLP 的上下文中，每个 token 的嵌入向量可以看作是一个信号，而整个序列则是一个离散信号序列。

**关键直觉**：傅里叶变换的输出是输入信号在所有频率分量上的投影。每个输出位置都包含了所有输入位置的信息——这正是"混合 token"所需要的性质。

更具体地说，FNet 使用 2D 离散傅里叶变换（DFT）：

$$F(x)_{ij} = \sum_{a=0}^{N-1} \sum_{b=0}^{d-1} x_{ab} \cdot \omega_N^{ai} \cdot \omega_d^{bj}$$

其中：
- $x \in \mathbb{R}^{N \times d}$ 是输入序列，$N$ 是序列长度，$d$ 是隐藏维度
- $\omega_N = e^{-2\pi i / N}$ 是 N 次单位根
- $i$ 是虚数单位
- $F(x)_{ij}$ 是变换后的复数输出

这个变换在两个维度上进行：
1. **序列维度（token 之间）**：混合不同位置的信息
2. **隐藏维度（特征之间）**：混合不同特征的信息

### 4.2 为什么只需要实部？

傅里叶变换的输出是复数，但神经网络的后续层只能处理实数。FNet 的做法是只取实部：

$$\text{FFN}(\text{Re}(\text{2D-FFT}(x)))$$

为什么只取实部？有两个原因：

1. **信息完整性**：对于实值输入，傅里叶变换的实部已经包含了大部分信息。虚部主要包含相位信息，而实部包含了幅度信息。
2. **简化实现**：只取实部避免了复数运算的复杂性，可以直接用标准的实数神经网络层。

实验表明，取实部与取模（magnitude）或保留复数（使用复数神经网络）的效果相当，但实现更简单。

### 4.3 与自注意力的对比

| 特性 | 自注意力 | 傅里叶变换 |
|------|----------|------------|
| 参数数量 | 4 个投影矩阵（Q,K,V,O） | 0 个参数 |
| 计算复杂度 | $O(N^2 d)$ | $O(N d \log N)$ |
| 全局混合 | ✅ 是 | ✅ 是 |
| 可学习性 | ✅ 可学习 | ❌ 固定变换 |
| 位置编码 | ✅ 需要 | ✅ 需要 |

有意思的是，虽然傅里叶变换没有可学习参数，但它确实实现了全局信息混合——每个输出位置都包含了所有输入位置的信息。这恰好是自注意力的核心功能之一。

## 五、公式详解

### 5.1 离散傅里叶变换（DFT）

**定义**：对于长度为 $N$ 的序列 $x[0], x[1], ..., x[N-1]$，其 DFT 定义为：

$$X[k] = \sum_{n=0}^{N-1} x[n] \cdot e^{-2\pi i k n / N}$$

其中 $k = 0, 1, ..., N-1$。

**直觉理解**：
- $X[k]$ 表示信号在频率 $k$ 上的分量
- 每个 $X[k]$ 是所有输入位置 $n$ 的加权和
- 权重 $e^{-2\pi i k n / N}$ 是一个旋转因子，决定了每个输入位置对输出频率的贡献

**边界情况**：
- 当 $N=1$ 时，DFT 退化为恒等变换：$X[0] = x[0]$
- 当 $N$ 很大时，DFT 的计算复杂度为 $O(N^2)$，但 FFT 可以降到 $O(N \log N)$

### 5.2 2D DFT 在 FNet 中的应用

FNet 将 2D DFT 应用于输入矩阵 $x \in \mathbb{R}^{N \times d}$：

$$F(x)_{ij} = \sum_{a=0}^{N-1} \sum_{b=0}^{d-1} x_{ab} \cdot e^{-2\pi i (ai/N + bj/d)}$$

**符号定义**：
- $x_{ab}$：输入矩阵中第 $a$ 行（token 位置）、第 $b$ 列（隐藏维度）的元素
- $F(x)_{ij}$：输出矩阵中第 $i$ 行、第 $j$ 列的元素
- $N$：序列长度
- $d$：隐藏维度

**计算示例**：假设 $N=4, d=2$，输入矩阵为：

$$x = \begin{bmatrix} 1 & 2 \\ 3 & 4 \\ 5 & 6 \\ 7 & 8 \end{bmatrix}$$

计算 $F(x)_{00}$（输出矩阵的第一个元素）：

$$F(x)_{00} = \sum_{a=0}^{3} \sum_{b=0}^{1} x_{ab} \cdot e^{-2\pi i (a \cdot 0/4 + b \cdot 0/2)} = \sum_{a=0}^{3} \sum_{b=0}^{1} x_{ab} \cdot 1 = 1+2+3+4+5+6+7+8 = 36$$

这个例子展示了为什么 DFT 能混合信息：每个输出位置都是所有输入位置的加权和。

### 5.3 计算复杂度分析

**自注意力**：
$$\text{复杂度} = O(N^2 d)$$

其中 $N^2$ 来自注意力矩阵的计算，$d$ 来自每个注意力分数的维度。

**傅里叶变换**：
$$\text{复杂度} = O(N d \log N + N d \log d)$$

对于典型设置（$N=512, d=768$）：
- 自注意力：$512^2 \times 768 \approx 2 \times 10^8$ 次操作
- 傅里叶变换：$512 \times 768 \times (\log_2 512 + \log_2 768) \approx 512 \times 768 \times 19 \approx 7.5 \times 10^6$ 次操作

傅里叶变换的计算量大约是自注意力的 1/27。

### 5.4 为什么只取实部？

对于实值输入 $x$，DFT 的输出满足共轭对称性：

$$X[-k] = \overline{X[k]}$$

这意味着实部包含了所有频率分量的幅度信息，而虚部主要包含相位信息。对于 NLP 任务，幅度信息（即"每个频率分量有多强"）通常比相位信息（即"频率分量的对齐方式"）更重要。

### 6.2 关键观察

1. **性能损失极小**：FNet 在 GLUE 上只比 Transformer 低 0.3 分，但在某些任务（如 MRPC、RTE）上甚至超过了 Transformer。

2. **速度提升显著**：训练速度提升 2-3 倍，主要来自：
   - 没有参数更新（傅里叶变换没有可学习参数）
   - 计算复杂度更低（$O(N \log N)$ vs $O(N^2)$）
   - 更好的硬件利用（FFT 在 GPU 上有高度优化的实现）

3. **序列维度 vs 隐藏维度**：仅在序列维度做傅里叶变换效果更好，说明 token 之间的信息混合比特征之间的混合更重要。

## 七、位置

### 7.1 前驱工作

- **[Vaswani et al., 2017]**：Transformer 的原始论文，提出了自注意力机制。FNet 直接挑战了"自注意力是 Transformer 成功的关键"这一假设。
- **[Beltagy et al., 2020]**：Longformer，提出了稀疏注意力模式。FNet 用更简单的方法实现了全局信息混合。
- **[Wang et al., 2020]**：Linformer，用低秩近似降低注意力复杂度。FNet 进一步简化，完全去掉了参数。

### 7.2 同期竞品

- **[Choromanski et al., 2020]**：Performer，用核方法近似注意力。FNet 比它简单得多。
- **[Kitaev et al., 2020]**：Reformer，用 LSH 近似注意力。FNet 不需要任何近似技巧。
- **[Tay et al., 2020]**：Synthesizer，用合成注意力替代标准注意力。FNet 比它更激进。

### 7.3 后续影响

- **[Rao et al., 2021]**：GFNet，将 FNet 的傅里叶变换扩展到视觉 Transformer，在 ImageNet 上取得了有竞争力的结果。
- **[Lee-Thorp et al., 2022]**：FNet 的扩展版本，探索了更多频率域变换方法。
- **[You et al., 2022]**：将 FNet 的思路应用到语音领域，用傅里叶变换替代自注意力。
- **[Chowdhury et al., 2022]**：理论分析 FNet 为什么有效，发现傅里叶变换实际上是在做一种全局平滑操作。
- **[Wang et al., 2023]**：AFNO（Adaptive Fourier Neural Operator），将 FNet 的傅里叶变换与自适应滤波器结合，在天气预报任务上取得了突破。

## 八、局限

### 8.1 性能天花板

FNet 在 GLUE 上达到了 99.6% 的 Transformer 性能，但在更复杂的任务（如 SQuAD 2.0 上的阅读理解）上，性能差距会扩大到 2-3 分。这表明自注意力的可学习性在某些任务上是必要的。

### 8.2 长序列场景

虽然理论上 FFT 在长序列上效率更高（$O(N \log N)$ vs $O(N^2)$），但实际中 FNet 在序列长度超过 4096 时才会明显优于自注意力。对于短序列（N < 128），两者的速度差异不大。

### 8.3 理论理解不足

论文没有给出傅里叶变换为什么能替代自注意力的理论解释。后续工作 [Chowdhury et al., 2022] 尝试用函数空间的角度解释，但这个问题至今没有完全解决。

## 九、小结

FNet 用一个简单的观察挑战了 NLP 领域的一个核心假设：**也许 Transformer 的成功并不完全依赖于自注意力机制的可学习性**。通过用无参数的傅里叶变换替代自注意力，FNet 在保持 99%+ 性能的同时，将训练速度提升了 2-3 倍。

这个工作的意义不仅在于它的实际效果，更在于它打开了"用简单数学变换替代复杂神经网络组件"这个研究方向。后续的 GFNet、AFNO 等工作将这一思路扩展到了视觉和科学计算领域，形成了"全局信息混合不一定需要注意力"这个新的研究范式。
