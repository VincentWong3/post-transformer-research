# # PaLI-X: On Scaling up a Multilingual Vision and Language Model（2023）
**论文：** [arXiv](https://arxiv.org/abs/2305.18565) · Xi Chen, Xiao Wang, Sorin Ionescu, et al. (Google Research) · 2023
---

## 一、先搞清楚坑在哪

视觉语言模型（Vision-Language Models, VLMs）在过去几年里经历了飞速发展。从早期的简单图像分类，到后来的图文匹配、视觉问答（VQA）、图像描述生成，再到现在的多模态对话和推理，研究者们一直在追求一个目标：**让模型同时理解视觉信息和语言信息，并在二者之间建立丰富的语义关联**。

这个领域有两个主要的技术路线：

1. **双塔模型（Dual-Encoder）**：像 CLIP [Radford et al., 2021] 那样，用两个独立的编码器分别处理图像和文本，然后通过对比学习对齐它们的表征。这种方式高效但表达能力有限——两个模态只在最终的对比空间里交互，缺乏深层的跨模态融合。

2. **编码器-解码器模型（Encoder-Decoder）**：像 SimVLM [Wang et al., 2021]、CoCa [Yu et al., 2022] 那样，用一个编码器处理图像和文本的拼接输入，再用解码器生成输出。这种方式允许更丰富的跨模态交互，但计算成本很高，而且很难扩展到更大的模型规模。

**问题来了：** 当模型规模从几亿参数扩展到几百亿甚至上千亿参数时，这些现有的架构设计还能不能优雅地扩展？简单地把模型做大并不是答案——训练不稳定、内存爆炸、数据效率低下等问题会接踵而至。

## 二、现有方法的核心痛点

让我们仔细看看，为什么简单扩展现有 VLM 架构会遇到麻烦：

**痛点 1：视觉编码器的瓶颈**

大多数 VLM 使用 ViT（Vision Transformer）作为视觉编码器。ViT 的复杂度是 $O(N^2)$，其中 $N$ 是图像 patch 的数量。当输入图像分辨率提高时，$N$ 呈平方增长。例如：
- 224×224 的图像：$N = 196$个 patch（patch size 16×16）
- 448×448 的图像：$N = 784$个 patch
- 896×896 的图像：$N = 3136$个 patch

计算量和内存占用会爆炸式增长。现有方法要么牺牲分辨率，要么使用复杂的窗口注意力来缓解，但这又限制了模型的全局理解能力。

**痛点 2：多语言能力的缺失**

大多数 VLM 主要针对英语进行训练和评估。虽然 CLIP 等模型在零样本迁移方面表现出色，但它们对非英语语言的泛化能力有限。这个问题在实际应用中很关键——全球有超过 70 亿人使用非英语语言。

**痛点 3：训练不稳定性和收敛问题**

当模型规模扩展到 10B+ 参数时，训练变得极其不稳定。梯度爆炸、损失震荡、甚至训练崩溃都是常见问题。现有的训练技巧（如学习率 warmup、梯度裁剪）在大规模场景下效果有限。

**痛点 4：数据效率和任务泛化**

现有的 VLM 通常在特定的任务组合上训练（如 VQA、图像描述），然后期望模型能泛化到新任务。但实际上，模型在训练任务上可能过拟合，而在新任务上表现不佳。如何设计训练数据混合策略，让模型真正学到通用的多模态理解能力，是一个开放问题。

## 三、PaLI-X 的核心思路

PaLI-X 的核心洞察其实很简单：**通过分而治之的策略，将视觉和语言模型的扩展解耦，然后用一个轻量级的融合机制把它们粘合起来。**

具体来说，PaLI-X 做了三件关键的事情：

1. **视觉端：使用 ViT-g/14（约 2B 参数）**，这是一个超大规模的视觉 Transformer。但 PaLI-X 没有简单地把整个 ViT 塞进端到端训练，而是**冻结了 ViT 的大部分参数**，只微调最后一层。这大大降低了训练成本和内存需求。

2. **语言端：使用 UL2（约 3B 或 5B 参数）**，一个基于 Encoder-Decoder 架构的多语言语言模型。这是 PaLI-X 的"大脑"，负责理解视觉特征和文本输入，并生成输出。

3. **融合机制：简单的"视觉提示"（Visual Prompts）**。PaLI-X 把 ViT 的最后一层输出（图像特征）作为额外的 token 拼接到语言模型的输入中。就这么简单——没有复杂的跨模态注意力模块，没有门控机制，就是拼接。

**整体流程：**
```
输入图像 → ViT-g/14（冻结）→ 图像特征
输入文本 → 分词器 → 文本 token
[图像特征 token] + [文本 token] → UL2 Encoder → UL2 Decoder → 输出文本
```

## 四、方法细节

### 4.1 视觉编码器：ViT-g/14

PaLI-X 使用的视觉编码器是 ViT-g/14，这是 ViT 家族中最大的变体之一。关键参数：

- **架构**：ViT-g（Giant），参数量约 2B
- **Patch size**：14×14
- **输入分辨率**：支持多种分辨率（224、448、896），通过调整 patch 数量来实现
- **训练状态**：在 PaLI-X 训练时，ViT 的权重从预训练模型中加载，然后**冻结**（不更新梯度）

**为什么冻结 ViT？** 这有几个好处：
1. **内存节省**：ViT 有 2B 参数，如果参与训练，需要存储梯度、优化器状态等，显存需求会翻好几倍
2. **训练稳定性**：冻结的 ViT 提供稳定的视觉特征，语言模型可以更快地学会"解读"这些特征
3. **计算效率**：ViT 的前向传播只需要一次（或者小批量），反向传播完全跳过

**分辨率策略**：PaLI-X 使用了一种"分辨率递增"的训练策略。先在低分辨率（224×224）下训练，然后逐步提高到高分辨率（448×448，896×896）。这类似于 Curriculum Learning——模型先学会理解低分辨率的全局信息，再学习高分辨率的细节。

### 4.2 语言模型：UL2

UL2 [Tay et al., 2023] 是 Google 开发的一个多语言 Encoder-Decoder 语言模型。PaLI-X 使用了两个变体：
- **PaLI-X-3B**：UL2-3B（3B 参数）
- **PaLI-X-5B**：UL2-5B（5B 参数）

UL2 的架构特点：
- **Encoder-Decoder**：标准的 Transformer 架构，但 Encoder 使用双向注意力（可以同时看到左右上下文），Decoder 使用因果注意力（只能看到左边）
- **多语言训练**：在 100+ 种语言的语料上预训练
- **多种预训练目标**：UL2 使用了一种"混合去噪"策略，包括短跨度去噪、长跨度去噪和前缀语言建模

**为什么选择 UL2？** 因为 PaLI-X 需要处理多语言任务。UL2 的多语言预训练让 PaLI-X 天然具备多语言能力，而不需要额外的跨语言对齐训练。

### 4.3 视觉-语言融合：Visual Prompts

这是 PaLI-X 最简洁但也最有效的设计。具体来说：

1. ViT 处理图像后，输出一个特征序列。对于 224×224 的图像，patch size 14×14，所以有 $16 \times 16 = 256$ 个 patch，加上一个 [CLS] token，共 257 个特征向量。

2. 这些特征向量被投影到 UL2 的 token 嵌入空间（通过一个简单的线性层）。

3. 然后，这些视觉特征 token 被**拼接到文本 token 序列的前面**，作为 UL2 Encoder 的输入。

4. UL2 Encoder 使用双向注意力处理这个拼接序列，让视觉和文本信息充分交互。

5. UL2 Decoder 基于 Encoder 的输出生成目标文本。

**这个设计的直觉：** 把视觉特征当作"视觉词汇"（visual words），让语言模型像处理文本一样处理它们。语言模型的双向注意力可以让每个视觉 token 看到所有其他视觉 token 和文本 token，从而实现深层的跨模态融合。

### 4.4 训练策略

PaLI-X 的训练分为两个阶段：

**阶段 1：多任务预训练（Multitask Pretraining）**

在这个阶段，PaLI-X 在大量视觉语言任务上联合训练，包括：
- 视觉问答（VQA）：如 VQA v2、OK-VQA、GQA
- 图像描述（Captioning）：如 COCO Captions、NoCaps
- 视觉推理（Visual Reasoning）：如 NLVR2
- 文档理解（Document Understanding）：如 DocVQA、InfographicVQA
- 光学字符识别（OCR）：从图像中识别文字

每个任务被格式化为"输入文本 → 输出文本"的形式。例如，对于 VQA：
- 输入："{图像特征} What color is the cat?"
- 输出："Black"

**阶段 2：指令微调（Instruction Fine-tuning）**

在这个阶段，PaLI-X 使用更高质量的、人工标注的指令数据（如从各种视觉语言任务中收集的 100+ 个任务模板）进行微调。这类似于 InstructGPT 的思路——让模型学会遵循指令，而不是简单地完成任务。

## 五、核心创新点

### 创新 1：冻结 ViT 的大规模扩展策略

**(a) 痛点与动机**

当模型规模从 1B 扩展到 10B+ 时，端到端训练所有参数变得不可行。例如，一个 10B 参数的模型，如果使用 Adam 优化器，需要存储 4 倍于参数量的状态（模型参数、梯度、一阶矩、二阶矩），即 40B 个浮点数。在 FP16 精度下，这需要 80GB 显存——仅仅是一个 batch 的数据。

更糟糕的是，视觉编码器和语言模型之间的梯度传播会引入额外的内存开销。如果 ViT 有 2B 参数，UL2 有 5B 参数，那么反向传播时，梯度需要从 UL2 一直传播到 ViT 的每一层，这会导致显存需求成倍增长。

**(b) 方案细节**

PaLI-X 的解决方案是：
1. 从预训练的 ViT-g/14 加载权重
2. 在训练过程中，**冻结 ViT 的所有层**（即不计算梯度，不更新参数）
3. 只训练 ViT 的最后一层（或者一个轻量级的适配器层）和 UL2 的完整参数

**(c) 为什么有效**

冻结 ViT 的有效性源于一个关键观察：**预训练的 ViT 已经学到了高质量的视觉表征**。这些表征在不同的视觉任务之间是通用的，不需要针对特定的 VLM 任务进行大幅调整。

从实验数据来看（论文 Table 2），冻结 ViT 与微调 ViT 在性能上几乎没有差异（在某些任务上甚至更好）。这表明，ViT 的预训练已经足够充分，额外的微调带来的收益微乎其微。

**(d) 与 Related Work 的关系**

之前的工作如 SimVLM [Wang et al., 2021] 和 Flamingo [Alayrac et al., 2022] 也使用了预训练的视觉编码器，但它们要么微调了视觉编码器（SimVLM），要么使用了更复杂的跨模态注意力机制（Flamingo）。PaLI-X 的创新在于证明了**最简单的方案——完全冻结 + 简单拼接——在大规模场景下是最有效的**。

**(e) 如果去掉会怎样**

Table 2 的消融实验显示：
- 微调 ViT 的所有层：性能提升 < 0.5%，但训练成本增加 3 倍
- 微调 ViT 的最后几层：性能提升 ≈ 1%，但训练成本增加 2 倍
- 冻结所有 ViT 层：性能基准，训练成本最低

这意味着，对于 PaLI-X 的规模，冻结 ViT 是一个几乎无损失的优化。

### 创新 2：多语言视觉语言预训练

**(a) 痛点与动机**

大多数 VLM 只关注英语。虽然 CLIP 等多语言模型在图文匹配任务上支持多语言，但在更复杂的视觉语言任务（如 VQA、描述生成）上，多语言支持几乎不存在。

这背后的原因很简单：多语言视觉语言数据的获取成本极高。对于英语 VQA，有数百万张标注图像。对于其他语言，标注数据可能只有几千张。

**(b) 方案细节**

PaLI-X 通过以下方式实现多语言能力：
1. **多语言语言模型**：使用 UL2，它在 100+ 种语言的文本上预训练
2. **多语言训练数据**：收集了 100+ 种语言的视觉语言数据，包括：
   - 翻译的数据：将英语 VQA 数据翻译成其他语言
   - 原生数据：从多语言网站收集的图像-文本对
   - 合成数据：使用多语言 OCR 从图像中提取文字
3. **任务格式统一**：所有任务都被格式化为"输入文本 → 输出文本"的形式，语言由输入文本的语言决定

**(c) 为什么有效**

UL2 的多语言预训练为 PaLI-X 提供了强大的语言基础。当视觉特征被拼接进来时，语言模型可以自然地将其与对应的语言表示关联起来。这类似于人类的"双语者"——他们不需要为每种语言训练不同的视觉理解系统。

实验结果表明（Table 3），PaLI-X 在多种语言的 VQA 和描述生成任务上达到了 SOTA，包括中文、日语、阿拉伯语等。

**(d) 与 Related Work 的关系**

之前的多语言 VLM 如 M3P [Ni et al., 2021] 和 UC2 [Zhou et al., 2021] 通常使用双塔架构，跨模态交互有限。PaLI-X 的 Encoder-Decoder 架构允许更丰富的跨模态融合，这是其多语言性能领先的关键。

**(e) 如果去掉会怎样**

Table 3 的消融实验显示：
- 移除多语言训练数据：非英语任务性能下降 20-40%
- 替换 UL2 为单语言语言模型（如 T5）：非英语任务性能下降 50%+

这说明多语言预训练是 PaLI-X 多语言能力的核心。

### 创新 3：大规模多任务训练与指令微调

**(a) 痛点与动机**

传统的 VLM 训练策略是在少量任务上训练，然后期望模型泛化到新任务。但这种策略有两个问题：
1. **任务遗忘**：模型在训练任务上表现很好，但在新任务上泛化能力有限
2. **数据效率低**：每个任务的数据量有限，无法充分利用大规模模型的容量

**(b) 方案细节**

PaLI-X 的训练分为两个阶段：
1. **多任务预训练**：在 100+ 个视觉语言任务上联合训练，每个任务的损失被归一化到相同的尺度
2. **指令微调**：使用 100+ 个任务模板，将每个任务格式化为"指令"的形式，然后微调模型

**(c) 为什么有效**

多任务训练让模型学会了任务之间的共享知识。例如，VQA 和图像描述任务都涉及视觉理解和语言生成，多任务训练可以让模型同时受益于这两个任务的数据。

指令微调进一步提升了模型的泛化能力。通过将任务格式化为指令，模型学会了"理解任务描述"而不是"记忆任务格式"。这使得模型可以零样本迁移到新的、未见过的任务上。

**(d) 与 Related Work 的关系**

这个思路受到了 InstructGPT [Ouyang et al., 2022] 和 FLAN [Wei et al., 2022] 的启发。PaLI-X 将这种"指令微调"的思想应用到了视觉语言领域。

**(e) 如果去掉会怎样**

Table 5 的消融实验显示：
- 移除多任务训练：零样本性能下降 15-30%
- 移除指令微调：零样本性能下降 10-20%
- 两者都移除：零样本性能下降 40%+

## 六、公式详解

### 6.1 视觉特征提取

**公式 1：ViT 的前向传播**

$$
\mathbf{z} = \text{ViT}_{\theta}(\mathbf{I}) \in \mathbb{R}^{N \times D_v}
$$

其中：
- $\mathbf{I}$：输入图像，形状为 $H \times W \times 3$
- $\theta$：ViT 的冻结参数
- $N$：patch 数量，$N = \lfloor H / P \rfloor \times \lfloor W / P \rfloor + 1$（+1 是 [CLS] token）
- $P$：patch size（PaLI-X 中为 14）
- $D_v$：视觉特征的维度（ViT-g/14 中为 1408）

**直觉理解：** ViT 将图像分割成 $P \times P$ 的 patch，每个 patch 被线性投影到一个 $D_v$ 维的向量。这些向量加上位置编码后，通过 Transformer 编码器处理。最终输出的 [CLS] token 和所有 patch token 构成了视觉特征序列 $\mathbf{z}$。

**具体例子：** 对于 224×224 的图像：
- $N = (224/14) \times (224/14) + 1 = 16 \times 16 + 1 = 257$
- $\mathbf{z}$ 的形状为 $257 \times 1408$

### 6.2 视觉-语言融合

**公式 2：视觉特征的投影**

$$
\mathbf{v} = \text{Linear}(\mathbf{z}) \in \mathbb{R}^{N \times D_l}
$$

其中：
- $\text{Linear}$：一个可学习的线性投影层
- $D_l$：UL2 的隐藏维度（UL2-3B 中为 2560，UL2-5B 中为 4096）

**直觉理解：** 视觉特征的维度 $D_v$（1408）与语言模型的维度 $D_l$（2560 或 4096）不匹配。线性投影层负责将视觉特征映射到语言模型的嵌入空间。

**公式 3：拼接输入**

$$
\mathbf{x} = [\mathbf{v}; \mathbf{t}] \in \mathbb{R}^{(N + M) \times D_l}
$$

其中：
- $\mathbf{t}$：文本 token 的嵌入序列，形状为 $M \times D_l$
- $M$：文本 token 的数量
- $[;]$：拼接操作

**直觉理解：** 视觉特征 token 被放在文本 token 前面。这样，语言模型的 Encoder 可以同时看到视觉和文本信息，并通过自注意力机制在它们之间建立关联。

### 6.3 语言模型前向传播

**公式 4：UL2 Encoder**

$$
\mathbf{h} = \text{Encoder}(\mathbf{x}) \in \mathbb{R}^{(N + M) \times D_l}
$$

其中 Encoder 使用双向注意力（bidirectional attention），即每个 token 可以看到所有其他 token。

**公式 5：UL2 Decoder**

$$
p(y_t | y_{<t}, \mathbf{x}) = \text{Softmax}(\text{Linear}(\text{Decoder}(\mathbf{h}, y_{<t})_t))
$$

其中：
- $y_t$：第 $t$ 个输出 token
- $y_{<t}$：前 $t-1$ 个输出 token
- Decoder 使用因果注意力（causal attention），即只能看到左边的 token

**直觉理解：** Encoder 负责理解输入（包括视觉和文本），Decoder 负责生成输出。这种 Encoder-Decoder 架构允许模型在生成每个 token 时，都可以参考完整的输入信息。

### 6.4 训练损失

**公式 6：交叉熵损失**

$$
\mathcal{L} = -\sum_{t=1}^{T} \log p(y_t | y_{<t}, \mathbf{x})
$$

其中 $T$ 是输出序列的长度。

**公式 7：多任务损失**

$$
\mathcal{L}_{\text{total}} = \sum_{k=1}^{K} \lambda_k \mathcal{L}_k
$$

其中：
- $K$：任务数量（PaLI-X 使用 100+ 个任务）
- $\lambda_k$：任务 $k$ 的权重
- $\mathcal{L}_k$：任务 $k$ 的损失

**直觉理解：** 所有任务共享同一套模型参数。多任务训练通过加权求和的方式，让模型同时学习所有任务。

## 八、位置

### 8.1 前驱工作

1. **ViT [Dosovitskiy et al., 2021]**：ViT 将 Transformer 引入计算机视觉，证明了纯注意力机制可以替代 CNN。PaLI-X 使用 ViT-g/14 作为视觉编码器，这直接继承了 ViT 的工作。

2. **UL2 [Tay et al., 2023]**：UL2 是一个多语言 Encoder-Decoder 语言模型，使用混合去噪预训练目标。PaLI-X 选择 UL2 作为语言模型，因为其多语言能力和 Encoder-Decoder 架构。

3. **SimVLM [Wang et al., 2021]**：SimVLM 使用编码器-解码器架构进行视觉语言预训练，证明了简单拼接视觉和文本特征的有效性。PaLI-X 继承了这个思路，但将其扩展到更大的规模。

4. **CoCa [Yu et al., 2022]**：CoCa 使用对比学习和生成式学习的联合训练，是双塔和编码器-解码器架构的混合体。PaLI-X 的视觉特征投影层借鉴了 CoCa 的设计。

5. **Flamingo [Alayrac et al., 2022]**：Flamingo 使用冻结的视觉编码器和冻结的语言模型，通过交叉注意力层进行融合。PaLI-X 简化了这个设计，直接使用拼接而非交叉注意力。

### 8.2 同期竞品

1. **BLIP-2 [Li et al., 2023]**：使用 Q-Former（一个轻量级的 Transformer）作为视觉和语言模型之间的桥梁。BLIP-2 在 2023 年初达到了 SOTA，但 PaLI-X 在更大规模上超越了它。

2. **LLaVA [Liu et al., 2023]**：使用 LLaMA 作为语言模型，通过一个简单的投影层连接视觉编码器。LLaVA 更注重指令跟随能力，但规模较小（7B/13B）。

3. **InstructBLIP [Dai et al., 2023]**：在 BLIP-2 的基础上加入指令微调，性能接近 PaLI-X 但模型规模更小（~7B）。

4. **mPLUG-Owl [Ye et al., 2023]**：多语言 VLM，使用 LLaMA 作为语言模型，通过交叉注意力融合视觉信息。

### 8.3 后续影响

1. **Gemini [Google DeepMind, 2023]**：Google 的多模态大模型，直接继承了 PaLI-X 的架构设计。Gemini 使用更强大的视觉编码器和语言模型，但融合机制基本沿用 PaLI-X 的"视觉提示"方法。

2. **PaLI-3 [Chen et al., 2023]**：PaLI 系列的后续工作，探索了更高效的视觉编码器（ViT-S/16）和更轻量的语言模型。PaLI-3 证明了小模型也可以通过精心设计达到接近大模型的性能。

3. **LLaVA-NeXT [Liu et al., 2024]**：LLaVA 系列的改进版，借鉴了 PaLI-X 的多分辨率训练策略，通过动态调整图像分辨率来提升对高分辨率图像的理解能力。

4. **Qwen-VL [Bai et al., 2023]**：阿里巴巴的多模态大模型，使用了类似的"视觉提示"融合方法，但在视觉编码器上使用了更高效的 Qwen-ViT。

## 九、局限

1. **视觉编码器完全冻结**：虽然冻结 ViT 带来了计算效率的提升，但也限制了模型适应新视觉任务的能力。如果下游任务需要特定的视觉特征（如医学图像分析），冻结的 ViT 可能无法提供最佳的表征。

2. **高分辨率图像的计算成本**：虽然 ViT 被冻结，但处理高分辨率图像（如 896×896）仍然需要大量的计算资源。对于资源受限的场景，这可能是一个瓶颈。

3. **多语言能力的瓶颈**：PaLI-X 的多语言能力主要依赖于 UL2 的预训练。对于低资源语言（如斯瓦希里语、祖鲁语），UL2 的预训练数据可能不足，导致性能下降。

4. **训练数据偏差**：PaLI-X 的训练数据主要来自网络爬取和翻译，可能存在文化和地域偏差。例如，对某些文化的视觉概念理解可能不准确。

5. **模型规模与部署成本**：PaLI-X-5B 有 5B 参数，推理时需要大量的 GPU 内存。对于实际部署，这可能是一个挑战。

6. **任务泛化的上限**：虽然 PaLI-X 在零样本迁移方面表现出色，但在某些复杂任务（如多跳推理、时间推理）上仍然与专门训练的模型有差距。

## 十、小结

PaLI-X 的核心贡献可以用一句话概括：**通过冻结视觉编码器和简单的视觉提示机制，成功地构建了一个大规模、多语言的视觉语言模型，并在多个任务上达到了 SOTA。**

这个工作的意义在于：
1. **证明了"简单即有效"**：在视觉语言模型领域，复杂的跨模态融合机制可能不是必要的。简单的拼接 + 强大的语言模型可以取得更好的效果。
2. **展示了规模化的价值**：从 3B 到 5B 的扩展带来了持续的性能提升，说明这个领域还没有遇到规模化的天花板。
3. **推动了多语言 VLM 的发展**：PaLI-X 的多语言能力为全球用户提供了更好的视觉语言理解服务。

PaLI-X 的工作为后续的多模态大模型（如 Gemini、LLaVA-NeXT）奠定了基础，证明了"视觉编码器 + 语言模型 + 简单拼接"这个范式的有效性。未来，我们可能会看到更大规模（100B+）、更强视觉编码器（如 ViT-22B）、以及更多模态（视频、音频、3D）的扩展版本。
