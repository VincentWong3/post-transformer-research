# Mask2Former: Masked-attention Mask Transformer for Universal Image Segmentation（2022）

**论文：** [arXiv:2112.01527](https://arxiv.org/abs/2112.01527) · Bowen Cheng, Ishan Misra, Alexander G. Schwing, Alexander Kirillov, Rohit Girdhar · FAIR + UIUC · **CVPR 2022**

**技术分支：** C2 — 视觉分割技术线 / C2-12 — mask classification segmentation 范式

**代码：** [bowenc0221/mask2former](https://github.com/facebookresearch/Mask2Former)

---

## 一、先搞清楚"问题在哪里"

2021 年，MaskFormer 证明了 mask classification 范式可以统一语义分割、实例分割和全景分割——同一个架构、同一个损失、同一个训练流程，在三个任务上都取得了有竞争力的效果。这看起来是"一个问题，一个解法"的完美故事。

但 MaskFormer 有一个致命短板：**它只统一了架构，没有统一效果。**

具体来说，MaskFormer 在语义分割和全景分割上达到了 SOTA，但在实例分割上严重落后——比当时最好的专用架构（Swin-HTC++）低了超过 9 AP。在训练效率上，MaskFormer 需要 300 个 epoch 才能收敛到 40.1 AP，而 Swin-HTC++ 只用 72 个 epoch 就做得更好。更糟的是，MaskFormer 在 32GB 的 GPU 上只能塞进一张图。

所以这篇论文要回答的问题是：**能不能让 universal architecture 不仅在架构上统一，在效果和效率上也全面超越专用架构？**

答案是 Mask2Former。

---

## 二、旧路线的真正问题

MaskFormer 的问题出在哪里？分析下来有三条：

### 2.1 标准 cross-attention 收敛慢

MaskFormer 的 Transformer decoder 使用标准的 cross-attention——每个 query 和全图的每个像素做 attention。这对于训练收敛来说是灾难性的。

原因在于：cross-attention 在训练初期需要很多 epoch 才能学会"关注到物体的局部区域"[46]。一开始 attention 权重是均匀分布的，模型看不到明确的物体边界。这就解释了为什么 MaskFormer 要训 300 个 epoch——前 100 个 epoch 可能只是在学"哪里是前景"。

### 2.2 单尺度特征对小物体不友好

MaskFormer 的 Transformer decoder 只处理 pixel decoder 输出的单分辨率特征图。这导致小物体的信息在高分辨率特征图上丢失了——而高分辨率特征又没有被用到 decoder 中。

### 2.3 训练显存爆炸

Mask classification 需要在 H×W 的高分辨率特征图上计算每个 query 的二进制 mask loss。当 H 和 W 比较大时（比如 COCO 的 1024×1024 输入），计算和存储成本都非常高。MaskFormer 只能塞 1 张图进 32GB 显存。

这三个问题构成了一个"不可能三角"：通用性、性能、效率，旧架构最多只能同时满足两个。Mask2Former 的任务是打破这个三角。

## 三、这篇论文的核心思路

Mask2Former 的核心思路可以用一句话概括：**在 MaskFormer 的 meta architecture 基础上，用 masked attention 替代标准 cross-attention，用高效多尺度策略替代单尺度特征，再加上一系列优化改进和 3× 显存节省技巧，使得 universal architecture 在性能、效率和收敛速度上都超越专用架构。**

具体的四个改进可以归纳如下：

```text
MaskFormer（C2-11, 2021）
    │
    ├── masked attention —— cross-attention 只在前景 mask 区域内计算
    ├── 多尺度特征 —— P3/P4/P5 逐层 round-robin
    ├── 优化改进 —— 调换 attention 顺序、learnable queries、去 dropout
    └── 点采样 loss —— 112×112 随机点算 mask loss，显存省 3×
    │
    ▼
Mask2Former（C2-12, 2022）
```

这条技术路线的谱系位置是：

```text
DETR（set prediction, 2020）
  ↓
MaskFormer（mask classification 统一分割, 2021）
  ↓
Mask2Former（masked attention + 多尺度 + 高效训练, 2022） ← 本篇
  ↓
OneFormer（单模型多任务, 2023）
```

## 四、整体结构与 forward 流程

### 4.1 整体架构

Mask2Former 沿用 MaskFormer 的 meta architecture，由三个组件构成：

1. **Backbone** — 提取低分辨率特征。可用 ResNet、Swin Transformer 等
2. **Pixel Decoder** — 逐步上采样 backbone 输出，生成高分辨率 per-pixel embeddings。默认使用 MSDeformAttn[66]
3. **Transformer Decoder** — 用 masked attention 处理 object queries，输出 mask predictions

```text
输入图像: H×W×3
    │
    ▼
[Backbone]                 e.g., ResNet-50 / Swin-L
    │                      backbone output 分辨率: H/32 × W/32
    ▼
[Pixel Decoder]            MSDeformAttn / FaPN / BiFPN
    │                      输出特征金字塔: P3 (1/8), P4 (1/16), P5 (1/32)
    ▼
[Transformer Decoder]      L=3 次重复，每次 3 层
    │                      → Layer 1: mask attn @ P5 (1/32)
    │                      → Layer 2: mask attn @ P4 (1/16)
    │                      → Layer 3: mask attn @ P3 (1/8)
    │                      → 重复 L 次，共 3L 层
    ▼
[Mask Predictions]         N 个 binary masks + N 个 class labels
```

### 4.2 前向过程

**Step 1 — Backbone 提取特征**

输入图像通过 backbone（ResNet-50 或 Swin-L），输出低分辨率的特征图。例如 ResNet-50 输出分辨率为 $H/32 \times W/32$ ，channel 数 2048。

**Step 2 — Pixel Decoder 生成特征金字塔**

Pixel Decoder 逐步上采样 backbone 输出，生成三个分辨率的特征金字塔：
- P5: $H/32 \times W/32$ （低分辨率，高语义）
- P4: $H/16 \times W/16$ （中等分辨率）
- P3: $H/8 \times W/8$ （高分辨率，空间细节丰富）

每个分辨率上加正弦位置编码 $e_{\text{pos}} \in \mathbb{R}^{H_lW_l \times C}$ 和可学习的尺度编码 $e_{\text{lvl}} \in \mathbb{R}^{1 \times C}$ 。

**Step 3 — Transformer Decoder 循环处理**

Transformer decoder 共 $3L$ 层（ $L=3$ 为默认值），每 3 层为一个循环。在第 1、4、7... 层使用 P5（1/32 分辨率），在第 2、5、8... 层使用 P4（1/16），在第 3、6、9... 层使用 P3（1/8）。

每层的处理顺序是：masked attention → self-attention → FFN（与标准 Transformer 的 self-attention → cross-attention → FFN 顺序不同）。

**Step 4 — Mask 预测**

每一层都输出对 N 个 query 的 mask 预测和类别预测。损失在所有中间层上计算，最终输出最后一层的预测。

## 五、核心模块逐个解释

### 5.1 Masked Attention（关键创新）

**问题：** 标准 cross-attention 中，每个 query 与全图所有位置的像素做 attention，导致训练收敛极慢，需要大量 epoch 才能学会关注物体的局部区域。

**解决方案：** Masked attention 限制 cross-attention 只在预测 mask 的前景区域内计算。

公式上，标准 cross-attention 是：

$$\mathbf{X}_l = \text{softmax}(\mathbf{Q}_l\mathbf{K}^T_l)\mathbf{V}_l + \mathbf{X}_{l-1}$$

Masked attention 改为：

$$\mathbf{X}_l = \text{softmax}(\mathcal{M}_{l-1} + \mathbf{Q}_l\mathbf{K}^T_l)\mathbf{V}_l + \mathbf{X}_{l-1}$$

其中 $\mathcal{M}_{l-1}$ 是 attention mask，在特征位置 $(x,y)$ 上的值为：

$$\mathcal{M}_{l-1}(x,y) = \begin{cases} 0 & \text{if } \mathbf{M}_{l-1}(x,y) = 1 \\ -\infty & \text{otherwise} \end{cases}$$

这里 $\mathbf{M}_{l-1} \in \{0,1\}^{N \times H_lW_l}$ 是上一层输出的二进制 mask prediction（阈值 0.5），被缩放到 $\mathbf{K}_l$ 同样的分辨率。

**为什么这个设计有效：**

论文的核心假设是：**局部特征足够更新 query 特征，上下文信息可以通过 self-attention 来聚合。** 不需要 cross-attention 看全图。

Masked attention 的直觉是：对第 $l$ 层的一个 query，它只关心上一层为自己预测的那个 mask 区域内的特征。例如，如果某个 query 在第 $l-1$ 层预测了一个"猫"的 mask，那么第 $l$ 层的 cross-attention 就只看猫的区域内——而不是全图。

**效果：**

消融实验（Table 4(a)）显示，去掉 masked attention 后 AP 下降 5.9，PQ 下降 4.8，mIoU 下降 1.7——在所有改进中贡献最大。

Table 4(c) 还显示，masked attention 全面优于其他 cross-attention 变体：标准 cross-attention（AP 37.8）、SMCA[22]（AP 37.9）、K-Net 的 mask pooling（AP 43.1），而 masked attention 达到 AP 43.7。

### 5.2 高效多尺度策略

**问题：** 高分辨率特征对小物体分割很重要，但如果在所有 decoder 层都使用高分辨率特征，计算量（FLOPs）会急剧增加。

**解决方案：** 利用 pixel decoder 产生的特征金字塔（P3/P4/P5，分辨率分别为 1/8、1/16、1/32），每次只把一个分辨率的特征送入当前 transformer decoder 层。具体是 round-robin 的方式：第一层用 P5（1/32），第二层用 P4（1/16），第三层用 P3（1/8），然后重复。

**为什么这个设计高效：**

论文在 Table 4(d) 中比较了不同策略：
- 单尺度 1/32：AP 41.5, PQ 50.2, mIoU 46.1, FLOPs 218G
- 单尺度 1/8：AP 44.0, PQ 51.8, mIoU 47.4, FLOPs 239G
- 朴素多尺度（3 scales 同时用）：AP 44.0, PQ 51.9, mIoU 46.3, FLOPs 247G
- **高效多尺度（round-robin）：AP 43.7, PQ 51.9, mIoU 47.2, FLOPs 226G**

关键发现：高效多尺度策略用比 1/8 单尺度更少的 FLOPs（226G vs 239G），在 PQ 和 mIoU 上更好。朴素多尺度虽然 AP 更高（44.0 vs 43.7），但 FLOPs 多了 21G。

### 5.3 优化改进

Mask2Former 还引入了三个零计算开销的优化：

**1. 调换 self-attention 和 cross-attention 的顺序。** 标准 Transformer decoder 的顺序是 self-attention → cross-attention → FFN。Mask2Former 改为 cross-attention（即 masked attention）→ self-attention → FFN。理由是：query 特征刚从上一层出来时还不包含图像信息，先做 self-attention 没有意义。

**2. Learnable query features。** 标准 DETR/MaskFormer 中 $\mathbf{X}_0$ （初始 query feature）被初始化为零，只让 positional embedding 可学习。Mask2Former 让 $\mathbf{X}_0$ 也可学习。这些 learnable query 在进入 Transformer decoder 之前就参与预测 $\mathbf{M}_0$ （初始 mask），相当于有了直接的梯度监督。

**3. 移除 dropout。** 标准 Transformer 中对残差连接和 attention map 应用 dropout。Mask2Former 发现在这些任务上去掉 dropout 不影响性能。

Table 4(b) 显示，三项改进一起贡献了 AP +1.4、PQ +1.1、mIoU +0.9 的提升，而 FLOPs 完全不变。

### 5.4 基于点的 mask loss（显存节省 3×）

**问题：** 在全分辨率图像（如 1024×1024）上计算 mask loss 极度耗显存，MaskFormer 只能塞 1 张图进 32GB 显存。

**解决方案：** 受 PointRend[30] 启发，只在 $K=12544$ （即 $112 \times 112$ ）个随机采样点上计算 mask loss。

实现上分两步：
1. **匹配损失（matching loss）：** 对所有预测和 GT mask 统一采样相同的 $K$ 个点，构成 cost matrix 做 bipartite matching
2. **最终损失（final loss）：** 匹配完成后，对每个匹配对使用 importance sampling[30] 采样不同的 $K$ 个点

效果：训练显存从 18GB 降至 6GB 每张图（3× 节省），且性能不下降。

## 七、训练细节

### 7.1 全景/实例分割（COCO）

- **优化器：** AdamW[38]
- **学习率：** 初始 0.0001，weight decay 0.05，backbone 学习率乘以 0.1
- **学习率衰减：** step schedule，在总训练 steps 的 0.9 和 0.95 处各衰减 10×
- **训练 epoch：** 默认 50 epochs，batch size 16
- **数据增强：** LSJ（large-scale jittering），随机缩放 0.1–2.0，然后裁剪到 1024×1024
- **推理：** 短边 resize 到 800，长边不超过 1333
- **推理性能测量：** V100 GPU，batch size 1，覆盖整个验证集（含后处理时间）

### 7.2 语义分割（ADE20K）

- 和 MaskFormer[14] 相同的训练设置，除了：
  - backbone（包括 Transformer backbone）学习率乘以 0.1（MaskFormer 只对 CNN backbone 做）
  - ResNet 和 Swin backbone 统一使用初始学习率 0.0001、weight decay 0.05

### 7.3 收敛速度

论文中最重要的数据点之一：Mask2Former (R50) 训练 **50 epochs** 达到了 51.9 PQ，而 MaskFormer (R50) 需要 **300 epochs** 才达到 46.5 PQ。这意味着：
- 性能更高（+5.4 PQ）
- 训练快了 **6×**
- 32GB GPU 每张图从 18GB 降到 6GB（3× 显存节省）

综合来看，Mask2Former 用 MaskFormer **1/6 的训练时间** 和 **1/3 的显存**，达到了明显更好的效果。

## 十一、范式意义

**Mask2Former 的范式意义：它完成了 universal segmentation architecture 从"可行"到"超越"的最后一步。**

具体来说，Mask2Former 在三个方面改变了社区对 universal segmentation 的认知：

### 11.1 打破了"通用必然牺牲性能"的认知

在 Mask2Former 之前，universal architecture 的认知是：一个架构做所有事，必然在每个任务上不如专用架构。Mask2Former 第一次用实验证明了：**universal architecture 可以不仅统一架构，而且在所有任务上超越专用架构。**

### 11.2 训练效率的突破

Mask2Former 用 MaskFormer 1/6 的 epoch 数（50 vs 300）达到了更高的性能。这个效率提升不是来自更好的硬件或更大的 batch，而是来自架构设计：

- Masked attention 让每个 query 只看自己的前景区域，减少了无效计算，也加快了收敛
- 点采样 loss 让训练显存从 18GB 降到 6GB

这让 universal architecture 从"只有 FAIR 级别的资源才能训"变成了"有单卡的人也能训"。

### 11.3 Masked attention 的设计哲学

Masked attention 的核心理念——**"先预测，再关注"**——有更深远的影响。它意味着 Transformer decoder 不再需要从零开始学习"哪里是物体"，而是利用自身的中间预测来引导后续 attention。这是一种递归式的自引导（self-bootstrapping）：第 $l-1$ 层的预测帮助第 $l$ 层更好地关注，第 $l$ 层的预测又帮助第 $l+1$ 层。

这区别于标准 DETR 中 cross-attention 的"白纸式"学习。论文引用了[46]的发现——"global context in cross-attention 是 Transformer-based 模型收敛慢的原因"——并给出了一个非常优雅的解决方式：不做全局，只做局部。

### 11.4 分类：foundation node

在技术树中，Mask2Former 是一个 **foundation node（基础节点）**——它建立了 universal segmentation 的实用基线。在它之后，做分割的社区可以放心用同一个架构处理所有分割任务，不需要再为每个任务设计不同的 head 和特征处理。

它不是 bridge node（不连接不同领域）。它是真正的 foundation node——一个把"可以做"变成"应该做"的工作。

从实际影响力来看，Mask2Former（截至 2024 年引用超过 3000 次）已经成为图像分割的事实标准之一，在 detectron2 和 OpenMMLab 中都有官方实现，被广泛应用于各类视觉系统中的分割模块。

## 十二、小结

Mask2Former 的核心贡献是回答了 MaskFormer 没能回答的问题：**一个 universal architecture 能不能在统一架构的同时，也在性能和效率上全面超越专用架构？**

答案是可以，方法是四个改进：

1. **Masked attention**：cross-attention 只看预测 mask 的前景区域，不关注全图——收敛速度和性能同时提升
2. **高效多尺度特征**：P5/P4/P3 round-robin，用可控的计算开销引入高分辨率特征
3. **优化改进**：调换 attention 顺序、learnable queries、去 dropout——零计算开销的性能提升
4. **点采样 mask loss**： $112 \times 112$ 个随机点，显存省 3×

结果是：57.8 PQ（COCO 全景）、50.1 AP（COCO 实例）、57.7 mIoU（ADE20K 语义）——全部 SOTA，且训练 epoch 从 300 降到 50（R50）或 100（Swin-L）。

Mask2Former 证明了 **universal segmentation architecture 不仅是可能的，而且是更好的选择**。在这之后，"为每个分割任务设计专用架构"这条路变得不那么理直气壮了。
