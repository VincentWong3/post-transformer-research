## 一、先搞清楚"问题在哪里"

Transformer 已经成为深度学习的事实标准架构。从 BERT、GPT 到 T5，几乎所有主流模型都在用 self-attention。它的核心机制——Scaled Dot-Product Attention——让每个 token 都能关注到序列中的所有其他 token，从而捕捉长程依赖。

但这里藏着一个巨大的隐患：**复杂度是 O(L²)**，其中 L 是序列长度。为什么？因为 attention 的计算需要显式构造一个 L×L 的 attention 矩阵，矩阵里的每个元素是 query 和 key 的点积再经过 softmax 归一化。对于长度为 4096 的序列，这个矩阵有 1600 万个元素；对于长度为 128K 的序列，它是一个 160 亿个元素的庞然大物，几乎不可能存进 GPU 显存。

这个平方复杂度在模型训练和推理中制造了两道墙。第一道墙：当序列长度超过几千 token，显存消耗会急剧膨胀，即使是最新的高端 GPU 也难以承受。第二道墙：即使显存够用，计算时间 的 O(L²d) 也意味着运行时间的二次增长，让长序列处理变得不切实际。

这就是 Performer 这篇论文要解决的根本问题——**能不能把 attention 的复杂度从二次降到线性，同时不牺牲标准 softmax attention 的表达能力？**

## 二、旧路线的真正问题

在 Performer 之前，学术界已经提出了多种方案来缓解 attention 的平方复杂度。但这些方案都有各自的硬伤。

**稀疏注意力（Sparse Attention）** 是早期最直观的思路。代表工作包括 Child 等人的 Sparse Transformer，以及 Beltagy 等人的 Longformer。这些方法限制每个 token 只能关注固定范围内的邻居，或者通过预定义的稀疏模式（如步长模式、窗口模式）来降低计算量。问题在于：稀疏模式是人工设计的，这意味着模型被迫"假设"重要信息总是出现在局部区域内。对于需要捕获全局长程依赖的任务（如蛋白质序列建模、文档级 NLP），这种假设往往不成立。

**基于哈希的方法（LSH Attention）** 以 Reformer（Kitaev et al., 2020）为代表。Reformer 通过局部敏感哈希将 query 和 key 分桶，使得每个 query 只需要和同一桶内的 key 做 attention。但这里有一个关键限制：它强制 query 和 key 共享相同的线性变换参数（即 Q=K），这极大地限制了模型的表达能力。而且 LSH 的分桶操作在 GPU 上并不高效，实际加速效果有限。

**基于聚类的方法**（如 Roy 等人的 Routing Transformer）使用 k-means 聚类来动态确定每个 token 的 attention 区域。虽然比固定稀疏模式更灵活，但聚类操作本身增加了额外开销，而且聚类的质量直接影响 attention 的效果，训练稳定性是一个问题。

**低秩核方法**（如 Katharopoulos 等人的 Linear Transformer，Shen 等人的 Efficient Attention）尝试用核函数替代标准 softmax。它们将 attention 矩阵分解为低秩形式，从而达到线性复杂度。但这些方法**不近似标准 softmax attention**，而是用不同的核函数取而代之。这意味着它们改变了对 attention 的定义——去掉了 softmax 归一化中的指数函数，或者使用了完全不同的核。因此，无法保证这些模型能够达到和标准 Transformer 相同的表达能力。

**有偏估计方法**（如 Wang 等人的 Nyströmformer）使用 Nyström 近似来估计 attention 矩阵。但这种方法只在双向（非因果）setting 下工作，而且均方误差（MSE）较大，没有提供无偏估计的理论保证。

总的来说，此前的方法要么牺牲表达能力和理论保证，要么只在特定任务上有效，**没有一个方法能同时做到三点：线性复杂度、标准 softmax attention 的近似、以及强有力的理论保证**。这个空白正是 Performer 要填补的。

## 三、这篇论文想干什么

Performer 这篇论文的目标非常清晰：设计一种新的 Transformer 架构，使得 attention 的计算复杂度从 O(L²d) 降到 O(Lmd)（m << L，m 为特征维度），同时保持对标准 softmax attention 的**无偏或近似无偏估计**，并且提供严格的理论保证。

更具体地说，作者想要做到四件事：

**第一，构建正确的随机特征映射。** 从核方法的角度重新理解 attention，找到一种随机特征映射 φ(x)，使得 softmax kernel K(x,y)=exp(xᵀy) 可以被写成 φ(x) 和 φ(y) 内积的期望。关键在于 φ(x) 的每个分量都必须是非负的，否则 attention 归一化会出问题。这是此前所有方法没有解决的核心难点。

**第二，保持理论严谨性。** 作者不仅想要一个能 work 的方法，还想要严格的证明：近似是无偏或近乎无偏的、近似误差有一致收敛性、并且估计方差有界且在目标值趋近于 0 时也趋近于 0。

**第三，支持双向和单向 attention。** 即同时支持 encoder 的双向 attention 和 decoder 的自回归单向 attention。后者因为 causal masking 的限制，实现更复杂。

**第四，让 FAVOR+ 成为 attention 的"通用加速器"。** 不仅针对 softmax，这套框架还能泛化到其他可核化的 attention 机制。这使得研究人员能够首次在大规模任务上公平地比较不同 attention kernel 的效果。

## 四、核心思路：把 attention 写成核函数

理解 Performer，首先要从 attention 的核视角说起。

标准 attention 的计算可以分为三步：
1. 计算 Q 和 K 的点积矩阵：S = QKᵀ / √d
2. 对 S 逐元素取指数：A = exp(S)（即 softmax 归一化之前）
3. 归一化：Att = D⁻¹AV，其中 D = diag(A·1_L)

如果我们观察矩阵 A 的第 i 行第 j 列元素，它实际上是 query_i 和 key_j 的一个"相似度得分"。这个得分通过 exp((q_i)ᵀk_j/√d) 来计算。这正是核方法的味道——两个向量的相似度由某个核函数 K(x,y) 决定。

如果我们能找到一种随机特征映射 φ: ℝᵈ→ℝ⁺ᵐ，使得：
K(x,y) = 𝔼[φ(x)ᵀφ(y)]

那么 attention 就可以被改写为：
Att = D⁻¹ · (Q' · (K')ᵀ) · V

其中 Q' 和 K' 的行分别是 φ(q_i)ᵀ 和 φ(k_i)ᵀ。注意这里的计算顺序：先算 (K')ᵀV（O(Lmd)），再算 Q'((K')ᵀV)（也是 O(Lmd)）。**整个过程的复杂度从 O(L²d) 降到了 O(Lmd)。**

这就是 FAVOR+ 中 FA（Fast Attention）部分的核心理念。

但关键在于找到合适的随机特征映射 φ——它必须将 softmax kernel 分解为 φ 内积的期望，同时保证 φ 输出为正。传统的随机特征方案为什么不 work？下面从基本原理说起。

## 五、随机特征映射的基本原理

随机特征映射（Random Feature Maps）并不是 Performer 原创的概念。它最早由 Rahimi 和 Recht 在 2007 年提出，用于高效近似平移不变核（shift-invariant kernels），如高斯核。

Rahimi-Recht 方法的核心洞察是 Bochner 定理：一个平移不变核 K(x-y) 如果正定，那么它的傅里叶变换是某个概率分布。通过从这个分布中采样随机频率 ω，我们可以构造如下形式的随机特征：
φ(x) = [cos(ω₁ᵀx), sin(ω₁ᵀx), cos(ω₂ᵀx), sin(ω₂ᵀx), ...]

可以证明，对于高斯核 K_gauss(x,y) = exp(-‖x-y‖²/2)：
K_gauss(x,y) = 𝔼[φ(x)ᵀφ(y)]

这种基于三角函数的随机特征（Trigonometric Random Features，简称 TrigRF）在高斯核近似中表现出色，且误差随特征数量 m 的增大以 O(1/√m) 的速度收敛。

但问题在于，softmax kernel 和高斯 kernel 虽然有关系，但不一样。我们有：
SM(x,y) = exp(xᵀy) = exp(‖x‖²/2) · K_gauss(x,y) · exp(‖y‖²/2)

这意味着我们可以通过给高斯核的随机特征乘上 exp(‖x‖²/2) 来近似 softmax kernel。这就是论文中提到的 SM̂_m_trig 方法。

但这个故事有一个关键漏洞，我们马上就会看到。

## 六、为什么三角函数不行——负值特征的陷阱

如果三角随机特征能很好地近似高斯核，那通过指数乘子来近似 softmax kernel 似乎也是顺理成章的。但作者指出，这个方案在实践中**完全不 work**。原因如下。

Attention 机制本质上是在为每个 token 计算一个凸组合（convex combination）——把 V 矩阵的行按 attention 权重加权求和。这就要求 attention 的权重必须是非负的，且归一化后和为 1。

问题出在 φ(x) 本身。三角随机特征产生的是正负交替的值（因为 cos 和 sin 的输出范围是 [-1,1]），这意味着 φ(q_i) 和 φ(k_j) 的内积**可能为负**。虽然是近似，但这个负值会导致标准化器 D̂ 中出现负数或异常值。

更严重的是，当 softmax kernel 的真实值趋近于 0（即两个 token 几乎不相关）时，三角随机特征的近似方差**反而变得非常大**。这是因为在真实值接近 0 的区域，三角函数的正负振荡造成的相对误差极大。结果就是归一化矩阵 D̂ 可能出现负的对角线元素，导致训练完全失败，或者收敛到次优解。

作者在论文中提供了理论分析和实验验证。他们展示了用三角随机特征（SM̂_m_trig）来近似 softmax attention 时，训练曲线要么根本不收敛，要么模型质量严重下降。这解释了为什么在过去几年里，虽然随机特征方法已经被广泛用于核近似，但**从来没有人成功用它来近似标准的 softmax attention**——问题不在随机特征这个 idea 本身，而在于选择了错误的特征函数。

## 七、正随机特征（PRF）的核心构造

Performer 的关键贡献之一，就是设计了一套**保证非负**的随机特征映射（Positive Random Features，简称 PRF），用于无偏估计 softmax kernel。

构造过程非常巧妙。作者从以下引理出发：

SM(x,y) = exp(xᵀy) = 𝔼[exp(ωᵀx - ‖x‖²/2) · exp(ωᵀy - ‖y‖²/2)]

其中 ω ~ N(0, I_d)。如果令：
ξ(x) = exp(ωᵀx - ‖x‖²/2)

那么显然 ξ(x) 是个标量，且永远为正（指数函数的值域是 (0,+∞)）。于是 φ(x) = ξ(x)/√m 就是一个正随机特征映射。

更一般地，通过采样 m 个独立的 ω_i ~ N(0, I_d)，我们可以构造：
φ(x) = (1/√m) · [exp(ω₁ᵀx - ‖x‖²/2), ..., exp(ωₘᵀx - ‖x‖²/2)]

可以证明：
𝔼[φ(x)ᵀφ(y)] = SM(x,y) = exp(xᵀy)

这是无偏的，且 φ(x) 的每个分量都是严格正的。这就是 PRF 的魔力。

这个构造看起来很简单，但它是整个 FAVOR+ 方法的基石。因为特征值恒为正，attention 的归一化操作 D̂⁻¹ 就有了稳定的数学基础——不会出现负值，不会出现除零错误。

## 八、FAVOR+ 的完整算法

现在我们把所有拼图拼起来。FAVOR+（Fast Attention Via positive Orthogonal Random features plus）的完整算法如下：

**步骤 1：输入预处理。** 输入三个矩阵 Q, K, V ∈ ℝ^(L×d)。将 Q 和 K 的每一行除以 √[4]√d（即 d 的四次方根），等价于在 softmax kernel 中完成 1/√d 的缩放。

**步骤 2：采样随机矩阵。** 从标准正态分布 N(0, I_d) 中采样 m 个 d 维随机向量，构成一个 m × d 的随机矩阵 Ω。这是随机频率矩阵。

**步骤 3：计算正随机特征。** 对每个 query q_i 和每个 key k_j，计算：
φ(q_i) = (1/√m) · exp(Ω · q_i - ‖q_i‖²/2 · 1_m)
φ(k_j) = (1/√m) · exp(Ω · k_j - ‖k_j‖²/2 · 1_m)

这里 exp 是逐元素指数运算，1_m 是全 1 向量。注意这里每个分量都是正的。

**步骤 4：线性时间 attention 计算。** 得到 Q' ∈ ℝ^(L×m) 和 K' ∈ ℝ^(L×m) 后，按如下顺序计算：
- 先算 K'ᵀV ∈ ℝ^(m×d)（O(Lmd)）
- 再算 Q'(K'ᵀV) ∈ ℝ^(L×d)（O(Lmd)）
- 同时计算 D̂ = Q'(K'ᵀ1_L)（O(Lm)）
- 最后归一化：D̂⁻¹ · [Q'(K'ᵀV)]

整个过程的空间复杂度为 O(Lm + Ld + md)，时间复杂度为 O(Lmd)。当 r = m << L 时，这就是线性复杂度。

**步骤 5（可选）：正交化。** 使用 Gram-Schmidt 或 QR 分解，对随机矩阵 Ω 的行进行正交化。正交化后的随机特征虽然不再严格独立，但近似方差更低。这是 FAVOR+ 中 OR（Orthogonal Random）部分的来源。

这里有一个容易被忽略的工程细节：正交化应该对 Ω 的行做，而不是对 φ(x) 做。因为正交化是在随机频率层面上完成的，正交后的 ω_i 仍然可以用来计算 exp(ω_iᵀx)，只是这些 ω_i 之间不再是独立的，而是满足 ω_i · ω_j = 0（i≠j）。这个正交性降低了不同随机特征之间的协方差，从而直接降低了最终 estimator 的方差。

另外值得注意的是，在实际实现中，QR 分解通常比 Gram-Schmidt 更数值稳定，特别是在 m 较大时。作者在代码中使用的是 QR 分解。

## 九、正交随机特征：降低方差的关键

独立同分布（i.i.d.）采样在理论上很干净，但在实践中会有一个问题：采样的随机特征之间可能高度相关，导致近似方差较大。

为此，作者引入了**正交随机特征**（Orthogonal Random Features）。做法是对采样的随机矩阵 Ω 进行 Gram-Schmidt 正交化（或者使用 QR 分解），得到行与行之间正交的矩阵 Ω_orth。然后用这个正交矩阵来计算 PRF。

正交化为什么有效？直觉上，正交化保证了随机特征之间的多样性，避免了多个特征捕捉到相同方向的信息。从理论角度看，正交化降低了近似 estimator 的方差。

作者在附录中给出了严格的理论分析：对于相同的特征数量 m，正交随机特征的近似误差（均方误差）**严格小于**独立同分布特征的误差。具体来说，对于高斯核的近似，方差可以降低到原来的 1/m 量级。

这个分析从随机矩阵理论的角度出发，利用了正交随机矩阵的特殊性质。独立随机向量的分布是各向同性的，但正交约束引入了负相关，使得特征向量更均匀地分散在空间中。

正交化的代价是多少？Gram-Schmidt 正交化的时间复杂度是 O(m²d)。但对于典型的设置（m ≈ 128-256，d ≈ 64-512），这个代价和 attention 本身的计算量（O(Lmd)）相比可以接受。而且正交化只需要在做 forward 之前做一次，不会在每次 forward 中重复计算。

PRF 加正交化，就是 FAVOR+ 中 OR+ 部分（Orthogonal Random features plus）的全部含义。

## 十、单向 Attention 也能做：Fast Prefix-Sum 技巧

前面讨论的都是双向（bidirectional）attention，即 encoder 中使用的 full attention。但在 decoder 中，我们需要的是**单向（unidirectional/causal）attention**——每个 token 只能关注它之前的 token，不能"偷看"后面的 token。

对于标准 Transformer，实现 causal attention 只需要对 attention 矩阵 A 取下三角矩阵 tril(A)。但对于 FAVOR+，事情没有这么简单。因为 FAVOR+ 是先做矩阵乘法再取结果，attention 矩阵从未被显式构造过，无法直接做 mask。

作者提出了一个精巧的解决方案：**Fast Prefix-Sum 方法**。

核心观察是：第 i 个 token 的 causal attention 结果可以写为：
Att→(q_i, K, V) = (Σ_(j≤i) φ(q_i)ᵀφ(k_j) · v_j) / (Σ_(j≤i) φ(q_i)ᵀφ(k_j))

可以改写为：
= φ(q_i)ᵀ · (Σ_(j≤i) φ(k_j) · v_j) / (φ(q_i)ᵀ · Σ_(j≤i) φ(k_j))

注意这里的核心是计算**前缀和**：
- S_i = Σ_(j≤i) φ(k_j) ⊗ v_j（外积的累积和，形状为 m×d）
- Z_i = Σ_(j≤i) φ(k_j)（累积和，形状为 m×1）

然后对于每个 i：
Att→(q_i) = φ(q_i)ᵀ · S_i / (φ(q_i)ᵀ · Z_i)

这里关键的是 S_i 和 Z_i 的累积和可以**并行计算**。作者提出了一种基于二分法的并行 prefix-sum 算法，时间复杂度为 O(Lmd)（和双向 attention 相同），且完全可以在 GPU 上高效执行。

这意味着 FAVOR+ 对 causal attention 的支持没有额外的时间复杂度开销，只是将普通的并行 prefix-sum 策略用在了 attention 上。这是 Performer 能够完整替代标准 Transformer 的关键——它同时支持 encoder 和 decoder。

## 十一、理论保证有多强

Performer 的一个突出优势是它的理论保证。作者提供了三层理论分析：

**第一层：无偏估计。** 对于 softmax kernel Kₛₘ(x,y) = exp(xᵀy)，PRF 给出的是无偏估计。即对任意 x,y ∈ ℝᵈ：
𝔼[φ(x)ᵀφ(y)] = Kₛₘ(x,y)
这个结果可以直接从高斯分布的矩生成函数推导出来。对于用正交随机特征的版本，偏差也是接近零的。

**第二层：一致收敛性。** 作者证明了近似 attention 矩阵和真实 attention 矩阵之间的误差（以某种矩阵范数度量）随着特征数量 m 的增大而一致收敛到 0。具体地说，对于任意的 ε > 0，存在一个 m₀ = O(d log(d/ε)/ε²)，使得当 m > m₀ 时，以高概率有：
||Â - A|| ≤ ε
其中 Â 是 FAVOR+ 的近似 attention 矩阵（归一化前）。这个结果保证了在足够多的特征下，近似质量是有保障的。

**第三层：估计方差。** 这是最关键的保障。在值趋近于 0 的区域（即两个 token 基本不相关），PRF 的估计方差也**趋近于 0**。这与三角函数特征恰恰相反——后者在值趋近于 0 时方差反而爆炸。

更精确地说，作者证明了对于任意 x,y，有：
Var(φ(x)ᵀφ(y)) = O(Kₛₘ(x,y)² · (exp(‖x-y‖²/m) - 1))

这意味着当 Kₛₘ(x,y) → 0 时方差也 → 0。这是一个优良的性质，保证了 attention 中那些很小的权重（对应不相关的 token）不会被近似过程随机放大，从而保持了 attention 分布的稳定性。

这三层理论保证加在一起，使得 Performer 成为第一个在理论上能够**可靠地近似标准 softmax attention**的线性 Transformer。

## 十二、实验验证——从像素到蛋白质

作者在三个差异很大的任务上测试了 Performer 的效果，涵盖了图像、文本和生物信息学领域。

**任务一：Pixel-Prediction（像素预测）。** 使用 ImageNet 64×64 数据集。这是一个序列预测任务——将图像展平为像素序列，预测下一个像素的值。序列长度 L=4096。由于输入序列较长，标准 Transformer 在这个任务上已经非常吃力。Performer 不仅成功训练，而且性能与基线 Transformer 相当。在无偏近似 + PRF 的设置下，Performer 的 test perplexity 几乎和基线重合。

**任务二：文本建模。** 使用 LM1B 文本数据集进行语言建模。在这个任务上，Performer 与 Reformer（LSH attention）和标准 Transformer 进行对比。结果显示：Performer 在采样几十个随机特征（m = 64-256）时即可接近 Transformer 的 perplexity，而 Reformer 受限于 LSH 分桶的近似质量，在某些设置下性能不如 Performer。

**任务三：蛋白质序列建模。** Performer 的一个亮点应用是在蛋白质序列上的大规模建模。蛋白质序列的长度可以从几百到几千氨基酸残基，标准的 Transformer 二次复杂度使得在这个领域的应用非常受限。作者基于 JAX 框架实现了 Performer 的大规模训练，处理的总 token 量达到数十亿级别，序列长度远超标准 Transformer 的能力范围。具体来说，作者在 UniRef 蛋白质数据库上训练了多种规模的 Performer 模型，展示了在保持线性复杂度的同时，模型质量（perplexity）与标准 Transformer 相当。这个实验特别有说服力，因为蛋白质序列中的长程相互作用（如远距离氨基酸残基之间的结构折叠依赖）恰恰是标准 Transformer 的 O(L²) 复杂度无法规模化的场景。Performer 让这类生物信息学应用第一次可以扩展到全长度蛋白质序列。

**消融实验。** 作者还进行了详细的消融研究，包括：
- 正交随机特征 vs 独立同分布随机特征：正交化在所有设置下都降低了方差，对性能有一致的正向贡献。
- 不同特征数量 m 的影响：随着 m 增大，近似质量提高，在 m ≈ 128-256 时基本达到收敛。
- PRF vs TrigRF 的对比：TrigRF 在两个任务上都出现了训练失败或严重退化，证实了正特征的重要性。
- 训练稳定性分析：作者跟踪了训练过程中 attention 矩阵近似误差的变化，发现 FAVOR+ 的估计误差在训练过程中保持稳定，没有出现累积漂移。

**训练分析与性能数据。** 作者报告了 Performer 在 TPU 上的训练吞吐数据。对于长序列（L ≥ 4096），Performer 相比标准 Transformer 的训练速度有明显提升，且显存占用显著降低。例如，当 L=4096 且 d=64 时，标准 Transformer 的 attention 复杂度为 O(L²d)，Performer 为 O(Lmd)（m 为特征维度，如 256），差距随 L 增大而扩大。论文在 TPU 上验证了 Performer 在长序列下的训练吞吐显著优于标准 Transformer。


## 十三、与现有方法的系统对比

Performer 不是唯一尝试降低 attention 复杂度的论文。我们来做一个系统的对比。

**Linear Transformer（Katharopoulos et al., 2020）：** Linear Transformer 将 softmax 替换为 elu(x)+1 的核函数。它同样是线性复杂度（通过特征映射），但它的核函数不是标准 softmax，因此不提供对标准 attention 的近似。Linear Transformer 在实验中也表现出色，但它改变了对 attention 的定义，不能保证在其他任务上同样有效。相比之下，Performer 是直接近似标准 softmax attention，通过 fine-tuning 可以无缝替换标准 Transformer。

**Efficient Attention（Shen et al., 2018）：** 使用了一个不同的核（基于 ReLU 或 Softplus），同样改变了 attention 的定义。不支持 causal masking。

**Reformer 使用 LSH 来做稀疏 attention，attention 复杂度约 O(L log L · d)。而且 Reformer 强制 Q=K，限制了表达能力。它也没有提供对标准 attention 近似误差的理论保证。

**Sparse Transformer（Child et al., 2019）：** 使用固定的稀疏模式（固定步长 + 局部窗口）。复杂度 O(L√L) 或 O(L√L · d)。需要手写 CUDA kernel 来获得实际加速，且无法处理非局部的长程依赖。

**Longformer（Beltagy et al., 2020）：** 滑动窗口 + 全局 token 的稀疏 attention。复杂度接近 O(Lw) 其中 w 是窗口大小。同样需要自定义 GPU 操作。稀疏模式是人工设计的。

**Big Bird（Zaheer et al., 2020）：** 结合随机稀疏、局部窗口和全局 token 的注意力模式。复杂度 O(L(w + r + g)) 其中 w、r、g 是超参数。在长序列上表现出色，但仍然是修改后的 attention，不是对标准 softmax 的近似。

**Nyströmformer（Wang et al., 2021）：** 使用 Nyström 近似来估计 attention。复杂度 O(Ld²) 或 O(L²/m)（m 是 landmarks 数量）。但它的近似有偏，且 MSE 较大。

Performer 的独特定位在于：它是**唯一一个提供无偏或近乎无偏估计、有统一收敛保证、且直接近似标准 softmax attention 的线性 Transformer**。但这并不代表它一定优于所有其他方法，因为近似误差在极小的 m 下可能较大，且实际加速效果取决于具体实现和硬件优化。此外，如果某个任务本身就不需要标准 softmax attention（例如语言建模中局部依赖占主导），那么像 Longformer 或 BigBird 那样的稀疏 attention 可能更加高效。这也引出了论文中提到的另一个研究方向：是否可以利用 FAVOR+ 的泛化能力，在大规模任务上比较不同 kernel 的效果，找出特定任务的最优注意力核。

## 十四、有哪些局限性

每篇论文都有自己的局限。Performer 也不例外。

**第一，特征维度 r 的选择是个 trade-off。** 理论上 r 可以远小于 L 来实现线性复杂度，但在实践中，为了达到足够的近似精度，m 通常不能太小（m ≈ 64-256）。对于中等长度的序列（如 L=512），这个 m 和 L 差别不大，实际加速有限甚至可能更慢。换句话说，Performer 在**极长序列**上才能真正发挥优势。在 L < 1000 的序列上，优化后的标准 Transformer 可能更快。

**第二，prefix-sum 的 causal masking 实现不够高效。** 虽然 prefix-sum 在理论上是 O(Lmd)，但双向前缀和需要额外计算和存储，实际实现中可能带来一定的开销。作者承认了这一点，并指出优化 prefix-sum 实现是一个未来方向。

**第三，蒸馏 / fine-tuning 的成本。** Performer 可以 fine-tune 从标准 Transformer 初始化的参数，但这个过程本身需要额外的计算资源。对于已经训练好的标准 Transformer 模型，直接换成 Performer 架构后需要重新训练或 fine-tune。

**第四，内存占用问题。** 虽然 FAVOR+ 的时间复杂度是线性的，但它的空间复杂度 O(Lm + Ld + md) 中，Lm 这一项可能不小。如果 m=256 且 L=65536，光 Q' 和 K' 就需要 65536×256×2×4≈134MB（FP32），加上其他缓存可能接近 1GB。

**第五，无法处理任意核。** FAVOR+ 要求 attention kernel 可以通过随机特征映射分解为 φ(x)ᵀφ(y) 的形式。虽然这涵盖了 softmax、高斯核、以及一批常见的核，但并非所有 attention 机制都可以这样分解。

**第六，与其他加速技术的兼容性。** 作者在论文中简要提到了 FAVOR+ 可以和 reversible layers、cluster-based attention 等技术结合，但并未给出实验验证。实际工程中如何与这些技术一同部署还是一个开放问题。

## 十五、小结

Performer 是一篇追求"既要又要"的经典论文。它的贡献可以概括为五点：

**1. 解决了关键难题。** 在此之前，从未有人成功用随机特征映射来高效近似标准的 softmax attention。核心障碍是三角函数的正负振荡导致 attention 归一化不稳定。Performer 通过精心设计的正随机特征（PRF）优雅地解决了这个问题。

**2. 提供严格的算法。** FAVOR+ 不仅理论完备（无偏估计、一致收敛、低方差），而且实用：支持双向和单向 attention，同时能泛化到其他可核化的 attention 机制。

**3. 理论保障扎实。** 相比 Sparse Transformer、Reformer、Longformer 等方法，Performer 提供了最强的理论保证——无偏性、一致收敛性，以及最重要的是在低相关区域的低方差保证。

**4. 实验验证充分。** 从图像到文本到蛋白质，三个差异显著的领域验证了 FAVOR+ 的通用性。尤其是在蛋白质序列建模上的应用，展示了线性复杂度 Transformer 在科学领域的巨大潜力。

**5. 引出了后续工作。** Performer 是线性注意力浪潮中的重要里程碑。它之后涌现了大量跟进工作，包括更多样的正随机特征设计、基于核方法的注意力统一框架、以及在实际系统（如生物序列处理、长文档 NLP）中的工程落地。
