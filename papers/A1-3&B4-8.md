## 一、先搞清楚"问题在哪里"

2018 年 BERT 在 NLP 领域取得了巨大的成功，它的核心思想极其简单：随机 mask 掉句子中 15% 的词，然后用 Transformer 的输出通过一个 softmax 分类器去预测被 mask 掉的词。这个 masked language modeling（MLM）预训练任务让模型学到了双向的上下文表示，在下游任务上只需微调就能大幅超越此前的方法。

2020 年，Vision Transformer（ViT）证明了 Transformer 架构也可以直接用在图像分类上——把一张图切成 16×16 的 patch，展平后作为 token 序列喂给标准 Transformer encoder。这让人不禁想问：**能不能把 BERT 的那套 MLM 预训练直接搬到视觉上？**

但这件事没那么容易。BERT 能成功，有一个关键技术前提：**它有一个定义良好的词表。** 自然语言中的"词"或者 BPE 子词单元都有明确的离散 ID，softmax 分类器的输出空间就是这个词表。图像 patch 呢？一个 16×16 的 RGB 图像块有 16×16×3 = 768 个像素值，每个值都在 [0, 255] 范围内。不存在一个"图像词表"来定义所有可能的 patch 类别。

一个最直接的替代方案是把任务当成回归问题：mask 掉一些 patch，让模型去预测被 mask 的像素值（像素级重建）。但这会带来另一个问题——**像素级重建任务会让模型把大部分能力浪费在建模短程依赖和高频细节上**（如纹理、边缘等低级特征），而不是学习语义级别的表示。这就像让一个学生去抄写句子的每个字母，而不是理解句子的含义。

那么问题就变成了：**如何为图像 patch 构建一个类似 NLP 词表的离散化方案，从而让 BERT-style 的预训练能在视觉上真正发挥作用？**

这就是 BEiT 要解决的核心问题。

## 二、核心思路：视觉词汇表 + 掩码图像建模

BEiT 的回答非常巧妙：**先用一个离散变分自编码器（dVAE）把图像"分词"成离散的 visual tokens，这样就构造出了一个"视觉词汇表"；然后在这个视觉词汇表上做 masked image modeling（MIM）——随机 mask 图像 patch，让 ViT 去预测对应的离散视觉 token ID。**

这个思路在抽象层面和 BERT 完全一致：

```text
BERT:      句子中的词    → 随机 mask 15% → Transformer → 预测被 mask 的词
BEiT:      图像中的 patch → 随机 mask 40% → ViT        → 预测被 mask 的 visual token
```

区别在于：BERT 的词是天然离散的（"苹果"就是"苹果"），而 BEiT 的"视觉词"需要通过第一阶段学习得到。

具体的两阶段流程如下：

**第一阶段 — 学习 dVAE tokenizer（图像分词器）：**
1. 把图像输入一个 tokenizer 网络 $q_\phi(z|x)$ ，输出每个 patch 对应的离散 token ID（来自一个大小为 8192 的视觉词表）
2. 用 decoder 网络 $p_\psi(x|z)$ 从这些离散 token 重建原始图像
3. 由于 token 是离散的，梯度无法直接回传，用 Gumbel-Softmax 松弛来解决
4. 训练完成后，tokenizer 就能把任何一张图"翻译"成 14×14 的离散 token 网格

**第二阶段 — Masked Image Modeling（预训练 ViT）：**
1. 对同一张图，保留两个表示：raw pixels 作为输入（image patches），离散 token 作为预测目标（visual tokens）
2. 随机 mask 40% 的 image patches，用 [M] embedding 代替
3. 把被破坏的 patch 序列喂给 ViT backbone
4. 在 ViT 的输出上接一个 softmax 分类器（8192 类），预测被 mask 位置的 visual token ID
5. 优化目标是最大化正确 visual token 的 log-likelihood

这里有一个关键的反直觉点：**输入是 raw pixels（连续的），预测目标是离散 token ID（分类问题）。** 这和 BERT 一模一样——BERT 的输入是 token embedding（连续向量），预测目标是离散 token ID。区别只是视觉的"tokenization"多了一个 dVAE 步骤。

## 三、为什么不能直接预测像素？

你可能在想：何必这么麻烦？直接让模型预测被 mask patch 的像素值不就行了吗？

论文明确指出这个方案的问题：**像素级 recovery 任务会让模型倾向于建模短程依赖和高频细节。**

这背后的直觉是：如果你让一个模型去预测一个 mask 掉的 16×16 区域的 RGB 值，模型最"省力"的策略是学习邻域像素的平滑插值——左边是蓝色天空，右边也是蓝色天空，那中间被 mask 的部分大概率也是蓝色天空。这种"抄邻居"的策略确实能降低像素级 loss，但学不到高层的语义信息——模型不需要知道"这是一只猫"就能很好地预测邻近像素。

而预测离散 visual token 则不同。dVAE 的 tokenizer 已经把图像压缩到了高层语义空间——一个 visual token ID 对应的是"一块带条纹的橙色区域"或者"一双黑色的眼睛"这样的语义概念。要预测这些，模型必须理解图像的语义内容，而不是靠纹理插值。

**消融实验（Section 3.3）也证实了这一点：BEiT 的离散 token 预测方案显著优于像素级 auto-encoding。**

## 四、Blockwise Masking：不是随机 mask

BEiT 的另一个技术细节是块状掩码（blockwise masking），而不是随机 mask 个别 patch。

```text
随机 mask（不好的）:         块状 mask（BEiT 用的）:

  [ ][✓][ ][ ] [ ][ ][✓][ ]     [✓][✓][ ][ ] [ ][ ][ ][ ]
  [ ][ ][✓][ ] [✓][ ][ ][ ]     [✓][✓][ ][ ] [ ][ ][ ][ ]
  [✓][ ][ ][ ] [ ][ ][✓][ ]     [ ][ ][ ][ ] [ ][✓][✓][ ]
  [ ][✓][ ][ ] [ ][ ][ ][✓]     [ ][ ][ ][ ] [ ][✓][✓][ ]
```

其中 [✓] 是被 mask 的位置。

Blockwise masking 的算法（Algorithm 1）：

1. 初始化空集 $\mathcal{M}$
2. 重复以下步骤直到 $\mathcal{M}$ 中的 patch 数量超过 $0.4N$ （ $N$ 是总 patch 数）：
   - 随机选择一个块大小 $s$ ，范围在 [16, 0.4N-|M|] 之间
   - 随机选择长宽比 $r$ ，范围在 [0.3, 1/0.3]
   - 根据 $s$ 和 $r$ 计算块的宽和高： $a=\sqrt{s \cdot r}$ ， $b=\sqrt{s/r}$
   - 在图像网格中随机选择一个起始位置 $(t, l)$
   - 把这个块加入 $\mathcal{M}$

为什么要用块状 mask 而不是随机 mask？论文的解释是：**块状 mask 迫使模型利用更远的上下文信息来推理被 mask 的区域**。如果一个 patch 周围全是可见的，模型可以靠邻近像素的纹理来推断；但如果一整块区域都被 mask 了，模型必须从远处的可见 patch 中收集信息来理解这个区域是什么。

这和 BERT 的 n-gram masking 思路一致——mask 掉连续的词比 mask 单个词更难，迫使模型做更深层的语义推理。

## 五、VAE 视角的理论解释

论文在 Section 2.4 提供了一个优雅的理论视角：**BEiT 的整体两阶段训练可以看作一个变分自编码器（VAE）的证据下界（ELBO）优化。**

设 $x$ 为原始图像， $\tilde{x}$ 为 mask 后的图像， $z$ 为 visual tokens。ELBO of $p(x|\tilde{x})$ 为：

$$\log p(x|\tilde{x}) \geq \underbrace{\mathbb{E}_{z \sim q_\phi(z|x)}[\log p_\psi(x|z)]}_{\text{Stage 1: Visual Token Reconstruction}} - \underbrace{D_{KL}[q_\phi(z|x), p_\theta(z|\tilde{x})]}_{\text{Stage 2: Masked Image Modeling}}$$

其中：
- $q_\phi(z|x)$ ：图像 tokenizer，将像素映射为离散 visual token
- $p_\psi(x|z)$ ：decoder，从 visual token 重建原始图像
- $p_\theta(z|\tilde{x})$ ：MIM 预训练任务，从被破坏的图像恢复 visual token

**Stage 1（dVAE 训练）：** 学习 $q_\phi$ 和 $p_\psi$ ，最小化重建损失 $-\mathbb{E}_{z \sim q_\phi(z|x)}[\log p_\psi(x|z)]$ 。训练完成后固定 $q_\phi$ 和 $p_\psi$ ，并用每个 patch 的最可能 token $\hat{z}_i = \arg\max_z q_\phi(z|x_i)$ 代替原始的离散分布。

**Stage 2（MIM 预训练）：** 学习 $p_\theta(z|\tilde{x})$ ，即等式中的 KL 散度项。在这一阶段， $q_\phi$ 和 $p_\psi$ 被固定，训练目标简化为最大化 $\log p_\theta(\hat{z}|\tilde{x})$ 。这就是公式 (1) 中的 MIM 目标。

这个视角清晰地解释了 BEiT 为什么是一个自洽的整体：**两个阶段联合优化同一个 ELBO，第一阶段构造视觉"语言"（visual tokens），第二阶段在这个"语言"上做 BERT 式预训练。**

## 六、架构细节

### 6.1 Backbone Transformer

BEiT 的 backbone 网络直接沿用了 ViT 的标准 Transformer encoder。这使得实验结果可以直接和 ViT、DeiT 等在架构层面进行公平比较。

具体配置：
- **Image patches：** 224×224 图像 → 14×14 网格，每个 patch 16×16 像素
- **Visual tokens：** 同样 14×14 网格，词表大小 $|\mathcal{V}|=8192$
- **Patch embedding：** 将 16×16×3=768 维的 patch 展平后线性投影到 $D$ 维
- **Special token：** 在序列开头添加 [S] 特殊 token（类似 BERT 的 [CLS]）
- **Position embedding：** 标准可学习的 1D 位置编码
- **Transformer 层数：** BEiT-Base 为 12 层（ $L=12$ ），隐藏维度 $D=768$ ，12 个 attention head

前向过程：
1. 输入图像 → 分割为 14×14 = 196 个 patch
2. 每个 patch 展平并线性投影 → 196 个 D 维 patch embedding
3. 添加 [S] token（最终序列长度 197）
4. 加上可学习位置编码
5. 通过 L 层 Transformer Block
6. 最后一层输出： $H^L = [h_{[S]}^L, h_1^L, ..., h_N^L]$

### 6.2 预训练配置

- **预训练数据：** ImageNet-1K（约 120 万张图像）
- **优化器：** AdamW
- **最大学习率：** 1.5e-3（经过 warmup 后 cosine 衰减）
- **Batch size：** 2,048
- **训练轮数：** 300 epochs
- **Warmup：** 10 epochs
- **Weight decay：** 0.05
- **Masking ratio：** 40%（即大约 78 个 patch 被 mask）

为什么用 ImageNet-22K 而不是 ImageNet-1K 做预训练？论文的动机是模拟 BERT 在大规模无标注语料上预训练的场景。ImageNet-22K 虽然也是有标签的（21K 类），但在 BEiT 的 MIM 预训练中 **完全没有使用标签**——它只把图像当作无标注数据来使用。这与后来直接用 ImageNet-1K 预训练的 MAE 形成了对比。

## 七、微调与下游任务

### 7.1 ImageNet-1K 图像分类

预训练完成后，BEiT 的 encoder 参数可以直接用于下游任务微调。对于图像分类，使用平均池化（average pooling）聚合所有 patch 的表征，再接一个线性分类头。

**微调配置：**
- **数据集：** ImageNet-1K（128 万张，1K 类）
- **输入分辨率：** 224×224（或 384×384）
- **优化器：** AdamW
- **最大学习率：** 2e-5
- **Batch size：** 1,024
- **训练轮数：** 100 epochs
- **Weight decay：** 0.05

**主要结果（Table 1）：**

| 模型 | 预训练数据 | 输入尺寸 | Top-1 准确率 |
|------|-----------|---------|-------------|
| DeiT-B (from scratch) | — | 224² | 81.8% |
| ViT-B/16 (from scratch) | — | 224² | 79.6% |
| **BEiT-Base** | **ImageNet-22K (无标签)** | **224²** | **83.2%** |
| BEiT-Base (384²) | ImageNet-1K | 384² | 84.6% |
| ViT-L/16 (supervised ImageNet-22K pretrain) | ImageNet-22K | 384² | 84.6% |
| **BEiT-Large (224²)** | **ImageNet-1K** | **224²** | **85.2%** |
| **BEiT-Large (384²)** | **ImageNet-1K** | **384²** | **86.3%** |

关键发现：

1. **BEiT-Base（83.2%） 显著超越从头训练的 DeiT-B（81.8%）**。注意 DeiT 在 81.8% 时已经用了大量数据增强（RandAugment、Mixup、CutMix 等），而 BEiT 的 83.2% 是在更简单的微调设置下达到的。这个差距完全来自 MIM 预训练。

2. **BEiT-Large（85.2%）只用 ImageNet-1K 微调 224²，就超越了 ViT-L（84.6%）在完整的 ImageNet-22K 上有监督预训练+微调到 384² 的效果。** 这是一个非常有力的结果——自监督预训练在有足够规模时，可以超越有监督预训练。

3. **BEiT 和有监督预训练是互补的。** 论文提到如果先用 ImageNet 标签做中间微调再微调下游任务，性能还可以进一步提升。这说明 BEiT 学习到的表示和有监督学习不冲突，而是互相补充。

### 7.2 ADE20K 语义分割

BEiT 在语义分割上也做了实验，采用 SETR（ViT-based 语义分割框架）作为 baseline。

| 模型 | 预训练 | mIoU |
|------|--------|------|
| SETR-Large (ViT-L) | 监督 ImageNet-22K 预训练 | 50.3 |
| **BEiT-Large** | **自监督 MIM** | **53.3** |
| BEiT-Large + intermediate fine-tuning | 自监督 MIM + ImageNet 标签 | **54.0** |

BEiT-Large 在 ADE20K 上的 53.3 mIoU 显著超越了监督预训练的 SETR-Large（50.3 mIoU），提升了 3 个百分点的 mIoU。加上中间微调后进一步提升到 54.0 mIoU。这验证了 MIM 预训练学到的表示对于 dense prediction 任务同样有效。

## 八、消融实验与分析

### 8.1 Masking 策略的影响

论文对比了不同的 masking 策略：

| 策略 | ImageNet Top-1 |
|------|---------------|
| 基线（blockwise masking, 40%） | [见论文 Table 3] |
| 随机 patch masking | ↓ |
| 像素级回归（预测 raw pixels） | ↓ |
| 不同 masking ratio | [见论文 Figure 3] |

关键的消融结论：**Blockwise masking 优于随机 mask**，且 **预测离散 visual token 远优于预测 raw pixels**。这验证了 BEiT 的两个核心设计选择的必要性。

### 8.2 Self-Attention 可视化

论文展示了一个非常有趣的现象：**BEiT 在没有使用任何人类标注的情况下，self-attention 机制自动学习到了区分语义区域和物体边界的能力。** 这一结果是通过可视化 [S] token 的 self-attention 权重得到的——[S] token 在 ViT 中会与其他所有 patch 计算注意力，其注意力分布自然地反映了哪些 patch 在语义上属于同一区域。

这和 DINO（2021 年稍后发表的另一篇自监督 ViT 论文）中观察到的"自监督 ViT 自动学到语义分割能力"的现象是一致的，但 DINO 和 BEiT 都观察到了这一现象。

### 8.3 微调收敛速度与稳定性

论文指出，经过 BEiT 预训练后，下游任务的微调收敛速度更快、训练更稳定。这意味着实际使用 BEiT 时，可以用更少的微调 epoch 获得更好的结果，降低了训练成本。

## 九、从 VAE 视角理解为什么 BEiT 有效

从 VAE 理论的视角，BEiT 的成功可以归结为三个层面的设计：

**1. 离散 token 提供了语义级别的监督信号。**

如果 MIM 的目标是预测 raw pixels，误差信号都来自像素级的 MSE。这个信号中，95% 的信息都是关于低级纹理的（"这个像素是 128 还是 129 的蓝色"），只有 5% 是关于语义的。dVAE 的离散化过程相当于一个信息瓶颈——它把大量低级的像素细节丢弃了，只保留了语义级别的信息。MIM 在这种压缩后的监督信号上训练，迫使模型关注语义理解。

**2. 两阶段训练刻意解耦了两个学习目标。**

第一阶段只学"如何把图像编码为离散 token"（数据压缩），第二阶段只学"从上下文推测被 mask 的 token"（语义理解）。这两个目标如果一起学习（end-to-end），可能会互相干扰——优化重建 loss 和优化分类 loss 的需求是冲突的。解耦后，每个阶段的目标都足够清晰。

**3. Blockwise masking 增加了任务难度，迫使模型做语义推理。**

随机 mask 可以让模型靠局部纹理线索来"作弊"。Blockwise masking 通过 mask 掉连续区域，迫使模型必须理解图像的全局语义结构——"这里有眼睛、有耳朵、有尾巴，所以这应该是一只猫，被 mask 的区域也应该是猫的一部分"。

## 十、时间线位置

```text
2017.06  Transformer (Vaswani et al.)              ← 序列建模基础
   ↓
2018.10  BERT (Devlin et al.)                       ← MLM 预训练在 NLP 的成功
   ↓
2020.10  ViT (Dosovitskiy et al.)                   ← Transformer 首次用于图像分类
   ↓
2021.01  DeiT (Touvron et al.)                      ← 训练策略改进，不依赖超大规模数据
   ↓
2021.06  BEiT (Bao et al.)                          ← ★ 本篇 ★ 第一个视觉 BERT
   ↓
2021.11  MAE (He et al.)                            ← 简化 MIM，像素级预测 + 高 masking ratio
   ↓
2022.03  SimMIM (Xie et al.)                        ← 简化 MIM，像素级回归
   ↓
2023.01  EVA (Fang et al.)                          ← MIM + CLIP 融合
```

BEiT 出现在 2021 年 6 月。这个时间点非常关键：

- 2020 年 10 月 ViT 发布后，社区知道 Transformer 可以做图像分类，但训练需要海量数据（JFT-300M）
- 2021 年 1 月 DeiT 通过蒸馏和数据增强解决了数据饥渴问题，但架构设计没有根本性改进
- **2021 年 6 月 BEiT 提出了第一个真正意义上的视觉 BERT 方案，把自监督预训练的力量引入视觉 Transformer**

在 BEiT 之后，MIM 范式快速演化：
- **MAE（2021.11）** 用了更激进的设计：不对称 encoder-decoder 架构、75% 的 masking ratio、像素级预测。MAE 证明了"用 raw pixels 做预测也可以很好"——关键是 masking ratio 要足够高（mask 掉大部分 token），迫使模型做语义推理而不是纹理插值。
- **SimMIM（2022.03）** 进一步简化，直接预测 raw pixels，效果也接近 BEiT。
- **EVA（2023.01）** 用 MIM + CLIP 特征蒸馏大范式，把 MIM 推到了新高度。

BEiT 作为开山之作，定义了 MIM 的基本范式：离散 token 预测 + blockwise masking + 两阶段训练。后续工作在这个框架内做了各种简化和改进。

## 十一、技术树中的位置

在本技术树中，BEiT 处于两个分支的交汇点：

**A1-3（encoder-only 预训练范式 · 视觉迁移支）：** 这是 NLP 中的 encoder-only 预训练（BERT 路线）向视觉迁移的第一步。在 BERT 之后，社区一直有一个自然的想法：这种"双向 mask 预测"的预训练范式能否超越模态的限制？BEiT 给出了肯定的答案，证明了这个范式可以在视觉上同样有效。

**B4-8（masked image modeling）：** 这是自监督视觉表征学习的一个新范式。在此之前，主流的方法是 contrastive learning（SimCLR, MoCo, BYOL, DINO）和 self-distillation（DeiT）。BEiT 开辟了第三条路——把 BERT 的 denoising auto-encoding 理念搬到视觉，用 mask-and-predict 的方式进行自监督学习。

具体看 B4-8 这条路线：

```text
Contrastive Learning (SimCLR, MoCo)   ← 对比学习
   ↓
Non-Contrastive (BYOL, SimSiam, DINO) ← 非对比学习
   ↓
Masked Image Modeling:
   ├── BEiT (2021.06)      ← ★ 本篇 ★
   ├── MAE (2021.11)       ← 像素级预测 + 高 mask ratio
   ├── SimMIM (2022.03)    ← 更简化的像素级回归
   └── EVA (2023.01)       ← MIM + CLIP 蒸馏
```

BEiT 在这个谱系中的定位是 **bridge node（桥梁节点）**——它把 NLP 中验证非常成功的 MLM 范式翻译到了视觉领域，建立了一座从 NLP 预训练到视觉预训练的桥梁。这座桥梁的核心设计（离散化 + mask-and-predict）定义了整个 MIM 范式的基本框架。

## 十二、可扩展性讨论

BEiT 的架构选择对可扩展性既有优势也有约束：

### 12.1 优势

1. **dVAE 的 tokenizer 是预训练好后固定的。** 这意味着在不同规模上训练 ViT 时，不需要重新训练 tokenizer。这一点在后来 MAE 中得到了延续（MAE 的 tokenizer 就是 identity mapping——不需要 tokenizer）。

2. **MIM 预训练天然支持扩展到更大的模型和数据。** 和对比学习（需要精心设计负样本）不同，MIM 只需要 mask 和预测，不需要负样本对。这在扩展到大规模模型时是一个重要优势——不需要担心负样本采样的 bias 或 batch size 限制。

3. **视觉词表一旦学好就是通用的。** dVAE 在大规模数据上训练好后，可以用于任何后续的 MIM 预训练和下游任务。这个设计理念和 NLP 中预训练 BPE 词表的思路一致。

### 12.2 约束

1. **两阶段训练增加了工程复杂度。** 需要先训练一个 dVAE tokenizer，然后才能开始 MIM 预训练。虽然可以直接复用 DALL-E 的 tokenizer（论文就是这么做的），但如果要自己训练，会增加成本。

2. **dVAE 的 tokenizer 质量成为上限。** 如果 tokenizer 把语义信息丢失了，MIM 阶段再努力也学不到好的表示。论文对比了 DALL-E 的 tokenizer 和自实现的 tokenizer，发现 DALL-E 的 tokenizer 更好——显示 tokenizer 质量对下游性能有显著影响。

3. **词表大小 8192 的选择。** 这个选择来自 DALL-E。如果词表太小，visual tokens 无法表达图像中的丰富语义；如果太大，softmax 分类的计算开销会很大。论文没有对这个超参进行系统消融。

4. **计算开销。** MIM 预训练时，每个被 mask 的 patch 都需要过一个 8192 类的 softmax 分类器。虽然这比像素级 MSE 计算更复杂，但论文的消融实验证明这个代价是值得的。

## 十三、局限性与后续方向

### 13.1 依赖外部 tokenizer

BEiT 的 MIM 依赖一个 dVAE tokenizer，而这个 tokenizer 来自另一个独立的预训练过程。这意味着：
- **tokenizer 引入了一个额外的超参空间**（词表大小、网络架构、训练数据等），这些超参的选择都会间接影响 MIM 的效果
- **tokenizer 的训练需要额外数据。** DALL-E 的 dVAE 是在 2.5 亿张图像上训练的，而 BEiT 的 MIM 预训练在 ImageNet-1K 上。这种"更大的先验"（tokenizer 在更多数据上学到）和"下游预训练"的数据规模不匹配问题值得关注。

MAE 完全绕开了这个问题——直接用 raw pixels 做预测，不需要任何 tokenizer。这证明了 tokenizer 不是 MIM 的必要条件，关键在于 training objective 的设计和高 masking ratio 的配合。

### 13.2 40% masking ratio 是不是最优

BEiT 固定使用 40% 的 masking ratio，而没有系统研究不同 ratio 的影响。后续的 MAE 发现 75% 才是最优的——mask 掉大部分 token 才是关键。为什么 BEiT 不需要那么高的 mask ratio？因为它的预测目标是离散 token（已经压缩后的表示），比预测 raw pixels 更容易，所以不需要那么高的 masking ratio 来增加难度。

### 13.3 只在 ImageNet-1K 上预训练

BEiT 使用 ImageNet-1K（120 万张图像）做预训练，在 ImageNet-22K 上微调的 BEiT+ 变体进一步提升了性能，没有尝试更大规模的数据。这可能是工程资源的限制。后续的 iBOT、EVA、DINOv2 等都在更大规模的数据上做了预训练，验证了 MIM 范式在更大数据规模下依然有效。

### 13.4 和对比学习的关系

BEiT 是纯的 MIM 方法，没有引入对比学习的组件。后续的 iBOT（2021）将 BEiT 的 MIM 和 DINO 的自蒸馏结合起来，在同一个框架中同时做 mask prediction 和 contrastive learning，效果显著优于单独使用任何一种。这提示 MIM 和对比学习可能不是竞争关系，而是互补关系。

## 十四、范式意义

**BEiT 的范式意义：它是视觉 BERT 的开山之作，第一次把 masked language modeling 的成功经验成功迁移到了视觉领域。**

具体来说，BEiT 完成了以下几件重要的事：

### 14.1 证明了 BERT-style 预训练在视觉上可行

在 BEiT 之前，社区有一个隐性的假设：图像和语言本质不同，BERT 的 MLM 范式可能不适合视觉。BEiT 用实验证明了这个假设是错的——只需要引入一个视觉 tokenizer 来"翻译"图像为离散 token，BERT 范式就可以在视觉上产生巨大的效果提升。

这个证明的连锁反应是巨大的：它开启了一整条 MIM 研究路线，催生了 MAE、SimMIM、iBOT、EVA 等一系列后续工作。

### 14.2 挑战了对比学习在视觉自监督范式中的主导地位

在 BEiT 之前（2020-2021 年初），视觉自监督学习几乎被对比学习范式（SimCLR、MoCo、BYOL、SwAV、DINO）主导。BEiT 提供了一个完全不同的路径——基于 denoising auto-encoding 的生成式预训练，而不是基于判别式的对比学习。这打破了对比学习一家独大的局面，丰富了整个领域的研究工具箱。

此后，社区逐渐形成了共识：**对比学习和 MIM 各有优势，互补使用效果更好。**

### 14.3 连接了 NLP 和 CV 的预训练范式

BEiT 最深远的影响可能不是技术层面的，而是**概念层面**的：它把 NLP 和 CV 的预训练范式拉到了一条主线上。BERT 的思路可以用于视觉，MIM 是视觉版的 MLM——这种类比不仅帮助理解，更重要的是它暗示了一个更深层的可能性：也许 NLP 和 CV 的底层原理是相通的，差异只在于数据的 tokenization 方式。

这种范式统一的想法，在后来多模态 foundation model（如 CLIP、Flamingo、Kosmos-1、LLaVA）中得到了更充分的体现。

### 14.4 分类：bridge node

在技术树中，BEiT 是一个 **bridge node（桥梁节点）**——它连接了 NLP 的 MLM 范式和视觉的自监督预训练，第一次让 BERT 式的预训练在视觉上真正落地（而不是仅仅作为概念类比）。

这不是 foundation node（基础节点），因为 BEiT 的开创性在于"首次证明可行性"，而 MAE 等后续工作才把这个范式变成了真正的通用方案。但如果没有 BEiT 的先导验证，MAE 的设计可能会被大大推迟。

## 十五、小结

直白说，BEiT 的重要性不只在于它提升了 1-2 个百分点的 ImageNet 准确率。它做得最重要的一件事是：**证明了 BERT 可以"看"懂图像。**

在它之前，视觉 Transformer 的自监督预训练走的都是对比学习路线——通过比较图像之间的相似性来学习表示。BEiT 提出了一个全新的思路：**让视觉 Transformer 像 BERT 读句子一样，去"填空"被 mask 掉的图像区域。** 这个思路的简洁性和优雅性在于——它几乎没有对 BERT 的原始设计做任何颠覆性的修改，只是多了一个将图像转换为"视觉词汇"的步骤。

这个"差不多就行"的翻译，最终带来了显著的效果提升：

1. **BEiT-Base（83.2%）超越 DeiT（81.8%）从头训练**，证明了 MIM 预训练的价值
2. **BEiT-Large（85.2%）超越 ViT-L 的 ImageNet-22K 有监督预训练（84.6%）**，证明了自监督可以做得比有监督更好
3. **[S] token 的 self-attention 自动学到语义分割**，证明了模型学到的是真正的语义理解而不是低级特征匹配

说它改变了视觉 Transformer 的命运也不为过——在 BEiT 之后，MIM 成为视觉自监督预训练的主流范式之一，各大实验室都在这个框架上做优化，efficiency 更高、效果更好的变体不断涌现。而 BEiT 作为第一条路，它的贡献已经超出了论文本身的实验数字。
