# BEiT: BERT Pre-Training of Image Transformers（2021）

**论文：** [arXiv](https://arxiv.org/abs/2106.08254) · Bao et al. · 2021

---

## 一、先搞清楚坑在哪

BERT 在 NLP 上的成功证明了一个范式：**在大规模无标注数据上做 Masked Language Model 预训练，然后 fine-tune 到下游任务**。但同样的问题在视觉上一直没有答案：**图像能不能也做类似 BERT 的 MLM 预训练？**

直接原因很直观：NLP 里的词是离散的（来自固定词表），图像的 pixel 是连续的、高维的空间信号。BERT 预测 [MASK] 后的词——输出是 30k 类别的分类问题；如果用同样的方法预测 [MASK] 后的像素，输出是 224×224×3 = 150528 个连续值的回归问题——难度完全不同，毫无意义。

## 二、已有方法的真正问题

ViT（Dosovitskiy et al., 2020）证明在 3 亿图片上预训练后，Transformer 可以超越 CNN。但 JFT-300M 是 Google 的内部数据，绝大多数研究者用不到。

当时最好的自监督视觉预训练方法是对比学习（SimCLR、MoCo、BYOL、DINO）——拉近同一图片的不同增强视图，推远不同图片。对比学习的核心依赖：
- 精心设计的数据增强（决定了学到的特征类型）
- 大 batch size 或 momentum encoder（MoCo/BYOL 缓解了这个问题）
- 负样本（SimCLR 需要大量负样本）

对比学习的问题是：**它学到的是图片级别的判别能力，不是像素级别的理解能力**。一张猫的图片和一个桌子的图片被拉远，但模型并不知道"猫"是什么——它只知道这两张图片不一样。

## 三、BEiT 的核心思路

一句话：**把图像建模成离散的视觉 token，然后直接套用 BERT 的 MLM 来做预训练。**

三个关键设计：
1. **Image Tokenizer（DALL-E 的 dVAE）**：把 224×224 的图片编码成 14×14=196 个离散 token（来自 8192 个视觉词表的索引），就像 BERT 把句子编码成 WordPiece token 一样
2. **Masked Image Modeling（MIM）**：随机 mask 40% 的图像 patch（远高于 BERT 的 15%），让 Transformer encoder 从可见 patches 中预测被 mask 的视觉 token
3. **Patch-Level 表示**：预训练后，模型学到的是每个 patch 的语义表示——不仅可以用于分类（取 [CLS] token），还可以用于检测和分割（取 patch 特征）

## 四、网络架构

### 整体流程

```
预训练阶段：
┌──────────────────────┐     ┌──────────────────┐
│  Image 224×224       │     │ 原始图像          │
│  → Patch 16×16       │     │  (用于 tokenizer)  │
│  → Mask 40% patches  │     └────────┬─────────┘
│  → ViT Encoder       │              │
│  → 预测 masked tokens │     ┌────────▼─────────┐
└──────────────────────┘     │ dVAE Tokenizer    │
         │                    │ 图像→14×14离散token│
         ▼                    │ 8192 视觉词表      │
  MLM Head (8192 classes)    └──────────────────┘
         │
         ▼
  Cross-Entropy Loss (预测被 mask 的视觉 token)
```

### 架构组件

**1. Image Tokenizer（dVAE）。** 使用 DALL-E（Ramesh et al., 2021）训练好的离散变分自编码器：编码器把 224×224 图像压缩成 14×14=196 个离散 code，每个 code 是 8192 个视觉 token 中的一个索引。解码器从离散 code 重建图像。

tokenizer 是预训练好的、冻结的——不参与 MIM 的训练。损失只来自预测 masked tokens，tokenizer 本身不更新。

**2. ViT Encoder。** 标准 ViT-Base 架构（12 层、768 dim、12 heads）。输入处理：
- 16×16 patch embedding → 196 个 patch
- 随机 mask 40% 的 patch（替换为可学习的 mask embedding）
- [CLS] token + patch embeddings + 位置编码

**3. MLM Prediction Head。** 在 ViT encoder 的输出上接一个线性层（768→8192），预测每个 masked patch 的视觉 token 类别。

### Fine-tune 阶段

去掉 MLM head，接下游任务头：
- **分类**：取 [CLS] token，接分类头 → 和 ViT 一样
- **检测/分割**：取所有 patch 的输出，接 FPN/UperNet → 和 Swin 一样

## 五、核心贡献

### 贡献 1: MIM（Masked Image Modeling）

**动机**：BERT 的 MLM 在 NLP 上前所未有地有效，但图像是连续信号不能直接套用。

**方案的推导过程**：
1. 图像是连续的 → 需要先离散化 → 需要 tokenizer
2. 用 VQ-VAE / dVAE 做离散化 → 得到视觉词表
3. 对 patches 做 MLM → 和 BERT 一样，预测被 mask 的 token

**关键参数**：
- Mask 比例：40%（远高于 BERT 的 15%）。为什么？因为图像包含大量冗余信息（相邻像素高度相关），15% 的 mask 太简单——模型可以从周围像素轻松推断
- Blockwise masking：mask 不是随机选单个 patch，而是随机选矩形区域（block）。这强制模型理解更大范围的上下文
- Tokenizer vocab size：8192（和 DALL-E 一致）

### 贡献 2: Blockwise Masking

```
随机 mask（15%, BERT风格）：         Blockwise mask（40%, BEiT风格）：
┌──┬──┬──┬──┬──┬──┐                ┌──┬──┬──┬──┬──┬──┐
│  │  │▓▓│  │  │  │                │  │  │▓▓│▓▓│▓▓│▓▓│
├──┼──┼──┼──┼──┼──┤                ├──┼──┼──┼──┼──┼──┤
│  │▓▓│  │  │▓▓│  │                │  │  │▓▓│▓▓│▓▓│▓▓│
├──┼──┼──┼──┼──┼──┤                ├──┼──┼──┼──┼──┼──┤
│  │  │  │  │  │  │                │▓▓│▓▓│▓▓│▓▓│  │  │
├──┼──┼──┼──┼──┼──┤                ├──┼──┼──┼──┼──┼──┤
│▓▓│  │  │▓▓│  │  │                │▓▓│▓▓│▓▓│▓▓│  │  │
└──┴──┴──┴──┴──┴──┘                └──┴──┴──┴──┴──┴──┘
     易从周围推测                           难，需要语义理解
```

Blockwise masking 的实现：随机生成一个 aspect ratio 随机的矩形，它的面积占图片的 40%。然后把这个矩形覆盖的区域全部 mask 掉。

**直觉**：如果只 mask 单个 patch，相邻 patch 的颜色和纹理几乎一样，模型只需要"填充"而不需要理解。Blockwise mask 强迫模型学会**语义级别的理解**——比如 mask 掉的是一个狗头，模型需要从狗身体和背景推断这里有狗头。

### 贡献 3: 预训练-微调的一致性

BEiT 的预训练和 fine-tune 使用完全相同的 Transformer encoder（ViT-Base），只是预训练时多了一个 MLM head。这和 BERT 的设计哲学完全一致。

对比之下，对比学习方法（SimCLR/MoCo）在预训练阶段需要的数据增强（裁剪、颜色失真、旋转）在 fine-tune 时完全不用，导致预训练和 fine-tune 之间的 mismatch。

## 六、实验结果

### ImageNet-1K 分类（Table 1）

| 模型 | 预训练数据 | 预训练方法 | ImageNet top-1 |
|------|-----------|-----------|---------------|
| ViT-B | JFT-300M | Supervised | 86.8 |
| ViT-B | ImageNet-1K | From scratch | 79.9 |
| DeiT-B | ImageNet-1K | Distillation | 83.8 |
| **BEiT-B** | **ImageNet-1K + DALL-E dVAE** | **MIM** | **85.2** |

BEiT-B 的 85.2% 比 DeiT-B 的 83.8% 高出 1.4 个百分点，而且**不需要 ImageNet 标签**（预训练阶段完全是无监督的），只用了 ImageNet-22K 上预训练好的 dVAE tokenizer。

### 语义分割（Table 3）

| Backbone | 方法 | ADE20K mIoU |
|---------|------|-------------|
| DeiT-B | UperNet | 47.1 |
| **BEiT-B** | **UperNet** | **48.4** |

BEiT 在分割任务上的提升验证了 patch-level MIM 预训练学到了比分类更好的空间特征。

### 消融实验关键发现

1. **Mask 比例**：40% 最优。低于 30% 任务太简单，高于 50% 信息太少学不到
2. **Blockwise vs random masking**：Blockwise 比 random 高 0.8%
3. **Tokenizer 的影响**：dVAE tokenizer 比 VQ-VAE 略好
4. **冻结 vs 更新 tokenizer**：冻结效果更好（更新 tokenizer 反而让编码退化）

## 七、位置

### 前驱工作

- **[Devlin et al., 2019] BERT**：BEiT 的核心灵感来源。MLM → MIM，完全对应
- **[Ramesh et al., 2021] DALL-E**：提供 dVAE tokenizer，是 BEiT 的关键基础设施
- **[van den Oord et al., 2017] VQ-VAE**：离散化图像表示的基础技术

### 后续影响

- **MAE（He et al., 2022）**：在 BEiT 的基础上，MAE 去掉了 tokenizer，直接在像素空间做 masked reconstruction，而且只对可见 patches 做编码（非对称设计）。
- **MaskFeat（Wei et al., 2022）**：用 HOG 特征作为预测目标替代离散 tokenizer。
- **SimMIM（Xie et al., 2022）**：简化版 MIM，直接从 raw pixels 回归。
- **EVA（Fang et al., 2022）**：把 BEiT 的 MIM 扩展到更大的模型和数据规模。
- BEiT 开启了"BERT-style 视觉预训练"这个方向，和对比学习路线并列为视觉自监督学习的两大主流。

## 八、局限

1. **依赖外部 tokenizer。** BEiT 需要 DALL-E 的 dVAE tokenizer（在 250M 图文对上训练的）。这意味着 BEiT 的预训练间接依赖了多模态数据。MAE 通过直接预测像素绕开了这个问题。

2. **预训练计算成本。** MIM 需要对所有 patch（包括可见和 masked）做编码，效率低。MAE 的非对称编码（只编码可见 patches）正是为了解决这个问题。

3. **不适用于高分辨率输入。** 和 ViT 一样，BEiT 的 $O(N^2)$ 复杂度限制了高分辨率应用。

## 九、小结

BEiT 证明了**BERT 的 MLM 预训练可以直接迁移到视觉领域**，关键在于找到一个合适的离散化方案（dVAE tokenizer）。它开启了 masked image modeling（MIM）这一研究方向，直接启发了 MAE、MaskFeat、SimMIM、EVA 等一系列工作。

在 BEiT 之后，视觉预训练的格局从"对比学习一家独大"变成了"对比学习 + MIM 双轨并行"。
