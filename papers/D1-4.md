# Learning Transferable Visual Models From Natural Language Supervision（2021）

**论文：** [arXiv](https://arxiv.org/abs/2103.00020) · Radford et al. · 2021（CLIP）

---

## 一、先搞清楚坑在哪

2021 年之前的计算机视觉，工作流程基本是这样的：先拿 ImageNet 或者其他人工标注数据集预训练一个模型，再针对具体任务做 fine-tune。这套流程有两个根深蒂固的假设。

第一个假设是 **"标注类别是固定的"**。ImageNet 有 1000 个类，JFT-300M 做了 18291 个类，Instagram hashtag 预训练覆盖了几个万量级的标签——不管做多大，始终是一个封闭集合。想识别一个不在训练集里的类别？得重新收集标注数据、重新训练或 fine-tune。

第二个假设是 **"需要人工标注才能学"**。高质量的数据标注（每张图一个 ground-truth label）又贵又慢。ImageNet-1K 的 128 万张标注图片背后是大量的人力成本。而同期的 NLP，GPT 系列、BERT 等模型已经证明了可以直接从互联网文本中学习，不需要人工标注。

这种对比让人忍不住想问：**视觉为什么不能用自然语言来监督？**

## 二、已有方案的根本问题

在 CLIP 之前，用文本做视觉学习的尝试已经有很多了。

**时序线回顾：**
- Mori et al. (1999)：训练模型预测图片关联文本中的名词和形容词
- Quattoni et al. (2007)：通过流形学习从 caption 预测训练分类器
- Srivastava & Salakhutdinov (2012)：深度多模态 Boltzmann 机
- Joulin et al. (2016)：把 YFCC100M 图片的标题/描述/hashtag 转成 bag-of-words 多标签分类任务，AlexNet 预训练后迁移表现和 ImageNet 预训练差不多
- Li et al. (2017)：Visual N-Grams，扩展到预测短语 n-gram，首次展示了 zero-shot 迁移到其他分类数据集
- VirTex (Desai & Johnson, 2020)、ICMLM (Bulent Sariyildiz et al., 2020)、ConVIRT (Zhang et al., 2020)：用 Transformer + 语言模型/掩码语言模型/对比学习做图像表示学习

**为什么这些方案没火起来？**

这几个问题的链条非常清晰：

1. **性能太差。** Li et al. (2017) 在 ImageNet zero-shot 上只做到 11.5%——当时 SOTA 是 88.4%，甚至不如 2012 年的经典方法（~50%）。性能差太多了，根本没法和有监督方法竞争。

2. **规模太小。** Mahajan et al. (2018) 用 Instagram 3.5B 图片训练了 19 GPU 年，Kolesnikov et al. (2019) 用 JFT-300M 训练了多加速器年。而 VirTex/ICMLM/ConVIRT 只在 10-20 万张图片上训练了几天。**规模差了 3-4 个数量级。**

3. **预测目标太难。** 精确预测图片对应的文字描述（caption generation）是个极其困难的任务——图片的文本描述千差万别，"一张狗的照片"、"我家狗子今天在公园"都能描述同一张图。模型花大量算力去建模文本中的细粒度差异，却对视觉表示学习帮助有限。

4. **静态分类器 + 封闭集合。** Instagram 预训练和 JFT 预训练虽然规模上去了，但用的是固定 softmax 分类器——训练时预测 1000 或 18291 个类，推理时也只能预测这些类，没有动态输出能力，"zero-shot" 能力严重受限。

## 三、CLIP 的核心思路

CLIP（Contrastive Language-Image Pre-training）的思路简单到令人怀疑：**不要求模型精确写出图片的描述，只让模型判断哪段文字配哪张图。**

就这么一个看似更简单的目标，配合 400M 图文对的规模和对比学习，效果却炸了。

```ascii
┌────────────────────────────────────────────────────────────────────┐
│                    CLIP 总体架构 (示意图)                            │
│                                                                    │
│    ┌──────────┐      ┌──────────────┐       ┌──────────┐          │
│    │  Image   │ ──▶  │   Image      │       │   Text   │          │
│    │          │      │   Encoder    │       │  Encoder │          │
│    │  Batch   │      │ (ResNet/ViT) │       │ (Transf.)│          │
│    │  N×HWC   │      │    ───▶      │       │  ───▶    │          │
│    └──────────┘      │  I_f [N,di]  │       │T_f [N,dt]│          │
│                      └──────┬───────┘       └─────┬────┘          │
│                             │                     │               │
│                      ┌──────▼───────┐       ┌─────▼──────┐        │
│                      │ Linear Proj  │       │Linear Proj │        │
│                      │  W_i [di,de] │       │W_t [dt,de] │        │
│                      └──────┬───────┘       └─────┬──────┘        │
│                             │                     │               │
│                      ┌──────▼─────────────────────▼──────┐        │
│                      │      L2 Normalize                  │        │
│                      │  I_e [N,de], T_e [N,de]           │        │
│                      └──────┬─────────────────────┬──────┘        │
│                             │                     │               │
│                      ┌──────▼─────────────────────▼──────┐        │
│                      │  Cosine Similarity: I_e @ T_e.T   │        │
│                      │  Logits [N,N] scaled by exp(τ)    │        │
│                      └──────────────┬────────────────────┘        │
│                                     │                             │
│                      ┌──────────────▼────────────────────┐        │
│                      │ Symmetric Cross-Entropy Loss      │        │
│                      │ loss_i + loss_t = 对角匹配        │        │
│                      └───────────────────────────────────┘        │
└────────────────────────────────────────────────────────────────────┘
```

**简要流程 (来自论文 Figure 1 的伪代码逻辑):**

1. 一个 batch 有 $N$ 个 (image, text) 对
2. Image Encoder（ResNet 或 ViT）提取图像特征 $I_f \in \mathbb{R}^{N \times d_i}$
3. Text Encoder（Transformer）提取文本特征 $T_f \in \mathbb{R}^{N \times d_t}$
4. 各自通过线性投影到公共嵌入空间，L2 归一化：$I_e, T_e \in \mathbb{R}^{N \times d_e}$
5. 计算 $N \times N$ 的余弦相似度矩阵，用可学习的温度参数 $\tau$ 缩放
6. 对每一行（image→text）和每一列（text→image）分别做 cross-entropy loss，取平均

这里有个关键设计：**对称损失**。图像到文本的匹配损失和文本到图像的匹配损失是分开算然后平均的，这迫使嵌入空间在两个方向上都要 align。

## 四、网络架构详解

### 4.1 图像编码器

CLIP 提供了两种图像编码器选择：

**ResNet 系列（改进版）：**
- 基础架构：ResNet-50 / ResNet-101
- 改进 1：采用 ResNet-D 改进 (He et al., 2019)——在 stem 部分用 7×7 卷积替换为 3 个 3×3 卷积，降低计算量
- 改进 2：抗锯齿 rect-2 blur pooling (Zhang, 2019)
- 改进 3：**用 Attention Pooling 替代全局平均池化**——单个 transformer-style 多头 QKV 注意力层，query 由全局平均池化的结果生成
- 缩放方式：受 EfficientNet 启发，同时增加 width、depth、resolution 三个维度，而不是像传统做法只增加一个维度

**Vision Transformer 系列：**
- ViT-B/32 (patch size=32)、ViT-B/16 (patch size=16)、ViT-L/14 (patch size=14)
- 在 ViT 实现基础上仅加了两个小修改：patch + positional embedding 后多一层 layer norm；略微不同的初始化方案

### 4.2 文本编码器

- **架构：** Transformer (Vaswani et al., 2017)，采用 GPT-2 的架构修改 (Radford et al., 2019)
- **基础尺寸：** 63M 参数，12 层，512 宽，8 注意力头
- **分词：** 小写化的 BPE，词表大小 49,152 (Sennrich et al., 2015)
- **序列长度：** 最大 76 个 token（为了计算效率）
- **表示提取：** 序列以 [SOS] 和 [EOS] 标记包裹，最高层在 [EOS] 处的激活作为文本特征表达，经 layer norm 后线性投影到多模态嵌入空间
- **注意：** 使用了 masked self-attention（保留未来 token 不可见），目的是兼容未来用预训练语言模型初始化或添加语言模型辅助目标的可能性

### 4.3 前向传播过程详解

以 ViT-L/14 为例（COLR 训练过程中最常用的版本）：

**输入：**
- 图像 batch：$I \in \mathbb{R}^{N \times 3 \times H \times W}$（$N=32768, H=W=224$，ViT-L/14@336px 版使用 336px）
- 文本 batch：$T \in \mathbb{R}^{N \times L}$（$L=76$ token IDs, 使用 BPE 词表 49152）

**Step 1: 图像特征提取**
```
ViT-L/14:
  输入: I [N, 3, 224, 224]
  → Patch Embedding (14x14 patches): [N, 256, 1024]    (256=224/14 × 224/14)
     (每个 patch 用 Conv2D 投影到 d=1024)
  → CLS token + Positional Embedding + LayerNorm
  → 24 层 Transformer encoder:
     - 每层: LayerNorm → Multi-Head Self-Attention (16 heads, d=1024)
            → Residual → LayerNorm → MLP (1024→4096→1024) → Residual
  → [CLS] token 的输出: I_f [N, 1024]
```

**Step 2: 文本特征提取**
```
Text Transformer (12层, 512宽):
  输入: T [N, 76] (token IDs)
  → Token Embedding (vocab 49152 → d=512): [N, 76, 512]
  → Positional Embedding + 12 层 masked self-attention + MLP
  → 取 [EOS] token (最后一个 token) 的输出: [N, 512]
  → LayerNorm: T_f [N, 512]
```

**Step 3: 投影到公共嵌入空间**
```
I_e = L2Norm(I_f @ W_i)    # W_i [1024, 512], I_e [N, 512]
T_e = L2Norm(T_f @ W_t)    # W_t [512, 512],  T_e [N, 512]
```

**Step 4: 计算相似度 & 损失**
```
logits = I_e @ T_e.T * exp(τ)    # [N, N], τ 是可学习的标量
loss_i = CrossEntropy(logits, labels, axis=0)  # 每张图去匹配正确的文本
loss_t = CrossEntropy(logits, labels, axis=1)  # 每段文本去匹配正确的图片
loss   = (loss_i + loss_t) / 2
```

## 五、核心创新点

### 创新 1：将对比学习引入图文预训练

#### (a) 痛点与动机

之前的工作（VirTex, ICMLM）试图让模型**精确预测**图片的文本描述。这带来了一个问题：图片的文本描述千差万别，"My dog playing in the park" 和 "A cute puppy" 都能描述同一张图。让模型去精确生成/预测这些词，等于让模型花大量算力去学习文本的词汇差异而不是视觉概念。

对比学习的思路来自两个观察：
- Tian et al. (2019) 发现对比学习在图像表示上优于预测式目标
- Chen et al. (2020a) 发现生成式图像模型比同性能对比模型需要多一个数量级的算力

#### (b) 方案细节

不预测具体词汇，而是做对比学习：给定一个 batch 的 $N$ 个 (image, text) 对，模型要从 $N \times N$ 个可能的配对中找出正确的 $N$ 个配对。

损失函数是对称的信息保留对比损失（InfoNCE 的变体）：

$$L = \frac{1}{2} \left[ \frac{1}{N} \sum_{i=1}^{N} \log \frac{\exp(s_{ii}/\tau)}{\sum_{j=1}^{N} \exp(s_{ij}/\tau)} + \frac{1}{N} \sum_{j=1}^{N} \log \frac{\exp(s_{jj}/\tau)}{\sum_{i=1}^{N} \exp(s_{ij}/\tau)} \right]$$

其中 $s_{ij} = \frac{I_{e_i}^\top T_{e_j}}{\|I_{e_i}\|\|T_{e_j}\|}$ 是余弦相似度。

#### (c) 为什么有效

从信息论角度：预测性的语言模型需要建模所有词的依赖关系，而对比学习只需要区分正负样本对。这是**一个简单的代理任务**，但足够好地驱动视觉表示学习。

从实验角度看，论文 Figure 2 展示了一个关键发现：
- Transformer 语言模型（caption prediction）：3× 慢于 BoW baseline
- Bag-of-Words 预测式目标：baseline
- **CLIP 对比学习：再快 4×**

也就是说，同样的计算量下，对比学习的 ImageNet zero-shot 迁移速度是预测式语言模型的 **12 倍**。

#### (d) 与 Related Work 的关系

- 这个对比学习目标最早由 Sohn (2016) 在多度量学习中提出（multi-class N-pair loss）
- 被 Oord et al. (2018) 以 InfoNCE 的形式推广到对比表示学习
- Zhang et al. (2020)（ConVIRT）第一次将其用于医学影像的图文对比学习
- CLIP 的贡献是：**在 400M 图文数据规模上验证了对比学习的缩放规律性**，并做了一系列简化

#### (e) 如果去掉会怎样

论文 Figure 2 明确对比了两种目标的效果。在同样的 ImageNet zero-shot 评估下：
- 对比学习：4× 效率提升
- Bag-of-Words 预测：baseline
这还没算加上 Transformer 语言模型预测 12 倍的差距。如果去掉对比学习，要么需要 4× 数据量来达到同样的 zero-shot 效果，要么得分大幅下降。

### 创新 2：用自然语言作为灵活的分类器接口

#### (a) 痛点与动机

有监督分类器的本质问题是：**训练时定义了一组固定类别，推理时只能在这组类别上预测**。要做到 zero-shot（零样本迁移），需要一种动态的分类器构建机制。

#### (b) 方案细节

CLIP 的做法非常优雅：**用文本编码器动态生成分类器的权重**。

具体来说，对于任意分类数据集，把类别名称转换为句子（例如 ImageNet 的 "dog" → "A photo of a dog."），通过 text encoder 得到该类的文本嵌入向量，作为分类器的权重向量。每张图片的图像嵌入和所有类别的文本嵌入做余弦相似度，取最高分的类别。

```
┌─────────────────────────────────────────────────────────────┐
│              CLIP Zero-shot 分类流程                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  图像编码:                                                  │
│   Image  →  Image Encoder  →  I_e ∈ R^{d_e}                │
│                                                             │
│  文本编码 (类别名 → 句子):                                  │
│   "A photo of a dog."    →  Text Encoder  →  T_e1 ∈ R^{d_e}│
│   "A photo of a cat."    →  Text Encoder  →  T_e2 ∈ R^{d_e}│
│   "A photo of a car."    →  Text Encoder  →  T_e3 ∈ R^{d_e}│
│   ...                                                      │
│                                                             │
│  概率:  p(class_i | image) = softmax(I_e · T_ei / τ)_i     │
│                                                             │
│  ★ 不需要任何训练数据，直接用类别名做推理                     │
└─────────────────────────────────────────────────────────────┘
```

#### (c) 为什么有效

这一点非常巧妙。每一张训练中的 (image, text) 对，对 CLIP 来说就是一个 **"1-shot 分类任务"**：batch size $N=32768$ 就是 32768 个不同的"类别"，每个类别只有 1 个训练样本。**CLIP 本质上在预训练阶段就在不断解不同任务的 zero-shot 分类问题。**

测试阶段的工作机制可以理解为：文本编码器是一个超网络（hypernetwork, Ha et al., 2016），它基于类别名称生成线性分类器的权重。Lei Ba et al. (2015) 最早引入了这种形式的 zero-shot 图像分类器，Elhoseiny et al. (2013) 提出了从自然语言生成分类器的想法。

#### (d) 与 Related Work 的关系

- Visual N-Grams (Li et al., 2017) 也探索了 zero-shot 迁移，但做法是把类名转成 n-gram 表示后计算概率，在 ImageNet 上只做到 11.5%
- CLIP 将这一能力推到了 76.2%（ViT-L/14@336px），并验证了其在 30+ 个数据集上的通用性

#### (e) 如果去掉会怎样

CLIP 就没有"零样本"能力了，就和普通的有监督预训练模型一样，需要每个下游数据集都 fine-tune。核心创新点就不存在了。

### 创新 3：Prompt Engineering & Ensembling

#### (a) 痛点与动机

CLIP 的 text encoder 预训练时接触的是完整句子（"A photo of my dog in the park"），但做 zero-shot 分类时，输入只是孤零零的类名（"dog"）。**分布不匹配**导致性能下降。此外，一词多义问题（crane 可以是起重机也可以是鹤）会导致歧义。

#### (b) 方案细节

- **Prompt 模板：** 使用 "A photo of a {label}." 作为默认提示，这在 ImageNet 上比直接用类名提升了 1.3%
- **任务定制提示：** 如细粒度分类用 "A photo of a {label}, a type of pet/food/aircraft."；OCR 用引号括起文本；卫星图用 "A satellite photo of a {label}."
- **Ensembling：** 生成多个不同 prompt 的文本嵌入，在嵌入空间取平均（而不是在概率空间），这样 ensemble 的推理成本和单 prompt 一样

#### (c) 为什么有效

论文 Figure 4 显示：prompt engineering + ensembling 在 36 个数据集上平均提升约 **5 个点**。这个收益相当于模型用 4× 更多计算量带来的提升——但 prompt engineering 的计算成本可忽略不计（嵌入预计算后缓存）。

#### (d) 与 Related Work 的关系

这与 GPT-3 的 prompt engineering 思路一脉相承 (Brown et al., 2020; Gao et al., 2020)。

#### (e) 如果去掉会怎样

ImageNet 上从 76.2% 降到约 71.2%，Stanford Cars 上降近 29 个点（Figure 4 可见性能退化）。

## 六、公式详解

### 6.1 对比损失函数

#### 符号定义

- $N$：batch size（论文中为 32768）
- $I_{e_i} \in \mathbb{R}^{d_e}$：第 $i$ 张图像的 L2 归一化嵌入
- $T_{e_j} \in \mathbb{R}^{d_e}$：第 $j$ 段文本的 L2 归一化嵌入
- $s_{ij} = I_{e_i}^\top T_{e_j}$：余弦相似度（因为 $I_e$ 和 $T_e$ 已 L2 归一化）
- $\tau$：温度参数，可学习，初始化为对应 0.07，clipped 防止 logits 缩放超过 100 倍

#### 公式与推导

**图像→文本损失（每张图去匹配正确的文本）：**

$$L_i = -\frac{1}{N} \sum_{i=1}^{N} \log \frac{\exp(s_{ii}/\tau)}{\sum_{j=1}^{N} \exp(s_{ij}/\tau)}$$

**文本→图像损失（每段文本去匹配正确的图片）：**

$$L_t = -\frac{1}{N} \sum_{j=1}^{N} \log \frac{\exp(s_{jj}/\tau)}{\sum_{i=1}^{N} \exp(s_{ij}/\tau)}$$

**总损失：** $L = (L_i + L_t) / 2$

#### 直觉理解

从行方向看：每个图像 $i$ 在所有 $N$ 个文本候选中找到正确的那个，是一个 $N$-way 分类问题。
从列方向看：每个文本 $j$ 在所有 $N$ 个图像候选中找到正确的那个，同样是一个 $N$-way 分类问题。

对称的关键意义：**强迫嵌入空间在两个方向上都要好。** 如果只做单向（比如只是 image→text），text encoder 可以偷懒——它只需要生成足够区分性的文本嵌入，而不需要让文本嵌入在图像端的视角下有好的结构。

#### 具体数值示例

以 batch size $N = 32768$（论文中的实际值），温度 $\tau \approx 0.07$ 的初始值。

每个 logits 矩阵是 $32768 \times 32768$，所以在一次前向中要计算约 10.7 亿个相似度分数（约 1B 对）。

梯度在正确配对的 $N=32768$ 个正样本上正向推动，在 $N^2 - N \approx 10.7$ 亿个负样本上反向推动。这个对比压力非常大——每个正样本对应的负样本数高达 32767 个。

#### 边界情况

当 $\tau \to \infty$ 时，softmax 分布趋近均匀分布，loss 趋近 $\log N$（完全不学习）。

当 $\tau \to 0$ 时，softmax 趋近 one-hot 分布，loss 趋近 0，但梯度会变得极小（饱和 region），导致训练停滞。论文将 $\tau$ 初始化为对应于 0.07，并 clipping 防止 logits scale 超过 100。

### 6.2 温度参数学习

温度 $\tau$ 在训练中被参数化为 $\exp(\alpha)$，其中 $\alpha$ 是可学习标量。这样做的好处是：

- 自动选择最优的 logit 缩放范围
- 避免手动调参
- $\alpha$ 无约束，但通过 clipping 防止 $\exp(\alpha) > 100$

### 6.3 训练的缩放规律

论文 Figure 9 展示了零样本错误率与模型计算量的关系：在一个 5 模型序列上（RN50 → RN50x64 覆盖 44× 计算量范围），log-log 尺度上近似线性。这与语言模型中的缩放规律 (Kaplan et al., 2020) 一致。

$$E_{\text{zero-shot}} \propto \text{FLOPs}^{-\alpha}$$

其中 $\alpha$ 由实验数据拟合得出。

- **GPT 系列** (Radford et al., 2018, 2019; Brown et al., 2020)：证明了任务无关预训练 + zero-shot 迁移在 NLP 上的可扩展性，CLIP 将这一理念迁移到视觉领域
- **Vision Transformer** (Dosovitskiy et al., 2020)：CLIP 实验并验证了 ViT 在对比学习框架下的计算效率优势

### 同期竞品

- **ALIGN** (Jia et al., 2021, Google)：几乎同期使用 1.8B 图文对 + 对比学习，使用更大的 EffcientNet-L2，在 ImageNet zero-shot 上达到 76.4%（略高于 CLIP 的 76.2%）

**直接扩展（同一团队）：**
- **DALL·E 2** (Ramesh et al., 2022)：使用 CLIP 文本编码器作为文本到图像生成的引导，CLIP 嵌入作为 prior
- **CLIP 后续工作**：CLIP 在视频理解（VideoCLIP, ActionCLIP）、3D 理解（PointCLIP）等方向的扩展

- **OpenCLIP** (Ilharco et al., 2021)：社区驱动的开源 CLIP 复现，包含了更丰富的训练配方

## 十、局限

### 1. 零样本性能在专业/复杂任务上仍然很差

**问题：** EuroSAT（卫星图像分类，-37.1%）、CLEVRCounts（对象计数）、PatchCamelyon（病理学）等任务上 CLIP 零样本表现远不及有监督基线。

**何时重要：** 需要高精度的垂直领域（医疗、遥感、自动驾驶等）。

**论文是否承认：** 是。论文明确讨论这些任务需要"专门的、密集的训练才能胜任"。

**后续改进：** CLIP 的 linear probe 在这些任务上有所改善，但整体上需要 domain-specific 的微调或扩展。

### 2. 数据效率问题

**问题：** CLIP 需要 400M 图文对训练。小数据量场景（如医疗、低资源语言）下收集如此大规模数据不现实。

**何时重要：** 数据稀缺领域。

**后续改进：** LiT (Zhai et al., 2021)、BLIP-2 等探索了用更少的图文对进行高效对齐。

### 3. Prompt Engineering 仍然需要人工

**问题：** 不同数据集的最佳 prompt 模板不同（Stanford Cars 需要指定 type，OCR 需要引号）。对 30+ 数据集逐一调整 prompt 有一定人力消耗。

**何时重要：** 大规模评估或自动化场景。

### 4. 只做了线性投影，非线性投影被跳过

**问题：** 论文说"没观察到效率差异"——但 SimCLR 等工作证明非线性投影对无监督表示学习有帮助。可能只是规模大了之后非线性投影的边际收益变小。

**后续改进：** 后续工作 (EVA-CLIP, SigLIP) 采用了更复杂的投影头设计。

### 5. Batch Size 巨大

**问题：** 32768 的 batch size 意味着需要大量 GPU 集群。这对于学术实验室来说门槛太高。论文提到如果不做相似度分片，单 GPU 内存装不下完整的 $N\times N$ 相似度矩阵。

**后续改进：** SigLIP 用 sigmoid 损失替代 softmax 对比损失，摆脱了对 global batch 的依赖。

### 6. 数据质量和过滤问题

**问题：** 400M WIT 数据虽然量大，但收集自互联网，存在 bias、低质量内容和有害内容。论文 Section 7 讨论了伦理影响。

## 十一、小结

**这篇论文做了什么：** CLIP 提出了一个极其简单的方法——用 400M 互联网图文对做对比学习——来训练可迁移的视觉模型。核心洞察是：不预测精确文字，只判断配对关系，加上大规模数据，对比学习就能学到通用的视觉表示。

**为什么有效：** 
1. 对比学习避免了 caption generation 的困难，效率提升了 4-12×
2. 400M 图文对提供了丰富的视觉概念覆盖
3. 自然语言作为分类器接口实现了灵活的 zero-shot 迁移
4. 对称损失确保了图像和文本嵌入空间的双向对齐

**改变了什么：** CLIP 从根本上变革了计算机视觉和多模态学习。
- 证明了 "scale + natural language" 是可以替代人工标注的视觉训练范式
- 使零样本分类从概念验证（11.5%）推进到实用水平（76.2%）
- 为文本到图像生成（DALL·E 2, Stable Diffusion）和多模态大模型（GPT-4V）提供了核心组件
- 视觉编码器+billion scale 图文对比学习成为现代多模态系统的"标配"

直到今天，CLIP 的嵌入和编码器仍然是被引用最多、使用最广的多模态基础模型之一。它不仅是方法的突破，更是一个**范式的转变**——从此视觉学习不再需要死板的人工标注，而是可以通过互联网级别的自然语言数据进行训练。
