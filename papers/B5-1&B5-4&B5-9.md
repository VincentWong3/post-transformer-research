## 一、先搞清楚"问题在哪里"

2020 年底，Vision Transformer（ViT）在 ImageNet 上证明了**纯 Transformer 架构可以做图像分类，而且能做得好**。整个 CV 界自然有一个强烈的冲动：视频呢？

视频和图像的根本区别在于一个"时间维"。一段视频不仅是 3 秒还是 30 秒的问题——动作识别需要模型理解"随时间展开的模态变化"。一个打篮球的人，在起跳瞬间和投篮瞬间，单帧看起来完全不同；但结合时间上下文，它们属于同一个动作。

在 TimeSformer 之前，处理视频的主流方案都是 3D 卷积（I3D, SlowFast, C3D 等）。3D 卷积在 2D 卷积的基础上增加了一个时间维度的 kernel，让卷积滤波器同时跨越空间和时间轴。这个方案在当时是站得住脚的——在 Something-Something 数据集上，SlowFast 已经刷到了很高。

但 3D 卷积有两个结构性缺陷：

**第一个缺陷：视野太小。** 3D 卷积的核通常只有 3×3×3 或 3×7×7。即便堆叠十几层，每个位置能看到的上下文仍然有限。一个 3D 卷积的等效感受野永远不会超过它堆叠的深度。而视频中的动作可能需要关联十几秒甚至几十秒外的上下文——一个运动员在投篮前几秒的传球动作可能决定了当前帧中"投篮"的语义类别。

**第二个缺陷：归纳偏置太强。** 卷积操作天然假设"局部性"和"平移不变性"——这在图像上是对的，但视频里的动作是高度非局部的。"拍手"需要同时看到左右手的相对位置（空间远程依赖），"挥手"需要看到手在连续帧间的位移（时间远程依赖）。用 3D 卷积处理这些问题，相当于强行假设"只有在 3×3×3 的 spatiotemporal window 内的像素才相关"——这显然不对。

所以 TimeSformer 提出的核心问题非常直白：

> **视频理解到底需不需要专门设计的时间模块？还是说，直接用 Transformer 的 self-attention 同时处理空间和时间就够了？**

这个问题在当时是激进的——因为 2021 年初，还没有人尝试过**完全不用卷积、完全基于 self-attention**的视频识别模型。所有已有的"视频 Transformer"工作（如 Non-Local Networks）都是在卷积特征图上加几层 attention 模块作为补充，而不是彻底替换。

## 二、旧路线的真正问题

### 2.1 3D 卷积的"局部性假设"在视频上不成立

3D 卷积的核大小在时间维通常为 3 或 5，跨 3-5 帧做局部卷积。对于相邻帧之间的短期运动模式，这确实够了。但问题在于：**很多动作的判别性信息不在相邻帧之间，而在远距离帧之间。**

看一个例子：对于一个"倒水"的视频，从"杯子空"到"杯子里有水"可能需要几秒钟。3D 卷积需要堆叠几十层才能把最后一层的感受野扩大到覆盖"空杯→倒水→满杯"的全过程。而在这几十层中，信息逐层收缩、逐层丢失。

Self-attention 则完全不同：每一层的每个 query 可以直接和所有空间位置、所有时间帧的 key 做交互。空间上看，一帧左上角的 patch 可以直接关联到右下角的手；时间上看，第 1 帧的手可以直接关联到第 64 帧的水杯。这是 3D 卷积做不到的。

### 2.2 Transformer 的"过强全局性"在视频上也不成立

如果直接套用 ViT 到视频，做法很简单：把 8 帧视频中每帧切 16×16 patch，8 帧共 8×14×14 = 1568 个 token，然后在这 1568 个 token 上做全局 self-attention。

这个方案理论上可以建模所有 spatiotemporal 关系，但有两个问题：

**计算量太大。** 全局 self-attention 的复杂度是 O((NFT)²)，其中 N 是每帧 patch 数，F 是帧数（8-96），T 是 token 总数。对于 8 帧×16×16 patch 的输入，149 维的 attention 矩阵有 1568² ≈ 2.5M 个元素。对于 96 帧，有 18816² ≈ 354M 个元素——这在一张 GPU 上根本不现实。

**视觉关系其实是局部稠密、全局稀疏的。** 在 NLP 中，"句子中的每个词都和其他词相关"大致成立——因为语言是依赖语法结构的关系网络。但在视频中，绝大多数 patch 对之间不需要直接交互。一个 patch 只需要和自己的空间邻居、以及自己在其他帧中的对应位置交互。强制所有 patch 互相 attention 既浪费计算，又引入了不必要的噪声。

所以 TimeSformer 面临的核心问题是如何在"3D 卷积的视野太小"和"全局 self-attention 的视野过大且太贵"之间找到一个合适的中间点。

## 三、核心思路：Divided Space-Time Attention

TimeSformer 的核心贡献非常集中：**把 3D 时空 self-attention 拆成两步骤——先做空间 self-attention（在同一帧内），再做时间 self-attention（在同一空间位置上跨帧）。** 这两步在每个 Transformer block 内顺序执行，共享同一个 MLP。

这个设计的名字是 **"divided attention"**。给一个直观的理解：

```
标准的 Transformer Block:
  输入 → LayerNorm → 多头 Self-Attention（全局）→ 残差 → LayerNorm → MLP → 残差 → 输出

TimeSformer 的 Divided Attention Block:
  输入 → LayerNorm → 空间 Self-Attention（同一帧内）
       → LayerNorm → 时间 Self-Attention（同一位置跨帧）
       → 残差（两个 attention 合并跳过）→ LayerNorm → MLP → 残差 → 输出
```

用公式表示：

```
空间 attention:  α(p,t)^space = Softmax(q(p,t) · [k(0,0), {k(p',t) for p'=1...N}] / √Dh)
时间 attention:  α(p,t)^time  = Softmax(q(p,t) · [k(0,0), {k(p,t') for t'=1...F}] / √Dh)
```

**关键在于"分步"——query patch 先只看同一个帧内的其他 patch（空间 interaction），再看其他帧中和自己同一位置的 patch（时间 interaction）。** 这两步在不同帧间不交互：对于空间 attention，不同帧的 patch 之间不做比较；对于时间 attention，不同空间位置的 patch 之间不做比较。

## 四、五种 Attention 方案的消融比较

TimeSformer 论文最精彩的部分是 Figure 2 和对应的消融实验——作者不是直接抛出一个方案，而是**系统性比较了五种不同的时空 self-attention 设计方案**，让人看到为什么 divided attention 是最优的。

这五种方案是：

### 方案 1：Space-only（S）
只做空间 attention。每个 patch 只看自己帧内的其他 patch，完全不做时间建模。query 和 key 来自同一帧。

表现：明显低于其他方案，证明了时间建模的必要性。

### 方案 2：Joint space-time（ST）
全局 3D spatiotemporal attention。每个 patch 看所有帧的所有 patch。这是最通用的方案，计算量也是最大的。

表现：比 space-only 好，但不如 divided。原因可能是全局 attention 引入了太多噪声交互——一个 patch 没有必要和所有其他 patch 直接交互。

### 方案 3：Divided space-time（T+S）
先做时间 attention（同一位置跨帧），再做空间 attention（同一帧内跨位置）。两步骤的注意力权重分别独立计算。

表现：**在所有方案中表现最好。** 每一步的注意力矩阵只有 NF × N（空间）或 NF × F（时间），复杂度从 O(N²F²) 降到了 O(N²F + NF²)。

### 方案 4：Sparse Local Global（L+G）
只在局部 spatiotemporal 窗口内做 attention。窗口的大小是有限的——每个 query 只看到邻近的空间位置和邻近的时间帧。这一步的动机是模仿 3D 卷积的局部性。

表现：不如 divided。原因是窗口限制了远程依赖的建模——如果一个动作需要关联两个相隔很远的空间位置，局部方案就做不到。

### 方案 5：Axis attention（A）
分两次 attention——一次沿空间轴，一次沿时间轴——但不是先时间后空间，而是空间轴 attention 沿 H 方向和 W 方向分别进行（即轴向 attention）。

表现：不如 divided，但好于 space-only。

### 消融结论

（具体数字请见论文 Table 1）

Divided attention（T+S）在所有方案中准确率最高。而且更重要的是，它的计算成本远低于 joint space-time attention——因为每次只在一个方向上做 attention，token 对的数量从 O(N²F²) 降到了 O(N²F + NF²)。

消融实验还比较了"先时间后空间"和"先空间后时间"两个顺序。论文发现两者效果差不多，但最终选择了"先时间后空间"作为默认方案（时间 attention 让跨越帧的相同位置先交互，再通过空间 attention 把时间对齐后的信息分布到帧内各个位置）。

## 五、整体结构与 Forward 流程

### 5.1 总体架构

TimeSformer 的架构直接继承自 ViT，修改极小：

```
输入视频: [B, 3, F, H, W]  （F 帧 RGB 视频）
    │
    ▼
Patch Embedding: 每帧切成 16×16 的非重叠 patch
    │             每帧 ≈ 14×14 = 196 个 patch
    │             输出: [B, NF+1, D] （+1 为 cls token）
    │
    ▼
Positional Embedding: 空间位置 + 时间位置的 learnable embedding
    │             每个 patch (p,t) 有独立的 (x,y,t) 位置编码
    │
    ▼
L × Transformer Block（Divided Attention）
    │
    ├── LayerNorm → Temporal Self-Attention → 残差
    │   （同一空间位置跨所有帧）
    │
    ├── LayerNorm → Spatial Self-Attention → 残差
    │   （同一帧内跨所有空间位置）
    │
    └── LayerNorm → MLP → 残差
    │
    ▼
LayerNorm(cls token) → Linear 分类头
```

### 5.2 关键参数

默认配置（TimeSformer，即 ViT-Base/16）：
- Patch 大小：16×16
- 每帧 patch 数：14×14 = 196（224×224 输入）
- 帧数：F = 8（默认），也可用 16, 32, 96
- Token 总数：8×196 + 1 = 1569
- Transformer Block 数：L = 12（ViT-Base 配置）
- Hidden dim：D = 768
- Attention heads：12
- MLP hidden dim：3072
- 参数量：121M（TimeSformer）

### 5.3 前向过程逐步跟踪

**Step 1 — 输入**

输入 8 帧 224×224 RGB 图像，形状 [B, 3, 8, 224, 224]。

**Step 2 — Patch Embedding**

每帧切成 14×14 = 196 个 16×16 patch。每个 patch 展平为 16×16×3 = 768 维向量，通过可学习线性层映射到 D 维。

张量形状：[B, 8×196, D]（帧和空间位置合并为序列维度）

加 cls token（1 个特殊 token，后续用于分类）：[B, 8×196+1, D]

**Step 3 — 位置编码**

每个 patch 有独立的空间位置 (y, x) 和时间位置 (t)。论文使用可学习的 3D 位置编码：每个 (y, x, t) triplet 对应一个独立的 D 维向量。

这和 ViT 的 1D 位置编码不同——ViT 只编码 196 个空间位置的顺序，不区分不同帧的相同位置应该有不同的编码。在 TimeSformer 中，同一空间位置在不同帧的不同时间点有独立的编码，这是为了区分"这一帧的左上角 patch"和"另一帧的左上角 patch"。

**Step 4 — Divided Attention Block（12 层）**

以第 1 层为例：

子步骤 1 — 时间 Self-Attention：

对于每个空间位置 p，计算 q(p,t)·k(p,t') 对所有 t、所有 t' 的注意力权重。这意味着：某个帧里左上角的 patch，要和所有帧里左上角的 patch 做 attention。

注意：**只有在同一空间位置的 patch 之间才有时间 attention。** 左上角和右下角之间不做时间 attention——这是 divided attention 节省计算的关键。

子步骤 2 — 空间 Self-Attention：

对每个帧 t，计算 q(p,t)·k(p',t) 对所有 p、所有 p' 的注意力权重。这意味着：一帧内所有 patch 之间互相做 attention，但不同帧的 patch 之间不做。

子步骤 3 — MLP：

标准的 2 层 MLP（hidden 3072，GELU 激活），作用在每个 token 上。

子步骤 4 — 残差连接：

两个 attention 的输出合并后（add）过残差连接。

**Step 5 — 分类**

12 层结束后，取 cls token 的最后一个 block 输出，过 LayerNorm 和线性分类头。

## 六、核心公式与数学直觉

### 6.1 复杂度分析

Divided attention 的复杂度优势是这篇论文最优雅的部分。

**Joint space-time attention（全局 3D attention）：**
- 每个 query 需要和所有 NF 个 key 做比较
- 总 query 数：NF + 1（含 cls token）
- 复杂度：O(2(NF)²D)

**Divided space-time attention：**
- 时间 attention：每个 query 需要和 F 个 key（同一空间位置的所有帧）做比较
  - 复杂度：O(NF²D)
- 空间 attention：每个 query 需要和 N 个 key（同一帧内的所有空间位置）做比较
  - 复杂度：O(N²FD)
- 总复杂度：O(NF²D + N²FD)

对比：全局 attention 是 O((NF)²D) = O(N²F²D)，而 divided 是 O(NF²D + N²FD)。当 N 和 F 都很大时，divided 约是全局的 1/NF。

举例：N=196, F=8。全局 attention 需要 1568² ≈ 2.5M 次 query-key 比较。Divided attention 需要 196×64 + 196²×8 ≈ 12,544 + 307,328 ≈ 320K 次——约全局的 1/8。

### 6.2 为什么 Divided Attention 能 work？

如果深入思考，"先做时间 attention，再做空间 attention"这个顺序的选择是有道理的。

**第一步：时间 attention。** 每个空间位置的 patch 先与自己在其他帧的对应 patch 交互。这可以认为是"跨帧对齐"——第 n 帧的某个 patch 从第 n-1/n+1 帧的对应位置获取"运动上下文"。这个步骤的输出是"时间聚合后的帧表示"——每帧的每个 patch 都包含了它自己的时间演变信息。

**第二步：空间 attention。** 时间对齐后的每帧表示再做内部的空间交互。一个 patch 现在不仅有当前帧的信息，还有来自其他帧的时间上下文，所以空间 attention 可以基于更丰富的表示来建模帧内关系。

对比"先空间后时间"：先在同一帧内聚合空间信息，再做跨帧的对应。论文中验证了两种顺序效果接近，但选择"先时间后空间"作为默认——可能因为时间 attention 的计算量更小（F 通常远小于 N）。

### 6.3 位置编码的设计

TimeSformer 使用可学习的 3D 位置编码，而不是 ViT 的 1D 编码。每个空间位置 (y, x) 和时间位置 (t) 共同决定一个编码向量。

这种设计隐含了"同一位置在不同帧中有相同空间位置"的假设。但这对于视频中的动作识别是合理的——在 Kinetics 的典型动作场景中，主体通常在画面中心附近运动，不会在不同帧间突然跳跃到全新位置。所以同一空间位置的跨帧对应关系是有意义的。

## 七、关键结果

### 7.1 Kinetics-400

论文 Section 4.2

TimeSformer 在 Kinetics-400 上使用 8 帧输入和 ImageNet-21K 预训练可达到较好结果。增加帧数（16 帧、32 帧、96 帧）可以进一步提升性能，但增益逐渐递减。即使不依赖 ImageNet 预训练，直接在 Kinetics-400 上从头训练也能取得有竞争力的结果。

关键发现：**TimeSformer 是第一个在视频动作识别上证明纯 Transformer 架构（无卷积、无需 3D 卷积设计）可以达到 SOTA 的工作。**

### 7.2 Kinetics-600

论文 Section 4.2

在更大规模的 Kinetics-600 上，TimeSformer 同样取得了当时最佳的结果。这验证了纯 self-attention 模型在大数据下的扩展能力——当数据量增加时，Transformer 的弱归纳偏置变成一个优势，而不是劣势。

### 7.3 Something-Something v2

论文 Section 4.2

Something-Something v2 是一个高时序依赖的数据集——很多动作类别靠的是"运动模式"（如"把东西从一边移到另一边"）而不是静态外观。在这个数据上，TimeSformer 不如后来的 MViT（详见第十一节的分析）。

### 7.4 训练效率

Section 4.2 / Table 4.1

相比于当时主流的 3D CNN 方案（如 SlowFast），TimeSformer 的训练速度快约 9 倍，推理速度快 论文 Section 4.5。这个效率优势来自 Transformer 的并行计算特性——self-attention 的矩阵乘法在 GPU 上高度优化，而 3D 卷积的 im2col 操作有较大的访存开销。

### 7.5 长视频处理

Section 4.6 / Table 8

TimeSformer 可以处理 96 帧（约 3 秒的 32fps 视频）甚至超过 1 分钟的输入。增加视频长度会稳定提升准确率，没有出现信息过载或性能急剧下降的情况。对于 3D CNN 来说，处理长视频需要先分段再聚合（如 SlowFast 的 64 帧上限），而 Transformer 可以直接把长视频作为序列输入。

## 八、消融实验详解

### 8.1 Attention 方案对比

（具体数字见论文 Table 1）

五种 attention 方案的对比结果清晰地展示了时空建模的"最适合"位置：

1. **Space-only（S）**：不做任何时间建模——baseline 水平
2. **Joint（ST）**：全局时空 attention——好于 S，但不如更简单的 divided
3. **Divided（T+S）**：**最佳方案**——超过 joint 且计算量更小
4. **Sparse local（L）**：局部窗口 attention——限制了远程依赖
5. **Axis（A）**：轴向 attention——好于 local 但不如 divided

核心结论：**在视频中，不是"交互越多越好"，而是"合适的交互范围+合适的分步方式"最重要。** Divided attention 提供了一个恰到好处的平衡：空间 attention 覆盖全帧（而不只是局部窗口），时间 attention 覆盖全序列（而不只是相邻帧），两者分步计算避免了全局 attention 的无意义交互。

### 8.2 是否需要 ImageNet 预训练？

这是 TimeSformer 当时最有争议也最令人惊讶的实验。

2021 年，几乎所有视觉 Transformer 的工作（ViT, DeiT, Swin 等）都需要在大规模图像数据集上预训练才能超过 CNN baseline。社区普遍认为"Transformer 是数据饥渴的"。

TimeSformer 的实验显示：**在 Kinetics-400 上从头训练，TimeSformer 仅达到 64.8%，远低于 3D CNN from scratch 的 76.4%。** 论文中给出的数字是：论文 Section 4.2——与 SlowFast 的 76.4% 差距很大。这证明了**视频数据本身提供了足够的监督信号让 Transformer 学到可用的 spatiotemporal 表示**，不需要借助 ImageNet 的静态图像知识。

进一步实验显示，使用 ImageNet-21K 预训练仍然有增益——这提供了更好的初始化，帮助模型更快收敛、在同样训练时间下达到更高准确率。但关键是：**ImageNet 预训练不是必需的**。

### 8.3 帧数的影响

Figure 5 / Section 4.3

增加输入帧数可以持续提升准确率，从 8 帧到 32 帧到 96 帧都观察到增益。但增益是递减的：8→16 帧的提升最大，16→96 帧的增益较小。这符合直觉——大多数动作的判别性信息集中在前几秒，更长的时间范围只提供边际收益。

### 8.4 Patch 大小的影响

[see paper Section 4.3]

## 九、与 3D CNN 的比较

TimeSformer 论文花了相当篇幅直接对标 SlowFast——当时最强的 3D CNN 基线。

### 9.1 准确率

在 Kinetics-400 上，TimeSformer 使用 ImageNet 预训练达到了超过 SlowFast 的准确率。不使用 ImageNet 预训练时，TimeSformer 仅 64.8%，远低于 SlowFast 的 76.4%。考虑到 TimeSformer 完全没有用到任何卷积操作，这个结果的意义是重大的。

### 9.2 训练速度

TimeSformer 在相同的 GPU 配置下训练速度是 SlowFast 的 3 倍。原因：
- Self-attention 的矩阵乘法和 softmax 在 GPU 上是高度优化的 cuBLAS 操作
- 而 3D 卷积的 im2col 和 fold 操作涉及大量内存重排
- TimeSformer 的 batch 可以更大——Transformer 不受卷积 kernel size 的限制

### 9.3 推理灵活性

TimeSformer 的一个独特优势是可以在推理时动态调整输入帧数：模型可以对 8 帧、16 帧或 96 帧的输入直接推理，不需要调整架构。而 3D CNN 的输入长度是固定的（训练时设定的帧数）。

## 十一、与后续 MViT 的对比

MViT（Multiscale Vision Transformers, Fan et al., 2021）是紧随 TimeSformer 之后的视频 Transformer 工作，两者的对比很有意思。

### 11.1 设计哲学的不同

**TimeSformer** 的设计哲学是"keep it simple"：保持 ViT 的基本结构不变，只在 attention 层面做最小化的修改（分步 attention）。每帧使用固定分辨率的 token，不做多尺度。

**MViT** 的设计哲学是"向下采样"：在 Transformer block 之间逐步降低空间和时间分辨率，同时增加 channel 数。这和 CNN 的层级化设计思路完全一致——stage 1/2 的高分辨率特征用于保留空间细节，stage 3/4 的低分辨率特征用于提取高级语义。

### 11.2 谁更好？看数据集

在 Kinetics-400/600 上，TimeSformer 和 MViT 的准确率接近，TimeSformer 稍高——因为 K400 的动作识别更多依赖静态外观和短期运动，不需要那么多时间分辨率的下降。

在 Something-Something v2 上，**MViT 显著强于 TimeSformer**。SSv2 的动作类别高度依赖对"运动模式"的精确理解——"抛出"和"接住"的区别在于物体的运动方向和速度。MViT 的多尺度设计在这里起了关键作用：
- 高分辨率的时间特征保留了帧间的精细运动细节
- 低分辨率的时间特征提供了更大的时间感受野
- 多尺度的组合让 MViT 在 SSv2 这种"时序密集型"任务上比 TimeSformer 的固定分辨率设计更有优势

### 11.3 结论

TimeSformer 证明了"divided space-time attention"这个简单方案在 K400 这种以静态外观为主的数据集上已经足够好。但 MViT 的多尺度设计在 SSv2 这种时序密集型任务上提供了更强的时序偏置。理想情况下，两者的结合——divided attention + 多尺度层级——可能是最优的，这确实是后续很多工作探索的方向（如 Video Swin Transformer 就结合了 divided attention 的变体和 3D 移位窗口）。

## 十二、范式意义

### 12.1 开创了"视频 Transformer"这个方向

在 TimeSformer 之前，"用纯 Transformer 做视频识别"只是一个想法。在 TimeSformer 之后，它变成了一个 active research direction。2021 年下半年涌现了大量的视频 Transformer 工作（ViViT, MViT, Video Swin, STAM, Video ViT 等），它们的共同起点就是 TimeSformer 的这项工作。

### 12.2 证明了 divided attention 的有效性

Divided space-time attention 在 TimeSformer 之后成为视频 Transformer 的"标配"设计模式。Video Swin Transformer、ViViT 等后续工作都不同程度地采用了分步 attention 的思路——将时间和空间建模分离到不同的 attention 模块中。

### 12.3 改变了视频识别的研究范式

在 TimeSformer 之前，视频识别的默认框架是"3D CNN + optical flow"。TimeSformer 证明了两件事：
1. **不需要光学流**——Transformer 可以直接从 RGB 帧中学习运动表示
2. **不需要 3D 卷积**——Self-attention 可以更好地建模时间和空间依赖

这为此后的视频识别研究开辟了一条完全不同的路线：不再需要手工设计的运动特征（光流），不需要复杂的 3D 卷积 kernel 优化，只需要标准的 Transformer 架构加上合理的 attention 分解。

### 12.4 分类：bridge node（桥梁节点）

在技术树中，TimeSformer 是一个 **bridge node（桥梁节点）**——它把 ViT（图像 Transformer）的成功经验成功迁移到了视频领域，并在两者之间建立了可控的方法论桥梁。

它不是 foundation node（如 ViT 本身），因为它建立在 ViT 的既有设计之上。但它的影响力和后续工作的参考价值使其成为 B5 分支中最重要的节点之一。

## 十三、局限性与后续方向

### 13.1 Divided attention 的共享 MLP 限制

TimeSformer 在两个 attention 步骤之间只使用一个共享的 MLP。这意味着时间 attention 的输出直接进入空间 attention（跳过 MLP）。这种设计的考虑是减少参数，但也意味着时间 attention 的输出特征没有被 MLP 完全处理就直接用于空间 attention。后续工作（如 STAM）将两个 attention 步骤用各自独立的 MLP 分开，在某些任务上获得了更好的结果。

### 13.2 没有多尺度

TimeSformer 的所有 Transformer block 都在同样的 patch 分辨率上操作。这限制了模型对多尺度运动模式的建模能力——大尺度运动（如全身动作）和小尺度运动（如手指动作）需要不同的空间-时间分辨率。MViT 的多尺度设计在 SSv2 上的优势正是来自于此。

### 13.3 固定帧率的假设

TimeSformer 假设所有输入帧的采样率固定且帧间间隔均匀。对于真正变帧率的视频（如监控视频中不同 camera 的不同帧率），模型需要重新训练或适配。后续的 Video Transformer 工作（如 TimeSformer 的扩展版本）探索了可变的帧采样策略。

### 13.4 高分辨率下的显存压力

虽然 divided attention 减少了 attention 矩阵的大小，但 patch embedding 后的 token 总数仍然是 NF+1。对于高分辨率输入（如 448×448，patch 为 16×16，则 N=784），即使只有 8 帧，token 总数也超过 6000，显存开销仍然很大。TimeSformer 在论文中使用了 gradient checkpointing 来缓解这个问题。

### 13.5 位置编码的设计空间

论文使用了可学习 3D 位置编码，但没有深入探讨不同的位置编码设计。后续很多工作（如 Rotary Position Embedding、相对位置偏置）在视频 Transformer 上做了更深入的探索，可能带来进一步的提升。

### 13.6 分割/检测等 dense prediction 任务的缺位

TimeSformer 只在分类任务上做了验证，没有扩展到视频中的 dense prediction 任务（如视频实例分割、视频目标检测）。在这类需要逐像素输出的任务上，简单的 divided attention + cls token 架构可能不够，需要更复杂的 decoder 设计（如 DETR 在图像上的扩展）。

## 十四、小结

TimeSformer 是一篇思路极其清晰、实验设计非常完整的论文。它的核心贡献可以总结为一句话：

> **在视频识别中，不要一次性把所有时间和空间位置两两配对做 attention。先看时间、再看空间——不仅计算量更小，而且效果更好。**

这个"分步 attention"的思路虽然简单，但它在 2021 年的提出时机非常关键——刚好在 ViT 验证了图像 Transformer 的可行性之后、社区开始向视频探索的节点上。TimeSformer 用最干净的实验设计（五种 attention 方案的完全消融）告诉了社区：视频 Transformer 应该怎么做。

这篇论文的影响不仅仅在于它提出的 divided attention 方案本身。它改变了一个研究社区的思考方式——此后的视频 Transformer 工作，不需要再问"是否需要 3D 卷积"这个问题了，因为 TimeSformer 已经给出了回答。后续的方向变成了："如何设计更好的视频 Transformer"——这本身就是一个巨大的范式转变。

在技术树的 B5 分支中，TimeSformer 是源头。它之后的 MViT、Video Swin、ViViT 等工作，都是在 TimeSformer 建立的基础上做改进——有的加多尺度，有的改 token 化方式，有的引入移位窗口。但所有的起点都是这篇论文提出的"divide and conquer"思路。
