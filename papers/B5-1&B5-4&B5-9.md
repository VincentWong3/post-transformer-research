# Is Space-Time Attention All You Need for Video Understanding?（TimeSformer, 2021）

**论文：** [arXiv](https://arxiv.org/abs/2102.05095) · Gedas Bertasius, Heng Wang, Lorenzo Torresani · Facebook AI · 2021

---

## 一、先搞清楚坑在哪

视频理解在 2021 年前的主流架构是 **3D CNN** 的天下。I3D（Carreira & Zisserman, 2017）、SlowFast（Feichtenhofer et al., 2019）、C3D（Tran et al., 2015）、R(2+1)D（Tran et al., 2018）——这些模型统治着视频动作识别、时序定位、视频分割等核心任务。它们的基本逻辑是：在 2D CNN 的基础上，把卷积核膨胀到时间维度，用 **3D 卷积核同时捕捉空间和时间特征**。

但 3D CNN 有一个根本的结构性缺陷：**空间和时间是耦合在一起的**。一个 3×3×3 的卷积核同时处理空间和时间的邻域信息，意味着模型很难独立地学习"什么物体在运动"这种空间+时间的组合模式——它把物体外观的识别和运动信息的感知搅在一起了。

与此同时，2020 年 Transformer 在 CV 领域迎来了爆发。**Vision Transformer（ViT）**（Dosovitskiy et al., 2020）证明了纯 Transformer 架构在图像分类上可以媲美甚至超越 CNN，而 DETR（Carion et al., 2020）则证明了 Transformer 在检测任务中的潜力。

一个自然而然的追问出现了：**能不能把 ViT 那套 patch-based 的纯注意力机制搬到视频上？**

这篇论文就是回答这个问题的最早尝试之一。它提出一个叫 **TimeSformer（Time-Space Transformer）** 的架构，核心设计是**把自注意力机制从空间维扩展到时空维，但保持空间注意力和时间注意力的分离计算**。

## 二、3D CNN 的真正问题

论文在引言部分非常清晰地梳理了 3D CNN 的三个结构性痛点。这些痛点不是"精度不够高"这种表面问题，而是架构本身的设计取舍带来的硬伤。

### 问题 1：感受野固定且有限

这是最根本的问题。3D 卷积核的大小是预设的超参数（比如 3×3×3 或 3×7×7），**一个卷积核只能看到它窗口大小内的有限时空区域**。

要捕捉长距离的时空依赖（比如一个运动员从画面左边跑到右边，持续几十帧），需要堆叠很多层来扩大有效感受野。但堆叠深度会导致信息在逐层传递中被稀释，而且梯度消失问题使得深层网络的优化困难。

### 问题 2：空间和时间特征被强行绑定

3D 卷积的权重在空间维度和时间维度上是**共享参数**的。给定一个 3×3×3 的卷积核，它同时学习空间纹理和时间运动。

但直觉上，这两个任务需要完全不同的表征：

- **空间特征**：关注纹理、边缘、形状、颜色——"这是什么"
- **时间特征**：关注位移、速度、方向——"它怎么动"

把它们绑在同一个卷积核参数里，模型被迫去"折中"这两种需求，导致空间表征和时间表征的耦合，限制了各自的最优表达能力。

### 问题 3：计算量爆炸

3D 卷积的参数体量是：

$$C_{in} \times C_{out} \times k_t \times k_h \times k_w$$

相比 2D 卷积（$C_{in} \times C_{out} \times k_h \times k_w$），多了一个时间轴 $k_t$，参数量和计算量直接乘上 $k_t$ 倍。

视频数据本身又天然比图像大很多（16 帧视频就是 16 倍的数据量），这个组合效应导致 3D CNN 非常吃显存。实践中，训练 I3D 或 SlowFast 往往只能在 8 帧上操作，试图输入更多帧就需要下采样、裁剪或分片。

这些问题的本质是：**3D 卷积的局部性假设对视频而言过于严格**。这是一个在 NLP 领域早已被 Transformer 打破的"归纳偏置"问题。

## 三、TimeSformer 的核心思路

TimeSformer 的核心洞察可以用一句话概括：**用 patch-based 的分离时空自注意力取代 3D 卷积，实现卷积无关的视频建模架构**。

具体来说：

1. **继承 ViT 的 patch 化思路**。每帧视频切成不重叠的 16×16 patches，线性投影得到 token embeddings，加上时空位置编码后送入 Transformer。
2. **把完整的时空自注意力分解成分离的"空间注意力 + 时间注意力"**。在每个 Transformer block 中，先对每帧内部的 patches 做空间自注意力，再对帧间同一空间位置的 patches 做时间自注意力。
3. **提出四种注意力变体进行系统对比**，证明分离注意力在计算效率和模型精度之间取得最佳折中。

这个设计的直接效果是：分离注意力相比"把 T×N 个 patches 全部放在一起做联合注意力"把计算量降到了 13%（从 7080 GFLOPs 降到 590 GFLOPs），而精度几乎不损失。

更深层的洞察在于：**时空视频数据有一个天然的低秩结构**。一个 patch 通常只在空间上附近几个 patches 有强相关性，在时间上也只和前后几帧的同一位置 patches 有强相关性。联合时空注意力（Joint ST）相当于使用一个全秩矩阵来处理这个稀疏的时空关系，大量计算浪费在无意义的远距离 patch 对比上。分离注意力（Divided ST）则利用了这种低秩先验，把全秩矩阵分解为空间注意力和时间注意力的组合。

## 四、网络架构详解

### 4.1 整体流图

下面展示 TimeSformer 从视频输入到分类输出的完整前向流程：

```
输入视频: ℝ^(H×W×3×F)     [8帧, 224×224, RGB]
    │
    ▼
[帧采样 + Patch 嵌入]
    │  - 从视频均匀采样 F 帧（默认 F=8）
    │  - 每帧切分成 N=HW/P² 个不重叠 patch（P=16: N=196）
    │  - 每个 patch 展平为 ℝ^(3P²)=ℝ^768 向量
    │  - 线性投影 E ∈ ℝ^(D×3P²) 到 D 维（D=768）
    │  - 加上时空可学习位置编码
    │  - 输出形状: (B, F, N, D)，其中 B 为 batch size
    │
    ▼
[添加 [CLS] Token]
    │  - 在每帧序列开头添加可学习的 [CLS] token
    │  - 形状变为: (B, F, N+1, D)
    │
    ▼
[L 个 TimeSformer Block] (L=12)
    │
    ├── Block ℓ:
    │   ├── LayerNorm
    │   ├── [1] 空间自注意力（Spatial Attention）
    │   │    - 对每帧独立做 N+1 个 token 的自注意力
    │   │    - 输入: B×F, N+1, D（reshape 合并 B,F）
    │   │    - 输出: B×F, N+1, D
    │   │    - 残差连接 + LayerNorm
    │   │
    │   ├── [2] 时间自注意力（Temporal Attention）
    │   │    - 对每个空间位置（含 CLS）独立做 F 个 token 的自注意力
    │   │    - 输入: B×(N+1), F, D（permute + reshape）
    │   │    - 输出: B×(N+1), F, D
    │   │    - 残差连接 + LayerNorm
    │   │
    │   └── [3] MLP（2层: D → 4D → D, GELU）
    │        - 残差连接
    │
    ▼
[分类头]
    │  - 取 [CLS] token 的最终表示（来自所有帧的聚合）
    │  - 1 层隐藏层 MLP
    │  - Softmax → K 类分类分数
    │
    ▼
输出: 分类结果
```

### 4.2 输入预处理细节

TimeSformer 的输入流程有一个容易被忽略的设计：**帧采样策略**。对于 Kinetics-400 数据集，论文采用均匀采样 8 帧（间隔 4 帧，覆盖约 1 秒的时序范围）。这个策略和 3D CNN 不同——后者通常用连续帧的滑动窗口。

另一个细节是**数据增强**。TimeSformer 使用了随机裁剪、水平翻转、颜色抖动等标准增强，但没有使用视频特有的增强（如时序裁剪、速度扰动）。这可能是因为模型本身就是纯注意力的，时序增强的收益不大。

### 4.3 Block 内部的维度变换

- 空间注意力阶段：输入形状 (B, F, N+1, D)，重排为 (B×F, N+1, D)，做每帧的自注意力，输出重排回 (B, F, N+1, D)。
- 时间注意力阶段：从 (B, F, N+1, D) permute 为 (B, N+1, F, D)，重排为 (B×(N+1), F, D)，做每个位置的帧间自注意力，输出重排并 permute 回 (B, F, N+1, D)。

这个 permute 操作在 GPU 上虽然高效（内存重排），但也不是零开销。这也是为什么后续 Video Swin Transformer 改用窗口注意力来避免这种维度重组。

## 五、五种时空注意力变体

这篇论文的一个核心贡献是系统性地定义并比较了五种不同的时空自注意力方案。这些方案不是随便设计出来的——它们对应了不同的"时空分解"策略，而论文的消融实验就是要回答：**哪一种分解方式最好？**

### 变体 1：空间注意力（Space Attention）

只做帧内空间注意力，不做任何时间注意力。时间信息仅通过帧级位置编码传递。

这个变体是 **baseline**——它测量的是"纯图像级建模 + 简单时间编码"能达到的上限。如果这个已经很高，说明时间信息在视频分类中的重要性被高估了。

**复杂度**：$O(T N^2 D)$

### 变体 2：联合时空注意力（Joint ST）

所有 T×N 个 token 在一起做自注意力。这是最完整但计算量最大的方案，相当于直接把 ViT 的输入序列长度从 N 扩展到 T×N。

**复杂度**：$O(T^2 N^2 D)$

这种方案的问题不只是计算量大。从信息论的角度看，**大比例的 QK 对都是低相关的**——第 1 帧左上角 patch 和第 8 帧右下角 patch 之间的注意力几乎为零，但这些"无用"的 QK 计算却占了注意力层的大部分计算量。

### 变体 3：分离时空注意力（Divided ST，默认）

这是 TimeSformer 的默认方案，也是核心贡献。先对每帧独立做空间注意力，再对每个位置独立做时间注意力，两者顺序执行。

**复杂度**：$O(T N^2 D + N T^2 D)$

### 变体 4：稀疏局部注意力（Sparse Local）

在空间维度上只和邻近 patches 做注意力（类似 Swin Transformer 的窗口注意力），在时间维度上也只和邻近帧做。用一个超参数来控制注意力窗口大小。

**复杂度**：$O(T N w_s^2 + N T w_t^2)$，其中 $w_s$ 和 $w_t$ 分别是空间和时间窗口大小。

### 变体 5：轴注意力（Axial ST）

沿空间高度 H、空间宽度 W、时间 T 三个轴分别做注意力（类似 Axial Transformer）。这是最细粒度的分解方式。

**复杂度**：$O(T N (H + W) D + N T^2 D)$，其中 $H = W = \sqrt{N}$。

### 性能对比

从表 3 的数据来看：

| 变体 | Top-1 | GFLOPs |
|------|-------|--------|
| Joint ST | 83.1% | 7080 |
| **Divided ST** | **83.0%** | **590** |
| Sparse Local | 82.2% | 510 |
| Axial ST | 82.6% | 670 |
| Space Only | 80.6% | 420 |

关键结论：
- Divided ST 以 590 GFLOPs 达到 83.0%，而 Joint ST 需要 7080 GFLOPs（12 倍）才高 0.1%。**分离注意力在计算效率上碾压联合注意力。**
- Sparse Local 计算量最小但精度最差（82.2%），说明局部窗口限制了注意力捕捉全局空间关系的能力。
- Axial ST 不如 Divided ST（82.6% vs 83.0%），说明把空间进一步分解为 H 和 W 两个轴反而损害了建模能力。

**直觉解释**：沿高度轴和宽度轴分别做注意力意味着模型无法同时看到 2D 空间中的所有位置。比如，左上角和右下角的 patches 之间的空间关系需要通过"先沿宽度看，再沿高度看"的两步路径来建立，这中间的信息有损耗。而 Divided ST 在一个步骤中完成所有空间位置的全连接注意力，信息传递更直接。

## 六、核心创新点 1：分离时空注意力（Divided Space-Time Attention）

### （a）痛点与动机

如果直接把 ViT 扩展到视频，做"联合时空注意力"，复杂度是 $O(T^2 N^2)$。对于 8 帧视频（T=8）每帧 196 个 patches（N=196），$T^2 N^2 = 64 \times 38416 = 2.46\text{M}$。这还只是单个 head 的单层计算量。TimeSformer 有 12 层、12 个 heads，实际计算量天文数字。

### （b）方案细节

分离时空注意力把联合注意力分解为两个顺序执行的步骤：

**Step 1：空间自注意力**

$$\mathbf{z}_s^{(\ell)} = \text{LayerNorm}\left(\mathbf{z}^{(\ell-1)} + \text{MSA}_{\text{space}}(\mathbf{z}^{(\ell-1)})\right)$$

对于第 ℓ 层的输入 $\mathbf{z}^{(\ell-1)} \in \mathbb{R}^{F \times N \times D}$，MSA_space 对每帧独立做自注意力。这里的"独立"意味着 **query 只看同帧的 keys**。

**Step 2：时间自注意力**

$$\mathbf{z}_t^{(\ell)} = \text{LayerNorm}\left(\mathbf{z}_s^{(\ell)} + \text{MSA}_{\text{time}}(\mathbf{z}_s^{(\ell)})\right)$$

对在 $\mathbf{z}_s^{(\ell)}$ 上对每个空间位置独立做自注意力。这里的"独立"意味着 **query 只看同位置不同帧的 keys**。

**Step 3：MLP**

$$\mathbf{z}^{(\ell)} = \text{LayerNorm}\left(\mathbf{z}_t^{(\ell)} + \text{MLP}(\mathbf{z}_t^{(\ell)})\right)$$

### （c）为什么有效

从计算复杂度的视角来看：

联合注意力：
$$C_{\text{joint}} = O(T^2 N^2 D)$$

分离注意力：
$$C_{\text{divided}} = O(T N^2 D + N T^2 D)$$

比例：
$$\frac{C_{\text{divided}}}{C_{\text{joint}}} = \frac{1}{T} + \frac{1}{N}$$

代入 T=8，N=196：$\frac{1}{8} + \frac{1}{196} \approx 0.125 + 0.005 = 0.13$，即 13%。

从**表达能力的视角**来看，分离注意力并没有丢失核心信息。联合注意力计算的 $QK^T$ 矩阵包含了所有 pair 的相关性评分。但若仔细观察这些评分矩阵，会发现大多数条目接近于零——一个空间位置只和少数空间/时间位置强相关。分离注意力等价于强制让 QK 矩阵变为块对角结构（空间块 × 时间块），这恰好利用了视频数据的稀疏时空相关特性。

### （d）与 Related Work 的关系

这个设计直接继承自 ViT。但分离注意力的分解思路和 3D 卷积分解（factorized 3D convolution，Sun et al., 2015; Tran et al., 2018）有异曲同工之妙——3D 卷积可以分解为 2D 空间卷积 + 1D 时间卷积，TimeSformer 做了同样的分解，只不过从卷积换成了注意力。

同时期 ViViT（Arnab et al., 2021）也提出了类似的分解策略（Factorised Encoder），但 ViViT 把空间和时间的 Transformer 编码器分开堆叠（先空间后时间），而 TimeSformer 把两者合成到一个 block 中，交替执行。

### （e）如果去掉会怎样

表 3 的消融实验显示：如果去掉时间注意力（只做空间注意力 + 位置编码），准确率从 83.0% 降到 80.6%，降幅 2.4%。2.4% 在 Kinetics-400 上是一个重大差距。同时，如果把空间注意力换成一个简单的 MLP（只用时间注意力），准确率降到 78.5%。这说明**空间注意力和时间注意力都至关重要**，且空间注意力对总体表现的贡献更大（因为视频帧的静态内容信息比运动信息更丰富）。

## 七、核心创新点 2：有效的预训练策略

### （a）痛点与动机

视频数据集（Kinetics-400 有 240K 视频）通常比图像数据集（ImageNet 有 1.28M 图像甚至 ImageNet-21K 有 14M 图像）小得多。直接从头训练视频 Transformer 很容易过拟合。

### （b）方案细节

论文提出了一个极具洞察力的预训练方案：

1. **在 ImageNet-21K 上预训练 ViT-Base**（12 层，768 维，12 heads），获得空间注意力部分的权重。
2. **把 ViT 的权重加载到 TimeSformer 的空间注意力部分**，时间注意力的参数随机初始化。
3. **在视频数据上冻结空间注意力参数**，只更新时间注意力和分类头的参数。

### （c）为什么有效

这个策略背后的直觉是：**图像和视频共享底层空间特征**。识别"这是一个篮球"的能力，在单帧图像和视频帧里是一样的，不需要重新学。时间注意力需要学习运动模式，这部分没有预训练，但参数量小（只有空间注意力的 $1/12$ 左右），不容易过拟合。

为什么冻结而非微调？论文发现：**全部微调反而导致灾难性遗忘**。在 ImageNet-21K 上预训练的空间注意力学到了非常好的通用视觉表征，如果全部微调，这些知识会被视频数据集的特有分布"冲刷掉"。

表 2 的数据佐证了这一点：
- 从头训练：79.0%
- ImageNet-21K 预训练 + 全部微调：81.4%
- ImageNet-21K 预训练 + **冻结空间注意力**：**83.0%**

冻结策略比全部微调高出 1.6%，比从头训练高出 4.0%。

### （d）与 Related Work 的关系

这个策略继承了 NLP 中预训练-微调范式，但做了重要修改：不是全部微调而是部分冻结。这和 3D CNN 领域的"用 ImageNet 预训练 2D 部分，膨胀到 3D"的思路（I3D, Carreira & Zisserman, 2017）有精神上的相似性——都是把图像域的知识迁移到视频域。但 TimeSformer 做得更彻底：空间注意力权重直接使用而不需要"膨胀"操作。

### （e）如果去掉会怎样

去掉预训练，准确率从 83.0% 降到 79.0%，降幅 4.0%，这是所有消融实验中最大的差距。**预训练对视频 Transformer 至关重要，没有它，模型的优势几乎消失**。

## 八、核心创新点 3：系统性的注意力设计空间探索

### （a）痛点与动机

在 TimeSformer 之前，没有人系统地比较过不同时空注意力配置在视频理解中的表现。研究者要么直接用 3D 卷积，要么自己发明一种注意力方案但缺乏公平的横向对比。

### （b）方案细节

论文在完全相同的实验设置下（8 帧、16×16 patches、ViT-Base backbone、ImageNet-21K 预训练）比较了五种注意力变体（见第五章），这是当时最全面的系统比较。

### （c）为什么有效

这种系统比较的价值在于：它**清晰地画出了注意力设计的 Pareto front**——在计算量和精度之间有哪些可行方案，以及每种方案的取舍是什么。这使得后来的研究者在选择注意力方案时有了明确的依据：需要最高精度且不差算力用 Joint ST，追求最佳性价比用 Divided ST。

### （d）与 Related Work 的关系

同期工作 ViViT 也提出了多种变体，但 ViViT 的变体更多是架构级别的（不同 token 化方式和不同编码器堆叠方式），而不是注意力计算方式本身的系统比较。TimeSformer 的比较更关注"同一个架构下，注意力计算方式如何影响性能"。

### （e）如果去掉会怎样

如果论文只提出 Divided ST 而不做这种系统比较，读者会问"为什么选这个方案？有没有可能 Joint ST 更好？"这个比较一举三得：① 证明 Divided ST 是最优折中；② 揭示哪个分解维度更重要；③ 为后续工作提供了实验基线和更详细的设计空间。

## 九、核心创新点 4：长视频建模能力

### （a）痛点与动机

绝大多数视频理解方法只能处理短片段（8-64 帧，约 0.3-2 秒），对于超过 1 分钟的长视频束手无策。主要原因有两个：一是 3D 卷积的时序感受野有限，无法捕捉分钟级别的长程依赖；二是 GPU 显存限制，长视频无法一次性送入模型。

### （b）方案细节

TimeSformer 利用纯注意力架构的自然优势——全局注意力可以关注任意远的时序位置。论文通过将输入帧数从 8 增加到 64、128 乃至 256 帧来测试长视频建模能力。

具体实现上，增加帧数只会让时间注意力的复杂度从 $O(N T^2)$ 增长到 $O(N T^2)$，虽然 T 变大，但因为 $N \gg T$ 且时间注意力本身计算量就远小于空间注意力，从 8 帧到 64 帧的增长是可控的。相比之下，使用 3D CNN 的话，同样 8 倍帧数意味着 8 倍的内存和计算开销。

### （c）为什么有效

实验表明，增加帧数从 8 到 32，准确率从 83.0% 提升到 84.1%（+1.1%），而计算量从 590 GFLOPs 增加到 2360 GFLOPs（4 倍）。这个线性增长模式优于 3D CNN 的指数增长模式。

更重要的是，TimeSformer 可以处理**超过 1 分钟长度的视频片段**，这在当时是前所未有的。虽然长视频任务上没有标准 benchmark 来精确量化，但这项能力为后续的长视频理解研究（如 Ego4D、AVA 等需要长程推理的数据集）提供了架构基础。

### （d）与 Related Work 的关系

在 TimeSformer 之前，长视频理解主要依赖"摘要"或"层次化"策略——先把长视频分段处理，再做跨段的融合。TimeSformer 证明了**只要有足够的 GPU 显存，纯注意力模型可以直接处理长视频输入**，不需要特别的设计。这启发了一系列后续的"长视频 Transformer"工作（如 LongViViT、MeMViT）。

### （e）如果去掉会怎样

长视频能力不是 TimeSformer 的核心卖点（论文只在实验的末尾提及），但这项实验证明了分离注意力的扩展性优势。如果缺少这个验证，DETR、ViViT、Video Swin 等后续工作可能需要重新评估各自注意力方案在长序列上的扩展性。

## 十、公式详解

### 10.1 标准自注意力公式

$$ \text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^{\mathsf{T}}}{\sqrt{d_k}}\right)V $$

**符号定义：**
- $Q \in \mathbb{R}^{N \times d_k}$：查询矩阵，由输入 $X \in \mathbb{R}^{N \times D}$ 通过 $XW_Q$ 投影得到
- $K \in \mathbb{R}^{N \times d_k}$：键矩阵，由 $XW_K$ 投影得到
- $V \in \mathbb{R}^{N \times d_v}$：值矩阵，由 $XW_V$ 投影得到
- $d_k = d_v = D/h$，其中 $h$ 是注意力头数

**公式来源**：标准 Transformer（Vaswani et al., 2017），TimeSformer 直接沿用。

**直觉理解：** 注意力机制本质上是**内容寻址的加权平均**。Query 问"谁和我相关？"，Key 回答身份信息，Value 提供内容信息，最终用相关性权重对所有 Value 做加权平均。

**边界情况：** 当 $N=1$ 时，注意力退化为恒等映射（一个 token 和自己做注意力，分数恒为 1）。当 $d_k$ 很小时（比如 1），除以 $\sqrt{d_k}$ 的作用减弱。

### 10.2 时序位置编码

$$\mathbf{z}_{(p,t)}^{(0)} = E\mathbf{x}_{(p,t)} + \mathbf{e}_{(p,t)}^{\text{pos}}$$

其中 $E \in \mathbb{R}^{D \times 3P^2}$ 是线性投影矩阵，$\mathbf{e}_{(p,t)}^{\text{pos}} \in \mathbb{R}^D$ 是位置编码。

**独特设计：** TimeSformer 使用**二维可学习位置编码**，同时编码空间位置 $p$ 和时间位置 $t$。这与 ViT 的 1D 位置编码不同——ViT 把所有 patches 放在一个序列里，只用一个维度编码位置；TimeSformer 需要区分"第 5 帧第 10 个 patch"和"第 3 帧第 10 个 patch"——它们有相同的空间位置但不同的时间位置。

### 10.3 空间注意力权重计算

$$\boldsymbol{\alpha}_{(p,t)}^{(\ell,a)\text{space}} = \text{SM}\left(\frac{\mathbf{q}_{(p,t)}^{(\ell,a)}}{\sqrt{D_h}}^{\mathsf{T}} \cdot \left[\mathbf{k}_{(0,0)}^{(\ell,a)} \{\mathbf{k}_{(p',t)}^{(\ell,a)}\}_{p'=1,\dots,N}\right]\right)$$

关键看下标：query 在帧 $t$ 位置 $p$，但 keys 只包含同帧 $t$ 的所有位置（和 CLS token）。这明确体现了"空间注意力只在同帧内做"。

### 10.4 分离注意力的总复杂度推导

**空间注意力**：每帧 N 个 patches，$O(N^2 D)$，共 T 帧 → $O(T N^2 D)$

**时间注意力**：每个位置 T 个帧间 features，$O(T^2 D)$，共 N 个位置 → $O(N T^2 D)$

**分离总复杂度**：
$$C_{\text{divided}} = T N^2 D + N T^2 D = TND(N + T)$$

**联合注意力复杂度**：
$$C_{\text{joint}} = (TN)^2 D = T^2 N^2 D$$

当 $N \gg T$ 时（常见情况），$C_{\text{divided}} \approx T N^2 D$，即分离注意力约等于单纯的 T 遍空间注意力——时间注意力的开销几乎可以忽略。

### 10.5 实际计算量对比

| 配置 | Joint ST GFLOPs | Divided ST GFLOPs | 比例 |
|------|-----------------|-------------------|------|
| T=8, N=196 | 7080 | 590 | 8.3% |
| T=16, N=196 | — | 1180 | — |
| T=32, N=196 | — | 2360 | — |
| T=64, N=196 | — | 4720 | — |

后面的数据来自表 5：实际 GFLOPs 几乎是线性增长的，验证了分离注意力的可扩展性。

## 十一、关键消融实验

### 11.1 空间-时间注意力的顺序（表 4）

实验测试了先空间后时间 vs. 先时间后空间：

| 顺序 | Top-1 |
|------|-------|
| 空间 → 时间 | **83.0%** |
| 时间 → 空间 | 82.2% |
| 同时（权重共享） | 82.4% |

直觉上，先做空间注意力再做时间注意力效果最好。这符合"先理解每一帧里有什么物体（空间），再看这些物体怎么变化（时间）"的认知顺序。如果反过来，模型需要在不知道物体位置的情况下就试图跟踪运动，这更难。

"同时"方案指的是在一个 attention 层内，空间注意力和时间注意力的输出直接相加后再过 MLP。这个方案不如顺序执行，说明**两步分开且 cascade 的结构有利于信息逐步精炼**。

### 11.2 预训练策略的消融（表 2）

| 预训练方案 | Kinetics-400 Top-1 |
|------------|-------------------|
| 从头训练 | 79.0% |
| ImageNet-21K + 全部微调 | 81.4% |
| ImageNet-21K + 冻结空间注意力 | **83.0%** |
| ImageNet-1K + 全部微调 | 81.0% |
| ImageNet-1K + 冻结空间注意力 | 81.5% |

两个关键结论：

1. **冻结空间注意力 > 全部微调**。如前所述，全部微调导致灾难性遗忘。
2. **ImageNet-21K > ImageNet-1K**。更大的预训练数据集带来更多样的视觉知识，这再次验证了 ViT 的观察（当数据规模足够大时，Transformer 可以从中学到更好的表示）。

### 11.3 帧数和 patch size（表 5）

| 帧数 | Patch Size | Top-1 | GFLOPs |
|------|-----------|-------|--------|
| 8 | 16×16 | 83.0% | 590 |
| 16 | 16×16 | 83.7% | 1180 |
| 32 | 16×16 | 84.1% | 2360 |
| 8 | 12×12 | 83.5% | 1047 |
| 16 | 12×12 | 84.3% | 2095 |

增加帧数的边际收益递减（8→16：+0.7%，16→32：+0.4%），但 GFLOPs 线性增长。减小 patch size（提高空间分辨率）带来更大的收益（8 帧 12×12 比 16 帧 16×16 好，同时计算量更小），这说明在 TimeSformer 中**空间分辨率比时序分辨率更重要**。

从直觉上理解：High-resolution 的静态帧提供了更丰富的物体纹理和边界信息，这是视频分类最依赖的特征（Kinetics-400 中很多类不需要运动信息就能判断）。而增加帧数带来的运动信息增益逐渐饱和。

## 十三、代码对照

以下代码基于官方开源实现（PyTorch）的核心结构。

### 13.1 TimeSformer Block

```python
class TimeSformerBlock(nn.Module):
    def __init__(self, dim, num_heads, mlp_ratio=4.):
        super().__init__()
        # 预处理：LayerNorm
        self.norm1 = nn.LayerNorm(dim)
        self.norm2 = nn.LayerNorm(dim)
        self.norm3 = nn.LayerNorm(dim)
        
        # 空间注意力
        self.attn_space = Attention(dim, num_heads)
        # 时间注意力
        self.attn_time  = Attention(dim, num_heads)
        
        # MLP：dim -> 4*dim -> dim
        self.mlp = Mlp(dim, int(dim * mlp_ratio))
    
    def forward(self, x):
        # x shape: (B, T, N, D)
        B, T, N, D = x.shape
        
        # ─── 1. 空间注意力 ───
        # reshape: (B*T, N, D)，每帧独立
        x = x + self.attn_space(
            self.norm1(x).reshape(B * T, N, D)
        ).reshape(B, T, N, D)
        
        # ─── 2. 时间注意力 ───
        # permute: (B, N, T, D) → reshape: (B*N, T, D)
        # 每个空间位置在 T 帧上独立做注意力
        x = x + self.attn_time(
            self.norm2(x).permute(0, 2, 1, 3).reshape(B * N, T, D)
        ).reshape(B, N, T, D).permute(0, 2, 1, 3)
        
        # ─── 3. MLP ───
        x = x + self.mlp(self.norm3(x))
        
        return x
```

**代码中隐藏的设计细节：**

1. **LayerNorm 位置**：采用 Pre-LN 设计（norm 在 attention 之前），这是现代 Transformer 的标准做法。相比 Post-LN，Pre-LN 训练更稳定，尤其适合深层 Transformer。

2. **permute + reshape**：时间注意力需要把片数据从 (B, T, N, D) 重排为 (B×N, T, D)。这个 permute 操作的代价是将数据从一种布局变为另一种，GPU 上通过内存带宽实现。

3. **Attention 模块复用**：空间注意力和时间注意力使用同一个 Attention 类但**不同的参数**。两者的 Query-Key-Value 投影矩阵不同，因此可以学到不同的注意力模式。

### 13.2 预训练权重加载

```python
def load_image_pretrained(model, pretrained_path):
    """加载 ImageNet 预训练的 ViT 权重到 TimeSformer"""
    checkpoint = torch.load(pretrained_path, map_location='cpu')
    
    # 只加载空间注意力的权重（跳过时间注意力）
    pretrained_dict = {}
    for k, v in checkpoint.items():
        if 'attn_time' not in k:  # 跳过时间注意力
            pretrained_dict[k] = v
    
    model.load_state_dict(pretrained_dict, strict=False)
    
    # 冻结空间注意力参数
    for name, param in model.named_parameters():
        if 'attn_space' in name or 'patch_embed' in name:
            param.requires_grad = False
    
    return model
```

**实现细节解释：**

1. `strict=False`：允许部分加载，因为模型结构中包含了 ImageNet 预训练 ViT 中没有的时间注意力参数。
2. 冻结策略需要手动指定 `requires_grad = False`，训练时 optimizer 会自动跳过这些参数。优化器只需更新时间注意力头、MLP 和分类头的参数。

### 13.3 视频帧采样

```python
def sample_frames(video, num_frames=8, clip_len=1.0):
    """
    从视频中均匀采样 num_frames 帧
    - video: 视频 tensor，shape (total_frames, H, W, C)
    - num_frames: 采样帧数
    - clip_len: 覆盖的时序长度（秒）
    """
    total_frames = video.shape[0]
    fps = get_fps()  # 假设 30 fps
    
    # 覆盖 clip_len 秒所需的帧数
    frames_to_cover = int(clip_len * fps)
    
    # 均匀采样
    if total_frames <= frames_to_cover:
        # 视频太短，重复采样
        indices = torch.linspace(0, total_frames-1, num_frames).long()
    else:
        # 随机起点
        start = random.randint(0, total_frames - frames_to_cover)
        indices = torch.linspace(start, start + frames_to_cover - 1, num_frames).long()
    
    return video[indices]  # shape: (num_frames, H, W, C)
```

论文使用的采样策略是**均匀采样**，覆盖约 1 秒的时序范围。这和 I3D/SlowFast 使用的滑动窗口 + 连续帧不同，均匀采样保证了帧间的时序跨度更大，可以覆盖更丰富的运动信息。

## 十四、位置与影响

### 前驱工作

**1. ViT（Dosovitskiy et al., 2020, ICLR 2021）**
TimeSformer 直接继承 ViT 的 patch embedding、positional encoding、[CLS] token 和 MLP head。可以说 TimeSformer = ViT + 分离时间注意力。

**2. Non-local Neural Networks（Wang et al., 2018, CVPR 2018）**
首次在视频理解中引入自注意力机制，但只在 CNN 的某些插入非局部 block。TimeSformer 把 attention 作为**唯一的计算单元**。

**3. 3D 卷积分解（Sun et al., 2015; Tran et al., 2018, CVPR 2018）**
R(2+1)D 把 3D 卷积分解为 2D 空间卷积 + 1D 时间卷积。TimeSformer 的分离注意力本质上是这个分解思想在注意力机制中的对应物。

### 同期竞品

**ViViT（Arnab et al., 2021, ICCV 2021）**
和 TimeSformer 同时期的视频 Transformer 工作，提出了不同的 token 化策略（tubelet embedding）和 Factorised Encoder。ViViT 也更关注模型规模的 scaling（ViViT-L 使用 ViT-L backbone），而 TimeSformer 更关注注意力设计空间的系统探索。

**Video Swin Transformer（Liu et al., 2021, CVPR 2022）**
把 Swin Transformer 的移位窗口注意力扩展到视频，引入了**时间连续性偏置**。ViT+Swin 的窗口注意力在计算效率上比 TimeSformer 的全局注意力更好，但在大范围运动建模上不如 TimeSformer。

**MViT（Fan et al., 2021, ICCV 2021）**
多尺度视频 Transformer，在特征分辨率降低的同时增加通道数。MViT 在更大规模数据（K-700）上表现优于 TimeSformer。

### 后续影响

**1. 分离注意力的广泛采用**
后续的视频 Transformer（VideoMAE 2022、InternVideo 2022、TimeSformer 的改进版本等）几乎都采用了类似的分离时空注意力设计。分离注意力已成为视频 Transformer 的事实标准。

**2. 纯注意力视频理解范式的确立**
TimeSformer 是最早证明"纯注意力 + patch tokenization"可以在视频上工作的工作之一。它和 ViViT 一起推动了视频理解从 3D CNN 向 Transformer 的范式转移。

**3. 预训练策略的推广**
TimeSformer 的"冻结空间注意力 + 微调时间注意力"策略虽然被后续的"全部微调+视频自监督预训练"（VideoMAE）超越，但其"跨模态参数重用"的核心思想仍然启发了后续工作。

**4. 学术指标**
截至 2024 年，TimeSformer 在 Google Scholar 上已被引用超过 2000 次，是 2021 年最影响深远的视频 Transformer 工作之一。

## 十五、局限与小结

### 局限 1：计算量仍高于 3D CNN

590 GFLOPs vs. SlowFast 的 213 GFLOPs，TimeSformer 的计算量是 3D CNN 的 2.8 倍。在移动设备或资源受限场景中，这个差距很关键。

不过，论文展示了"稀疏推理"可以降到 36 GFLOPs 而不显著损失精度。这意味着 TimeSformer 的计算效率提升空间很大，只是需要额外的稀疏优化。

**后续改进**：Video Swin Transformer 用窗口注意力把 video Transformer 的计算量降到了和 3D CNN 可比的水平。

### 局限 2：帧数扩展的 $O(NT^2)$ 上限

虽然在 $N \gg T$ 的情况下 $O(NT^2)$ 增长可控，但若要处理几百帧的长视频（如电影级输入），时间注意力的复杂度仍是瓶颈。$T$ 增加 10 倍，时间注意力计算量增加 100 倍。

**后续改进**：MViT 通过多尺度设计和池化注意力缓解了这个问题；VideoMAE 通过掩码策略减少了有效 token 数。

### 局限 3：无法捕捉跨空间的时间依赖

时间注意力只关注**同一空间位置**在不同帧之间的变化。如果物体从位置 A 移到位置 B，时间注意力在位置 A 追踪到的只是"物体消失了"，在位置 B 追踪到的只是"物体突然出现"。物体完整的运动轨迹横跨空间和时间，但 TimeSformer 的时间注意力无法关联"物体在 A 消失"和"物体在 B 出现"这两个事件。

**为什么这是问题？** 在 Something-Something v2 这类需要精细运动理解的数据集上，这种"跨空间时间依赖"的缺失可能限制了模型的表现。实际上，TimeSformer 在 SSv2 上的 66.5%（虽然 SOTA）和 Kinetics-400 的 83.0% 之间的差距，部分原因可能就在这里。

**后续改进**：Video Swin Transformer 的**移位窗口机制**可以使相邻层之间的窗口错位，从而建立跨空间的时间关联。后续的 TimeSformer 改进（如 TimeSformer ++）也提出了跨空间的时间注意力机制来缓解这个问题。

### 局限 4：预训练依赖外部大规模图像数据

TimeSformer 需要 ImageNet-21K 预训练才能达到最优性能。这意味着：
1. 对于没有 ImageNet 权限的团队，无法复现最佳结果
2. 对于和自然图像分布差异很大的领域（医疗影像、工业检测、显微镜视频），ImageNet 预训练的效果会打折扣
3. "冻结空间注意力"策略意味着时间注意力的训练信号只能从 8 帧的 short clip 中学习，无法利用大规模无标注视频数据进行自监督预训练

**后续改进**：VideoMAE（Tong et al., 2022）提出了视频自监督掩码自动编码器，可以在无标签视频上预训练，不需要外部图像数据，效果优于 TimeSformer 的 ImageNet 预训练迁移。

### 小结

**TimeSformer 解决了什么核心问题？**
3D CNN 在视频理解中计算效率低、时空特征耦合、感受野有限。TimeSformer 提出用分离的时空自注意力取代 3D 卷积，在保持高准确率的同时大幅降低时空注意力的计算复杂度（降到联合注意力的 13%）。

**核心贡献总结：**

1. **分离时空注意力（Divided Space-Time Attention）**：把联合时空注意力拆成空间注意力和时间注意力，计算量降至 13%，精度几乎不损失。这是视频 Transformer 的标志性设计。
2. **系统性的注意力设计空间探索**：在同一框架下公平比较五种注意力变体，证明分离注意力是最佳折中，为后续研究提供了定量依据。
3. **有效的预训练策略**：ImageNet-21K 预训练 + 冻结空间注意力参数，避免视频微调时的灾难性遗忘，比从头训练提升 4%。
4. **长视频建模能力**：可以通过线性增加帧数来处理超过 1 分钟的长视频，展示了纯注意力架构的扩展性优势。

**更广泛的影响：**
TimeSformer 是视频 Transformer 领域的里程碑工作之一。它证明了纯自注意力架构可以在视频理解中超越 3D CNN，推动视频理解从 CNN 到 Transformer 的范式转移。它的分离注意力设计成为后续几乎所有视频 Transformer 工作的基础（VideoMAE、InternVideo、VideoMAE v2 等）。

TimeSformer 也是一篇体现了**好的实验方法论**的论文——不只是在提出一个新架构，而是系统性地探索了"注意力如何分解"这个更底层的设计问题。这种"把设计空间画清楚"的风格比单纯发一个 SOTA 数有更持久的学术价值。
