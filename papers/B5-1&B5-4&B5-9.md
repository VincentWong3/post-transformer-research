# # Is Space-Time Attention All You Need for Video Understanding?（2021）

**论文：** [arXiv](https://arxiv.org/abs/2102.05095) · Gedas Bertasius, Heng Wang, Lorenzo Torresani · 2021

---

## 一、先搞清楚坑在哪

视频理解在2021年前后，主流架构还是3D CNN的天下。I3D、SlowFast、C3D这些模型统治着动作识别、时序定位等任务。它们的基本逻辑是：在2D CNN的基础上，把时间维度和空间维度一起卷积，用3D卷积核同时捕捉空间和时间特征。

这带来了一个根本问题：**空间和时间是耦合在一起的**。3D卷积核同时处理空间和时间维度，意味着模型很难独立地学习"某个物体在移动"这种空间+时间的组合模式——它把物体的外观和运动搅在一起了。

与此同时，Transformer在NLP领域已经大杀四方，ViT（Vision Transformer，2020）刚刚证明纯Transformer架构在图像分类上可以媲美甚至超越CNN。一个自然的问题是：**能不能把ViT那套纯注意力机制搬到视频上？**

## 二、3D CNN的真正问题

3D CNN有三个具体痛点，这篇论文在引言里说得很清楚：

**1. 感受野固定且有限**

3D卷积核的大小是预设的超参数（比如3×3×3或3×7×7）。一个卷积核只能看到它窗口大小的时空区域。要捕捉长距离的时空依赖（比如一个运动员从画面左边跑到右边，持续几十帧），需要堆叠很多层，这又带来梯度消失和计算开销问题。

**2. 空间和时间特征混在一起学**

3D卷积的权重在空间和时间维度上是共享的。这意味着"这个物体是什么"（空间特征）和"它怎么动"（时间特征）被同一个卷积核参数处理。但直觉上，这两个任务需要不同的表征：识别物体需要纹理、边缘、形状信息；理解运动需要光流、位移信息。把它们绑在一起学，模型效率低。

**3. 计算量爆炸**

一个3D卷积核的参数是 $C_{in} \times C_{out} \times k_t \times k_h \times k_w$。相比2D卷积（$C_{in} \times C_{out} \times k_h \times k_w$），多了一个时间维度$k_t$，参数量和计算量直接乘上$k_t$倍。而视频数据天然比图像大很多（16帧视频就是16倍数据量），这个组合效应导致3D CNN非常吃显存。

## 三、TimeSformer的核心思路

这篇论文的核心洞察非常简洁：**把自注意力机制从空间扩展到时空，但保持空间和时间注意力的分离计算**。

具体来说，TimeSformer（Time-Space Transformer）把视频帧切成patches，然后对每个patch在空间维度和时间维度分别做自注意力。这样做的好处是：

1. 模型可以灵活地关注任意位置的时空关系（不像3D CNN受限于固定核大小）
2. 空间注意力和时间注意力解耦，可以独立学习外观和运动
3. 计算效率比3D CNN高（后文会详细分析复杂度）

**直白说**：TimeSformer不是把3D CNN换成3D Transformer，而是把"空间注意力"和"时间注意力"分开做，先让每个patch关注同一帧内其他patch（空间），再让它关注不同帧的同一位置patch（时间）。

## 四、网络架构详解

### 4.1 整体架构

TimeSformer的整体架构基于ViT，但增加了时间注意力模块。图1展示了完整的处理流程：

![图1：TimeSformer架构示意图，展示了从视频帧输入到分类输出的完整流程](figures/timesformer/fig1.png)

**前向过程（逐模块，带形状标注）：**

```
输入视频: T × H × W × 3
    │
    ▼
[帧采样 + Patch嵌入]
    │  - 从视频中均匀采样T帧 (T=8)
    │  - 每帧切分成N个patches (N=H/P × W/P, P=16)
    │  - 每个patch线性投影到D维 (D=768)
    │  - 输出形状: T × N × D
    │
    ▼
[位置编码]
    │  - 空间位置编码: 为每个patch在帧内的位置添加可学习编码
    │  - 时间位置编码: 为每帧添加可学习编码
    │  - 输出形状: T × N × D (不变)
    │
    ▼
[L个TimeSformer Block]
    │
    ├── [空间注意力] (每个block)
    │   │  - 对每帧内的N个patches做自注意力
    │   │  - 输入: T × N × D → 输出: T × N × D
    │   │
    ├── [时间注意力] (每个block)
    │   │  - 对帧间同一位置的patches做自注意力
    │   │  - 输入: N × T × D → 输出: N × T × D
    │   │
    └── [MLP + LayerNorm + 残差连接]
    │      - 标准Transformer block结构
    │
    ▼
[CLS Token分类头]
    │  - 取第一个token (CLS token) 经过MLP分类
    │  - 输出: K类分类分数
    │
    ▼
输出: 分类结果
```

### 4.2 TimeSformer Block详解

每个TimeSformer Block包含两个注意力模块和一个MLP。图2展示了block内部结构：

![图2：TimeSformer Block内部结构，展示了空间注意力和时间注意力的顺序执行](figures/timesformer/fig2.png)

**关键设计决策：为什么先空间后时间？**

论文实验发现，先做空间注意力再做时间注意力效果最好（表4）。直觉上，这符合"先理解每一帧里有什么，再看这些物体怎么随时间变化"的认知顺序。如果反过来（先时间后空间），模型需要在不知道物体位置的情况下就试图跟踪运动，这更难。

### 4.3 四种注意力变体

论文提出了四种不同的时空注意力配置（图3），这也是论文的核心贡献之一：

![图3：四种时空注意力变体的示意图](figures/timesformer/fig3.png)

**（a）联合时空注意力（Joint ST）**：把所有T×N个patch放在一起做自注意力。这相当于一个3D Transformer，复杂度$O(T^2 N^2)$，计算量巨大。

**（b）分离时空注意力（Divided ST，即TimeSformer默认）**：先做空间注意力（复杂度$O(T N^2)$），再做时间注意力（复杂度$O(N T^2)$）。总复杂度$O(T N^2 + N T^2)$，比联合注意力小很多。

**（c）稀疏全局注意力（Sparse Local）**：只在空间上做注意力，但每个patch只看它周围的局部区域（类似Swin Transformer的窗口注意力）。时间上不做注意力，只通过帧间位置编码传递信息。

**（d）轴注意力（Axial ST）**：沿空间高度、空间宽度、时间三个轴分别做注意力。这是最细粒度的分离方式。

### 4.4 为什么分离注意力有效

这里有个关键洞察：**时空视频数据有一个天然的低秩结构**。一个patch通常只在空间上附近几个patch有强相关性，在时间上也只和前后几帧的同一位置patch有强相关性。把联合注意力拆成空间+时间，相当于利用了这种低秩先验，大幅降低了计算复杂度，同时没有丢失太多信息。

有意思的是，论文实验显示（表3），分离注意力（Divided ST）在准确率上几乎不输联合注意力（Joint ST），但计算量只有后者的几分之一。

## 五、核心创新点

### 创新点1：分离的时空注意力（Divided Space-Time Attention）

**（a）痛点与动机**

3D CNN的痛点前面说过了。即使是Transformer路线，如果直接把ViT扩展到视频，做"联合时空注意力"，复杂度是$O(T^2 N^2)$。对于8帧视频、每帧196个patch，联合注意力的复杂度是$O(8^2 \times 196^2) = O(64 \times 38416) = O(2.46M)$。这还只是单个head的计算量，实际模型有12个head、12层，计算量天文数字。

**（b）方案细节**

分离注意力把联合注意力拆成两步：
1. **空间注意力**：对每帧内部的N个patch做自注意力
   $$z_{s}^{(l)} = \text{LN}(y^{(l-1)} + \text{MSA}_{\text{space}}(y^{(l-1)}))$$
   其中MSA是空间多头自注意力，LN是LayerNorm
   
2. **时间注意力**：对帧间同一位置的patch做自注意力
   $$z_{t}^{(l)} = \text{LN}(z_{s}^{(l)} + \text{MSA}_{\text{time}}(z_{s}^{(l)}))$$
   其中MSA是时间多头自注意力

3. **前馈网络**：
   $$y^{(l)} = \text{LN}(z_{t}^{(l)} + \text{MLP}(z_{t}^{(l)}))$$

**（c）为什么有效**

复杂度分析：联合注意力 $O(T^2 N^2)$ vs 分离注意力 $O(T N^2 + N T^2)$

代入实际值（T=8, N=196）：
- 联合注意力：$8^2 \times 196^2 = 64 \times 38416 = 2,458,624$
- 分离注意力：$8 \times 196^2 + 196 \times 8^2 = 8 \times 38416 + 196 \times 64 = 307,328 + 12,544 = 319,872$

分离注意力只有联合注意力的13%。而且实验证明（表3），这种分离几乎不损失准确率（分离注意力83.0% vs 联合注意力83.1%）。

**（d）与Related Work的关系**

这个设计直接继承自ViT [Dosovitskiy et al., 2020] 的图像分类架构。ViT把图像切patches做自注意力，TimeSformer把它扩展到视频。同时，分离注意力的思路和3D卷积分解（factorized 3D convolution）有异曲同工之妙——3D卷积可以分解成2D空间卷积+1D时间卷积 [Sun et al., 2015; Tran et al., 2018]，TimeSformer也做了类似的分解，只不过从卷积换成了注意力。

**（e）如果去掉会怎样**

表3的消融实验显示：
- 去掉时间注意力（只用空间注意力 + 位置编码）：准确率从83.0%降到80.6%，降幅2.4%
- 把空间注意力换成MLP（只用时间注意力）：准确率降到78.5%，降幅4.5%
- 两者都去掉（只用位置编码 + MLP）：准确率降到76.8%，降幅6.2%

这说明空间注意力和时间注意力都很重要，且空间注意力贡献更大（因为视频里空间信息比时间信息更丰富）。

### 创新点2：四种注意力变体的系统对比

**（a）痛点与动机**

之前的工作要么用3D CNN（固定感受野），要么用3D Transformer（计算量太大）。没有人系统地比较过不同的时空注意力配置。

**（b）方案细节**

论文在相同的实验设置下（8帧、16×16 patches、ViT-Base backbone）比较了四种变体（图3）：
- Joint ST：联合时空注意力
- Divided ST：分离时空注意力（默认）
- Sparse Local：稀疏局部注意力
- Axial ST：轴注意力

**（c）为什么有效**

表3的结果：
| 变体 | Top-1准确率 | GFLOPs |
|------|-------------|--------|
| Joint ST | 83.1% | 7080 |
| Divided ST | 83.0% | 590 |
| Sparse Local | 82.2% | 510 |
| Axial ST | 82.6% | 670 |

Divided ST以590 GFLOPs达到了83.0%，而Joint ST需要7080 GFLOPs（12倍计算量）才高0.1%。这说明分离注意力在计算效率上碾压联合注意力。

**（d）与Related Work的关系**

这个系统性对比填补了空白。之前的视频Transformer工作要么只用了联合注意力 [Arnab et al., 2021]，要么只用了分离注意力 [Fan et al., 2021]，但没有在同一框架下公平比较。

**（e）如果去掉会怎样**

如果论文只提出Divided ST而不做这种系统比较，读者会怀疑"是不是有其他更好的配置"。这个比较确立了Divided ST作为最优折中的地位。

### 创新点3：预训练策略

**（a）痛点与动机**

视频数据集（Kinetics-400 有240K视频，Something-Something v2有220K视频）比图像数据集（ImageNet有1.28M图像）小得多。直接从头训练视频Transformer很容易过拟合。

**（b）方案细节**

论文提出在ImageNet-21K上预训练空间注意力部分（即ViT的权重），然后冻结空间注意力参数，只微调时间注意力参数。具体来说：
1. 在ImageNet-21K上预训练ViT-Base（12层，768维，12 heads）
2. 把预训练权重加载到TimeSformer的空间注意力部分
3. 在视频数据上训练时，冻结空间注意力，只更新时间注意力和分类头

**（c）为什么有效**

这个策略利用了图像和视频共享空间特征的先验——物体识别在图像和视频里是一样的。时间注意力需要学习运动模式，这部分没有预训练，但参数量小（只有空间注意力的1/12左右），不容易过拟合。

表2显示：
- 从头训练：79.0%
- ImageNet-21K预训练+微调全部参数：81.4%
- ImageNet-21K预训练+冻结空间注意力：83.0%

有意思的是，冻结空间注意力反而比微调全部参数好。论文推测这是因为微调全部参数会导致灾难性遗忘——空间注意力忘记了在ImageNet上学到的物体识别知识。

**（d）与Related Work的关系**

这个策略借鉴了NLP的预训练-微调范式 [Devlin et al., 2019; Brown et al., 2020]，但做了适配：不是全部微调，而是部分冻结。这和3D CNN领域的"用ImageNet预训练2D部分" [Carreira & Zisserman, 2017] 思路一致。

**（e）如果去掉会怎样**

表2的消融实验：去掉预训练，准确率从83.0%降到79.0%，降幅4.0%。这个差距在视频理解任务里很大。说明预训练对视频Transformer至关重要。

## 六、公式详解

### 6.1 标准自注意力公式

**公式1：自注意力计算**

$$\text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

**（a）符号定义**
- $Q \in \mathbb{R}^{N \times d_k}$：查询矩阵，由输入线性投影得到
- $K \in \mathbb{R}^{N \times d_k}$：键矩阵，由输入线性投影得到
- $V \in \mathbb{R}^{N \times d_v}$：值矩阵，由输入线性投影得到
- $d_k$：查询和键的维度（通常等于$d_v$）
- $N$：token数量（在空间注意力中$N=H/P \times W/P$，在时间注意力中$N=T$）

**（b）公式来源**

这是标准的Transformer自注意力公式 [Vaswani et al., 2017]。TimeSformer直接沿用。

**（c）推导过程**

1. 输入序列 $X \in \mathbb{R}^{N \times d_{model}}$
2. 三个线性投影：$Q = XW_Q$, $K = XW_K$, $V = XW_V$
3. 计算注意力分数：$S = QK^T / \sqrt{d_k}$，形状为$N \times N$
4. Softmax归一化：$A = \text{Softmax}(S)$，每行和为1
5. 加权聚合：$O = AV$，形状为$N \times d_v$

**（d）直觉理解**

注意力机制的本质是**加权平均**。每个token通过$Q$询问其他token："你和我有多相关？"其他token通过$K$回答。答案是注意力分数。然后根据分数加权平均所有token的$V$。

除以$\sqrt{d_k}$是为了防止内积过大导致softmax梯度消失。当$d_k$较大时，$QK^T$的元素方差约为$d_k$，除以$\sqrt{d_k}$后方差变为1。

**（e）边界情况**

- 当$N=1$时，注意力退化为恒等映射（因为只有一个token和自己做注意力）
- 当$d_k$很小时（比如1），除以$\sqrt{d_k}$的效果减弱，softmax可能进入饱和区

### 6.2 多头自注意力（MSA）

**公式2：多头自注意力**

$$\text{MSA}(X) = \text{Concat}(\text{head}_1, ..., \text{head}_h)W_O$$

其中 $\text{head}_i = \text{Attention}(XW_Q^i, XW_K^i, XW_V^i)$

**（a）符号定义**
- $h$：head数量（TimeSformer默认$h=12$）
- $W_Q^i \in \mathbb{R}^{d_{model} \times d_k}$：第$i$个head的查询投影
- $W_O \in \mathbb{R}^{h d_v \times d_{model}}$：输出投影

**（b）公式来源**

标准的Transformer多头注意力 [Vaswani et al., 2017]。

**（c）推导过程**

1. 将$d_{model}$维的输入投影到$h$个不同的子空间
2. 每个子空间独立做自注意力
3. 拼接所有head的输出，再投影回$d_{model}$维

**（d）直觉理解**

多头注意力让模型在不同的表示子空间里同时学习不同的注意力模式。比如一个head可能学习"物体之间的空间关系"，另一个head学习"颜色相似性"，第三个head学习"运动一致性"。

**（e）边界情况**

- 当$h=1$时，退化为单头注意力
- 当$d_k$太小（比如8，对应$d_{model}=96, h=12$），每个head的表示能力受限

### 6.3 空间注意力复杂度

**公式3：空间注意力的计算复杂度**

$$C_{\text{space}} = O(T \cdot N^2 \cdot d)$$

**（a）符号定义**
- $T$：帧数（TimeSformer默认$T=8$）
- $N$：每帧的patch数（输入224×224，patch size 16，则$N=14 \times 14 = 196$）
- $d$：隐藏维度（TimeSformer默认$d=768$）

**（b）公式来源**

自注意力的计算复杂度是$O(N^2 d)$，其中$N$是token数。空间注意力对每帧独立做，所以乘以$T$。

**（c）推导过程**

1. QKV投影：$3N d^2$（每个token从$d$维投影到$d$维，三个投影矩阵）
2. 注意力分数计算：$N^2 d$（$QK^T$矩阵乘法）
3. 加权聚合：$N^2 d$（$AV$矩阵乘法）
4. 输出投影：$N d^2$
5. 总复杂度：$3N d^2 + 2N^2 d + N d^2 = O(N^2 d + N d^2)$

对于空间注意力，每帧独立做，所以总复杂度是$T \cdot O(N^2 d + N d^2)$。

**（d）直觉理解**

第一项$N^2 d$来自注意力分数计算和聚合，这是$O(N^2)$的，当$N$很大时占主导。第二项$N d^2$来自投影，当$d$很大时占主导。

**（e）边界情况**

代入实际值（$T=8, N=196, d=768$）：
- $T N^2 d = 8 \times 196^2 \times 768 = 8 \times 38416 \times 768 = 236, 073, 984$
- $T N d^2 = 8 \times 196 \times 768^2 = 8 \times 196 \times 589, 824 = 924, 844, 032$

可见$N d^2$项更大，说明投影操作比注意力计算更耗时（当$d > N$时）。

### 6.4 时间注意力复杂度

**公式4：时间注意力的计算复杂度**

$$C_{\text{time}} = O(N \cdot T^2 \cdot d)$$

**（a）符号定义**
- $T$：帧数
- $N$：每帧的patch数
- $d$：隐藏维度

**（b）公式来源**

时间注意力对每个空间位置独立做。每个空间位置有$T$个帧间patch，所以复杂度是$O(T^2 d)$，共有$N$个位置。

**（c）推导过程**

1. 重排数据：从形状$T \times N \times d$重排为$N \times T \times d$
2. 对每个$N$位置独立做自注意力：$N \cdot O(T^2 d)$
3. 重排回$T \times N \times d$

**（d）直觉理解**

时间注意力关注的是"同一个位置在不同帧之间的变化"。比如一个物体从第1帧到第8帧的运动轨迹。因为每个位置独立处理，所以复杂度是$N$个独立的$T^2$注意力。

**（e）边界情况**

代入实际值（$T=8, N=196, d=768$）：
- $N T^2 d = 196 \times 8^2 \times 768 = 196 \times 64 \times 768 = 9, 634, 176$

对比空间注意力的$T N^2 d = 236M$，时间注意力只有9.6M，小了一个数量级。这说明时间注意力计算量远小于空间注意力。

### 6.5 总复杂度

**公式5：TimeSformer的总计算复杂度**

$$C_{\text{total}} = C_{\text{space}} + C_{\text{time}} = T N^2 d + N T^2 d = TNd(N + T)$$

**（a）符号定义**
- $T, N, d$同上

**（b）公式来源**

分离注意力的总复杂度是空间注意力和时间注意力的和。

**（c）推导过程**

$$C_{\text{total}} = T N^2 d + N T^2 d = TNd(N + T)$$

**（d）直觉理解**

当$N \gg T$时（视频帧数少但每帧分辨率高），总复杂度约等于$T N^2 d$，空间注意力占主导。当$T \gg N$时（帧数多但分辨率低），总复杂度约等于$N T^2 d$，时间注意力占主导。

对于TimeSformer的默认设置（$T=8, N=196$），$N \gg T$，所以空间注意力占主导。

**（e）边界情况**

对比联合注意力复杂度：$C_{\text{joint}} = O(T^2 N^2 d)$

分离注意力 vs 联合注意力：
$$\frac{C_{\text{divided}}}{C_{\text{joint}}} = \frac{T N^2 + N T^2}{T^2 N^2} = \frac{1}{T} + \frac{1}{N}$$

代入$T=8, N=196$：
$$\frac{1}{8} + \frac{1}{196} = 0.125 + 0.005 = 0.13$$

分离注意力只有联合注意力的13%。

### 8.1 模型定义

```python
# 核心模块：TimeSformerBlock
class TimeSformerBlock(nn.Module):
    def __init__(self, dim, num_heads, mlp_ratio=4.):
        super().__init__()
        self.norm1 = nn.LayerNorm(dim)
        self.attn_space = Attention(dim, num_heads)  # 空间注意力
        self.norm2 = nn.LayerNorm(dim)
        self.attn_time = Attention(dim, num_heads)   # 时间注意力
        self.norm3 = nn.LayerNorm(dim)
        self.mlp = Mlp(dim, int(dim * mlp_ratio))
    
    def forward(self, x):
        # x shape: (B, T, N, D)
        B, T, N, D = x.shape
        
        # 空间注意力：对每帧独立做
        x = x + self.attn_space(self.norm1(x).reshape(B*T, N, D)).reshape(B, T, N, D)
        
        # 时间注意力：对每个位置独立做
        x = x + self.attn_time(self.norm2(x).permute(0, 2, 1, 3).reshape(B*N, T, D)).reshape(B, N, T, D).permute(0, 2, 1, 3)
        
        # MLP
        x = x + self.mlp(self.norm3(x))
        return x
```

**代码中隐藏的细节：**
1. 代码里把空间注意力输出形状从(B*T, N, D) reshape回(B, T, N, D)，然后permute成(B, N, T, D)做时间注意力。这个permute操作是计算密集型的，但在GPU上可以通过内存重排高效实现。
2. 时间注意力输入形状是(B*N, T, D)，即把每个空间位置独立出来，在T帧上做注意力。这对应了论文中"对每个空间位置独立做时间注意力"的描述。
3. 代码里没有显式的"CLS token"分类头，而是用全局平均池化或取第一个patch作为分类特征。

## 九、位置

### 前驱工作

1. **ViT [Dosovitskiy et al., 2020]**：把Transformer用在图像分类上，证明了纯注意力机制可以取代CNN。TimeSformer直接继承ViT的patch嵌入、位置编码、分类头等设计。

2. **Non-local Neural Networks [Wang et al., 2018]**：首次在视频理解中使用自注意力机制，但只作为CNN的补充模块。TimeSformer把注意力作为主要计算单元。

3. **3D卷积分解 [Sun et al., 2015; Tran et al., 2018]**：把3D卷积分解成2D空间卷积+1D时间卷积。TimeSformer的分离注意力是这种分解思想在注意力机制中的对应。

### 同期竞品

1. **ViViT [Arnab et al., 2021]**：另一个视频Transformer，使用联合时空注意力。TimeSformer的分离注意力在计算效率上明显优于ViViT。

2. **Video Swin Transformer [Liu et al., 2021]**：把Swin Transformer的窗口注意力扩展到视频。TimeSformer在全局注意力方面有优势。

3. **MViT [Fan et al., 2021]**：多尺度视频Transformer。TimeSformer更简单，但MViT在更大规模上表现更好。

### 后续影响

1. **TimeSformer的分离注意力设计被广泛采用**：后续的视频Transformer工作（如VideoMAE、InternVideo）都采用了类似的分离注意力设计。

2. **预训练策略的推广**：TimeSformer的"在ImageNet预训练空间部分，冻结空间注意力微调时间部分"策略被后续工作（如VideoMAE）改进和推广。

3. **视频理解领域的范式转移**：TimeSformer和同期工作一起推动了视频理解从3D CNN向Transformer的转变。2022年之后，视频Transformer成为主流。

4. **多模态扩展**：TimeSformer的设计被扩展到视频-语言任务（如CLIP4Clip、VideoCLIP）。

5. **高效视频理解**：TimeSformer的分离注意力启发了后续的高效视频Transformer（如EfficientVideoTransformer、TimeSformer的量化版本）。

## 十、局限

### 局限1：计算量仍高于3D CNN

**问题**：TimeSformer（590 GFLOPs）比SlowFast（213 GFLOPs）计算量大2.8倍。虽然比联合注意力（7080 GFLOPs）好很多，但在资源受限的设备上仍然不实用。

**何时重要**：在移动设备、边缘计算等场景下，590 GFLOPs是不可接受的。

**论文是否承认**：论文在结论部分提到"计算效率仍有提升空间"。

**后续工作**：Video Swin Transformer通过窗口注意力进一步降低了计算量。

### 局限2：帧数扩展性有限

**问题**：增加帧数时，时间注意力的复杂度是$O(T^2)$。从8帧到32帧，计算量增长16倍（时间注意力部分）。

**何时重要**：需要处理长视频（如60帧以上）时，时间注意力成为瓶颈。

**论文是否承认**：论文在消融实验中展示了帧数增加带来的计算量增长，但没有提出解决方案。

**后续工作**：VideoMAE通过掩码策略减少了有效帧数，MViT通过多尺度设计缓解了这个问题。

### 局限3：对长距离时间依赖的建模能力有限

**问题**：时间注意力只关注同一空间位置在不同帧之间的变化，无法捕捉"物体移动到不同位置"这种跨空间的时间依赖。

**何时重要**：在需要跟踪物体运动的任务（如动作检测、视频分割）中，这种限制很明显。

**论文是否承认**：论文没有直接讨论这个问题，但实验中Something-Something v2的结果（66.5%）相比Kinetics-400（83.0%）明显偏低，暗示了这个问题。

**后续工作**：Video Swin Transformer的移位窗口注意力可以捕捉跨空间的时间依赖。

### 局限4：预训练策略依赖ImageNet

**问题**：预训练需要ImageNet-21K（1400万图像），这不是所有研究者都能获取的。而且ImageNet的分布和视频数据不同（ImageNet是单帧、静态物体，视频包含运动）。

**何时重要**：在医疗视频、工业检测等特定领域，ImageNet预训练可能不是最优的。

**论文是否承认**：论文提到"ImageNet预训练是有效的"，但没有讨论领域适应问题。

**后续工作**：VideoMAE提出了视频自监督预训练，不需要外部数据。

## 十一、小结

**论文解决了什么问题**：3D CNN在视频理解中计算效率低、时空特征耦合、感受野有限。TimeSformer提出用分离的时空自注意力取代3D卷积，在保持高准确率的同时大幅降低计算复杂度。

**核心贡献**：
1. 提出了分离时空注意力（Divided Space-Time Attention），把联合注意力拆成空间注意力和时间注意力，计算量降至13%
2. 系统比较了四种时空注意力配置，证明分离注意力是最优折中
3. 提出了有效的预训练策略（ImageNet预训练+冻结空间注意力）

**更广泛的影响**：TimeSformer是视频Transformer领域的里程碑工作之一。它证明了纯注意力机制可以在视频理解中超越3D CNN，推动了视频理解从CNN到Transformer的范式转移。后续的视频Transformer工作（VideoMAE、InternVideo、Video Swin等）都直接或间接受到TimeSformer的启发。
