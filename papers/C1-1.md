# Faster R-CNN: Towards Real-Time Object Detection with Region Proposal Networks（2015）

**论文：** [arXiv](https://arxiv.org/abs/1506.01497) · Shaoqing Ren, Kaiming He, Ross Girshick, Jian Sun · 2015

---

## 一、先搞清楚坑在哪

目标检测在 2015 年之前一直面临一个尴尬的局面：**准确率高的方法跑得慢，跑得快的方法准确率低。**

具体来说，当时的检测范式是这样的：

1. **Selective Search + R-CNN**：先用 Selective Search 生成约 2000 个候选区域（region proposals），然后对每个候选区域分别送入 CNN 提取特征、分类、回归边框。准确率高（mAP 66% 在 PASCAL VOC 2007），但每张图片要跑 2000 次 CNN，速度只有 14 秒/张。

2. **SPPnet / Fast R-CNN**：把整张图一次送入 CNN 提取共享特征图（feature map），然后在特征图上通过 ROI Pooling 提取每个候选区域的特征。速度提升到 2-3 秒/张，但候选区域生成仍然使用 Selective Search，这个步骤本身就要 2 秒。

3. **OverFeat / YOLO（同期）**：尝试端到端的回归方法，直接预测边框坐标。速度快（YOLO 达到 45 FPS），但准确率明显低于基于候选区域的方法。

**核心矛盾**：候选区域生成（region proposal）是检测的瓶颈。Selective Search 和 EdgeBoxes 这些基于低级视觉线索的方法，虽然不需要训练，但：
- 计算速度慢（CPU 上跑 2 秒/张）
- 生成的候选区域质量有限（召回率不够高）
- 无法利用 GPU 加速（都是 CPU 算法）

**有意思的是**：CNN 在分类任务中已经证明了能学习到高质量的视觉特征，那为什么不用 CNN 本身来生成候选区域呢？

## 二、[Selective Search / EdgeBoxes] 的真正问题

### 2.1 计算效率问题

Selective Search 基于超像素合并（superpixel merging）算法，在 CPU 上处理一张 1000×600 的图片需要约 2 秒。这个时间虽然比 R-CNN 的 14 秒少，但对于追求实时的应用来说完全不可接受。

更糟糕的是，**这个计算开销是固定的**——不管图片内容简单还是复杂，Selective Search 都要跑完整个合并过程。

### 2.2 与检测网络脱节

候选区域生成和检测是**两个独立的阶段**：

```
图片 → Selective Search（CPU）→ 候选区域 → CNN 特征提取（GPU）→ 分类 + 回归（GPU）
```

这里有个严重的问题：Selective Search 完全不知道后面的 CNN 在做什么。它基于颜色、纹理、大小、形状等低级线索生成候选区域，但这些线索可能和 CNN 学到的语义特征完全不相关。

**举个例子**：如果一张图片里有一个白色的狗在雪地里，Selective Search 可能因为颜色相似而把狗和雪地合并成一个区域，但 CNN 知道狗和雪地是不同物体。

### 2.3 无法端到端优化

因为候选区域生成是不可微的（基于规则的算法），整个检测流程无法端到端训练。这意味着：
- 候选区域生成的质量无法通过检测损失来改进
- 检测网络的梯度无法回流到候选区域生成阶段
- 两个阶段的优化目标是割裂的

## 三、Faster R-CNN 的核心思路

**一句话概括**：让 CNN 在特征图上直接生成候选区域，把候选区域生成和检测融合到同一个网络中，实现端到端训练和实时检测。

更具体地说，Faster R-CNN 提出了 **Region Proposal Network（RPN）**，它是一个轻量级的全卷积网络，在共享特征图上滑动一个小窗口，在每个位置预测多个候选区域（用不同尺度和长宽比的 anchor boxes 表示）。

整个检测流程变成：

```
图片 → CNN 特征提取（共享）→ RPN 生成候选区域 → ROI Pooling → 分类 + 回归
                                    ↑
                              全部在 GPU 上完成
```

关键洞察：**RPN 和检测网络共享卷积层**，因此 RPN 的计算开销几乎为零——它只是在已有的特征图上多加了几层卷积。

## 四、网络架构详解

### 4.1 整体架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                          输入图片 (H × W × 3)                        │
└─────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────┐
│                   共享卷积层 (VGG-16 / ZF-Net)                       │
│               输出特征图 (H/16 × W/16 × 512)                        │
└─────────────────────────────────────────────────────────────────────┘
                                    │
                    ┌───────────────┴───────────────┐
                    ▼                               ▼
┌───────────────────────────────┐   ┌───────────────────────────────┐
│       Region Proposal Network │   │     ROI Pooling + 检测网络      │
│                               │   │                               │
│   3×3 卷积 (512 → 512)        │   │    ROI Pooling (7×7)           │
│           │                   │   │           │                   │
│     ┌─────┴─────┐             │   │    全连接层 (4096)             │
│     ▼           ▼             │   │           │                   │
│  分类分支     回归分支         │   │    全连接层 (4096)             │
│  (1×1 卷积)   (1×1 卷积)      │   │           │                   │
│  2k 个输出    4k 个输出        │   │     ┌─────┴─────┐             │
│                               │   │     ▼           ▼             │
│  候选区域分数  候选区域坐标     │   │  分类 (C+1)   回归 (4C)       │
└───────────────────────────────┘   └───────────────────────────────┘
```

### 4.2 前向传播过程（逐层分析）

**输入**：任意大小的 RGB 图片，通常缩放到短边 600 像素（PASCAL VOC），长边不超过 1000 像素。

**Step 1：共享卷积层**

使用 VGG-16（或 ZF-Net）的前 13 个卷积层（5 个 stage），通过 5 次 2×2 最大池化，将输入缩小 32 倍。

```
输入: 600 × 800 × 3
↓
Conv1_1 → Conv1_2 → Pool (2×)
输出: 300 × 400 × 64
↓
Conv2_1 → Conv2_2 → Pool (2×)
输出: 150 × 200 × 128
↓
Conv3_1 → Conv3_2 → Conv3_3 → Pool (2×)
输出: 75 × 100 × 256
↓
Conv4_1 → Conv4_2 → Conv4_3 → Pool (2×)
输出: 38 × 50 × 512
↓
Conv5_1 → Conv5_2 → Conv5_3
输出: 38 × 50 × 512
```

实际处理时，VGG-16 的 Conv5 输出 stride 为 16（因为从 Conv5 开始不再池化），所以特征图大小是 H/16 × W/16。

**Step 2：RPN 生成候选区域**

RPN 在 Conv5 特征图上滑动一个 3×3 的窗口，每个窗口位置预测 k 个 anchor boxes（默认 k=9，3 种尺度 × 3 种长宽比）。

```
输入特征图: 38 × 50 × 512
↓
3×3 卷积 (pad=1, stride=1): 38 × 50 × 512
↓
分类分支: 1×1 卷积 → 38 × 50 × 2k (每个 anchor 的 object/non-object 分数)
回归分支: 1×1 卷积 → 38 × 50 × 4k (每个 anchor 的 4 个坐标偏移)
```

每个位置有 k 个 anchor，所以：
- 分类分支输出通道: 2k = 18
- 回归分支输出通道: 4k = 36

总共生成的 anchor 数量：38 × 50 × 9 = 17,100 个

**Step 3：候选区域筛选**

RPN 生成的所有 anchor 经过以下筛选步骤：
1. 去掉超出图片边界的 anchor
2. 按 objectness 分数排序，取前 N 个（训练时 2000，测试时 300）
3. 对选出的 anchor 应用 NMS（IoU 阈值 0.7）
4. NMS 后取前 M 个（训练时 2000，测试时 300）

**Step 4：ROI Pooling**

将 RPN 生成的候选区域映射到 Conv5 特征图上，通过 ROI Pooling 提取固定大小（7×7）的特征。

```
候选区域坐标 (在原图上的坐标) → 映射到特征图 (除以 stride 16)
→ ROI Pooling (7×7 网格，每个网格内做最大池化)
→ 输出: 每个候选区域 7 × 7 × 512 的特征向量
```

**Step 5：检测网络**

对每个候选区域的特征向量，经过两个全连接层（4096 维），最后输出分类和回归结果。

```
输入: 7 × 7 × 512 → 展平 → 25088 维
↓
FC1: 25088 → 4096 (ReLU + Dropout)
↓
FC2: 4096 → 4096 (ReLU + Dropout)
↓
分类分支: 4096 → (C+1) 类分数 (C 个物体类 + 1 个背景类)
回归分支: 4096 → 4C 个坐标偏移 (每个物体类 4 个坐标)
```

### 4.3 为什么这样设计

**共享卷积层**：这是 Faster R-CNN 的关键创新。RPN 和检测网络共用同一个特征提取器，避免了重复计算。一张图片只需要一次前向传播，就能同时得到候选区域和检测结果。

**Anchor 机制**：RPN 在每个位置预测多个尺度和长宽比的候选区域，而不是像 Selective Search 那样生成任意形状的区域。这有几个好处：
1. **计算可预测**：每个位置的输出通道数是固定的（2k + 4k）
2. **多尺度覆盖**：通过不同尺度的 anchor，RPN 可以检测大小差异很大的物体
3. **训练稳定**：anchor 提供了明确的监督信号——每个 anchor 要么是正样本（与 ground truth 的 IoU > 0.7），要么是负样本（IoU < 0.3）

**3×3 滑动窗口**：这个窗口大小在特征图上对应原图的 48×48 像素区域（16×3=48），加上 anchor 的尺度变化，可以覆盖从几十到几百像素的物体。

## 五、核心创新点

### 创新点 1：Region Proposal Network (RPN)

#### （a）痛点与动机

**问题**：候选区域生成是检测的瓶颈。Selective Search 每张图 2 秒，且无法用 GPU 加速。

**为什么现有方法失败**：
- Selective Search [Uijlings et al., 2013]：基于超像素合并，计算复杂度 O(N log N)，N 是超像素数量
- EdgeBoxes [Zitnick & Dollár, 2014]：基于边缘密度，虽然快一些（0.25 秒/张），但召回率不如 Selective Search
- 两者都是 CPU 算法，与后续的 CNN 特征提取完全割裂

**动机**：既然 CNN 已经在分类任务中证明了能学习到高质量的视觉特征，那应该也能学习到哪些区域可能包含物体。如果让 CNN 同时负责特征提取和候选区域生成，就能实现端到端优化。

#### （b）方案细节

RPN 是一个全卷积网络（FCN），在共享特征图上工作：

```
输入特征图 (H/16 × W/16 × 512)
    │
    ▼
3×3 卷积 (512 → 512, padding=1, stride=1)
    │
    ├──→ 1×1 卷积 (512 → 2k) → reshape → softmax → 每个 anchor 的 objectness 分数
    │
    └──→ 1×1 卷积 (512 → 4k) → 每个 anchor 的坐标偏移 (tx, ty, tw, th)
```

每个 anchor 的坐标回归目标：
```
tx = (x - xa) / wa,   ty = (y - ya) / ha
tw = log(w / wa),     th = log(h / ha)
```

其中 (x, y, w, h) 是预测框的中心坐标和宽高，(xa, ya, wa, ha) 是 anchor 框。

#### （c）为什么有效

**直觉**：RPN 的 3×3 卷积在特征图上滑动，每个位置看到的感受野（在原图上约 48×48 像素）足以判断该区域是否包含物体。而且由于共享卷积层已经学习了丰富的语义特征，RPN 的分类分支只需要在特征图的每个位置判断"有没有物体"，比直接检测物体类别简单得多。

**实验证据**：
- 论文 Table 2 显示，RPN 在 PASCAL VOC 2007 上达到 69.9% mAP，比 Selective Search（66.9%）高 3 个百分点
- RPN 的召回率在 300 个 proposals 时达到 90% 以上（IoU=0.5），而 Selective Search 需要 2000 个 proposals 才能达到类似水平

#### （d）与 Related Work 的关系

**前驱工作**：
- R-CNN [Girshick et al., 2014]：首次将 CNN 用于目标检测，但使用 Selective Search 生成候选区域
- OverFeat [Sermanet et al., 2014]：尝试用 CNN 直接回归边框，但缺乏候选区域生成机制
- MultiBox [Erhan et al., 2014]：用 CNN 预测候选区域，但需要单独训练一个网络，不与检测共享

**与 MultiBox 的区别**：MultiBox 使用全连接层输出候选区域坐标，输入是整张图片的特征，无法在特征图的每个位置独立预测。RPN 使用全卷积结构，在特征图的每个位置独立预测，计算效率更高，且能与检测网络共享卷积层。

#### （e）如果去掉会怎样

如果去掉 RPN，回到 Selective Search 或 EdgeBoxes：
- 速度下降：每张图增加 2 秒（Selective Search）或 0.25 秒（EdgeBoxes）
- 准确率下降：论文 Table 2 显示，用 Selective Search + Fast R-CNN 是 66.9%，而 RPN + Fast R-CNN 是 69.9%，提升了 3 个百分点
- 无法端到端训练：候选区域生成和检测的梯度无法互相传播

### 创新点 2：Anchor 机制

#### （a）痛点与动机

**问题**：如何让 CNN 预测不同尺度和长宽比的候选区域？传统方法（图像金字塔、特征金字塔）计算量大。

**为什么现有方法失败**：
- 图像金字塔（多尺度输入）：每张图片需要缩放多次，计算量线性增加
- 特征金字塔（多尺度特征）：需要设计复杂的特征融合结构

**动机**：在特征图每个位置预定义一组不同尺度和长宽比的"参考框"（anchors），让网络学习的是"相对于参考框的偏移"，而不是直接预测绝对坐标。

#### （b）方案细节

每个滑动窗口位置定义 k 个 anchors，默认 k=9：
- 3 种尺度：128², 256², 512²（在原图上的像素面积）
- 3 种长宽比：1:1, 1:2, 2:1

对于特征图上的一个点 (i, j)，对应在原图上的中心点为：
```
x_center = (j + 0.5) × stride
y_center = (i + 0.5) × stride
```
其中 stride = 16（VGG-16 的 Conv5 下采样倍数）。

然后以这个中心点生成 9 个 anchors，每个 anchor 的面积和长宽比由上述参数决定。

#### （c）为什么有效

**直觉**：Anchor 机制把候选区域预测问题从"直接回归任意坐标"简化为"在有限候选集中选择并微调"。因为：
1. **先验知识**：物体通常有固定的尺度和长宽比（人一般是 1:2 或 2:1，车一般是 2:1 或 3:2）
2. **回归更容易**：预测相对偏移比预测绝对坐标更容易收敛
3. **多尺度覆盖**：9 个 anchor 的组合足以覆盖常见物体的尺寸

**实验证据**：
- 论文 Table 3 显示，使用 3 种尺度和 3 种长宽比（9 个 anchors）效果最好，增加更多尺度或长宽比收益递减
- 单尺度（1 anchor）的 mAP 为 62.8%，9 个 anchors 提升到 69.9%，说明多尺度多长宽比确实有帮助

#### （d）与 Related Work 的关系

**前驱工作**：
- DPM [Felzenszwalb et al., 2010]：使用不同长宽比的"部件"（parts）来检测物体，类似 anchor 的思想
- MultiBox [Erhan et al., 2014]：使用 k-means 聚类预定义候选框，但只用了 800 个全局候选框，没有在特征图每个位置设置

**与 MultiBox 的区别**：MultiBox 的候选框是全局的（整张图共享），而 RPN 的 anchors 是局部的（每个位置独立）。这使得 RPN 可以处理不同位置的不同尺度物体。

#### （e）如果去掉会怎样

如果去掉 anchor 机制，直接在特征图每个位置回归任意坐标：
- 训练不稳定：没有先验约束，网络可能预测出各种奇怪形状的框
- 收敛困难：需要更多训练数据和更复杂的损失函数
- 准确率下降：论文 Table 3 显示，1 个 anchor（无多尺度）的 mAP 下降 7 个百分点

### 创新点 3：共享卷积层的 4-Step 交替训练

#### （a）痛点与动机

**问题**：RPN 和 Fast R-CNN 检测网络共享卷积层，但两者需要不同的训练数据——RPN 需要 anchor 级别的标注（正负样本），Fast R-CNN 需要 proposal 级别的标注（物体类别和边框）。

**为什么现有方法失败**：
- 简单联合训练：RPN 和 Fast R-CNN 的损失函数不同，训练目标有冲突
- 分阶段训练：先训练 RPN，再训练 Fast R-CNN，但共享层的参数会变化

**动机**：设计一种交替训练策略，让 RPN 和 Fast R-CNN 交替优化，同时保持共享层的参数一致性。

#### （b）方案细节

4 步训练过程：

**Step 1**：训练 RPN（使用 ImageNet 预训练的 VGG-16 初始化）
- 冻结共享卷积层的前 2 个 stage
- 训练 RPN 的 3×3 卷积和 1×1 卷积层
- 损失函数：分类损失（二分类：object/non-object）+ 回归损失（Smooth L1）

**Step 2**：训练 Fast R-CNN 检测网络（使用 Step 1 的 RPN 生成 proposals）
- 使用 ImageNet 预训练的 VGG-16 初始化
- 共享卷积层用 Step 1 的 RPN 结果初始化
- 训练检测网络的 ROI Pooling 和全连接层
- 此时两个网络的卷积层没有共享（参数不同）

**Step 3**：再次训练 RPN（使用 Step 2 的检测网络初始化共享层）
- 固定共享卷积层
- 只微调 RPN 特有的层（3×3 和 1×1 卷积）
- 这确保了 RPN 和检测网络的共享层参数一致

**Step 4**：再次训练 Fast R-CNN 检测网络（使用 Step 3 的 RPN）
- 固定共享卷积层
- 只微调检测网络特有的层（全连接层）
- 最终得到统一的网络

#### （c）为什么有效

**直觉**：这种交替训练策略解决了两个问题：
1. **参数冲突**：RPN 需要学习"哪些区域有物体"，检测网络需要学习"物体是什么类别"，两者目标不同但共享底层特征
2. **数据依赖**：RPN 需要 anchor 标注，检测网络需要 proposal 标注，两者不能同时生成

**实验证据**：
- 论文 Table 4 显示，4-step 交替训练（69.9%）比分别训练（68.5%）高 1.4 个百分点
- 联合训练（近似端到端）能达到 69.2%，但实现更复杂

#### （d）与 Related Work 的关系

**前驱工作**：
- R-CNN [Girshick et al., 2014]：分阶段训练，每个阶段独立，没有共享层
- Fast R-CNN [Girshick, 2015]：端到端训练检测网络，但候选区域生成仍然独立

**后续影响**：后来的 SSD [Liu et al., 2016] 和 YOLO [Redmon et al., 2016] 实现了真正的端到端联合训练，不再需要交替训练。

#### （e）如果去掉会怎样

如果直接联合训练（近似端到端）：
- 论文 Table 4 显示，联合训练达到 69.2%，比交替训练低 0.7 个百分点
- 训练过程更不稳定，需要更多调参技巧

## 六、公式详解

### 6.1 多任务损失函数

Faster R-CNN 的损失函数是 RPN 损失和 Fast R-CNN 损失的加权和。

#### （1）RPN 损失函数

$$L(\{p_i\}, \{t_i\}) = \frac{1}{N_{cls}} \sum_i L_{cls}(p_i, p_i^*) + \lambda \frac{1}{N_{reg}} \sum_i p_i^* L_{reg}(t_i, t_i^*)$$

**符号定义**：
- $i$：mini-batch 中 anchor 的索引
- $p_i$：anchor i 是物体的预测概率
- $p_i^*$：ground truth 标签（1 表示正样本，0 表示负样本）
- $t_i$：预测的 4 个坐标偏移参数 $(t_x, t_y, t_w, t_h)$
- $t_i^*$：ground truth 的坐标偏移参数
- $L_{cls}$：二分类的交叉熵损失
- $L_{reg}$：Smooth L1 损失
- $N_{cls}$：mini-batch 大小（256）
- $N_{reg}$：anchor 位置数量（约 2400）
- $\lambda$：平衡权重（默认 10，使两项损失大致等权重）

**公式来源**：继承自 Fast R-CNN [Girshick, 2015] 的多任务损失设计，但 RPN 的分类是二分类（object vs. non-object），而 Fast R-CNN 是多分类（C 个物体类 + 背景）。

**直觉理解**：
- 第一项 $\frac{1}{N_{cls}} \sum_i L_{cls}$：分类损失，让 RPN 学会区分"有物体"和"没有物体"
- 第二项 $\frac{1}{N_{reg}} \sum_i p_i^* L_{reg}$：回归损失，只对正样本（$p_i^* = 1$）计算，让 RPN 学会精调边框位置

**具体例子**：
假设一个 mini-batch 有 256 个 anchors，其中 128 个正样本、128 个负样本。

对于正样本 anchor，分类损失：
$$L_{cls}(p_i, 1) = -\log(p_i)$$
如果预测概率 $p_i = 0.9$，损失为 $-\log(0.9) \approx 0.105$

对于负样本 anchor，分类损失：
$$L_{cls}(p_i, 0) = -\log(1 - p_i)$$
如果预测概率 $p_i = 0.1$，损失为 $-\log(0.9) \approx 0.105$

回归损失（Smooth L1）：
$$L_{reg}(t_i, t_i^*) = \sum_{j \in \{x,y,w,h\}} \text{smooth}_{L1}(t_{i,j} - t_{i,j}^*)$$

其中：
$$\text{smooth}_{L1}(x) = \begin{cases} 0.5x^2 & \text{if } |x| < 1 \\ |x| - 0.5 & \text{otherwise} \end{cases}$$

**边界情况**：
- 当 $\lambda = 0$：只优化分类，不优化回归，RPN 只能判断有无物体，不能精调边框
- 当 $\lambda$ 很大：回归损失主导，分类被忽略，RPN 可能过拟合边框位置

#### （2）Fast R-CNN 损失函数

$$L(p, u, t^u, v) = L_{cls}(p, u) + \lambda [u \geq 1] L_{reg}(t^u, v)$$

**符号定义**：
- $p$：预测的类别概率分布 $(p_0, p_1, ..., p_C)$
- $u$：ground truth 类别（0 表示背景）
- $t^u$：预测的边框坐标 $(t_x^u, t_y^u, t_w^u, t_h^u)$
- $v$：ground truth 边框坐标 $(v_x, v_y, v_w, v_h)$
- $[u \geq 1]$：指示函数，当 $u \geq 1$（即非背景）时为 1，否则为 0

**公式来源**：直接继承自 Fast R-CNN [Girshick, 2015]，Faster R-CNN 没有修改这个损失函数。

### 6.2 Anchor 回归公式

#### （1）预测框计算

给定 anchor 框 $(x_a, y_a, w_a, h_a)$ 和预测偏移 $(t_x, t_y, t_w, t_h)$，预测框的坐标：

$$x = t_x \cdot w_a + x_a$$
$$y = t_y \cdot h_a + y_a$$
$$w = w_a \cdot \exp(t_w)$$
$$h = h_a \cdot \exp(t_h)$$

**直觉理解**：
- 中心点偏移 $(t_x, t_y)$：线性缩放，因为中心点偏移与 anchor 大小成正比
- 宽高缩放 $(t_w, t_h)$：指数缩放，确保预测的宽高始终为正数

**具体例子**：
假设 anchor 在特征图位置 (10, 10)，对应原图中心 (160, 160)，anchor 大小为 128×128（1:1 比例）。

如果预测偏移为 $t_x = 0.1, t_y = -0.05, t_w = 0.2, t_h = 0.1$：

预测框中心：
$$x = 0.1 \times 128 + 160 = 172.8$$
$$y = -0.05 \times 128 + 160 = 153.6$$

预测框宽高：
$$w = 128 \times \exp(0.2) = 128 \times 1.221 = 156.3$$
$$h = 128 \times \exp(0.1) = 128 \times 1.105 = 141.4$$

所以预测框为 (172.8, 153.6, 156.3, 141.4)。

#### （2）Ground Truth 偏移计算

给定 ground truth 框 $(x^*, y^*, w^*, h^*)$ 和 anchor 框 $(x_a, y_a, w_a, h_a)$：

$$t_x^* = (x^* - x_a) / w_a$$
$$t_y^* = (y^* - y_a) / h_a$$
$$t_w^* = \log(w^* / w_a)$$
$$t_h^* = \log(h^* / h_a)$$

**边界情况**：
- 当 ground truth 框与 anchor 完全重合时，所有偏移为 0
- 当 ground truth 框是 anchor 的两倍大小时，$t_w^* = \log(2) \approx 0.693$
- 当 ground truth 框远大于 anchor 时，$t_w^*$ 可能很大，但指数函数会放大误差，所以需要 Smooth L1 来抑制异常值

### 6.3 NMS（非极大值抑制）

NMS 用于去除冗余的候选区域：

```
输入: 所有候选区域 B = {b₁, b₂, ..., bₙ}，对应分数 S = {s₁, s₂, ..., sₙ}
输出: 筛选后的候选区域 D

1. 按分数降序排序 B
2. 选择分数最高的候选区域 bᵢ，加入 D
3. 从 B 中移除 bᵢ
4. 计算 bᵢ 与 B 中所有剩余候选区域的 IoU
5. 移除 IoU > 阈值 (0.7) 的候选区域
6. 重复步骤 2-5，直到 B 为空
```

IoU 计算公式：
$$\text{IoU}(A, B) = \frac{|A \cap B|}{|A \cup B|} = \frac{|A \cap B|}{|A| + |B| - |A \cap B|}$$

**直觉理解**：NMS 假设一个物体只应该有一个检测框，如果多个框高度重叠（IoU 高），只保留分数最高的那个。

## 九、位置

### 9.1 前驱工作

1. **R-CNN [Girshick et al., 2014]**：首次将 CNN 用于目标检测，但使用 Selective Search 生成候选区域，每张图片需要处理 2000 次 CNN。

2. **SPPnet [He et al., 2015]**：提出空间金字塔池化（SPP），允许任意大小输入，但候选区域生成仍然独立。

3. **Fast R-CNN [Girshick, 2015]**：引入 ROI Pooling 和多任务损失，实现端到端训练检测网络，但候选区域生成仍然是瓶颈。

4. **OverFeat [Sermanet et al., 2014]**：用 CNN 同时预测物体类别和边框，但使用滑动窗口和图像金字塔，计算量大。

5. **MultiBox [Erhan et al., 2014]**：用 CNN 预测候选区域，但使用全连接层输出全局候选框，不能与检测网络共享卷积层。

### 9.2 同期竞品

1. **YOLO [Redmon et al., 2016]**：提出统一的端到端检测框架，将检测视为回归问题，速度更快（45 FPS），但准确率较低（63.4% mAP on VOC 2007）。

2. **SSD [Liu et al., 2016]**：结合 Faster R-CNN 的 anchor 机制和 YOLO 的端到端思想，在多个特征图尺度上预测，达到 74.3% mAP on VOC 2007（300×300 输入），速度 46 FPS。

3. **R-FCN [Dai et al., 2016]**：提出位置敏感 ROI 池化（Position-Sensitive ROI Pooling），解决 Faster R-CNN 中 ROI 池化后的计算瓶颈，达到 83.6% mAP on COCO。

### 9.3 后续影响

**直接扩展**：
1. **Mask R-CNN [He et al., 2017]**：在 Faster R-CNN 基础上增加分割分支（mask head），实现实例分割。FCIS 和 PANet 等后续工作也基于 Faster R-CNN 架构。

2. **Cascade R-CNN [Cai & Vasconcelos, 2018]**：使用级联的检测头，每个阶段使用递增的 IoU 阈值，解决正负样本不平衡问题。

3. **TridentNet [Li et al., 2019]**：使用不同空洞率的并行分支处理不同尺度的物体，在 Faster R-CNN 框架上改进多尺度检测。

**架构改进**：
4. **FPN [Lin et al., 2017]**：特征金字塔网络，在 Faster R-CNN 基础上增加自上而下的路径和横向连接，显著提升小物体检测性能。

5. **Libra R-CNN [Pang et al., 2019]**：系统分析 Faster R-CNN 的训练不平衡问题，提出 IoU-balanced sampling、balanced feature pyramid 等改进。

6. **Grid R-CNN [Lu et al., 2019]**：使用网格点回归替代传统的边框回归，提高定位精度。

**范式融合**：
7. **DETR [Carion et al., 2020]**：基于 Transformer 的端到端检测，使用匈牙利算法匹配预测和 ground truth，完全抛弃 anchor 和 NMS，但速度较慢。

8. **Sparse R-CNN [Sun et al., 2021]**：使用可学习的 proposal boxes 替代 RPN，结合 Transformer 进行特征交互。

**框架采纳**：
9. **MMDetection [Chen et al., 2019]**：基于 Faster R-CNN 设计模块化检测框架，支持多种检测器，成为最流行的目标检测代码库。

10. **Detectron2 [Wu et al., 2019]**：Facebook AI Research 的检测平台，核心架构继承自 Mask R-CNN（Faster R-CNN 的扩展）。

## 十、局限

### 10.1 小物体检测能力有限

**问题**：Faster R-CNN 在检测小物体（< 32×32 像素）时效果不佳。

**原因**：VGG-16 的下采样倍数为 16，小物体在特征图上可能只有 1-2 个像素，信息严重丢失。

**何时重要**：在需要检测微小物体的场景（如遥感图像、医学图像、监控视频）中，这个限制影响很大。

**论文是否承认**：论文没有直接讨论小物体检测问题，但后续工作（FPN [Lin et al., 2017]）专门解决这个问题。

**后续改进**：FPN 通过多尺度特征金字塔，在多个分辨率上检测物体，显著提升小物体检测性能。

### 10.2 计算瓶颈在 ROI 处理

**问题**：ROI Pooling 和后续的全连接层是计算瓶颈，特别是当候选区域数量很多时。

**原因**：每个候选区域都需要独立通过 ROI Pooling 和全连接层，这部分计算无法批量优化。

**何时重要**：在需要大量候选区域（> 1000）的场景中，ROI 处理时间可能超过 RPN 的计算时间。

**后续改进**：R-FCN [Dai et al., 2016] 使用位置敏感 ROI 池化，将大部分计算移到共享卷积层，ROI 处理只需要简单的平均池化。

### 10.3 Anchor 设计依赖先验知识

**问题**：Anchor 的尺度和长宽比需要根据数据集手动调整。

**原因**：不同的数据集有不同的物体尺寸分布（如 COCO 包含大量小物体，Cityscapes 包含大量大物体）。

**何时重要**：在迁移到新数据集或新任务时，需要重新设计 anchor 参数。

**后续改进**：MetaAnchor [Yang et al., 2018] 提出动态生成 anchor，Guided Anchoring [Wang et al., 2019] 根据特征图内容自适应生成 anchor。

### 10.4 训练过程复杂

**问题**：4-step 交替训练过程繁琐，需要多次切换训练阶段。

**原因**：RPN 和检测网络共享卷积层，但两者需要不同的训练数据和损失函数。

**何时重要**：在资源受限的环境中，复杂的训练流程增加了实验成本。

**后续改进**：SSD [Liu et al., 2016] 和 YOLO [Redmon et al., 2016] 实现了真正的端到端联合训练，YOLOv2 还引入了 anchor 机制但不需要交替训练。

### 10.5 速度仍不够实时

**问题**：0.198 秒/张（约 5 FPS）虽然比 Fast R-CNN 快很多，但离真正的实时（30 FPS）还有差距。

**原因**：VGG-16 的计算量大，ROI Pooling 和全连接层也是瓶颈。

**何时重要**：在需要实时处理视频流的应用中（如自动驾驶、视频监控），5 FPS 不够。

**后续改进**：YOLO [Redmon et al., 2016] 达到 45 FPS，SSD [Liu et al., 2016] 达到 46 FPS，都远远超过 Faster R-CNN。

## 十一、小结

Faster R-CNN 的核心贡献是提出了 **Region Proposal Network（RPN）**，让 CNN 在特征图上直接生成候选区域，把候选区域生成和检测融合到同一个网络中。这个看似简单的创新解决了目标检测领域长期存在的瓶颈问题——候选区域生成的速度和质量。

**为什么它成功了**：
1. **计算效率**：RPN 在共享特征图上运行，几乎零额外计算开销
2. **端到端训练**：候选区域生成和检测可以联合优化，互相促进
3. **Anchor 机制**：提供了合理的先验，简化了回归问题
4. **共享卷积层**：避免了重复计算，实现了速度和质量的双赢

**后续影响**：Faster R-CNN 奠定了两阶段检测器的标准范式，后续几乎所有两阶段检测器（Mask R-CNN、Cascade R-CNN、FPN 等）都基于这个架构。它也启发了单阶段检测器（SSD、YOLOv2 等）使用 anchor 机制。可以说，Faster R-CNN 是深度学习目标检测领域最具影响力的工作之一，它让目标检测从"慢速实验"走向了"实时应用"。
