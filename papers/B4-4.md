# Exploring Simple Siamese Representation Learning（2020）

**论文：** [arXiv](https://arxiv.org/abs/2011.10566) · 作者 · 年份

---

## 一、先搞清楚坑在哪

自监督学习的目标是让模型在没有人工标注的情况下，从数据本身学习到有用的视觉表征。在 2020 年前后，对比学习（contrastive learning）成为主流范式，核心思想是让同一张图片的不同增强视图（positive pairs）在特征空间中靠近，让不同图片的视图（negative pairs）互相远离。

然而，对比学习有一个绕不开的痛点：**需要负样本**。MoCo [He et al., 2020] 用动量编码器维护一个大的负样本队列，SimCLR [Chen et al., 2020] 用大 batch size 提供足够多的负样本。这些设计增加了系统复杂度，也带来了工程上的麻烦——负样本的数量和质量直接决定了性能好坏。

那问题来了：**能不能不用负样本？** 直觉上，如果只用正样本做"拉近"，模型会崩溃——把所有图片都映射到同一个点（collapse），损失函数就失效了。防止 collapse 成了自监督学习的核心难题。

## 二、现有方法的真正问题

在 SimSiam 之前，已经有几种方法尝试解决 collapse 问题，但各有各的工程包袱：

1. **MoCo**：用动量编码器+大队列提供负样本，需要额外的编码器副本和队列维护逻辑。
2. **SimCLR**：用大 batch size 提供负样本，需要 4096 的 batch，对 GPU 显存要求极高。
3. **BYOL [Grill et al., 2020]**：首次提出只用正样本也能学，用动量编码器+预测器防止 collapse。但为什么动量编码器有效？理论解释不清晰，更像是一个工程 trick。
4. **SwAV [Caron et al., 2020]**：用聚类中心替代负样本，需要在线聚类和 Sinkhorn-Knopp 算法，训练流程复杂。

这些方法的核心区别在于"如何防止 collapse"这个问题的不同解法。但一个自然的疑问是：**这些复杂的机制是否都是必要的？** 有没有可能，一个更简单的结构就能解决 collapse 问题？

## 三、SimSiam 的核心思路

SimSiam 的答案出奇简单：**什么都不做**——没有负样本，没有动量编码器，没有大 batch，没有聚类。只需要一个 Siamese 网络 + 一个预测器 + stop-gradient 操作，就能避免 collapse。

具体来说，SimSiam 对同一张图片做两次随机增强，得到两个视图 x1 和 x2。两个视图通过同一个编码器 f（共享权重），然后一个视图的编码结果通过一个预测器 h，另一个视图直接输出，两者计算负余弦相似度损失。关键操作是：**梯度只更新预测器那一侧，另一侧停止梯度传播（stop-gradient）**。

就这么简单。没有动量，没有队列，没有负样本。

## 四、方法详解

### 4.1 网络结构

SimSiam 由两个核心组件构成：

- **编码器 f**：通常是一个 ResNet-50（去掉最后的全连接层），输出一个 2048 维的特征向量。两个视图共享这个编码器，权重完全相同。
- **预测器 h**：一个小的 MLP，结构为 FC(2048) → BN → ReLU → FC(2048) → BN。注意输出层没有 ReLU，最后一个 BN 层在推理时可去掉。

### 4.2 前向过程

```
输入图片 x
    │
    ├── 随机增强 T1 ──→ x1
    │       │
    │       └── 编码器 f (共享权重) ──→ z1 = f(x1)  [2048-d]
    │               │
    │               └── 预测器 h ──→ p1 = h(z1)  [2048-d]
    │
    └── 随机增强 T2 ──→ x2
            │
            └── 编码器 f (共享权重) ──→ z2 = f(x2)  [2048-d]
                    │
                    └── 停止梯度 ──→ sg(z2)
                            │
                            └── 计算损失：-cosine(p1, sg(z2))
```

关键点：
- 编码器 f 对 x1 和 x2 是**完全共享权重**的，不是动量编码器那种"缓慢更新"。
- 预测器 h 只加在其中一个视图上。
- **stop-gradient 是唯一防止 collapse 的机制**——如果没有它，模型会迅速崩溃到所有输出相同的 trivial solution。

### 4.3 损失函数

损失函数是**负余弦相似度**：

$$D(p_1, z_2) = -\frac{p_1}{\lVert p_1 \rVert_2} \cdot \frac{z_2}{\lVert z_2 \rVert_2}$$

对称化处理：两个视图都过一遍，但每次只梯度更新预测器那一侧：

$$L = \frac{1}{2} D(p_1, \text{sg}(z_2)) + \frac{1}{2} D(p_2, \text{sg}(z_1))$$

其中 $\text{sg}$ 表示 stop-gradient。

### 4.4 为什么 stop-gradient 有效？

这是 SimSiam 最核心的发现。论文给出了一个理论分析框架，把 SimSiam 看作一种**隐式的 EM 算法**。

想象一下，如果我们把优化目标写成：

$$\min_{\theta, \eta} \sum_i \mathbb{E}_{T} \left\lVert F_\theta(T(x_i)) - \eta_{x_i} \right\rVert_2^2$$

其中 $F_\theta$ 是编码器+预测器，$\eta_{x_i}$ 是每张图片的"表征"（类似聚类中心）。这个目标可以交替优化：
- **E-step**：固定 $\theta$，优化 $\eta$，即 $\eta_{x_i} = \mathbb{E}_T [F_\theta(T(x_i))]$
- **M-step**：固定 $\eta$，优化 $\theta$，即最小化 $\lVert F_\theta(T(x_i)) - \eta_{x_i} \rVert^2$

SimSiam 的 stop-gradient 恰好模拟了**一步 E-step 和一步 M-step 的交替**。当我们在 x1 上做预测、x2 上停止梯度时，x2 的输出就相当于一个"伪标签" $\eta$，而 x1 的预测器则在尝试拟合这个伪标签。下一次迭代时，伪标签更新了（因为编码器变了），预测器再去追新的伪标签。

这个交替过程天然防止了 collapse，因为伪标签 $\eta$ 是通过 stop-gradient 固定的，不会随着预测器一起坍缩。

## 五、核心创新点

### 5.1 创新一：stop-gradient 作为唯一的 collapse 防止机制

**(a) 痛点与动机**

在 SimSiam 之前，所有正样本学习方法（BYOL、SwAV）都依赖额外的机制防止 collapse。BYOL 用动量编码器，SwAV 用聚类。这些机制增加了系统复杂度，也模糊了"到底什么才是防止 collapse 的关键因素"这个问题的答案。SimSiam 的动机很简单：**如果去掉所有额外机制，只保留最核心的 stop-gradient，模型还能学吗？**

**(b) 方案细节**

SimSiam 在计算损失时，对其中一个视图的输出施加 stop-gradient 操作。在 PyTorch 中就是一行代码：

```python
# x1 和 x2 是两个增强视图
z1 = f(x1)  # 编码器输出
z2 = f(x2)
p1 = h(z1)  # 预测器输出
p2 = h(z2)

# 对称损失，但梯度只更新预测器那一侧
loss = D(p1, z2.detach()) / 2 + D(p2, z1.detach()) / 2
```

`z2.detach()` 就是 stop-gradient——z2 作为目标，不接收梯度。

**(c) 为什么有效**

论文通过大量消融实验证明：
1. **没有 stop-gradient，模型立即 collapse**：训练 loss 降到 -1（余弦相似度的最小值），所有输出变成常数。
2. **有 stop-gradient，即使没有动量编码器、没有负样本，模型也能稳定训练**。
3. **stop-gradient 的效果与动量编码器类似，但更简单**：动量编码器本质上是"缓慢更新的 stop-gradient"。

**(d) 与 Related Work 的关系**

- BYOL [Grill et al., 2020] 用动量编码器，其 online 网络更新 target 网络的动量版本。SimSiam 证明动量不是必要的，stop-gradient 就够。
- SwAV [Caron et al., 2020] 用在线聚类生成伪标签。SimSiam 证明聚类也不是必要的，stop-gradient 就够。
- 更早的 Bootstrap Your Own Latent [Grill et al., 2020] 其实也隐含了 stop-gradient，但被动量编码器的叙事掩盖了。

**(e) 如果去掉会怎样**

论文 Table 2 做了关键消融：
- 去掉 stop-gradient：loss 立即 collapse 到 -1，top-1 accuracy 降到 0.1%（随机猜测水平）。
- 保留 stop-gradient：top-1 accuracy 达到 67.4%（在 ImageNet 上，100 轮训练）。

### 5.2 创新二：预测器是必要条件

**(a) 痛点与动机**

如果只用 stop-gradient，但不用预测器，模型会怎样？直觉上，如果没有预测器，损失就变成了直接对齐两个编码器的输出。但因为 stop-gradient 的存在，梯度只更新一侧，这会导致不对称的优化——被更新的那一侧编码器会不断"适应"固定目标，最终可能还是 collapse。

**(b) 方案细节**

预测器 h 是一个小 MLP，加在其中一个视图的编码器输出后面。结构为：FC(2048) → BN → ReLU → FC(2048) → BN。

**(c) 为什么有效**

预测器的作用是**打破对称性**。没有预测器时，两个分支完全对称（都是编码器 f），stop-gradient 只是让一个分支不更新，但两个分支的映射函数是一样的。加上预测器后，p1 = h(f(x1)) 和 z2 = f(x2) 是两个不同的函数，这种不对称性让 stop-gradient 的 EM 解释更自然——预测器在"拟合"另一个视图的编码器输出。

**(d) 与 Related Work 的关系**

BYOL 也有预测器，但 BYOL 的预测器是加在 online 网络上，target 网络没有预测器。SimSiam 的预测器设计与 BYOL 类似，但 SimSiam 的两个分支共享编码器权重（BYOL 的 online 和 target 权重不同），这进一步简化了架构。

**(e) 如果去掉会怎样**

论文 Table 2 显示：
- 去掉预测器：top-1 accuracy 从 67.4% 降到 55.0%，性能显著下降。
- 但即使没有预测器，模型也没有 collapse（55% 远高于随机猜测），这说明 stop-gradient 本身才是防止 collapse 的关键，预测器是提升性能的辅助。

### 5.3 创新三：共享编码器权重

**(a) 痛点与动机**

BYOL 的 online 和 target 网络权重不同，需要动量更新。这增加了参数量和工程复杂度。一个自然的问题是：**能否让两个分支共享同一个编码器？**

**(b) 方案细节**

SimSiam 的两个视图通过同一个编码器 f，权重完全共享。这是通过 PyTorch 的 `nn.Module` 共享实现的——同一个模型实例处理两个输入。

**(c) 为什么有效**

共享权重有两个好处：
1. **减少参数量**：编码器只有一份，而不是两份。
2. **简化训练**：不需要动量更新，标准 SGD 即可。

为什么共享权重不会导致 collapse？因为 stop-gradient 打破了对称性——虽然编码器权重相同，但一个分支有预测器，另一个没有，损失函数不是对称的。

**(d) 与 Related Work 的关系**

- MoCo [He et al., 2020] 的编码器是动量更新的，query 和 key 编码器不同。
- BYOL [Grill et al., 2020] 也是动量更新。
- SimCLR [Chen et al., 2020] 虽然共享权重，但需要负样本。
- SimSiam 是第一个在正样本学习方法中实现完全共享权重的。

**(e) 如果去掉会怎样**

论文实验了两种变体：
1. **动量编码器**（类似 BYOL）：性能与共享权重版本相当，但参数更多。
2. **两个独立编码器**（不共享权重，不动量更新）：模型 collapse。

这说明共享权重不是 collapse 的原因，关键还是 stop-gradient。

## 六、公式详解

### 损失函数

$$D(p_1, z_2) = -\frac{p_1}{\lVert p_1 \rVert_2} \cdot \frac{z_2}{\lVert z_2 \rVert_2}$$

- **p1**: 预测器输出，维度 2048。p1 = h(f(x1))。
- **z2**: 编码器输出，维度 2048。z2 = f(x2)，且施加了 stop-gradient。
- **分母**: L2 范数，将向量归一化到单位球面。

**直觉**：这个损失就是两个单位向量之间的负余弦相似度。当两个向量方向一致时，余弦相似度为 1，损失为 -1；方向相反时，余弦相似度为 -1，损失为 1。所以最小化损失就是让两个向量方向尽可能一致。

**具体数值例子**：
假设 p1 = [2, 1, 0, ...], z2 = [1, 2, 0, ...]（简化到 2 维便于计算）

- ||p1|| = sqrt(4 + 1) = sqrt(5) ≈ 2.236
- ||z2|| = sqrt(1 + 4) = sqrt(5) ≈ 2.236
- p1/||p1|| = [0.894, 0.447]
- z2/||z2|| = [0.447, 0.894]
- D = -(0.894 × 0.447 + 0.447 × 0.894) = -(0.4 + 0.4) = -0.8

**边界情况**：
- 当 p1 和 z2 完全对齐时，D = -1（最小值）。
- 当 p1 和 z2 正交时，D = 0。
- 当 p1 和 z2 方向相反时，D = 1（最大值）。

**对称损失**：

$$L = \frac{1}{2} D(p_1, \text{sg}(z_2)) + \frac{1}{2} D(p_2, \text{sg}(z_1))$$

这个对称形式确保两个视图都得到利用。注意每次只有一个预测器分支参与梯度更新（另一个分支的 z 被 stop-gradient）。

## 八、代码对照

论文的官方实现（PyTorch）在 `/simsiam` 目录下。核心代码在 `builder.py` 中：

```python
class SimSiam(nn.Module):
    def __init__(self, base_encoder, dim=2048, pred_dim=512):
        super(SimSiam, self).__init__()
        self.encoder = base_encoder(num_classes=dim)  # ResNet-50 输出 2048 维
        self.predictor = nn.Sequential(
            nn.Linear(dim, pred_dim, bias=False),  # 2048 -> 512
            nn.BatchNorm1d(pred_dim),
            nn.ReLU(inplace=True),
            nn.Linear(pred_dim, dim),  # 512 -> 2048
        )

    def forward(self, x1, x2):
        z1 = self.encoder(x1)  # 编码器输出
        z2 = self.encoder(x2)
        
        p1 = self.predictor(z1)  # 预测器输出
        p2 = self.predictor(z2)
        
        # 对称损失 + stop-gradient
        loss = -(F.cosine_similarity(p1, z2.detach(), dim=-1).mean() / 2
                 + F.cosine_similarity(p2, z1.detach(), dim=-1).mean() / 2)
        return loss
```

代码中值得注意的细节：
1. **预测器的中间维度是 512**（不是 2048），这是一个 bottleneck 设计，减少参数量。
2. **预测器的最后一层没有 BN**，这与论文描述一致（"输出层没有 ReLU，最后一个 BN 层在推理时可去掉"）。
3. **损失直接用负余弦相似度**，没有温度参数（SimCLR 需要温度）。
4. **batch normalization 的位置很关键**：编码器的最后一层有 BN，预测器中间层也有 BN。如果去掉预测器的 BN，性能会下降（Table 2 显示从 67.4% 降到 65.8%）。

## 九、位置

### 9.1 前驱工作

- **MoCo [He et al., 2020]**：提出了动量编码器+负样本队列的范式。SimSiam 继承了对"表征一致性"的追求，但去掉了负样本。
- **BYOL [Grill et al., 2020]**：首次证明只用正样本也能学。SimSiam 进一步简化，去掉了动量编码器。
- **SwAV [Caron et al., 2020]**：用聚类替代负样本。SimSiam 证明聚类也不是必要的。
- **SimCLR [Chen et al., 2020]**：建立了对比学习的标准框架（数据增强+NT-Xent 损失）。SimSiam 受其增强策略的启发。

### 9.2 同期竞品

- **Barlow Twins [Zbontar et al., 2021]**：用互相关矩阵的对角化替代对比学习。与 SimSiam 同期，也追求简洁性。
- **VICReg [Bardes et al., 2021]**：用方差-不变性-协方差正则化防止 collapse。与 SimSiam 同期，理论更清晰。

### 9.3 后续影响

1. **直接扩展**：
   - **DINO [Caron et al., 2021]**：将 SimSiam 的 stop-gradient 思想扩展到 Vision Transformer，用自蒸馏实现自监督学习。
   - **MoCo v3 [Chen et al., 2021]**：将 SimSiam 的预测器设计引入 MoCo 框架，发现 stop-gradient 对 ViT 训练至关重要。

2. **架构改进**：
   - **BYOL 的后续工作**：受 SimSiam 启发，后续研究开始重新审视 BYOL 的动量编码器是否必要。
   - **SimSiam + ViT**：SimSiam 的简单架构使其成为研究 ViT 自监督学习的理想平台。

3. **范式融合**：
   - **Masked Autoencoders [He et al., 2021]**：虽然 MAE 走的是掩码重建路线，但其"预测缺失部分"的思路与 SimSiam 的"预测另一个视图"有异曲同工之妙。
   - **CLIP [Radford et al., 2021]**：多模态领域的对比学习，其"拉近匹配对"的思想与 SimSiam 的单模态自监督有相似之处。

4. **框架采用**：
   - **Self-Supervised Learning Benchmark**：SimSiam 成为自监督学习的标准基线之一。
   - **Hugging Face Transformers**：一些视觉模型的自监督预训练采用了 SimSiam 的 stop-gradient 策略。

## 十、局限
