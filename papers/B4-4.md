# Exploring Simple Siamese Representation Learning: SimSiam（2020）

**论文：** [arXiv:2011.10566](https://arxiv.org/abs/2011.10566) · Xinlei Chen, Kaiming He · Facebook AI Research (FAIR) · 2020

**技术分支：** B4 — 是否可以形成统一视觉 encoder 基座模型？ / B4-2 — non-contrastive representation learning

**代码：** [github.com/facebookresearch/simsiam](https://github.com/facebookresearch/simsiam)

---

## 一、先搞清楚"问题在哪里"

2020 年，自监督视觉表征学习已经取得了显著进展。以 SimCLR、MoCo、BYOL、SwAV 为代表的方法，在 ImageNet 线性评估上把 top-1 准确率从两年前的个位数推到了 70% 以上。这些方法的性能差距在缩小，但它们的**复杂度**差距却在扩大——每个方法都引入了一套防止表征坍塌（collapse）的机制。

回顾一下这些机制的各自"配置"：

| 方法 | 防坍塌机制 | 需要的额外组件 | 工程成本 |
|------|-----------|--------------|---------|
| SimCLR | 负样本对比 | 大 batch（4096） | 32+ GPU，LARS 优化器 |
| MoCo | 负样本队列 | 动量编码器、队列 | 双编码器、队列管理 |
| SwAV | 在线聚类 + 均衡分配 | Sinkhorn-Knopp 变换 | 额外优化步骤 |
| BYOL | 不对称预测 + EMA | 动量编码器 | 双网络参数存储 |

问题在于：**所有这些多余的复杂度，真的都是必要的吗？**

这篇论文的核心追问是：Siamese 网络本身是否已经足够强大，以至于我们只要挂上最简单的 stop-gradient（停止梯度传播），就能自动防止坍塌？带着这个追问，作者 Chen 和 He 把那些"花哨"的组件——负样本、动量编码器、大 batch、在线聚类——逐个去掉，最后发现了一个令人惊讶的事实：**在完整的 Siamese 网络 + 预测器 + stop-gradient 这三件套下，什么都不加也能学到有竞争力的表征。**

## 二、旧路线的真正问题

### 2.1 对比学习的"负样本诅咒"

对比学习的核心范式是 InfoNCE 损失：

$$ \ell_{i,j} = -\log \frac{\exp(\text{sim}(z_i, z_j) / \tau)}{\sum_{k=1}^{2N} \mathbb{1}_{k \neq i} \exp(\text{sim}(z_i, z_k) / \tau)} $$

分母中的 $2N-1$ 个负样本提供了"推开"力，防止模型把所有的表征都坍缩到同一个点。但这也意味着：

**问题 1：负样本数量决定了性能天花板。** SimCLR 实验表明，batch size 从 256 增加到 4096，top-1 准确率从 61.5% 提高到 69.3%。更大的 batch = 更多的负样本 = 更好的表征，但这需要 32 张 V100。

**问题 2：负样本质量不可控。** 随机采样到的负样本大多是"猫"vs"汽车"这种显而易见的区别，对模型提升有限。真正的困难负样本——"猫"vs"猫科动物雕塑"——被随机采到的概率太低。模型花大量计算在简单的负样本上，效率低下。

**问题 3：一致性困境。** MoCo 试图用 memory bank + 动量编码器来解决 batch size 限制，但 memory bank 中存储的表示由旧编码器生成，而当前编码器已经更新——新旧表示之间存在偏差。想要大字典，就得接受过时的表示。

### 2.2 BYOL 的成功与残留问题

BYOL 证明了"没有负样本也能学"，这是一个突破。但 BYOL 引入了两个新的依赖：

1. **动量编码器**：目标网络通过 EMA 缓慢跟踪在线网络。BYOL 论文声称动量编码器是"防止坍塌的关键"——去掉它，表 5 显示准确率从 74.3% 降到 0.3%（实际是 66.9%，BYOL v3 纠正后）。
2. **不对称预测器**：在线网络有预测器，目标网络没有。

这引出了 SimSiam 要回答的更本质问题：**如果动量编码器本身也有 stop-gradient（因为 momentum update 不传梯度），那防止坍塌的到底是谁的功劳——EMA 还是 stop-gradient？** 没有人问过这个问题，因为动量编码器天然带有 stop-gradient，两者被始终绑定在一起。

## 三、这篇论文的核心思路

**这篇论文的核心思路是：在完全对称的 Siamese 网络（共享权重）上，去掉负样本、去掉动量编码器、去掉大 batch，只保留预测器 + stop-gradient 这一对关键设计，证明这样极简的结构就能有效防止坍塌并学到有竞争力表征。**

SimSiam 的位置在技术谱系中的定位异常有趣——它有点像"每个先行者减去一个核心组件"：

- **BYOL 去掉动量编码器** = SimSiam（保留预测器 + stop-gradient）
- **SimCLR 去掉负样本** = SimSiam（加上预测器 + stop-gradient）
- **SwAV 去掉在线聚类** = SimSiam（加上预测器 + stop-gradient）

```
有监督 ImageNet 预训练（2012-2018）
    ↓
对比学习范式（2019-2020）
    ├── SimCLR（大 batch，负样本）
    ├── MoCo（队列+动量编码器）
    └── BYOL（EMA+预测器，无负样本，2020）
        ↓
"最少组件"探索（2020）
    └── SimSiam（停止梯度+预测器）← 本篇
        ↓
后续扩展
    ├── DINO（EMA+自蒸馏+center+sharpening，ViT，2021）
    └── iBOT（MIM+在线 tokenizer，2021）
```

## 四、整体结构与 forward 流程

### 4.1 整体架构

SimSiam 的架构极度简洁，所有组件加起来只有三个：

```
图像 x
    │
    ├── 增强 T₁ ──→ x₁ ──→ 编码器 f（共享权重）──→ z₁
    │                          │
    │                          ├──→ 投影 MLP（2048-d 隐层，3 层）
    │                          │    （每层有 BN，输出 2048-d）
    │                          │
    │                          └──→ 输出 2048-d 投影向量 z
    │
    │                          ┌──→ 预测 MLP（512-d 瓶颈隐层，2 层）
    │                          │    （隐层有 BN，输出无 BN 或 ReLU）
    │                          │
    │                          ▼
    │                       p₁ = h(z₁)   ← 预测值
    │
    │                         ┌──────────────┐
    │                         │  L = D(p₁, sg(z₂))/2  │
    │                         │     + D(p₂, sg(z₁))/2 │
    │                         └──────────────┘
    │                                     ▲
    ▼                                     │
    │                                     │
    └── 增强 T₂ ──→ x₂ ──→ 编码器 f（共享）──→ z₂ ──→ stop-gradient ──→ sg(z₂)
```

**关键符号：**
- $f$：编码器 = ResNet-50 backbone + 3 层投影 MLP（2048-d，每层 BN）
- $h$：预测器 = 2 层 MLP（512-d 瓶颈隐层，隐层 BN，输出无 BN/ReLU）
- $x_1, x_2$：同一图像的两个随机增强视图
- $z_1, z_2$：编码器输出（$\mathbb{R}^{n \times 2048}$）
- $p_1, p_2$：预测器输出（$\mathbb{R}^{n \times 2048}$）
- sg：stop-gradient 操作

### 4.2 前向过程逐步跟踪

**Step 1 — 数据增强**

对图像 $x$ 独立采样两次随机增强 $T_1, T_2 \sim \mathcal{T}$，得到 $x_1 = T_1(x)$，$x_2 = T_2(x)$。

增强策略遵循 SimCLR 流程：随机裁剪 → resize 224×224 → 颜色抖动 → 灰度化 → 水平翻转 → 高斯模糊。

输入形状：`[B, 3, 224, 224]`，两个视图各 B 张。

**Step 2 — 编码器前向（共享权重）**

两个增强视图通过**同一个编码器 $f$**（共享权重），分别输出投影向量 $z_1, z_2 \in \mathbb{R}^{B \times 2048}$：

$$ z_1 = f(x_1), \quad z_2 = f(x_2) $$

编码器 $f$ 的构成为 ResNet-50 backbone → 全局平均池化（2048-d）→ 3 层投影 MLP。投影 MLP 的结构：
- 第 1 层：Linear(2048, 2048) + BN + ReLU
- 第 2 层：Linear(2048, 2048) + BN + ReLU
- 第 3 层：Linear(2048, 2048) + BN（**输出层也带 BN**，但无 ReLU）

最终输出 $z$ 的维度是 $2048$。

**Step 3 — 预测器前向（仅一个视图）**

其中一个视图的输出 $z_1$ 经过预测器 $h$，得到预测值 $p_1$。预测器是一个 2 层**瓶颈结构** MLP：
- 第 1 层：Linear(2048, 512) + BN + ReLU
- 第 2 层：Linear(512, 2048)（**无 BN、无 ReLU**）

$$ p_1 = h(z_1) \in \mathbb{R}^{B \times 2048} $$

为什么是瓶颈结构（2048→512→2048）？论文在补充材料中的消融表明：瓶颈结构比等宽结构（2048→2048→2048）效果更好。可能的解释是：瓶颈迫使预测器学习更紧凑的映射，而不是简单地复制输入。

**Step 4 — 对称计算**

对称地，$z_2$ 经过 $h$ 得到 $p_2 = h(z_2)$。

注意：两个视图**都经过预测器**（对称结构的体现），但 stop-gradient 对**目标侧**（被预测的 $z$）施加。

**Step 5 — 损失计算**

对于每一对 $(p_1, z_2)$，计算**负余弦相似度**（negative cosine similarity）：

$$ \mathcal{D}(p_1, z_2) = -\frac{p_1}{\|p_1\|_2} \cdot \frac{z_2}{\|z_2\|_2} $$

注意 $z_2$ 在这里经过了 `detach()`（stop-gradient），在梯度计算中被视为常数。

整体对称损失为：

$$ \mathcal{L} = \frac{1}{2} \mathcal{D}(p_1, \text{stopgrad}(z_2)) + \frac{1}{2} \mathcal{D}(p_2, \text{stopgrad}(z_1)) $$

损失最小值是 $-1$（当 $p$ 和 $z$ 完全同向时）。

**Step 6 — 梯度反向传播与参数更新**

只更新编码器 $f$ 和预测器 $h$。使用标准 SGD 优化器——不需要 LARS。

这里有一个关键细节：**编码器 $f$ 从两个途径接收梯度**。以 $x_2$ 经过的编码器为例：
- 从第一项 $\mathcal{D}(p_1, \text{stopgrad}(z_2))$ 中，$z_2$ 被 stop-grad，**不贡献梯度**
- 从第二项 $\mathcal{D}(p_2, \text{stopgrad}(z_1))$ 中，$p_2 = h(f(x_2))$，**梯度正常传播**

也就是说，同一个编码器对 $x_2$ 的处理，在"被预测"时不收梯度，在"作为预测源"时正常收梯度。这正是 stop-gradient 的精妙之处——它打破了对称性，让优化变成了一种隐式的交替过程。

## 五、核心模块逐个解释

### 5.1 Stop-gradient：唯一不可缺的魔法

**这是整篇论文最重要的发现。**

去掉 stop-gradient 的效果：训练损失直接降到理论最小值 $-1$，所有样本的编码器输出坍塌到同一个常数向量——从 68.1% 降到 0.1%（ImageNet 的随机猜测水平）。

保留 stop-gradient 的效果：输出的标准偏差稳定在 $1/\sqrt{d}$ 附近——这是各向同性高斯分布的理论值——意味着表征散布在整个单位超球面上，没有坍塌。

**一个极其重要的观察：** 去掉 stop-gradient 时，架构中的其他所有设计（预测器、BN、L2 归一化）都**原封不动**——但它们没有任何一个能独立防止坍塌。这说明防止坍塌的全部功劳在 stop-gradient 上，而不是在其他组件上。

**但 stop-gradient 到底在做什么？**

直观理解：stop-gradient 强迫优化过程变成一个"两步走"的交替。第一侧（预测侧）要努力追赶另一侧的输出，但另一侧（目标侧）是"冻住"的。目标侧只在轮到它充当预测侧时才更新。这类似于交替优化两个子问题——而"同时优化所有变量"正是导致坍塌的原因。

**为什么这在直觉上合理？** 如果不用 stop-gradient，梯度会把两条分支的编码器 $f$ 向同一个方向更新，同时包括 $z_1$ 和 $z_2$ 的梯度。这就给了模型一个简单的"捷径"：把所有 $z$ 推向同一个值。用了 stop-gradient，一条分支的 $f$ 从另一条分支的预测误差中收到梯度——预测误差大，说明预测器的映射还不够好，而不是编码器直接需要把 $z$ 拉近——这样就打破了"大家一起走捷径"的路径。

### 5.2 预测器（Predictor MLP）

预测器 $h$ 的作用是创建不对称性。没有 $h$ 的话，损失变为：

$$ \mathcal{L} = \frac{1}{2}\mathcal{D}(z_1, \text{stopgrad}(z_2)) + \frac{1}{2}\mathcal{D}(z_2, \text{stopgrad}(z_1)) $$

有趣的是，这个损失的梯度方向等价于去掉 stop-gradient 后 $\mathcal{D}(z_1, z_2)$ 的梯度方向（只是大小减半），所以效果和"没有 stop-gradient"一样——坍塌。

**三个消融实验清楚地说明了预测器的作用：**

| 配置 | 准确率 | 故障模式 |
|------|--------|---------|
| 基线（有 $h$） | **67.7%** | 正常训练 |
| 无 $h$ | 0.1% | 坍塌（梯度分析如上） |
| $h$ 固定为随机初始化 | 1.5% | 不收敛（不是坍塌，loss 持续偏高） |
| $h$ 学习率不衰减 | **68.1%** | 好于基线！ |

**为什么 $h$ 的学习率不衰减反而更好？** 论文的假设是：$h$ 的作用是适应编码器的最新表征。如果 $h$ 的学习率过早衰减，它就被"固化"了，无法跟踪表征的变化。让 $h$ 一直保持高的学习率，反而能让它更好地拟合当前的表征空间——这实际上是论文 EM 假设的一个佐证：$h$ 是在做"一步预测"的逼近，需要持续学习。

### 5.3 Batch size 无关性

SimSiam 不需要大 batch：

| Batch size | 64 | 128 | 256 | 512 | 1024 | 2048 | 4096 |
|-----------|-----|-----|-----|-----|------|------|------|
| 准确率 | 66.1% | 67.3% | 68.1% | 68.1% | 68.0% | 67.9% | 64.0% |

batch=64 到 2048 之间，性能几乎不变（67-68%）。这和使用标准 SGD 优化器（而非 LARS）密切相关——batch=4096 时的下降是因为 SGD 在大 batch 下本就不稳定（监督学习也有此问题）。

对比之下：SimCLR 的 batch size 从 256 到 4096，准确率变化了 8 个百分点。SimSiam 在这方面的鲁棒性意味着**它可以在单卡 8GB 显存上训练**，大幅降低了自监督学习的硬件门槛。

## 六、核心公式与数学直觉

### 6.1 损失函数

$$ \mathcal{D}(p_1, z_2) = -\frac{p_1}{\|p_1\|_2} \cdot \frac{z_2}{\|z_2\|_2} $$

**这个式子在做什么：** L2 归一化后将两个向量投影到单位球面上，取它们的余弦相似度并取负。当两个向量方向完全一致时，值为 $-1$；正交时 $0$；反向时 $1$。

**和 MSE 的关系：**

$$ \| \overline{p_1} - \overline{z_2} \|_2^2 = 2 - 2\cos(\theta) = 2 + 2\mathcal{D}(p_1, z_2) $$

所以优化负余弦相似度等价于优化 L2 归一化后的 MSE——两者的梯度方向完全相同，相差一个常数缩放。论文选择负余弦相似度形式的原因是：数值范围已知（$[-1, 1]$），方便监控训练进度。

### 6.2 Stop-gradient 的隐式分析

论文的核心技术洞察是：

$$ \mathcal{L} = \frac{1}{2}\mathcal{D}(p_1, \text{sg}(z_2)) + \frac{1}{2}\mathcal{D}(p_2, \text{sg}(z_1)) $$

**梯度流动分析：**

令 $\mathcal{L}_1 = \frac{1}{2}\mathcal{D}(h(f(x_1)), \text{sg}(f(x_2)))$。梯度：

$$ \frac{\partial \mathcal{L}_1}{\partial \theta_f} = \frac{1}{2} \frac{\partial \mathcal{D}(h(f(x_1)), \text{sg}(f(x_2)))}{\partial f(x_1)} \cdot \frac{\partial f(x_1)}{\partial \theta_f} $$

注意这里对 $f(x_2)$ **不求导**（因为 sg），但对 $f(x_1)$ 正常求导。

整体上，编码器参数 $\theta_f$ 同时从 $\mathcal{L}_1$（通过 $f(x_1)$）和 $\mathcal{L}_2$（通过 $f(x_2)$ 的 $p_2 = h(f(x_2))$ 部分）接收梯度。但关键的区别是：**目标侧（被预测的那条分支的输出）永远是冻结的**。这意味着：

1. $x_1$ 侧的编码器：从 $\mathcal{L}_1$（作为预测源）和 $\mathcal{L}_2$（作为被预测目标 → 但被冻结）接收梯度 → 实际仅从 $\mathcal{L}_1$ 得到有效梯度
2. $x_2$ 侧的编码器：同样情况，仅从 $\mathcal{L}_2$ 得到有效梯度

**所以实际上，每步梯度更新只作用于"预测源"那一侧。** 这就是"交替优化"的实质——两个视图轮流扮演"追赶者"和"被追赶者"。

### 6.3 与 BYOL 的梯度对比

BYOL 的梯度路径：在线网络 $\theta$ 通过标准梯度下降更新，目标网络 $\xi$ 通过 EMA 更新（不传梯度）。

SimSiam 的梯度路径：两个分支的参数通过同一个 $\theta_f$ 共享，stop-gradient 让某些梯度路径被阻断。

**关键区别：** BYOL 用的是**双参数集**（$\theta, \xi$），SimSiam 用的是**单参数集**（$\theta_f$）但通过 stop-gradient 模拟双参数集的效果。SimSiam 更简单但更巧妙——它证明了阻止坍塌的机制可以不在于"参数数量的翻倍"，而在于"梯度路径的选择性阻断"。

## 七、重要图表解释与 ASCII 重构

### 7.1 架构图（论文 Figure 1）

```text
   Image x
      │
      ├── T₁(x)=x₁ ──→ f（共享编码器）──→ z₁ ──→ h（预测器）──→ p₁
      │                                                   │
      │                                                   ▼
      │                                          ┌────────────────┐
      │                                          │ L = -cosine    │
      │                                          │ (p₁, sg(z₂))  │
      │                                          │ + symmetrized  │
      │                                          └────────────────┘
      │                                                          ▲
      │                                                          │
      └── T₂(x)=x₂ ──→ f（共享编码器）──→ z₂ ──→ stop-gradient ──┘
```

**论文 Figure 1 在说什么：** 这幅图是整个论文最关键的视觉信号。它的要点是：
- 编码器 $f$ 是**共享权重**的（图中只画了一个 $f$ 框）——区别于 BYOL 的双编码器
- 预测器 $h$ 只在一侧——但对称损失让两个视图都经过 $h$（只是 $z$ 一侧被 stop-grad）
- stop-gradient 符号 $\text{sg}$——这是唯一的"特殊机制"

### 7.2 stop-gradient 消融（论文 Figure 2）

```
损失曲线（左）：                    输出标准差（中）：              kNN 准确率（右）：
loss                                std(z/||z||₂)                 kNN top-1
│                                   │                             │
│  w/o sg:  迅速降至 -1              │  w/o sg:  → 0（坍塌）        │  w/ sg:  → 持续提升
│  （模型坍塌了）                    │                               │
│                                   │  w/ sg: ≈ 1/√d               │  w/o sg: → 0（随机）
│  w/ sg:  缓慢下降                  │  （完美散布在球面）            │
│                                   │                             │
└───→ epoch                        └───→ epoch                   └───→ epoch
```

**这个图真正在说什么：** 它用三个互不依赖的证据证明了 stop-gradient 的作用：
1. **损失曲线**：w/o sg 立即降到最小值，表明模型找到了"捷径"
2. **输出标准差**：w/o sg 降为零，直接证明了坍塌（所有样本映射到同一点）
3. **kNN 准确率**：w/ sg 稳步提升，说明表征质量在持续改善

这三个指标的组合是极其有力的——任何单一指标都可能被误读，但三者同时指向同一个结论。

准确率（%）

68 │    ●    ●    ●    ●    ●

SimSiam 在 64-2048 的 batch size 范围内的准确率波动只有 ~2 个百分点。对比 SimCLR 在同样范围内的 8+ 个百分点波动，SimSiam 的稳定性令人印象深刻。

## 八、复杂度、效率与可扩展性分析

### 8.1 计算复杂度

SimSiam 每步计算量：
- 编码器 $f$ 前向：两个视图分开算，实际上因为共享参数，等价于 2 倍 batch 的编码器一次前向
- 预测器 $h$ 前向：两个视图各一次
- 损失计算：只需计算 B 对余弦相似度（$O(B)$），不需要 SimCLR 的 $O(B^2)$ 对比矩阵

**定量对比（ResNet-50）：**

| 方法 | 编码器前向 | 损失复杂度 | 优化器 |
|------|----------|-----------|--------|
| SimCLR | $2B$ 次 | $O(B^2)$ | LARS（需要） |
| MoCo | $2B$ 次（在线+动量） | $O(BK)$（K=queue size） | SGD |
| BYOL | $2B$ 次（在线+目标） | $O(B)$ | LARS（默认） |
| **SimSiam** | **$2B$ 次** | **$O(B)$** | **SGD（标准）** |

**关键点：** SimSiam 是所有方法中**最简单**的——标准 SGD、$O(B)$ 损失计算、无动量编码器、无队列。

### 8.2 显存消耗

SimSiam 只需要一组编码器参数（单参数集），没有动量编码器的双倍参数。对于 ResNet-50：
- 编码器 $f$：~24M 参数（包括投影 MLP）
- 预测器 $h$：~2.6M 参数（2048-512-2048，瓶颈结构）
- 总参数量：~27M（远小于 BYOL 的 ~50M）
- 梯度图大小：只有 ~27M 参数对应的梯度 —— 单张 8GB GPU 可以训练 batch=256

### 8.3 训练效率

论文中所有消融实验在 100 epoch 下完成。100 epoch 的 SimSiam 在 8 张 V100 上大约需要 2 天。

对比 BYOL 的 1000 epoch（5-7 天），SimSiam 的训练周期更短。但需要注意的是，SimSiam 的 800 epoch 结果（71.3%）不及 BYOL 的 800 epoch（74.3%），说明 SimSiam 的长训练收益不如 BYOL——可能因为缺少 EMA 提供的平滑更新。

### 8.4 超参数鲁棒性

| 超参数 | 范围 | 性能变化 |
|--------|------|---------|
| Batch size | 64-4096 | 66.5%-68.1%（< 2% 波动） |
| 投影 MLP 隐层维度 | 1024, 2048, 4096 | ~1% 波动 |
| 预测器 LR 是否衰减 | 衰减/不衰减 | 均有效（不衰减略优） |
| 对称损失 vs 非对称 | 对称/非对称 | 对称优 3.3% |

SimSiam 是已发表的自监督方法中超参数最鲁棒的一个——这与其简单的设计直接相关。

### 9.1 ImageNet 线性评估

所有方法基于 ResNet-50，两个 224×224 视图预训练，单 crop 线性评估：

| 方法 | 100 ep | 200 ep | 400 ep | 800 ep | 负样本 | 动量编码器 | 大 batch |
|------|--------|--------|--------|--------|--------|------------|---------|
| SimCLR (repro.) | 66.5% | 68.3% | 69.8% | 70.4% | ✅ | ❌ | ✅ (4096) |
| MoCo v2 (repro.) | 67.4% | 69.9% | 71.0% | 72.2% | ✅ | ✅ | ❌ (256) |
| BYOL (repro.) | 66.5% | 70.6% | 73.2% | 74.3% | ❌ | ✅ | ✅ (4096) |
| SwAV (repro.) | 66.5% | 69.1% | 70.7% | 71.8% | ❌ | ❌ | ✅ (4096) |
| **SimSiam** | **68.1%** | **70.0%** | **70.8%** | **71.3%** | ❌ | ❌ | ❌ (256) |

**关键发现：**
- SimSiam 在 **100 epoch** 下是**所有方法中最高**的（68.1%）——短训练优势明显
- 长训练后（800 ep：71.3%）落于 BYOL（74.3%）和 MoCo v2（72.2%），但超过了 SimCLR（70.4%）
- SimSiam 的**长训练收益递减**最明显：100→800 ep，SimSiam 提升 3.2%，BYOL 提升 7.8%
- SimSiam 是唯一一个**不需要**任何辅助机制（负样本/动量编码器/大 batch）的方法

### 9.2 迁移学习（VOC 检测 / COCO 检测和分割）

200 epoch 预训练后在下游任务微调（Mask R-CNN / Faster R-CNN）：

| 方法 | VOC07 AP₅₀ | VOC07+12 AP₅₀ | COCO AP | COCO AP₅₀ (mask) |
|------|-----------|-------------|---------|-----------------|
| ImageNet 监督 | 74.4 | 81.3 | 38.2 | 33.3 |
| SimCLR | 75.9 | 81.8 | 37.9 | 33.3 |
| MoCo v2 | 77.1 | 82.3 | 39.2 | 34.3 |
| BYOL | 77.1 | 81.4 | 37.9 | 33.2 |
| SwAV | 75.5 | 81.5 | 37.6 | 33.1 |
| **SimSiam (base)** | **75.5** | **82.0** | **37.9** | **33.2** |
| **SimSiam (optimal)** | **77.3** | **82.4** | **39.2** | **34.4** |

**关键发现：**
- SimSiam (optimal) 在 VOC 和 COCO 检测上达到了**所有方法中最高**（与 MoCo v2 并列）
- **所有自监督方法在迁移学习上都超过或等同于 ImageNet 监督预训练**——这证明了自监督学习在下游任务上的价值
- "optimal"配方（lr=0.5, wd=1e-5）是论文中发现的另一个超参数设置，ImageNet 线性评估与 baseline 相同但迁移更好——说明线性评估不能完全反映表征质量

### 9.3 跨熵相似度实验（Section 4.5）

SimSiam 也可用跨熵替代余弦相似度，不坍塌但准确率从 68.1% 降至 63.2%。这减弱了"防坍塌依赖余弦相似度特性"的猜测，同时建立了一个连接 SwAV 的桥梁——SwAV 正是使用跨熵相似度 + Sinkhorn-Knopp 变换。

### 9.4 EM 假设验证实验（Section 5.2）

**实验 1：多步交替优化**

将 SimSiam 的单步交替扩展为 $k$ 步的内循环：

| 交替步长 | 准确率 |
|---------|--------|
| 1-step（= SimSiam） | 68.1% |
| 10-step | 68.7% |
| 100-step | 68.9% |
| 1-epoch | 67.0% |

多步交替效果更好，说明 SimSiam 确实可以理解为交替优化的一个特例（1-step truncation）。1-epoch 步长性能反而下降，说明过多的内循环更新可能过拟合到固定的 $\eta$。

**实验 2：移动平均替代预测器**

如果无法一步到位逼近期望 $\mathbb{E}_{\mathcal{T}}[\cdot]$，可以用移动平均近似。该方法（不需要预测器 $h$）得 55.0%——相比去掉 $h$ 时的 0.1%，证明 $h$ 的实际作用确实是在补足"期望逼近"的缺口。

### 10.2 核心类：SimSiam

```python
# main_simsiam.py（关键片段）
class SimSiam(nn.Module):
    def __init__(self, backbone, dim=2048, pred_dim=512):
        super().__init__()
        # 编码器 f：backbone + 3 层投影 MLP
        self.encoder = nn.Sequential(
            backbone,              # ResNet-50 -> 2048-d
            # 投影 MLP（3 层，输出带 BN）
            nn.Linear(2048, 2048, bias=False),
            nn.BatchNorm1d(2048),
            nn.ReLU(),
            nn.Linear(2048, 2048, bias=False),
            nn.BatchNorm1d(2048),
            nn.ReLU(),
            nn.Linear(2048, dim, bias=False),
            nn.BatchNorm1d(dim)    # 输出层 BN，无 ReLU
        )
        
        # 预测器 h：2 层瓶颈 MLP
        self.predictor = nn.Sequential(
            nn.Linear(dim, pred_dim, bias=False),  # 2048 → 512
            nn.BatchNorm1d(pred_dim),
            nn.ReLU(),
            nn.Linear(pred_dim, dim)  # 512 → 2048
            # 输出层没有 BN，没有 ReLU
        )
    
    def forward(self, x1, x2):
        # 编码器输出
        z1 = self.encoder(x1)  # [B, 2048]
        z2 = self.encoder(x2)  # [B, 2048]
        
        # 预测器输出
        p1 = self.predictor(z1)  # [B, 2048]
        p2 = self.predictor(z2)  # [B, 2048]
        
        return p1, p2, z1, z2
```

### 10.3 损失函数

```python
def D(p, z):
    """负余弦相似度 + stop-gradient"""
    z = z.detach()  # stop gradient：z 在梯度计算中被视为常数
    p = F.normalize(p, dim=1)  # L2 归一化
    z = F.normalize(z, dim=1)  # L2 归一化
    return -(p * z).sum(dim=1).mean()  # 负余弦相似度

# 对称损失
loss = D(p1, z2) / 2 + D(p2, z1) / 2
```

### 10.4 训练循环

```python
for epoch in range(args.epochs):
    for images, _ in train_loader:
        # 随机增强：同一图像两次
        x1 = aug(images)  # [B, 3, 224, 224]
        x2 = aug(images)  # [B, 3, 224, 224]
        
        # 前向
        p1, p2, z1, z2 = model(x1, x2)
        loss = -(F.normalize(p1, dim=1) * F.normalize(z2.detach(), dim=1)).sum(dim=1).mean() / 2
             - (F.normalize(p2, dim=1) * F.normalize(z1.detach(), dim=1)).sum(dim=1).mean() / 2
        
        # 反向传播
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
```

**这段代码的最重要细节：**

1. **`z.detach()` 的位置**——它放在损失函数 $D$ 内部，而不是在 forward 函数中调用。这意味着编码器的反向路径被选择性阻断。

2. **对称性**——虽然代码中 $p$ 和 $z$ 看起来只计算了两个方向的损失，但注意 $p1 = h(f(x_1))$ 和 $p2 = h(f(x_2))$，所以两次前向共享编码器 $f$ 的全部梯度——每个 view 既贡献预测梯度也贡献目标终止。

3. **无动量编码器，无 EMA**——在训练循环中没有任何对"目标网络"的更新步骤。这就是最简化的 SimSiam。

### 10.5 配置细节

```python
# 优化器：标准 SGD，不是 LARS
optimizer = torch.optim.SGD(
    model.parameters(),
    lr=args.lr,           # base_lr * BatchSize / 256
    momentum=0.9,
    weight_decay=1e-4
)

# 学习率调度：余弦衰减
lr_scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(
    optimizer,
    T_max=args.epochs
)

# 默认 batch size = 512（8 GPU，每 GPU 64）
# base_lr = 0.05，线性缩放
```

配置层级的简洁程度与架构本身的简洁度一致——没有 LARS、没有特殊的 warmup 策略（除了 batch≥1024 时使用 10 epoch warmup）、没有动量编码器超参数——这是自我监督学习代码历史上最简洁的主流程。

## 十一、Related Work 与技术谱系位置

### 11.1 它继承了谁？

**脉络 1：Siamese 网络的基线回归（Bromley et al., 1994）**

SimSiam 是"回归本质"的尝试——回到最原始的 Siamese 网络定义：**共享权重的双分支网络**。所有 2020 年的方法（SimCLR、MoCo、BYOL、SwAV）都在这个基础上加了各种防坍塌机制，SimSiam 逐个拆除后发现 stop-gradient 就够了。

**脉络 2：BYOL 的预测头架构（Grill et al., 2020）**

SimSiam 的直接灵感来源是 BYOL。BYOL 证明了无负样本自监督学习的可行性，而 SimSiam 进一步问了一个更尖锐的问题：**动量编码器是必要的吗？** 答案：不是。只要 stop-gradient 和预测头就够了。

**脉络 3：交替优化（k-means, EM 算法）**

论文的假设部分（Section 5）将 SimSiam 解释为 EM 类算法的实现——$f$ 类似于 k-means 的聚类中心，$\eta$ 类似于分配向量。Stop-gradient 是交替优化的自然结果。

### 11.2 它修正了谁？

SimSiam 修正了 BYOL 的一个隐含假设：**EMA 动量编码器是防止坍塌的必要条件。** BYOL 原文做了"去掉动量编码器"的实验，结果是 0.3%（后纠正为 66.9%），让作者认为动量编码器不可或缺。SimSiam 证明在相同的架构配置下，只保留 stop-gradient 就可以得到 68.1%——高于 BYOL 在同样 epoch 下的 66.5%。

### 11.3 它启发了谁？

```text
非对比自监督学习范式（BYOL → SimSiam）
    │
    ├── DINO（Caron et al., 2021）
    │   ViT + 自蒸馏 + EMA（借用了 stop-gradient 的思想）
    │   将"预测器"替换为"centering + sharpening"
    │
    ├── iBOT（Zhou et al., 2021）
    │   MIM + 在线 tokenizer（同样使用了 stop-gradient 对 tokenizer 做交替）
    │
    ├── DINOv2（Oquab et al., 2023）
    │   大规模蒸馏 + EMA + 多视角
    │
    └── Barlow Twins（Zbontar et al., 2021）
        互相关矩阵约束（不同的防坍塌机制，但同样简单）
```

DINO（2021）直接继承了 SimSiam 的"stop-gradient + 预测器"核心设计，但对齐到 ViT 架构，并增加了 centering 和 sharpening 两个操作来替代预测器的非线性映射。可以说没有 SimSiam 在 ResNet 上证明 stop-gradient 的充分性，DINO 在 ViT 上的设计会缺少理论支撑。

### 11.4 它是主干路线还是旁支路线？

SimSiam 属于 **主干路线的关键节点**。它虽然不是性能最高（800 ep 的 71.3% 低于 BYOL 的 74.3%），但它的**概念简洁性**和 **"组件分析"的方法论**对后来的工作产生了巨大影响。它把自监督学习的研究焦点从"怎么工程调整防坍塌"转移到了"什么才是最本质的机制"。

### 11.5 它在整棵树上的位置

```text
B4. 是否可以形成统一视觉 encoder 基座模型？
  ├── B4-1. contrastive representation learning
  │   ├── SimCLR（2020）
  │   └── MoCo（2020）
  │
  └── B4-2. non-contrastive representation learning  ← ✦
      ├── BYOL（2020）—— EMA + 预测器    
      ├── SimSiam（2020）—— stop-gradient + 预测器（无 EMA） ← 本篇
      ├── DINO（2021）—— ViT + 自蒸馏
      ├── iBOT（2021）—— MIM + 在线 tokenizer
      └── DINOv2（2023）—— 大规模蒸馏
```

## 十三、范式意义

### 13.1 这属于什么类型的贡献？

SimSiam 的贡献可以定位为 **机制发现 + 算法简化**。

**机制发现**：发现了 stop-gradient 是防止 Siamese 网络坍塌的**充分必要条件**（在预测器存在的前提下）。其他所有防坍塌机制——负样本、动量编码器、在线聚类——都是**工程冗余**。

**算法简化**：将自监督学习的训练流程从"多个组件组合"简化为"编码器 + 预测器 + stop-gradient"三件套。这个简化不是表演性质的——它证明了核心的归纳偏置来自 Siamese 架构本身，而不是围绕它搭建的防坍塌脚手架。

### 13.2 具体改变了什么？

**改变 1：自监督学习的"最小基线"被重新定义**

在 SimSiam 之前，自监督学习的"最小可工作基线"至少需要负样本（SimCLR）或 EMA（BYOL）。此后，"简单 Siamese + stop-gradient"成了 60%+ 的底线——如果你连这个都做不到，说明你的设计有问题。后续所有方法都会做这个消融。

**改变 2：对"为什么能防坍塌"的思考方向被改变**

SimSiam 的 EM 假设虽然不是严格的数学证明，但它提供了一个**概念框架**：stop-gradient = 交替优化。这个框架在 DINO（centering）、iBOT（tokenizer 冻结）等后续工作中都被借用。

**改变 3：一种"减一组件"的研究方法论**

## 十四、局限性与后续方向

### 14.1 长训练收益递减

SimSiam 从 100 到 800 epoch 提升 3.2%，而 BYOL 提升 7.8%。可能的原因是缺少 EMA 提供的平滑更新。EMA 在长训练中相当于一个"低通滤波器", 表征质量随时间平滑提升, 而 SimSiam 的单步交替优化可能更容易进入局部停滞。

**后续工作：** DINO 重新引入了 EMA，但将预测器替换为更稳健的 centering + sharpening 机制, 在 ViT 上取得了优异的 scaling 性能。

### 14.2 EM 假设未完全验证

论文的假设（Section 5）是启发性而非证明性的。虽然多步交替实验支持该假设，但它没有解释为什么交替优化能防坍塌而联合优化不能。这需要更严格的优化理论分析。

**后续工作：** Tian et al. (2021, "Understanding self-supervised learning dynamics without contrastive pairs") 提供了一些理论见解，证明 SimSiam 的预测器 + stop-gradient 等价于了一个隐式的局部对比性目标。

### 14.3 仍然需要预测器和数据增强

SimSiam 虽然不需要负样本/动量编码器，但仍然依赖：
- **预测器 $h$**——去掉它即坍塌（0.1%）
- **精心设计的数据增强**：尽管论文没有系统地消融增强策略，但增强质量仍然影响结果

这意味着 SimSiam 的"简单"是相对于已有方法而言，绝对意义上它仍然有一组不可缺的组件。

### 14.4 在 ViT 上的直接迁移可能需要调整

SimSiam 的设计是在 ResNet-50 上验证的。后来的 DINO 发现，简单的 SimSiam 在 ViT 上效果不如在 CNN 上好，需要额外的 centering 和 sharpening 来防止 ViT 的 attention 坍塌。

**潜在原因：** ResNet 的卷积结构天然具有局部性和空间层次性，这提供了额外的归纳偏置。ViT 的全注意力缺乏这种偏置，更容易坍塌——所以在 ViT 上需要额外的约束。

### 14.5 理论优于实践的微妙定位

SimSiam 的**概念重要性**大于其**实际应用价值**。在 800 epoch 下，BYOL（74.3%）和 MoCo v2（72.2%）的 ImageNet 线性评估都高于 SimSiam（71.3%），迁移学习上三者基本持平。因此实际工程中很难说"用 SimSiam 代替 BYOL/MoCo"有明显的收益。它的价值在于**教给我们了一个教训**：核心的归纳偏置来自 Siamese 架构，工程冗余不是必要的。
