## 一、先搞清楚"问题在哪里"

2019 年，视觉语言（V+L）领域出现了一个新趋势：不再为每个下游任务（VQA、图像检索、指代表达等）从头训练模型，而是走"预训练 + 微调"的路线，就像 BERT 在 NLP 中做的那样。

这个想法听起来很自然——既然 BERT 可以在纯文本上预训练一个通用的文本表征，那能不能也在图文数据上预训练一个通用的多模态表征？

但问题在于，图像和文本是两种完全不同的模态。文本是离散的 token 序列，图像是连续的像素空间。**"通用多模态表征"需要解决一个核心难题：图像和文本如何进入同一个表征空间，使得跨模态的语义对齐成为可能？**

2019 年上半年，ViLBERT（Lu et al.）和 LXMERT（Tan & Bansal）率先迈出了第一步。它们的思路是：保留两个独立的 Transformer encoder（一个处理文本、一个处理图像），只在中间层用 co-attention 做跨模态交互。这就是"双流架构"——文本流和图像流各自独立，通过注意力机制互相"看"对方。

这个方案有效，但有两个问题：

1. **双流的参数更多。** 两个 encoder 各自有一套独立的参数，模型大小翻倍。
2. **跨模态交互的深度有限。** co-attention 只在特定层发生，两边的信息融合不够充分。

那么，能不能换个思路——**把图像和文本直接塞进同一个 Transformer，让它们在一个统一的 encoder 里自己学习怎么交互？**

这就是 UNITER 要回答的问题。

## 二、旧路线的真正问题

在 UNITER 之前，ViLBERT 和 LXMERT 代表了"双流（two-stream）"路线。这个设计选择不是偶然的——它的直觉基础是：文本和图像是不同模态，理应由不同的 encoder 分别处理，然后在高层融合。

这个直觉有道理，但也带来了结构性的问题。

### 2.1 双流的冗余

ViLBERT 用了两个独立的 BERT 架构（12 层文本 encoder + 6 层视觉 encoder），LXMERT 则是三个 Transformer（文本 encoder + 视觉 encoder + 跨模态 encoder）。这些 encoder 的参数是独立的，不能共享。

从参数效率来看，双流意味着每个模态都在"重新学习"Transformer 的基本能力——self-attention、position encoding、feed-forward——而这些能力本质上是模态无关的。如果你相信 Transformer 的 self-attention 可以处理任意序列数据（无论文本还是图像），那么理论上一个统一的 Transformer 就够用了。

### 2.2 跨模态交互的局限

在双流架构中，跨模态交互通过 co-attention 完成。以 ViLBERT 为例，文本侧的 key/value 被送到视觉侧的 attention 层，反之亦然。但 co-attention 只发生在特定的"交互层"，两个流的大部分层在独立处理自己的模态。

这意味着：**大部分网络容量被用在了单模态表征上，跨模态的表征只在少数层被学习。** 而视觉语言任务（如 VQA 中的"那个红色的球是什么？"）的核心恰恰是跨模态的理解——模型需要把"红色的"和图像中特定的区域联系起来。

### 2.3 预训练任务的设计空间

更重要的是，双流架构限制了预训练任务的设计。

ViLBERT 用了两个预训练任务：Masked Language Modeling（MLM，遮住文本词，用图像信息预测）和 Image-Text Matching（ITM，判断图文是否配对）。LXMERT 增加了 Masked Region Modeling（MRM，遮住图像区域，用文本信息预测）。

但这些工作对"遮罩策略"没有足够深入地探索。比如：遮住文本词的时候，模型是否应该完全看到图像？遮住图像区域的时候，模型是否应该完全看到文本？这就是 "conditional masking" 和 "joint random masking" 之间的选择。

**ViLBERT 和 LXMERT 都用了联合随机遮罩（同时遮住两边的部分 token），但 UNITER 论文将会证明，条件遮罩（遮一边时完全保留另一边）的效果更好。**

### 2.4 细粒度对齐的缺失

现有的预训练任务都只做了"全局对齐"——ITM 只判断整张图是否匹配整句话，没有机制来学习"word"和"region"之间的细粒度对应关系。

比如对于句子 "一个男人牵着一条狗在公园散步"，模型需要知道 "男人" 对应图像中的人的区域，"狗" 对应狗的区域，"公园" 对应背景。**这种词-区域的对齐是许多下游任务（如指代表达、视觉推理）的基础，但没有被显式地建模。**

## 三、这篇论文的核心思路

**这篇论文的核心思路是：用单流（single-stream）架构取代双流（two-stream）架构，用条件遮罩代替联合随机遮罩，并引入基于最优传输（Optimal Transport）的词-区域对齐任务。**

具体来说，UNITER 做了三个关键设计决策：

**1. 单流 Transformer：** 图像和文本进入同一个 Transformer encoder。图像 region 特征 + 位置编码 → linear projection 到和文本词 embedding 相同的空间，然后和文本 token 拼接在一起送入一个统一的 Transformer。

**2. 条件遮罩（Conditional Masking）：** 做 MLM 时，只遮住文本 token，图像完全可见；做 MRM 时，只遮住图像 region，文本完全可见。这迫使模型在遮住一边时，主动去"读"另一边的信息。

**3. 四种预训练任务：** MLM（遮词预测）+ MRM（遮区域预测，含三个变体）+ ITM（图文匹配）+ WRA（词-区域对齐，基于最优传输）。其中 WRA 是 UNITER 的独特贡献——通过最优传输的距离来显式地学习词和区域的细粒度对齐。

整个架构可以简单理解为：

```text
图像 region (Faster R-CNN) → ROI 特征 + 位置编码 → linear projection
文本 token → BERT tokenizer → embedding lookup
        ↓
    拼接 → [CLS] tok1 tok2 ... [SEP] reg1 reg2 ... [SEP]
        ↓
   Transformer encoder (12层 / 24层)
        ↓
   MLM head / MRM head / ITM head / WRA loss
```

这条技术路线的谱系位置是：

```text
BERT (2018, pure text pretraining)
   ↓
ViLBERT/LXMERT (2019, 双流跨模态 encoder)
   ↓
UNITER (2019, 单流跨模态 encoder) ← 本篇
   ↓
Oscar / VinVL / ALBEF (2020+, 单流 + 检测标签 / 单流 + 对比学习)
```

UNITER 和 ViLBERT/LXMERT 的最大区别就是单流 vs 双流。这个选择让 UNITER 的参数量更小、推理速度更快，而且在 6 个下游任务上实现了当时的最优结果。

## 四、整体结构与 forward 流程

### 4.1 文本端输入

```
句子: "a dog is running on the grass"
→ BERT tokenizer → ["[CLS]", "a", "dog", "is", "running", "on", "the", "grass", "[SEP]"]
→ embedding lookup + 位置编码 + 段落编码
→ 文本 embedding: N_text × d  (d=768 for base)
```

文本端的处理和标准 BERT 完全一致：[CLS] 放在开头、[SEP] 放在结尾、learned 位置编码 + token type embedding。

### 4.2 视觉端输入

这是 UNITER 和 ViLBERT 差异最大的部分。

**ViLBERT 的做法：** Faster R-CNN 提取 ROI 特征 → 视觉端单独用一个 attention 层来处理，然后通过 co-attention 和文本交互。

**UNITER 的做法：** Faster R-CNN 提取 ROI 特征 → linear projection + 位置编码 → 直接拼接到文本序列中，所有 token（文本 + 图像）一起过同一个 Transformer。

具体来说，视觉端的输入由两部分组成：

**ROI 特征（visual features）：** 用预训练的 Faster R-CNN 检测图像中的物体区域（object region）。每张图像检测 36 个 ROI（自适应采样，实际数量可变，通过 padding 到固定长度）。每个 ROI 经过 RoI Pooling 后输出 2048 维的特征向量，然后通过一个 linear projection 降维到 d=768，与文本 embedding 对齐。

**位置编码（positional encoding）：** 每个 ROI 的边界框 (x1, y1, x2, y2) 归一化到 [0,1]，再加上宽度和高度，构成 7 维特征。这个 7 维向量通过一个 linear projection 映射到 d 维，然后和 ROI 特征相加（或拼接，取决于实现版本）。

```python
# 视觉端伪代码
roi_features = faster_rcnn(image)  # [36, 2048]
roi_boxes = format_bbox(x1, y1, x2, y2)  # [36, 7] 或 [36, 5]

vis_embed = Linear(2048, d)(roi_features)  # [36, d]
pos_embed = Linear(7, d)(roi_boxes)        # [36, d]
vis_input = vis_embed + pos_embed          # [36, d]
```

然后，文本 embedding 和视觉 embedding 被拼接：

```python
# 单流拼接
input_embed = [
    text_embedding  # [L_text, d]
    + position_embedding_text               # 文本位置编码
    + segment_embedding_text,               # 文本段 ID=0

    vis_input,                              # [L_vis, d]
]  # 整体: [L_text + L_vis, d]

# text segment = 0, visual segment = 1
segment_ids = [0] * L_text + [1] * L_vis
```

### 4.3 整体架构

UNITER 使用单流 Transformer，base 版本为 12 层、768 维、12 头；large 版本为 24 层、1024 维、16 头。

完整的前向流程：

```text
输入图像 + 输入文本
    │
    ├── 文本路径: BERT tokenizer → embedding lookup
    │   → token_embed [L_text, d]
    │
    └── 视觉路径: Faster R-CNN → ROI features [36, 2048]
        → linear(2048, d) + position encoding → vis_embed [36, d]
    │
    ↓
[CLS] tok1 tok2 ... [SEP] reg1 reg2 ... [SEP]
    │
    ↓
Transformer encoder (12× / 24×)
    │
    ↓
Encoder output: [N_total, d]
    │
    ├── [CLS] → MLP → ITM 分类头 (是否匹配)
    ├── 每个 text token → MLP → MLM 分类头 (预测遮住的词)
    ├── 每个 vis region → MLP → MRM 分类头 (预测遮住的 region)
    └── word-region 对齐 → WRA loss (最优传输)
```

### 4.4 UNITER-base vs UNITER-large

| 配置 | Base | Large |
|------|------|-------|
| Transformer 层数 | 12 | 24 |
| 隐层维度 | 768 | 1024 |
| Attention 头数 | 12 | 16 |
| 参数总量 | ~86M | ~300M |
| 视觉 ROI | 36 个/padding | 36 个/padding |
| 文本长度 | 最多 60 token | 最多 60 token |

## 五、核心模块逐个解释

### 5.1 单流 Transformer（Single-Stream vs Two-Stream）

这是 UNITER 最核心的设计决策。

**单流的直觉：** 文本和图像的 token 本质上都是"序列元素"，Transformer 的 self-attention 可以处理任意类型的序列。与其让两个独立的 encoder 先各自处理再通过 co-attention 融合，不如让所有 token 在同一个 Transformer 里直接交互。

**单流的好处：**
- 参数更少（只需要一个 Transformer）
- 交互更充分（每一层 attention 都是跨模态的）
- 推理更快（一个前向传播 vs 两个或三个）

**单流的担心：** 模态之间的"gap"会不会让训练不稳定？毕竟文本是离散语义单元，图像 region 是连续视觉特征。

UNITER 的实验结果表明，这个担心是多余的——单流的效果至少不差于双流（在论文的 6 个下游任务上全面超越当时的 SOTA）。

实际上，**单流在跨模态交互的深度上天然优于双流。** 在双流架构中，文本和图像只在特定层做 co-attention；而在单流架构中，每一层的 self-attention 都在做跨模态交互。这意味着跨模态信息可以在所有层之间自由流动。

### 5.2 条件遮罩（Conditional Masking）

这是 UNITER 和 ViLBERT/LXMERT 在预训练细节上的关键区别。

假设输入是：文本 `[CLS] a dog is [MASK] on the grass [SEP]` + 图像 ROI。

**ViLBERT/LXMERT 的做法（联合随机遮罩）：**
- 文本遮罩率 15%
- 图像区域遮罩率 15%（随机）
- 遮罩是独立进行的——遮住文本 token 时，部分图像 region 也被遮住了

**UNITER 的做法（条件遮罩）：**
- 做 MLM 时：遮住 15% 的文本 token，**但所有图像 region 全部可见**
- 做 MRM 时：遮住 15% 的图像 region，**但所有文本 token 全部可见**

条件遮罩迫使模型在预测被遮住的内容时，必须依赖另一模态的完整信息。

```text
条件遮罩 — MLM:
  文本: [CLS] a [MASK] is running on the [MASK] [SEP]
  图像: reg1 reg2 reg3 ... reg36 (全部可见)
  → 模型需要利用完整图像信息来预测 "dog" 和 "grass"

条件遮罩 — MRM:
  文本: a dog is running on the grass (全部可见)
  图像: reg1 [MASK] reg3 ... [MASK] ... reg36
  → 模型需要利用完整文本信息来预测遮住 region 的语义
```

**直觉上，条件遮罩更合理：** 预训练的目标是让模型学会"跨模态理解"，而不是"猜谜"。如果两边都被遮住了，模型只能靠单模态的信息来做预测——那不是真正的跨模态学习。

### 5.3 MLM（Masked Language Modeling）

标准 BERT 的 MLM 任务。随机遮住 15% 的文本 token（其中 80% 替换为 [MASK]、10% 替换为随机 token、10% 保持不变），模型通过 Transformer 输出的编码来预测原始 token。

与 BERT 的唯一区别是：预测时模型可以同时看到图像 region 的信息。这使得模型的预测能力超越了纯语言模型——它不仅能根据上下文猜词，还能根据图像内容猜词。

比如对于 "a [MASK] is running on the grass"，如果图像中是一只狗，模型可以输出 "dog"；如果图像中是一匹马，模型输出 "horse"。

### 5.4 MRM（Masked Region Modeling）

Masked Region Modeling 是 MLM 的图像版本——随机遮住 15% 的图像 region（将特征替换为零向量或用 [IMG] embedding 替换），模型需要从编码中恢复原始 region 信息。

UNITER 提出了 MRM 的三种变体：

**MRFR（Masked Region Feature Regression）：** 回归原始 ROI 特征。目标函数是 L2 距离（均方误差），模型直接预测被遮住 region 的 2048 维 Faster R-CNN 特征。

$$L_{\text{MRFR}} = \mathbb{E}_{(w, v) \sim D} \sum_{i \in M} ||f_{\text{dec}}(h_i) - v_i||_2^2$$

其中 $M$ 是被遮住的 region 集合， $h_i$ 是 region $i$ 的 Transformer 输出， $f_{\text{dec}}$ 是一个 MLP 解码器（2048-d 输出）， $v_i$ 是原始 ROI 特征。

**MRC-kl（Masked Region Classification with KL-Divergence）：** 从检测器（Faster R-CNN）的 soft label 中蒸馏物体类别信息。Faster R-CNN 对每个 region 输出一个类别分布（经过 softmax 的 logits），MRC-kl 让模型预测的类别分布和这个分布做 KL 散度最小化。

$$L_{\text{M RC-kl}} = \mathbb{E}_{(w, v) \sim D} \sum_{i \in M} D_{\text{KL}}(c_i^{\text{detector}} || c_i^{\text{model}})$$

其中 $c_i^{\text{detector}}$ 是 Faster R-CNN 对该 region 预测的类别分布， $c_i^{\text{model}}$ 是模型输出的分布。

**MRC（Masked Region Classification）：** 硬标签版本——使用 Faster R-CNN 的 argmax 类别作为 ground truth，做标准的交叉熵分类。这个变体在消融中效果比 MRC-kl 略差（可能是因为硬标签丢失了分布的不确定性信息）。

**直觉：** MRM 让模型学习的不仅仅是"某个区域被遮住了"，而是"被遮住的区域应该是怎样的视觉特征 / 怎样的物体类别"。这迫使模型从文本线索（"running on the grass" → 这个 region 可能是"狗"或"猫"）推断视觉内容。

### 5.5 ITM（Image-Text Matching）

标准的图文匹配二分类任务。输入一个图文对，模型输出匹配（positive）或不匹配（negative）的概率。

关键设计：使用 [CLS] token 的最终编码作为全局图文表征输入二类分类器。

负样本构建策略：从训练集中随机采样不匹配的图文对（negative batch），正负比例 1:1。这是最基础的 hard negative mining 策略（用随机采样而非精心选择的 hard negative）。

ITM 的位置：ITM loss 的权重在所有预训练任务中最大（λ_ITM > λ_MLM > λ_MRM），因为匹配/不匹配的判别是最基础的跨模态理解能力。

### 5.6 WRA（Word-Region Alignment，基于最优传输）

这是 UNITER 的独特贡献——**用最优传输（Optimal Transport, OT）来显式地学习词和图像 region 之间的细粒度对齐。**

**为什么需要 WRA？**

ITM 只做了全局对齐——判断整张图和整句话是否匹配。模型不需要知道哪个词对应哪个 region，只要能区分全局"是否匹配"就行。

但很多任务需要细粒度的词-区域对齐。比如指代表达（Referring Expression Comprehension）："坐在右边的穿红衣服的女人"——模型需要知道 "右边"、"红衣服"、"女人" 各自对应图像中的哪个 region。

**最优传输的思路：**

把一个图文对中的文本词和图像 region 看作两个概率分布。假设我们有一堆"词"和一堆"region"，词-区域对齐的本质是"如何把词的信息传输到 region 空间"。

具体来说：

1. **文本侧：** 每个词 token 有一个 embedding（来自 BERT 编码器输出），归一化为概率分布（所有词的概率和为 1）。

2. **图像侧：** 每个 region 有一个 embedding，同样归一化。

3. **传输成本：** 定义一个成本矩阵 $C$ ，其中 $C_{ij} = 1 - \cos(e_i^{\text{word}}, e_j^{\text{region}})$ ——余弦相似度越小，成本越高。

4. **最优传输：** 找到一个传输方案（transport plan） $T \in \mathbb{R}^{N_{\text{word}} \times N_{\text{region}}}$ ，使得从文本侧传输到图像侧的总成本最小，且满足边际约束（行和 = 文本分布、列和 = 图像分布）。

$$L_{\text{WRA}} = \text{OT}(p_w, p_v) = \min_{T \in \Pi(p_w, p_v)} \sum_{i,j} T_{ij} C_{ij}$$

这个最小值是 Wasserstein distance（推土机距离，earth mover's distance）。

**为什么最优传输有效？**

最优传输的解 $T^*$ 天然是一个对齐矩阵—— $T^*_{ij}$ 表示词 $i$ 和 region $j$ 之间的"传输量"，也就是对齐强度。

对于匹配的图文对，最优传输距离应该很小（词和 region 可以很好地对齐）；对于不匹配的图文对，最优传输距离应该很大（词找不到对应的 region，或者对齐成本很高）。

**实现细节：**

UNITER 使用 IPOT（Inexact Proximal point method for Optimal Transport）算法近似求解 OT，因为精确的 OT 求解是 O(n³) 的（匈牙利算法），而 IPOT 可以在 O(n²) 内近似。

IPOT 是一个迭代算法（论文中迭代 20 次），每次迭代更新传输矩阵 $T$ ：

$$T^{(k+1)} = \text{diag}(\mathbf{u}^{(k)}) K_T^{\odot} \text{diag}(\mathbf{v}^{(k)})$$

其中 $K_T = \exp(-C/\tau)$ （ $\tau$ 是温度系数）， $\mathbf{u}$ 和 $\mathbf{v}$ 通过 Sinkhorn-like 迭代得到。

**WRA 的训练技巧：**

- WRA 只对同一 batch 内的正样本计算 OT loss（不匹配的图文对不计算）
- WRA loss 的权重比 ITM 和 MLM 小，作为辅助任务
- 需要和 ITM 一起训练（论文中两个变体："ITM+WRA" 和 "ITM然后WRA"）

### 5.7 预训练任务的组合策略

UNITER 的一个重要贡献是通过消融实验找到了最佳的预训练任务组合。论文探索了多种组合：

| 组合 | 任务 |
|------|------|
| 基线 | MLM + ITM |
| +MRM | MLM + MRM + ITM |
| +WRA | MLM + ITM + WRA |
| 全任务 | MLM + MRM + ITM + WRA |
| ITM→WRA | MLM + MRM + ITM（先训练），然后 WRA（后训练） |
| ITM+WRA | MLM + MRM + (ITM+WRA 同时) |

最终的最佳组合是 **MLM + MRFR + MRC-kl + ITM+WRA**。

### 5.8 预训练数据集

UNITER 在四个图文数据集上预训练：

| 数据集 | 图文对数量 | 特点 |
|--------|-----------|------|
| COCO | 113K 图像，5 个描述/图像 | ~565K 对 |
| Visual Genome | 108K 图像，~50 个区域描述 | ~5.4M 对 |
| Conceptual Captions | ~3.3M 图像 | 网络爬取，压缩描述 |
| SBU Captions | ~860K 图像 | 基于 Flickr 数据构造 |

预训练总共使用了约 1000 万图文对。

**预训练任务组合的消融（见论文 Table 2）：**

- 组合所有任务达到最佳效果

**条件遮罩 vs 联合随机遮罩（见论文 Table 3）：**

| 遮罩策略 | MLM | MRM |
|---------|-----|-----|
| 条件遮罩 | ✓ 更好 | 更好 ✓ |
| 联合随机遮罩 | ✗ 差 | 差 ✗ |

具体来说：在 MLM 任务中，条件遮罩比联合随机遮罩高 `Table 3 展示了下游任务精度` 个点。在 MRFR 和 MRC-kl 等任务上条件遮罩也一致优于联合随机遮罩。这验证了论文的核心论点——**遮罩时保留另一模态的全部信息，比两边都遮住更有利于跨模态学习。**

**单流 vs 双流对比（见论文 Table 4）：**

在相同预训练设置下（预训练数据相同、任务相同），UNITER 的单流架构和带有共注意力的双流变体（UNITER-two-stream）对比：

|------|-----|-------|---------|
| 双流（UNITER-two-stream） | 72.5 | 78.6 | 69.9 |
| 单流（UNITER） | **72.8** | **77.85** | **72.5** |

单流在所有任务上一致优于双流，而且参数量更少、速度更快。这说明：**对 V+L 任务来说，单流比双流更有效。**

**WRA 消融（见论文 Table 5）：**

在 Flickr30K 检索上，WRA 的具体贡献：

| 设置 | 图文检索 R@1 | 图-文检索 R@1 |

OT-based WRA 显著优于简单的余弦相似度对齐（+1.9/+1.3），证明 OT 的全局优化（考虑所有词-区域对的匹配）确实比点对点的相似度更有效。

## 七、核心公式

### 7.1 总损失函数

$$L = \lambda_{\text{MLM}} L_{\text{MLM}} + \lambda_{\text{MRFR}} L_{\text{MRFR}} + \lambda_{\text{MRC-kl}} L_{\text{MRC-kl}} + \lambda_{\text{ITM}} L_{\text{ITM}} + \lambda_{\text{WRA}} L_{\text{WRA}}$$

权重配置（论文中最佳的配置）：
- λ_MLM = 1.0
- λ_MRFR = 1.0
- λ_MRC-kl = 1.0
- λ_ITM = 5.0（ITM 的权重最大，因为匹配/不匹配的判别是跨模态学习的基础）
- λ_WRA = 1.0

### 7.2 最优传输（OT）公式

给定词 embedding 矩阵 $E_{\text{word}} \in \mathbb{R}^{N_{\text{word}} \times d}$ 和 region embedding 矩阵 $E_{\text{region}} \in \mathbb{R}^{N_{\text{region}} \times d}$ ：

1. 归一化得到概率分布：
   $$p_w = \text{SoftMax}(W_w E_{\text{word}})$$
   $$p_v = \text{SoftMax}(W_v E_{\text{region}})$$

2. 成本矩阵：
   $$C_{ij} = 1 - \cos(\mathbf{E}_{\text{word}}^{(i)}, \mathbf{E}_{\text{region}}^{(j)})$$

3. 最优传输距离（Wasserstein-1）：
   $$L_{\text{WRA}} = \min_{T \in \Pi(p_w, p_v)} \langle T, C \rangle = \sum_{i,j} T_{ij} C_{ij}$$

4. 通过 IPOT 迭代求解 $T$ ，固定 20 次迭代。

### 7.3 MRFR 和 MRC-kl 损失

MRFR（回归）：
$$L_{\text{MRFR}} = \frac{1}{|M|} \sum_{i \in M} ||h_{\text{MRFR}}(h_i) - v_i||_2^2$$

MRC-kl（KL 散度）：
$$L_{\text{MRC-kl}} = \frac{1}{|M|} \sum_{i \in M} \sum_{k=1}^{K} c_{i,k}^{\text{detector}} \log \frac{c_{i,k}^{\text{detector}}}{c_{i,k}^{\text{model}}}$$

其中 $K$ 是检测器类别数（Faster R-CNN 的 1600 个物体类别）。

## 八、Related Work 与技术谱系位置

### 8.1 技术树中的位置

```text
BERT (Devlin et al., 2018) — 纯文本预训练
   │
   ├── ViLBERT (Lu et al., 2019) — 双流跨模态，2019
   │      │
   │      └── 影响: UNITER, ALBEF, VLMo
   │
   ├── LXMERT (Tan & Bansal, 2019) — 三流跨模态，2019
   │      │
   │      └── 影响: UNITER（具体任务对比）
   │
   ├── VisualBERT (Li et al., 2019) — 单流跨模态，2019
   │      │
   │      └── 类似架构，但预训练任务更少、数据更少
   │
   └── UNITER (Chen et al., 2019) — ★ 本篇 ★ 单流 + OT-based WRA
          │
          ├── Oscar (Li et al., 2020) — 加入检测标签作为锚点
          ├── VinVL (Zhang et al., 2021) — 更好的视觉 backbone
          ├── ALBEF (Li et al., 2021) — 对比学习 + momentum distillation
          └── VLMo (Bao et al., 2021) — 单流/双流混合（MoE）
```

### 8.2 前驱工作

**BERT（Devlin et al., 2018）：** 所有 V+L 预训练工作的起源。MLM 和 NSP（Next Sentence Prediction）的思路被直接继承——MLM 变成了跨模态版的 MLM+MRM，NSP 变成了 ITM。

**ViLBERT（Lu et al., 2019）：** 第一个将 BERT 扩展到视觉语言的预训练工作。提出了双流架构（文本 BERT + 视觉 BERT，通过 co-attention 交互）和两个预训练任务（MLM + ITM）。UNITER 继承了 MLM 和 ITM 的设计，但用单流替代了双流，增加了 MRM 和 WRA。

**LXMERT（Tan & Bansal, 2019）：** 三流架构（文本 encoder + 视觉 encoder + 跨模态 encoder）。首次引入了 MRM 预训练任务。UNITER 将 LXMERT 的三流简化为单流，并将 MRM 加以扩展（MRFR + MRC-kl）。

**VisualBERT（Li et al., 2019）：** 和 UNITER 几乎同时期的单流工作，但预训练数据更少（只用 COCO）、预训练任务更少（只有 MLM + ITM）、没有引入 WRA。

### 8.3 同期竞品

**ViLBERT（2019）：** 双流的代表。参数量更大、速度更慢。在 UNITER 的对比中，单流的一致优于双流。

**LXMERT（2019）：** 三流的代表，参数量最大。在大部分任务上被 UNITER 超越，但在 VQA 上两者最为接近（LXMERT 72.5% vs UNITER-base 72.8%）。

**VisualBERT（2019）：** 最朴素的单流方案。效果比 UNITER 差很多，主要差在预训练任务的设计（没有 MRM、WRA）和预训练数据的规模。

### 8.4 后续影响

**Oscar（Li et al., 2020, 发表于 2022 TPAMI）：** 在 UNITER 的基础上，为每个图像 region 加上检测标签（object tag）作为"锚点"。标签既和文本对齐、又和图像 region 对齐，相当于在文本和图像之间加了一个"中间层"字典。VQA 上达到 73.6%（base）/ 76.9%（large）[see paper for exact numbers]。

**VinVL（Zhang et al., 2021）：** 用更好的视觉 backbone（更大的 Faster R-CNN + 更大的检测数据集）替换了 UNITER 的视觉编码器。这个改进的主要信号：**视觉端的质量可能是比预训练任务设计更重要的瓶颈。**

**ALBEF（Li et al., 2021）：** 引入了动量蒸馏（momentum distillation）和对比学习。在单流基础上融合了双流的优点（图像和文本先做对齐再进融合 encoder）。

**VLMo（Bao et al., 2021）：** 用 MoE（Mixture-of-Experts）在单流和双流之间切换。部分专家处理自己的模态，部分专家做跨模态融合。

**SOHO（Huang et al., 2020）：** 提出了"Seeing Out of the Box"——端到端学习视觉 token，避免对预训练检测器的依赖。

## 九、范式意义

**UNITER 的范式意义：它是早期跨模态 encoder 范式中"单流路线"的最强验证。**

具体来说，UNITER 完成了三件事：

### 9.1 验证了单流架构的可行性

在 UNITER 之前，社区的主流直觉是"视觉和语言是不同的东西，需要不同的 encoder 来处理"。UNITER 用实验证明：**统一的 Transformer encoder 可以同时处理两个模态，而且效果更好。**

这个结论的核心意义在于：它意味着 Transformer 确实是"模态无关"的——只要你能把视觉信息也 token 化、embedding 化，Transformer 就能处理。这对后续的 all-in-one 架构（OFA、Flamingo、Kosmos-1 等）产生了深远影响。

### 9.2 回答了"预训练任务该怎么设计"的问题

UNITER 系统性地对比了不同预训练任务的组合，得出的结论是：

1. **条件遮罩 > 联合随机遮罩。** 这个结论不仅适用于 UNITER 自身，也影响了后续几乎所有 V+L 预训练工作的遮罩策略。

2. **细粒度对齐（WRA）和全局对齐（ITM）是互补的。** 两者同时训练时，效果比单独使用任何一个都好。

3. **回归型 MRM（MRFR）+ 分类型 MRM（MRC-kl）的组合比任何一个单独使用都有效。** 这说明视觉预训练任务需要同时覆盖"特征级"和"语义级"的信息。

### 9.3 将 OT（最优传输）引入了 V+L 领域

UNTITER 是第一篇将最优传输用于词-区域对齐的 V+L 论文。这个想法后来被 Unicoder-VL、SOHO 等后续工作继承和发展。

OT 的核心优势：**它不像余弦相似度那样只比较单个词-区域对，而是全局求解最优的"词→区域"的传输方案。** 这个全局优化的性质让 WRA 能更好地捕捉语义对应关系（比如 "dog is running" 应该对应多个 region 的区域组合，而不是独立的区域）。

### 9.4 分类：foundation node

在技术树中，UNITER 是一个 **foundation node（基础节点）**。它与 ViLBERT、LXMERT 一起，共同定义了"跨模态 encoder 预训练"这个技术方向。UNITER 在单流路线上的成功为后续几乎所有单流 V+L 模型（Oscar、VinVL、ALBEF、VLMo）奠定了基础。

它与 ViLBERT 的关系是"竞争并最终胜出的路线"。在今天回头看，单流已经成为了 V+L encoder 预训练的主流选择。

## 十、局限性与后续方向

### 10.1 对预训练检测器的依赖

UNITER 严重依赖 Faster R-CNN 的检测输出。这意味着：

1. **视觉信息的上限受限于检测器。** 如果检测器漏检了某个物体，UNITER 就永远看不到它。
2. **检测器本身的训练数据（Visual Genome 的 1600 类）限制了模型能识别的物体范围。** 超出这 1600 类的物体（如罕见的动物、工业设备）就检测不到。
3. **Faster R-CNN 是单独训练的，和 UNITER 不是 end-to-end。** 整个 pipeline 是两阶段的。

后续工作（如 SOHO、VinVL）尝试解决这个问题：SOHO 用 grid feature 替代 region feature（避免检测），VinVL 换用更强的检测器。

### 10.2 非端到端训练

视觉编码器和文本编码器分开训练（Faster R-CNN 是预训练的，固定不更新）。这意味着跨模态学习只在 Transformer 部分发生，视觉 backbone 的表征不能被微调。如果 Faster R-CNN 提取的 ROI 不够好，Transformer 也只能将就。

### 10.3 预训练成本

在 4 个数据集（约 10M 图文对）上预训练 UNITER-base 需要相当多的计算资源（论文没有报具体时间，但类似规模的模型在 2019 年需要约 8-16 个 V100 GPU × 数天）。

### 10.4 模态数量限制

UNITER 只支持两个模态（图像 + 文本）。如果加入视频、音频、3D 等更多模态，单流架构的序列长度会迅速增长，带来计算和效率问题。

### 10.5 生成能力

UNITER 是一个纯 encoder 模型，没有 decoder。它只能做理解类任务（VQA、分类、检索），不能生成图像描述或对话。这个问题在后续的 BLIP、OFA 等工作得到了解决。

## 十一、小结

UNITER 在 2019 年这个时间点，做了几件重要的事：

1. **简洁就是力量。** 用一个统一的 Transformer encoder 替代 ViLBERT/LXMERT 的多流架构，参数更少、速度更快、效果更好。这个选择在后来被证明是 V+L encoder 预训练的正确方向。

2. **预训练任务设计不是越多越好，是要找到正确的组合。** 条件遮罩、MRFR+MRC-kl 的组合、OT-based WRA——这三个设计的消融实验清晰地展示了每个组件的贡献。

3. **细粒度对齐是跨模态理解的关键。** ITM 只能做全局判断，而 WRA 提供了词-区域级别的细粒度监督。最优传输是为这个目标设计的合适工具。

在技术树中，UNITER 是 D1-1（早期跨模态 encoder 范式）的重要节点之一。它和 ViLBERT、LXMERT 一起定义了"V+L 预训练"这个范式的早期形态。后续的 Oscar、VinVL、ALBEF 等模型——虽然在架构、预训练任务、训练策略上有各种改进——基本都延续了单流（或改进的单流）的设计路径。

UNITER 的代码开源（[ChenRocks/UNITER](https://github.com/ChenRocks/UNITER)），包含 PyTorch 实现、预训练脚本和下游任务微调脚本。
