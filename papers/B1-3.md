## 一、先搞清楚"问题在哪里"

2017 年 Transformer 在机器翻译上横空出世之后，NLP 领域几乎是肉眼可见地被它统一了。BERT 把双向上下文玩明白了，GPT 把生成式预训练做大了，到 2020 年已经出现超过 100B 参数的语言模型（Brown et al., 2020; Lepikhin et al., 2020），而且模型越大效果越好——没有饱和的迹象。

反观计算机视觉，CNN 仍然是绝对的主导（LeCun et al., 1989; Krizhevsky et al., 2012; He et al., 2016）。虽然已经有不少工作尝试把 self-attention 引入 CNN（Wang et al., 2018; Carion et al., 2020），甚至完全用 self-attention 替代卷积（Ramachandran et al., 2019; Wang et al., 2020a），但这些模型要么是"在 CNN 上加 attention"，要么是把卷积替换成特殊的 attention 模式——始终没有摆脱 CNN 的总体结构。而且，那些纯粹基于 attention 的模型虽然理论上高效，但在现代硬件加速器上由于使用了特殊 attention 模式而无法有效扩展。

到 2020 年，在大规模图像识别领域，经典 ResNet 类架构仍然是 state of the art（Mahajan et al., 2018; Xie et al., 2020; Kolesnikov et al., 2020）。

所以问题是：**能不能直接把标准 Transformer 拿到图像上，不做任何架构层面的修改？**

这个问题听起来简单，但在此之前没有人真正试过。不是不想试，有两个看似无解的原因。

## 二、旧路线的真正问题

### 2.1 像素级 self-attention 的 O(N²) 诅咒

Self-attention 的 naive 应用要求每个像素关注每一个其他像素。对于一张 224×224 的图片，这大约是 50k 个像素，attention 矩阵就有 50k² ≈ 2.5B 个元素——一张 GPU 也放不下。所以之前的所有工作都在做近似：局部邻域 attention（Parmar et al., 2018）、稀疏 Transformer（Child et al., 2019）、轴向 attention（Ho et al., 2019; Wang et al., 2020a）、可变大小块 attention（Weissenborn et al., 2019）。这些方案各有道理，但都需要复杂的工程优化才能在硬件上高效实现。

### 2.2 "把像素当词"的直觉被忽视

其实最直接的想法早就有人试过：Cordonnier et al.（2020）从输入图像中提取 2×2 的 patch，然后应用完整的 self-attention。这个模型几乎就是 ViT。但它有两个问题：一是 patch 太小（2×2），只适用于小分辨率图像；二是没有在大规模预训练中验证。

### 2.3 iGPT 的路没有走通

另一个相关工作 image GPT（iGPT; Chen et al., 2020a）把 Transformer 直接应用到像素级别——降低图像分辨率和颜色空间后，把像素排成序列做生成式预训练。最终在 ImageNet 上只达到 72%——远远不够。

### 2.4 核心矛盾

之前的思路困在一个死循环里：要么用近似 attention（各种复杂的局部/稀疏 attention 模式），要么用像素级输入但是效果很差。**没有人想到可以把图像切成 16×16 的 patch 然后用标准 Transformer——因为 16×16 意味着 196 个 token，和 NLP 里句子的 token 数量差不多。** 这个洞察看起来简单，但它是打破 O(N²) 诅咒的关键。

## 三、这篇论文的核心思路

**核心思路极其简单：把一张图切成 16×16 的非重叠 patch，拉直成向量序列，加上位置编码，然后直接用标准 Transformer encoder 处理。**

```text
输入图像: H×W×C
    │
    ▼
[Patch Extraction]    切成 N = H×W/P² 个非重叠的 P×P patch
    │                 默认 P=16, 224² 输入 → 196 个 patch
    ▼
[Linear Projection]   每个 patch 展平 → 线性投影到 D 维
    │                 输出: N×(P²·C) → N×D
    ▼
[+ [CLS] Token]       像 BERT 一样, 在序列前加一个可学习的 [class] token
    │                 序列长度: N+1
    ▼
[+ Position Embedding] 加可学习的 1D 位置编码
    │
    ▼
[Transformer Encoder] L 层标准 Transformer (MSA + MLP + LN + 残差)
    │
    ▼
[Classification Head]  取 [CLS] token 的输出 → 分类
```

这个方案的最大优势：**NLP 领域的 scalable Transformer 架构和高效实现可以直接拿来用。**

## 四、整体架构与公式

### 4.1 Vision Transformer（ViT）

ViT 的架构基本就是 BERT 的 encoder 直接搬过来。

**Patch Embedding：**

给定输入图像 $\mathbf{x} \in \mathbb{R}^{H \times W \times C}$ ，切分成固定大小的 2D patch $\mathbf{x}_p \in \mathbb{R}^{N \times (P^2 \cdot C)}$ ，其中 $P$ 是 patch 大小（默认 16）， $N = HW/P^2$ 是 patch 数量（224² 输入下 $N=196$ ）。Transformer 使用恒定隐层维度 $D$ ，因此用可训练的线性投影 $\mathbf{E}$ 把每个 patch 映射到 $D$ 维。

$$\mathbf{z}_0 = [\mathbf{x}_{\text{class}}; \mathbf{x}_p^1\mathbf{E}; \mathbf{x}_p^2\mathbf{E}; \cdots; \mathbf{x}_p^N\mathbf{E}] + \mathbf{E}_{pos}$$
$$\mathbf{E} \in \mathbb{R}^{(P^2 \cdot C) \times D},\; \mathbf{E}_{pos} \in \mathbb{R}^{(N+1) \times D}$$

**[CLS] Token：** 和 BERT 的 `[class]` token 完全一样，在 patch embedding 序列前加一个可学习的嵌入 $\mathbf{x}_{\text{class}}$ 。该 token 在 Transformer encoder 输出层的状态 $\mathbf{z}_L^0$ 被用作图像表示 $\mathbf{y}$ 。

**Position Embedding：** 使用标准的可学习 1D 位置编码。论文附录 D.4 的消融实验表明，更复杂的 2D-aware 位置编码（2D 位置编码、相对位置编码）相比简单的 1D 可学习位置编码几乎没有性能提升（Table 8 显示 1-D Pos. Emb. 在 5-shot ImageNet 上为 0.64206，2-D 为 0.64001，Rel. Pos. Emb. 为 0.64032）。因为 patch 级别的输入空间分辨率已经很小（14×14），学会空间关系对各种位置编码策略来说是同样容易的。

**Transformer Encoder：** 标准的 Transformer 层——LayerNorm 放在每个子层之前（Pre-LN），残差连接放在每个子层之后。MLP 包含两层，中间使用 GELU 非线性激活。

$$\mathbf{z'}_\ell = \text{MSA}(\text{LN}(\mathbf{z}_{\ell-1})) + \mathbf{z}_{\ell-1},\quad \ell=1\ldots L$$
$$\mathbf{z}_\ell = \text{MLP}(\text{LN}(\mathbf{z'}_\ell)) + \mathbf{z'}_\ell,\quad \ell=1\ldots L$$
$$\mathbf{y} = \text{LN}(\mathbf{z}_L^0)$$

**三个模型变体（Table 1）：**

| 模型 | 层数 | Hidden D | MLP 大小 | Heads | 参数量 |
|------|------|----------|---------|-------|--------|
| ViT-Base | 12 | 768 | 3072 | 12 | 86M |
| ViT-Large | 24 | 1024 | 4096 | 16 | 307M |
| ViT-Huge | 32 | 1280 | 5120 | 16 | 632M |

Base 和 Large 直接沿用 BERT 的配置，Huge 是新增的。

模型命名规则：ViT-L/16 表示 Large 变体 + 16×16 的输入 patch 大小。注意 Transformer 的序列长度与 patch 大小的平方成反比——patch 越小，序列越长，计算成本越高。

**分类头：** 预训练时用含一个隐藏层的 MLP + tanh 激活，微调时替换为单层线性层。

### 4.2 Fine-tuning and Higher Resolution

微调时，移掉预训练的分类头，附加一个零初始化的 $D \times K$ 线性层（ $K$ 是下游类别数）。

提高分辨率微调（fine-tune at higher resolution）时，保持 patch 大小不变 → 有效序列长度增加。ViT 可处理任意序列长度（受显存限制），但预训练的位置编码在新分辨率下没有意义。因此对预训练的位置编码进行 2D 插值，按其在原始图像中的位置对应调整。

注意：**分辨率调整和 patch 提取——是 ViT 中唯二手动注入 2D 图像结构归纳偏置的地方。**

### 4.3 Hybrid Architecture

混合架构：输入序列来自 CNN 特征图，而不是原始图像。patch embedding 投影 $\mathbf{E}$ 作用于从 CNN 特征图中提取的 patch。特殊情况：patch 的空间大小为 1×1，即直接把特征图空间维度展平后投影到 Transformer 维度。

## 五、归纳偏置——少到惊人的先验知识

论文专门用了一个小节讨论归纳偏置（Inductive bias），这是 ViT 区别于所有之前视觉模型最根本的地方。

**在 CNN 中：** 局部性（locality）、二维邻域结构（two-dimensional neighborhood structure）、平移等变性（translation equivariance）——这些先验被嵌入到每一层中，从模型第一层到最后层都在发挥作用。

**在 ViT 中：** 
- 只有 MLP 层是局部且平移等变的
- Self-attention 是全局的——没有局部性假设
- 2D 邻域结构的利用极为有限：只在模型开始时把图像切成 patch，以及在微调时调整位置编码来适配不同分辨率的图像
- 除此之外，位置编码在初始化时完全不含任何关于 patch 2D 位置的信息——**所有空间关系都必须从数据中从头学习**

论文的核心结论：**大规模训练（large scale training）可以碾压归纳偏置（trumps inductive bias）。**

当在中等规模数据集（如 1.3M 的 ImageNet）上训练而不加强正则化时，ViT 的准确率比同量级 ResNet 低几个百分点——这符合预期，因为 Transformer 缺乏 CNN 的归纳偏置，在数据不足时无法很好地泛化。

但当模型在更大的数据集（14M-300M 图像）上训练时，情况就彻底变了。

## 六、实验配置

### 6.1 数据集

- **ImageNet（ILSVRC-2012）：** 1k 类，1.3M 图像
- **ImageNet-21k：** 21k 类，14M 图像（Deng et al., 2009）
- **JFT：** 18k 类，303M 高分辨率图像（Sun et al., 2017）

下游任务：ImageNet（原始验证标签和清洗后的 ReaL 标签）、CIFAR-10/100、Oxford-IIIT Pets、Oxford Flowers-102，以及 19-task VTAB 分类套件（每任务 1,000 个训练样本），分 Natural、Specialized 和 Structured 三组。

### 6.2 训练配置

所有模型（包括 ResNet）使用 Adam 优化器（Kingma & Ba, 2015）， $\beta_1=0.9, \beta_2=0.999$ ，batch size 4096，weight decay 0.1。附录 D.1 显示，在他们的设置下 Adam 对 ResNet 也略优于 SGD——这个反常规发现很有意思。

线性学习率预热 + 线性衰减。微调时使用 SGD with momentum，batch size 512。Table 2 的 ImageNet 结果在 512（ViT-L/16）或 518（ViT-H/14）分辨率下微调，并使用 Polyak 平均（因子 0.9999）。

对比基线：ResNet（He et al., 2016）但用 Group Normalization 替换 Batch Normalization，使用标准化卷积（Qiao et al., 2019）——论文称修改后的模型为 "ResNet (BiT)"。

## 七、核心实验结果

### 7.1 与 SOTA CNN 对比（Table 2）

| | ViT-H/14 (JFT) | ViT-L/16 (JFT) | ViT-L/16 (I21k) | BiT-L (ResNet152x4) | Noisy Student (EfficientNet-L2) |
|---|---|---|---|---|---|
| **ImageNet** | **88.55%** | 87.76% | 85.30% | 87.54% | 88.4/88.5%* |
| **ImageNet ReaL** | **90.72%** | 90.54% | 88.62% | 90.54% | 90.55% |
| **CIFAR-100** | **94.55%** | 93.90% | 93.25% | 93.51% | - |
| **CIFAR-10** | **99.50%** | 99.42% | 99.15% | 99.37% | - |
| **Oxford-IIIT Pets** | **97.56%** | 97.32% | 94.67% | 96.62% | - |
| **Oxford Flowers-102** | 99.68% | **99.74%** | 99.61% | 99.63% | - |
| **VTAB (19 tasks)** | **77.63%** | 76.28% | 72.72% | 76.29% | - |
| **TPUv3-core-days** | **2.5k** | **0.68k** | **0.23k** | 9.9k | 12.3k |

关键发现：
- ViT-H/14（JFT-300M 预训练）在所有数据集上全面超越 BiT-L，且预训练算力仅为后者的 1/4（2.5k vs 9.9k TPUv3-core-days）
- ViT-L/16（JFT-300M）在几乎所有任务上超过 BiT-L，预训练算力仅为 0.68k——**1/15 的算力达到相当或更好的效果**
- 即使只在 ImageNet-21k（公开数据集）上预训练，ViT-L/16 也取得了有竞争力的结果——可在 8 核 TPUv3 上约 30 天内完成训练
- Noisy Student 在 ImageNet 上达到 88.4/88.5%（稍微优于 ViT-H/14 的 88.55%），但这是通过半监督学习 + JFT 去标签数据 + 大 Ensemble 实现的——训练成本是 12.3k 个 TPUv3-core-days

VTAB 分解（Figure 2）：ViT-H/14 在 Natural 和 Structured 任务组上表现最好；Specialized 组上与 BiT-R152x4 相近。

### 7.2 预训练数据规模的影响（Figure 3 & 4）

这是论文最有洞察力的实验：

1. **直接对比不同规模预训练（Figure 3）：** 在 ImageNet（最小）上预训练时，大 ViT 模型比同尺寸 ResNet 差；在 ImageNet-21k（中间）上训练时，两者接近；只有在 JFT-300M 上，大 ViT 才真正超越 ResNet。

2. **随机子集实验（Figure 4）：** 在 JFT 的 9M、30M、90M 和完整 303M 子集上训练。ViT-B/32（略快于 ResNet50）在 9M 子集上远差于 ResNet50，但在 90M+ 子集上反超。ViT-L/16 和 ResNet152x2 也有同样模式。**这说明 CNN 的卷积归纳偏置在小数据上有用，但数据足够大时，直接从数据学习相关模式不仅足够，而且更好。**

### 7.3 性能 vs 算力的扩展性分析（Figure 5）

论文对 7 个 ResNet、6 个 ViT、5 个 Hybrid 模型做了系统的 scaling 实验（在 JFT-300M 上预训练，评估多个下游任务）。

**核心发现：**
- ViT 在性能/算力权衡上全面优于 ResNet——**ViT 用约 2-4 倍的更少算力即可达到相同的性能（5 个数据集平均）**
- Hybrid 在小算力预算时略微优于纯 ViT，但大模型时差距消失
- ViT 在所尝试的范围内未出现饱和——这是鼓励继续 scaling 的信号

### 7.4 自监督实验

论文做了一个初步的 masked patch prediction 实验（类似 BERT 的 masked language modeling）：破坏 50% 的 patch embedding（80% 替换为 [mask] 嵌入，10% 替换为随机其他 patch 嵌入，10% 保持不变），预测每个被破坏 patch 的 3-bit 平均颜色（共 512 种颜色）。

结果：ViT-B/16 在 ImageNet 上达到 79.9%，比从头训练好 2%，但仍比有监督预训练差 4%。论文将对比学习预训练留作未来工作。

### 7.5 注意力可视化分析

论文分析了 ViT 的内部工作机制（Figure 6 & 7）：

- **Embedding filters（Figure 7 左）：** 第一层线性投影的主成分分析显示，学习的滤波器与低维 patch 细结构的基函数相似
- **Position embedding 相似度（Figure 7 中）：** 模型学到的位置编码中，空间距离近的 patch 有更相似的位置编码，且呈现行/列结构
- **Attention distance（Figure 7 右）：** 一些 head 在最低层就已经关注到图像的大部分区域——全局信息整合能力从一开始就在使用。其他 head 在低层保持很小的 attention distance，这可能起到了 CNN 中早期卷积层的作用。attention distance 随深度增加而增加
- **Attention maps（Figure 6）：** 模型关注与分类语义相关的图像区域

## 八、核心设计决策与消融

### 8.1 位置编码（Appendix D.4）

| 位置编码方案 | ImageNet 5-shot |
|------------|:------------:|
| 无位置编码 | 0.61382 |
| 1-D 可学习 | 0.64206 |
| 2-D 可学习 | 0.64001 |
| 相对位置编码 | 0.64032 |

无位置编码 → 有位置编码差距巨大（0.61382 vs 0.64206），但不同位置编码形式之间几乎没有差距。论文推测，在 patch 级别输入上（空间分辨率仅为 14×14），学会空间关系对各种编码策略都同样容易。

### 8.2 [CLS] Token vs Global Average Pooling（Appendix D.3）

初始实验中发现 GAP 效果很差。但进一步实验发现这不是 [CLS] Token 或 GAP 本身的问题——而是它们需要不同的学习率。调整合适的学习率后，两者效果相当。

### 8.3 Transformer Shape Scaling（Appendix D.2）

对不同维度做 scaling 消融（固定其他维度），发现：
- **深度（depth）** 的 scaling 收益最大（直到 64 层才饱和）
- **宽度（width）** 的 scaling 收益最小
- **减小 patch size**（增加序列长度）在不增加参数的情况下持续带来改进

结论：计算量（compute）可能比参数量（parameters）更能预测性能。Scaling 应优先加深而非加宽。

### 8.4 训练细节（Table 3）

所有模型用 batch size 4096 + 10k 步预热训练。ImageNet 从头训练需要强正则化（weight decay 0.3，dropout 0.1，gradient clipping global norm 1）。Dropout 应用在除 qkv 投影外的所有稠密层之后。训练分辨率 224。

## 九、后续影响与技术谱系位置

### 9.1 技术树中的位置

```text
[Transformer, Vaswani et al., 2017]
   │
   ├── [BERT, Devlin et al., 2019]  ← [CLS] token + Pre-LN 的灵感来源
   │
   └── [ViT, 2020]  ← ★ 本篇 ★ 第一个无卷积的视觉 Transformer
          │
          ├── [DeiT, Touvron et al., 2021]  ← 训练策略改进，数据蒸馏
          │      │
          │      ├── 不需要 JFT-300M，用 ImageNet 即可
          │      └── 知识蒸馏 + 强数据增强
          │
          ├── [PVT, Wang et al., 2021]  ← 金字塔 Transformer
          │
          ├── [Swin Transformer, Liu et al., 2021]  ← 层级局部窗口
          │      │
          │      ├── 解决了 ViT 最致命的两个问题：
          │      │   1. 单分辨率输出（只有 1/16 分辨率特征图）
          │      │   2. O(N²) 全局 attention 复杂度
          │      └── 把 Transformer backbone 变成通用 backbone
          │
          ├── [MAE, He et al., 2021]  ← 自监督走向巅峰
          │
          └── [CLIP, Radford et al., 2021]  ← 图文对齐
                 │
                 └── ViT 的文章 encoder 成为 CLIP 的标准视觉 backbone
```

ViT 是整个视觉 Transformer 路线的 **根节点**。这个简单到极致的架构开启了一条完全不同的技术路线。

### 9.2 前驱工作

- **Transformer（Vaswani et al., 2017）：** ViT 几乎直接照搬了原始 Transformer 的 encoder 设计，只改动了输入层
- **BERT（Devlin et al., 2019）：** [CLS] token 的使用直接来自 BERT，同时 ViT 的模型变体配置也直接来自 BERT（Base 和 Large）
- **Cordonnier et al.（2020）：** 最接近的前驱工作，也已经用 2×2 patch + 全局 self-attention。但 patch 太小（只适用于小分辨率），且没有验证大规模预训练的价值

### 9.3 同期与后续

- **DeiT（Touvron et al., 2021）：** 解决了 ViT "依赖于 JFT-300M 这种超大数据集" 的痛点。通过知识蒸馏（RegNet作为 teacher）+ 强数据增强，DeiT 可以在 ImageNet-1K 上从头训练达到有竞争力的结果。DeiT-B 在 ImageNet 上达到 81.8%（无蒸馏）/ 83.8%（有蒸馏 vs RegNetY-16GF）。

- **Swin Transformer（Liu et al., 2021）：** 在 ViT 之后迅速被发现的"ViT 不能做检测/分割"的问题上给出了答案。Swin 用 4 级金字塔（Patch Merging）+ 局部窗口 attention + 移位窗口方案，把 Transformer backbone 从分类扩展到通用 dense prediction backbone。

- **MAE（He et al., 2021）：** 在 ViT 的自监督路线上走了更远。MAE 用极高比例的 masking（75%）+ decoder-only 重构，在 ImageNet 上让 ViT 的自监督预训练首次追上了有监督。

- **CLIP（Radford et al., 2021）：** ViT 的架构被直接选为 CLIP 的视觉 encoder。ViT-L/14 成为 CLIP 的标准配置。这是 ViT 架构影响力最广的落地——几乎影响了后续所有 multimodal 模型。

### 9.4 范式意义

**ViT 的范式意义是"zero to one"级别的。**

在 2020 年之前，视觉领域的共识是：CNN 的归纳偏置——平移等变性、局部性、层级结构——是处理图像数据所必需的。即使有人试图引入 self-attention，也都是在 CNN 的框架内做加法或局部替换。

ViT 用最极端的实验设计证明了三件事：

1. **Transformer 可以纯靠 attention 做视觉，不需要 CNN 的任何帮助。** 不是"在 CNN 上加 attention"，而是完全不用 CNN，直接拿 patch 序列给 Transformer。这个"去卷积"的结果本身就是一个强烈的信号。

2. **归纳偏置不是必需的——数据够多就能学出来。** 在小数据上 ViT 不如 ResNet，但那是因为 CNN 的先验帮你省了数据。数据足够多（14M-300M）时，Transformer 不仅学会了 CNN 已经有的模式，还做得更好。论文的原话是：*"large scale training trumps inductive bias."*

3. **视觉 Transformer 的扩展效率远高于 CNN。** ViT 用 2-4 倍更少的 FLOPs 达到同样精度（Figure 5）。在 TPUv3 上，ViT-H/14 的预训练算力（2.5k core-days）只有 BiT-L（9.9k core-days）的 1/4，但效果更好。这个扩展优势决定了后续的走向——当模型规模继续增大时，ViT 路线的成本优势会越来越明显。

## 十、局限性与未解决的问题

### 10.1 "只能做分类"

论文自己承认了这点。Conclusion 中写道：*"One is to apply ViT to other computer vision tasks, such as detection and segmentation."* ViT 的单分辨率输出（ $\frac{H}{16} \times \frac{W}{16}$ 的特征图）天然不适合 dense prediction 任务。这个问题在 Swin Transformer 中被解决。

### 10.2 数据饥渴

ViT 在中小数据集（如 ImageNet）上不如同量级 ResNet。虽然大模型在大数据上有优势，但这对算力不够的团队来说是一个准入门槛。DeiT 后来通过知识蒸馏和强数据增强解决了这个问题。

### 10.3 自监督距离有监督还有差距

Masked patch prediction 只做到 79.9%，比有监督预训练差 4%。直到 MAE（2021）才把这个差距缩小。

### 10.4 全局 attention 的计算瓶颈

虽然 patch 化把 token 数降到了 196（224² 输入，16² patch），但对更高分辨率输入或更小 patch 时， $O(N^2)$ 的复杂度仍然是一个问题。ViT-L/16 在 512 分辨率下微调的 token 数是 $(512/16)^2 = 1024$ ，这个还算可控。但再大就不行了。

### 10.5 对训练细节敏感

ViT 需要精心调参：weight decay、learning rate、dropout、label smoothing 在小数据集上都要仔细调节。论文在 ImageNet 上用了 strong regularization（weight decay 0.3, dropout 0.1, gradient clipping 1）才得到可接受的结果。

### 10.6 两层分类头问题

预训练用了 2 层 MLP + tanh，但微调时替换为单层线性层。这意味着预训练学到的分类表示在下游任务上没有被直接利用。后续 DeiT 和 MAE 等工作中，微调策略得到了更系统的研究。

## 十一、小结

ViT 在 2020 年提交 （2010.11929，2020 年 10 月）的时候，可能连作者自己都没有完全意识到这篇文章会变成什么。

它是一个极其简单的架构——把 Transformer encoder 从 NLP 里抄过来，改一下输入层就行。论文原文坦率地承认：*"In model design we follow the original Transformer as closely as possible."* 不加新东西，不针对视觉做特殊设计，几乎是刻意地做减法。

但就是这个 "不做特殊设计" 的设计，卡住了整个 CV 社区之前 3-4 年探索视觉 Transformer 路上的所有弯路。之前大家想的是怎么在 attention 机制上做文章——局部 attention、稀疏 attention、轴向 attention、relational attention——每种方案都有自己的理由和至少一篇顶会论文。ViT 告诉你：**这些东西都不需要。把 patch 切大一点，给够数据，标准 Transformer 自己就学会了。**

从实际影响看，ViT 是过去十年 CV 领域最重要的论文之一：
- 它开启了 "视觉 Transformer backbone" 这条技术路线，孕育了 Swin、DeiT、MAE、DINO 一批后续工作
- 它的架构（ViT-L/14）成为 CLIP 的视觉 encoder，从而间接影响了多模态基础模型的发展
- 它打破了 "视觉必须有卷积归纳偏置" 这个长达几十年的学术共识

技术树的 B1-3 节点标注为 "里程碑节点"——这个定位是准确的。ViT 不是终点，它是起点。
