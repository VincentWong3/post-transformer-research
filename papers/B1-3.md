# An Image is Worth 16x16 Words: Transformers for Image Recognition at Scale（2020）

**论文：** [arXiv](https://arxiv.org/abs/2010.11929) · Dosovitskiy et al. · 2020


## 一、先搞清楚坑在哪

2020 年以前，计算机视觉几乎被卷积神经网络（CNN）统治。从 AlexNet（Krizhevsky et al., 2012）到 ResNet（He et al., 2016）再到 EfficientNet（Tan & Le, 2019），CNN 架构不断进化，但核心假设不变：**局部连接 + 平移等变性 + 层级下采样**。

与此同时，NLP 领域已经被 Transformer（Vaswani et al., 2017）彻底改变。BERT（Devlin et al., 2019）和 GPT（Radford et al., 2018; Brown et al., 2020）证明了：在足够数据上预训练的大规模 Transformer，可以碾压一切精心设计的 NLP 架构。

一个自然的问题是：**Transformer 能不能也取代 CNN 成为视觉领域的基础架构？**

此前不是没人试过。但已有的尝试都是"在 CNN 上加 attention"（如 SE-Net, Hu et al., 2019; Non-local Networks, Wang et al., 2018），或者用自注意力完全替代卷积但在小模型上验证（如 Ramachandran et al., 2019）。关键问题是：**纯 Transformer 直接怼图片，在大规模数据上到底能不能行？**

## 二、ViT 之前尝试的真正问题

在 ViT 之前，把 attention 用到视觉上的工作可以分为几类：

**一类：在 CNN 内部加 attention 模块。** 如 SE-Net、CBAM、Non-local Networks。这些方法保留了 CNN 的骨干，只是在某些层穿插了 attention 操作。性能有提升，但本质上还是 CNN——Transformer 的潜力没真正释放。

**二类：用局部自注意力完全替代卷积。** 如 Stand-Alone Self-Attention（Ramachandran et al., 2019）和 Axial Attention（Ho et al., 2019）。这些工作在理论上证明了 attention 可以替代卷积，但在实际硬件上的效率极差——滑动窗口的 key set 不固定导致内存访问散乱，GPU 利用率极低。

**三类：Sparse Transformer（Child et al., 2019）和 Image Transformer（Parmar et al., 2018）**。尝试用稀疏 attention 模式把 self-attention 扩展到图像，但工程复杂度过高，难以规模化。

**四类：Cordonnier et al.（2020）。** 最接近 ViT 的工作——也是把图片切成 patch（2×2），然后做全局 self-attention。但受限于 2×2 的极小 patch size（导致 token 量极大，只能处理小分辨率），且只在 ImageNet 上验证，没有大规模预训练。

ViT 的核心理念与这些工作不同：**不优化 attention 机制，而是优化数据规模。** 只要数据够多、模型够大，CNN 的归纳偏置（inductive bias）可以被大规模训练覆盖。

## 三、ViT 的核心思路

一句话：**把图片切成 16×16 的 patch，拉直成序列，直接用标准 Transformer encoder做分类，大规模预训练是关键。**

要点：
- **最小改动原则**：尽可能复用 NLP Transformer 的架构——同一个 Transformer encoder、同样的 [class] token 机制、同样的 pre-LN 结构
- **Patch 化是关键设计**：图片 224×224→16×16 patch→196 个 token。这个数字和 NLP 里 "BERT-base 处理 512 token" 是同量级的
- **数据规模 > 架构设计**：在 ImageNet-1K（1.3M 图片）上从头训练，ViT 打不过 ResNet。但在 JFT-300M（3 亿图片）上预训练后迁移，ViT 碾压 ResNet

## 四、网络架构详解

### 架构总览

```
Input: RGB 224×224×3
    │
    ▼
┌──────────────────────────────────────┐
│ Patch Embedding                      │
│ 图片 → 16×16 不重叠 patch → 196 patches│
│ 每个 patch 展平: 16×16×3 = 768 维     │
│ 线性投影: 768 → D (D=768 for ViT-B)  │
└──────────────────┬───────────────────┘
                   │
                   ▼
┌──────────────────────────────────────┐
│ Embedding 拼接 + 位置编码              │
│ [class] token + 196 patch embeddings │
│ + 位置编码 (1D learnable)            │
│ 总序列长度: 197, 每 token 维度: D    │
└──────────────────┬───────────────────┘
                   │
                   ▼
┌──────────────────────────────────────┐
│ Transformer Encoder ×L (L=12 for B)  │
│ ┌─────────────────────────────────┐  │
│ │  Layer Norm                      │  │
│ │  Multi-Head Self-Attention (MSA) │  │
│ │  Residual                        │  │
│ │  Layer Norm                      │  │
│ │  MLP (D→4D→D, GELU)             │  │
│ │  Residual                        │  │
│ └─────────────────────────────────┘  │
└──────────────────┬───────────────────┘
                   │
                   ▼
        ┌──────────────────┐
        │ 取 [class] token  │
        │  第 0 位置输出    │
        └────────┬─────────┘
                 │
                 ▼
        ┌──────────────────┐
        │ Classification   │
        │ Head: D → K     │
        └──────────────────┘
```

### 前向过程逐步走（以 ViT-B/16 为例）

**Step 1 — Patch Embedding.** 输入 224×224×3 → 切成 16×16 的不重叠 patch。patch 数量：$(224/16)^2 = 14×14 = 196$。每个 patch 展平成 768 维（16×16×3）。线性投影 $E \in \mathbb{R}^{768 \times 768}$ 将其映射到 D=768。

实现上，这等价于一个 `nn.Conv2d(3, 768, kernel_size=16, stride=16)`——卷积核 16×16，步长 16，输出形状 `1×768×14×14`，然后 flatten 成 `196×768`。用卷积实现 patch embedding 比手动切 patch 快得多。

**Step 2 — 拼接 Embedding。**

$$\mathbf{z}_0 = [\mathbf{x}_{\text{class}}; \; \mathbf{x}_p^1\mathbf{E}; \; \mathbf{x}_p^2\mathbf{E}; \; \cdots ; \; \mathbf{x}_p^N\mathbf{E}] + \mathbf{E}_{\text{pos}}$$

- $\mathbf{x}_{\text{class}}$：一个可学习的 [class] token，BERT 风格的分类标记。形状 1×D
- $\mathbf{x}_p^i\mathbf{E}$：第 i 个 patch 经过线性投影后的 embedding，共 196 个。形状 196×D
- 拼接后：197×D
- $\mathbf{E}_{\text{pos}}$：可学习的 1D 位置编码。形状 197×D

**为什么用 1D 位置编码而不是 2D？** 论文 Appendix D.4 验证了 2D-aware 位置编码（行+列偏移）没有显著提升。直觉：self-attention 是全局的，模型可以在 attention 权重里自己学习 2D 空间关系，不需要先验地编码 2D 结构。

**Step 3 — Transformer Encoder ×12（ViT-B）。** 每一层：

$$\mathbf{z}'_\ell = \text{MSA}(\text{LN}(\mathbf{z}_{\ell-1})) + \mathbf{z}_{\ell-1}, \quad \ell=1...L$$

$$\mathbf{z}_\ell = \text{MLP}(\text{LN}(\mathbf{z}'_\ell)) + \mathbf{z}'_\ell, \quad \ell=1...L$$

12 层保持形状 197×768 不变。每层维度不变——这是 Transformer 的特点，不像 CNN 分辨率逐层递减。

**Step 4 — 分类输出。**

$$\mathbf{y} = \text{LN}(\mathbf{z}_L^0)$$

只取序列中第 0 个位置（[class] token），过 LayerNorm，然后接分类头。

### 模型变体

| 变体 | Layers | Hidden D | MLP size | Heads | 参数量 |
|------|--------|----------|----------|-------|--------|
| ViT-Base | 12 | 768 | 3072 | 12 | 86M |
| ViT-Large | 24 | 1024 | 4096 | 16 | 307M |
| ViT-Huge | 32 | 1280 | 5120 | 16 | 632M |

ViT-B 直接照搬 BERT-Base 的超参数。这不是巧合——作者刻意复用 NLP 上的成功配置。

## 五、核心创新点

严格来说，ViT 没有"创新"——它用的是标准 Transformer encoder、标准 patch embedding、标准 [class] token。它的贡献在于**设计选择和实验验证**。

### (a) 设计选择 1: Patch Size = 16×16

**动机：** 更小的 patch（如 8×8）产生更多 token（28×28=784），计算量更大；更大的 patch（如 32×32）token 太少（7×7=49），空间信息丢失。16×16 是 ImageNet 分类上精度和效率的最佳平衡。

**为什么不在 patch 之间用 overlap？** 不重叠使实现简单（直接 Conv2d stride=kernel_size），而且重叠引入的额外计算对 ImageNet-1K 分类没有明显帮助。

### (b) 设计选择 2: [class] token

**动机：** BERT 用 [CLS] token 做分类，ViT 照搬。替代方案是对所有 patch 的输出做全局平均池化（类似 ViT 后的很多工作采用的方式）。论文 Appendix D.1 验证了两者效果相近，[class] token 和 GAP 差别不大。之所以用 [class] token，只是因为"和 BERT 保持一致"。

### (c) 设计选择 3: 1D 位置编码

**动机：** 2D 位置编码（把 H 和 W 的位置分别编码）理论上更好，但实验证明 1D 也够用——因为模型可以从 patch embedding 的内容中推断出 2D 布局。细想也合理：注意力权重里，图像左上角的 patch 和右下角的 patch 自然会互相关性弱，不需要硬编码。

### (d) 设计选择 4: Pre-LN（LayerNorm 放在 MSA/MLP 之前）

**动机：** 原始 Transformer（Vaswani et al., 2017）用 Post-LN（LayerNorm 在残差之后），但 Post-LN 在大模型上训练不稳定。Pre-LN（Xiong et al., 2020; Wang et al., 2019）把 LN 放在子层之前，梯度更稳定。ViT 全用 Pre-LN。

**如果不用会怎样：** 用 Post-LN 的大 ViT 模型训练更容易发散，需要更多的 warmup 和学习率调参。

### (e) 大规模预训练才是真正的创新

ViT 最重要的贡献不是架构设计，而是**实验证明**：只要数据足够大，标准的 Transformer 可以直接学习图片，不需要 CNN 的归纳偏置。

| 预训练数据 | ImageNet Top-1 |
|-----------|---------------|
| ImageNet-1K (1.3M) | 79.9 (ViT-B, 从零训练) |
| ImageNet-21K (14M) | 85.4 (ViT-B, 预训练) |
| JFT-300M (303M) | 88.55 (ViT-H/14, 预训练) |

从 ImageNet-1K 到 JFT-300M，数据量扩大 230 倍，精度提升 8.6 个百分点。这印证了论文的核心论点：**大规模训练可以弥补归纳偏置的缺失。**

## 六、公式详解

### 公式 1: Patch Embedding

$$\mathbf{x}_p \in \mathbb{R}^{N \times (P^2 \cdot C)}, \quad N = \frac{HW}{P^2}$$

$$\mathbf{z}_0 = [\mathbf{x}_{\text{class}}; \; \mathbf{x}_p^1\mathbf{E}; \; \mathbf{x}_p^2\mathbf{E}; \; \cdots ; \; \mathbf{x}_p^N\mathbf{E}] + \mathbf{E}_{\text{pos}}$$

$$\mathbf{E} \in \mathbb{R}^{(P^2 \cdot C) \times D}, \quad \mathbf{E}_{\text{pos}} \in \mathbb{R}^{(N+1) \times D}$$

**(a) 符号定义：**
- $H, W, C$: 输入图像的高、宽、通道数（224, 224, 3）
- $P$: patch size（16）。$N=HW/P^2=196$ 个 patch
- $D$: Transformer 的隐层维度（ViT-B: 768）
- $\mathbf{E}$: patch embedding 投影矩阵
- $\mathbf{E}_{\text{pos}}$: 1D 位置编码（可学习）
- $\mathbf{x}_{\text{class}}$: [class] token 起始向量（可学习）

**(b) 公式来源：** 直接改编自 BERT 的输入表示（Devlin et al., 2019），唯一的改动是用线性投影代替 BERT 的 token embedding lookup。

**(c) 推导过程：** 输入 224×224×3 → 切 16×16 patch → 每个 patch 3×16×16 = 768 维 → 与 $E$ 做矩阵乘法：$\mathbf{x}_p^i \times \mathbf{E}$ 得到 D 维向量 → 拼接 [class] token 得 (N+1)×D → 加位置编码。

**(d) 直觉：** $P=16$ 的选择有深层原因：224/16=14，14×14=196 个 token，和 BERT 的 512 token 是同一个数量级。如果 $P=8$，token 数 784——虽然信息更丰富，但计算量翻 4 倍（attention 是 $O(N^2)$）。如果 $P=32$，token 数 49——信息丢失。16×16 碰巧是最佳平衡点。

**(e) 边界情况：** 当调整输入分辨率时，$N$ 变化，$\mathbf{E}_{\text{pos}}$ 需要 2D 插值。例如在 384×384 上 fine-tune 时，$N=(384/16)^2=576$，位置编码需从 14×14 插值到 24×24。

### 公式 2: Multi-Head Self-Attention (MSA)

标准 scaled dot-product attention：

$$\text{Attention}(Q, K, V) = \text{SoftMax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

ViT 的 multi-head：

$$\text{MSA}(\mathbf{z}) = [\text{head}_1; \text{head}_2; \cdots; \text{head}_h]\mathbf{W}_O$$

$$\text{head}_i = \text{Attention}(\mathbf{z}\mathbf{W}_i^Q, \mathbf{z}\mathbf{W}_i^K, \mathbf{z}\mathbf{W}_i^V)$$

**(a) 符号定义：** $Q,K,V \in \mathbb{R}^{197 \times d_k}$（每 head）。$d_k = D/h$，$h=12$（ViT-B），所以 $d_k = 768/12 = 64$。

**(b) 公式来源：** 标准 Transformer（Vaswani et al., 2017），没有修改。这是 ViT "最小改动原则"的体现。

**(c) 直觉：** Attention 矩阵 $QK^T$ 的形状是 197×197——每一个 patch 对每一个其他 patch 的注意力权重。这就是全局 self-attention。相比之下，CNN 的卷积核只看到 $k \times k$ 的局部邻域（如 ResNet 的 3×3）。全局 attention 是 ViT 对比 CNN 的根本区别。

**数值举例：** $QK^T$ 的计算量：197×768 乘 768×197 → 197×197 → $197^2 = 38809$ 个 attention 权重。每个权重表示一个 patch 对另一个 patch 的关注程度。如果输入是 800×1333（$P=16$，$N \approx 4160$），$4160^2 \approx 17M$ 个权重——这就是 Swin Transformer 要解决的计算量问题。

### 公式 3: Transformer Block（Pre-LN）

$$\mathbf{z}'_\ell = \text{MSA}(\text{LN}(\mathbf{z}_{\ell-1})) + \mathbf{z}_{\ell-1}, \quad \ell=1...L$$

$$\mathbf{z}_\ell = \text{MLP}(\text{LN}(\mathbf{z}'_\ell)) + \mathbf{z}'_\ell, \quad \ell=1...L$$

$$\mathbf{y} = \text{LN}(\mathbf{z}_L^0)$$

**(a) 符号：** $\ell$ 是层索引（ViT-B: $\ell=1...12$），$\mathbf{z}_L^0$ 是第 $L$ 层输出的第 0 个位置（[class] token）。

**(b) 来源：** Pre-LN Transformer（Xiong et al., 2020; Wang et al., 2019）。原始 Transformer 的 Post-LN 是 $\mathbf{z}'_\ell = \text{LN(MSA}(\mathbf{z}_{\ell-1}) + \mathbf{z}_{\ell-1})$。ViT 把 LN 移到了子层**之前**。

**(c) 为什么 Pre-LN 更好：** 在 Post-LN 中，LayerNorm 对残差流的输出做归一化——但如果某一个 MSA 层的输出非常大，LN 会把整个残差流压缩，导致梯度爆炸或消失。Pre-LN 的 LN 只作用于子层的输入，残差连接直接相加不被 LN 影响，梯度更稳定。

**(d) 边界情况：** 最后一层的 $\mathbf{y} = \text{LN}(\mathbf{z}_L^0)$ 取了第 0 个位置后过 LN，这和中间层的 LN 用法不一致——中间层 LN 在 MSA/MLP 之前，最后一层 LN 在输出之后。原因是 [class] token 经过 12 层 attention 后可能没有归一化到和分类头兼容的分布范围，一个额外的 LN 能稳定 fine-tuning。

### 公式 4: Fine-tuning 分辨率调整

Fine-tune 时如果需要更高分辨率，保持 $P$ 不变，$N$ 增大。位置编码通过 2D 插值调整：

$$\mathbf{E}_{\text{pos}}^{(new)} = \text{Interpolate}\left(\mathbf{E}_{\text{pos}}^{(orig)}; \; (H_{new}/P, W_{new}/P)\right)$$

例如：224→384 输入，位置编码网格从 14×14 插值到 24×24。这是 ViT 唯一注入 2D 归纳偏置的地方。

## 七、实验结果

### ImageNet 分类（Table 2）

| 模型 | 预训练数据 | ImageNet Top-1 | 预训练成本 (TPUv3-days) |
|------|-----------|---------------|------------------------|
| ResNet-152 (BiT) | ImageNet-21K | 86.0 | — |
| EfficientNet-L2 (Noisy Student) | ImageNet + JFT | 88.4 | — |
| **ViT-B/16** | **JFT-300M** | **86.8** | **0.25k** |
| **ViT-L/16** | **JFT-300M** | **88.1** | **0.72k** |
| **ViT-H/14** | **JFT-300M** | **88.55** | **2.5k** |

核心发现：ViT-H/14 在 ImageNet 上达到 88.55%，持平当时 SOTA 的 EfficientNet-L2（88.4%），但训练成本低得多。

### VTAB 迁移学习（Table 3）

| 模型 | Natural | Specialized | Structured | 平均 |
|------|---------|-------------|-----------|------|
| BiT-ResNet-152 × 3 | 82.1 | 86.1 | 64.0 | 77.4 |
| **ViT-H/14** | **84.2** | **87.2** | **69.5** | **80.3** |

ViT 在 Structured 任务上比 ResNet 高出 5.5 个百分点——这部分任务需要几何理解（如计数、深度预测），说明全局 attention 比 CNN 的局部感受野更擅长捕捉空间关系。

### 规模效应（Figure 3）

论文最关键的可视化：随着预训练数据量增大，ViT 的性能提升速度比 ResNet 快得多：
- ImageNet-1K 上：小数据，ViT 打不过 ResNet（缺少归纳偏置）
- ImageNet-21K 上：中期，ViT 追上 ResNet
- JFT-300M 上：大数据，ViT 反超 ResNet——并且差距随模型增大拉大

ViT-H/14 在 JFT-300M 上预训练后，CIFAR-100 达到 94.55%、CIFAR-10 达到 99.50%。

## 八、代码对照（GitHub: google-research/vision_transformer）

官方实现是 JAX / Flax 版本的 `vit_jax/models_vit.py`。几个代码层面的发现：

**1. Patch Embedding 用 Conv2d 实现。** 不是把图片切 patch 再矩阵乘，而是 `nn.Conv2D(in_chans=3, out_chans=D, kernel_size=P, strides=P, padding='VALID')`。等价于把每个 16×16 区域分别做线性投影——物理学和数学上等价，但 GPU 效率高得多。

**2. 初始化策略。** 所有线性层的权重用 `nn.initializers.xavier_uniform()` 初始化，偏置归零。位置编码用 `nn.initializers.normal(stddev=0.02)`。这些细节在论文附录中有提及但未在所有实现中保持一致。

**3. Dropout 的使用。** 论文使用了 embedding dropout（对位置编码 dropout）和 attention dropout，但没有使用 stochastic depth（DropPath）。对比之下，DeiT（Touvron et al., 2021）和 Swin 都大量使用 DropPath。这是 ViT 在 ImageNet-1K 从头训练精度受限的原因之一。

**4. 学习率 warmup + cosine decay。** 预训练使用 AdamW，基础学习率 0.001（ViT-B）或 0.0003（ViT-L/H），线性 warmup 10k steps，然后 cosine decay。和 BERT 的训练配置基本一致。

**5. 权重衰减。** ViT 对位置编码和 [class] token 不做权重衰减，仅对 attention 和 MLP 的权重矩阵做。这是因为位置编码等参数"需要更大的灵活性"。

## 九、位置

### 前驱工作

- **[Vaswani et al., 2017] Attention Is All You Need**：Transformer 的基础架构。ViT 直接复用了 encoder 部分，不做任何修改。
- **[Devlin et al., 2019] BERT**：引入了 [CLS] token 做分类。ViT 沿用这个设计。
- **[Cordonnier et al., 2020]**：最接近 ViT 的前驱工作。同样把图片切成 patch + 全局 self-attention，但 patch size = 2×2（只适合低分辨率），且没有在大规模数据上验证。

### 同期竞品

- **[Touvron et al., 2021] DeiT（Data-efficient Image Transformers）**：一个月后发表在 ICML 2021。在 ViT 基础上加了知识蒸馏、数据增强和正则化，使得 ViT 可以在 ImageNet-1K 上直接从零训练（Swin 也用了 DeiT 的训练技巧）。
- **[Liu et al., 2021] Swin Transformer**：5 个月后发表（arXiv 2021.03）。解决了 ViT 的计算复杂度和尺度两大问题，将 Transformer backbone 从分类扩展到检测/分割。
- **[Chen et al., 2021] Pre-Training Image Paths (BEiT)**：把 BERT 的 Masked Language Modeling 引入视觉，用 ViT 架构做 Masked Image Modeling。

### 后续影响

ViT 是 CV 领域 2020s 最具开创性的工作之一（引用量超过 5 万次），它的影响分层如下：

**直接扩展（Google 系列）**
- **[ViT-G, Zhai et al., 2022] ViT-Giant**：ViT 的极限版本，20 亿参数，用 30 亿弱标注数据预训练，在 ImageNet 上达到 90.45% top-1。
- **[LiT, Zhai et al., 2022]**：用文本监督学习视觉特征，ViT 作为图像编码器。多模态方向的扩展。
- **[SigLIP, Zhai et al., 2023]**：对比学习的改进，也用 ViT 架构。

**架构改进**
- **[DeiT III, Touvron et al., 2022] ResNet Strikes Back**：改良 ViT 训练流程，证明了 ViT 在 ImageNet-1K 从零训练可以达到 Swin 级别的精度。
- **[DINO, Caron et al., 2021] & DINOv2, Oquab et al., 2023**：用 ViT 做自监督学习。DINO 发现 ViT 的注意力图自然分割出物体的各个部分——这是 CNN 自监督方法做不到的。
- **[MAE, He et al., 2022] Masked Autoencoders**：用 ViT 做 masked image modeling——mask 掉 75% 的 patches，只让模型从可见 patches 重建。比 BERT 式的 15% mask 激进得多，ViT 的架构自然支持这种非对称编码。

**训练范式变化**
- ViT 的 patch embedding 和 Transformer encoder 形成了一个事实标准，后续大量工作（CLIP、DALL·E、Stable Diffusion）都用 ViT 做视觉编码器。
- ViT 证明的"大规模数据可以覆盖归纳偏置"这一论点，直接影响了 CLIP（400M 图文对）、DALL·E（250M 图文对）、GPT-4V 等视觉语言模型的训练策略。

## 十、局限

1. **计算复杂度 $O(N^2)$。** ViT 做全局 self-attention，patch 数量 $N$ 随分辨率平方增长。检测/分割所需的 800×1333 输入产生约 4160 个 patch，$4160^2 \approx 17M$ 的 attention 矩阵完全不可行。这是 Swin Transformer 和后续局部 attention 方法的核心动机。

2. **小数据训练差。** 在 ImageNet-1K（1.3M 图片）上从头训练，ViT-B 只有 79.9% top-1，同期 ResNet-50 可以达到 76-78% 但参数量少得多，DeiT（加入蒸馏训练）的 ViT-B 达到 83.8%。没有大规模预训练时，ViT 的归纳偏置不足是个硬伤。

3. **缺少多尺度特征。** ViT 输出单一分辨率（如 $\frac{H}{16}\times\frac{W}{16}\times D$），无法直接做 dense prediction。后续的 Swin、PVT、MViT 都通过层级设计解决了这个问题。

4. **位置编码的处理方式粗糙。** 调整输入分辨率时通过 2D 插值调整 1D 位置编码——这个处理方式语义上不够优雅。ViT 的位置编码长度必须插值，而这引入的误差在高分辨率 fine-tune 时会影响性能。

5. **需要大量计算资源预训练。** ViT-H/14 在 JFT-300M 上预训练需要 2.5k TPUv3-core-days（约 10 天在 250 块 TPU 上）。这是 Google 的算力门槛，不是所有人都能复现的。

## 十一、小结

ViT 的核心贡献不在于发明什么新技术——它的 Transformer encoder、patch embedding、[class] token 全都是已有的。它的贡献在于一个**实验性的发现**：

> 当预训练数据足够大（3 亿图片），标准的 Transformer 可以直接学习 2D 图像结构，不需要 CNN 式的归纳偏置。

这个发现的意义超出了 ViT 本身的精度。它开启了一个新的范式：NLP 和 CV 可以用同样的架构（Transformer）处理不同模态的数据，为后来的多模态模型（CLIP、DALL·E、GPT-4V）铺平了道路。ViT 之后的"CV 架构进化"中，所有主流工作要么是 ViT 的改良（DeiT、MAE、DINO），要么是 ViT 缺点的补全（Swin、PVT），要么是用 ViT 的发现反推 CNN（ConvNeXt）。

如果说 ResNet 定义了 2010s 的视觉架构，ViT 定义了 2020s。
