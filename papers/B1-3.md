# An Image is Worth 16x16 Words: Transformers for Image Recognition at Scale（2020）

## 一、先搞清楚坑在哪

2020 年之前，卷积神经网络（CNN）是计算机视觉的绝对主流。从 AlexNet（Krizhevsky et al., 2012）到 ResNet（He et al., 2016）再到 EfficientNet（Tan & Le, 2019），CNN 架构不断进化——但核心假设不变：局部连接、平移等变性、层级下采样。

与此同时，NLP 领域已经被 Transformer（Vaswani et al., 2017）彻底改变。BERT（Devlin et al., 2019）和 GPT（Radford et al., 2018）证明了：在足够数据上预训练的大规模 Transformer，可以碾压精心设计的 NLP 架构。

一个自然的问题：Transformer 能不能也取代 CNN 成为视觉领域的基础架构？

此前已有尝试把 attention 用在视觉上——在 CNN 内部加 attention（SE-Net、Non-local），或用局部 attention 替代卷积（Ramachandran et al., 2019）。但没有一个工作证明了"纯 Transformer 直接处理图像就能超越 CNN"。

## 二、ViT 之前尝试的真正问题

已有方法主要分为三类：

**1. CNN + attention 混合。** SE-Net（Hu et al., 2018）在卷积特征图上加通道注意力；Non-local Networks（Wang et al., 2018）在 CNN 中间穿插全局 self-attention。这些方法保留了 CNN backbone，attention 只是点缀——Transformer 的潜力没释放。

**2. 局部自注意力替代卷积。** Stand-Alone Self-Attention（Ramachandran et al., 2019）用滑动窗口自注意力替代 3×3 卷积。验证了 attention 可以替代卷积，但实际 GPU 效率极低（key set 不固定导致内存访问散乱）。

**3. 小 patch 全局 attention。** Cordonnier et al.（2020）在 2×2 的极小 patch 上做全局 attention——但 token 量极大（对于 224×224 输入，token 数 12544），只能处理低分辨率。

核心矛盾：注意力机制可以处理图像但效率太低（局部 attention = 内存效率差，全局 attention = 计算量爆炸）。

## 三、ViT 的核心思路

一句话：把图片切成 16×16 的 patch，拉直成序列，直接用标准 Transformer encoder 做分类。

三个关键选择：

1. **最小改动原则**——尽可能复用 NLP Transformer：标准的 Transformer encoder（Vaswani et al., 2017）、[class] token、pre-LN
2. **Patch Size = 16×16**——平衡 token 数（196）和信息粒度。196 个 token 和 BERT 的 512 token 同数量级
3. **规模 > 架构**——在 ImageNet-1K（1.3M 图片）上从头训练，ViT 打不过 ResNet。在 JFT-300M（3 亿图片）上预训练后，ViT 反超

## 四、架构详解

### 整体结构

```
Input: 224×224×3
    │
    ▼
[Patch Embedding]  ─── Conv2d(3, 768, 16, 16) → 196×768
    │
    ▼
[Concat + Position]  ─── [class] token (1×768) + 位置编码
    │                  197×768
    ▼
┌─────────────────────────────────┐
│ Transformer Encoder ×12         │
│ ┌─ LN → MSA → + ─┐             │
│ └─ LN → MLP → + ─┘             │  保持 197×768
└─────────────┬───────────────────┘
              │
              ▼
[class] token output → LN → Classification Head
```

### 模型变体

| 模型 | Layers | Hidden D | MLP | Heads | Params |
|------|--------|----------|-----|-------|--------|
| ViT-Base | 12 | 768 | 3072 | 12 | 86M |
| ViT-Large | 24 | 1024 | 4096 | 16 | 307M |
| ViT-Huge | 32 | 1280 | 5120 | 16 | 632M |

### 输入表示

$$\mathbf{z}_0 = [\mathbf{x}_{\text{class}}; \; \mathbf{x}_p^1\mathbf{E}; \; \mathbf{x}_p^2\mathbf{E}; \; \cdots \; ; \; \mathbf{x}_p^N\mathbf{E}] + \mathbf{E}_{\text{pos}}$$

其中 $\mathbf{x}_p \in \mathbb{R}^{N \times (P^2\cdot C)}$ 是展平的 patches，$\mathbf{E} \in \mathbb{R}^{(P^2\cdot C)\times D}$ 是投影矩阵，$\mathbf{E}_{\text{pos}}$ 是 1D 可学习位置编码。

为什么用 1D 位置编码？Appendix D.4 验证了 2D-aware 位置编码没有显著提升——self-attention 的全局性使得模型可以在注意力权重里自己学习 2D 空间关系。

## 五、核心创新点

严格来说，ViT 没有发明新架构——它的 Transformer encoder、[class] token、位置编码都是已有的。它的贡献是**设计选择和实验验证**。

### (a) Patch Size = 16×16

P=16 产生 N=196 tokens。如果 P=8，N=784（计算量翻 4 倍）。如果 P=32，N=49（空间信息损失太大）。16×16 是 ImageNet 分类上精度与计算量的最佳平衡点。

### (b) [class] token vs 全局平均池化

和 BERT 一样用 [class] token。Appendix D.1 验证了 [class] token 和 GAP 效果相近。

### (c) Pre-LN

原始 Transformer（Vaswani et al., 2017）用 Post-LN（LN 在残差之后），但 Post-LN 在大模型上训练不稳定。Pre-LN（Xiong et al., 2020）把 LN 移动到 MSA/MLP 之前，梯度更稳定。ViT 全用 Pre-LN。

### (d) 大规模预训练 > 归纳偏置

这是 ViT 最重要的发现——论文 Figure 3 的可视化：

```
ImageNet-1K 上：  ViT < ResNet  (小数据，偏置重要)
ImageNet-21K 上： ViT ≈ ResNet  (中等数据)
JFT-300M 上：    ViT > ResNet  (大数据，偏置不重要)
```

在 JFT-300M 上，ViT-H/14 达到 88.55%，持平当时 SOTA 的 EfficientNet-L2（88.4%），且训练成本低得多。

## 六、公式详解

### MSA 复杂度

$$\Omega(\text{MSA}) = 4hwC^2 + 2(hw)^2C$$

QKV 投影（$3hwC^2$）+ 输出投影（$hwC^2$）= $4hwC^2$（线性项）。Attention 矩阵计算（$QK^T: hw \times hw$, $V \times A$）= $2(hw)^2C$（平方项）。

对 ViT-B/16 处理 224×224 输入：$hw = 196$。$2(hw)^2C = 2 \times 196^2 \times 768 \approx 59M$ FLOPs。如果输入 800×1333（COCO 常见），$N \approx 4160$，平方项 $2 \times 4160^2 \times 768 \approx 26.6B$——无法计算。这就是 ViT 不能做检测/分割的原因。

### Transformer Block

$$\mathbf{z}'_\ell = \text{MSA}(\text{LN}(\mathbf{z}_{\ell-1})) + \mathbf{z}_{\ell-1}$$
$$\mathbf{z}_\ell = \text{MLP}(\text{LN}(\mathbf{z}'_\ell)) + \mathbf{z}'_\ell$$
$$\mathbf{y} = \text{LN}(\mathbf{z}_L^0)$$

最后一层取 [class] token（位置 0）过 LN。Pre-LN 的关键：梯度可以直接通过残差连接无损传播。

## 八、代码对照

官方实现是 JAX/Flax 版本。发现论文未写的细节：

**1. Patch Embedding 用 Conv2d 实现。** `nn.Conv2D(in_chans=3, out_chans=D, kernel_size=P, strides=P)` — 和切 patch 再矩阵乘等价，但 GPU 效率高得多。

**2. 初始化策略。** 线性层用 xavier_uniform，位置编码用 normal(stddev=0.02)。

**3. 权重衰减。** 对位置编码和 [class] token 不做权重衰减，仅对 attention 和 MLP 的权重矩阵做。

**4. 学习率调度。** 使用 AdamW，warmup + cosine decay，和 BERT 配置一致。

## 九、位置

### 前驱

- **[Vaswani et al., 2017] Transformer**：基础架构。ViT 直接复用了 encoder，不做修改。
- **[Cordonnier et al., 2020]**：最接近的前驱——patch + 全局 attention，但 patch=2×2 只适合低分辨率。
- **[Devlin et al., 2019] BERT**：引入 [CLS] token，ViT 沿用。

### 后续影响

- **[DeiT, Touvron et al., 2021]**：给 ViT 加上蒸馏训练，使 ImageNet-1K 从头训练可行
- **[Swin Transformer, Liu et al., 2021]**：解决 ViT 的多尺度和计算复杂度
- **[MAE, He et al., 2022]**：ViT + Masked Autoencoding，自监督预训练 SOTA
- **CLIP、DALL·E、Stable Diffusion** 均使用 ViT 作为视觉编码器
- **ViT-G、SigLIP、LiT** 等 Google 系列将 ViT 扩展到更大规模

## 十、局限

1. **$O(N^2)$ 复杂度**。全局 self-attention 在处理高分辨率时不可行。检测/分割需 800×1333 → 4160 tokens → $4160^2 \approx 17M$ 的注意力矩阵。

2. **缺少多尺度特征**。单分辨率输出（$H/16 \times W/16$），不能直接做 dense prediction。

3. **小数据训练差**。ImageNet-1K 上从头训练只有 79.9%，不如 ResNet-50（76.2% 但参数少得多）。

4. **位置编码插值粗糙**。调整分辨率时对位置编码做 2D 插值，语义上不够优雅。

## 十一、小结

ViT 的核心贡献不是发明任何新技术——它的所有组件都是已有的。它的贡献在于一个实验性发现：

$$\text{大规模预训练} + \text{标准 Transformer} > \text{CNN 归纳偏置}$$

当数据从 1.3M 增长到 300M，ViT 从落后 ResNet 到反超 8%——这个发现直接开启了"CV 架构 Transformer 化"的时代。ViT 之后的 CV 架构进化中，所有主流工作要么是 ViT 的改良（DeiT、MAE、DINO），要么是 ViT 缺点的补全（Swin、PVT），要么是用 ViT 的发现反推 CNN（ConvNeXt）。
