## 一、先搞清楚"问题在哪里"

2021 年，视觉自监督学习领域正处在两种范式的角力中。

一边是以 SimCLR、MoCo v3、BYOL 为代表的对比学习（contrastive learning）路线。这些方法的核心思路是：给同一张图片做不同的数据增强得到两个 view，然后拉近它们的表征、推开不同图片的表征。这条路效果很好——MoCo v3 在 ImageNet 线性探针上能到 77.6%——但它极度依赖数据增强。用 BYOL 和 SimCLR 的论文自己的话来说，如果只用 cropping 作为增强（不加 color jitter），精度分别掉 13% 和 28%。

另一边是 masked image modeling（MIM）路线。BERT 在 NLP 上的成功让很多人尝试把"mask 一部分 token 然后预测"这个思路搬到视觉上。2021 年 6 月，BEiT 首次在 ViT 上实现了这个范式的有效训练，证明了"看完剩下的 patch 猜遮掉的 patch"可以学到有用的视觉表征。但 BEiT 做的并不简单——它需要用 DALL-E 的 dVAE 把图片 tokenize 成离散 token，然后像 BERT 一样预测被 mask token 的索引。

这就在视觉自监督领域留下了一个很尴尬的 gap：**对比学习靠的是复杂的增强策略，MIM 靠的是复杂的 tokenizer。有没有一种方法，像 BERT 那样简单直接——mask 掉一部分输入，直接预测原始内容（比如像素），就能学到好表征？**

MAE 就是在这个问题背景下出现的。

在这之前，还有几件看似不相关但给了 FAIR 团队关键线索的事：

- **ViT（2020 年 10 月）** 证明了纯 Transformer encoder 可以做图像 backbone——这就意味着 BERT 式的 mask tokens 可以原生地插入 patch 序列里，不再受限于卷积的规则网格。
- **NLP 的经验**表明，**mask 比例越高，任务越难，学到的表征越好**。BERT 的 mask ratio 是 15%，但这是语言——信息密度极高，15% 的 mask 已经能创造足够好的预训练任务。图像有严重的空间冗余，邻接像素可以通过简单的插值补全。所以一个直觉是：**视觉上的 mask ratio 应该远高于 15%**。
- **iGPT（2020）** 尝试了 pixel-level 的自回归预测，但 1362M 参数的 iGPT-L 在 ImageNet 线性探针上只有 69.0%，远低于对比学习方法。它在像素级做预测，模型参数主要花在了记忆像素模式上，而不是学高级语义。

所以 MAE 的作者要回答的核心问题是：**如何设计一个 vision 版的 masked autoencoder，让它既简单（不要 tokenizer）、又高效（训练快）、还能学到比对比学习更好的表征？**

答案浓缩成三个设计选择：**高比例 mask + 非对称 encoder-decoder + 预测原始像素**。

## 二、旧路线的根本问题

MAE 论文的第一节非常精彩，作者提出了方法论层面的三个视角，解释了为什么 NLP 的 masked autoencoding 在视觉上一直不太成功。

### 2.1 架构的 barrier 已经不存在了

第一个视角（(i)）很直接：在卷积网络里，"被遮掉的区域"很难原生的作为特殊 token 插入。卷积的输入是一张规整的网格，没有"序列"的概念。ViT 把图片切成 patch 序列，这使得 BERT 风格的 mask tokens 可以原生的加进来。所以从 ViT 出现开始，架构层面的障碍已经不存在了。

### 2.2 信息密度不同：视觉的冗余是语言无法比的

第二个视角（(ii)）是最关键的分析。作者指出：

> 语言是人类生成的信号，语义密度极高。训练模型预测一个句子里少数几个被遮掉的词，这个任务本身就能诱导出复杂的语言理解。相反，图像是自然信号，有严重的空间冗余——一个缺失的 patch 可以从邻近 patch 通过简单的插值恢复，不需要对物体有高级理解。

这句话直击本质。BERT 在 NLP 上 mask 15% 的 token 就够了，因为每个被遮的词都有丰富的语义。但在图像上，如果你只 mask 15% 的 patch，剩下的 85% 足以让模型通过局部平滑和纹理延续来"猜"出被遮的内容，不需要理解"这是一只猫"。

**所以 MAE 的核心策略是：mask 一个非常高的比例（75%），把空间冗余大幅缩减，迫使模型去做更高级的推理。**

### 2.3 视觉的 decoder 和 NLP 的作用完全不同

第三个视角（(iii)）更有意思。在 BERT 中，decoder（其实就是一个 MLP）做的事情是"预测被遮掉的词"，输出的是一个语义 token。但在视觉 autoencoder 中，decoder 要重建像素——像素的语义层级远低于识别任务所需要的表征。这意味着：

- **如果要线性探针效果好**（即 encoder 的输出已经高度线性可分），decoder 必须足够深，把"重建像素"相关的专有化表征吸收掉，让 encoder 的输出更抽象。
- **但如果是端到端微调**（fine-tune encoder 的最后几层），那么 decoder 浅一点也无所谓，因为微调过程会让 encoder 适应识别任务。

这个观察直接驱动了 MAE 的**非对称 encoder-decoder 设计**——encoder 很深（ViT-L 是 24 层），decoder 很浅（默认 8 层，宽度 512-d）。

## 三、这篇论文的核心思路

**MAE 的核心思路可以浓缩成一句话：随机 mask 输入图像的 75% 的 patch，让一个非对称的 ViT encoder（只处理 visible patches）和轻量 decoder（同时处理 encoded visible + mask tokens）重建被遮掉的像素。**

核心有三板斧：

```text
高比例随机 mask（75%）         →  消除空间冗余，创造有意义的 self-supervisory task
       ↓
非对称 encoder-decoder        →  25% 的 patch 进 encoder → 3×+ 训练加速
encoder 只看 visible patches
decoder 用 mask tokens + visible
       ↓
预测原始像素（不是离散 token） →  不需要 dVAE，不需要额外数据，更简单
```

三个设计共同构成了一个"win-win"场景：

- **对准确率好**：75% mask 是最优的 mask ratio（Figure 5），在 fine-tuning 和 linear probing 上都最好
- **对效率好**：encoder 只处理 25% 的 patch，计算量和内存降到 1/4 以下，可以轻松 scale 到 ViT-Huge

这条技术路线的谱系位置：

```text
iGPT (2020)                     ← 像素级自回归生成，不缩放
   ↓
ViT (2020)                      ← 证明 Transformer 可以做视觉 backbone
   ↓
BEiT (2021.06)                  ← MIM 先驱，预测离散 token
   ↓
MAE (2021.11)                   ← ★ 本篇 ★ 预测像素，非对称设计，最简洁的 MIM
   ↓
EVA / EVA-02 (2022/2023)        ← MIM 尺度扩展
```

MAE 和 BEiT 的两个核心区别：

| 对比维度 | BEiT | MAE |
|---------|------|-----|
| 预测目标 | 离散 token（dVAE tokenizer） | 原始像素（可直接归一化后使用） |
| Encoder 输入 | 所有 patch（含 mask tokens） | 只有 visible patches（无 mask tokens） |
| Decoder | 跟 encoder 融合在一起 | 独立的轻量 decoder |
| 额外数据依赖 | 依赖 DALLE 的 250M 图片预训练 | 不需要 |
| 单 epoch 训练速度 | 慢（100% patch 进 encoder） | 快约 3-4 倍（25% patch 进 encoder） |

## 四、整体架构与 forward 流程

### 4.1 整体架构

MAE 的架构出奇的简单，比 BEiT 和对比学习都简单：

```text
输入图像: H×W×3 (e.g., 224×224×3)
    │
    ▼
[Patch Embedding]              ViT: 16×16 patches → 14×14 = 196 tokens
    │                          线性投影 + position embedding
    ▼
[随机 Shuffle + 去尾]           保证只有 25% 的 patch（49/196 tokens）保留
    │                          mask ratio = 75%
    ▼
┌═══════════════════════════════┐
║  ViT Encoder (大)             ║
║  只处理 visible patches       ║
║  ViT-L: 24 层 Transformer    ║
║  ViT-H: 32 层 Transformer    ║
║  输出: encoded visible tokens ║
└════════┬══════════════════════┘
         │
         ▼
[拼接 mask tokens + unshuffle]   mask token 是共享可学习向量
    │                          加上 position embedding
    ▼
┌───────────────────────────────┐
│  MAE Decoder (小)             │
│  默认 8 层 Transformer        │
│  宽度 512-d（<9% ViT-L FLOPs）│
│  处理所有 196 个 token        │
│  输出: 每个 patch 的像素值     │
└───────────────┬───────────────┘
                │
                ▼
[MSE Loss]                    只对被 mask 的 patch 算 loss
```

### 4.2 前向过程逐步跟踪

**Step 0 — 输入准备**

输入 RGB 图像，尺寸 224×224×3。用 16×16 的 patch size，得到 14×14=196 个 patch。

**Step 1 — 随机采样（masking）**

关键操作：**随机 shuffle token 列表，保留前 25%，扔掉后 75%**。

```python
# 伪代码：shuffle + 去尾
tokens = linear_proj(patches) + pos_embed  # [B, 196, dim]
shuffle_indices = random.shuffle(range(196))
keep = tokens[shuffle_indices[:49]]         # 25%
masked = tokens[shuffle_indices[49:]]       # 75%
```

为什么不直接用"mask ratio 75% 的 mask"而是构造这样的采样？因为不需要任何稀疏操作——直接在常规密集矩阵上操作即可。shuffle + 去尾是实现"没有 replacement 的随机采样"最干净的方式。

**Step 2 — Encoder 前向**

Encoder 只处理那 49 个 visible token。它是一个标准的 ViT（ViT-L 24 层 / ViT-H 32 层），包含：

- Multi-head self-attention（patch 之间）
- MLP block（2 层 FC + GELU）
- Pre-LN + residual connection

全程操作在 49 个 token 上（而不是 196 个）。对于 ViT-L（1024-d），这是大约 25% 的序列长度和 100% 的 model width。计算量减少了 3.3×（attention 层 O(n²) 意味着减少约 16 倍的计算量）。

**Step 3 — 拼接 mask tokens + unshuffle**

编码完成后，49 个 encoded visible tokens 和 147 个 mask tokens 拼接在一起，然后 **unshuffle**——即反方向的 shuffle，恢复原始的 196 个位置的顺序。这样每个位置都对应回原始的 patch 位置。

Mask token 是一个共享的可学习向量——所有被遮掉的 patch 都用同一个 mask token 初始值，但它会在训练中学会携带"这里被遮掉了"的信号。

**关键细节：** Position embeddings 是加在这个 full set 上的（196 个位置都有），而不是只在 encoder 部分加。如果没有这个，mask tokens 不知道自己在图像中的位置。

**Step 4 — Decoder 前向**

Decoder 处理 196 个 tokens（49 encoded visible + 147 mask tokens）。它也更浅更窄：

- 默认：8 层 Transformer，宽度 512
- 对比 ViT-L：24 层 Transformer，宽度 1024

按作者说法，decoder 每个 token 的计算量不到 encoder 的 10%（"<10% computation per token vs. the encoder"）。具体数字：9% FLOPs per token vs ViT-L。

**Step 5 — 重建 + Loss**

Decoder 最后一层是一个线性投影，输出通道数等于一个 patch 的像素数（16×16×3=768）。reshape 后得到完整的重建图像。

Loss 是 MSE，**只对被遮掉的 patch 计算**（147 个）。作者发现如果在所有 patch 上算 loss，精度掉大约 0.5%（Figure 5 脚注提到的）。

### 4.3 为什么这种非对称设计好

核心洞察：**把 mask tokens 移到小 decoder 里处理，而不是在 encoder 里处理**。好处是双重的：

1. **准确率提升**：Encoder 始终只看到真实 patch（没有 mask token），这消除了预训练和部署之间的 gap——推理时 encoder 也是输入全部真实 patch。Table 1c 显示，如果在 encoder 中使用 mask tokens（像 BERT 那样），线性探针从 73.5% 掉到 59.6%，掉了 14 个点。

2. **计算量节省**：Encoder 只处理 25% 的 patch，自注意力的二次复杂度 O(n²) 被砍到原来的约 1/16。综合来看，**整体训练 FLOPs 减少了 3.3×**（Table 1c）。实际 wall-clock 加速效果：ViT-L 从 42.4 小时减少到 15.4 小时（2.8× 加速），如果 decoder 再浅一点（1-block），能到 11.6 小时（3.7× 加速）。对于 ViT-H，用 1-block decoder 直接到 4.1× 加速（Table 2）。

## 五、核心模块逐个解释

### 5.1 高比例随机 mask

这是 MAE 最重要的消融实验之一（Figure 5）。

**核心发现：mask ratio 75% 是最优点。**

```text
Fine-tuning 精度随 mask ratio 变化:
  Ratio  10%   20%   30%   40%   50%   60%   70%   75%   80%   90%
  Acc    83.0  83.4  83.4  84.5  84.7  84.9  85.0  84.9  84.9  83.2

Linear probing 精度随 mask ratio 变化:
  Ratio  10%   20%   30%   40%   50%   60%   70%   75%   80%   90%
  Acc    54.6  58.9  61.7  66.1  67.0  69.9  71.8  73.5  73.2  71.8
```

几个关键观察：

**第一，最优 mask ratio 远比 BERT 的 15% 高。** 这是视觉和语言信息密度差异的直接结果。图片的局部邻域 patch 高度相关，必须 mask 得足够多才能迫使模型做"整体理解"而不是"纹理外推"。

**第二，fine-tuning 对 mask ratio 相对不敏感**（40%-80% 范围内都超过 84.5%），但 linear probing 高度敏感——75% 比 10% 高了近 19 个点。这说明高 mask ratio 让 encoder 学到了更抽象的表征，微调可以"纠正"掉一些不好的特征，但线性分类器无能为力。

**第三，100% mask 的极端情况也没试**（90% 已经很极端了，只有 19/196 个 patch），精度开始下降。这说明尽管要消除冗余，也还是需要有足够的可见信息支撑推理。

### 5.2 随机采样策略

MAE 用最简单的均匀随机采样（random sampling）。论文 Table 1f 对比了两种替代方案：

**Block-wise masking（BEiT 的方式）**：遮掉大的连续区域。在 50% ratio 时工作良好（83.9%），但到 75% 时显著下降（82.8% fine-tuning, 63.9% linear probing）。训练 loss 更高，重建更模糊。这说明 block masking 在 75% 时制造的任务太难了——连续大块区域被遮掉，剩下的上下文完全不足以做合理推断。

**Grid-wise masking**：规则地每四个 patch 保留一个。这是一个更容易的任务（训练 loss 更低，重建更清晰），但是表征质量也更低（84.0% fine-tuning, 66.0% linear probing）。因为规律网格允许模型依赖"已知的结构"来插值，不需要真正理解语义。

随机采样在中间找到了最好的平衡——既足够具有挑战性，又有足够的 random structure 避免任务变得 trivial。

### 5.3 Decoder 设计选择

**Decoder depth（Table 1a）：**

| Decoder layers | Fine-tuning | Linear probing |
|---------------|-------------|----------------|
| 1 | 84.8 | 65.5 |
| 2 | 84.9 | 70.0 |
| 4 | 84.9 | 71.9 |
| 8 | 84.9 | 73.5 |
| 12 | 84.4 | 73.3 |

这个消融非常说明问题：

- 对于 fine-tuning，**1 层 decoder 就够了**（84.8% vs 8 层的 84.9%，基本持平）。说明微调过程中，encoder 最后几层会被重新调整来适应识别任务，一个浅 decoder 留下的"像素专化层"很少。
- 对于 linear probing，depth 非常关键。1 层只有 65.5%，8 层 73.5%（+8%）。因为线性探针冻结了 encoder，encoder 输出必须已经足够抽象，而"像素重建"的专化层需要被 decoder 吸收掉。

**Decoder width（Table 1b）：**

| Decoder width | Fine-tuning | Linear probing |
|--------------|-------------|----------------|
| 128 | 84.8 | 69.1 |
| 256 | 84.8 | 71.3 |
| 512 | 84.9 | 73.5 |
| 768 | 84.4 | 73.1 |
| 1024 | 84.3 | 73.1 |

Width 512 是最优的平衡点——再宽反而下降（可能是因为过拟合到了像素重建任务上）。注意 decoder 的宽度和 encoder（1024）形成了鲜明的不对称。

**默认 decoder 设置：8 层，512 width，每个 token 的 FLOPs 约为 ViT-L 的 9%。**

### 5.4 Mask Token 处理

MAE 最重要的设计选择之一：**不在 encoder 中用 mask token**。

Table 1c 的数据非常有力：

```text
                        Fine-tuning    Linear probing    FLOPs 比
encoder w/ [M] token     84.2           59.6              3.3×
encoder w/o [M] token    84.9           73.5              1× (baseline)
```

两种设计差异巨大：

- 如果在 encoder 中使用 mask token，encoder 在预训练时看到的是"真实 patch + mask token"的混合输入。但部署时，所有 patch 都是真实的。这个 gap 导致了线性探针从 73.5% 掉到 59.6%（-14%）。Fine-tuning 可以部分补偿这个 gap（因为微调过程中 encoder 会重新适应），但 84.2% vs 84.9% 仍然有 0.7 个点的差距。
- 去掉 mask token 后，计算量减少了 3.3×——不仅因为序列长度变短（25%），还因为 self-attention 的二次复杂度。

### 5.5 重建目标：像素 vs token

MAE 最革命性的结论之一：**直接用像素做重建目标就足够了**，不需要 BEiT 式的离散 token。

Table 1d 的数据：

```text
                              Fine-tuning    Linear probing
pixel (w/o normalization)      84.9           73.5
pixel (w/ normalization)       85.4           73.9
PCA                            84.6           72.3
dVAE token                     85.3           71.6
```

几个关键点：

- **归一化后的像素是最优的**（85.4% fine-tuning, 73.9% linear probing）。所谓的 per-patch normalization：对每个 patch 内所有像素算均值和标准差，然后用它们归一化整个 patch。这个操作增强了 patch 内部的局部对比度。
- **dVAE token 和归一化像素相比没有优势**（85.3% vs 85.4% fine-tuning, 71.6% vs 73.9% linear probing）。在 Table 7（下游迁移实验）中，两者的差异也是统计上不显著的（"statistically insignificant"）。
- **PCA 降维到 96 个系数效果反而更差**（84.6%, 72.3%）。作者推断，高频成分（在 PCA 中被舍弃了）对 MAE 的表征学习是有用的。

**像素目标带来的巨大好处：更简单**。

dVAE tokenizer 需要额外的预训练阶段（在 250M 图片上），tokenizer 本身的 encoder 就是一个大型卷积网络（"40% FLOPs of ViT-L"）。用像素做目标完全不需要这些东西，开箱即用。**这是 MAE 相比 BEiT 最大的实战优势。**

### 5.6 数据增强：出奇地"不需要"

对比学习方法极度依赖数据增强——SimCLR、BYOL、MoCo 全都需要 aggressive 的 color jitter、grayscale、Gaussian blur 等来创造不同的 view。MAE 的作者发现了一个令人惊讶的事实：

**MAE 只用 cropping（甚至不用 color jitter）就能工作得很好。**

Table 1e 的数据：

```text
Augmentation                    Fine-tuning    Linear probing
none (center-crop only, no flip)  84.0          65.7
crop, fixed size                   84.7          73.1
crop, random size                  84.9          73.5
crop + color jitter                84.3          71.9
```

不加任何增强（只做中心裁剪，不做翻转）居然有 84.0%——这对比之下 SimCLR/BYOL 如果只用 cropping，精度掉了 13%-28%。

**为什么 MAE 不太需要数据增强？** 因为 **random masking 本身就起到了类似增强的作用**。每次迭代的 mask pattern 都不一样，相当于每张图被"随机遮挡"成了不同的训练样本。masking 让预训练任务本身变得更难（需要更多正则化），相比之下数据增强的作用就弱化了。

### 5.7 训练计划（scaling behavior）

Figure 7 显示了一个关键现象：**MAE 准确率随训练 epoch 持续提升，没有饱和迹象**，即使到了 1600 epoch 也还在涨。

这与对比学习形成鲜明对比：MoCo v3 在 ViT-L 上 300 epoch 就饱和了。为什么？因为 **MAE encoder 每个 epoch 只看到 25% 的 patch**，而对比学习 encoder 每个 epoch 看到 200%（two-crop）甚至更多（multi-crop）的 patch。所以 MAE 的每个 epoch 信息量更少，需要更多 epoch 来达到收敛。

这也意味着：**MAE 的预训练时间并不比对比学习长**，尽管 epoch 数大很多。具体数字：MAE ViT-L 1600 epoch 在 128 TPU-v3 上需要 31 小时，MoCo v3 300 epoch 需要 36 小时。对，你没看错——MAE 跑 1600 epoch 比 MoCo v3 跑 300 epoch 还快。

## 六、核心公式与数学直觉

### 6.1 前向公式

给定输入图像 $x \in \mathbb{R}^{H \times W \times 3}$ ，划分成 $N = HW / P^2$ 个 patch（默认 ViT 的 $P=16, N=196$ ）。

随机采样 mask 集 $M \subset \{1, ..., N\}$ ， $\left|M\right| = \lfloor rN \rfloor$ 。默认 mask ratio $r = 0.75$ ，保留可见集 $V = \{1, ..., N\} \setminus M$ 。

Encoder $f_\theta$ 只处理可见 patch：

$$z_V = f_\theta(\{x_i + pos_i\}_{i \in V})$$

Decoder $g_\phi$ 接收 $z_V$ 和 mask tokens $m_i$ （共享可学习向量，共 $\left|M\right|$ 个），拼接并按原始位置排序：

$$\hat{x} = g_\phi(\text{unshuffle}(\{z_i\}_{i \in V} \cup \{m_i\}_{i \in M}) + pos_{1:N})$$

Loss 是 MSE，只对 mask patch 计算：

$$\mathcal{L} = \frac{1}{\left|M\right|} \sum_{i \in M} \| \hat{x}_i - x_i \|_2^2$$

### 6.2 计算复杂度分析

non-asymmetric encoder（用 mask token，所有 patch 进 encoder）的 attention 复杂度：

$$\Omega(\text{MSA}_{\text{standard}}) = 4 N C^2 + 2 N^2 C$$

MAE encoder 只处理 $N_V = (1-r)N$ 个 token：

$$\Omega(\text{MSA}_{\text{MAE}}) = 4 N_V C^2 + 2 N_V^2 C$$

当 $r = 0.75$ 时， $N_V = 0.25 N$ 。Attention 项 $2 N^2 C \rightarrow 2 (0.25 N)^2 C = 0.125 N^2 C$ ，减少了 **8 倍**。

实际加速比（包括 QKV 投影等其他操作）在 Table 2 中测量约为 2.8-4.1×（取决于 decoder 深度和 encoder 大小）。这个加速随 mask ratio 升高而增大，因为 self-attention 的二次复杂度将小序列的优势放大了。

### 6.3 为什么 high mask ratio 有效

这是一个信息论直觉。图像的空间冗余可以被描述为：给定一个邻域像素的集合，预测被遮像素的条件熵 $H(X_{\text{masked}} | X_{\text{visible}})$ 。

当 mask ratio 较低时（如 15%）， $X_{\text{visible}}$ 包含了几乎所有需要的局部信息， $H$ 很小——模型可以通过简单的纹理外推来解决，不需要理解物体级语义。

当 mask ratio 达到 75% 时，可见信息大幅减少， $H$ 变大。这时模型必须依赖"整体认知"——比如"这是一个猫的图片，猫有眼睛、耳朵、毛皮，所以最可能是这样的"—才能做合理的重建。**这个"整体认知"正是我们想让 encoder 学到的语义表征。**

作者通过 Figure 4 做了定性验证：用一个 75% mask ratio 预训练的 MAE 去重建 85%、95% mask 的图片，模型的预测虽然和 ground truth 不同，但语义上是合理的（比如推测了一个合理的物体形状）。

**Table 1b — Decoder width**
- 同样，fine-tuning 对 width 不敏感

**Table 1c — Mask token in encoder**
- **最重要的消融**：encoder 用 mask token 会让 linear probing 从 73.5% 掉到 59.6%

- 不需要 color jitter，甚至几乎不需要数据增强
- No augmentation 仍有 84.0%

**Table 1f — Mask sampling**
- Random > Grid > Block

### 7.5 Figure 9 — Partial Fine-tuning

在 $4.3 节中，作者做了一个非常有意思的实验：**只微调 encoder 的最后几层，冻结前面所有层**。

```text
# blocks tuned    0      1      4      6      12     18     24
MAE                73.5   81.0   84.2   84.4   84.7   84.9   84.9
MoCo v3            77.6   79.9   81.6   81.9   83.2   83.8   84.1
```

关键发现：

1. **MAE 在 linear probing 上低于 MoCo v3**（73.5% vs 77.6%）。这印证了 MAE 的表征不是线性可分的，而是更"非线性"的。

2. **但只要微调一个 Transformer block（不是全部 24 个），MAE 就反超了**（81.0% vs 79.9%）。微调"半个"block（只有 MLP 部分）就能到 79.1%，远超线性探针。

3. **微调的 block 越多，MAE 的优势越大**。4 个 block 时领先 2.6%（84.2% vs 81.6%），全微调时领先 0.8%（84.9% vs 84.1%）。

4. **MAE 的线性可分性差，但非线性特征更强**。这也符合直觉：masked autoencoder 学到的表征是"理解图像"的，而不是"让线性分类器好分"的。NLP 中 BERT 的 linear probing 同样比不过对比学习方法，但 fine-tuning 更好。

这个实验对社区的 linear probing 风潮是一个重要提醒：**线性探针只是表征质量的一个维度，不是全部。**

### 10.2 核心模型 `models_mae.py`

```python
class MaskedAutoencoderViT(nn.Module):
    def __init__(self, img_size=224, patch_size=16, in_chans=3,
                 embed_dim=1024, depth=24, num_heads=16,        # encoder
                 decoder_embed_dim=512, decoder_depth=8,        # decoder
                 decoder_num_heads=16, mlp_ratio=4.,
                 norm_layer=nn.LayerNorm, norm_pix_loss=False):

        # Patch embedding
        self.patch_embed = PatchEmbed(img_size, patch_size, in_chans, embed_dim)

        # Encoder
        self.blocks = nn.ModuleList([
            Block(embed_dim, num_heads, mlp_ratio, qkv_bias=True)
            for _ in range(depth)
        ])

        # Decoder
        self.decoder_embed = nn.Linear(embed_dim, decoder_embed_dim)
        self.mask_token = nn.Parameter(torch.zeros(1, 1, decoder_embed_dim))
        self.decoder_pos_embed = nn.Parameter(...)  # 可学习的位置编码
        self.decoder_blocks = nn.ModuleList([
            Block(decoder_embed_dim, decoder_num_heads, mlp_ratio, qkv_bias=True)
            for _ in range(decoder_depth)
        ])
        self.decoder_pred = nn.Linear(decoder_embed_dim, patch_size**2 * in_chans)
```

前向函数 `forward_encoder` 实现了 shuffle + keep 的核心操作：

```python
def forward_encoder(self, x, mask_ratio):
    # patch embedding
    x = self.patch_embed(x)  # [B, N, D] 其中 N = 196, D = embed_dim

    # add pos embed w/o cls token
    x = x + self.pos_embed[:, 1:, :]  # 去掉 cls token 位置

    # masking: 生成噪音排列，保留前 keep 个
    N = x.shape[1]
    keep = int(N * (1 - mask_ratio))

    noise = torch.rand(B, N, device=x.device)  # 随机噪音
    ids_shuffle = torch.argsort(noise, dim=1)  # 排序得到随机排列索引
    ids_keep = ids_shuffle[:, :keep]            # 取前 keep 个
    x_masked = torch.gather(x, dim=1, index=ids_keep.unsqueeze(-1).repeat(1,1,x.shape[-1]))

    # encoder transformer
    for blk in self.blocks:
        x_masked = blk(x_masked)

    return x_masked, ids_restore  # ids_restore 用于 decoder 的 unshuffle
```

`forward_decoder` 接收 encoded tokens，加上 mask tokens，unshuffle，然后过 decoder：

```python
def forward_decoder(self, x, ids_restore):
    # 用线性投影把 encoder 输出映射到 decoder 维度
    x = self.decoder_embed(x)  # [B, keep, decoder_dim]

    # 拼接 mask tokens
    mask_tokens = self.mask_token.repeat(x.shape[0], ids_restore.shape[1] - x.shape[1], 1)
    x_ = torch.cat([x, mask_tokens], dim=1)

    # unshuffle: 恢复原始顺序
    x = torch.gather(x_, dim=1, index=ids_restore.unsqueeze(-1).repeat(1, 1, x.shape[2]))

    # 加 decoder 位置编码
    x = x + self.decoder_pos_embed

    # decoder transformer
    for blk in self.decoder_blocks:
        x = blk(x)

    # 预测每个 patch 的像素值
    x = self.decoder_pred(x)  # [B, N, 768]

    return x
```

### 10.3 论文 vs 代码：几个实现细节

**1. Shuffle 的方式。** 代码用的是 `torch.argsort(torch.rand(...))` 而不是 `torch.randperm`。原因是这样可以通过 `ids_shuffle` 和其逆映射 `ids_restore` 精准控制 keep/remove 的索引对应关系，不像 `randperm` 需要额外记录。

**2. Position embeddings 选择 sine-cosine。** 作者说"our MAE adds positional embeddings (the sine-cosine version) to both the encoder and decoder inputs"。相比 ViT 的可学习位置编码，sine-cosine 不需要额外参数，而且天然支持更长序列（可以插值到不同的输入尺寸）。

**3. 没有 cls token。** MAE 训练时在 encoder 输入前加了一个 dummy token（辅助线性分类时做 class token），但推理时用 average pooling 效果一样好。

**4. 没有相对位置编码或 layer scaling。** 作者明确说"our MAE does not use relative position or layer scaling (which are used in the code of [2])"，但微调时为了对齐 在 seg 等任务中加上了相对位置偏置（初始化为 0）。

**5. Decoder 线性投影。** 由于 encoder 和 decoder 宽度不同（1024 vs 512），中间有一个 `decoder_embed` 线性投影层做维度变化。

**6. Per-patch normalization 的计算。** 在代码中，如果 `norm_pix_loss=True`，loss 计算前会对每个 patch 内的像素做：

```python
mean = target_patch.mean(dim=-1, keepdim=True)
var = target_patch.var(dim=-1, keepdim=True)
target = (target_patch - mean) / (var + 1e-6)**.5
```

MAE 处于 **A1-4（encoder-only 视觉预训练）** 和 **B4-9（masked image modeling）** 的交汇点。它同时回答了两个问题：

1. **A1 路线的问题**：BERT 风格的 encoder-only 预训练能否直接迁移到视觉？答案是能——只要做对 mask 比例和架构设计。
2. **B4 路线的问题**：不依赖标签的情况下，能否学到通用的视觉表征？答案也是能——MIM 可以作为 foundation model 的训练范式，且比对比学习更简单。

### 11.2 前驱工作：MAE 继承了谁？

**ViT（Dosovitskiy et al., 2020）：** MAE 直接使用了 ViT 的 patch embedding 和 Transformer block 结构。Encoder 就是不加修改的 ViT。可以说 MAE 是 **ViT 的自监督训练方案**。

**BERT（Devlin et al., 2019）：** MAE 继承了两个关键设计：
- 随机 mask 一部分输入，预测被遮内容
- 使用 mask token
- Loss 只算在被遮部分上

但 MAE 在 BERT 的基础上做了两个重要改动：mask 比例从 15% 提升到 75%，以及 decoder 的独立性增强。

**BEiT（Bao et al., 2021.06）：** BEiT 证明了 masked image modeling 在 ViT 上可行。MAE 在 BEiT 的基础上做了关键简化：去掉 dVAE tokenizer，用像素做目标；采用非对称 encoder-decoder 省计算。可以说是 **BEiT 的"极简主义"版本**。

**Denoising Autoencoders（Vincent et al., 2008）：** MAE 在形式上是 DAE 的一种——输入被破坏（mask），输出重建原始信号。但不同之处在于 DAE 算所有像素的 loss，MAE 只算被遮部分的 loss。

### 11.3 同期竞品：MAE 和谁竞争？

**MoCo v3（Chen et al., 2021.08）：** 对比学习的代表。MoCo v3 把 MoCo 的对比框架适配到 ViT 上，在 ViT-L 上达到 84.1%（比 MAE 的 85.9% 低 1.8%）。但 MoCo v3 的 linear probing 更好（77.6% vs 73.5%），而且训练更省时（300 epoch vs 1600 epoch）。

**DINO（Caron et al., 2021.11）：** 自蒸馏方法。在 ViT-B 上 82.8%，比 MAE 的 83.6% 低 0.8%。DINO 的一个有趣发现是 ViT 的自注意力图在某些层表现出物体分割的能力，这在 MAE 中不显著。

**BEiT（Bao et al., 2021.06）：** MAE 最重要的竞品和对比对象。MAE 在几乎所有任务上超过了 BEiT（且更简单更快），标志着 MIM 范式从"预测 token"转变为"预测像素"。

### 11.4 后续影响：MAE 启发了谁？

**SimMIM（Xie et al., 2022）：** 作者把 MAE 的思路移植到 Swin Transformer 上，验证了 MIM 在层级 backbone 上的有效性。SimMIM 使用更大的 mask（窗口大小的块而不是 patch），证明了 MAE 的核心洞察（高比例 mask + 预测像素）可以推广到非 ViT 架构。

**MaskFeat（Wei et al., 2022）：** 提出用 HOG 特征（方向梯度直方图）作为重建目标，而不是原始像素。HOG 特征具有较弱的语义，但降低了重建难度。结果：在 ViT-L 上达到 86.7% 的微调精度，略高于 MAE。

**EVA / EVA-02（Fang et al., 2022/2023）：** 把 MAE 扩展到更大的数据规模和模型规模。EVA 引入 MIM + CLIP 蒸馏的方式，EVA-02 进一步简化了 MIM 的设计。

**ConvNeXt（Liu et al., 2022）：** 虽然不直接是 MAE 的后继，但 ConvNeXt 的文章中用了 SimMIM（Swin 版的 MAE）做自监督预训练，证明了 MAE 模式对现代 CNN 同样有效。

**VideoMAE（Tong et al., 2022）：** 将 MAE 扩展到视频领域。视频的时空冗余比图像更大，所以 VideoMAE 用了更高的 mask ratio（90-95%）。在 Kinetics-400 上达到 SOTA。

**整体影响：**
- 截至 2024 年已被大量引用，成为 MIM 领域最具影响力的工作之一
- 重新唤醒了 masked image modeling 这一研究路线
- 推动社区从对比学习回归到更简单的自监督范式
- 证明了"直接预测像素"是可行的，不需要复杂的 tokenization
- 启发了后续大量 MIM 变体（mask ratio，mask 策略，reconstruction target 的多个方向）

## 十三、范式意义

**MAE 的范式意义：它证明了"最简单的设计往往是最好的"——在 MIM 这个领域把复杂度降到了最低，同时效果提到了最高。**

具体来说，MAE 完成了三件事：

### 13.1 视觉版 BERT，而且更简单

BERT 的 masked LM 在 NLP 上取得了巨大成功，但社区一直困惑为什么在视觉上效果不佳。MAE 给出了清晰的答案：**不是方法的问题，是细节的问题**——mask ratio 不够高 + encoder 设计不合理。

MAE 证明了：只要 mask ratio 足够高（75%）、encoder 只处理 visible patch，视觉版的 BERT 不仅有效，而且比对比学习更好。

### 13.2 去掉 tokenizer 是可行的

2021 年的 MIM 研究有一个隐含假设：**因为像素是低级的，所以需要 tokenizer 把像素变成高级语义单元（token）**。BEiT 依赖于 DALL-E 的 dVAE tokenizer，iGPT 也用了 pixel-level 的 tokenization。

MAE 打破了这个假设。它直接预测归一化像素，效果和用 dVAE token 完全一样。这证明了一个反直觉的事实：**即使训练目标是低级的像素重建，encoder 仍然可以学到高级的语义表征。** 关键在于 MAE 的高比例 mask 迫使 encoder 去做"推理"而不是"插值"。

### 13.3 MIM 是可以 scale 的

对比学习在 ViT-H 上出现性能趋缓（MoCo v3 未在标准设置下报告 ViT-H 结果），但 MAE 在 ViT-H 上继续提升（83.6% → 85.9% → 86.9%）。MIM 的 scaling behavior 更像 NLP：模型越大、数据越多，增益越大。

这意味着 MIM 有潜力成为视觉 foundation model 的预训练范式。

### 13.4 分类：foundation node

在技术树中，MAE 是一个 foundation node。它建立了 "predict pixels after high-ratio masking" 这个 MIM 新基线，后续大量 MIM 变体（SimMIM, MaskFeat, VideoMAE, EVA, EVA-02）都是在这个基线上做增量改进。

## 十四、局限性与后续方向

### 14.1 Linear probing 不如对比学习

MAE 在 linear probing 上明显低于 MoCo v3（75.8% vs 77.6% for ViT-L, 见 Table 12）。这限制了它在"特征需要做线性分类"的场景（如一些迁移学习基准）的使用。

但作者通过 partial fine-tuning 实验（Figure 9）很好地回应了这个问题：**linear probing 只是一个维度，微调一个 block 后 MAE 就全面反超了**。

### 14.2 需要大量 epoch 才能收敛

MAE 在 1600 epoch 远未饱和（linear probing 仍在上升），这意味着如果计算资源有限，MAE 可能不是最优选择——MoCo v3 在 300 epoch 就能得到体面的结果。

不过训练效率（每 epoch 的时间）部分补偿了这个问题：MAE 1600 epoch = 31 小时，MoCo v3 300 epoch = 36 小时。

### 14.3 没有探索更大的模型和数据

论文做到 ViT-H（632M 参数）和 1600 epoch 的 IN1K 预训练。更大的模型（ViT-G、ViT-22B）和更大的数据（JFT-300M 或 IG-3.5B 以上）是否会有更好的 scaling behavior？后续的 EVA 系列做了部分探索。

### 14.4 重建质量 vs 表征质量的关系不明确

高 mask ratio（75%）比中等 mask ratio（50%）产生了更好的表征，但重建质量更差。这意味着 "重建得像不像" 和 "encoder 有没有学到好表征" 可能是矛盾的。在 Figure 6 中，grid-wise masking 的重建更清晰但表征更差。这个 trade-off 的机制还没有完全解码。

### 14.5 Uniform random mask 是否最优

MAE 使用最简单的 uniform random masking。后续工作探索了替代方案：
- **Adversarial masking（A-MAE, 2022）**：让 mask 选择最大化 loss 的位置
- **Attentive masking**：用 attention 引导 mask 到"重要"区域
- **Semantic masking（PeCo, 2022）**：用 perceptual loss 指导 mask

这些变体在特定任务上可能有增益，但 uniform random 的简单性本身就是 MAE 的核心优势之一。

## 十五、小结

直白说，MAE 的价值在于它做了一个无比优雅的减法。

在 MAE 之前，做视觉自监督学习是一件很复杂的事：
- 对比学习需要精心设计的数据增强 pipeline、memory bank / momentum encoder / predictor 等组件
- BEiT 需要 dVAE tokenizer（在 250M 额外图片上预训练）、block-wise masking、离散 token 预测

MAE 把一切都简化了：**随机 mask 75% 的 patch，直接预测像素值。没了。**

这个简单带来的不仅仅是用起来方便，更是一个深层认知的转变：**视觉自监督学习的关键不是"设计更复杂的任务"，而是"让简单的任务变得足够困难"。** 75% mask ratio 就是那个魔法参数——它把视觉的空间冗余消除到临界点，让最简单的像素重建任务变成了需要语义理解的挑战。

在技术树中，MAE 是 A1-4 和 B4-9 的双重位置节点。它既是 BERT 范式在视觉上的成功落地，也是 MIM 路线的奠基之作之一。它所建立的 "mask 高比例 → 非对称 encoder-decoder → 预测像素" 三件套，定义了一个新的坐标系——后续的 MIM 工作（SimMIM, MaskFeat, VideoMAE, EVA）都围绕这个坐标系做调整和扩展。

这和 Swin Transformer（B2-2）的范式意义类似：**Swin 证明了局部 attention + 层级结构可以取代全局 attention 成为视觉 backbone 的标配；MAE 证明了最简单的 MIM 设计可以取代对比学习成为视觉 foundation model 的预训练范式。**

不过两者也有重要差异：Swin 是架构上的创新，MAE 是训练范式上的创新。Swin 改变的是 backbone 的"天花板"，MAE 改变的是预训练数据的"天花板"——让视觉模型可以在无标注数据上训练到足够高的水平。
