# Masked Autoencoders Are Scalable Vision Learners（2021）

**论文：** [arXiv](https://arxiv.org/abs/2111.06377) · He et al. · 2021（MAE）

---

## 一、先搞清楚坑在哪

BEiT 证明了 Masked Image Modeling 在视觉上有效，但有两个问题：

1. **计算效率低**：BEiT 对所有 patches（visible + masked）都做编码。如果 mask 40% 的 patches，实际上 40% 的计算量浪费在了被 mask 的 patches 上
2. **依赖 tokenizer**：需要 DALL-E 的 dVAE 把图像离散化，这引入了额外依赖和损失

更本质的问题：**在 NLP 中 BERT 的 MLM 预测离散 token 很自然（词是离散的），在视觉中预测像素或离散 code 哪个更好？**

## 二、已有方法的真正问题

**BEiT 型 MIM**：对所有 patches 编码 → 预测离散 token。计算浪费在 masked patches 上。MAE 的核心洞察：对 masked patches 做编码是浪费——不是"处理 masked patches"，而是"直接不让 masked patches 进入编码器"。

**对比学习（SimCLR/MoCo/DINO）**：需要精心设计的数据增强，学到的更多是图像级别的判别能力，而不是 patch 级别的理解。MAE 的 pixel-level reconstruction 天然学习细粒度表示。

**MAE 的灵感直接来自 BERT 和 denoising autoencoders**：在 NLP 里 mask 掉部分词并让模型恢复，在视觉里 mask 掉部分 patch 并让模型恢复像素值。区别只在于：
- NLP 用 softmax 分类预测词（离散）
- MAE 用 MSE 回归预测像素（连续）

## 三、MAE 的核心思路

一句话：**mask 掉 75% 的图像 patches，只编码可见的 25%，然后用轻量解码器从编码特征重建原始图像。**

三个关键设计：
1. **非对称编码器-解码器设计**：大的编码器（ViT）只处理可见 patches（25% 计算量），小的解码器（轻量 Transformer）处理完整序列（含 mask tokens）
2. **极高 mask 比例（75%）**：远高于 BERT 的 15% 和 BEiT 的 40%。强冗余性使得图像可以从极少信息重建
3. **直接预测像素值**：不需要 tokenizer，不需要离散化——简单地对 normalized pixel 做 MSE 回归

## 四、网络架构

### 整体流程

```
原始图像 (224×224)
    │
    ▼
Patch Partition (16×16) → 196 patches
    │
    ▼
随机 mask 75% of patches（只保留 49 个 visible patches）
    │
    ▼
┌───────────────────────────────────────────────────────┐
│ ViT Encoder (只处理 visible patches)                    │
│ 输入: 49 patches × 768 dim                             │
│ 输出: 49 个编码后的表示                                  │
│ 复杂度: (25% × 196)² = 49² = 2401 （对比全编码 196²=38k）│
│ → 计算量减少到原来的 1/16                               │
└───────────────────────┬───────────────────────────────┘
                        │
          49 个可见 patch 的编码表示
                        │
    ┌───────────────────┴───────────────────┐
    │ 用 <MASK> 占位符填充回 196 个位置      │
    │ (49 个编码 + 147 个可学习 mask token)   │
    └───────────────────┬───────────────────┘
                        │
                        ▼
┌───────────────────────────────────────────────────────┐
│ 轻量 Decoder（8 层 Transformer，512 dim）               │
│ 输入: 196 tokens (49 visible + 147 mask) + 位置编码    │
│ 输出: 196 个 patch 的像素预测（每个 patch 147 维）       │
└───────────────────────┬───────────────────────────────┘
                        │
                        ▼
              MSE Loss vs 原始 Image 的 patch 像素
```

### 架构细节

**编码器：** 标准 ViT（如 ViT-B: 12 层, 768 dim, 12 heads）。**关键差异：只处理可见 patches**。这意味着编码器的序列长度只有 49（75% mask），标准 ViT 是 196。计算量下降到原来的 $\frac{1}{16}$。

编码器输出的位置编码也需要对应——只对可见 patches 添加位置编码，masked patches 不参与。

**解码器：** 轻量 Transformer（8 层, 512 dim, 16 heads）。输入是把编码器的 49 个输出和 147 个可学习 mask token 按照原始位置拼接成 196 的序列，再加位置编码。解码器通过 self-attention 让 mask tokens 从可见 patches 获取信息。

任务：每个 mask token 输出一个 147 维的向量（16×16×3/2 = 147，不对不对——16×16×3 = 768。哦，看论文具体怎么设置。MAE 把每个 patch 的像素做了归一化，然后预测归一化后的像素值。输出维度是 3×16×16 = 768。）

**为什么解码器这么轻量够了？** 因为重建像素是"低级"任务，不需要深层编码器的语义抽象能力。解码器只需要把编码器的语义特征转换为像素级别的预测。

## 五、核心贡献

### 贡献 1: 非对称编码-解码

MAE 最聪明的设计：编码器只看可见 patches（25%），解码器看完整序列（可见 + mask）。这个非对称设计带来了巨大的效率提升。

**计算量对比：**

| 方法 | Encoder tokens | Decoder tokens | 总计算量（归一化） |
|------|---------------|---------------|-----------------|
| BEiT (mask 40%) | 196 | — | 1.0 |
| MAE (mask 75%) | 49 | 196 | 0.25 (enc) + 0.1 (dec) ≈ 0.35 |

MAE 的总计算量只有 BEiT 的 35%，而且编码器（最重的部分）只处理 25% 的数据。

**为什么可以只编码可见 patches？** Attention 是 set operation——不依赖顺序。编码器不需要知道 masked patches 在什么位置，只需要知道可见 patches 之间的相对关系。位置编码在解码器阶段再加回去。

### 贡献 2: 75% mask 比例（极高 mask 率）

BERT 用 15%，BEiT 用 40%，MAE 用 **75%**。为什么这么高？

直觉：图像包含大量冗余信息。相邻 patches 之间的像素高度相关。如果用 50% 的 masks，模型可以通过周围 patches 的插值来填充——不需要语义理解。75% 的 mask 强迫模型进行**语义级别的推理**：给定 25% 的可见 patches（可能是猫的耳朵尖、胡须、背景窗帘），模型必须推断出整个猫的结构。

**实验验证（Table 1a）：** MAE 的 mask 比例消融：
| Mask 比例 | Reconstruction error | Fine-tune top-1 |
|-----------|-------------------|----------------|
| 50% | 低 | 82.3 |
| 75% | 中 | **83.6** |
| 90% | 高 | 79.8 |

75% 最佳——重建更难但学到的表征更好。90% 信息太少，重建质量太差，损害表征。

### 贡献 3: 直接预测像素值（不需要 tokenizer）

MAE 的输出是每个 patch 的 L2-normalized 像素值。Loss 是 MSE——简单到极致。

对比 BEiT（需要 dVAE tokenizer + 8192 类分类损失），MAE 完全消除了对 tokenizer 的依赖。这不仅简化了流程，还意味着 MAE 可以用在任意数据集上——不需要多模态数据训练 tokenizer。

### 贡献 4: 数据增强几乎不用

MAE 的预训练几乎不需要数据增强——只用了随机 resize crop。对比之下，对比学习方法（SimCLR 需要 RandAugment + ColorJitter + GaussianBlur 等多达 10 种增强）和 DeiT 都需要大量数据增强。

为什么？因为 MIM 任务的难度来自 mask 本身，不是来自数据增强。如果只给 25% 的 patches，重建 75% 的 patches 本身就是个足够难的任务，不需要额外增加难度。

## 六、实验结果

### ImageNet-1K 分类（Table 2）

| 模型 | 预训练 | Epochs | ViT-B | ViT-L | ViT-H |
|------|--------|--------|-------|-------|-------|
| From scratch | — | — | 82.3 | 82.6 | 83.1 |
| MoCo v3 | Unsup | 300 | 83.4 | 84.0 | — |
| BEiT | MIM | 800 | 84.0 | 85.2 | — |
| **MAE** | **MIM** | **800** | **85.2** | **86.9** | **87.9** |
| **MAE** | **MIM** | **1600** | **85.6** | **87.4** | **88.3** |

MAE 在 ViT-L 上达到 86.9%，比 BEiT 的 85.2% 高出 1.7%。在 ViT-H 上达到 87.9%，接近有监督的 ViT-H（88.55%），但 MAE 不需要 JFT-300M 标签。

**更长的训练持续提升**：MAE 在 1600 epoch 后没有饱和——说明 MIM 是一个可以持续受益于更多数据的预训练范式。

### 目标检测（Table 5）

| Backbone | 预训练 | COCO box AP |
|---------|--------|-------------|
| ViT-B | Supervised (JFT) | 44.5 |
| ViT-B | MAE | **46.1** |
| ViT-B | MAE (1600ep) | **48.4** |

MAE 预训练的 ViT 在检测上甚至超过了有监督的 JFT-300M 版本——MIM 学习到的 patch 级表示天然适合 dense prediction 任务。

### 语义分割（Table 6）

| Backbone | 预训练 | ADE20K mIoU |
|---------|--------|-------------|
| ViT-B | Supervised (JFT) | 35.1 (Linear) |
| ViT-B | MAE | **44.0** (Linear) |
| ViT-B | MAE (1600ep) | **46.3** (Linear) |

线性探测（Linear probe）结果尤其惊艳——直接用 MAE 预训练的 ViT 特征训练一个线性分类/分割头，就能达到 44.0 mIoU。这验证了 MAE 学习的表征是**线性可分的（下游任务不需要重新学习特征）**。

## 七、代码对照（GitHub: facebookresearch/mae）

**1. 非对称编码的实现。** 编码器只接收可见 patches：`x = x[visible_indices]`。然后经过 ViT encoder 后，编码结果通过 `scatter_` 或 `padding` 恢复到 196 长度的序列，mask 位置填充 mask token。

**2. Mask 生成。** 随机选择 patches 的索引。不是 blockwise（如 BEiT 的矩形区域），而是完全随机的独立采样。在 75% 比例下，即使随机采样也会自然形成块状分布。

**3. Normalized pixels。** 每个被 mask 的 patch 的像素值先做 per-patch 归一化（均值=0，标准差=1），然后作为预测目标。这避免了模型直接预测 raw pixel 值的困难。

**4. Positional embeddings 的关键使用。** 编码器和解码器使用相同的位置编码。编码器只需要可见 patches 的位置编码，解码器需要全部 196 个位置的位置编码。

## 八、位置

### 前驱工作

- **[Devlin et al., 2019] BERT**：MAE 在视觉上实现了 BERT 的思想——mask 并重建
- **[Bao et al., 2021] BEiT**：第一个 MIM 工作，MAE 继承了 MIM 的核心思想但有三大改进：非对称编码、更高 mask 比例、直接像素预测
- **[Vincent et al., 2008] Denoising Autoencoders**：实际上 MIM 是 denoising autoencoders 的 Transformer 版本

### 后续影响

- **MAE 成为 ViT 预训练的事实标准**：后续几乎所有 ViT 变体的自监督预训练都基于 MAE。
- **VideoMAE**：把 MAE 扩展到视频（mask 更多的 3D patches）。
- **EVA（Fang et al., 2022）**：把 MAE 扩展到 1B 参数规模，证明 MAE + MIM 可以持续放大。
- **ConvNeXt V2**：把 MAE 应用到 ConvNeXt（卷积架构），证明了 MIM 的有效性不取决于 Transformer 架构。
- **DINOv2（Oquab et al., 2023）**：把 MAE 和对比学习结合起来，得到了更好的视觉特征。

## 九、局限

1. **需要大量资源预训练。** MAE 的 1600 epoch 训练在 ViT-H 上需要大量的计算资源（32 TPUv3 × 几天）。如果只训练 400 epoch，性能会下降 1-2 个百分点。

2. **Linear probe 的 gap。** 虽然 MAE 在 fine-tune 上表现出色，但 linear probe（线性分类评估冻结特征）结果显著低于对比学习（DINO）。这说明 MAE 的特征空间不是线性可分的——需要 fine-tune 才能释放潜力。理论上，MAE 学到的是**生成式表征**（适合重建），对比学习学到的是**判别式表征**（适合分类）。

3. **不直接适用于 dense prediction。** MAE 基于 ViT（单分辨率输出），本身不做多尺度。下游检测/分割仍需要 FPN/UperNet。但 MIM 学习到的 patch 级表示确实优于 ImageNet 分类监督。

## 十、小结

MAE 的核心贡献是**去掉了 MIM 中的一切冗余**：
- 不编码 masked patches（计算量节省 16×）
- 不需要 tokenizer（直接预测像素）
- 不需要数据增强（mask 本身提供难度）

它证明了三个反直觉的发现：
1. mask 75% 的 patches 效果最好——图像冗余远超直觉
2. 轻量解码器足够做像素重建——低级任务不需要深层网络
3. MIM 预训练可以超越有监督预训练——不需要标签也能学到更好的表示

MAE 和 BEiT 一起，确立了 MIM 作为视觉自监督学习的主流范式，与对比学习路线（SimCLR → MoCo → DINO）并驾齐驱。
