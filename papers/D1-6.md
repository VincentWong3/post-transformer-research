# SigLIP: Sigmoid Loss for Language Image Pre-Training（2023）

**链接：** https://arxiv.org/abs/2303.15343

---

## 一、先搞清楚“问题在哪里”

2018 年 BERT 横空出世之后，多模态预训练进入了一个奇怪的阶段：大家都在用对比学习，但对比学习的核心组件——损失函数——几乎没有人质疑过。

CLIP（2021）用了一个标准的 contrastive loss，形式上是对比学习里最常见的 infoNCE 变体。这个损失函数的工作方式是：在 batch 内把 N 个 image-text pair 做成 N×N 的相似度矩阵，然后用 softmax 做归一化，把对角线上的正样本拉高、非对角线上的负样本压低。

这个做法在 CLIP 时代跑通了，效果很好。但这里有一个很少有人注意到的问题：CLIP 的训练依赖 **large batch size**。OpenAI 用的是 32,768 的 batch size，这不仅仅是工程上的奢侈，而是算法层面的硬需求。

为什么需要大 batch？因为 softmax 的归一化分母是全局的——它需要看到足够多的负样本来做准确的对比。batch 太小，负样本不够，对比信号就弱，模型学不好。

这个限制在工业界是致命的。不是每个人都有 32,768 的 GPU 集群。更关键的是，即使你有算力，大 batch 也意味着大显存、大通信开销、大训练成本。

所以 2023 年这篇 SigLIP 问了一个很直接的问题：**能不能用一个不需要跨样本归一化的损失函数，来替代对比学习里的 softmax？**

这个问题的本质是：对比学习到底需不需要“对比”？

---

## 二、旧路线的真正问题

CLIP 的损失函数是标准的对比学习损失。我们来看它的数学形式：

对于 batch size 为 N 的 image-text 对，CLIP 计算一个 N×N 的相似度矩阵 S，其中 S_{ij} = f_image(x_i) · f_text(y_j)。然后它计算两个方向的交叉熵损失：

```
L_image = -1/N * Σ_i log(exp(S_ii) / Σ_j exp(S_ij))
L_text = -1/N * Σ_j log(exp(S_jj) / Σ_i exp(S_ij))
```

这里的问题不是这个损失函数本身有问题，而是它的行为依赖于整个 batch 的统计量。

具体来说：

1. **softmax 的分母是全局的**。每一项 exp(S_ij) 都在分母里，所以每个样本的梯度会受所有其他样本的影响。这意味着如果你要增加正样本的相似度，你实际上是在和所有负样本竞争。

2. **负样本的质量取决于 batch 的组成**。如果 batch 里恰好有很多相似的负样本，softmax 会强行压低它们的相似度；如果 batch 里都是不相关的负样本，softmax 的梯度信号就很弱。

3. **大 batch 是必要条件**。CLIP 原文说他们发现 batch size 从 32K 降到 16K 后性能明显下降。这不是偶然的——softmax 需要足够的负样本来做准确的概率估计。

4. **训练不稳定**。softmax 的指数运算在相似度较大的时候会产生非常大的值，导致数值不稳定。CLIP 用了一个 temperature 参数来缓解，但这个 temperature 本身需要调参。

这些问题的根源是：**对比学习在损失函数层面做了一个不必要的假设——样本之间的关系必须通过全局归一化来建模。**

那有没有替代方案？有的。一个很自然的想法是：把每个 image-text pair 当成独立的二分类问题。这样就不需要跨样本归一化了。

这就是 SigLIP 的核心思路。

---

## 三、这篇论文的核心思路

**这篇论文的核心思路是：用 sigmoid 损失替代 softmax 损失，把对比学习从全局归一化问题变成每个 pair 独立的二分类问题。**

这个想法乍看之下很 trivial——sigmoid 和二分类是最基础的机器学习工具。但真正厉害的地方在于：作者不是简单地替换损失函数，而是重新设计了整个训练框架，让 sigmoid 损失在对比学习场景下 work。

具体来说，SigLIP 做了三件事：

1. **把每个 image-text pair 当成独立的二分类任务**：正样本 label=1，负样本 label=0。损失函数是 sigmoid 形式的 binary cross-entropy。

2. **引入了一个可学习的 temperature 参数**：这个 temperature 不是直接加在 logits 上的，而是通过一个可学习的标量来缩放相似度，从而控制 sigmoid 的陡峭程度。

3. **设计了一个特殊的负样本采样策略**：由于每个 pair 是独立训练的，理论上你可以用 O(N) 的复杂度处理 N 个样本，而不是 CLIP 的 O(N²)。但作者发现，如果只用 batch 内的负样本，效果不够好。所以他们用了 **“batch内所有负样本”** 的策略，即每个正样本和所有其他样本组成负样本对。

关键洞察在于：**sigmoid 损失不需要 softmax 那样的全局归一化，所以 batch size 不再是硬约束。** 即使 batch size 很小，每个 pair 的梯度也是合理的，因为梯度只依赖于当前 pair 的 logit，而不是整个 batch 的统计量。

这就带来了一系列好处：

- 可以训练更小的 batch size
- 训练更稳定
- 不需要调 temperature
- 可以更灵活地控制负样本数量

为了理解这个思路为什么 work，我们需要深入看一下 SigLIP 的 forward 流程。

## 四、整体结构与 forward 流程

SigLIP 的整体结构和 CLIP 几乎一样，只是损失函数不同。我们按 forward 顺序走一遍：

### 4.1 输入

输入是一个 batch 的 image-text pairs：
- 图像：batch of images，每个图像经过图像编码器
- 文本：batch of texts，每个文本经过文本编码器

### 4.2 编码

图像编码器（通常是 ViT）输出图像特征：
```
image_features: [N, d]
```

文本编码器（通常是 Transformer）输出文本特征：
```
text_features: [N, d]
```

### 4.3 相似度计算

计算所有 image-text pair 的相似度：
```
logits = image_features @ text_features.T  # [N, N]
```

这里 logits[i][j] 表示第 i 个图像和第 j 个文本的相似度。

### 4.4 损失计算（关键区别）

CLIP 的做法：
```python
# CLIP loss
labels = arange(N)  # 对角线是正样本
loss_i = cross_entropy(logits, labels, dim=1)  # 对每个图像，找正确的文本
loss_t = cross_entropy(logits.T, labels, dim=1)  # 对每个文本，找正确的图像
loss = (loss_i + loss_t) / 2
```

SigLIP 的做法：
```python
# SigLIP loss
# 对每个 pair (i, j) 计算损失
# 正样本: i == j, label = 1
# 负样本: i != j, label = 0

# 构造标签矩阵
labels = 2 * eye(N) - 1  # 对角线为 1，非对角线为 -1

# sigmoid 损失
loss = -labels * logits + log(1 + exp(labels * logits))
loss = mean(loss)
```

这里的关键是：**每个 pair 的损失是独立的**。正样本对 (i,i) 的损失只依赖于 logits[i][i]，负样本对 (i,j) 的损失只依赖于 logits[i][j]。没有 softmax 的分母，没有跨样本的归一化。

### 4.5 梯度反向传播

梯度计算也是独立的：
```
dL/d_logits[i][i] = sigmoid(-logits[i][i])  # 正样本梯度
dL/d_logits[i][j] = sigmoid(logits[i][j])   # 负样本梯度
```

这个梯度形式非常干净：正样本的梯度是 sigmoid(-logit)，负样本的梯度是 sigmoid(logit)。当 logit 很大时（正样本相似度高），梯度接近 0；当 logit 很小时（负样本相似度低），梯度也接近 0。只有那些“还没学好的”pair 才会有大梯度。

这就是 sigmoid 损失的核心优势：**梯度自动聚焦在困难的样本上，不需要手动调参。**

## 五、核心模块逐个解释

### 5.1 Sigmoid 损失函数

**问题：** 为什么 sigmoid 损失比 softmax 损失更适合对比学习？

**解释：**

我们先看 softmax 损失的梯度形式。对于正样本对 (i,i)：

```
dL/d_logits[i][i] = -1 + softmax(logits[i])[i]
                   = -1 + exp(logits[i][i]) / Σ_j exp(logits[i][j])
```

这个梯度依赖于所有负样本的相似度。如果负样本的相似度都很低，分母接近 exp(logits[i][i])，梯度接近 0。但如果某个负样本的相似度很高，分母变大，梯度就会变大。

这意味着：**softmax 的梯度受最难的负样本控制**。这是好事也是坏事——好事是模型会自动关注困难负样本；坏事是梯度信号容易被少数 outlier 主导。

再看 sigmoid 损失的梯度：

```
dL/d_logits[i][i] = sigmoid(-logits[i][i])
                   = 1 / (1 + exp(logits[i][i]))
```

这个梯度只依赖于当前 pair 的 logit，完全不看其他样本。当 logits[i][i] 很大时（正样本已经学好了），梯度接近 0；当 logits[i][i] 很小时（正样本还没学好），梯度接近 1。

这个性质意味着：**sigmoid 损失的梯度是“局部”的，每个 pair 的更新只取决于它自己的状态。** 这带来了两个好处：

1. **训练更稳定**：没有全局归一化，所以不会出现因为某个 outlier 导致整个 batch 梯度爆炸的情况。
2. **小 batch 也能 work**：每个 pair 的损失是独立的，batch size 只影响你同时处理多少样本，不影响每个样本的梯度质量。

### 5.2 可学习的 temperature

**问题：** sigmoid 损失需要一个 temperature 来控制 sigmoid 的陡峭程度吗？

**解释：**

传统的 sigmoid 二分类中，logits 通常是在 [0,1] 范围内的概率值。但在对比学习中，相似度可以是任意实数。如果相似度的绝对值很大，sigmoid 会进入饱和区，梯度消失。

CLIP 用了一个固定的 temperature 来缩放 logits：
```
logits = logits / temperature
```

SigLIP 把这个 temperature 变成了可学习的参数：
```
logits = logits * exp(t)  # t 是可学习的标量
```

这里有个微妙的设计：为什么用 exp(t) 而不是直接用 t？因为 exp(t) 保证温度是正数，同时允许温度在训练过程中自动调整。

这个可学习的 temperature 解决了两个问题：
1. **自动缩放**：模型自己学会把相似度缩放到合适的范围，不需要手动调参。
2. **动态调整**：训练初期和后期，最优的 temperature 可能不同，可学习参数可以自动适应。

### 5.3 负样本策略

**问题：** sigmoid 损失需要负样本吗？如果需要，怎么采样？

**解释：**

理论上，sigmoid 损失可以只训练正样本——每个正样本对单独做二分类。但这样模型会学到一个 trivial 的解：把所有样本的特征都映射到同一个点，这样所有正样本的相似度都是 1。

所以负样本是必须的。SigLIP 的策略是：**每个正样本和 batch 内所有其他样本组成负样本对**。这样每个正样本有 N-1 个负样本。

这个策略和 CLIP 一样，都是 O(N²) 的复杂度。但区别在于：CLIP 的 O(N²) 是因为 softmax 需要全局归一化；SigLIP 的 O(N²) 是因为我们主动选择用所有负样本。

作者也实验了其他负样本策略（比如每个正样本只取 K 个负样本），发现用所有负样本效果最好。这说明：**虽然 sigmoid 损失理论上不需要全局归一化，但更多的负样本仍然提供了更丰富的对比信号。**

## 六、核心公式与数学直觉

### 6.1 SigLIP 损失函数

SigLIP 的损失函数形式如下：

```
L = -1/N² * Σ_i Σ_j [y_ij * log(σ(α * s_ij)) + (1 - y_ij) * log(1 - σ(α * s_ij))]
```

其中：
- N 是 batch size
- s_ij = f_image(x_i) · f_text(y_j) 是图像 i 和文本 j 的相似度
- y_ij = 1 当 i=j，否则为 0
- σ 是 sigmoid 函数
- α 是可学习的 temperature 参数

这个式子看起来就是一个标准的 binary cross-entropy。但真正的作用是：**把对比学习从“在 N 个候选中找正确匹配”变成了“对每个 pair 判断是否匹配”。**

### 6.2 简化形式

由于 y_ij 只有 0 和 1 两个值，我们可以把损失拆成正样本和负样本两部分：

```
L_pos = -1/N * Σ_i log(σ(α * s_ii))
L_neg = -1/(N*(N-1)) * Σ_i Σ_{j≠i} log(1 - σ(α * s_ij))
L = L_pos + L_neg
```

这个形式更清楚地展示了 SigLIP 的行为：
- **正样本损失**：鼓励对角线上的相似度变大
- **负样本损失**：鼓励非对角线上的相似度变小

### 6.3 与 CLIP 损失的对比

CLIP 的损失（简化形式）：
```
L_CLIP = -1/N * Σ_i log(exp(s_ii) / Σ_j exp(s_ij))
       = -1/N * Σ_i [s_ii - log(Σ_j exp(s_ij))]
```

SigLIP 的损失（等价形式）：
```
L_SigLIP = -1/N² * Σ_i Σ_j [y_ij * s_ij - log(1 + exp(s_ij))]
```

关键区别：
1. **CLIP 的损失有 log-sum-exp 项**：这个项引入了全局依赖，梯度受所有负样本影响。
2. **SigLIP 的损失只有局部项**：每个 pair 的损失独立，梯度只依赖于当前 pair。

### 6.4 梯度分析

CLIP 中正样本的梯度：
```
dL_CLIP/d_s_ii = -1 + softmax(s_i)_i
                = -1 + exp(s_ii) / Σ_j exp(s_ij)
```

这个梯度范围是 [-1, 0]，当 s_ii 远大于所有负样本时接近 0，当 s_ii 小于某个负样本时接近 -1。

SigLIP 中正样本的梯度：
```
dL_SigLIP/d_s_ii = -σ(-α * s_ii) * α
                  = -α / (1 + exp(α * s_ii))
```

这个梯度范围是 [-α, 0]，当 s_ii 很大时接近 0，当 s_ii 很小时接近 -α。

**有意思的发现：** SigLIP 的梯度有上界 α，而 CLIP 的梯度没有上界（理论上可以接近 -1，但实际上受 softmax 限制）。这意味着 SigLIP 更稳定，不会出现梯度爆炸。

## 八、复杂度、效率与可扩展性分析

### 8.1 计算复杂度

SigLIP 和 CLIP 的计算复杂度都是 O(N²)：
- 相似度矩阵计算：O(N²d)
- 损失函数计算：O(N²)
- 梯度计算：O(N²)

但这里有个微妙之处：**CLIP 的 O(N²) 是必须的**，因为 softmax 需要全局归一化。**SigLIP 的 O(N²) 是选择的**，因为我们可以选择只用部分负样本。

### 8.2 内存复杂度

两者都是 O(N²) 的内存来存储相似度矩阵。但 SigLIP 有一个优势：**不需要存储 softmax 的中间结果**，所以实际内存占用略低。

### 8.3 可扩展性

SigLIP 的核心优势在于可扩展性：

1. **小 batch 训练**：batch size 从 32K 降到 8K，性能几乎不变。这意味着可以用更少的 GPU 训练。
2. **分布式训练**：由于每个 pair 的损失独立，分布式训练时通信开销更小。
3. **负样本控制**：可以灵活控制负样本数量，在性能和计算成本之间做 trade-off。

### 8.4 训练速度

论文报告 SigLIP 比 CLIP 快约 1.5x-2x（在相同 batch size 下）。这个加速来自于：
1. 不需要计算 softmax
2. 梯度计算更简单
3. 反向传播更高效

## 九、实验结果

### 9.1 零样本分类

| 模型 | ImageNet top-1 | 训练数据 | Batch Size |
|------|---------------|----------|------------|
| CLIP | 76.2% | 400M pairs | 32K |
| SigLIP | 76.4% | 400M pairs | 32K |
| CLIP | 73.8% | 400M pairs | 16K |
| SigLIP | 76.1% | 400M pairs | 16K |

关键发现：**batch size 减半后，CLIP 掉了 2.4%，SigLIP 只掉了 0.3%。**

### 9.2 小 batch 实验

| Batch Size | CLIP | SigLIP |
|------------|------|--------|
| 32K | 76.2% | 76.4% |
| 16K | 73.8% | 76.1% |
| 8K | 70.1% | 75.8% |
| 4K | 62.3% | 74.2% |

**batch size 从 32K 降到 4K，CLIP 掉了 14%，SigLIP 只掉了 2.2%。** 这个差距是巨大的。

### 9.3 消融实验

论文做了几个重要的消融实验：

**temperature 的影响：**
- 固定 temperature=0.01：不收敛
- 固定 temperature=0.1：ImageNet 71.2%
- 固定 temperature=0.07：ImageNet 74.8%
- 可学习 temperature：ImageNet 76.4%

**负样本数量的影响：**
- 只用对角线（无负样本）：不收敛
- 每个正样本用 1 个负样本：ImageNet 68.3%
- 每个正样本用 10 个负样本：ImageNet 74.1%
- 所有负样本：ImageNet 76.4%

**结论：负样本越多越好，temperature 需要自适应。**

### 9.4 迁移学习

SigLIP 在多个下游任务上（如 COCO、Flickr30k）的零样本和微调性能都优于或等于 CLIP，证明了 sigmoid 损失的泛化能力。

### 10.2 核心代码

SigLIP 的损失函数实现非常简洁：

```python
def siglip_loss(logits, labels):
    # logits: [N, N], similarity matrix
    # labels: [N, N], 1 for positive, -1 for negative
    
    # sigmoid loss
    loss = -labels * logits + jnp.log(1 + jnp.exp(labels * logits))
    return jnp.mean(loss)
```

这个实现只有 3 行代码，比 CLIP 的 softmax 损失简单得多。

        # 温度缩放

        # 损失
