## 一、先搞清楚"问题在哪里"

2021 年，CLIP 做了一件让整个 CV 社区震动的事：用 4 亿图文对训练一个双塔模型，在零样本分类上追平了有监督的 ResNet-50。这证明了「自然语言监督」可以替代人工标注，成为视觉表征学习的 scalable 信号源。

但 CLIP 的成功有一个隐藏的前提：**它需要非常大的 batch size**。CLIP 原文用了 32,768 的 batch size，因为它使用的是 **softmax-based contrastive loss**——每个 batch 内，所有正负样本对的 similarity 会被一起 softmax 归一化，然后算交叉熵。这意味着：

1. **batch size 决定了负样本数量。** 一个 batch 有 N 个图像-文本对，每张图像只能看到同一 batch 内的 N-1 个负文本。负样本太少，对比学习的信号太弱。
2. **softmax 归一化是全局操作。** 每个样本的 loss 依赖于同 batch 内所有其他样本的相似度分数。这让 loss 的计算无法独立化。
3. **batch size 和大规模训练的绑定。** CLIP 发现，只有在 batch size ≥ 32,768 时，模型才能学到好的表征。小 batch 下，负样本不足，效果断崖式下跌。

这三个问题合在一起，意味着做 CLIP 式训练需要巨大的算力——32K+ 的 batch size 意味着至少数百张 GPU/TPU。这直接把大多数研究者和中小团队挡在了门外。

所以到 2023 年，图文预训练面临的核心问题是：**能不能设计一种 loss，让正负样本对的比较不依赖于 batch 内的全局归一化？** 如果可以，小 batch 也能高效训练，大 batch 反而可以成为一个可选的优化项而不是硬性要求。

SigLIP 就是在这个节点上出现的。

## 二、旧路线的真正问题

要理解 SigLIP 为什么重要，得先搞懂 CLIP 的 loss 为什么绑定大 batch——这不是一个工程上的"选择"，而是 softmax 的数学性质决定的。

### 2.1 Softmax-based contrastive loss 的数学结构

CLIP 的 loss 本质是一个 **N-way contrastive loss**：

$$\mathcal{L}_{\text{CLIP}} = -\frac{1}{N}\sum_{i=1}^{N}\log\frac{\exp(s_{ii})}{\sum_{j=1}^{N}\exp(s_{ij})} - \frac{1}{N}\sum_{j=1}^{N}\log\frac{\exp(s_{jj})}{\sum_{i=1}^{N}\exp(s_{ij})}$$

其中 $s_{ij}$ 是第 $i$ 张图像和第 $j$ 个文本的相似度（通常是 cosine similarity 乘以温度系数）。

这个公式里有两个关键点：

**第一，分母是对所有配对做的求和。** 每个图像 $i$ 的 loss 不仅要看它和自己的文本 $j=i$ 的匹配度（分子），还要看它和所有其他文本 $j\neq i$ 的匹配度（分母）。这意味着，同 batch 内增加一张图像 $N+1$ ，所有 $N$ 张图像的 loss 都会变化——因为它们的 softmax 分母变了。

**第二， $N$ 越大，负样本越多样。** CLIP 中，真实的负样本只有同 batch 内的其他 $N-1$ 个文本。如果 $N=32$ ，每张图像只有 31 个负文本；如果 $N=32,768$ ，则有 32,767 个负文本。更多样的负样本让对比学习的 margin 更大，表征更鲁棒。

### 2.2 为什么小 batch 不行

CLIP 原文的实验中，随着 batch size 从几百增长到 32,768，zero-shot ImageNet 准确率持续提升。小 batch 下的表现明显弱于大 batch。

小 batch 下效果差的核心原因在于 **softmax 的归一化是全局的，负样本的数量和质量直接受限于 batch size**。这不是训练技巧能解决的问题——这是 loss 函数的设计问题。

### 2.3 更大的 batch ≠ 更好的效果

即使在大 batch regime 下，CLIP 也有一个隐藏矛盾：**更大的 batch 提供更多负样本，但同 batch 内的正负样本对来自同一分布，多样性有限。** 更根本的是，softmax 的归一化本质上是一个 batch 内竞争——模型只强制图像和自己的文本在 batch 内"脱颖而出"，而不关心这个"脱颖而出"的输出值在绝对值上意味着什么。

换言之，CLIP 的 loss 只关心排名（ranking），不关心置信度（calibration）。这论文在实验中会进一步验证这一点。

### 2.4 小结：需要一个什么样的 loss？

一个理想的对齐式预训练 loss 应该满足：

1. **独立计算** — 每个图文对的 loss 不依赖于同 batch 内其他配对
2. **batch size 无关** — 小 batch 和大 batch 一样有效
3. **灵活的负样本控制** — 可以独立调节正负样本比例，而不是被 batch size 绑定

现有的 softmax loss 一条都不满足。SigLIP 的任务就是设计一个满足这三条的 loss。

## 三、这篇论文的核心思路

**这篇论文的核心思路是：把 CLIP 的 softmax-based N-way contrastive loss 替换为一个 pairwise sigmoid loss，让每个图像-文本对独立计算二分类损失，不再依赖 batch 内的全局归一化。**

具体来说，就是用一个极其简单的公式替换掉复杂的 softmax：

$$\mathcal{L}_{\text{SigLIP}} = -\frac{1}{N}\sum_{i=1}^{N}\sum_{j=1}^{N}\ell(s_{ij}, y_{ij})$$

其中 $\ell(s, y)$ 是逐对的 sigmoid 损失：

$$\ell(s, y) = -y \log\sigma(s \cdot t + b) - (1-y) \log(1-\sigma(s \cdot t + b))$$

或者更简洁的等价形式（使用 log-sigmoid）：

$$\ell(s_{ij}, y_{ij}) = \begin{cases}
-\log\sigma(s_{ij} \cdot t + b), & y_{ij}=1 \text{ (匹配)} \\
-\log\sigma(-s_{ij} \cdot t - b), & y_{ij}=0 \text{ (不匹配)}
\end{cases}$$

这里 $y_{ij} \in \{0, 1\}$ 是标签（1 表示匹配对 $i=j$ ，0 表示不匹配对 $i\neq j$ ）， $s_{ij}$ 是图像 $i$ 和文本 $j$ 的余弦相似度， $t$ 是可学习的温度参数， $b$ 是偏置项。

这条技术路线的谱系位置是：

```text
Softmax-based contrastive loss (CLIP, 2021)
   ↓
ALIGN: 双塔 + 较大 batch, 12.8K batch size (2021)
   ↓
SigLIP: sigmoid loss, batch size 灵活 (2023) ← 本篇
```

SigLIP 和 CLIP 的最大区别是：CLIP 的 loss 需要先对 $\{s_{i1}, s_{i2}, \dots, s_{iN}\}$ 做 softmax，然后算交叉熵——这意味着 $s_{i1}$ 的 loss 依赖于 $s_{i2}$ 到 $s_{iN}$ 的值。SigLIP 的 sigmoid loss 直接在每个 $s_{ij}$ 上独立计算二分类交叉熵，** $s_{i1}$ 的 loss 完全不依赖于 $s_{i2}$ **。

这个简单的改变，带来了深远的影响。

## 四、整体结构与 forward 流程

### 4.1 整体架构

SigLIP 的架构和 CLIP 几乎完全相同——都是双塔结构：

```text
图像输入                          文本输入
   │                                │
   ▼                                ▼
图像 Encoder (ViT/ResNet)         文本 Encoder (Transformer)
   │                                │
   ▼                                ▼
图像 Embedding                    文本 Embedding
   │                                │
   └─────────── dot product ────────┘
                     │
                     ▼
              pairwise sigmoid loss
              (per pair, independent)
```

唯一的架构差异在 loss 函数层面。编码器部分没有引入任何新的架构设计。

### 4.2 前向过程逐步跟踪

以 batch size $N=16$ 为例：

**Step 1 — 编码**

- 输入：16 张图像 + 16 段文本
- 图像 encoder 输出： $I \in \mathbb{R}^{16 \times d}$
- 文本 encoder 输出： $T \in \mathbb{R}^{16 \times d}$

**Step 2 — 相似度矩阵计算**

- 计算所有 256 对图像-文本相似度
- 输出： $S = I \cdot T^\top / \sqrt{d} \in \mathbb{R}^{16 \times 16}$

**Step 3 — 构建 target 标签**

- $y_{ij} = 1$ 当 $i = j$ （对角线上的匹配对）
- $y_{ij} = 0$ 当 $i \neq j$ （非对角线的不匹配对）

**Step 4 — 计算 pairwise sigmoid loss**

- 对 256 个 $s_{ij}$ 分别计算 sigmoid loss
- 每个 loss 只依赖 $s_{ij}$ 、温度 $t$ 和偏置 $b$
- **完全不依赖同 batch 内其他配对的相似度**

**Step 5 — 求平均 loss 后反向传播**

```python
# SigLIP loss 的简单实现（伪代码）
def siglip_loss(image_embeds, text_embeds, t, b):
    N = image_embeds.shape[0]
    
    # 计算所有配对相似度
    logits = t * image_embeds @ text_embeds.T + b  # [N, N]
    
    # 构建标签：对角线上为 1，其余为 0
    labels = 2 * torch.eye(N) - 1  # 1 for match, -1 for non-match
    # 或者用二元标签：labels_01 = torch.eye(N)
    
    # 逐对 sigmoid loss
    # 等价于：labels * logits 作为 logit，再算 log-sigmoid
    loss = -torch.sum(torch.log(torch.sigmoid(labels * logits))) / N
    
    return loss
```

**关键差异：** CLIP 的 loss 需要对 logits 按行/列做 softmax（ $\frac{1}{N}\sum_i \text{cross-entropy}(\text{softmax}(I_i \cdot T))$ ），softmax 的输入是整行 16 个相似度。SigLIP 直接对 256 个独立的 logit 算 sigmoid，logit 之间没有相互依赖。

### 4.3 温度参数和偏置项

论文引入了两个可学习的标量参数：

- **温度 $t$ ：** 缩放相似度分数，控制 softness/hardness。和 CLIP 中的 temperature 作用相同。
- **偏置 $b$ ：** 调节匹配/不匹配对的先验概率。当 $b > 0$ 时，sigmoid 的决策边界会偏向于"匹配"的决策，这可以在训练初期缓解正类信号稀疏的问题。

论文中提到，偏置 $b$ 的初始值会影响训练动态。使用合适的初始值（如 $b \approx -2$ 左右，具体看实验配置）可以让训练更稳定。

## 五、核心公式与数学直觉

### 5.1 对比两种 loss 的数学结构

**CLIP (softmax-based)：**

一个图像 $i$ 对 batch 内所有文本的 loss：

$$\mathcal{L}_i^{\text{CLIP}} = -\log\frac{\exp(s_{ii})}{\sum_{j=1}^{N}\exp(s_{ij})}$$

展开后可以看到：

$$\mathcal{L}_i^{\text{CLIP}} = -s_{ii} + \log\sum_{j=1}^{N}\exp(s_{ij})$$

第二项 $\log\sum\exp(s_{ij})$ 包含所有 $s_{i1}, s_{i2}, \dots, s_{iN}$ ，所以 $\partial \mathcal{L}_i / \partial s_{ij}$ 对所有 $j$ 都是非零的。

**SigLIP (sigmoid-based)：**

对所有 $N^2$ 个配对的 loss 求和：

$$\mathcal{L}^{\text{SigLIP}} = -\frac{1}{N}\sum_{i=1}^{N}\sum_{j=1}^{N}\left[y_{ij}\log\sigma(t\cdot s_{ij} + b) + (1-y_{ij})\log\sigma(-t\cdot s_{ij} - b)\right]$$

或者用更常见的等价形式：

$$\mathcal{L}^{\text{SigLIP}} = -\frac{1}{N}\sum_{i=1}^{N}\sum_{j=1}^{N}\log\sigma(y_{ij} \cdot (t\cdot s_{ij} + b))$$

其中 $\log\sigma(x)$ 是 log-sigmoid 函数（ $\log\sigma(x) = -\log(1+e^{-x})$ ）。

关键区别：** $\partial \mathcal{L}^{\text{SigLIP}} / \partial s_{ij}$ 只依赖于 $s_{ij}$ 本身，不依赖于任何其他配对。** 梯度是逐元素独立的。

### 5.2 Batch size 解耦的数学直觉

为什么 sigmoid loss 能让 batch size 变得灵活？

**从梯度的角度看：**

CLIP 的梯度中，样本 $i$ 对 $s_{ij}$ 的梯度是：

$$\frac{\partial \mathcal{L}_i^{\text{CLIP}}}{\partial s_{ij}} = \frac{\exp(s_{ij})}{\sum_k\exp(s_{ik})} - \delta_{ij} = \text{softmax}_j(s_i) - \delta_{ij}$$

这个梯度的 magnitude 受 softmax 分母中所有 $s_{ik}$ 的影响。如果 $N$ 很小，softmax 分布不够尖锐，梯度信号弱。如果 $N$ 很大，softmax 分布更尖锐，正对的梯度 signal-to-noise ratio 更高。

SigLIP 的梯度是：

$$\frac{\partial \mathcal{L}^{\text{SigLIP}}}{\partial s_{ij}} = -t \cdot y_{ij} \cdot \sigma(-y_{ij}(t\cdot s_{ij} + b))$$

$= \begin{cases}
-t \cdot \sigma(-t\cdot s_{ij} - b), & i=j \text{ (正对)} \\
+t \cdot \sigma(t\cdot s_{ij} + b), & i\neq j \text{ (负对)}
\end{cases} $

这个梯度只依赖于 $s_{ij}$ 和温度 $t$ 和偏置 $b$ ——batch size $N$ 完全不出现。这意味着 **sigmoid loss 的梯度信号强度不随 batch size 改变**，小 batch 一样可以有强梯度。

**从信息论的角度看：**

CLIP 的 loss 本质是在 batch 内做 N-way 判别——模型要区分"哪个文本和这张图像最匹配"。这天然依赖 batch 中的负样本数量来提供判别信息。

SigLIP 的 loss 本质是在做 **pairwise verification**——模型要判断"这一对图像和文本是否匹配"。每个配对独立提供二分类监督信号。负样本可以来自同 batch（ $N^2 - N$ 个负对），也可以通过跨 batch 或者 memory bank 引入。

### 5.3 对角线与全矩阵

CLIP 只计算对角线上 N 个正对的 softmax loss：

$$\mathcal{L}_{\text{CLIP}} = -\frac{1}{2N}\sum_{i=1}^N \left[\log\frac{\exp(s_{ii})}{\sum_j\exp(s_{ij})} + \log\frac{\exp(s_{ii})}{\sum_j\exp(s_{ji})}\right]$$

SigLIP 计算全部 $N^2$ 个配对的 loss——包括对角线上的 $N$ 个正对和非对角线的 $N^2 - N$ 个负对：

$$\mathcal{L}_{\text{SigLIP}} = -\frac{1}{N}\sum_{i=1}^N\sum_{j=1}^N \ell(s_{ij}, y_{ij})$$

**为什么 CLIP 不用全矩阵？** CLIP 的 softmax 归一化已经隐含地利用了非对角元素——它们在分母中起作用。SigLIP 的 sigmoid 没有分母，因此需要显式地遍历负对来计算 loss。

**这意味着 SigLIP 的每个 step 有 $N^2$ 个 loss 项，而 CLIP 有 $2N$ 个 loss 项。** 显存和计算量上 SigLIP 略高（ $O(N^2)$ vs $O(N)$ ），但由于不需要 softmax 归一化，实际速度差异并不大。而且 batch size 可以设得很小， $O(N^2)$ 在 small batch 下根本不是问题。

### 5.4 偏置项 $b$ 的作用

偏置 $b$ 的存在有一个重要的实际意义。在 sigmoid loss 中， $t \cdot s_{ij} + b$ 的符号决定了这个配对是被推近还是推远。当 $s_{ij}=0$ （随机初始化的相似度）时， $\sigma(b)$ 决定了模型对这个配对的初始预测。

论文通过实验发现，使用合适的偏置初始化（如 $b \approx 0$ 附近）可以让训练初期更稳定。如果没有偏置项，初始化时正负对的 logit 都是 0，模型会陷入"不确定"的状态。

## 六、实验结果

### 6.1 SigLiT：SigLIP + Locked-image Tuning

论文中最令人印象深刻的结果之一来自 **SigLiT**，即 SigLIP + LiT（Locked-image Tuning）。LiT 的做法是：先在一个大规模图文数据集上训练双塔，然后在微调阶段**冻结图像 encoder，只训练文本 encoder**。

SigLiT 使用 4 个 TPUv4 芯片，**在两天内**训练了一个模型，在 ImageNet 上达到了 **84.5% 的 zero-shot top-1 准确率**。

这个结果的意义：CLIP 原文训练类似规模的模型需要数百个 TPU 天。SigLiT 把计算成本压缩了两个数量级。

### 6.2 Batch size 的消融实验

论文对 batch size 做了系统性的研究，这是其中一个核心实验：

| Loss | Batch size | ImageNet zero-shot top-1 |
|------|-----------|-------------------------|
| Softmax (CLIP) | 16 | 较低（~40-50%） |
| Softmax (CLIP) | 32,768 | 基准 |
| **Sigmoid (SigLIP)** | **16** | **和 CLIP 32,768 相当** |
| **Sigmoid (SigLIP)** | **32,768** | **略高于基线** |
| Sigmoid (SigLIP) | 1,000,000 | 和 32,768 接近（收益递减） |

**核心发现：**

1. **sigmoid loss 在 batch size 16 时效果和 CLIP 在 batch size 32,768 时相当。** 这意味着 batch size 缩减了 2,000×，且性能没有明显损失。
2. **sigmoid loss 也能从更大的 batch 中受益。** 从 16 增加到 32,768 仍有提升，只是提升幅度不如 CLIP 显著。
3. **batch size 继续增大时收益递减。** 论文将 batch size 推到 1,000,000，发现效果提升已经很小，32,768 已经足够。

### 6.3 与 CLIP 的直接对比

在相同的模型架构、相同的训练数据和相同的计算预算下，SigLIP 在 zero-shot 分类上的表现与 CLIP 持平或略好。论文在多组实验中都验证了这一点。

### 6.4 负样本比例的影响

由于 sigmoid loss 的 batch size 解耦特性，论文能够独立研究 **负样本比例（negative-to-positive ratio）** 的影响——这在 CLIP 中是无法做到的，因为 batch size 决定了负样本数量。

关键的观察是：**适度的负样本比例就足够了。** 更多负样本带来的边际收益递减很快，这解释了为什么 batch size 32,768 已经足够——全矩阵 $N \times N$ 中，正对只有 $N$ 个，负对有 $N^2 - N$ 个，负正比随 $N$ 线性增长。当 $N=32,768$ 时，负正比约为 32,767:1，已经接近饱和。

### 6.5 消融实验

论文对 sigmoid loss 的各项设计选择做了系统消融：

**温度和偏置的影响：**

- 使用合适的温度 $t$ 和偏置 $b$ 对最终效果有显著影响
- 固定偏置 $b=0$ 或温度 $t=1$ 都会导致效果下降
- 两者的联合优化是重要的

**对比 CLIP 变体的总览：**

| Method | ImageNet zero-shot | 备注 |
|--------|-------------------|------|
| CLIP (softmax) | 基准 | 需要大 batch |
| SigLIP (sigmoid) | 持平或略好 | 小 batch 即可 |
| SigLiT (SigLIP + LiT) | 84.5% | 4 TPUv4, 2 天训练 |

## 七、Related Work 与技术谱系位置

### 7.1 技术树中的位置

```text
[Transformer, 2017]
   │
   └── [双塔跨模态预训练路线]
         │
         ├── [CLIP, Radford et al., 2021]  ← softmax contrastive loss，需大 batch
         │      │
         │      ├── [ALIGN, Jia et al., 2021]  ← 用 noisy data 扩展数据源
         │      ├── [LiT, Zhai et al., 2022]  ← 冻结图像 encoder 提高效率
         │      └── [SigLIP, Zhai et al., 2023]  ← ★ 本篇 ★ sigmoid loss 替代 softmax
         │
         └── [BLIP / BLIP-2 / Flamingo]  ← 跨模态理解的另一条路线
```

SigLIP 处于 "对齐式大规模预训练" 这个路线上，是对 CLIP 最直接的改进——不改变架构，只改变 loss 函数。

### 7.2 前驱工作：SigLIP 继承了谁？

**CLIP（Radford et al., 2021）：** SigLIP 继承了 CLIP 的双塔架构、对比学习范式、温度缩放、zero-shot 评估方式。核心改进只在一个点上——loss 函数从 softmax-based 变成 sigmoid-based。

**LiT（Zhai et al., 2022）：** 来自同一个团队。LiT 发现冻结预训练好的图像 encoder，只训练文本 encoder，可以在极大减少计算量的前提下达到接近 SOTA 的 zero-shot 效果。SigLiT 就是把 SigLIP 套进了 LiT 的训练框架中。

**ALIGN（Jia et al., 2021）：** 在 CLIP 基础上用了 18 亿更嘈杂的图文对，验证了数据量可以补偿噪声。ALIGN 也使用对比学习框架，没有尝试修改 loss 函数。

### 7.3 同期与后续影响

## 九、范式意义

**SigLIP 的范式意义：它是一个"约束解除器（constraint remover）"级别的贡献。**

具体来说，SigLIP 完成了以下几件事：

### 9.1 打破了"大 batch 是对比学习的必要条件"的认知

CLIP 发表后，社区的一个无形共识是：图像-文本对比学习需要大 batch。很多工作把算力的大部分（如果不是全部）花在维持大 batch size 上。

SigLIP 用数学分析和实验证明：**这个约束来自 loss 函数的设计，而不是对比学习本身。** 把 softmax 换成 sigmoid，N-way contrastive loss 变成 pairwise verification loss，batch size 就不再是关键瓶颈。

这就像发现一个锁其实没锁上——只是所有人都以为它是锁着的。

### 9.2 打开了小算力研究者的参与空间

CLIP 的 32,768 batch size 需要大规模的分布式训练基础设施。SigLIP 可以在 **4 个 TPUv4 芯片上用两天时间** 训练出一个极具竞争力的模型。这让更多算力有限的研究团队和机构可以参与到图文预训练的研究中来。

### 9.3 提供一个极其简单但有效的改进

SigLIP 的贡献是极简的——不引入新的模型架构、不引入新的训练技巧、不依赖新的数据源。唯一的变化是几行代码：

```python
# CLIP 的 loss
loss = cross_entropy(logits / temperature, labels)
# 需要 softmax，需要 labels 是 one-hot 索引

# SigLIP 的 loss  
loss = -mean(log_sigmoid(labels * (logits * t + b)))
# 不需要 softmax，每对独立
# labels 是 ±1 而不是 one-hot
```

这种"最简单但有效"的改进是最有力的一种科学发现——它说明之前的路径有一个非必要的复杂度。

### 9.4 分类：constraint removal node

在技术树中，SigLIP 是一个 **constraint removal node（约束解除节点）**——它不是在已有的方法上做增量改进，而是移除一个被社区默认为必要但实际上不必要的限制。

它不是 foundation node（基础节点），因为 SigLIP 没有建立新的技术范式。它也不是 refinement node（精化节点），因为它的贡献不是调优或工程化。它揭示了一个"共识假象"——一个看起来必要的条件实际上并不必要。

## 十、局限性与后续方向

### 10.1 并非对所有任务都好

SigLIP 的零样本分类效果与 CLIP 持平或略好，但论文没有系统评估在 retrieval、visual grounding 等其他任务上的表现。sigmoid loss 的 pairwise nature 可能对需要细粒度排序的任务有不同影响。

### 10.2 $O(N^2)$ 的 loss 计算

虽然小 batch 下 $O(N^2)$ 不是问题，但当 batch size 增大到 32K 或百万级别时， $N^2$ 个 pair 的 loss 计算需要一定的工程优化（如稀疏化或采样）。论文通过实验说明了适度的 batch size（32K）已经足够，但极端大 batch 场景下 sigmoid loss 的计算开销可能高于 softmax。

### 10.3 偏置和温度的调参

sigmoid loss 引入了额外的可学习参数 $b$ （偏置），增加了超参数空间。虽然论文给出了引导性的初始化建议，但最佳设置可能随数据分布变化。

### 10.4 纯数学解释的缺乏

论文没有给出一个完整的理论分析来说明为什么 sigmoid loss 能比 softmax loss 在更小 batch 下达到同样效果。梯度分析是直观的（梯度从 $N$ 中解耦），但一个更严格的理论分析——比如从对比学习的 sample complexity 角度看——可能能揭示更深层的原因。

### 10.5 稳定性与训练动态

论文提到了偏置项 $b$ 的初始值会影响训练稳定性，但没有深入分析 sigmoid loss 的训练动态——比如它是否比 softmax loss 更容易陷入坏局部最优，或在某些数据分布下会有不同的收敛行为。
