# BLIP: Bootstrapping Language-Image Pre-training for Unified Vision-Language Understanding and Generation（2022）

## 一、先搞清楚“问题在哪里”

在 BLIP 出现之前，视觉-语言预训练（Vision-Language Pre-training, VLP）已经是一个相当活跃的方向。CLIP（2021）证明了图文对比学习在大规模数据上的有效性，ALBEF（2021）引入了动量蒸馏和多模态编码器，VinVL（2021）则强调了目标检测特征的重要性。这些工作让多模态模型在图文检索、视觉问答（VQA）、图像描述（Captioning）等任务上取得了显著进展。

但是，仔细看这些工作，会发现一个奇怪的现象：**大多数模型只能做理解（Understanding），不能做生成（Generation）；或者反过来，只能做生成，不能做理解。**

具体来说，CLIP 和 ALBEF 这类模型的核心是图文对比学习，它们学习的是图文对齐的表示，擅长做检索、分类、VQA 这类判别式任务。但它们没有生成能力——你不能让 CLIP 直接写出一句图像描述。而像 Oscar（2020）这类模型，虽然能做图像描述生成，但它们的训练目标和架构设计并没有针对理解任务做优化。

这就引出一个很尴尬的局面：如果你想做一个同时需要理解和生成能力的应用——比如一个能看图说话、还能回答问题的助手——你得部署两个不同的模型，分别处理理解和生成任务。这不仅增加了部署成本，还意味着两个模型学到的知识是割裂的。

更致命的问题是：**数据质量**。VLP 模型通常从互联网上抓取图文对（image-text pairs）进行训练。互联网数据的噪声非常大——一张图片配的文字可能只有几个标签，也可能是一段完全不相关的叙述，还可能包含明显的描述错误。CLIP 和 ALBEF 的做法是直接在大规模噪声数据上训练，然后依赖对比学习的目标函数来“忽略”噪声。但这种方法并不彻底：噪声仍然会影响模型的表示质量，尤其是在需要精确语义理解的下游任务上。

所以，BLIP 这篇论文要解决的核心问题是：

> **如何设计一个统一的视觉-语言预训练框架，既能做理解又能做生成，同时还能自动清洗训练数据中的噪声？**

这个问题的难点在于：理解和生成对模型架构的要求是不同的。理解任务（如检索、VQA）需要模型学习图文之间的细粒度对齐，通常使用编码器架构（如 BERT）或编码器-解码器架构（如 ALBEF）。生成任务（如图像描述）需要模型能够自回归地生成文本，通常使用解码器架构（如 GPT）或编码器-解码器架构（如 Oscar）。如果要统一两者，架构设计上必须找到一个既能编码又能解码的平衡点。

第二个难点是数据清洗。传统的做法是人工标注高质量图文对，但这成本极高，无法规模化。另一种做法是用规则过滤，但规则太死板，无法适应多样化的噪声模式。理想的做法是**自动**从大规模噪声数据中筛选出高质量样本，同时还要能不损失数据量——因为数据量本身就是 VLP 模型的关键优势。

BLIP 的核心贡献就是同时解决了这两个问题。它提出了一个统一的多模态混合编码器-解码器（Multimodal mixture of Encoder-Decoder, MED）架构，以及一种名为 CapFilt（Captioning and Filtering）的自动数据清洗方法。

## 二、旧路线的真正问题

要理解 BLIP 的贡献，先得看清楚之前的方法到底卡在哪里。

### 2.1 架构上的割裂

VLP 模型在架构上大致可以分为三类：

1. **编码器模型（Encoder-only）**：典型代表是 CLIP 和 ALBEF。它们使用双流或单流编码器架构，学习图文对齐的表示。这类模型在理解任务上表现很好，但无法生成文本。

2. **解码器模型（Decoder-only）**：典型代表是 GPT 系列的多模态变体。它们可以生成文本，但缺乏对图像输入的细粒度编码能力，在理解任务上表现不佳。

3. **编码器-解码器模型（Encoder-Decoder）**：典型代表是 Oscar 和 VL-T5。它们有编码器和解码器两部分，理论上既能理解也能生成。但问题在于，这类模型的编码器和解码器通常是**独立训练**的——编码器负责理解，解码器负责生成，两者之间的信息流动受到限制。而且，它们的训练目标通常是单一的（要么是理解任务，要么是生成任务），而不是同时优化的。

这就意味着，在 BLIP 之前，没有一种架构能够同时做到三件事：
- 高效编码图文信息（理解）
- 自回归生成文本（生成）
- 在同一个训练过程中同时优化理解和生成目标

### 2.2 数据噪声问题

互联网图文对的数据质量可以用“惨不忍睹”来形容。以 Conceptual Captions（CC）数据集为例，很多图片配的文本是：

- “图片 12345.jpg”——这是文件名而不是描述
- “点击这里查看详情”——这是广告文案
- “我家的猫”——这是部分描述，但信息不完整
- 一段完全不相关的新闻标题

CLIP 的做法是直接用对比学习来训练，认为只要数据量足够大，模型就能自动学会忽略噪声。但这个假设并不完全成立。实际上，噪声数据会导致模型学到错误的图文关联。比如，一张猫的图片配了“狗”的文字，模型就会学到猫和狗之间的错误关联。

ALBEF 引入了动量蒸馏（Momentum Distillation）来缓解这个问题。具体做法是维护一个动量模型（momentum model），用动量模型生成伪标签（soft labels），然后用这些伪标签来训练主模型。这样做的目的是让模型对噪声不那么敏感。但动量蒸馏并不能真正去除噪声数据——它只是让模型在噪声数据上学习时更“平滑”一些。

更根本的问题是：**噪声数据不仅影响表示学习，还会影响生成任务的质量**。如果你用噪声数据训练一个图像描述生成器，它可能会学到“看图说话”的错误模式——比如看到一张风景图，却生成了“这是广告”这样的文本。

### 2.3 数据利用率低

另一个被忽视的问题是：大量可用的图文数据没有被充分利用。互联网上除了图文对（image-text pairs），还有大量的纯图像和纯文本数据。传统 VLP 模型通常只使用图文对进行训练，忽略了单模态数据。但实际上，单模态数据也可以用来增强模型的表示能力——比如用图像自监督学习来增强视觉编码器，用文本自监督学习来增强文本编码器。

BLIP 之前的模型要么只用了图文对，要么用了图文对加单模态数据，但没有系统性地设计一个训练目标来同时利用所有数据。

### 2.4 小结

所以，BLIP 要解决的三个核心问题是：
1. **架构不统一**：需要一种能同时做理解和生成的架构
2. **数据噪声大**：需要一种自动清洗数据的方法
3. **数据利用率低**：需要一种能同时利用图文对和单模态数据的训练方法

这三个问题相互关联：架构不统一导致无法同时优化理解和生成目标；数据噪声大导致生成质量差；数据利用率低导致模型能力受限。BLIP 的解决方案是同时出击，用一个统一的框架解决所有问题。

## 三、这篇论文的核心思路

BLIP 的核心思路可以用一句话概括：

> **设计一个统一的多模态混合编码器-解码器架构，配合自举式的数据清洗方法，实现高质量的统一视觉-语言理解和生成。**

这个思路包含两个关键组件：

### 3.1 多模态混合编码器-解码器（MED）

MED 架构的核心设计是：**同一个 Transformer 模型，通过不同的注意力掩码模式，同时扮演编码器、解码器和多模态编码器三种角色。**

具体来说，MED 有三种工作模式：

1. **单模态编码器（Unimodal Encoder）**：文本和图像分别编码，使用双向自注意力（bidirectional self-attention）。这对应理解任务中的图文对齐学习。

2. **图像条件文本解码器（Image-grounded Text Decoder）**：文本部分使用因果自注意力（causal self-attention），交叉注意力层接收图像特征。这对应生成任务。

3. **多模态编码器（Multimodal Encoder）**：文本和图像通过交叉注意力层交互，文本部分使用双向自注意力。这对应需要细粒度图文交互的理解任务（如 VQA）。

这种设计的巧妙之处在于：**三种模式共享大部分参数**。单模态编码器和多模态编码器共享文本编码器的参数；图像条件文本解码器和多模态编码器共享交叉注意力层的参数。这样，模型在不同任务上学习到的知识可以相互迁移。

### 3.2 CapFilt：自举式数据清洗

CapFilt 的思路也很直接：**用一个模型生成高质量描述，再用另一个模型过滤噪声数据。**

具体来说，CapFilt 包含两个模块：

1. **Captioner（描述器）**：一个 MED 解码器，负责为图像生成文本描述。Captioner 在高质量数据（如 COCO Captions）上微调，然后对大规模噪声数据（如 Conceptual Captions）中的图像生成新的描述。

2. **Filter（过滤器）**：一个 MED 多模态编码器，负责判断图文对是否匹配。Filter 在高质量数据上微调，然后对原始噪声数据和 Captioner 生成的描述进行过滤，只保留匹配的图文对。

这样，通过 Captioner 和 Filter 的协同工作，可以从噪声数据中自动提取出高质量的图文对。然后，这些高质量数据被用来重新训练模型，形成**自举（bootstrapping）**循环——模型自己生成数据，自己过滤数据，再用过滤后的数据训练自己。

### 3.3 训练目标

BLIP 同时优化三个目标：

1. **图文对比学习（Image-Text Contrastive Loss, ITC）**：对齐图文表示，用于理解任务
2. **图文匹配（Image-Text Matching Loss, ITM）**：学习细粒度图文交互，用于理解任务
3. **语言建模（Language Modeling Loss, LM）**：自回归生成文本，用于生成任务

这三个目标对应 MED 的三种工作模式：ITC 用单模态编码器，ITM 用多模态编码器，LM 用图像条件文本解码器。

### 3.4 整体流程

```text
大规模噪声图文数据
 ↓
Captioner（生成新描述）
 ↓
Filter（过滤不匹配的图文对）
 ↓
高质量图文数据
 ↓
MED 模型（同时优化 ITC + ITM + LM）
 ↓
理解和生成任务
```

这个流程的核心是：**用模型自己生成的数据来训练自己，形成一个正向循环**。

## 四、整体结构与 forward 流程

现在来看 BLIP 的 forward 流程。我们假设输入是一个图文对 (image, text)。

### 4.1 输入表示

**图像编码**：
- 使用 ViT（Vision Transformer）作为图像编码器
- 图像被分割成 patches，每个 patch 通过线性投影变成 patch embedding
- 加上位置编码和 [CLS] token
- 输出：图像特征序列（包括 [CLS] token 和 patch features）

**文本编码**：
- 使用 BERT 的 tokenizer 将文本分词
- 每个 token 加上位置编码和 token 类型编码
- 加上 [CLS] 和 [SEP] token
- 输出：文本 token 序列

### 4.2 三种工作模式的 forward 流程

#### 模式一：单模态编码器（用于 ITC 损失）

```text
图像: [CLS_img] patch1 patch2 ... patchN
文本: [CLS_text] token1 token2 ... tokenM
```

在这个模式下，图像和文本分别通过独立的编码器（共享参数但输入不同）：
- 图像编码器输出图像特征：`I = [i_cls, i_1, i_2, ..., i_N]`
- 文本编码器输出文本特征：`T = [t_cls, t_1, t_2, ..., t_M]`

注意：这里的文本编码器使用**双向自注意力**，即每个 token 可以看到所有其他 token。这是标准的 BERT 编码方式。

ITC 损失的目的是对齐 `i_cls` 和 `t_cls` 的表示。具体来说，在一个 batch 内，计算所有图文对的余弦相似度，然后用对比学习（InfoNCE）损失来拉近匹配的图文对，推远不匹配的图文对。

#### 模式二：多模态编码器（用于 ITM 损失）

```text
图像: [CLS_img] patch1 patch2 ... patchN
文本: [CLS_text] token1 token2 ... tokenM
```

在这个模式下，图像和文本通过交叉注意力层交互。具体流程：

1. 图像编码器输出图像特征（同模式一）
2. 文本编码器的每一层都插入交叉注意力层，接收图像特征作为 key 和 value
3. 文本部分的注意力仍然是双向的
4. 输出：图文融合后的文本特征

ITM 损失是一个二分类任务：给定一个图文对，判断它们是否匹配。具体做法是取 [CLS_text] 的表示，通过一个线性分类器预测匹配概率。

ITM 的难点在于负样本的构造。如果随机采样负样本，模型很容易学会“这些不匹配的图文对很容易区分”，而忽略了细粒度的不匹配。BLIP 使用了**硬负样本挖掘（hard negative mining）**：在一个 batch 内，选择与正样本图文对比相似度最高的图文对作为负样本。这样模型必须学会识别更细微的不匹配。

#### 模式三：图像条件文本解码器（用于 LM 损失）

```text
图像: [CLS_img] patch1 patch2 ... patchN
文本: [CLS_text] token1 token2 ... tokenM
```

在这个模式下，文本解码器使用**因果自注意力**（causal self-attention），即每个 token 只能看到它自己和之前的 token。同时，解码器的每一层都插入交叉注意力层，接收图像特征作为 key 和 value。

LM 损失就是标准的自回归语言建模损失：预测下一个 token 的概率，最小化负对数似然。

### 4.3 训练时的 forward 顺序

在训练时，BLIP 并不是依次执行三种模式，而是**同时计算三个损失**。具体来说：

1. 图像编码器一次 forward，输出图像特征
2. 文本编码器一次 forward（使用双向注意力），输出文本特征
3. 从文本特征中计算 ITC 损失
4. 文本编码器再次 forward（这次使用交叉注意力和双向注意力），输出图文融合特征
5. 从图文融合特征中计算 ITM 损失
6. 文本解码器 forward（使用因果注意力和交叉注意力），输出生成概率
7. 从生成概率中计算 LM 损失
8. 三个损失加权求和，反向传播

注意：步骤 2 和步骤 4 的文本编码器是同一个模型，但注意力模式不同。在实现上，这通常意味着需要两次 forward 文本编码器（一次不带交叉注意力，一次带交叉注意力）。

### 4.4 CapFilt 的 forward 流程

CapFilt 的 forward 流程分为两个阶段：

**第一阶段：训练 Captioner 和 Filter**

1. 在高质量数据（如 COCO Captions）上微调一个 MED 解码器作为 Captioner
2. 在高质量数据上微调一个 MED 多模态编码器作为 Filter
3. 这两个模型是独立训练的

**第二阶段：数据清洗**

1. Captioner 对噪声数据中的图像生成新的描述（使用 beam search 或 nucleus sampling）
2. Filter 对原始噪声图文对和 Captioner 生成的图文对进行匹配判断
3. 只保留 Filter 判断为匹配的图文对
4. 用清洗后的数据重新训练 MED 模型

这个流程的关键在于：Captioner 生成的描述可能比原始噪声文本更准确，而 Filter 可以过滤掉不匹配的图文对。两者结合，可以从噪声数据中提取出高质量的图文对。

## 五、核心模块逐个解释

### 5.1 图像编码器：ViT

BLIP 使用 ViT 作为图像编码器，而不是传统的 CNN。这个选择有几个原因：

1. **ViT 的全局感受野**：CNN 需要堆叠很多层才能看到全局信息，而 ViT 在第一层就能看到所有 patch。这对于图文对齐任务很重要——模型需要理解图像的整体语义，而不是局部特征。

2. **ViT 的序列化输出**：ViT 输出的是 token 序列，可以直接被 Transformer 处理，无需额外的特征映射。

3. **ViT 的可扩展性**：ViT 在大规模数据上表现更好，符合 BLIP 的大规模预训练需求。

BLIP 使用的是 ViT-B/16 和 ViT-L/14 两种配置，与 CLIP 相同。实际上，BLIP 的图像编码器初始化自 CLIP 的 ViT 权重。这是一个很实际的选择：CLIP 已经在大规模图文对上预训练过，初始化权重可以提供更好的起点。

### 5.2 文本编码器/解码器：BERT 的变体

BLIP 的文本编码器/解码器基于 BERT 架构，但做了两个关键修改：

1. **交叉注意力层**：在每层 Transformer 中插入交叉注意力层，用于接收图像特征。这个交叉注意力层的 key 和 value 来自图像编码器的输出，query 来自文本 token。

2. **注意力掩码**：根据工作模式，使用不同的注意力掩码：
   - 单模态编码器模式：全双向掩码（所有 token 可见）
   - 图像条件文本解码器模式：因果掩码（每个 token 只看到自己和之前的 token）
   - 多模态编码器模式：全双向掩码 + 交叉注意力

这个设计的巧妙之处在于：**只需要修改注意力掩码，就可以在编码器和解码器之间切换**。这意味着同一个模型可以同时处理理解和生成任务，而且大部分参数是共享的。

### 5.3 交叉注意力层

交叉注意力层是 BLIP 的核心模块。它的输入是：

- **Query**：来自文本 token
- **Key 和 Value**：来自图像 token（包括 [CLS] 和 patch features）

交叉注意力的计算方式与标准注意力相同：

```
Attention(Q, K, V) = softmax(QK^T / sqrt(d)) V
```

其中 Q 是文本 token 的表示，K 和 V 是图像 token 的表示。

交叉注意力层的位置在每层 Transformer 中，通常在自注意力层之后、前馈网络之前。这样，文本 token 首先通过自注意力层与其他文本 token 交互，然后通过交叉注意力层与图像 token 交互，最后通过前馈网络进行非线性变换。

### 5.4 Captioner 和 Filter

Captioner 和 Filter 是 CapFilt 的两个核心模块。

**Captioner**：
- 架构：MED 图像条件文本解码器
- 输入：图像特征 + [CLS] token
- 输出：文本描述
- 训练：在高质量数据（如 COCO Captions）上微调，使用 LM 损失
- 推理：使用 beam search 或 nucleus sampling 生成描述

Captioner 的设计目标是：**生成高质量的、与图像匹配的描述**。在高质量数据上微调后，Captioner 学会了“看图说话”的能力。然后，它对噪声数据中的图像生成新的描述，这些描述可能比原始噪声文本更准确。

**Filter**：
- 架构：MED 多模态编码器
- 输入：图像特征 + 文本 token
- 输出：匹配概率
- 训练：在高质量数据上微调，使用 ITM 损失
- 推理：对图文对进行二分类判断

Filter 的设计目标是：**判断图文对是否匹配**。在高质量数据上微调后，Filter 学会了识别图文不匹配的模式。然后，它对原始噪声图文对和 Captioner 生成的图文对进行过滤，只保留匹配的样本。

### 5.5 动量蒸馏（Momentum Distillation）

BLIP 还使用了动量蒸馏（来自 ALBEF）来进一步缓解噪声问题。动量蒸馏的核心思想是：

1. 维护一个动量模型（momentum model），其参数是主模型参数的指数移动平均（EMA）
2. 动量模型对同一个输入生成伪标签（soft labels）
3. 主模型的学习目标不仅包括真实标签，还包括动量模型的伪标签

具体来说，在 ITC 损失中，动量模型生成图文对之间的相似度分布，主模型学习去匹配这个分布。在 ITM 损失中，动量模型生成图文匹配的概率，主模型学习去匹配这个概率。

动量蒸馏的作用是：**让模型的学习过程更平滑，减少对噪声标签的过拟合**。因为动量模型是主模型的 EMA，它比主模型更稳定，生成的伪标签比真实标签更可靠。

## 六、核心公式与数学直觉

### 6.1 ITC 损失（图文对比学习）

ITC 损失的目标是拉近匹配图文对的距离，推远不匹配图文对的距离。

给定一个 batch 的 N 个图文对，计算图像 [CLS] token 和文本 [CLS] token 的余弦相似度：

```
s(I_i, T_j) = (i_cls_i · t_cls_j) / (||i_cls_i|| * ||t_cls_j||)
```

然后计算图像到文本的对比损失：

```
L_itc_img = -1/N * sum_i log( exp(s(I_i, T_i) / τ) / sum_j exp(s(I_i, T_j) / τ) )
```

和文本到图像的对比损失：

```
L_itc_txt = -1/N * sum_j log( exp(s(I_j, T_j) / τ) / sum_i exp(s(I_i, T_j) / τ) )
```

最终的 ITC 损失是两者的平均：

```
L_itc = (L_itc_img + L_itc_txt) / 2
```

其中 τ 是温度参数，控制分布的平滑程度。

**这个公式的直觉是**：在一个 batch 内，每个图像应该与它对应的文本更相似，与其他文本更不相似。对比损失实际上是在做一个 N 分类任务——对于每个图像，从 N 个文本中找出正确的那个。

但这里有一个问题：**对比损失假设 batch 内的负样本都是真正的负样本**。但实际上，一个 batch 内可能存在“伪负样本”——比如两张不同的图片都描述了“一只猫”，它们对应的文本可能是相似的。在这种情况下，对比损失会把它们推开，导致模型学到错误的表示。

动量蒸馏可以缓解这个问题：动量模型生成的伪标签会告诉主模型，某些“负样本”实际上与正样本很相似，不应该被完全推开。

### 6.2 ITM 损失（图文匹配）

ITM 损失是一个二分类任务：给定一个图文对，判断它们是否匹配。

具体来说，取多模态编码器的 [CLS_text] 表示，通过一个线性分类器预测匹配概率：

```
p_match = softmax(MLP(h_[CLS_text]))
```

损失函数是标准的交叉熵损失：

```
L_itm = -1/N * sum_i [y_i * log(p_match_i) + (1-y_i) * log(1-p_match_i)]
```

其中 y_i 是标签（1 表示匹配，0 表示不匹配）。

**关键设计：硬负样本挖掘**

ITM 的难点在于负样本的构造。如果随机采样负样本，模型很容易学会“这些不匹配的图文对很容易区分”，而忽略了细粒度的不匹配。

BLIP 使用硬负样本挖掘：在一个 batch 内，对于每个图像，选择与它图文对比相似度最高但实际不匹配的文本作为负样本。这样，模型必须学会识别更细微的不匹配。

具体做法是：

1. 计算 batch 内所有图文对的 ITC 相似度矩阵 S（N×N）
2. 对于每个图像 i，从 S[i, :] 中选出相似度最高但 j≠i 的文本 j 作为负样本
3. 对于每个文本 j，从 S[:, j] 中选出相似度最高但 i≠j 的图像 i 作为负样本

这个设计的直觉是：**如果两个图文对的 ITC 相似度很高，但它们实际上不匹配，那么它们一定存在某种细粒度的不匹配**。模型必须学会识别这种不匹配，而不是简单地依赖全局相似度。

### 6.3 LM 损失（语言建模）

LM 损失是标准的自回归语言建模损失：

```
L_lm = -1/N * sum_i sum_t log P(t_i_t | t_i_<t, I_i)
```

其中 t_i_t 是第 i 个文本的第 t 个 token，t_i_<t 是前 t-1 个 token，I_i 是第 i 个图像的表示。

这个损失的目标是：**给定图像，生成正确的文本描述**。在生成时，模型使用因果自注意力，每个 token 只能看到它自己和之前的 token。

**这个公式的直觉是**：模型需要学习“看图说话”的能力。给定图像特征，模型需要预测下一个最可能的 token。这个过程中，模型不仅需要理解图像内容，还需要掌握语言的语法和语义。

### 6.4 总损失

BLIP 的总损失是三个损失的加权和：

```
L = L_itc + L_itm + L_lm
```

这里没有权重系数，因为三个损失的量级是接近的。

**为什么同时优化三个损失？**

- ITC 损失强制图文表示对齐，这是理解和生成的基础
- ITM 损失强制模型学习细粒度的图文交互，这对理解任务（如 VQA）很重要
- LM 损失强制模型学习生成能力，这对生成任务（如图像描述）很重要

三者互补：ITC 提供全局对齐，ITM 提供局部交互，LM 提供生成能力。

### 6.5 CapFilt 的过滤公式

CapFilt 中，Filter 对图文对进行匹配判断。给定一个图文对 (I, T)，Filter 输出匹配概率 p_match(I, T)。

过滤策略是：保留 p_match(I, T) > 0.5 的图文对。

但这里有一个微妙的问题：**Captioner 生成的描述可能比原始噪声文本更准确，但 Filter 可能对 Captioner 生成的描述有偏见**。因为 Filter 是在高质量数据上训练的，它可能更倾向于保留与高质量数据分布相似的描述。

BLIP 的实验表明，这种偏见并不严重。实际上，Captioner 生成的描述通常比原始噪声文本更接近高质量数据分布，因此 Filter 对它们更友好。

### 6.6 动量蒸馏的公式

动量蒸馏的核心是：主模型学习去匹配动量模型生成的伪标签。

对于 ITC 损失，动量模型生成伪标签：

```
q_itc(I_i, T_j) = softmax(s_m(I_i, T_j) / τ_m)
```

其中 s_m 是动量模型的相似度函数，τ_m 是动量模型的温度参数。

主模型的学习目标是最小化 KL 散度：

```
L_itc_distill = KL(p_itc || q_itc)
```

其中 p_itc 是主模型预测的相似度分布。

类似地，对于 ITM 损失，动量模型生成伪标签：

```
q_itm(I_i, T_j) = softmax(logits_m(I_i, T_j))
```

主模型的学习目标是最小化交叉熵：

```
L_itm_distill = CE(p_itm, q_itm)
```

**这个设计的直觉是**：动量模型是主模型的 EMA，它比主模型更稳定、更少受噪声影响。因此，动量模型生成的伪标签比真实标签更可靠。主模型学习去匹配这些伪标签，相当于从更稳定的老师那里学习。

## 八、复杂度、效率与可扩展性分析

### 8.1 计算复杂度

BLIP 的计算复杂度主要来自三个方面：

1. **图像编码器（ViT）**：O(N_patches^2 * d)，其中 N_patches 是 patch 数量（224×224 图像约 196 个 patch），d 是隐藏维度

2. **文本编码器/解码器**：O(N_tokens^2 * d)，其中 N_tokens 是文本 token 数量（通常 30-50）

3. **交叉注意力层**：O(N_tokens * N_patches * d)，其中 key/value 来自图像特征，query 来自文本 token

从复杂度来看，交叉注意力层的复杂度是 O(N_tokens * N_patches)，比自注意力的 O(N^2) 要低（因为 N_tokens 和 N_patches 通常都远小于 196 或 50 的平方）。但交叉注意力层需要额外的计算，因为 key/value 来自不同的模态。

### 8.2 参数量

BLIP 的参数量与 ViT 和 BERT 相当：

- ViT-B/16：86M 参数
- BERT-base：110M 参数
- 交叉注意力层：约 40M 参数（每层约 2M，共 12 层）
- 总计：约 235M 参数

这个参数量与 ALBEF 相当，比 CLIP（428M）小，但性能更好。

### 8.3 训练效率

BLIP 的训练效率受到三个因素的影响：

1. **三个损失同时计算**：需要三次 forward 文本编码器/解码器，增加了训练时间

2. **硬负样本挖掘**：需要计算 ITC 相似度矩阵，增加了计算量

3. **动量蒸馏**：需要维护动量模型，增加了内存消耗

但 BLIP 的作者在实现上做了优化：
- 使用梯度累积来模拟更大的 batch size
- 使用混合精度训练
- 使用分布式训练

### 8.4 可扩展性

BLIP 的可扩展性很好：

1. **模型规模**：可以从 ViT-B 扩展到 ViT-L，参数量从 235M 增加到 470M

2. **数据规模**：CapFilt 可以处理任意规模的数据，数据量越大，清洗后的数据质量越高

3. **任务范围**：MED 架构可以适应各种理解和生成任务

### 10.2 核心模型：med.py

`med.py` 是 MED 架构的核心实现。主要类：

- `BertEncoder`：BERT 编码器，支持三种注意力模式
- `BertLayer`：单层 Transformer，包含自注意力、交叉注意力、前馈网络
- `BertAttention`：自注意力或交叉注意力
- `BertSelfAttention`：自注意力实现
- `BertCrossAttention`：交叉注意力实现

关键代码片段（forward 函数）：

```python
class BertLayer(nn.Module):
    def forward(self, hidden_states, attention_mask, 
                encoder_hidden_states, encoder_attention_mask):
        # 自注意力
        self_attention_output = self.attention(
            hidden_states, attention_mask
        )
        # 交叉注意力（如果有 encoder_hidden_states）
        if encoder_hidden_states is not None:
            cross_attention_output = self.crossattention(
                self_attention_output, 
                encoder_hidden_states, 
                encoder_attention_mask
            )
        else:
            cross_attention_output = self_attention_output
        # 前馈网络
        layer_output = self.feedforward(cross_attention_output)
        return layer_output
```

这个实现非常简洁：通过 `encoder_hidden_states` 是否为 None 来控制是否使用交叉注意力。当 `encoder_hidden_states` 为 None 时，模型就是单模态编码器；当不为 None 时，模型就是多模态编码器或解码器。

### 10.4 CapFilt 实现：capfilt.py

`capfilt.py` 实现了 CapFilt 的数据清洗流程。

关键代码片段：

```python
class CapFilt:
    def __init__(self, captioner, filter, device):
        self.captioner = captioner
        self.filter = filter
        self.device = device
    
    def generate_captions(self, images):
        # Captioner 生成描述
        captions = self.captioner.generate(
            images, 
            sample=True,  # 使用采样生成多样化的描述
            top_p=0.9,    # nucleus sampling
            max_length=50
        )
        return captions
    
    def filter_pairs(self, images, texts):
        # Filter 判断图文对是否匹配
        scores = self.filter(images, texts)
        # 保留匹配的图文对
        keep = scores > 0.5
        return keep
```

### 10.5 训练脚本

`train_retrieval.py` 展示了 BLIP 的训练流程：

```python
# 前向传播
image_features = model.visual_encoder(images)
text_features = model.text_encoder(texts)

# 计算 ITC 损失
itc_loss = model.compute_itc_loss(image_features, text_features)

# 计算 ITM 损失（含硬负样本挖掘）
hard_negatives = model.hard_negative_mining(image_features, text_features)
itm_loss = model.compute_itm_loss(image_features, text_features, hard_negatives)

# 计算 LM 损失
lm_loss = model.compute_lm_loss(image_features, texts)

# 总损失
loss = itc_loss + itm_loss + lm_loss
loss.backward()
optimizer.step()
```

## 十四、局限性与后续方向

### 14.1 局限性

BLIP 的局限性包括：

1. **计算成本高**：同时优化三个损失需要三次 forward 文本编码器/解码器，训练成本较高

2. **数据依赖性强**：CapFilt 依赖高质量数据（如 COCO）来训练 Captioner 和 Filter。如果高质量数据不足，CapFilt 的效果会下降

3. **生成质量有限**：BLIP 的生成能力受限于模型规模和训练数据。在复杂场景下，生成的描述可能不够准确

4. **架构灵活性不足**：MED 架构虽然统一了理解和生成，但三种模式之间的切换需要修改注意力掩码，实现起来比较复杂

5. **缺乏推理能力**：BLIP 主要做图文理解和生成，缺乏复杂的推理能力（如多步推理、常识推理）

### 14.2 后续方向

BLIP 的后续工作包括：

1. **BLIP-2**：使用 Q-Former 连接冻结的图像编码器和语言模型，大幅降低了训练成本，同时保持了性能

2. **InstructBLIP**：基于 BLIP-2 的指令微调模型，可以执行更复杂的多模态任务

3. **LLaVA**：使用视觉编码器 + 语言模型的多模态大模型，进一步提升了多模态理解和生成能力

4. **数据清洗的自动化**：CapFilt 的思路被很多后续工作采用，如使用大语言模型生成描述、使用多模态模型过滤噪声数据

### 14.3 未解决的问题

BLIP 没有解决的问题包括：

1. **长文本生成**：BLIP 生成的描述通常较短（50 token 以内），无法生成详细的段落

2. **视频理解**：BLIP 只处理图像，没有扩展到视频

3. **多语言支持**：BLIP 主要针对英文，没有支持多语言

4. **实时推理**：BLIP 的推理速度较慢，不适合实时应用

## 十五、小结

BLIP 是视觉-语言预训练领域的一个重要工作。它的核心贡献是：

1. **提出了 MED 架构**：统一了理解和生成，让同一个模型可以同时做图文检索、VQA 和图像描述

2. **提出了 CapFilt**：实现了自动数据清洗，从噪声数据中提取高质量图文对

3. **证明了多任务学习的有效性**：同时优化 ITC + ITM + LM 三个损失，三个任务相互促进

BLIP 的价值不只是提出了一个更好的模型，而是把 VLP 这条路线从“只能做理解或只能做生成”推进到了“既能做理解又能做生成”的阶段。它的架构设计和数据策略为后来的 BLIP-2、InstructBLIP、LLaVA 等模型奠定了基础。

直白说，BLIP 的价值在于：

> **它让多模态模型从“偏科生”变成了“全科生”，而且证明了“全科”比“偏科”学得更好。**
