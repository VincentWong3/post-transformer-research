## 一、先搞清楚"问题在哪里"

2017 年 Transformer 横空出世之后，NLP 领域几乎被它统治了。机器翻译、文本分类、问答系统……只要能把任务写成序列，Transformer 就能刷出 state-of-the-art 的结果。BERT、GPT-2、RoBERTa 一个接一个地刷新榜单。

但这套架构有个硬伤：**self-attention 的计算量跟序列长度的平方成正比**。

式子很简单。对于一个长度为 $n$ 的序列，self-attention 要算一个 $n \times n$ 的 attention 矩阵 $P$：

$$P = \text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right)$$

$Q$ 和 $K$ 都是 $n \times d$ 的矩阵，$QK^T$ 是 $n \times n$ 的矩阵乘法。所以每算一次 self-attention，时间和空间复杂度都是 $O(n^2)$。

对短序列（比如 $n=128$）来说这不是问题。但一旦序列变长，问题就来了：

- BERT 的默认长度是 512，这还算能忍
- 但要处理长文档、科学文献、蛋白质序列、或者长视频特征，$n$ 可能到 4096、8192 甚至更多
- 此时 $n^2$ 项会爆炸：$n=4096$ 时 attention 矩阵有 1600 万个元素，$n=16384$ 时有 2.68 亿个

更要命的是，这不只是推理时的问题。训练时每个 step 都要算这个矩阵，显存和算力都扛不住。BERT-Large 要在 16 个 Cloud TPU 上训练四天，GPT-3 的算力开销比 GPT-2 高了几个数量级。

所以到了 2020 年，一个核心问题变得越来越紧迫：

> **Transformer 必须全量计算 attention 吗？能不能在不显著牺牲性能的前提下，把 $O(n^2)$ 降下来？**

这不是一个"锦上添花"的问题。如果解决不了，Transformer 在处理长序列方面的潜力就被锁死了。这就是 Linformer 这篇论文出场时面对的战场。

## 二、旧路线的真正问题

在 Linformer 之前，已经有好几条路线尝试解决 attention 的效率瓶颈。

### 稀疏 attention

思路很直观：不让每个 token 跟所有 token 做 attention，只跟一部分做。比如 Sparse Transformer（Child et al., 2019）只计算 attention 矩阵对角线附近的位置，或者按某种固定模式（strided、fixed）选择 attention 对。

复杂度可以降到 $O(n\sqrt{n})$。听起来很不错，对吧？

但问题在于：**稀疏 pattern 是手工设计的，不是学出来的**。你事先规定了每个 token 只能看哪些位置，那万一任务需要的长距离依赖不在你的 pattern 里呢？实验也证实了这一点——Qiu et al.（2019）的 blockwise self-attention 只获得了 20% 的加速，却掉了 2% 的性能。

换句话说，稀疏 attention 在"省计算"和"保留信息"之间做了一个人工折中，而这个折中往往两头不讨好。

### LSH attention（Reformer）

Reformer（Kitaev et al., 2020）走了另一条路：用局部敏感哈希（locally-sensitive hashing）把 token 分到不同的 bucket 里，每个 token 只跟同 bucket 的 token 做 attention。

理论上复杂度是 $O(n \log n)$。但论文自己承认，实际上只有当 $n > 2048$ 时才能看到效率提升。而且 LSH 需要多轮哈希来保证召回率，多轮哈希又增加了顺序操作的数量，反而拖慢了实际速度。

### 其他通用技术

还有一些不针对 attention 的通用优化：

- **混合精度训练**：用 fp16 代替 fp32，对 attention 本身的 $O(n^2)$ 瓶颈没有直接影响
- **知识蒸馏**：把大模型压缩成小模型，但训练时 teacher 模型仍然很慢，而且 student 模型通常有 2.5% 左右的性能损失
- **梯度检查点**：用时间换内存，不加速推理

这些方法各有各的局限，但有一个共同点：**它们没有真正解决 self-attention 的 $O(n^2)$ 结构性问题**。它们只是在现有的 $O(n^2)$ 框架下打补丁。

所以真正需要的是：**重新思考 self-attention 的计算结构本身**。能不能在数学上找到一种方式，让 attention 的计算量跟序列长度成线性关系，同时又不明显损失信息？

## 三、这篇论文的核心思路

> 这篇论文的核心思路是：**self-attention 的 context mapping matrix $P$ 本质上是低秩的，所以可以用两个线性投影把 Key 和 Value 压缩到低维空间，再把 attention 算在压缩后的空间里。**

这句话拆开来看：

- 标准 attention 的 $P \in \mathbb{R}^{n \times n}$ 是 $n \times n$ 的矩阵
- 作者发现 $P$ 的奇异值分布呈长尾状——前几百个奇异值就占了绝大部分能量
- 这意味着 $P$ 的信息可以用一个低秩矩阵近似
- 基于这个发现，作者在计算 attention 之前先把 Key 和 Value 投影到低维空间（$n \rightarrow k$，$k \ll n$）
- 这样 attention 矩阵就变成 $n \times k$ 的了，复杂度从 $O(n^2)$ 降到 $O(nk)$

这个思路的漂亮之处在于：**它不靠稀疏性来省计算，而是靠数学上的低秩近似**。你不必手工设计 attention pattern，也不必用哈希做近似匹配，一切都在连续的可微空间里完成。

用流程图画出来大概是这样的：

```text
标准 Transformer:
Q (n×d) ──┐
          ├──→ QK^T (n×n) ──softmax──→ P (n×n) ──×V──→ output (n×d)
K (n×d) ──┘
V (n×d) ──────────────────────────────────────────┘

Linformer:
Q (n×d) ──┐
          ├──→ Q·(E·K)^T (n×k) ──softmax──→ P̄ (n×k) ──×(F·V)──→ output (n×d)
K (n×d) ──┤                          ↑                    ↑
          └──→ E·K (k×d) ────────────┘                    │
V (n×d) ─────────────────────→ F·V (k×d) ────────────────┘
```

其中 $E, F \in \mathbb{R}^{k \times n}$ 是线性投影矩阵，可学习参数，把 Key 和 Value 从 $n$ 维压缩到 $k$ 维。

⚠️ 因果注意力的限制：E 和 F 在序列维度上做全局投影（整个 n 维序列→k 维），因此标准的 causal masking（每个 token 只能看前面的 token）不能直接应用。Linformer 论文主要验证了 BERT 风格的编码器任务；对于 GPT 风格的生成任务，需要额外设计。

本质上它做的是：**用 Key 和 Value 侧的线性压缩来代替 attention 矩阵的低秩分解，避免了直接做 SVD 的计算开销**。

## 四、整体结构与 forward 流程

Linformer 不是一个全新的架构，而是在标准 Transformer 的 self-attention 层里做了手术。整体结构和 BERT/RoBERTa 几乎一样，唯一的区别就是 Multi-Head Attention 内部的实现方式。

### 整体结构

```text
input tokens
    ↓
Token Embedding + Position Embedding
    ↓
[Encoder Layer × L]  ← 跟标准 Transformer 一样的堆叠方式
    ├── Multi-Head Self-Attention (Linformer)
    │     └── 这里换成了线性 attention
    ├── LayerNorm + Residual
    ├── Feed-Forward Network (FFN)
    └── LayerNorm + Residual
    ↓
[CLS] token → classifier
```

### Forward 流程（以 single head 为例）

假设输入序列长度 $n=512$，隐藏维度 $d=768$，投影维度 $k=128$（论文的默认配置）。

**Step 1: 输入投影**

输入 $X \in \mathbb{R}^{n \times d}$ 通过三个不同的权重矩阵得到 Q, K, V：

$$Q = XW^Q,\quad K = XW^K,\quad V = XW^V$$

其中 $W^Q, W^K \in \mathbb{R}^{d \times d_k}$，$W^V \in \mathbb{R}^{d \times d_v}$。在论文的实现中 $d_k = d_v = d / h$，$h$ 是 head 数量。

这一步跟标准 Transformer 完全一样。Q, K, V 的形状是 $n \times d$（准确说是 $n \times d_k$，为了简化讨论这里用 $d$ 统一表示）。

**Step 2: Key 和 Value 的线性投影（这是 Linformer 的核心改动）**

在标准 Transformer 中，接下来要算 $QK^T$，得到一个 $n \times n$ 的矩阵。

但在 Linformer 里，Key 和 Value 先分别被两个投影矩阵 $E, F \in \mathbb{R}^{k \times n}$ 压缩：

$$K_{\text{proj}} = EK \in \mathbb{R}^{k \times d}$$
$$V_{\text{proj}} = FV \in \mathbb{R}^{k \times d}$$

注意这里 $E$ 和 $F$ 是把序列长度维度从 $n$ 压缩到 $k$：原始的 Key 是 $n$ 个 token 每个 $d$ 维，投影后是 $k$ 个"压缩后的上下文表示"每个 $d$ 维。

如上一节所述，这种全局投影方式与 causal masking 不兼容，限制了它在生成任务中的直接使用。

**Step 3: 计算线性 attention**

现在 Q 仍然是 $n$ 个 token 的 query 表示（$n \times d$），但 Key 被压缩成了 $k$ 个"原型"（$k \times d$）。计算 attention 得分：

$$P̄ = \text{softmax}\left(\frac{Q K_{\text{proj}}^T}{\sqrt{d}}\right) \in \mathbb{R}^{n \times k}$$

这个 $P̄$ 是 $n \times k$ 的矩阵，远小于标准 attention 的 $n \times n$。

**Step 4: 加权求和**

用 $P̄$ 对压缩后的 Value 做加权求和：

$$\text{head}_i = P̄ \cdot V_{\text{proj}} \in \mathbb{R}^{n \times d}$$

**Step 5: Multi-Head 合并**

所有头的输出拼接起来，通过输出投影 $W^O$：

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) W^O$$

这一步跟标准 Transformer 完全一样。

### 形状变化一览

```text
标准 Transformer:
Q (n×d), K (n×d), V (n×d)
    → QK^T (n×n) → softmax → P (n×n)
    → PV (n×d)
复杂度: O(n²·d)

Linformer:
Q (n×d), K (n×d), V (n×d)
    → EK (k×d), FV (k×d)          ← 投影压缩
    → Q(EK)^T (n×k) → softmax → P̄ (n×k)
    → P̄(FV) (n×d)
复杂度: O(n·k·d)
```

当 $k \ll n$ 时（比如 $k=128$ 对 $n=512$），复杂度从 $O(n^2)$ 变成了 $O(n)$。

## 五、核心模块逐个解释

### 5.1 低秩发现（Section 3）

这是整篇论文的出发点。作者先做了实验观察，再做了理论证明。

**实验观察：** 作者在 RoBERTa-base（12层）和 RoBERTa-large（24层）上，对 Wiki103 和 IMDB 两个数据集做了 spectrum analysis。具体做法是：对每层每个 head 的 attention 矩阵 $P$ 做奇异值分解，然后画累计奇异值占比。

结果非常清楚：$P$ 的奇异值呈长尾分布。前几十个奇异值就占了总能量的绝大部分。在高层（靠近输出层），这种长尾分布比低层更严重——说明高层的 attention 矩阵秩更低。

具体来说，当 $n=512$ 时，累计到第 128 个奇异值时就已经能恢复 $P$ 的大部分信息。

**理论证明（Theorem 1）：** 作者用 Johnson–Lindenstrauss 引理证明了：对于任何 $Q, K, V \in \mathbb{R}^{n \times d}$，存在一个低秩矩阵 $\tilde{P}$，满足：

$$\Pr(\|\tilde{P}w^T - Pw^T\| \leq \epsilon \|Pw^T\|) > 1 - o(1)$$

且 $\text{rank}(\tilde{P}) = \Theta(\log n)$。

这个定理说的是：attention 的输出 $Pw^T$（对任意 Value 列向量 $w$）可以被一个秩为 $\Theta(\log n)$ 的矩阵 $\tilde{P}$ 以高概率近似。注意这里要求的秩是 $\log n$，远小于 $n$。

证明的核心技巧是用一个随机投影矩阵 $R \in \mathbb{R}^{k \times n}$ 来构造 $\tilde{P} = \exp(A) \cdot D_A^{-1} R^T R$，然后利用 JL 引理证明这个构造以高概率保持 pairwise 距离。

**理论到实践的差距：** Theorem 1 理论上的秩要求是 $\Theta(\log n)$（$n=512$ 时约 $\log 512 \approx 9$），但论文实验用 $k=128$ 而不是 $k=9$。这是为什么？几个原因：(1) JL 引理的常数因子 —— 保证 $\epsilon$ 近似需要 $k = O(\epsilon^{-2} \log n)$，$\epsilon$ 越小常数越大；(2) 多头分解 —— $k$ 被分配到 $h$ 个 head 上，每个 head 的实际压缩维度是 $k/h$；(3) 实际优化中，$k$ 过小时 softmax 后的分辨能力不足，$k=128$ 是一个在理论上行得通的边界和实际上最优的平衡点。论文的消融实验（Table 4 及 Figure 3）也验证了 $k$ 从 32 到 256 的精度曲线：$k=128$ 时精度与 full attention 基本持平，$k=32$ 则有明显下降。

### 5.2 线性投影模块（Section 4）

理论告诉我们可以用低秩近似，但怎么实现呢？直接对每个 attention 矩阵做 SVD 是不行的——SVD 本身的计算量就很大。

所以作者换了一个思路：**不直接去近似 $P$，而是让 Key 和 Value 先在序列维度上被压缩**。

具体来说，对每个 head $i$，引入两个可学习的投影矩阵 $E_i, F_i \in \mathbb{R}^{k \times n}$：

$$\overline{\text{head}_i} = \text{Attention}(QW_i^Q, E_i K W_i^K, F_i V W_i^V)$$

$$= \underbrace{\text{softmax}\left(\frac{QW_i^Q (E_i K W_i^K)^T}{\sqrt{d_k}}\right)}_{\bar{P}: n \times k} \cdot \underbrace{F_i V W_i^V}_{k \times d}$$

这里的 $E_i$ 和 $F_i$ 是两个独立的投影矩阵。$E_i$ 投影 Key，$F_i$ 投影 Value。为什么用两个而不是一个？因为 Key 和 Value 在 attention 中的角色不同——Key 决定了 attention 的分配权重，Value 决定了实际加权的内容，用不同的投影可以给它们不同的自由度。

### 5.3 参数共享策略

引入额外的投影矩阵 $E$ 和 $F$ 会增加参数量，虽然不多（每层每个 head 多了 $2nk$ 个参数）。为了减少参数开销，作者实验了多种共享策略：

- **无共享：** 每层每个 head 有自己的 $E_i, F_i$
- **headwise 共享：** 同一层内所有 head 共享同一对 $E, F$
- **key-value 共享：** $E = F$（Key 和 Value 用同一个投影）
- **layerwise 共享：** 所有层共享同一对 $E, F$

实验结果很有意思：

> **layerwise 共享效果最好，甚至略好于不共享。**

乍看之下这反直觉——按理说参数越多表达力越强。但作者的解释是：layerwise 共享起到了正则化的效果，减少了过拟合。而且在高层，attention 的秩本身就更低，用统一的低维投影也够了。

这其实是个好消息：**layerwise 共享意味着几乎不增加额外参数**，整个模型除了 attention 计算方式变了，参数量跟标准 Transformer 基本一样。

## 六、核心公式与数学直觉

### 6.1 标准 self-attention

$$P = \text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right)$$

$QK^T$ 是 $n \times n$ 的矩阵，每个元素 $(QK^T)_{ij}$ 表示第 $i$ 个 query 和第 $j$ 个 key 的相似度。softmax 按行归一化，让每行的 attention 权重和为 1。

计算 $P$ 后再乘以 $V$：$\text{output} = PV$，其中 $P \in \mathbb{R}^{n \times n}$，$V \in \mathbb{R}^{n \times d}$。

这个计算的核心开销是 $QK^T$ 这个 $n \times n$ 的矩阵乘法——**$O(n^2)$ 的来源**。

### 6.2 Linformer 的线性 attention

$$P̄ = \text{softmax}\left(\frac{Q(EK)^T}{\sqrt{d}}\right)$$

$$= \text{softmax}\left(\frac{QK^T E^T}{\sqrt{d}}\right)$$

注意 $Q \in \mathbb{R}^{n \times d}$，$EK \in \mathbb{R}^{k \times d}$，所以 $Q(EK)^T \in \mathbb{R}^{n \times k}$。

这里 $E$ 做的是：把 $n$ 个 key 的 $d$ 维表示压缩成 $k$ 个"代表"的 $d$ 维表示。数学上相当于在序列维度上做了一个线性降维。

然后：

$$\text{output} = P̄ \cdot (FV)$$

$P̄ \in \mathbb{R}^{n \times k}$，$FV \in \mathbb{R}^{k \times d}$，输出是 $n \times d$。

### 6.3 复杂度分析

标准 self-attention 的时间复杂度：

$$T_{\text{standard}} = O(n^2 d + n d^2)$$

其中 $n^2 d$ 来自 $QK^T$ 和 $PV$ 的矩阵乘法，$n d^2$ 来自 $Q, K, V$ 的投影。

Linformer 的时间复杂度：

$$T_{\text{linformer}} = O(n k d + n d^2)$$

其中 $n k d$ 来自 $Q(EK)^T$ 和 $P̄(FV)$ 的矩阵乘法。当 $k \ll n$ 时，$n k d$ 远小于 $n^2 d$。

内存复杂度类似：

$$M_{\text{standard}} = O(n^2 + nd)$$

$$M_{\text{linformer}} = O(nk + nd)$$

### 6.4 为什么 $k$ 不需要随 $n$ 增长？

Theorem 1 说 $\text{rank}(\tilde{P}) = \Theta(\log n)$。所以理论上需要的投影维度 $k$ 只跟 $\log n$ 成正比，跟 $n$ 本身无关。

实验也验证了这一点：当 $n$ 从 512 增加到 4096，固定 $k=256$ 时，Linformer 的 perplexity 几乎不变（Figure 3d）。这说明 $k$ 确实不依赖 $n$——随着序列变长，attention 矩阵的有效秩并没有显著增加。

这也意味着 Linformer 在长序列上的优势会越来越大：$n$ 越大，$k$ 固定，$nk$ 相对于 $n^2$ 的节省就越显著。

```text
       ▼        │                       │         │
     P(n×n)     │                    softmax      │
       │        │                       │         │
       └──┬─────┘                       ▼         │
          │                          P̄ (n×k)     │
          ▼                            │          │
       output (n×d)                    └────┬─────┘
                                             │
                                             ▼
                                          output (n×d)
```

### 图 3：Pretraining 曲线

论文的 Figure 3 包含四个子图：

- **(a)** $n=512$ 时不同 $k$ 值的 validation perplexity 曲线
- **(b)** $n=1024$ 时不同 $k$ 值的 validation perplexity 曲线
- **(c)** 不同参数共享策略的 validation perplexity 对比
- **(d)** 固定 $k=256$ 时不同 $n$ 值的 validation perplexity 曲线

关键发现已经在之前的章节说过了，最值得强调的是图 (d)：当 $k$ 固定为 256 时，$n$ 从 512 增长到 4096，perplexity 基本不变。这是"$k$ 不随 $n$ 缩放"这个核心结论的直接证据。

### 表 1：各架构复杂度对比

| 模型架构 | 每层复杂度 | 最少顺序操作数 |
|---------|----------|------------|
| 循环神经网络 | $O(n)$ | $O(n)$ |
| Transformer | $O(n^2)$ | $O(1)$ |
| Sparse Transformer | $O(n\sqrt{n})$ | $O(1)$ |
| Reformer | $O(n\log n)$ | $O(\log n)$ |
| **Linformer** | **$O(n)$** | **$O(1)$** |

这个表很说明问题：Linformer 是唯一一个同时做到 $O(n)$ 复杂度和 $O(1)$ 顺序操作的架构。循环神经网络虽然也是 $O(n)$，但它的顺序操作是 $O(n)$——必须按时间步逐步计算。Reformer 虽然复杂度低，但顺序操作数是 $O(\log n)$，因为它需要多轮哈希。

## 八、复杂度、效率与可扩展性分析

### 标准 vs Linformer 的显存占用对比

论文在 16GB Tesla V100 上做了推理效率 benchmark（Table 3）。以下是从论文 Table 3 中提取的速度提升数据（layerwise 共享）：

**推理速度提升（相对于标准 Transformer）：**

| 序列长度 $n$ | $k=128$ | $k=256$ |
|------------|---------|---------|
| 512 | 1.5× | 1.3× |
| 1024 | 1.7× | 1.6× |
| 2048 | 2.6× | 2.4× |
| 4096 | 3.4× | 3.2× |
| 8192 | 5.5× | 5.0× |
| 16384 | 8.6× | 7.8× |
| 32768 | 13× | 12× |
| 65536 | 20× | 18× |

**显存节省：**

| 序列长度 $n$ | $k=128$ | $k=256$ |
|------------|---------|---------|
| 512 | 1.7× | 1.5× |
| 1024 | 3.0× | 2.9× |
| 2048 | 6.1× | 5.6× |
| 4096 | 14× | 13× |
| 8192 | 28× | 26× |
| 16384 | 56× | 48× |
| 32768 | 56×[^1] | 48× |
| 65536 | 60× | 52× |

[^1]: Tesla V100 有 16GB HBM2 显存。$n=65536$ 时标准 attention 的 $n \times n$ 矩阵（float32）约 16GB，刚好超过 V100 容量。因此 $n=32768$ 和 $n=16384$ 之间节省倍数持平，到 $n=65536$ 时通过公式外推得到更高节省倍数。

注意几个现象：

1. **$n$ 越大，加速比和节省越明显。** 在 $n=65536$, $k=128$ 时，推理速度提升 20 倍，显存节省 60 倍。这完美体现了 $O(n)$ vs $O(n^2)$ 的巨大差异。

2. **$k$ 越小，加速越大。** $k=128$ 比 $k=256$ 在每个 $n$ 上都快一些，但差距不是数量级的——这是 $n$ vs $nk$ 的差距还没大到压过常数因子的结果。

3. **显存的节省比速度的提升更显著。** 因为 $n \times n$ 的 attention 矩阵本身是显存瓶颈（对 $n=65536$，$n^2 = 42$ 亿个 float，约 16GB），而实际矩阵乘法有 GPU 的优化硬件支持，速度损失没有显存那么剧烈。

### 新的瓶颈在哪？

Linformer 引入的线性投影 $E$ 和 $F$ 本身需要额外的矩阵乘法（$EK$ 和 $FV$），各需 $O(nkd)$ 的计算量。这个额外开销跟 $O(nkd)$ 的 attention 计算在一个量级，所以整体复杂度还是 $O(nkd)$。

但对很短的序列（比如 $n=128$），$k$ 和 $n$ 差不多大，Linformer 的加速效果不显著，甚至可能因为额外投影而更慢。

这也是为什么论文默认 $k=128$ 且主要在 $n \geq 512$ 的场景下评估——Linformer 的优势在长序列上才能真正发挥。

## 九、实验结果

### 9.1 Pretraining Perplexity

**实验设置：** 在 BookCorpus + English Wikipedia（33 亿词）上做 MLM pretraining，64 块 Tesla V100，250k steps。

**投影维度消融（Figure 3a, 3b）：**

- $n=512$ 时，$k=128$ 就几乎跟标准 Transformer 的 perplexity 持平
- $n=1024$ 时，$k=256$ 持平
- $k$ 越大性能越好，但收益递减——$k=256$ 比 $k=128$ 好一点点，$k=512$（等于 $n$）就跟标准 Transformer 一样了

**共享策略消融（Figure 3c）：**

- 无共享 ≈ headwise 共享 ≈ key-value 共享 ≈ layerwise 共享
- 几乎没差别
- 说明投影矩阵的参数冗余很大，共享不影响表达力

**序列长度消融（Figure 3d）：**

- 固定 $k=256$，$n$ 从 512 取到 4096
- 最终 perplexity 几乎不变
- **直接证据：Linformer 的性能由 $k$ 决定，跟 $n$ 无关**

### 9.2 Downstream 任务性能

论文在 GLUE benchmark 的三个任务（SST-2、QNLI、QQP）以及 IMDB 情感分析数据集上做了 fine-tune 评测（Table 2）。

| 模型 | SST-2 | IMDB | QNLI | QQP | Average |
|------|-------|------|------|-----|---------|
| RoBERTa-base（Liu et al., 2019） | 93.1 | 94.1 | 90.9 | 90.9 | 92.25 |
| Linformer, $n=512$, $k=128$ | 92.4 | 94.0 | 90.4 | 90.2 | 91.75 |
| Linformer, $n=512$, $k=128$, shared kv | 93.4 | 93.4 | 90.3 | 90.3 | 91.85 |
| Linformer, $n=512$, $k=128$, shared kv+layer | 93.2 | 93.8 | 90.1 | 90.2 | 91.83 |
| Linformer, $n=512$, $k=256$ | 93.2 | 94.0 | 90.6 | 90.5 | 92.08 |
| Linformer, $n=512$, $k=256$, shared kv | 93.3 | 93.6 | 90.6 | 90.6 | 92.03 |
| Linformer, $n=512$, $k=256$, shared kv+layer | 93.1 | 94.1 | 91.2 | 90.8 | **92.30** |
| BERT-base（Devlin et al., 2019） | 92.7 | 93.5 | 91.8 | 89.6 | 91.90 |
| Distilled BERT, 6-layer（Sanh et al., 2019） | 91.3 | 92.8 | 89.2 | 88.5 | 90.45 |

关键观察：

1. **Linformer 在 $k=256$ 时全面追平甚至微超 RoBERTa-base**（92.30 vs 92.25）
2. **layerwise 共享 + key-value 共享的效果最好**，这跟直觉可能相反——按理说不共享的参数更多、表达力更强，但共享起到了正则化作用
3. **$k=128$ 时 Linformer 略低于 RoBERTa-base（91.75 vs 92.25），但差距不大**
4. BERT-base 和 Distilled BERT 的表现都比 RoBERTa-base 和 Linformer 差一截

结论很清楚：**当 $k$ 够大时（$k=256$ 对于 $n=512$，即压缩比 2:1），Linformer 的性能完全跟标准 Transformer 相当。** 而 $k$ 只需 256 就意味着 attention 计算量减少了 50%，显存节省更多。

### 9.3 这些实验证明了什么

实验的核心结论有三个：

1. **低秩假设成立。** 谱分析 → 理论证明 → 实验验证，三个环节互相支撑
2. **$k$ 不随 $n$ 缩放。** 固定 $k$，增大 $n$，perplexity 不变。这是 $O(n)$ 复杂度的前提条件
3. **参数共享可行。** layerwise 共享不仅不降性能，还略优于不共享，意味着几乎零额外参数代价

论文官方没有提供独立的代码仓库。Linformer 的实现被包含在 [fairseq](https://github.com/facebookresearch/fairseq) 框架中。

由于 fairseq 相对复杂，这里梳理一下核心实现思路，帮助读者理解如何把论文公式翻译成代码。

### 核心实现逻辑

Linformer 的自注意力层实现大致如下：

```python
class LinformerSelfAttention(nn.Module):
    def __init__(self, embed_dim, num_heads, seq_len, k=128):
        super().__init__()
        self.embed_dim = embed_dim
        self.num_heads = num_heads
        self.head_dim = embed_dim // num_heads

        # 标准 Q, K, V 投影
        self.q_proj = nn.Linear(embed_dim, embed_dim)
        self.k_proj = nn.Linear(embed_dim, embed_dim)
        self.v_proj = nn.Linear(embed_dim, embed_dim)

        # Linformer 的低秩投影
        self.e_proj = nn.Linear(seq_len, k, bias=False)
        self.f_proj = nn.Linear(seq_len, k, bias=False)

        self.out_proj = nn.Linear(embed_dim, embed_dim)

    def forward(self, x):
        # x: (batch, seq_len, embed_dim)
        bsz, n, d = x.shape

        q = self.q_proj(x)  # (b, n, d)
        k = self.k_proj(x)  # (b, n, d)
        v = self.v_proj(x)  # (b, n, d)

        # transpose 到 (b, h, n, d/h)
        q = q.view(bsz, n, self.num_heads, self.head_dim).transpose(1, 2)
        k = k.view(bsz, n, self.num_heads, self.head_dim).transpose(1, 2)
        v = v.view(bsz, n, self.num_heads, self.head_dim).transpose(1, 2)

        # 对 K 和 V 做低秩投影（在序列维度上）
        # k: (b, h, n, d/h) → e_proj(k): (b, h, k, d/h)
        k_proj = self.e_proj(k.transpose(-2, -1)).transpose(-2, -1)
        v_proj = self.f_proj(v.transpose(-2, -1)).transpose(-2, -1)

        # 计算线性 attention
        # attention score: (b, h, n, k)
        attn = torch.matmul(q, k_proj.transpose(-2, -1)) / math.sqrt(self.head_dim)
        attn = F.softmax(attn, dim=-1)

        # 加权求和: (b, h, n, d/h)
        out = torch.matmul(attn, v_proj)

        out = out.transpose(1, 2).contiguous().view(bsz, n, d)
        return self.out_proj(out)
```

这里 `e_proj` 和 `f_proj` 就是论文中的 $E$ 和 $F$，是一个 `Linear(seq_len, k, bias=False)`。注意 `nn.Linear` 默认在最后一维做变换，而我们需要在序列维度上投影，所以需要对 `k` 和 `v` 做 transpose。

对于 layerwise 共享的实现，只需要在模型外层定义一个共享的 `e_proj` 和 `f_proj`，然后在所有 Linformer 层中传入同一个投影层即可。

### 参数共享的代码映射

论文中提到的几种共享策略在代码层面的实现差异如下：

- **headwise 共享：** 在 Multi-Head Attention 中，投影在 head 维度展开之后进行，所以每个 head 可以有自己的 `e_proj` 和 `f_proj`。共享就是所有 head 用同一个。
- **key-value 共享：** 用同一个 `nn.Linear` 作为 `e_proj` 和 `f_proj`。
- **layerwise 共享：** 在 Transformer 的 `__init__` 中定义一个 `shared_e` 和 `shared_f`，然后传给每一层的 `LinformerSelfAttention`。

## 十一、Related Work 与技术谱系位置

Linformer 在高效 Transformer 的发展谱系中处于一个非常关键的位置。

### 它继承了谁？

Linformer 的核心洞察——attention 是低秩的——本质上继承并发展了 kernel-based attention 的思路。在 Linformer 之前，已经有工作尝试用核方法（kernel methods）来近似 attention，但通常需要复杂的特征映射。

它也直接对标了 Sparse Transformer 和 Reformer，试图解决它们各自的问题：Sparse Transformer 的 pattern 固定问题，Reformer 的哈希开销问题。

### 它修正了谁？

Linformer 明确修正了"attention 必须全量计算"这个隐含假设。之前的优化工作（稀疏 attention、LSH）都是在"仍然需要近似全量 attention"的框架下做文章，而 Linformer 直接说：**attention 矩阵的信息冗余很大，不需要算 $n^2$ 个元素**。

### 它替代了谁？

在需要处理长序列的场景下，Linformer 可以替代标准 Transformer 的 self-attention 层。特别是对于 $n \geq 512$ 的场景，Linformer 提供了几乎无性能损失的加速方案。

### 它启发了谁？

Linformer 对后续工作产生了广泛影响：

- **Performer（2021）：** 用随机正交矩阵做 attention 的线性化，也用到了低秩近似的思想
- **Nyströmformer（2021）：** 用 Nyström 方法做 attention 的低秩近似
- **Longformer（2020）：** 结合了局部 attention 和全局 attention
- **BigBird（2020）：** 结合了随机 attention、局部 attention 和全局 attention

### 技术谱系树

```text
Transformer (2017) —— 标准 self-attention: O(n²)
    │
    ├── Sparse Transformer (2019) —— 固定稀疏 pattern: O(n√n)
    │       └── 问题: 手工 pattern 不够灵活
    │
    ├── Reformer (2020) —— LSH 哈希: O(n log n)
    │       └── 问题: 多轮哈希开销大
    │
    ├── Linformer (2020) —— 低秩投影: O(n)    ← 本篇
    │       └── 低秩假设 + 线性投影
    │
    ├── Performer (2021) —— 核方法线性化: O(n)
    ├── Nyströmformer (2021) —— Nyström 近似: O(n)
    ├── Longformer (2020) —— 滑动窗口 + 全局: O(n)
    └── BigBird (2020) —— 稀疏 + 全局 + 随机: O(n)
```

Linformer 在这个谱系里属于 **"低秩近似"路线**的开创者。它首次系统性地提出了"self-attention 是低秩的"这一观点，并用理论和实验双重验证了这个假设。

## 十二、时间线位置

```
2017  ─────────── Transformer（Vaswani et al.）
                        │
2019  ─── Sparse Transformer ─── BERT ─── GPT-2
                        │
2020  ─── Reformer ─── Linformer ─── Longformer ─── BigBird
                        │
2021  ─── Performer ─── Nyströmformer ─── Efficient Transformers Survey
```

Linformer 在 2020 年 6 月出现在 arXiv 上。这个时间点很有意思：

1. **BERT 革命之后两年。** BERT（2018）让预训练语言模型成了 NLP 的标配，但大家开始意识到长序列训练的成本太高。
2. **GPT-3 刚刚发布（2020 年 5 月）。** 175B 参数的大模型把训练和推理成本推到了新高度，效率优化变得空前重要。
3. **Sparse Transformer（2019）和 Reformer（2020）已经探过路。** 它们证明了 attention 效率优化的方向是对的，但也暴露了各自的不足。
4. **Longformer（2020 年 4 月）和 BigBird（2020 年 7 月）也是同期工作。** 2020 年是"高效 Transformer"的爆发年，Linformer 是其中最关键的一篇之一。

为什么这个时间点会出现 Linformer？因为当时市场已经验证了 Transformer 在 NLP 上的统治力，但 $O(n^2)$ 的瓶颈越来越明显。之前的工作（稀疏、哈希）提供了思路但不够彻底，而 Linformer 提供了一个更优雅的数学解释和更简洁的工程实现。

## 十三、范式意义

Linformer 的范式意义可以从几个层面来看：

### 它是"attention 矩阵低秩"概念的开创者

在 Linformer 之前，很少有人认真问过：attention 矩阵有没有冗余？能不能压缩？Linformer 不仅问了这个问题，还用谱分析和定理给出了答案。

这个贡献的意义在于：**它重新定义了我们对 attention 计算的理解**。Attention 不再是"必须看所有位置"的操作，而是"可以只看低维子空间中的信息"的操作。这个概念开启了后续一系列低秩近似 attention 的工作（Performer、Nyströmformer 等）。

### 它是"线性 attention"路线的奠基节点

Linformer 属于"高效 attention 范式"中的**低秩近似子路线**。它不是用稀疏性来省计算，也不是用核技巧来线性化，而是直接用线性投影压缩序列维度。这条路线的核心就是：

> **在 Key 和 Value 侧做线性压缩，让 attention 矩阵变成 $n \times k$ 而不是 $n \times n$。**

这个范式的优雅之处在于：它几乎不改变 Transformer 的整体架构，只改了 self-attention 内部的一小部分。这意味着现成的 Transformer 代码库可以很方便地迁移到 Linformer。

### 它没有解决的问题

Linformer 也有一些被后续工作改进的地方：

1. **它仍然需要计算 $QK^T$。** 虽然 Key 被压缩了，但这个乘法仍然是 $O(nkd)$ 的。对极度长的序列，这可能仍然是瓶颈。
2. **投影矩阵 $E$ 和 $F$ 是可学习的。** 这意味着它们需要训练数据来学习——从头训练或 fine-tune 都需要。
总的来说，Linformer 是一个 **foundation node**（基础节点）——它提出了一种新的技术路线，后续工作在这条路线上做了更多扩展和优化。（其他局限性详见第十四节。）

## 十四、局限性与后续方向

### 局限

1. **仅验证了 NLP 任务。** Linformer 的实验全部在 NLP 上（GLUE、IMDB），没有在视觉、语音或多模态上验证。虽然低秩假设可能具有普遍性，但这需要实验支持。

2. **对极短序列无效。** 当 $n \leq 128$ 时，$E$ 和 $F$ 的额外投影开销可能使 Linformer 比标准 Transformer 更慢。论文没有讨论这个情况。

3. **$k$ 的选择需要手动调。** 论文实验了不同的 $k$ 值，但 $k$ 本身是一个超参数，需要根据任务和数据调整。后续工作（如自适应 $k$）可以解决这个问题。


5. **理论证明有局限。** Theorem 1 的证明依赖于 JL 引理，给出了概率保证，但没有直接给出 $k$ 的具体值和 $n$ 的关系。论文中的 $k=128$ 主要是实验经验值。

### 后续方向

这些局限也自然地指出了后续工作的方向：

- **Performer（Choromanski et al., 2021）：** 用随机正交矩阵（FAVOR+）做 attention 的线性化，不需要可学习的投影矩阵，而且天然支持 causal masking。
- **Nyströmformer（Xiong et al., 2021）：** 用 Nyström 方法做 attention 的低秩近似，在序列维度上进行 landmark 采样。
- **FlashAttention（Dao et al., 2022）：** 从硬件层面优化 attention 的 IO 效率，跟 Linformer 是正交的。
- **自适应 $k$：** 根据输入自适应地选择投影维度，或者对不同层不同 head 使用不同的 $k$。

## 十五、小结

Linformer 这篇论文的核心贡献可以总结为一句话：

> **它发现并证明了 self-attention 的 context mapping matrix 是低秩的，并利用这个发现设计了一个线性复杂度的 attention 机制。**

具体来说：

- **它改变了一个认知：** attention 矩阵 $P$ 的信息高度冗余，不需要算 $n^2$ 个元素。这个认知改变了后续所有高效 attention 工作的方向。
- **它把一条技术路线往前推了一步：** "低秩近似"路线从"理论上可能"推进到了"可以作为通用模块使用"的阶段。
- **在技术树中的节点类型：** 它是一个 **foundation node**（基础节点）。它不是对已有范式的修修补补，而是提出了一种新的范式思考方式。

直白说，这篇论文的价值不只是提出了一个线性 attention 机制，而是让整个社区意识到：

> **Self-attention 的平方复杂度不是一个结构性的必然，而是一个可以被破除的假设。**
