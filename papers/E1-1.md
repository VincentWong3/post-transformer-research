# Linformer: Self-Attention with Linear Complexity（2020）

**论文：** [arXiv](https://arxiv.org/abs/2006.04768) · Wang et al. · 2020

---

## 一、先搞清楚坑在哪

Transformer 的核心——自注意力机制——有一个众所周知的"死穴"：它的计算复杂度与序列长度 $n$ 的平方成正比，即 $O(n^2)$。这意味着，当你尝试处理长序列时，计算量和内存占用会爆炸式增长。

让我们具体算一笔账。假设序列长度 $n=512$（BERT 的标准长度），注意力矩阵是 $512 \times 512$，大约 26 万个元素。但当 $n=4096$ 时，注意力矩阵膨胀到 1600 万个元素——增长了 64 倍。如果 $n=16384$（一些文档级任务的需求），注意力矩阵包含 2.68 亿个元素，单是存储这个矩阵就需要超过 1GB 的内存。

这就产生了一个核心矛盾：**自注意力是 Transformer 最强大的机制，但它也是扩展序列长度的根本瓶颈。**

研究人员一直在问：能不能保留自注意力的表达能力，同时把复杂度从 $O(n^2)$ 降到 $O(n)$？

## 二、现有方法的真正问题

在 Linformer 之前，已经有一些尝试来解决这个问题：

### 稀疏注意力（Sparse Attention）
[Child et al., 2019] 提出只计算部分位置之间的注意力，比如固定间隔或局部窗口。问题在于：**稀疏模式需要人工设计，而且可能丢失长距离依赖信息**。比如两个相距很远的 token 恰好不在稀疏模式中，它们的交互就被忽略了。

### 低秩近似（Low-rank Approximation）
有一些工作尝试用低秩矩阵分解来近似注意力矩阵，但要么近似质量不够好，要么需要复杂的训练技巧。

### 核方法（Kernel Methods）
[Choromanski et al., 2020] 的 Performer 用随机特征映射来近似 softmax 注意力。这种方法在理论上很优雅，但实际实现中需要仔细调整超参数，而且近似误差在某些任务上可能不可忽略。

**关键问题：** 这些方法要么牺牲了注意力机制的灵活性（稀疏模式），要么引入了额外的近似误差（核方法），要么实现复杂。

## 三、Linformer 的核心思路

**一句话总结：注意力矩阵是低秩的，所以我们可以用一个线性投影把序列长度从 $n$ 压缩到 $k$，从而实现 $O(nk)$ 的复杂度，其中 $k \ll n$。**

这个洞察来自一个观察：在预训练好的 Transformer 模型中，注意力矩阵的奇异值衰减很快，说明它的有效秩远小于序列长度。换句话说，注意力矩阵的大部分信息集中在少数几个主成分上。

Linformer 的做法很直接：在计算注意力之前，先把 $n \times d$ 的 Key 和 Value 矩阵投影到 $k \times d$（其中 $k$ 是投影维度），然后在这个压缩后的空间里计算注意力。这样，注意力矩阵的大小就从 $n \times n$ 变成了 $n \times k$，复杂度从 $O(n^2)$ 降到了 $O(nk)$。

## 四、网络架构详解

### 4.1 整体架构

Linformer 基本上就是 Transformer 编码器，唯一的改动在多头注意力模块内部。让我们看看改动前后的对比：

```
标准 Transformer 多头注意力：

输入: n x d_model
    │
    ├─→ Q: n x d_k
    ├─→ K: n x d_k
    └─→ V: n x d_v
    │
    Attention(Q,K,V) = softmax(QK^T/√d_k) V
    │
    ┌────── n x n 注意力矩阵 ──────┐
    │  复杂度: O(n²)               │
    └──────────────────────────────┘
    │
    输出: n x d_v


Linformer 多头注意力：

输入: n x d_model
    │
    ├─→ Q: n x d_k
    ├─→ K: n x d_k ──→ 线性投影 ──→ K_proj: k x d_k
    └─→ V: n x d_v ──→ 线性投影 ──→ V_proj: k x d_v
    │
    Attention(Q,K_proj,V_proj) = softmax(QK_proj^T/√d_k) V_proj
    │
    ┌────── n x k 注意力矩阵 ──────┐
    │  复杂度: O(nk)               │
    └──────────────────────────────┘
    │
    输出: n x d_v
```

### 4.2 前向过程详解

让我们用一个具体的例子来走一遍前向过程。假设：
- 序列长度 $n = 1024$
- 投影维度 $k = 128$（论文中常用 $k=128$ 或 $k=256$）
- 模型维度 $d_{model} = 768$
- 注意力头数 $h = 12$
- 每个头的维度 $d_k = d_v = 64$

**步骤 1：输入嵌入**
输入形状：$[batch, 1024, 768]$

**步骤 2：QKV 投影**
三个独立的线性层，每个将 $768$ 维投影到 $12 \times 64 = 768$ 维。
- Q 形状：$[batch, 1024, 768]$
- K 形状：$[batch, 1024, 768]$
- V 形状：$[batch, 1024, 768]$

**步骤 3：分头**
将最后一维拆成 12 个头：
- Q 形状：$[batch, 12, 1024, 64]$
- K 形状：$[batch, 12, 1024, 64]$
- V 形状：$[batch, 12, 1024, 64]$

**步骤 4：线性投影（核心创新）**
对 K 和 V 进行投影。每个头有自己的投影矩阵 $E_i$ 和 $F_i$，形状为 $[1024, 128]$。
- K 经过投影：$[batch, 12, 128, 64]$（注意，这里对序列维度进行压缩）
- V 经过投影：$[batch, 12, 128, 64]$

**步骤 5：计算注意力**
- 计算 Q 和 K_proj 的点积：$[batch, 12, 1024, 128]$
- 除以 $\sqrt{d_k} = \sqrt{64} = 8$
- softmax：形状不变，$[batch, 12, 1024, 128]$
- 乘以 V_proj：$[batch, 12, 1024, 64]$

**步骤 6：合并头**
$[batch, 1024, 768]$

**步骤 7：输出投影**
线性层，$[batch, 1024, 768]$

**关键变化：** 标准注意力中，步骤 5 的中间矩阵是 $[batch, 12, 1024, 1024]$，而 Linformer 中是 $[batch, 12, 1024, 128]$。内存节省了 $1024/128 = 8$ 倍。

### 4.3 为什么这个设计有效

这个设计的巧妙之处在于：**投影发生在计算注意力之前，而不是之后**。如果先计算完整的 $n \times n$ 注意力矩阵再压缩，那就已经付出了 $O(n^2)$ 的代价。通过在计算之前压缩 K 和 V，Linformer 直接避免了生成大矩阵。

## 五、核心创新点

### 创新点 1：注意力矩阵的低秩性假设

#### (a) 痛点与动机

标准自注意力的 $O(n^2)$ 复杂度来自于计算 $QK^T$ 矩阵。这个矩阵的大小随序列长度平方增长，但它的信息真的是满秩的吗？

论文作者做了一个实验：取一个预训练好的 RoBERTa 模型，计算不同层、不同头的注意力矩阵，然后做奇异值分解（SVD）。结果如图 1 所示：

```
注意力矩阵奇异值衰减（示意图）：

奇异值大小
│
1.0 ┤█
    │█
0.8 ┤ █
    │ █
0.6 ┤  █
    │  █
0.4 ┤   █
    │   █
0.2 ┤    █
    │    ██
0.0 ┤     ████
    │         ████████
    └──────────────────▶ 奇异值序号
    0   50  100 150 200 250
```

**图 1：注意力矩阵的奇异值衰减曲线。可以看到，前几十个奇异值就包含了绝大部分信息。**

这个观察意味着：注意力矩阵的有效秩远小于 $n$。也就是说，我们不需要完整的 $n \times n$ 矩阵来表达注意力关系。

#### (b) 方案细节

Linformer 用两个可学习的投影矩阵 $E$ 和 $F$ 来压缩 Key 和 Value：

$$K_{proj} = EK, \quad V_{proj} = FV$$

其中 $E, F \in \mathbb{R}^{k \times n}$，$k \ll n$。

注意力计算变为：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK_{proj}^T}{\sqrt{d_k}}\right)V_{proj}$$

#### (c) 为什么有效

低秩性是一个经验观察，但它的存在有其内在原因：在自然语言中，每个 token 只与少数其他 token 有强相关性。大多数 token 对之间的注意力权重接近于零。注意力矩阵本质上是一个"稀疏"矩阵，只是以稠密形式存储。

通过投影到 $k$ 维，我们相当于自动学习了一个"软性"的稀疏模式——不需要人工设计，模型自己决定哪些信息值得保留。

论文的实验（Table 1）显示，当 $k=128$ 时，Linformer 在 GLUE 基准上的性能与标准 Transformer 几乎相同，但复杂度从 $O(n^2)$ 降到了 $O(n)$。

#### (d) 与 Related Work 的关系

这个思路与 [Cho et al., 2019] 的"矩阵分解注意力"有相似之处，但 Cho 的方法是对已经计算好的注意力矩阵做低秩近似，而 Linformer 是在计算之前就压缩，避免了 $O(n^2)$ 的中间矩阵。

与 Performer [Choromanski et al., 2020] 的核方法相比，Linformer 更直接：核方法用随机特征映射来近似 softmax，而 Linformer 直接用线性投影压缩 K 和 V。

#### (e) 如果去掉会怎样

论文的消融实验（Table 3）显示，如果不使用投影（即 $k=n$），Linformer 退化为标准 Transformer，性能完全一致。这说明投影本身没有引入信息损失（至少在 $k$ 足够大时）。

### 创新点 2：共享投影参数

#### (a) 痛点与动机

如果每个头都有自己的投影矩阵 $E_i$ 和 $F_i$，参数数量会随着头数线性增长。对于 12 头注意力，这意味著额外的 $12 \times 2 \times n \times k$ 个参数。当 $n$ 很大时，这仍然是一个不小的开销。

#### (b) 方案细节

论文探索了三种共享策略：
1. **头间共享（Head-wise sharing）**：所有头共享同一组 $E$ 和 $F$
2. **Key-Value 共享（KV-wise sharing）**：Key 和 Value 共享同一个投影矩阵 $E = F$
3. **层间共享（Layer-wise sharing）**：所有层共享同一组投影矩阵

#### (c) 为什么有效

实验（Table 2）表明，头间共享几乎没有性能损失，Key-Value 共享有轻微下降，层间共享的下降更明显一些。这说明投影矩阵主要学习的是"如何压缩序列"这个通用知识，而不是针对特定头或特定层的知识。

#### (d) 与 Related Work 的关系

这种共享策略在 Transformer 中并不罕见。例如，ALBERT [Lan et al., 2020] 在层间共享参数，大大减少了参数量。Linformer 将类似的思想应用于投影矩阵。

#### (e) 如果去掉会怎样

论文的消融实验（Table 2）显示，从不共享到共享，性能下降不到 0.5 个点（在 GLUE 上），但参数量减少了 12 倍（12 个头）。这是一个非常划算的 trade-off。

### 创新点 3：参数化的投影矩阵

#### (a) 痛点与动机

为什么不用固定的投影矩阵（比如随机投影或平均池化）？因为固定投影可能丢失重要的位置信息。比如，平均池化会把所有位置的信息混合在一起，使得注意力无法区分不同位置。

#### (b) 方案细节

投影矩阵 $E$ 和 $F$ 是可学习的参数，通过反向传播训练。这允许模型学习到"哪些位置的信息应该被保留"。

#### (c) 为什么有效

可学习的投影矩阵比固定投影更灵活。论文的实验（Table 3）比较了可学习投影与固定投影（如平均池化、随机投影），可学习投影的效果明显更好。

## 六、公式详解

### 公式 1：标准自注意力

$$ \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V $$

**符号定义：**
- $Q \in \mathbb{R}^{n \times d_k}$：查询矩阵
- $K \in \mathbb{R}^{n \times d_k}$：键矩阵
- $V \in \mathbb{R}^{n \times d_v}$：值矩阵
- $n$：序列长度
- $d_k$：查询/键的维度
- $d_v$：值的维度

**复杂度分析：**
$QK^T$ 的计算需要 $O(n^2 d_k)$ 次乘法。当 $n$ 很大时，这就是瓶颈。

### 公式 2：Linformer 自注意力

$$ \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{Q(EK)^T}{\sqrt{d_k}}\right) FV $$

**符号定义：**
- $E \in \mathbb{R}^{k \times n}$：Key 的投影矩阵
- $F \in \mathbb{R}^{k \times n}$：Value 的投影矩阵
- $k$：投影维度，$k \ll n$

**复杂度分析：**
- $EK$：$O(n d_k k)$
- $Q(EK)^T$：$O(n d_k k)$
- 总复杂度：$O(n d_k k)$，当 $k \ll n$ 时，近似为 $O(n)$

**直觉理解：**
投影矩阵 $E$ 将 $n$ 个位置压缩成 $k$ 个"虚拟位置"。每个虚拟位置是原始位置的加权和，权重由 $E$ 学习得到。注意力机制只在这 $k$ 个虚拟位置上操作。

### 公式 3：复杂度对比

标准注意力：
$$ O(n^2 d_k + n^2 d_v) $$

Linformer：
$$ O(nk d_k + nk d_v + n d_k k + n d_v k) $$

**具体数值计算：**
设 $n=1024, k=128, d_k=64, d_v=64$：

标准注意力：
- $QK^T$：$1024 \times 1024 \times 64 = 67,108,864$ 次乘法
- 乘以 $V$：$1024 \times 1024 \times 64 = 67,108,864$ 次乘法
- 总计：约 $1.34 \times 10^8$ 次乘法

Linformer：
- $EK$：$1024 \times 64 \times 128 = 8,388,608$ 次乘法
- $Q(EK)^T$：$1024 \times 64 \times 128 = 8,388,608$ 次乘法
- $FV$：$1024 \times 64 \times 128 = 8,388,608$ 次乘法
- 乘以 $V_{proj}$：$1024 \times 128 \times 64 = 8,388,608$ 次乘法
- 总计：约 $3.35 \times 10^7$ 次乘法

加速比：$1.34 \times 10^8 / 3.35 \times 10^7 \approx 4$ 倍

当 $n=4096, k=128$ 时：
- 标准注意力：约 $2.15 \times 10^9$ 次乘法
- Linformer：约 $1.34 \times 10^8$ 次乘法
- 加速比：约 16 倍

**边界情况：**
- 当 $k = n$ 时，Linformer 退化为标准注意力
- 当 $k$ 趋近于 1 时，所有位置的信息被压缩成一个点，注意力退化为全局平均

## 八、代码对照

论文的官方代码在 GitHub 上（https://github.com/tatp22/linformer-pytorch）。关键实现细节：

### 8.1 投影矩阵的实现

```python
class LinformerAttention(nn.Module):
    def __init__(self, dim, seq_len, k=256, heads=8):
        super().__init__()
        self.heads = heads
        self.seq_len = seq_len
        self.k = k
        
        # 投影矩阵
        self.E = nn.Linear(seq_len, k, bias=False)
        self.F = nn.Linear(seq_len, k, bias=False)
        
        # QKV 投影
        self.to_q = nn.Linear(dim, dim * heads)
        self.to_k = nn.Linear(dim, dim * heads)
        self.to_v = nn.Linear(dim, dim * heads)
        
    def forward(self, x):
        # x: (batch, seq_len, dim)
        q = self.to_q(x)
        k = self.to_k(x)
        v = self.to_v(x)
        
        # 分头
        q = q.view(batch, seq_len, heads, -1).transpose(1, 2)
        k = k.view(batch, seq_len, heads, -1).transpose(1, 2)
        v = v.view(batch, seq_len, heads, -1).transpose(1, 2)
        
        # 投影（在序列维度上）
        k_proj = self.E(k.transpose(-2, -1)).transpose(-2, -1)
        v_proj = self.F(v.transpose(-2, -1)).transpose(-2, -1)
        
        # 计算注意力
        attn = torch.matmul(q, k_proj.transpose(-2, -1)) / math.sqrt(d_k)
        attn = F.softmax(attn, dim=-1)
        out = torch.matmul(attn, v_proj)
        
        return out
```

**代码中的关键点：**
1. 投影是在序列维度上进行的，而不是特征维度
2. 投影矩阵是 `nn.Linear`，意味着它包含可学习的权重和偏置（虽然论文中说没有偏置）
3. 投影矩阵在所有头上共享（如果设置 `shared=True`）

### 8.2 训练细节

从代码中可以看到的一些训练技巧：
- 使用 LayerNorm 和 Dropout（与标准 Transformer 相同）
- 学习率调度使用 Noam 调度器
- 投影矩阵的初始化使用 Kaiming 初始化

## 九、位置

### 9.1 前驱工作

1. **Transformer [Vaswani et al., 2017]**：Linformer 的基础框架
2. **Low-rank Matrix Factorization [Cho et al., 2019]**：首次提出注意力矩阵的低秩性质，但方法是在计算后压缩
3. **Sparse Transformer [Child et al., 2019]**：用稀疏模式解决长序列问题，但需要人工设计模式
4. **Reformer [Kitaev et al., 2020]**：用 LSH（局部敏感哈希）实现近似注意力，但实现复杂

### 9.2 同期竞品

1. **Performer [Choromanski et al., 2020]**：用随机特征映射近似 softmax，复杂度 $O(n)$
2. **Longformer [Beltagy et al., 2020]**：结合局部注意力和全局注意力，复杂度 $O(n)$
3. **Big Bird [Zaheer et al., 2020]**：随机注意力 + 局部注意力 + 全局注意力，复杂度 $O(n)$

### 9.3 后续影响

1. **Linformer 的思路被广泛采用**：许多后续工作直接使用或改进了线性投影方法
2. **Transformer-XL [Dai et al., 2019]** 结合了 Linformer 的投影思想来处理更长的上下文
3. **LongNet [Ding et al., 2023]**：扩展了 Linformer 的投影思想，实现了 $O(n)$ 复杂度的超长序列处理
4. **FlashAttention [Dao et al., 2022]**：虽然方法不同（IO 感知），但目标相同——降低注意力的计算成本
5. **Mamba [Gu & Dao, 2023]**：完全抛弃注意力，用状态空间模型实现线性复杂度，但 Linformer 的低秩投影思想为其提供了理论基础

## 十、局限

### 局限 1：投影维度 $k$ 的选择

## 十一、小结

Linformer 的核心贡献在于提出了一个简单而有效的洞察：**自注意力矩阵是低秩的，因此可以用线性投影来压缩 Key 和 Value，从而将复杂度从 $O(n^2)$ 降到 $O(n)$。**

这个方法的优雅之处在于它的简洁性：不需要复杂的稀疏模式设计，不需要随机特征映射，只需要在标准 Transformer 中插入两个可学习的投影矩阵。实验证明，在 $k=128$ 或 $k=256$ 时，Linformer 在 GLUE 基准上的性能与标准 Transformer 几乎相同，但可以处理 4 倍甚至 16 倍长的序列。

Linformer 的意义不仅在于它本身，更在于它推动了"线性注意力"这一研究方向的发展。它与其他同期工作（Performer、Longformer、Big Bird）一起，证明了 Transformer 可以扩展到长序列任务，为后续的许多工作奠定了基础。
