# Momentum Contrast for Unsupervised Visual Representation Learning（MoCo，2020）

**论文：** [arXiv:1911.05722](https://arxiv.org/abs/1911.05722) · Kaiming He, Haoqi Fan, Yuxin Wu, Saining Xie, Ross Girshick · Facebook AI Research (FAIR) · CVPR 2020

**技术分支：** B4 — 统一视觉 encoder 基座模型 / B4-1 — contrastive representation learning

**代码：** [facebookresearch/moco](https://github.com/facebookresearch/moco)

---

## 一、先搞清楚"问题在哪里"

2019 年底，自然语言处理已经用无监督预训练（GPT, BERT）取得了革命性进展，但计算机视觉的主流做法仍然是有监督预训练——先在 ImageNet 上做有监督分类预训练，然后把 backbone 迁移到下游任务上微调。为什么 CV 的无监督预训练迟迟追不上 NLP？

论文给了一个很直觉的解释：**NLP 的输入信号空间是离散的。** 词和子词单元本身就是天然的 token，可以构建词典，无监督学习（如语言建模）本质上是在做字典查询。但视觉的原始信号是连续的、高维的像素空间，不像文字那样天然有结构。

那有没有办法把视觉的无监督学习也统一到"字典查询"的框架下？这就是 MoCo 的出发点。

在这个框架下，视觉无监督学习被抽象为：

1. 从数据中采样"key"（如图像或 patch），用 encoder 网络编码为表示向量
2. 构造一个 query，约束它与其匹配的 key（positive）相似、与不匹配的 key（negative）不相似
3. 用对比损失（contrastive loss）来优化 encoder

MoCo 之前已有的对比学习方法在这个范式下有一个共同的问题：**它们构建的字典要么不够大，要么不够一致（consistent）。** 一个理想的字典应该同时满足两个条件：(i) **大**——更好地采样连续高维视觉空间；(ii) **一致**——字典中的 keys 应该由相同或相似的 encoder 编码，这样它们与 query 的比较才有一致的参考系。

## 二、旧路线的真正问题

论文梳理了两种已有的对比学习机制，这两种机制分别遇到了"大"和"一致"中的一个瓶颈。

### 2.1 端到端（End-to-End）机制

这是最自然的方式：query encoder 和 key encoder 共享参数（或不同参数），通过反向传播同时更新。每次取当前 mini-batch 中的样本作为字典 keys。

**优点：** keys 由同一个 encoder 编码，语义一致。

**瓶颈：** 字典大小被 mini-batch size 绑定。GPU 内存限制了 batch size 的上限（在 8 张 32GB V100 上最大能跑 1024）。更大的 mini-batch 本身也是个开放问题——需要线性学习率缩放规则，而且大 batch 优化本身更难。

### 2.2 记忆银行（Memory Bank）机制

[Wu et al., 2018]（InstDisc）提出的方案：维护一个存储了**数据集中所有样本**表示的记忆银行。每个 mini-batch 从记忆银行中随机采样作为字典 keys。

**优点：** 字典大小不受 mini-batch size 限制，可以很大。

**瓶颈：** 记忆银行中每个样本的表示是在它上一次被看到时更新的，所以采样出来的 keys 来自过去整个 epoch 中多个不同步的 encoder，不一致性高。

论文复现了这种机制，在使用 InfoNCE 损失和 K=65536 时得到 **58.0%** 的准确率（比 MoCo 低 2.6%）。

### 2.3 两者的核心矛盾

两种机制的本质矛盾可以概括为：

```
端到端:  字典 × 大 (受batch size限制)  ✓ 一致 (同一encoder)
记忆银行: ✓ 大 (不受限制)              × 不一致 (不同步的encoder)
```

MoCo 要同时解决这两个问题。

## 三、这篇论文的核心思路

MoCo 的核心思路就一句话：**用队列构建大字典，用动量更新保证一致性。**

具体来说两个设计：

1. **队列做字典。** 把字典维护成一个先进先出的队列。当前 mini-batch 的编码结果入队，最早的 mini-batch 出队。字典大小和 mini-batch size 完全解耦，可以独立设置为超参数（如 65536）。队列始终只维护数据的一个采样子集，计算开销可控。而且移除最旧的 mini-batch 还有额外好处——它们经编码后已经过时，一致性最差，移除是合理的。

2. **动量更新 key encoder。** 队列涉及到历史多个 mini-batch 的 keys，无法通过反向传播来更新 key encoder（梯度需要传播到队列中所有样本）。最直接的方案是复制 query encoder 的参数给 key encoder，但实验显示这个方案效果很差。原因是 encoder 变化太快，导致 keys 的一致性被破坏。MoCo 的解决方案是动量更新：

   $$\theta_{\text{k}} \leftarrow m\theta_{\text{k}} + (1 - m)\theta_{\text{q}}$$

   其中 $m \in [0,1)$ 是动量系数。只有 query encoder $\theta_{\text{q}}$ 通过反向传播更新，key encoder $\theta_{\text{k}}$ 通过动量缓慢追赶 query encoder。动量越大，key encoder 变化越慢，生成的 keys 就越一致。

   默认 $m=0.999$ 。实验中 $m=0.999$ （59.0% acc）远好于 $m=0.9$ （55.2% acc）， $m=0$ 时训练发散。

### 3.1 伪代码

```python
# f_q, f_k: encoder networks for query and key
# queue: dictionary as a queue of K keys (C×K)
# m: momentum
# t: temperature

f_k.params = f_q.params  # initialize

for x in loader:  # load a minibatch x with N samples
    x_q = aug(x)  # a randomly augmented version
    x_k = aug(x)  # another randomly augmented version

    q = f_q.forward(x_q)  # queries: N×C
    k = f_k.forward(x_k)  # keys: N×C
    k = k.detach()         # no gradient to keys

    # positive logits: N×1
    l_pos = bmm(q.view(N,1,C), k.view(N,C,1))
    # negative logits: N×K
    l_neg = mm(q.view(N,C), queue.view(C,K))
    # logits: N×(1+K)
    logits = cat([l_pos, l_neg], dim=1)

    # contrastive loss, Eqn.(1)
    labels = zeros(N)
    loss = CrossEntropyLoss(logits/t, labels)

    # SGD update: query network
    loss.backward()
    update(f_q.params)

    # momentum update: key network
    f_k.params = m*f_k.params + (1-m)*f_q.params

    # update dictionary
    enqueue(queue, k)
    dequeue(queue)
```

### 3.2 损失函数

使用 InfoNCE 损失：

$$\mathcal{L}_q = -\log \frac{\exp(q \cdot k_+ / \tau)}{\sum_{i=0}^K \exp(q \cdot k_i / \tau)}$$

其中 $\tau=0.07$ 是温度超参数。这个损失本质上是一个 (K+1)-way softmax 分类器，试图把 query $q$ 分类为正样本 $k_+$ 。

损失包含 1 个正样本和 K 个负样本（来自队列）。在 MoCo 的默认设置中 K=65536，即 65536 个负样本，形成一个超大分类问题。

## 四、Pretest Task 与实现细节

### 4.1 实例判别（Instance Discrimination）

MoCo 采用了一个简单的 pretext task（论文不做 pretext 层面的创新）：同一张图像的两个不同增强视图构成正样本对，不同图像的视图互为负样本。具体做法和 [Wu et al., 2018]（InstDisc）一致，同时参考了 [Ye et al., 2019] 和 [Bachman et al., 2019] 使用的双视图增强策略。

### 4.2 编码器设计

- Backbone：标准 ResNet（无特殊架构设计，如 patchified inputs 或定制感受野）
- 输出层：全局平均池化后接一个固定维度（128-D）的全连接层
- L2 归一化：输出向量做 L2 归一化
- 数据增强：224×224 crop + 随机颜色抖动 + 随机水平翻转 + 随机灰度化

这和之前需要特殊架构的方法不同——CPC 需要 patchify 输入，AMDIM 需要定制感受野，CMC 需要组合两个网络。MoCo 用标准 ResNet，方便迁移到各种下游任务。

### 4.3 Shuffling BN（关键工程发现）

论文发现一个有趣的问题：当 encoder 使用 Batch Normalization 时，模型会"作弊"。原因是 BN 在 mini-batch 内引入了跨样本的信息泄露（batch statistics 来自于所有样本），模型可以利用这个信息来降低 pretext task 的难度，但学不到好的语义表示。

解决方案：**Shuffling BN**。在多 GPU 训练时，对于 key encoder $f_k$ ，先在 GPU 间打乱样本顺序，编码完再恢复；query encoder $f_q$ 的样本顺序不变。这样 query 和其 positive key 的 batch statistics 来自两个不同的数据子集，堵住了作弊路径。

这个技巧在 MoCo 和端到端的对比实验中都使用。

### 4.4 训练配置

**ImageNet-1M（IN-1M）：** ~128 万张图片。SGD 优化器，weight decay 0.0001，SGD momentum 0.9，mini-batch 256（8 GPU），初始学习率 0.03。训练 200 个 epoch，在 120 和 160 epoch 学习率×0.1。ResNet-50 约需 53 小时。

**Instagram-1B（IG-1B）：** ~9.4 亿张 Instagram 公开图片，来自 ~1500 个 ImageNet 相关 hashtags，分布长尾不平衡。mini-batch 1024（64 GPU），学习率 0.12，指数衰减（每 62500 步 ×0.9），训练 125 万次迭代（~1.4 epoch），ResNet-50 约需 6 天。

## 五、实验结果

### 5.1 ImageNet 线性分类协议

这是无监督表征学习的标准评测方式：无监督预训练后冻结特征，训练一个线性 softmax 分类器（100 epoch），报告 ImageNet 验证集 top-1 准确率。

**三种对比机制的比较（Figure 3）：**

在相同的 pretext task 和 InfoNCE 损失下，三种机制表现如下：

- **端到端：** 受限于 mini-batch size（最大 1024），增大 batch 需要线性学习率缩放
- **记忆银行：** 可以达到 58.0%（论文的改进复现版本，使用 InfoNCE + K=65536）
- **MoCo：** 在相同条件下优于前两者

三种机制都验证了：**更大的字典（K 更大）带来更好的结果**，这个趋势对所有机制都成立。

**动量（m）的消融（K=4096）：**

- m=0：训练发散，失败
- m=0.9：55.2%
- m=0.99：57.8%
- m=0.999：59.0%
- m=0.9999：58.9%

m 在 0.99~0.9999 之间表现良好，m=0.999 为最佳。缓慢演进的 key encoder 是 MoCo 有效性的关键。

**MoCo 在不同规模模型上的结果：**

| 模型 | 参数量 | Top-1 准确率 |
|------|--------|-------------|
| MoCo R50 | 24M | 60.6% |
| MoCo RX50 (ResNeXt-50-32×8d) | 46M | 63.9% |
| MoCo R50w 2× | 94M | 65.4% |
| MoCo R50w 4× | 375M | 68.6% |

MoCo 使用标准 ResNet-50（24M 参数）就达到了有竞争力的 60.6%，在同类模型大小的方法中表现最佳。更大模型带来进一步改善（68.6%）。论文强调，这些结果来自标准的 ResNet 架构、不需要特殊设计，便于迁移到下游任务。

论文还注释了，后续的 "MoCo v2"（对数据增强和投影头做了小改动）将 R50 的准确率从 60.6% 提升到了 71.1%，说明了 MoCo 框架的通用性和稳健性。

### 5.2 PASCAL VOC 目标检测

**设置：** Faster R-CNN，backbone 为 R50-dilated-C5 或 R50-C4，微调所有层，验证方法是否能在"小数据"（VOC trainval07+12 ~16.5k 张）上展示迁移优势。

**R50-C4 backbone（Table 2b）：**

| 预训练 | AP50 | AP | AP75 |
|--------|------|----|------|
| Random init | 60.2 | 33.8 | 33.1 |
| Supervised IN-1M | 81.3 | 53.5 | 58.8 |
| **MoCo IN-1M** | **81.5 (+0.2)** | **55.9 (+2.4)** | **62.6 (+3.8)** |
| **MoCo IG-1B** | **82.2 (+0.9)** | **57.2 (+3.7)** | **63.7 (+4.9)** |

C4 backbone 上 MoCo 在有监督预训练之上展现了明显增益（最高 +4.9 AP75）。论文指出一个有趣的现象：迁移精度依赖于检测器的结构——C4 backbone 上无监督预训练的优势比 dilated-C5 更大。这种预训练 vs 检测器结构的关系在过去未被充分关注。

**R50-dilated-C5 backbone（Table 2a）：**

MoCo IN-1M 与有监督基线基本持平（81.1 vs 81.4 AP50），MoCo IG-1B 超越基线（81.6 vs 81.4 AP50）。

**对比机制在 VOC 检测上的差异（Table 3）：** 相同设置下，MoCo 在 R50-C4 backbone 上以 81.5 AP50 全面领先端到端（80.4）和记忆银行（80.6），证明了 MoCo 机制本身的贡献。而且，端到端和记忆银行机制如何扩展到更大规模的数据（如 IG-1B）仍是开放问题。

**更少训练数据场景（VOC trainval2007 ~5k 张，Table 4）：**

对于 AP50 指标，之前所有方法都无法赶上各自的有监督预训练基线。而 MoCo 在 IN-1M、IN-14M、YFCC-100M、IG-1B 上预训练后**全部超越**有监督基线。在更严格的 AP75 指标上，MoCo IG-1B 获得了高达 **+9.0** 的增益。

### 5.4 更多下游任务（Table 6）

论文在 7 个检测/分割任务上验证了 MoCo：

**COCO 关键点检测：** 有监督预训练对比随机初始化没有明显优势（65.8 vs 65.9 APkp），而 MoCo 在所有指标上超越了有监督基线（**+1.0 APkp** IN-1M, **+1.1 APkp** IG-1B）。

**COCO DensePose 估计：** MoCo 大幅超越有监督预训练，在 AP75dp 上最高提升 **+3.7**。这是一个对定位精度要求极高的任务。

**LVIS v0.5 实例分割：** ~1000 个长尾类别。MoCo IG-1B 在最佳设置下（24.9 APmk）超越了有监督基线（24.4 APmk）。

**Cityscapes 实例分割：** MoCo IG-1B 与有监督基线持平（32.9 APmk），AP50mk 更高（60.3 vs 59.6）。

**Cityscapes 语义分割：** MoCo 超越有监督基线最多 **+0.9** 点。

**VOC 语义分割：** 论文承认这是 MoCo 唯一表现不如有监督的任务（低 0.8~1.9 点），这是一个负面案例。

论文总结：MoCo 在 **7 个检测/分割任务**中可以超越其 ImageNet 有监督预训练基线。（7 个任务包括：VOC/COCO 目标检测、COCO/LVIS 实例分割、COCO 关键点检测、COCO DensePose、Cityscapes 语义分割。）此外在 Cityscapes 实例分割上持平，在 VOC 语义分割上落后。

**值得注意的趋势：** 在所有任务中，MoCo IG-1B（10 亿级 Instagram 数据）都一致地优于 MoCo IN-1M（120 万级 ImageNet 数据）。这说明 MoCo 能在大规模、低质量管控的数据上表现良好，是一种朝向真实世界无监督学习的有效方案。

### 5.5 附加实验

**COCO 更长微调（6× schedule, Table A.1）：** 6× schedule（~72 epoch）下，从随机初始化训练大幅追赶（41.4 APbb），有监督预训练仍有改进（41.9 APbb），而 MoCo IG-1B 进一步提升（42.8 APbb），差距从 2× 的 +0.5 扩大到 **+0.9**。这说明 MoCo 预训练的特征在更充分的微调中保持了优势。

**iNaturalist 细粒度分类：** MoCo IN-1M（65.6%）与有监督基线（66.1%）非常接近，MoCo IG-1B（65.8%）也基本相当，比随机初始化（61.8%）高约 4 个点。

**ImageNet 端到端微调：** 在 IG-1B 上预训练后在 ImageNet 上微调，MoCo（77.3%）比从随机初始化训练（76.5%）高 0.8%。

## 六、范式意义

MoCo 在对比学习的发展中占据了一个特殊的位置：**它不是第一个做视觉对比学习的，但它提出了一个优雅且通用的机制设计，把"字典的大小"和"字典的一致性"这两个矛盾目标统一了起来。**

### 6.1 视角转变：从损失函数设计到字典构建

在 MoCo 之前，无监督对比学习的改进主要聚焦在 pretext task 设计和损失函数形式。MoCo 放弃了这个方向上的创新，而是**从一个更高的抽象层次——字典构建——来审视问题。** 它把对比学习统一描述为"动态字典查找"，然后发现所有已有方法在这个框架下都受限于"大"和"一致"之间的权衡。

这个视角的转变是 MoCo 最根本的贡献。它把问题从"设计更好的损失函数"重构为"设计更好的字典构建机制"。

### 6.2 两个设计的精妙之处

**队列解耦。** 用 FIFO 队列做字典，可能在论文写成时看起来很简单，但其内涵很丰富：队列天然地实现了字典大小的任意扩缩，解耦了负样本数量与 batch size；同时队列的先进先出天然保证了字典中始终是最新的样本，最旧的样本（最可能不一致）被自动移除。

**动量 encoder。** 动量更新不是新概念（在 supervised learning 中也有用），但 MoCo 是第一个把这个机制系统引入对比学习的 key encoder。其巧妙之处在于：动量 encoder 不需要反向传播，解决了队列带来的梯度传播不可行的问题；同时它提供了一个连续体：从 m=0（copy weights，fail）到 m=0.999（稳定演进，work），展示了一致性对对比学习的重要性。

### 6.3 简单性与通用性

MoCo 使用了标准的 ResNet 架构，不需要 CPC 的 patchify、AMDIM 的定制感受野、CMC 的双网络组合。这意味着模型迁移到下游检测/分割任务时没有架构障碍——在 VOC/COCO 上微调可以直接替换预训练 backbone，zero adaptation cost。

论文在 7 个不同任务上验证了 MoCo 的迁移能力，其中多个任务超越了有监督基线。这在当时（2019 年末）是一个强有力的结果——它表明无监督预训练在视觉任务上**已经可以在实践中有竞争力地替代有监督预训练。**

### 6.4 技术树位置

在技术树中，MoCo 是 B4（统一视觉 encoder 基座模型）下 B4-1（contrastive representation learning）范式的奠基性工作之一。它与 SimCLR（B4-1）分别从不同角度推进了对比学习：

- **SimCLR** 证明了简单正负例对比 + 大量数据增强就足够好（不需要 memory bank 或动量 encoder）
- **MoCo** 系统性地解决了对比学习中字典的规模与一致性问题
