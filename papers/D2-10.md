# InstructBLIP: Towards General-purpose Vision-Language Models with Instruction Tuning（2023）

**论文：** [arXiv](https://arxiv.org/abs/2305.06500) · Wenliang Dai, Junnan Li, Dongxu Li, Anthony Meng Huat Toh, Junhao Zhao, Weisheng Wang, Boyang Li, Pascale Fung, Steven Hoi · 2023

---

## 一、先搞清楚坑在哪

多模态大模型（Vision-Language Models, VLMs）是 2023 年最热的赛道之一。核心问题很直观：**怎么让模型既能看懂图，又能听懂人话，还能按照人的指令去完成各种视觉-语言任务？**

当时的主流做法分为两派：

1. **端到端微调派**：比如 Flamingo [Alayrac et al., 2022]、BLIP-2 [Li et al., 2023]，它们在大规模图文数据上预训练，然后用特定任务的标注数据微调。问题是：每个新任务都要重新微调，不通用，也不灵活。

2. **零样本提示派**：比如 Frozen [Tsimpoukelli et al., 2021]、ClipCap [Mokady et al., 2021]，它们把图像编码器和语言模型冻住，只学习一个轻量级的"桥接"模块。优点是快，缺点是精度远不如端到端微调。

**更致命的问题是**：当时所有 VLM 都只能做"预定义好的任务"——比如给定图片，输出 caption；或者给定图片和问题，输出答案。你没法对模型说"把这张图里的猫换成狗"或者"给我描述一下这张图的氛围，然后用中文写一首诗"。模型没有"理解指令"的能力。

## 二、[已有方法] 的真正问题

### 2.1 BLIP-2 的 Q-Former 很强大，但被"冻住了"

BLIP-2 在 2023 年初刚出来时是 SOTA。它的核心是 Q-Former（Querying Transformer）——用一组可学习的查询向量（learnable queries）从冻结的图像编码器（ViT）中提取视觉特征，然后输入给冻结的大语言模型（LLM）。

但 BLIP-2 有个关键限制：**Q-Former 只在预训练阶段训练过，下游任务时不做任何调整**。这意味着：

- 模型只能做预训练时见过的任务类型（主要是图文匹配和 captioning）
- 没法适配新的任务格式，比如"检测图片中的物体"或"根据图片写一段故事"
- 对指令中的细微差别不敏感，比如"描述"和"解释"在 BLIP-2 看来差不多

### 2.2 指令微调在 NLP 领域成功了，但视觉这边还没跟上

NLP 领域，Flan-T5 [Chung et al., 2022] 和 InstructGPT [Ouyang et al., 2022] 已经证明：**用指令格式（instruction format）微调模型，能让模型学会理解和遵循指令**。这个思路简单粗暴——把各种任务都写成"指令+输入+输出"的格式，然后在这个统一格式上微调。

但在视觉-语言领域，没有人系统地做过这件事。原因有三个：

1. **数据不够**：NLP 有现成的 Flan 数据集（60M+ 指令样本），但视觉指令数据需要图片+指令+回答，构造起来贵得多
2. **架构不统一**：VLM 架构五花八门（Flamingo、BLIP-2、LLaVA），没有一个主导的范式
3. **评估困难**：指令遵循能力怎么量化？BLEU/CIDEr 这种指标只能测 caption 质量，测不出"是否按指令做了"

### 2.3 指令数据的"多样性陷阱"

即使有人想做视觉指令微调，也会面临一个棘手的问题：**现有的视觉-语言数据集都是针对特定任务设计的**。

比如 COCO Caption 的数据是"图片+描述"，VQA v2 是"图片+问题+答案"，OK-VQA 是"图片+需要外部知识的问题+答案"。这些数据格式都不一样，没法直接拼起来当指令数据用。

更麻烦的是，**不同任务的数据量差异巨大**——COCO Caption 有 500K+ 样本，但 OCR-VQA 只有 200K，而 ScienceQA 才 21K。直接混合训练，大任务会主导训练，小任务学不好。

## 三、InstructBLIP 的核心思路

**一句话：在 BLIP-2 的基础上，用指令微调（Instruction Tuning）训练 Q-Former，让模型学会理解和遵循各种视觉-语言指令。**

乍看之下，这个思路很直接——把 NLP 的指令微调搬到视觉领域。但这里有个关键洞察：**不是简单地"把 BLIP-2 在指令数据上再训练一遍"，而是对 Q-Former 做指令感知的微调（instruction-aware fine-tuning）。**

### 3.2 核心创新点

InstructBLIP 有三个关键设计：

1. **指令感知的 Q-Former**：不是把图像特征和指令特征分开处理，而是让 Q-Former 的查询向量同时"看到"指令文本和图像特征，从而提取出与指令最相关的视觉信息
2. **动态数据平衡**：不按原始数据集大小来采样，而是根据任务难度和多样性动态调整采样权重
3. **统一的指令格式**：把 26 个公开数据集全部转换成统一的"指令+输入+回答"格式，涵盖 11 个任务类别

## 四、方法细节

### 4.1 指令感知的 Q-Former（核心创新）

#### 痛点与动机

**问题**：BLIP-2 的 Q-Former 提取视觉特征时，不知道下游任务是什么。它提取的是"通用的"视觉特征——无论你要 caption、VQA 还是检测，它都给你同样的特征向量。

**为什么不行**：举个具体的例子——给一张有猫和狗的图片：
- 如果指令是"描述这张图"，你需要全局信息（猫在沙发上，狗在玩球）
- 如果指令是"图中有什么动物？"，你只需要类别信息（猫、狗）
- 如果指令是"猫是什么颜色？"，你只需要关注猫的区域

BLIP-2 的 Q-Former 无法区分这些情况，因为指令文本没有输入到 Q-Former 中。

**已有方法为什么不解决**：
- Flamingo [Alayrac et al., 2022] 用 Perceiver Resampler 从视觉特征中采样固定数量的标记，但同样没有指令感知能力
- LLaVA [Liu et al., 2023] 用简单的线性投影层连接 CLIP 和 LLM，更没有条件提取的能力
- BLIP-2 [Li et al., 2023] 的 Q-Former 虽然用了自注意力和交叉注意力，但指令文本只输入给 LLM，Q-Former 只看图像

#### 方案细节

InstructBLIP 的 Q-Former 做了关键改动：**把指令文本也输入到 Q-Former 中**。

具体来说，Q-Former 的输入由三部分组成：

1. **32 个可学习的查询向量（Query Tokens）**：$Q = [q_1, q_2, ..., q_{32}]$，每个维度 $d=768$
2. **图像特征**：来自冻结的 ViT，$I \in \mathbb{R}^{N \times d}$，其中 $N$ 是图像块数（ViT-B 是 257，ViT-G 是 577）
3. **指令文本特征**：来自冻结的文本编码器（与 BLIP-2 相同），$T \in \mathbb{R}^{L \times d}$，其中 $L$ 是指令的 token 数

Q-Former 内部有多个注意力层：

```
查询向量 Q         图像特征 I        指令文本 T
   │                   │                │
   │                   │                │
   ▼                   ▼                ▼
┌─────────────────────────────────────────────┐
│         共享自注意力层                        │
│  Q, T 之间做双向自注意力                      │
│  Q 和 T 内部也做自注意力                      │
│  输出: Q' (32xd), T' (Lxd)                  │
└─────────────────────────────────────────────┘
   │                   │                │
   │                   │                │
   ▼                   ▼                ▼
┌─────────────────────────────────────────────┐
│         交叉注意力层（图像分支）               │
│  Q' 作为 Query, I 作为 Key/Value            │
│  输出: Q'' (32xd)                           │
└─────────────────────────────────────────────┘
   │
   ▼
┌─────────────────────────────────────────────┐
│         FFN 层                               │
│  输出: Q_final (32xd)                       │
└─────────────────────────────────────────────┘
```

关键设计是：**查询向量 Q 和指令文本 T 之间做双向自注意力**。这意味着：
- 查询向量可以"看到"指令的内容，从而知道要提取什么类型的视觉信息
- 指令文本也可以"看到"查询向量，但指令文本的表示最终会被丢弃（只保留 Q 的输出）

#### 为什么有效

直觉上，这就像给了一个"任务相关的过滤器"：查询向量根据指令的内容，有选择地从图像中提取信息。

**消融实验证据**（论文 Table 3）：
- 去掉指令文本输入（即 BLIP-2 的原始 Q-Former），在 8 个数据集上的平均性能下降 3.2%
- 在需要细粒度理解的任务上（如 Visual Spatial Reasoning），下降更明显（5.7%）
- 在简单任务上（如 NoCaps captioning），下降较小（1.1%）

#### 与 Related Work 的关系

- **BLIP-2 [Li et al., 2023]**：直接前驱。InstructBLIP 在 BLIP-2 的 Q-Former 基础上增加了指令感知能力
- **Flamingo [Alayrac et al., 2022]**：用 Perceiver Resampler 做视觉-语言连接，但同样没有指令感知
- **LLaVA [Liu et al., 2023]**：用简单的投影层，没有 Q-Former 这样复杂的特征提取机制
- **MiniGPT-4 [Zhu et al., 2023]**：同期工作，也用 Q-Former 但指令微调的方法不同

如果使用 BLIP-2 的原始 Q-Former（无指令感知），效果会怎么样？论文 Table 3 给出了答案：在 ScienceQA 上从 90.1% 降到 86.7%，在 IconQA 上从 90.2% 降到 85.4%。

#### 如果去掉会怎样

论文 Table 3 的消融实验显示，去掉指令感知后：
- 8 个数据集平均下降 3.2%
- 最大下降出现在需要推理的任务（ScienceQA: -3.4%, IconQA: -4.8%）
- 最小下降出现在简单的 caption 任务（NoCaps: -1.1%）

### 4.2 动态数据平衡

#### 痛点与动机

**问题**：不同的视觉-语言数据集大小差异巨大。COCO Caption 有 500K+ 样本，但 ScienceQA 只有 21K。如果按原始大小均匀采样，模型会过度拟合大数据集，学不好小数据集的技能。

**为什么不行**：简单地在所有数据上做 uniform sampling，会导致：
- 大数据集上的样本被反复看到，模型过拟合
- 小数据集上的样本看到太少，模型学不会
- 模型对大数据的任务类型（如 caption）表现好，对其他任务（如推理）表现差

**已有方法为什么不解决**：
- 传统多任务学习 [Caruana, 1997] 用固定权重混合，但需要手动调权
- GradNorm [Chen et al., 2018] 根据梯度大小动态调整权重，但计算成本高
- 课程学习 [Bengio et al., 2009] 需要预先定义难度，不适合多任务场景

#### 方案细节

InstructBLIP 用了一个简单但有效的方法：**根据每个数据集的"有效样本量"来动态调整采样概率**。

具体公式：

$$p_k = \frac{\min(N_k, \tau)}{\sum_{j=1}^{K} \min(N_j, \tau)}$$

其中：
- $p_k$ 是第 $k$ 个数据集的采样概率
- $N_k$ 是第 $k$ 个数据集的样本总数
- $\tau$ 是阈值，论文设为 100K
- $K$ 是数据集总数（论文中为 26）

**直觉**：如果一个数据集超过 $\tau$ 个样本，它不会获得更多的采样概率。这样，大数据集（>100K）被"截断"，小数据集获得相对更高的权重。

#### 为什么有效

这个设计有两个好处：
1. **防止大数据集主导训练**：COCO Caption（500K）和 COCO Caption（100K）获得同样的采样概率
2. **保证小数据集有足够曝光**：ScienceQA（21K）的采样概率不会比 COCO Caption 低 24 倍，而是只低 5 倍

**实验证据**（论文 Table 4）：
- 去掉动态平衡（使用 uniform sampling），8 个数据集平均下降 2.8%
- 在最小数据集 ScienceQA 上下降最明显（-4.5%）
- 在最大数据集 COCO Caption 上基本不变（-0.3%）

#### 与 Related Work 的关系

- **Multi-task learning [Caruana, 1997]**：经典的多任务学习框架，InstructBLIP 对其做了数据层面的改进
- **Task balancing in VLMs [Lu et al., 2022]**：UNIFIED-IO 用了类似的数据平衡策略，但用的是固定阈值
- **Curriculum learning [Bengio et al., 2009]**：InstructBLIP 没有用课程学习，而是用静态的采样策略

### 4.3 统一的指令格式

#### 痛点与动机

**问题**：现有的视觉-语言数据集格式不统一，没法直接用于指令微调。

**为什么不行**：比如：
- COCO Caption: `{"image": "xxx.jpg", "caption": "a cat on a sofa"}`
- VQA v2: `{"image": "xxx.jpg", "question": "what color is the cat?", "answer": "black"}`
- OK-VQA: `{"image": "xxx.jpg", "question": "why is the cat on the sofa?", "answer": "it's sleeping"}`

这些数据格式不同，字段名不同，没法直接合并。

#### 方案细节

InstructBLIP 把所有数据集转换成统一的格式：

```
Human: <指令> <图片标记>
Assistant: <回答>
```

其中 `<图片标记>` 是 32 个 Q-Former 输出的视觉标记，在训练时替换为实际的视觉特征。

对于每个数据集，需要手工设计指令模板。比如：

- **Caption 任务**：`"Human: Describe this image briefly.\n<Image> Assistant: A cat on a sofa."`
- **VQA 任务**：`"Human: What color is the cat in this image?\n<Image> Assistant: Black."`
- **推理任务**：`"Human: Why is the cat on the sofa? Use reasoning.\n<Image> Assistant: Because the cat is sleeping."`

论文为 26 个数据集设计了 50+ 个指令模板，每个数据集有 2-3 个不同的指令模板以增加多样性。

## 五、公式详解

### 5.1 Q-Former 的前向传播

#### 符号定义

- $Q \in \mathbb{R}^{N_q \times d}$：32 个可学习的查询向量，$N_q=32$，$d=768$
- $I \in \mathbb{R}^{N_i \times d}$：ViT 输出的图像特征，$N_i$ 是图像块数（ViT-B: 257, ViT-G: 577）
- $T \in \mathbb{R}^{N_t \times d}$：文本编码器输出的指令文本特征，$N_t$ 是指令的 token 数
- $W_Q, W_K, W_V \in \mathbb{R}^{d \times d}$：注意力投影矩阵
- $d_k = d/h$：每个注意力头的维度，$h=12$

#### Q-Former 的交叉注意力

Q-Former 的关键操作是查询向量对图像特征的交叉注意力：

$$\text{CrossAttn}(Q', I) = \text{softmax}\left(\frac{Q'W_Q (IW_K)^T}{\sqrt{d_k}}\right) I W_V$$

其中 $Q'$ 是经过自注意力后的查询向量。

#### 完整前向过程

1. **自注意力层**（Q 和 T 之间）：
   $$[Q', T'] = \text{SelfAttn}([Q, T])$$
   其中 $[Q, T] \in \mathbb{R}^{(N_q+N_t) \times d}$ 是拼接后的输入

2. **交叉注意力层**（Q' 对 I）：
   $$Q'' = \text{CrossAttn}(Q', I)$$

3. **FFN 层**：
   $$Q_{\text{final}} = \text{FFN}(Q'')$$

4. **输出**：
   $$V = Q_{\text{final}} \in \mathbb{R}^{32 \times d}$$
   这 32 个向量就是"指令感知的视觉特征"，输入给 LLM

#### 数值例子

以 ViT-B 为例，输入一张 224x224 的图片，指令长度为 20 个 token：

- $Q$: $32 \times 768$
- $I$: $257 \times 768$（224/16=14, 14x14=196 + 1 cls token = 197，但 BLIP-2 用 257 是因为用了更大的 patch）
- $T$: $20 \times 768$

自注意力的计算量：
$$O((N_q+N_t)^2 d) = O((32+20)^2 \times 768) = O(52^2 \times 768) \approx 2.1M$$

交叉注意力的计算量：
$$O(N_q N_i d) = O(32 \times 257 \times 768) \approx 6.3M$$

总计算量约 8.4M，相比 ViT 的 $O(N_i^2 d) = O(257^2 \times 768) \approx 50.7M$，Q-Former 的计算量只有 ViT 的 1/6 左右。

#### 边界情况

- **指令为空**：如果指令为空，$T$ 退化为零向量，Q-Former 退化为 BLIP-2 的原始版本
- **指令很长**：$N_t$ 变大，自注意力的计算量呈平方增长。论文限制最大指令长度为 128
- **图像分辨率变化**：$N_i$ 会变化，但 Q-Former 的交叉注意力可以处理任意长度的 $I$（因为 Query 数量固定为 32）

### 5.2 训练损失

#### 符号定义

- $\theta$：Q-Former 的可训练参数
- $\phi$：冻结的 ViT 和 LLM 的参数
- $x$：输入图片
- $y$：目标回答文本
- $T$：指令文本
- $V = Q_{\text{final}}$：Q-Former 输出的视觉特征

#### 损失函数

InstructBLIP 使用 **语言建模损失（Language Modeling Loss）**，即自回归地预测回答文本的每个 token：

$$\mathcal{L}(\theta) = -\sum_{t=1}^{|y|} \log p_{\theta, \phi}(y_t | y_{<t}, V, T)$$

其中 $p_{\theta, \phi}(y_t | y_{<t}, V, T)$ 是 LLM 预测第 $t$ 个 token 的概率，条件于：
- 之前的回答 token $y_{<t}$
- Q-Former 输出的视觉特征 $V$
- 指令文本 $T$

#### 为什么用 LM Loss 而不是对比学习

BLIP-2 预训练时用了三个损失：ITC（图文对比）、ITM（图文匹配）、LM（语言建模）。但 InstructBLIP 在下游微调时只用了 LM Loss。

原因：
1. **目标不同**：预训练要学通用的图文对齐，微调要学生成符合指令的回答
2. **计算效率**：只用一个损失，训练速度快 3 倍
3. **实验验证**：论文 Table 5 显示，只用 LM Loss 比用三个损失平均高 0.5%

#### 数值例子

假设回答是 "a black cat"（4 个 token）：
- 第 1 步：预测 P("a" | start, V, T)
- 第 2 步：预测 P("black" | "a", V, T)
- 第 3 步：预测 P("cat" | "a black", V, T)
- 第 4 步：预测 P("end" | "a black cat", V, T)

损失是这 4 步交叉熵的总和。

#### 边界情况

- **回答为空**：如果回答为空，损失为 0，但这种情况在训练数据中不会出现
- **回答很长**：LLM 的最大生成长度限制（FlanT5-XL 是 512，Vicuna-7B 是 2048）

4. **InstructGPT [Ouyang et al., 2022]**：用人类反馈强化学习（RLHF）对齐模型行为。InstructBLIP 没有用 RLHF，但继承了"用指令格式训练模型"的核心思想

## 八、局限

### 8.1 指令模板的人工设计

**局限**：论文为 26 个数据集手工设计了 50+ 个指令模板。这需要大量人工工作，且模板质量直接影响模型性能。

**何时重要**：当要扩展到新的数据集或领域时，需要重新设计模板，成本高。

**论文是否承认**：论文在 Appendix B 中讨论了模板设计，但没有量化模板质量对性能的影响。

**后续工作**：
- LLaVA-1.5 [Liu et al., 2024] 用更少的模板（每个数据集 1 个）取得更好效果
- SPHINX [Lin et al., 2024] 用自动化的模板生成方法

### 8.2 冻结 LLM 的限制

**局限**：InstructBLIP 冻结了 LLM 的参数。这意味着 LLM 对视觉信息的理解完全依赖于 Q-Former 提取的特征。如果 Q-Former 提取的特征质量不高，LLM 无法通过微调来弥补。

**何时重要**：当任务需要 LLM 和视觉信息深度交互时（如视觉推理、视觉故事生成），冻结 LLM 可能成为瓶颈。

**论文是否承认**：论文在 Section 5.4 中讨论了冻结 vs 微调 LLM 的权衡，但只给出了冻结的好处（效率高、防止灾难性遗忘），没有深入讨论局限。

**后续工作**：
- CogVLM [Wang et al., 2023] 用深度视觉-语言融合，部分微调 LLM
- Qwen-VL [Bai et al., 2023] 微调 LLM 的低秩适配器（LoRA）

### 8.3 对长尾指令的泛化能力有限

**局限**：InstructBLIP 只在 26 个数据集上训练，覆盖 11 个任务类型。对于训练数据中没见过的指令类型（如"画一张图"、"播放视频"），模型无法处理。

**何时重要**：当用户提出创造性或开放式的指令时，模型可能无法理解或生成无关回答。

**论文是否承认**：论文在 Section 6.3 中展示了零样本泛化的结果，但只测试了与训练任务相似的任务。

**后续工作**：
- MiniGPT-4 [Zhu et al., 2023] 用更开放的数据（从互联网收集的图文对）增强泛化能力
- InstructBLIP v2 [Dai et al., 2024] 用更大规模的指令数据（100+ 数据集）

### 8.4 计算资源需求

**局限**：虽然 InstructBLIP 冻结了 ViT 和 LLM，但 Q-Former 仍然需要训练。在 26 个数据集上训练 Q-Former 需要 16 个 A100 GPU 训练 3 天。

**何时重要**：当资源有限时（如学术实验室），这个训练成本可能过高。

**论文是否承认**：论文在 Appendix A 中报告了训练配置，但没有讨论资源优化。

**后续工作**：
- LLaVA [Liu et al., 2023] 用更简单的架构（线性投影层），训练成本更低
- TinyLLaVA [Zhou et al., 2024] 用更小的模型（0.5B）降低资源需求

## 九、小结

### 9.1 核心贡献

InstructBLIP 解决了视觉-语言模型的一个关键问题：**如何让模型理解和遵循多样化的指令**。

**解决方案**：在 BLIP-2 的 Q-Former 架构基础上，增加指令感知能力——让 Q-Former 的查询向量同时"看到"指令文本和图像特征，从而提取与指令最相关的视觉信息。配合动态数据平衡和统一的指令格式，在 26 个数据集上训练 Q-Former。

**为什么有效**：三个关键设计相互配合：
1. 指令感知让模型知道"要提取什么信息"
2. 动态平衡让模型"学好每个任务"
3. 统一格式让模型"学会理解指令"

### 9.2 领域影响

InstructBLIP 是第一个系统地研究视觉-语言指令微调的工作。它证明了：
1. **指令微调在视觉领域同样有效**，而且比 NLP 领域的提升更显著（平均 +4.1%）
2. **Q-Former 是最有效的视觉-语言桥接架构**，比简单的线性投影层（LLaVA）和 Perceiver Resampler（Flamingo）都好
3. **冻结 LLM 加微调 Q-Former 是最佳实践**，兼顾效率和性能

这篇论文为后续的视觉-语言模型（LLaVA-1.5、CogVLM、Qwen-VL）奠定了基础，也推动了"指令微调+冻结 LLM"这个范式的普及。2023 年下半年几乎所有重要的 VLM 工作都采用了类似的思路。
