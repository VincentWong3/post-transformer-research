# InstructBLIP: Towards General-purpose Vision-Language Models with Instruction Tuning（2023）

**论文：** [arXiv:2305.06500](https://arxiv.org/abs/2305.06500) · Wenliang Dai, Junnan Li, Dongxu Li, Anthony Meng Huat Tiong, Junqi Zhao, Weisheng Wang, Boyang Li, Pascale Fung, Steven Hoi · Salesforce Research, HKUST, NTU · 2023

**代码：** [salesforce/LAVIS/projects/instructblip](https://github.com/salesforce/LAVIS/tree/main/projects/instructblip)

## 一、先搞清楚"问题在哪里"

2023 年初，多模态大模型领域处在一个有趣但尴尬的节点。

一方面，BLIP-2 已经证明了一个极具吸引力的范式：**保持图像编码器和 LLM 冻结，只训练一个轻量的 Q-Former 来做桥接**。这个方案训练成本极低（单卡 A100 就能跑），却能让冻结的 LLM "看懂"图像——在 VQA 和图像描述任务上达到当时的 SOTA。更重要的是，BLIP-2 使用 FlanT5 这种 instruction-tuned LLM，天然具备一定的指令跟随能力。

但另一方面，BLIP-2 的能力是**窄的**。给它一张图让它描述，它做得很好；让它做 VQA，也不错。但如果问它"这张图里有什么不合理的地方？"或者"这张图告诉你什么故事？"——它就不知道该怎么处理了。原因很简单：BLIP-2 的训练数据是图像描述（caption）数据，它学到的只是"看图说话"这一种技能。

与此同时，NLP 领域已经经历了一场革命。2022 年的 FlanT5 和 ChatGPT 证明了 instruction tuning 的力量：把大量任务转换成"指令+输入→输出"的统一格式来训练，模型就能学会"执行指令"这种通用能力。你不需要为翻译训练一个模型、为摘要训练另一个模型——同一个模型，换条指令就能干不同的事。

所以自然的问题来了：**这个"指令跟随"的超能力，能不能迁移到多模态场景？** 能不能让一个模型既会看图说话，又会做 VQA，又会做视觉推理，又会做视觉对话……而且面对没见过的任务也能泛化？

这听起来像 BLIP-2 的自然延续——它已经有 instruction-tuned LLM 了，给它多加点任务数据不就行了？但事情没那么简单。论文指出了两个核心挑战。

**第一个挑战：多模态指令数据的缺失。** NLP 领域有现成的 instruction tuning 数据集（如 FLAN、Super-NaturalInstructions），但多模态领域没有。你要自己造：收集各种 VQA、caption、分类、对话数据集，然后为每个数据集设计指令模板。26 个数据集，每个 10-15 个模板——这还不算数据格式转换、去重、防污染等工作。

**第二个挑战：视觉特征提取是 instruction-agnostic 的。** BLIP-2 的 Q-Former 不管你的指令是什么，提取的都是同一套视觉特征。这对于"描述这张图"来说够用，但对"这张图里有什么不寻常的东西？"来说就不够了——后者需要视觉系统关注异常区域而不是整个场景。也就是说，**视觉特征提取本身需要知道"你要关注什么"**——它必须是 instruction-aware 的。

InstructBLIP 就是在这个节点上出现的。它的核心想法很简洁：**在 BLIP-2 的 Q-Former 里加入指令信息，让视觉特征提取根据指令动态调整；然后用 26 个数据集做大规模 instruction tuning。**

## 二、旧路线的真正问题

在 InstructBLIP 之前，多模态统一的尝试可以归为两条路线，各有各的局限。

### 2.1 路线一：多任务学习（Multitask Learning）

这条路线的思路是把各种任务统一成相同的 input-output 格式，然后一起训练。例如，把 caption、VQA、分类都表示成"图像+文本→文本"的形式，共享同一个 encoder-decoder 架构。

**问题是什么？**

论文在实验部分（Figure 4）给出了明确的答案。直接用 BLIP-2 FlanT5XL backbone 做多任务训练（不加指令），在 held-in 数据集上和 instruction tuning 效果差不多，但在 held-out 数据集上——**多任务训练和原始 BLIP-2 表现相当，而 instruction tuning 大幅领先**。

这个结果很能说明问题：多任务训练的模型学会的是"在特定数据分布上输出特定格式"，而不是"理解指令并执行对应的任务"。当测试数据的分布变了（另一个数据集的 VQA，甚至完全不同的任务类型），多任务训练就没有泛化能力了。

### 2.2 路线二：视觉适配器 + 少量描述数据

这条路线以 BLIP-2 和 Flamingo 为代表。训练视觉适配器（Q-Former 或类似模块）来把视觉特征喂给冻结的 LLM，但训练数据主要来自图像描述（caption）数据。

**问题又是什么？**

caption 数据只能教会模型"描述图片"这一种技能。当任务变成 VQA、推理、对话、分类时，模型照猫画虎，能力严重受限。这就像一个只会写说明文的人突然被要求写议论文——语法没问题，但格式和内容对不上。

### 2.3 所以 InstructBLIP 要解决什么？

核心问题其实是一个：**如何在保持 BLIP-2 训练效率优势（冻结大部分参数）的前提下，让多模态模型学会"指令跟随"这种通用能力？**

这需要三个层面的突破：
1. **数据层面** — 造一个大规模、多样化的多模态 instruction tuning 数据集
2. **模型层面** — 让视觉特征提取模块能够感知指令
3. **训练层面** — 协调 26 个差异巨大的数据集之间的学习进度

## 三、这篇论文的核心思路

**核心思路就一句话：在 BLIP-2 的 Q-Former 里加入指令信息做 instruction-aware 视觉特征提取，然后在 26 个多模态数据集上做 instruction tuning。**

我用一个简单的类比来帮助理解：

BLIP-2 的 Q-Former 像一个"通用翻译器"——无论你要什么信息，它都从图片里提取同样的一组特征。InstructBLIP 把这个翻译器升级成了"智能助理"——它能根据你的问题（指令）来决定从图片里提取什么信息。你问"这是什么颜色？"它就关注颜色区域；你问"这里有几个行人？"它就关注物体计数。

这个"智能"来自一个很小的改动：**把指令文本也喂给 Q-Former**，让 Q-Former 的 self-attention 层在 query embedding 和 instruction token 之间做交互。这样，query embedding 在提取视觉特征时就知道"该注意什么"。

## 四、整体结构

### 4.1 架构总览

InstructBLIP 的架构和 BLIP-2 几乎一样，只有一个关键区别：

```text
[Image Encoder (frozen)]            [Text Instruction]
       │                                   │
       ▼                                   ▼
[Q-Former (finetuned)]          ←──  instruction token 也输入进来
       │
       ▼
[Linear Projection]
       │
       ▼
[Frozen LLM (FlanT5 / Vicuna)]
       │
       ▼
[Generated Response]
```

所有组件（图像编码器 ViT-g/14、Q-Former、LLM）都和 BLIP-2 一致。唯一的架构创新在 Q-Former 内部——它的 self-attention 层不仅对 32 个 learnable query embedding 做 attention，还同时 attention 到 instruction 的文本 token。

具体来说，Q-Former 包含两个子模块：
- **Self-attention**：query 之间 + query 与 instruction token 之间
- **Cross-attention**：query 与图像编码器输出之间

instruction token 通过 self-attention 影响了 query embedding 的内容，从而间接影响了 cross-attention 阶段从图像里提取的信息。

### 4.2 四个模型变体

论文训练了四种 InstructBLIP 模型，都用相同的图像编码器 ViT-g/14，但使用不同的冻住 LLM：

| 变体 | 冻住 LLM | 参数量 | LLM 类型 |
|------|---------|--------|---------|
| InstructBLIP FlanT5XL | FlanT5-XL | 3B | encoder-decoder |
| InstructBLIP FlanT5XXL | FlanT5-XXL | 11B | encoder-decoder |
| InstructBLIP Vicuna-7B | Vicuna-7B | 7B | decoder-only |
| InstructBLIP Vicuna-13B | Vicuna-13B | 13B | decoder-only |

FlanT5 是基于 T5 的 instruction-tuned encoder-decoder 模型，Vicuna 是基于 LLaMA 的 instruction-tuned decoder-only 模型。两种 LLM 类型各有利弊，后面实验部分会详细分析。

因为原始的 BLIP-2 没有 Vicuna 的预训练 checkpoint，论文用和 BLIP-2 相同的两阶段预训练流程，对 Vicuna 版本也做了 Q-Former 预训练。

### 4.3 训练配置

所有模型的训练配置都一样：
- 最多 60K 步，每 3K 步验证一次
- batch size：192（3B）、128（7B）、64（11/13B）
- 优化器：AdamW，β1=0.9，β2=0.999，weight decay 0.05
- 前 1,000 步 linear warmup，学习率从 10⁻⁸ 升到 10⁻⁵，之后 cosine decay 到 0
- 16 张 Nvidia A100（40G）GPU，1.5 天内完成训练

### 4.4 推理方法

推理时有两种策略。对大多数任务（caption、open-ended VQA），直接让模型生成文本。对分类和多选题，用 vocabulary ranking 方法——限制模型的词汇表为候选列表，算每个候选的 log-likelihood，选最高的。这个方法应用于 ScienceQA、IconQA、A-OKVQA（多选题）、HatefulMemes、Visual Dialog、MSVD 和 MSRVTT。

对视频 QA 任务，每段视频均匀采样 4 帧，每帧独立过图像编码器和 Q-Former，然后把提取的视觉特征拼接起来喂给 LLM。

## 五、数据：26 个数据集，11 个任务类别

这是论文最"工程"但也很关键的部分。

### 5.1 数据集收集

论文收集了 26 个公开多模态数据集，覆盖 11 个任务类别：

| 任务类别 | 数据集 |
|---------|--------|
| 图像描述（Image Captioning） | COCO Caption, NoCaps, Flickr30K |
| 阅读理解型图像描述 | TextCaps |
| 视觉推理（Visual Reasoning） | ScienceQA, VSR |
| 图像问答（Image QA） | VQAv2, GQA |
| 知识型图像问答（Knowledge-grounded QA） | OKVQA, A-OKVQA, OKVQA |
| 阅读理解型图像问答 | OCR-VQA, TextVQA |
| 图像问题生成 | VQAv2、GQA 等 QA 数据集改造 |
| 视频问答（Video QA） | MSVD-QA, MSRVTT-QA, iVQA |
| 视觉对话（Visual Conversational QA） | Visual Dialog |
| 图像分类（Image Classification） | HatefulMemes |
| LLaVA-Instruct-150K | 多轮对话数据 |

### 5.2 Held-in / Held-out 划分

为了评估零样本泛化能力，26 个数据集被分成两组：
- **13 个 held-in 数据集**：用于 instruction tuning 训练（黄色标记）
- **13 个 held-out 数据集**：用于零样本评估（白色标记）

held-out 数据包括两种类型：1）训练时见过任务类型但没见过数据集的；2）训练时完全没见过的**整个任务类别**。论文特意保留了 4 个完整的任务类别用于零样本评估——视觉推理（ScienceQA, VSR）、视频问答（MSVD-QA, MSRVTT-QA, iVQA）、视觉对话（Visual Dialog）和图像分类（HatefulMemes）。

这意味着 InstructBLIP 在视频问答上的零样本能力是**纯泛化**——训练时连视频数据都没见过。

### 5.3 指令模板

对每个数据集，论文精心设计了 10 到 15 个不同的自然语言指令模板。例如，对 VQA 任务可能的指令包括："Answer the question based on the image."、"What is the answer to the following question?"、"Short answer: [question]" 等。

对天然倾向于短回答的数据集，会在指令中加入 "short"、"briefly" 等词汇，防止模型过度生成短输出。对 LLaVA-Instruct-150K 不做额外处理，因为它本身就是指令格式。

### 5.4 平衡采样策略

26 个数据集的大小差异巨大——从几千样本到几十万样本不等。如果均匀混合，小数据集会被欠拟合，大数据集会被过拟合。

论文提出了基于平方根的概率采样：给定 D 个数据集，大小分别为 {S₁, S₂, ..., S_D}，数据集 d 的样本被选中的概率为：

$$p_d = \frac{\sqrt{S_d}}{\sum_{i=1}^{D}\sqrt{S_i}}$$

平方根比线性更平滑——大数据集不会被过度采样，小数据集也不会被完全忽略。在此基础上还做了手动调整——降低 A-OKVQA（多选题为主）的权重，提高 OKVQA（开放式生成）的权重。

## 六、实验结果

### 6.1 零样本评估（Table 1）

论文在 13 个 held-out 数据集上对比了 InstructBLIP 与 BLIP-2、Flamingo 的零样本表现。这是 InstructBLIP 最核心的结果。

**主要指标对比（蓝色高亮为最高值）：**

| 数据集 | BLIP-2 (FlanT5XL) | InstructBLIP (FlanT5XL) | 差距 |
|--------|------------------|------------------------|------|
| NoCaps (CIDEr) | 104.5 | 119.9 | +15.4 |
| Flickr30K (CIDEr) | 76.1 | 84.5 | +8.4 |
| GQA (acc) | 44.0 | 48.4 | +4.4 |
| VSR (acc) | 60.5 | 64.8 | +4.3 |
| IconQA (acc) | 45.5 | 50.0 | +4.5 |
| TextVQA (acc) | 43.1 | 46.6 | +3.5 |
| VisDial (MRR) | 45.7 | 46.6 | +0.9 |
| HatefulMemes (AUC) | 53.0 | 56.6 | +3.6 |
| VizWiz (acc) | 29.8 | 32.7 | +2.9 |
| ScienceQA IMG (acc) | 54.9 | 70.4 | +15.5 |
| MSVD-QA (acc) | 33.7 | 43.4 | +9.7 |
| MSRVTT-QA (acc) | 16.2 | 25.0 | +8.8 |
| iVQA (acc) | 40.4 | 53.1 | +12.7 |

关键观察：

1. **InstructBLIP 在所有 13 个 held-out 数据集上都达到了新 SOTA。** 这是一个非常一致的结果——不只是某些任务好，是所有任务都好。

2. **平均相对提升 15.0%（InstructBLIP FlanT5XL vs BLIP-2 FlanT5XL）。** 论文明确报告了这个数字。

3. **对完全没见过任务类别的数据集，泛化能力更强。** 例如视频问答 MSRVTT-QA，InstructBLIP 比之前 SOTA 提高了约 47.1% 的相对提升——尽管训练时从没见过时序视频数据。

4. **用 Vicuna 的版本在某些任务上更强。** 例如 NoCaps（123.1 CIDEr, Vicuna-7B）、IconQA（50.7, Vicuna-13B）、HatefulMemes（59.6, Vicuna-7B）。

5. **超越更大参数量模型。** InstructBLIP Vicuna-7B 在很多任务上超过了 Flamingo-80B，而 7B 版本的参数量只有 80B 版本的 1/10 不到。

### 6.2 消融实验（Table 2）

论文做了两个关键消融：去掉 instruction-aware 视觉特征提取，和去掉平衡采样策略。

**FlanT5XL 版本：**

| 变体 | Held-in Avg. | GQA | ScienceQA | IconQA | VizWiz | iVQA |
|------|:----------:|:---:|:---------:|:------:|:------:|:----:|
| InstructBLIP (FlanT5XL) | **94.1** | **48.4** | **70.4** | **50.0** | **32.7** | **53.1** |
| w/o Instruction-aware | 89.8 (-4.3) | 45.9 (-2.5) | 63.4 (-7.0) | 45.8 (-4.2) | 25.1 (-7.6) | 47.5 (-5.6) |
| w/o Data Balancing | 92.6 (-1.5) | 46.8 (-1.6) | 66.0 (-4.4) | 49.9 (-0.1) | 31.8 (-0.9) | 51.1 (-2.0) |

**Vicuna-7B 版本：**

| 变体 | Held-in Avg. | GQA | ScienceQA | IconQA | VizWiz | iVQA |
|------|:----------:|:---:|:---------:|:------:|:------:|:----:|
| InstructBLIP (Vicuna-7B) | **100.8** | **49.2** | **60.5** | **43.1** | **34.5** | **52.2** |
| w/o Instruction-aware | 98.9 (-1.9) | 48.2 (-1.0) | 55.2 (-5.3) | 41.2 (-1.9) | 32.4 (-2.1) | 36.8 (-15.4) |
| w/o Data Balancing | 98.8 (-2.0) | 47.8 (-1.4) | 59.4 (-1.1) | 43.5 (+0.4) | 32.3 (-2.2) | 50.3 (-1.9) |

held-in 平均分来自四个数据集：COCO Caption、OKVQA、A-OKVQA、TextCaps。

**两个核心发现：**

1. **Instruction-aware 视觉特征提取是关键贡献。** 去掉后，FlanT5XL 版本 held-in 平均从 94.1 降到 89.8（-4.3），ScienceQA 从 70.4 降到 63.4（-7.0），VizWiz 从 32.7 降到 25.1（-7.6）。Vicuna-7B 版本的 iVQA 从 52.2 骤降到 36.8（-15.4），说明视频问答这类需要时空推理的任务尤其依赖 instruction-aware 特征——指令帮助 Q-Former 关注视频中与问题相关的帧区域。

2. **平衡采样对稳定训练更关键。** 去掉后性能下降较小但持续存在，而且训练过程变得不稳定——不同数据集在不同的训练步达到最优，导致无法用一个 checkpoint 同时服务于所有任务。

### 6.3 Instruction Tuning vs. Multitask Learning（Figure 4）

论文专门设计实验来回答一个关键问题：**InstructBLIP 的泛化能力到底是来自"多任务训练"还是来自"指令格式"？**

设计了两种多任务基线：
- **Vanilla multitask**：只用原始 input-output 格式训练，不提供指令
- **[Task:Dataset] multitask**：在输入前加 `[Task:Dataset]` 标识符

结果很清晰：
- 在 held-in 数据集上，三种方法表现类似——模型只要训练过就能拟合
- 在 held-out 数据集上，instruction tuning 显著优于两种多任务方法，而多任务方法和原始 BLIP-2 零样本性能相当

结论：**instruction 格式本身是泛化能力的关键来源**——它教会了模型"理解指令、执行对应任务"的通用能力，而不是"记住训练时的 input-output 映射"。

### 6.4 微调结果（Table 3）

InstructBLIP 还可以作为下游任务微调的初始化。和 BLIP-2 微调相比，一个重要的差异是：**InstructBLIP 保持 224×224 的输入分辨率，且冻结图像编码器**，而 BLIP-2 和 Flamingo 通常会增加输入分辨率并微调视觉编码器。

这带来了极致的训练效率：可训练参数从 1.2B（BLIP-2 微调）降到了 188M。

**微调结果对比（ScienceQA IMG 准确率）：**

| 模型 | ScienceQA IMG | OCR-VQA | OKVQA | A-OKVQA (DA) | A-OKVQA (MC) |
|------|:------------:|:-------:|:-----:|:------------:|:------------:|
| BLIP-2 (FlanT5XL) | — | — | — | — | — |
| InstructBLIP (FlanT5XL) | **90.7** | **72.8** | 62.1 | 64.0 | 75.7 |
| InstructBLIP (Vicuna-7B) | 79.5 | 72.8 | 62.1 | 64.0 | 73.4 |

论文报告 InstructBLIP 在 ScienceQA (IMG)、OCR-VQA、A-OKVQA 上达到了新 SOTA（OKVQA 被 562B 参数的 PaLM-E 超过）。

**一个有趣的发现：FlanT5 版本的 InstructBLIP 在多选题上更强，Vicuna 版本在开放生成任务上更强。** 原因是 FlanT5 的主要 instruction tuning 数据来自 NLP 基准测试，包含大量多选题和分类数据集；而 Vicuna 的 instruction tuning 数据是开放式的指令跟随数据。

### 6.5 定性评估

论文展示了 InstructBLIP 的几种关键能力：

1. **复杂视觉推理**：模型能根据图像推理出可能发生了什么——例如，从场景位置（棕榈树）推断灾难类型。

2. **知识关联**：能连接视觉输入和嵌入的文本知识，生成信息丰富的回答（如介绍一幅名画）。

3. **隐喻理解**：在描述整体氛围时，展现了理解视觉图像隐喻含义的能力。

4. **多轮对话**：能进行多轮视觉对话，在生成新回复时有效考虑对话历史。

## 八、范式意义

**InstructBLIP 的范式意义：它证明了"冻结大部分参数 + 大规模 instruction tuning"是构建通用多模态模型的一条有效路径。**

具体来说，它完成了三件事：

### 8.1 证明了 BLIP-2 架构的扩展性

BLIP-2 的 Q-Former + 冻结组件的设计，不仅适用于 caption 和 VQA，还能通过 instruction tuning 扩展到 11 个任务类别。这是对 BLIP-2 设计哲学的有力验证。

更重要的是，**InstructBLIP 的训练效率优势**——4 个模型变体各在 16 A100 GPU 上 1.5 天完成——使得这个范式对算力有限的团队也具有可行性。

### 8.2 定义了一个可复现的实验范式

论文的 held-in / held-out 划分策略、平衡采样方法、指令模板设计原则——这些方法论层面的贡献可能比具体的模型权重更有影响力。后续的多模态 instruction tuning 工作几乎都沿用了类似的实验设计。

### 8.3 验证了 instruction-aware 视觉特征的不可替代性

通过消融实验，论文清楚地证明：**不是任何形式的特征提取都能和 instruction tuning 配合**。instruction-agnostic 的视觉特征提取在 instruction tuning 下表现显著更差——尤其是需要空间推理（ScienceQA 下降 7.0 点）和时序推理（iVQA 下降 5.6-15.4 点）的任务。

这意味着"把指令也传进视觉编码器"不是一个锦上添花的技巧，而是 instruction tuning 成功的关键组件。

### 8.4 分类：bridge node

在技术树中，InstructBLIP 是一个 **bridge node（桥梁节点）**。

它连接了两条此前独立发展的技术路线：NLP 领域的 instruction tuning（FlanT5 等）和多模态领域的冻结 LLM + 视觉适配器（BLIP-2、Flamingo）。InstructBLIP 证明这两条路线可以结合，而且结合后产生了 1+1>2 的效果。

它不是 foundation node——因为它依赖 BLIP-2 的预训练权重和架构设计。但它是一个重要的 bridge node，因为它打开了一个新的研究方向。

## 九、局限性与后续方向

### 9.1 视觉编码器粒度限制

InstructBLIP 使用 224×224 的输入分辨率，远低于许多下游任务的最佳实践（如 OCR-VQA 通常使用更高分辨率）。论文在微调时保持了这一分辨率，这限制了模型在细粒度视觉任务上的上限——例如场景文字识别、小物体检测等。

### 9.2 LLM 仍然是瓶颈

论文观察到 FlanT5 版本在多选题上更好、Vicuna 版本在开放生成上更好的现象，说明**冻结 LLM 的选择直接决定了模型的能力上限**。但 Q-Former 只有 188M 参数，和 7B-13B 的 LLM 相比太小——即便 instruction tuning 也只能在"如何提取视觉特征"上做文章，无法改变 LLM 本身的推理能力和知识储备。

### 9.3 视频泛化的局限性

虽然 InstructBLIP 在视频 QA 上展现了泛化能力（47.1% 的相对提升），但它处理视频的方式（均匀采样 4 帧，独立处理，拼接特征）非常粗糙。这种策略丢失了帧间的时序依赖信息——对于需要理解动作顺序、事件因果的视频任务，这种近似是不够的。

### 9.4 多模态对话能力的深度

InstructBLIP 展现了一定程度的多轮对话能力，但和后来专门优化的模型（如 LLaVA-1.5）相比，在多轮指令跟随的流畅性上还有差距。这主要是因为 LLaVA-Instruct-150K 在 26 个数据集中只是其中一个，占比不大。

### 9.5 开放生成的质量评估

论文的定量评估集中在标准基准上（VQA、caption 等），但这些基准大多使用短答案或选择题。对于 open-ended 指令跟随的质量（如多轮对话、长文本生成），论文主要依赖定性分析，缺乏系统性的自动评估指标。

## 十、小结

InstructBLIP 是一个看起来"没加多少新东西"但实际上影响深远的工作。

从架构上看，它的创新只有一：把 instruction token 也喂给 Q-Former 的 self-attention。从方法上看，它做的事情也很简单：收集 26 个数据集，统一成指令格式，然后训一个 Q-Former。

但这个"简单"的组合产生了很显著的效果——**13 个 held-out 数据集上全部达到 SOTA**。更重要的是，这篇论文用完善的实验设计（held-in/held-out 划分、独立保留 4 个任务类别、instruction tuning vs multitask 对比）证明了一个关键的结论：**instruction 本身——而不是多任务训练——是零样本泛化的关键**。

在技术树中，InstructBLIP 扮演的是 bridge node 的角色：它连接了 NLP instruction tuning 和多模态冻结 LLM 范式这两条此前独立的路线。这个连接在 2023 年上半年影响了后续几乎所有多模态 instruction tuning 工作——LLaVA-1.5、mPLUG-Owl、Qwen-VL-Chat 都在不同程度上沿用了 InstructBLIP 建立的实验范式。

**如果你只能记住三件事：**
1. **InstructBLIP = BLIP-2 + instruction tuning**，训练效率极高（1.5 天，16 A100），全模型开源
2. **核心创新是 instruction-aware Q-Former**——把 instruction token 输入 Q-Former 的 self-attention，让视觉特征提取根据指令动态调整。去掉这个机制，ScienceQA 性能下降 7.0 点，iVQA 下降 5.6 点
3. **13 个 held-out 数据集全部 SOTA**——包括 4 个训练时完全没见过的任务类别（视频 QA、视觉对话、视觉推理、图像分类）
