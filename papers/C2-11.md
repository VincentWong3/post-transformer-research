# MaskFormer: Per-Pixel Classification is Not All You Need for Semantic Segmentation（2021）

**论文：** [arXiv:2107.06278](https://arxiv.org/abs/2107.06278) · Bowen Cheng, Alexander G. Schwing, Alexander Kirillov · FAIR + UIUC · NeurIPS 2021 Spotlight

**技术分支：** C2 — 视觉分割技术线 / C2-11 — mask classification segmentation 范式

**代码：** [bowenc0221/maskformer](https://github.com/bowenc0221/maskformer)

---

## 一、先搞清楚"问题在哪里"

语义分割的目标是把图像的每个像素分到不同的语义类别中。从 2015 年 FCN 开始，深度学习时代的语义分割几乎全部采用 **per-pixel classification（每像素分类）** 范式——对输出特征图的每个像素位置应用一个分类损失，让每个像素独立预测自己的类别标签。

与此同时，实例分割（instance segmentation）走的完全是另一条路。Mask R-CNN、DETR 等方法使用 **mask classification（掩码分类）** 范式：预测一组二进制掩码，每个掩码关联一个类别标签。这个范式天然支持可变数量的输出，适合实例级任务。

所以到 2021 年，CV 分割领域面临一个尴尬的局面：

- **语义分割**：DeepLab、PSPNet、OCRNet 等，全部走 per-pixel classification
- **实例分割**：Mask R-CNN、DETR、YOLACT 等，全部走 mask classification
- **全景分割**：需要同时做语义和实例，所以往往是一个混合系统（如 Panoptic FPN）

同一个视觉任务族，因为范式不同而产生了完全不同的模型体系。这导致模型设计、训练流程和评估方式都不统一。

但 MaskFormer 的作者们注意到了一个被忽略的历史事实：**在 FCN 之前，最好的语义分割方法（如 O2P、SDS）恰恰用的是 mask classification 范式。** FCN 的 per-pixel classification 在 2015 年凭借更优的 mIoU 接管了语义分割之后，mask classification 被完全驱逐出了语义分割领域。

于是作者们提出了一个核心问题：**能否用同一个 mask classification 模型统一语义分割、实例分割和全景分割？如果做得到，它在语义分割上能否超越统治了六年的 per-pixel classification？**

## 二、旧路线的真正问题

### 2.1 Per-pixel classification 的静态输出假设

Per-pixel classification 把图像分割视为一个 $H \times W$ 的分类问题：对每个像素 $i$ ，预测它在 $K$ 个类别上的概率分布 $p_i \in \Delta^K$ ，然后通过 per-pixel cross-entropy loss 训练：

$$\mathcal{L}_{\text{pixel-cls}}(y, y^{\text{gt}}) = \sum_{i=1}^{H \cdot W} -\log p_i(y_i^{\text{gt}})$$

这个公式隐含了一个关键假设：**输出空间是静态的**。对于一张给定的图像，模型始终输出 $H \times W \times K$ 的分类 logits。这在语义分割上没问题——每张图都需要分类所有像素。

但实例分割不行。一张图里可能有 0 个、3 个或 20 个实例，per-pixel classification 无法返回可变数量的预测。这就是为什么它天然不适用于实例级任务。

### 2.2 每个像素单独分类 vs. 整体识别

Per-pixel classification 的另一个问题是：**它鼓励逐像素决策，而不是区域级别的识别。** 每个像素独立计算其类别概率，即使相邻像素属于同一个物体，它们之间也没有任何隐式的"区域一致性"约束。

更根本地说，per-pixel classification 把"图像划分"和"区域分类"揉在一起——分类就是划分，划分就是分类。而 mask classification 把这两步解耦了：先划分图像为 $N$ 个区域（用二进制掩码表示），再把每个区域作为一个整体进行分类。

### 2.3 语义分割和实例分割的"墙"

因为两个任务用了完全不同的范式，实践中需要用完全不同的模型分别处理。全景分割试图统一两者，但要么用 Mask R-CNN + FPN 的混合架构（Panoptic FPN），要么在 per-pixel 预测上加后处理做实例分组。

这种割裂意味着：语义分割的进步（如更深的 backbone、更好的 context aggregation）不能直接惠及实例分割，反之亦然。

### 2.4 已有 mask classification 方案的局限

虽然 Mask R-CNN 和 DETR 都用了 mask classification，但它们依赖边界框预测——Mask R-CNN 需要 box head 做 proposal 分类和回归，DETR 需要 box prediction 做 bipartite matching 的匹配成本计算。

Max-DeepLab（2021）去掉了 box 依赖，但引入了复杂的辅助损失（instance discrimination loss、mask-ID cross entropy loss、标准的 per-pixel classification loss），不够简洁。

所以问题是：**能不能做一个"纯粹"的 mask classification 模型，只用 mask classification loss（分类 + 二值掩码损失），不用任何辅助损失或 box 监督？**

## 三、这篇论文的核心思路

**核心洞察：mask classification 范式本身就足够通用，可以用同一个模型、同一个损失、同一个训练流程统一解决语义分割和实例分割任务。**

具体来说，MaskFormer 的思路可以拆解成三层：

1. **从形式化角度**：把 segmentation 重新定义为 $N$ 个 `(prob, mask)` pair 的集合预测问题。语义分割和实例分割的区别只在于推理时如何组装这些 pair。

2. **从架构角度**：用 pixel decoder（负责提取 per-pixel embedding）+ transformer decoder（负责产生 per-segment embedding）来生成这些 pair。任何现有的 per-pixel classification 模型都可以通过替换分类头"无缝转换"成 MaskFormer。

3. **从训练角度**：用 bipartite matching 在预测和 GT segment 之间建立匹配，仅用一个 mask classification loss（cross-entropy 分类 + focal/dice 掩码损失）端到端训练，不需要任何辅助损失或 box 监督。

```text
Per-pixel classification（DeepLab/PSPNet/OCRNet）
        ↓
任何 per-pixel 模型都可以通过 MaskFormer 模块转为 mask classification
        ↓
同一个架构 + 同一个损失 → 语义分割 SOTA + 实例分割 SOTA + 全景分割 SOTA
```

这条技术路线的谱系位置是：

```text
FCN（per-pixel classification范式, 2015）
  ↓
DeepLab / PSPNet / OCRNet（per-pixel的各种改进, 2015-2021）
  ↓
MaskFormer（重新发现mask classification的通用性, 2021） ← 本篇
  ↓
Mask2Former（masked attention改进, 2022）
```

## 四、整体结构与 forward 流程

### 4.1 整体架构

MaskFormer 有三个模块（见 Fig. 2）：

```text
输入图像: H×W×3
    │
    ▼
┌────────────────────────────────────────────┐
│ Pixel-level Module（像素级模块）               │
│  - Backbone（ResNet / Swin-Transformer）     │
│    → 低分辨率特征图 ℱ: C_ℱ × H/S × W/S      │
│       (S=32)                                │
│  - Pixel Decoder（FPN-style 逐步上采样）      │
│    → 逐像素嵌入 ℰ_pixel: C_ℰ × H × W        │
│       (C_ℰ=256)                             │
└────────────────────┬───────────────────────┘
                     │
    ┌────────────────┴────────────────┐
    │                                 │
    ▼                                 │
┌──────────────────────┐              │
│ Transformer Module    │              │
│  - N=100 个可学习 query │             │
│  - 6 层 Transformer   │              │
│    decoder 层         │              │
│  - 对 ℱ 做 cross-     │              │
│    attention          │              │
│  → per-segment 嵌入    │              │
│    𝒬: C_𝒬 × N         │              │
└──────────┬───────────┘              │
           │                          │
           ▼                          ▼
┌────────────────────────────────────────────┐
│ Segmentation Module（分割模块）               │
│  - Class head: Linear → softmax            │
│    → N 个类别预测 {p_i ∈ Δ^{K+1}}          │
│  - Mask head: MLP (2 hidden, 256 channels) │
│    → N 个掩码嵌入 ℰ_mask: C_ℰ × N          │
│    然后用点积 + sigmoid 生成掩码:           │
│    m_i[h,w] = sigmoid(ℰ_mask[:,i]^T ·      │
│                       ℰ_pixel[:,h,w])      │
│  → N 个 (p_i, m_i) pair                    │
└────────────────────────────────────────────┘
```

### 4.2 前向过程逐步跟踪

**Step 1 — Backbone 提取特征**
输入图像 $H \times W \times 3$ ，backbone 输出低分辨率特征图 $\mathcal{F} \in \mathbb{R}^{C_\mathcal{F} \times \frac{H}{32} \times \frac{W}{32}}$ 。

**Step 2 — Pixel Decoder 产生 per-pixel embedding**
用 FPN-style 上采样：从 stride-32 特征图开始，逐步 2× 上采样，将对应分辨率的 backbone 特征图用 1×1 conv + GroupNorm 投影后相加。每个特征图经过 3×3 conv + GN + ReLU 融合，最终得到 stride-4 的特征图，再用 1×1 conv 得到 256 维 per-pixel embedding $\mathcal{E}_{\text{pixel}}$ 。

所有特征图在 pixel decoder 中都有 256 个 channels。

**Step 3 — Transformer Decoder 产生 per-segment embedding**
初始化 $N=100$ 个零向量作为 query，每个 query 关联一个可学习的位置编码。6 层 Transformer decoder 层对 backbone 特征 $\mathcal{F}$ 做 cross-attention，输出 $N$ 个 per-segment embedding $\mathcal{Q}$ 。所有 predictions 并行输出（同 DETR）。

**Step 4 — 生成类别预测**
在 $\mathcal{Q}$ 上加 linear classifier → softmax → 每个 segment 在 $K+1$ 个类别上的概率分布（含 $\varnothing$ "no object"）。

**Step 5 — 生成掩码预测**
MLP（2 hidden layers, 256 channels）将 $\mathcal{Q}$ 映射为 $N$ 个 mask embedding $\mathcal{E}_{\text{mask}} \in \mathbb{R}^{256 \times N}$ 。然后：

$$m_i[h,w] = \text{sigmoid}(\mathcal{E}_{\text{mask}}[:,i]^T \cdot \mathcal{E}_{\text{pixel}}[:,h,w])$$

**关键设计选择**：不用 softmax 强制掩码互斥，而是用 sigmoid 独立预测每个掩码。这让掩码可以重叠，适用于实例分割（同一类可以有多个重叠的实例掩码）。

### 4.3 推理策略：如何把 mask classification 输出转成任务相关格式

**语义分割（推荐）**：
用 marginalization 方式做软聚合：

$$\text{argmax}_{c \in \{1,\dots,K\}} \sum_{i=1}^N p_i(c) \cdot m_i[h,w]$$

即对每个类别 $c$ ，把所有 mask 上该位置的 $m_i$ 用 $p_i(c)$ 加权求和，然后取 argmax。这等价于把 mask classification 的输出"投影"回 per-pixel classification 格式。

**全景分割（general inference）**：
硬分配每个像素到具体的 probability-mask pair $i$ ：

$$\text{argmax}_{i: c_i \neq \varnothing} \; p_i(c_i) \cdot m_i[h,w]$$

其中 $c_i = \text{argmax}_c p_i(c)$ 。属于同一个 $i$ 的像素组成一个 segment。语义分割时合并同类 segment；实例分割时用 $i$ 区分不同实例。

在后处理中，过滤掉置信度低于 0.8 的 mask，并移除那些超过 80% 面积被其他预测遮挡的 mask。

## 五、核心模块逐个解释

### 5.1 Pixel Decoder：为什么用 FPN 而不是 ASPP

大多数 per-pixel classification 模型（如 DeepLabV3+ 的 ASPP、PSPNet 的 PPM）在 backbone 顶部用一个巨大的 context aggregation 模块来收集全局上下文。

但 MaskFormer 的 Transformer decoder 已经承担了全局上下文收集的任务——它通过 cross-attention 在整张图像的特征图上收集信息来生成类别预测。因此，pixel decoder 不需要再做 heavy context aggregation，可以做得更轻量。

所以 MaskFormer 用了 FPN-style 的上采样结构。这比 ASPP 更简单、更高效，而且生成的 per-pixel embedding 保留了高分辨率空间信息。

公式上，FPN 式 decoder 逐步上采样：

$$\mathcal{F}_4 \xrightarrow{2\times \text{upsample}} \mathcal{F}'_4 + \text{Proj}(\mathcal{F}_8) \xrightarrow{\text{fuse}} \mathcal{F}_8' \xrightarrow{\text{repeat}} \cdots \xrightarrow{} \mathcal{E}_{\text{pixel}}$$

其中 $\mathcal{F}_s$ 是 stride $s$ 的特征图。这与标准 FPN 不同的是，MaskFormer 直接用它产生 per-pixel embedding，而不是做多尺度检测。

### 5.2 Transformer Decoder：同时处理语义和实例的关键

Transformer decoder 用 100 个可学习 query，每个通过 cross-attention 在 backbone 特征图上收集信息，最终输出一个 per-segment embedding。

关键特性：
- **Parallel decoding**（同 DETR）：所有 query 的输出同时生成，没有 autoregressive 依赖
- **Learnable queries**：不是来自图像内容，而是固定数量的可学习 embedding，在训练中学会"关注"某种类型的图像区域
- **6 层 transformer decoder**：每层后应用相同 loss
- **语义分割只需 1 层**：实验发现语义分割用 1 层 decoder 就够用；实例/全景分割需要多层来去除重复预测

### 5.3 Bipartite Matching：不用 box 的匹配

MaskFormer 的 matching cost 直接基于类别和掩码预测，不像 DETR 那样需要边界框：

$$\text{cost}(z_i, z_j^{\text{gt}}) = -p_i(c_j^{\text{gt}}) + \mathcal{L}_{\text{mask}}(m_i, m_j^{\text{gt}})$$

其中 $\mathcal{L}_{\text{mask}}$ 是 binary mask loss（focal + dice）。

论文还试了另一种匹配方式：**fixed matching**。当预测数 $N$ 等于类别数 $K$ 时，可以直接把第 $i$ 个 prediction 匹配到类别 $i$ 的 GT region（不匹配 $\varnothing$ ）。实验发现 bipartite matching 比 fixed matching 更好——不仅更灵活（ $N$ 可以不等于 $K$ ），而且结果更好（+0.5 mIoU, +3.1 PQ $^{\text{St}}$ ）。

### 5.4 Mask Head：共享 per-pixel embedding vs. per-query 独立上采样

DETR 中，每个 query 独立生成一个 mask：先在低分辨率上产生 attention map，然后每个 query 用一个独立的上采样模块恢复到高分辨率。这意味着 mask head 的计算量是 $N$ 倍于单 query。

MaskFormer 的 mask head 完全不同：pixel decoder 一次性生成整张图的 per-pixel embedding（所有 query 共享），然后每个 query 只需产生一个 256 维的 mask embedding，通过点积得到 mask。

$$
\text{DETR mask head cost}: \mathcal{O}(N \times f_{\text{upsample}})
$$
$$
\text{MaskFormer mask head cost}: \mathcal{O}(f_{\text{upsample}} + N \times 256)
$$

在 $N=100$ 时，这个差异非常大。这就是为什么 MaskFormer 比 DETR 更高效。

### 5.5 Loss：仅用 mask classification loss，无辅助损失

$$\mathcal{L}_{\text{mask-cls}}(z, z^{\text{gt}}) = \sum_{j=1}^N \left[-\log p_{\sigma(j)}(c_j^{\text{gt}}) + \mathbb{1}_{c_j^{\text{gt}} \neq \varnothing} \mathcal{L}_{\text{mask}}(m_{\sigma(j)}, m_j^{\text{gt}})\right]$$

其中：
- $\sigma$ 是最优匹配
- 第一项：cross-entropy 分类损失
- 第二项：binary mask loss
- $\mathcal{L}_{\text{mask}} = \lambda_{\text{focal}} \mathcal{L}_{\text{focal}} + \lambda_{\text{dice}} \mathcal{L}_{\text{dice}}$ ， $\lambda_{\text{focal}}=20.0$ , $\lambda_{\text{dice}}=1.0$
- "no object" 类别的分类损失权重设为 0.1（同 DETR）

==核心卖点==：这是当时**唯一**不需要任何辅助损失（bounding box loss、instance discrimination loss、per-pixel classification loss）就能同时做好语义、实例、全景分割的 mask classification 模型。

## 六、核心公式与数学直觉

### 6.1 Per-pixel vs. mask classification 的损失差异

Per-pixel classification loss 对每个像素独立算损失，和周围像素无关：

$$\mathcal{L}_{\text{pixel-cls}} = \sum_{i=1}^{H\cdot W} -\log p_i(y_i^{\text{gt}})$$

Mask classification loss 把损失分解成"分类"和"分割"两部分：

$$\mathcal{L}_{\text{mask-cls}} = \underbrace{\sum_{j=1}^N -\log p_{\sigma(j)}(c_j^{\text{gt}})}_{\text{分类}} + \underbrace{\sum_{j: c_j^{\text{gt}} \neq \varnothing} \mathcal{L}_{\text{mask}}(m_{\sigma(j)}, m_j^{\text{gt}})}_{\text{分割}}$$

主要区别：
1. **分类不是在像素级别做的，而是在 segment 级别** —— 每个 segment 只有一个类别标签，该标签适用于整个 mask 区域。这提供了一个"区域级"的识别信号，而不是"逐像素"的。
2. **mask loss 关注的是形状和边界** —— focal + dice loss 组合让模型同时关注难分类像素（focal）和大/小区域（dice）。
3. **匹配机制 $\sigma$ ** —— 模型需要先学会"这个 query 应该关注图像中的哪个区域"，才能算损失。这隐式地训练了模型做 proposal 生成。

### 6.2 Matching cost 的数学直觉

Matching cost 不依赖边界框，直接使用 mask 和 class score：

$$\text{cost}_{ij} = -p_i(c_j^{\text{gt}}) + \mathcal{L}_{\text{mask}}(m_i, m_j^{\text{gt}})$$

这里的关键洞察是：**用 masks 做匹配比用 boxes 更精确，尤其对于 stuff 类别**。stuff 类别（天空、草地、墙壁）通常覆盖大块不规则区域，边界框很难表达——一个天空的 box 可能和另一个天空的 box 完全一样，但它们的 mask 可以完全不同。

Table 5 验证了这一点：使用 mask-based matching 的 PQ 是 46.5，而使用 box-based matching 的 PQ 只有 43.7（DETR 基线是 43.4）。主要差距来自 stuff 类别的 PQ $^{\text{St}}$ ：39.8 vs. 35.3/36.3。

$$\Delta \text{PQ}^{\text{St}} = 39.8 - 36.3 = 3.5$$

### 6.3 从 set prediction 到语义分割的数学变换

MaskFormer 的语义推理用了一种 marginalization 技巧：

$$\text{score}(c)[h,w] = \sum_{i=1}^N p_i(c) \cdot m_i[h,w]$$

这个公式直观含义是：像素 $[h,w]$ 属于类别 $c$ 的置信度，等于所有 mask 在该位置的值乘以该 mask 被分类为 $c$ 的概率的加权和。

这个操作本质上是：**把 mask classification 的 $N$ 个概率-掩码 pair "投影"回 per-pixel classification 的 $K$ 个类别置信度图。** 模型没有直接做 per-pixel 分类，但在推理时可以通过 marginalization 得到 per-pixel 结果。

论文对比了这种软聚合 vs. 硬分配（general inference）：semantic marginalization 在 mIoU 上更好（44.5 vs. 42.4 on ADE20K），因为它没有丢弃信息——每个像素的最终标签考虑了所有 mask 的"投票"。

### 6.4 Query 数量 $N=100$ 的合理性

直观上， $N$ 应该至少等于一张图中可能出现的最大类别数+实例数。但 ADE20K 有 150 个类别，每张图平均只有 8.2 个类别出现；COCO-Stuff-10K 有 171 个类，平均 6.6；ADE20K-Full 有 847 个类，平均 9.1。

用 100 个 query 时，MaskFormer 在所有数据集上表现最好。有趣的是：
- 即使只用 20 个 query，MaskFormer 也超过了 per-pixel baseline
- 超过 100 个 query 后效果反而下降（300 query 时 43.5 mIoU, 1000 query 时 35.4 mIoU）

论文认为这是因为**每个 query 不一定要只对应一个类别**。实验发现，不同 query 预测的唯一类别数分布不均匀：有些 query 预测多种类别，"house"和"building"可能被同一个 query 覆盖，但"water"和"sofa"也可能被同一个 query 覆盖。

固定 $N=100$ 在所有数据集上表现良好，说明**不需要根据类别数调整 query 数量**——这在 847 类的 ADE20K-Full 上非常实用，因为 per-pixel classification 需要 847 维的分类头（训练显存很大），而 MaskFormer 只需要 100 维。

## 七、重要图表解释

### 7.1 Figure 1 — Per-pixel vs. mask classification 范式对比

原图左侧展示 per-pixel classification： $H \times W$ 个像素，每个像素输出一个 $K$ 维分类向量，然后所有像素的 cross-entropy loss 相加。

右侧展示 mask classification：输出 $N$ 个 pair（class + mask），用 bipartite matching 或 fixed matching 匹配到 GT segments，每个 pair 的 loss 是分类损失 + mask 损失。

核心差别：左侧在"哪里"和"是什么"上没有区分——每个位置独立决策；右侧显式地先把"哪里"（mask）学出来，然后判断"是什么"（class）。

### 7.2 Figure 2 — MaskFormer 整体架构

论文的三模块设计清晰展示在 Figure 2 中：

1. **Pixel-level module（上半部分）**：backbone → pixel decoder → 逐像素嵌入 $\mathcal{E}_{\text{pixel}}$
2. **Transformer module（中间）**：backbone features + learnable queries → Transformer decoder → per-segment embedding $\mathcal{Q}$
3. **Segmentation module（下方）**： $\mathcal{Q}$ 分两路——class head 产生类别概率，mask head 产生 mask embedding $\mathcal{E}_{\text{mask}}$ ，然后 $\mathcal{E}_{\text{mask}}$ 和 $\mathcal{E}_{\text{pixel}}$ 点积生成最终 mask

图中的乘法符号 $\bigotimes$ 表示矩阵乘法： $\mathcal{E}_{\text{mask}}^{T} \cdot \mathcal{E}_{\text{pixel}} \rightarrow \mathbb{R}^{N \times H \times W}$ ，再 sigmoid 得到二值掩码。

## 九、Related Work 与技术谱系位置

### 9.1 技术树中的位置

```text
FCN（Long et al., 2015）
  ├── DeepLab系列（v1/v2/v3/v3+, 2015-2018）
  ├── PSPNet（Zhao et al., 2017）
  ├── OCRNet（Yuan et al., 2021）
  └── Swin-UperNet（Liu et al., 2021）
        ↓ 全部是 per-pixel classification

MaskFormer（Cheng et al., 2021）← 本篇
  └── 重新引入 mask classification 到语义分割
        ↓
  Mask2Former（Cheng et al., 2022）
  └── masked attention + 改进架构
```

MaskFormer 不处在某个范式的内部分支上（不是 per-pixel 的某个改进），而是提出了一个**范式转换**——从 per-pixel classification 回到 mask classification。

### 9.2 前驱工作

**FCN 之前的 mask classification 方法：** O2P（2012）和 SDS（2012）用 mask proposal + 分类器做语义分割，在 PASCAL VOC 上曾是 SOTA。MaskFormer 的贡献不是"发明"mask classification，而是证明它**在今天的大规模深度学习中同样有效**，甚至更好。

**DETR（Carion et al., 2020）：** MaskFormer 的 set prediction、bipartite matching、Transformer decoder 均继承自 DETR。关键区别：MaskFormer 用 mask-based matching 替代 box-based matching，移除了 box prediction head。

**Max-DeepLab（Wang et al., 2021）：** 同期工作，首次用纯 mask classification 做全景分割。但 Max-DeepLab 需要 instance discrimination、mask-ID cross-entropy、per-pixel classification 等多个辅助损失。MaskFormer 只用 $\mathcal{L}_{\text{mask-cls}}$ 。

**Mask R-CNN（He et al., 2017）：** 实例分割的基座方法。使用 box proposals + mask head。MaskFormer 去掉了 box 依赖。

### 9.3 后续影响

**Mask2Former（Cheng et al., 2022）：** MaskFormer 的直接后继。核心改进：
1. **Masked attention**：Transformer decoder 的 cross-attention 只关注预测 mask 内部的区域（用上一步的 mask prediction 做 attention mask），而不是整张图。大幅降低计算量。
2. **多尺度高分辨率特征**：从 FPN 的多尺度特征中逐级做 masked attention，提高小物体性能。
3. **更小的 Transformer decoder**：3 层替代 6 层，每个 decoder layer 只处理 0.1 × 分辨率的特征。
4. 在所有三个任务（语义、实例、全景）上全面超越 MaskFormer。

**Unified mask classification 成为分割主流：**
- MaskFormer 证明了"一个架构 + 一个损失 + 三个任务"的可行性
- Mask2Former 用 masked attention 使之高效
- 后续的 OneFormer、FC-CLIP 等都基于 mask classification 范式

## 十、范式意义

### 10.1 挑战了"per-pixel classification 是语义分割的最佳范式"的预设

从 2015 年到 2021 年，语义分割社区默认了 per-pixel classification 就是语义分割的标准做法。DeepLab、PSPNet、OCRNet 等大量工作都在这个范式下改进 context aggregation 模块。

MaskFormer 证明：**仅仅把 per-pixel classification 换成 mask classification，其他所有设计不变，就能在 ADE20K 上提升 1.8 mIoU（fixed matching）到 2.6 mIoU（bipartite matching）。**

这是一个"范式级别的提升"——它不是在已有路线内优化某个模块，而是换了一条路线。

### 10.2 统一了三个分割任务的训练流程

MaskFormer 之前，一个研究组如果要同时做语义、实例、全景分割，至少需要维护三套不同的训练代码和模型配置。

MaskFormer 之后，只用一套代码、一套配置就能弄完。Transformer decoder + mask classification 的灵活性使得训练时只需根据任务改变 supervision 格式（语义：category region masks；实例：object instance masks；全景：两者的混合），模型结构、loss、训练流程完全不动。

### 10.3 Bridge node：连接了分割范式和 set prediction 范式

MaskFormer 成功地把 DETR 的 set prediction 机制从检测/全景分割领域扩展到了语义分割领域。证明 Transformer decoder 的 set prediction 不只能做物体检测，还能做语义分割——它的输出 query 可以代表"图像中的一个区域"，而不仅是"图像中的一个物体"。

这个认知后来被 masked attention（Mask2Former）进一步拓展，最终形成了现在基于 mask classification 的分割方法家族。

## 十一、局限性与后续方向

### 11.1 小数据集上优势不明显

在 Cityscapes（19 类）上，MaskFormer 的 mIoU 和 per-pixel baseline 持平。论文的分析是：当类别少且识别容易时，mask classification 的 segment-level 分类没有明显优势，而像素级精度（SQ $^{\text{St}}$ ）反而略低（82.0 vs. 82.9）。后续的 Mask2Former 通过多尺度 high-resolution masked attention 部分解决了这个问题。

### 11.2 计算瓶颈：Transformer decoder 的全局 attention

MaskFormer 的 Transformer decoder 对 backbone 特征图 $\mathcal{F}$ 做全局 cross-attention。虽然 $\mathcal{F}$ 是 stride-32 的低分辨率特征图，但当输入是高分辨率图像（如 Cityscapes 的 1024×2048，Mapillary 的 4000×6000）时， $32\times$ 下采样后的特征点数仍然很大。

Mask2Former 的 masked attention 直接解决了这个问题——只关注 mask 内部的区域，将 cross-attention 复杂度从 $\mathcal{O}(HW)$ 降到 $\mathcal{O}(\text{mask area})$ 。

### 11.3 小物体性能

MaskFormer 使用 FPN-style pixel decoder，但 Transformer decoder 只在 stride-32 的低分辨率特征上做 cross-attention。低分辨率特征图对小物体的空间信息不敏感，可能导致小物体检测效果不佳。

Mask2Former 的多尺度设计（在 FPN 的不同分辨率上依次做 masked attention）克服了这个问题。

### 11.4 推理后处理非端到端

虽然 MaskFormer 的训练是端到端的，但推理时全景分割需要一些启发式后处理（过滤低置信度 mask、移除被遮挡的 mask）。这些后处理引入了手工设计的阈值（置信度 0.8，面积比例 80%）。

### 11.5 对初始化的敏感性

MaskFormer 从 DETR 继承了 100 个可学习 query。这些 query 的初始化质量影响训练稳定性和最终性能。DETR/Deformable DETR 后续工作表明，query 的初始化和辅助 loss 设计对 set prediction 模型的性能有显著影响。

## 十二、小结

MaskFormer 的核心贡献非常清晰：**它证明了 mask classification 范式不仅可以统一语义、实例、全景分割，而且在语义分割上超越了统治六年的 per-pixel classification。**

这个结论不是靠设计复杂的模块或训练技巧达成的——恰恰相反，MaskFormer 是一个极其简单的模型。它的架构（backbone + pixel decoder + transformer decoder）几乎全部来自已有的工作（FPN、DETR），它的 loss 只有两项（分类 + 掩码），没有任何辅助损失。论文中很少出现 "novel module" 之类的词——它做的一件事就是：**把语义分割从 per-pixel classification 换成了 mask classification，然后看看会发生什么。**

结果表明，这一换带来了：
- ADE20K 语义分割 SOTA（55.6 mIoU）
- COCO 全景分割 SOTA（52.7 PQ）
- 比 per-pixel 方法更少的参数（-10%）和 FLOPs（-40%）
- 同一套代码做三个任务

在技术树中，MaskFormer 是一个 **bridge node（桥梁节点）**——它把 DETR 的 set prediction 机制和语义分割连接起来，把掩码分类范式从实例分割领域扩展到了语义分割领域。它开启的 "unified mask classification" 路线，成为后来 Mask2Former、OneFormer 等工作的基础。
