# ViViT: A Video Vision Transformer（2021）

**论文：** [arXiv](https://arxiv.org/abs/2103.15691) · Anurag Arnab, Mostafa Dehghani, Georg Heigold, Chen Sun, Mario Lučić, Cordelia Schmid · Google Research · 2021

## 一、先搞清楚坑在哪

视频理解的目标很简单：给一段视频，让模型判断它"在干什么"——是打篮球、做蛋糕，还是两个人握手。这个任务叫**视频动作识别（Video Action Recognition）**，是计算机视觉最核心的问题之一。主流 benchmark 包括 **Kinetics-400/600**（400/600 类，YouTube 上的日常人类活动）、**Something-Something V2**（强调时序关系的细粒度动作）、**Epic Kitchens 100**（厨房场景的 egocentric 动作识别）。

在 ViViT 之前（2021 年初），视频理解领域的王者是 **3D CNN**。从 I3D（Carreira & Zisserman, 2017）把 2D 卷积"膨胀"成 3D 开始，到 SlowFast（Feichtenhofer et al., 2019）的双路径设计，再到 TSM（Lin et al., 2019）的时序移位模块——这些方法共享同一个核心假设：**用 3D 卷积核同时处理空间（高、宽）和时间（帧）三个维度**。

与此同时，2020 年 Transformer 在视觉领域迎来了爆发。**Vision Transformer（ViT）**（Dosovitskiy et al., 2020）证明了纯 Transformer 架构在图像分类上可以媲美甚至超越 CNN。一个自然的问题是：**Transformer 的全局自注意力机制能不能打破 3D CNN 的局部感受野瓶颈，把视觉 Transformer 从图像扩展到视频？**

但事情没有这么简单。视频比图像多了一个时间维度，直接把 ViT 搬过来会导致计算量爆炸。一张 224×224 的图片切成 16×16 的 patch 有 196 个 token，一段 32 帧的视频就有 32×196=6272 个 token。**Self-Attention 的计算复杂度是 O(N²)，N 翻 32 倍意味着计算量翻约 1000 倍**。这不是工程优化能解决的问题——它需要架构层面的根本设计。

所以 ViViT 的核心挑战非常明确：**如何让 Transformer 在视频上保持全局建模能力的同时，让计算量变得可行？**

## 二、3D CNN 的真正问题

在进入 ViViT 的方案之前，我们先看看 3D CNN 具体痛在哪。这不是"精度不够高"这种表面问题，而是**架构层面的结构性限制**。

### 问题 1：局部感受野的硬上限

3D 卷积核的大小是预设的超参数——比如 3×3×3 或 3×7×7——一个卷积核只能看到它窗口大小内的有限时空区域。要建模"一个人先拿起杯子，然后走到门口，最后离开房间"这种跨几十帧的长程因果依赖，需要堆叠很多层来扩大有效感受野。但深度堆叠不仅带来梯度消失问题，还会导致中间特征不断下采样，信息逐渐丢失。

Non-local Neural Networks（Wang et al., 2018）试图在 CNN 中插入非局部模块来解决这个问题，但 Non-local block 本质上就是一个 self-attention，计算量同样不小，而且和 CNN 主干网络的整合并不自然——它更像是"打了补丁"，而不是从底层重新设计。

### 问题 2：空间和时间的耦合绑定

这是最容易被忽视却最根本的问题。3D 卷积的权重在空间维度和时间维度上是共享参数的。给定一个 3×3×3 的卷积核，它同时学习空间纹理和时间运动。但直觉上，这两个任务需要完全不同的表征：

- **空间特征**：纹理、边缘、形状、颜色——"这是什么物体"
- **时间特征**：位移、速度、方向——"它怎么动"

把它们绑在同一个卷积核参数里，模型被迫去"折中"这两种需求。这就像让同一个学生同时考数学和语文——数学好的学生不一定语文好，但 3D 卷积强迫它们用同一套权重。

### 问题 3：预训练的不便

3D CNN 很少能从图像预训练中直接受益。I3D 的做法是把 2D 卷积核"膨胀"成 3D（在时间维度上复制权重），这需要额外的时间初始化策略，且效果并不理想。相比之下，Transformer 在 NLP 领域的预训练-微调范式非常成熟——如果能用上在 ImageNet-21K 上预训练好的 ViT 权重，会是巨大的优势。

### 问题 4：计算量爆炸

3D CNN 的参数体量是 $C_{in} \times C_{out} \times k_t \times k_h \times k_w$，相比 2D 卷积多了一个时间轴 $k_t$。视频数据本身又比图像大很多倍，这个组合效应导致 3D CNN 非常吃显存。实践中，训练 I3D 或 SlowFast 往往只能在 8-32 帧上操作，增加帧数就需要下采样或分片。

这些问题的本质是：**3D 卷积的局部性假设对视频而言过于严格**。如果能用 Transformer 的全局自注意力替代卷积的局部感受野，视频理解的上限可能会被重新定义。

## 三、ViViT 的核心思路

ViViT 的核心思路可以概括为：**把视频看作一个 3D 时空网格（T × H × W），用 3D 卷积提取时空块（tubelet）作为 token，再用分解式的自注意力（Factorised Self-Attention）高效处理这些 token，从而在保持全局建模能力的同时让计算量变得可控**。

这个思路包含三个紧密耦合的设计决策：

**第一，用 3D 卷积做 token 化，而不是 2D 卷积。** 相邻帧之间不是独立的——第 t 帧左上角的 patch 和第 t+1 帧左上角的 patch 在物理上对应同一空间位置。3D 卷积核能同时捕捉这两帧的局部运动信息，生成更紧凑、信息更丰富的 token 表示。这对应文中的 **Tubelet Embedding**。

**第二，把完整的时空自注意力分解成空间注意力和时间注意力。** 不是一次性地让所有 token 互相做 attention，而是先让同一帧内的 token 互相做 attention（空间注意力），再让同一空间位置不同帧的 token 互相做 attention（时间注意力）。计算量从 O((N_t × N_s)²) 降到 O(N_t × N_s² + N_s × N_t²)，减少了约 15 倍。这对应文中的 **Factorised Self-Attention**。

**第三，把三维位置编码分解成空间位置编码和时间位置编码。** 完整的 3D 位置编码需要为每个时空位置独立学习一个向量，参数多且无法泛化到不同的帧数或分辨率。分解后，空间编码在帧间共享，时间编码在所有空间位置共享，参数减少约 15 倍，且天然支持变长视频输入。

这三个设计不是孤立的——它们共同回答了"视频和图像的本质区别是什么"这个问题。视频比图像多了一个时间维度，但这个维度不是简单的"更多帧"，而是有独特的结构——帧间连续性、运动轨迹、时序因果性。ViViT 的设计正是为了捕捉这些结构。

## 四、Tokenization：Uniform Frame Sampling vs Tubelet Embedding

ViViT 提出了两种将视频映射到 token 序列的方法。

### 方法 1：Uniform Frame Sampling（均匀帧采样）

这是最简单直接的方法：从输入视频中均匀采样 N_t 帧，每帧用 ViT 的 2D patch embedding（16×16 非重叠 patch + 线性投影）独立处理，然后把所有帧的 token 拼接成一个长序列。

输入视频形状 T×H×W×3，输出 token 序列形状 (N_t × N_h × N_w) × D，其中 N_h = H/16, N_w = W/16。对于 32 帧 224×224 的视频，N_t=32, N_h=N_w=14，总 token 数 = 32×14×14 = 6272。

这种方法的优点是简单、实现方便，并且可以直接复用 ViT 的 2D patch embedding 权重。缺点是完全没有利用帧间的局部运动信息——相邻帧的时间对应关系需要由 Transformer 从零开始学习。

### 方法 2：Tubelet Embedding

与此相对，Tubelet Embedding 用 3D 卷积同时提取时空块。3D 卷积核大小为 (t, h, w) = (2, 16, 16)，步长为 (τ, δ, δ) = (2, 16, 16)。每个 tubelet 覆盖 2 帧、16×16 的空间区域，相邻 tubelet 在时间和空间上都不重叠。

输出形状：N_t = T/τ = 16, N_h = H/δ = 14, N_w = W/δ = 14，总 token 数 = 16×14×14 = 3136。注意，Tubelet Embedding 的时间步长是 2，所以 token 数只有 Uniform Frame Sampling 的一半（3136 vs 6272），但每个 token 包含了两帧的时空信息。

论文 Table 1a 的消融实验显示，在 Kinetics-400 上，Tubelet Embedding 比 Uniform Frame Sampling 高约 1.5% 的 top-1 准确率（78.5% vs 77.0%）。这个差距在 Something-Something V2 这种对时序关系更敏感的 benchmark 上可能会更大。

### 为什么 Tubelet 更好？

**直觉：** 相邻帧之间的运动通常是平滑的——物体不会瞬移。3D 卷积核能同时看到两帧的同一空间区域，可以学习到"这个像素在 t 帧是红色，在 t+1 帧变成了蓝色"这样的时序变化模式。这种信息在后续的 self-attention 中会被利用，帮助模型理解运动。

**数学对应：** 3D 卷积的操作等价于一个 3D 线性投影，其输出特征自然地融合了相邻帧的时空信息。如果使用 2D 卷积（每帧独立），相邻帧的 token 在序列中是相邻的，但 self-attention 需要从数据中学习这种对应关系——这比 3D 卷积直接编码进来更难。

## 五、四种模型变体

ViViT 提出了四种模型变体，从简单到复杂排列。这不是随意提出的，而是系统地展示了设计空间——从"最直接的 ViT 扩展"到"最高效的分解架构"。

### Model 1：Spatio-temporal Attention（ViViT-S）

这是最直接的 baseline：把所有时空 token 交给一个完整的 Transformer encoder，不做任何分解。每个 transformer 层的 MSA 在所有 token 之间计算全连接注意力。

这个模型的优点是**全局建模能力最强**——从第一层开始，每个 token 就可以关注到视频中所有帧的所有位置。缺点是计算量巨大——O((N_t × N_s)²) 的复杂度对视频来说完全不现实。论文中使用这个模型作为 upper-bound baseline，但不作为主要方法。

论文中对应的变体是 **ViViT-M**（M 代表 Model 1）。

### Model 2：Factorised Encoder

这个模型使用两个独立的 Transformer encoder：首先是一个**空间编码器**，只建模同一帧内 token 之间的交互；然后是一个**时间编码器**，建模帧与帧之间的交互。

具体来说：
1. 空间编码器接收所有 token，但 attention 只在同一帧内计算（不跨帧）。
2. 每个帧的 token 经过 L_s 层空间编码器后，用 [CLS] token 或全局平均池化得到帧级特征 h_i ∈ ℝ^d。
3. 所有帧的帧级特征组成序列 H ∈ ℝ^{N_t × d}，输入时间编码器。
4. 时间编码器经过 L_t 层后输出最终的分类特征。

这个架构对应的是一个"**late fusion**"策略——先独立理解每帧的内容，再融合时序信息。它的计算复杂度是 O(N_t × N_s² + N_t²)，相比 Model 1 大幅降低。

### Model 3：Factorised Self-Attention

这个模型更精巧——不在编码器层面做分解，而是在**每个 transformer block 内部**把 MSA 操作分解成空间注意力和时间注意力两个步骤。

输入 token 先做空间注意力（同一帧内），再做时间注意力（同一空间位置、不同帧）。两个注意力步骤共享同一组 LayerNorm 和 skip-connection。关键区别是：注意力的计算范围被限制在空间域或时间域，而不是完整的时空域。

这种做法的计算复杂度和 Model 2 相同（O(N_t × N_s² + N_t² × N_s)），但因为注意力是在每层内部做的，模型可以更早地融合时空信息，而不像 Model 2 那样需要先处理完所有空间层才进入时间层。

论文中 Model 3 对应 **ViViT-L**（L 代表 Large）。

### Model 4：Factorised Encoder with Local Attention

这个模型是对 Model 2 的进一步改进：空间编码器使用局部注意力（只在局部窗口内做注意力，如 Swin Transformer 的窗口注意力），而不是全局注意力。

这进一步降低了计算量，但论文中没有作为主要变体重点讨论。

### 四种变体的定位

| 变体 | 论文命名 | Tokenization | 注意力机制 | 复杂度（单层） |
|------|---------|-------------|-----------|--------------|
| Model 1 | ViViT-M | Tubelet | 完整时空 | O(N_t²N_s²) |
| Model 2 | — | Tubelet | 空间编码器+时间编码器 | O(N_tN_s² + N_t²) |
| Model 3 | ViViT-L | Tubelet | 每层内分解 | O(N_tN_s² + N_sN_t²) |
| Model 4 | — | Tubelet | 局部空间+全局时间 | 更低 |


论文的实验主要聚焦在 **ViViT-S**（最小的单帧 Spatial-only baseline）和 **ViViT-L/Model 3**（Factorised Self-Attention）上。

## 六、网络架构详解

### 6.1 ViViT-L（Model 3）的完整前向过程

```
输入视频: [B, 32, 224, 224, 3] ──────────────────────────────────┐
                                                                  │
Step 1: Tubelet Embedding                                         │
  3D Conv: kernel=(2,16,16), stride=(2,16,16)                     │
  → [B, 16, 14, 14, D]                                           │
  → flatten: [B, 3136, D]                                        │
                                                                  ▼
Step 2: 位置编码 + [CLS] Token                                     │
  空间位置编码: [1, 1, 14, 14, D] → 广播到所有帧                     │
  时间位置编码: [1, 16, 1, 1, D] → 广播到所有空间位置                  │
  相加: P(t,h,w) = P_space(h,w) + P_time(t)                       │
  [CLS] token 拼接到序列头部                                      │
  → [B, 3137, D]                                                  │
                                                                  ▼
Step 3: L 层 Factorised Transformer Block                          │
  每层包含:                                                        │
  ┌─────────────────────────────────────────────────────┐          │
  │  LayerNorm → Space MSA (帧内) → 残差连接               │          │
  │  LayerNorm → Time MSA (帧间同位置) → 残差连接          │          │
  │  LayerNorm → MLP → 残差连接                          │          │
  └─────────────────────────────────────────────────────┘          │
  → [B, 3137, D]                                                  │
                                                                  ▼
Step 4: 分类头                                                     │
  [CLS] token → LayerNorm → Linear(num_classes)                   │
  → [B, num_classes]                                              │
```

### 6.2 ViViT-H 的架构

ViViT-H（Huge）是论文中性能最好的变体，它结合了 Model 2 和 Model 3 的优点——使用独立的时序和空间编码器，同时在各个编码器内部使用分解注意力。

ViViT-H 的架构更复杂：
- **空间编码器**：12 层 transformer，处理每帧内的 token。每帧一个 [CLS] token。
- **时间编码器**：8 层 transformer（比空间层少），接收 16 个帧级 [CLS] token 再加上一个全局 [CLS] token。

这种设计的直觉是：空间建模需要更多层（每帧 196 个 token，信息密度高），而时间建模需要更少层（16 个帧级 token，信息已经高度抽象）。这种非对称设计进一步提高了效率。

ViViT-H 在 Kinetics-400 上达到 79.5% 的 top-1 准确率，是 2021 年时的 SOTA。

### 6.3 关键设计细节

**1. 参数共享：** 空间编码器在处理不同帧时使用**共享权重**。这意味着模型学会的"如何理解一帧图像"可以复用到所有帧，减少了过拟合风险。

**2. 位置编码的广播策略：** 空间位置编码的形状是 (1, 1, N_s, D)，通过 broadcasting 扩展到所有帧。时间位置编码的形状是 (1, N_t, 1, D)，通过 broadcasting 扩展到所有空间位置。两者相加得到每个 token 的完整位置编码。这种广播机制使得模型天然支持变长输入。

**3. [CLS] token 的策略：** ViViT-L 使用全局 [CLS] token（在序列头部，和所有 token 做 attention），而 ViViT-H 使用每帧一个 [CLS] token（空间编码器）和全局 [CLS] token（时间编码器）。前者的架构更简单，后者的分离度更高。

## 七、Factorised Self-Attention 详解

这是 ViViT 的核心贡献，也是论文被广泛引用的主要原因。我来详细拆解这个设计。

### 7.1 为什么要分解？

假设视频有 N_t 个时间位置和 N_s 个空间位置，总 token 数是 N = N_t × N_s。标准 Self-Attention 的复杂度是 O(N²) = O(N_t² × N_s²)。

具体数值：N_t = 16, N_s = 196, N = 3136。完整注意力单层的浮点运算量约为：
$$N_t^2 \times N_s^2 \times d_k = 16^2 \times 196^2 \times 64 \approx 6.3 \times 10^8$$

这还只是一层。一个 12 层的 Transformer 就是 75 亿次浮点运算——没有任何实际可用。

### 7.2 分解后的复杂度

Factorised Self-Attention 把此操作分解成两步：

1. **空间注意力**（每帧内部）：O(N_t × N_s² × d_k)
2. **时间注意力**（同空间位置、不同帧之间）：O(N_s × N_t² × d_k)

总复杂度：O(N_t × N_s² × d_k + N_s × N_t² × d_k) = O(N_t × N_s × d_k × (N_s + N_t))

具体数值：16 × 196 × 64 × (196 + 16) ≈ 4.25 × 10^7

对比完整注意力的 6.3×10^8，减少了约 **15 倍**。

### 7.3 分解的有效性

更反直觉的是：**分解后性能反而更好**。论文 Table 2a 显示，在 Kinetics-400 上，ViViT-L（Factorised，78.5%）超过了 ViViT-M（完整注意力，77.8%）。

为什么"更简单"的模型比"更完整"的模型效果好？三个原因：

**第一，参数效率。** 完整注意力的参数量更大（需要更大维度的 QKV 投影），在有限的数据下更容易过拟合。分解后每个注意力模块的参数更少，但反而更专注于自己的任务——空间注意力专注识别物体，时间注意力专注追踪运动。

**第二，优化更容易。** 完整时空注意力要让一个注意力矩阵同时编码空间关系（左上 vs 右下）和时间关系（第 3 帧 vs 第 10 帧），这对注意力头的压力很大。分解后，空间注意力的注意力矩阵只需编码空间结构，时间注意力的只需编码时序关系，每个任务都更容易学习。

**第三，正则化效果。** 分解本身就是一种结构正则化——它告诉模型"空间信息应该由空间注意力处理，时间信息应该由时间注意力处理"。这相当于在模型架构中注入了先验知识，减少了模型需要从数据中学习的自由度。

### 7.4 两种实现方式

**方式 A：层内分解（Model 3 / ViViT-L）**

在每个 transformer block 内部，顺序执行空间 MSA 和时间 MSA。两者共享同一组残差连接和 LayerNorm。输入 token 先后经过空间注意力和时间注意力的"串行"处理。

**方式 B：编码器级分解（Model 2 / ViViT-H）**

使用两个独立的 transformer encoder——空间编码器先处理完所有层，输出帧级特征；时间编码器再处理帧级特征。注意力的分解发生在编码器层面，而不是层层面。

两者的计算复杂度相同（都在 O(N_s² + N_t²) 量级），但架构不同。方式 A 更"早融合"——空间和时间信息在每层内就交互了。方式 B 更"晚融合"——空间处理完才进入时间。论文中发现方式 B（即 ViViT-H）性能更好，可能是因为空间信息和时间信息的抽象级别不同，用独立编码器处理更优。

## 八、分解位置编码

### 8.1 问题

视频 token 的位置是三维的（时间 × 高度 × 宽度）。如果使用完整的三维位置编码——为每个时空位置独立学习一个向量——参数数量是 N_t × N_s × D。以 ViViT-L 为例：16 × 196 × 768 ≈ 240 万个参数。

但这还不是主要问题。**主要问题是泛化性：** 完整的三维编码无法泛化到不同的分辨率或帧数。如果测试时帧数从 16 变成 32，或者分辨率从 224 变成 448，完整的三维位置编码就失效了——它只能处理预定义的时空网格。

### 8.2 方案

ViViT 把三维位置编码分解成空间位置编码和时间位置编码：

$$P_{t,h,w} = P^{\text{space}}_{h,w} + P^{\text{time}}_t$$

其中：
- $P^{\text{space}} \in \mathbb{R}^{H_s \times W_s \times D}$：空间位置编码，在帧间共享
- $P^{\text{time}} \in \mathbb{R}^{N_t \times D}$：时间位置编码，在所有空间位置共享

参数数量：H_s × W_s × D + N_t × D = 14×14×768 + 16×768 = 162,816。相比完整编码的 2,408,448 减少了 14.8 倍。

### 8.3 为什么有效

**直觉：** 空间编码告诉模型"这个 patch 在帧的左上角还是右下角"，时间编码告诉模型"这是第几帧"。这两者是独立且正交的——"左上角"不依赖于这是第几帧，"第 5 帧"也不依赖于它出现在屏幕的哪个位置。加法操作足够表达它们的组合。

**实验证据：** 论文 Table 1b 显示，分解位置编码比完整 3D 编码高 0.5% 的 top-1 准确率（78.5% vs 78.0%）。分解编码不仅参数更少，性能反而更好。

### 8.4 变长输入的天然支持

分解编码还有一个隐藏优势：**支持变长输入**。在训练时可以用 16 帧，测试时可以用 32 帧——只需增加时间编码的数量（在已学习的编码之间插值或直接学习更多编码）。空间编码同理。完整 3D 编码则无法做到这一点。

## 九、公式详解

### 9.1 Self-Attention 的计算复杂度

标准 Self-Attention：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

计算复杂度：O(N² × d_k + N × d_v)，主要开销在 QK^T 的矩阵乘法，复杂度 O(N² × d_k)。

对于视频，N = N_t × N_s，完整注意力的复杂度：

$$O((N_t N_s)^2 \cdot d_k) = O(N_t^2 N_s^2 d_k)$$

Factorised Attention 的复杂度：

$$\text{Space: } O(N_t \cdot N_s^2 \cdot d_k), \quad \text{Time: } O(N_s \cdot N_t^2 \cdot d_k)$$

$$\text{Total: } O(N_t N_s d_k (N_s + N_t))$$

加速比：

$$\frac{N_t^2 N_s^2}{N_t N_s (N_s + N_t)} = \frac{N_t N_s}{N_s + N_t}$$

当 N_t = 16, N_s = 196 时，加速比 = (16 × 196)/(196 + 16) ≈ 14.8 倍。

### 9.2 Tubelet Embedding 的输出形状

$$X_{\text{tubelet}} = \text{Conv3D}_{\text{kernel}=(t,h,w), \text{stride}=(\tau,\delta,\delta)}(X_{\text{video}})$$

输出形状：B × T/τ × H/δ × W/δ × D

参数数量：C_in × D × t × h × w + D（偏置）= 3 × 768 × 2 × 16 × 16 + 768 = 1,179,648

这个参数数量远小于后续 transformer 编码器的参数（如 ViViT-L 有 87M 总参数），说明 tokenization 层不是主要开销。

### 9.3 Factorised Positional Encoding

$$P_{t,h,w} = P^{\text{space}}_{h,w} + P^{\text{time}}_t$$

符号说明：
- $P_{t,h,w} \in \mathbb{R}^D$：位置 (t, h, w) 的完整位置编码
- $P^{\text{space}} \in \mathbb{R}^{H_s \times W_s \times D}$：空间位置编码，帧间共享
- $P^{\text{time}} \in \mathbb{R}^{N_t \times D}$：时间位置编码，空间位置间共享

在实现中，这是通过 broadcasting 完成的——空间编码从 (1, 1, H_sW_s, D) 广播到 (B, N_t, H_sW_s, D)，时间编码从 (1, N_t, 1, D) 广播到 (B, N_t, H_sW_s, D)，然后逐元素相加。

## 十、训练策略与正则化

Transformer 模型（尤其是视觉 Transformer）有一个众所周知的特性：**它们需要大量数据或者很强的正则化才能训练好**。ViT 在 JFT-300M（3 亿图片）上预训练才能达到 SOTA，就是因为 Transformer 缺少 CNN 的平移不变性等归纳偏置。

视频数据集的规模比图像小得多——Kinetics-400 只有 24 万视频片段，ImageNet 有 140 万图片。如何在小数据集上训练好视频 Transformer 是 ViViT 必须解决的工程问题。

### 10.1 从 ViT 加载预训练权重

ViViT 的策略是：**先在 ImageNet-21K（1400 万图片）上预训练 ViT，然后把 ViT 的权重迁移到 ViViT 中**。

迁移方式：
1. **Patch embedding 层**：ViT 的 2D patch embedding（Conv2D）需要变成 3D tubelet embedding（Conv3D）。做法是在时间维度上复制权重——把 Conv2D 的权重在时间轴上复制 t 次（tubelet 的时间长度），然后除以 t 做归一化。
2. **Transformer 层**：ViT 的 transformer 层可以直接复用。如果使用 Factorised Self-Attention，把原来的 MSA 替换成空间 MSA + 时间 MSA 的序列。
3. **Positional encoding**：ViT 的 2D 位置编码不能直接复用，需要初始化 ViViT 的分解位置编码。空间编码可以从 ViT 的位置编码中提取，时间编码从头初始化。

这种方法的好处是：**ViT 已经学会了强大的图像表示，ViViT 只需要学习"如何把帧间信息关联起来"**，大大减少了对视频数据的需求。

### 10.2 正则化策略

论文实验部分使用了多种正则化技术：

1. **Dropout**：在 MLP 块中应用 dropout，dropout rate 在 {0.1, 0.2, 0.3} 中调优。
2. **Stochastic Depth（DropPath）**：随机跳过一些 transformer 层，drop rate 在 {0.1, 0.2} 中调优。
3. **Weight Decay**：使用 AdamW 优化器，weight decay 在 {0.01, 0.03} 中调优。
4. **Label Smoothing**：标签平滑系数设为 0.1。
5. **RandAugment**：应用 RandAugment 数据增强策略。

这些正则化技术对视频 Transformer 特别重要，因为视频模型的参数量大但训练数据有限。

### 10.3 训练配置

论文没有详细公布所有训练配置，但从实验描述中可以推断：
- **优化器**：AdamW
- **学习率**：余弦衰减 + 线性 warmup
- **Batch size**：1024（在 TPU v3-256 上训练）
- **训练 epoch**：30-50 个 epoch（Kinetics-400）
- **预训练数据**：ImageNet-21K（用于 ViT 权重初始化）

### 10.4 数据增强

对于视频数据，ViViT 使用了：
1. **Random cropping**：在时间维度随机裁剪 32 帧，空间维度随机裁剪 224×224。
2. **Random flipping**：水平随机翻转。
3. **Color jitter**：颜色抖动增强。

这些增强策略和图像领域的标准做法基本相同，没有针对视频的特殊设计（如时序一致的增强）。

## 十二、与同期工作的比较

ViViT 的 arxiv 日期是 2021 年 3 月 26 日。它有三个主要的同期竞品：

### TimeSformer（Bertasius et al., 2021，2021 年 2 月）

TimeSformer 比 ViViT 早约一个月发布，也提出了分解时空注意力的方案（Divided Space-Time Attention）。两者在注意力机制上的思路非常相似。

关键区别：
- **Tokenization**：TimeSformer 使用 2D patch embedding（每帧独立），ViViT 使用 3D tubelet embedding（时空一体）。
- **注意力顺序**：TimeSformer 先做时间注意力再做空间注意力，ViViT 的 Model 3 先做空间再做时间。
- **实验基准**：TimeSformer 在 Kinetics-400 上达到 78.0%（ViViT-L 是 78.5%）。

时间顺序上 TimeSformer 更早，但 ViViT 的设计更系统、消融实验更充分。

### Video Swin Transformer（Liu et al., 2021，2021 年 6 月）

Video Swin 把 Swin Transformer 的移位窗口机制扩展到 3D。它的核心思路不是"全局注意力"，而是"3D 窗口注意力 + 窗口移位"。

关键区别：
- **计算效率**：Video Swin 的窗口注意力比 ViViT 的分解注意力更高效（约 200 GFLOPs）。
- **感受野**：窗口注意力是局部的，需要跨窗口信息交换（通过窗口移位），不如 ViViT 的全局注意力。
- **性能**：Video Swin 在 Kinetics-400 上达到 78.8%，略高于 ViViT-L 的 78.5%。

### MViT（Fan et al., 2021，2021 年 4 月）

MViT（Multiscale Vision Transformers）提出了多尺度特征金字塔，在计算效率和性能之间取得了更好的平衡。MViT 在 Kinetics-400 上达到约 79.0%。后来许多视频 Transformer 工作（如 VideoMAE）都基于 MViT 而非 ViViT。

三个工作的定位不同：TimeSformer 是"先发"，ViViT 是"系统"，Video Swin 和 MViT 是"高效"。ViViT 的价值在于它的系统性——它提出了四种模型变体、完整的消融实验和明确的"分解注意力"设计原则，为后续工作提供了清晰的设计空间参考。

## 十三、位置

### 13.1 前驱工作

1. **ViT（Dosovitskiy et al., 2020）**：ViViT 最直接的前驱。ViT 证明了 Transformer 在图像分类上的有效性，ViViT 将其扩展到视频领域。ViViT 的模型架构、tokenization、位置编码都直接继承自 ViT。

2. **Transformer（Vaswani et al., 2017）**：自注意力机制的起源。ViViT 的 Factorised Attention 是对原始 Transformer 中 Multi-Head Attention 的时空分解。

3. **I3D（Carreira & Zisserman, 2017）**：3D CNN 在视频理解中的奠基工作。ViViT 的 tubelet embedding 借鉴了 I3D 中"把 2D 卷积膨胀成 3D"的思想。

4. **Non-local Neural Networks（Wang et al., 2018）**：在 CNN 中引入自注意力机制的先驱。ViViT 的 Factorised Attention 可以看作 Non-local block 的 Transformer 版本。

5. **C3D（Tran et al., 2015）**：3D 卷积在视频中的早期工作。ViViT 的 tubelet embedding 在概念上与 C3D 的 3D 卷积类似，但用途不同——C3D 用 3D 卷积极做特征提取，ViViT 用 3D 卷积做 tokenization。

### 13.2 后续影响

1. **VideoMAE（Tong et al., 2022）**：把 MAE（Masked Autoencoder）扩展到视频领域，使用高比例掩码策略（90%-95%），在 ViViT 的架构上进行自监督预训练。VideoMAE 在 Kinetics-400 上达到 87.4% 的 top-1 准确率，证明了 ViViT 架构在自监督学习中的有效性。

2. **ActionFormer（Zhang et al., 2022）**：在时序动作检测中使用 ViViT 作为视频编码器。

3. **Video-LLaMA（Zhang et al., 2023）**：在视频大语言模型中使用 ViViT 作为视觉编码器，连接视频和文本。

4. **Scenic 框架**：ViViT 的代码被整合到 Google 的 Scenic 视觉 Transformer 库中，成为视频 Transformer 的标准实现。

### 13.3 在技术树中的位置

在本技术树中，ViViT 出现在 B5 分支的三个位置：
- **B5-1（视频 token 化）**：Step B5-2，和 TimeSformer（B5-1）一起构成了 spatiotemporal tokenization 的两条路线。
- **B5-2（降低 attention 复杂度）**：Step B5-5，通过 Factorised Self-Attention 解决了视频 attention 的计算瓶颈。
- **B5-4（统一时空 encoder）**：Step B5-10，作为视频 foundation encoder 的一个方案。

这三个位置不是重复——它们代表了 ViViT 贡献的三个不同侧面：**视频如何 token 化**、**时空 attention 如何高效**、**统一 encoder 如何设计**。

## 十四、局限

### 局限 1：计算量仍然很大

即使使用了 Factorised Attention，ViViT-H 的计算量仍然高达 1992 GFLOPs。相比之下，SlowFast 只有 213 GFLOPs，TSM 只有 65 GFLOPs。这意味着 ViViT 在移动设备、无人机、自动驾驶等资源受限的场景中难以部署。

后续的 Video Swin Transformer 通过局部窗口注意力把计算量降到约 200 GFLOPs，MViT 通过多尺度设计也实现了更好的效率。但必须承认：视频 Transformer 从第一天起就面临计算成本问题。

### 局限 2：对长视频的支持有限

ViViT 的设计假设输入是固定长度的视频片段（通常 32 帧）。对于几分钟到几小时的长视频，ViViT 无法直接处理，需要额外的分段策略。在实际应用中（如监控视频分析、电影理解），长视频才是常态。后续的 ActionFormer 使用 ViViT 作为 backbone，但需要额外的时序定位模块来处理长视频。

### 局限 3：预训练数据需求大

ViViT 需要在大规模数据集 ImageNet-21K（1400 万图片）上预训练 ViT 的权重，然后在 Kinetics-400 上微调。如果没有这些预训练权重，性能会明显下降。在医学视频、特定领域监控视频等数据稀缺的场景下，ViViT 难以直接使用。

后续的 VideoMAE 通过自监督预训练（只需要视频数据，不需要标签），降低了对大规模标注数据的依赖。

### 局限 4：时序建模的粒度有限

ViViT 的 tubelet embedding 使用步长 2，把 32 帧压缩成 16 个时间位置。这意味着模型无法捕捉帧级别的精细运动（如微表情、快速手势）。每个时间 token 包含 2 帧信息，时间分辨率只有 16 个位置。对于需要精细时序理解的任务，这个粒度可能不够。

### 局限 5：没有跨视频建模

ViViT 处理的是单个视频片段，没有考虑视频之间的关联。在需要跨视频推理的任务（如视频检索、视频摘要）中，ViViT 需要再搭配额外的检索或聚类模块。

## 十五、小结

ViViT 是视频 Transformer 领域的奠基性工作。它系统性地回答了"如何把 ViT 扩展到视频"这个核心问题，提出了三个紧密耦合的关键创新：

1. **Tubelet Embedding**：用 3D 卷积提取时空块，让每个 token 天然包含相邻帧的局部运动信息。相比 2D patch embedding 提升 1.5% top-1 准确率。

2. **Factorised Self-Attention**：把完整的时空注意力分解成空间注意力和时间注意力，计算量减少约 15 倍的同时性能反而提升 0.7%。这不是"妥协"，而是结构正则化带来的增益。

3. **分解位置编码**：把三维位置编码分解成空间编码和时间编码，参数减少 14.8 倍，泛化性更好，天然支持变长输入。

这些创新的共同特征是对**视频的四维结构（T×H×W×C）做了合理的因子分解**——把"时空一体"的问题拆解成"空间"和"时间"两个相对独立但协同的子问题。

ViViT 的**更广泛影响**在于：它开启了视频 Transformer 时代的大门。在 ViViT 之后，视频 Transformer 成为了视频理解的主流范式，涌现了 TimeSformer、Video Swin、MViT、VideoMAE 等一系列工作。即使到今天，ViViT 的分解注意力思想仍然被大多数视频 Transformer 继承——不管是窗口注意力、多尺度注意力还是掩码建模，底层的"空间和时间分开建模"原则都源自 ViViT 和同期工作的开创性探索。

当然，ViViT 也有局限性——计算量大、对长视频支持有限、需要大规模预训练。但这些局限不是 ViViT 独有的，而是早期视频 Transformer 领域面临的共同挑战。后续工作在效率（Video Swin）、自监督学习（VideoMAE）、长视频理解（ActionFormer）等方向上不断改进，把视频理解的边界持续向前推进。
