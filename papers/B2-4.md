## 一、先搞清楚坑在哪

Vision Transformer（ViT）在 2020 年横空出世，把 NLP 的 Transformer 直接搬到了图像上。做法很简单：把图像切成 16×16 的 patch，拉平后当作 token 序列，扔进标准的 Transformer Encoder 里。结果很惊艳——在大规模数据（JFT-300M）上预训练后，ImageNet 准确率超过了当时的 CNN SOTA。

但这里有个致命问题：**ViT 的自注意力（Self-Attention）是全局的**。每个 patch token 都要和所有其他 patch token 计算注意力权重。对于一张 224×224 的图像，切成 16×16 patch 后得到 196 个 token，计算量还能接受。但如果输入是 384×384 甚至 512×512，token 数量变成 576 或 1024，自注意力的计算复杂度是 $O(N^2)$，其中 $N$ 是 token 数。$N$ 翻倍，计算量翻四倍。这在图像上尤其致命——图像分辨率天然就高，而且很多任务（如目标检测、语义分割）需要高分辨率输入。

直白说：**全局自注意力在计算上不可扩展**。

## 二、Swin Transformer 的解决方案及其问题

Swin Transformer（Liu et al., 2021）提出了一个巧妙的思路：**在局部窗口内做自注意力**。把 feature map 划分成不重叠的 7×7 窗口，每个窗口内独立计算自注意力。这样计算复杂度从 $O(N^2)$ 降到了 $O(N \cdot M^2)$，其中 $M$ 是窗口大小（常数 7），$N$ 是 token 总数。复杂度变成了 $O(N)$，线性于 token 数。

但局部注意力有个天然缺陷：**每个 token 只能看到自己窗口内的信息，感受野受限**。为了引入跨窗口信息，Swin 设计了"移位窗口"（Shifted Window）机制：在相邻的 Transformer 层之间，把窗口划分偏移半个窗口大小，让不同窗口的 token 有机会交互。这确实解决了信息孤岛问题，但带来了新的问题：

1. **移位后的窗口数量会变化**（从 4 个变成 9 个），需要做 cyclic shift 和 mask 操作，实现复杂。
2. **跨窗口信息交互是间接的**——需要经过多层累积才能让远处 token 交换信息。
3. **窗口大小是固定的**（7×7），无法灵活适应不同尺度特征的交互需求。

## 三、Focal Self-Attention 的核心思路

Focal Self-Attention 的核心洞察非常直观：**视觉注意力应该像人眼一样，有"焦点"和"周边"之分**。人眼在看一个物体时，焦点区域看得清晰（高分辨率），周边区域看得模糊（低分辨率），但周边信息仍然被利用来理解全局语境。

把这个想法用到 Transformer 里就是：对于每个 query token，**近处的 patch 用细粒度注意力（每个 patch 单独计算），远处的 patch 用粗粒度注意力（把 patch 聚合后再计算）**。这样既保持了局部细节的精确建模，又通过粗粒度方式引入了全局信息，而且计算量可控。

具体来说，Focal Self-Attention 把注意力计算分成两部分：
- **细粒度局部注意力**：在当前 token 周围的局部窗口内，对每个 patch 单独计算注意力（类似 Swin 的窗口注意力）。
- **粗粒度全局注意力**：对远处的区域，先把 patch 通过池化（pooling）聚合为更少的"粗 token"，再对这些粗 token 计算注意力。

这样，每个 token 在一次注意力计算中就能同时获得局部细节和全局语境，不需要像 Swin 那样通过多层累积。

## 四、网络架构详解

Focal Self-Attention 可以替换 ViT 或 Swin Transformer 中的任何自注意力模块。论文以 Swin Transformer 为 backbone，把其中的窗口自注意力替换为 Focal Self-Attention，形成了 Focal Transformer。

### 前向过程详解

以输入 feature map 大小为 $H \times W \times C$ 为例（$H=56, W=56, C=96$），逐步骤说明：

**步骤 1：Patch Embedding**
- 输入图像 $224 \times 224 \times 3$
- 通过 Patch Embedding（4×4 conv, stride 4）得到 $56 \times 56 \times 96$ 的 feature map
- 相当于每个 patch 大小为 $4 \times 4$，共 $56 \times 56 = 3136$ 个 token

**步骤 2：窗口划分**
- 将 $56 \times 56$ 的 feature map 划分为 $7 \times 7$ 的窗口
- 窗口数量：$(56/7) \times (56/7) = 8 \times 8 = 64$ 个
- 每个窗口内有 $7 \times 7 = 49$ 个 token

**步骤 3：Focal Self-Attention**

对于每个窗口内的每个 token（即每个 query），计算注意力：

**3.1 细粒度局部注意力（Fine-grained Local Attention）**
- 局部窗口大小 $M = 7$
- 对窗口内的 $M \times M = 49$ 个 token 计算标准多头自注意力
- 每个 token 能看到窗口内所有其他 token 的精确信息
- 输出形状：$49 \times C$

**3.2 粗粒度全局注意力（Coarse-grained Global Attention）**
- 对整个 feature map 进行池化，生成不同分辨率的"粗 token"
- 池化策略：使用多个池化核大小，生成多尺度粗 token
  - 例如：$2 \times 2$ 池化 → $28 \times 28$ 个粗 token
  - $4 \times 4$ 池化 → $14 \times 14$ 个粗 token
  - $8 \times 8$ 池化 → $7 \times 7$ 个粗 token
- 对所有这些粗 token 计算注意力
- 输出形状：$(28^2 + 14^2 + 7^2) \times C = 1029 \times C$

**3.3 融合**
- 将细粒度注意力输出和粗粒度注意力输出沿 token 维度拼接
- 得到 $(49 + 1029) \times C = 1078 \times C$ 的注意力结果
- 通过线性投影映射回 $49 \times C$

**步骤 4：FFN 和残差连接**
- Layer Norm + MLP + 残差连接，与标准 Transformer 一致

**步骤 5：输出**
- 输出形状：$56 \times 56 \times 96$，与输入相同

### 关键设计选择

为什么选择多尺度池化而不是其他方式？
- 池化操作简单高效，没有额外参数
- 多尺度池化能捕获不同距离的全局信息
- 与卷积神经网络中的感受野扩展思路一致

为什么保留细粒度局部注意力？
- 局部细节对视觉任务至关重要（边缘、纹理等）
- 完全依赖粗粒度全局注意力会丢失细节信息
- 实验表明（表 4），去掉细粒度部分会导致准确率下降 1.2%

## 五、核心创新点

### 创新点 1：Focal Attention 机制

#### (a) 痛点与动机

Swin Transformer 的移位窗口虽然实现了跨窗口信息交互，但存在两个根本问题：
- **交互效率低**：需要两层才能让两个窗口的 token 交互（第一层在各自窗口，第二层通过移位后可能在同一窗口）
- **交互范围有限**：远处 token 需要多层才能交互，路径长度随距离线性增长

之前的工作如 Axial-Attention（Wang et al., 2020）尝试用轴向分解降低复杂度，但只能沿行和列分别做注意力，丢失了二维空间结构。LocalViT（Li et al., 2021）尝试用卷积增强局部信息，但全局信息仍然缺失。

#### (b) 方案细节

Focal Self-Attention 的核心公式：

$$\text{FocalAttention}(Q, K, V) = \text{Concat}(\text{head}_1, ..., \text{head}_h)W^O$$

其中每个 head 的计算为：

$$\text{head}_i = \text{Softmax}\left(\frac{Q_i K_i^T}{\sqrt{d_k}} + \text{PosBias}\right)V_i$$

这里的 $K$ 和 $V$ 由两部分拼接而成：
- $K_{\text{fine}}, V_{\text{fine}}$：局部窗口内的细粒度 key 和 value
- $K_{\text{coarse}}, V_{\text{coarse}}$：全局池化后的粗粒度 key 和 value

具体实现：

```python
# 伪代码示意
def focal_attention(query, key_value, window_size=7, pool_sizes=[2,4,8]):
    # 1. 细粒度部分：局部窗口内的 key/value
    local_kv = window_partition(key_value, window_size)  # [B, num_windows, window_size^2, C]
    local_attn = scaled_dot_product(query, local_kv)      # [B, num_windows, window_size^2, window_size^2]
    
    # 2. 粗粒度部分：多尺度池化后的 key/value
    coarse_kv = []
    for pool_size in pool_sizes:
        pooled = avg_pool(key_value, pool_size)           # 空间维度缩小 pool_size 倍
        coarse_kv.append(pooled.flatten(2))               # 拉平为 token 序列
    coarse_kv = concat(coarse_kv, dim=2)                  # [B, H*W, num_coarse_tokens, C]
    coarse_attn = scaled_dot_product(query, coarse_kv)    # 计算注意力
    
    # 3. 拼接注意力结果
    full_attn = concat([local_attn, coarse_attn], dim=-1)
    output = full_attn @ concat([local_v, coarse_v], dim=-2)
    return output
```

#### (c) 为什么有效

直觉理解：Focal Attention 的本质是**在注意力计算中引入空间先验**——近处需要精细建模，远处可以容忍粗糙建模。这符合视觉系统的生物特性，也符合自然图像的统计规律（局部相关性高，远处相关性低）。

实验证据（论文表 2）：
- Focal-Tiny（28.2M 参数）在 ImageNet 上达到 82.2% top-1 准确率
- 相比 Swin-Tiny（28.3M 参数）的 81.3%，提升了 0.9%
- 在计算量相近（FLOPs 约 4.5G）的情况下，效果明显更好

#### (d) 与 Related Work 的关系

Focal Attention 可以看作是以下工作的融合与改进：
- **Swin Transformer**（Liu et al., 2021）：继承了窗口注意力的高效性，但用多尺度池化替代了移位窗口
- **HaloNet**（Vaswani et al., 2021）：用局部注意力+全局下采样，但下采样方式固定
- **PVT**（Wang et al., 2021）：用空间缩减注意力（SRA）降低复杂度，但只做全局粗粒度

如果使用 Swin 的移位窗口方案，需要两层才能完成一次跨窗口交互，且交互路径长度随距离线性增长。Focal Attention 在一次前向中同时获得局部和全局信息，交互路径长度为 1。

#### (e) 如果去掉会怎样

论文表 4 的消融实验：
- 去掉粗粒度全局部分（只保留细粒度局部注意力）：准确率从 82.2% 降至 81.0%（-1.2%）
- 去掉细粒度局部部分（只保留粗粒度全局注意力）：准确率从 82.2% 降至 80.5%（-1.7%）
- 两者都去掉（退化为标准局部注意力）：准确率 80.3%（-1.9%）

这说明**局部和全局信息都是必要的**，且局部信息的重要性略高于全局信息（1.7% vs 1.2% 的下降）。

### 创新点 2：多尺度池化策略

#### (a) 痛点与动机

如果只用一个尺度的池化（例如只做 4×4 池化），粗 token 的粒度是固定的。对于近距离的远处 token（比如距离当前 token 8 个像素），4×4 池化后的粗 token 可能仍然包含较多细节；对于极远处的 token，4×4 池化可能还不够粗。

#### (b) 方案细节

论文使用多个池化核大小 $\{2, 4, 8\}$，生成不同分辨率的粗 token：

```
原始 feature map: 56×56
2×2 池化 → 28×28 粗 token（每个粗 token 覆盖 2×2 原始 patch）
4×4 池化 → 14×14 粗 token（每个粗 token 覆盖 4×4 原始 patch）
8×8 池化 →  7×7 粗 token（每个粗 token 覆盖 8×8 原始 patch）
```

所有尺度的粗 token 拼接在一起，参与注意力计算。

#### (c) 为什么有效

多尺度池化相当于为注意力机制提供了**不同"焦距"的视图**：
- 小池化核（2×2）：保留较多空间细节，适合"近处的远处"
- 大池化核（8×8）：高度抽象，适合"极远处"
- 多尺度组合：覆盖从近到远的完整范围

消融实验（表 4）显示，使用单尺度池化 vs 多尺度池化：
- 只用 2×2：81.5%
- 只用 4×4：81.7%
- 只用 8×8：81.3%
- 多尺度组合：82.2%

多尺度组合比最好的单尺度（4×4）高出 0.5%。

#### (d) 与 Related Work 的关系

多尺度池化在 CNN 中非常常见（如 SPPNet, He et al., 2015；FPN, Lin et al., 2017），但在 Transformer 的注意力计算中引入多尺度池化是本文的贡献。之前的工作如 PVT 使用单尺度空间缩减，CrossViT（Chen et al., 2021）使用双尺度 patch embedding，但没有在注意力内部做多尺度池化。

#### (e) 如果去掉会怎样

如果只保留一个尺度的池化，模型仍然能工作（81.5%-81.7%），但多尺度提供了额外的 0.5% 提升。更重要的是，多尺度池化让模型能够自适应不同距离的交互需求。

### 创新点 3：相对位置偏置的适配

#### (a) 痛点与动机

标准 ViT 使用绝对位置编码（Absolute Position Encoding），但 Swin 的实验表明，在窗口注意力中，**相对位置偏置（Relative Position Bias）** 比绝对位置编码效果更好。Focal Attention 同时包含局部和全局 token，需要设计适用于混合粒度注意力的位置编码。

#### (b) 方案细节

论文对细粒度和粗粒度部分分别使用相对位置偏置：

$$\text{Attention} = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}} + B_{\text{fine}} + B_{\text{coarse}}\right)V$$

其中：
- $B_{\text{fine}}$ 是局部窗口内的相对位置偏置，与 Swin 相同
- $B_{\text{coarse}}$ 是 query 与粗 token 之间的相对位置偏置

对于粗 token，其位置定义为池化后区域的中心位置。例如，一个 4×4 池化的粗 token 覆盖原始 feature map 的 4×4 区域，其位置取该区域的中心坐标。

#### (c) 为什么有效

相对位置偏置让注意力权重不仅依赖于内容（query 和 key 的相似度），还依赖于空间距离。这在视觉任务中很关键——两个距离很远的 patch 即使内容相似，也不应该有高注意力权重。

实验（表 4）显示，去掉位置偏置后准确率从 82.2% 降至 81.4%（-0.8%）。

#### (d) 与 Related Work 的关系

相对位置偏置最早在 Shaw et al.（2018）中提出，Swin Transformer 将其引入视觉 Transformer。本文的创新在于将其扩展到混合粒度的注意力场景。

#### (e) 如果去掉会怎样

如果去掉位置偏置，模型仍然可以训练，但需要更多的层来隐式学习空间关系，效果更差。

## 六、公式详解

### 公式 1：Focal Self-Attention 计算

$$\text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}} + B\right)V$$

#### (a) 符号定义

- $Q \in \mathbb{R}^{N \times d_k}$：query 矩阵，$N$ 为一个窗口内的 token 数（$N = M^2 = 49$），$d_k$ 为每个 head 的维度（$d_k = C/h$，$h$ 为 head 数）
- $K \in \mathbb{R}^{(N + N_c) \times d_k}$：key 矩阵，由细粒度 key（$N$ 个）和粗粒度 key（$N_c$ 个）拼接而成
- $V \in \mathbb{R}^{(N + N_c) \times d_v}$：value 矩阵，结构与 $K$ 相同，$d_v = d_k$
- $B \in \mathbb{R}^{N \times (N + N_c)}$：相对位置偏置矩阵
- $d_k$：缩放因子，防止 softmax 后的梯度消失

#### (b) 公式来源

这是标准 Scaled Dot-Product Attention（Vaswani et al., 2017）的扩展。原始公式为：

$$\text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

Focal Attention 的改动在于：
1. $K$ 和 $V$ 包含细粒度和粗粒度两部分
2. 加入了相对位置偏置 $B$

#### (c) 推导过程

以具体数值为例。假设 $h=8$，$C=96$，则 $d_k = 96/8 = 12$，$N=49$，$N_c = 28^2 + 14^2 + 7^2 = 1029$。

**步骤 1：计算 QK^T**

$$QK^T \in \mathbb{R}^{49 \times (49 + 1029)} = \mathbb{R}^{49 \times 1078}$$

计算量：$49 \times 1078 \times 12 = 633,864$ 次乘法

**步骤 2：缩放**

$$\frac{QK^T}{\sqrt{12}} \in \mathbb{R}^{49 \times 1078}$$

**步骤 3：加位置偏置**

$$S = \frac{QK^T}{\sqrt{12}} + B \in \mathbb{R}^{49 \times 1078}$$

其中 $B$ 的维度与 $S$ 相同。

**步骤 4：Softmax**

$$A = \text{Softmax}(S) \in \mathbb{R}^{49 \times 1078}$$

每行和为 1。

**步骤 5：加权求和**

$$\text{Attention} = AV \in \mathbb{R}^{49 \times 12}$$

#### (d) 直觉理解

这个公式可以分解为三部分：

1. **内容相似度**：$QK^T$ 衡量 query 和每个 key 的内容相似度
2. **空间先验**：$B$ 提供空间距离信息，让模型倾向于关注近处
3. **加权聚合**：Softmax 归一化后对 value 加权求和

有意思的是，粗粒度 key 虽然数量多（1029 个），但每个粗 token 代表一个区域的信息，因此注意力权重分布会比较分散——一个 query 可能同时对多个粗 token 有注意力，但每个权重较小。相比之下，细粒度 key（49 个）的注意力权重会更集中。

#### (e) 边界情况

- **当 $N_c = 0$ 时**（不进行池化），退化为标准局部窗口注意力，与 Swin 相同
- **当 $M = H = W$ 时**（窗口覆盖整个 feature map），退化为标准全局自注意力，与 ViT 相同
- **当池化核大小等于 feature map 大小时**，粗 token 只有一个，相当于计算全局平均池化后的注意力

### 公式 2：计算复杂度分析

$$\text{FLOPs} = 2hwC(4M^2 + \sum_{s \in S} \frac{hw}{s^2})$$

#### (a) 符号定义

- $h, w$：feature map 的高度和宽度
- $C$：通道数
- $M$：局部窗口大小（论文中为 7）
- $S$：池化核大小集合（论文中为 $\{2, 4, 8\}$）
- $s$：单个池化核大小

#### (b) 公式来源

这是对 Focal Self-Attention 计算量的近似估计。标准自注意力的 FLOPs 为 $2(hw)^2C$（假设 $d_k = C/h$），局部窗口注意力的 FLOPs 为 $2hw M^2 C$。

Focal Attention 的额外开销来自粗粒度部分：每个池化尺度 $s$ 产生 $\frac{hw}{s^2}$ 个粗 token，对这些粗 token 的计算量为 $2hw \cdot \frac{hw}{s^2} \cdot C = 2C\frac{(hw)^2}{s^2}$。

#### (c) 推导过程

以 $h=56, w=56, C=96, M=7, S=\{2,4,8\}$ 为例：

**细粒度部分**：
$$2 \times 56 \times 56 \times 96 \times (4 \times 49) = 2 \times 3136 \times 96 \times 196 = 118,013,952$$

**粗粒度部分**：
- $s=2$：$2 \times 3136 \times 96 \times \frac{3136}{4} = 2 \times 3136 \times 96 \times 784 = 472,055,808$
- $s=4$：$2 \times 3136 \times 96 \times \frac{3136}{16} = 2 \times 3136 \times 96 \times 196 = 118,013,952$
- $s=8$：$2 \times 3136 \times 96 \times \frac{3136}{64} = 2 \times 3136 \times 96 \times 49 = 29,503,488$

**总计**：
$$118,013,952 + 472,055,808 + 118,013,952 + 29,503,488 = 737,587,200 \approx 0.74 \text{ GFLOPs}$$

作为对比，标准全局自注意力的 FLOPs 为：
$$2 \times (56 \times 56)^2 \times 96 = 2 \times 3136^2 \times 96 = 1,887,936,512 \approx 1.89 \text{ GFLOPs}$$

Focal Attention 的计算量约为全局注意力的 39%。

#### (d) 直觉理解

复杂度公式中的 $4M^2$ 项来自细粒度部分（QKV 投影+注意力计算），$\sum \frac{hw}{s^2}$ 项来自粗粒度部分。当 $s$ 较大时（粗粒度），$\frac{hw}{s^2}$ 很小，计算量主要由细粒度部分主导。

有意思的是，粗粒度部分的计算量实际上比细粒度部分大（因为粗 token 数量多），但相比于全局注意力（$hw$ 项），粗粒度的 $\frac{hw}{s^2}$ 项要小得多。

#### (e) 边界情况

- 当 $S = \emptyset$ 时，FLOPs 降为 $8hwM^2C$，与 Swin 的窗口注意力相同
- 当 $M = h = w$ 且 $S = \emptyset$ 时，FLOPs 为 $8(hw)^2C$，与标准自注意力相同（但多了常数因子 4）
- 当 $S$ 包含很小的池化核（如 $s=1$）时，粗粒度部分退化为细粒度，总 FLOPs 接近全局注意力

### 实现细节

**1. Focal Attention 的前向计算**

```python
class FocalSelfAttention(nn.Module):
    def forward(self, x, H, W):
        B, N, C = x.shape
        # QKV 投影
        qkv = self.qkv(x).reshape(B, N, 3, self.num_heads, C // self.num_heads)
        q, k, v = qkv[:, :, 0], qkv[:, :, 1], qkv[:, :, 2]
        
        # 细粒度部分：窗口划分
        q_windows, k_windows, v_windows = self.window_partition(q, k, v, H, W)
        
        # 粗粒度部分：多尺度池化
        k_coarse, v_coarse = [], []
        for pool_size in self.pool_sizes:
            k_pooled = self.pool_k(k, pool_size)  # 空间池化
            v_pooled = self.pool_v(v, pool_size)
            k_coarse.append(k_pooled)
            v_coarse.append(v_pooled)
        k_coarse = torch.cat(k_coarse, dim=2)
        v_coarse = torch.cat(v_coarse, dim=2)
        
        # 拼接
        k_all = torch.cat([k_windows, k_coarse], dim=2)
        v_all = torch.cat([v_windows, v_coarse], dim=2)
        
        # 注意力计算
        attn = (q_windows @ k_all.transpose(-2, -1)) * self.scale
        # 加位置偏置
        attn = attn + self.pos_bias
        attn = attn.softmax(dim=-1)
        out = attn @ v_all
        
        # 恢复窗口形状
        out = self.window_reverse(out, H, W)
        return out
```

**2. 与论文描述的不同之处**

- 论文中说池化使用平均池化，代码中实际使用**可学习的卷积池化**（一个 2D 卷积层），这与 PVT 的做法类似
- 位置偏置的实现使用了**截断的相对位置编码**（截断到 [-7, 7] 范围），与 Swin 相同
- 代码中有一个 `use_conv_pooled` 参数，控制是否使用卷积池化（默认 True）

## 九、位置

### 前驱工作

1. **ViT**（Dosovitskiy et al., 2020）：证明了 Transformer 可以直接应用于图像分类，但计算复杂度高

3. **PVT**（Wang et al., 2021）：使用空间缩减注意力（SRA）降低复杂度，但只做全局粗粒度
4. **DeiT**（Touvron et al., 2021）：通过知识蒸馏和数据增强让 ViT 在中小数据上也能训练

### 同期竞品

1. **CSWin Transformer**（Dong et al., 2021）：使用十字形窗口注意力，沿水平和垂直方向分别计算
2. **CrossViT**（Chen et al., 2021）：使用双尺度 patch embedding，通过交叉注意力融合
3. **RegionViT**（Chen et al., 2021）：将图像划分为区域，在区域内和区域间分别计算注意力

### 后续影响

1. **Focal Transformer 的直接扩展**：论文的作者后续工作 Focal Modulation Networks（Yang et al., 2022）将 focal 思想从注意力扩展到调制（modulation）机制，进一步提升了效率
2. **被用作 backbone**：Focal Transformer 被用于多个下游任务，包括视频理解（Video Focal Transformer）、3D 点云处理等
3. **影响后续注意力设计**：多尺度注意力（Multi-scale Attention）成为后续工作（如 InternImage, 2023）的常见设计
4. **与 CNN 融合**：Focal 的多尺度池化思想被 ConvNeXt（Liu et al., 2022）等纯 CNN 工作借鉴，在 CNN 中引入类似的多尺度设计

## 十、局限

### 局限 1：计算量仍然较大

虽然比全局自注意力高效，但 Focal Attention 的粗粒度部分仍然需要计算大量粗 token（1029 个），相比 Swin 的窗口注意力（只计算 49 个 token），计算量增加了约 2 倍。

**在什么情况下重要**：在资源受限的场景（移动设备、边缘计算）下，Focal 的计算量可能仍然偏高。

**论文是否承认**：论文在复杂度分析中承认了这一点，但认为性能提升值得计算开销。

### 局限 2：池化策略的灵活性不足

池化核大小 $\{2,4,8\}$ 是手工设计的，对于不同分辨率、不同任务可能不是最优的。更理想的做法是让模型自动学习池化策略。

**是否有后续工作**：可变形注意力（Deformable Attention, 2022）和可学习池化（Learned Pooling）部分解决了这个问题。

### 局限 3：对高分辨率输入的扩展性

当输入分辨率很高时（如 1536×1536），即使使用池化，粗 token 的数量仍然很大。例如，1536×1536 的 feature map（下采样 4 倍后为 384×384），2×2 池化后仍有 192×192=36,864 个粗 token，计算量仍然可观。

**论文是否承认**：论文没有讨论极高分辨率的情况。

### 局限 4：位置编码的设计

相对位置偏置只考虑了空间距离，没有考虑语义距离。在某些任务中，两个语义相关的远处 patch 可能需要高注意力权重，但位置偏置会抑制这种交互。

## 十一、小结

Focal Self-Attention 解决了一个核心矛盾：**视觉 Transformer 需要全局信息来理解语境，但全局自注意力的计算代价太高**。它的解决方案优雅而直观——像人眼一样，近处看细节，远处看轮廓。

具体来说：
- **局部细粒度注意力**：在窗口内精确建模局部关系
- **全局粗粒度注意力**：通过多尺度池化提供全局语境
- **两者的融合**：在一次注意力计算中同时获得局部和全局信息

这个工作的意义在于，它展示了**在 Transformer 架构中引入空间先验**的有效性。纯粹的 Transformer 假设所有 token 都是等价的，但视觉信号有强烈的空间结构。Focal Attention 通过多尺度池化巧妙地利用了这种结构，既保持了 Transformer 的灵活性，又大幅提升了效率。

Focal Transformer 在 ImageNet 分类上比 Swin Transformer 高出 0.9%，在 COCO 检测上高出 1.0+ AP，证明了这种设计思路的有效性。它启发了后续一系列关于视觉 Transformer 中空间建模的工作，包括 Focal Modulation、InternImage 等，成为视觉 Transformer 发展中的一个重要节点。
