**论文：** [arXiv:2107.00641](https://arxiv.org/abs/2107.00641) · Jianwei Yang, Chunyuan Li, Pengchuan Zhang, Xiyang Dai, Bin Xiao, Lu Yuan, Jianfeng Gao · Microsoft Research · 2021

**技术分支：** B2 — 视觉 backbone 层级化 / B2-4 — hierarchical backbone 范式

**代码：** [microsoft/Focal-Transformer](https://github.com/microsoft/Focal-Transformer)

## 一、先搞清楚"问题在哪里"

2021 年上半年，Swin Transformer 刚刚证明了"层级式局部窗口注意力"可以成为通用视觉 backbone，在 ImageNet 分类、COCO 检测、ADE20K 分割上全面超越了 CNN 基线。

Swin 的核心创新是：**用不重叠的局部窗口替代全局自注意力**。每个 token 只在 7×7 的窗口内做 attention，复杂度从 $O(N^2)$ 降到 $O(N)$。然后用"移位窗口"（Shifted Window）机制在相邻层之间错开窗口划分，让信息跨窗口流动。

这个方案漂亮地解决了 ViT 的两个核心矛盾——复杂度太高、缺乏多尺度特征。Swin 也因此成为了 2021 年 CVPR 的最佳论文。

但 Swin 的移位窗口有个结构性缺陷：**跨窗口信息交互是间接的**。

一个处在窗口 A 左上角的 token，想要和窗口 B 右下角的 token 交互，需要经过多层累积：第一层看不到，第二层通过移位可能进入同一窗口，第三层才是真正的有效交互。这种间接交互路径随距离线性增长——远处的 token 需要更多层才能通信。

换个角度看：**Swin 的每次注意力计算，每个 token 只能看到窗口内的 49 个 token（7×7），不到总 token 数的 2%（对于 56×56 的 feature map）**。感受野被窗口大小硬性绑定了。

Focal Transformer 要解决的问题很明确：**能否在一次注意力计算中，同时获得局部细节和全局语境？**

---

## 二、旧路线的真正问题

要理解 Focal 为什么比 Swin 更好，得先搞清楚 Swin 的移位窗口方案在哪儿不舒服。

### 2.1 Swin 的"两层才能交互"问题

Swin 实现跨窗口信息流动的方式是 W-MSA 和 SW-MSA 交替排列：

```
层 1 (W-MSA):  每个 token 只看自己 7×7 窗口内的 49 个 token
层 2 (SW-MSA): 窗口偏移半个窗口，token 所在窗口改变，看到新的 49 个 token
```

这意味着：**一个 token 要和距离较远的 token 交互，至少需要经过两层。**

量化分析一下：

假设 feature map 大小为 $H \times W$，窗口大小为 $M$（Swin 中 M=7）。在阶段 1（56×56 分辨率），8×8=64 个窗口。一个窗口内的 token 到相邻窗口中心的 token，最短路径需要经过 1 层（通过 SW-MSA 进入同一窗口）。但到对角方向的窗口，需要经过 2 层。

对于更深的层（阶段 3 和 4，特征图更小但感受野更大），窗口数量变少，跨窗口交互路径也相应缩短。但关键问题是：**这种"间接交互"不是由注意力机制直接完成的，而是依赖架构的累积效应。**

### 2.2 移位窗口的实现复杂度

SW-MSA 的实现并不简单。窗口偏移后，特征图的划分不再均匀——原本均匀的 4 个窗口变成了 9 个大小不等的窗口。为了在 GPU 上高效计算，Swin 使用了 cyclic shift（循环移位），把右下角的碎片移动到左上角，凑成规则窗口，然后通过 attention mask 屏蔽掉不该注意力计算的位置。

cyclic shift + mask 在工程层面可行，但：

- 需要额外的 mask 数据结构和计算
- cyclic shift 的 memory copy 操作增加了延迟
- 窗口数量从 $(\frac{H}{M})^2$ 变成 $(\frac{H}{M} + 1)^2$，计算量增加约 50%

### 2.3 窗口大小不可扩展

Swin 的窗口大小 M=7 是固定的。M 决定了感受野的物理大小——不管模型规模多大，每个 token 每次 attention 最多看到 49 个 token。增加层数可以间接扩大感受野，但每层之间的交互效率没有提升。

对于需要全局理解的任务（如实例分割中的实例间关系、全景分割中的背景建模），Swin 这种"等待多层积累"的方式既慢又不直接。

### 2.4 并行工作的尝试

同一时期，其他工作也在尝试不同的路径：

- **PVT**（Wang et al., 2021）：用 Spatial Reduction Attention（SRA）做稀疏全局注意力，复杂度仍保留 $O(N^2)$ 的平方项
- **HaloNet**（Vaswani et al., 2021）：用固定下采样做全局上下文，但下采样粒度不可控
- **Axial-Attention**（Wang et al., 2020）：分解为行和列的一维注意力，丢失二维结构信息

这些方案各有不足。PVT 的计算量随分辨率增长仍然很快；HaloNet 的下采样是固定的，无法自适应不同区域的交互需求；Axial-Attention 丢失了空间结构。

### 2.5 小结：需要一个"一步到位"的方案

理想的方案应该满足：

1. **局部精细** — 近处的 token 要能细粒度交互，保留空间细节
2. **全局覆盖** — 远处的信息也要能捕获，不依赖多层累积
3. **计算可控** — 复杂度 $O(N)$ 或接近 $O(N)$
4. **硬件高效** — 内存访问模式友好，避免散乱的 gather 操作

Focal Self-Attention 的方案是：**近处用细粒度注意力，远处用粗粒度注意力，一次计算同时获得两者。**

## 三、这篇论文的核心思路

**核心思路：让每个查询 token 在一次注意力计算中，同时看到两种粒度的信息——局部区域的每个 patch（细粒度）和全局区域的聚合 patch（粗粒度）。**

打个比方：人眼看一幅画时，视线焦点处的细节是清晰的（fine-grained），余光扫到的区域是模糊的（coarse-grained），但模糊的余光仍然提供了画布的布局和整体颜色信息。Focal Self-Attention 就是把这个"焦点-余光"机制模型化了。

具体来说，Focal Self-Attention 把注意力计算分成多个粒度层级（Focal Levels）：

```
等级 1（细粒度）: 局部窗口内的每个 patch，不做任何池化
等级 2（中粒度）: 稍远的区域，2×2 池化后得到粗 token
等级 3（粗粒度）: 更远甚至全图范围，4×4 或 8×8 池化后得到更粗的 token
```

所有粒度的 token 拼接在一起，作为 key 和 value 参与标准的多头自注意力计算。查询 token 的注意力权重会自然分配给不同粒度的 token——对于需要精密定位的任务（如目标检测中的 bounding box 预测），细粒度 token 的权重会更高；对于需要全局语境的任务（如语义分割中的类别预测），粗粒度 token 的权重会更高。

从谱系上看，这条路线是从 Swin 出发的改进：

```text
ViT（全局 attention, 单尺度, 2020）
   ↓
Swin Transformer（窗口 attention + 移位, 层级化, 2021）
   ↓
Focal Transformer（窗口 attention + 多粒度, 层级化, 2021） ← 本篇
   ↓
Focal Modulation（从 attention 到调制机制, 2022）
```

Focal 和 Swin 的核心区别在于：Swin 依赖"层间间接交互"来扩大感受野，Focal 在"层内直接做全局交互"——但通过池化把计算量控制在可接受范围。

## 四、整体结构与 forward 流程

### 4.1 整体架构（Focal-Tiny 为例）

```
输入图像: 224×224×3
   │
   ▼
Patch Embedding（4×4 conv, stride 4）
   → 56×56×96 feature map
   │
   ▼
Stage 1: [Focal Block × 2]   56×56, 维度 96
   │
   ▼
Patch Embedding（2×2 conv, stride 2）
   │
   ▼
Stage 2: [Focal Block × 2]   28×28, 维度 192
   │
   ▼
Patch Embedding（2×2 conv, stride 2）
   │
   ▼
Stage 3: [Focal Block × 6]   14×14, 维度 384
   │
   ▼
Patch Embedding（2×2 conv, stride 2）
   │
   ▼
Stage 4: [Focal Block × 2]   7×7, 维度 512
   │
   ▼
分类头（LayerNorm + Global Avg Pool + Linear）
```

与 Swin 完全相同的 4 级金字塔结构（Swin-T 也是 2-2-6-2 层数配置），唯一的区别是每个 Stage 内部的注意力计算从 Swin 的"窗口注意力 + 移位窗口交替"替换为"Focal Self-Attention"。

Focal Transformer 没有移位窗口。每个 Focal Block 的结构是：

```
Input
   │
   ▼
LayerNorm
   │
   ▼
Focal Self-Attention（替代 Swin 的 W-MSA / SW-MSA）
   │
   ▼
残差连接 (+)
   │
   ▼
LayerNorm
   │
   ▼
MLP (GELU, 4× expansion)
   │
   ▼
残差连接 (+)
   │
   ▼
Output
```

### 4.2 Focal Self-Attention 的 forward 流程

以 Stage 1（56×56×96）为例，一个 Focal Block 的注意力计算分以下步骤：

**步骤 1：子窗口池化（Sub-window Pooling）**

对输入 feature map $x \in \mathbb{R}^{56 \times 56 \times 96}$，按 $L=3$ 个 focal level 分别做子窗口池化：

- Level 1（细粒度）：子窗口大小 $s_w^1 = 1$（不池化，保留原始精度）
  - 输出：$56 \times 56 \times 96$（不变）
- Level 2（中粒度）：子窗口大小 $s_w^2 = 2$
  - 把 56×56 划分为 28×28 个 2×2 子窗口
  - 每个子窗口通过线性层池化为 1 个 token
  - 输出：$28 \times 28 \times 96$
- Level 3（粗粒度）：子窗口大小 $s_w^3 = 4$
  - 把 56×56 划分为 14×14 个 4×4 子窗口
  - 输出：$14 \times 14 \times 96$

池化的方式不是简单平均或最大池化，而是使用一个**线性层**对子窗口内的 token 做加权求和。这个线性层的参数跨子窗口共享，引入的参数量很小（子窗口大小平方 × 输入维度 → 输出维度，对于 $s_w=2$ 是 $4 \times 96 \rightarrow 96$，约 384 个参数，几乎可忽略）。

**步骤 2：窗口划分**

将原始 feature map（Level 1）划分为不重叠的窗口，窗口大小 $s_p = 7$：

- 56×56 → 8×8 个窗口，每个窗口 7×7=49 个 token

查询（Query）只从 Level 1 的原始 token 中生成。Key 和 Value 从所有 3 个 level 的 token 中提取。

**步骤 3：为每个窗口提取多粒度 surroundings**

以中间的一个窗口（如图 4 中的蓝色窗口）为例，提取其周围 token：

- Level 1 区域：$s_r^1 \times s_r^1$（如 8×8）个细粒度 token
  - 从原始 feature map 中提取窗口附近的 8×8=64 个 token
- Level 2 区域：$s_r^2 \times s_r^2$（如 6×6）个中粒度 token
  - 从 Level 2 池化后的 28×28 特征图中提取对应区域附近的 6×6=36 个 token
- Level 3 区域：$s_r^3 \times s_r^3$（如 4×4）个粗粒度 token
  - 从 Level 3 池化后的 14×14 特征图中提取 4×4=16 个 token

三个 level 的 token 数量合计：$s = 64 + 36 + 16 = 116$ 个 token。

相比于标准注意力中每个 query 需要和 $56 \times 56 = 3136$ 个 token 计算注意力，Focal Attention 只需要和 116 个 token 计算——不到原始数量的 4%。

**步骤 4：注意力计算**

对窗口内的 49 个 query token，和拼接后的 116 个 key/value token 计算标准 Scaled Dot-Product Attention：

$$\text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}} + B\right)V$$

其中 $B$ 是每个 level 独立的相对位置偏置。

**步骤 5：窗口重组**

将每个窗口的输出拼回原始的 56×56 形状。

### 4.3 关键设计选择

**为什么用子窗口池化（线性层）而不是平均池化？**

- 线性层是可学习的，能根据任务调整池化策略
- 平均池化会丢失空间细节，线性层保留了通道级别的权重分配能力
- 参数量极低（子窗口大小相关的线性层），几乎不影响模型大小

**为什么窗口内 token 共享相同 surroundings？**

- 每个 token 单独提取 surroundings 会导致大量重复的 gather 操作，内存开销巨大
- 窗口内的 token 位置接近，它们的 surroundings 高度重叠
- 以窗口为单位提取，大幅减少了 gather 次数

**为什么保留 level 间的重叠区域？**

- 严格来说，细粒度的 inner level 区域应该从 outer level 中排除
- 但作者选择保留重叠，让模型可以通过注意力权重自动决定使用哪个层级的特征
- 实验表明，保留重叠比不保留效果好

## 五、核心创新点

### 创新点 1：Focal Self-Attention 机制

#### (a) 痛点与动机

Swin 的移位窗口让跨窗口信息交互需要多层积累。以一个处在 Stage 1（56×56）的 query token 为例：

- Swin 层 1（W-MSA）：看到 49 个局部 token
- Swin 层 2（SW-MSA）：移位后看到另一个 49 个 token（可能跨越原窗口边界）
- 要看到距离 28 个像素之外的 token，需要经过约 4-6 层

问题是：**Transformer 的设计初衷是"每次前向都能全局交互"。如果每次只能看到局部，那 Transformer 和 CNN 的区别在哪？**

Focal Self-Attention 的答案是：**每次仍然只和部分 token 交互，但通过多粒度覆盖，用更少的 token 达到更大的等效感受野。**

#### (b) 方案细节

Focal Self-Attention 的核心公式与标准注意力相同，只是 key 和 value 的来源不同：

$$Q = f_q(x^1), \quad K = \{f_k(x^1), f_k(x^2), ..., f_k(x^L)\}, \quad V = \{f_v(x^1), f_v(x^2), ..., f_v(x^L)\}$$

其中 $x^1$ 是原始特征图（细粒度），$x^2, ..., x^L$ 是池化后的特征图（粗粒度）。

查询只从细粒度特征图中生成（保留定位精度）。Key 和 Value 从所有 level 的特征图中提取。

```python
# Focal Self-Attention 伪代码
def focal_self_attention(x, H, W, pool_sizes=[2,4]):
    """
    x: [B, H*W, C] — 展平的 feature map
    pool_sizes: 子窗口大小列表，对应 focal levels
    """
    B, N, C = x.shape
    
    # 1. 多尺度池化
    pooled_maps = [x]  # level 1: 原始分辨率
    for s in pool_sizes:
        # 将 feature map 划分为 s×s 子窗口，用线性层池化
        x_pooled = sub_window_pool(x, s, H, W)  # 空间缩小 s 倍
        pooled_maps.append(x_pooled)
    
    # 2. QKV 投影（对每个 level 分别投影）
    Q = q_proj(pooled_maps[0])      # 只用 level 1 生成 Q
    K_list = [k_proj(m) for m in pooled_maps]
    V_list = [v_proj(m) for m in pooled_maps]
    
    # 3. 窗口划分
    Q_windows = window_partition(Q, H, W)        # [B, num_windows, M^2, C]
    K_windows = [window_partition(k, H, W) for k in K_list]
    V_windows = [window_partition(v, H, W) for v in V_list]
    
    # 4. 为每个窗口收集 surroundings
    K_surround, V_surround = [], []
    for l in range(L):  # L = len(pool_sizes) + 1
        Ki, Vi = extract_surroundings(
            K_windows[l], V_windows[l],
            region_size=sr[l],  # 每个 level 的周围区域大小
            window_size=sp      # 窗口大小
        )
        K_surround.append(Ki)
        V_surround.append(Vi)
    
    K_all = torch.cat(K_surround, dim=-2)  # 拼接所有 level 的 key
    V_all = torch.cat(V_surround, dim=-2)  # 拼接所有 level 的 value
    
    # 5. 标准注意力计算
    attn = (Q_windows @ K_all.transpose(-2, -1)) / sqrt(d_k)
    attn = attn + position_bias              # 相对位置偏置
    attn = attn.softmax(dim=-1)
    out = attn @ V_all
    
    # 6. 恢复空间形状
    out = window_reverse(out, H, W)
    return out
```

#### (c) 为什么有效

**理论分析：** Focal Self-Attention 在一次计算中的等效感受野远大于 Swin 的窗口注意力。

以 Stage 1 为例（56×56 feature map）：

| 方案 | 每次交互的 token 数 | 等效感受野 |
|------|-------------------|-----------|
| 标准全局注意力 | 3136 | 56×56（全图） |
| Swin W-MSA | 49 | 7×7 |
| Swin SW-MSA | 49 | 14×14（2 层累积） |
| Focal（3 level） | 116 | 56×56（全图） |

更直观的对比图是论文的 Figure 3：随着使用的 token 数量增加，标准注意力的感受野按 $O(N)$ 增长（因为每个 token 加一个位置），Focal 的感受野按 $O(N \times \text{pool\_factor})$ 增长（因为粗粒度 token 代表更大区域）。

**实验证据：**
- Focal-Tiny（29.1M, 82.2%）比 Swin-Tiny（28.3M, 81.3%）高 0.9%
- Focal-Small（51.1M, 83.5%）比 Swin-Small（50M, 83.0%）高 0.5%
- 参数量和 FLOPs 几乎相同，性能提升完全来自注意力机制的改进

#### (d) 与 Related Work 的关系

Focal Self-Attention 在注意力设计谱系中的位置：

```
全局注意力 (ViT) — 每个 token 和所有 token 交互
    ↓
空间缩减注意力 (PVT-SRA) — 下采样后再做粗粒度全局注意力
    ↓
局部窗口注意力 (Swin W-MSA) — 只和窗口内 token 交互
    ↓  ← 以上工作要么全局但粗糙，要么精细但局部
Focal Attention (本篇) — 局部精细 + 全局粗糙，两者兼得
```

与 PVT 的区别：PVT 的空间缩减注意力是做完下采样后做全局注意力，所有区域同粒度。Focal 是**不同距离不同粒度**，更符合自然图像的统计规律。

与 HaloNet 的区别：HaloNet 用固定的下采样 stride 做全局上下文，下采样方式不可学习。Focal 使用可学习的线性层做子窗口池化。

与 Swin 的区别：Focal 不移位窗口，而是通过多粒度池化一次性覆盖全图。

#### (e) 如果去掉会怎样

从论文表 4 的消融实验看：

- 去掉粗粒度部分（只保留 Level 1 局部注意力）：82.2% → 81.0%（-1.2%）
- 去掉细粒度部分（只保留多粒度池化后的粗注意力）：82.2% → 80.5%（-1.7%）
- 两者都去掉（相当于 Swin 而不用移位）：82.2% → 80.3%（-1.9%）

这说明：
- 细粒度局部信息（贡献 1.7%）比粗粒度全局信息（贡献 1.2%）更重要
- 但两者缺一不可，组合起来效果更好

### 创新点 2：子窗口池化（Sub-window Pooling）

#### (a) 痛点与动机

远程信息的粗粒度表示需要一个池化策略来把多个 token 聚合成一个粗 token。最直接的做法是平均池化或最大池化，但问题是：**固定池化策略不能根据不同区域的内容自适应调整。**

例如，一个子窗口包含背景（天空）和前景（飞鸟），平均池化会把两者混在一起，丢失关键语义信息。可学习的池化可以让模型自己决定如何聚合。

#### (b) 方案细节

对于每个子窗口大小 $s_w^l$，论文定义了一个线性层 $f_p^l$：

$$x^l = f_p^l(\text{Reshape}(x)) \in \mathbb{R}^{\frac{M}{s_w^l} \times \frac{N}{s_w^l} \times d}$$

具体操作：
1. 把 feature map 划分为 $(M/s_w^l) \times (N/s_w^l)$ 个子窗口
2. 每个子窗口内展平为 $(s_w^l \times s_w^l)$ 个 token
3. 通过线性层 $f_p^l$（输入维度：$s_w^l \times s_w^l$，输出维度：1）对子窗口内的 token 做加权聚合
4. 每个子窗口得到一个粗 token

注意这个线性层不改变特征维度 $d$——它只对空间维度做池化。

池化层的参数量：对于 Stage 1（$d=96$），Level 2 的子窗口大小 $s_w^2=2$，线性层参数为 $(4, 1)$，共 $4 \times 96 = 384$ 个参数。Level 3 的 $s_w^3=4$，参数为 $(16, 1)$，共 $16 \times 96 = 1536$ 个参数。相对于 Focal-Tiny 的 29.1M 参数，这两个池化层引入的参数量（<2K）几乎可以忽略不计。

#### (c) 为什么有效

可学习的池化比固定池化（平均池化）有显著优势：

- **自适应聚合**：可以学习到"对某些通道取平均、对某些通道取最大值、对某些通道取加权和"等不同策略
- **任务自适应**：不同任务对池化的需求不同——分类任务可能需要保留更多的类别相关特征，检测任务需要保留空间定位信息
- **网络深度自适应**：浅层可能需要保留边缘细节，深层可能更需要语义信息

论文的消融实验中，可学习池化比平均池化好大约 0.3-0.4%。

#### (d) 与 Related Work 的关系

子窗口池化与 CNN 中的池化层（MaxPool, AvgPool）类似，但有两个关键区别：
- 它是**可学习的**（用线性层），不是固定的
- 它在**注意力计算内部**使用，不是独立的下采样操作

与 ViT 中的 class token 类似的思路：ViT 的 class token 通过注意力机制"池化"全局信息，Focal 的子窗口池化通过线性层聚合局部区域信息。

#### (e) 如果去掉会怎样

如果使用平均池化替换可学习池化，论文报告的准确率大约下降 0.3%。这个差距相对较小——说明池化方式本身不是 Focal Attention 最重要的贡献，多粒度覆盖才是。

### 创新点 3：窗口级注意力 + 相对位置偏置

#### (a) 痛点与动机

在原始的 Focal Self-Attention 设计中，每个 token 独立提取其 surroundings。这在概念上很干净，但实现起来有严重的内存效率问题：token 的 surroundings 区域高度重叠，如果每个 token 独立 gather，同一块区域会被反复复制多次。

例如，两个相邻 token 距离 1 个像素，它们的 surroundings（比如 8×8 区域）重叠了 56 个 token 中的 49 个。如果每个 token 独立 gather，这 49 个重叠 token 会被重复读取——浪费内存带宽。

#### (b) 方案细节

论文采用 Swin 的窗口划分思路：**先划分窗口，然后以窗口为单位提取 surroundings。**

给定 $s_p \times s_p$ 的窗口，窗口内所有 $s_p^2$ 个 token 共享相同的 surroundings 集合。对于每个窗口 $i$：

$$K_i = \{K_i^1, ..., K_i^L\}, \quad V_i = \{V_i^1, ..., V_i^L\}$$

其中 $K_i^l$ 是窗口 $i$ 在 level $l$ 上的 surroundings key，区域大小为 $s_r^l \times s_r^l$。

相对位置偏置也按 level 独立计算：
- Level 1（细粒度）：与 Swin 相同，在窗口内 $(2s_p - 1) \times (2s_p - 1)$ 的范围内计算
- Level 2+（粗粒度）：考虑到粗粒度 token 和细粒度 query 之间的尺度差异，需要计算两个 token 之间的实际空间距离（以原始 feature map 的像素为单位）

#### (c) 为什么有效

窗口级 attention 在两个层面提升了效率：
1. **减少内存访问次数**：一个窗口内的 $s_p^2$ 个 token 共享 surroundings，gather 操作减少为原来的 $1/s_p^2$
2. **提高计算并行度**：窗口之间的注意力计算可以完全独立并行（batch 维度）

窗口大小 $s_p$ 的选择需要在计算效率和注意力效果之间权衡。论文默认 $s_p=7$（与 Swin 一致），因为 7×7 是一个在硬件上效率较高的窗口大小（49 个 token 的注意力矩阵适中，且不触发 GPU 的共享内存限制）。

#### (d) 与 Related Work 的关系

窗口级注意力 + 相对位置偏置的设计直接继承自 Swin Transformer。Focal 的创新在于：
1. 不移位窗口（简化实现，减少 mask 开销）
2. 为不同 level 设计不同的位置偏置（处理粗细粒度的空间关系）

与 Swin 的关键区别：Focal 不需要 cyclic shift 和 attention mask，因为不需要移位——所有全局信息通过多粒度池化直接获得。

#### (e) 如果去掉会怎样

如果去掉相对位置偏置（表 4）：82.2% → 81.4%（-0.8%）。位置偏置为注意力提供了空间先验，对于视觉任务尤其重要。

如果去掉窗口级操作（每个 token 独立做 Focal Attention）：显存和速度会显著下降，论文估计大约慢 2-3 倍。但准确率可能略有提升（每个 token 的 surroundings 位置更精确）。

## 六、公式详解

### 公式 1：子窗口池化

$$x^l = f_p^l(\hat{x}) \in \mathbb{R}^{\frac{M}{s_w^l} \times \frac{N}{s_w^l} \times d}, \quad \hat{x} = \text{Reshape}(x) \in \mathbb{R}^{(\frac{M}{s_w^l} \times \frac{N}{s_w^l} \times d) \times (s_w^l \times s_w^l)}$$

#### (a) 符号定义

- $x \in \mathbb{R}^{M \times N \times d}$：输入 feature map，$M \times N$ 空间大小，$d$ 特征维度
- $s_w^l$：level $l$ 的子窗口大小
- $f_p^l$：可学习的池化线性层
- $\hat{x}$：reshape 后的中间结果，每行对应一个子窗口内的所有 token

#### (b) 公式来源

这是标准"unfold → linear projection → fold"操作：

1. **Unfold**：把 $M \times N$ 的 feature map 展开为 $\frac{M}{s_w^l} \times \frac{N}{s_w^l}$ 个子窗口，每个子窗口包含 $s_w^l \times s_w^l$ 个 token
2. **Project**：线性层把每个子窗口的 $s_w^l \times s_w^l$ 个 token 投影为 1 个
3. **Fold**：将投影结果重新排列为 $\frac{M}{s_w^l} \times \frac{N}{s_w^l}$ 的粗特征图

#### (c) 推导过程

以 $M=N=56, d=96, s_w^2=2$ 为例：

1. 空间划分：56×56 → 28×28 个子窗口，每个子窗口 2×2=4 个 token
2. Reshape：$(56 \times 56 \times 96) \rightarrow (28 \times 28 \times 96, 4)$
   - 解释：把每个子窗口的 4 个 token 展平到最后一维
3. 线性投影：$f_p^2: \mathbb{R}^4 \rightarrow \mathbb{R}^1$
   - 这是个 $(4, 1)$ 的线性层，作用于每个子窗口的展平向量
   - 参数量：$4 \times 96 + 96 = 480$（可以进一步优化为 depthwise 操作）
4. 输出形状：$28 \times 28 \times 96$

#### (d) 直觉理解

这等价于一个轻量化的 **learnable spatial pooling**。和 CNN 中的平均池化（核大小 2×2, stride 2）相比，区别在于：
- 平均池化给每个位置等权重（1/4）
- 可学习池化让模型学习分配权重——某些像素的重要性可能更高

比如在图像边缘，靠近边缘的像素可能获得更高的池化权重，因为边缘信息对物体识别更重要。

#### (e) 边界情况

- **当 $s_w^l = 1$ 时**：池化层退化为恒等映射，$x^l = x$
- **当 $s_w^l = M = N$ 时**：全局池化，得到 1×1 的 global token
- **当 $s_w^l$ 不能被 $M$ 整除时**：需要 padding，但论文使用的 multi-scale 架构保证 $M, N$ 始终被 $s_w^l$ 整除

### 公式 2：Focal Self-Attention 计算

$$\text{Attention}(Q_i, K_i, V_i) = \text{Softmax}\left(\frac{Q_i K_i^T}{\sqrt{d}} + B\right)V_i$$

#### (a) 符号定义

- $Q_i \in \mathbb{R}^{s_p^2 \times d_k}$：窗口 $i$ 内的所有 query token
- $K_i \in \mathbb{R}^{s \times d_k}$：窗口 $i$ 的 surroundings（所有 level 的 key 拼接）
  - $s = \sum_{l=1}^L (s_r^l)^2$，所有 level 的 token 总数
- $V_i \in \mathbb{R}^{s \times d_v}$：对应的 value
- $B \in \mathbb{R}^{s_p^2 \times s}$：相对位置偏置矩阵
- $\sqrt{d}$：缩放因子

#### (b) 公式来源

这是标准 Scaled Dot-Product Attention 的窗口版本。假设 $s_p=7, s_r^1=8, s_r^2=6, s_r^3=4$：

- $s = 8^2 + 6^2 + 4^2 = 64 + 36 + 16 = 116$
- 注意力矩阵 $Q_i K_i^T \in \mathbb{R}^{49 \times 116}$，形状合理

#### (c) 推导过程

**第一步：计算注意力分数**

$$S = \frac{Q_i K_i^T}{\sqrt{d}} \in \mathbb{R}^{s_p^2 \times s}$$

$S$ 的每个元素 $S_{jk}$ 表示窗口内第 $j$ 个 query 和第 $k$ 个 key 的相似度。

**第二步：加位置偏置**

$$\tilde{S} = S + B \in \mathbb{R}^{s_p^2 \times s}$$

$B_{jk}$ 表示 query $j$ 和 key $k$ 之间的相对位置偏置。对于不同 level 的 key，偏置参数不同。

**第三步：Softmax 归一化**

$$A_{jk} = \frac{\exp(\tilde{S}_{jk})}{\sum_{k'=1}^s \exp(\tilde{S}_{jk'})}$$

**第四步：加权求和**

$$\text{Output}_j = \sum_{k=1}^s A_{jk} V_{ik}$$

#### (d) 直觉理解

Focal Attention 的智能之处在于：**注意力权重会自动学习到"什么时候看重局部，什么时候看重全局"。**

对于需要精确定位的信息（如物体的边界），query 会给细粒度 key 更高的权重。对于需要语境理解的信息（如判断类别），query 会给粗粒度 key 更高的权重。

可视化（论文未提供，但从类 Swin 注意力模式可以推断）：分类任务中，粗粒度全局 token 的权重在深层更大；检测任务中，细粒度局部 token 在浅层更大。

#### (e) 边界情况

- **当 $L=1$ 时**：退化为标准局部窗口注意力（类似 Swin 的 W-MSA）
- **当 $s_r^1 = M/g$ 且 $L=1$ 时**：退化为全局注意力（因为窗口大小覆盖全图）
- **当 $s_w^2 = \cdots = s_w^L = 0$ 时**（不池化）：等价于扩大局部窗口

### 公式 3：计算复杂度分析

Focal Self-Attention 的 FLOPs 估计：

$$\text{FLOPs} = \mathcal{O}\left(HW\left(4M^2 + \sum_{l=1}^L \frac{HW}{(s_w^l)^2} \cdot (s_r^l)^2\right)d\right)$$

#### (a) 符号定义

- $H, W$：feature map 的高度和宽度
- $M = s_p$：窗口大小
- $d$：特征维度
- $s_w^l$：level $l$ 的子窗口池化大小
- $s_r^l$：level $l$ 的周围区域大小（以 pooled feature map 上的 token 为单位）

#### (b) 复杂度推导

**标准全局注意力：**
$$\text{FLOPs} = 2(HW)^2 d$$

**Swin 窗口注意力（4 项对应 QKV 投影 + 注意力计算，估计）：**
$$\text{FLOPs} = 2HW(4M^2)d$$

**Focal 注意力：**
细粒度部分（Level 1）：
$$2HW \cdot (s_r^1)^2 \cdot d$$

粗粒度部分（Level 2+）：
$$2HW \cdot \sum_{l=2}^L \frac{HW}{(s_w^l)^2} \cdot (s_r^l)^2 \cdot d$$

其中 $\frac{HW}{(s_w^l)^2}$ 是池化后的 token 总数，$(s_r^l)^2$ 是每个窗口的 surroundings 大小。

#### (c) 数值示例

Stage 1（$H=W=56, d=96, M=7, L=3$）：

**细粒度 Level 1**（$s_w^1=1, s_r^1=8$）：
$$2 \times 56 \times 56 \times 64 \times 96 = 38,535,168$$

**Level 2**（$s_w^2=2, s_r^2=6$）：
$$2 \times 56 \times 56 \times \frac{56 \times 56}{4} \times 36 \times 96 / 3136$$
简化计算：每个窗口在 Level 2 上提取 36 个 token，共 64 个窗口
$$= 2 \times 64 \times 36 \times 49 \times 96 = 21,692,416$$

**Level 3**（$s_w^3=4, s_r^3=4$）：
$$2 \times 64 \times 16 \times 49 \times 96 = 9,641,216$$

**总计**：$38.5 + 21.7 + 9.6 \approx 69.9\text{M FLOPs} \approx 0.07\text{ GFLOPs}$

作为对比，标准全局注意力需要：
$$2 \times (56 \times 56)^2 \times 96 \approx 1.89\text{ GFLOPs}$$

Focal 约为全局的 3.7%。

#### (d) 直觉理解

Focal 的复杂度由两部分组成：
1. **细粒度项** $4M^2$：与 Swin 相同，与 feature map 大小无关，只和窗口大小有关
2. **粗粒度项** $\sum \frac{HW}{s_w^2} \cdot s_r^2$：与 feature map 大小相关，但池化因子 $s_w^2$ 起到了折扣作用

粗粒度项实际上比细粒度项更大（因为粗 token 数量多），但相比于全局注意力的 $HW$ 项，粗粒度的 $\frac{HW}{s_w^2}$ 项小了 $s_w^2$ 倍。

#### (e) 边界情况

- **当 $s_w^l \to \infty$ 时**：池化倍数极大，粗粒度项趋近于 0，复杂度 = Swin
- **当 $s_w^l = 1, L=1$ 时**：退化为全局注意力
- **当 $L$ 很大但 $s_w^l$ 很小时**：粗粒度项膨胀，复杂度可能接近全局注意力

论文的实验配置（$L=3, s_w=\{1,2,4\}$）在复杂度和性能之间取得了良好的平衡。

### 8.1 关键实现细节

**子窗口池化：**

```python
class Mlp(nn.Module):
    """论文中的子窗口池化线性层 f_p^l"""

    def __init__(self, in_features, hidden_features=None):
        super().__init__()
        self.fc1 = nn.Linear(in_features, hidden_features)
        self.act = nn.GELU()
        self.fc2 = nn.Linear(hidden_features, in_features)

    def forward(self, x):
        x = self.fc1(x)
        x = self.act(x)
        x = self.fc2(x)
        return x
```

**窗口式 Focal Attention 的核心：**

```python
class GroupFocalAttention(nn.Module):
    """窗口级 Focal Self-Attention"""

    def __init__(self, dim, num_heads, window_size, focal_levels,
                 focal_windows, focal_regions, pool_method="fc"):
        super().__init__()
        self.dim = dim
        self.num_heads = num_heads
        self.window_size = window_size  # sp
        self.focal_levels = focal_levels  # L
        self.focal_windows = focal_windows  # [sw_1, sw_2, ..., sw_L]
        self.focal_regions = focal_regions  # [sr_1, sr_2, ..., sr_L]

        # K, V 投影（每个 level 独立）
        self.kv_projs = nn.ModuleList([
            nn.Linear(dim, dim * 2) for _ in range(focal_levels)
        ])
        # Q 投影（只在 level 1）
        self.q_proj = nn.Linear(dim, dim)

        # 池化层（level >= 2 时才需要）
        self.pool_convs = nn.ModuleList()
        for i in range(focal_levels):
            if focal_windows[i] > 1:
                pool = nn.Conv2d(dim, dim,
                                 kernel_size=focal_windows[i],
                                 stride=focal_windows[i])
                self.pool_convs.append(pool)
            else:
                self.pool_convs.append(nn.Identity())
```

关键发现：
- 代码中实际使用了 **2D 卷积**（`nn.Conv2d`）而不是线性层来做池化。卷积核大小等于子窗口大小 $s_w^l$，步长也相同，等价于不重叠的滑动窗口操作
- 这与论文描述（"a simple linear layer"）不完全一致，但卷积实现更自然、更高效

## 九、位置

### 前驱工作

1. **ViT**（Dosovitskiy et al., 2020）：首个纯 Transformer 图像分类模型，定义了"patch → token"范式，但全局注意力 $O(N^2)$ 不适用于高分辨率输入
2. **Swin Transformer**（Liu et al., 2021）：窗口注意力 + 移位 + 层级化，第一次让 Transformer backbone 全面超越 CNN backbone。Focal 的直接基线和主要对比对象
3. **PVT**（Wang et al., 2021）：用 Spatial Reduction Attention 降低复杂度，是 Swin 之外的另一种层级化路线。Focal 的吸收点：空间池化降低注意力计算量
4. **DeiT**（Touvron et al., 2021）：通过数据增强和知识蒸馏让 ViT 在 ImageNet 上从零训练。Focal 继承了大部分训练配置（RandAugment, Mixup 等）

### 同期竞品

1. **CSWin Transformer**（Dong et al., 2021, arXiv 2107.00652——与 Focal 同期）：十字形窗口注意力，水平和垂直条带分开计算。每个 token 可以和整行/整列交互，跨窗口能力强于 Swin
2. **CrossViT**（Chen et al., 2021）：双尺度 patch embedding + 交叉注意力融合，用不同分辨率的 token 做多尺度建模
3. **RegionViT**（Chen et al., 2021）：引入区域 token 和局部 token 的双层结构，区域 token 负责全局交互，局部 token 负责精细细节

Focal 与同期竞品的核心区别：竞品大多采用**显式的双分支/多分支**设计（两个分辨率的 token 序列分别处理，然后融合），Focal 在**单次注意力计算内部**完成多粒度建模，架构更简洁。

### 后续影响

1. **Focal Modulation Networks**（Yang et al., 2022, arXiv:2203.11926）：同一团队的后续工作，将"focal"思想从注意力机制扩展到调制（modulation）机制。核心变化：用 element-wise 的门控/调制替代 attention 中的 QK^T 乘法，进一步提升效率。在参数量和计算量相当的条件下，Focal Modulation 比 Focal Attention 快约 2 倍
2. **Focal 作为 backbone**：被大量下游工作使用，包括视频理解、3D 点云、医疗图像分割等
3. **多粒度注意力成为标准设计**：后续的 InternImage（2023）、ConvNeXt V2（2023）等都采用了类似的多尺度/多粒度设计思想
4. **对高效注意力的影响**：Focal 验证了"注意力不一定要是全局的、也不一定要是纯局部的——可以用不同粒度混合"这一设计原则，影响了后续的 Efficient Attention 工作

## 十、局限

### 局限 1：计算量仍然高于 Swin

虽然 Focal 的复杂度远低于全局注意力，但比 Swin 的窗口注意力高约 1.5-2 倍（取决于粗粒度 level 的数量和区域大小）。

具体来说，Stage 1（56×56）中，Swin 的窗口注意力需要 49 个 key/value token 每个窗口，Focal（3 levels）需要 116 个——多出 2.4 倍。这个差距在浅层（Stage 1-2）更明显，因为浅层的分辨率高、窗口数量多。

对推理速度的影响：在 GPU 上，Focal-T 比 Swin-T 约慢 15-20%（作者报告中）。这是一个"用速度换精度"的权衡。

**在什么情况下重要：** 对实时性要求高的应用（移动端、自动驾驶）中，Focal 的计算开销可能是瓶颈。

### 局限 2：窗口大小和 level 数的手工设置

$s_p=7, L=3, s_w=\{1,2,4\}, s_r=\{8,6,4\}$ 这些超参数都是手工选择的，没有自适应的机制。对于不同分辨率、不同层级、不同任务，可能存在不同的最优配置。

例如，Stage 4 的 feature map 只有 7×7，此时 3 个 level 的池化已经没有意义（因为 4×4 池化后只剩 1×1）。虽然代码中会在不同 stage 自动调整池化参数，但最优的超参组合仍然是手工调出来的。

**是否有后续工作：** 可变形注意力（Deformable Attention）和可学习的池化策略部分解决了这个问题，但整体来说，Focal 超参的手工设置是一个固有局限。

### 局限 3：细粒度 token 的 surroundings 提取效率

每个窗口需要从不同 level 的特征图中提取 surroundings，涉及了跨 level 的 gather 操作。虽然窗口级提取比 token 级提取高效，但仍然比 Swin 的简单窗口划分复杂。

内存分析：对于 Stage 1 的每个窗口（49 个 token），除了自身 49 个 key 外，还要额外 gather 67 个粗粒度 key。如果做 batch 且窗口数量多（64 个窗口），额外内存大约是 $64 \times 67 \times 96 \approx 0.4\text{MB}$——不算大，但增加了数据加载和重排的开销。

### 局限 4：可扩展性未充分验证

论文中最大的模型是 Focal-L（193M 参数），在 ImageNet-22K 上预训练。但相比于 Swin V2（后来扩展到 3B 参数），Focal 缺乏大规模验证。Focal 的计算密度更高（每个 attention 计算更多的 key/value），在大规模训练中的扩展曲线未知。

### 局限 5：池化信息可能丢失

虽然在注意力计算中通过多粒度补偿了池化的信息丢失，但池化本身是一个"多对一"的压缩操作，不可避免会丢失信息。两个不同的子窗口模式可能池化出相同的粗 token 表示，导致歧义。

这个问题在浅层更严重，因为浅层的特征还在处理边缘/纹理等精细结构，不适合粗粒度的信息压缩。

## 十一、小结

Focal Self-Attention 解决了一个看似矛盾的问题：**如何让视觉 Transformer 的每次注意力计算同时做到"局部精细"和"全局覆盖"？**

它的方案是：**近处用细粒度注意力，远处用粗粒度注意力，通过多尺度池化把计算量控制在可接受范围。** 这个灵感来自人眼的视觉机制——焦点区域看得清楚，余光区域虽然模糊但提供了全局语境。

从技术上说，Focal 在 Swin Transformer 的基础上做了一项精准的改进：**把 Swin 的"间接跨窗口交互"（需要多层累积）替换为"直接多粒度交互"（单层内完成）。** 这个改动不需要改变整体架构（仍然是 4 级金字塔 + 窗口注意力），只替换了注意力模块本身，却带来了稳定的性能提升（ImageNet +0.4~0.9%，COCO +1.0~1.1 AP，ADE20K +1.4~3.3 mIoU）。

但 Focal 也有代价：**计算量比 Swin 高约 1.5-2 倍**。虽然远低于全局注意力，但在实时场景和资源受限设备上，这个开销需要考虑。

在技术树中，Focal Transformer 是一个 **refinement node（精化节点）**。它不是 Swin 那样的"开辟新范式的节点"，而是在 Swin 建立的"局部窗口 + 层级"框架内，对注意力机制做了精准改进。这个工作的价值在于：它验证了一个重要设计原则——**注意力不需要在所有距离上保持相同粒度，根据距离自适应调整粒度是可行且有效的。**

后续的 Focal Modulation Networks 沿着这个思路更进一步，用调制机制替代注意力，实现了更好的效率。而 Focal 提出的"多粒度注意力"设计，成为了后续一大批视觉 Transformer 工作的标准组件。
