# Focal Self-attention for Local-Global Interactions in Vision Transformers（2021）

**论文：** [arXiv:2107.00641](https://arxiv.org/abs/2107.00641) · Jianwei Yang, Chunyuan Li, Pengchuan Zhang, Xiyang Dai, Bin Xiao, Lu Yuan, Jianfeng Gao · Microsoft Research · 2021

**技术分支：** B2 — 视觉 backbone 层级化 / B2-4 — hierarchical backbone 范式

**代码：** [microsoft/Focal-Transformer](https://github.com/microsoft/Focal-Transformer)

---

## 一、先搞清楚"问题在哪里"

2021 年上半年，Swin Transformer 刚刚证明了"层级式局部窗口注意力"可以成为通用视觉 backbone。在 ImageNet 分类、COCO 检测、ADE20K 分割上，Swin 全面超越了 CNN 基线（ResNe(X)t），获得了 ICCV 2021 最佳论文奖。

但 Swin 有一个结构性局限：**跨窗口的信息交互是间接的。**

要理解这个局限，得先看 Swin 的设计逻辑。Swin 用不重叠的 7×7 窗口做自注意力（W-MSA），每个 token 每次只能看到同窗口内的 49 个 token。为了让信息跨窗口流动，Swin 在相邻层之间做窗口偏移（SW-MSA），实现相邻窗口间的间接通信。

问题是：**角落里一个 token 想和另一个远处的 token 交互，需要经过多层累积。** 56×56 的 feature map 上，一次注意力只能覆盖不到 2% 的区域。远处信息的传递依赖深度——这是一个"用层数换感受野"的方案。

Focal Transformer 要解决的正是这个问题：**能否在一次注意力计算中，同时捕捉局部细节和全局语境？**

## 二、旧路线的真正问题

### 2.1 两条路线各自有坑

在 Focal 出现之前，降低视觉 Transformer 注意力复杂度的已有方案可以归为两类：

**路线一：粗粒度全局注意力。** 代表工作是 PVT（Pyramid Vision Transformer）。PVT 用 Spatial Reduction Attention（SRA）——先对 feature map 做空间降维（把 2×2 的 token 拼成一个），然后在降维后的稀疏特征图上做全局注意力。复杂度是 $O((hw)(hw/R^2))$ ，其中 $R$ 是降维率。这个方案的问题是：降维后的粗 token 丢失了精细的空间细节，无法满足目标检测这种需要精确定位的任务。

**路线二：细粒度局部注意力。** 代表工作是 Swin Transformer。每个 token 只在 7×7 窗口内做注意力，复杂度 $O(M^2 hw)$ 是线性的。方案的问题前文已述：跨窗口通信是间接的，每层只能覆盖小范围。

两条路线各自只解决了复杂度问题的一半——要么全局但粗糙，要么精细但局部。**标准全注意力本来能同时做好局部和全局，但它的 $O(N^2)$ 复杂度在高分辨率输入下不可行。** Focal 的思路是：用"不同距离、不同粒度"的方式，在保持线性复杂度的同时，恢复全注意力的局部+全局双重能力。

### 2.2 Swin 的移位窗口到底哪里不舒服

具体到 Swin，三个工程层面的细节值得注意：

1. **偶数层数约束。** 因为 W-MSA 和 SW-MSA 必须交替排列，每个 stage 的层数必须是偶数。这限制了架构设计的灵活性——有时模型可能在奇数层达到最优，但被这个约束卡死了。

2. **cyclic shift 的工程开销。** SW-MSA 的实现使用了循环移位（`torch.roll`）把碎片窗口拼合成规则形状，然后用 attention mask 屏蔽不该交互的区域。虽然相比朴素实现的滑动窗口快了 40 倍以上（Swin 论文 Table 5），但仍然比纯 W-MSA 多了约 2% 的延迟开销，并且需要管理额外的 mask 数据。

3. **窗口扩大时效率下降。** 窗口大小 M 翻倍时，Swin 的 attention 计算量增长 $M^2$ （窗口内 token 数）。而 Focal 的 coarse 级别通过池化让计算增长慢得多。

### 2.3 抽象矛盾：注意力一定是"全有或全无"的吗？

标准视觉 Transformer 的注意力计算隐含了一个假设：**所有 query-key 对都应该在相同的粒度下计算相似度。** 不论是近在咫尺的两个像素还是相隔很远的两个区域，它们的注意力计算都走同样的 $QK^T$ 流程。这在概念上很干净，但效率很低——远处 token 之间不需要同等精度的信息。

Focal 的洞察是：**相邻像素的交互需要高精度（空间细节），而远处像素的交互可以容忍低精度（语义信息）。** 就像人眼看一幅画——视线焦点处的细节是清晰的，余光扫到的区域虽然模糊，但足以提供画布布局和整体色彩信息。

## 三、这篇论文的核心思路

**核心思路：在单次注意力计算中，用近处的细粒度 token 和远处的粗粒度 token 来覆盖全图，把计算量控制在可接受的范围内。**

具体说，Focal Self-Attention 定义了两个粒度层级（ $L=2$ ）：

- **Level 0（细粒度）：** 窗口附近区域的原始分辨率 token。子窗口大小 $s_w^0=1$ ，即不池化。关注区域大小 $s_r^0=13$ ，相当于原始 feature map 上一块 $13 \times 13$ 的区域。这个区域覆盖了 query 窗口（ $7 \times 7$ ）本身以及向四周扩展 3 个 token 的范围。

- **Level 1（粗粒度）：** 更远区域的池化 token。子窗口大小 $s_w^1=7$ （与窗口大小相同），即把 $7 \times 7$ 的区块通过线性层池化为 1 个粗 token。关注区域大小 $s_r^1$ 随 stage 变化：Stage 1 为 7，Stage 2 为 5，Stage 3 为 3，Stage 4 为 1。

用前文的类比：Level 0 是"焦点区域"（清晰），Level 1 是"余光区域"（模糊但覆盖全图）。两者在一次注意力计算中同时完成。

从谱系位置看：

```text
ViT（全局 attention, 单尺度, 2020）
   ↓
Swin Transformer（窗口 attention + 移位, 层级化, 2021）
   ↓
Focal Transformer（窗口 attention + 多粒度, 层级化, 2021） ← 本篇
   ↓
Focal Modulation Networks（从 attention 到调制机制, 2022）
```

Focal 和 Swin 的核心区别是：**Swin 用"移位窗口"间接实现跨窗口信息流动，需要偶数层数和工程 hack（cyclic shift + mask）；Focal 用"多粒度池化"直接为每个窗口提供全局信息，不需要移位，也不需要偶数层数。**

## 四、整体结构与 forward 流程

### 4.1 整体架构

Focal Transformer 的整体架构与 Swin Transformer 几乎一致：4 级金字塔结构，每级之间用 Patch Embedding（stride-2 卷积或拼接+线性投影）做下采样。

```
输入图像: 224×224×3
   │
   ▼
Patch Embedding（4×4 conv, stride 4） → 56×56×96
   │
   ▼
Stage 1: [Focal Block × 2]    56×56, 维度 96
   │
   ▼
Patch Embedding（stride 2）    → 28×28×192
   │
   ▼
Stage 2: [Focal Block × 2]    28×28, 维度 192
   │
   ▼
Patch Embedding（stride 2）    → 14×14×384
   │
   ▼
Stage 3: [Focal Block × 6]    14×14, 维度 384
   │
   ▼
Patch Embedding（stride 2）    → 7×7×768
   │
   ▼
Stage 4: [Focal Block × 2]    7×7, 维度 768
   │
   ▼
分类头（LayerNorm + Global Avg Pool + Linear）
```

Focal-Tiny 的层数配置为 [2,2,6,2]，与 Swin-Tiny 完全相同。但每个 Block 内部的注意力计算从 Swin 的"W-MSA ↔ SW-MSA 交替"替换为"Focal Self-Attention"。

每个 Focal Block 的结构是标准的 Pre-LN Transformer Block：

```
Input
   │
   ▼
LayerNorm
   │
   ▼
Focal Self-Attention（只在空间维度做，不跨通道）
   │
   ▼
残差连接 (+)
   │
   ▼
LayerNorm
   │
   ▼
MLP（GELU, 4× expansion）
   │
   ▼
残差连接 (+)
   │
   ▼
Output
```

所有 Focal Block 内的注意力都是 Focal Self-Attention，没有交替模式，没有移位操作，没有 attention mask。**这是一个比 Swin 更简洁的 Block 设计。**

### 4.2 Focal Self-Attention 的 forward 流程

以 Stage 1（56×56×96, 窗口大小 $s_p=7$ ）的一个 query 窗口为例，完整流程分 5 步：

**步骤 1 — 子窗口池化（Sub-window Pooling）**

对输入 feature map $x \in \mathbb{R}^{56 \times 56 \times 96}$ ，做 $L=2$ 个 level 的子窗口池化：

- **Level 0（细粒度）：** $s_w^0=1$ ，不池化。输出特征图尺寸不变： $56 \times 56 \times 96$ 。
- **Level 1（粗粒度）：** $s_w^1=7$ ，与窗口大小相同。把 56×56 划分为 $8 \times 8$ 个 7×7 子窗口，每个子窗口用线性层池化为 1 个粗 token。输出尺寸： $8 \times 8 \times 96$ 。

池化公式很简单：对每个 7×7 子窗口，把 49 个 token 展平，通过一个可学习的线性层 $f_p$ 聚合为 1 个 token。 $f_p$ 的输入维度是 $49 \times 96$ ，输出维度是 $1 \times 96$ 。参数数量： $49 \times 96 = 4704$ 。相比于 Focal-Tiny 的 29.1M 参数，这部分几乎可忽略。

**步骤 2 — 窗口划分**

将原始 feature map（Level 0）划分为不重叠的 7×7 窗口。56×56 → $8 \times 8 = 64$ 个窗口，每个窗口 49 个 token。

Query 只从 Level 0（原始分辨率）生成。Key 和 Value 从 Level 0 和 Level 1 共同生成。

**步骤 3 — 为每个窗口提取 multi-level surroundings**

对第 $i$ 个 query 窗口（大小 $7\times7$ ），从两个 level 的特征图中提取其 surroundings 的 key 和 value：

- **Level 0 surroundings：** 在 Level 0 特征图中，提取窗口周围的 $s_r^0 \times s_r^0 = 13 \times 13$ 个 fine token。这覆盖了窗口本身（7×7）以及向四周各扩展 3 个 token 的范围。对于 Stage 1 来说， $13 \times 13 = 169$ 个 token 可以覆盖大部分局部信息。
- **Level 1 surroundings：** 在 Level 1 池化后的 $8 \times 8$ 特征图中，提取对应窗口附近的 $s_r^1 \times s_r^1 = 7 \times 7$ 个 coarse token。由于 Level 1 的一个 token 对应原始 feature map 上 7×7 的区域（池化因子 7）， $7 \times 7$ 个 coarse token 已经覆盖了整个特征图（ $56 \times 56$ ）。这就实现了**全局覆盖**。

两个 level 的 key/value 数量合计： $s = 13^2 + 7^2 = 169 + 49 = 218$ 。

相比于标准全注意力中每个 query 需要和 $56 \times 56 = 3136$ 个 token 计算，Focal 只需和 218 个 token 计算——仅为 7%。

**步骤 4 — 注意力计算**

对窗口内的 49 个 query token，和拼接后的 218 个 key/value token 做标准 Scaled Dot-Product Attention：

$$\text{Attention}(Q_i, K_i, V_i) = \text{Softmax}\left(\frac{Q_i K_i^T}{\sqrt{d_k}} + B\right)V_i$$

其中 $B$ 是按 level 分别维护的相对位置偏置。

**步骤 5 — 窗口重组**

将每个窗口的输出拼接回原始形状。

其余 3 个 Stage 的流程完全相同，只是 $s_r^1$ 不同：Stage 2 为 5，Stage 3 为 3，Stage 4 为 1。Stage 4 时特征图只有 $7 \times 7$ ，Level 1 的粗粒度注意力 $s_r^1=1$ 等价于最后一个粗 token（全图池化后的单个 token）——最极端情况下的全局交互。

### 4.3 与 Swin 的 forward 流程对比

| 步骤 | Swin Transformer | Focal Transformer |
|------|-----------------|-------------------|
| 窗口划分 | 规则窗口 → 循环移位窗口 → 规则窗口 | 始终规则窗口 |
| 注意力 | W-MSA（层 1,3,5...）↔ SW-MSA（层 2,4,6...）交替 | 每层都用 Focal Self-Attention |
| 跨窗口通信 | 通过移位间接实现，需偶数层 | 通过粗粒度 level 直接实现，无层数限制 |
| 额外操作 | cyclic shift + attention mask | 子窗口池化（线性层） |
| key/value 数（Stage 1） | 49 per window | 218 per window（细 169 + 粗 49） |
| 复杂度 | $O(M^2 hw d)$ | $O((s_r^0)^2 + (s_r^1)^2) hw d$ |

### 4.4 复杂度分析

Focal Self-Attention 的总复杂度：

$$\text{FLOPs} = \mathcal{O}\left(hw\left(4dC + \sum_{l=0}^{1} (s_r^l)^2 d\right)\right)$$

前一项 $4hwdC$ 是 QKV 投影和 output 投影，与 Swin 相同。后一项 $\sum_l (s_r^l)^2 hw d$ 是 attention 计算。

数值对比（Stage 1, $h=56, w=56, d=96$ ）：
- 标准全局注意力： $2(hw)^2 d = 1.89 \text{GFLOPs}$
- Swin 窗口注意力： $2M^2 hw d = 29.5 \text{MFLOPs}$
- Focal 注意力： $2(s_r^0)^2 hw d + 2(s_r^1)^2 hw d = 2 \times 169 + 2 \times 49 = 436 \text{MFLOPs}$ 的 attention 部分。加上 QKV 投影（与 Swin 相同），总计约 $0.6 \text{GFLOPs}$ 。

Focal 的 attention 计算量是 Swin 的 14.8 倍，但只有全局注意力的 23%。**多出的计算量换来了每层都能直接获取全局信息的能力，省掉了 Swin 需要多层积累才能获得全局感受野的代价。**

论文的实际报告 FLOPs（Focal-T 4.9G, Swin-T 4.5G）证实了这一点——Focal 比 Swin 多约 8-10% 的 FLOPs，主要来自粗粒度 level 的额外 attention 计算。

## 五、核心模块详解

### 5.1 子窗口池化（Sub-window Pooling）

**做什么：** 把 $s_w^l \times s_w^l$ 的区块池化为单个代表 token，用于 coarse-level 的注意力计算。

**怎么做：** 对每个子窗口，展平后通过线性层加权求和。公式：

$$x^l = f_p^l(\hat{x}) \in \mathbb{R}^{\frac{M}{s_w^l} \times \frac{N}{s_w^l} \times d}, \quad \hat{x} = \text{Reshape}(x) \in \mathbb{R}^{(\frac{M}{s_w^l} \times \frac{N}{s_w^l} \times d) \times (s_w^l \times s_w^l)}$$

**为什么这样设计：**

1. **可学习池化：** 不同于固定池化（mean/max），线性层的权重可以在训练中学习到"哪些 spatial 位置更重要"。例如，对于 Level 1 的池化（ $s_w=7$ ），模型可能学到窗口中心区域的 token 权重更高（因为距离 query 窗口更近）。
2. **参数高效：** 池化层的参数量为 $s_w^2 \times d$ ，对于 $s_w=7, d=96$ 的情况，仅 4704 个参数。相对于整个模型的数千万参数，几乎可以忽略。
3. **通道无关的池化：** 每个通道独立池化，保留了 channel 维度的信息。最大池化或平均池化对通道一视同仁，而可学习池化可以学习到"某些通道需要保留中心区域、某些通道需要保留边缘区域"等不同策略。

**实际实现：** 代码中使用 `nn.Conv2d`（kernel = $s_w^l$ , stride = $s_w^l$ ）+ 2 层 MLP（含 GELU）。卷积核大小的滑窗对不重叠子窗口做卷积，等价于论文描述的子窗口线性池化。2 层 MLP 增强了池化表达能力。

### 5.2 窗口级 surroundings 提取

**做什么：** 为每个 query 窗口提取其 surroundings 的 key/value，用于注意力计算。

**怎么做：** 对每个窗口 $i$ ，在其 Level 0 和 Level 1 的特征图中，分别框出 $s_r^0 \times s_r^0$ 和 $s_r^1 \times s_r^1$ 的区域，提取其中的 key/value。

对于 Stage 1（56×56）：
- Level 0 范围内：窗口中心确定后，取上下左右各扩展 3 个 token 的范围，即 13×13。由于窗口本身是 7×7，extract 范围是在窗口周围 $(-3,-3) \sim (+3,+3)$ 的扩展。对于处于 feature map 中心的窗口，很好覆盖。对于边角窗口，会超出边界，此时使用 padding（论文中通过 zero-padding 处理边界情况）。
- Level 1 范围内：在 $8 \times 8$ 的池化特征图上，取 7×7 的范围——已经覆盖了整个池化空间，等价于全局覆盖。

**为什么是窗口级而不是 token 级：** 同一窗口内的 token 位置接近，它们各自的 surroundings 高度重叠。如果每个 token 独立做 gather，会产生大量重复的内存读取。以窗口为单位共享 surroundings 提取结果，内存访问次数减少到原来的 $1/s_p^2 = 1/49$ 。

**Stage 4 的特殊情况：** 特征图大小只有 7×7，Level 0 区域 $s_r^0=7$ （因为 7×7 是整个图），Level 1 区域 $s_r^1=1$ （单个粗 token）。此时 Level 1 退化为一颗"全局粗 token"，等价于对整个 feature map 做一次全局池化后作为额外的 key/value。

### 5.3 多 level 相对位置偏置

**做什么：** 为注意力添加位置信息，告诉模型 query 和 key 之间的空间关系。

**怎么做：** 两个 level 使用不同的 position bias 表：

- **Level 0（细粒度）：** 与 Swin 相同的 $(2s_p - 1) \times (2s_p - 1) = 13 \times 13$ 的偏置表 $B^0$ 。因为 query 和 key 都在原始分辨率下，相对位置范围是 $[-s_p+1, s_p-1] = [-6, 6]$ ，共 13 个偏移值。对于 49 个 query token 和 169 个 Level 0 key token，每个 query-key 对的偏移量映射到 $13 \times 13$ 表中取值。

- **Level 1（粗粒度）：** 偏置表大小为 $s_r^1 \times s_r^1$ 。因为粗粒度 key 的尺度较小（ $8 \times 8$ 池化网格），只需要这个大小的偏置表即可覆盖所有可能的偏移组合。 $\{B^1 \in \mathbb{R}^{7 \times 7}, \mathbb{R}^{5 \times 5}, \mathbb{R}^{3 \times 3}, \mathbb{R}^{1 \times 1}\}$ （对应 Stage 1-4）。

两个 level 的偏置在注意力计算时拼接到一起，与 key/value 的拼接方式一致。

**为什么不同粒度需要不同的偏置策略：** Level 0 的 key 是原始分辨率下的 token，query-key 对之间的偏移单位是"像素"。Level 1 的 key 是池化后的粗 token，每个粗 token 代表了原始空间中的 $7\times7$ 区域。如果使用相同的偏置策略，粗 token 之间的偏移距离被放大了 7 倍——所以需要独立维护偏置表。

## 六、核心公式

### 公式 1：子窗口池化

$$x^l = f_p^l(\hat{x}) \in \mathbb{R}^{\frac{M}{s_w^l} \times \frac{N}{s_w^l} \times d}, \quad \hat{x} = \text{Reshape}(x) \in \mathbb{R}^{(\frac{M}{s_w^l} \times \frac{N}{s_w^l} \times d) \times (s_w^l \times s_w^l)}$$

**解释：** 输入 $x \in \mathbb{R}^{M \times N \times d}$ 。对 level $l$ ，用 $s_w^l \times s_w^l$ 的子窗口将空间划分网格，每个网格内的 $s_w^l \times s_w^l$ 个 token 通过线性层 $f_p^l$ 聚合为 1 个 token。输出空间尺寸变为 $M/s_w^l \times N/s_w^l$ 。

$L=2$ 时：
- $l=0$ : $s_w^0=1$ ， $x^0 = x$ （恒等映射，不做池化）
- $l=1$ : $s_w^1=7$ ， $x^1$ 是 $56/7 \times 56/7 = 8 \times 8$ 的粗特征图

### 公式 2：Focal Self-Attention

$$Q = f_q(x^0), \quad K = \{f_k(x^0), f_k(x^1)\}, \quad V = \{f_v(x^0), f_v(x^1)\}$$

$$\text{Attention}(Q_i, K_i, V_i) = \text{Softmax}\left(\frac{Q_i K_i^T}{\sqrt{d_k}} + B\right)V_i$$

其中 $Q_i \in \mathbb{R}^{s_p^2 \times d_k}$ 是第 $i$ 个窗口的 query（来自 Level 0）， $K_i$ 和 $V_i$ 是该窗口在两个 level 上的 surroundings 的拼接。

**关键设计点：Query 只从 Level 0 生成。** 因为只有 Level 0 保留了原始分辨率下的精确定位信息。粗粒度 Key 提供了语义信息，但不需要从粗粒度的 Query 出发——Query 的定位精度决定了最终注意力输出的空间精度。

### 公式 3：复杂度

全局注意力（baseline）：

$$\Omega(\text{MSA}) = 4hwC^2 + 2(hw)^2 C$$

Swin 窗口注意力：

$$\Omega(\text{W-MSA}) = 4hwC^2 + 2M^2 hw C$$

Focal 注意力（ $L=2$ ）：

$$\Omega(\text{Focal}) = 4hwC^2 + 2\big((s_r^0)^2 + (s_r^1)^2\big) hw C$$

Stage 1 数值代入（ $h=w=56, C=96, M=7, s_r^0=13, s_r^1=7$ ）：

| 方法 | Attention FLOPs | QKV FLOPs | 总计 |
|------|----------------|-----------|------|
| 全局 MSA | $2 \times 3136^2 \times 96 = 1.89 \text{G}$ | 0.12G | 2.01G |
| Swin W-MSA | $2 \times 49 \times 3136 \times 96 = 29.5 \text{M}$ | 0.12G | 0.15G |
| Focal | $2 \times (169+49) \times 3136 \times 96 = 0.13 \text{G}$ | 0.12G | 0.25G |

Focal 在 Stage 1 的 FLOPs 约为 Swin 的 1.7 倍，全局注意力的 12.5%。考虑到后 3 个 Stage 分辨率更低，Focal 的额外开销占比更小（Stage 4 的 $s_r^1=1$ ，粗粒度部分只贡献 $1 \times 1 = 1$ 个 token）。全模型 FLOPs 的差距约为 8-10%（Focal-T 4.9G vs Swin-T 4.5G）。

## 八、消融实验

### 8.1 细粒度 vs 粗粒度的贡献（Figure 5）

论文做了 4 组消融来解开局部和全局注意力的贡献：

| 配置 | ImageNet Top-1 | COCO Mask R-CNN AP^b |
|------|---------------|---------------------|
| Focal-T（完整，局部+全局） | 82.2 | 44.8 |
| Focal-T-Window（仅窗口内，砍掉所有跨窗口交互） | 80.1 | 38.3 |
| Focal-T-Local（仅局部跨窗口） | ~81.5 | ~43.0 |
| Focal-T-Global（仅粗粒度全局） | ~81.2 | ~42.0 |

**分析：**

1. **砍掉所有跨窗口交互（Window only）**：精度暴跌 2.1% 分类、6.5 AP 检测。这证实了跨窗口信息是必不可少的——只做窗口内的局部注意力，感受野被死死限制在 7×7 范围内。

2. **仅局部跨窗口（Local fine-grain）：** 开启 $s_r^0=13$ 的细粒度跨窗口交互，不开启粗粒度全局部分。分类恢复至 ~81.5%，检测恢复至 ~43.0 AP。这说明仅靠局部扩展的 fine-grained surroundings 就能恢复大部分性能——毕竟 13×13 的覆盖范围已经是窗口自身的近 4 倍。

3. **仅粗粒度全局（Global coarse-grain）：** 不做细粒度跨窗口，只开启粗粒度 Level 1 的全局交互。分类恢复至 ~81.2%，检测恢复至 ~42.0 AP。粗粒度全局信息的恢复程度稍低，特别是在检测任务上——说明全局语境的粗略信息不能完全替代局部扩展的精细信息。

4. **两者都开（完整模型）：** 分类 82.2%，检测 44.8 AP。两个组件的提升完全叠加，说明局部精细信息和全局粗略信息是互补的。细粒度局部对检测的贡献更大，粗粒度全局对分类的贡献更大——这符合直觉：检测需要精确定位边界，分类更需要理解整个图像的语义内容。

### 8.2 窗口大小（Table 8）

| 配置 | Window Size | ImageNet Top-1 | COCO Mask R-CNN AP^b | ADE20K mIoU |
|------|------------|---------------|---------------------|-------------|
| Swin-T | 7 | 81.3* | 43.7 | 44.5 |
| Focal-T | 7 | 82.2 | 44.8 | 45.8 |
| Swin-T | 14 | ~81.6 | ~44.3 | ~45.0 |
| Focal-T | 14 | ~82.4 | ~45.0 | ~46.2 |

*注：Swin-T 窗口 7 的 81.3 是 Swin 论文原数，Focal 论文使用相同的训练配置得到 81.2（正常波动范围内）。

**关键结论：窗口 14 时 Swin-T 的提升**来自更大的局部感受野，但 Focal-T 用 7 窗口已经超过 Swin-T 用 14 窗口（82.2 > ~81.6）——说明 Focal 的粗粒度全局交互比单纯放大窗口更有效。

### 8.3 窗口移位必要性（Table 9）

| 配置 | ImageNet Top-1 | COCO Mask R-CNN AP^b |
|------|---------------|---------------------|
| Swin-T（有移位） | 81.2 | 43.7 |
| Swin-T（无移位） | ~80.2 | ~38.8 |
| **Focal-T（无移位，原始）** | **82.2** | **44.8** |
| Focal-T（加上移位） | ~81.9 | ~44.5 |

Swin 去掉移位后精度大跌（81.2→80.2, 43.7→38.8），因为跨窗口通信完全中断。Focal 加上移位后反而稍微下降（82.2→81.9）。这说明：**Focal 的多粒度注意力已经提供了足够的跨窗口信息，移位操作不仅多余，还可能引入噪声。**

这个结果有一个重要推论：**Focal 不需要偶数层数约束。** Swin 的每个 stage 必须偶数层（W / SW 交替），Focal 的每层都可以独立配置。这为架构搜索、剪枝等后续优化提供了更大的自由度。

### 8.4 模型深度（Table 10）

| 配置 | Stage 3 层数 | ImageNet Top-1 | FLOPs |
|------|------------|---------------|-------|
| Swin-T | 6（原始） | 81.2 | 4.5G |
| Swin-T | 4 | ~80.5 | ~4.1G |
| Swin-T | 2 | ~79.5 | ~3.6G |
| Focal-T | 6（原始） | 82.2 | 4.9G |
| Focal-T | 4 | **81.4** | ~4.4G |
| Focal-T | 2 | ~80.5 | ~3.8G |

**Focal-T 用 4 层（81.4%）就超过了 Swin-T 用 6 层（81.2%）。** 这说明 Focal 的每层信息效率更高——因为每层都直接做了全图粗粒度交互，所以在更少的层数下能达到相同的效果。如果用层数换效率，Focal 可以做到更小更快的同时保持高精度。

## 九、代码阅读

官方仓库：`microsoft/Focal-Transformer`（[GitHub](https://github.com/microsoft/Focal-Transformer)）

核心文件：`models/focal_transformer.py`

### 9.1 核心类层次

```text
FocalTransformer
 ├── PatchEmbed              # Patch Partition（4×4 conv, stride 4）
 ├── BasicLayer (×4)         # 4 stages
 │   ├── PatchMerging         # Stage 间下采样（2×2 concat + Linear）
 │   └── FocalTransformerBlock (×N)
 │       ├── FocalAttention   # Focal Self-Attention 核心实现
 │       └── Mlp              # 2-layer MLP + GELU
 └── norm + head              # LayerNorm + Linear 分类器
```

### 9.2 FocalAttention 核心代码路径

1. **QKV 投影：** `self.qkv = nn.Linear(dim, dim*3, bias=qkv_bias)`。和标准 Transformer 一样，三个投影共享一个线性层，拆分后分别作为 Q, K, V。

2. **子窗口池化：** 使用 `nn.Conv2d(dim, dim, kernel_size=sw, stride=sw)`。卷积核大小等于子窗口大小，步长也一样，达到了"不重叠的滑动窗口池化"效果。比论文描述的线性层方案更高效（可以直接在 2D 特征图上操作，不需要 reshape）。

   池化层内部使用 2 层 MLP（Conv2d + GELU + Conv2d），不是论文中说的单层线性层。这增强了池化的表达能力。

3. **Level 0 surroundings 提取：** 在原始 2D 特征图上，通过 `F.unfold` 提取窗口周围 $(s_r^0 + s_p - 1) \times (s_r^0 + s_p - 1)$ 的范围，然后从中选出对应窗口的 surroundings。

4. **Level 1 surroundings 提取：** 在池化后的粗特征图上，通过类似操作提取 $s_r^1 \times s_r^1$ 的范围。由于池化因子是 7，这一步的 gather 范围覆盖了整个粗特征图。

5. **Relative position bias：** 两个 level 各自维护独立的位置偏置表。Level 0 使用 $(2s_p-1)^2 \times \text{nH}$ 的偏置表，通过预计算的索引表查值。Level 1 使用 $(2s_r^1-1)^2 \times \text{nH}$ 。

6. **Fused attention 计算：** Level 0 和 Level 1 的 key/value 在 token 维度拼接，然后做标准的 scaled dot-product attention。由于两个 level 的 token 在 concat 后共享相同的计算路径，没有任何特殊处理——注意力权重会根据内容自动分配。

## 十、技术谱系

### 前驱

- **ViT (2020):** 定义 patch→token 范式，但全局注意力不适合高分辨率任务
- **Swin Transformer (2021):** 窗口注意力 + 层级化，Focal 的直接基线和修改对象
- **PVT (2021):** Spatial Reduction Attention 提供了"空间池化降低注意力复杂度"的思路参考
- **DeiT (2021):** 训练配置（RandAugment, Mixup, CutMix）被 Focal 继承

### 同期竞品

- **CSWin Transformer (2021, arXiv 2107.00652 — 与 Focal 同一天发布):** 十字形窗口注意力，水平+垂直条带分开计算。每个 token 可以和整行/整列交互，跨窗口能力强于 Swin
- **CrossViT (2021):** 双尺度 patch embedding + 交叉注意力融合
- **RegionViT (2021):** 区域 token + 局部 token 的双层结构

Focal 与同期竞品的核心区别：竞品大多采用**显式双分支**设计（两个分辨率的 token 序列分别处理再融合），Focal 在**单次注意力内部**完成多粒度建模，架构更简洁。

### 后续

- **Focal Modulation Networks (2022, arXiv:2203.11926):** 同一团队将"focal"思想从注意力扩展到调制机制，用 element-wise 门控替代 $QK^T$ 乘法，参数量相当下速度快约 2 倍
- **InternImage (2023):** 使用动态卷积近似大核注意力，继承了"不同区域不同处理"的设计哲学

## 十二、范式意义

Focal Transformer 的核心贡献不在于开辟了新范式，而在于**验证了一个重要的注意力设计原则：注意力不一定要在所有距离上保持相同粒度。**

这个原则看似简单，但直到 Focal 之前，没有人系统性验证过。全局注意力假设"所有 query-key 对需要同精度"，局部注意力假设"只需要局部细节就够了"——这两种极端都被 Focal 的工作证明是次优的。最优方案在不同距离上用不同粒度。

在技术树中，Focal Transformer 是一个 **refinement node（精化节点）**。它没有推翻 Swin 的"局部窗口+层级"框架（实际上完全沿用了这个框架），而是在框架内部替换了注意力机制。这种"小改动、大效果"的工作特点意味着：**当范式的核心框架已被验证有效时，细致的组件改进可以带来显著的边际收益。**

Focal 发现的另一个重要事实：**视觉 Transformer 的学习速度与跨层通信效率有关。** Focal-T 在训练第 100 epoch 时达到 75.7%，而 Swin-T 只有 73.9%——快 1.8%。论文将此归因于 Focal 每层的全局交互让信息在前向/后向传播中流动更通畅。这个观察后来被证明对大规模训练很重要（Swin V2 花了很大功夫解决训练不稳定问题，而 Focal 的多粒度设计可能天然更稳定）。

## 十三、局限

### 13.1 计算量比 Swin 高

Focal-T（4.9G FLOPs）比 Swin-T（4.5G FLOPs）多约 9% 的 FLOPs，主要来自粗粒度 attention 的额外计算。在 GPU 上的实际推理速度比 Swin 慢约 15-20%。

这个"用速度换精度"的权衡在实时应用和资源受限设备上可能是瓶颈。对于移动端部署或自动驾驶等场景，Swin 的简单局部窗口方案可能更合适。

### 13.2 超参手工设置

$s_p=7, s_r^0=13, s_r^1 \in \{7,5,3,1\}$ 这些超参都是手工选的。对于不同分辨率、不同 stage、不同任务，最优配置可能不同。虽然论文通过消融（窗口大小 7 vs 14）部分验证了配置的鲁棒性，但缺乏自适应机制。

### 13.3 池化信息丢失

Level 1 用 7×7 池化把 49 个 token 压缩为 1 个粗 token。池化是"多对一"操作，不可避免地丢失了子窗口内的空间结构信息。两个完全不同的空间模式可能池化出相同的粗 token（例如"左上角有一个亮点"和"右下角有一个亮点"在池化后可能无法区分）。

在浅层 stage 中，特征还在处理边缘/纹理等精细结构，此时粗粒度的信息压缩可能丢失对后续任务有用的细节。Focal 通过重叠区域（显式保留 Level 0/1 的重叠区域）来部分缓解这个问题，但并未完全解决。

### 13.4 大规模未充分验证

Focal-Large（~193M 参数）的 ImageNet-22K 预训练使用了 Swin-Large 的权重做初始化（"部分初始化"），而非从零训练。Focal 的完整扩展曲线（scaling law）没有被研究。相比于 Swin V2 后来扩展到 3B 参数并系统研究了大模型训练稳定性，Focal 在大规模方向的探索比较有限。

### 13.5 聚焦于视觉，未探索语言域

论文在 Future Work 中提到相信 Focal Attention 也适用于 NLP，但没有做任何 NLP 实验。实际上，NLP 中的 token 之间没有"空间邻近性"的自然概念——"距离"在 NLP 中是抽象的（位置的索引差，不是物理距离）。Focal 依赖的"近处细粒度、远处粗粒度"直觉在 NLP 中是否成立，是未验证的。

## 十四、小结

Focal Transformer 在 Swin Transformer 的"局部窗口+层级"框架上，做了一项精准的改进：**把"间接跨窗口交互"换成"直接多粒度交互"。**

具体改动很小——每个注意力模块加一个子窗口池化层（线性层，参数不到 5000），引入一个粗粒度 attention level。但收益显著：ImageNet 上 +1.0%（Tiny），COCO 上 +1.7 AP（RetinaNet 1×），ADE20K 上 +1.3 mIoU（单尺度）。而且这个收益不需要偶数层数约束，不需要移位窗口，不需要 attention mask——架构反而更简化了。

从更大的视角看，Focal 做的是一次"效率-能力"的重新平衡：Swin 牺牲了每层的感受野来换取 $O(N)$ 复杂度，Focal 用多出的 10% FLOPs 把感受野恢复到全局。这 10% 的额外计算换来了 1% 的分类精度和 1-2 AP 的检测精度——在大多数场景下是划算的买卖。

Focal 没有推翻什么，但它验证了一个至今仍有参考价值的设计原则：**注意力不需要在所有距离上保持相同粒度。** 这个原则后来被 Focal Modulation、InternImage 等工作继承，成为了视觉 Transformer 中处理 attention 质量和效率权衡的标准思路之一。
