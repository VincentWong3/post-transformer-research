# MLP-Mixer: An all-MLP Architecture for Vision（2021）

**论文：** [arXiv](https://arxiv.org/abs/2105.01601) · Ilya Tolstikhin, Neil Houlsby, Alexander Kolesnikov, Lucas Beyer, Xiaohua Zhai, Thomas Unterthiner, Jessica Yung, Andreas Steiner, Daniel Keysers, Jakob Uszkoreit, Mario Lucic, Alexey Dosovitskiy · 2021

---

## 一、先搞清楚坑在哪

2021 年的视觉领域，卷积神经网络（CNN）已经统治了将近十年。从 AlexNet 到 ResNet 再到 EfficientNet，CNN 架构的演化史几乎就是图像识别的进化史。但就在大家以为"视觉就得靠卷积"的时候，Vision Transformer（ViT）证明了一件事：**注意力机制也能干好视觉**，而且在大规模数据下甚至比 CNN 更好。

这就引出了一个更根本的问题：到底什么才是视觉任务真正需要的？

- CNN 带了很强的归纳偏置——局部性、平移不变性、层次化特征。ViT 说"这些不是必需的"。
- 但 ViT 自己也有复杂度——自注意力机制。MLP-Mixer 的答案是更激进的：**连注意力都不需要，用最简单的 MLP 就可以了。**

## 二、CNN 和 Transformer 的真正问题

**CNN** 的核心操作是卷积核滑窗。这个操作背后的假设是：相邻像素的关系比远距离像素更重要。它有效地利用了局部性，但在建模长距离依赖时效率很低——要扩大感受野，要么堆更多层，要么用大卷积核，成本都很高。

**ViT** 把图像切成 patch，用自注意力在 patch 之间建立全局连接，解决了长距离依赖的问题。但自注意力的计算复杂度是 $O(N^2)$（$N$ 为 patch 数量），图像分辨率一上去就扛不住了。而且 ViT 的归纳偏置太少，需要超大规模数据（如 JFT-300M，300M 张图像、18k 类别）才能充分发挥。

## 三、MLP-Mixer 的核心思路

**一句话总结**：用两个交替的 MLP 分别处理「空间位置之间」和「通道之间」的信息交互，完全替代卷积和自注意力。

具体来说，输入图像被切成 $S$ 个不重叠的 patch 后，每个 patch 通过线性投影映射到 $C$ 维隐藏空间，得到一个 $S \times C$ 的"特征表"（$S$ 个位置，每个 $C$ 维）。MLP-Mixer 在每个层做了两个正交的操作：

1. **Token-mixing MLP（空间混合）**：在 $S$ 个空间位置之间混合信息，作用于特征表的列上（即跨 patch 混合同通道的信息）
2. **Channel-mixing MLP（通道混合）**：在每个空间位置内部混合 $C$ 个通道的信息，作用于特征表的行上（即跨通道混合同位置的信息）

这两个 MLP 交替执行，配合残差连接（skip-connection）和 Layer Normalization（逐层归一化）。

## 四、方法细节

### 整体架构

图 1 展示了 MLP-Mixer 的整体架构。输入图像先经 patch 提取和线性投影得到一个特征表，然后通过若干 Mixer 层（每个包含空间混合和通道混合两个 MLP），最后经全局平均池化和分类头输出。

```
输入图像 (H x W x 3)
    │
    ▼ Patch 提取 + 线性投影 (patch_size=P, S = H×W / P²)
    │
    ▼ 特征表 U ∈ ℝ^{S×C}
    │
    ┌─────────────────────────────────────┐
    │        Mixer Layer (重复 L 次)        │
    │                                      │
    │  ┌─ LayerNorm ─ Token-mixing MLP ──┐ │
    │  │          (转置后做 MLP)          │ │
    │  └────────── + 残差连接 ────────────┘ │
    │  ┌─ LayerNorm ─ Channel-mixing MLP ┐ │
    │  │           (直接做 MLP)           │ │
    │  └────────── + 残差连接 ────────────┘ │
    └─────────────────────────────────────┘
    │
    ▼ 全局平均池化
    │
    ▼ 分类头 (线性层)
```

**图 1：MLP-Mixer 整体架构。输入图像先切 patch 并投影到 C 维，再通过 L 个结构相同的 Mixer 层。每个 Mixer 层中，Token-mixing MLP 在空间维度（跨 patch）做混合，Channel-mixing MLP 在通道维度（跨特征）做混合，两者交替执行。**

### 关键设计思路

这里的关键洞察是：**把空间混合和通道混合解耦**。传统的 CNN 卷积核同时做两件事（卷积核同时跨越空间和通道维度），ViT 的自注意力也在混合空间信息的同时通过 value 投影混合通道信息。MLP-Mixer 把这两件事拆开了，每个操作只做一件事，用最简单的 MLP 实现。

**Token-mixing MLP 替代自注意力**：在 ViT 中，自注意力计算所有 patch 对之间的相关性，权重是"数据相关"的（由 query 和 key 的点积决定）。MLP-Mixer 的 token-mixing MLP 也做类似的事情——让每个 patch 的信息与其他所有 patch 混合——但权重是"静态的、可学习的"，对所有输入都一样。相当于用固定的混合矩阵替代了动态的注意力矩阵。

**Channel-mixing MLP 替代 1×1 卷积**：在每个空间位置内部混合通道信息，等价于一个 1×1 卷积 + GELU 非线性激活（一种平滑版的 ReLU，[Hendrycks & Gimpel, 2016]）。和 CNN 中的 pointwise convolution 做的完全一样。

**不需要位置编码**：ViT 需要位置编码是因为自注意力是置换不变的（permutation invariant），不知道 patch 的顺序。但 MLP-Mixer 的 token-mixing MLP 权重矩阵本身与 patch 索引绑定——$W_{ij}$ 就是从 patch $j$ 到 patch $i$ 的连接强度——所以隐式地编码了位置信息。论文中的消融实验也证实了这一点：加位置编码反而略降（76.4% → 76.2%）。

### 参数共享设计

所有 Mixer 层使用"等宽"（isotropic）设计——每层的输入输出尺寸相同，这类似于 Transformer 而不是大多数 CNN 的金字塔结构。而且，同一层内的所有行共享同一个 channel-mixing MLP，所有列共享同一个 token-mixing MLP。论文提到，这种参数共享并未损害性能（见 Supplementary A.1），但显著减少了参数量。

## 五、公式详解

### Token-mixing MLP

$$U_{*,i} = X_{*,i} + \mathbf{W}_2\, \sigma(\mathbf{W}_1\, \text{LayerNorm}(X)_{*,i}), \quad i = 1 \ldots C$$

- $X \in \mathbb{R}^{S \times C}$：输入特征表，$S$ 个 patch，每个 $C$ 维
- $X_{*,i}$：第 $i$ 列（一个通道在所有 $S$ 个位置上的值），是一个长度为 $S$ 的向量
- $\mathbf{W}_1 \in \mathbb{R}^{D_S \times S}$，$\mathbf{W}_2 \in \mathbb{R}^{S \times D_S}$：token-mixing MLP 的两个权重矩阵
- $D_S$：token-mixing MLP 隐藏层宽度（论文中独立于 $S$ 设定，B/16 为 384）
- $\sigma$：逐元素的 GELU 非线性激活

**工作方式**：对输入 $X$ 先做 LayerNorm（在通道维度做归一化），然后转置得到 $X^\top \in \mathbb{R}^{C \times S}$，对每行（一个通道的所有空间位置值）独立做 $S \rightarrow D_S \rightarrow S$ 的 MLP，再转置回来加残差。所有通道共享同一个 token-mixing MLP 的参数。

关键性质：计算复杂度与 $S$ **线性相关**（因为 $D_S$ 独立于 $S$ 设定），而不是像 ViT 的自注意力那样 $O(S^2)$。

### Channel-mixing MLP

$$Y_{j,*} = U_{j,*} + \mathbf{W}_4\, \sigma(\mathbf{W}_3\, \text{LayerNorm}(U)_{j,*}), \quad j = 1 \ldots S$$

- $U \in \mathbb{R}^{S \times C}$：token-mixing 后的输出
- $U_{j,*}$：第 $j$ 行（一个空间位置在所有 $C$ 个通道上的值），是一个长度为 $C$ 的向量
- $\mathbf{W}_3 \in \mathbb{R}^{D_C \times C}$，$\mathbf{W}_4 \in \mathbb{R}^{C \times D_C}$
- $D_C$：通道 MLP 隐藏层宽度（B/16 为 $3072 = 4 \times 768$）

**直觉**：对每个空间位置独立地做 $C \rightarrow 4C \rightarrow C$ 的 MLP，等价于逐位置的 1×1 卷积 + GELU。

### 参数量计算（以 Mixer-B/16 为例）

论文 Table 1 给出了各模型规格：B/16 是 $C=768$，$S=196$，$L=12$，$D_C=3072$，$D_S=384$。

- 每个 channel-mixing MLP：$2 \times 768 \times 3072 + 3072 + 768 \approx 4.72M$
- 12 层 channel-mixing 小计：$4.72 \times 12 \approx 56.7M$
- 每个 token-mixing MLP：$2 \times 196 \times 384 + 384 + 196 \approx 0.15M$（注意论文中 $D_S=384$，不是 $4S$）
- 12 层 token-mixing 小计：$0.15 \times 12 \approx 1.8M$
- 加上 patch 投影和分类头：总参数量约 59M

参数量大头在 channel-mixing MLP（占约 96%）。注意 token-mixing 的参数量较小，因为 $D_S$ 独立于 $S$ 设定且远小于 $S \times 4$。

## 六、实验结果

论文在 ImageNet-21k（14M 图像、21k 类别）和 JFT-300M（300M 图像、18k 类别）上预训练，然后 fine-tune 到各下游任务。以下数据来自论文 Table 2（所有模型在 448 分辨率 fine-tune）。

| 模型 | 预训练数据 | ImageNet Top-1 | 推理吞吐 (img/s/core) |
|-----|-----------|---------------|---------------------|
| HaloNet [Vaswani et al., 2021] | ImageNet-21k | 85.8% | 120 |
| **Mixer-L/16** | ImageNet-21k | **84.15%** | **105** |
| ViT-L/16 [Dosovitskiy et al., 2020] | ImageNet-21k | 85.30% | 32 |
| BiT-R152x4 [Kolesnikov et al., 2020] | ImageNet-21k | 85.39% | 26 |
| NFNet-F4+ [Brock et al., 2021] | JFT-300M | 89.2% | 46 |
| **Mixer-H/14** | JFT-300M | **87.94%** | **40** |
| BiT-R152x4 | JFT-300M | 87.54% | 26 |
| ViT-H/14 | JFT-300M | 88.55% | 15 |

（数据来源：论文 Table 2，所有 Mixer 模型在 448 分辨率 fine-tune，三次运行平均标准差 < 0.1）

**关键发现**：

1. **ImageNet-21k 上 Mixer-L/16 略低于 ViT-L/16**：84.15% vs 85.30%（约 1.1 个百分点差距）
2. **JFT-300M 上差距缩小**：Mixer-H/14 (87.94%) 距 ViT-H/14 (88.55%) 仅 0.6 个百分点
3. **推理吞吐优势显著**：Mixer-L/16 是 105 img/s/core，同级的 ViT-L/16 只有 32。论文将此归因于 MLP 的矩阵乘法比注意力计算更适合硬件加速
4. **VTA 迁移表现**：Mixer-L/16 在 VTAB-1k 上达到 74.95，甚至超过 ViT-L/16 的 72.72

**消融实验结果**（论文未编号的消融部分 + Supplementary A.1）：
- 去掉 token-mixing MLP → 准确率大幅下降（约 -4.3%）
- 去掉 channel-mixing MLP → 类似幅度下降（约 -4.9%）
- 两个 MLP 缺一不可，说明空间混合和通道混合都是必要的
- 位置编码 → 无增益或略降
- 用 1×1 卷积替代 channel-mixing → 结果几乎相同，验证了两者的等价性

## 七、位置

**前驱工作**：
- **[ViT, Dosovitskiy et al., 2020]**：首次将纯 Transformer 应用于图像分类，证明了注意力可以替代卷积。MLP-Mixer 直接继承了 ViT 的 patch 提取和投影方式
- **[ResMLP, Touvron et al., 2021]**：几乎同时提出的纯 MLP 架构，用仿射变换替代 LayerNorm，思路异曲同工

**同期竞品**：
- **gMLP [Liu et al., 2021]**：Google 同一团队，在空间 MLP 中引入门控机制来增强表达能力
- **FNet [Lee-Thorp et al., 2021]**：用傅里叶变换替代自注意力，思路更简洁，参数量更少

**后续影响**：
- **MetaFormer [Yu et al., 2022]**：提出 Transformer/MLP 的成功可能主要来自"架构池"（残差连接 + LayerNorm + token/channel mixing），而非特定的 token-mixing 操作——用平均池化替代注意力也有效
- **ConvNeXt [Liu et al., 2022]**：将 CNN 现代化，采用了深度可分离卷积（= token-mixing + channel-mixing 的 CNN 版）
- **PoolFormer [Yu et al., 2022]**：进一步验证，用简单的平均池化替代 token-mixing MLP 也能达到不错的效果
- **Swin Transformer [Liu et al., 2021]** 和 **ConvMixer [Trockman et al., 2022]** 都在不同程度上吸收了"分离空间混合和通道混合"的设计范式

MLP-Mixer 最持久的贡献是提供了一个"架构分离"的视角。这个设计范式后来被几乎所有现代视觉架构（无论基于 CNN、Transformer 还是 SSM）所采纳。

## 八、局限

1. **性能天花板低于 ViT 和顶级 CNN**：在相同数据和规模下，MLP-Mixer 的准确率始终低于 ViT。论文明确承认这一点，强调其目标是"证明 MLP 也能用"而非"MLP 更好"。

2. **输入分辨率必须固定**：Token-mixing MLP 的权重矩阵大小与 patch 数量 $S$ 绑定。Fine-tuning 时若改变分辨率，论文采用权重块对角插值（block-diagonal copy）的技巧来扩展矩阵，但只支持 $S'=K^2S$ 的缩放倍数。

3. **需要更多数据才能充分释放潜力**：在 ImageNet-21k 上 Mixer-L/16 比 ViT-L/16 差约 1%，但在 JFT-300M 上差距缩小到 0.6%。这说明 MLP 的固定混合矩阵需要更多数据才能学到有效的空间混合模式。对于数据有限的任务，MLP-Mixer 不是最佳选择。

## 九、小结

MLP-Mixer 的回答是：**卷积不需要，注意力也不需要，纯 MLP 就够了。**

它不是最强的（性能不如 ViT 和顶级 CNN），但它是最简单的之一。它用最基础的操作——矩阵乘法、转置、非线性——拼出了一个在 ImageNet 上能达到 87.94% top-1 准确率的视觉模型。这个结果告诉我们两件事：第一，视觉任务的"复杂度"可能没有想象中那么高；第二，"空间混合 + 通道混合"分离的设计范式是真正本质的东西，具体用什么操作（卷积、注意力、MLP、甚至池化）反而是次要的。

MLP-Mixer 的代码已开源在 [google-research/vision_transformer](https://github.com/google-research/vision_transformer)（JAX/Flax 实现）。
