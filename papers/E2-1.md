# MLP-Mixer: An all-MLP Architecture for Vision（2021）

**论文：** [arXiv](https://arxiv.org/abs/2105.01601) · Ilya Tolstikhin, Neil Houlsby, Alexander Kolesnikov, Lucas Beyer, Xiaohua Zhai, Thomas Unterthiner, Jessica Yung, Andreas Steiner, Daniel Keysers, Jakob Uszkoreit, Mario Lucic, Alexey Dosovitskiy · 2021

---

## 一、先搞清楚坑在哪

2021 年的视觉领域，卷积神经网络（CNN）几乎统治了一切。从 2012 年的 AlexNet 到 2020 年的 EfficientNet，CNN 架构的进化史就是视觉识别能力的进化史。但就在同一年，Vision Transformer（ViT）横空出世，证明了纯 Transformer 架构（即只有自注意力机制，没有卷积）也能在图像分类上达到 SOTA 水平。

这就引出了一个根本性的问题：**CNN 的自带归纳偏置（局部性、平移不变性、层级特征）到底是不是视觉任务必需的？** ViT 告诉我们"不是必需的"，但它引入了一个新的复杂组件——自注意力机制。那么，**注意力机制本身是不是必需的？**

MLP-Mixer 的答案更加激进：**连注意力都不需要，纯 MLP 就够了。**

## 二、CNN 和 Transformer 的真正问题

### 2.1 CNN 的问题

CNN 的核心操作是卷积核在空间维度上滑动。这个操作隐含了两个强假设：
1. **局部性**：像素之间的相关性随着距离增加而衰减
2. **平移不变性**：一个模式出现在哪里不重要，重要的是它出现了

这两个假设在大多数视觉任务中确实成立，但它们在建模**长距离依赖**时效率很低。要扩大感受野，CNN 只能通过堆叠更多层（深层网络）或使用更大的卷积核（如 7×7），但前者需要极深的网络，后者参数量爆炸。

### 2.2 Transformer（ViT）的问题

ViT 把图像切成 patch，用自注意力机制在 patch 之间建立全局连接。这解决了 CNN 长距离依赖的问题，但自注意力机制的计算复杂度是 $O(N^2)$，其中 $N$ 是 patch 数量。对于高分辨率图像，这个计算开销是不可接受的。

更微妙的是，ViT 的**归纳偏置更少**，这意味着它需要更多的数据来训练。Google 团队在 ViT 论文中明确指出：如果不在 JFT-300M 这种超大规模数据集上预训练，ViT 的效果还不如同等级别的 CNN。

## 三、MLP-Mixer 的核心思路

**一句话总结**：用两个交替的 MLP 层分别处理空间维度和通道维度，完全替代卷积和自注意力。

更具体地说，MLP-Mixer 把图像处理拆成两个正交的操作：
1. **通道混合（Channel-mixing）**：在每个空间位置内部，混合不同通道的信息（类似于 1×1 卷积）
2. **空间混合（Space-mixing）**：在不同空间位置之间，混合相同通道的信息（类似于深度可分离卷积的 depthwise 操作）

这两个操作交替进行，构成了 Mixer 层的基础。

## 四、网络架构详解

### 4.1 整体架构图

```
输入图像 (H x W x 3)
    │
    ▼
Patch提取 + 线性投影 (将图像切成 S×S 个 patch，每个 patch 展平后投影到 C 维)
    │
    ▼
Patch 序列 (S x C)，其中 S = H*W / (patch_size^2)
    │
    ▼
┌─────────────────────────────────────────────────┐
│              Mixer Layer (重复 L 次)              │
│                                                 │
│   ┌───────────────┐    ┌───────────────┐        │
│   │ Layer Norm    │    │ Layer Norm    │        │
│   │      ↓        │    │      ↓        │        │
│   │ Token-mixing  │    │ Channel-mixing│        │
│   │ MLP (空间混合) │    │ MLP (通道混合) │        │
│   │      ↓        │    │      ↓        │        │
│   │ 残差连接 +     │    │ 残差连接 +     │        │
│   └───────────────┘    └───────────────┘        │
└─────────────────────────────────────────────────┘
    │
    ▼
全局平均池化 (Global Average Pooling)
    │
    ▼
分类头 (MLP → 类别数)
    │
    ▼
类别预测
```

### 4.2 前向过程详解

**输入**：图像 $X \in \mathbb{R}^{H \times W \times 3}$

**Step 1: Patch 提取和投影**

将图像切成不重叠的 patch，每个 patch 的大小为 $P \times P$。patch 数量为：
$$S = \frac{H}{P} \times \frac{W}{P}$$

每个 patch 展平后通过一个线性投影（全连接层）映射到 $C$ 维：
$$U = [u_1, u_2, ..., u_S]^T \in \mathbb{R}^{S \times C}$$

这里的 $U$ 可以看作是一个 $S \times C$ 的矩阵，$S$ 行表示 $S$ 个空间位置，$C$ 列表示 $C$ 个通道。

**Step 2: Mixer Layer（重复 L 次）**

每个 Mixer 层由两个子层组成：

**子层 1: Token-mixing MLP（空间混合）**

输入：$U \in \mathbb{R}^{S \times C}$

先做 Layer Normalization（在通道维度上做）：
$$U' = \text{LayerNorm}(U)$$

然后对 $U'$ 进行转置：$U'^T \in \mathbb{R}^{C \times S}$

为什么转置？因为我们要在**空间维度**上做 MLP。$U'^T$ 的每一行对应一个通道在所有空间位置上的值，形状为 $1 \times S$。对这个 $1 \times S$ 的向量做 MLP，就是在混合不同空间位置的信息。

Token-mixing MLP 的结构：
$$\text{MLP}(x) = \text{GELU}(x \cdot W_1 + b_1) \cdot W_2 + b_2$$

其中 $W_1 \in \mathbb{R}^{S \times S_{expand}}$，$W_2 \in \mathbb{R}^{S_{expand} \times S}$，$S_{expand}$ 是 MLP 的隐藏层大小（通常 $S_{expand} > S$，比如是 S 的 4 倍）。

做完 MLP 后，再转置回来：
$$U_{mixed} = \text{TokenMLP}(U'^T)^T \in \mathbb{R}^{S \times C}$$

最后加上残差连接：
$$U \leftarrow U + U_{mixed}$$

**子层 2: Channel-mixing MLP（通道混合）**

输入：$U \in \mathbb{R}^{S \times C}$

先做 Layer Normalization：
$$U' = \text{LayerNorm}(U)$$

然后在通道维度上做 MLP。注意这里**不需要转置**，因为 $U'$ 的每一行对应一个空间位置在所有通道上的值，形状为 $1 \times C$。

Channel-mixing MLP 的结构：
$$\text{MLP}(x) = \text{GELU}(x \cdot W_1 + b_1) \cdot W_2 + b_2$$

其中 $W_1 \in \mathbb{R}^{C \times C_{expand}}$，$W_2 \in \mathbb{R}^{C_{expand} \times C}$，$C_{expand}$ 是 MLP 的隐藏层大小（通常是 C 的 4 倍）。

加上残差连接：
$$U \leftarrow U + \text{ChannelMLP}(U')$$

**Step 3: 全局平均池化和分类**

经过 L 个 Mixer 层后，对 $U$ 在空间维度上做全局平均池化：
$$y = \frac{1}{S} \sum_{i=1}^{S} U_i \in \mathbb{R}^{C}$$

最后通过一个分类头（一个全连接层）输出类别预测：
$$\hat{y} = \text{FC}(y) \in \mathbb{R}^{N_{classes}}$$

### 4.3 为什么这样设计？

**Token-mixing MLP 替代了自注意力**：在 ViT 中，自注意力机制计算所有 patch 对之间的相关性，复杂度 $O(S^2)$。Token-mixing MLP 也做类似的事情——它让每个空间位置的信息与其他所有空间位置的信息混合，但通过一个固定的 MLP 而不是数据相关的注意力权重。这相当于用**静态的、可学习的混合权重**替代了**动态的、输入相关的注意力权重**。

**Channel-mixing MLP 替代了 1×1 卷积**：在每个空间位置内部混合通道信息，这和 1×1 卷积做的事情完全一样。实际上，Channel-mixing MLP 等价于一个 1×1 卷积 + 一个非线性激活函数。

**两个操作的分离是关键**：在 CNN 中，标准卷积同时混合空间和通道信息（每个卷积核同时跨越空间和通道维度）。在 ViT 中，自注意力也在混合空间信息（注意力权重）的同时通过 value 投影混合通道信息。MLP-Mixer 把这两个操作完全解耦，让每个 MLP 只做一件事。

## 五、核心创新点

### 5.1 纯 MLP 架构

**（a）痛点与动机**

在 MLP-Mixer 之前，视觉领域的 SOTA 模型要么基于卷积（CNN），要么基于自注意力（ViT）。这两种架构都包含复杂的操作：
- CNN 需要精心设计的卷积核、池化层、归一化层等
- ViT 需要多头自注意力、位置编码、分类 token 等

直觉上，如果连最简单的 MLP 都能达到可比的效果，那说明视觉任务可能不需要那么复杂的归纳偏置。

**（b）方案细节**

整个网络只有两种操作：MLP 和残差连接。没有卷积、没有注意力、没有位置编码。每个 Mixer 层包含两个 MLP，一个在空间维度上操作，一个在通道维度上操作。

**（c）为什么有效**

实验表明，MLP-Mixer 在 ImageNet 上达到了与 ViT 和 CNN 可比的效果。这说明：
1. **卷积和注意力都不是必需的**，它们可以被更简单的操作替代
2. **通道混合和空间混合的分离**是一种有效的设计原则
3. MLP 的**静态权重**（对所有输入都一样）在足够多的数据下可以学到有效的空间混合模式

**（d）与 Related Work 的关系**

- [LeCun 1998] 的 LeNet 使用卷积，是局部连接 + 权重共享
- [Vaswani 2017] 的 Transformer 使用自注意力，是全局连接 + 动态权重
- MLP-Mixer 使用全连接 + 静态权重，是全局连接 + 静态权重

有意思的是，MLP-Mixer 在连接模式上更像 Transformer（全局连接），但在权重模式上更像 CNN（静态权重）。

**（e）如果去掉会怎样**

论文中的消融实验（Table 3）显示：
- 如果去掉 token-mixing MLP（只用 channel-mixing），模型退化为在每个空间位置上独立做 MLP，ImageNet top-1 准确率从 76.4% 降到 72.1%
- 如果去掉 channel-mixing MLP（只用 token-mixing），模型退化为在每个通道上独立做 MLP，准确率降到 71.5%
- 两个 MLP 同时使用，效果最好

### 5.2 无需位置编码

**（a）痛点与动机**

ViT 需要位置编码来告诉模型 patch 在图像中的位置。这是因为自注意力是置换不变的（permutation invariant）——如果把输入 patch 的顺序打乱，自注意力的输出也会相应打乱，但模型无法区分"打乱后的顺序"和"原始顺序"。位置编码提供了空间位置信息。

**（b）方案细节**

MLP-Mixer 的 token-mixing MLP 在空间维度上操作，它的权重 $W_1$ 和 $W_2$ 的维度与 patch 数量 $S$ 相关。这意味着 MLP 的权重矩阵本身就是**位置相关的**——第一个 patch 的输入对应权重矩阵的第一列，第二个 patch 对应第二列，等等。这个固定的权重矩阵隐式地编码了位置信息。

**（c）为什么有效**

不需要位置编码的原因很直接：token-mixing MLP 的权重矩阵 $W \in \mathbb{R}^{S \times S}$ 中的每个元素 $W_{ij}$ 对应从 patch $j$ 到 patch $i$ 的连接强度。这个强度是**可学习的**，并且**依赖于 patch 的索引**。所以 MLP 本身就编码了"哪个 patch 和哪个 patch 有关系"的信息。

相比之下，自注意力机制中的权重是动态计算的（query 和 key 的点积），它本身是置换不变的，所以需要额外的位置编码来打破对称性。

**（d）与 Related Work 的关系**

- [Dosovitskiy 2020] 的 ViT 使用可学习的位置编码或正弦位置编码
- [Wang 2021] 的 MLP-Mixer 证明位置编码可以被 MLP 的固定权重隐式编码

**（e）如果去掉会怎样**

论文中对比了有/无位置编码的变体（Table 3）：
- MLP-Mixer 不加位置编码：76.4%
- MLP-Mixer 加位置编码：76.2%（反而略低）

这说明位置编码对 MLP-Mixer 不仅不是必需的，甚至可能有点干扰。

### 5.3 对称的设计

**（a）痛点与动机**

CNN 和 ViT 的架构设计通常不对称：有专门的 stem 层、下采样层、分类头等。MLP-Mixer 追求极简，所有 Mixer 层结构完全相同。

**（b）方案细节**

每个 Mixer 层包含：
1. Layer Norm → Token-mixing MLP → 残差连接
2. Layer Norm → Channel-mixing MLP → 残差连接

这个结构在所有层中重复，没有任何变化。唯一的例外是第一层（patch 投影）和最后一层（分类头）。

**（c）为什么有效**

对称设计的优势在于**简单**和**可扩展性**。如果所有层结构相同，那么增加层数只是简单的重复，不需要重新设计架构。这也让 MLP-Mixer 更容易实现和调试。

**（d）与 Related Work 的关系**

- [He 2016] 的 ResNet 使用残差连接，但不同层的结构不同（有下采样层）
- [Vaswani 2017] 的 Transformer 使用相同的 block 重复，但每个 block 包含注意力 + FFN
- MLP-Mixer 比 Transformer 更对称：所有 block 完全相同，没有注意力

**（e）如果去掉会怎样**

论文没有做这方面的消融，但直觉上如果打破对称性（比如在浅层用更大的 MLP 隐藏层），可能会提升性能，但会牺牲简洁性。

## 六、公式详解

### 6.1 Patch 投影

**公式**：
$$U = X_{\text{patches}} \cdot W_{\text{proj}} + b_{\text{proj}}$$

**（a）符号定义**
- $X_{\text{patches}} \in \mathbb{R}^{S \times (P^2 \cdot 3)}$：将图像切成 $S$ 个 $P \times P$ 的 patch，每个 patch 展平为 $P^2 \cdot 3$ 维向量（3 是 RGB 通道）
- $W_{\text{proj}} \in \mathbb{R}^{(P^2 \cdot 3) \times C}$：投影矩阵，将 patch 向量映射到 $C$ 维
- $b_{\text{proj}} \in \mathbb{R}^{C}$：偏置项
- $U \in \mathbb{R}^{S \times C}$：投影后的 patch 序列

**（b）公式来源**

这是标准的线性投影，和 ViT 中的 patch embedding 完全一样。实际上，这个操作等价于一个卷积核大小为 $P \times P$、步长为 $P$、输出通道为 $C$ 的卷积层。

**（c）推导过程**

以输入图像 $224 \times 224$、patch 大小 $16 \times 16$ 为例：
- $S = (224/16) \times (224/16) = 14 \times 14 = 196$
- 每个 patch 展平后维度：$16 \times 16 \times 3 = 768$
- $W_{\text{proj}} \in \mathbb{R}^{768 \times C}$，其中 $C$ 是模型宽度（如 512 或 768）
- 输出 $U \in \mathbb{R}^{196 \times C}$

**（d）直觉理解**

这个公式做的事情就是把图像切成小块，然后把每个小块"翻译"成一个固定长度的向量。$W_{\text{proj}}$ 学习的是"什么样的像素模式对应什么样的特征向量"。

**（e）边界情况**
- 当 $P = H = W$ 时，$S = 1$，整个图像变成一个 patch，MLP-Mixer 退化为一个标准的 MLP
- 当 $P = 1$ 时，$S = H \times W$，每个像素是一个 patch，此时 token-mixing MLP 的权重矩阵大小为 $(H \times W) \times (H \times W)$，计算量极大

### 6.2 Token-mixing MLP

**公式**：
$$U_{\text{token}} = \text{MLP}_{\text{token}}(\text{LayerNorm}(U)^T)^T$$

其中：
$$\text{MLP}_{\text{token}}(x) = \text{GELU}(x \cdot W_1 + b_1) \cdot W_2 + b_2$$

**（a）符号定义**
- $U \in \mathbb{R}^{S \times C}$：输入
- $\text{LayerNorm}(U) \in \mathbb{R}^{S \times C}$：在最后一维（通道维度）上做 LayerNorm
- $U' = \text{LayerNorm}(U)^T \in \mathbb{R}^{C \times S}$：转置后，每一行对应一个通道在所有空间位置上的值
- $W_1 \in \mathbb{R}^{S \times S_{\text{expand}}}$：第一个全连接层的权重
- $W_2 \in \mathbb{R}^{S_{\text{expand}} \times S}$：第二个全连接层的权重
- $b_1 \in \mathbb{R}^{S_{\text{expand}}}$，$b_2 \in \mathbb{R}^{S}$：偏置项
- $S_{\text{expand}}$：MLP 隐藏层大小，通常 $S_{\text{expand}} = 4 \times S$

**（b）公式来源**

这是标准的两层 MLP，但应用在转置后的数据上。GELU 激活函数来自 [Hendrycks 2016]。

**（c）推导过程**

以 $S = 196$，$C = 768$，$S_{\text{expand}} = 4 \times 196 = 784$ 为例：

1. 输入 $U \in \mathbb{R}^{196 \times 768}$
2. LayerNorm：$U_{\text{norm}} \in \mathbb{R}^{196 \times 768}$
3. 转置：$U_{\text{norm}}^T \in \mathbb{R}^{768 \times 196}$
4. 第一层全连接：$U_{\text{norm}}^T \cdot W_1$，其中 $W_1 \in \mathbb{R}^{196 \times 784}$，输出 $\in \mathbb{R}^{768 \times 784}$
5. GELU 激活：输出 $\in \mathbb{R}^{768 \times 784}$
6. 第二层全连接：$\text{GELU}(\cdot) \cdot W_2$，其中 $W_2 \in \mathbb{R}^{784 \times 196}$，输出 $\in \mathbb{R}^{768 \times 196}$
7. 转置回来：输出 $\in \mathbb{R}^{196 \times 768}$

**（d）直觉理解**

这个 MLP 的输入是 $C$ 个长度为 $S$ 的向量，每个向量对应一个通道在所有空间位置上的值。MLP 学习的是如何混合这些空间位置的信息，即"对于某个通道，哪些空间位置应该被组合在一起"。

注意这个 MLP 的权重矩阵 $W_1$ 和 $W_2$ 的维度与 $S$ 相关，这意味着**所有通道共享同一个空间混合模式**。也就是说，不管是什么通道（R、G、B 或更高层的特征），空间混合的方式是一样的。

**（e）边界情况**
- 当 $S = 1$ 时，token-mixing MLP 的输入是 $C \times 1$ 的向量，MLP 无法混合任何空间信息，退化为恒等映射
- 当 $S$ 很大时（如 $196$），$W_1 \in \mathbb{R}^{196 \times 784}$ 的参数量为 $196 \times 784 \approx 154K$，这是可接受的
- 但如果 $S$ 非常大（如 $4096$），$W_1$ 的参数量会达到 $4096 \times 16384 \approx 67M$，变得不可接受。这就是为什么 MLP-Mixer 需要固定 patch 大小，不能处理任意分辨率的图像

### 6.3 Channel-mixing MLP

**公式**：
$$U_{\text{channel}} = \text{MLP}_{\text{channel}}(\text{LayerNorm}(U))$$

其中：
$$\text{MLP}_{\text{channel}}(x) = \text{GELU}(x \cdot W_1 + b_1) \cdot W_2 + b_2$$

**（a）符号定义**
- $U \in \mathbb{R}^{S \times C}$：输入
- $W_1 \in \mathbb{R}^{C \times C_{\text{expand}}}$：第一个全连接层的权重
- $W_2 \in \mathbb{R}^{C_{\text{expand}} \times C}$：第二个全连接层的权重
- $C_{\text{expand}}$：MLP 隐藏层大小，通常 $C_{\text{expand}} = 4 \times C$

**（b）公式来源**

同样来自标准的两层 MLP。注意这里的 MLP 直接在原始数据上操作，不需要转置。

**（c）推导过程**

以 $S = 196$，$C = 768$，$C_{\text{expand}} = 4 \times 768 = 3072$ 为例：

1. 输入 $U \in \mathbb{R}^{196 \times 768}$
2. LayerNorm：$U_{\text{norm}} \in \mathbb{R}^{196 \times 768}$
3. 第一层全连接：$U_{\text{norm}} \cdot W_1$，其中 $W_1 \in \mathbb{R}^{768 \times 3072}$，输出 $\in \mathbb{R}^{196 \times 3072}$
4. GELU 激活：输出 $\in \mathbb{R}^{196 \times 3072}$
5. 第二层全连接：$\text{GELU}(\cdot) \cdot W_2$，其中 $W_2 \in \mathbb{R}^{3072 \times 768}$，输出 $\in \mathbb{R}^{196 \times 768}$

**（d）直觉理解**

这个 MLP 的输入是 $S$ 个长度为 $C$ 的向量，每个向量对应一个空间位置在所有通道上的值。MLP 学习的是如何混合这些通道的信息，即"对于某个空间位置，哪些通道应该被组合在一起"。

注意这个 MLP 的权重矩阵 $W_1$ 和 $W_2$ 的维度与 $C$ 相关，这意味着**所有空间位置共享同一个通道混合模式**。这和 CNN 中的 1×1 卷积完全一样。

**（e）边界情况**
- 当 $C = 1$ 时，channel-mixing MLP 无法混合任何通道信息
- 当 $C$ 很大时（如 $768$），$W_1 \in \mathbb{R}^{768 \times 3072}$ 的参数量为 $768 \times 3072 \approx 2.36M$，这部分是 MLP-Mixer 参数量的主要来源

### 6.4 参数量计算

**公式**：
$$\text{Params} = \underbrace{(P^2 \cdot 3 + 1) \cdot C}_{\text{patch 投影}} + L \cdot \left( \underbrace{2 \cdot S \cdot S_{\text{expand}} + S_{\text{expand}} + S}_{\text{token-mixing MLP}} + \underbrace{2 \cdot C \cdot C_{\text{expand}} + C_{\text{expand}} + C}_{\text{channel-mixing MLP}} \right) + \underbrace{C \cdot N_{\text{classes}} + N_{\text{classes}}}_{\text{分类头}}$$

**（a）符号定义**
- $P$：patch 大小
- $C$：模型宽度（通道数）
- $S$：patch 数量
- $L$：Mixer 层数
- $S_{\text{expand}}$：token-mixing MLP 隐藏层大小
- $C_{\text{expand}}$：channel-mixing MLP 隐藏层大小
- $N_{\text{classes}}$：类别数

**（b）公式来源**

这是标准 MLP 的参数计算公式。每个全连接层的参数为 $(\text{输入维度} \times \text{输出维度}) + \text{输出维度}$（权重 + 偏置）。

**（c）推导过程**

以 MLP-Mixer-B/16 为例（$P=16$，$C=768$，$S=196$，$L=12$，$S_{\text{expand}}=4 \times 196 = 784$，$C_{\text{expand}}=4 \times 768 = 3072$，$N_{\text{classes}}=1000$）：

1. Patch 投影：$(16 \times 16 \times 3 + 1) \times 768 = 768 \times 768 + 768 = 590,592$
2. 每个 Mixer 层：
   - Token-mixing MLP：$2 \times 196 \times 784 + 784 + 196 = 307,328 + 980 = 308,308$
   - Channel-mixing MLP：$2 \times 768 \times 3072 + 3072 + 768 = 4,718,592 + 3,840 = 4,722,432$
   - 每层总参数：$308,308 + 4,722,432 = 5,030,740$
3. 12 层总参数：$12 \times 5,030,740 = 60,368,880$
4. 分类头：$768 \times 1000 + 1000 = 769,000$
5. 总参数量：$590,592 + 60,368,880 + 769,000 = 61,728,472 \approx 59M$

**（d）直觉理解**

从推导可以看出，参数量的主要来源是 channel-mixing MLP（占总参数的约 92%）。这是因为 $C$ 远大于 $S$（768 vs 196），所以 $C \times C_{\text{expand}}$ 远大于 $S \times S_{\text{expand}}$。

这也意味着 MLP-Mixer 的参数量主要由**通道维度**决定，而不是空间维度。这和 CNN 类似（通道数越深，参数越多），但与 ViT 不同（ViT 的注意力部分参数量与 $S^2$ 成正比）。

**（e）边界情况**
- 如果增大 patch 大小 $P$，$S$ 减小，token-mixing MLP 的参数量减少，但 patch 投影的参数量不变
- 如果增大 $C$，channel-mixing MLP 的参数量以 $O(C^2)$ 增长，这是主要的计算瓶颈
- 如果增大 $L$，总参数量线性增长

## 八、代码对照

### 8.1 官方实现（JAX/Flax）

Google 官方实现地址：https://github.com/google-research/vision_transformer

核心代码在 `vit_jax/models.py` 中，MLP-Mixer 的实现非常简洁：

```python
class MlpBlock(nn.Module):
  mlp_dim: int
  
  @nn.compact
  def __call__(self, x):
    y = nn.Dense(self.mlp_dim)(x)
    y = nn.gelu(y)
    y = nn.Dense(x.shape[-1])(y)
    return y

class MixerBlock(nn.Module):
  """Mixer block."""
  tokens_mlp_dim: int
  channels_mlp_dim: int
  
  @nn.compact
  def __call__(self, x):
    y = nn.LayerNorm()(x)
    y = jnp.swapaxes(y, -2, -1)  # 转置：S x C → C x S
    y = MlpBlock(self.tokens_mlp_dim, name='token_mixing')(y)
    y = jnp.swapaxes(y, -2, -1)  # 转置回来：C x S → S x C
    x = x + y
    y = nn.LayerNorm()(x)
    y = MlpBlock(self.channels_mlp_dim, name='channel_mixing')(y)
    x = x + y
    return x
```

### 8.2 代码中的关键细节

**1. `jnp.swapaxes` 实现转置**

代码用 `jnp.swapaxes(x, -2, -1)` 来实现 $S \times C$ 到 $C \times S$ 的转置。这是 JAX 中的数组轴交换操作。

**2. MlpBlock 是共享的**

在 `MixerBlock` 中，`token_mixing` 和 `channel_mixing` 是独立的 `MlpBlock` 实例，但每个 `MlpBlock` 内部的参数是共享的（所有通道共享同一个 token-mixing MLP，所有空间位置共享同一个 channel-mixing MLP）。

**3. 没有显式的位置编码**

代码中没有任何位置编码的添加，这验证了论文的声明。

**4. LayerNorm 的位置**

LayerNorm 在每个 MLP 之前应用（Pre-LN 结构），而不是之后。这和 Transformer 的 Pre-LN 变体一致。

**5. 残差连接**

每个子层都有残差连接，和 ResNet 一样。

### 8.3 与论文描述的一致性和差异

**一致性**：
- 架构完全匹配：token-mixing MLP → channel-mixing MLP 的顺序
- MLP 隐藏层大小是输入大小的 4 倍
- 使用 GELU 激活函数

**差异**：
- 论文中没有明确说明 LayerNorm 的具体实现（是 Pre-LN 还是 Post-LN），代码显示是 Pre-LN
- 论文中没有提到 `jnp.swapaxes` 的具体实现细节

## 九、位置

### 9.1 前驱工作

1. **Vision Transformer (ViT)** [Dosovitskiy et al., 2020]：首次将纯 Transformer 应用于图像分类，证明了自注意力可以替代卷积。MLP-Mixer 直接继承了 ViT 的 patch 提取和投影方法，以及残差连接和 LayerNorm 的使用。

2. **ResMLP** [Touvron et al., 2021]：几乎与 MLP-Mixer 同时提出的纯 MLP 架构。ResMLP 也使用两个 MLP 分别处理空间和通道维度，但 ResMLP 使用仿射变换（affine transformation）替代 LayerNorm，并且在空间 MLP 中使用了更复杂的门控机制。

3. **ConvMixer** [Trockman et al., 2022]：后续工作，证明深度可分离卷积（depthwise convolution + pointwise convolution）可以达到与 MLP-Mixer 类似的效果。这从另一个角度验证了"空间混合 + 通道混合"的设计范式。

### 9.2 同期竞品

1. **ResMLP** [Touvron et al., 2021]：Facebook AI 团队提出，与 MLP-Mixer 几乎同时。两者的核心思路相同，但在具体实现上有差异（ResMLP 使用仿射变换，MLP-Mixer 使用 LayerNorm）。

2. **gMLP** [Liu et al., 2021]：Google 团队提出，在 MLP 中引入门控机制（gating）来增强表达能力。gMLP 在空间 MLP 中使用了类似自注意力的门控结构。

3. **FNet** [Lee-Thorp et al., 2021]：使用傅里叶变换替代自注意力，证明更简单的线性操作也能达到类似效果。

### 9.3 后续影响

1. **直接扩展**：
   - **Swin Transformer** [Liu et al., 2021]：虽然使用了自注意力，但其窗口划分和移动策略受到了 MLP-Mixer 的"局部 vs 全局"分离思路的影响
   - **MetaFormer** [Yu et al., 2022]：提出 Transformer/MLP 的成功可能主要来自架构（残差连接、LayerNorm、token-mixing + channel-mixing），而不是特定的 token-mixing 操作

2. **架构改进**：
   - **ConvNeXt** [Liu et al., 2022]：将 CNN 现代化，采用了与 MLP-Mixer 类似的"深度可分离卷积 + 大卷积核"设计
   - **PoolFormer** [Yu et al., 2022]：用简单的平均池化替代 token-mixing MLP，进一步验证了"token-mixing 操作可以是简单的池化"

3. **范式融合**：
   - **MLP-Mixer 的"空间混合 + 通道混合"范式被广泛采用**，成为现代视觉架构的标准设计模式
   - **S4** [Gu et al., 2021] 和 **Mamba** [Gu & Dao, 2023]：在序列建模中使用状态空间模型（SSM），其设计思路与 MLP-Mixer 的"分离空间和通道混合"有异曲同工之处

4. **框架采用**：
   - **timm 库** [Wightman, 2019]：包含了 MLP-Mixer 的 PyTorch 实现，作为标准模型之一
   - **Hugging Face Transformers**：也包含了 MLP-Mixer 的实现

## 十、局限

### 10.1 性能不如 ViT 和 CNN

**局限**：在相同规模和相同预训练数据下，MLP-Mixer 的准确率略低于 ViT 和 EfficientNet。

**何时重要**：当追求极致性能时（如竞赛、高精度要求的应用），MLP-Mixer 不是最佳选择。

**论文是否承认**：是的，论文中明确承认"MLP-Mixer 在 ImageNet 上的表现不如 ViT"。

**后续工作**：ConvNeXt [Liu et al., 2022] 通过改进 CNN 架构达到了更好的效果，但 MLP-Mixer 的"分离空间和通道混合"思路被保留。

### 10.2 无法处理任意分辨率

**局限**：Token-mixing MLP 的权重矩阵大小与 patch 数量 $S$ 相关，因此输入图像的尺寸必须固定。这限制了 MLP-Mixer 在处理不同分辨率图像时的灵活性。

**何时重要**：当需要在训练和推理时使用不同分辨率时（如目标检测、语义分割等密集预测任务）。

**论文是否承认**：论文中提到了这个限制，并指出这是一个未来工作方向。

**后续工作**：Swin Transformer [Liu et al., 2021] 通过引入层次化结构和窗口注意力解决了这个问题，但 MLP-Mixer 本身没有很好的解决方案。

### 10.3 需要更多数据

**局限**：由于 MLP-Mixer 的归纳偏置更少（没有卷积的局部性假设，没有注意力的动态权重），它需要更多的数据才能达到与 CNN 和 ViT 相当的性能。

**何时重要**：当数据量有限时（如医学图像、小样本学习），MLP-Mixer 可能不如 CNN 有效。

**论文是否承认**：是的，论文中的实验表明，在 ImageNet-21k 上预训练时，MLP-Mixer 与 ViT 的差距较大（约 4-5%），但在 JFT-300M 上预训练后，差距缩小到 1-2%。

**后续工作**：ConvMixer [Trockman et al., 2022] 通过引入深度可分离卷积（更强的归纳偏置）在更少的数据上取得了更好的效果。

### 10.4 计算效率问题

**局限**：虽然 MLP-Mixer 的参数量比 ViT 少，但它的计算量（FLOPs）并不一定更少。Channel-mixing MLP 的计算复杂度是 $O(S \cdot C^2)$，对于大 $C$ 来说计算量很大。

**何时重要**：当推理速度是关键约束时（如移动端部署、实时应用）。

**论文是否承认**：论文中没有详细讨论计算效率问题。

## 十一、小结

MLP-Mixer 的核心贡献是证明了**纯 MLP 架构在视觉任务上也能达到接近 SOTA 的效果**。这个结果打破了两个"常识"：
1. 视觉任务需要卷积（CNN 的局部连接假设）
2. 视觉任务需要注意力（Transformer 的动态权重假设）

MLP-Mixer 告诉我们，更简单的操作——静态的、全连接的 MLP——在足够多的数据下也能学到有效的视觉表征。

从更广的角度看，MLP-Mixer 开启了"极简视觉架构"的研究方向。后续的 ResMLP、gMLP、ConvMixer、MetaFormer 等工作都在探索：**到底什么才是视觉架构中真正必不可少的组件？** 这个问题的答案至今仍在争论中，但 MLP-Mixer 无疑是一个重要的里程碑。
