## 一、先搞清楚"问题在哪里"

Vision Transformer（ViT）在 2020 年底横空出世时，整个计算机视觉界都受到了冲击。这篇论文（Dosovitskiy et al., 2020）证明了：如果你有足够多的训练数据（比如 3 亿张 JFT-300M 图片），把图像切成 patch 喂进标准的 Transformer encoder，就能在 ImageNet 上追平甚至超过当时最好的 CNN。这意味着自 2012 年 AlexNet 以来统治了视觉领域八年的卷积操作，并不是视觉任务"必需"的归纳偏置。

但 ViT 的出现也带来了一个更深层的追问。ViT 的成功依赖两个核心组件：**自注意力（self-attention）** 和 **MLP（多层感知机）**。自注意力负责让不同 patch 之间的信息互相交流——所谓 "token mixing"。MLP 负责在同一个 token 的不同 channel 之间做非线性变换——所谓 "channel mixing"。但问题是，自注意力真的是 token mixing 的唯一有效方式吗？

2021 年初，这个问题的答案已经初现端倪。Swin Transformer（Liu et al., 2021）把窗口限定的局部注意力和层级化设计引入 Transformer，在 ImageNet 上取得了显著提升。CaiT（Touvron et al., 2021）在 DeiT 的基础上增加了 class-attention 层。这些工作都是在 attention 框架内部做文章——改进 attention 的计算方式、减少 attention 的复杂度、或者改变 attention 的组织结构。

但有一个更激进的问题一直悬而未决：**如果我们彻底去掉 attention，只用 MLP 去做 token mixing，还能不能行？**

这个问题本质上是在拷问 ViT 成功的根本原因。ViT 之所以有效，是因为 attention 这个机制本身有多重要，还是因为"把图像切 patch + 在大规模数据上训练 + 足够的模型容量"才是真正的原因？如果 MLP 就能搞定 token mixing，那就说明 attention 的特有属性——pairwise 交互、动态权重、内容相关性——在视觉任务中可能并不是关键。关键可能只是"让不同空间位置的信息能够互相交流"这个更基础的操作，而具体怎么交流、用什么机制，是次要的。

MLP-Mixer 就是来回答这个问题的。它的结论足够直接，也足够挑衅：**Attention 和 convolution 都不是必需的，只用 MLP 就能达到竞争性能。**

## 二、旧路线的真正问题

要理解 MLP-Mixer 的动机，需要重新审视 2020-2021 年视觉模型设计的核心思路。

### 2.1 CNN 路线的隐含假设

CNN 的核心操作是卷积。一个 $3\times3$ 的卷积核在空间维度上滑动，同时在所有 channel 上累加。这个设计隐含了三个重要的归纳偏置（inductive bias）：

1. **局部性**：输出像素只与输入中一小块邻域有关。
2. **平移不变性**：同样的卷积核在图像的所有位置共享参数。
3. **层级化**：通过池化或 stride 卷积构建特征金字塔。

这些偏置在 ImageNet 级的数据量上（百万级图片）非常有效，它们限制了模型需要学习的自由度，让 CNN 可以在中等规模数据上泛化得很好。但当数据规模增大到数亿张图片时，这些偏置反而可能成为上限——它们限制了模型学到更长距离、更灵活的空间模式的能力。

### 2.2 ViT 路线的隐含假设

ViT 抛弃了 CNN 的局部性和层级化偏置，直接用全局注意力让所有 patch 之间可以做 pairwise 交互。这让模型具备了更大的灵活性——它可以学习任意复杂的空间关系，不需要被局部邻域限制。

但 ViT 引入了另一个隐含假设：**自注意力是让空间位置之间信息交互的最佳方式**。自注意力的特性包括：

- **内容相关的权重**：每个 query 和 key 的相似度决定了信息流动的强度。
- **全局感受野**：所有位置之间都能互相看到。
- **动态计算**：权重不是固定的，而是随输入内容变化。
- **二次复杂度**：计算量和显存随 token 数平方增长。

ViT 的成功让人们不自觉地接受了这个假设：如果你要做全局的 token mixing，自注意力就是默认选择。MLP-Mixer 的论文问的就是：**这个假设真的成立吗？**

### 2.3 真正的瓶颈在哪里

回顾一下 2021 年之前 ViT 的已知问题：

- **二次复杂度**：ViT 的 self-attention 在 $O(S^2)$ 的复杂度（$S$ 是 patch 数），当输入分辨率增加时，计算量急剧增长。
- **位置编码**：ViT 需要额外的 absolute position embeddings（绝对位置编码）来告诉模型 patch 的顺序——attention 本身是 permutation-invariant（排列不变）的，它不知道 patch 谁先谁后。
- **数据饥渴**：ViT 在没有大规模预训练时，性能显著低于同等规模的 CNN。

前两个问题直接和 attention 机制相关。如果 token mixing 可以采用其他机制，也许能同时解决复杂度问题和位置编码问题。第三个问题——数据饥渴——则更像是 ViT 缺少 CNN 式归纳偏置的副作用。如果 MLP 能提供合适的偏置，也许能让模型在中等规模数据上也更有效。

这引出了 MLP-Mixer 的设计动机：**既然 token mixing 的本质只是"让空间位置之间交换信息"，那我们为什么不能用更简单的机制来实现它？**

有意思的是，从某种角度来看，MLP 在 token mixing 中的角色可以理解为"一个特殊的 CNN"：它等价于一个 kernel size 等于整个特征图大小、且所有 channel 共享权重参数的 depthwise convolution（深度可分离卷积）。这个观察非常重要——它暗示了 MLP-Mixer 并非与卷积路线彻底割裂，而是把卷积的一个特例做到了极致。

## 三、这篇论文的核心思路

**一句话核心思路：把图像建模为 $S\times C$ 的二维表格（$S$ 个空间位置，$C$ 个特征通道），分别用两种 MLP 在表格的行方向和列方向上做信息混合——token-mixing MLP 作用于列（spatial 维度），channel-mixing MLP 作用于行（feature 维度），两者交替堆叠即可实现媲美 attention 和 convolution 的特征学习能力。**

这个设计突破了两个根深蒂固的假设：

1. **Attention 不是 token mixing 的唯一选择**：MLP 在 spatial 维度上同样可以做全局信息交互，而且复杂度是线性的（$O(S)$ vs $O(S^2)$）。
2. **Channel mixing 和 token mixing 可以显式分离**：CNN 中两者是耦合的（一个卷积核同时做 spatial 和 channel 的混合），ViT 中的 self-attention 也是跨 channels 的（multi-head attention 在空间和通道维度同时做 mixing）。Mixer 把它们完全拆开，各自用独立的 MLP 处理。

从技术谱系看，这篇论文属于 **E2 分支（"attention 是否唯一有效的 token mixing 机制？"）下的 E2-1 范式（非 attention mixing 基线）**。它的历史角色是确立"attention 不是必要的"这个论断，为后来一系列非 attention 的 token mixing 方法（如 ConvMixer、MetaFormer 等）打开了一扇门。

## 四、整体结构与 forward 流程

MLP-Mixer 的架构非常简洁。以下是从输入到输出的完整 forward 流程：

### 4.1 宏观流程

```
输入图像: H × W × 3
    │
    ├─ 切 patch：将图像划分成不重叠的 P×P 小块
    │
    ├─ 展平 + 线性投影：每个 patch → C 维向量
    │   └─ 结果：S × C 的表格，其中 S = H×W/P²
    │
    ├─ (Mixer Layer × N) ───────────┐
    │   ├─ LayerNorm                 │
    │   ├─ Token-mixing MLP          │  ← 沿着 S 维度（列方向）操作
    │   ├─ 残差连接 (+)              │
    │   ├─ LayerNorm                 │
    │   ├─ Channel-mixing MLP        │  ← 沿着 C 维度（行方向）操作
    │   └─ 残差连接 (+) ─────────────┘
    │
    ├─ LayerNorm
    │
    ├─ 全局平均池化（Global Average Pooling）
    │
    └─ 线性分类器 → 类别概率
```

### 4.2 数据结构的理解

关键是要理解 Mixer 把输入组织成了一个 $S \times C$ 的二维表格（矩阵 $\mathbf{X}$）：

```
            ┌─ C 个 channel ─────────────────────────────┐
            │                                            │
            │  channel 1  channel 2 ...  channel C       │
            │                                            │
 S 个 patch  │  x₁,1    x₁,2    ...    x₁,C  ── row 1 ──→ channel-mixing MLP
 (token)    │  x₂,1    x₂,2    ...    x₂,C  ── row 2 ──→ channel-mixing MLP
            │  ...                                      │
            │  x_S,1   x_S,2   ...    x_S,C ── row S ──→ channel-mixing MLP
            │    │        │              │
            │    ↓        ↓              ↓
            │   token-mixing MLP (shared across all columns)
            │   作用于列，即对每个 channel 在所有 patch 上做 MLP
            └────────────────────────────────────────────┘
```

- **行方向（每行对应一个 patch）**：channel-mixing MLP 作用于每个 patch 的所有 channel，让一个位置内部的不同特征之间交换信息。这个操作是"per-patch"的——每个 patch 独立处理，共享 MLP 参数。
- **列方向（每列对应一个 channel）**：token-mixing MLP 作用于每个 channel 的所有 patch，让不同空间位置之间交换信息。这个操作是"per-channel"的——每个 channel 独立处理，共享 MLP 参数。

### 4.3 伪代码

```
输入: X ∈ R^{S×C}
输出: Y ∈ R^{S×C}（每个 Mixer layer）

def mixer_layer(X):
    # Token-mixing: 作用于 spatial 维度
    # 先转置为 C×S，在 S 维度上做 MLP，再转置回来
    U = X + TokenMixingMLP(LayerNorm(X))    # U ∈ R^{S×C}

    # Channel-mixing: 作用于 feature 维度
    Y = U + ChannelMixingMLP(LayerNorm(U))  # Y ∈ R^{S×C}

    return Y

def TokenMixingMLP(X):
    # X ∈ R^{S×C}
    X_t = transpose(X)          # X_t ∈ R^{C×S}
    H = GELU(W1 @ X_t + b1)    # W1 ∈ R^{D_S×S}, H ∈ R^{C×D_S}
    Y_t = W2 @ H + b2          # W2 ∈ R^{S×D_S}, Y_t ∈ R^{C×S}
    Y = transpose(Y_t)          # Y ∈ R^{S×C}
    return Y

def ChannelMixingMLP(X):
    # X ∈ R^{S×C}
    H = GELU(W3 @ X + b3)      # W3 ∈ R^{D_C×C}, H ∈ R^{S×D_C}
    Y = W4 @ H + b4            # W4 ∈ R^{C×D_C}, Y ∈ R^{S×C}
    return Y
```

这里 $D_S$ 和 $D_C$ 分别是 token-mixing MLP 和 channel-mixing MLP 的隐层维度。典型配置中 $D_C = 4C$（与 Transformer FFN 的扩展比例相同），$D_S$ 则独立于 $S$ 设定。

## 五、核心模块逐个解释

### 5.1 Patch Embedding（patch 嵌入层）

**解决的问题：** 将图像从像素空间映射到 token 空间。

**为什么需要：** 图像是 $H \times W \times 3$ 的连续像素值，而 Mixer 需要处理的是离散的 token 序列。需要一个将局部像素块压缩为特征向量的初始处理层。

**如何工作：** 与 ViT 完全相同——将图像划分为 $P \times P$ 的不重叠 patch，每个 patch 展平后通过一个可学习的线性投影层（全连接层），投影到 $C$ 维隐藏空间。结果是 $S \times C$ 的 token 表格，其中 $S = HW/P^2$。

**关键设计选择：** 所有 patch 共享同一个投影矩阵（linear projection），这与 CNN 中滑动卷积核共享参数的设计一致。Mixer 不使用额外的位置编码（position embeddings），因为 token-mixing MLP 本身对输入顺序敏感——这个区别值得展开说。

**与 ViT 的关键差异：** ViT 将图像分割后需要加上位置编码，因为 self-attention 是 permutation-invariant（对排列不敏感）的——如果把 patch 的顺序打乱，attention 的输出平均值不会变。而 MLP（全连接层）天然对输入顺序敏感：如果将输入维度的顺序打乱，输出的值会完全不同。因此 token-mixing MLP 隐含地编码了每个 token 的位置信息（通过其在输入向量中的位置索引来区分），不需要额外加位置编码。

一个直观的理解：token-mixing MLP 接受一个 $S$ 维向量（某个 channel 在所有 patch 上的值），用权重矩阵 $\mathbf{W}_1 \in \mathbb{R}^{D_S \times S}$ 做线性变换。如果 $S$ 维输入的顺序被改变，这个线性变换的结果也会被改变。所以模型能够隐式地感知 patch 的顺序。

### 5.2 Token-mixing MLP（token 混合 MLP）

**解决的问题：** 让不同空间位置（patch）之间交换信息。

**为什么需要：** 这是 Mixer 中实现"全局感受野"的操作。只靠 per-patch 的 channel mixing，每个 patch 的表示是孤立的，模型无法知道图像中其他位置有什么。就像你只看到了猫耳朵的像素块，但不知道它旁边是猫头还是背景。

**如何工作：** 对每个 channel $i$，取所有 $S$ 个 patch 在该 channel 上的值 $\mathbf{X}_{*, i} \in \mathbb{R}^{S}$，送入一个两层的 MLP（$S \rightarrow D_S \rightarrow S$），前后各接 GELU 非线性激活：

```text
输入: X_{*,i} ∈ ℝ^S    (某个 channel 在所有 patch 上的值)
 ↓
全连接层: W1 ∈ ℝ^{D_S×S}
 ↓
GELU 激活
 ↓
全连接层: W2 ∈ ℝ^{S×D_S}
 ↓
输出: U_{*,i} ∈ ℝ^S     (mixing 后的该 channel)
```

**为什么这个设计合理：** 仔细想想 token-mixing MLP 的权重矩阵 $\mathbf{W}_1 \in \mathbb{R}^{D_S \times S}$。$S$ 是 patch 的个数，每个输入维度对应一个具体的 patch 位置。因此 $\mathbf{W}_1$ 的第 $j$ 行可以理解为"对第 $j$ 个 patch 和其他 patch 之间关系的加权组合"。当 MLP 训练完成后，$\mathbf{W}_1$ 就在所有 channel 间共享，等价于学习了所有空间位置之间的一个"静态关系图"。

**关键设计选择：** 参数在 channel 之间共享。同一层内所有 channel 共用同一个 token-mixing MLP。这意味着无论 hidden dimension $C$ 有多大，token-mixing MLP 的参数数始终是 $2 \times D_S \times S$，与 $C$ 无关。论文在消融实验中验证了这一点——即便放弃参数共享（每个 channel 用自己的 token-mixing MLP），性能提升也非常微小，但模型大小会显著增加。

**复杂度分析：** Token-mixing MLP 的复杂度是 $O(S \times D_S)$，即对 $C$ 个通道、每个通道做 $S \times D_S$ 的矩阵乘法。由于 $D_S$ 是固定常数（论文中典型为 256 或 384），复杂度实际上是 $O(S)$——与 patch 数成线性关系。对比 ViT 的 self-attention 复杂度 $O(S^2)$，这是一个显著的优势。

### 5.3 Channel-mixing MLP（通道混合 MLP）

**解决的问题：** 让同一个 patch 内不同 channel 之间交换信息。

**为什么需要：** 一个 patch 经过 patch embedding 后得到 $C$ 维向量，各 channel 编码了该局部区域的不同特征（如边缘、纹理、颜色等）。这些特征需要被进一步组合和变换才能形成高层次语义。

**如何工作：** 对每个 patch $j$，取其所有 $C$ 个 channel 上的值 $\mathbf{U}_{j, *} \in \mathbb{R}^{C}$，送入一个两层的 MLP（$C \rightarrow D_C \rightarrow C$），中间接 GELU 激活：

```text
输入: U_{j,*} ∈ ℝ^C    (某个 patch 在所有 channel 上的值)
 ↓
全连接层: W3 ∈ ℝ^{D_C×C}
 ↓
GELU 激活
 ↓
全连接层: W4 ∈ ℝ^{C×D_C}
 ↓
输出: Y_{j,*} ∈ ℝ^C     (channel mixing 后的该 patch)
```

**这本质上是 "per-patch FFN"：** Transformer 的 encoder 中也包含一个 per-token 的 FFN（通常两层，$C \rightarrow 4C \rightarrow C$），同样使用 GELU 激活。Mixer 的 channel-mixing MLP 就是直接复用了这个设计。$D_C = 4C$ 是典型的配置，与 Transformer 的 FFN 扩展比例一致。

**参数共享：** 所有 $S$ 个 patch 共用同一个 channel-mixing MLP，这与 CNN 中卷积核的权重共享逻辑一致。

### 5.4 残差连接（Skip Connections）

**解决的问题：** 防止深层网络的梯度消失和信息退化。

**模式：** Mixer 使用了标准的残差连接：每个 MLP 块的输入和输出相加。公式为：

```text
U = X + TokenMixingMLP(LayerNorm(X))
Y = U + ChannelMixingMLP(LayerNorm(U))
```

这种"先 Norm 后操作"（pre-normalization）的顺序与 ViT 一致，都是受 Transformer 改进版本（如 Pre-LN Transformer）的影响。

### 5.5 Layer Normalization

**解决的问题：** 稳定训练过程。

**位置：** 每个 MLP 块之前（pre-norm 模式）。注意 LayerNorm 是作用在 channel 维度上的（对每个 token 的 $C$ 维向量做 normalization），而不是在 spatial 维度上。

### 5.6 分类头（Classification Head）

**解决的问题：** 从 $S \times C$ 的 token 表中提取全局分类信息。

**设计：** 先对所有 token 做**全局平均池化**（global average pooling, GAP），将 $S$ 个 token 平均汇集为 $C$ 维向量，然后接一个线性分类器（全连接层）映射到类别数。

**与 ViT 的差异：** ViT 使用一个额外的 [CLS] token（类似于 BERT 的分类 token），其最终表示作为分类依据。Mixer 直接使用全局平均池化来聚合所有 patch 的信息。

## 六、核心公式与数学直觉

### 6.1 Mixer Layer 的完整数学表达式

论文中给出了 Mixer layer 的完整公式（省略了层索引）：

**Token-mixing 步:**

$$\mathbf{U}_{*,i} = \mathbf{X}_{*,i} + \mathbf{W}_2 \sigma(\mathbf{W}_1 \text{LayerNorm}(\mathbf{X})_{*,i}), \quad \text{for } i = 1 \dots C$$

**Channel-mixing 步:**

$$\mathbf{Y}_{j,*} = \mathbf{U}_{j,*} + \mathbf{W}_4 \sigma(\mathbf{W}_3 \text{LayerNorm}(\mathbf{U})_{j,*}), \quad \text{for } j = 1 \dots S$$

其中：

- $\mathbf{X} \in \mathbb{R}^{S \times C}$ 是 Mixer layer 的输入
- $\mathbf{U} \in \mathbb{R}^{S \times C}$ 是 token-mixing 后的中间表示
- $\mathbf{Y} \in \mathbb{R}^{S \times C}$ 是最终输出
- $\sigma(\cdot)$ 是 GELU 激活函数
- $\mathbf{W}_1 \in \mathbb{R}^{D_S \times S}$, $\mathbf{W}_2 \in \mathbb{R}^{S \times D_S}$ 是 token-mixing MLP 的权重
- $\mathbf{W}_3 \in \mathbb{R}^{D_C \times C}$, $\mathbf{W}_4 \in \mathbb{R}^{C \times D_C}$ 是 channel-mixing MLP 的权重
- $\mathbf{X}_{*,i}$ 表示矩阵 $\mathbf{X}$ 的第 $i$ 列（所有 patch 在第 $i$ 个 channel 上的值）
- $\mathbf{Y}_{j,*}$ 表示矩阵 $\mathbf{Y}$ 的第 $j$ 行（第 $j$ 个 patch 在所有 channel 上的值）

### 6.2 公式的直觉解释

**Token-mixing 步（公式 1）:**

这个步操作的是矩阵的**列**。对每个 channel $i$：

1. 先对输入做 LayerNorm（稳定训练）
2. 取第 $i$ 列 $\text{LayerNorm}(\mathbf{X})_{*,i}$，它是一个 $S$ 维向量——即不同空间位置在某单个 channel 上的响应
3. 用 $\mathbf{W}_1 \in \mathbb{R}^{D_S \times S}$ 将其投影到 $D_S$ 维空间
4. 经过 GELU 非线性激活
5. 用 $\mathbf{W}_2 \in \mathbb{R}^{S \times D_S}$ 将其投影回 $S$ 维
6. 加上原始的 $\mathbf{X}_{*,i}$（残差连接）

最终效果：**某个 channel 上不同位置的值被线性+非线性地重新组合**，组合权重在所有 channel 间共享。这与"全局 depthwise convolution"在数学上等价——卷积核大小等于整张特征图（$S$ 个位置），且所有通道共用相同卷积核。

**Channel-mixing 步（公式 2）:**

这个步操作的是矩阵的**行**。对每个 patch（token）$j$：

1. 对 $\mathbf{U}$ 做 LayerNorm
2. 取第 $j$ 行 $\text{LayerNorm}(\mathbf{U})_{j,*}$，它是一个 $C$ 维向量——即某个空间位置在所有 channel 上的响应
3. 用 $\mathbf{W}_3 \in \mathbb{R}^{D_C \times C}$ 将其投影到 $D_C$ 维空间（通常 $D_C = 4C$）
4. 经过 GELU 非线性激活
5. 用 $\mathbf{W}_4 \in \mathbb{R}^{C \times D_C}$ 将其投影回 $C$ 维
6. 加上原始的 $\mathbf{U}_{j,*}$（残差连接）

最终效果：**某个位置的 channel 被重新组合**，与 Transformer 的 per-token FFN 完全一致。

### 6.3 为什么要显式分离两种 mixing？

这是一个重要的设计决策。CNN 中 $3\times3$ 卷积同时做 spatial 和 channel mixing，ViT 中 self-attention 也是跨 channels 做 global mixing。Mixer 把它们分开，每个只做一件事。

好处是**简单和可控**：

- Token-mixing MLP 只在 spatial 维度上混合，channel 之间没有干扰。这相当于在每个 channel 上独立地学习一个"空间对应表"。
- Channel-mixing MLP 只做 per-location 的特征变换，不涉及空间信息。这就和 Transformer 的 FFN 完全相同。

这种分离也意味着每层的参数可以独立控制：$D_S$ 控制空间混合的容量，$D_C$ 控制特征变换的容量。在实验中，$D_S$ 远小于 $S$（例如 $D_S=384$ 而 $S=196$），说明空间混合不需要太多容量；而 $D_C$ 较大（例如 $D_C=3072$），说明特征变换需要更大的容量。

### 7.3 线性复杂度推导

论文反复强调的一个关键优势是 Mixer 的线性复杂度。这里用数学说明：

**Token-mixing MLP 的计算量（FLOPs）：**

对每个 channel，做两次矩阵乘法：
- 第一层：$2 \times S \times D_S$ FLOPs（乘法和加法各一次）
- 第二层：$2 \times D_S \times S$ FLOPs

对 $C$ 个 channel 合计：$4 \times C \times S \times D_S$ FLOPs

**Channel-mixing MLP 的计算量：**

对每个 token，做两次矩阵乘法：
- 第一层：$2 \times C \times D_C$ FLOPs
- 第二层：$2 \times D_C \times C$ FLOPs

对 $S$ 个 token 合计：$4 \times S \times C \times D_C$ FLOPs

**总计算量（每个 Mixer layer）：** $4SC(D_S + D_C)$

由于 $D_S$ 和 $D_C$ 都是常数（不随输入大小变化），且 $S = HW/P^2$，总计算量 $O(S) = O(HW)$，与像素数成线性关系。

对比 ViT 中 multi-head self-attention 的 $O(S^2 C)$ 复杂度（假设 $C = d_{\text{model}}$ 且 $d_k$ 约为 $C/H$），差异是本质性的。

## 八、复杂度、效率与可扩展性分析

### 8.1 计算复杂度比较

Mixer 的核心优势在于其计算复杂度相对于输入 patch 数 $S$ 是线性的，而非二次的。

| 操作 | 复杂度 | 备注 |
|------|--------|------|
| ViT self-attention | $O(S^2 \cdot C)$ | 二次于 patch 数 |
| Token-mixing MLP | $O(S \cdot C \cdot D_S)$ | 线性于 patch 数 |
| Channel-mixing MLP | $O(S \cdot C \cdot D_C)$ | 线性于 patch 数 |
| 整体 Mixer layer | $O(S \cdot C \cdot (D_S + D_C))$ | 线性于 patch 数 |

这意味着当输入分辨率增加时（例如从 224 提升到 448），ViT 的 token 数翻 4 倍、注意力计算翻 16 倍；而 Mixer 的 token 数翻 4 倍、计算量只翻 4 倍。

### 8.2 参数效率

另一个值得注意的特点是 Mixer 的参数效率。从 Table 1 可以看出：

- **Token-mixing MLP 的参数数**：$2 \times D_S \times S$。对 Mixer-B/16，$D_S=384, S=196$，token-mixing MLP 只有 $2 \times 384 \times 196 \approx 150K$ 参数。相比之下，一个 ViT-B/16 的 self-attention 有 multi-head 的 $Q, K, V, O$ 四个投影层，参数数约为 $4 \times C^2 = 4 \times 768^2 \approx 2.36M$，但注意 self-attention 参数量与 $S$ 无关——关键差异在计算量上而非参数量。

- **Channel-mixing MLP 的参数数**：$2 \times C \times D_C = 2 \times C \times 4C = 8C^2$。对 Mixer-B/16：$8 \times 768^2 \approx 4.7M$。这部分占模型主体的主要参数。

### 8.3 实际吞吐量

论文 Table 2 提供了模型在 TPU-v3 上的吞吐量（images/sec/core）：

- **Mixer-L/16**：105 img/sec/core
- **ViT-L/16**：32 img/sec/core
- **HaloNet**（一种混合 attention+convolution 架构）：120 img/sec/core
- **BiT-R152x4**（ResNet 变体）：26 img/sec/core

Mixer-L/16 的吞吐量（105 img/sec）比 ViT-L/16（32 img/sec）高 3 倍以上，仅略低于 HaloNet（120 img/sec）。这个结果说明：**去掉 self-attention 不仅在理论复杂度上有优势，在实际硬件上也有实实在在的速度提升**。

### 8.4 训练成本

论文给出了预训练成本（TPU-v3 core-days）：

- **Mixer-H/14**（JFT-300M 预训练）：1.01k core-days
- **ViT-H/14**（JFT-300M 预训练）：2.30k core-days
- **BiT-R152x4**（JFT-300M 预训练）：9.90k core-days

Mixer 的训练成本比 ViT 低一半以上，比 BiT（ResNet）低近一个数量级。

### 8.5 扩展性问题

尽管 Mixer 的 token-mixing 复杂度是线性的，但它有一个新的瓶颈：**批量间推理的内存开销**。在高分辨率输入时，$S$ 增加，每个 channel 上的 MLP 输入维度 $\mathbf{W}_1 \in \mathbb{R}^{D_S \times S}$ 也随之增加。这限制了对高分辨率输入的 fine-tuning 方式——论文在 fine-tuning 更高分辨率时使用了 block-diagonal 初始化的方式来调整 $\mathbf{W}_2$ 矩阵的尺寸，且只支持 $S' = K^2 S$ 的缩放倍数。

## 九、实验结果

论文的核心实验是回答：**在公平的预训练-微调（pre-train then fine-tune）范式中，纯 MLP 的 Mixer 能否与当时的 CNN 和 attention-based 模型竞争？**

### 9.1 在 ImageNet-21k 上预训练后的结果

| 模型 | ImageNet top-1 | ReaL top-1 | Avg 5 | VTAB-1k | 吞吐量(img/s/core) | 训练成本(core-days) |
|------|---------------|-----------|-------|---------|-------------------|-------------------|
| HaloNet | 85.8 | — | — | — | 120 | 0.10k |
| **Mixer-L/16** | **84.15** | **87.86** | **93.91** | **74.95** | **105** | **0.41k** |
| ViT-L/16 | 85.30 | 88.62 | 94.39 | 72.72 | 32 | 0.18k |
| BiT-R152x4 | 85.39 | — | 94.04 | 70.64 | 26 | 0.94k |

解释：

- **直接比较 Mixer-L/16 vs ViT-L/16**：Mixer 的 ImageNet top-1（84.15）比 ViT（85.30）低约 1 个百分点，ReaL（87.86 vs 88.62）和 Avg 5（93.91 vs 94.39）也有类似差距。但在大部分下游任务中，Mixer 在 VTAB-1k 上反而比 ViT 高了约 2 个百分点（74.95 vs 72.72）。**这意味着 Mixer 在细粒度迁移学习上不比 ViT 差——甚至在某些场景更好。**
- **吞吐量优势**：Mixer-L/16 的吞吐量（105 img/s/core）是 ViT-L/16（32）的 3.3 倍。这是 token-mixing MLP 的线性复杂度带来的实际收益。
- **与 HaloNet 的比较**：HaloNet（一个 hybrid attention+convolution 架构）在吞吐量上略优（120 vs 105），精度也更高（85.8 vs 84.15）。但 Mixer 的完全 MLP 性质在概念简洁性上有优势。

### 9.2 在 JFT-300M 上预训练后的结果

| 模型 | ImageNet top-1 | ReaL top-1 | Avg 5 | VTAB-1k | 吞吐量(img/s/core) | 训练成本(core-days) |
|------|---------------|-----------|-------|---------|-------------------|-------------------|
| NFNet-F4+ | 89.2 | — | — | — | 46 | 1.86k |
| **Mixer-H/14** | **87.94** | **90.18** | **95.71** | **75.33** | **40** | **1.01k** |
| BiT-R152x4 | 87.54 | 90.54 | 95.33 | 76.29 | 26 | 9.90k |
| ViT-H/14 | 88.55 | 90.72 | 95.97 | 77.63 | 15 | 2.30k |

解释：

- 在更大规模数据上，Mixer-H/14（87.94）与 ViT-H/14（88.55）的差距缩小到约 0.6 个百分点。
- Mixer-H/14 的训练成本（1.01k core-days）不到 ViT-H/14（2.30k core-days）的一半。
- Mixer-H/14 的吞吐量（40 img/s/core）是 ViT-H/14（15）的 2.7 倍。
- 在 ReaL 指标上，ViT-H/14（90.72）略高于 Mixer-H/14（90.18），差距不到 0.6 个百分点。Mixer 在多数指标上与 ViT 接近。
- BiT-R152x4（ResNet 变体）在 VTAB-1k 上表现非常出色（76.29），甚至超过 Mixer（75.33），但低于 ViT（77.63），说明 CNN 在小样本迁移学习上仍有优势。

### 9.3 消融实验（论文 Supplementary A.1）

论文在补充材料中做了参数共享的消融研究——即让每个 channel 使用独立的 token-mixing MLP 而不是共享参数。结论是：虽然不共享参数后模型容量显著增加，但性能提升非常微小。这验证了 token-mixing MLP 中参数共享的设计是正确的——它能大幅减少参数量而不降低性能。

### 9.4 对输入排列的敏感性分析（论文 Section 3.4）

论文 v3 添加了一个有趣的实验：对输入 patch 的顺序进行随机排列（shuffle），观察对性能的影响。由于 token-mixing MLP 对输入顺序敏感，这个实验可以检验 Mixer 是否真的编码了空间位置信息。

实验发现，打乱 patch 顺序后性能急剧下降到随机水平——这证明 Mixer 确实通过 token-mixing MLP 学习到了隐式位置信息，不需要额外的位置编码。

### 9.5 实验总结

这些实验的核心结论是：

1. **Attention 确实不是必要的**——纯 MLP 的 Mixer 在 ImageNet-21k 预训练 + fine-tune 的设定下可以达到约 84% 的 top-1 精度，在大规模（JFT-300M）预训练下可达到 87.94%。
2. **精度差距随数据量增加而缩小**——从 ImageNet-21k 到 JFT-300M，Mixer 与 ViT 的差距从约 1.2% 缩小到约 0.6%。
3. **效率优势显著**——Mixer 的吞吐量是 ViT 的 2-3 倍，训练成本是 ViT 的一半左右。
4. **CNN 在小样本迁移中仍有优势**——在 VTAB-1k（每个任务仅 1000 个样本）上，BiT-R152x4 仍然表现强劲。

## 十、官方代码实现阅读

### 10.1 代码位置

MLP-Mixer 的官方代码放在 Google Research 的 vision_transformer 仓库中。代码使用 JAX/Flax 实现，主要实现在 `vit_jax/models_mixer.py` 文件中。

### 10.2 核心实现：MlpBlock

```python
class MlpBlock(nn.Module):
  mlp_dim: int

  @nn.compact
  def __call__(self, x):
    y = nn.Dense(self.mlp_dim)(x)
    y = nn.gelu(y)
    return nn.Dense(x.shape[-1])(y)
```

这是最基本的 MLP 块：一个两层的全连接网络，隐层维度为 `mlp_dim`，中间接 GELU 激活。输出维度与输入最后一维相同（通过 `x.shape[-1]` 获取），配合残差连接使用。

这个 `MlpBlock` 是 token-mixing 和 channel-mixing 两个 MLP 的公共构建块——区别仅在于 `mlp_dim` 参数不同。

### 10.3 核心实现：MixerBlock

```python
class MixerBlock(nn.Module):
  """Mixer block layer."""
  tokens_mlp_dim: int
  channels_mlp_dim: int

  @nn.compact
  def __call__(self, x):
    y = nn.LayerNorm()(x)
    y = jnp.swapaxes(y, 1, 2)
    y = MlpBlock(self.tokens_mlp_dim, name='token_mixing')(y)
    y = jnp.swapaxes(y, 1, 2)
    x = x + y
    y = nn.LayerNorm()(x)
    return x + MlpBlock(self.channels_mlp_dim, name='channel_mixing')(y)
```

这是 Mixer 层的核心实现。关键行是 `jnp.swapaxes(y, 1, 2)`：

- 输入 `x` 的形状是 `(batch, S, C)`，其中 `S` 是 patch 数（spatial token 维度），`C` 是 hidden dimension。
- `swapaxes(y, 1, 2)` 将形状变为 `(batch, C, S)`——现在最后一维成了 spatial 维度 $S$。
- `MlpBlock(self.tokens_mlp_dim)` 的 `nn.Dense` 作用于最后一维，即对每个 channel 在 spatial 上做全连接——这实现了 token-mixing。
- 第二次 `swapaxes(y, 1, 2)` 将形状恢复为 `(batch, S, C)`，保持与原始输入的维度顺序一致。
- channel-mixing 阶段：直接在 `(batch, S, C)` 上做 `MlpBlock(self.channels_mlp_dim)`，即对每个 token 在 channel 上做全连接。

### 10.4 整体模型：MlpMixer

```python
class MlpMixer(nn.Module):
  """Mixer architecture."""
  patches: Any
  num_classes: int
  num_blocks: int
  hidden_dim: int
  tokens_mlp_dim: int
  channels_mlp_dim: int
  model_name: Optional[str] = None

  @nn.compact
  def __call__(self, inputs, *, train):
    del train
    x = nn.Conv(self.hidden_dim, self.patches.size,
                strides=self.patches.size, name='stem')(inputs)
    x = einops.rearrange(x, 'n h w c -> n (h w) c')
    for _ in range(self.num_blocks):
      x = MixerBlock(self.tokens_mlp_dim, self.channels_mlp_dim)(x)
    x = nn.LayerNorm(name='pre_head_layer_norm')(x)
    x = jnp.mean(x, axis=1)
    if self.num_classes:
      x = nn.Dense(self.num_classes, kernel_init=nn.initializers.zeros,
                   name='head')(x)
    return x
```

关键细节：

- **Patch embedding（`stem`）**：使用了一个 `nn.Conv` 层，卷积核大小为 `self.patches.size`（即 patch 尺寸），stride 也等于 patch 尺寸。这等价于将每个 patch 卷积成一个通道数为 `hidden_dim` 的特征向量。然后用 `einops.rearrange` 将 `(batch, H', W', C)` 的空间组织展开为 `(batch, S, C)` 的 token 序列。
- **全局平均池化**：`jnp.mean(x, axis=1)`—对 $S$ 维度取平均，得到 `(batch, C)` 的全局特征。
- **分类头**：用 `nn.Dense(self.num_classes)` 做最终分类。分类头的权重初始化为 0（`kernel_init=nn.initializers.zeros`）。
- **不使用 [CLS] token**：与 ViT 不同，Mixer 不使用额外的 [CLS] token，而是直接对所有 token 做平均池化。

```text
│
├─ E3: 状态空间模型和线性复杂度 token mixing
│   ├─ S4 / Mamba (2023-24)
│   └─ Vision Mamba / VMamba (2024)
│
└─ E4: 频域 mixing 和 Fourier/matrix mixing
    └─ GFNet / FNet (2021): 用傅里叶变换做 token mixing
```

### 11.3 关键关系

- **ConvMixer（2021）**：Trockman & Kolter 的工作，用 depthwise convolution 代替了 token-mixing MLP，其余结构和 Mixer 几乎相同。它等价于验证了"token mixing 可以用任意 per-channel 全局操作"这个命题。
- **gMLP（2021, NLP）**：将 Mixer 的 token-mixing MLP 改为门控线性单元（gated linear unit），在 NLP 任务上证明 MLP 也能做序列 token mixing。
- **MetaFormer / PoolFormer（Yu et al., 2022）**：进一步实验发现，即使把 token-mixing 操作替换为简单的 $3\times3$ 平均池化（average pooling），模型仍然表现不错。这说明在"transformer-like"架构中，**真正重要的可能是 skip connection + FFN + layer norm 的总体"骨架"（MetaFormer），而不是具体的 mixing 算子**。
- **ViT（2020）**：Mixer 的"对手"——用于证明 attention 更具优势的 baseline。不是直接继承，而是平行竞争。
- **ResMLP（2021）**：与 MLP-Mixer 同期的工作（Facebook AI），也是 MLP-based architecture。区别在于 ResMLP 用了 affine transformation 代替 LayerNorm，并且在 channel-mixing MLP 中使用了更简单的线性层。

### 11.4 启发

MLP-Mixer 最大的学术贡献不在于"MLP 能比 attention 更好"，而在于**打破了"视觉模型必须用卷积或注意力"这个预设**。它释放了一个概念信号：既然 MLP 可以做到，也许 token mixing 的具体机制并不重要，重要的是整体的架构形态和学习范式。这个信号直接催生了后来一系列追求"最小归纳偏置"或"最简单 token mixing"的工作。

### 12.2 为什么在这个时间点出现？

MLP-Mixer 出现在 ViT 发布后约 7 个月。这个时间点非常微妙：

1. **ViT 已经验证了"patch-based image classification"范式的可行**。在 ViT 之前，没有人敢把图像切成 patch 然后直接喂进 Transformer 做分类。ViT 打开了这个范式，Mixer 复用但替换核心机制。

2. **2020-2021 年正是"attention 是否必要"的辩论期**。Swin Transformer 和其他注意力工作的成功掀起了 attention 热潮，但残影是：视觉界开始追问 attention 的作用边界在哪。Mixer 正好在这个节点回答。

3. **Google 团队在 ViT 上的积累**——MLP-Mixer 的作者中，Tolstikhin、Houlsby、Kolesnikov、Beyer、Zhai、Uszkoreit、Dosovitskiy 也出现在 ViT（Dosovitskiy et al., 2020）的作者列表中。这是一个在验证了 ViT 有效后，"更进一步"的自然探索。

4. **计算资源允许大规模预训练**——100M~300M 级图片的预训练范式已经成熟，使得训练无归纳偏置的纯 MLP 模型变得可能。

### 12.3 后续影响

尽管 MLP-Mixer 本身的直接实用性受到了争议（很多人说"可以用卷积达到类似效果"），但它引发的"Attention is not all you need"讨论在 2021-2022 年成为热点。其最重要的遗产是推动了 MetaFormer 类架构的诞生，后者直接启发了后来 PoolFormer、IdentityFormer、CAFormer 等一系列简化架构。

## 十三、范式意义

### 13.1 范式级别：Backbone Paradigm Challenge（主干范式挑战）

MLP-Mixer 不属于"范式确立"——它没有创建一个被广泛采用的新型架构族（不像 ViT 那样被普遍接受为 CNN 的替代品）。但它属于 **范式挑战（paradigm challenge）**——它质疑了一个被广泛接受的前提假设。

量化来说：

- **全 MLP 骨干网络**这一子方向没有成为主流，但它作为论证工具的价值远大于其实用价值。
- Mixer 本身没有被广泛部署或替代 ViT/CNN，但它的论证——**"convolution 和 attention 都不是视觉任务必需的"**——深刻地改变了社区对"什么在起作用"的理解。
- 这篇论文最重要的是它的**否定性（negative）结果**，而不是其肯定性（positive）结果。

### 13.2 在技术树中的节点类型：Bridge Node（桥梁节点）

MLP-Mixer 的节点类型更接近"桥梁节点"而非"主干节点"或"应用节点"：

- 它不是主干节点，因为后续模型并没有直接继承它的架构。
- 它是桥梁节点，因为它连接了两条技术路线——ViT/Transformer 路线和"最小归纳偏置"路线。
- 它为后来的 MetaFormer/PoolFormer 等更激进的简化工作提供了概念基础。

### 13.3 与 ViT 的范式对比

| 维度 | ViT | MLP-Mixer |
|------|-----|-----------|
| 核心机制 | Multi-head self-attention | MLP (token-mixing + channel-mixing) |
| token mixing | Query-Key 配对计算的动态权重 | 静态的线性变换（学习到的空间关系矩阵） |
| 复杂度 | $O(S^2 C)$ | $O(S C D_S)$（线性） |
| 位置编码 | 需要显式添加 | 隐式编码在 MLP 权重中 |
| 归纳偏置 | 极弱（排列不变性需位置编码） | 较弱（MLP 对顺序敏感为其注入偏置） |
| 社区影响 | 开创了 vision transformer 子领域 | 提供了"attention 不是必须"的论证 |
| 实际采用 | 广泛采用（ImageNet 和下游任务） | 有限（主要作为概念验证） |

### 13.4 "Attention is not all you need"的起点

MLP-Mixer 可以与另一个著名的 NLP 工作——gMLP——一起被视为"Attention is not all you need"时代的起点。它们共同证明了一个事实：**Transformer 的成功可能更多来自于架构的整体形态（残差连接、LayerNorm、FFN 的大小比例、大规模数据预训练），而不是 self-attention 这一单个机制**。

这个洞见在后续的 MetaFormer 论文中被进一步形式化：如果你保留 transformer 的宏架构形态（spatial mixing + channel mixing + residuals + layer norm），把 spatial mixing 替换为任意简单的操作（包括 global average pooling），效果仍然可观。

## 十四、局限性与后续方向

### 14.1 数据依赖性强

与 ViT 一样，Mixer 在中等规模数据（ImageNet-1K，1.3M 图片）上从头训练时性能有限。论文报告 Mixer-B/16 从零训练可达约 76.4% top-1 精度（与 ResNet-50 相近），但远低于同等规模、使用了强 augmentation 的 CNN（如我们的方案需要更复杂的 regularization）。它需要 ImageNet-21k 或 JFT-300M 级别的预训练才能充分发挥潜力。

这一点限制了 Mixer 的实际应用——大多数场景下，你没有 1 亿张标注图片来预训练模型。

### 14.2 高分辨率 fine-tuning 需要特殊处理

fine-tuning 高分辨率（如 448x448）时，Mixer 需要调整 token-mixing MLP 的权重矩阵大小（因为 patch 数 $S$ 增加了）。论文使用的 block-diagonal 初始化方案虽然可行（$S' = K^2S$），但不如 ViT 的位置编码插值那般灵活。

### 14.3 没有突破非线性信息交互的上限

Token-mixing MLP 学到的是静态的、对所有输入共享的空间关系矩阵。这比卷积（也共享、但局部）的灵活性更强，但比 self-attention（动态、内容相关）的灵活性更弱。论文虽然没有直接比较 stateless vs content-aware mixing 的上限差异，但不难推断：在需要内容相关的 token mixing 的任务中（例如一个对象在不同图像中位于不同位置），静态 MLP 可能不如动态 attention 有效。

### 14.4 仅验证了分类任务

论文仅在图像分类和下游分类任务上做了实验。在目标检测、语义分割等密集预测任务上，Mixer 的表现没有被充分验证。从后续工作来看，纯 MLP 骨干在这些任务上确实遇到了更大的挑战。

### 14.5 后续可能的改进方向

- **结合局部-全局信息**：Token-mixing MLP 是全局的，但缺少局部的先验。后续的 ConvMixer 直接使用 depthwise convolution 来同时做局部和全局 mixing。
- **动态 token-mixing**：如果能把 MLP 的权重变成输入相关的（conditional），也许能结合 attention 的动态性和 MLP 的效率。
- **金字塔结构**：Mixer 使用 isotropic 设计（所有层保持相同大小），而 CNN 的金字塔设计已被证明更高效。后续的分层式 MLP 可以探索这种方向。
- **更高效的 fine-tuning 机制**：解决高分辨率下 token-mixing MLP 维度不匹配的问题。

## 十五、小结

MLP-Mixer 的核心论点是简短而挑衅的：**你可以只用矩阵乘法和非线性激活来实现视觉分类，得到的结果与当时最好的卷积神经网络和基于注意力机制的模型具有竞争力。**

这篇论文的学术价值不在于它提出了一个实际被大规模采用的架构（事实上它没有），而在于它完成了那个时代需要的"否定性实验"（falsification experiment）。在 ViT 引爆视觉 Transformer 热潮之后，所有人都默认 attention 是 token mixing 的最佳选择。MLP-Mixer 证明这不是唯一的道路。它把一个隐含的假设——"视觉任务中成功的特征提取必须使用卷积或注意力"——从"不可置疑的真理"变成了"值得检验的命题"。

放在技术树中，MLP-Mixer 是一个 E2-1 类节点。它属于 post-transformer-research 中 E2 分支（"attention 是否唯一有效的 token mixing 机制？"），担任非 attention mixing 基线的角色。它的直接后续不是架构上的持续演化，而是概念上的扩散——ConvMixer 验证了卷积也能做简单的 token mixing，MetaFormer 摘出了 transformer 架构的"骨架"，PoolFormer 进一步用 average pooling 完成了终极简化。

直白说，这篇论文的价值不是给出了一个新的模型让你部署使用，而是让整个领域停下来想了一想：**我们在 attention 上投入这么多研究精力，是因为它真的特别有效，还是因为我们默认它是有效的，从没想过其他选项？**

对这个问题的重新思考，构成了 post-transformer-research 中 E2 分支的整个研究动机，也是 MLP-Mixer 在技术谱系中最本质的贡献。
