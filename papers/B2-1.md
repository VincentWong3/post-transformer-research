## 一、先搞清楚"问题在哪里"

2020 年底，ViT（Vision Transformer）横空出世，证明了不用卷积、只用 Transformer 就能在 ImageNet 分类上和 ResNet 扳手腕。这件事在 CV 界激起了一个巨大的问号：**Transformer 能不能像 ResNet 一样，成为所有视觉任务的通用 backbone？**

但冷静下来一看，发现事情没这么简单。ViT 的架构有个本质问题：它是为分类设计的，不是为 dense prediction 设计的。

分类和 dense prediction 对特征表示的要求完全不同：

- **分类**只需要一个全局特征（CLS token 或 GAP），不在乎 spatial 细节
- **检测和分割**需要高分辨率、多尺度的特征图（1/4、1/8、1/16、1/32），才能检测大物体和小物体

ViT 把 224×224 图片切成 16×16 的 patch，输出永远是 14×14（224/16）的单分辨率特征图。你要做检测？好，接 FPN 需要 4 级特征图，ViT 只给一级。你要做分割？好，U-Net 需要 skip connection 吃多尺度特征，ViT 也只给一级。所有现有的 dense prediction 框架都假设 backbone 输出的是金字塔结构——但 ViT 不是。

那 ViT 能不能用更小的 patch（比如 4×4）来保留高分辨率？理论上可以，但实际不行。原因很简单：**小 patch → 更多 token → attention 复杂度 $O(N^2)$。** 4×4 patch 在 224×224 下是 3136 个 token，attention 矩阵 3136² ≈ 983 万——已经很大了。换成检测常用的 800×1333 输入，token 数超过 6 万，直接显存爆炸。

所以问题很清楚：ViT 证明了"Transformer 可以做视觉"，但它的"columnar"结构（单分辨率 + 全局 attention）限制了它只能做分类。**社区需要一个既能生成多尺度金字塔特征图、又能控制 attention 计算复杂度的 Transformer backbone。** PVT 就是在回答这个问题的。

## 二、旧路线的真正问题

在深入 PVT 之前，得先拆解 ViT 的两条结构性瓶颈，理解为什么"缝缝补补"解决不了它们。

### 2.1 尺度瓶颈：NLP 和 CV 的本质差异

NLP 里，句子的语义单位（词/子词）天然是单尺度的——"apple" 和 "watermelon" 的 token 长度差不多。但在 CV 里，尺度可以差几十倍。一张街景图像里，近处的行人可能占据 400×200 像素，远处的红绿灯可能只有 20×30 像素。检测模型靠 FPN 来应对这个问题：在低层高分辨率特征图上检测小物体，在高层的语义特征图上检测大物体。

ViT 给不了这个。它的所有 stage 都是同质化的，输出分辨率永远是 input_stride 的固定值。SETR（2021，Zheng et al.）尝试在 ViT 后面接 CNN decoder 做 upsampling 来生成多尺度特征——但这本质上是"事后补救"。CNN 骨架的天然多层下采样结构，在纯 Transformer 里不存在，必须重新设计。

### 2.2 复杂度瓶颈：全局 attention 的二次灾难

标准 Transformer 的自注意力复杂度是 $O((hw)^2C)$。在 NLP 里，hw 是句子长度（几百），扛得住。在 CV 里，hw 是像素数——对于 224×224 图像用 4×4 patch：hw = 56×56 = 3136，平方后接近 1000 万。

如果 COCO 输入的短边 800 像素：Stage 1 特征图 200×333 ≈ 66,600 个 token——全局 attention 算 66,600² × C 的 FLOPs，约 851G，一张 V100 都装不下。

这就是为什么 ViT 被迫用 16×16 甚至 32×32 的粗 patch——不是因为粗 patch 更好，而是因为细 patch 计算上不可行。但粗 patch 就意味着丢掉空间细节，而 dense prediction 最需要的就是空间细节。

### 2.3 结构瓶颈：为什么 CNN 的金字塔结构是个好设计

CNN 的 backbone（如 ResNet）用分层下采样（stride-2 conv 或 pooling）逐步降低分辨率、增加 channel 数，自然形成一个特征金字塔：Stage 1 输出 4× 下采样，Stage 2 是 8×，Stage 3 是 16×，Stage 4 是 32×。FPN、U-Net、Mask R-CNN 都依赖这个金字塔结构。

ViT 的问题在于：**它的结构是"columnar"的，即所有 Transformer encoder 层的输入输出序列长度相同。** 没有分层下采样，就没有金字塔。

要让 Transformer 产生金字塔特征图，需要在不同 stage 之间主动降低序列长度。这就引出了 PVT 的核心思路：**把 CNN 的金字塔结构翻译到 Transformer 的语言里。**

## 三、这篇论文的核心思路

**PVT 的核心思路是：给 Transformer 加入"渐进式缩小的金字塔结构"，让它在不同 stage 输出不同分辨率的特征图。关键设计有三个：渐进式 patch embedding 下采样、空间降维注意力（SRA）、以及和 ResNet 对齐的 4 级输出格式。**

简单来说，PVT 做了三件事：

```text
渐进式 patch embedding     → 每经过一个 stage，分辨率减半（4→8→16→32）
         ↓
Spatial-Reduction Attention → 在 attention 之前对 K 和 V 做空间降维，减少计算量
         ↓
4 级金字塔输出（F1~F4）     → 和 ResNet 的输出格式一致，可即插即用
```

技术路线谱系位置：

```text
CNN backbone (ResNet, 2015)
   ↓
ViT (2020) — 全局 attention, columnar 结构, 只做分类
   ↓
PVT (2021, ICCV) — ★ 本篇 ★ 第一个金字塔 Transformer backbone
   ↓
PVTv2 (2022) — 线性复杂度改进版
   ↓
Swin (2021, 同期) — 局部窗口 attention + 移位
```

PVT 和同期工作 Swin Transformer 都出自 2021 年，都在回答同一个问题："如何让 Transformer 做 dense prediction？"但走了不同的路：PVT 选择保留全局 attention（通过 SRA 降维 K 和 V 来降低复杂度），Swin 则完全放弃跨窗口的全局 attention。这两条路线在后续的社区反馈中有了不同的命运，但 PVT 作为第一个提出金字塔结构的 Transformer backbone，它的贡献是开创性的。

## 四、整体结构与 forward 流程

### 4.1 整体架构

PVT 有 4 个 stage，和 ResNet 的 4 个 stage 完全对齐。每个 stage 由两部分组成：

1. **Patch Embedding 层**：将输入特征图划分成 patch，做线性投影，同时降低分辨率
2. **$L_i$ 个 Transformer Encoder 层**：每个 Encoder 层由 SRA + FFN 构成

```text
输入图像: H×W×3 (e.g., 224×224×3)
    │
    ▼
┌─────────────────────────────────────────┐
│ Stage 1 (4×下采样, H/4×W/4, C1=64)    │
│  Patch Embedding: P1=4                  │
│  SRA: R1=8, N1=1                        │
│  × L1 个 Transformer Encoder            │
│  输出: H/4 × W/4 × 64                   │
└───────────────┬─────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│ Stage 2 (8×下采样, H/8×W/8, C2=128)   │
│  Patch Embedding: P2=2                  │
│  SRA: R2=4, N2=2                        │
│  × L2 个 Transformer Encoder            │
│  输出: H/8 × W/8 × 128                  │
└───────────────┬─────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│ Stage 3 (16×下采样, H/16×W/16, C3=320)│
│  Patch Embedding: P3=2                  │
│  SRA: R3=2, N3=5                        │
│  × L3 个 Transformer Encoder            │
│  输出: H/16 × W/16 × 320                │
└───────────────┬─────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│ Stage 4 (32×下采样, H/32×W/32, C4=512)│
│  Patch Embedding: P4=2                  │
│  SRA: R4=1 (降维率为 1 → 全局 attention)│
│  N4=8                                   │
│  × L4 个 Transformer Encoder            │
│  输出: H/32 × W/32 × 512                │
└─────────────────────────────────────────┘
```

### 4.2 前向过程逐步跟踪（PVT-Small, 224×224 输入）

**Step 0 — 输入形状**

`[B, 3, 224, 224]`

**Stage 1**

- **Patch Embedding**：把 224×224 图像划分成 4×4 的 patch。patch 数 = (224/4)² = 3136。每个 patch 展平为 4×4×3=48 维向量。线性投影到 C1=64 维。
  - 张量形状：`[B, 3, 224, 224]` → `[B, 3136, 64]` → reshape → `[B, 56, 56, 64]`

- 加 position embedding（1D learned PE）。
- 经过 L1=3 个 Transformer Encoder 层，每层包含 SRA + FFN。
  - SRA: R1=8。Q 保持 56×56=3136 长度；K 和 V 通过 SR 操作降维到 3136/64 ≈ 49 个 token（实际 reshape 为 56/8=7 行，56/8=7 列，29 channel 拼接）。
  - 输出形状不变：`[B, 56, 56, 64]` → 这是 F1，4×下采样。

**Stage 2**

- **Patch Embedding**：P2=2。在 56×56 的特征图上，用 2×2 patch 进行划分。每个 patch 4 个 token，拼接后线性投影到 C2=128。
  - 张量形状：`[B, 56, 56, 64]` → `[B, 28, 28, 128]`

- 经过 L2=3 个 Encoder 层。SRA 的 R2=4，K 和 V 降维到 (28/4)² = 49 个 token。
- 输出：`[B, 28, 28, 128]` — F2，8×下采样。

**Stage 3**

- **Patch Embedding**：P3=2。`[B, 28, 28, 128]` → `[B, 14, 14, 320]`
- 经过 L3=6 个 Encoder 层。R3=2，K 和 V 降维到 (14/2)² = 49 个 token。
- 输出：`[B, 14, 14, 320]` — F3，16×下采样。

**Stage 4**

- **Patch Embedding**：P4=2。`[B, 14, 14, 320]` → `[B, 7, 7, 512]`
- 经过 L4=3 个 Encoder 层。R4=1，降维率为 1，即不做降维——此时 7×7=49 个 token，直接做全局 attention。
- 输出：`[B, 7, 7, 512]` — F4，32×下采样。

**Final Output**

金字塔特征图 `{F1, F2, F3, F4}`，分辨率为 4×、8×、16×、32×下采样，可以直接喂给 FPN、Mask R-CNN、RetinaNet 等框架。

## 五、核心模块逐个解释

### 5.1 渐进式 Patch Embedding

**问题：** 标准 Transformer Encoder 的输入和输出序列长度相同。要实现金字塔结构，需要在不同 stage 之间降低序列长度。

**PVT 的方案：** 用 Patch Embedding 层来做下采样。在 stage i 的入口，将输入特征图划分为 $P_i \times P_i$ 的 patch，展平后通过线性投影映射到目标 channel 数。

```python
# 伪代码：PVT 的 Patch Embedding
def patch_embedding(x, patch_size, out_channels):
    """
    x: [B, H, W, C_in] — 输入特征图
    patch_size: P_i — patch 的大小
    out_channels: C_i — 目标 channel 数
    """
    B, H, W, C = x.shape
    # 1) 划分 patch: [B, H//P, P, W//P, P, C]
    x = x.reshape(B, H // P, P, W // P, P, C)
    # 2) 合并 patch 内维度: [B, H//P, W//P, P*P*C]
    x = x.permute(0, 1, 3, 2, 4, 5).contiguous()
    x = x.reshape(B, H // P, W // P, P * P * C)
    # 3) 线性投影到 out_channels
    x = nn.Linear(P * P * C, out_channels)(x)
    return x  # [B, H//P, W//P, out_channels]
```

**Stage 1 的特殊处理：** Stage 1 在原始图像上做 patch embedding，输入没有 channel 维（3个 RGB 通道），所以 P1=4, C1=64：每个 4×4×3 的 patch 展平为 48 维，线性投影到 64 维。这类似于 ViT 的 patch embedding，但 patch 更小（4×4 vs 16×16）。

**为什么 P1=4？** 论文中说"为保持和 ResNet 一致的 4-stride 最高分辨率输出"。ResNet 的 Stage 1（conv1+bn+relu+maxpool）输出是 4×下采样，PVT 对齐这个设定。

**后续 stage 的 P 设置：** P2=P3=P4=2。这是因为后续 stage 的输入已经是特征图（有 channel 维），用 2×2 的 patch 相当于 stride=2 的下采样。

### 5.2 空间降维注意力（SRA）

**问题：** PVT 使用 4×4 的细粒度 patch，Stage 1 的输出是 56×56=3136 个 token。如果直接做全局 attention，attention 矩阵是 3136×3136 ≈ 1000 万——比 ViT 的 196×196 大了 25 倍。加上 Stage 1 是 64 channel，FLOPs 已经很高了。

**解决方案：** 在计算 attention 之前，对 K 和 V 做空间降维（Spatial Reduction）。Q 保持原大小，K 和 V 被 reshape 并线性投影到更少的 token 数。

SRA 的操作流程：

```python
# SRA 的 SR 操作 (Spatial Reduction)
def spatial_reduction(x, reduction_ratio):
    """
    x: [B, H*W, C] — 输入序列 (K 或 V)
    reduction_ratio: R — 降维率
    """
    B, N, C = x.shape
    H = W = int(sqrt(N))
    R = reduction_ratio
    # 1) reshape: [B, H, W, C] → [B, H/R, R, W/R, R, C]
    x = x.view(B, H // R, R, W // R, R, C)
    # 2) permute & reshape: [B, H/R, W/R, R*R*C]
    x = x.permute(0, 1, 3, 2, 4, 5).contiguous()
    x = x.view(B, (H//R)*(W//R), R*R*C)
    # 3) 线性投影: [B, H*W/R², C] — 恢复到 C 维
    x = nn.Linear(R*R*C, C)(x)
    x = nn.LayerNorm(C)(x)
    return x  # [B, H*W/R², C]
```

**公式表达：**

$${\rm SR}(\mathbf{x}) = {\rm Norm}({\rm Reshape}(\mathbf{x}, R_i) W^S)$$

其中 $\mathbf{x} \in \mathbb{R}^{(H_i W_i) \times C_i}$，$R_i$ 是 stage i 的降维率。$\rm Reshape$ 将序列 reshape 为 $\frac{H_i W_i}{R_i^2} \times (R_i^2 C_i)$，$W^S \in \mathbb{R}^{(R_i^2 C_i) \times C_i}$ 线性投影回到 $C_i$ 维。

**完整 SRA：**

$${\rm head}_j = {\rm Attention}(QW_j^Q, {\rm SR}(K)W_j^K, {\rm SR}(V)W_j^V)$$

$${\rm SRA}(Q, K, V) = {\rm Concat}({\rm head}_0, ..., {\rm head}_{N_i}) W^O$$

其中 attention 是 standard scaled dot-product：

$${\rm Attention}(Q, K, V) = {\rm SoftMax}\left(\frac{QK^\top}{\sqrt{d_{\rm head}}}\right)V$$

**关键洞察：** 降维后的 K 和 V 序列长度变为原来的 $1/R_i^2$。Stage 1 中 R1=8，K 的 token 数从 3136 降到 3136/64 ≈ 49——整整 64 倍降维。这意味着 attention 矩阵从 3136×3136 变成 3136×49 ≈ 154K 元素——减少了两个数量级。

### 5.3 SRA 的复杂度分析

**全局 MHA 的复杂度：** $O((H_i W_i)^2 C_i)$

**SRA 的复杂度：** $O\left(\frac{(H_i W_i)^2 C_i}{R_i^2}\right)$

因为 K 和 V 的序列长度变为 $H_i W_i / R_i^2$，所以 $QK^\top$ 的矩阵乘法复杂度从 $(H_i W_i) \times (H_i W_i)$ 降为 $(H_i W_i) \times (H_i W_i / R_i^2)$。

具体来说，以 PVT-Small 的 Stage 1（56×56=3136 token, C=64, R=8）为例：

- MHA attention 部分 FLOPs: $2 \times 3136 \times 3136 \times 64 \approx 1.26\text{G}$
- SRA attention 部分 FLOPs: $2 \times 3136 \times 49 \times 64 \approx 19.7\text{M}$
- SR 操作额外开销: reshape (零成本) + linear (3136, R²C→C) ≈ R²C = 64×64 = 4096 FLOPs/token ≈ 12.8M FLOPs

**总注意力 FLOPs: ~32.5M vs 全局 attention 的 1.26G——快了约 39 倍。**

### 5.4 FFN（前馈网络）

和标准 Transformer 一致：两层线性层，中间 GELU 激活，带残差连接。

PVT 的 FFN 有个细微变化：中间层的 expansion ratio $E_i$ 在不同 stage 不同。Stage 1-2 用 E=8（常见值是 4），Stage 3-4 用 E=4。

Stage 1 为什么要用更大的 E=8？因为 Stage 1 的 channel 数最小（64），即使 expansion 到 8 倍（512），计算总量也不大。更大的 FFN 给了 Stage 1 更多的特征表达能力。

### 5.5 Position Embedding

PVT 使用 1D 可学习的 position embedding。对于检测和分割任务（输入分辨率可变），position embedding 通过双线性插值调整大小。

和 ViT 一样，每个 token 加一个可学习的 position embedding。但 PVT 的处理更灵活：当输入分辨率变化时，position embedding 矩阵不是直接复用（因为矩阵大小变了），而是通过插值来适应新的分辨率。

## 六、核心公式与数学直觉

### 6.1 SRA 复杂度公式

**标准 MHA 单头复杂度：**

$$\Omega({\rm head}_j) = 4 \cdot C^2 \cdot (HW) + 2 \cdot (HW)^2 \cdot C$$

第一项来自 QKV 投影（$3C^2$）和 output 投影（$C^2$）。第二项来自 attention 矩阵（$(HW)^2$）和 value aggregation（$(HW)^2$）。

**SRA 单头复杂度：**

$$\Omega({\rm head}_j^{\rm SRA}) = 4 \cdot C^2 \cdot (HW) + 2 \cdot \frac{(HW)^2}{R^2} \cdot C + \frac{(HW) \cdot (R^2C) \cdot C}{R^2}$$

相比 MHA，SRA 多了第三项：SR 操作的线性投影开销（$R^2 C^2$ 的矩阵乘法）。但前两项中，第二项（attention）从 $(HW)^2$ 降为 $(HW)^2 / R^2$，这是最显著的节省。

**直觉：** 在 attention 中，"谁注意谁"的数量级从 N×N 降为 N×(N/R²)。Q 保留了所有位置，这些位置互相之间仍然不直接 attention——而是各自 attention 到降维后的"summary token"集合。这种设计假设了：**K 和 V 的空间冗余足够高，R² 倍的降维不会损失关键信息。** 在图像中，这个假设通常是合理的——相邻像素的特征很相似，用 8×8 块的平均/线性投影能保留大部分信息。

### 6.2 SRA 中的 $R_i$ 设置

不同 stage 的降维率不同，遵循一个直觉规律：**高分辨率 stage 用高降维率，低分辨率 stage 用低降维率。**

| Stage | 分辨率（H/4×W/4 输入 224²） | Token 数 | R | K 降维后 token 数 |
|-------|--------------------------|---------|---|----------------|
| 1 | 56×56 | 3136 | 8 | ~49 |
| 2 | 28×28 | 784 | 4 | ~49 |
| 3 | 14×14 | 196 | 2 | ~49 |
| 4 | 7×7 | 49 | 1 | 49 (不降维) |

有趣的是，**所有 stage 降维后的 K token 数都约等于 49**（7×7）。这意味着 attention 矩阵的列数始终在 49 左右，无论在哪个 stage。这是设计者的刻意安排：保证每个 stage 的 attention 计算量相当，而不是让 Stage 1 扛大头、Stage 4 几乎不干活。

### 6.3 公式的直觉总结

PVT 的数学直觉可以概括为：

1. **用细 patch（4×4）保留高分辨率** → 空间细节不丢
2. **渐进式 patch embedding 做下采样** → 序列长度逐步减少，形成金字塔
3. **SRA 降维 K 和 V** → attention 复杂度从 O(N²) 降到 O(N²/R²)，R 越大节省越多
4. **保持降维后 K token 数大致恒定（~49）** → 各 stage 计算负载均匀

这个设计比 ViT 聪明的地方在于：ViT 用大 patch（16×16）来暴力减少 token 数，但代价是丢失空间细节。PVT 用小 patch 保留细节，再用 SRA 把 attention 的计算量压下来。

原图对比了三种架构设计：

  H×W×3
    │ Conv+Pool (stride 2)
    ▼
  H/4 × W/4 × C1  ←── 需要设置目标尺寸 (4-stride)
    │ Conv+Pool (stride 2)
    ▼

    ▼

    │ ...
    ▼

**ViT（b）：** Patch Embedding（16×16）→ Transformer Encoder × L → 分类头。所有的 Encoder 层共享相同的序列长度。输出是单分辨率（16-stride 或 32-stride）。Columnar 结构。

## 八、复杂度、效率与可扩展性分析

### 8.1 时间复杂度

PVT 的 FLOPs 控制是它最聪明的工程决策之一。

**对比 ViT-S/16 和 PVT-Small：**

| 模型 | 参数量 | GFLOPs | ImageNet Top-1 Err |
|------|-------|--------|-------------------|
| ViT-S/16 | 48.8M | 9.9 | 19.2% |
| PVT-Small | 24.5M | 3.8 | 20.2% |

PVT-Small 的 FLOPs 只有 ViT-S/16 的 38%，参数量只有一半。这是 SRA 的功劳——Stage 1 是 FLOPs 大户（56×56=3136 token），但 R=8 把 attention 部分压缩了 64 倍。

**与 ResNet 对比：** PVT-Small（24.5M, 3.8GFLOPs）和 ResNet-50（25.6M, 4.1GFLOPs）相当，但参数量反而更少。在 2021 年，一个 Transformer backbone 能在参数量和 FLOPs 上"打平"ResNet，已经是重大进展。

### 8.2 Attention 复杂度分解

以 PVT-Small 的 Stage 1（C=64, R=8, H=W=56）为例，分解 FLOPs：

```text
Stage 1 总计 FLOPs 估算:
  Patch Embedding: 3136 × (48×64 + 64) ≈ 9.7M
  3 层 Transformer Encoder:
    SRA layer 每层:
      QKV 投影: 3 × 64 × 64 × 3136 ≈ 38.5M
      SR(K) + SR(V) 额外开销:
        2 × 3136 × 64 × 64 ≈ 25.7M  (线性投影 R²C→C)
      Attention 矩阵:
        (3136 × 49 + 49 × 3136) × 64 ≈ 19.7M
      Output 投影: 64 × 64 × 3136 ≈ 12.8M
    FFN layer 每层:
      Expansion: 2 × 64 × 512 × 3136 ≈ 205M
  三层共计: 3 × (38.5 + 25.7 + 19.7 + 12.8 + 205)M ≈ 903M
  ≈ 0.9 GFLOPs
```

Stage 1 的 FFN 用了 E=8（64→512→64），因此 FFN 占了大头（~205M/层）。如果改 E=4，Stage 1 FLOPs 能再减半。但作者选择在浅层用大 expansion ratio，可能因为浅层需要更丰富的特征表达能力。

### 8.3 新引入的瓶颈

PVT 解决了 ViT 的"单分辨率和高复杂度"问题，但也带来了新的 trade-off：

**1. SRA 的信息损失。** R=8 意味着每个 K token 聚合了周围 8×8=64 个像素的信息——这相当于一个很大感受野的 avg pooling。对于 Stage 1（4×下采样）的 4×4 patch 来说，R=8 意味着 K 的每个 token 覆盖 32×32 原始像素范围。这会模糊细粒度细节，可能影响小物体的检测。

**2. Q 仍然是全局的。** SRA 只降维了 K 和 V，Q 保留全部 token。所以 attention 矩阵是 Q(N) × K(N/R²)，计算量仍是 N×(N/R²) 而非 N×N。以 COCO 检测输入（短边 800）为例，Stage 1 特征图约 200×333 ≈ 66,600 个 token，即使 R=8，attention 也有约 66,600 × (66,600/64) ≈ 69M 个元素。虽然比全局的 44 亿好了三个数量级，但和 Swin 的 O(N) 相比仍有差距。

> ⚠️ **编者注：** SRA 的降维因子是 $R^2$ 而非 $R$，因为 SR 操作在 H 和 W 两个维度各缩小 R 倍，K 的 token 数变为 N/R²。以 800×1333 输入为例，Stage 1 经 4×下采样后约有 66,600 个 token，R=8 时 K 降维至 66,600/64 ≈ 1,041 个 token，attention 矩阵约为 69M 个元素。

**3. 全局 attention 的硬件开销。** 即使 token 数降到 49，attention 矩阵是 3136×49 而不是 49×49——Q 保留全部分辨率。这意味着 attention 计算的 GPU 并行效率可能不如纯局部 attention（每个查询看固定邻居）。

**4. Stage 1 与后续 stage 的下采样步幅不均衡。** P1=4 直接从原始图像跳至 4×下采样（224→56），而后续 stage 的 P2=P3=P4=2 各降一半。这意味 Stage 1 的 patch embedding 承担了最大的下采样跨度（stride-4），同时也产生了最多的 token（3136）。这个设计是合理的（为了对齐 ResNet 的 4-stride 输出），但它也使得 Stage 1 成为全 pipeline 的计算瓶颈。

### 9.5 消融实验

论文做了几组消融来分析 SRA 和金字塔结构的贡献：

**SRA 降维率的影响：** R 越大，FLOPs 越低，但精度也越低。论文没有提供完整的消融数据表格，但指出了 R=8 在 Stage 1 是最优平衡点。

**分类 token vs GAP：** PVT 在分类任务上使用 CLS token（类似 ViT）。论文没在这块做详细的消融，但后续的 PVTv2 证实了 GAP 简单有效。

**位置编码插值的影响：** 对于变分辨率输入（检测/分割），position embedding 需要从预训练的 56×56 网格插值到目标分辨率。论文发现双线性插值对最终精度影响很小（<0.5%），说明了 PVT 对分辨率变化的鲁棒性。

### 10.3 核心代码片段分析

**SRA 的 Attention 实现：**

```python
class Attention(nn.Module):
    def __init__(self, dim, num_heads=8, qkv_bias=False, qk_scale=None,
                 attn_drop=0., proj_drop=0., sr_ratio=1):
        super().__init__()
        self.num_heads = num_heads
        head_dim = dim // num_heads
        self.scale = qk_scale or head_dim ** -0.5
        self.sr_ratio = sr_ratio

        self.q = nn.Linear(dim, dim, bias=qkv_bias)
        self.kv = nn.Linear(dim, dim * 2, bias=qkv_bias)
        self.proj = nn.Linear(dim, dim)

        if sr_ratio > 1:
            self.sr = nn.Conv2d(dim, dim, kernel_size=sr_ratio, stride=sr_ratio)
            self.norm = nn.LayerNorm(dim)

    def forward(self, x, H, W):
        B, N, C = x.shape
        q = self.q(x).reshape(B, N, self.num_heads, C // self.num_heads)
        q = q.permute(0, 2, 1, 3)  # [B, nH, N, C//nH]

        if self.sr_ratio > 1:
            x_ = x.permute(0, 2, 1).reshape(B, C, H, W)
            x_ = self.sr(x_).reshape(B, C, -1).permute(0, 2, 1)  # downsampled
            x_ = self.norm(x_)
            kv = self.kv(x_).reshape(B, -1, 2, self.num_heads, C // self.num_heads)
        else:
            kv = self.kv(x).reshape(B, -1, 2, self.num_heads, C // self.num_heads)
        kv = kv.permute(2, 0, 3, 1, 4)
        k, v = kv[0], kv[1]

        attn = (q @ k.transpose(-2, -1)) * self.scale
        attn = attn.softmax(dim=-1)
        x = (attn @ v).transpose(1, 2).reshape(B, N, C)
        x = self.proj(x)
        return x
```

**关键发现：**

1. **SR 实现用的是 conv2d 而不是 reshape+linear。** `nn.Conv2d(dim, dim, kernel_size=sr_ratio, stride=sr_ratio)` 等价于空间降维——stride=sr_ratio 把 H 和 W 各缩小 sr_ratio 倍。这比 reshape+linear 更简洁、更高效（利用了卷积硬件优化），但语义上是等价的。

2. **Q 和 K/V 的线性投影独立。** Q 用 `self.q`，K/V 共享 `self.kv`（输出 dim*2）。这和标准 Transformer 一样，但 K/V 作用于降维后的序列。

3. **sr_ratio=1 时不降维。** Stage 4 的 sr_ratio=1，这时 Attention 退化为标准全局 attention，因为没有空间降维操作。

4. **LayerNorm 在 SR 之后。** 降维后的序列先过 LN 再过 K/V 投影——这和 Pre-LN 的设计一致。

**PatchEmbed 实现：**

```python
class PatchEmbed(nn.Module):
    def __init__(self, img_size=224, patch_size=16, in_chans=3, embed_dim=768):
        # Stage 1: img_size=224, patch_size=4
        # Stage 2: img_size=56,  patch_size=2
        self.proj = nn.Conv2d(in_chans, embed_dim,
                              kernel_size=patch_size, stride=patch_size)

    def forward(self, x):
        x = self.proj(x)  # Conv2d 实现 patch embedding
        return x
```

Stage 1 用卷积（kernel=4, stride=4）实现 patch embedding。这比手动 reshape+linear 更高效——卷积自动处理了划分和投影两个操作。

### 10.4 论文 vs 代码的细节差异

**1. SR 的实现选择：** 论文公式里 SR = Reshape + Linear + Norm。代码里 SR = Conv2d (stride=R) + Norm。两者等价但 conv2d 更快。

**2. FFN 的 expansion ratio：** 论文 Table 1 中 Stage 1-2 的 E=8，Stage 3-4 的 E=4。代码中通过隐藏层维度来控制。

**3. Position embedding 的插值：** 用于变分辨率输入时，position embedding 通过 `F.interpolate` 插值。

**4. Classification head：** PVT 在 Stage 4 的输入前加 CLS token，这和 ViT 一致。

### 11.1 技术树中的位置

PVT（B2-1）和 Swin（B2-2）是平行的两棵分支。PVT 先出现（2021年 2 月 arXiv），Swin 随后（2021年 3 月 arXiv）。两者都在解决"Transformer 做 dense prediction"的问题，但走了不同的技术路线。

### 11.2 前驱工作：PVT 继承了谁

**ViT（Dosovitskiy et al., 2020）：** PVT 继承了 ViT 的 patch embedding、Transformer encoder block、Pre-LN 结构、CLS token 等核心组件。PVT 的贡献不是发明新 attention 机制，而是把 ViT 的"columnar"结构改成了"pyramid"。

**DeiT（Touvron et al., 2021）：** PVT 的训练策略（AdamW、cosine schedule、RandAugment、Mixup、CutMix）基本来自 DeiT。没有 DeiT 的 knowledge distillation，但用了 DeiT 的 data augmentation pipeline。

**ResNet（He et al., 2016）：** PVT 的 4 级金字塔结构直接对齐 ResNet 的输出格式。PVT 的目标不是取代 ViT，而是提供一个和 ResNet 格式兼容的 Transformer backbone。

**CNN 金字塔结构（FPN, Lin et al., 2017）：** PVT 的 4 级输出可以直接对接 FPN，这和 ResNet+FPN 的兼容性完全一致。可以说，PVT 通过金字塔结构把 Transformer 接入了 CNN 时代的整条工具链。

### 11.3 同期竞品：PVT vs Swin

PVT 和 Swin 几乎是同时提交到 arXiv 的工作（PVT 在 2 月 24 日，Swin 在 3 月 30 日），两者有着共同的目标但走了截然不同的技术路线：

| 维度 | PVT | Swin |
|------|-----|------|
| 核心思想 | 全局 attention + SRA 降维 K/V | 局部窗口 attention + 移位 |
| Attention 复杂度 | $O(N^2/R^2)$ | $O(M^2 N)$ (M=7) |
| 感受野 | 准全局（Q 看到所有降维 K） | 局部（通过多层移位扩大） |
| 实现复杂度 | 简单（加一个 SR 层） | 较复杂（shift + mask + cyclic） |
| 硬件效率 | 一般（Q 仍然 N 个 token） | 好（窗口内连续内存访问） |

从实际效果看，Swin 的精度通常高于 PVT（Swin-L 在 COCO 上的 RetinaNet AP 更高）。但这两者使用的检测 framework 不完全相同，直接对比精度数字需要谨慎。Swin 的局部窗口方案在硬件效率和精度上都更优。

但 PVT 的价值在于：**它是第一个把金字塔结构引入 Transformer 的工作。** 在 PVT 之前，没有人系统地想过"如何在 Transformer 中构建金字塔特征图"。PVT 证明了这条路是可行的，为后续工作（包括 Swin 在内）奠定了概念基础。

### 11.4 后续影响：PVT 启发了谁

**PVTv2（Wang et al., 2022, arXiv:2106.13797）：** PVT 的后续版本。主要改进：
- **线性空间降维注意力（Linear SRA）**：用 depthwise separable convolution 替代 SR 的 avg pooling，进一步降低复杂度到 $O(N)$
- **重叠 patch embedding**：Stage 1 的 patch embedding 用 stride=4, kernel=7 的卷积代替 kernel=4, stride=4，增加重叠区域，提供更好的位置信息
- **移除了 position embedding**：用卷积的 zero padding 隐式编码位置信息
- 去掉分类 token，统一用 GAP
- COCO 上 PVTv2-B2（25M, 46.5 AP）比 PVT-S（34.2M, 40.4 AP）显著提升

**DETR + PVT 的组合（论文中实验）：** PVT 和 DETR 配合构建了第一个无卷积的端到端检测系统，证明 PVT 可以完全摆脱卷积。

**SegFormer（Xie et al., 2021）：** 基于层级 Transformer backbone 的分割框架，使用类似 PVT 的渐进式下采样设计。

**InternImage（2023）：** 用 deformable conv 替代 attention，但其多尺度金字塔设计思路和 PVT 一脉相承。

## 十二、范式意义

**PVT 的范式意义：它打破了"Transformer backbone 必须像 ViT 一样是 columnar 结构"的思维定势，证明了金字塔结构可以移植到 Transformer 上。**

具体来说，PVT 实现了三个认知转变：

### 12.1 "Transformer 也可以像 CNN 一样做多尺度"

PVT 之前，Transformer 的视觉应用要么是（1）用 Transformer decoder 做检测 head（DETR），保留 CNN backbone；要么是（2）用 ViT 做分类 backbone（ViT）。没有人想过 Transformer backbone 也可以产生多尺度金字塔特征。

PVT 证明：**patch embedding 层的 stride 和 kernel 大小，可以替代卷积的 stride，实现层级下采样。** 这个洞察本身看似简单，但在当时是需要打破的思维习惯——所有 Transformer 模型的 encoder 都是输入输出等长的，改变这一点需要勇气。

### 12.2 "全局 attention 可以被空间压缩"

ViT 的设计哲学隐含了一个假设：全局 attention 是必要的。PVT 用 SRA 挑战了这个假设：**K 和 V 的信息密度足够高，可以大幅压缩而不影响最终性能。**

具体来说，R=8 意味着每个 K token 代表 8×8=64 个原始 token —— 压缩率 64 倍而精度基本不下降。这证明了 K 和 V 有巨大的空间冗余。这个发现也为后来的工作（Swin 的局部窗口、Focal Transformer 的粗细粒度、各种稀疏注意力机制）打开了思路。

### 12.3 "Transfomer backbone 可以即插即用"

PVT 的输出格式（4 级金字塔特征图 F1~F4 的 stride 4/8/16/32）和 ResNet 完全一致。这意味着所有基于 ResNet 的 downstream framework（RetinaNet、Mask R-CNN、FPN、UperNet、Semantic FPN）都可以直接用 PVT 替换 backbone，零适配成本。

这个"即插即用"的设计理念非常重要——它降低了社区实验 Transformer backbone 的门槛。你不需要为 Transformer 重新设计检测头或分割头，取掉 ResNet，插入 PVT，发现效果更好，仅此而已。

### 12.4 分类：bridge node

PVT 在技术树中的分类是 **bridge node（桥梁节点）**。它连接了 ViT（第一个视觉 Transformer）和后来的 hierarchical Transformer backbone 浪潮。它本身不是 Swin 那样的大规模精化节点，但它开辟了"pyramid in Transformer"这个方向。

PVT 的核心贡献可以用一句话概括：**第一个让 Transformer backbone 输出多尺度金字塔特征图的工作。** 没有这句话，后来的 Swin、CSWin、Focal Transformer 等工作的表述都要重新写。

## 十三、局限性与后续方向

### 13.1 SRA 的复杂度瓶颈

SRA 的复杂度是 $O(N^2/R^2)$——虽然比 MHA 的 $O(N^2)$ 好了一个数量级（对于 R=8 是 64 倍），但毕竟不是 $O(N)$。当输入分辨率继续增大（如 800×1333 的 COCO 输入），PVT 仍然面临计算压力。

**对比 Swin 的 $O(N)$：** Swin 的局部窗口 attention 复杂度相对于 token 数是线性的。这意味着在极端高分辨率输入下，Swin 的扩展性优于 PVT。这是为什么后续的 PVTv2 采用了线性 SRA（基于 depthwise separable convolution 的降维方案）来进一步降低复杂度。

**数字对比：** 对于 COCO 输入的 Stage 1（800×1333 经 4×下采样后 ≈ 200×333 ≈ 66600 token）：

- SRA (R=8): attention 矩阵 66600 × (66600/64) ≈ 66600 × 1041 ≈ 69M 元素
- Swin (M=7): attention 矩阵 ≈ 49 × 49 × nWindows ≈ 49 × 49 × (200/7 × 333/7) ≈ 49 × 49 × 1359 ≈ 3.3M 元素

Swin 仍然有约 20 倍的优势。

### 13.2 重叠信息的缺失

PVT 的 patch embedding 使用非重叠 patch（kernel=stride），这意味着 patch 边界的信息在 patch 划分过程中被丢弃。相邻像素虽然很近，但如果分属不同 patch，它们的交互只能通过 Transformer 的 SRA+FFN 间接传递。

**为什么这个可能是问题：** CNN 的卷积核在相邻区域有重叠（kernel_size > stride），因此边界信息不会丢失。重叠 patch embedding 是 PVTv2 的一个关键改进（kernel=7, stride=4 的卷积代替原来的 kernel=4, stride=4）。

### 13.3 分类精度不是优势

PVT（PVT-Large 61.4M, 18.3% top-1 err）在分类上被同期工作超越（DeiT-B 86.6M, 18.2%；TNT-B 66M, 17.2%）。这在意料之中——如前所述，金字塔结构和 SRA 是为 dense prediction 设计的，分类任务不需要多尺度特征，而 SRA 的 K/V 压缩在分类任务中反而可能造成信息损失。

### 13.4 SRA 的 head 数和降维率的组合不够灵活

Table 1 中 Stage 1 的 attention 只有 1 个 head（N1=1）。单头 attention 的表达能力有限——不同 head 可以关注不同的空间区域，1 个 head 意味着只能用一种方式解释 Q 和 K 的关系。这可能限制了 PVT 的表达能力。

**对比：** ViT 的 Base 模型有 12 个头，DeiT 的 Small 有 6 个头。PVT 在 Stage 1 只有一个头，可能是因为 channel 数太小（C=64），再分多个头后每个头的维度太低（如 8 个头各 8 维），效益有限。

### 13.5 非卷积管道的不成熟

论文提到了 PVT+DETR 构建的第一个端到端无卷积检测系统，但实验并不充分。原文只在两个句子中提及了这个组合，没有完整的实验表格。这表明在 2021 年初，这个"无卷积"愿景更多是一个概念证明，而非成熟系统。

后续的 DETR 系列（DAB-DETR、DN-DETR、DINO）主要使用 ResNet backbone，直到很晚才有人系统性地探索全 Transformer 检测管道（如 UViT、YOLOS）。

### 13.6 缺乏理论解释

为什么 ReLU≈pooled K 和 V 可以做到 64 倍压缩而不损失信息？论文没有给出理论分析。直观的理解是"相邻像素的特征高度相似"，但这更多是一个经验观察而非严格论证。

后续的一些理论工作（如 Attention Entropy、信息瓶颈框架）尝试对 attention 中的冗余进行量化分析，但并没有直接针对 SRA 的理论解释。

## 十四、小结

PVT 是一个"概念先行"的工作。它的历史意义不在于刷榜（实验上确实被 Swin 超越了），而在于它定义了"金字塔 Transformer"这个概念。

在 PVT 之前，Visual Transformer 只有两种用法：作为 ViT 做分类，或者和 CNN backbone 配合。没有人想过 Transformer backbone 本身可以产生多尺度特征图。PVT 想了一个最直接的办法：**把 ResNet 的 4 级金字塔结构复制到 Transformer 里**——Stage 1 输出 4×下采样，Stage 2 输出 8×，Stage 3 输出 16×，Stage 4 输出 32×。然后发现，这个简单的想法就行得通。

这个贡献比它看上去要有价值得多。在 2021 年初，把一个"非标准"的 Transformer 变体部署到检测/分割框架上是有风险的——你不知道 transformer 的输出格式是否兼容 FPN，也不知道 position embedding 在变分辨率输入下是否工作。PVT 系统性地回答了这些问题，为后续工作扫清了障碍。

SRA 的设计也很聪明。它没有完全放弃全局 attention，而是用"降维 K 和 V"的方式保留了 Q 的全局视野。这个设计在理论上有说服力：Q 保留了所有位置，K 和 V 被压缩但仍然覆盖全图——attention map 是 Q(N) × K(N/R²)，Q 仍然能看到所有位置的概要信息。

如果硬要说 PVT 的问题，那就是它"不够彻底"。SRA 的复杂度是 $O(N^2/R^2)$ 而非 $O(N)$，在极高分辨率输入时仍然有瓶颈。这个局限性在后续的 PVTv2 和 Swin 中被解决了。但作为第一个吃螃蟹的人，PVT 已经做得足够好了——它让社区看到了"Transformer backbone 也能做 dense prediction"这条路，而这条路最终通向的是超越 ResNet 的全面胜利。
