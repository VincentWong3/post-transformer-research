## 一、先搞清楚"问题在哪里"

2020–2021 年，视觉 Transformer 的发展走到一个十字路口。

一边是 ViT 在图像分类上的成功（An Image is Worth 16x16 Words, 2020），证明了 Transformer encoder 可以脱离卷积，在足够的数据量下达到甚至超过 CNN。另一边，社区很快认识到一个棘手的差异：**视觉和语言，在底层表征上是两种完全不同性质的数据。**

### 1.1 语言的"稀疏序列" vs 视觉的"稠密张量"

NLP 里，transformer 处理的序列是一个个离散的 token。一个句子可能只有 20–100 个 token，每个 token 承载的语义信息密度极高。"apple" 三个字符代表一个具体概念，"running" 六个字符代表一个动作。语言序列是**稀疏且语义密集的**——每个 token 都有独立含义，token 之间的关系（语法、指代、逻辑）可以用 attention 高效地学习。

图像和视频完全相反。一张 224×224 的图像包含 50,176 个像素——这已经是极其稠密的信息。每个像素本身几乎没有语义（它是一个 RGB 数值），语义只出现在像素与像素之间的**结构关系**中。ViT 的 patch embedding 把这个稠密张量压缩成 $(224/16)^2 = 196$ 个 patch token——但这 196 个 token 的语义密度仍然远低于 NLP 中的 token。绝大多数 patch 是背景、纹理或物体的一部分，而不是"一个独立的概念"。

视频的问题就更极端了。一段 16 帧、224×224 的视频有 $16 \times 224 \times 224 \times 3 = 2.4M$ 个像素值。即便用 ViT 的方式切成 16×16 的 patch，token 数也是 $16 \times 196 = 3136$——是一张图像的 16 倍。而且视频中的大部分帧间变化是连续的、冗余的：相邻两帧之间可能只有 1% 的像素发生变化。

所以第一个核心问题是：**语言模型面对的 token 序列是短而语义密集的，视觉模型面对的 token 序列是长而语义稀疏的。** 同一套 Transformer 架构，在视觉上必须处理几个数量级更长的序列、更低的信噪比。

### 1.2 纯 Transformer 的"均匀分辨率"困境

ViT 的设计有一个深层假设：序列中所有 token 在语义上同等重要，所以所有层用相同的 token 数量。这在 NLP 中合理——每个词对句子语义的贡献大致相当。

但视觉不是这样的。一张图像里，某些区域（如人脸、文字）包含极多信息，另一些区域（如天空、地面）信息量极低。而且，**视觉理解天然是多尺度的**——小物体需要高分辨率特征（几×几像素的 patch 就能表示），大物体需要大感受野（几十×几十的区域才能判断它是什么），场景理解需要全局上下文。

CNN 解决这个问题的方式是特征金字塔：浅层在高分辨率上提取低级特征（边缘、纹理），深层在低分辨率上提取高级语义（物体类别）。Transformer 没有这个机制——ViT 的 12 层 block 全部用 196 个 token，每层看到的信息量是相同的。这就像用同一把放大镜去看蚂蚁和大象——对蚂蚁来说太大，对大象来说太小。

### 1.3 视频引入的时间维度——压垮骆驼的最后一根稻草

视频理解是视觉问题的终极形式：**它不仅有多尺度的空间信息，还有一个额外的稠密维度——时间。**

视频的帧率（24–30 fps）意味着大多数相邻帧之间只有微小变化。真正有用的"时间信息"不是单帧的内容（那些 CNN/ViT 已经在做了），而是帧与帧之间的**运动模式**——物体在移动、形状在变化、交互在进行。

但现有 Transformer 方法在视频上做什么？最直接的做法（VTN, 2020）是在 ViT 的顶部加一个时间 attention 层，或者像 TimeSformer 把空间 attention 和时间 attention 分开计算。这些做法的共同问题是：**它们把时间信息当成一个"装饰"加在空间 backbone 上**，而不是把它嵌入到 backbone 的基本结构中。结果，模型缺乏对时间信息的天然敏感性。

作者做了一个非常漂亮的实验来证明这一点：拿 TimeSformer 模型在 Kinetics 上测试——准确率正常。把测试视频的帧随机打乱，再测——准确率几乎不变。这意味着模型根本没在使用时间信息！它是在用单帧的外观线索做动作分类。MViT 的相同实验：帧打乱后，准确率大幅下降。说明 MViT 真正地利用了时间结构。

### 1.4 计算效率瓶颈

视频 Transformer 还有一个更残酷的现实问题：计算量。视频 token 数量是图像的 16 倍，而 self-attention 的复杂度是 $O(N^2)$——所以计算量是图像的 256 倍。一块 V100 连 16 帧 224² 视频的全局 attention 都放不下。

这引出一个根本选择：要么牺牲空间分辨率（降低 H/W），要么牺牲时间分辨率（降低帧数），要么牺牲模型容量（减少通道）。但三者都会降低性能。**MViT 的回答是：都不牺牲，而是在 Transformer 内部做多尺度降采样，让每个阶段的计算都精准匹配当下的信息需求。**

## 二、旧路线的真正问题

在 MViT 之前，视频理解领域的 Transformer 路线有三条，每条都有结构性的不足。

### 2.1 VTN 方案——"ViT + 时间 embedding"的投机取巧

Visual Transformer Network（VTN, 2020）的方案是：对每帧独立运行 ViT，得到帧级别的特征序列，然后在帧序列上再跑一层时间 attention。这是最直接的想法——把时间当成一个"post-processing"步骤。

这里的问题是：**空间特征和时间特征被完全解耦了。** 如果第一帧的某个 patch 和第二帧的对应 patch 之间存在运动关系，这个信息在独立的空间编码阶段会被丢掉。时间层只能在"帧级别的全局特征"上做 attention——它无法捕捉细粒度的时空对应关系（如"这个人影在第一帧的左上角，在第二帧的中间"）。这就像看连环画而不是看电影——每幅画的内容是清晰的，但你不知道人物是怎么移动过来的。

此外，VTN 依然继承了 ViT 的所有问题：单分辨率、$O(N^2)$ 复杂度、缺乏多尺度特征。

### 2.2 TimeSformer——"分开注意力"的妥协

TimeSformer（2021，几乎与 MViT 同时）做了更精巧的设计：把时空 attention 分解为空间 attention（同一帧内的 patch 之间）和时间 attention（不同帧间的同一空间位置之间）。这样 $O((THW)^2)$ 的复杂度降为 $O(T(HW)^2 + HW(T)^2)$——虽然还是平方，但不再是所有维度的联合平方。

但 TimeSformer 有两个深层问题：

**第一，空间和时间仍然是分开处理的**。空间 attention 看不到时间上下文，时间 attention 看不到空间上下文。联合的时空模式（如"一只狗从左向右跑"——既是空间位移又是时间变化）被拆成了两个独立步骤。模型无法学到"位置随时间变化"这个完整的运动概念。

**第二，帧打乱测试揭露了根本缺陷**。TimeSformer 在打乱帧后性能几乎不变（K400 上 77.7% vs 77.5%），说明时间 attention 模块的作用实际上很有限。作者推测原因：时间 attention 是在空间 attention 之后的，而空间 attention 已经对单帧做出了足够好的分类。时间 attention 只是锦上添花——甚至可有可无。

### 2.3 ViViT——"昂贵但全面"的暴力方案

ViViT（2021）对时空建模做了最全面的尝试：它设计了四种不同的时空 attention 变体（从联合到分解的各种组合），试图找到最佳平衡。但 ViViT-L 的参数量高达 89M，FLOPs 也达到 399G——是 MViT-B 的 2 倍多参数量、接近 2.5 倍 FLOPs。

更关键的是，**ViViT 沿用 ViT 的固定分辨率设计**。它在不同层之间不改变 token 数量，只是在注意力计算方式上做文章。这意味着它的高分辨率层和低分辨率层用完全相同的计算预算——如果浅层只需要低通道数就能处理边缘特征，深层需要高通道数来处理语义特征，ViViT 把计算资源均匀分配了。

所有三条路线的共同盲点是：**它们假设 Transformer 的"定长序列、固定通道"范式可以直接移植到视频**。没有考虑到视觉——尤其是视频——对多尺度表征的自然需求。

## 三、这篇论文的核心思路

**MViT 的核心思路非常简洁：把 CNN 的多尺度特征金字塔引入 Transformer，通过 Pooling Attention 让 Transformer block 内部具备尺度变换能力，然后堆叠多个"scale stage"构成一个从高分辨率→低分辨率、从低通道→高通道的层级结构。**

这个想法的基础洞察也很直接：Transformer block 的 MHA 中，query/key/value 三个矩阵本质上描述的是同一组 token 在不同"角色"上的表示。如果对它们做不同粒度的空间-时间池化，就可以让 attention 在一个尺度上查询（query 分辨率高）、在另一个尺度上聚合上下文（key/value 分辨率低）。这样，**attention 的计算过程本身就蕴含了多尺度建模**。

具体路线：

```text
输入视频 (T×H×W×3)
    │
    ├── 3D Patch Embedding → 把视频切成时空 tubelet
    │
    ├── Scale Stage 1:  N1×MViT Block (高分辨率/低通道)
    │      │   ↓ Pooling Attention (Q: 1×, KV: 1×)
    │      └── Pooling down 2×, channel ×2
    │
    ├── Scale Stage 2:  N2×MViT Block (中分辨率/中通道)
    │      │   ↓ Pooling Attention (Q: 1×, KV: 2×)
    │      └── Pooling down 2×, channel ×2
    │
    ├── Scale Stage 3:  N3×MViT Block (低分辨率/高通道)
    │      │   ↓ Pooling Attention (Q: 2×, KV: 2×)
    │      └── Pooling down 2×, channel ×2
    │
    └── Scale Stage 4:  N4×MViT Block (极低分辨率/极高通道)
           │   ↓ Pooling Attention (Q: 2×, KV: 2×)
           └── 全局平均池化 → 分类头
```

这和我们熟悉的 ResNet 的 4 个 stage（conv1 → conv2_x → conv3_x → conv4_x）在结构上完全对齐，但在计算上完全不同——MViT 的每个 block 仍然是用 self-attention 和 MLP 组成的纯 Transformer 结构，只是 attention 的操作方式变了。

## 四、架构细节

### 4.1 核心组件：Multi Head Pooling Attention（MHPA）

MHPA 是 MViT 最核心的设计，也是唯一与传统 Transformer 不同的组件。它在标准 MHA 的基础上，在注意力计算之前多加了一步——对 query、key、value 分别做空间-时间池化。

**标准 MHA：** 对输入 $X \in \mathbb{R}^{L \times D}$ 线性投影得到 $Q, K, V \in \mathbb{R}^{L \times D}$，计算 $\text{Softmax}(QK^T/\sqrt{d})V$。输入的序列长度 $L$ 在输入和输出侧保持不变。

**MHPA：** 对输入 $X$ 线性投影后，分别对 $Q, K, V$ 做池化 $Q = \mathcal{P}(Q; \Theta_Q), K = \mathcal{P}(K; \Theta_K), V = \mathcal{P}(V; \Theta_V)$，再计算 $\text{Softmax}(QK^T/\sqrt{d})V$。池化后各矩阵的序列长度可以不同。

**池化操作 $\mathcal{P}(\cdot; \Theta)$：**

池化算子接受一个形状为 $T \times H \times W$ 的空间-时间张量（展平前），用池化核 $k=[k_T, k_H, k_W]$、步长 $s=[s_T, s_H, s_W]$、填充 $p=[p_T, p_H, p_W]$ 做三维池化：

$$\tilde{T} = \left\lfloor\frac{T  + 2p_T - k_T}{s_T}\right\rfloor + 1$$

同理对 $H$ 和 $W$。展平后得到长度为 $\tilde{T}\tilde{H}\tilde{W}$ 的序列。

默认使用重叠池化核（kernel size > stride）和保形填充，所以序列长度约降低 $s_T \cdot s_H \cdot s_W$ 倍。

**Pooling Attention 的关键自由度：**

核心洞察是：**Q 的池化决定了输出分辨率，K/V 的池化决定了注意力计算的输入上下文规模。** 独立控制它们可以同时实现两种目标：

- **维持分辨率（对 Q 做较小 pool 或不做 pool）**：输出特征图保持精细结构
- **降低计算量（对 K/V 做 pool）**：注意力矩阵 $QK^T$ 的实际尺寸是 $(L/s_Q) \times (L/s_K)$，在保持 Q 分辨率的同时降低 K/V 数量，可以大幅降低复杂度

约束条件：$s_K = s_V$（K 和 V 必须使用相同的池化步长，因为它们需要对齐到相同的序列长度）。

**复杂度分析：**

标准 MHA 的复杂度（对 $T \times H \times W$ 的输入）：

$$\Omega(\text{MHA}) = 4 \cdot T H W \cdot C^2 + 2 \cdot (THW)^2 \cdot C$$

MHPA 的复杂度：

$$\Omega(\text{MHPA}) = 4 \cdot T H W \cdot C^2 + \frac{2 \cdot (THW)^2 \cdot C}{f_Q f_K}$$

其中 $f_Q = s_T^Q s_H^Q s_W^Q$，$f_K = s_T^K s_H^K s_W^K$。

第一项（QKV 投影）不变，第二项（注意力计算）降低 $f_Q f_K$ 倍。在 MViT 的典型设置中，$f_K$ 可以是 4（空间 $2\times2$ pool）或 8（时空 $1\times2\times2$ pool），注意力计算的 FLOPs 减少 4–8 倍。

### 4.2 整体网络结构

以 MViT-B 为例，输入为 $16 \times 224 \times 224 \times 3$（16 帧，帧间隔 $\tau=4$，约 2 秒的帧率降采样）：

| 阶段 | 操作 | 输出尺寸 |
|------|------|----------|
| Data layer | 时间步长 $\tau=4$ | $16 \times 224 \times 224 \times 3$ |
| Patch embedding | 3D Conv $3\times7\times7$, stride $2\times4\times4$, 96 通道 | $96 \times 8 \times 56 \times 56$ |
| Stage 1 | $1 \times$ MViT Block (Q pool=1, KV pool=1) | $96 \times 8 \times 56 \times 56$ |
| Down 1→2 | Pool $1\times3\times3$, stride $1\times2\times2$ + $1\times1$ Conv 192 | $192 \times 8 \times 28 \times 28$ |
| Stage 2 | $1 \times$ MViT Block (Q pool=1, KV pool=1) | $192 \times 8 \times 28 \times 28$ |
| Down 2→3 | Pool $3\times3\times3$, stride $2\times2\times2$ + $1\times1$ Conv 384 | $384 \times 4 \times 14 \times 14$ |
| Stage 3 | $1 \times$ MViT Block (Q pool=1, KV pool=2) | $384 \times 4 \times 14 \times 14$ |
| ...（MViT-L 的中间 block 会逐步增加 pool 强度（MViT-B 此阶段仅 1 个 block 无 pool 变化）） | | |
| Stage 3 (last) | $1 \times$ MViT Block (Q pool=2, KV pool=2) | $384 \times 4 \times 14 \times 14$ |
| Down 3→4 | Pool $3\times3\times3$, stride $2\times2\times2$ + $1\times1$ Conv 768 | $768 \times 2 \times 7 \times 7$ |
| Stage 4 | $2 \times$ MViT Block (Q pool=2, KV pool=2) | $768 \times 2 \times 7 \times 7$ |
| Head | 全局平均池化 + 线性分类器 | $K$ (类别数) |

各配置变体的通道数和 block 数量：

| 模型 | 通道数 | Stage1 | Stage2 | Stage3 | Stage4 | 参数量 | FLOPs |
|------|--------|--------|--------|--------|--------|--------|-------|
| MViT-B | 96 | 1 | 1 | 1 | 2 | 36.6M | 170G |
| MViT-L | 96 | 1 | 1 | 3 | 3 | 71.5M | 340G |
| MViT-L (32×56²) | 144 | 1 | 1 | 3 | 3 | 103.5M | 680G |

### 4.3 跨阶段降采样（Stage Transition）

MViT 通过时间-空间平均池化实现跨阶段的降采样（类似 CNN 的 max-pooling downsampling）。池化核大小与步长相同（非重叠池化），且同时作用于时间和空间维度。

例如 $3\times3\times3$ 核、$2\times2\times2$ 步长 → 输入 $8\times28\times28$ → 输出 $4\times14\times14$。

降采样后紧跟 $1\times1$ 卷积将通道翻倍（$96\to192\to384\to768$），这是标准 CNN 中"resolution / 2, channel × 2"策略的 Transformer 版本。

### 4.4 与 ViT 结构对比

| 特性 | ViT | MViT |
|------|-----|------|
| Token 序列长度 | 所有 block 固定 | 从高到低逐步减少 |
| 通道维度 | 所有 block 固定 | 从低到高逐步增加 |
| Attention 类型 | MHA（固定分辨率） | MHPA（可池化） |
| 视频输入方式 | 单帧或简单拼接 | 3D tubelet |
| 多尺度特征 | 无 | 4 级金字塔 |
| 参数效率 | 低（高 FLOPs/参数比） | 高 |
| 时间归纳偏置 | 弱（帧打乱不受影响） | 强（帧打乱性能大幅下降） |

## 五、核心创新点

### 创新点 1：Pooling Attention（多尺度自注意力操作）

#### (a) 痛点与动机

标准 MHA 在一个固定的 token 集合上操作。对于一组 $N$ 个 token，它让每个 token 和其他所有 $N-1$ 个 token 交互，得到 $N$ 个加权和。这在 $N$ 较小时（如 196 个 token 的图像分类）是合理的，但对于 $N=3136$ 的视频，$(3136)^2 \approx 10M$ 个 attention 权重需要计算和存储——每一步都如此，多层堆叠后不可承受。

更深层的动机是：**视觉信号中，不同尺度的特征需要不同的 token 密度。** 高频细节（边缘、纹理）需要高密度 token 来精确捕捉位置；低频语义（物体类别、场景）只需要低密度 token，但每个 token 需要足够大的感受野来"看到"整个物体。MHA 对两者一视同仁——所有 token 在所有层都有相同的分辨率和感受野——这就导致了非最优的计算分配。

#### (b) 方案细节

MHPA 在三个层面引入 pooling：

1. **Query Pooling ($\mathcal{P}_Q$)**：决定注意力输出的空间-时间分辨率。$s_Q=1$ 时输出分辨率不变；$s_Q=2$ 时输出分辨率减半。

2. **Key Pooling ($\mathcal{P}_K$)**：决定注意力计算时 tokens 的相互作用范围。$s_K>1$ 会降低 key 数量，每个 query 关注更少但更"浓缩"的 key。

3. **Value Pooling ($\mathcal{P}_V$)**：必须与 $\mathcal{P}_K$ 一致（$s_K=s_V$），因为 attention 权重直接作用在 value 上。

Pooling 实现为 3D 平均池化（对 $T\times H\times W$ 维度），使用重叠核。例如，对 $8\times56\times56$ 的输入：
- $k=(1,3,3), s=(1,2,2)$ → 输出 $8\times28\times28$（空间降采样 2×，时间不变）
- $k=(3,3,3), s=(2,2,2)$ → 输出 $4\times14\times14$（时空同时降采样 2×）

#### (c) 为什么有效

**机制 1：计算效率与感受野的同步增长**

当 $s_Q=1, s_K=2, s_V=2$ 时，每个 query 关注 $(H/s_K) \times (W/s_K)$ 个 key，相比标准 attention 少了 4 倍。同时，$s_K=2$ 的池化核覆盖了 $3\times3$ 的原始区域（如果使用 $k=(1,3,3)$ 的 pool），所以每个 key 已经聚合了局部邻域的信息。结果是：**计算量降低 4 倍，但每个 query 看到的感受野等价于 $3\times3$ 的卷积 + 全局 attention。**

**机制 2：多尺度特征的自然形成**

不同 block 使用不同的 $(s_Q, s_K)$ 组合。浅层 stage 使用 $(1,1)$——精细的特征图，局部注意力。深层 stage 使用 $(2,2)$——粗化的特征图，全局注意力（因为 token 数已经很少了）。这等价于 CNN 特征金字塔中，浅层大特征图 + 小卷积核 → 深层小特征图 + 大卷积核，但实现方式更简洁——只需改变池化参数。

**机制 3：注意力头之间的多尺度多样性**

不同注意力头可以用不同的池化参数。论文实验表明，在多头的多头池化注意力中（h=8 或 16），允许不同头使用不同的 pooling 步长（mix of 1, 2, 3）可以进一步提升性能，因为不同的头关注不同尺度的模式。

#### (d) 与 Related Work 的关系

- **ViT (Dosovitskiy et al., 2021)**：标准的 MHA 没有 pooling 机制，所有层使用固定 token 数。MHPA 是对 MHA 的最小改动——只增加了池化步骤——就获得了多尺度能力。

- **Swin Transformer (Liu et al., 2021)**：几乎同期提出的 Swin 使用局部窗口 attention + 窗口移位来实现跨层信息流动。MViT 的方案不需要窗口分区，也不需要移位，而是用学习好的注意力权重自然确定跨区域连接——具有更强的表达灵活性。

- **PVT (Wang et al., 2021)**：Pyramid Vision Transformer 通过空间降采样减少 token 数，但在注意力机制内不做 pooling——只在 stage 之间降采样。MViT 的 MHPA 在 attention 计算内部做 pooling，粒度更细。

#### (e) 如果去掉会怎样

论文 Table 5 的消融实验非常清晰地展示了每个设计的效果：
- 将 MHPA 替换为标准 MHA：Top-1 从 80.2% 降至 78.7%（−1.5%）
- 固定所有 stage 使用 $(s_Q=1, s_K=1)$（无多尺度）：从 80.2% 降至 79.1%（−1.1%）
- 同时去掉 MHPA 和固定分辨率：从 80.2% 降至 77.9%（−2.3%）

这说明：MHPA 带来的不是微调增益（+1.5%），而是结构性改进。同时池化时间和空间维度比只池化空间维度要好（+0.7%），说明时空协同池化至关重要。

### 创新点 2：多尺度特征金字塔

#### (a) 痛点与动机

CNN 的归纳偏置之一是通道-分辨率协同缩放：每个 stage 的分辨率减半、通道翻倍。这使 CNN 在计算量可控的前提下，从低层到高层逐步增加表示容量（通道数）和缩减空间冗余。

Transformer 没有这个设计。ViT 在所有层使用固定的序列长度和通道数——深层 token 仍然在精细的空间位置上做 attention，但语义上已经不需要这些空间细节了。这导致了计算浪费：深层 block 用同样的计算预算（$O(N^2)$ attention）去处理可以用更少 token 表示的语义特征。

#### (b) 方案细节

MViT 定义了 4 个 scale stage，每个 stage 内的 MViT block 保持相同的分辨率-通道配置。stage 之间通过**池化降采样**和 **1×1 卷积升通道** 过渡：

```
Stage 1:  8×56×56,  C=96   (1.7M tokens, 微结构/边缘)
Stage 2:  8×28×28,  C=192  (0.4M tokens, 局部纹理/形状)
Stage 3:  4×14×14,  C=384  (78K tokens, 物体部件)
Stage 4:  2×7×7,    C=768  (98 tokens, 物体语义)
```

token 数量从 Stage 1 到 Stage 4 降低了约 ~256 倍（从 1.7M 到 98），而通道数增加了 8 倍。这意味着深层 98 个 token、每个 token 768 维的表示，其**信息容量**（$98 \times 768 \approx 75K$）比浅层 ~25K 个 token 各 96 维降低了约 22 倍，但每个 token 的语义丰富度大大提高了——这正是"浓缩"的含义。

#### (c) 为什么有效

**计算分配的效率：**
注意力复杂度 $O(N^2)$ 决定了 token 减少的直接收益。Stage 4 有 98 个 token，其 attention 计算只有 $98^2 \approx 9,604$；如果保持 Stage 1 的 1.7M token，attention 计算是 $(1.7M)^2 \approx 2.9 \times 10^{12}$——相差 3 亿倍。当然，实际不会这么极端，但这个数量级差异说明了多尺度的计算优势。

**表示空间的适配性：**
深层特征只需要在较粗的空间网格上表示语义概念（"这是一只狗"不需要精确到像素位置），所以低分辨率是足够的。多尺度金字塔让每个 stage 的计算都聚焦于当前最需要的细节度。

**与下游任务的兼容性：**
检测、分割等 dense prediction 任务需要多尺度特征图——小物体在高分辨率特征图上检测，大物体在低分辨率特征图上检测。MViT 的 4 级输出可以直接作为 FPN（或者更精确地，Feature Pyramid）的输入，不需要额外的上采样/降采样。

### 创新点 3：强时间归纳偏置

#### (a) 痛点与动机

视频理解的最尴尬事实之一是：**很多"视频 Transformer"实际上并不理解运动。** 它们只是用更强大的图像 backbone 做单帧分类，然后取平均或简单聚合。

图 5 是 MViT 论文里最具洞察力的实验。作者拿 TimeSformer（一个专门设计的视频 Transformer）在 Kinetics-400 上测试——正常准确率 77.7%。然后把测试视频的所有帧按随机顺序排列，再跑一次——准确率 77.5%，只下降了 0.2%。这意味着模型的分类几乎完全依赖单帧外观，时间信息基本没用。

同样的实验在 MViT 上：正常准确率 80.2%，打乱帧后降至 ~76%（约 −4%）。说明 MViT 确实在学习时间模式。

#### (b) 方案细节

时间归纳偏置通过三个设计实现：

1. **3D Patch Embedding**：使用 3D 卷积核（$3\times7\times7$，stride $2\times4\times4$），把时空 tubelet 投影到 token。相邻两帧的对应区域被编码到同一个 tubelet 中，跨帧的共享信息从第一层就被捕捉。

2. **时间 Pooling**：跨 stage 的降采样同时对时间维度做 2× 池化（$8\to4\to2\to1$）。每个 stage 的时间分辨率减半，但每个 token 覆盖的时间跨度翻倍。Stage 1 的每个 token 覆盖 ~2 帧信息，Stage 3 覆盖 ~8 帧。

3. **Pooling Attention 的时空协同**：池化核覆盖 $T\times H\times W$ 三个维度，注意力计算过程中自然地建立时空对应关系。这是关键的：不像 TimeSformer 先空间后时间（两步独立），MViT 的 attention 同时考虑时空邻域。

#### (c) 为什么有效

**时间感受野的自然增长：**
Stage 1（8 帧，精细）：适合建模快速运动（~0.25秒内），如挥手、投掷。Stage 3（4 帧，粗化）：适合建模中等时序运动（~0.5秒），如走路、跑步。Stage 4（2 帧，最粗）：适合建模慢速语义（~1秒），如阅读、等待。这个多时间分辨率让模型能同时捕捉不同时间尺度的运动。

**帧乱序测试揭示的归纳偏置强度：**
表格数据：
- TimeSformer：正常 77.7% → 乱序 77.5%（−0.2%，几乎没有时间偏置）
- ViViT：正常 77.5% → 乱序 ~77%（微量时间偏置）
- MViT-B：正常 80.2% → 乱序 ~76%（有明显增强的时间偏置）
- MViT-L：正常 82.3% → 乱序 ~78%（大模型时间利用更好）

这个差异的本质原因：MViT 在每个块内部都做时空注意力（Pooling Attention 的池化核覆盖三个维度），而 TimeSformer 做的是"空间注意力 + 时间注意力"的解耦。MViT 的注意力模式天然强制了时间对齐。

#### (d) 如果去掉会怎样

消融实验：
- 去掉时间 pooling（所有池化只作用在空间维度）：79.3%（−0.9%）
- 用独立的空间和时间 attention 替换 MHPA：79.6%（−0.6%）
- 使用 MViT 原设计：80.2%（最优）

## 六、公式详解

### 公式 1：MHPA 的计算流程

给定输入 $X \in \mathbb{R}^{B \times T \times H \times W \times D}$：

**Step 1：线性投影**
$$Q = XW_Q, \quad K = XW_K, \quad V = XW_V$$
其中 $W_Q, W_K, W_V \in \mathbb{R}^{D \times D}$。

**Step 2：Pooling（每头独立）**
$$\tilde{Q}_h = \mathcal{P}_h(Q_h; \Theta_Q^h), \quad \tilde{K}_h = \mathcal{P}_h(K_h; \Theta_K^h), \quad \tilde{V}_h = \mathcal{P}_h(V_h; \Theta_V^h)$$
其中 $Q_h \in \mathbb{R}^{B \times T \times H \times W \times (D/h)}$ 是第 $h$ 个头，$\Theta_Q^h = (k_Q^h, s_Q^h, p_Q^h)$ 是池化参数。

**Step 3：注意力计算**
$$\text{Attn}_h = \text{Softmax}\left(\frac{\tilde{Q}_h \tilde{K}_h^T}{\sqrt{D/h}}\right) \tilde{V}_h$$

**Step 4：输出投影**
$$\text{Out} = \text{Concat}(\text{Attn}_1, \dots, \text{Attn}_h) W_O$$

**直觉理解：**
- Pooling 操作可以看作是对 $Q, K, V$ 的"预处理"——把 token 数量的维度压缩到适合注意力计算的规模
- 不同 head 可以有不同的池化参数，实现多头多尺度

### 公式 2：复杂度对比

**MHA（标准）：**
注意力矩阵 $A \in \mathbb{R}^{(THW) \times (THW)}$。

计算瓶颈是矩阵乘法 $QK^T$：
$$\Omega_{\text{MHA}} = (THW)^2 (D/h) \cdot h = (THW)^2 D$$

**MHPA（MViT）：**
$\tilde{Q} \in \mathbb{R}^{(THW/s_Q) \times (D/h)}$，$\tilde{K} \in \mathbb{R}^{(THW/s_K) \times (D/h)}$。

计算瓶颈是：
$$\Omega_{\text{MHPA}} = \frac{(THW)^2}{s_Q s_K} D$$

所以加速比为 $s_Q s_K$。

对 MViT-B 的 Stage 2（$8\times28\times28$）：
- $s_Q=1, s_K=1$ → MHA: $L=6272$, 注意力计算 ≈ 200M ops
- 标准 MHA 在相同 $L$ 下: ≈ 200M ops（这里相同，因为 Stage 2 的 Q/KV 都不做 pool）

对 Stage 3（$4\times14\times14$，$s_Q=1, s_K=2$）：
- $s_Q=1, s_K=2$ → 注意力计算降到 1/2
- 实际值: token 约 784, MHA 需要 ~600K ops, MHPA 需要 ~150K ops

对 Stage 4（$2\times7\times7$，$s_Q=2, s_K=2$）：
- $s_Q=2, s_K=2$ → 注意力计算降到 1/4
- token 数 ~98 vs 392 ops→节省 4×（约 370K vs 95K）

### 公式 3：通道-分辨率缩放关系

$$C_{i+1} = 2C_i, \quad R_{i+1} = \frac{R_i}{2}$$

其中 $R_i$ 表示 stage $i$ 的 $H \times W$ 空间分辨率，$C_i$ 是通道数。

这个关系来自 CNN 的"计算守恒"法则：
- Stage $i$ 的 FLOPs 约为 $\mathcal{O}(C_i^2 \cdot R_i)$
- Stage $i+1$ 的 FLOPs 约为 $\mathcal{O}((2C_i)^2 \cdot (R_i/4)) = \mathcal{O}(4C_i^2 \cdot R_i/4) = \mathcal{O}(C_i^2 \cdot R_i)$

所以每个 stage 的 FLOPs 大致相等——计算资源在深度方向上均衡分配。

## 八、代码对照

从官方代码库 mvit/attention.py 可以看出几个论文未展开的关键实现细节：

### 8.1 MHPA 的工程实现

```python
# mvit/attention.py (simplified)
class PoolAttention(nn.Module):
    def __init__(self, dim, num_heads, 
                 q_stride=1, kv_stride=2,
                 pool_first=True):
        self.num_heads = num_heads
        self.qkv = nn.Linear(dim, dim * 3, bias=qkv_bias)
        self.proj = nn.Linear(dim, dim)
        self.q_pool = nn.AvgPool3d(
            kernel_size=q_stride, stride=q_stride, 
            padding=0) if q_stride > 1 else nn.Identity()
        self.kv_pool = nn.AvgPool3d(
            kernel_size=kv_stride, stride=kv_stride,
            padding=0) if kv_stride > 1 else nn.Identity()
    
    def forward(self, x):
        B, N, C = x.shape  # N = T*H*W
        # 线性投影
        qkv = self.qkv(x).reshape(B, N, 3, self.num_heads, -1)
        q, k, v = qkv.unbind(2)
        
        # reshape to (B, head, T, H, W, C//head)
        q = q.reshape(B, T, H, W, -1)
        k = k.reshape(B, T, H, W, -1)
        v = v.reshape(B, T, H, W, -1)
        
        # Pooling (may be identity if stride=1)
        q = self.q_pool(q).flatten(1, 3)
        k = self.kv_pool(k).flatten(1, 3)
        v = self.kv_pool(v).flatten(1, 3)
        
        # Attention
        attn = (q @ k.transpose(-2, -1)) / math.sqrt(q.size(-1))
        attn = attn.softmax(dim=-1)
        out = (attn @ v).transpose(1, 2).flatten(2)
        return self.proj(out)
```

**值得注意的细节：**

1. **Pooling 默认对 $T\times H\times W$ 做 AvgPool3d**，不是只对空间维度。$T$ 维度也会被池化，这是保持时间多尺度的关键。

2. **$s_Q=s_K=1$ 时 Pooling 替换为 Identity**——这不是"不降采样"，而是选择在这个 block 中跳过降采样。

3. **Pooling 发生在 Attention 计算之前，不在之后**。这和 ResNet 的跨 stage 池化不同——MViT 把池化嵌入了 attention 内部，而非分离的操作。

### 8.2 位置编码

MViT 使用**可学习的绝对位置编码**（sin-cos 初始化），但只在第一个 stage 的输入处添加。后续 stage 的位置信息通过池化操作自然传递——池化后的特征已经保留了空间位置关系，不需要在每个 stage 重新加位置编码。

这不同于 ViT（只在输入层加）和 Swin（使用相对位置编码）。MViT 的"只在第一层加位置编码"实际上依赖于池化的保位置特性（平均池化保留位置的相对顺序），这在严格意义上也有信息丢失的风险——但在实践中被证明足够好。

### 8.3 训练配置

- 优化器：AdamW（lr=1e-3, weight decay=0.05）
- 预热：5 epoch 线性 warmup
- 学习率调度：Cosine decay（300 epoch）
- 数据增强：RandAugment (m=9, n=2), Mixup (alpha=0.8), CutMix (alpha=1.0)
- 正则化：Stochastic Depth (rate=0.1), Label Smoothing (rate=0.1)
- 视频采样：时间步长 $\tau=4$（每 4 帧取一帧），16 帧输入 = 64 帧原始视频长度

## 九、论文在技术树中的位置

### 9.1 前驱工作

1. **ResNet (He et al., 2016)** — 4-stage 特征金字塔的设计是 MViT stage 级别的直接灵感。MViT 的通道-分辨率缩放策略（$C \times 2, R/2$）完全继承了 ResNet 的计算公平分配原则。

2. **ViT (Dosovitskiy et al., 2021)** — MViT 的基本组件（MHA, MLP, LayerNorm, residual connection）全部来自 ViT。MViT 没有发明新的 Transformer 操作，而是改造了 attention 的计算方式。

3. **SlowFast (Feichtenhofer et al., 2019)** — MViT 的作者中包括了 SlowFast 的原创者，所以 SlowFast 的"多时间分辨率"思想（一条慢路径处理低帧率、一条快路径处理高帧率）直接影响了 MViT 的时间多尺度设计。

4. **FPN (Lin et al., 2017)** — 特征金字塔网络的多尺度输出概念被 MViT 融入 Transformer 中，但 FPN 是在 CNN 特征图上额外构建的，MViT 则直接把多尺度集成到 backbone 中。

### 9.2 同期竞争

- **Swin Transformer (Liu et al., 2021)** — 2021 年 3 月（比 MViT 早 1 个月），Swin 用移位窗口实现了层级注意力。MViT 和 Swin 都解决了"Transformer 如何层级化"的问题，但方案完全不同：Swin 靠窗口分区 + 移位，MViT 靠池化 attention。两者在这个问题的解法上各有千秋，但 MViT 天然支持视频（因为有 3D pooling），Swin 需要做视频扩展（Video Swin Transformer）。

- **TimeSformer (Bertasius et al., 2021)** — 将时空 attention 分解为分离的空间 + 时间 attention。MViT 与其同期但方式不同：不做分解，让池化 attention 自然处理时空。

- **ViViT (Arnab et al., 2021)** — 最接近的竞争产品，也探索了多种时空 attention 分解方式。但 ViViT 的最大弱点是依赖 ImageNet-21K 预训练，而 MViT 只需要 ImageNet-1K，说明多尺度设计本身替代了部分大数据需求。

### 9.3 后续影响

1. **MViTv2 (Li et al., 2022)** — 直接继承，引入 decomposed relative position encoding 和 residual pooling connections，在视频和图像上进一步提升性能。

2. **VideoMAE (Tong et al., 2022)** — 使用 MViT-Huge 作为 backbone 进行视频自监督学习，在 Kinetics-400 上达到 87.4% 的 SOTA。这是 MViT 作为通用视频 backbone 的最佳证明。

3. **InternVideo (Wang et al., 2022)** — 结合 VideoMAE（MViT 架构）和另一种方案，形成统一的视频 foundation model。

4. **OmniMotion / 3D 重建** — MViT 的 3D pooling attention 思想也被应用于 3D 点云理解、光流预测等需要时空建模的任务。

5. **ConvNeXt (Liu et al., 2022)** — 将 Swin Transformer 的高级设计反向移植到 CNN。MViT 的池化 attention 也被一些 conv-transformer hybrid 工作借鉴。

## 十、局限

1. **位置编码的单层使用**。MViT 只在第一个 stage 的输入处添加位置编码，虽然通过池化的位置留存性可以部分保持位置信息，但在深层 stage（时间分辨率极低时）位置编码的精确度可能不足以区分空间邻接关系。MViTv2 通过引入 decomposed relative position encoding 解决了这个问题。

2. **Pooling 参数的手工设计**。每个 stage 的 pooling kernel size、stride、padding 需要手工调节，缺乏自动化或自适应机制。不同数据集、不同输入分辨率可能需要不同的 pooling 策略，调参成本较高。

3. **长时间视频的建模限制**。MViT 的输入是固定长度的视频片段（16–32 帧，覆盖 1–2 秒），无法直接处理更长的时间跨度。虽然可以通过滑动窗口扩展，但窗口之间的时序关系丢失了。后续工作（如 TimeSformer 的 64 帧版本、VideoMAE 的 masked reconstruction）尝试了不同的解决方案。

4. **运动建模并非显式**。MViT 的时间归纳偏置是隐式的——通过池化迫使 attention 在时空上约束——而不是显式的运动建模（如光流或 3D 卷积）。虽然打乱帧测试显示了 MViT 的有效性，但对于极端快速运动或复杂的时空交互（如多人交互），隐式建模可能不如显式运动模块。

5. **Patch Embedding 的时间分辨率不灵活**。首层 3D 卷积的时间步长 $\tau$ 是固定的超参数（默认 $\tau=4$）。对于慢速变化和快速变化的视频，理想的采样率不同，但 MViT 使用固定的帧间隔。

## 十一、小结

MViT 是视频 Transformer 发展中一个关键的**桥梁节点**。它不是在 Transformer 架构上做功能补丁（如 VTN 在 ViT 顶部加时间层），也不是从头设计全新的 Transformer 变体（如 Swin 的移位窗口），而是在**最小化架构改动**的前提下，把一个被忽略的信息——视觉信号的多尺度本质——通过池化 attention 注入 Transformer。

它的贡献可以这样概括：

1. **对视频理解领域**：提出了第一个不需要 ImageNet-21K 预训练、在参数量和计算量都小于 3D CNN 的前提下达到 SOTA 的视频 Transformer。这证明了视频 Transformer 可以像图像 ViT 一样有实用价值。

2. **对 Transformer 架构发展**：证明了"多尺度"不是 CNN 的专属设计——Transformer 通过修改 attention 的内部计算就可以实现层级化。这为后来的 Swin Transformer、ConvNeXt 等"Transformer 层级化"工作提供了技术谱系上的合理性和信心。

3. **对视觉表征学习**：多尺度 attention 的自然副产品——强时间归纳偏置——揭示了"架构设计本身可以替代部分预训练数据需求"。MViT 不需要大型外部预训练（ImageNet-21K），因为多尺度设计提供了更强的归纳偏置。

**一句话总结：MViT 用 Pooling Attention 在 Transformer 内部实现了 CNN 的多尺度特征金字塔，在不增加参数量的前提下，同时提升计算效率和模型性能——是视频 Transformer 的奠基之作。**
