## 一、先搞清楚"问题在哪里"

2020 — 2021 年初，视觉 Transformer 的进展主要沿着两条路线展开。

**第一条路线：单尺度全局 attention。** ViT（Dosovitskiy et al., 2020）证明了 Transformer 可以不做任何卷积就能做图像分类，但它有一个根本性的设计假设——**所有层的通道宽度和空间分辨率保持不变**。对于 224×224 输入，ViT 切成 16×16 patch（196 个 token），每层都在这个序列长度上做全局 self-attention。这是个与 NLP 对齐的设计——句子中的 token 本来就没有空间尺度的概念。

**第二条路线：视频 Transformer。** TimeSformer、ViViT、VTN 等工作把 ViT 扩展到视频，但核心思路是"先拿 ImageNet-21K 预训练一个 ViT，然后移植到视频上做微调"。这种方法有两个问题：第一，依赖大规模外部数据（ImageNet-21K 有约 1400 万张图片，是 Kinetics-400 的 60 倍）；第二，ViT 的架构本身没有对视觉信号的密度和时空连续性做任何适配。

MViT 的切入点非常清晰：**视觉信号和语言信号的根本差异在于密度和尺度**。一张 224×224 图片有 50176 个像素，而一句话平均只有 20-30 个 token。视觉信息是连续、密集、有层级结构的——浅层需要高空间分辨率建模边缘和纹理，深层需要大通道容量建模语义概念。用"NLP 的架构不做修改直接做视觉"这个假设，在 MViT 看来需要重新审视。

同期工作 Swin Transformer（Liu et al., 2021，B2-2）也看到了层级化的必要性，但走的是"局部窗口 + 移位"的路线。MViT 走的是另一条路：**不在 attention 中引入局部性假设，而是通过 pooling attention 渐进地降低分辨率并扩展通道维度**。两条路线都实现了多尺度特征金字塔，但机制完全不同。

## 二、旧路线的真正问题

### 2.1 ViT 的单分辨率瓶颈

ViT 在所有层维持相同的序列长度和通道维度。以 ViT-Base 为例（表 1 在论文中为 Table 1 架构表）：输入 8×224×224（视频场景），经过 1×16×16 patch projection 后，token 数为 8×14×14 = 1568（加上 class token 为 1569），通道数 768。12 层 Transformer block 全部在 768×8×14×14 的维度上运行。

这意味着**浅层和深层的计算量是均匀的**，但视觉任务中浅层不需要大量通道来建模简单特征。这既浪费计算，也限制了深层的能力——如果浅层能少用一些通道，就可以把"预算"留给深层。

### 2.2 视频 Transformer 的"伪时序建模"

论文做了一个非常直接的实验来揭露问题：在推理时打乱输入帧的时间顺序。对于 ViT-B（从 ImageNet-21K 预训练微调来的视频模型），打乱帧序后准确率从 79.3% 几乎不变（仍然是 79.3%）。对于从零训练的 ViT-B，准确率从 68.5% 变为 68.4%（仅下降 0.1%）。

**这说明 ViT 根本没有建模时序信息。** 它本质上在做"bag-of-frames"分类——每帧独立提取特征然后平均，时序位置编码完全被忽略了。

而 MViT-B 在同样的测试下，准确率从 77.2% 下降到 70.1%，降幅 7.1%。这是因为多尺度结构中，帧与帧之间的时空信息通过 pooling attention 的时空池化操作被强制融合了。

### 2.3 外部预训练的依赖性

2021 年的视频 Transformer 工作（VTN、TimeSformer、ViViT）全部依赖 ImageNet-21K 预训练。VTN 的消融实验显示，从 ImageNet-1K 预训练（75.6% top-1）换成 ImageNet-21K（78.6% top-1），提升 3 个百分点——说明性能很大程度上来自预训练数据量，而非架构设计。

MViT 的所有模型都是**在 Kinetics 上从零训练**，不使用任何外部数据，在同等计算量下达到了更高或相当的准确率。

## 三、这篇论文的核心思路

**核心思路：构建一个多尺度特征金字塔的 Transformer，通过 pooling attention 让每一层/每一阶段的时空分辨率和通道容量动态变化。**

具体来说不是用一个固定的架构处理所有尺度，而是设计了 4-5 个 scale stage：

- **早期 stage：** 高空间分辨率（如 H/4 × W/4）+ 小通道容量（如 D=96），适合建模细粒度的低级视觉特征（边缘、纹理）
- **中期 stage：** 逐步降低分辨率、增加通道数，特征从"大量细粒度"变为"少量高维语义"
- **晚期 stage：** 低空间分辨率（如 H/32 × W/32）+ 大通道容量（如 8D=768），建模全局语义

这个设计直接继承自 CNN 的设计原则（VGG、ResNet 的 stage 结构），但用 Transformer 的方式实现——核心操作不是卷积步进，而是**Multi Head Pooling Attention（MHPA）**。

论文在多处强调一个关键洞察：**多尺度结构天然带来了强时序归纳偏置**。因为浅层的高分辨率特征必须在空间和时间维度上都被池化才能进入下一 stage，这个过程中帧之间的对应关系被强制建立了。

## 四、整体结构与 forward 流程

### 4.1 Multi Head Pooling Attention（MHPA）

MHPA 是 MViT 的核心算子，代替了标准 Transformer 中的 Multi Head Attention（MHA）。

**标准 MHA 的流程：**
1. 对输入 X 做线性投影得到 Q, K, V（维度不变）
2. 在 Q 和 K 之间计算 attention 矩阵（序列长度 L × L）
3. 用 attention 矩阵加权 V

**MHPA 的改进：**
1. 对输入 X 做线性投影得到 Q̂, K̂, V̂
2. **对 Q̂, K̂, V̂ 分别做空间/时间 pooling**（池化核大小、步长、padding 可独立配置）
3. 在缩短后的序列上计算 attention

重要约束：K 和 V 的步长必须相同（ $s_K \equiv s_V$ ），否则 attention 矩阵的维度会不匹配。

Pooling 算子 𝒫(·; Θ) 在 T×H×W 三个维度上分别做池化，输出序列长度为：
L̃ = ⌊(L + 2p - k)/s⌋ + 1（按坐标分别计算）

**为什么 pooling attention 同时降低了计算和内存：**
- Attention 计算复杂度与序列长度的平方成正比
- 对 K, V 做池化（stride > 1），attention 矩阵大小从 $L^2$ 变成 $(L/f_Q)\cdot(L/f_K)$
- 对 Q 做池化，输出序列长度也缩短了

论文给出了具体复杂度公式（复杂度分析节）：单头的 runtime 复杂度为 $O(THWD/h \cdot (D + THW/f_Q f_K))$ ，其中 $f_Q$ , $f_K$ 是 Q 和 K 的序列缩减因子。

### 4.2 MViT 整体架构（MViT-Base 为例）

论文 Table 3(b)（架构表）给出了 MViT-Base 的完整结构，共有 5 个阶段（包括数据层）：

```text
输入视频: T×H×W×3
    │
    ▼
[data layer]                stride τ×1×1
    │                       输出: T×H×W
    ▼
[cube₁]                     3×7×7 时空 cube, D=96
     stride 2×4×4
    │                       输出: 96×T/2×H/4×W/4
    ▼
┌─────────────────────────────────────────┐
│ scale₂  (MHPA + MLP × N₂=1)            │
│   通道: 96, 分辨率: T/2×H/4×W/4         │
│   MHPA pool K,V: adaptive stride        │
└────────────────┬────────────────────────┘
                 │   MLP 输出维度扩展: 96 → 384
                 ▼
[scale transition]  Q pool stride (1,2,2)
                 │   通道加倍: 96 → 192
                 ▼
┌─────────────────────────────────────────┐
│ scale₃  (MHPA + MLP × N₃=2)            │
│   通道: 192, 分辨率: T/2×H/8×W/8        │
└────────────────┬────────────────────────┘
                 │   MLP 输出: 192 → 768
                 ▼
[scale transition]  Q pool stride (1,2,2)
                 │   通道: 192 → 384
                 ▼
┌─────────────────────────────────────────┐
│ scale₄  (MHPA + MLP × N₄=11)           │
│   通道: 384, 分辨率: T/2×H/16×W/16      │
│   最深的主干 stage                      │
└────────────────┬────────────────────────┘
                 │   MLP 输出: 384 → 1536
                 ▼
[scale transition]  Q pool stride (1,2,2)
                 │   通道: 384 → 768
                 ▼
┌─────────────────────────────────────────┐
│ scale₅  (MHPA + MLP × N₅=2)            │
│   通道: 768, 分辨率: T/2×H/32×W/32      │
│   最粗分辨率, 最大通道容量              │
└─────────────────────────────────────────┘
```

关键设计点：

**通道扩展与分辨率缩减的关系：**
- 每个 stage 之间，分辨率减半（H×W → H/2 × W/2）
- 通道加倍（D → 2D）
- 这种设计保持了各 stage 间的计算量大致平衡，和 ResNet 的 stage 设计原则一致

**MHPA 的使用策略：**
- 每个 stage 的第一个 block 做 Q pooling（开始降分辨率）
- 同 stage 内的其余 block 只做 K, V pooling（保持分辨率但降低计算量）
- K, V pooling 的 stride 随 stage 自适应调整，使得 K, V 序列长度在所有 stage 中保持一致（具体策略见 §8.4 消融实验：scale₂ 到 scale₅ 的 K,V stride 逐步从 $1\times8\times8$ 缩小到 $1\times2\times2$ ）

**Head 数的增长：**
- scale₁ 只有 h=1 个头
- 随通道数增加，每头的通道数保持约 96，因此 head 数也增长
- 这控制了 attention 的内存占用

### 4.3 模型变体

> **关于池化类型的说明：** max-pool 指池化操作取窗口内最大值做下采样；conv-pool 指用可学习的 channel-wise 卷积 + LayerNorm 代替最大池化。后者效果更好但速度略慢，完整消融见 §5.1。

**MViT-Base（表 A.1b，附录架构表）：**
- 总层数 16（1+2+11+2）
- 70.5G FLOPs（16×4 采样），36.5M 参数
- K400 top-1: 77.2%（max-pool）/ 78.4%（conv-pool）

**MViT-Small（表 A.1c）：**
- 只用了 3 个 scale stage（无最高分辨率 stage）
- 初始 cube: 3×8×8, stride 2×8×8, D=128
- N 分布: [3, 7, 6]，总层数 16
- 32.9G FLOPs，26.1M 参数
- K400 top-1: 74.3%（max-pool）/ 76.0%（conv-pool）

**与 ViT-Base 的对比（表 A.1a）：**
- ViT-B: 179.6G FLOPs, 87.2M 参数, K400 top-1 68.5%（从零训练）
- MViT-B: 70.5G FLOPs, 36.5M 参数, K400 top-1 77.2%（从零训练）
- MViT-B 用 60% 更少的 FLOPs、58% 更少的参数，高出 8.7 个百分点

## 五、核心模块逐个解释

### 5.1 Pooling 算子：为什么需要"重叠"池化

论文对池化核大小（k）和步长（s）的关系做了仔细消融（表 13）：

| kernel k 与 stride s 的关系 | top-1 |
|---------------------------|-------|
| k = s（非重叠） | 76.1 |
| k = 2s + 1（大重叠） | 75.5 |
| **k = s + 1（小重叠）** | **77.2** |
| average pool (k=s+1) | 75.4 |
| conv pool (k=s+1) | 78.3 |
| conv pool (3×3×3) | **78.4** |

关键发现：
- 非重叠池化（k=s）去掉了很多信息
- 适当的重叠（k=s+1）是最好的——这保证了池化窗口之间有平滑的过渡
- 重叠太多（k=2s+1）反而有害——可能是过度平滑了空间信息
- **平均池化远不如最大池化**（75.4 vs 77.2）
- **可学习的卷积池化（channel-wise conv + LN）最好**（78.3-78.4），在所有实验中最终都用了这个

### 5.2 通道扩展与 MLP 输出维度

在每个 stage 的最后一个 block，MLP 的输出维度从标准配置（4D）增加到更大的值。具体来说，在 stage 到 stage 的过渡中，前一个 stage 的 MLP 输出维度被设置为下一 stage 输入维度的一半加上标准扩张。

例如 MViT-B 中：
- scale₂ 的 MLP 输出维度：4×96=384（等于下一 stage 的输入 192 的两倍）
- scale₃ 的 MLP 输出维度：4×192=768
- scale₄ 的 MLP 输出维度：4×384=1536
- scale₅ 的 MLP 输出维度：4×768=3072

### 5.3 Skip connection 的通道适配

当 stage 过渡导致通道数变化时，残差连接也需要适配。论文消融了三种策略（附录表 A.4）：

| 方案 | K400 top-1 |
|------|-----------|
| (a) LN + 线性层（默认） | **77.2** |
| (b) 直接线性变换（无 LN） | 74.6 |
| (c) 无 skip connection | 74.7 |

**(a) 比 (b) 高 2.6 个百分点**——说明在通道扩展时，先做 LN 再线性变换对优化至关重要。推测原因：不做 LN 直接线性变换会放大输入的尺度，造成梯度不稳定。

### 5.4 位置编码：分离的空间与时间编码

论文消融了四种位置编码配置（表 8）：

| 方案 | 参数 | top-1 |
|------|------|-------|
| (i) 无位置编码 | 36.2M | 75.8 |
| (ii) 仅空间位置编码 | 36.5M | 76.7 |
| (iii) 联合时空编码 | 38.6M | 76.5 |
| **(iv) 分离时空编码** | **36.5M** | **77.2** |

分离编码（iv）最佳，且比联合编码少 2.1M 参数。说明空间位置和时间位置可以用不同的编码方式处理，没必要耦合在一起。

### 5.5 Cube embedding：从 2D patch 到 3D cube

不同于 ViT 的 2D patch（1×16×16），MViT 使用了时空 cube（如 3×7×7）来做输入投影。消融实验（表 9）显示，**cube 在时间维上的扩展（cT > 1）明显有益**：

| 采样 T×τ | cube 大小 (cT×cH×cW) | top-1 |
|----------|---------------------|-------|
| 8×8 | 1×4×4（仅空间 patch） | 74.5 |
| 8×8 | 1×7×7 | 75.6 |
| 8×8 | **3×7×7**（时空 cube） | **75.9** |
| **16×4** | **3×7×7（默认）** | **77.2** |
| 32×2 | 3×7×7 | 77.2 |
| 32×2 | 7×7×7 | 77.3 |

如果只做空间 patch（1×4×4），性能比 3×7×7 cube 差了 1.4 个百分点。更大的时间 cube（7×7×7）没有进一步提升，说明 3 帧的时间范围已经足够捕获局部运动信息。

## 六、核心公式与数学直觉

### 6.1 Pooling Attention 的计算复杂度

先定义公式中的变量： $T$ = 时间帧数， $H, W$ = 空间高和宽， $D$ = 通道维度， $f_Q$ = Q 的序列缩减因子（= $s_T^Q \cdot s_H^Q \cdot s_W^Q$ ）， $f_K$ = K 的序列缩减因子。

举个具体例子帮助直觉：假设输入 $T=8, H=14, W=14, D=96$ ，标准 MHA 的 attention 矩阵大小为 $(THW)^2 = (1568)^2 \approx 2.5\text{M}$ 个元素。如果 $f_Q f_K = 1\times8\times8 = 64$ ，则 MHPA 的 attention 矩阵缩小到 $(1568/8)^2 \approx 38\text{K}$ 个元素，降低约 65 倍。

标准 MHA 的复杂度（以视频输入 D×T×H×W 为例）：
- QKV 投影: O(THWD²)
- Attention 矩阵: O((THW)² D)
- 总复杂度: O(THWD² + (THW)² D)

MHPA 的复杂度：
- QKV 投影 + Pooling: $O(THWD^2)$ （Pooling 开销 $O(THWD\cdot k_T k_H k_W / s_T s_H s_W)$ 可忽略）
- 缩短后的 Attention 矩阵: $O((THW)^2 D / f_Q f_K)$
- 总复杂度: $O(THWD^2 + (THW)^2 D / f_Q f_K)$

如果 $f_Q f_K = 64$ （8×8 的时空降采样），attention 部分的理论加速比是 64 倍。

### 6.2 复杂度与通道数的权衡

从复杂度公式中可以看到两个关键项：
- 通道项: $THWD^2$ （随 $D^2$ 增长）
- 序列项: $(THW)^2 D / f_Q f_K$ （随序列长度平方增长）

MViT 的设计核心是管理这两项的平衡：
- 早期 stage: $THW$ 大， $f_Q f_K$ 小（序列项主导），所以 $D$ 设小
- 后期 stage: $THW$ 小（池化后）， $f_Q f_K$ 大，所以 $D$ 可以设大

用最简单的数值例子帮助直觉：如果 $D=96$ 时分辨率为 $T/2\times H/4\times W/4$ ，通道加倍到 $D=192$ 时分辨率变为 $T/2\times H/8\times W/8$ ，则 $192 \times (H/8 \times W/8) \approx 96 \times (H/4 \times W/4) \times 2/4 = \text{原量的一半}$ 。再加上同一 stage 内的 Q, K, V pooling 压缩，计算量大致保持稳定。

当通道加倍且分辨率减半时（各减 $2\times$ ），计算量大致保持稳定： $2D \times (THW/4) \approx D \times THW$ 。

3. **MViT-B 64×3（81.2%）** 与 **ViT-L-ViViT（81.3%，IN-21K，310.8M 参数）** 持平，但 FLOPs 少 91%（455×3×3=4.1T vs 3992×3×4=47.9T），参数少 8.5 倍

**总体趋势总结：** MViT-B 在 16×4 输入（70.5G FLOPs，无外部预训练）下达到 78.4%，虽然略低于部分 IN-21K 预训练的 ViT 变体（TimeSformer 80.7%、ViViT 81.3%），但 FLOPs 仅为它们的 1/14–1/34，参数少 3–8 倍。增加帧数（32×3、64×3）带来的边际收益递减（每翻倍帧数提升约 1-2%），但计算量增长可控。MViT-B 64×3（81.2%）与 ViT-L-ViViT（81.3%，310.8M 参数）持平，计算量仅为后者的 1/11。

## 八、关键消融实验

### 8.1 帧打乱测试（表 7）

| 模型 | 正常 | 打乱帧 | 差异 |
|------|------|--------|------|
| **MViT-B** | **77.2** | **70.1** | **−7.1** |
| ViT-B（从零） | 68.5 | 68.4 | −0.1 |
| ViT-B（IN-21K，79.3） | 79.3 | 79.3 | 0.0 |

结论：ViT 的视频模型基本在做"bag-of-frames"，时序位置编码被完全忽略。MViT 的多尺度架构强制建模了时序信息。

### 8.2 ViT 中加入一个 scale stage（表 A.2）

| 模型 | FLOPs | 内存 | top-1 |
|------|-------|------|-------|
| ViT-B（12 blocks, 1 stage） | 179.6G | 16.8G | 68.5 |
| 2-scale ViT-B（6+6 blocks，Q pool） | 111.1G（−38%） | 9.8G（−41%） | **71.0（+1.5）** |
| ViT-B + K,V pool | 148.4G | 8.9G | 69.1（+0.6） |

仅仅将 ViT-B 的最后 6 层放在一个降低分辨率的 stage 中，就同时提升了准确率（+1.5%）并大幅降低计算量（−38% FLOPs，−41% 内存）。

## 九、相关工作原理与技术谱系

### 9.1 技术树中的位置

```text
[Transformer, Vaswani et al., 2017]
   │
   ├── [ViT, Dosovitskiy et al., 2020] ← 首个视觉 Transformer
   │      │
   │      ├── [DeiT, Touvron et al., 2021] ← 改进训练策略
   │      │
   │      ├── [PVT, Wang et al., 2021] ← 金字塔 Transformer
   │      │
   │      ├── [Swin Transformer, Liu et al., 2021] ← B2-2，局部窗口+层级
   │      │
   │      └── [MViT, Fan et al., 2021] ← ★ 本篇，pooling attention + 层级
   │             │
   │             ├── [MViTv2, Li et al., 2022] ← 改进版
   │             ├── [Video MAE, Tong et al., 2022] ← 视频 MAE 基于 MViT
   │             └── [InternVideo, Wang et al., 2022] ← 统一视频表示
   │
   └── 视频 Transformer
          ├── [VTN, Neimark et al., 2021] ← ViT + 时序聚合
          ├── [TimeSformer, Bertasius et al., 2021] ← 分离时空 attention
          ├── [ViViT, Arnab et al., 2021] ← 多帧 tubelet + 变体
          └── [MViT, Fan et al., 2021] ← ★ 本篇
```

MViT 同时位于 **B2**（视觉 backbone 层级化）和 **B5**（视频 Transformer）两条技术线上。它是少有的同一篇论文同时回答"视觉 backbone 是否需要层级化"和"视频/时空能否统一建模"两个问题的作品。

### 9.2 与 Swin Transformer 的对比

MViT 和 Swin Transformer 都是 2021 年推出的层级化视觉 Transformer，但路线完全不同：

| 维度 | Swin Transformer | MViT |
|------|-----------------|------|
| 降低复杂度的方式 | 局部窗口内做 attention（7×7 window） | 对序列做 pooling 缩减后做 attention |
| 信息跨窗口/跨区域流动 | 通过 shifted window 交错实现 | 通过 Q pool 降分辨率时融合 |
| 层级化实现 | Patch Merging（拼接 → 线性投影） | MLP 输出维度扩展 + Q pool 降采样 |
| 位置编码 | 相对位置偏置 | 分离的时空绝对位置编码 |
| 核心归纳偏置 | 局部性（相邻像素更相关） | 多尺度（视觉信号从密集到稀疏） |
| 视频支持 | 默认 2D，需扩展为 Video Swin | 原生 3D（时空 cube + 时空 attention） |
| ImageNet top-1 | Swin-B: 83.5% (224²) | MViT-B-24-wide: 84.8% (320²) |
| K400 top-1 | Video Swin: 84.8（后续工作） | MViT-B-24: 83.8（K600） |

**两者的共同点**：都看到了 ViT 单分辨率、O(N²) 复杂度的限制，都引入了 4-5 级金字塔结构，都在分类和检测任务上超越了 ViT。

**根本差异**：Swin 通过"限制 attention 范围"来降低复杂度，MViT 通过"缩小 attention 对象（pooling 后的 K,V）"来降低复杂度。Swin 更偏工程优化（cache-friendly 的内存访问），MViT 更偏信号处理（从密集到稀疏的多尺度金字塔）。

### 9.3 与 TimeSformer / ViViT 的对比

MViT 是 2021 年唯一不依赖 ImageNet-21K 预训练的视频 Transformer：

- **VTN**: 给每帧提取 ViT 特征，再加时序聚合模块
- **TimeSformer**: 在 ViT 上做分离时空 attention（先空间后时间）
- **ViViT**: 多种变体（tubelet + 因子化/稀疏 attention）

这些工作都用"在 ViT 基础上加视频适配"的思路。MViT 是从头设计了一个适合视频的 Transformer 架构——多尺度金字塔对视频的时空密集信号天然合适。

## 十、时序偏置的定性分析

论文在附录中做了一个非常有趣的定性实验（附录图 A.5）：**可视化每一层每个 head 的"平均 attention 距离"**（即 attention 权重的加权平均距离）。

在初始化时：
- ViT-B：所有层、所有 head 的平均 attention 距离几乎完全一致
- MViT-B：不同层的平均 attention 距离差异巨大——部分 head 关注近距离（局部），部分 head 关注远距离（全局）

训练收敛后：
- ViT-B：attention 距离随层数单调递增（浅层局部、深层全局）
- MViT-B：attention 距离在各层之间非单调，且即使是同一层的不同 head 也有显著差异

**结论：MViT 的多尺度结构在初始化时就提供了更强的归纳偏置——不同 stage 天然关注不同尺度的特征。而 ViT 需要从头学出这种多尺度行为（并且不一定能学得高效）。**

## 十一、官方代码实现阅读

MViT 的代码在 [facebookresearch/SlowFast](https://github.com/facebookresearch/SlowFast) 仓库中。核心实现在 `slowfast/models/` 目录下。

### 11.1 核心模块

**MHPA（Multi Head Pooling Attention）**是 MViT 最核心的层，其前向流程：

1. 输入 X，通过 QKV 线性投影得到 Q̂, K̂, V̂
2. 用 `nn.AvgPool3d` / `nn.MaxPool3d` / 自定义 conv-pool 对 Q̂, K̂, V̂ 分别做 pooling
3. 在缩短后的序列上做标准 attention（Multi-Head Attention）
4. 如果 Q 被池化了，输出序列长度变短

### 11.2 关键实现细节

**Pooling 类型**：论文消融了 max-pool、avg-pool、conv-pool。最终所有主要实验使用 conv-pool（channel-wise 卷积 + LayerNorm），对最大池化来说效果更好（≈+1.2%）。

**Cube embedding 实现**：使用 `nn.Conv3d` 实现时空 cube 投影（kernel=3×7×7, stride=2×4×4），这与 ViT 的 `nn.Conv2d` patch projection 类似但扩展到了时间维度。

**Positional embedding**：空间使用 2D sine-cosine 位置编码，时间使用 1D 可学习位置编码，两者相加而不是拼接。

**Stage transition**：在每个 stage 的最后，MLP 的输出维度被设置为下一 stage 的输入通道数（通常是 2×）。然后在 stage 的第一个 MHPA block 中做 Q pooling（降分辨率）。

### 11.3 论文与实现的差异（少数发现）

- 论文描述中使用 "1 个头" 在 scale₁，实际代码中可以根据配置调整
- conv-pool 使用 channel-wise convolution（分组数 = 通道数），而非标准 2D/3D 卷积
- 分类头使用 class token（与 ViT 一致），而非 Swin 那种 GAP（全局平均池化）

## 十三、范式意义

**MViT 的范式意义：它证明了"视觉 Transformer 不需要全局 attention，也不需要局部窗口——多尺度金字塔本身就是有效的视觉先验。"**

具体来说，MViT 完成了三件事：

### 13.1 建立视频 Transformer 的新基线

在 MViT 之前，视频 Transformer 是"拿现成 ViT + 微调"的思路。MViT 证明了从零训练的、架构适配的 Transformer 可以比依赖大规模外部预训练的 ViT 变体更好。这改变了社区对视频 Transformer 的认知——不需要 ImageNet-21K 预训练也能做好的视频 Transformer。

### 13.2 多尺度 + Transformer 的优雅结合

MViT 证明了多尺度金字塔不仅是 CNN 的特有设计，也是 Transformer 的有效设计。Pooling attention 作为一个灵活的操作，可以同时在时间和空间维度上调整分辨率，同时保持 Transformer 的全局 attention 能力（只是 attention 作用在缩小的序列上而非原序列上）。

### 13.3 提供了强时序偏置的架构解释

帧打乱实验清晰地展示了：**多尺度架构本身提供了时序偏置**。这是因为早期 stage 中的时空 cube 投影强制融合了帧间信息，而池化操作进一步建立了跨帧的对应关系。这不依赖于手工设计的时序模块或额外的 loss——是架构设计带来的自然结果。

### 13.4 后续影响

MViT 启发了大量后续工作：
- **MViTv2**（2022）：改进版，提升了速度和准确性
- **Video MAE**（2022）：使用 MViT 作为视频 masked autoencoding 的 backbone
- **InternVideo**（2022）：视频理解的统一框架，使用 MViT 作为 base encoder

## 十四、局限性与后续方向

### 14.1 Pooling 操作的信息损失

虽然 pool attention 降低了计算量，但对 K, V 做池化不可避免地会丢失信息。特别是当 stride 较大（如 scale₁ 中 K,V pool stride 1×8×8）时，池化窗口覆盖了 8×8=64 个空间位置——一个 token 只能看到聚合后的信息。论文的消融实验显示没有 K,V pooling 的变体准确率最高（77.6%），但 FLOPs 和内存也最高（130.8G, 16.3G）。这是一个 trade-off。

### 14.2 对小数据集可能不够友好

MViT 的架构参数（stage 数、通道数、层数）以 Kinetics 为基准优化。在小数据集上可能需要调整。不过论文在 ImageNet 上的结果（从零训练）表明它至少可以在 1.28M 级别数据上有效工作。

### 14.3 推理时延 vs FLOPs 的关系

论文报告的 FLOPs 优势显著，但在实际推理时延方面数据较少。ViT 可以用高度优化的矩阵乘法库（无分支、无池化），而 MViT 的池化操作增加了控制流和内存访问模式的复杂度。量化这种差异需要更多分析。

### 14.4 与检测/分割的深度集成

与 Swin 被广泛用于 Mask R-CNN 等检测框架不同，论文在检测任务上（AVA）的适配相对简单——从 backbone 特征图做 RoIAlign（一种从特征图提取任意大小区域特征的双线性插值方法，常用于检测任务）。这种适配方式是否最优可以进一步探讨。后续的 Mask2Former 等工作展示了更好的检测/分割集成方案。

### 14.5 训练成本

MViT-B（16×4）使用 128 GPU 训练 200 个 epoch，batch size 512。这在 FAIR 级别的算力上是可行的，但对资源有限的团队来说复制起来成本不低。

## 十五、小结

MViT 的核心贡献不在于提出了一个"更好的注意力变体"，而在于证明了：**视觉 Transformer 应该针对视觉信号的密度和尺度特性做架构调整，而不是照搬 NLP 的 Transformer 设计。**

它通过三个设计解决了视觉 Transformer 的核心问题：
1. **多尺度金字塔**解决了"视觉信息有层级结构"的问题——从密集到稀疏的渐进式处理
2. **Pooling attention** 解决了"视觉信号太密集"的问题——通过对 K,Q,V 的空间/时间池化降低 attention 成本
3. **时空 cube** 解决了"时序信息被忽略"的问题——通过在输入层就融合帧间信息建立时序偏置

在技术树中，MViT 是一个 **bridge node**——它连接了卷积网络的多尺度金字塔传统和 Transformer 架构，同时跨越了图像和视频两个视觉子领域。它不是某个路线的唯一开创者（Swin 同期做了类似的层级化工作，但走的是不同路径），而是在"视频"这个特定领域做出了最彻底的实验验证的工作。
