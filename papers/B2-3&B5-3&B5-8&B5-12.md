## 一、先搞清楚"问题在哪里"

2020 年 ViT 出世之后，整个 CV 社区做了一个思想实验：**如果完全不用卷积，只用 Transformer 能不能做视觉？** ViT 给了个初步答案——在 ImageNet 分类上，用足够大的数据（JFT-300M）训练，ViT 可以追上甚至超过当时的 CNN SOTA。

但社区很快发现 ViT 有一个微妙的"偏科"问题。

ViT 的设计哲学非常简单：把图片切成 16×16 的 patch，展平成 token，扔进标准 Transformer encoder。encoder 内部保持恒定的 channel 维度和分辨率。12 层 Transformer block 完事后，用 cls token 做分类。就这么简单。

这个设计有一个隐含假设：视觉信号的密度和 NLP 序列是一样的，所以同样的架构可以直接搬过来用。

但实际情况是，**视觉信号的密度远高于 NLP**。一张 224×224 图片有 50176 个像素，而一句话平均只有 20 个词。就算切成 16×16 的 patch，也有 196 个 token——比大多数句子长。如果换成视频呢？一个 16 帧、224×224 的视频，用 16×16×2 的时空 patch，token 数轻松上千。

**更关键的问题：视觉信号天然的层级结构，Transformer 默认没有利用。** Hubel 和 Wiesel 在猫的视皮层实验里早就发现：视觉处理是分级的——低级区域（V1）处理边缘和条状特征，高级区域处理更抽象的语义。CNN 通过逐层下采样和 channel 扩展来模仿这个结构。但 ViT 的 Transformer encoder 没有做任何类似的事情——所有层级都一样的分辨率和 channel 数。

MViT 的核心论点就是：**视觉信号这么稠密，你这么"平铺直叙"地处理是浪费的。应该在早期低分辨率高空间细节，在后期高语义低空间细节。** 这不就是多尺度特征金字塔的老思路吗？但关键是——怎么把它引入 Transformer 呢？

## 二、旧路线的真正问题

在 MViT 之前，视觉 Transformer 的发展基本沿着两条路走：

### 2.1 全局 attention 的性价比问题

ViT 的全局 self-attention 在所有 token 之间做 pairwise 计算。复杂度是 O(N²)。196 个 token（224² 输入，16² patch）勉强能跑，但一旦分辨率提高或 patch 变小，N 增长，平方项马上爆炸。

更微妙的是，**全局 attention 对视觉来说是 overkill 的。** 语言序列中，任意两个词之间都可能存在语法或语义关系（比如句子开头的词和结尾的词通过谓语联系），所以全局 attention 在 NLP 中有意义。但视觉中，相距很远的两个像素之间很少有直接关系——猫的眼睛和猫的尾巴不会直接交互，它们通过"身体"这个中间介质间接相关。

也就是说，你用 O(N²) 的代价做了大量实际上不需要的交互计算。

### 2.2 固定分辨率的问题

ViT 在所有 block 保持相同的序列长度和 channel 维度。这在图像分类上没问题——你只需要一个全局表示来分类。但对于检测、分割这类需要多尺度特征的任务，ViT 的单尺度输出就是硬伤。

回头看看 CNN：ResNet 有 4 个 stage，输出 1/4、1/8、1/16、1/32 四种分辨率的特征图。FPN、U-Net、Mask R-CNN 都是基于这个多尺度设计。ViT 只有一个 1/16 的输出，要做 dense prediction 就必须额外设计（比如用 SETR 的上采样解码器），而且效果不如 ResNet 的天然金字塔。

### 2.3 视频的困境：不仅仅是 3D 化

如果图像已经难搞，视频更难。视频增加了时间维度 T，token 数一下子变成 T×H×W。如果用 ViT 的全局 attention，token 数一翻几十倍，计算量翻几百倍。

当时有些工作（VTN、TimeSformer、ViViT）直接用 Video ViT 方案：把视频帧看成更长的 token 序列，或者做分离的时空 attention（先空间再时间）。但这些方案有两个共同问题：

1. **依赖大规模外部预训练。** ViViT-L 需要在 ImageNet-21K（约 1400 万张图，是 Kinetics-400 的 60 倍）上预训练才能达到好效果。
2. **对时间信息的使用不足。** MViT 论文做了一个很厉害的实验：把测试视频的帧顺序打乱（shuffled frames），ViT 模型的表现几乎不掉点。这说明 ViT 实际上在"看单帧图片做投票"而不是真正理解视频中的运动信息。相反，MViT 在这种测试下精度大幅下降——表明它真的在建模时间依赖关系。

### 2.4 小结

所以 MViT 要解决的问题可以总结为三条：
1. **视觉信号的密度**要求一个更高效的信息处理方式，而不是"一刀切"的固定分辨率
2. **视觉任务的多样性**要求 backbone 输出多尺度特征，而不是单分辨率
3. **视频中的时间信息**需要被真正建模，而不是被静态外观淹没

## 三、这篇论文的核心思路

**核心思路极简版：给 Transformer 加上多尺度特征金字塔，每一层同时做 channel 扩张和分辨率下采样，让 Transformer 自己学会从低层细节到高层语义的层级表示。**

具体来说，MViT 做了两件事：

### 3.1 多尺度阶段（Multiscale Stages）

把网络分成若干个"尺度阶段"（scale stages）。每个 stage 有固定的 channel 数和分辨率。从第一个 stage 到最后一个 stage，channel 逐渐扩张，分辨率逐渐降低。最后输出一个多尺度特征金字塔。

这和 CNN 的 stage 设计（ResNet 的 `conv2_x`~`conv5_x`）完全对齐。但区别在于：CNN 是通过 stride 卷积或池化来下采样的，MViT 是在 attention 内部做下采样。

### 3.2 池化注意力（Pooling Attention）

这是整篇论文最核心的改动。标准 MHA 中，Q、K、V 的序列长度和输入一样。MViT 的 Multi Head Pooling Attention（MHPA）在计算 attention 之前，先对 Q、K、V 做池化操作——降低序列长度（分辨率），然后才在缩短后的序列上算 attention。

具体来说：
- Q 的池化决定了**输出分辨率**（因为 attention 结果的序列长度等于 Q 的序列长度）
- K 和 V 的池化决定了**attention 计算的复杂度**（K 和 V 越短，QK^T 矩阵越小）

这个设计的优雅之处在于：**分辨率降低和 attention 计算是一体的**。你不需要单独做池化再跑 attention —— pool 和 attend 在一个操作里完成。

### 3.3 隐式时间偏置

对于视频，MViT 在池化操作中同时池化时间和空间维度。这使得模型天然地聚合了时空信息。论文展示了 MViT 对帧顺序打乱非常敏感（精度大幅下降），而标准 ViT 几乎不受影响——证明 MViT 真的在学习时间维度的模式。

## 四、整体结构与 forward 流程

### 4.1 整体架构

MViT 有若干个"尺度阶段"，每个阶段包含若干 MHPA block。典型的 MViT（以 MViT-B 为例）有 4 个 stage：

```text
输入视频: T×H×W×3 (e.g., 16×224×224×3)
    │
    ▼
[Patch Embedding]         1×16×16 patch + 线性投影
    │                     输出: T×14×14×D  (D=96)
    ▼
┌────────────────────────────────────────────────────────┐
│ Stage 1 (T×14×14, D=96)                               │
│  N₁ 个 MHPA Block                                      │
│  每个 Block: Pooling Q: 无变化                          │
│             Pooling K/V: 2×2×2 → 长度/8               │
│  分辨率: T×14×14 → T×14×14 (Q 决定输出分辨率)           │
└────────────────┬───────────────────────────────────────┘
                 │
                 ▼  ← 下采样：Q pooling 引入 stride
┌────────────────────────────────────────────────────────┐
│ Stage 2 (T×7×7, D=192)  [Channel ↑, 分辨率 ↓]         │
│  N₂ 个 MHPA Block                                      │
│  Q pool: 1×2×2                                         │
│  分辨率: T×14×14 → T×7×7                               │
└────────────────┬───────────────────────────────────────┘
                 │
                 ▼
┌────────────────────────────────────────────────────────┐
│ Stage 3 (T/2×4×4, D=384)  [继续扩张]                   │
│  N₃ 个 MHPA Block                                      │
│  Q pool: 2×2×2                                         │
└────────────────┬───────────────────────────────────────┘
                 │
                 ▼
┌────────────────────────────────────────────────────────┐
│ Stage 4 (T/4×2×2, D=768)  [最高层]                     │
│  N₄ 个 MHPA Block                                      │
└────────────────┬───────────────────────────────────────┘
                 │
                 ▼
[全局平均池化] → [MLP 分类头]
```

### 4.2 前向过程逐步跟踪

**Step 1 — Patch Embedding**

视频输入 16×224×224×3。切成 1×16×16 的时空 patch（时间上不切，只切空间）。每个 patch 展平为 16×16×3=768 维。线性投影到 D=96。

张量形状：`[B, 16, 224, 224, 3]` → `[B, 16, 14, 14, 96]`

如果用图像模型（去掉时间维度），就是 224×224×3 → 14×14×96。

**Step 2 — Stage 1（N₁ 个 MHPA Block）**

第一个 stage 保持输入分辨率 T×14×14。每个 block 内，Q 不做池化，K 和 V 做池化来降低 attention 的复杂度。

假设每个 MHPA Block 内部：Q pool: 1×1×1（无变化），K/V pool: 2×2×2（时间×空间×空间各 2 倍下采样）。那么对于长度为 L = T×14×14 的序列，K 和 V 被池化到 L/8 = (T/2)×7×7。

Attention 计算的 QK^T 矩阵大小：L × (L/8) ——比标准 MHA 的 L × L 小 8 倍。

**Step 3 — Stage 2（下采样）**

从 Stage 1 到 Stage 2 之间，通过 Q pooling 实现分辨率降低（类似 CNN 的 stride-2 卷积）。通常用 Q pool stride 1×2×2，让空间分辨率从 14×14 降到 7×7，channel 从 96 扩展到 192。

Stage 2 block 内部，K/V pool 继续做池化。

**Step 4 — Stage 3/4（继续扩展）**

重复这个过程。Stage 3 可能同时做时间下采样（Q pool stride 2×2×2），把时间维从 T 降到 T/2，空间从 7×7 降到 4×4，channel 扩张到 384。

Stage 4 进一步，时间 T/4，空间 1×1 或 2×2，channel 768。

**Step 5 — 分类**

最后一个 stage 的输出做全局平均池化，再过分类头。

### 4.3 输出特征图

| Stage | 分辨率（时空） | Channel | 说明 |
|-------|-------------|---------|------|
| 1 | T×14×14 | 96 | 高分辨率，低语义 |
| 2 | T×7×7 | 192 | 中等分辨率 |
| 3 | T/2×4×4 | 384 | 低分辨率，高语义 |
| 4 | T/4×2×2 | 768 | 最粗，最抽象 |

和 ResNet 的 conv2~conv5 对齐，可以适配 FPN、UperNet 等检测/分割框架。

### 4.4 架构变体

| 变体 | D | Stage blocks | 参数量 | FLOPs（video） |
|------|---|------------|--------|--------------|
| MViT-T | 64 | [1,1,1,1] | 15M | 36G |
| MViT-S | 64 | [1,2,5,2] | 25M | 64G |
| MViT-B | 96 | [1,2,5,2] | 37M | 102G |
| MViT-B-16 | 96 | [2,3,16,3] | 57M | 170G |
| MViT-L | 96 | [2,6,36,4] | 216M | 636G |

D 是初始 channel 数。Stage block 数指每个 stage 的 MHPA block 数量。FLOPs 是视频模型在 Kinetics 上的计算量。

## 五、核心模块逐个解释

### 5.1 池化操作（Pooling Operator）

池化操作 $\mathcal{P}(\cdot; \mathbf{\Theta})$ 是 MHPA 的基石。参数 $\mathbf{\Theta} = (\mathbf{k}, \mathbf{s}, \mathbf{p})$ 是池化核大小、步长、填充。

输入是一个 D 维 tensor，形状为 $D \times T \times H \times W$。池化操作将序列长度从 $\mathbf{L} = T \times H \times W$ 降到 $\mathbf{\tilde{L}} = \tilde{T} \times \tilde{H} \times \tilde{W}$，其中：

$$\mathbf{\tilde{L}} = \left\lfloor \frac{\mathbf{L} + 2\mathbf{p} - \mathbf{k}}{\mathbf{s}} \right\rfloor + 1$$

注意这个池化**不是在 channel 维度上做的**（不像 CNN 的 pooling 是 spatial-only）。它是沿着时空维度（T, H, W）分别做池化，保持 channel 维不变。

默认使用**重叠核**（overlapping kernels）——核大小大于步长，确保相邻池化窗口之间有重叠，减少信息丢失。

最关键的细节：这个池化是**可学习的**吗？论文中用的是池化（pooling）不是卷积——池化操作没有可学习参数。但它在 Q、K、V 投影之后做池化，所以 $\hat{Q}$ 和 $\hat{K}$ 已经通过 $W_Q$、$W_K$ 做了可学习的线性投影，池化只是压缩序列长度。

### 5.2 Multi Head Pooling Attention（MHPA）

标准 MHA 的流程：

```
输入 X → Q=XW_Q, K=XW_K, V=XW_V → Attn(Q,K,V) → concat → proj
```

MHPA 的流程：

```
输入 X → Q=XW_Q, K=XW_K, V=XW_V
       → Q_pool = 𝒫(Q; 𝚯_Q), K_pool = 𝒫(K; 𝚯_K), V_pool = 𝒫(V; 𝚯_V)
       → Attn(Q_pool, K_pool, V_pool) → concat → proj
```

关键：Q、K、V 的池化 stride 可以不一样。

- **Q pooling**（stride s_Q）：决定输出分辨率。如果 s_Q > 1，输出的序列长度比输入短。
- **K/V pooling**（stride s_K = s_V）：决定 attention 计算的规模。s_K 越大，K 和 V 的序列越短，QK^T 矩阵越小。

最优雅的部分：你可以把**分辨率下采样**和**注意力计算**融合在一起。在 CNN 中，你需要用 stride 卷积或池化来降低分辨率，然后用另一个模块做特征交互。在 MViT 中，这两个事情在 MHPA 一步完成。

**为什么需要 Q pool 和 K/V pool 不同步？**

以 Stage 1 为例：输入分辨率 T×14×14，你想保持这个分辨率（Q 不池化），但你又不想做全局 attention（因为 O(N²) 太贵）。于是你对 K 和 V 做池化，让他们变短——attention 计算的是"高分辨率 query"对"低分辨率 key/value 的投影"。

这相当于：**每个 token 不是和所有其他 token 交互，而是和一组局部聚合后的 token 交互。** 这种不对称的 Q/K/V 分辨率设计，在信息保留度和计算效率之间做了巧妙的平衡。

### 5.3 多尺度 stage 设计

每个 stage 由 N 个 MHPA block 组成。Stage i 到 Stage i+1 的过渡中，Q pooling 的 stride 被设为大于 1（典型值 1×2×2 或 2×2×2），实现分辨率下采样。同时 channel 维度翻倍。

Stage 内部的 MHPA block 中，Q 的 stride 通常为 1（不改变分辨率），只对 K 和 V 做池化。

这种设计对应了 CNN 规律：stage 边界处做 stride-2 下采样 + channel 加倍，stage 内部保持分辨率。

### 5.4 位置编码

MViT 使用了**可学习的绝对位置编码**（learned absolute positional embedding），和 ViT 一致。位置编码加在 patch embedding 的输出上。

不过，由于 MViT 在不同 stage 有不同分辨率，位置编码需要针对每个 stage 独立学习。Stage 1 的 14×14 位置编码和 Stage 2 的 7×7 位置编码不一样。这比 ViT 的单一分辨率位置编码要稍微复杂一些。

### 5.5 分类方案

对于视频分类，MViT 采用全局时空平均池化 + MLP 分类头。没有使用 cls token。

论文的消融实验（Table 4a）比较了不同池化方案：
- 全局平均池化：基准性能
- 单 cls token：效果较差——可能因为 MViT 的多尺度设计使得 cls token 在不同分辨率之间迁移时不太好
- 多 cls token：和平均池化相当

最终选择全局平均池化，简练干净。

## 六、核心公式与数学直觉

### 6.1 池化注意力公式

池化注意力的完整数学表达：

$$\text{PA}(Q, K, V) = \text{Softmax}\left(\frac{\mathcal{P}(Q;\Theta_Q)\mathcal{P}(K;\Theta_K)^T}{\sqrt{d}}\right)\mathcal{P}(V;\Theta_V)$$

其中 $\mathcal{P}$ 是池化操作。

对比标准注意力：

$$\text{Attn}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d}}\right)V$$

区别非常直接：在点积之前，对 Q、K、V 分别做池化。

### 6.2 复杂度分析

输入张量维度：$D \times T \times H \times W$，序列长度 $L = T \cdot H \cdot W$。

**标准 MHA 每 head 复杂度：**

$$O\left(\frac{L D}{h} \left(D + L\right)\right)$$

第一项 $L D^2 / h$ 来自 QKV 投影，第二项 $L^2 D / h$ 来自 attention 矩阵计算。

**MHPA 每 head 复杂度：**

$$O\left(\frac{L D}{h} \left(D + \frac{L}{f_Q f_K}\right)\right)$$

其中 $f_Q = s_Q^T \cdot s_Q^H \cdot s_Q^W$ 是 Q 的序列长度缩减因子，$f_K$ 同理（$f_K = f_V$ 因为 K 和 V stride 必须一致）。

关键差异：第二项从 $L^2$ 降到了 $L^2 / (f_Q f_K)$。如果 f_Q = 1（Q 不池化）且 f_K = 8（K/V 被 2×2×2 池化），第二项变成 $L^2 / 8$——attention 矩阵的 FLOPs 降到 1/8。

**内存复杂度更惊人：**

标准 MHA 内存：$O(L^2 / h)$，attention 矩阵就是 L×L。

MHPA 内存：$O(L^2 / (h f_Q f_K))$，attention 矩阵是 (L/f_Q) × (L/f_K)。

对于视频输入 L = 16×14×14 = 3136，f_Q = 1，f_K = 8：
- 标准 MHA attention 矩阵：3136² ≈ 9.8M 元素
- MHPA attention 矩阵：3136 × 392 ≈ 1.2M 元素
- 减少 8 倍内存

对于更大的输入（比如更高帧率或分辨率），这个差距可以到几百倍。

### 6.3 MHPA Block 完整公式

第 $l$ 个 MHPA Block 的计算：

$$\hat{\mathbf{z}}^l = \text{MHPA}(\text{LN}(\mathbf{z}^{l-1})) + \mathbf{z}^{l-1}$$
$$\mathbf{z}^l = \text{MLP}(\text{LN}(\hat{\mathbf{z}}^l)) + \hat{\mathbf{z}}^l$$

标准的 Pre-LN Transformer Block，只是把 MHA 换成了 MHPA。

### 6.4 Stage 过渡公式

从 Stage i 到 Stage i+1 的下采样，通过在 stage 边界的第一个 MHPA block 设置 Q pool stride > 1 来实现。

Stage i 最后一个 block 输出：序列长度 $L_i$，channel $D_i$

Stage i+1 第一个 block 输入：经过一个额外的池化操作后，序列长度 $L_{i+1} = L_i / (s_Q^T \cdot s_Q^H \cdot s_Q^W)$，channel $D_{i+1} = 2D_i$

这个 channel 扩张是通过调整 MHPA 的投影权重维度自动实现的，不需要额外的线性层。

## 七、重要图表解释与 ASCII 重构

### 7.1 Figure 1 — 多尺度特征层次概念图

原图 Figure 1 展示了一个图像通过 MViT 的多尺度阶段，从"密集且简单"（高分辨率低通道）到"稀疏且复杂"（低分辨率高通道）的特征演化。

```
输入:  密集空间信息, 简单通道 (3 channel RGB, 高密度像素)
         │
         ▼
Stage 1: 高时空分辨率, 低通道 (T×14×14, D=96)
         │
         ▼
Stage 2: 中分辨率, 中通道 (T×7×7, D=192)
         │
         ▼
Stage 3: 低分辨率, 高通道 (T/2×4×4, D=384)
         │
         ▼
Stage 4: 最粗分辨率, 最高通道 (T/4×2×2, D=768)
         │
输出:  稀疏空间信息, 复杂通道 (高级语义特征)
```

这个图在概念层面传达的核心信息是：**MViT 不是扔掉空间信息，而是把空间信息"换"成了通道信息。** 早期分辨率高所以能看到细节，但通道少所以特征简单。后期分辨率低但通道多，能表示更复杂的语义。

### 7.3 Figure 3 — 池化注意力机制

原图 Figure 3 是 MHPA 的图示。这是整篇论文最核心的示意图。

```
输入序列 L=T×H×W
     │
     ├─→ Q_proj (D×D) ─→ 𝒫(Q; 𝚯_Q) ──→ Q (L_Q×D)
     ├─→ K_proj (D×D) ─→ 𝒫(K; 𝚯_K) ──→ K (L_K×D)
     └─→ V_proj (D×D) ─→ 𝒫(V; 𝚯_V) ──→ V (L_V×D)
                                      │
                               ┌──────┴──────┐
                               │  Q × K^T     │
                               │  ↓ softmax   │
                               │  × V         │
                               └──────┬──────┘
                                      │
                                      ▼
                              输出序列 L_Q×D
```

其中 L_Q ≤ L（等于或小于输入长度），L_K = L_V < L（通常小于输入长度）。

三种模式：
1. **Q 池化（L_Q < L）：** 输出分辨率降低，等效于 stride 卷积
2. **K/V 池化（L_K, L_V < L）：** 降低 attention 计算量，但保持输出分辨率
3. **两者都池化：** 同时降低分辨率和计算量

## 八、复杂度、效率与可扩展性分析

### 8.1 注意力复杂度对比

| 方法 | Attention 复杂度 | 对视频 L=3136 |
|------|-----------------|--------------|
| 标准 MHA（ViT） | O(L²D) | O(9.8M × D) |
| 稀疏 attention（Swin） | O(M²LD) | O(49L × D) |
| 池化 attention（MViT） | O(L²D / f) | O(1.2M × D) 当 f=8 |
| 分离 attention（TimeSformer） | O(LD(L_spatial + L_time)) | O(~2.5M × D) |

MViT 的 l 方法不是最简单的也不是最激进的，但它在设计上有一个独特优势：**它不需要修改 attention 的数学形式**，只是通过池化改变了输入长度。这意味着你可以直接复用标准 Transformer 的工程优化（加速库、混合精度训练等）。

### 8.2 参数量和 FLOPs

论文 Table 1 给出了各个变体的详细对比：

| 模型 | 参数量 | 总 FLOPs（Kinetics-400, 单 clip） |
|------|--------|----------------------------------|
| MViT-T | 15M | 36G |
| MViT-S | 25M | 64G |
| MViT-B | 37M | 102G |
| MViT-B-16 | 57M | 170G |
| MViT-L | 216M | 636G |
| ViViT-L | 175M | 1159G（6.8× MViT-B-16 的 FLOPs） |
| TimeSformer-L | 121M | 714G |

ViViT-L 有 175M 参数（比 MViT-B-16 的 57M 多 3 倍），FLOPs 是 MViT-B-16 的 6.8 倍，而且还需要 IN-21K 预训练。

### 8.3 推理时的 temporal clips

论文的一个工程亮点是对 temporal clips 的处理。在推理时，每个视频会采样多个 temporal clips（不同时间段的帧序列），分别通过模型后取平均预测。

MViT 的优势在于：它的多尺度设计让**每个 clip 的推理成本更低**，因此可以用更多的 clips 来投票，获得更高的精度，而总成本仍然低于竞争对手。

在 Figure 2 中可以看到，MViT 曲线随着 clip 数量增加稳步上升，而 ViViT 和 TimeSformer 的曲线已经趋于平缓——可能因为它们单个 clip 的计算成本太高，无法支持足够多的 clips 来充分覆盖整个视频。

### 8.4 瓶颈与限制

1. **池化操作不是免费的。** 对 Q、K、V 做池化需要额外的 reshape 和空间池化计算。虽然比全 attention 便宜得多，但相比一些更高效的方法（如 Swin 的局部窗口）还是有一定开销。

2. **多 stage 设计带来工程复杂度。** 不像 ViT 的"一个堆栈吃到饱"，MViT 需要为每个 stage 设计不同的 resolution 和 channel，增加了调参空间。

3. **K/V 池化 vs 全局信息。** K/V 池化降低了 attention 计算的序列长度，但也损失了信息粒度。如果池化率太大，模型可能丢失重要的细粒度 token-token 关系。

4. **视频下采样的时间维处理。** 如何在时间维上做池化是一个需要仔细设计的选择——时间下采样太快会丢失高频运动信息，太慢则效率提升有限。

---

尽管有这些限制，但实验结果表明池化 attention 在实际场景中的收益远大于损失。

### 10.2 核心类结构

```python
class MViT(nn.Module):
    """
    Multiscale Vision Transformers
    """
    def __init__(self, cfg):
        # cfg 包含所有超参数：patch size, pool strides, 
        # num_heads, embed_dim 等
        
        # Patch Embedding
        self.patch_embed = PatchEmbed(cfg)
        
        # Positional Encoding
        self.pos_embed = nn.Parameter(...)
        
        # 4 个 Scale Stage
        self.stages = nn.ModuleList()
        for stage_cfg in cfg.MVIT.STAGES:
            stage = ScaleStage(stage_cfg)
            self.stages.append(stage)
        
        # 分类头
        self.head = MViTHead(...)
    
    def forward(self, x):
        x = self.patch_embed(x)          # [B,C,T,H,W] → [B,N,D]
        x = x + self.pos_embed
        for stage in self.stages:
            x = stage(x)
        x = self.head(x)
        return x
```

### 10.3 MHPA 实现

```python
class MultiScaleBlock(nn.Module):
    """
    MHPA + MLP 的标准 Pre-LN Block
    """
    def __init__(self, dim, dim_out, 
                 pool_q_kernel, pool_q_stride, pool_q_padding,
                 pool_kv_kernel, pool_kv_stride, pool_kv_padding,
                 num_heads):
        # dim → dim_out （stage 边界处 dim != dim_out）
        self.norm1 = nn.LayerNorm(dim)
        self.norm2 = nn.LayerNorm(dim_out)
        
        # MHPA
        self.attn = MultiHeadPoolingAttention(
            dim, dim_out, num_heads,
            pool_q=(pool_q_kernel, pool_q_stride, pool_q_padding),
            pool_kv=(pool_kv_kernel, pool_kv_stride, pool_kv_padding)
        )
        
        # MLP（4× expansion）
        self.mlp = Mlp(dim_out, int(dim_out * 4))
        
        # 下采样时的 shortcut
        if dim != dim_out:
            self.shortcut = nn.Linear(dim, dim_out)
    
    def forward(self, x):
        # Pre-LN + MHPA + 残差
        shortcut = x
        x = self.norm1(x)
        x = self.attn(x)
        if hasattr(self, 'shortcut'):
            shortcut = self.shortcut(shortcut)
        x = x + shortcut
        
        # Pre-LN + MLP + 残差
        shortcut = x
        x = self.norm2(x)
        x = self.mlp(x)
        x = x + shortcut
        return x
```

### 10.4 MultiHeadPoolingAttention 实现

```python
class MultiHeadPoolingAttention(nn.Module):
    def __init__(self, dim, dim_out, num_heads,
                 pool_q, pool_kv):
        self.num_heads = num_heads
        head_dim = dim_out // num_heads
        self.scale = head_dim ** -0.5
        
        # QKV 投影
        self.qkv = nn.Linear(dim, dim_out * 3, bias=False)
        
        # 池化核参数
        self.pool_q = nn.AvgPool3d(
            pool_q[0], pool_q[1], pool_q[2], count_include_pad=False)
        self.pool_kv = nn.AvgPool3d(
            pool_kv[0], pool_kv[1], pool_kv[2], count_include_pad=False)
        
        # 输出投影
        self.proj = nn.Linear(dim_out, dim_out)
    
    def forward(self, x, B, T, H, W):
        """
        x: [B*N, D] where N=T*H*W
        但需要 reshape 成 5D tensor 来做池化
        """
        # Reshape: [B*N, D] → [B, D, T, H, W]
        x = x.reshape(B, T, H, W, -1).permute(0, 4, 1, 2, 3)
        
        # QKV 投影
        qkv = self.qkv(x.permute(0, 2, 3, 4, 1).reshape(-1, x.shape[1]))
        qkv = qkv.reshape(B, T, H, W, 3, self.num_heads, -1)
        qkv = qkv.permute(4, 0, 5, 1, 2, 3, 6)  # [3, B, h, T, H, W, head_dim]
        q, k, v = qkv[0], qkv[1], qkv[2]
        
        # Q 池化: [B, h, T, H, W, head_dim] → 池化
        q = self.pool_q(q)
        k = self.pool_kv(k)
        v = self.pool_kv(v)
        
        # Flatten 时空维
        _, _, Tq, Hq, Wq, _ = q.shape
        _, _, Tkv, Hkv, Wkv, _ = k.shape
        q = q.reshape(B, self.num_heads, -1, q.shape[-1])  # [B, h, Nq, d]
        k = k.reshape(B, self.num_heads, -1, k.shape[-1])  # [B, h, Nkv, d]
        v = v.reshape(B, self.num_heads, -1, v.shape[-1])  # [B, h, Nkv, d]
        
        # Attention: Q × K^T
        attn = (q @ k.transpose(-2, -1)) * self.scale
        attn = attn.softmax(dim=-1)
        x = (attn @ v)  # [B, h, Nq, d]
        
        # 合并 head, 输出投影
        x = x.transpose(1, 2).reshape(B, Nq, -1)
        x = self.proj(x)
        
        return x
```

### 10.5 论文 vs 代码的注意点

1. **池化用的是 AvgPool3d，不是 MaxPool。** 平均池化保持信息的平滑表示，更适合 attention 中的 softmax 分布——如果做了 max pooling 再 softmax，会引入偏差。

2. **Q/K/V 池化核和 stride 是超参数。** 论文在不同 stage 使用不同的池化配置。这些配置在 MViT 的 4 个 stage 中不同，由实验调节得到。

3. **PyTorch 的 conv3d 没有被用到。** MViT 明确使用的是 `nn.AvgPool3d`，这是关键设计区别——它不是在做"用可学习 conv 降维"而是在做"用固定 pooling 降维"。

4. **dim_in ≠ dim_out 时做 shortcut。** 在 stage 边界，channel 维翻倍，使用额外的 Linear 层做 shortcut，和标准残差网络的降采样 shortcut 类似。

## 十一、Related Work 与技术谱系

### 11.1 技术树中的位置

MViT 同时覆盖了四条技术分支：

```
B2: 视觉 backbone 层级化
  └── B2-3: hierarchical backbone 范式
       ├── PVT (2021, 金字塔 Transformer)
       ├── Swin Transformer (2021, 移位窗口)
       ├── MViT (2021, 本篇)
       └── CSWin / Focal Transformer (2021-2022)

B5: Transformer 处理时空视觉任务
  ├── B5-1: 视频 tokenization (TimeSformer, ViViT)
  ├── B5-2: 时空 attention 降复杂度
  ├── B5-3: 视频 backbone 的时间归纳偏置
  └── B5-4: 统一时空 encoder 基座模型
       ├── TimeSformer (2021)
       ├── ViViT (2021)
       ├── Video Swin (2022)
       └── MViT (2021, 本篇)
```

MViT 的独特之处：它是**唯一一个同时在"层级 backbone"和"视频建模"两条路线上都做了核心贡献的工作**。Swin 主要在 2D 图像上做层级 backbone，TimeSformer 主要在视频上做 attention 设计——MViT 同时覆盖了两边。

### 11.2 前驱工作

**ViT (Dosovitskiy et al., 2020)：** MViT 的起点。ViT 证明了纯 Transformer 可以做视觉，但它的单分辨率设计不适合多尺度建模。MViT 保留了 ViT 的 patch embedding 和 transformer block 基本结构，但增加了多尺度 stage 设计。

**多尺度特征金字塔 (Burt & Adelson, 1983; Lin et al., 2017 FPN)：** MViT 在概念层面上延续了这个经典视觉思想——多尺度处理是视觉理解的"已知最优解"。MViT 的创新在于把这个思想用 Transformer 的方式重新实现。

**ResNet (He et al., 2016)：** ResNet 的 stage 设计（逐步下采样 + channel 扩张）是 MViT 的多尺度 stage 的直接灵感。MViT 实际上是在回答：如果把 ResNet 的 ResBlock 换成 Transformer Block，同时保持 stage 结构，能行吗？

**SlowFast (Feichtenhofer et al., 2019)：** 有趣的是，MViT 的几位作者也是 SlowFast 的作者。SlowFast 用了两个不同帧率的 path（慢速高精度 + 快速低精度）来处理视频。MViT 可以视为 SlowFast 思想的 Transformer 版本——多尺度就是不同的 "path"，只不过在同一个网络内分层实现。

### 11.3 同期竞品

**Swin Transformer (Liu et al., 2021, CVPR)：** Swin 和 MViT 都是 2021 年的层级 Transformer，但走了完全不同的技术路线。Swin 用局部窗口 + 移位来降低复杂度；MViT 用池化 attention 来降低复杂度。Swin 偏好在图像 backbone（ImageNet + COCO + ADE20K），MViT 偏好在视频（Kinetics + SSv2 + AVA + Charades）。两者不是竞争关系而是互补——Swin 在 2D 检测/分割上更强，MViT 在视频上更强。

**PVT (Wang et al., 2021)：** 同期金字塔 Transformer，用 Spatial Reduction Attention 在低分辨率上做稀疏全局 attention。复杂度比标准 MHA 低但不如 MViT 的池化 attention 高效。PVT 在检测任务上有不错表现但没有覆盖视频。

**TimeSformer (Bertasius et al., 2021, ICML)：** 分离的时空 attention——先做空间 attention，再做时间 attention。是第一个纯 Transformer 的视频架构。但 TimeSformer 的单尺度设计和对 IN-21K 预训练的依赖，在 MViT 面前显得不够经济。

**ViViT (Arnab et al., 2021, NeurIPS)：** 全时空注意力的视频 ViT。ViViT-L 有 175M 参数 + IN-21K 预训练 + 1159G FLOPs，才到 81.3% Kinetics-400，而 MViT-B 用 37M 参数 + 零外部数据 + 102G FLOPs 就到了 81.2%。差距主要来自多尺度设计和更好的效率。

### 11.4 后续影响

**MViTv2 (Li et al., 2022)：** 直接的后继工作。在 MViT 基础上加入了分解的相对位置编码、残差池化连接、Swish-GLU MLP 等改进。Kinetics-400 达到 86.1%，SSv2 达到 73.3%。

**VideoMAE (Tong et al., 2022)：** 把 MAE 的自监督范式推广到视频，使用 MViT 作为 backbone。Tube masking 的时空掩码策略建立在 MViT 的多尺度设计之上。

**MViT 作为自监督学习的 backbone：** DINO、MAE、BEiT 等一系列自监督方法都实验过基于 MViT backbone 的版本，验证了 MViT 作为通用 backbone 的适配性。

**OmniMAE (Goyal et al., 2022)：** 单 encoder 同时处理图像和视频，一个 MViT 架构统一了图片和视频的预训练。这直接验证了 MViT 的设计——图像是特化的视频（T=1），可以用同一套架构。

## 十三、范式意义

### 13.1 证明了"好的视觉 backbone 需要多尺度"这条规则在 Transformer 时代仍然成立

ViT 的设计哲学有点"唯 attention 论"：只要 attention 够强，它就能从像素学到一切，不需要任何架构上的归纳偏置。MViT 用实验证明了这个想法过于乐观——引入多尺度结构这个"视觉先验"不仅没有损害性能，反而带来了显著的提升。

**实际意义：** 不要迷信 attention 的"万能性"。视觉信号是结构化的，好的架构应该编码这种结构。

### 13.2 池化 attention：一个优雅的效率策略

MViT 的池化 attention 在概念上非常简洁：你在 attention 之前做一次 pooling，同时完成了分辨率降低和信息聚合。不需要专门的 downsampling layer（如 CNN 的 stride conv），不需要特殊的 attention 变体（如 Swin 的窗口）。就是 standard attention + pooling。

这有一个连锁好处：**你可以用同一个 attention 实现处理不同分辨率的输入**。只需要调整 Q/K/V 的 pool stride 即可。

### 13.3 视频上的"时间归纳偏置"是自然的副产品

MViT 最初的设计动机是效率（降低 attention 复杂度），但它在视频上意外获得了一个重要的副产品：**隐式的时间归纳偏置**。

当你在时空域上做 pooling 时，时间和空间被同样地处理——没有谁优先。这迫使模型在每一层都同时考虑时间和空间信息。对比之下，TimeSformer 的"先空间后时间"设计，让时间信息在每一层都滞后于空间信息；而 Swin 的 2D 窗口+3D 扩展版本，时间维度处理是附加的。

**实际意义：** 好的架构应该把时间视为视觉的基本维度，而不是一个附加维度。

### 13.4 分类：foundation node（基础节点）

在技术树中，MViT 是一个 foundation node。它的核心贡献不在某个具体的 state-of-the-art 刷新（虽然也刷新了），而在于建立了一个**新的架构模板**——池化 attention + 多尺度 stage。

后续的 MViTv2、OmniMAE、VideoMAE 等工作都是在这个模板上继续迭代。它也在更广的维度上推动了"视频 Transformer backbone"这个子领域的成熟。

## 十四、局限性与后续方向

### 14.1 池化操作损失信息粒度的权衡

K/V pool 虽然大幅降低了 attention 的计算复杂度，但也损失了细粒度的 token 级交互信息。高池化率意味着每个被池化后的 token 代表了原来多个 token 的"平均"——精细的局部模式可能被平均掉。

这个 trade-off 在 MViT 中通过调节池化 stride 来控制，但没有系统性地研究最优池化率。

### 14.2 Q pool 和 K/V pool 的不对称性

在 Stage 1，MViT 对 Q 不做池化（保持高分辨率输出），但对 K/V 做池化（降低 attention 复杂度）。这个设计隐含的假设是"query 需要高分辨率，key/value 可以容忍低分辨率"。但这个假设的合理性论文没有深入验证。

在某些场景下（比如需要高精度的语义分割），可能 K/V 的细粒度信息也很重要。MViTv2 的残差池化连接部分地解决了这个问题。

### 14.3 视频 backbone 的定位限制

虽然 MViT 在视频分类上表现出色，但它的多尺度设计在更复杂的视频 dense prediction 任务（如视频实例分割、视频全景分割）上的验证不够充分。论文只做了 AVA（时空动作检测），没有深入探索视频分割。

### 14.4 与 Swin 的对比不够直接

论文在视频上没有和 Swin 做直接对比（Swin 的视频扩展是 2022 年的 Video Swin Transformer），在图像上也没有直接对比（Swin 在 ImageNet 上更优）。这使得"视频上 MViT 更好还是 Swin 更好"这个问题的答案在发表时是不确定的。

### 14.5 训练成本仍然不低

虽然 MViT 比 ViViT 和 TimeSformer 高效很多，但 MViT-L 仍然需要 636G FLOPs 和大量 GPU 时间。对于一个 216M 参数的模型，训练在 2021 年仍然是大规模工程。对于零外部预训练，这个成本还行，但对小型实验室来说还是有门槛。

## 十五、小结

MViT 这篇论文讲的道理其实很简单，但讲得很到位。

**简单来说：视觉处理需要多尺度，CNN 有，Transformer 也应该有。** 不是说 Transformer 的 attention 不好，而是说单尺度的 attention 处理视觉信号像用渔网捞珍珠——大网格是快了但小珍珠全漏了，小网格保住了细节但效率太低。多尺度就是不同尺寸的网眼组合。

MViT 解决这个问题的办法也很直接：在 attention 里面加 pooling。让 query 的高分辨率得以保持，让 key/value 的序列得以压缩——attention 矩阵的大小降到 1/8 甚至更低，而模型效果反而提高。

这个"反而提高"的发现是最有趣的部分。池化不只是省计算，它还引入了有益的正则化——迫使模型在更紧凑的表示上做推理。这和 DenseNet 的"特征复用"、EfficientNet 的"复合缩放"背后的直觉相似：**有时候限制信息通路反而让网络学得更好**。

在技术树中，MViT 的定位非常清晰：它是 B2（层级 backbone）和 B5（时空 Transformer）的交汇点。在层级 backbone 这边，它和 Swin、PVT 一起完成了"从单尺度到多尺度"的范式转换。在时空 Transformer 这边，它用池化 attention 证明了视频不需要分离的时空 attention——只要池化操作同时覆盖时空维度就好。

最后，如果你问我 MViT 和 Swin 最大的区别是什么，我会说：**Swin 关注的是"如何让 attention 更高效"，而 MViT 关注的是"如何让 attention 更适合视觉"。** Swin 的窗口移位是工程层面的创新，MViT 的池化 attention 是知觉层面的选择——它回归到了视觉科学对"多尺度处理"的基本理解。两种思路都在推动 Transformer 视觉 backbone 的发展，而 MViT 的独特贡献在于把多尺度这个"CV 的老智慧"用最自然的方式引入了 Transformer 架构。
