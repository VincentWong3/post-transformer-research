# Multiscale Vision Transformers（2021）

**论文：** [arXiv](https://arxiv.org/abs/2104.11227) · Facebook AI Research (FAIR) · 2021

---

## 一、先搞清楚坑在哪

视频理解这个领域，长期以来被两个范式统治着：

1. **3D CNN**（如 I3D、SlowFast、C3D）：用 3D 卷积核同时处理空间和时间维度。效果好，但计算量大得离谱——视频比图像多了一个时间轴，参数量和 FLOPs 直接跟着立方增长。

2. **2D CNN + 光流**（如 Two-Stream）：用 2D CNN 处理单帧图像，再用光流网络处理运动信息。问题在于光流计算本身就很慢，而且两个流各自为政，特征融合不够充分。

2020 年 Vision Transformer (ViT) 在图像分类上横空出世，证明了纯 Transformer 也能搞定视觉任务。这给视频理解带来了新思路：能不能把 ViT 扩展到视频？

但这里有个坑：**ViT 的 patch embedding 是对单张图像做的**。视频是 T 帧图像序列，如果简单地把每帧独立处理，时间维度就丢了；如果直接把所有帧的 patches 拼起来，序列长度暴涨 N 倍（N = 帧数），自注意力的 $O(N^2)$ 复杂度会直接炸掉。

**这就是 MViT 要解决的核心问题：如何在保持 Transformer 架构优势的同时，高效地建模视频的时空信息？**

## 二、ViT 在视频上的真正问题

先看 ViT 在图像上的工作方式：

1. 把 $H \times W$ 的图像切成 $P \times P$ 的 patches，得到 $N = HW/P^2$ 个 patches
2. 每个 patch 线性投影到 D 维，加上位置编码
3. 输入 Transformer encoder，每个 block 包含多头自注意力（MSA）和 MLP

扩展到视频时，最简单的做法是：把 T 帧视频的每一帧都切成 patches，得到 $N = T \times H \times W / P^2$ 个 patches。

**问题 1：序列长度爆炸**
- 图像：$224\times224$ 图像，$16\times16$ patches → 196 个 tokens
- 视频：16 帧 $224\times224$ → 3136 个 tokens
- 自注意力复杂度 $O(N^2)$ → 计算量增加 $(3136/196)^2 \approx 256$ 倍

**问题 2：时间维度的建模方式**
- 直接拼 patches 会丢失帧之间的对应关系
- 时间位置编码怎么设计？每帧独立编码还是全局编码？

**问题 3：多尺度特征缺失**
- ViT 只在单一分辨率下操作（所有 block 的 token 数相同）
- CNN 天然具有多尺度特征金字塔（浅层高分辨率捕捉细节，深层低分辨率捕捉语义）
- 视频理解尤其需要多尺度：小物体需要高分辨率特征，大运动需要大感受野

**MViT 的核心思路：把多尺度特征金字塔的思想引入 Transformer，同时在空间和时间维度上逐步降采样，让模型从局部细节逐步过渡到全局语义。**

## 三、MViT 的核心思路

一句话概括：**在 Transformer 的每个 block 中，对空间-时间维度逐步降采样（pooling），同时增加通道维度，形成一个从细到粗的特征金字塔。**

这和 CNN 的特征金字塔异曲同工——浅层 block 处理高分辨率特征（小感受野、细节信息），深层 block 处理低分辨率特征（大感受野、语义信息）。

但 MViT 的巧妙之处在于：**它不是简单地在输入层做 pooling，而是在每个 Transformer block 内部，对 query、key、value 分别做不同粒度的 pooling，让自注意力机制本身就能适应多尺度。**

具体来说：
- 每个 MViT block 包含一个 **Pooling Attention** 模块
- 对 query 做空间-时间 pooling（降低分辨率）
- 对 key 和 value 做空间-时间 pooling（进一步降低分辨率）
- 通道维度随分辨率降低而增加（保持信息容量）

## 四、网络架构详解

### 整体架构

```
输入视频: T x H x W x 3
    │
    ▼
[Patch Embedding] → 将视频切成 3D patches (t x h x w)
    │ 输出: T/t x H/h x W/w x D
    │
    ▼
┌─────────────────────────────────────────────────┐
│  Stage 1: N1 个 MViT Block                      │
│  输入分辨率: T/t x H/h x W/w, 通道 D            │
│  输出分辨率: T/t x H/h x W/w, 通道 D            │
└─────────────────────────────────────────────────┘
    │
    ▼ (空间-时间 pooling + 通道扩展)
┌─────────────────────────────────────────────────┐
│  Stage 2: N2 个 MViT Block                      │
│  输入分辨率: T/t x H/h x W/w, 通道 2D           │
│  输出分辨率: T/(2t) x H/(2h) x W/(2w), 通道 2D  │
└─────────────────────────────────────────────────┘
    │
    ▼ (空间-时间 pooling + 通道扩展)
┌─────────────────────────────────────────────────┐
│  Stage 3: N3 个 MViT Block                      │
│  输入分辨率: T/(2t) x H/(2h) x W/(2w), 通道 4D  │
│  输出分辨率: T/(4t) x H/(4h) x W/(4w), 通道 4D  │
└─────────────────────────────────────────────────┘
    │
    ▼ (空间-时间 pooling + 通道扩展)
┌─────────────────────────────────────────────────┐
│  Stage 4: N4 个 MViT Block                      │
│  输入分辨率: T/(4t) x H/(4h) x W/(4w), 通道 8D  │
│  输出分辨率: T/(8t) x H/(8h) x W/(8w), 通道 8D  │
└─────────────────────────────────────────────────┘
    │
    ▼ (全局平均池化 + 分类头)
    分类结果
```

### 前向传播详解

以输入视频 $T \times H \times W \times 3 = 16 \times 224 \times 224 \times 3$ 为例：

**第 0 步：Patch Embedding**

论文使用 3D 卷积实现 patch embedding，卷积核大小为 $3 \times 7 \times 7$（时间 x 空间 x 空间），步长为 $2 \times 4 \times 4$。

```
输入: 16 x 224 x 224 x 3
3D Conv (3x7x7, stride 2x4x4, 96 channels)
输出: 8 x 56 x 56 x 96
```

这里的时间步长 2 意味着把相邻两帧合并成一个时间 patch，空间步长 4 意味着空间分辨率降低 4 倍。

**第 1 阶段：N1 个 MViT Block**

每个 MViT Block 包含：
1. **Pooling Attention**（核心创新）
2. **Layer Normalization**
3. **MLP**（2 层全连接，中间有 GELU 激活）
4. **残差连接**

第一个 block 输入形状：$8 \times 56 \times 56 \times 96$
- 空间维度：$56 \times 56 = 3136$ 个空间位置
- 时间维度：8 个时间步
- 总 token 数：$8 \times 3136 = 25088$

**Pooling Attention 详解：**

对于输入 $X \in \mathbb{R}^{T \times H \times W \times C}$：

1. 线性投影得到 Q、K、V：
   - $Q = XW_Q$，形状不变
   - $K = XW_K$，形状不变  
   - $V = XW_V$，形状不变

2. **对 Q 做 pooling**：$Q_p = \text{Pool}_Q(Q)$
   - Pooling 核大小：$k_Q \times k_Q$（空间），池化步长 $s_Q \times s_Q$
   - 输出形状：$T \times (H/s_Q) \times (W/s_Q) \times C$
   - 在 MViT 中，$k_Q = s_Q = 1$（不对 Q 做降采样），或者 $k_Q = s_Q = 2$（降采样 2 倍）

3. **对 K 做 pooling**：$K_p = \text{Pool}_K(K)$
   - Pooling 核大小：$k_K \times k_K$，池化步长 $s_K \times s_K$
   - 输出形状：$T \times (H/s_K) \times (W/s_K) \times C$
   - 在第一个 block 中，$k_K = s_K = 1$；后续 block 中 $k_K = s_K = 2$

4. **对 V 做 pooling**：$V_p = \text{Pool}_V(V)$
   - 和 K 使用相同的 pooling 参数
   - 输出形状：$T \times (H/s_K) \times (W/s_K) \times C$

5. **计算注意力**：
   $$\text{Attention}(Q_p, K_p, V_p) = \text{Softmax}\left(\frac{Q_p K_p^T}{\sqrt{C/h}}\right) V_p$$

其中 $h$ 是注意力头的数量。

**为什么对 Q、K、V 做不同粒度的 pooling？**

这里有个关键洞察：query 的粒度决定了输出特征的分辨率，key/value 的粒度决定了模型能够感知的上下文范围。

- **对 Q 不做或少量 pooling**：输出特征保持较高分辨率，保留细节
- **对 K、V 做 pooling**：key/value 数量减少，降低了注意力计算的复杂度；同时 pooling 增大了感受野，让每个 query 能看到更大范围的上下文

这就像 CNN 中的 dilated convolution——卷积核不变，但感受野变大了。

**第 1 阶段输出**：$8 \times 56 \times 56 \times 96$

**第 2 阶段：分辨率降低，通道增加**

在进入第 2 阶段前，对特征做空间-时间 pooling：
- Pooling 核：$2 \times 2 \times 2$（时间 x 空间 x 空间）
- 步长：$2 \times 2 \times 2$
- 通道数从 96 翻倍到 192

```
输入: 8 x 56 x 56 x 96
↓ Pooling (2x2x2, stride 2x2x2)
中间: 4 x 28 x 28 x 96
↓ 1x1 Conv (通道扩展)
输出: 4 x 28 x 28 x 192
```

第 2 阶段的 MViT Block 中：
- Q pooling：$k_Q = s_Q = 1$（保持分辨率）
- K、V pooling：$k_K = s_K = 2$（降低 key/value 分辨率）

**第 2 阶段输出**：$4 \times 28 \times 28 \times 192$

**第 3 阶段：进一步降采样**

```
输入: 4 x 28 x 28 x 192
↓ Pooling (2x2x2, stride 2x2x2) + 1x1 Conv
输出: 2 x 14 x 14 x 384
```

第 3 阶段的 MViT Block 中：
- Q pooling：$k_Q = s_Q = 2$（对 query 也做降采样）
- K、V pooling：$k_K = s_K = 2$

**第 3 阶段输出**：$2 \times 14 \times 14 \times 384$

**第 4 阶段：最终特征**

```
输入: 2 x 14 x 14 x 384
↓ Pooling (2x2x2, stride 2x2x2) + 1x1 Conv
输出: 1 x 7 x 7 x 768
```

第 4 阶段的 MViT Block 中：
- Q pooling：$k_Q = s_Q = 2$
- K、V pooling：$k_K = s_K = 2$

**第 4 阶段输出**：$1 \times 7 \times 7 \times 768$

**分类头**：
- 全局平均池化：$1 \times 1 \times 1 \times 768$
- 线性层：768 → 类别数

## 五、核心创新点

### 创新点 1：Pooling Attention（多尺度自注意力）

#### (a) 痛点与动机

标准 Transformer 的自注意力对所有 token 一视同仁，计算复杂度 $O(N^2)$ 随 token 数平方增长。在视频中，$N = T \times H \times W / P^2$，即使是中等分辨率视频也有数千个 token。

更根本的问题是：**标准自注意力缺乏多尺度感知能力**。每个 token 只能关注到其他所有 token，但没有机制让模型在不同尺度上提取特征。这就像用固定大小的卷积核去处理图像——要么感受野太小看不到整体，要么感受野太大丢失细节。

#### (b) 方案细节

Pooling Attention 在计算注意力之前，对 Q、K、V 分别做池化：

$$Q_p = \text{Pool}(Q, k_Q, s_Q)$$
$$K_p = \text{Pool}(K, k_K, s_K)$$
$$V_p = \text{Pool}(V, k_V, s_V)$$

其中 Pool 是 3D 池化（时间 + 空间），$k$ 是池化核大小，$s$ 是步长。

然后计算：
$$\text{Attn}(Q_p, K_p, V_p) = \text{Softmax}\left(\frac{Q_p K_p^T}{\sqrt{d}}\right) V_p$$

#### (c) 为什么有效

1. **计算效率**：对 K、V 做 pooling 后，key/value 数量减少，注意力矩阵 $Q_p K_p^T$ 的尺寸从 $N \times N$ 变为 $(N/s) \times (N/s)$，复杂度降低 $s^2$ 倍。

2. **多尺度特征**：不同 block 使用不同的 pooling 参数，形成从细到粗的特征金字塔。浅层 block（高分辨率）关注局部细节，深层 block（低分辨率）关注全局语义。

3. **增大感受野**：对 K、V 做 pooling 等效于增大了每个 query 的感受野——每个 query 能看到的 key 覆盖了更大的空间区域。

4. **平滑降采样**：相比在 block 之间突然做 pooling，Pooling Attention 在注意力计算内部逐步降低分辨率，信息损失更小。

#### (d) 与 Related Work 的关系

- **ViT [Dosovitskiy et al., 2021]**：ViT 在所有 block 中使用相同的 token 数，没有多尺度。MViT 引入了分辨率变化。
- **CNN 特征金字塔 [Lin et al., 2017]**：FPN 在 CNN 特征图上构建金字塔，但需要额外的侧向连接。MViT 把多尺度直接集成到注意力机制中。
- **Swin Transformer [Liu et al., 2021]**：Swin 通过窗口移位实现局部注意力，并在不同 stage 降低分辨率。MViT 的 pooling 机制更灵活，可以独立控制 Q、K、V 的分辨率。

#### (e) 如果去掉会怎样

论文 Table 5 的消融实验显示：
- 去掉 Pooling Attention（使用标准注意力）：在 Kinetics-400 上 top-1 准确率下降 1.5%
- 使用固定 pooling（所有 block 相同）：准确率下降 0.8%
- 使用不同 pooling 策略（论文方案）：最佳

### 创新点 2：多尺度特征金字塔（Resolution Reduction）

#### (a) 痛点与动机

视频理解需要同时处理不同尺度的信息：
- 小物体（如远处的行人）需要高分辨率特征
- 大运动（如摄像机移动）需要大感受野
- 动作识别需要全局语义（如"跑步"和"走路"的区别在于整体运动模式）

CNN 通过池化层自然形成特征金字塔，但 Transformer 缺乏这种机制。ViT 在所有 block 中使用相同的 token 数，导致：
- 浅层 token 数太多，计算量大
- 深层 token 数不变，无法聚焦于语义信息

#### (b) 方案细节

MViT 设计了 4 个 stage，每个 stage 内部保持分辨率不变，stage 之间进行降采样：

```
Stage 1: 8 x 56 x 56 x 96    (分辨率最高)
Stage 2: 4 x 28 x 28 x 192   (降采样 2x，通道翻倍)
Stage 3: 2 x 14 x 14 x 384   (降采样 2x，通道翻倍)
Stage 4: 1 x 7 x 7 x 768     (降采样 2x，通道翻倍)
```

降采样通过 3D 平均池化实现，池化核大小等于步长（避免重叠）。

#### (c) 为什么有效

1. **计算-精度平衡**：高分辨率阶段 token 多但计算量可控（因为 key/value 被 pooling 了），低分辨率阶段 token 少但通道多（信息容量大）。

2. **逐步抽象**：模型从细节逐步过渡到语义，和人类的视觉感知过程一致。

3. **参数效率**：通道数随分辨率降低而增加，总参数量增长可控。相比 ViT 在固定分辨率下增加通道数，MViT 的参数量更少。

#### (d) 与 Related Work 的关系

- **SlowFast [Feichtenhofer et al., 2019]**：Slow 路径处理低帧率高分辨率，Fast 路径处理高帧率低分辨率。MViT 的多尺度更细粒度，每个 stage 都在变化。
- **ResNet [He et al., 2016]**：ResNet 的 stage 设计（conv2_x → conv3_x → conv4_x → conv5_x）是 MViT 的灵感来源。

#### (e) 如果去掉会怎样

论文 Table 5 显示：
- 固定分辨率（所有 stage 保持 8 x 56 x 56）：准确率下降 2.3%，FLOPs 增加 40%
- 去掉通道翻倍：准确率下降 1.1%
- 论文方案：最佳平衡

### 创新点 3：时间维度的多尺度处理

#### (a) 痛点与动机

视频中的时间信息是多尺度的：
- 快速动作（如挥手）需要高时间分辨率
- 慢速动作（如走路）需要长时间跨度

标准做法（如 TimeSformer）把时间注意力单独处理，但计算量仍然很大。

#### (b) 方案细节

MViT 在时间维度上也进行多尺度处理：
- Patch embedding 时用时间步长 2，把相邻帧合并
- 每个 stage 对时间维度做 2x 降采样
- Pooling Attention 中，pooling 核同时覆盖时间和空间

#### (c) 为什么有效

1. **时间感受野递增**：浅层 stage 时间分辨率高（8 帧），能捕捉快速动作；深层 stage 时间分辨率低（1 帧），但通道数多，能建模长程依赖。

2. **时空协同**：pooling 同时覆盖时间和空间，让模型学习时空联合特征，而不是分开处理。

#### (d) 与 Related Work 的关系

- **TimeSformer [Bertasius et al., 2021]**：分开的空间注意力和时间注意力。MViT 的联合时空注意力更紧凑。
- **ViViT [Arnab et al., 2021]**：多种时空分解方式。MViT 的多尺度是另一种思路。

#### (e) 如果去掉会怎样

论文 Table 6 显示：
- 去掉时间 pooling（只在空间上 pooling）：准确率下降 0.9%
- 使用固定时间分辨率：准确率下降 0.5%

## 六、公式详解

### 公式 1：Pooling Attention 复杂度

标准自注意力的复杂度：
$$\Omega(\text{MSA}) = 4hwC^2 + 2(hw)^2C$$

其中 $h, w$ 是空间分辨率，$C$ 是通道数。

**符号定义**：
- $h, w$：特征图的高度和宽度
- $C$：通道数
- 第一项 $4hwC^2$：QKV 投影和输出投影的矩阵乘法
- 第二项 $2(hw)^2C$：注意力矩阵计算和加权求和

MViT Pooling Attention 的复杂度：
$$\Omega(\text{PoolAttn}) = 4hwC^2 + \frac{2(hw)^2C}{s_K s_Q}$$

**推导过程**：
1. QKV 投影不变：$3hwC^2 + hwC^2 = 4hwC^2$（三个投影 + 输出投影）
2. Q、K、V 的 pooling 操作：$\mathcal{O}(hwC)$，可以忽略
3. 注意力矩阵：$Q_p$ 有 $hw/s_Q$ 个 query，$K_p$ 有 $hw/s_K$ 个 key
   - 注意力矩阵大小：$(hw/s_Q) \times (hw/s_K)$
   - 计算复杂度：$(hw/s_Q)(hw/s_K)C = (hw)^2C / (s_Q s_K)$
4. 加权求和：同样 $(hw)^2C / (s_Q s_K)$

**直觉理解**：
- 第一项 $4hwC^2$ 是投影的代价，和标准注意力相同
- 第二项是注意力计算的代价，通过 pooling 降低了 $s_Q s_K$ 倍

**具体数值例子**：
以第 1 阶段为例：$h=56, w=56, C=96, s_Q=1, s_K=1$
- 标准注意力：$4 \times 56 \times 56 \times 96^2 + 2 \times (56 \times 56)^2 \times 96$
- 第一项：$4 \times 3136 \times 9216 \approx 115.6M$
- 第二项：$2 \times 3136^2 \times 96 \approx 1.89B$
- 总计：约 2.0B

以第 2 阶段为例：$h=28, w=28, C=192, s_Q=1, s_K=2$
- MViT 注意力：$4 \times 28 \times 28 \times 192^2 + 2 \times (28 \times 28)^2 \times 192 / (1 \times 2)$
- 第一项：$4 \times 784 \times 36864 \approx 115.6M$
- 第二项：$2 \times 784^2 \times 96 \approx 118.0M$
- 总计：约 233.6M

相比标准注意力（$s_Q=s_K=1$）的 2.0B，MViT 降低了约 8.5 倍。

**边界情况**：
- 当 $s_Q = s_K = 1$ 时，退化为标准注意力
- 当 $s_Q$ 或 $s_K$ 为 0 时无意义（pooling 核大小至少为 1）
- 当 $s_Q$ 很大时，query 数量很少，注意力矩阵很小，但可能丢失空间信息

### 公式 2：通道扩展策略

$$C_{l+1} = 2 \times C_l$$

**符号定义**：
- $C_l$：第 $l$ 个 stage 的通道数
- $C_{l+1}$：第 $l+1$ 个 stage 的通道数

**推导过程**：
每次空间-时间分辨率降低 2 倍时，通道数翻倍。这是 CNN 的经典设计（如 ResNet），目的是保持信息容量大致不变。

**直觉理解**：
- 空间分辨率降低 4 倍（$2 \times 2$），像素数减少为 1/4
- 通道数增加 2 倍，总信息量：$(1/4) \times 2 = 1/2$
- 信息量减少但更"浓缩"，语义更强

**具体数值例子**：
- Stage 1：$8 \times 56 \times 56 \times 96$，信息量 = $8 \times 56 \times 56 \times 96 \approx 2.4M$
- Stage 2：$4 \times 28 \times 28 \times 192$，信息量 = $4 \times 28 \times 28 \times 192 \approx 0.6M$
- Stage 4：$1 \times 7 \times 7 \times 768$，信息量 = $1 \times 7 \times 7 \times 768 \approx 37K$

**边界情况**：
- 如果通道数不翻倍，深层特征的信息容量不足
- 如果通道数翻倍太多，参数量增长过快

### 1. Pooling 的实现

```python
class PoolAttention(nn.Module):
    def __init__(self, dim, num_heads, q_stride=1, kv_stride=2):
        self.q_pool = nn.AvgPool3d(kernel_size=q_stride, stride=q_stride)
        self.kv_pool = nn.AvgPool3d(kernel_size=kv_stride, stride=kv_stride)
    
    def forward(self, x):
        # x: (B, T, H, W, C)
        B, T, H, W, C = x.shape
        
        # QKV projection
        q = self.q_proj(x)
        k = self.k_proj(x)
        v = self.v_proj(x)
        
        # Pooling
        q = self.q_pool(q)
        k = self.kv_pool(k)
        v = self.kv_pool(v)
        
        # Reshape for attention
        # ... (reshape to (B, num_heads, N_q, C//h) etc.)
        
        # Attention
        attn = torch.matmul(q, k.transpose(-2, -1)) / math.sqrt(dim // num_heads)
        attn = attn.softmax(dim=-1)
        out = torch.matmul(attn, v)
        
        return out
```

### 前驱工作

1. **ViT [Dosovitskiy et al., 2021]**：证明了 Transformer 在图像上的可行性，是 MViT 的基础
2. **ResNet [He et al., 2016]**：stage 设计和多尺度特征金字塔的灵感来源
3. **SlowFast [Feichtenhofer et al., 2019]**：视频理解中多时间分辨率的代表性工作

### 同期竞品

1. **TimeSformer [Bertasius et al., 2021]**：分开的空间和时间注意力，和 MViT 同期发表，但计算效率不如 MViT
2. **ViViT [Arnab et al., 2021]**：多种时空分解方式，参数量大，效率低
3. **Swin Transformer [Liu et al., 2021]**：图像上的局部注意力+移位窗口，MViT 在视频上的多尺度思路有异曲同工之妙

### 后续影响

1. **MViTv2 [Li et al., 2022]**：MViT 的直接扩展，引入了 decomposed relative position encoding 和 residual pooling connection，在视频和图像任务上都取得了更好的结果
2. **VideoMAE [Tong et al., 2022]**：使用 MViT 作为 backbone 进行视频自监督学习，证明了 MViT 作为通用视频 backbone 的能力
3. **UniFormer [Li et al., 2022]**：结合了 3D CNN 和 Transformer 的优势，其中多尺度设计借鉴了 MViT
4. **InternVideo [Wang et al., 2022]**：使用 MViT 作为基础模型之一进行视频表示学习
5. **Video Swin Transformer [Liu et al., 2022]**：Swin Transformer 的视频扩展，和 MViT 在多尺度设计上互相启发

## 十、局限

1. **预训练数据需求大**：MViT 在 ImageNet-21K 上预训练才能达到最佳效果，这对计算资源要求很高。相比之下，3D CNN 可以直接在视频数据集上从头训练。

2. **长视频处理困难**：MViT 的输入是固定长度的视频片段（16-32 帧），无法直接处理更长的视频。虽然可以通过滑动窗口处理，但会丢失帧间的长程依赖。

3. **空间-时间 pooling 的参数选择**：不同 stage 的 pooling 参数（核大小、步长）需要手动设计，缺乏自适应机制。MViTv2 通过 residual pooling connection 缓解了这个问题。

4. **位置编码的局限性**：使用可学习位置编码，无法处理任意长度的视频。后续工作（如 VideoMAE）使用相对位置编码来解决这个问题。

5. **对快速运动的处理**：虽然 MViT 有多尺度时间处理，但对极快运动（如体育视频中的快速动作）的处理能力有限，因为高层的时间分辨率已经很低。

## 十一、小结

MViT 是视频理解领域的一个里程碑式工作。它解决了 ViT 扩展到视频时的三个核心问题：计算复杂度爆炸、缺乏多尺度特征、时间维度建模困难。

核心贡献是 **Pooling Attention**——在 Transformer block 内部对 Q、K、V 做不同粒度的池化，同时实现多尺度特征提取和计算效率提升。这个设计简单而优雅，不需要复杂的架构修改，就能把 CNN 的特征金字塔思想引入 Transformer。

**更广泛的启示**：MViT 证明了 Transformer 在视频理解上的潜力，开启了一个新的研究方向。后续的 VideoMAE、MViTv2 等工作都在 MViT 的基础上进一步发展。同时，MViT 的多尺度设计思想也被图像领域的 Swin Transformer 等模型借鉴。

**一句话总结：MViT 通过 Pooling Attention 在 Transformer 中实现了多尺度时空特征金字塔，以 CNN 级别的参数量达到了 ViT 级别的准确率，是视频 Transformer 的奠基之作。**
