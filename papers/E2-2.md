# # Pay Attention to MLPs（2021）

**论文：** [arXiv](https://arxiv.org/abs/2105.08050) · Hanxiao Liu, Zihang Dai, David R. So, Quoc V. Le (Google Research, Brain Team) · 2021

---

## 一、先搞清楚坑在哪

Vision Transformer（ViT）在 2020 年横空出世，第一次证明了纯 Transformer 架构（只有 attention + MLP）在图像分类上可以媲美甚至超越 CNN。这引发了一个很自然的问题：**attention 到底是不是 ViT 成功的必要条件？**

直觉上，attention 层负责捕捉 token 之间的全局依赖关系，而 MLP 层负责在每个 token 内部做特征变换。两者缺一不可——至少当时大家是这么认为的。

但这篇论文的作者提出了一个反直觉的假设：**也许真正起作用的不是 attention，而是 MLP？** 或者更精确地说——也许 attention 只是在做一种特殊的 token 间通信，而这种通信完全可以用更简单的 MLP 替代？

## 二、ViT 的真正问题

ViT 的核心架构可以简化为交替堆叠的两个模块：
1. **Multi-Head Self-Attention (MHSA)**：让 patch token 之间互相"看"对方，捕捉全局关系
2. **MLP**：对每个 token 独立做非线性变换

这里有一个隐藏的假设：**attention 是唯一能实现 token 间通信的机制**。MLP 只做 per-token 的处理，不涉及 token 间的信息交换。

但 attention 有几个让人不舒服的地方：
- **计算复杂度是 $O(N^2)$**，N 是 patch 数量。对高分辨率图像，patch 数轻松上千，二次复杂度很疼
- **需要位置编码**，因为 attention 本身是置换不变的（permutation-invariant），不告诉它位置它不知道谁是谁
- **实现复杂**，相比 MLP 的矩阵乘法，attention 需要 QKV 投影、softmax、scaled dot-product 等一堆操作

如果 attention 真的是 ViT 成功的核心，那这些代价是值得的。但如果不是呢？

## 三、gMLP 的核心思路

这篇论文的核心洞察只有一句话：**一个简单的 MLP，只要设计成能实现 token 间通信，就能达到甚至超越 attention 的效果。**

具体来说，作者提出了 **gMLP（gated MLP）**，一个完全由 MLP 层组成的架构。它的核心模块是 **Spatial Gating Unit (SGU)**——一个在空间维度（即 token 之间）做门控线性变换的层。

整个 gMLP 没有 attention，没有位置编码，只有：
- 标准的 per-token 线性投影
- SGU：在 token 间做线性变换 + 门控
- 标准的 per-token 非线性 MLP

就这么简单。

## 四、网络架构详解

### 4.1 整体架构

gMLP 的整体架构如下：

```
输入图像: 224x224x3
    │
    ▼
Patch Embedding: 16x16 patches → 196 tokens, 每个 token 维度 d_model
    │
    ▼
gMLP Block × L 层 (默认 L=12, d_model=384)
    │
    ▼
Layer Norm → 分类头 (Linear → Softmax)
```

每个 gMLP Block 的结构：

```
输入: x ∈ R^{N×d} (N=196 tokens, d=384)
    │
    ▼
Layer Norm
    │
    ▼
通道投影 (线性层): d → d_ff (默认 d_ff=384)
    │
    ▼
┌─────────────────────────────────────┐
│         Spatial Gating Unit         │
│  ┌──────────┐     ┌──────────┐     │
│  │ 分支 u   │     │ 分支 v   │     │
│  │ (N×d_ff) │     │ (N×d_ff) │     │
│  └────┬─────┘     └────┬─────┘     │
│       │                │           │
│       │      ┌─────────▼──────┐    │
│       │      │ 线性层: d_ff→d_ff│   │
│       │      │ (在 N 维度上)   │    │
│       │      └─────────┬──────┘    │
│       │                │           │
│       └────────┬───────┘           │
│                ▼                   │
│          逐元素相乘 (门控)          │
└─────────────────────────────────────┘
    │
    ▼
通道投影 (线性层): d_ff → d
    │
    ▼
残差连接: 输出 = 输入 + 投影输出
```

### 4.2 前向过程详解

我们一步一步走，跟踪每个张量的形状：

**Step 1: Patch Embedding**
- 输入: `224×224×3` 图像
- 切成 16×16 patches: `(224/16)×(224/16) = 14×14 = 196` 个 patch
- 每个 patch 展平并投影到 d_model=384 维
- 输出: `(196, 384)`，记作 `x`

**Step 2: 进入第一个 gMLP Block**

**Step 2a: Layer Norm**
- 对每个 token 的 384 维做 Layer Normalization
- 形状不变: `(196, 384)`

**Step 2b: 通道投影 (第一个线性层)**
- `W: 384 → d_ff`，这里 d_ff=384（默认相等）
- 输出: `(196, 384)`，记作 `z`

**Step 2c: Spatial Gating Unit (SGU)**

SGU 的核心思想是把张量分成两半，一半直接传递，另一半经过空间维度上的线性变换后再做门控。

```
z: (196, 384)
    │
    ├── 前一半: u = z[:, :d_ff//2] = z[:, :192]  → (196, 192)
    │
    └── 后一半: v = z[:, d_ff//2:] = z[:, 192:] → (196, 192)
    
v' = (W_v @ v.T).T  → 线性变换在 N 维度上
   = v @ W_v.T      → (196, 192) @ (192, 192) = (196, 192)
   
输出 = u ⊙ v'  → (196, 192) ⊙ (196, 192) = (196, 192)
```

这里 `W_v` 是一个 `(192, 192)` 的可学习矩阵，作用在 token 维度上。**这就是 token 间通信发生的地方**——每个 token 的新表示是所有 token 的线性组合。

**Step 2d: 通道投影 (第二个线性层)**
- 将 SGU 的输出从 d_ff/2=192 投影回 d=384
- 输出: `(196, 384)`

**Step 2e: 残差连接**
- `输出 = x + 投影输出` → `(196, 384)`

**Step 3: 重复 L=12 个 Block**

**Step 4: 分类头**
- 取第一个 token（class token）或对所有 token 做平均
- 线性层: 384 → 1000 (ImageNet 类别数)
- Softmax

### 4.3 为什么 SGU 能替代 Attention？

这里有个关键：SGU 中的线性层 `W_v` 是在空间维度（token 维度）上做变换的。这意味着：

- 每个 token 的输出是所有 token 的加权和
- 权重由 `W_v` 学习得到，是静态的（input-independent）
- 对比 attention：attention 的权重是动态计算的（依赖于 query 和 key 的相似度）

**静态 vs 动态** 是 gMLP 和 Transformer 的核心区别。gMLP 说：静态权重就够了，不需要复杂的动态注意力。

## 五、核心创新点

### 创新点 1：Spatial Gating Unit (SGU)

**(a) 痛点与动机**

Transformer 中 token 间通信依赖 attention，但 attention 有两个问题：
1. **计算复杂度 $O(N^2)$**：对长序列不友好
2. **需要位置编码**：attention 是置换不变的，必须额外注入位置信息

作者观察到：也许 token 间通信不需要这么复杂。一个简单的线性变换 + 门控就够了。

**(b) 方案细节**

SGU 的实现极其简单：
```python
def spatial_gating_unit(x):
    u, v = x.chunk(2, dim=-1)  # 沿通道维度分成两半
    v = v @ W_v.T  # 在 token 维度上做线性变换
    return u * v    # 逐元素相乘（门控）
```

就这么三行代码。

**(c) 为什么有效**

直觉上，SGU 做的事情是：
1. **分支 u**：保留原始信息（identity mapping）
2. **分支 v**：收集所有 token 的信息（线性变换）
3. **门控**：用 v 的信息来调制 u 的信息

这本质上是一种**上下文调制**：每个 token 的最终表示取决于它自己（u）和所有其他 token 的加权组合（v'）。门控机制让模型可以动态决定哪些信息需要被"放大"或"抑制"。

实验结果（Table 1）显示：gMLP-B（12层，d=768）在 ImageNet 上达到 82.0% top-1 准确率，只比同等大小的 ViT-B（82.3%）低 0.3%。这说明 SGU 确实能有效替代 attention。

**(d) 与 Related Work 的关系**

SGU 与以下工作有联系：
- **MLP-Mixer [Tolstikhin et al., 2021]**：同期工作，也提出用 MLP 替代 attention。不同点在于 MLP-Mixer 在空间维度上使用两个 MLP（一个 per-patch，一个 per-channel），而 gMLP 使用单个门控单元
- **FNet [Lee-Thorp et al., 2021]**：用傅里叶变换替代 attention，也是线性变换
- **Synthesizer [Tay et al., 2020]**：提出用合成注意力（synthetic attention）替代标准注意力

**(e) 如果去掉会怎样**

Table 3 显示：如果把 SGU 替换为恒等映射（即只保留 MLP 部分，去掉所有 token 间通信），gMLP 在 ImageNet 上的准确率从 82.0% 降到 79.2%，掉了 2.8 个点。这说明 SGU 对性能至关重要。

### 创新点 2：消除位置编码

**(a) 痛点与动机**

ViT 需要位置编码，但位置编码的设计很麻烦：需要决定是绝对位置还是相对位置，是学习式的还是固定的，以及如何处理不同分辨率的输入。

**(b) 方案细节**

gMLP 不需要任何位置编码。这是因为 SGU 中的线性层 `W_v` 是**非置换不变的**——`W_v` 的每一行对应一个特定的 token 位置，所以 token 的顺序信息天然被编码在权重矩阵中。

**(c) 为什么有效**

直观理解：在 attention 中，`QK^T` 计算的是 token 之间的相似度，这个相似度与位置无关（除非加位置编码）。但在 SGU 中，`W_v` 的第 i 行第 j 列直接决定了 token j 对 token i 的贡献权重，这个权重是固定的（对每个位置对有一个固定的权重），所以位置信息隐含在权重矩阵中。

Table 3 验证了这一点：gMLP 去掉位置编码后性能不变（82.0% vs 82.0%），而 ViT 去掉位置编码后性能大幅下降。

**(d) 与 Related Work 的关系**

- **ViT [Dosovitskiy et al., 2020]**：需要可学习的位置编码
- **DeiT [Touvron et al., 2021]**：同样需要位置编码
- **MLP-Mixer [Tolstikhin et al., 2021]**：也不需要位置编码，原因类似

**(e) 如果去掉会怎样**

实际上 gMLP 天生就不需要位置编码，所以不存在"去掉"的问题。但如果强行给 gMLP 加位置编码会怎样？Table 3 显示：加了位置编码后性能反而略有下降（82.0% → 81.9%），说明位置编码对 gMLP 来说不仅是多余的，还可能有害。

## 六、公式详解

### 公式 1：gMLP Block

$$y = \text{MLP}(\text{SGU}(\text{LN}(x))) + x$$

**符号定义：**
- $x \in \mathbb{R}^{N \times d}$：输入，N 个 token，每个 d 维
- $\text{LN}$：Layer Normalization
- $\text{SGU}$：Spatial Gating Unit
- $\text{MLP}$：两层线性层 + GELU 激活
- $y$：输出，形状与 x 相同

**公式来源：**
这是标准残差连接的形式，来自 [He et al., 2016] 的 ResNet。gMLP 只是把内部的卷积替换为 SGU + MLP。

**直觉理解：**
残差连接让梯度可以直接流回浅层，解决了深层网络训练难的问题。gMLP 沿用这个设计。

### 公式 2：Spatial Gating Unit

$$\text{SGU}(z) = u \odot (W_v v + b_v)$$

其中 $z = [u, v]$，$u, v \in \mathbb{R}^{N \times d/2}$，$W_v \in \mathbb{R}^{N \times N}$，$b_v \in \mathbb{R}^{N}$

**符号定义：**
- $z$：SGU 的输入，沿通道维度分成两半 $u$ 和 $v$
- $u$：前一半通道，直接传递（identity mapping）
- $v$：后一半通道，经过空间线性变换
- $W_v$：空间维度上的线性变换矩阵，形状为 $N \times N$
- $b_v$：偏置项，形状为 $N$
- $\odot$：逐元素相乘（Hadamard product）

**公式来源：**
这是作者提出的新设计。门控机制来自 [Dauphin et al., 2017] 的 Gated Linear Unit (GLU)，但 SGU 在空间维度上做门控，而 GLU 在通道维度上做门控。

**推导过程：**
1. 输入 $z \in \mathbb{R}^{N \times d}$ 被分成两半：
   $$u = z[:, :d/2], \quad v = z[:, d/2:]$$
   
2. 对 $v$ 做空间线性变换：
   $$v' = W_v v + b_v$$
   注意这里 $W_v$ 作用在 $N$ 维度上：$(W_v v)_{ij} = \sum_{k=1}^N (W_v)_{ik} v_{kj}$

3. 门控操作：
   $$\text{SGU}(z) = u \odot v'$$

**直觉理解：**
- $u$ 保留了每个 token 自身的特征信息
- $v'$ 聚合了所有 token 的信息（通过 $W_v$ 的线性组合）
- 逐元素相乘实现了"用全局信息调制局部信息"的效果

**具体例子：**
假设 N=4（4 个 token），d=6（每个 token 6 维），则 d/2=3。

```
z = [
  [1, 2, 3, 4, 5, 6],    # token 1
  [7, 8, 9, 10, 11, 12],  # token 2
  [13, 14, 15, 16, 17, 18], # token 3
  [19, 20, 21, 22, 23, 24]  # token 4
]

u = [
  [1, 2, 3],
  [7, 8, 9],
  [13, 14, 15],
  [19, 20, 21]
]

v = [
  [4, 5, 6],
  [10, 11, 12],
  [16, 17, 18],
  [22, 23, 24]
]

假设 W_v 为 4×4 的单位矩阵（简化），b_v = 0：
v' = W_v v = v

SGU(z) = u ⊙ v = [
  [1×4, 2×5, 3×6],
  [7×10, 8×11, 9×12],
  [13×16, 14×17, 15×18],
  [19×22, 20×23, 21×24]
] = [
  [4, 10, 18],
  [70, 88, 108],
  [208, 238, 270],
  [418, 460, 504]
]
```

**边界情况：**
- 当 $W_v = I$（单位矩阵）且 $b_v = 0$ 时，SGU 退化为逐元素平方：$u \odot v$
- 当 $W_v$ 的所有行相等时，所有 token 的输出都相同（退化为全局平均池化）
- 当 $d$ 为奇数时，需要特殊处理（实际实现中通常保证 d 为偶数）

### 公式 3：计算复杂度分析

$$\text{复杂度}_{\text{SGU}} = O(N^2 d/2) = O(N^2 d/2)$$

$$\text{复杂度}_{\text{Attention}} = O(N^2 d)$$

**符号定义：**
- N：token 数量
- d：模型维度

**公式来源：**
标准的大 O 复杂度分析。

**推导过程：**

SGU 的计算主要来自矩阵乘法 $W_v v$：
- $W_v \in \mathbb{R}^{N \times N}$，$v \in \mathbb{R}^{N \times d/2}$
- 计算量：$N \times N \times d/2 = N^2 d/2$

Attention 的计算：
- QKV 投影：$3 \times N \times d^2$
- Attention score：$N \times N \times d$
- Softmax：$N \times N$
- 加权求和：$N \times N \times d$
- 总计算量：$2N^2 d + 3Nd^2$

**直觉理解：**
SGU 的复杂度是 $O(N^2 d/2)$，attention 是 $O(N^2 d)$。但更重要的是，SGU 不需要 $O(Nd^2)$ 的投影计算，所以实际计算量更小。

**具体例子：**
以 gMLP-B 为例，N=196（14×14 patches），d=768：
- SGU：$196^2 \times 768/2 \approx 14.7M$ 次浮点运算
- Attention：$2 \times 196^2 \times 768 + 3 \times 196 \times 768^2 \approx 59.0M + 346.6M \approx 405.6M$ 次浮点运算

SGU 的计算量只有 attention 的约 3.6%。

**边界情况：**
- 当 N 很小（如 N=1）时，SGU 和 attention 的计算量都可忽略
- 当 N 很大（如 N=4096）时，SGU 的 $O(N^2)$ 仍然是个问题，但比 attention 的 $O(N^2 d)$ 要好

## 八、位置

### 前驱工作

1. **Transformer [Vaswani et al., 2017]**：提出了 attention + MLP 的架构范式，gMLP 挑战了 attention 的必要性
2. **ViT [Dosovitskiy et al., 2020]**：将 Transformer 应用于图像分类，证明了纯 Transformer 可以超越 CNN
3. **GLU [Dauphin et al., 2017]**：提出了门控线性单元，SGU 借鉴了门控思想
4. **ResNet [He et al., 2016]**：残差连接，gMLP 沿用了这个设计

### 同期竞品

1. **MLP-Mixer [Tolstikhin et al., 2021]**：同时期发表，也用 MLP 替代 attention。不同点在于 MLP-Mixer 使用两个 MLP（空间 MLP 和通道 MLP），而 gMLP 使用单个门控单元
2. **FNet [Lee-Thorp et al., 2021]**：用傅里叶变换替代 attention
3. **Synthesizer [Tay et al., 2020]**：提出合成注意力

### 后续影响

1. **ConvNeXt [Liu et al., 2022]**：现代化 CNN 设计，借鉴了 gMLP 的简单设计理念
2. **MetaFormer [Yu et al., 2022]**：提出"池化可以做注意力"的观点，进一步简化了 token 间通信
3. **Mamba [Gu & Dao, 2023]**：状态空间模型，用更高效的线性复杂度替代 attention
4. **RWKV [Peng et al., 2023]**：RNN 风格的 Transformer 替代方案
5. **Hyena [Poli et al., 2023]**：用隐式 MLP 替代 attention

gMLP 最重要的贡献是证明了 **attention 不是 token 间通信的唯一方式**，打开了"后 Transformer"时代的大门。

## 九、局限

### 局限 1：静态权重 vs 动态权重

**问题：** SGU 中的 $W_v$ 是静态的（input-independent），对所有输入图像使用相同的权重。而 attention 的权重是动态计算的，可以针对不同输入自适应调整。

**影响：** 在需要强上下文理解的任务（如细粒度分类、目标检测）中，静态权重可能不够灵活。

**论文是否承认：** 论文在 Section 4.3 中讨论了这一点，承认 attention 的动态权重是一个优势。

**后续工作：** ConvNeXt 和 MetaFormer 通过混合使用 attention 和 MLP 来平衡静态和动态。

### 局限 2：$O(N^2)$ 复杂度

**问题：** SGU 的复杂度是 $O(N^2)$，虽然比 attention 的 $O(N^2 d)$ 好，但对长序列仍然不够友好。

**影响：** 对于需要处理大量 token 的任务（如高分辨率图像、视频理解），gMLP 仍然会遇到计算瓶颈。

**论文是否承认：** 论文在 Section 5 中讨论了未来工作方向，提到了更高效的 token 间通信机制。

**后续工作：** Mamba、RWKV 等模型用线性复杂度替代了二次复杂度。

### 局限 3：在 NLP 任务上表现一般

**问题：** gMLP 在图像任务上表现良好，但在 NLP 任务（如语言建模、机器翻译）上不如 Transformer。

**影响：** 说明 attention 在 NLP 中可能确实有独特优势（如处理长距离依赖、建模语法结构）。

**论文是否承认：** 论文在 Section 4.4 中报告了 NLP 实验结果，gMLP 在 SuperGLUE 上比 BERT 低约 2-3 个点。

**后续工作：** 后续的 MLP 变体（如 MLP-Mixer）主要在视觉任务上应用，NLP 仍然是 Transformer 的天下。

### 局限 4：对超参数敏感

**问题：** gMLP 中 d_ff 的选择（等于 d 还是 2d 或 4d）对性能影响较大。

**影响：** 需要仔细调参才能达到最佳性能。

**论文是否承认：** 论文在 Section 3.2 中讨论了 d_ff 的选择，但没有深入分析敏感性。

## 十、小结

gMLP 这篇论文的核心贡献是：**证明了 attention 不是 ViT 成功的必要条件**。通过简单的 Spatial Gating Unit（一个 $N \times N$ 的线性层 + 门控），gMLP 实现了与 ViT 相当的 ImageNet 分类性能，同时更简单、更高效、不需要位置编码。

这篇论文的 broader impact 是：它打开了"后 Transformer"时代的大门，激发了后续一系列替代 attention 的工作（ConvNeXt、MetaFormer、Mamba、RWKV、Hyena 等）。它告诉我们：**不要被已有的成功架构限制想象力，最简单的方案往往是最优雅的。**

当然，gMLP 也有局限：静态权重不如 attention 灵活、在 NLP 任务上表现一般。但它的核心洞察——"也许我们不需要 attention"——已经深刻影响了深度学习架构的设计方向。
