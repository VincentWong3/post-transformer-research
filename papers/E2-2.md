## 一、论文基本信息

### 1.1 标题的幽默

论文的标题"Pay Attention to MLPs"是一个双关语。字面意思是"关注一下 MLP"，但"Pay Attention"同时也是对 Transformer 核心机制——self-attention——的调侃。标题的潜台词是：**别把目光全放在 attention 上，MLP 可能才是真正重要的组件。**

这种命名本身就暗示了论文的激进立场。它不是要改进 Transformer，而是要挑战 Transformer 的一个基本假设：self-attention 是必不可少的。

### 1.2 作者阵容

这篇论文的作者全部来自 Google Research，阵容极具分量：

- **Hanxiao Liu**：AutoML 和 NAS 方向的知名研究者，先前参与了 EfficientNet 等高效架构的研究。
- **Zihang Dai**：Transformer-XL 的作者之一，对 Transformer 架构的改进有深厚积累。
- **David R. So**：同样来自 NAS 和 Transformer 方向，参与了 Evolved Transformer 的工作。
- **Quoc V. Le**：Google Brain 的核心人物，AutoML、EfficientNet 等大量重要工作的领导者。

四位作者在不同方向上都有大量经验，但共同点是都深入了解 Transformer 的内部工作原理。这使得论文的结论——self-attention 并非 Transformer 成功的必要条件——更加有分量。

### 1.3 发表背景

gMLP 发表于 2021 年 5 月。这个时间点很有趣：就在几个月前（2021 年 3 月），Google 的另一个团队刚发表了 MLP-Mixer，提出了一个完全基于 MLP 的视觉架构。2021 年初的几个月里，"MLP 能否替代 Transformer"成为了深度学习社区的热门话题。MLP-Mixer 证明了在 ImageNet 上可以用纯 MLP 达到有竞争力的结果，但它的设计比较笨重——需要两个独立的 MLP 层分别处理 channel mixing 和 token mixing，且需要大量的训练技巧。

gMLP 在这个基础上往前走了一步：它保留了 MLP 作为基础构件，但引入了一种更优雅的 token mixing 机制——**Spatial Gating Unit（SGU）**——使得架构更简洁、更容易训练，并且在视觉和 NLP 两个领域都证明了与 Transformer 相当的性能。

这实际上是对"MLP vs Transformer"辩论的一个回答：**你不需要 self-attention，但你确实需要某种形式的 token mixing——SGU 就是一个足够好的替代方案。**

## 二、研究动机：Self-Attention 到底是不是必须的？

### 2.1 Transformer 的成功源于什么？

到 2021 年，Transformer 在 NLP（BERT、GPT-3）和视觉（ViT、DeiT）两个领域都取得了巨大成功。但在成功背后，学界对一个问题一直争论不休：

**Transformer 的成功到底来自 self-attention，还是来自其整体架构设计（残差连接、LayerNorm、FFN 的堆叠）？**

这个问题不是纯粹的学术好奇。如果 Transformer 的成功主要来自架构设计的整体优势，而非 attention 本身，那就意味着：

1. 研究者可能浪费了大量精力在优化 attention 机制上——而 attention 可能并不是贡献最大的部分。
2. 很多所谓的"attention 改进"工作，改进的可能并非 attention 本身，而是附带的正则化或表征效果。
3. 如果可以用更简单的机制替代 attention，模型可能会更高效、更容易训练。

### 2.2 已有线索

在 gMLP 之前，已经有多个工作暗示 attention 可能不是唯一的解决方案：

- **MLP-Mixer（2021）**：用两个独立的 MLP 层分别做 channel mixing 和 token mixing，在 ImageNet 上达到了与 ViT 相当的精度。但 Mixer 的 token-mixing MLP 实际上是一个空间维度上的全连接层，需要输入序列长度固定，且不自然。

- **ResMLP（2021）**：进一步简化了 MLP-Mixer，去掉了 layer norm 中的可学习参数，用仿射变换替代。ResMLP 也在 ImageNet 上取得了不错的结果。

- **Synthesizer（2020）**：Tay et al. 的工作，尝试用可学习的 attention 权重替代输入依赖的 attention 权重，发现效果仍然不错。

但这些工作都留了一个尾巴：**在 NLP 领域，MLP 架构还能和 Transformer 抗衡吗？** MLP-Mixer 和 ResMLP 都只在视觉任务上做了实验，而在 NLP 上，大家的共识是 Transformer 不可替代。

gMLP 正是为了回答这个问题。它的目标是在**视觉和 NLP 两个领域**都证明：self-attention 不是必需的。

### 2.3 研究问题的精确表述

gMLP 要回答的问题是：

**在标准 Transformer 架构中，如果用一种简单的、无 attention 的空间混合机制替代 self-attention，同时在视觉和 NLP 两个领域都保持与 Transformer 相当的精度，是否可能？**

注意这个问题的精确性："替代 self-attention"意味着不走 attention 路线——不需要 QKV 分解，不需要点积相似度，不需要 softmax 归一化。"在两个领域都保持相当精度"意味着泛化能力要够宽——不只在一个领域有效。

## 三、核心思路：用线性门控替代 Attention

### 3.1 核心思想

gMLP 的核心思想可以用一句话概括：

**用线性门控（linear gating）做空间维度上的 token mixing，替代 self-attention 做同一件事的角色。**

"门控"在深度学习中是一个经典操作。LSTM 年代就已经广泛使用——通过一个 sigmoid 门的输出控制信息流的通断。在 gMLP 中，门控机制被用作 token 之间信息交换的方式：一个 token 的输出信号受其他 token 的影响而缩放，从而实现跨 token 的信息流通。

### 3.2 为什么门控可以替代 Attention？

Self-attention 的本质操作是：**对于每个 token，收集所有其他 token 的信息，按相似度加权求和。** 即：

$$\text{Output}_i = \sum_j \alpha_{ij} V_j$$

其中 $\alpha_{ij}$ 是 query i 和 key j 的相似度经过 softmax 归一化之后的值。

门控的操作是：**对于每个 token，将其信息投射到两个子空间，让其中一个子空间接受空间维度的线性变换，然后与另一个子空间逐元素相乘。** 即：

$$\text{Output}_i = Z_{1,i} \odot (W Z_{2})_i$$

其中 $W$ 是一个 $n \times n$ 的可学习矩阵（$n$ 是序列长度），$\odot$ 是逐元素乘法。

两种操作都能让 token 之间产生信息交互——attention 通过加权求和，门控通过线性投影+逐元素相乘。门控的交互方式更简单（没有 softmax，没有相似度计算），但足够让信息在序列维度流通。

### 3.3 整体思路的展开

gMLP 的设计思路大致如下：

1. **保留 Transformer 的 channel mixing**：每个 token 独立通过一个 FFN（MLP），将信息从输入维度映射到更高的中间维度，再映射回来。这等价于 Transformer 中 FFN 子层的功能。

2. **用 SGU 替代 self-attention 做 spatial mixing**：在 channel mixing 之间插入一个门控单元，该单元在空间维度（token 之间）进行信息交换。

3. **去掉不需要的组件**：不再需要位置编码（空间信息由 SGU 的权重矩阵隐式编码），不再需要 masking padding tokens。

这种设计的优雅之处在于：**你不需要改变 Transformer 的整体结构（堆叠 block、残差连接、LayerNorm），只需要替换其中的 attention 子层。** 这使得 gMLP 既是一个独立的新架构，也是 Transformer 的一个极端消融实验。

## 四、整体架构与 Forward 流程

### 4.1 gMLP Block 结构

一个标准的 gMLP block 的结构如下：

```text
输入 x ∈ ℝ^{n×d}
    │
    ├── LayerNorm
    │
    ├── Channel Projection (d → d_ffn)     ← 等价于 Transformer 中 FFN 的第一层
    │       线性变换 + 偏置
    │
    ├── GELU 激活函数
    │
    ├── Spatial Gating Unit (SGU)           ← 替代 Self-Attention 的关键模块
    │       在空间维度（n×n）做线性变换 + 门控
    │
    ├── Channel Projection (d_ffn/2 → d)  ← 将 SGU 输出的 d_ffn/2 维投影回模型维度 d     ← 等价于 Transformer 中 FFN 的第二层
    │       线性变换 + 偏置
    │
    └── + 残差连接

输出 LayerNorm(x + block_output)
```

注意这里一个关键的细节：**gMLP 的架构设计去掉了标准的"两个子层"结构**。在 Transformer 中，一个 block 包含两个子层（self-attention 子层 + FFN 子层），每个子层都有自己的残差连接和 LayerNorm。在 gMLP 中，**一个 block 只有一个残差连接和一个 LayerNorm**——SGU 被嵌入到 FFN 的中间，成为了一个整体。

这样的设计比 Transformer 更紧凑。每个 block 是：

```
output = x + LayerNorm → Linear(d→d_ffn) → GELU → SGU (输出 d_ffn/2) → Linear(d_ffn/2→d)
```

### 4.2 Forward 流程详解

以一个长度为 n、隐藏维度为 d 的序列为例：

**1. LayerNorm**
输入 $x \in \mathbb{R}^{n \times d}$，对每个 token（行）做层归一化。

**2. 第一次 Channel Projection（$d \to d_{ffn}$）**
$$
U = \text{GELU}(X W_1 + b_1)
$$
其中 $W_1 \in \mathbb{R}^{d \times d_{ffn}}$，$b_1 \in \mathbb{R}^{d_{ffn}}$。

论文中通常设置 $d_{ffn}$ 比 $d$ 大几倍（类似 Transformer 中的 FFN expansion ratio）。

**3. SGU 空间门控**
$$
Z = \text{SGU}(U)
$$
这是核心操作，下一节会详细展开。

**4. 第二次 Channel Projection（$d_{ffn} \to d$）**
$$
S = Z W_2 + b_2
$$
其中 $W_2 \in \mathbb{R}^{d_{ffn} \times d}$，$b_2 \in \mathbb{R}^{d}$。

**5. 残差连接**
$$
Y = X + S
$$

### 4.3 与 Transformer 的对比

```text
Transformer Block:
    x → LN → MHA  → + → LN → FFN → + → y
                          ↑              ↑
                       残差①          残差②

gMLP Block:
    x → LN → Linear(d→d_ffn) → GELU → SGU (d_ffn/2) → Linear(d_ffn/2→d) → + → y
                                                              ↑
                                                           残差①
```

两个关键区别：

1. **子层数量不同**：Transformer 有两个子层（MHA + FFN），各自带残差。gMLP 只有一个残差路径。

2. **SGU 替代 MHA 的位置**：SGU 不是在 MHA 子层的位置，而是嵌入在 FFN 中间。这实际上把"做 channel mixing 的 FFN"和"做 spatial mixing 的 SGU"组合成了一个完整的计算块，更高效。

## 五、Spatial Gating Unit（SGU）——核心模块详解

### 5.1 SGU 的定义

Spatial Gating Unit 是 gMLP 的核心创新。它的输入是一个矩阵 $U \in \mathbb{R}^{n \times d_{ffn}}$，其中 $n$ 是序列长度，$d_{ffn}$ 是中间隐藏维度。

SGU 的完整计算如下：

```text
输入 U ∈ ℝ^{n×d_ffn}
    │
    ├── 沿 channel 维度（列）split
    │
    ├── Z₁ ∈ ℝ^{n×(d_ffn/2)}     ← 第一部分，直接用于门控乘法
    │
    ├── Z₂ ∈ ℝ^{n×(d_ffn/2)}     ← 第二部分，经过空间投影
    │       │
    │       ├── 可选：LayerNorm（在 Z₂ 上）
    │       │
    │       └── 线性变换: Z₂' = (W ⊙ S) Z₂ + b
    │              W ∈ ℝ^{n×n} 可训练
    │              S 是固定的缩放因子（可选，用于初始化约束）
    │              b ∈ ℝ^{n} 可训练偏置项
    │              注：W 和输入 U 无关——是静态参数
    │
    └── 门控乘法: Z₁ ⊙ Z₂'       ← 逐元素相乘
        │
        └── 输出 Z ∈ ℝ^{n×d_ffn}
```

**数学表达式：**

$$
Z = Z_1 \odot (W Z_2 + b)
$$

其中：
- $[Z_1, Z_2] = \text{split}(U)$，$Z_1, Z_2 \in \mathbb{R}^{n \times d_{ffn}/2}$
- $W \in \mathbb{R}^{n \times n}$ 是可训练的参数矩阵
- $b \in \mathbb{R}^n$ 是偏置项
- $\odot$ 表示逐元素乘法（广播到所有 channel）
- $W Z_2$ 的计算是 $n \times n$ 乘以 $n \times (d_{ffn}/2)$，复杂度 $O(n^2 \cdot d_{ffn}/2)$

### 5.2 SGU 的关键设计决策

**决策 1：静态投影 vs 动态投影**

这是 SGU 与 self-attention 最根本的区别。

在 self-attention 中，token 之间的交互权重是**动态计算**的：$\alpha_{ij} = \text{softmax}(q_i^\top k_j / \sqrt{d})$。这个权重依赖于当前输入的内容——交互模式随着输入的变化而变化。

在 SGU 中，投影矩阵 $W$ 是**静态的**：它在训练过程中学习，一旦训练完成就固定下来。$W$ 不会根据当前输入动态调整。从某种程度上说，$W$ 编码的是空间位置上的一种"先验交互模式"——它学习的是哪些位置应该彼此影响，而不是影响的方式应该取决于内容。

有趣的是，论文的实验证明：**静态投影已经足够好了。** 在大部分任务上，gMLP 与 Transformer 的表现相差无几。这说明很多任务中，token 之间的交互模式更多由位置关系决定（局部性），而非由语义内容决定。

**决策 2：Channel splitting**

SGU 没有使用输入的全部 channel。它只使用了一半的 channel 做空间投影，另一半直接参与乘法。

这实际上是一种**资源分配**策略：
- $Z_1$ 只携带逐 token 的信息，不做空间交互
- $Z_2$ 携带空间信息，经过 $W$ 的投影后与 $Z_1$ 交互

这种 split 设计有两个好处：
1. **计算效率**：只需要对一半的 channel 做 $n \times n$ 的空间投影，减少了计算量。
2. **避免过度混合**：保留了信息直通的路径（$Z_1$ 直接参与输出），防止信息被过度混合后丢失。

**决策 3：不需要 softmax**

SGU 的门控操作使用 $\odot$（逐元素乘法），不需要 softmax。这意味着：
- 没有指数运算（更高效）
- 没有归一化
- 数值稳定性更好
- 不需要 mask padding token（因为 $W$ 是直接学习的，不会被 padding 影响）

### 5.3 SGU 的计算复杂度

对于序列长度 $n$、中间维度 $d_{ffn}$：

- Spatial projection: $O(n^2 \cdot d_{ffn}/2)$
- Element-wise multiply: $O(n \cdot d_{ffn}/2)$
- Broadcasting bias: $O(n \cdot d_{ffn}/2)$

总复杂度：$O(n^2 d_{ffn})$。这与 standard attention 的 $O(n^2 d)$ 在同一量级（$d_{ffn}$ 通常比 $d$ 大几倍）。

所以 SGU 不是一种"高效 attention"——它没有像 Linformer 或 Performer 那样试图把复杂度降到线性。gMLP 的目的是**简化**而非**加速**。

## 六、初始化策略与训练稳定性

### 6.1 初始化的重要性

SGU 使用了一个精心设计的初始化策略。这对于确保训练稳定至关重要，尤其是在模型较深时。

如果不做特殊初始化，随机初始化的 $W$ 矩阵可能会在训练初期产生过强的空间混合，导致梯度爆炸或消失。

### 6.2 初始化公式

论文将 $W$ 初始化为接近零，将 $b$ 初始化为 1：

$$
W \approx 0, \quad b \approx \mathbf{1}
$$

更精确地说，$W$ 的每个元素被初始化为一个非常小的随机值（接近零），偏置项 $b$ 的所有元素被初始化为 1。

### 6.3 初始化后的 SGU 行为

当 $W \approx 0$ 且 $b \approx 1$ 时：

$$
W Z_2 + b \approx 0 \cdot Z_2 + \mathbf{1} = \mathbf{1}
$$

于是门控乘法变为：

$$
Z_1 \odot (W Z_2 + b) \approx Z_1 \odot \mathbf{1} = Z_1
$$

这意味着在训练初期，SGU 近似一个恒等映射：W 接近零使得 f_W,b(Z₂) ≈ 1，输出 Z = Z₁ ⊙ 1 ≈ Z₁。由于 Z 只有 d_ffn/2 维，后续用 Linear(d_ffn/2 → d) 投影回 d 维。

更精确地说，$Z_1 \in \mathbb{R}^{n \times d_{ffn}/2}$ 与 $b \in \mathbb{R}^n$ 做广播乘法时：$Z_1 \odot (W Z_2 + b)$ 中 $b$ 是 $n \times 1$ 的向量（广播到 $d_{ffn}/2$ 维度）。当 $b_i = 1$ 时，第 $i$ 行乘以 1，没有变化。

所以在训练开始时，每个 gMLP block 的输出接近于：

```
output ≈ x + LayerNorm → Linear(d→d_ffn) → GELU → SGU (d_ffn/2) → Linear(d_ffn/2→d)
```

这等价于去掉 SGU 后的标准 FFN。**训练过程逐渐学习 $W$ 矩阵，使 SGU 逐步发挥空间混合的作用。**

### 6.4 为什么要这样初始化？

这种初始化策略的核心目的是**稳定训练**。

在深度学习中，从恒等映射开始训练是一种常见的技巧（如 ResNet）。它确保了在网络很深时，梯度可以顺畅地通过 shortcut 路径传播。

对于 SGU 来说，从恒等映射开始还有另一个好处：它避免了训练初期空间混合强度过大导致的梯度问题。如果 $W$ 在初始化时就产生强烈的空间交互（比如随机矩阵的线性变换），整个门控的输出可能完全失控——尤其当 $n$ 较大时，$WZ_2$ 中每个元素是 $n$ 个随机值的和，方差会随着 $n$ 增长。

通过将 $W$ 初始化为接近零，网络可以**逐步**学习空间混合模式，而不是一开始就必须适应剧烈的空间变换。

### 6.5 与 Transformer 的初始化对比

Transformer 中的 attention 初始化也需要特殊处理。标准 Transformer 的 attention 输出在初始化时通常也是接近零的——因为 QKV 投影是随机初始化的，两两点积的和趋近于零均值。然后通过残差连接，初始输出也接近输入。

但 Transformer 的各个组件之间存在一个重要的交互：如果 attention 矩阵的熵太高（softmax 分布太平坦），多个 attention head 的叠加大幅增加了初始阶段的交互强度；如果熵太低（太尖锐），只有少数 token 能交互，信息流通受阻。

SGU 的初始化更直接——$W \approx 0$ 意味着几乎没有混合，训练过程完全可控。

## 七、位置编码与 Mask：不需要了

### 7.1 为什么不需要位置编码？

Transformer 的 self-attention 是**置换等变的（permutation equivariant）**——如果不加位置编码，打乱输入 token 的顺序不会改变 attention 的输出（只是 token 的位置也跟着变了）。

所以 Transformer 必须通过显式的位置编码（绝对位置编码、相对位置编码、或旋转位置编码等）来引入位置信息。

gMLP 的 SGU 不存在这个问题。原因很简单：**SGU 中的投影矩阵 $W$ 是有结构的**——它的每一行对应一个固定的空间位置。

具体来说，$W \in \mathbb{R}^{n \times n}$ 是一个全连接矩阵。第 $i$ 行和第 $j$ 列的元素 $W_{ij}$ 编码了"位置 $j$ 的信息对位置 $i$ 的输出有多大影响"。这个影响模式是在训练过程中学到的——模型可以学到相邻位置的影响更强（局部性），也可以学到对称位置的影响更强（空间不变性），取决于任务的需要。

所以 $W$ 天然是**位置敏感的**。交换两个 token 的位置，$W$ 矩阵的行列索引也会跟着变化——但 $W_{ij}$ 本身是固定的，不是输入动态的，所以它天然编码了**绝对位置之间的交互关系**。

这就是为什么 gMLP 不需要显式的位置编码。

### 7.2 对 Mask 的影响

在 NLP 中，padding tokens 是由于序列长度不够而填充的空 token。在 Transformer 中，这些 padding tokens 必须被 mask 掉——否则 attention 会错误地将它们纳入加权求和。

gMLP 的 SGU 同样不关心 padding。原因是 $W$ 矩阵是学习得到的——在训练过程中，网络会学会忽略 padding 位置对应的行和列。因为 padding token 在所有序列中的行为是一致的（都是零或特殊标记），网络可以自然而然地学到 $W_{ij}$ 中对 padding 位置赋予接近零的权重。

论文给出过一个有趣的观察：**在 MLM 预训练任务中，gMLP 的 SGU 不需要显式 mask，但 Transformer 必须 mask 掉 padding tokens。** 这侧面说明了 $W$ 矩阵可以学到哪些位置是"真实的序列位置"。

### 7.3 代价：序列长度固定

SGU 使用了一个 $n \times n$ 的全连接矩阵 $W$，这意味着它有一个隐含假设：**序列长度是固定的**。

如果序列长度可变，$W$ 矩阵的尺寸需要适应最长序列。对于短序列，SGU 可以通过 $W$ 的子矩阵（前 $k$ 行 $k$ 列）来操作，但这并不自然。对于长序列，$W$ 矩阵无法直接适应。

在 Transformer 中，attention 权重 $\alpha_{ij}$ 是动态计算的，天然支持任意长度的序列——只需要 mask 处理 padding 即可。

这是 gMLP 的一个固有限制：**它不原生支持变长序列或超长序列**。在实践中有一些变通方案：
1. 将长序列切分为固定长度的 segment
2. 对不同长度的序列使用不同的 $W$ 矩阵
3. 对 $W$ 矩阵做结构化约束（如 Toeplitz 矩阵），使其支持可变长度

论文在 NLP 实验中使用了固定长度为 128 的序列。

## 八、ImageNet 实验

### 8.1 实验设置

gMLP 在 ImageNet（ILSVRC 2012，1000 类，约 128 万张训练图 + 5 万张验证图）上进行了图像分类实验。在这个设置中，图像被划分为固定大小的 patches（$16 \times 16$），每个 patch 线性投影到 embedding 空间，然后输入到堆叠的 gMLP block 中。

gMLP 的视觉架构与 ViT/DeiT 几乎完全相同——唯一的区别是用 gMLP block 替代了 Transformer block。其他设计（patch embedding、class token、蒸馏等）保持不变。

### 8.2 模型变体

论文定义了两种规模的 gMLP 模型：

| 模型 | 层数 | 隐藏维度 d | FFN 维度 d_ffn | 参数 |
|------|------|-----------|---------------|------|
| gMLP-S | 30 | 256 | 1536 | 19M |
| gMLP-B | 30 | 512 | 3072 | 73M |

### 8.3 主要结果

gMLP 与 DeiT（采用 DeiT 的训练技巧，但使用 Transformer block）对比：

**gMLP-S vs DeiT-S：**
- gMLP-S（19M 参数）：79.6% top-1
- DeiT-S（22M 参数）：79.8% top-1
- 结果非常接近，gMLP 仅落后 0.2%

**gMLP-B vs DeiT-B：**
- gMLP-B（73M 参数）：81.6% top-1
- DeiT-B（86M 参数）：81.8% top-1
- gMLP 少了 13M 参数，精度仅落后 0.2%

这些结果是一个强有力的证据：**在视觉任务中，self-attention 不是取得高性能的必要条件。** gMLP 在 ImageNet 上达到了与 DeiT 几乎相同的精度，参数更少——这意味着它没有显著能力损失。

### 8.4 与 MLP-Mixer 的对比

在同一时间点，MLP-Mixer 在 ImageNet 上达到了约 76.4%（Mixer-B/16），明显低于 gMLP-B 的 81.6%。这个差距有多个原因：

1. **更先进的训练策略**：gMLP 使用了 DeiT 的训练策略（更强的高斯优化增强、正则化等），而 MLP-Mixer 使用的是相对基础的训练策略。
2. **架构差异**：gMLP 的 SGU 可能比 Mixer 的 token-mixing MLP 更有效——SGU 通过门控机制引入了非线性交互，而 Mixer 的 token-mixing MLP 只是两个线性层加一个 GELU。
3. **更好的初始化**：SGU 的精心初始化确保了训练的稳定性。

但需要注意，两者放在一起对比并不完全公平，因为训练策略不同。

## 九、NLP 实验：Masked Language Model 预训练

### 9.1 实验设置

gMLP 在 BERT 规模的 masked language model（MLM）预训练任务上进行了实验。设置如下：

- **数据**：BooksCorpus（约 800M 词）和 English Wikipedia（约 2,500M 词），与 BERT 的训练数据一致。
- **最大序列长度**：128（固定），这正好是 SGU 的 $W$ 矩阵尺寸。
- **训练步数**：1M 步 MLM 预训练。
- **batch size/bpe**：使用了与 BERT-base 大致相同的配置。

gMLP 的 NLP 版本与视觉版本的架构基本相同，但有一些小的调整：
- 去掉了 GELU（或保留，取决于具体配置）
- 输入为 subword token 序列，加上 embedding 层
- 输出为 masked token 的预测

### 9.2 主要实验结果

**预训练困惑度（perplexity）：**

论文报告，gMLP 在 MLM 预训练中达到了与 Transformer（BERT-base 架构）几乎相同的 perplexity。具体来说，经过 1M 步训练后：

- Transformer（BERT-base）：perplexity ≈ 4.28
- gMLP（相同规模）：perplexity ≈ 4.28

两者几乎完全持平。

这本身就是一个非常重要的结果：**在 language modeling 中，self-attention 也不是必需的。** gMLP 没有使用任何形式的 attention，但在 MLM pretraining 的任务上可以做到与 Transformer 一样好。

### 9.3 数据效率

论文还检查了 gMLP 在不同数据量下的表现。一个常见的担忧是：MLP 架构需要更多的数据才能达到与 Transformer 相同的效果——因为 attention 的归纳偏差（pairwise interaction）可能更有效。

实验结果否定了这个假设。在 1M 步的预训练过程中，随着训练数据的增加，gMLP 和 Transformer 的 perplexity 下降曲线几乎重合。这暗示 gMLP 的数据效率与 Transformer 相当——至少在 MLM 预训练这个任务上。

### 9.4 训练稳定性

在 MLM 预训练中，gMLP 的训练非常稳定。论文没有报告任何训练崩溃或不收敛的问题。这与 SGU 的初始化策略密切相关——$W \approx 0$ 确保了训练初期没有剧烈的空间混合，梯度可以顺畅回传。

这是一个对 Transformer 而言相对不重要的细节（Transformer 的 attention 初始化天生稳定），但对 MLP 架构至关重要。

## 十、NLP 下游任务微调

### 10.1 GLUE 基准

论文将 gMLP 模型在 MLM 预训练后，在 GLUE（General Language Understanding Evaluation）基准上进行微调和评估。GLUE 包含多种 NLP 任务，包括：
论文只报告了 SST-2（情感分类）、MNLI（自然语言推理）和 SQuAD（问答）的结果，没有扩展到完整的 GLUE benchmark。在 SST-2 和 MNLI 上，gMLP 与 BERT 表现相当。在 SQuAD v2.0 上，gMLP 需要更大的模型才能匹敌 BERT。

### 10.3 跨句对齐任务的差距

MNLI 是最能体现差距的任务。MNLI 要求模型判断一个句子是否蕴含（entailment）、矛盾（contradiction）或中立（neutral）于另一个句子。这需要模型做**跨句对齐**——将两个句子中的对应部分匹配起来并推理它们的关系。

Self-attention 天然适合这种任务，因为 attention 可以动态计算任意两个 token（包括跨句的 token）之间的交互权重。对于需要精确对齐两个句子的任务，attention 的"内容感知"能力可能提供了更多帮助。

SGU 中的静态 $W$ 矩阵无法实现这一点——无论两个句子之间的关系如何变化，$W_{ij}$ 都是一样的。它只能编码位置之间的固定交互模式，无法根据内容调整交互强度。

这正是 gMLP 最大的局限性所在：**在处理需要内容感知的跨句子/跨序列对齐任务时，静态的 token mixing 不如动态的 attention 灵活。**

### 10.4 混合模型：gMLP + 一点 Attention

论文做了一个非常有趣的实验：在 gMLP 的基础上，仅仅添加一个很小的 self-attention head（single head, 128 维），看看能否弥补跨句对齐的差距。

结果非常惊人：

- gMLP + 一个小的 attention head（128 dim, 1 head）：在 MNLI 上**超越**了完整 BERT-base。
- 也就是说，gMLP 加上极其有限的 attention 能力，就可以在跨句对齐任务上战胜拥有全尺寸 attention 的 BERT。

这揭示了一个关键结论：**模型的大部分能力来自 MLP 的容量（channel mixing + 空间门控），attention 只是在跨句对齐这样的特定任务上提供了关键的补充能力。** 而且只需要很少的 attention——一个 head 就足够了。

### 10.5 扩大模型能否弥补差距？

论文接着测试了另一种策略：不添加 attention，而是单纯扩大 gMLP 的规模，看看更大的模型能否在 MNLI 上追平 BERT。

结果显示：**是的，扩大模型可以缩小差距。** 当 gMLP 的参数量增加到远大于 BERT-base 时，它在 MNLI 上的表现可以接近甚至达到 BERT 的水平。

这与一些理解一致：静态 token mixing 的有效性受限于模型的容量——如果模型足够大，它可以通过其他参数间接模拟

## 十一、学习到的空间投影权重分析

### 11.1 视觉任务中的 $W$ 权重可视化

论文对 ImageNet 上训练的 gMLP 模型中的 $W$ 矩阵进行了可视化分析。由于 $W$ 是 $n \times n$ 的矩阵（在 ImageNet 中，一张 $224 \times 224$ 的图像被划分成 $14 \times 14 = 196$ 个 $16 \times 16$ 的 patches，所以 $n = 196$），$W$ 矩阵可以被可视化为 $14 \times 14$ 的二维权重图。

**观察 1：局部性（Locality）**

论文的可视化显示，$W$ 矩阵学到的权重表现出强烈的局部性——对角线附近的元素权重最大，远离对角线的元素权重接近零。也就是说，SGU 学到的空间交互模式是：**相邻的 patches 之间交互最强，相距较远的 patches 之间交互很弱。**

这种行为与卷积神经网络中的感受野非常类似。在 CNN 中，每个卷积核只关注局部区域，通过堆叠层来扩大感受野。gMLP 的 SGU 在单层中即表现出类似的局部性，只不过它在一个 $n \times n$ 的矩阵中实现了全局可达但局部重权重的交互模式。

**观察 2：空间不变性（Spatial Invariance）**

更深入的分析显示，$W$ 矩阵的权重具有近似空间不变性——在图像的不同区域，局部交互模式是相似的。例如，patch (0,0) 与其相邻 patch (0,1) 之间的交互权重，与 patch (5,10) 与其相邻 patch (5,11) 之间的交互权重类似。

这种空间不变性在视觉任务中非常重要，因为它使得模型能够泛化到图像中不同位置的相似结构。

### 11.2 NLP 任务中的 $W$ 权重分析

在 MLM 预训练任务中，$W$ 的表现形式有所不同。

由于序列是**一维的**（token 序列），$W$ 矩阵是一个 $n \times n$ 的二维矩阵。论文对它的分析揭示了一个有趣的现象：

**SGU 自动学习了近似的 Toeplitz 矩阵。**

Toeplitz 矩阵是指对角线为常数的矩阵——即 $W_{ij} = f(i - j)$，矩阵元素的值只取决于位置之间的**相对位移**，而不取决于绝对位置。

这意味着在 MLM 中，$W$ 自动学到了**移位不变性（shift invariance）**：token $i$ 和 token $j$ 之间的交互权重只取决于它们的距离 $|i - j|$，而不取决于 $i$ 和 $j$ 的具体位置。

这种性质在 NLP 中是有意义的：语言中的模式往往只依赖于局部上下文之间的距离，而不是绝对位置。一个 token 与它的前一个 token 之间的交互模式，在整个序列中的任何位置都是相似的。

### 11.3 Toeplitz 矩阵的近似程度

论文没有给出 Toeplitz 程度的定量度量，但从可视化结果看，$W$ 矩阵的接近对角线区域表现出明显的 Toeplitz 性质。这可能是由于 MLM 任务的特性——在 mask 语言模型中，每个 token 主要从它的局部上下文中获取信息，而这种局部交互在序列的不同位置应该是相似的。

### 11.4 意义

这些分析有重要的启示：

1. **Self-attention 的学习结果与 SGU 的静态投影很相似**：在 Transformer 训练好后，attention 权重的模式可能也表现出局部性和空间不变性——只不过 attention 是在输入条件下动态计算的。SGU 直接把这种模式固定在了参数中，用更简单的方式实现了类似的效果。

2. **归纳偏差的差异**：Transformer 没有局部性的归纳偏差，但可以通过数据学习到局部性。SGU 也没有局部性偏差，但它学到的结果与 Transformer 类似。这说明对于视觉和语言任务，局部性是数据本身的性质，而不是某种特定架构的特权。

3. **静态投影的局限性**：虽然静态投影在大多数任务上足够好，但在需要内容感知交互的任务（如跨句对齐）上，静态投影缺乏灵活性。这解释了 gMLP 在 MNLI 上略差的原因。

## 十二、Related Work：gMLP 在"后 Transformer"演化中的位置

### 12.1 直接前驱

**MLP-Mixer（Tolstikhin et al., 2021）**

2021 年 3 月发表的 MLP-Mixer 是第一个在 ImageNet 上证明"纯 MLP 也能做视觉任务"的工作。它的架构包含两个交替的 MLP：一个做 channel mixing（逐 token），一个做 token mixing（逐 channel）。

MLP-Mixer 的问题：
- Token-mixing MLP 是在每个 channel 上独立做 $n \times n$ 的全连接——相当于把 $n$ 个 token 的每个 channel 视为一个独立样本，做一次全连接。这意味着 token-mixing MLP 无法在 channel 之间共享信息。
- 训练需要更多的技巧和更长的周期。
- 在 NLP 上未被验证。

gMLP 的改进：
- 用 SGU 替代了 token-mixing MLP，引入了门控机制，使 channel 之间可以共享信息。
- 将 token-mixing 整合到 channel-mixing 的中间，形成了一个更紧凑的 block。
- 在 NLP 上也做了验证。

**ResMLP（Touvron et al., 2021）**

ResMLP 进一步简化了 MLP-Mixer：去掉了 LayerNorm 的可学习参数，使用仿射变换替代。在视觉任务上取得了不错的结果。

ResMLP 的问题：
- 同样只验证了视觉任务。
- 架构的可扩展性没有充分探索。

### 12.2 同期工作

在 gMLP 发表的同期，还有其他探索"无 attention Transformer"的工作：

**FNet（Lee-Thorp et al., 2021, Google）**

FNet 用傅里叶变换替代 self-attention，在空间维度做 token mixing。它证明了离散傅里叶变换（DFT）这个完全无参数、无学习的方法也可以在大部分任务上达到与 Transformer 相近的结果。

FNet 的思路更加激进：它根本不需要学习空间交互——傅里叶变换已经足够了。FNet 的 GPU 效率非常高（FFT 是高度优化的），但在某些任务上（如 MNLI）的损失比 gMLP 更大。

**Synthesizer（Tay et al., 2020, Google）**

Synthesizer 探索了"可学习的 attention 权重"——去掉 QKV 的点积计算，直接用可学习的参数矩阵（或低秩分解）生成 attention 权重。它本质上也是去掉 attention 的动态性。

Synthesizer 观察到，在某些任务中，**随机初始化但训练好的 static attention 效果几乎与标准 attention 一样好**——这与 gMLP 的结论一致。

### 12.3 技术谱系树

```text
"Transformer 的 success 到底来自哪里？"
         │
         ├── "来自 attention" → 改进 attention（大部分 Transformer 改进工作）
         │
         └── "可能不来自 attention" → 去掉 attention
              │
              ├── Synthesizer (Tay et al., 2020) — 可学习 attention 权重
              │
              ├── MLP-Mixer (Tolstikhin et al., 2021) — 两个 MLP 做两种 mixing
              │     └── ResMLP (Touvron et al., 2021) — 简化 LayerNorm
              │           ↓
              ├── gMLP (Liu et al., 2021) — 用 SGU 做空间混合 ★ 本篇
              │     └── 混合路线: gMLP + 少量 attention
              │           ↓
              ├── FNet (Lee-Thorp et al., 2021) — 傅里叶变换做 mixing
              │
              └── 后续影响
                    ├── ConvMixer (Trockman & Kolter, 2022) — 纯卷积
                    ├── MetaFormer (Yu et al., 2022) — Transformer 的通用架构框架
                    └── RWKV (Peng et al., 2023) — 线性 attention + 门控
```

### 12.4 gMLP 的独特贡献

与其他"去掉 attention"的工作相比，gMLP 的独特贡献在于：

1. **跨模态验证**：同时证明了在视觉和 NLP 上都能工作。MLP-Mixer 只在视觉上验证，Synthesizer 只在 NLP 上验证。gMLP 是第一个在两个领域都证明"attention 不是必须的"的工作。

2. **更好的架构设计**：SGU 比 MLP-Mixer 的 token-mixing MLP 更优雅——门控机制提供了更强的表示能力，channel split 实现了效率优化。

3. **丰富的消融分析**：论文做了大量消融实验，包括不同初始化的影响、不同中间维度的影响、训练动态分析等，使得结论更加坚实。

4. **"混合模型"的启示**：gMLP + 一点 attention 超越完整 Transformer 的发现，为后续的混合架构研究提供了灵感。

```text
  └── MetaFormer (Yu et al.) — 抽象为通用框架
```

2021 年上半年是"MLP 是否可以替代 Transformer"辩论的高峰期。gMLP 在这个辩论中扮演了关键角色：它用更严密的实验设计（同时在视觉和 NLP 上验证）给出了一个更明确的答案——**可以，但在跨句对齐任务上需要少量 attention 辅助。**

### 13.2 范式分类

- **方法类型**：架构改进（核心模块替换）
- **范式意义**：**范式级工作**——从一个之前没人认真质疑的基本假设（self-attention 对 Transformer 的成功是必需的）中提出了一个反例
- **路线角色**：**主干路线节点**——非 attention mixing 这条路线的关键工作，连接了 MLP-Mixer 和后续的混合架构
- **影响范围**：NLP + CV + 架构设计理论

gMLP 的范式意义在于它不仅仅是提出了一个新结构——它提出的是一个**反事实论证**（counterfactual argument）：如果 self-attention 真的是 Transformer 成功的核心原因，那么去掉它应该导致性能显著下降。但实验证明，去掉 self-attention 后性能基本保持不变——这意味着 Transformer 成功的因素比"attention"这个词要复杂得多。

### 13.3 对后续工作的影响

gMLP 直接影响了多个后续工作：

1. **MetaFormer（Yu et al., 2022）**：将 Transformer 抽象为"token mixer + channel mixer"的通用框架，指出 token mixer 可以是任何东西（attention、SGU、FFT、甚至池化），而真正重要的是 MetaFormer 这个通用架构。

2. **ConvMixer（Trockman & Kolter, 2022）**：用深度可分离卷积替代 token mixer，本质上与 SGU 做着同样的事——只是用卷积矩阵替代了全连接矩阵。

3. **RWKV（Peng et al., 2023）**：在 NLP 中结合了线性 attention 和门控机制，可以看作是 gMLP+SGU 与 attention 的混合延续。

## 十四、局限性

### 14.1 计算复杂度

SGU 的 $W$ 矩阵是 $n \times n$ 的，计算 $WZ_2$ 的复杂度是 $O(n^2 \cdot d_{ffn}/2)$。这与标准 self-attention 的 $O(n^2 \cdot d)$ 在同一量级。

如果 $n$ 很大（例如 $n > 1024$），SGU 的计算开销会变得很大。虽然它可以利用成熟的矩阵乘法库（cuBLAS 等），但复杂度始终是平方级。

这意味着 gMLP 不适用于超长序列任务——例如需要处理 4096 或更长 token 的文档理解任务。对于这些任务，Performer 或 Longformer 之类的线性或近似线性方法更合适。

### 14.2 序列长度固定

这是 SGU 最根本的限制。$W \in \mathbb{R}^{n \times n}$ 是一个固定大小的矩阵，无法适应不同的序列长度或动态顺序。

在实践中，可以通过以下方式绕过：
- 使用固定长度的输入（如 128 或 512）
- 将长序列切分为 segment
- 对 $W$ 做特定结构化约束（如 Toeplitz 变体）

但这些都不是原生解决方案。Transformer 的 attention 可以无缝处理不同长度的输入，这是它的一个重要优势。

### 14.3 跨句对齐任务的不足

论文实验明确显示，gMLP 在需要精确跨句对齐的任务（MNLI）上略逊于 Transformer。差距不大（约 0.8-0.9%），但确实存在。

这种差距的原因是：**SGU 的 $W$ 是静态的，难以实现"内容感知"的跨句交互。** 对于两个句子融合的任务，attention 可以动态计算两个句子中任意位置的匹配权重，而 SGU 只能使用固定的空间投影。

这个局限性在"混合模型"实验中得到了印证——加一个小 attention head 就可以弥补差距。所以这不是 gMLP 框架的本质缺陷，而是纯静态 mixing 的固有限制。

### 14.4 实验覆盖范围有限

尽管 gMLP 在 ImageNet、MLM、GLUE 等主要基准上做了充分实验，但它的实验范围仍有限制：

- **跨句对齐任务表现不如 Transformer。** gMLP 在需要跨句对齐的任务（如 MNLI）上比 Transformer 差约 0.8%，但通过增加模型规模或添加微量的 self-attention 可以弥补。
- **未在生成任务上验证**：论文主要做的是理解和分类任务（MLM、GLUE、ImageNet），未覆盖机器翻译、文本生成等自回归生成任务。在生成任务中，attention 的作用可能更加关键。
- **未在更长序列上验证**：所有 NLP 实验使用最大 128 的序列长度，不足以反映长文档理解场景。

这些实验的空白意味着 gMLP 的结论——self-attention 不是必须的——可能在某些场景下成立，但在其他场景下可能不成立。

### 14.5 泛化性的疑问

gMLP 的 $W$ 矩阵在训练过程中学习到的是一种"固定的空间交互模式"。这种模式对于与训练数据分布相似的输入有效，但对于分布外（OOD）的输入是否仍然有效？当输入的空间结构发生重大变化时（例如图像中出现了一个高度异常的位置分布），SGU 可能无法有效适应。

与此相比，Transformer 的 attention 是内容感知的——它在推理时可以动态调整交互模式，以适应新的输入分布。这是静态 mixing 的一个潜在弱点。

## 十五、总结

### 15.1 核心贡献回顾

gMLP 的核心贡献是：**提出了 Spatial Gating Unit（SGU），用线性门控机制替代 self-attention 做空间维度上的 token mixing，并且在视觉和 NLP 两个领域证明了该架构可以在大多数任务上与 Transformer 达到相当的性能。**

这个结论的意义在于：**self-attention 不是 Transformer 成功的关键。** Transformer 的大部分能力来自于它的整体架构设计——MLP 的容量、残差连接、LayerNorm 的堆叠——而不是来自于 attention 提供的"内容感知的 token 间交互"。

### 15.2 SGU vs Self-Attention

```text
Self-Attention:
  - QKV 分解 → 点积相似度 → softmax → 加权求和
  - 权重是动态计算的（内容感知）
  - 天然支持变长序列
  - O(n²d) 复杂度

SGU:
  - Channel split → 一半经线性投影 → 门控乘法
  - 权重是静态学习的（位置感知）
  - 序列长度需固定
  - O(n²d_ffn) 复杂度（d_ffn 约为 d 的 3-6 倍，所以 SGU 的复杂度高于 standard attention
```

两者最大的区别是**动态 vs 静态**。Attention 的交互权重随输入变化，SGU 的权重是固定的。但在大多数任务上，这种动态性带来的优势很小——模型大部分时候只需要一个不错的静态交互模式。

### 15.3 关键数据点

- **ImageNet**：gMLP-B 81.6% vs DeiT-B 81.8%，参数更少（73M vs 86M）
- **MLM Pretraining**：perplexity 几乎完全持平
- **GLUE**：大部分任务持平，MNLI 略差（~0.8-0.9%）
- **gMLP + 1 attention head**：在 MNLI 上超越完整 BERT-base

这些数据共同勾勒出一条清晰的结论：**Transformer 的精髓在 MLP，不在 attention。**

### 15.4 启示

gMLP 给了社区三个重要的启示：

**第一，"注意力"这个概念可能被过度浪漫化了。** self-attention 被称为"让模型学会关注重要内容"的优雅机制。但 gMLP 表明，大部分时候你只需要一个静态的、学习到的空间交互模式，就足够了。这让人怀疑，Transformer 的成功是否真的来自于"attention is all you need"中描述的"内容感知交互"，还是只是 MLP 的大量参数起了作用。

**第二，"无 attention"架构是一个值得探索的方向。** gMLP 之后，有更多工作在探索"non-attention mixers"——例如 ConvMixer（卷积）、FNet（傅里叶变换）、PoolFormer（平均池化）等。这些工作共同指向了一个结论：**很多简单的操作都可以做 token mixing，效果并不比 attention 差。**

**第三，混合架构可能比纯架构更好。** gMLP + 少量 attention 超越完整 Transformer 的发现，暗示了混合架构的潜力。用少量的 attention（一个 head、128 维）补充静态 mixing 的不足，可以在大部分任务上达到更好的效率-精度平衡。这个思路影响了后续的多种混合架构探索。

### 15.5 更广泛的视角

gMLP 不是要取代 Transformer——它是要回答一个更根本的问题："Transformer 的成功究竟来自哪里？"

到 2023-2024 年，回看 gMLP 及其同期工作，学界对这个问题有了更清晰的认识：**Transformer 的强大来自多个因素的组合，包括残差连接的稳定性、Scale 的可预测性、计算效率（矩阵乘法的高度并行化）、以及适合 Transformer 的训练技巧（AdamW、学习率调度、数据增强等）。Self-attention 只是其中的一个组件，而且是可替换的。**

这个认识深刻影响了后续的架构设计。例如：
- **RWKV** 和 **Mamba** 在语言模型中探索了非 attention 的 token mixing 方式
- **Hyena** 和 **S4** 等结构化状态空间模型用更复杂的线性算子替代 attention
- **Gemma**、**Llama** 等主流模型虽然在保留 attention，但越来越重视 MLP 层的设计

gMLP 在这个"后 Transformer"的多元演化中，扮演了一个关键的反问角色——不是发明了最好的架构，而是帮助整个领域理解了自己正在使用的工具。
