**论文：** [arXiv](https://arxiv.org/abs/2104.14294) · 作者：Mathilde Caron, Hugo Touvron, Ishan Misra, Hervé Jégou, Julien Mairal, Piotr Bojanowski, Armand Joulin · 年份：2021

## 一、先搞清楚坑在哪

在 2020-2021 年，视觉领域的自监督学习（Self-Supervised Learning, SSL）已经有了显著进展。主流方法可以归为两类：

1. **对比学习（Contrastive Learning）**：如 SimCLR [Chen et al., 2020]、MoCo [He et al., 2020]，通过拉近同一图像的不同增强视图（positive pairs），推远不同图像的视图（negative pairs）来学习表征。这些方法在 ImageNet 上已经能达到接近监督学习的性能。

2. **非对比方法（Non-contrastive methods）**：如 BYOL [Grill et al., 2020]、SimSiam [Chen & He, 2021]，不依赖负样本，而是通过预测器（predictor）或 stop-gradient 机制防止坍塌。

这些方法都基于 CNN 骨干网络（ResNet 等）。但与此同时，Vision Transformer（ViT）[Dosovitskiy et al., 2021] 刚被提出不久，在监督学习上展现出了与 CNN 竞争甚至超越的性能。一个自然的问题是：**自监督学习在 ViT 上表现如何？**

## 二、已有方法在 ViT 上的真正问题

乍看之下，直接把现有的自监督方法（如 SimCLR、BYOL）套用到 ViT 上似乎可行。但实验表明有几个问题：

1. **ViT 缺乏 CNN 的归纳偏置（inductive bias）**：CNN 有局部性（locality）和平移等变性（translation equivariance），这让它天然适合视觉任务。ViT 的 Transformer 架构是全局的，patch embedding 后全靠自注意力，没有这些先验知识。

2. **对比学习在 ViT 上效果一般**：直接使用对比学习训练 ViT（如 MoCo v3 [Chen et al., 2021] 所做的），性能不如在 CNN 上。ViT 似乎难以从对比目标中学到好的视觉特征。

3. **ViT 的自注意力机制需要大量数据**：ViT 在 ImageNet-1K 上训练时，如果不使用强正则化或大规模数据，性能不如 ResNet。自监督场景下数据量有限，ViT 更难训练。

4. **没有利用 ViT 的独特优势**：现有方法把 ViT 当作"另一个特征提取器"，没有思考 ViT 的架构特性（如 patch 级表征、自注意力图）能在自监督中带来什么额外价值。

这里有个关键：**DINO 的作者意识到，自监督学习的目标不仅仅是学习好的全局特征（global features），更重要的是学习到对下游任务有用的结构化表征（structured representations）**。ViT 的 patch 级输出和注意力机制恰好提供了这种可能性。

## 三、DINO 的核心思路

DINO 的核心思想出奇地简单：**用自蒸馏（self-distillation）来训练 ViT，不需要负样本也不需要聚类**。

具体来说：
- 一个图像的两个增强视图（一个全局视图 + 多个局部视图）分别送入两个网络：**student** 和 **teacher**。
- student 网络试图匹配 teacher 网络的输出（soft label）。
- teacher 网络通过 student 参数的指数移动平均（EMA）更新。
- 输出是一个 d 维的特征向量，通过 softmax 转化为概率分布。

乍看之下，这和 BYOL、SimSiam 非常相似。但 DINO 有两个关键差异：
1. **输出是分类头（classification head）的 logits，而不是投影头（projection head）的嵌入向量**。
2. **使用 centering 和 sharpening 来防止坍塌**，而不是 predictor 或 stop-gradient。

有意思的是，DINO 并没有专门为 ViT 设计，但实验发现它在 ViT 上展现出了一些 CNN 上不存在的"涌现性质"（emergent properties）——最惊艳的是：**自监督 ViT 的注意力图会自动分割出前景物体**，无需任何语义标签。

## 四、网络架构详解

DINO 的整体架构是一个师生蒸馏框架，如图 1 所示。

![图1: DINO 整体架构。来自论文 Figure 1。两个视图（global + local）通过 student 和 teacher 网络，输出经过 softmax 的概率分布，通过交叉熵损失进行匹配。teacher 参数由 student 的 EMA 更新。](https://ar5iv.labs.arxiv.org/html/2104.14294/assets/figs/fig1.png)

**图 1：DINO 的自蒸馏框架。** 输入图像经过数据增强生成两个全局视图（分辨率 224×224）和多个局部视图（分辨率 96×96）。所有视图都送入 student 网络，只有全局视图送入 teacher 网络。student 的输出试图匹配 teacher 的输出。teacher 的参数 $θ_t$ 通过 student 参数 $θ_s$ 的 EMA 更新：$θ_t ← λθ_t + (1-λ)θ_s$。

### 前向过程详解

**Step 1: 数据增强与视图生成**

输入图像 $x$ 经过随机数据增强，生成多组视图：
- 2 个全局视图（global views）：分辨率 224×224，覆盖图像的 50%-100% 区域
- 多个局部视图（local views）：分辨率 96×96，覆盖图像的 10%-50% 区域

**为什么这样设计？** 全局视图提供整体语义信息，局部视图提供细粒度细节。让 student 从局部视图预测 teacher 从全局视图得到的输出，迫使模型学习"局部-全局"对应关系。

**Step 2: 特征提取**

每个视图独立通过 ViT 网络（student 或 teacher）。ViT 的处理流程：

1. **Patch Embedding**：将 $224×224×3$ 的图像分割成 $14×14$ 个 $16×16$ 的 patch，每个 patch 线性投影到 $D$ 维空间。输出：$196 × D$（196 = 14×14 个 patch）。

2. **Class Token 与位置编码**：在 patch 序列前拼接一个可学习的 [CLS] token，加上位置编码。输出：$197 × D$（197 = 196 patches + 1 [CLS] token）。

3. **Transformer Encoder**：L 层 Transformer 块，每层包含：
   - Multi-Head Self-Attention（MSA）：$O(N^2D)$ 复杂度，N=197
   - MLP：两个全连接层，中间有 GELU 激活
   - Layer Normalization 和 Residual Connection

4. **输出**：取 [CLS] token 的输出（$D$ 维向量）作为全局表征。

**Step 3: 投影头（Projection Head）**

[CLS] token 的输出经过一个 3 层 MLP（隐藏层 2048 维，L2 归一化），输出 $K$ 维 logits（论文中 K=65536）。

**为什么要用投影头？** 直接将 [CLS] token 的输出用于蒸馏可能维度太高、噪声太大。投影头将特征映射到更低维的空间，同时也增加了非线性变换能力。

**Step 4: Softmax 与损失计算**

student 和 teacher 的输出分别经过 softmax：

$$P_s(x) = \frac{\exp(g_{\theta_s}(x) / \tau_s)}{\sum_{k=1}^K \exp(g_{\theta_s}(x)_k / \tau_s)}$$

$$P_t(x) = \frac{\exp(g_{\theta_t}(x) / \tau_t)}{\sum_{k=1}^K \exp(g_{\theta_t}(x)_k / \tau_t)}$$

其中 $\tau_s$ 和 $\tau_t$ 分别是 student 和 teacher 的温度参数。$\tau_s=0.1$（较低，输出更"尖锐"），$\tau_t=0.04$（更低，输出更"尖锐"）。

损失函数是交叉熵：

$$\min_{\theta_s} \sum_{x \in \{x_1^g, x_2^g, ..., x_1^l, x_2^l, ...\}} H(P_t(x^g), P_s(x))$$

其中 $x^g$ 是全局视图，$x$ 可以是全局或局部视图。注意：**只有全局视图送入 teacher**。

**Step 5: Teacher 更新**

teacher 参数不通过梯度更新，而是通过 EMA：

$$\theta_t ← \lambda \theta_t + (1-\lambda) \theta_s$$

其中 $\lambda$ 从 0.996 开始，在训练过程中逐渐增加到 1（cosine schedule）。

### 防止坍塌的机制

DINO 使用两种机制防止所有输出坍塌到同一个值：

1. **Centering（中心化）**：从 teacher 的 logits 中减去一个动量更新的中心向量 $c$：

   $$g_{\theta_t}(x) \leftarrow g_{\theta_t}(x) - c$$

   $$c ← m c + (1-m) \frac{1}{B} \sum_{i=1}^B g_{\theta_t}(x_i)$$

   其中 $m=0.9$，$B$ 是 batch size。Centering 防止某一维度的 logits 一直过大。

2. **Sharpening（锐化）**：使用低温度参数 $\tau_t=0.04$ 使 teacher 的 softmax 输出更"尖锐"（更接近 one-hot）。Sharpening 防止所有类别的概率太均匀。

**为什么需要两者结合？** 单独使用 centering 会导致输出均匀分布（所有维度相等），单独使用 sharpening 会导致输出塌缩到单一维度。两者结合达到平衡：centering 防止 collapse，sharpening 防止 uniform。

## 五、核心创新点

### 创新点 1：自蒸馏框架（Self-Distillation）

**(a) 痛点与动机**

在自监督学习中，防止表征坍塌是一个核心问题。对比学习使用负样本（SimCLR、MoCo），非对比方法使用 predictor（BYOL）或 stop-gradient（SimSiam）。但这些方法都需要精心设计的网络结构（如 predictor 需要特定的初始化）或超参数（如负样本数量）。

DINO 的作者观察到：**自蒸馏本身就是一个天然的自监督框架**。如果让两个相同架构的网络互相学习，只要防止它们坍塌，就能学到有用的表征。

**(b) 方案细节**

DINO 的自蒸馏框架：
- student 和 teacher 网络架构完全相同（ViT + projection head）
- teacher 参数通过 student 的 EMA 更新
- 只有全局视图送入 teacher，所有视图送入 student
- 使用 centering + sharpening 防止坍塌

**(c) 为什么有效**

自蒸馏的有效性可以从两个角度理解：
- **信息瓶颈（Information Bottleneck）**：student 从局部视图预测 teacher 从全局视图得到的输出，迫使模型学习"局部→全局"的映射，这需要捕获图像的语义结构。
- **正则化效应**：EMA 更新的 teacher 提供更稳定的目标分布，类似于"教师信号"的平滑版本。

论文中的消融实验（Table 3）显示：**去除蒸馏框架（用对比学习替代）导致性能下降 3-5%**。

**(d) 与 Related Work 的关系**

DINO 的自蒸馏框架与 BYOL [Grill et al., 2020] 和 SimSiam [Chen & He, 2021] 密切相关：
- BYOL 也使用 EMA teacher，但输出是嵌入向量，且使用 predictor 防止坍塌
- SimSiam 使用 stop-gradient，没有 EMA
- DINO 的关键区别是：输出是分类 logits（不是嵌入），且使用 centering + sharpening

**(e) 如果去掉会怎样**

Table 3 的消融实验：
- 去掉蒸馏（用 SimCLR 式对比损失）：ImageNet linear probe 从 72.7% 降到 69.8%
- 固定 teacher（不用 EMA）：性能降至 68.5%
- 去掉局部视图：性能降至 70.1%

### 创新点 2：Centering + Sharpening 的平衡机制

**(a) 痛点与动机**

非对比方法（BYOL、SimSiam）使用 predictor 或 stop-gradient 来防止坍塌。但 DINO 的作者发现，在分类头输出（logits）上，简单的 centering 和 sharpening 就能达到同样的效果。

这里有个关键观察：**在对比学习框架中，防止坍塌通常需要不对称的网络结构（teacher/student 不同或 predictor）。而在蒸馏框架中，如果输出是概率分布，centering 和 sharpening 就足够了。**

**(b) 方案细节**

Centering：从 teacher logits 中减去动量更新的均值。
Sharpening：使用低温度参数使 softmax 输出更尖锐。

两者必须同时使用：
- 只有 centering → 输出均匀分布（所有维度相等）
- 只有 sharpening → 输出塌缩到单一维度

**(c) 为什么有效**

直觉上，centering 防止了"某个维度一直很大"（避免输出偏向某个类），sharpening 防止了"所有维度都一样大"（避免输出均匀）。两者互相制衡，达到一个动态平衡。

论文中的消融实验（Table 2）：
- 只有 centering：性能 66.5%（坍塌到均匀分布）
- 只有 sharpening：性能 59.3%（坍塌到单一维度）
- 两者结合：性能 72.7%

**(d) 与 Related Work 的关系**

Centering 的想法来自 DeepCluster [Caron et al., 2018] 和 SwAV [Caron et al., 2020]，这些方法在聚类分配中使用 centering 来平衡聚类大小。DINO 的创新在于将其应用到蒸馏框架中。

Sharpening 类似于知识蒸馏 [Hinton et al., 2015] 中的温度缩放，但 DINO 将其用于防止坍塌而非控制知识迁移的"软硬"程度。

**(e) 如果去掉会怎样**

Table 2 清晰展示了：去掉任何一个，性能大幅下降（66.5% 和 59.3% vs 72.7%）。两者缺一不可。

### 创新点 3：利用 ViT 的 patch 级输出进行自监督

**(a) 痛点与动机**

之前的所有自监督方法（SimCLR、BYOL、MoCo、SwAV）都只使用全局特征（global features），即 [CLS] token 或全局平均池化后的特征。但 ViT 的一个独特优势是：**它天然输出 patch 级表征**（196 个 patch 的 token 输出），这些表征包含空间信息。

**(b) 方案细节**

DINO 在训练时只使用 [CLS] token 的输出进行损失计算，但实验发现，**训练后的 ViT 的自注意力图（self-attention maps）会自动分割出前景物体**，如图 2 所示。

![图2: DINO 训练后的 ViT 自注意力图。来自论文 Figure 3。每行显示不同图像，每列是不同注意力头（head）的注意力图。可以看到，某些注意力头自动聚焦在前景物体上。](https://ar5iv.labs.arxiv.org/html/2104.14294/assets/figs/fig3.png)

**图 2：DINO 训练后的 ViT-S/8 的注意力图。** 每个图像显示 4 个注意力头的输出（从最后一层提取）。可以看到，某些头（如第 1 列第 2 行）自动聚焦在主体物体上，而另一些头关注背景。这是完全无监督的——模型从未见过任何语义标签。

**(c) 为什么有效**

论文给出了一种解释：**自蒸馏迫使模型学习"局部→全局"的对应关系**。当 student 从局部视图预测 teacher 从全局视图得到的输出时，模型必须理解哪些 patch 属于同一个语义实体。这种理解最终体现在自注意力图上：某些注意力头学会关注前景物体，另一些关注背景。

更具体地说，DINO 的局部视图（96×96，覆盖 10%-50% 区域）让 student 只能看到图像的一部分。要正确预测 teacher 的全局输出，student 需要推断"我看到的这部分属于什么物体"，这迫使注意力机制学会区分前景和背景。

**(d) 与 Related Work 的关系**

之前的工作中，iGPT [Chen et al., 2020] 在 ViT 上进行自回归预测，但需要大量计算。CLIP [Radford et al., 2021] 使用文本监督，不是纯视觉自监督。DINO 是第一个发现纯自监督 ViT 能产生语义分割性质的工作。

**(e) 如果去掉会怎样**

这里的关键不是"去掉"这个创新，而是**这是 DINO 训练后的涌现性质**，不是显式设计的。如果使用 CNN 骨干（如 ResNet-50），DINO 也能学到好的特征，但不会产生这种语义分割效果。这说明 ViT 的架构（自注意力 + patch 级表征）与 DINO 的自蒸馏框架之间存在某种协同效应。

## 六、公式详解

### 公式 1：蒸馏损失

$$L = -\sum_{k=1}^K P_t(x^g)_k \log P_s(x)_k$$

其中：
- $K=65536$：输出维度（分类头的大小）
- $x^g$：全局视图（仅送入 teacher）
- $x$：任意视图（全局或局部，送入 student）
- $P_t$：teacher 的 softmax 输出
- $P_s$：student 的 softmax 输出

**(a) 符号定义**
- $P_t(x^g)_k$：teacher 对全局视图 $x^g$ 的第 $k$ 维输出概率
- $P_s(x)_k$：student 对任意视图 $x$ 的第 $k$ 维输出概率

**(b) 公式来源**

这是标准的知识蒸馏损失 [Hinton et al., 2015]，即交叉熵损失。区别在于：teacher 的目标不是真实标签，而是 teacher 自身对全局视图的输出。

**(c) 推导过程**

交叉熵损失 $H(p,q) = -\sum_k p_k \log q_k$。这里 $p = P_t(x^g)$（teacher 的输出作为"目标分布"），$q = P_s(x)$（student 的输出作为"预测分布"）。

**(d) 直觉理解**

这个损失让 student 的输出概率分布尽可能接近 teacher 的分布。由于 teacher 只看到全局视图，而 student 看到局部视图，student 必须从局部信息推断全局语义。

**具体例子**：假设一张狗的图像。teacher 看到整只狗，输出概率分布 $P_t = [0.8, 0.1, 0.05, 0.05]$（维度 1 对应"狗"）。student 只看到狗的耳朵，输出 $P_s = [0.3, 0.4, 0.2, 0.1]$。损失计算：
- 项 1：$0.8 \times \log(0.3) = 0.8 \times (-1.204) = -0.963$
- 项 2：$0.1 \times \log(0.4) = 0.1 \times (-0.916) = -0.092$
- 项 3：$0.05 \times \log(0.2) = 0.05 \times (-1.609) = -0.080$
- 项 4：$0.05 \times \log(0.1) = 0.05 \times (-2.303) = -0.115$
- 总和：$-1.250$，取负得 $L=1.250$

为了降低损失，student 需要让"狗"维度的概率更高。

**(e) 边界情况**

- 当 $P_t$ 是 one-hot（某个维度为 1，其余为 0）时，退化为标准分类交叉熵
- 当 $P_t$ 是均匀分布时，损失对所有维度同等惩罚，模型无法学到有用信息（sharpening 防止这种情况）

### 公式 2：Teacher 的 EMA 更新

$$\theta_t ← \lambda \theta_t + (1-\lambda) \theta_s$$

其中 $\lambda$ 从 0.996 开始，cosine schedule 增加到 1。

**(a) 符号定义**
- $\theta_t$：teacher 网络参数
- $\theta_s$：student 网络参数
- $\lambda$：动量系数

**(b) 公式来源**

EMA 更新在 BYOL [Grill et al., 2020] 中首次用于自监督学习。DINO 直接沿用。

**(c) 推导过程**

初始：$\lambda=0.996$，此时 teacher 参数 99.6% 来自自身、0.4% 来自 student。
随着训练进行，$\lambda$ 逐渐增加到 1，teacher 停止更新。

**(d) 直觉理解**

EMA 让 teacher 的变化比 student 慢得多。这提供了稳定的训练目标——student 追赶的是"过去一段时间的平均版本"，而不是瞬息万变的当前版本。

**具体例子**：假设 student 在 step 100 时参数为 $\theta_s^{(100)}$，teacher 参数为 $\theta_t^{(100)}$。更新后：
$$\theta_t^{(101)} = 0.996 \times \theta_t^{(100)} + 0.004 \times \theta_s^{(100)}$$
teacher 只向 student 方向移动了 0.4%，变化非常平滑。

**(e) 边界情况**

- $\lambda=0$：teacher 直接复制 student，相当于自训练（self-training），容易坍塌
- $\lambda=1$：teacher 完全不更新，相当于固定教师网络（知识蒸馏中的常见做法），性能下降（Table 3 显示下降 4.2%）
- 论文使用 cosine schedule 让 $\lambda$ 从 0.996 到 1，早期 teacher 快速适应，后期稳定

### 公式 3：Centering 更新

$$g_{\theta_t}(x) \leftarrow g_{\theta_t}(x) - c$$

$$c ← m c + (1-m) \frac{1}{B} \sum_{i=1}^B g_{\theta_t}(x_i)$$

其中 $m=0.9$，$B$ 是 batch size。

**(a) 符号定义**
- $g_{\theta_t}(x)$：teacher 对输入 $x$ 的 logits 输出（$K$ 维向量）
- $c$：中心向量（$K$ 维），动量更新
- $m$：动量系数

**(b) 公式来源**

Centering 在 DeepCluster [Caron et al., 2018] 和 SwAV [Caron et al., 2020] 中用于聚类分配平衡。DINO 将其直接应用于 logits。

**(c) 推导过程**

每步训练：
1. 计算当前 batch 所有 teacher 输出的均值：$\bar{g} = \frac{1}{B} \sum_{i=1}^B g_{\theta_t}(x_i)$
2. 更新中心：$c \leftarrow 0.9c + 0.1\bar{g}$
3. 对每个 teacher 输出减去中心：$g_{\theta_t}(x_i) \leftarrow g_{\theta_t}(x_i) - c$

**(d) 直觉理解**

Centering 防止 logits 的某个维度持续偏大。如果某个维度的 logits 一直很大（比如所有图像都倾向于被分到"背景"类），centering 会逐渐减去这个偏置。

**具体例子**：假设 $K=3$，batch 中 4 张图像的 teacher logits 为：
- 图像 1：[10, 2, 1]
- 图像 2：[8, 3, 2]
- 图像 3：[9, 1, 3]
- 图像 4：[11, 2, 2]

均值 $\bar{g} = [9.5, 2.0, 2.0]$
假设 $c$ 初始为 0，更新：$c = 0.9 \times 0 + 0.1 \times [9.5, 2.0, 2.0] = [0.95, 0.2, 0.2]$
减去中心后：
- 图像 1：[9.05, 1.8, 0.8]
- 图像 2：[7.05, 2.8, 1.8]
- 图像 3：[8.05, 0.8, 2.8]
- 图像 4：[10.05, 1.8, 1.8]

第 1 维的偏置被部分移除。

**(e) 边界情况**

- $m=0$：中心直接等于当前 batch 均值，不稳定
- $m=1$：中心永不更新，相当于固定偏置
- 当 batch size 很小时，均值估计有噪声，$m$ 较大（0.9）可以平滑

### 公式 4：Softmax 与温度缩放

$$P(x)_k = \frac{\exp(g(x)_k / \tau)}{\sum_{j=1}^K \exp(g(x)_j / \tau)}$$

其中 $\tau_s=0.1$（student），$\tau_t=0.04$（teacher）。

**(a) 符号定义**
- $g(x)_k$：第 $k$ 维 logits
- $\tau$：温度参数

**(b) 公式来源**

标准 softmax 函数，温度参数来自知识蒸馏 [Hinton et al., 2015]。

**(c) 推导过程**

温度 $\tau$ 控制 softmax 输出的"尖锐程度"：
- $\tau \to 0$：softmax 趋近于 argmax（one-hot）
- $\tau \to \infty$：softmax 趋近于均匀分布
- $\tau=1$：标准 softmax

**(d) 直觉理解**

DINO 使用 $\tau_t=0.04$（非常低）让 teacher 输出尖锐的分布（接近 one-hot），$\tau_s=0.1$（相对较高）让 student 输出更平滑的分布。这迫使 student 学习 teacher 的"确定性"判断。

**具体例子**：假设 logits 为 [10, 5, 0]：
- $\tau=0.04$：softmax 输出 ≈ [1.0, 0.0, 0.0]（几乎 one-hot）
- $\tau=0.1$：softmax 输出 ≈ [0.993, 0.007, 0.000]（仍然尖锐但稍平滑）
- $\tau=1$：softmax 输出 ≈ [0.993, 0.007, 0.000]（注意：这里 logits 差异大，温度影响不大）

实际上，logits 值通常在 [-5, 5] 范围，低温度的影响更明显。

**(e) 边界情况**

- 如果 $\tau_t$ 太低（如 0.001），teacher 输出几乎 one-hot，梯度消失（log 函数在 0 附近趋于负无穷）
- 如果 $\tau_t$ 太高（如 0.1），teacher 输出不够尖锐，无法提供足够的监督信号
- 论文通过实验选择 $\tau_t=0.04$（Table 2 的消融）

### 8.1 投影头的实现

```python
# 来自 vision_transformer.py
class DINOHead(nn.Module):
    def __init__(self, in_dim, out_dim, use_bn=False, norm_last_layer=True, nlayers=3, hidden_dim=2048, bottleneck_dim=256):
        super().__init__()
        nlayers = max(nlayers, 1)
        if nlayers == 1:
            self.mlp = nn.Linear(in_dim, bottleneck_dim)
        else:
            layers = [nn.Linear(in_dim, hidden_dim)]
            if use_bn:
                layers.append(nn.BatchNorm1d(hidden_dim))
            layers.append(nn.GELU())
            for _ in range(nlayers - 2):
                layers.append(nn.Linear(hidden_dim, hidden_dim))
                if use_bn:
                    layers.append(nn.BatchNorm1d(hidden_dim))
                layers.append(nn.GELU())
            layers.append(nn.Linear(hidden_dim, bottleneck_dim))
            self.mlp = nn.Sequential(*layers)
        self.apply(self._init_weights)
        self.last_layer = nn.utils.weight_norm(nn.Linear(bottleneck_dim, out_dim, bias=False))
        self.last_layer.weight_g.data.fill_(1)
        if norm_last_layer:
            self.last_layer.weight_g.requires_grad = False
```

注意点：
- 投影头使用 weight normalization（不是 layer norm），且最后一层的 weight_g 被冻结（norm_last_layer=True）
- 输出维度 $K=65536$，论文中称为"prototypes"
- 隐藏层 2048 维，bottleneck 256 维

## 十、局限

### 局限 1：对 ViT 架构的依赖

**问题**：DINO 的涌现性质（语义分割注意力图）只在 ViT 上出现，在 CNN 上没有。

**影响**：如果下游任务需要全局特征（如分类），DINO 在 CNN 上表现也很好（75.3%），但无法获得结构化的空间表征。

**论文是否承认**：是。论文明确提到"这些性质是 ViT 特有的"。

**后续工作**：iBOT [Zhou et al., 2021] 和 DINOv2 [Oquab et al., 2023] 进一步探索了 ViT 的潜力。

### 局限 2：需要大量计算资源

**问题**：DINO 需要 8 张 V100 GPU、300 epochs 训练。训练 ViT-S/8（高分辨率）需要更多资源。

**影响**：对资源有限的团队不友好。

**论文是否承认**：间接承认。论文提到训练细节但没有讨论计算成本。

**后续工作**：DINOv2 通过更高效的训练策略降低了成本。

### 局限 3：输出维度 K 的选择

**问题**：$K=65536$ 是经验选择的，没有理论指导。

**影响**：如果 $K$ 太小（如 1000），模型可能无法区分足够多的语义概念；如果太大，训练不稳定。

**论文是否承认**：否。论文没有讨论 $K$ 的选择。

### 局限 4：注意力图的语义分割是涌现的，不是显式监督的

**问题**：虽然注意力图能分割前景物体，但这是定性观察，没有量化指标。

**影响**：无法直接用于下游任务（如分割、检测），需要额外的微调。

**论文是否承认**：是。论文提到"这些性质是涌现的"，但没有量化。

**后续工作**：DINOv2 通过对比学习进一步提升了注意力图的质量。

## 十一、小结

DINO 这篇论文的核心贡献是：**在 ViT 上应用自蒸馏框架，发现了自监督 ViT 的涌现性质——自注意力图自动分割前景物体**。

具体来说：
- **问题**：现有自监督方法在 ViT 上效果一般，没有利用 ViT 的架构优势。
- **解决方案**：使用自蒸馏框架（student 从局部视图预测 teacher 从全局视图的输出），配合 centering + sharpening 防止坍塌。
- **为什么有效**：自蒸馏迫使模型学习"局部→全局"的对应关系，ViT 的 patch 级表征和自注意力机制恰好能捕获这种结构。

**更广泛的启示**：DINO 改变了人们对自监督学习的认知——它不仅是一个特征学习方法，还能揭示模型内部的结构化表征。这启发了后续一系列工作（iBOT、DINOv2、MAE），推动了自监督学习从"特征学习"向"表征理解"的转变。

DINO 也证明了：**有时候，简单的方法（自蒸馏）加上合适的架构（ViT），就能产生意想不到的涌现性质**。这不是偶然的——当训练目标与架构特性相匹配时，模型会自发地学习到我们期望的结构。
