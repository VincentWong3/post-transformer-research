# Swin Transformer: Hierarchical Vision Transformer using Shifted Windows（2021）

**论文：** [arXiv:2103.14030](https://arxiv.org/abs/2103.14030) · Ze Liu, Yutong Lin, Yue Cao, Han Hu et al. · Microsoft Research Asia · 2021

**技术分支：** B2 — 视觉 backbone 层级化 / B2-2 — hierarchical backbone 范式

**代码：** [microsoft/Swin-Transformer](https://github.com/microsoft/Swin-Transformer)

---

## 一、先搞清楚"问题在哪里"

2020 年，Vision Transformer（ViT）第一次做到了一个看起来不可能的事：**完全不用卷积，只靠标准 Transformer encoder，就能在 ImageNet 分类上追上 ResNet**。这对 CV 界来说是一个强烈的信号——或许 Transformer 不只是 NLP 的专属架构，它也可以成为视觉任务的通用基座。

但 ViT 之后的几个月，社区很快发现了一个尴尬的事实：**ViT 只能做分类**。把它接到目标检测上，效果不如 ResNet；接到语义分割上，性能也不理想。为什么？因为 ViT 有两条根本性的短板。

**第一条短板：它只有单分辨率输出。** ViT 把 224×224 图片切成 16×16 的 patch（共 196 个 token），经过 12 层 Transformer，输出永远是一个 $\frac{H}{16} \times \frac{W}{16}$ 的特征图。而检测和分割这类 dense prediction 任务天然需要多尺度特征——CNN 靠 ResNet 的 4 个 stage 逐步下采样，输出 1/4、1/8、1/16、1/32 四个分辨率的特征图，FPN 和 U-Net 就是吃这个的。ViT 只有一个分辨率，要做 dense prediction 要么上采样（丢信息）、要么加额外的 deconv 模块（增复杂度）。

**第二条短板：计算复杂度是 $O(N^2)$。** ViT 的全局 self-attention 中，每个 token 要和所有其他 token 算 attention。224×224 输入（196 个 token）还能扛，但 COCO 检测的典型输入是 800×1333，patch 数超过 4000，平方项 $4000^2 = 16M$ 直接炸显存。也就是说，ViT 在图像分类这个有限场景下证明了自己，但在更广泛的视觉任务上——尤其是高分辨率输入的任务——它的架构从根本上就不适合。

所以到 2021 年初，CV 界面临的问题是：**Transformer 到底能不能成为像 ResNet 那样的通用 backbone？** 如果可以，它必须具备两个能力：多尺度特征表示和线性复杂度。Swin Transformer 就是在这个节点上出现的。

## 二、旧路线的真正问题

要理解 Swin 为什么重要，得先搞清楚 ViT 的两条短板为什么是"结构性的"——不是换个训练技巧就能解决的。

### 2.1 尺度问题：NLP 和 CV 的根本差异

Transformer 最初是为 NLP 设计的。在 NLP 中，"词"是天然的基本语义单位，词与词之间没有明显的尺度差异。"apple" 和 "Apple Inc." 在语义上有区别，但它们的 token 长度几乎一样。

但在 CV 中，视觉实体在图像上的尺度可以差几十倍。一张图片里，大象可能占据几百个像素，而一个茶杯可能只有几十个像素。检测模型（如 Faster R-CNN）依赖 backbone 输出多个尺度的特征图，让不同大小的物体在不同分辨率的特征图上被检测到——小物体靠高分辨率特征图（Stage 1/2），大物体靠低分辨率语义特征图（Stage 3/4）。

ViT 的单分辨率输出 $\frac{H}{16} \times \frac{W}{16}$ 天然没有这个能力。有工作尝试在 ViT 的顶层接 deconvolution 做上采样来生成多尺度特征（SETR, Zheng et al., 2021），但这本质上是在"补救"而不是"设计"——丢失的细粒度信息无法通过上采样完美恢复。

### 2.2 复杂度问题：视觉的"序列长度"是天生的

NLP 中，一个典型句子的 token 数在 100-500 之间，所以 $O(N^2)$ 的 attention 是可以接受的。但图像不一样。一张 224×224 的图片，如果用 ViT 的 16×16 patch，token 数为 196；如果用 4×4 patch（保留更多空间细节），token 数就是 3136——是 ViT 的 16 倍。

做检测或分割时，输入分辨率还要更高。COCO 的典型输入 800×1333，如果用 4×4 patch，token 数超过 25000。全局 attention 的 attention 矩阵 $QK^T$ 有 $25000^2 \approx 6.25$ 亿个元素——一张卡也放不下。

这就是为什么 ViT 之后，很多工作试图在不牺牲空间分辨率的前提下降低 attention 复杂度。但全局 attention 的这个瓶颈是结构性的：**任何每对 token 之间都要交互的计算范式，其复杂度都是 $O(N^2)$**。要突破这个限制，必须从 attention 的计算方式本身入手。

### 2.3 效率问题：滑动窗口的"慢"不是巧合

在 Swin 之前，已经有人试过用局部 attention 来降低复杂度。[Ramachandran et al., 2019] 在 Stand-Alone Self-Attention 中提出了滑动窗口自注意力——每个像素只和局部邻域内的像素做 attention。理论上是 $O(N)$ 复杂度，但实际延迟比卷积大很多。

原因很简单：**卷积的权重是跨空间共享的**（一个 kernel 的权重在整张图上滑动复用），所以硬件内存访问是连续的。但滑动窗口 self-attention 中，每个查询像素的 key set 都不一样——即使邻域大小相同，不同位置的像素"看"到的邻居也是不同的。这意味着 key 的 gather 操作是散乱的，无法用连续内存访问来实现。

换句话说，**卷积快的本质不是因为它的 FLOPs 低，而是因为它的内存访问是 cache-friendly 的。** 滑动窗口 self-attention 的 FLOPs 不高，但实际慢在内存访问模式。

### 2.4 小结：需要一个什么样的设计？

综合以上三点，一个可用的视觉 Transformer backbone 必须满足：

1. **多尺度特征** — 输出 4 级金字塔，能和 FPN/U-Net 即插即用
2. **线性复杂度** — attention 复杂度 $O(N)$ 而不是 $O(N^2)$
3. **硬件高效** — 内存访问连续，不要在 gather 操作上浪费带宽

现有的方案（全局 attention、滑动窗口 attention）各自只满足其中一个或两个条件。Swin Transformer 的任务是找一个方案，同时满足三个条件。

## 三、这篇论文的核心思路

**这篇论文的核心思路是：用非重叠的局部窗口做 self-attention，把复杂度从 $O(N^2)$ 降到 $O(N)$；然后用窗口偏移（shifted window）让信息跨窗口流动，解决局部 attention 的连通性问题；再加上 Patch Merging 实现层级特征图，对齐 CNN backbone 的输出格式。**

具体来说，三个设计组合在一起：

```text
局部窗口 attention        →  线性复杂度，硬件高效的连续内存访问
       ↓
交错移位窗口（W/SW交替）   →  跨窗口信息流动，等效感受野随层数增长
       ↓
Patch Merging 下采样      →  4 级金字塔特征图（1/4, 1/8, 1/16, 1/32）
```text

这条技术路线的谱系位置是：

```text
CNN backbone ×
   ↓
ViT（全局 attention, 单分辨率, 2020）
   ↓
PVT（金字塔 Transformer, 2021, 同期竞品）
   ↓
Swin Transformer（局部窗口 + 移位 + 层级, 2021） ← 本篇
   ↓
Swin V2（大规模版本, 2022）
```

Swin 和同期工作 PVT 的最大区别是：PVT 用 Spatial Reduction Attention（空间降维后做稀疏全局 attention），复杂度仍是 $O(N^2)$ 的低阶项；Swin 完全放弃跨窗口的全局 attention，只在局部窗口内算，靠窗口偏移来间接实现跨窗口连接。这个选择让 Swin 在硬件效率上显著优于 PVT。

## 四、整体结构与 forward 流程

### 4.1 整体架构（Swin-T 为例）

Swin Transformer 有 4 个 stage，对应 4 级分辨率，和 ResNet 的 ``conv2_x``—``conv5_x`` 完全对齐。

```text
输入图像: H×W×3  (e.g., 224×224×3)
    │
    ▼
[Patch Partition]         4×4 patches → 特征展开
    │                     输出: 56×56×48
    ▼
[Linear Embedding]        48 → C (C=96 for Swin-T)
    │                     输出: 56×56×96
    ▼
┌─────────────────────────────────────────┐
│ Stage 1 (4×下采样, 56×56, C=96)        │
│  ┌─────────────────────────────┐        │
│  │ Swin Block #1: W-MSA (规则) │        │
│  │ LN → W-MSA → 残差 → LN → MLP → 残差 │
│  └─────────────────────────────┘        │
│  ┌─────────────────────────────┐        │
│  │ Swin Block #2: SW-MSA (偏移)│        │
│  │ LN → SW-MSA → 残差 → LN → MLP → 残差 │
│  └─────────────────────────────┘        │
└────────────────┬────────────────────────┘
                 │
                 ▼
[Patch Merging]     2×2邻域拼接 → 4C → 线性层 → 2C
                 │   输出: 28×28×192 (2×下采样)
                 ▼
┌─────────────────────────────────────────┐
│ Stage 2 (8×下采样, 28×28, 2C=192)      │
│  Swin Block #3: W-MSA                   │
│  Swin Block #4: SW-MSA                  │
└────────────────┬────────────────────────┘
                 │
                 ▼
[Patch Merging]     28×28 → 14×14
                 │   输出: 14×14×384
                 ▼
┌─────────────────────────────────────────┐
│ Stage 3 (16×下采样, 14×14, 4C=384)     │
│  6个 Swin Blocks: W/SW 交替...          │
└────────────────┬────────────────────────┘
                 │
                 ▼
[Patch Merging]     14×14 → 7×7
                 │   输出: 7×7×768
                 ▼
┌─────────────────────────────────────────┐
│ Stage 4 (32×下采样, 7×7, 8C=768)       │
│  2个 Swin Blocks: W/SW 交替             │
└─────────────────────────────────────────┘
```

### 4.2 前向过程逐步跟踪（Swin-T, 224×224 输入）

**Step 1 — Patch Partition**

输入 224×224×3 的 RGB 图像。切成 4×4 的非重叠 patch。每个 patch 的 RGB 值展平为 4×4×3=48 维向量。

张量形状：`[B, 3, 224, 224]` → `[B, 48, 56, 56]`

关键点：ViT 用 16×16 patch（输出 14×14=196 token），Swin 用 4×4 patch（输出 56×56=3136 token）。token 数量是 ViT 的 16 倍，保留了更多的空间细节。

**Step 2 — Linear Embedding**

将 48 维投影到 C=96。使用 `nn.Linear(48, 96)` + `LayerNorm(96)`。

张量形状：`[B, 48, 56, 56]` → `[B, 96, 56, 56]`

后续的 Swin Blocks 内部将张量视为 `[B, 56, 56, 96]`（即 HW 维度和 C 维度互换），因为窗口划分操作需要 HW 维在最右侧。

**Stage 1（2 个 Swin Block, 分辨率 56×56, channel 96）**

Swin Block 的标准结构：

```python
# Swin Block (Pre-LN)
shortcut = x
x = LN(x)
x = window_attention(x)   # W-MSA 或 SW-MSA
x = x + shortcut           # 残差
shortcut = x
x = LN(x)
x = MLP(x)                 # 2层 MLP, GELU, 4× expansion
x = x + shortcut           # 残差
```

Block #1 用规则窗口（W-MSA），Block #2 用移位窗口（SW-MSA）。两者交替出现。窗口大小 M=7。

56×56 的网格，用 7×7 的窗口划分：窗口数量 = (56/7)² = 64。每个窗口内 tokens = 49。

**Step 3 — Patch Merging（Stage 1→2）**

在 2×2 邻域内，4 个 patch 的 channel 维拼接（4C=384），然后 `nn.Linear(384, 2C=192)`。

张量形状：`[B, 56, 56, 96]` → `[B, 28, 28, 192]`

等效于 CNN 里的 stride-2 卷积 + channel 加倍，但实现更轻量。

**Stage 2（2 个 Swin Block, 28×28, 192）**

窗口数量 = (28/7)² = 16。窗口内 tokens = 49。

**Step 4 — Patch Merging（Stage 2→3）**

28×28×192 → 14×14×384

**Stage 3（6 个 Swin Block, 14×14, 384）**

窗口数量 = (14/7)² = 4。这是最深的主干 stage。为什么 stage 3 的 block 最多？因为在检测/分割中，1/16 分辨率的特征图通常最重要——它平衡了语义信息和空间分辨率。

**Step 5 — Patch Merging（Stage 3→4）**

14×14×384 → 7×7×768

**Stage 4（2 个 Swin Block, 7×7, 768）**

注意 7×7 = 7×7（M=7），此时整张特征图只有一个窗口，所以窗口只有一个，W-MSA 退化为全局 MSA。

### 4.3 输出格式（4 级特征图）

| Stage | 下采样率 | 分辨率 | Channel | 类比 ResNet |
|-------|---------|--------|---------|-------------|
| 1 | 4× | 56×56 | C (96) | `conv2_x` |
| 2 | 8× | 28×28 | 2C (192) | `conv3_x` |
| 3 | 16× | 14×14 | 4C (384) | `conv4_x` |
| 4 | 32× | 7×7 | 8C (768) | `conv5_x` |

所有基于 ResNet 的检测/分割框架（Mask R-CNN, UperNet, FPN, HTC）都可以直接替换 backbone，零适配成本。

### 4.4 架构参数（4 个变体）

| 变体 | C | Layer nums | 参数 | FLOPs | 类比 |
|------|---|-----------|------|-------|------|
| Swin-T | 96 | [2,2,6,2] | 29M | 4.5G | ResNet-50 |
| Swin-S | 96 | [2,2,18,2] | 50M | 8.7G | ResNet-101 |
| Swin-B | 128 | [2,2,18,2] | 88M | 15.4G | ViT-B/DeiT-B |
| Swin-L | 192 | [2,2,18,2] | 197M | 34.5G | — |

窗口大小 M=7，每个 head 的 query 维度 d=32，MLP 扩展率 α=4。

## 五、核心模块逐个解释

### 5.1 W-MSA：窗口自注意力（为什么是局部窗口）

**问题：** 全局 MSA 的 attention 复杂度是 $O((hw)^2)$，对于高分辨率输入不可行。

**解决方案：** 把特征图均匀分成 $M \times M$（默认 7×7）的非重叠窗口，只在每个窗口内部做 self-attention。

**为什么这个设计是合理的：**

局部窗口 attention 背后的直觉是：**图像的相邻像素间有很强的局部相关性**。猫的耳朵和它的尾巴可能不需要直接交互，但耳朵和它旁边的毛发需要。CNN 靠堆叠 3×3 卷积来建模这种局部性，Swin 用 7×7 的局部窗口来做 self-attention 覆盖类似的范围。

**工作方式：**

```python
# Window partition: [B, H, W, C] → [B*nW, M, M, C]
def window_partition(x, window_size):
    B, H, W, C = x.shape
    x = x.view(B, H // window_size, window_size,
               W // window_size, window_size, C)
    # permute: B, H/ws, W/ws, ws, ws, C → B*H/ws*W/ws, ws, ws, C
    windows = x.permute(0, 1, 3, 2, 4, 5).contiguous()
    windows = windows.view(-1, window_size, window_size, C)
    return windows
```

这里的维度变换非常巧妙：先用 `view` 把 H 和 W 轴分解为 `(H/ws, ws)` 和 `(W/ws, ws)`，然后用 `permute` 把窗口索引和窗口内位置分开，最终把每个窗口展平为独立的 batch 项。相比 `torch.nn.Unfold`（需要构造滑动窗口矩阵），这个操作完全是 view + contiguous，没有数据复制开销。

### 5.2 SW-MSA：移位窗口（为什么需要交错）

**问题：** W-MSA 的每个 token 只和同一个 7×7 窗口内的邻居做 attention，不同窗口之间完全没有信息流通。这等价于把图片切成 7×7 的独立小块分别处理——全局建模能力为零。

**解决方案：** 相邻两层用不同的窗口划分。第 $l$ 层用规则网格（W-MSA），第 $l+1$ 层把网格偏移 $(\lfloor M/2\rfloor, \lfloor M/2\rfloor)=(3,3)$ 个 patch（SW-MSA）。

```text
Layer l (W-MSA, 规则划分):       Layer l+1 (SW-MSA, 偏移 3):

  ┌──────┬──────┬──────┬──────┐     ┌──────┬──────┬──────┬──────┐
  │      │      │      │      │     │  ────┼──────┼──────┤      │
  │  A   │  A   │  B   │  B   │     │  A/A │  A/B │  B/B │  C/D │
  │      │      │      │      │     │  ────┼──────┼──────┤      │
  ├──────┼──────┼──────┼──────┤     ├──────┼──────┼──────┼──────┤
  │      │      │      │      │     │  ────┼──────┼──────┤      │
  │  A   │  A   │  B   │  B   │     │  A/A │  A/B │  B/B │  C/D │
  │      │      │      │      │     │  ────┼──────┼──────┤      │
  ├──────┼──────┼──────┼──────┤     ├──────┼──────┼──────┼──────┤
  │      │      │      │      │     │      │      │      │      │
  │  C   │  C   │  D   │  D   │     │  A/C │  A/D │  B/D │  D/D │
  │      │      │      │      │     │      │      │      │      │
  ├──────┼──────┼──────┼──────┤     ├──────┼──────┼──────┼──────┤
  │      │      │      │      │     │      │      │      │      │
  │  C   │  C   │  D   │  D   │     │  C/C │  C/D │  D/D │  D/D │
  │      │      │      │      │     │      │      │      │      │
  └──────┴──────┴──────┴──────┘     └──────┴──────┴──────┴──────┘
```

偏移后，Layer $l+1$ 的每个窗口跨了 Layer $l$ 中多个不同窗口的边界。以左上角窗口为例：它包含了来自 Layer $l$ 的 A 区域（左上）、A/B 边界区域、B 区域（右上）的信息——这些区域在 Layer $l$ 中属于不同窗口，互相看不到，但在 Layer $l+1$ 中被放到了同一个窗口内做 attention。

**为什么比滑动窗口快：**

滑动窗口 self-attention 需要为每个查询像素准备不同的 key set。这意味着 key 的 gather 操作是非连续的——不同像素的邻域在内存中不相邻。

Swin 的窗口方案则不同：同一窗口内的所有查询共享同一个 key set（该窗口内的所有 token）。这使得 attention 计算的矩阵乘法是连续的——$QK^T$ 是规整的 $M^2 \times M^2$ 矩阵乘法。

论文 Table 5 的延迟数据证实了这一点：滑窗法在 Stage 1（56×56）需要 122ms（naive 实现），而 Swin 的窗口法只需要 3ms——快了 40 倍。

### 5.3 Cyclic Shift + Attention Mask（高效实现技巧）

**问题：** SW-MSA 窗口偏移后，窗口形状不再规则——一些窗口跨越了 4 个原本的规则窗口的边界，直接在这些不规则窗口上做 attention 会导致数据内存排列不连续。

**解决方案：** 对特征图做循环移位（`torch.roll`），把分散到四角的碎片重新聚拢成规则的窗口。然后用 attention mask 屏蔽掉不应该互相 attention 的 query-key 对。

**实现步骤：**

1. **循环移位（cyclic shift）：** `torch.roll(x, shifts=(-M//2, -M//2), dims=(1, 2))`。把特征图向左上方向平移 M/2 个像素。平移后，原来分散在四个角的碎片被拼合到左上角，形成和规则划分一样数量的规则窗口。

2. **规则窗口划分：** 在移位后的特征图上做标准窗口划分。

3. **Attention Mask：** 每个窗口内包含来自不同原始区域的 tokens。例如，一个窗口中可能有 4 个来自不同区域（A/B/C/D）的 tokens 块。在 attention 计算中，不同区域的 tokens 之间不应该互相看到。因此构造一个 mask 矩阵（大小 `[num_windows, M², M²]`），跨区域的 query-key 对加 -100（softmax 后变为 0）。

4. **反向循环移位：** attention 计算完成后，对输出做逆向的 `torch.roll` 恢复位置。

Mask 的构建是静态的——在模型初始化时一次性算好，推理时只需查表。论文 Figure 4 展示了 9 种 mask 模式。

**为什么高效：**

- `torch.roll` 是 $O(HWC)$ 的连续内存操作，没有数据复制或 gather 开销
- 窗口数量与规则划分时相同（$(\lceil h/M\rceil) \times (\lceil w/M\rceil)$），不增加窗口数
- 没有 padding 带来的额外计算

论文 Table 5 显示：cyclic 实现相比 naive padding 在整个网络上快 13%（Swin-T）到 18%（Swin-B）。

### 5.4 Patch Merging：层级特征图

**问题：** ViT 输出是单分辨率特征图，无法对接 FPN/U-Net 这类需要多尺度特征的模块。

**解决方案：** 在 stage 之间插入 Patch Merging 层：取 2×2 邻域的 4 个 patch，channel 维拼接（4C），然后线性投影到 2C。分辨率减半，channel 加倍。

```python
def patch_merging(x):
    """
    x: [B, H, W, C]
    output: [B, H/2, W/2, 2C]
    """
    # 取 2×2 邻域
    x = x.reshape(B, H//2, 2, W//2, 2, C)
    x = x.permute(0, 1, 3, 2, 4, 5).contiguous()
    x = x.reshape(B, H//2, W//2, 4*C)
    x = nn.Linear(4*C, 2*C)(x)
    return x
```

这个操作完全等效于 CNN 里的 stride-2 卷积 + channel 加倍，但实现了更简洁。

设计直觉：相邻 4 个 patch 之间有高度冗余（视觉信号的局部连续性）。拼接后的 4C 维特征既有冗余信息也有互补信息，线性投影到 2C 相当于做了一次信息压缩，保留重要的空间-语义信息。

### 5.5 相对位置偏置

**问题：** self-attention 本身是排列等变的——它不知道 token 之间的空间关系。ViT 用绝对位置编码（learned 1D positional embedding）来注入位置信息，但这种方法假设每个位置有独立的位置编码，且在不同输入分辨率下无法直接迁移。

**解决方案：** 在 attention 的 logits 上加一个可学习的相对位置偏置 $B$。

标准 self-attention：$\text{Attention}(Q, K, V) = \text{SoftMax}(\frac{QK^T}{\sqrt{d}})V$

Swin 的 self-attention：$\text{Attention}(Q, K, V) = \text{SoftMax}(\frac{QK^T}{\sqrt{d}} + B)V$

其中 $B \in \mathbb{R}^{M^2 \times M^2}$，表示窗口内每对 token 之间的相对位置关系。

但 $B$ 不直接学会 $M^2 \times M^2$ 的矩阵——那有 2401 个参数（M=7），不同窗口尺寸下要重新训练。Swin 用了更紧凑的表示：参数化一个 $(2M-1) \times (2M-1) = 13 \times 13$ 的偏置表 $\hat{B}$，然后通过索引从表中取值。

索引方式：对于窗口内 token i 和 token j，计算它们在 x 和 y 轴上的偏移量 $(\Delta x, \Delta y)$，映射到 $[0, 2M-2]$ 的范围。实际代码中使用的是：

```python
# 计算所有 token 对的 x/y 偏移
coords_h = torch.arange(M)
coords_w = torch.arange(M)
coords = torch.stack(torch.meshgrid([coords_h, coords_w]))  # 2, M, M
coords_flatten = coords.flatten(1)  # 2, M²
relative_coords = coords_flatten[:, :, None] - coords_flatten[:, None, :]  # 2, M², M²
relative_coords = relative_coords.permute(1, 2, 0).contiguous()  # M², M², 2
# 偏移量映射到 [0, 2M-2]
relative_coords[:, :, 0] += M - 1
relative_coords[:, :, 1] += M - 1
# 合并为单维索引
relative_position_index = relative_coords[:, :, 0] * (2*M-1) + relative_coords[:, :, 1]
# 查表
B = relative_position_bias_table[relative_position_index]  # M², M², nH
```

**为什么有效：**

- 相对位置偏置比绝对位置编码更符合视觉信号的平移不变性——$B$ 只依赖于两个 token 之间的偏移量，和它们在图像中的绝对位置无关。这在检测和分割任务中特别重要（Table 4 显示绝对位置编码在 COCO 上反而降效果）。
- 紧凑的参数表（$13 \times 13 \times \text{nH}$ vs $M^2 \times M^2 \times \text{nH} = 2401 \times \text{nH}$）减少了 14 倍参数。
- 通过双三次插值，预训练好的偏置表可以迁移到不同窗口尺寸（如从 M=7 迁移到 M=12），不需要重新训练。

Table 4 消融显示，去掉相对位置偏置从 81.3 掉到 80.3（-1.0%）。

## 六、核心公式与数学直觉

### 6.1 复杂度公式：为什么 W-MSA 是 $O(N)$

**全局 MSA：**

$$\Omega(\text{MSA}) = 4hwC^2 + 2(hw)^2C$$

**窗口 MSA：**

$$\Omega(\text{W-MSA}) = 4hwC^2 + 2M^2 hwC$$

其中 $h,w$ 是特征图的高和宽，$C$ 是 channel 数，$M$ 是窗口大小。

**公式推导（直观理解）：**

全局 MSA 的单 token 计算过程：

```
Q = x @ W_Q        # C×C → C FLOPs (每个 token)
K = x @ W_K        # C FLOPs
V = x @ W_V        # C FLOPs (QKV 共 3C)
attention = Q @ K^T  # C × hw → hw FLOPs
output = attention @ V  # hw × C → C FLOPs
out_proj = output @ W_O  # C×C → C FLOPs
```

每 token 共 $4C$（QKV+out_proj）+ $2hw$（attention+aggregate）FLOPs。

对所有 $hw$ 个 token 求和：
- QKV 投影 + output 投影：$4hwC^2$
- attention 矩阵 + value 聚集：$2(hw)^2C$

这就是两项的来源。

W-MSA 的 attention 只在 $M \times M$ 窗口内计算。一个窗口有 $M^2$ 个 token，attention FLOPs = $2M^4C$。共有 $\frac{hw}{M^2}$ 个窗口。总 FLOPs = $\frac{hw}{M^2} \times 2M^4C = 2M^2 hwC$。

**数字对比：** 设 $h=56, w=56, C=96, M=7$：

- 全局 MSA：$4(56^2)(96^2) + 2(56^4)(96) = 115.6M + 1888.2M = 2003.8M$ FLOPs
- W-MSA：$4(56^2)(96^2) + 2(7^2)(56^2)(96) = 115.6M + 29.5M = 145.1M$ FLOPs

**W-MSA 比全局 MSA 快 14 倍。**

关键洞察：第一项 $4hwC^2$（QKV 投影）在两者中相同，差异全在第二项。W-MSA 的第二项 $2M^2 hwC$ 中，$M$ 是常数（7），所以对 $hw$ 是线性。全局 MSA 的第二项 $2(hw)^2C$ 对 $hw$ 是二次。

如果输入是 800×1333（COCO 检测典型输入，4×下采样后约 200×333 = 66,600 个 patch），全局 MSA 的 attention FLOPs $2 \times (66600)^2 \times C \approx 851$G——完全不可行。W-MSA 的 attention FLOPs $2 \times 49 \times 66600 \times C \approx 627$M——差了超千倍。

### 6.2 相对位置偏置公式

$$\text{Attention}(Q, K, V) = \text{SoftMax}\left(\frac{QK^T}{\sqrt{d}} + B\right)V, \quad B \in \mathbb{R}^{M^2 \times M^2}$$

这个公式表面上只是在 attention logits 上加了一个偏置项，但背后有更深的设计直觉。

**如果不加 $B$：** attention 只由内容相似度 $(QK^T)$ 决定。两张完全相同的图片，其中一张把猫从左侧移到右侧，attention 会不同（因为 Q 和 K 的位置编码变了）。这在语义上不合理——猫的位置变了，但猫和周围像素的关系模式没变。

**加上 $B$ 后：** attention 同时由"内容相似度"和"相对空间位置"决定。两个像素在空间上相近，即使内容不完全相似，也能有较高的 attention 权重。这引入了 CNN 中隐含的空间连续性归纳偏置（邻近像素的特征应该更容易交互）。

**为什么不直接用绝对位置编码？**

Table 4 的消融结果很有说服力：
- 不加位置编码：80.1%
- 加绝对位置编码：80.5%（+0.4%）
- 加相对位置偏置：81.3%（+1.2%）
- 绝对 + 相对：81.3%（没有叠加增益）

更有趣的是在 COCO 检测上：绝对位置编码反而降了 0.2 box AP。原因可能是绝对位置编码对物体位置敏感——训练集上某个位置的偏置可能在测试集不成立。而相对位置偏置只关心偏移量，不受绝对位置影响。

### 6.3 Swin Block 计算图

$$\hat{\mathbf{z}}^l = \text{W-MSA}(\text{LN}(\mathbf{z}^{l-1})) + \mathbf{z}^{l-1}$$
$$\mathbf{z}^l = \text{MLP}(\text{LN}(\hat{\mathbf{z}}^l)) + \hat{\mathbf{z}}^l$$
$$\hat{\mathbf{z}}^{l+1} = \text{SW-MSA}(\text{LN}(\mathbf{z}^{l})) + \mathbf{z}^{l}$$
$$\mathbf{z}^{l+1} = \text{MLP}(\text{LN}(\hat{\mathbf{z}}^{l+1})) + \hat{\mathbf{z}}^{l+1}$$

这是标准的 Pre-LN Transformer Block 结构。和标准 Transformer（Post-LN）的区别是：LayerNorm 在子层之前而不是之后。

**为什么用 Pre-LN？** [Xiong et al., 2020] 证明 Pre-LN 比 Post-LN 训练更稳定，尤其是在深层 Transformer 中。Post-LN 的梯度流经 LN 会引入额外的方差，导致深层训练不稳定。Pre-LN 的残差路径更干净（直接从输入到输出跳过一个 block），梯度可以更顺畅地传播。

形式上，（S）W-MSA 对应 MSA 模块的替代——这是整个设计中最关键的替换。MLP 部分和标准 Transformer 一致：2 层线性层，中间 GELU 激活，残差连接。

## 八、复杂度、效率与可扩展性分析

### 8.1 时间复杂度

Swin 的核心复杂度优势来自 W-MSA 的 $O(M^2 hwC)$ 替代了全局 MSA 的 $O((hw)^2 C)$。

对于不同输入分辨率：

| 输入分辨率 (像素) | Stage 1 Patch 数 (4×下采样) | 全局 MSA FLOPs (attention 部分) | W-MSA FLOPs (attention 部分) | 加速比 |
|-----------------|---------------------------|-------------------------------|-------------------------------|--------|
| 224² ImageNet | 56²=3136 | 2×(3136)²×96 ≈ 1.89G | 2×49×3136×96 ≈ 29.5M | 64× |
| 512² 常见检测 | 128²=16384 | 2×(16384)²×96 ≈ 51.5G | 2×49×16384×96 ≈ 154M | 334× |
| 800×1333 COCO 典型 | 200×333≈66,600 | 2×(66,600)²×96 ≈ 851G (不可行) | 2×49×66,600×96 ≈ 627M | ~1300× |

加速比随输入分辨率急剧增长——这正是 Swin 设计的核心目标之一。

### 8.2 内存复杂度

全局 MSA 的 attention 矩阵大小：$O((hw)^2)$。对于 512² 输入，矩阵占 16384² × sizeof(float16) ≈ 512MB。

W-MSA 的 attention 矩阵大小：$O(M^4 \times \text{nWindows})$。对于 7² 窗口和 512² 输入：窗口数 = (128/7)² ≈ 334，每个窗口 49²=2401 元素，总 334×2401×2 bytes ≈ 1.6MB。

内存减少了 300 倍。这解释了为什么 ViT 做高分辨率 dense prediction 需要特殊优化（梯度检查点、碎片化 attention），而 Swin 可以直接做。

### 8.3 实际延迟对比（Table 5）

论文 Table 5 给出了关键延迟数据（V100 GPU）：

| 方法 | Stage 1 (ms) | Stage 2 | Stage 3 | Stage 4 | 总 (FPS) |
|------|-------------|---------|---------|---------|----------|
| 滑窗 (naive) | 122.5 | 38.3 | 12.1 | 7.6 | 183 (T) |
| 滑窗 (kernel) | 7.6 | 4.7 | 2.7 | 1.8 | 488 (T) |
| Performer | 4.8 | 2.8 | 1.8 | 1.5 | 638 (T) |
| W-MSA (无偏移) | 2.8 | 1.7 | 1.2 | 0.9 | 770 (T) |
| SW-MSA (cyclic) | 3.0 | 1.9 | 1.3 | 1.0 | 755 (T) |

关键发现：
- **滑窗 naive 和 kernel 的巨大差距**（122ms vs 7.6ms）说明优化实现（fused kernel）对滑窗至关重要。而没有专门 kernel 的 naive 实现基本不可用。
- **SW-MSA (cyclic) 比 Performer 快**（755 vs 638 FPS 在 Swin-T）——Performer 是 2020 年最快的 transformer 变体之一。
- **W-MSA 和 SW-MSA 之间只差 2% 的延迟**（770 vs 755 FPS）——shift + mask 的开销几乎可以忽略。

### 8.4 新引入的瓶颈

Swin 解决了 ViT 的计算瓶颈，但引入了新的限制：

1. **窗口大小限制感受野。** 虽然交错移位实现了跨窗口连接，但每次只能跨越相邻窗口。24 层 Swin-B 需要 12 次移位才能覆盖 84 个 patch（1344 像素）。对于需要全局感受野的任务（如全景分割），这不如直接做全局 attention。

2. **Cyclic shift 的边界效应。** 移位后窗口碎片化，需要 mask 屏蔽跨区域 attention。当窗口数量很少时（如 Stage 3 只有 4 个窗口），mask 的复杂度占比更大。

3. **窗口划分的硬件依赖。** 虽然连续内存访问比滑窗好，但 window partition 的 reshape + permute 操作在某些硬件（如 ARM CPU 或移动端 GPU）上可能不如 DWConv 高效。

4. **Stage 4 的边界情况。** 当分辨率 ≤ M×M 时（Stage 4 的 7×7），W-MSA 退化为全局 MSA。这在架构上不是问题，但意味着 Stage 4 的 FLOPs 没有线性优势。

### 9.4 消融实验（Table 4）

| 变体 | ImageNet Top-1 | COCO box AP | ADE20K mIoU |
|------|---------------|-------------|-------------|
| 基线 (W-MSA + SW-MSA + RelPos) | 81.3 | 50.5 | 46.1 |
| 只用 W-MSA（无移位） | 80.2 (-1.1) | 47.7 (-2.8) | 43.3 (-2.8) |
| 无位置编码 | 80.1 (-1.2) | 49.2 (-1.3) | 43.8 (-2.3) |
| 绝对位置编码 | 80.5 (-0.8) | 49.0 (-1.5) | 43.2 (-2.9) |
| 绝对 + 相对位置偏置 | 81.3 (0) | 50.2 (-0.3) | 44.0 (-2.1) |

消融实验说明三个关键事实：

1. **移位窗口贡献了 1.1 个分类点、2.8 box AP、2.8 mIoU。** 这是单一最大的贡献项。在检测和分割上，移位窗口比分类上更重要——因为 dense prediction 需要跨窗口的信息融合来建模物体边界和上下文。

2. **相对位置偏置在密集预测上尤其重要（+2.3 mIoU）。** 在检测/分割中，"空间邻近性"是极强的先验——相邻像素几乎总是属于同一个物体。相对位置偏置编码了这个先验。

3. **绝对位置编码在检测/分割上反而有害。** 分类任务受益于绝对位置信息（知道物体的绝对位置有助于识别特定类别，如"太阳"通常在图像上半部分），但检测/分割需要平移不变性——物体在一张图像的左上角还是右下角不应影响检测结果。

### 10.3 前向路径

`SwinTransformer.forward_features()` 是主入口：

```python
def forward_features(self, x):
    x = self.patch_embed(x)          # [B,3,224,224] → [B,56,56,96]
    if self.ape:
        x = x + self.absolute_pos_embed
    x = self.pos_drop(x)

    for layer in self.layers:         # 4个BasicLayer
        x = layer(x)                  # Stage内部处理
        # 保存每级输出用于 FPN/UperNet
        # layer_out[x] = x  （在完整实现中）

    x = self.norm(x)                  # LayerNorm
    x = self.avgpool(x.transpose(1, 3))  # 全局平均池化
    x = torch.flatten(x, 1)
    x = self.head(x)                  # Linear 分类头
    return x
```

### 10.4 WindowAttention 实现

这是最核心的类：

```python
class WindowAttention(nn.Module):
    def __init__(self, dim, window_size, num_heads, qkv_bias=True, qk_scale=None):
        self.window_size = window_size  # (M, M)
        self.num_heads = num_heads
        head_dim = dim // num_heads
        self.scale = qk_scale or head_dim ** -0.5

        # 相对位置偏置表: (2M-1) × (2M-1), num_heads
        self.relative_position_bias_table = nn.Parameter(
            torch.zeros((2 * window_size[0] - 1) * (2 * window_size[1] - 1), num_heads))

        # 预计算相对位置索引
        # ... (meshgrid + 偏移映射)

    def forward(self, x, mask=None):
        B_, N, C = x.shape      # B_ = num_windows * batch, N = M²
        qkv = self.qkv(x)       # [B_, N, 3*num_heads*head_dim]
        qkv = qkv.reshape(B_, N, 3, self.num_heads, -1).permute(2, 0, 3, 1, 4)
        q, k, v = qkv[0], qkv[1], qkv[2]

        attn = (q @ k.transpose(-2, -1)) * self.scale  # [B_, nH, N, N]

        # 加相对位置偏置
        relative_position_bias = self.relative_position_bias_table[
            self.relative_position_index.view(-1)].view(
            self.window_size[0] * self.window_size[1],
            self.window_size[0] * self.window_size[1], -1)
        relative_position_bias = relative_position_bias.permute(2, 0, 1)
        attn = attn + relative_position_bias.unsqueeze(0)

        # 加 mask（SW-MSA 时使用）
        if mask is not None:
            nW = mask.shape[0]  # num_windows
            attn = attn.view(B_ // nW, nW, self.num_heads, N, N) + mask.unsqueeze(1).unsqueeze(0)
            attn = attn.view(-1, self.num_heads, N, N)

        attn = attn.softmax(dim=-1)
        x = (attn @ v).transpose(1, 2).reshape(B_, N, C)
        x = self.proj(x)
        return x
```

### 10.5 论文 vs 代码对照：几个实现细节

**1. 相对位置偏置初始化。** 论文没提初始化方式。代码中使用 `trunc_normal_(self.relative_position_bias_table, std=.02)`——和 ViT 中的 position encoding 初始化一致。

**2. DropPath（随机深度）。** 每个 Swin Block 的残差连接后使用 `timm.layers.DropPath`。`dpr`（Drop Path Rate）从 0 线性增长到最后一层。Swin-T 的 dpr=0.2，意味着最后几层大约 20% 的 batch 样本会跳过残差路径。这在论文中没有显式讨论，但来自 DeiT 的训练技巧。

**3. window_partition 的 reshape 技巧。**

```python
x = x.view(B, H // window_size, window_size,
           W // window_size, window_size, C)
windows = x.permute(0, 1, 3, 2, 4, 5).contiguous().view(-1, window_size, window_size, C)
```

比 `torch.nn.functional.unfold` 更加高效——unfold 会产生重复的内存访问（scatter 操作），而 view + permute 是纯视图操作。

**4. 全局平均池化（GAP）替代 class token。** ViT 使用 cls token（特殊 token 用于全局分类），Swin 在 Stage 4 输出上做 GAP。论文指出，GAP 的精度和 cls token 一样好（81.3% vs 81.3%），而且更简洁——不需要额外的 position embedding 或特殊 token 处理。

### 11.1 技术树中的位置

   │      │             │
   │      │             ├── [CSWin Transformer, 2022]  ← 十字形窗口

```

Swin 处于 "hierarchical vision transformer backbone" 这个路线上，和 PVT 几乎是同期工作（都发表于 2021 年上半年）。

### 11.2 前驱工作：Swin 继承了谁？

**Transformer（Vaswani et al., 2017）：** Swin 保留了标准的 QKV attention、multi-head、FFN、residual connection、LayerNorm。核心修改只在 attention 的计算范围（从全局变成局部窗口）。

**ViT（Dosovitskiy et al., 2020）/ DeiT（Touvron et al., 2021）：** Swin 继承了 patch embedding 和 Pre-LN Transformer Block 的基本结构。训练配置（AdamW、cosine LR、RandAugment、Mixup、Cutmix）基本来自 DeiT。但 Swin 解决了 ViT 的两个核心问题：单分辨率输出和 $O(N^2)$ 复杂度。

**Stand-Alone Self-Attention（Ramachandran et al., 2019）：** 首次用局部自注意力替代卷积。验证了局部 attention 在视觉任务中的可行性，但滑动窗口方案延迟高。Swin 用非重叠窗口 + 偏移替代滑动窗口，解决了延迟问题。

**ResNet（He et al., 2016）：** Swin 的 4 级金字塔结构直接对齐 ResNet 的输出格式。这不是巧合——Swin 的设计目标之一就是"即插即用替换 ResNet"。

### 11.3 同期竞品：Swin 和谁竞争？

**PVT（Wang et al., 2021）：** 同期（ICCV 2021）提出的金字塔 Transformer。PVT 用 Spatial Reduction Attention（SRA）在每个 stage 降维特征图，然后做稀疏全局 attention。SRA 的复杂度是 $O((hw)(hw/R^2))$，其中 $R$ 是降维率——复杂度介于 $O(N)$ 和 $O(N^2)$ 之间，不如 Swin 的 $O(N)$。从实际效果看，Swin 在检测和分割上的增益显著大于 PVT。

**DeiT（Touvron et al., 2021）：** 和 Swin 同样是 2021 年 CVPR 的视觉 Transformer 工作。DeiT 主要关注训练策略（数据增强、蒸馏），架构上沿用 ViT 的全局 attention + 单分辨率。Swin 和 DeiT 不是竞争关系，而是互补关系——Swin 解决了 DeiT 不能解决的 dense prediction 问题。

**T2T-ViT（Yuan et al., 2021）：** T2T-ViT 在 ViT 之前加了一个 CNN-style 的 token 渐进融合模块，但后续仍然是标准 ViT 结构。精度不如 Swin，且没有层级特征图。

### 11.4 后续影响：Swin 启发了谁？

**Swin V2（Liu et al., 2022）：** 直接扩展。解决大规模训练（3B 参数）下的不稳定问题。核心改进：Post-Norm（解决深层梯度问题）、Log-Spaced CPB（解决窗口尺寸插值问题）、余弦相似度 attention、SimMIM 自监督预训练。ADE20K 上达到 61.4 mIoU。

**Video Swin Transformer（Liu et al., 2022）：** 把 2D shifted window 扩展到 3D 时空域。时间维度上也做移位窗口，实现跨时间帧的信息流动。Kinetics-400 上 84.8 top-1。这证明了 Swin 的核心设计不限于 2D 图像，可以推广到视频任务。

**CSWin Transformer（Dong et al., 2022）：** 十字形窗口——水平条带 + 垂直条带的 attention。感受野比 7×7 更大，可以用来替代 Swin 在需要更大感受野的场景。CSWin-B ImageNet 85.4%。

**Focal Transformer（Yang et al., 2021）：** 粗细粒度窗口——窗口中心区域做细粒度 attention，外围做粗粒度 attention。解决了 Swin 的局部限制，在不增加复杂度的情况下扩大了等效感受野。

**ConvNeXt（Liu et al., 2022）：** 这不是 Swin 的后继，而是**从 Swin 反推现代 CNN 设计**。作者用 Swin 的设计理念（大 kernel size、GELU、LayerNorm、patchify stem）逐步改良 ResNet，最终得到一个纯卷积架构，效果和 Swin 相当。核心启示：CNN 和 Transformer 之间的 gap 主要来自训练技巧和架构设计细节，不是本质差异。

**整体影响：**
- 引用量（截至 2026 年）超过 25,000 次
- 成为 Mask R-CNN、Cascade R-CNN、HTC、UperNet 等框架的标准 backbone
- 推动了 "ConvNeXt 化" 运动——很多 CNN 架构参考 Swin 的设计理念
- 为后来的视觉 foundation model（如 InternImage、DeiT III）提供了设计参照

## 十三、范式意义

**Swin Transformer 的范式意义：它是"backbone paradigm shift"级别的贡献。**

具体来说，Swin 完成了三件事，每一件放在当时来看都是重要突破：

### 13.1 打破了 "Transformer 只能做分类" 的认知

2020 年 ViT 之后，社区的一个自然疑问是：Transformer backbone 到底能不能做检测和分割？如果能，效果能追上 CNN 吗？

Swin 用实验给出了明确答案：**不仅能，而且在三个主流 benchmark 上全面超越 CNN backbone（ResNe(X)t）**。COCO 上 +4.2 box AP，ADE20K 上 +4.4 mIoU——这些差距不是"略好"或"相当"，而是有明显优势。

这从根本上改变了社区对 Transformer backbone 的认知：它不是 CNN 的补充或替代品中的"备选"，而是可以**全面接管** CNN backbone 生态位的架构。

### 13.2 提出了一个通用的 backbone 设计模板

Swin 的 4 级金字塔 + 局部 attention + 交错移位的设计模板，被后续大量工作沿用：
- CSWin 用十字形窗口替代方形窗口
- Focal Transformer 用粗细粒度的窗口划分
- Video Swin 把 2D 窗口扩展到 3D
- ConvNeXt 把 Swin 的设计理念翻译回 ConvNet

这些工作的共性：**都是围绕 Swin 的"局部窗口 + 层级"范式做的改进或变体**。也就是说，Swin 定义了一个坐标系，后续工作在这个坐标系里做调整。

### 13.3 证明了局部性在视觉 Transformer 中的重要性

ViT 的设计哲学是：attention 是万能的，全局 attention 可以学习到所有需要的模式。Swin 证明并非如此——**引入局部性假设（window attention）反而更好**。这不仅降低了复杂度，还引入了有益于视觉任务的归纳偏置。

这为后续的视觉 Transformer 工作提供了重要参考：不必追求全局 attention 的理想化假设，局部 + 层级的设计在实践中更有效。

### 13.4 分类：foundation node

在技术树中，Swin 是一个 **foundation node（基础节点）**——它建立了一个新的技术基线，后续大量工作都是在这个基线上做增量改进。

它不是 bridge node（桥梁节点），因为 Swin 并不连接两个不同的研究领域。它也不是 refinement node（精化节点），因为它不是对已有范式的微调改进。它是一个开辟了新方向的 foundation node。

## 十四、局限性与后续方向

### 14.1 局部感受野的理论上限

虽然交错移位窗口实现了跨窗口连接，但每个 token 每次只能跨越一个窗口边界。Swin-B（24 层，12 对 W/SW）的有效感受野在 84×84 像素左右（按 16× 下采样计）。对于需要全局感受野的任务（如全景分割中的实例间关系建模、远程依赖的语义关系），这不如直接做全局 attention。

后续工作试图解决这个问题：
- **Focal Transformer（2021）：** 粗细粒度注意力——窗口中心区域做细粒度 attention，外围做粗粒度 attention，等效感受野更大
- **CSWin Transformer（2022）：** 十字形窗口——水平条带 + 垂直条带，每个 token 可以和整行/整列交互
- **InternImage（2023）：** 用 deformable convolution 替代 attention，灵活控制感受野

### 14.2 分类精度并非极致

Swin-B（83.5%, 224² ImageNet-1K）在分类上虽然好于 DeiT-B（81.8%），但不如有蒸馏的 DeiT-B（83.8%）和 CvT-21（82.5% 但参数量小 3 倍）。

这不是 Swin 的设计缺陷——它的设计目标从来不是刷分类 SOTA，而是做通用 backbone。在分类任务上，全局 attention（ViT/DeiT）没有多分辨率和效率的约束，可以做更全局的建模。但在需要多尺度特征和效率的检测/分割任务上，Swin 的优势就体现出来了。

### 14.3 窗口大小 $M=7$ 的经验性选择

论文没有系统研究 M 的不同取值对效果的影响。Table 4 只对比了 M=7 和 M=4（81.3 vs 80.9, -0.4）。更大的 M 可能带来更好的性能，但也会增加 $O(M^2)$ 的计算成本。

后续的 Swin V2 对 M 的选择做了更系统的分析，发现更大的窗口（M=12 甚至 M=24）需要配合 log-spaced CPB（相对位置偏置的插值方案）。

### 14.4 Attention Mask 的推理开销

SW-MSA 的 cyclic shift + mask 操作虽然比滑动窗口快，但仍然有一定开销。每个窗口需要维护 $M^4$ 大小的 attention mask（$\text{nWindows} \times M^4$）。对于 Swin-T 的 Stage 1（64 个窗口，M=7），mask 矩阵约 64 × 2401 ≈ 153K 个元素——不是瓶颈但非零成本。

### 14.5 训练成本

- Swin-L（197M 参数）在 ImageNet-22K 上预训练需要 64 个 V100 GPU × 7 天
- 完整的实验（ImageNet + COCO + ADE20K + 消融）估计需要数百个 GPU 天
- 这在 2021 年属于正常的大规模实验，但对没有充足算力的研究组来说很难复现

### 14.6 与 NLP unified 的目标并未实现

论文在 Introduction 中提到：希望 Swin 能推动 CV 和 NLP 的统一架构。但后续的发展表明，CV 和 NLP 走的是不同的方向——NLP 走向了 decoder-only（GPT 路线），CV 走向了 encoder-only 或 encoder-decoder 混合。Swin 的 shifted window 方案并没有被 NLP 社区采纳。这个"统一"的目标在实践上并未实现。

## 十五、小结

直白说，Swin Transformer 的价值不只是提出了一个能打的视觉 backbone，而是把 **"Transformer 可以做通用视觉 backbone" 这条路线从"理论上可行"推进到了"实践上主导"的阶段**。

在它之前，社区对 Transformer backbone 的认知是：分类不错，但检测/分割不如 CNN。在它之后，情况反转了——检测/分割也可以用 Transformer，而且效果比 CNN 更好。

这个转变靠的是两个核心洞察：

1. **视觉的特征表示需要多尺度，而 Transformer 不需要全局 attention。** 局部窗口 attention + 层级下采样就够了，而且更高效。这个洞察打破了 ViT 隐含的"全局 attention 最好"的假设。

2. **信息跨窗口流动可以通过结构设计（窗口偏移）实现，不需要改变 attention 机制本身。** 窗口偏移 + cyclic shift 的工程实现既保持了 $O(N)$ 复杂度，又解决了窗口之间的连通性问题。这个设计的巧妙之处在于：它把"如何解决跨窗口连接"这个问题从 attention 机制内部转移到了架构设计层面。

在技术树中，Swin 是一个 foundation node。它所建立的"层级局部 attention"范式，影响了后续所有视觉 Transformer backbone 的设计。而 Swin 本身也被证明可以一直 scale up 到 3B 参数（Swin V2），证明了这条路可以走得很远。
