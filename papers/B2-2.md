# Swin Transformer: Hierarchical Vision Transformer using Shifted Windows（2021）

**论文：** [arXiv](https://arxiv.org/abs/2103.14030) · Ze Liu, Yutong Lin, Yue Cao, Han Hu et al. · 2021

---

# 一、论文解决的问题

在 ViT（2020）成功将 Transformer 引入视觉之后，一个核心矛盾浮现出来：

- **NLP 的 Transformer**：输入是固定长度的 token 序列，self-attention 在全局范围内计算，复杂度 $O(N^2)$
- **视觉任务的需求**：图像分辨率远高于文本序列长度（$224\times224$ 的图像对应 196 个 16×16 patch），且视觉实体存在**多尺度**特性（大物体 vs 小物体需要不同粒度的表示）

ViT 的做法是：
1. 用固定大小的 patch（如 16×16）将图像分块
2. 将所有 patch 送入全局 Transformer

这带来了两个问题：
- **计算量爆炸**：$O(N^2)$ 的 attention 对高分辨率图像不可扩展（$O((H/16 \cdot W/16)^2)$）
- **缺少多尺度层级结构**：CNN 固有的金字塔结构（stage 1→2→3→4）被抛弃了，这对于检测/分割等密集预测任务至关重要

**Swin Transformer 的核心问题**：能否设计一种 Transformer 架构，既保留 self-attention 的表达能力，又具备 CNN 的层级多尺度特性和线性计算复杂度？

# 二、核心方法 / 贡献

Swin Transformer 提出了两个关键设计：

### 1. 层级化架构（Hierarchical Feature Maps）

不像 ViT 那样只输出单一分辨率的特征图，Swin Transformer 像 CNN 一样逐 stage 下采样：

```
输入图像 (H×W×3)
    ↓
Stage 1: H/4 × W/4 × C      （类似 ResNet stage 1）
    ↓
Stage 2: H/8 × W/8 × 2C
    ↓
Stage 3: H/16 × W/16 × 4C
    ↓
Stage 4: H/32 × W/32 × 8C
```

这样就能方便地接入 FPN、U-Net 等需要多尺度特征的检测/分割框架。

### 2.  shifted window 自注意力（Shifted Window Self-Attention）

这是论文的核心创新：

- **W-MSA（Window-based MSA）**：将特征图划分为不重叠的窗口（如 7×7），只在每个窗口内计算 self-attention。复杂度从 $O(H^2W^2)$ 降到 $O(M^2HW)$，其中 $M$ 是窗口大小——**与图像尺寸呈线性关系**。
- **SW-MSA（Shifted Window MSA）**：在连续的两层之间，将窗口偏移 $\lfloor M/2 \rfloor$ 个像素。通过这种"错位"，原本不在同一窗口的 patch 可以互相通信，实现了跨窗口的信息交互。

```
Layer 1 (W-MSA)      Layer 2 (SW-MSA)
┌────┬────┬────┐     ┌────┬────┬────┐
│  1  │  1  │  2  │     │  1  │  1  │  1  │
├────┼────┼────┤  →  ├────┼────┼────┤
│  1  │  1  │  2  │     │  1  │  2  │  2  │
├────┼────┼────┤     ├────┼────┼────┤
│  3  │  3  │  4  │     │  2  │  3  │  4  │
└────┴────┴────┘     └────┴────┴────┘
```

图中数字代表窗口归属。Layer 1 中四个窗口各自独立做 self-attention；Layer 2 通过 shift 操作让边界区域实现跨窗口通信。

# 三、公式解释

### 公式 1：标准 MHA vs 窗口 MHA 复杂度对比

标准多头自注意力（global MSA）的复杂度：

$$\Omega(\text{MSA}) = 4hwC^2 + 2(hw)^2C$$

窗口多头自注意力（W-MSA）的复杂度：

$$\Omega(\text{W-MSA}) = 4hwC^2 + 2M^2hwC$$

其中 $h,w$ 是特征图的高和宽，$C$ 是通道数，$M$ 是窗口大小。

**关键差异**：global MSA 的第二项是 $(hw)^2$（图像面积平方），而 W-MSA 的第二项是 $M^2 \cdot hw$（窗口面积 × 图像面积）。当 $M=7$ 时，$M^2=49$，而 $(hw)^2$ 在 $h=w=56$ 时是 3136——差了 64 倍。

### 公式 2：窗口划分与合并的 patch merging

相邻 stage 之间通过 Patch Merging 实现 2× 下采样：

- 将 2×2 邻域的 4 个 patch 拼接（concatenate）
- 经过 LayerNorm + 线性层投影到 $2C$ 维
- 特征图分辨率减半，通道数翻倍

这类似于 CNN 中的 stride=2 卷积 + 通道翻倍。

# 五、实验结果

| 任务 | 数据集 | Swin-B | 前 SOTA | 提升 |
|------|--------|--------|---------|------|
| 任务 | 数据集 | Swin-L + 框架 | 前 SOTA | 提升 |
|------|--------|---------------|---------|------|
| 图像分类 | ImageNet-1K top-1 | 87.3%（384×384 输入） | - | 超越 DeiT-B |
| 目标检测 | COCO test-dev box AP | 58.7（HTC++） | 56.0 | +2.7 |
| 实例分割 | COCO test-dev mask AP | 51.1（HTC++） | 48.5 | +2.6 |
| 语义分割 | ADE20K val mIoU | 53.5（UperNet） | 50.3 | +3.2 |

注意：87.3% 是 **Swin-L** 在 384×384 高分辨率输入下的结果；Swin-B 在 224×224 下为 83.5%。检测/分割的结果均使用 **HTC++**（检测）和 **UperNet**（分割）作为下游框架，数字反映的是 Swin backbone + 强框架的整体性能。Swin Transformer 在密集预测任务上的优势极其显著，验证了层级多尺度设计的价值。

# 六、工程意义

1. **通用的视觉 backbone**：Swin Transformer 成了 Mask R-CNN、Cascade R-CNN、UperNet 等框架的直接替代品，即插即用
2. **线性复杂度使得高分辨率输入成为可能**：为后续 Vision Transformer 处理大尺寸图像（如 1536×1536）铺平了道路
3. **Shifted Window 的延展性**：这一设计也被 ConvNeXt 等后续工作借鉴（ConvNeXt 的 7×7 深度可分离卷积实质上模拟了 Swin 的局部窗口操作）
4. **Backbone 竞赛的终结者**：Swin 之后，"CNN vs Transformer"的争论基本结束——Transformer 也可以做好密集预测，CNN 也可以从 Transformer 借鉴设计。两条路线开始融合
