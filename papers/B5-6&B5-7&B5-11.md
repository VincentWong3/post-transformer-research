## 一、先搞清楚"问题在哪里"

2020 年 Vision Transformer（ViT）横空出世，宣布 CNN 不再是对视觉做自注意力的必须前置。ViT 直接把图像切成 patch 序列丢进 Transformer encoder，在 ImageNet 上跟 CNN 打平甚至反超。但 ViT 有一个公开的秘密：它严重依赖大规模预训练（JFT-300M），数据量不够时反而比不过 ResNet。

这个矛盾在视频领域被放大了。

视频比图像多一个时间维度，标准的做法是把 $T$ 帧画面当成输入。如果用 ViT 直接处理视频，patch 数量会暴涨到 $T \times (H/16) \times (W/16)$ 全局自注意力的复杂度 $O(N^2)$，让 $N$ 几乎不可接受。2021 年的 Timesformer 和 ViViT 都在解决这个问题——核心思路是做时空分解：空间自注意力只看单帧内部，时间自注意力只跨帧对齐，把 $O((T \times S)^2)$ 拆成 $O(T \times S^2 + T^2 \times S)$。这在复杂度上确实是进步，但它们保留了一个关键假设：**注意力在全图上算**。

"全图"的意思是：空间注意力在一帧内，每个 patch 和所有其他 patch 做交互。这在图像上勉强可行（224×224 约 196 个 patch），但在视频里 $T \times S$ 的总量仍然很大。更本质的问题是：全局注意力缺少归纳偏置。CNN 靠参数共享和局部连接天然告诉模型"空间近的像素应该先交互"，而 ViT 类的全局注意力完全依靠数据去学习这种结构。

问题来了：**当数据量不够大或者任务对局部运动模式敏感时，全局注意力到底是不是最优的？**

Convolutional 路线上，I3D、C3D 等 3D CNN 用 3D 卷积核直接建模时空局部关系，效果好但感受野固定、扩大感受野靠堆层。Transformer 路线上，全局注意力感受野大但计算贵、缺乏局部偏置。两条路线各有所长，但没有一个方法能把两者的优点合起来——直到 Video Swin Transformer 出现。

## 二、旧路线的真正问题

在 Video Swin Transformer 之前，视频 Transformer 的工作主要分两类。

**第一类：时空分解型。**Timesformer、ViViT 为代表。它们把自注意力拆成空间和时间两个步骤。以 Timesformer 为例，交替做空间自注意力（同一帧内 patch 之间）和时间自注意力（不同帧的同位置 patch 之间）。复杂度从 $O((T \times S)^2)$ 降到约 $O(T \times S^2 + T^2 \times S)$ 

但这个分解依赖一个很强的假设：空间和时间可以解耦。对于像 Kinetics 这种以场景外观为主的数据集（"弹钢琴"——看画面就知道在弹钢琴，不需要追踪手指的时序运动），分解问题不大。但对于 Something-Something 这类高度依赖时序运动模式的数据集（"把东西从一边推到另一边"——单帧画面几乎无法判断动作类别），分解空间的 patch 交互和时间的 patch 交互可能丢失重要的**空时联合特征**。

**第二类：全局注意力型。**直接在整个视频上做全局自注意力。ViViT 的 Factorised Encoder 变体、Video Vision Transformer 等都属于这类。感受野最大，但计算量也最大， $O((T \times S)^2)$ 的复杂度在视频里几乎不可扩展。

更根本的问题是：**全局注意力没有局部性归纳偏置。** 在图像领域，Swin Transformer（2021 年 3 月）已经证明了一件重要的事：在视觉 Transformer 里引入局部窗口注意力不仅节省计算，而且能提升性能。局部窗口迫使模型先关注近邻区域，再通过移位窗口实现跨窗口信息交换。这个设计把 CNN 的局部连接和 Transformer 的动态权重结合在了一起——而且效果好于纯全局注意力。

但 Swin Transformer 是 2D 的。它处理的是单张图像，没有时间维度。视频里真正需要的是 3D 的局部性：**同时在空间和时间上限制注意力范围**。比如一个动作可能持续几帧，局部区域在空间上相邻、在时间上也相邻。如果只用 2D 空间局部注意力+全局时间注意力，时间维度的交互仍然没有局部性偏置。

所以问题进一步收窄为：**怎么把 Swin Transformer 的局部窗口 + 移位窗口机制从 2D 空间扩展到 3D 时空域？**

## 三、这篇论文的核心思路

**这篇论文的核心思路是：将 Swin Transformer 的局部窗口自注意力和移位窗口机制从 2D 扩展到 3D，在视频 Transformer 中引入时空局部性归纳偏置。**

具体来说，Video Swin Transformer 做了三件事：

1. **3D patch 分割**：把视频的 $T$ 帧分成 3D patches（patch size 为 $2 \times 4 \times 4$ 分别是时间×高×宽×3 通道）。
2. **3D 窗口自注意力**：在 3D 窗口（大小为 $P \times M \times M$ 时间×高×宽）内部计算自注意力，而不是在全视频所有 patches 上算。
3. **3D 移位窗口**：在相邻层之间沿时间和空间方向各移位半个窗口大小，让不同窗口之间的信息可以流通。

这个设计的核心洞察是：**视频里的局部性不止是空间上的——时间上相邻的帧之间，同一个物体的运动也是局部的。** 一个在 $t$ 帧位于 $(x,y)$ 的物体，大概率在 $t+1$ 帧还在 $(x+\Delta x, y+\Delta y)$ 附近。因此把时间维也划入局部窗口，比先做空间再做时间的分解更自然。

从技术谱系来看：

```text
CNN backbone route
   ↓
ViT (global attention)
   ↓
Swin Transformer (2D local window + shift)
   ↓
Video Swin Transformer (3D local window + shift)
```

它直接把 Swin 在图像上的成功复制到了视频——而且因为可以复用 ImageNet 预训练的 Swin 权重，预训练数据需求大大降低。

## 四、整体结构与 forward 流程

Video Swin Transformer 的整体架构继承了 Swin Transformer 的 4 阶段层级设计，但所有关键操作都扩展到了 3D。

forward 流程如下：

```text
输入视频: T × 3 × H × W
 ↓
3D patch 分割 (patch size: 2×4×4×3)
 ↓
3D 线性嵌入 → tokens: (T/2) × (H/4) × (W/4) × C
 ↓
Stage 1: 3D Swin Transformer blocks (window size P×M×M)
   交替: 3DW-MSA → FFN → 3DSW-MSA → FFN
 ↓
3D Patch Merging (空间 2× 降采样，时间维不变)
 ↓
Stage 2: 同上结构，分辨率减半，通道加倍
 ↓
3D Patch Merging
 ↓
Stage 3: 同上
 ↓
3D Patch Merging
 ↓
Stage 4: 同上
 ↓
全局平均池化 → 分类头
```

关键的设计选择：

- **不沿时间维降采样**：整个网络保留 $T/2$ 个时间 token（论文中用 32 帧输入，每 2 帧合为一个 patch，得到 16 个时间 token），只在空间维度做 $2\times$ 降采样。这么做是为了保持时序分辨率，让网络保留足够的时间信息来建模运动。
- **4 阶段层级结构**：同 Swin-T/S/B/L，输出特征图的多尺度金字塔，便于与下游任务（检测、分割）结合。
- **3D 窗口大小**： $P \times M \times M$ 默认 $P=8, M=7$（论文 Table 1 中的默认配置）。$P$ 控制时间范围， $M$ 控制空间范围（消融实验中 $P=2$ 或 $P=4$ 也可选）。

## 五、核心模块逐个解释

### 5.1 3D Patch 分割与 3D 线性嵌入

Swin Transformer 把 $H \times W \times 3$ 的图像切成 $4 \times 4$ patches（stride 4），每个 patch 映射为 $4 \times 4 \times 3 = 48$ 维的特征向量，再线性投影到 $C$ 维。

Video Swin Transformer 对 $T \times 3 \times H \times W$ 的视频做 **3D patch 分割**：用 $2 \times 4 \times 4$ 的 3D patch（时间上 2 帧、空间上 $4 \times 4$ 像素），stride 也为 $2 \times 4 \times 4$ 每个 3D patch 被展平为 $2 \times 4 \times 4 \times 3 = 96$ 维向量，然后线性投影到 $C$ 维。

为什么要用 $2 \times 4 \times 4$ 而不是 $1 \times 4 \times 4$ 因为时间维的 stride 为 2 意味着相邻帧的信息在 patch 层就已经被融合了——这对于建模短时运动模式有利。同时时间 token 数量从 $T$ 减少到 $T/2$ 也降低了后续计算的负担。

### 5.2 3D Shifted Window based MSA

这是 Video Swin Transformer 最核心的模块。

**常规 3D window MSA（3DW-MSA）**：把所有的 3D tokens 分成不重叠的 3D 窗口，每个窗口大小为 $P \times M \times M$ 假设特征图大小为 $T' \times H' \times W'$ 那么窗口数量为 $(T'/P) \times (H'/M) \times (W'/M)$ 在每个 3D 窗口内部做标准的自注意力。

**为什么只做窗口内部？** 因为这样复杂度从全局的 $O((T' \times H' \times W')^2)$ 降到了 $O((T' \times H' \times W') \times (P \times M \times M))$ 即 $O(N \times K)$ $N$ 是总 token 数， $K$ 是窗口大小）。对于默认窗口 $P=8, M=7$，每个窗口内 $8 \times 7 \times 7 = 392$ 个 tokens，计算量远小于全局。

**移位 3D window MSA（3DSW-MSA）**：只在窗口内部做注意力的问题是窗口之间没有信息交换。Swin 的解决方案是：在相邻的 block 之间把窗口移位半个窗口大小，即移位量 $(\lfloor P/2 \rfloor, \lfloor M/2 \rfloor, \lfloor M/2 \rfloor)$ 这样，前一层的窗口边界在新一层的窗口中心，跨窗口信息通过这个"重叠"机制传递。

两个连续 block 的交替模式：

```text
Block 1: 3DW-MSA → LayerNorm → FFN → LayerNorm
Block 2: 3DSW-MSA → LayerNorm → FFN → LayerNorm
```

### 5.3 3D 相对位置偏置

在计算自注意力时，Video Swin Transformer 为注意力矩阵添加了一个 3D 相对位置偏置项 $B$ 

$$\text{Attention}(Q, K, V) = \text{SoftMax}(QK^T / \sqrt{d} + B) V$$

$B$ 的形状是 $P^2 \times M^2 \times M^2$ 对于每个 query 位置，相对于其他 key 位置的偏移在 3D 空间中有 $P \times M \times M$ 种可能，而注意力矩阵是二维的 $(\text{window tokens}) \times (\text{window tokens})$ 所以 $B$ 是 $\text{tokens}^2$ 维的。具体构造方法：先计算每个 token 对之间的 3D 相对位置偏移 $(t_{rel}, h_{rel}, w_{rel})$ 然后查表得到偏置值。

这个偏置的作用是告诉模型：**这些 patches 在时间和空间上是怎么排列的**。因为窗口内的 patches 没有用绝对位置编码，如果没有相对位置偏置，模型会把窗口内的 patches 当成一个无序集合来对待——这在视觉任务中是不合理的，因为相邻像素天然更相关。

### 5.4 3D Patch Merging

Swin Transformer 在每个阶段结束时做 Patch Merging（类似 CNN 的池化），把相邻的 $2 \times 2$ patches 合并为一个，通道数翻倍。

Video Swin Transformer 做 **3D Patch Merging**：只做空间维度的 $2\times$ 降采样（ $H'$ 和 $W'$ 各减半），时间维度保持不变（ $T'$ 不变）。具体的，把 $2 \times 2$ 空间邻域的 4 个 tokens 拼接到一起（ $4C$ 维），然后线性投影到 $2C$ 维。

为什么不在时间维也做降采样？论文的设计意图是：视频的分类和时序建模需要足够的时间分辨率，过早降低时间分辨率会丢失运动的细节。这个选择在 Something-Something v2 这种时序敏感数据集上被验证是正确的。

## 六、核心公式与数学直觉

### 6.1 Attention 公式

$$\text{Attention}(Q, K, V) = \text{SoftMax}(QK^T / \sqrt{d} + B) V$$

其中 $Q, K, V \in \mathbb{R}^{N \times d}$ 是 query、key、value 矩阵（ $N$ 是窗口内的 token 数）， $d$ 是 head 维度， $B \in \mathbb{R}^{N \times N}$ 是 3D 相对位置偏置。

这个公式和标准的多头自注意力唯一的不同就是 $+B$。$B$ 的值并非直接可学习——论文定义了一个更小的可学习参数表 $\hat{B} \in \mathbb{R}^{(2P-1) \times (2M-1) \times (2M-1)}$，覆盖时间、高、宽三个轴上各 $(-P+1, P-1)$ 和 $(-M+1, M-1)$ 的相对偏移范围。在运行时，根据每对 token 的实际 3D 相对偏移从 $\hat{B}$ 中查表得到对应的 $B$ 值。

### 6.2 3D 窗口注意力与移位窗口

两个连续 block 的交替公式：

$$\hat{z}^l = 3\text{DW-MSA}(LN(z^{l-1})) + z^{l-1}$$
$$z^l = FFN(LN(\hat{z}^l)) + \hat{z}^l$$
$$\hat{z}^{l+1} = 3\text{DSW-MSA}(LN(z^{l})) + z^{l}$$
$$z^{l+1} = FFN(LN(\hat{z}^{l+1})) + \hat{z}^{l+1}$$

这是标准的 Transformer block 结构（LayerNorm + attention + residual + FFN），只是把 attention 分别替换为 3D window 和 3D shifted window MSA。

### 6.3 计算复杂度分析

假设特征图大小为 $T' \times H' \times W'$ 通道数为 $C$ 窗口大小为 $P \times M \times M$ 

- **全局 3D attention**： $O(T' H' W' \times T' H' W' \times C)$
- **3D window attention**： $O(T' H' W' \times P M M \times C)$

因为 $P M M \ll T' H' W'$ 且复杂度是 token 数乘以窗口大小（线性），而不是 token 数的平方（平方）。具体数值上看，在 stage 1 中 $T' = 16, H' = W' = 56$，全局注意力的 $N = 16 \times 56 \times 56 = 50176$。默认窗口 $8 \times 7 \times 7 = 392$ 个 tokens 时的复杂度是全局注意力的 $N / K = 50176 / 392 \approx 128$ 分之一（若使用更小的 $P=2$ 窗口则是约 500 倍）。

这就是局部窗口核心的价值：把不可扩展的平方复杂度变成了可扩展的线性复杂度，同时保留了 Transformer 的动态权重计算。

### 7.2 3D 移位窗口示意

论文的 Figure 3 展示了 3D 窗口在两个连续 block 之间的移位过程（论文中称为"Cyclic Shift"的概念从 2D Swin 扩展到 3D）：

```text
Block 1 (regular window):
┌─────┬─────┬─────┐
│W(1,1)│W(1,2)│W(1,3)│
├─────┼─────┼─────┤
│W(2,1)│W(2,2)│W(2,3)│
├─────┼─────┼─────┤
│W(3,1)│W(3,2)│W(3,3)│
└─────┴─────┴─────┘
        ↓ 移位 (P/2, M/2, M/2)
Block 2 (shifted window):
┌─────┬─────┬─────┐
│W'(1,1)│W'(1,2)│W'(1,3)│
├─────┼─────┼─────┤
│W'(2,1)│W'(2,2)│W'(2,3)│
├─────┼─────┼─────┤
│W'(3,1)│W'(3,2)│W'(3,3)│
└─────┴─────┴─────┘
```

3D 版本把上面的 2D 网格扩展到 3D 立方体，每个维度（ $T, H, W$ 各跨半个窗口移位。

实际上，移位后的窗口数量会增加（在边界处产生不完整的窗口），Swin Transformer 通过 Cyclic Shift（循环移位）+ masked attention 来解决这个效率问题——不需要手动处理边界窗口，而是通过循环移位把所有 tokens 对齐成完整窗口，再通过 mask 屏蔽不应该相连的位置。这个技巧在 3D 版本中同样适用。

## 八、复杂度、效率与可扩展性分析

### 8.1 复杂度对比

| 方法 | 注意力复杂度 | 特点 |
|------|-------------|------|
| ViT（全局） | $O(N^2)$ | 平方复杂度，不可扩展 |
| Timesformer（时空分解） | $O(S^2 + T^2)$ | 线性复杂度，空间时间解耦 |
| Video Swin（3D 窗口） | $O(N \cdot P M M)$ | 线性复杂度，保留时空局部性 |

其中 $N = T' \times H' \times W'$ 为总 token 数， $S = H' \times W'$ 为单帧空间 token 数， $P \times M \times M$ 为 3D 窗口大小。

### 8.2 模型参数与计算量

从论文实验结果和官方 GitHub 数据：

**Kinetics-400 基准：**

| 模型变体 | 参数量 | FLOPs | ImageNet 预训练 | top-1 acc@1 |
|---------|-------|-------|----------------|-----------|
| Swin-T | 28M | 87.9G | IN-1K | 78.8% |
| Swin-S | 50M | 165.9G | IN-1K | 80.6% |
| Swin-B | 88M | 281.6G | IN-1K | 80.6% |
| Swin-B | 88M | 281.6G | IN-22K | 82.7% |
| Swin-L | ~200M | 604.8G | IN-21K | 84.9% |

关键观察：**Swin-L（约 200M 参数，IN-21K 预训练）在 Kinetics-400 上达到 84.9% top-1。** 这是一个非常重要的对比数字——ViViT-H 也达到了 84.9%，但 ViViT-H 有 647.5M 参数，预训练数据是 JFT-300M（3 亿张图）。而 Swin-L 只有约 200M 参数，预训练数据是 IN-21K（约 1400 万张图）。**Swin-L 用约 1/20 的预训练数据和约 1/3 的模型参数达到了同样的精度。**

### 8.3 效率的意义

这个结果说明两件事：

1. **局部性归纳偏置有效**：在数据量有限的情况下（IN-21K vs JFT-300M），带局部偏置的模型学得更好。

### 关键 forward 流程

简化版本的 `SwinTransformer3D.forward_features()` 伪流程（从真实代码提取结构）：

```python
def forward_features(self, x):
    x = self.patch_embed(x)           # 3D patch embedding: (B, T, 3, H, W) → (B, C, T/2, H/4, W/4)
    x = self.pos_drop(x)              # 可选 dropout
    
    for i, layer in enumerate(self.layers):
        x = layer(x)                  # 每个 BasicLayer 处理一个 stage
    
    x = self.norm(x)
    x = self.avgpool(x)               # 全局平均池化
    return x
```

`SwinTransformerBlock3D` 的核心 forward 代码如下（简化）：

```python
def forward(self, x):
    shortcut = x
    x = self.norm1(x)
    
    # 3D window attention (regular or shifted)
    x = self.attn(x, shift_size=self.shift_size)
    
    x = shortcut + self.drop_path(x)
    
    # FFN
    x = x + self.drop_path(self.norm2(self.mlp(x)))
    return x
```

### 3D 窗口注意力的实现

`WindowAttention3D` 是真正的核心。它的 `forward` 函数执行以下步骤：

1. **将特征图划分成 3D 窗口**（使用 `window_partition_3d` 函数）
2. **在每个窗口内部计算 QKV 注意力**
3. **加上 3D 相对位置偏置**
4. **将窗口重新拼接回特征图**（使用 `window_reverse_3d` 函数）

对于 shifted window 版本，会先对特征图做 `torch.roll`（循环移位），再执行窗口划分，最后在 attention 计算中通过 mask 屏蔽不应该相连的边界位置。

### 如何加载 ImageNet 预训练权重

这是 Video Swin Transformer 的一个工程亮点。`init_weights()` 函数在加载 2D Swin 权重时，会将 2D 权重适配到 3D 空间：

- 对于 patch embedding 层：将 2D 卷积权重在时间维上重复初始化
- 对于注意力层的 qkv 权重：直接复用（因为特征维度一致）
- 对于相对位置偏置：通过插值将 2D 偏置表扩展到 3D

这个设计使得视频模型可以"站在巨人的肩膀上"——直接利用海量 ImageNet 预训练的图像 Swin 权重，而不是从头训练。

## 十四、局限性与后续方向

### 14.1 局限

1. **3D 窗口大小是固定的超参数。** 默认 $P=8$ 的时间窗口虽然能覆盖较长的时域范围，但仍然是固定大小的。某些长程运动模式可能需要更大的时域感受野，而增大 $P$ 会直接增加计算量。

2. **时间维不做降采样可能不是最优的。** 对于某些任务（如动作识别），更多的时间 token 确实保留更多信息，但对于图像级别的下游任务（如视频分割），在高层可能也需要时序池化。

3. **依赖 ImageNet 预训练。** 虽然这是优势（训练需求低），但也意味着如果 ImageNet 上没有对应的 2D Swin 权重，某些自定义架构无法直接受益。

4. **全局感受野依赖于窗口移位。** 虽然多层移位后理论上能覆盖全局，但实际效果取决于 block 的层数。浅层网络的感受野可能仍然有限。

### 14.2 后续方向

Video Swin Transformer 启发了几个后续方向：

- **Video Swin v2**：改进相对位置偏置、添加 log-spaced continuous position bias，使得模型可以适应不同的输入分辨率。
- **将局部窗口应用于视频预训练**：VideoMAE 等自监督方法使用 Video Swin 作为 backbone。
- **混合架构**（如 UniFormer）：结合 3D 卷积和 3D 自注意力的混合设计。
