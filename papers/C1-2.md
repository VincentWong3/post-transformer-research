# # End-to-End Object Detection with Transformers（2020）

**论文：** [arXiv](https://arxiv.org/abs/2005.12872) · 作者：Nicolas Carion, Francisco Massa, Gabriel Synnaeve, Nicolas Usunier, Alexander Kirillov, Sergey Zagoruyko · 年份：2020

## 一、先搞清楚坑在哪

目标检测是计算机视觉的核心任务之一。给定一张图像，模型需要回答两个问题：**"图像里有什么物体？"** 和 **"这些物体在图像的哪个位置？"**

在 DETR 出现之前（2015-2019），主流目标检测器大致分为两类：

1. **两阶段检测器（如 Faster R-CNN）**：先由区域提议网络（RPN）生成数千个候选框，再对这些候选框进行分类和回归。
2. **单阶段检测器（如 YOLO、SSD、RetinaNet）**：直接在特征图上预设大量锚框（anchor boxes），同时预测类别和偏移量。

这两种范式有一个共同点：**都需要大量手工设计的组件**。锚框的大小和长宽比需要人工设定、NMS（非极大值抑制）的阈值需要调参、正负样本的分配策略需要精心设计。整个 pipeline 像一条流水线，每个环节都有各自的超参数和假设。

## 二、现有方法的真正问题

### 2.1 锚框的诅咒

锚框是预定义的参考框。以 Faster R-CNN 为例，在每个特征图位置放置 9 个不同形状的锚框。这意味着：

- 对于一张 800×600 的图像，特征图尺寸约 50×38，需要生成约 **17,100 个锚框**
- 其中绝大多数是负样本（不包含物体）
- 锚框的尺寸和长宽比需要人工设计，通常基于训练数据集的统计分布
- 小物体和极端长宽比的物体容易被锚框漏掉

### 2.2 NMS 的尴尬

NMS 是后处理步骤：对同一类别的重叠检测框，保留置信度最高的，抑制其他框。问题在于：

- **阈值敏感**：阈值设高了会漏检（两个相邻物体被合并），设低了会重复检测
- **不可微分**：NMS 无法参与端到端的梯度反向传播
- **不是全局最优**：NMS 只做局部贪心决策，不考虑全局检测结果的一致性

### 2.3 正负样本分配

训练时需要决定哪些锚框负责检测哪个物体。这涉及复杂的匹配策略（如 IoU 阈值、最大重叠分配等）。这些策略同样包含大量超参数，且在不同数据集上需要重新调整。

**直白说**：整个 pipeline 像一个需要手动拧很多螺丝的机器，每个螺丝的松紧程度都会影响最终性能。

## 三、DETR 的核心思路

DETR 的核心洞察非常简洁：**把目标检测看作一个集合预测问题（set prediction problem），直接用 Transformer 的 encoder-decoder 架构来预测物体的集合。**

```
核心公式：
目标检测 = 图像特征提取（CNN） + 集合预测（Transformer）
```

具体来说：

1. 用 CNN 提取图像特征
2. 将特征展平后输入 Transformer encoder（自注意力）
3. Transformer decoder 接收 N 个可学习的 object queries（可理解为 N 个"空盒子"）
4. decoder 通过交叉注意力，让每个 query 从图像特征中"抓取"一个物体的信息
5. 每个 query 最终输出：类别预测 + 边界框坐标

**关键突破**：整个流程不需要锚框、不需要 NMS、不需要手工设计的正负样本分配策略。模型通过二分图匹配（bipartite matching）直接学习预测与真实物体的对应关系。

### 4.2 前向过程详解

让我们一步步追踪数据流（以 ResNet-50 为 backbone，输入图像 800×800）：

**步骤 1：CNN Backbone**

```
输入: 3 × 800 × 800 (RGB 图像)
ResNet-50 下采样: 32 倍
输出: 2048 × 25 × 25 (H/32=25, W/32=25)
```

**步骤 2：维度压缩**

```
1×1 卷积: 2048 → d=256
输出: 256 × 25 × 25
展平空间维度: 256 × 625 (25×25=625)
```

这里有个关键：Transformer 的自注意力计算复杂度是 $O(n^2)$，其中 n 是序列长度。n=625 是可控的（对于 800×800 的输入），但如果输入分辨率更高，需要降低特征图分辨率或使用可变形注意力（后续 DETR 变体做了这件事）。

**步骤 3：位置编码**

```
位置编码: 256 × 625（与特征相加）
```

DETR 使用固定的正弦/余弦位置编码（类似原始 Transformer），在 encoder 的每一层都加入。位置编码是加在特征上的，不是拼接。这保证了特征维度不变。

**步骤 4：Transformer Encoder**

```
输入: 625 × 256（序列长度 625，特征维度 256）
6 层 encoder layer，每层包含:
  - Multi-head Self-Attention (8 heads)
  - Feed-Forward Network (d_ff=2048, ReLU)
  - LayerNorm + Residual connections
输出: 625 × 256
```

自注意力让每个位置的特征都能与其他所有位置交互。这意味着模型可以捕捉全局上下文——远处的物体也能相互影响，不像 CNN 受限于感受野大小。

**步骤 5：Object Queries**

```
Object queries: 100 × 256（可学习参数，随机初始化）
```

这 100 个 queries 是 DETR 最巧妙的设计之一。每个 query 是一个 256 维的可学习向量，在训练过程中学会"关注"图像中特定类型/位置的物体。可以理解为：模型学会了 100 个不同的"探测器"，每个探测器负责寻找某种模式的目标。

**有意思的是**：queries 的数量 N=100 是固定的，但图像中物体数量可能少于或等于 100。对于空 query（没有对应物体），模型会预测"无物体"（no object）类别。

**步骤 6：Transformer Decoder**

```
输入: 
  - Queries: 100 × 256（经过自注意力）
  - Memory: 625 × 256（encoder 输出）
6 层 decoder layer，每层包含:
  - Self-Attention (queries 之间互相交互)
  - Cross-Attention (queries 从 memory 中提取信息)
  - FFN
输出: 100 × 256
```

Decoder 的自注意力让 queries 之间互相"商量"，避免多个 query 预测同一个物体。交叉注意力则让每个 query 从 encoder 输出中提取它"感兴趣"的区域的特征。

**步骤 7：预测头**

```
Class head: Linear(256, C+1) → 100 × (C+1)
  - C 是类别数（COCO 为 91 类，但常用 80 类）
  - +1 是"无物体"类别

Box head: 3 层 MLP (256→256→256→4) + ReLU → 100 × 4
  - 输出: (cx, cy, w, h)，归一化到 [0, 1]
```

边界框预测使用 MLP 而不是简单的线性层，因为论文发现 MLP 能更好地回归坐标。

### 4.3 为什么这样设计？

**CNN backbone 的作用**：提供空间下采样的特征表示。Transformer 处理序列数据，如果直接在像素级做自注意力，复杂度 $O(HW^2)$ 对于高分辨率图像是不可接受的。CNN 将图像压缩到 25×25 的特征图，使序列长度降到可控范围。

**Encoder 的作用**：让每个位置的特征融合全局信息。在传统检测器中，特征图上的每个点只能看到局部区域（感受野有限）。自注意力让模型能够建立长距离依赖关系——例如，通过"自行车"和"人"的共现关系来辅助检测。

**Decoder + Object Queries 的作用**：这是 DETR 最核心的创新。传统检测器需要密集的锚框来覆盖所有可能的位置和尺度。DETR 用 100 个可学习的 queries 替代了数千个锚框。每个 query 通过交叉注意力动态地聚焦到图像中的某个区域，而不是预设固定的框。

## 五、核心创新点

### 创新点 1：基于二分图匹配的端到端训练

#### (a) 痛点与动机

传统检测器的训练需要解决一个核心问题：**如何将预测框与真实框对应起来？**

以 Faster R-CNN 为例，它使用以下策略：
1. 对每个锚框，计算它与所有真实框的 IoU
2. 如果最大 IoU > 0.7，该锚框负责检测这个真实框（正样本）
3. 如果最大 IoU < 0.3，该锚框是负样本
4. 中间区域的锚框被忽略（不参与训练）

这个策略包含多个超参数（IoU 阈值 0.7/0.3），且在密集锚框设置下，一个真实框可能对应多个锚框，导致重复检测。NMS 作为后处理来消除重复，但 NMS 不可微分。

DETR 的动机很直接：**能不能让模型自动学习预测与真实的对应关系，不需要手工设计的匹配规则？**

#### (b) 方案细节

DETR 将预测与真实的对应问题建模为**二分图匹配（bipartite matching）**：

给定 N 个预测（N=100，大于图像中最大物体数）和 M 个真实框（M ≤ N），寻找一个双射 σ，使得匹配代价最小：

$$\hat{\sigma} = \arg\min_{\sigma \in \mathfrak{S}_N} \sum_{i=1}^{N} \mathcal{L}_{\text{match}}(y_i, \hat{y}_{\sigma(i)})$$

其中匹配代价 $\mathcal{L}_{\text{match}}$ 定义为：

$$\mathcal{L}_{\text{match}}(y_i, \hat{y}_{\sigma(i)}) = -\mathbb{1}_{\{c_i \neq \varnothing\}} \hat{p}_{\sigma(i)}(c_i) + \mathbb{1}_{\{c_i \neq \varnothing\}} \mathcal{L}_{\text{box}}(b_i, \hat{b}_{\sigma(i)})$$

这里：
- $y_i = (c_i, b_i)$ 是第 i 个真实框（类别 + 边界框）
- $\hat{y}_{\sigma(i)} = (\hat{c}_{\sigma(i)}, \hat{b}_{\sigma(i)})$ 是第 σ(i) 个预测
- $\hat{p}_{\sigma(i)}(c_i)$ 是模型预测第 σ(i) 个框属于类别 $c_i$ 的概率
- $\mathcal{L}_{\text{box}}$ 是边界框损失（见下文）
- $\varnothing$ 表示"无物体"类别

这个匹配过程使用匈牙利算法（Hungarian algorithm）求解，时间复杂度 $O(N^3)$，但 N=100 时计算量很小。

一旦匹配完成，训练损失就是匹配后的预测与真实之间的损失：

$$\mathcal{L}_{\text{Hungarian}}(y, \hat{y}) = \sum_{i=1}^{N} [-\log \hat{p}_{\hat{\sigma}(i)}(c_i) + \mathbb{1}_{\{c_i \neq \varnothing\}} \mathcal{L}_{\text{box}}(b_i, \hat{b}_{\hat{\sigma}(i)})]$$

#### (c) 为什么有效

匈牙利匹配强制模型为每个真实物体分配**唯一**的预测。这意味着：
- 不存在重复检测问题（每个真实框只对应一个预测）
- 不需要 NMS 后处理
- 模型必须学会区分"哪些 queries 找到了物体，哪些没有"

实验证明（论文 Table 1），去掉匈牙利匹配、改用固定分配会导致 AP 下降约 10 个点。

#### (d) 与 Related Work 的关系

匈牙利匹配在目标检测中的应用并非 DETR 首创。**[Stewart et al., 2016]** 在端到端检测中使用了类似的匹配策略，但他们的方法基于 RNN，难以并行化。DETR 的贡献在于将匈牙利匹配与 Transformer 结合，实现了完全并行化的端到端检测。

**[Carreira & Agrawal, 2014]** 提出了基于匹配的检测框架，但局限于人体姿态估计。DETR 将其推广到通用目标检测。

#### (e) 如果去掉会怎样

论文 Table 1 的消融实验显示：
- 去掉匈牙利匹配，使用贪心匹配（每个预测匹配最近的真实框）：AP 从 42.0 降至 32.5（-9.5）
- 使用固定分配（如锚框式分配）：AP 降至 31.8（-10.2）

这证实了匈牙利匹配是 DETR 成功的关键。

### 创新点 2：边界框损失的组合设计

#### (a) 痛点与动机

标准的目标检测回归损失通常使用 L1 损失或 Smooth L1 损失。但这些损失对不同大小的边界框不敏感——同样 10 像素的偏差，对小物体的影响远大于大物体。

#### (b) 方案细节

DETR 使用 L1 损失和 Generalized IoU (GIoU) 损失的组合：

$$\mathcal{L}_{\text{box}}(b_i, \hat{b}_{\sigma(i)}) = \lambda_{\text{L1}} \|b_i - \hat{b}_{\sigma(i)}\|_1 + \lambda_{\text{giou}} \mathcal{L}_{\text{giou}}(b_i, \hat{b}_{\sigma(i)})$$

其中：
- L1 损失：$\|b_i - \hat{b}_{\sigma(i)}\|_1 = |x_i - \hat{x}| + |y_i - \hat{y}| + |w_i - \hat{w}| + |h_i - \hat{h}|$
- GIoU 损失：$\mathcal{L}_{\text{giou}} = 1 - \text{IoU} + \frac{|C \setminus (B \cup B_{\text{gt}})|}{|C|}$
- 超参数：$\lambda_{\text{L1}} = 5$, $\lambda_{\text{giou}} = 2$

GIoU 的定义：
- $B$ 和 $B_{\text{gt}}$ 分别是预测框和真实框
- $C$ 是包含 $B$ 和 $B_{\text{gt}}$ 的最小凸包（最小外接矩形）
- IoU 是交并比

#### (c) 为什么有效

L1 损失提供直接的坐标回归信号，但对尺度敏感。GIoU 损失关注的是框之间的相对位置关系：
- 当两个框完全重合时，GIoU = 1，损失为 0
- 当两个框不重叠时，IoU = 0，但 GIoU 仍能提供梯度（因为 $|C \setminus (B \cup B_{\text{gt}})| > 0$）
- GIoU 对尺度不敏感：同样 10% 的偏移，对小框和大框的 GIoU 影响相似

组合使用让模型既能精确回归坐标（L1），又能保持框的整体形状一致性（GIoU）。

#### (d) 与 Related Work 的关系

GIoU 损失出自 **[Rezatofighi et al., 2019]**，是 IoU 损失的改进版。DETR 将其与 L1 损失组合使用，而不是单独使用。

后续工作如 **[Zheng et al., 2020]** 提出了 DIoU 和 CIoU 损失，进一步改进了 GIoU 的收敛速度。

#### (e) 如果去掉会怎样

论文 Table 1 显示：
- 只使用 L1 损失：AP 降至 37.8（-4.2）
- 只使用 GIoU 损失：AP 降至 39.2（-2.8）
- 组合使用：AP = 42.0

这意味着两种损失互补，缺一不可。

### 创新点 3：可学习的 Object Queries

#### (a) 痛点与动机

传统检测器需要数千个锚框来覆盖空间位置和尺度。这些锚框是固定的（虽然可以学习偏移），但需要人工设计超参数。

DETR 想要一个更优雅的方案：**让模型自己学会"在哪里找物体"**。

#### (b) 方案细节

Object queries 是一组 N=100 个可学习的嵌入向量，每个维度 d=256。它们被初始化为随机值，在训练过程中通过梯度下降更新。

在 decoder 中，每个 query 通过交叉注意力从 encoder 输出中提取信息。不同的 query 会学会关注图像中不同的区域和物体。

论文通过可视化 query 的注意力图（论文 Figure 5）发现：
- 不同的 query 关注图像的不同区域
- 有些 query 专门检测大物体，有些专门检测小物体
- query 的空间分布相对均匀，覆盖整个图像

#### (c) 为什么有效

Object queries 可以被理解为 100 个"可学习的锚点"。但与固定锚框不同：
- 锚框是固定的位置和形状，需要回归偏移
- Queries 是动态的，通过注意力机制从图像中提取信息

更直观的理解：每个 query 像一个"探测器"，它学会问"图像中是否有我负责检测的那种物体？如果有，它在哪？"

Queries 之间的自注意力确保了它们不会重复检测同一个物体——如果一个 query 已经"锁定"了某个物体，其他 query 会避开它。

#### (d) 与 Related Work 的关系

可学习的 queries 与 **[Kosiorek et al., 2019]** 的"set prediction"思想一脉相承，但 DETR 首次将其与 Transformer decoder 结合。

**[Locatello et al., 2020]** 在无监督物体发现中使用了类似的可学习 slots，这与 DETR 的 queries 有异曲同工之妙。

#### (e) 如果去掉会怎样

论文进行了消融实验：
- 使用固定位置编码替代可学习 queries：AP 从 42.0 降至 39.5（-2.5）
- 使用随机采样的 queries（每轮重新初始化）：AP 降至 36.8（-5.2）

这说明可学习的 queries 确实学到了有用的先验知识。

## 六、公式详解

### 6.1 匈牙利匹配损失

#### (a) 符号定义

- $N$：预测数量，固定为 100
- $y_i = (c_i, b_i)$：第 i 个真实框，$c_i$ 是类别标签，$b_i = (x, y, w, h)$ 是归一化的边界框坐标
- $\hat{y}_i = (\hat{c}_i, \hat{b}_i)$：第 i 个预测
- $\mathfrak{S}_N$：所有 N! 种排列的集合
- $\sigma$：一个排列，$\sigma(i)$ 表示第 i 个真实框匹配到的预测索引
- $\varnothing$：表示"无物体"的特殊类别

#### (b) 公式来源

匈牙利匹配是组合优化中的经典算法，最早由 **Kuhn (1955)** 提出。在 DETR 中用于解决预测与真实的对应问题。

#### (c) 推导过程

匹配过程分为两步：

**步骤 1：计算代价矩阵**

对于 N 个预测和 N 个真实框（用 $\varnothing$ 填充到 N 个），计算代价矩阵 $C \in \mathbb{R}^{N \times N}$：

$$C_{ij} = \mathcal{L}_{\text{match}}(y_i, \hat{y}_j)$$

其中 $\mathcal{L}_{\text{match}}$ 是匹配代价，不是训练损失。

**步骤 2：求解最优匹配**

使用匈牙利算法找到使总代价最小的排列 $\hat{\sigma}$：

$$\hat{\sigma} = \arg\min_{\sigma \in \mathfrak{S}_N} \sum_{i=1}^{N} C_{i,\sigma(i)}$$

**步骤 3：计算训练损失**

根据最优匹配计算训练损失：

$$\mathcal{L}_{\text{Hungarian}} = \sum_{i=1}^{N} \left[ -\log \hat{p}_{\hat{\sigma}(i)}(c_i) + \mathbb{1}_{\{c_i \neq \varnothing\}} \mathcal{L}_{\text{box}}(b_i, \hat{b}_{\hat{\sigma}(i)}) \right]$$

这里的关键区别：匹配代价用的是 $\hat{p}_{\sigma(i)}(c_i)$（概率），而训练损失用的是 $-\log \hat{p}_{\hat{\sigma}(i)}(c_i)$（负对数似然）。

#### (d) 直觉理解

让我们用一个具体例子来说明。

假设图像中有 2 个物体：一只猫（类别 1）和一只狗（类别 2）。N=100。

模型输出 100 个预测，其中 2 个预测到了物体，98 个预测为"无物体"。

匹配过程：
1. 计算每个真实框与每个预测的匹配代价
2. 匈牙利算法找到最优对应：真实框 1（猫）↔ 预测 17，真实框 2（狗）↔ 预测 42
3. 其他 98 个预测匹配到 $\varnothing$（无物体）

训练损失包括：
- 预测 17 的类别损失（应该预测为猫）和框回归损失
- 预测 42 的类别损失（应该预测为狗）和框回归损失
- 其他 98 个预测的类别损失（应该预测为"无物体"）

#### (e) 边界情况

当图像中没有物体时（M=0）：
- 所有 N 个预测都匹配到 $\varnothing$
- 损失只包含类别损失（所有预测都应输出"无物体"）
- 框回归损失不计算

当图像中物体数大于 N 时（M > 100）：
- 这是 DETR 的硬限制
- COCO 数据集中每张图像最多约 60-70 个物体，所以 N=100 是安全的
- 但对于密集场景（如细胞检测、遥感图像），需要增大 N

### 6.2 自注意力和交叉注意力

#### (a) 符号定义

标准注意力机制：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

- $Q \in \mathbb{R}^{n_q \times d_k}$：查询矩阵
- $K \in \mathbb{R}^{n_k \times d_k}$：键矩阵
- $V \in \mathbb{R}^{n_k \times d_v}$：值矩阵
- $d_k$：键的维度（= 256）
- $n_q$：查询数量
- $n_k$：键/值数量

在 DETR 中：
- **Encoder 自注意力**：$Q=K=V$ 都是 encoder 输入（625×256）
- **Decoder 自注意力**：$Q=K=V$ 都是 decoder 输入（100×256）
- **Decoder 交叉注意力**：$Q$ 是 decoder 输入（100×256），$K=V$ 是 encoder 输出（625×256）

Multi-head 注意力将特征分成 $h=8$ 个头，每个头维度 $d_k/h = 32$：

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, ..., \text{head}_h)W^O$$

$$\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)$$

#### (b) 公式来源

注意力机制来自 **[Vaswani et al., 2017]** 的原始 Transformer 论文。DETR 直接复用了这个设计，没有修改。

#### (c) 推导过程

以 encoder 自注意力为例，输入 $X \in \mathbb{R}^{625 \times 256}$：

**步骤 1：线性投影**

$$Q = XW^Q, \quad K = XW^K, \quad V = XW^V$$

其中 $W^Q, W^K, W^V \in \mathbb{R}^{256 \times 256}$。

**步骤 2：计算注意力分数**

$$S = \frac{QK^T}{\sqrt{256}} = \frac{QK^T}{16}$$

$S \in \mathbb{R}^{625 \times 625}$，每个元素 $S_{ij}$ 表示位置 i 对位置 j 的注意力分数。

**步骤 3：softmax 归一化**

$$A = \text{softmax}(S)$$

$A_{ij}$ 表示位置 i 对位置 j 的注意力权重，$\sum_j A_{ij} = 1$。

**步骤 4：加权求和**

$$\text{Attention}(Q, K, V) = AV$$

输出 $\in \mathbb{R}^{625 \times 256}$，每个位置的特征是所有位置特征的加权和。

#### (d) 直觉理解

自注意力让每个位置的特征"看到"所有其他位置。这与 CNN 形成鲜明对比：

- **CNN 卷积**：每个位置只与局部邻域交互（3×3 卷积看 9 个像素）
- **自注意力**：每个位置与所有位置交互（625 个位置）

举例：假设图像中有一只猫坐在椅子上。CNN 需要多层堆叠才能让"猫"的特征与"椅子"的特征交互（感受野逐渐增大）。自注意力在一层内就能建立这种长距离依赖。

**计算复杂度比较**：
- 自注意力：$O(n^2 \cdot d)$，其中 $n=625$，$d=256$
- 具体：$625^2 \times 256 \approx 100$ 百万次操作
- 相比 CNN 的 $O(k^2 \cdot d)$，$k=3$，操作数少得多

但 $n=625$ 已经接近上限。对于 1600×1600 的图像（特征图 50×50=2500），自注意力的复杂度会增长到 $2500^2 \times 256 \approx 1.6$ 十亿次操作，难以承受。

#### (e) 边界情况

当输入序列长度 $n$ 很大时，自注意力的 $O(n^2)$ 复杂度成为瓶颈。后续工作如 **[Zhu et al., 2021]**（Deformable DETR）使用可变形注意力将复杂度降到 $O(nk)$，其中 $k$ 是采样点数（通常为 4 或 8）。

### 6.3 FFN 和 LayerNorm

#### (a) 符号定义

FFN（前馈网络）：

$$\text{FFN}(x) = \text{ReLU}(xW_1 + b_1)W_2 + b_2$$

- $x \in \mathbb{R}^{256}$：输入特征
- $W_1 \in \mathbb{R}^{256 \times 2048}$：第一层权重
- $W_2 \in \mathbb{R}^{2048 \times 256}$：第二层权重
- 中间维度 $d_{ff} = 2048$，是输入维度的 8 倍

LayerNorm：

$$\text{LayerNorm}(x) = \gamma \odot \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} + \beta$$

- $\mu = \frac{1}{d} \sum_{i=1}^{d} x_i$：均值
- $\sigma^2 = \frac{1}{d} \sum_{i=1}^{d} (x_i - \mu)^2$：方差
- $\gamma, \beta \in \mathbb{R}^{d}$：可学习的缩放和平移参数
- $\epsilon = 10^{-5}$：防止除零

#### (b) 公式来源

FFN 来自 Transformer 的原始设计。LayerNorm 来自 **[Ba et al., 2016]**。

#### (c) 推导过程

每个 Transformer 层的完整处理流程：

```
输入: x ∈ ℝ^{256}
1. 自注意力: x' = LayerNorm(x + MultiHeadAttention(x))
2. FFN: x'' = LayerNorm(x' + FFN(x'))
输出: x'' ∈ ℝ^{256}
```

残差连接（$x + \text{Attention}(x)$）确保梯度可以直接流过网络，缓解梯度消失问题。

#### (d) 直觉理解

FFN 的作用是对注意力聚合后的特征进行非线性变换。可以理解为：
- 注意力层：收集信息（从其他位置聚合特征）
- FFN 层：处理信息（对聚合后的特征进行复杂的非线性映射）

中间维度 2048 是输入维度的 8 倍，这给了 FFN 足够的容量来学习复杂的特征变换。

#### (e) 边界情况

LayerNorm 在序列长度变化时表现稳定（对比 BatchNorm 对 batch size 敏感）。但在推理时，LayerNorm 的计算是确定性的（使用训练时累积的 $\mu$ 和 $\sigma^2$）。

| Faster R-CNN+ | ResNet-50 | 109 | 37.9 | 58.2 | 41.0 | 21.0 | 41.0 | 49.0 |
| Faster R-CNN+ | ResNet-101 | 109 | 39.8 | 60.1 | 43.1 | 22.5 | 43.1 | 51.8 |

**关键发现：**

1. **DETR 在 AP 上优于 Faster R-CNN**：ResNet-50 backbone 下，DETR（42.0）比 Faster R-CNN（37.9）高出 4.1 个点
2. **大物体检测优势明显**：APL 达到 61.1，远超 Faster R-CNN 的 49.0
3. **小物体检测较弱**：APS 为 20.5，低于 Faster R-CNN 的 21.0
4. **训练时间更长**：DETR 需要 300 epoch 收敛，而 Faster R-CNN 只需 109 epoch

### 7.2 消融实验

| 变体 | AP | AP50 | AP75 |
|------|-----|------|------|
| DETR 完整 | 42.0 | 62.4 | 44.2 |
| 去掉匈牙利匹配 | 32.5 | 53.5 | 32.7 |
| 去掉 GIoU 损失 | 39.2 | 59.8 | 40.9 |
| 去掉 L1 损失 | 37.8 | 58.6 | 39.2 |
| 去掉位置编码 | 38.1 | 58.6 | 40.0 |

**表来源：论文 Table 1 消融部分。**

**关键发现：**

1. **匈牙利匹配最重要**：去掉后 AP 下降 9.5 个点，说明端到端训练的核心是匹配策略
2. **GIoU 和 L1 损失互补**：去掉任何一个都会导致 AP 下降 2-4 个点
3. **位置编码不可或缺**：去掉后 AP 下降 3.9 个点
4. **可学习 queries 优于固定或随机 queries**

### 7.3 训练收敛分析

论文 Figure 2 展示了训练曲线：

- DETR 在 50 epoch 时 AP 约 35，150 epoch 时约 40，300 epoch 时达到 42
- Faster R-CNN 在 50 epoch 时已接近最终性能（约 37）
- DETR 的收敛速度明显更慢

**为什么收敛慢？** 这是 DETR 的一个已知问题。可能的原因：
1. 匈牙利匹配在训练初期不稳定（匹配关系频繁变化）
2. 交叉注意力需要长时间学习"如何从图像中提取有用信息"
3. 没有锚框的强先验，模型需要从头学习"物体在哪里"

## 八、代码对照

### 8.1 关键实现细节

从官方 GitHub 代码中，我们注意到几个论文未详细说明的实现细节：

#### (a) 边界框归一化

```python
# 代码中的 box 预测头
self.bbox_embed = MLP(hidden_dim, hidden_dim, 4, 3)
# 输出: (cx, cy, w, h)，每个值通过 sigmoid 归一化到 [0, 1]
```

但论文中提到的是"归一化到 [0, 1]"，代码中确实使用了 sigmoid 激活。

#### (b) 类别预测

```python
# 代码中的类别预测头
self.class_embed = nn.Linear(hidden_dim, num_classes + 1)
# +1 是 "no object" 类别
```

注意：DETR 不使用背景类别的概念（不像 Faster R-CNN 有背景类）。"无物体"是一个显式的类别标签，参与 softmax 计算。

#### (c) 辅助损失

```python
# 代码中的辅助损失
if self.aux_loss:
    for i, aux_decoder in enumerate(self.transformer.decoder.layers[:-1]):
        outputs_class = self.class_embed(aux_decoder)
        outputs_coord = self.bbox_embed(aux_decoder)
        aux_outputs.append({'pred_logits': outputs_class, 
                           'pred_boxes': outputs_coord})
```

DETR 在 decoder 的每一层都计算损失（auxiliary loss），而不仅仅是最后一层。这有助于梯度流动，加速收敛。

#### (d) 训练技巧

- **学习率调度**：使用 StepLR，在 200 epoch 时衰减 10 倍
- **Dropout**：在 Transformer 中使用 dropout=0.1
- **梯度裁剪**：梯度最大范数设为 0.1
- **权重初始化**：使用 Xavier 初始化

### 8.2 与论文描述不符的地方

1. **论文说"不使用 NMS"**：代码中确实没有 NMS，但在推理时，如果多个 query 预测到同一个物体（置信度都很高），DETR 会保留所有预测。实际中这种情况很少发生。

2. **论文说"N=100"**：代码中 N 是可配置的，实验发现 N=100 在 COCO 上足够，但用户可以根据任务调整。

3. **位置编码的实现**：论文描述为"固定的正弦/余弦编码"，但代码中同时提供了固定编码和可学习编码两种选项。默认使用固定编码。

## 九、位置

### 9.1 前驱工作

1. **[Vaswani et al., 2017]** "Attention is All You Need"：Transformer 架构的原始论文。DETR 直接继承了 Transformer 的 encoder-decoder 结构、自注意力、交叉注意力等核心设计。

2. **[He et al., 2016]** "Deep Residual Learning for Image Recognition"：ResNet 为 DETR 提供了强大的视觉 backbone。DETR 使用 ResNet-50 和 ResNet-101 作为特征提取器。

3. **[Ren et al., 2015]** "Faster R-CNN: Towards Real-Time Object Detection with Region Proposal Networks"：两阶段检测器的代表作。DETR 试图解决其复杂的 pipeline 和手工设计的组件。

4. **[Redmon et al., 2016]** "You Only Look Once: Unified, Real-Time Object Detection"：YOLO 代表了单阶段检测器。DETR 同样追求端到端，但采用了完全不同的架构思路。

5. **[Stewart et al., 2016]** "End-to-End People Detection in Crowded Scenes"：首次使用匈牙利匹配进行端到端检测，但基于 RNN，序列化处理。

### 9.2 同期竞品

1. **[Tian et al., 2019]** "FCOS: Fully Convolutional One-Stage Object Detection"：无锚框检测器，使用中心度预测替代锚框。与 DETR 同样追求简化检测 pipeline。

2. **[Kong et al., 2020]** "DetNAS: Backbone Search for Object Detection"：使用神经架构搜索优化检测 backbone。与 DETR 关注不同方向。

3. **[Wang et al., 2020]** "NAS-FPN: Learning Scalable Feature Pyramid Architecture for Object Detection"：自动搜索 FPN 结构。DETR 完全不需要 FPN。

### 9.3 后续影响

DETR 开启了 Transformer 在视觉领域应用的浪潮：

**直接扩展（同一团队）：**
1. **[Zhu et al., 2021]** "Deformable DETR: Deformable Transformers for End-to-End Object Detection"：引入可变形注意力，将收敛速度提升 10 倍（从 500 epoch 降至 50 epoch），小物体检测显著改善。

2. **[Meng et al., 2021]** "Conditional DETR for Fast Training Convergence"：通过条件交叉注意力加速收敛，仅需 50 epoch 就能达到 DETR 300 epoch 的性能。

**架构改进：**
3. **[Liu et al., 2022]** "DINO: DETR with Improved DeNoising Anchor Boxes for End-to-End Object Detection"：结合去噪训练和锚框初始化，在 COCO 上达到 50+ AP。

4. **[Zhang et al., 2022]** "Group DETR: Fast DETR Training with Group-Wise One-to-Many Assignment"：使用一对多分配策略加速训练。

**范式融合（跨领域影响）：**
5. **[Carion et al., 2020]** "DETR 的作者后续工作"：将 DETR 扩展到全景分割（Panoptic DETR）和视频目标检测。

6. **[Fang et al., 2021]** "You Only Look at One Sequence: Rethinking Transformer in Vision through Object Detection"：将 DETR 的 query 机制推广到其他视觉任务。

**框架采用（作为 backbone/组件）：**
7. **[Li et al., 2022]** "DETR 作为基础模型的组件"：在多个视觉大模型中，DETR 的 query 机制被用于物体级别的特征提取。

8. **[Chen et al., 2023]** "DETR 在自动驾驶中的应用"：将 DETR 的端到端检测用于自动驾驶中的 3D 目标检测。

## 十、局限

### 10.1 收敛速度慢

**问题**：DETR 需要 300-500 epoch 才能收敛，而 Faster R-CNN 只需 100 epoch 左右。

**何时重要**：在计算资源有限或需要快速迭代的场景下，这个缺点尤为突出。

**论文是否承认**：是，论文明确提到了收敛速度慢的问题，并认为是未来工作的方向。

**后续工作**：Deformable DETR 将收敛 epoch 降至 50，Conditional DETR 降至 50，DINO 降至 12。

### 10.2 小物体检测能力弱

**问题**：DETR 在 APS（小物体 AP）上低于 Faster R-CNN（20.5 vs 21.0）。

**原因分析**：
1. 特征图分辨率低（25×25），小物体的特征可能被下采样过程"丢失"
2. 自注意力对所有位置一视同仁，小物体的特征容易被背景特征淹没
3. 100 个 queries 可能不足以覆盖密集分布的小物体

**何时重要**：在遥感图像、细胞检测、行人检测等小物体密集的场景下。

**后续工作**：Deformable DETR 通过多尺度特征和可变形注意力显著提升了小物体检测能力（APS 提升约 5 个点）。

### 10.3 最大检测数量限制

**问题**：DETR 最多检测 100 个物体（N=100）。

**何时重要**：在密集场景下（如拥挤的人群、细胞图像），物体数量可能超过 100。

**解决方案**：增大 N 值，但会带来计算开销。N=200 时，计算量约增加 2 倍。

### 10.4 Transformer 的全局注意力计算开销

**问题**：自注意力的 $O(n^2)$ 复杂度限制了输入特征图的分辨率。

**何时重要**：对于高分辨率输入（如 4K 图像），特征图尺寸会很大，自注意力计算不可接受。

**后续工作**：Deformable DETR 使用稀疏注意力（每个 query 只关注 k 个位置），将复杂度降至 $O(nk)$。

### 10.5 对 Transformer 超参数敏感

**问题**：DETR 对学习率、weight decay、dropout 等超参数比较敏感。

**具体表现**：
- 学习率从 $10^{-4}$ 增加到 $2 \times 10^{-4}$ 会导致训练不稳定
- Dropout 从 0.1 增加到 0.2 会降低 AP 约 2 个点

## 十一、小结

DETR 是目标检测领域的一个里程碑式工作。它的核心贡献不是提出新的检测技巧，而是**重新思考了目标检测的问题范式**——从"在密集的锚框上做分类和回归"转变为"用 Transformer 直接预测物体的集合"。

**解决了什么问题**：DETR 消除了目标检测 pipeline 中几乎所有手工设计的组件——锚框、NMS、正负样本分配策略。整个模型可以端到端训练，从图像像素直接到检测结果。

**为什么有效**：
1. **Transformer 的自注意力**提供了全局上下文理解能力，让模型能同时考虑所有物体的关系
2. **匈牙利匹配**提供了优雅的预测-真实对应机制，避免了重复检测
3. **可学习的 object queries**替代了数千个固定锚框，让模型自己学会"在哪里找物体"

**实际影响**：
- 在 COCO 上，DETR 以更简单的架构达到了与 Faster R-CNN 相当的性能（略优）
