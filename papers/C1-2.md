# End-to-End Object Detection with Transformers（2020）

**论文：** [arXiv](https://arxiv.org/abs/2005.12872) · Nicolas Carion, Francisco Massa, Gabriel Synnaeve, Nicolas Usunier, Alexander Kirillov, Sergey Zagoruyko (Facebook AI) · 2020（DETR）

---

## 一、先搞清楚"问题在哪里"

目标检测是计算机视觉最核心的任务之一：给一张图片，输出图片中所有物体的类别和包围框。听起来简单，但在 DETR 之前，检测系统的流程极其复杂。

主流方案是 Faster R-CNN（Ren et al., 2015）及其衍生系统。想象一个检测流水线：

```text
输入图片 ──► CNN backbone ──► Region Proposal ──► RoI Align ──►
分类/回归头 ──► NMS 去重 ──► 最终结果
```

这个流水线里几乎每个环节都有**手工设计的组件**：
- **Anchor boxes**：预先定义一组尺寸和比例的矩形框作为"候选"
- **NMS（非极大值抑制）**：因为 anchor 框会为同一个物体输出多个重复预测，需要一个后处理步骤来合并
- **正负样本分配规则**：哪个框匹配哪个 GT 物体，靠 IoU 阈值+启发式规则
- **Proposal 生成**：RPN（Region Proposal Network）本身又是一个独立的检测器

这些组件各自引入了大量的超参数。2020 年一篇研究（[Zhang et al., 2020]）证明：仅仅改变 anchor 的分配规则，就能让同一检测器的 AP 差距达到几个点。换句话说，**目标检测的复杂性和不稳定性，很大程度来自这些手工设计的选择**。

这篇论文的核心问题就是：**能不能把目标检测变得像翻译一样——输入一张图，直接并行输出一组物体，全程 differentiable，不要手工组件？**

## 二、旧路线的真正问题

再细拆 Faster R-CNN 的 pipeline 里的每个痛点：

**问题 1：Anchor 的缺陷**
- Anchor 是预先定义的一组框（Faster R-CNN 用 3 种尺度 × 3 种比例 = 9 种），它们和数据集分布强相关
- COCO 上的 9 种 anchor 对汽车和人合适，但对长颈鹿、火车这类长宽比极端的物体就不行
- 论文引用了 [Zhang et al., 2020] "Bridging the Gap Between Anchor-based and Anchor-free Detection" 来说明 anchor 设计对最终性能的巨大影响

**问题 2：NMS 是"补丁"**
- Anchor 本质上会产生大量冗余预测。同一个物体多个 anchor 都预测到了，需要 NMS 去掉重复
- NMS 本质是贪心的：按置信度排序，保留最高分的框，移除与其 IoU 过大的其他框
- 阈值靠拍脑袋（典型值 0.5），太松漏检测、太紧错杀。在密集场景（如人群）中 NMS 尤其糟糕

**问题 3：多阶段训练的复杂性**
- Faster R-CNN 有 4 个训练步骤的替代训练方案（后改为联合训练）
- RPN 的分类/回归 + Fast R-CNN 的分类/回归，每个都有自己的 loss，超参数多
- 推理时也需要先跑 RPN、再 RoI Align、再分类/回归——无法一步到位

**问题 4：启发式正负样本分配**
- 一个 GT 框分配给多个 anchor（IoU > 0.7 为正样本），导致一个物体对应多个预测
- 这种分配规则让模型学到的不是"唯一检测"，而是"冗余检测+NMS收拾残局"
- 训练目标和推理目标存在不一致（train time: 多个正样本; inference: 去掉多的）

**问题 5：跨模态不自然**
- RPN 用的是卷积做 proposal 的回归，但 proposal 之间的关系（哪些是同一个物体的重复）没有被显式建模
- 靠 NMS 这种非学习的后处理来建模，意味着模型自己学不会"去重"

所有这些问题的共同根源是：**检测器输出的不是一个"集合"，而是一个"带冗余的候选池"。** 冗余是因为模型不知道其他预测在干什么。

## 三、这篇论文的核心思路

DETR 的思路可以表述为一段话：

> 不要预测一堆带冗余的候选框再靠后处理筛选。
> 直接预测一组唯一的物体，一次到位。
> 关键在于两件事：一个能去重的 loss 和一个能去重的架构。

具体来说，DETR 把目标检测看作一个**直接的 set prediction 问题**。

**去重的 loss：** 用匈牙利算法（Hungarian Algorithm）在预测集合和 GT 集合之间做最优二分匹配。每个 GT 物体只分配给唯一的预测。匹配不到的预测学"no object"（∅）。这样模型天然不需要 NMS——它不需要学冗余。

**去重的架构：** 用 Transformer 的 encoder-decoder。Encoder 通过 self-attention 在整张图的特征上做全局推理。Decoder 把一组**可学习的 object queries**（数量固定，如 N=100）作为输入，通过 cross-attention 从 encoder 特征中检索信息，再通过 decoder self-attention 看到其他 query 的输出，从而学会不重复预测。

整个流程是全程 differentiable 的：输入图片 → backbone → transformer encoder → transformer decoder → FFN → 输出 N 个 (class, box)。不需要 RPN，不需要 anchor，不需要 NMS。

## 四、整体结构与 forward 流程

### 架构总览

```text
输入图片 (3, H, W)
    │
    ▼
[CNN Backbone (ResNet-50)] ───► feature map (C, H/32, W/32), C=2048
    │
    ▼
[1×1 Conv] ───► d_model=256 维特征 (256, H/32, W/32)
    │
    ▼ (flatten: H/32 × W/32 = HW 个 token + position encoding)
    │
    ▼
┌───────────────────────────────────────────────────────────┐
│  Transformer Encoder（6 层）                             │
│  每层: self-attention + FFN                              │
│  Token 数 = H/32 × W/32（约 800×1333÷32² ≈ 1082 个）   │
│  输出: 同上形状的 encoded feature                        │
└───────────────────────────────────────────────────────────┘
    │
    ▼ (decoder 的 cross-attention 的 key/value 来源)
    │
┌───────────────────────────────────────────────────────────┐
│  Transformer Decoder（6 层）                             │
│  输入: N=100 个 object queries (可学习的 embedding)      │
│  每层: self-attention(query间) → cross-attention(检索) → FFN  │
│  输出: N=100 个 decoded embeddings                       │
└───────────────────────────────────────────────────────────┘
    │
    ▼
[Prediction FFN] ───► 每个 query 输出:
    ├── class logits (COCO: 91 类 + ∅ = 92)
    └── box (cx, cy, w, h) 归一化坐标 [0,1]
```

### 前向过程逐步走

以 DETR with ResNet-50（COCO 输入 800×1333）为例：

**Step 1 — Backbone 提取特征。** ResNet-50（去掉最后的 avgpool 和 fc）输出下采样 32× 的特征图。假如输入是 800×1333，特征图大小 ≈ 25×42（向下取整 800/32=25, 1333/32=41.66→42），channel C=2048。

**Step 2 — Channel 压缩。** 1×1 Conv 把 2048 维压缩到 d_model=256。输出形状：(256, 25, 42)。

**Step 3 — Flatten + Position Encoding。** 特征图展平成 token 序列：25×42 = 1050 个 token，每个 token 256 维。加上 position encoding（位置编码也是 256 维，逐元素相加）。

**Step 4 — Encoder。** 6 层标准 Transformer encoder。每层做：self-attention（1050×1050 的 attention 矩阵）→ FFN。输出是 1050×256 的编码特征。

**Step 5 — Decoder。** 100 个 object queries（可学习参数，形状 100×256）进入 decoder。6 层 decoder 的每一层：
- **Self-attention:** 100 个 query 之间互相看。这是去重的关键——如果两个 query 关注到了同一个物体，self-attention 矩阵可以告诉它们"他已经占了这个位置"。
- **Cross-attention:** 每个 query 从 encoder 特征（1050 个 key）中检索与自己最相关的区域。
- **FFN:** 非线性变换。

**Step 6 — Prediction FFN。** 每个 decoder 输出通过一个 3 层 MLP（ReLU，隐藏层 256 维）+ 线性投影：
- class head: 输出 92 维 logits（COCO 91 个物体类 + 1 个 ∅）
- box head: 输出 4 维 (cx, cy, w, h)，经 sigmoid 归一化到 [0,1]

最终输出：100 个预测，包含冗余但模型训练后天然只有 ≤ 实际物体数 + 少量噪声 的非 ∅ 预测。

## 五、核心模块逐个解释

### 5.1 CNN Backbone

没什么特殊设计：标准的 ImageNet 预训练 ResNet（有 ResNet-50 和 ResNet-101 两个版本）。去掉最后的 avgpool 和 fc 层，输出下采样 32× 的特征图后面接了 1×1 conv 做 channel 压缩。

但这里有一个很重要的设计选择：**为什么只用了一个单分辨率 32× 下采样的特征图，而不用 FPN（Feature Pyramid Networks）那种多尺度架构？**

答案是：DETR 的设计哲学就是"足够少的手工设计"。FPN 本身就是一个手工设计的模块。Transformer 的全局 self-attention 理论上可以从单分辨率特征中捕捉多尺度信息——因为它可以看到整张图。实践证明，DETR 在大物体上确实有效，但在小物体上效果差（因为 32× 下采样后小物体只剩几个像素的特征）。

### 5.2 Positional Encoding

DETR 的 positional encoding（PE）是**固定（non-learnable）的正弦编码**，按 spatial 位置生成：

```python
pos_y = torch.arange(h)[:, None].float().repeat(1, w) / h * 2 * math.pi
pos_x = torch.arange(w)[None, :].float().repeat(h, 1) / w * 2 * math.pi
pe = cat([sin(pos_y * freq_k), cos(pos_y * freq_k),
          sin(pos_x * freq_k), cos(pos_x * freq_k)], dim=0)
```

其中 `freq_k = 1 / (10000^(2k/d_model))`，和原始 Transformer（Vaswani et al., 2017）的做法一致，只是从 1D 扩展到了 2D。

注意 PE 是**加到 encoder 的每个 self-attention 层的输入上**，而不是只加一次。这和原始 Transformer 只加在 embedding 上的做法不同——因为 encoder 有 6 层，每层都需要知道 token 的位置信息。

### 5.3 Transformer Encoder

标准的 Transformer encoder，6 层。每层包含：
- Multi-head self-attention（8 heads, d_model=256, head_dim=32）
- FFN（2048 维隐藏层，ReLU）
- Pre-LN residual（LayerNorm 放在 residual 之前，训练更稳定）

Encoder 的输入 token 数取决于特征图大小 + 位置编码。在 800×1333 输入下约 1050 个 token，attention 矩阵大小约 1050² ≈ 1.1M，对 GPU 来说完全可以接受。

Encoder 的作用是**全局上下文编码**。通过 self-attention，每个 token 可以看到整张图的所有其他 token。这比卷积的感受野大得多——卷积每次只能看 3×3 或 7×7 的局部区域。

### 5.4 Object Queries

Object queries 可能是 DETR 最神秘的设计。它们到底是什么？

**形式上：** Object queries 是一个可学习的参数矩阵，形状 `(N, d_model)`，N=100。它们和 word embedding 类似——每个 query 是一个 256 维的向量，在训练开始时随机初始化，随训练反向传播更新。

**功能上：** 每个 query 学习去关注图像的某个区域。经过训练，不同的 query 会分化——有的专门检测大物体，有的专门检测小物体，有的关注左上角，有的关注中心区域。Decode 之后，每个 query 对应输出一个检测结果。

**类比：** 可以理解为模型学会的 100 个"探测器"，每个探测器负责一类特定的空间位置+尺寸组合的物体。和 anchor 的区别是：anchor 是手工定义死的，object queries 是从数据中**自动学习**的。

**为什么是 100 个？** COCO 数据集每张图片平均约 7 个物体，最多约 60-70 个。取 N=100 确保有足够余量覆盖各种极端情况。剩下的 query 会学会输出 ∅（无物体）。

### 5.5 Transformer Decoder

标准的 Transformer decoder，6 层，但**没有 masked self-attention**（因为不需要自回归）。每层有三个子层：
1. **Self-attention（query 之间）：** N 个 queries 互相做 attention。表面上看是"每个 query 看到其他 query 的输出"，实际上是去重机制的核心——如果两个 query 高度相似（关注同一物体），self-attention 可以抑制其中一个。
2. **Cross-attention（query → encoder）：** 每个 query 从 encoder 特征（key/value）中检索信息。这是"哪里有物体"的决定性步骤——query 通过 cross-attention 定位到图像上与之最相关的区域。
3. **FFN：** 非线性变换。

**关键设计：** DETR 的 decoder 使用了**辅助损失（auxiliary losses）**——每个 decoder 层的输出都会过 prediction FFN 计算 loss。这有效缓解了 Transformer 深层训练的梯度消失问题，对最终精度贡献显著（Table 6 显示去掉辅助损失掉约 0.8 AP）。

### 5.6 Prediction FFN

每个 decoder 输出通过一个 3 层 MLP + 两个线性头：
- 隐藏层 256 维，ReLU
- LayerNorm
- Class head: Linear(256, num_classes + 1)
- Box head: Linear(256, 4) + sigmoid

注意 box head 输出的是直接坐标（归一化的 cx, cy, w, h），而不是像 Faster R-CNN 那样预测相对 anchor 的偏移量（Δx, Δy, Δw, Δh）。这是"无 anchor"的核心——预测绝对值而非相对值。

## 六、核心公式与数学直觉

### 公式 1: Hungarian Matching（匈牙利匹配）

$$\hat{\sigma} = \arg\min_{\sigma \in \mathfrak{S}_N} \sum_{i}^{N} \mathcal{L}_{\text{match}}(y_i, \hat{y}_{\sigma(i)})$$

**符号说明：**
- $y_i = (c_i, b_i)$: 第 $i$ 个 GT 物体，$c_i$ 是类别（可能是 ∅），$b_i \in [0,1]^4$ 是归一化 box 坐标
- $\hat{y}_{\sigma(i)} = (\hat{p}_{\sigma(i)}, \hat{b}_{\sigma(i)})$: 第 $\sigma(i)$ 个预测，$\hat{p}_{\sigma(i)}$ 是类别概率分布
- $\mathfrak{S}_N$: N 个元素的所有排列（N=100），共 100! 种可能——但匈牙利算法能在 $O(N^3)$ 内求解
- $\mathcal{L}_{\text{match}}$: 匹配代价，用于匹配阶段

**匹配代价的具体形式**（注意：这不是最终的 loss，只是"找配对"的代价）：

$$\mathcal{L}_{\text{match}}(y_i, \hat{y}_{\sigma(i)}) = -\mathbb{1}_{\{c_i \neq \varnothing\}} \,\hat{p}_{\sigma(i)}(c_i) + \mathbb{1}_{\{c_i \neq \varnothing\}} \,\mathcal{L}_{\text{box}}(b_i, \hat{b}_{\sigma(i)})$$

直觉：GT 物体 i 对预测 σ(i) 的"代价"是负的分类概率 + box 损失。代价越小，匹配越合适。

这里有一个设计细节：匹配代价里用的是 **probability** 而不是 log-probability。为什么？因为 log-probability 的数值范围（通常是 -1 到 -10）和 box loss 的数值范围差异太大，直接相加会让匹配纯粹由 box loss 主导。使用 probability（0~1 范围）可以让两者"可通约"（commensurable）。

**类比理解：** 这个匹配过程就像 HR 给 100 个 job candidates（预测）分配到 5 个岗位（GT 物体）。匈牙利算法找到全局最优的分配方案——每个岗位只分配一个人，确保总体匹配效果最好。

### 公式 2: Hungarian Loss（匈牙利损失）

$$\mathcal{L}_{\text{Hungarian}}(y, \hat{y}) = \sum_{i=1}^{N} \left[ -\log \hat{p}_{\hat{\sigma}(i)}(c_i) + \mathbb{1}_{\{c_i \neq \varnothing\}} \mathcal{L}_{\text{box}}(b_i, \hat{b}_{\hat{\sigma}(i)}) \right]$$

和匹配代价的区别：
- 匹配代价用 $\hat{p}$（probability），loss 用 $-\log \hat{p}$（negative log-likelihood）
- 匹配找最优排列 $\hat{\sigma}$ 用 $\mathcal{L}_{\text{match}}$，但计算梯度时用 $\mathcal{L}_{\text{Hungarian}}$

**类别不均衡处理：** 对 $c_i = \varnothing$ 的样本，$\log$ 项乘以因子 $1/10$（论文原文："down-weight the log-probability term when $c_i=\varnothing$ by a factor $10$"），以降低背景类在总 loss 中的比重。这和 Faster R-CNN 中对正负 proposal 做 1:3 采样的思路一致。

### 公式 3: 边界框损失

$$\mathcal{L}_{\text{box}}(b_i, \hat{b}_{\sigma(i)}) = \lambda_{\text{iou}} \mathcal{L}_{\text{iou}}(b_i, \hat{b}_{\sigma(i)}) + \lambda_{\text{L1}} \|b_i - \hat{b}_{\sigma(i)}\|_1$$

- $\mathcal{L}_{\text{iou}}$: Generalized IoU loss（Rezatofighi et al., 2019），尺度不变
- $\ell_1$ loss: 衡量绝对偏差
- $\lambda_{\text{iou}} = 2$, $\lambda_{\text{L1}} = 5$（论文设定的超参数，训练时归一化到 batch 内物体数）

为什么需要两个 loss？
- $\ell_1$ 对**尺度敏感**：同样的相对误差，大物体的 $\ell_1$ 比小物体大得多
- GIoU 是**尺度不敏感的**：不管框多大，IoU 都是 0~1
- 两者互补：$\ell_1$ 提供精确的坐标约束，GIoU 保证跨尺度公平

### 公式 4: 标准 Transformer Self-Attention

$$\text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

DETR 用的是标准版本，没有任何特殊魔改。$\sqrt{d_k}$ ($\sqrt{32}$) 的除法是标准化注意力权重方差的技巧，避免 softmax 后的梯度太小。

### 公式 5: 辅助损失（Auxiliary Loss）

每个 decoder 层 $l$ 的输出 $\mathbf{z}_l$ 都过一个共享的预测头：

$$\mathcal{L}_{\text{total}} = \sum_{l=1}^{L} \mathcal{L}_{\text{Hungarian}}(y, \text{FFN}(\mathbf{z}_l))$$

其中 $L=6$ 是 decoder 层数。这种方法也称为"deep supervision"（深度监督），在 dense prediction 任务中常见。DETR 的消融实验证实辅助损失对收敛和最终精度有显著帮助。


每个 query 的 cross-attention 权重在 encoder 特征图上形成热力分布。论文的实验（Figure 5）显示：
- 不同 query 关注不同区域，自然分出"负责不同物体"的 query
- 某些 query 关注物体整体（如整只熊），某些关注局部（如熊头、熊爪）
- 背景 query 的 attention 分布模糊，没有集中区域

这正是 object queries 工作机制的可视化证明。

### 图 3: Encoder Self-Attention（论文 Fig. 4）

DETR 的 encoder self-attention 可视化展示了全局推理能力：

```text
输入图片: 一群斑马在草地上

某个空间位置(x,y)的 query 在 self-attention 中关注了:
  ● 同一只斑马的其他部分       ── 强关注（同一物体）
  ● 周围的斑马                ── 中强关注（同类别）
  ● 草地背景                 ── 弱关注（背景）
  ● 天空                     ── 几乎无关注
```

论文发现：encoder 的 self-attention 已经展现了**实例分割的雏形**——同一物体的特征像素互相强关注，不同物体之间关注度弱一些。这说明 Transformer 的全局推理可以自然学会"物体图"的表示。

## 八、复杂度、效率与可扩展性分析

### 计算复杂度

DETR 的计算主要包括三部分：

**1. CNN Backbone:**
- ResNet-50 的标准 FLOPs：约 4.1 GFLOPs（224×224），实际输入 800×1333 时约 40-50 GFLOPs

**2. Transformer Encoder:**
$$\Omega(\text{Encoder}) = L \cdot (4hwC^2 + 2(hw)^2C)$$
其中 $h=25, w=42, C=256, L=6$：
- 每层 $4 \times 1050 \times 256^2 \approx 275M$（QKV + output 投影）
- 每层 $2 \times 1050^2 \times 256 \approx 564M$（attention 矩阵）
- 6 层总计约：$6 \times (275 + 564) \approx 5.0B$ FLOPs

Encoder 的 attention 复杂度 $O((hw)^2)$ 在这里是关键瓶颈。当输入分辨率翻倍时，token 数翻 4 倍，attention 翻 16 倍。这就是 DETR 高分辨率输入困难的根本原因。

**3. Transformer Decoder:**
Decoder 的计算包括每层的 self-attention ($N=100$ 个 query 互相做 attention)、cross-attention (每个 query 与 $hw \approx 1050$ 个 encoder 特征做 attention) 和 FFN。由于 $N \ll hw$，decoder 的总 FLOPs 约为 encoder 的 10-20%，远不是瓶颈。

**总占比：** Encoder 的 attention 是绝对算力瓶颈，约占总 FLOPs 的 60-70%。

### 训练成本

DETR 的显著缺点是训练慢：
- 需要 300 epoch（基准）甚至 500 epoch（最优）的 COCO 训练
- Faster R-CNN+ 只需 12× 对应约 36 epoch（3× 数据增强）
- DETR 的训练时间是 Faster R-CNN 的 **8-14 倍**
- 论文中使用 16 张 V100 训练 300 个 epoch 约需 3 天（单个 V100 时远超此时间）

### 推理速度

| 模型 | 骨干 | 推理时间 (ms) | FPS | AP |
|------|------|-------------|-----|----|
| Faster R-CNN+ | R50-FPN | 39 | 25.6 | 40.2 |
| Faster R-CNN+ | R101-FPN | 49 | 20.4 | 42.0 |
| DETR | R50 | 52 | 19.2 | 42.0 |
| DETR | R101 | 86 | 11.6 | 43.5 |

数据来自论文 Table 2（在 V100 上测量，batch size=1）。DETR 的推理速度大约是 Faster R-CNN 的 60-75%。原因：Transformer encoder 的 attention 计算量大，且无像卷积那样成熟的算子优化。

### 可扩展性分析

**正方面：**
- Object queries 数量 N 可以灵活调整（论文试验了 N=50、100、150、300）
- Encoder/decoder 层数可扩展
- 不需要 NMS 后处理，推理流程简单

**负方面：**
- Encoder attention 的 $O((hw)^2)$ 复杂度限制了高分辨率输入
- 100 个 queries 并行输出，大 batch 时显存压力大
- 训练 epoch 数随骨干网络加深反而需要更多（R101 需要 500 epoch 才能充分发挥）

DETR 在全景分割上显著超越当时的 SOTA，尤其在 stuff 类别上（PQ$^{st}$ 37.1 vs 33.4）。这是因为 object queries 的"集合预测"天然适合同时处理 things（物体）和 stuff（背景区域）的像素级预测。

### 9.6 大物体 vs 小物体的差距（论文分析）

论文给出的解释很直观：

> DETR 使用单个 32× 下采样特征图。大物体在特征图上占据几十个像素，有足够的信息被 query 检索到。小物体在特征图上可能只占 1-2 个像素，encoder 的 self-attention 可以"看到它们"，但 decoder 的 cross-attention 难以精确定位到这么小的区域。

这本质上是**分辨率-感受野的矛盾**：高分辨率特征图（如 8× 下采样）保留了小物体的空间细节，但 token 数量剧增（4×4=16 倍），encoder attention 的 $O((hw)^2)$ 复杂度会爆炸。低分辨率特征图 token 少但丢失了小物体。

### 关键代码片段解读

**1. Hungarian Matcher（matcher.py, ~40 行）**

```python
class HungarianMatcher(nn.Module):
    def forward(self, outputs, targets):
        # outputs: {'pred_logits': (B, N, C+1), 'pred_boxes': (B, N, 4)}
        # targets: list of dicts, len=B
        bs, num_queries = outputs["pred_logits"].shape[:2]

        # Flatten batch: (B*N, C+1) and (B*N, 4)
        out_prob = outputs["pred_logits"].flatten(0, 1).softmax(-1)
        out_bbox = outputs["pred_boxes"].flatten(0, 1)

        # 为每个样本计算匹配代价矩阵
        indices = []
        for i in range(bs):
            tgt_ids = targets[i]["labels"]    # (M,) 当前图片 M 个 GT
            tgt_bbox = targets[i]["boxes"]    # (M, 4)
            # 代价: -class_prob + box_loss
            cost_class = -out_prob[:, tgt_ids]  # (N, M)
            cost_bbox = torch.cdist(out_bbox, tgt_bbox, p=1)  # (N, M)
            cost_giou = -giou(out_bbox, tgt_bbox)  # (N, M)
            C = cost_bbox * 5 + cost_giou * 2 + cost_class
            # Hungarian algorithm (for loop over batch)
            from scipy.optimize import linear_sum_assignment
            row_ind, col_ind = linear_sum_assignment(C[i].detach().cpu().numpy())
            indices.append((row_ind, col_ind))
        return indices
```

注意 `scipy.optimize.linear_sum_assignment` 实现了匈牙利算法，是 $O(N^3)$ 复杂度。N=100 时约 1M 次操作，在批处理时会成为可感知的开销。

另一个设计细节：cost_class 用的是 **softmax probability** 而不是 logits——论文解释这样让分类代价和 box 代价的量级更为接近。

**2. DETR 主模型构造函数（detr.py）**

```python
class DETR(nn.Module):
    def __init__(self, backbone, transformer, num_classes, num_queries, aux_loss=True):
        super().__init__()
        self.transformer = transformer
        hidden_dim = transformer.d_model
        self.class_embed = nn.Linear(hidden_dim, num_classes + 1)
        self.bbox_embed = MLP(hidden_dim, hidden_dim, 4, 3)  # 3-layer MLP
        self.query_embed = nn.Embedding(num_queries, hidden_dim)
        self.input_proj = nn.Conv2d(backbone.num_channels, hidden_dim, kernel_size=1)
        self.backbone = backbone
        self.aux_loss = aux_loss
```

整个模型最复杂的部分就是这 ~20 行。论文声称"few hundred lines"实现了整个检测器，不是虚言。

**3. 位置编码生成（position_encoding.py）**

```python
class PositionEmbeddingSine(nn.Module):
    def forward(self, tensor):
        # tensor: (B, C, H, W)
        mask = tensor.new_zeros((tensor.shape[0], tensor.shape[2], tensor.shape[3]))
        # mask: 0=valid, 1=padding
        y_embed = mask.cumsum(1, dtype=torch.float32)  # 逐行累加
        x_embed = mask.cumsum(2, dtype=torch.float32)  # 逐列累加
        # 这里用输入图像的原始宽高比做归一化，不是特征图大小
        y_embed = y_embed / (y_embed[:, -1:, :] + 1e-6) * 2 * math.pi
        x_embed = x_embed / (x_embed[:, :, -1:] + 1e-6) * 2 * math.pi
        # ... 生成 sin/cos 编码 ...
```

位置编码是**以原始图像坐标归一化**后生成的，**不是在特征图上简单地用网格坐标**。这意味着不同宽高比的图像（如 800×1333 和 600×600 的 batch 混合）也能获得一致的相对位置表示。

**4. Transformer forward（transformer.py）**

```python
class Transformer(nn.Module):
    def forward(self, src, mask, query_embed, pos_embed):
        # src: (B, C, H, W) from backbone
        bs, c, h, w = src.shape
        # flatten spatial dims
        src = src.flatten(2).permute(2, 0, 1)  # (HW, B, C)
        pos_embed = pos_embed.flatten(2).permute(2, 0, 1)
        query_embed = query_embed.unsqueeze(1).repeat(1, bs, 1)  # (N, B, C)
        tgt = torch.zeros_like(query_embed)  # decoder 初值

        # Encoder
        memory = self.encoder(src, pos=pos_embed, src_key_padding_mask=mask)

        # Decoder (auxiliary outputs for aux_loss)
        hs = self.decoder(tgt, memory, query_pos=query_embed,
                         pos=pos_embed, src_key_padding_mask=mask)
        return hs, memory  # hs: (N_dec, B, N_q, C)
```

注意 decoder 的初始输入 `tgt` 是**全零张量**，object queries 通过 `query_pos` 参数注入 decoder 的 cross-attention。这和原始 Transformer decoder 的架构（将 target sequence 的 embedding 作为输入）不同——DETR 的 object queries 是作为"位置查询"而不是"内容"注入的。

### 论文未提及的实现细节

**1. 训练时的 gradient checkpointing。** 代码中包含可选的 checkpointing 来节省显存（`detr.py` 中 `checkpoint_meta_forward`）。

**2. Mask padding。** 输入图像被 padding 到 32 的倍数（DETR 要求严格除尽 32），padding 区域在 mask 中被标记，跳过 self-attention 计算。

**3. 损失归一化。** Box loss 的值被 batch 中的物体总数归一化（`denominator = max(1, n_objects)`），确保不同 batch 的损失量级一致。

**4. 训练时的数据增强。** DETR 使用了相对简单的数据增强（随机裁剪、颜色抖动、水平翻转），没有使用一些当时先进的增强（如 MixUp、CutMix 或 Mosaic）。

**5. Dropout。** Transformer 层默认使用了 0.1 的 dropout。论文正文没有讨论 dropout 的选择。

## 十一、related work 与技术谱系位置

### 直接前驱

- **[Vaswani et al., 2017] Attention Is All You Need**：Transformer 的基础。DETR 直接使用了标准 Transformer 的 encoder-decoder 架构。
- **[Ren et al., 2015] Faster R-CNN**：最经典的 two-stage 检测器。DETR 试图取代其复杂 pipeline。
- **[Lin et al., 2017] Feature Pyramid Networks (FPN)**：多尺度检测的标准做法。DETR 故意没使用 FPN——保持了架构简洁，但付出了小物体性能的代价。
- **[Redmon et al., 2016] YOLO**：最早的单阶段检测器，但依赖 anchor 和 grid。
- **[Rezatofighi et al., 2019] Generalized IoU**：DETR 使用的 box loss 之一。GIoU 在预测框完全脱离 GT 框时仍提供梯度。
- **[Stewart et al., 2016]（RNN-based detection）**：用 LSTM 自回归输出检测框，在较小的数据集上实验。DETR 用 transformer + 并行解码取代了 RNN，这是关键差异。

### 直接后继

DETR 开启了"transformer 检测器"这个子领域：
- **[Deformable DETR, Zhu et al., 2021]**：用可变形 attention 替代全局 attention，降低 encoder 复杂度，同时多尺度特征解决小物体问题。训练收敛从 300 epoch 降到 50 epoch。
- **[Conditional DETR, Meng et al., 2021]**：将 decoder cross-attention 分解成内容查询和空间查询，加速训练收敛。
- **[DAB-DETR, Liu et al., 2022]**：将 object queries 替换为 4D anchor box（动态 anchor boxes），让 queries 有明确的物理意义（中心坐标 + 宽高）。
- **[DN-DETR, Li et al., 2022]**：引入去噪训练（类似 MAE 的 mask 重建）来稳定 bipartite matching 的训练。
- **[DINO, Zhang et al., 2022]**：将 DN-DETR + DAB-DETR 融合 + contrastive denoising + look forward twice，在 COCO 上达到 SOTA。
- **[RT-DETR, Zhao et al., 2023]**：基于 DETR 的实时检测器，用高效混合编码器实现了 DETR 系的实时推理。

### 同期相关工作

- **[YOLOv4, Bochkovskiy et al., 2020]**（2020年4月，早于 DETR 约1个月）：anchor-based one-stage detector，大量 trick 但架构仍是卷积系。
- **[EfficientDet, Tan et al., 2020]**（2020年4月）：EfficientNet + BiFPN + compound scaling，仍是卷积系。
- **CenterNet / FCOS**（2019-2020）：anchor-free 检测器，但依然依赖 center/point 的启发式设计，没有完全端到端。

- ViT（Dosovitskiy et al., 2020）同样是 2020 年 10 月发表，但 DETR 的投稿（5 月）比 ViT 还早
- DETR 是第一篇**将 Transformer 用于核心视觉任务的论文**（在此之前 Transformer 在 CV 中主要用于关系建模的辅助模块，如 Relation Networks）
- 它和 ViT 一起为 2021 年的"Vision Transformer 大爆发"铺平了道路

## 十三、范式意义

DETR 的范式意义远不止于"又一个检测器"。

**1. 证明了 Transformer 可以做通用的视觉 decoder。** 在此之前，Transformer 在 CV 中的角色是 backbone（分类）、关系网络的辅助模块，或 attention 层的替换。DETR 展示了 Transformer decoder 可以作为一个通用的"视觉输出解码器"——给定一组 queries，从图像特征中检索并输出结构化预测。

**2. "Set prediction"范式的确立。** DETR 将目标检测从 dense prediction + post-processing 范式转变为 sparse set prediction 范式。这个范式后来被分割（MaskFormer, Mask2Former）、深度估计、多目标跟踪等任务广泛采纳。

**3. 手工设计的终点。** DETR 证明了"不需要 anchor、不需要 NMS、不需要 RPN"也能做出性能可接受的检测器。这改变了检测领域对"什么是必不可少的"的认知边界。

**4. 全景分割的统一框架。** DETR 的 segmentation 扩展展示了 object queries 可以同时做 thing 和 stuff 的分割——同一个 query 可以在一个任务中同时检测和分割。这为 MaskFormer/CondInst 等统一分割框架提供了思路。

**5. "简洁性优先"的设计哲学。** DETR 是整个后 Transformer 时代中架构最简洁的检测器之一。它的完整 PyTorch 实现不到 2000 行（包括分割扩展），而同时代的 Detectron2 有超过 10 万行。这种"简洁即优雅"的哲学后来影响了 Mask2Former、OneFormer 等。

## 十四、局限性与后续方向

### 主要局限

**1. 训练太慢。** 300 epoch 的训练时间是 Faster R-CNN 的 8-14 倍。这是 DETR "简洁"的最大的实用代价——训练成本高到大多数研究组无法随意实验。后续的 Deformable DETR（50 epoch）和 Conditional DETR（50 epoch）主要解决了这个问题。

**2. 小物体性能差。** AP$_S$ 比 Faster R-CNN+ 低约 1.5 个点。根本原因是 32× 下采样的单分辨率特征图信息量不足。后续改进（DC5、多尺度 deformable attention）解决了这个问题。

**3. Encoder attention 的可扩展性问题。** $O((hw)^2)$ 的复杂度让 DETR 在高分辨率输入（如 4K 视频帧）上不可行。Deformable DETR 的可变形 attention 将复杂度降到 $O(hwk)$ 是关键的改进方向。

**4. 硬编码的 N=100 上限。** 每张图片最多输出 100 个检测框，这在大多数 COCO 场景下够用，但在密集场景（如人群计数、细胞检测，可能有上千个物体）中会成为硬限制。

**5. Hungarian matching 的不可微分性。** 匈牙利算法本身是离散的（argmin），梯度只能通过 box/class loss 传播，不能通过匹配过程本身传播。这意味着模型无法"学习如何匹配"。这本质上是一个损失函数设计上的限制。

**6. 空间定位的不精确性。** DETR 直接预测绝对坐标（cx, cy, w, h），没有使用像 YOLO 或 Faster R-CNN 中的偏移量预测。论文发现 DETR 的 AP$_{75}$（高 IoU 阈值）低于 Faster R-CNN，说明在精确的空间定位上 DETR 仍有差距。

### 论文明确指出的后续方向

原文："We expect that future work will improve this aspect in the same way the development of FPN did for Faster R-CNN."——作者预见到了小物体问题可以通过多尺度特征改善。

"Training settings for DETR differ from standard object detectors in multiple ways. The new model requires extra-long training schedule and benefits from auxiliary decoding losses in the transformer."——作者承认训练效率是主要问题。

### 未在论文中讨论但重要的局限

**1. 对 backbone 分辨率的敏感性。** 大幅改变输入分辨率时，DETR 需要重新训练位置编码和 object queries，灵活性和 YOLO 系列相比有差距。

**2. Decoder 的并行输出特性。** 虽然并行输出是 DETR 的卖点，但这也意味着模型无法"先看全局再聚焦"。自回归方法（如 DenseCap 用的 RNN）可以在输出第二个框时看到第一个框的结果，而 DETR 在一开始就必须为所有 query 做决定。后续的 iterative refinement（如 Deformable DETR 的 iterative box refinement）部分缓解了这个问题。

**3. Object queries 的可解释性。** 虽然论文展示了 attention 可视化，但 object queries 的"学习分布"仍然不够直观——有些 query 始终在输出 ∅，但它们"知道"自己在做什么吗？后续 DAB-DETR 通过将 queries 替换为 4D anchors 解决了可解释性问题。

## 十五、小结

DETR 是目标检测领域的一座里程碑，尽管它不是 "全面超越" 的 SOTA 方案，但它做了一件更重要的事——**重新定义了"检测器可以做得多简单"**。

可以这样理解 DETR 的贡献层次：

**最底层：技术贡献。** 一个简单的 encoder-decoder transformer + Hungarian loss，完全端到端，不需要 anchor 和 NMS。在 COCO 上达到和优化充分的 Faster R-CNN+ 同等级别的 AP（42.0）。代码不到 2000 行。

**中间层：范式贡献。** "Set prediction" 范式——把结构化输出建模为 sets，用二分匹配做监督。这个范式在分割（MaskFormer, Mask2Former）、深度估计、多目标跟踪等任务中得到了推广。

**最上层：哲学贡献。** "简洁即力量"——当所有人都认为检测器必须复杂时，DETR 证明了最简洁的方案也能 work。它挑战了 anchor、NMS、RPN 的"必要性"，迫使社区重新思考"每个组件的真正价值是什么"。

DETR 之后，检测领域的重心从"设计更好的 anchor/NMS"转向了"如何让 transformer 检测器更高效、更稳定"。Deformable DETR 解决了复杂度和训练时间，DAB-DETR 解决了 queries 的可解释性，DN-DETR 和 DINO 解决了训练的稳定性。6 年后回看，DETR 开启了一条全新的技术路线，其影响力远远超出了它作为一个检测器的实际性能。

**DETR 的公式可以浓缩成一句话：**

$$\text{目标检测} = \text{CNN 特征} + \text{Transformer encoder} + \text{Transformer decoder} + \text{Hungarian loss}$$

**简单到极致，所以足够优雅。**
