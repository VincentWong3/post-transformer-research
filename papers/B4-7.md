# DINOv2: Learning Robust Visual Features without Supervision（2023）

**论文：** [arXiv](https://arxiv.org/abs/2304.07193) · Meta AI Research · 2023

---

## 一、先搞清楚坑在哪

自监督学习（Self-Supervised Learning, SSL）的目标是：**让模型在没有人工标注的数据上，学到通用的视觉特征**，然后在下游任务（分类、分割、检测）中直接使用或微调。

在 DINOv2 之前，自监督视觉特征学习有两条主要路线：

1. **对比学习**（Contrastive Learning）：如 SimCLR [Chen et al., 2020]、MoCo [He et al., 2020]。核心思想是让同一张图的不同增强视图在特征空间中靠近，不同图的视图远离。问题在于：需要大量负样本、batch size 很大、对数据增强敏感。

2. **掩码图像建模**（Masked Image Modeling, MIM）：如 MAE [He et al., 2022]、SimMIM [Xie et al., 2022]。核心思想是遮住图像的大部分 patch，让模型预测被遮住的内容。问题在于：学到的特征偏向纹理和低频信息，对高级语义（如物体类别）不够鲁棒。

这两条路线各有优劣，但有一个共同瓶颈：**数据规模**。当时的自监督方法大多在 ImageNet-1K（128 万张图）上训练。这远远不够——人类视觉系统的学习需要海量、多样化的视觉经验。

**DINOv2 的核心假设**：如果自监督方法本身已经足够好，那么限制其性能的瓶颈就是训练数据。只要把数据规模从百万级提升到亿级，特征质量就会有质的飞跃。

## 二、现有方法的真正问题

### 2.1 数据问题：不是方法不行，是数据不够

现有自监督方法（DINO [Caron et al., 2021]、iBOT [Zhou et al., 2022]）在 ImageNet-1K 上训练时，已经展现出不错的特征质量。但作者发现一个关键现象：

> **当训练数据从 ImageNet-1K 扩展到更大规模（如 Instagram 10 亿张图），特征质量的提升并不明显。**

这说明不是数据量的问题，而是**数据的多样性和质量**问题。ImageNet-1K 虽然干净，但类别有限（1000 类），场景单一。而互联网数据虽然量大，但噪声多、分布偏。

### 2.2 方法问题：自监督训练的不稳定性

现有自监督方法在扩展数据时面临两个技术挑战：

1. **训练不稳定**：大模型（ViT-g/14 有 11 亿参数）在大规模数据上训练时，loss 容易发散，需要精心调参。
2. **计算成本**：对比学习需要大量负样本，MIM 需要处理掩码 token，计算量随数据量线性增长。

### 2.3 评估问题：缺乏统一的评测基准

不同论文使用不同的下游任务、不同的微调策略、不同的评估协议，导致无法公平比较。需要像 NLP 领域 GLUE 那样的统一基准。

## 三、DINOv2 的核心思路

一句话概括：**用精心筛选的大规模数据集 + 改进的自蒸馏框架 + 高效训练策略，训练出一个通用的视觉特征提取器。**

具体来说，DINOv2 做了三件事：

1. **数据工程**：构建了一个包含 1.42 亿张图的高质量数据集 LVD-142M，通过去重、去噪、筛选来保证数据质量。
2. **方法改进**：结合 DINO 和 iBOT 两种自监督目标，加入新的正则化技术（Sinkhorn-Knopp 居中、KoLeo 正则化），让训练更稳定。
3. **高效训练**：使用 Flash Attention、混合精度训练、模型分片等技术，使 11 亿参数的模型能在 512 个 GPU 上训练。

最终得到的 ViT-g/14 模型在 16 个视觉任务上达到或超越 SOTA，且不需要任何标注数据。

## 四、网络架构详解

DINOv2 使用 Vision Transformer（ViT）作为骨干网络，没有使用卷积网络。核心架构是 ViT-g/14，即 giant 版本的 ViT，patch size 为 14x14。

### 4.1 整体架构图

```
输入图像: 224x224x3
    │
    ▼
┌─────────────────────────────────────────────────┐
│                  Patch Embedding                │
│  图像 → 16x16 patches (224/14=16)               │
│  每个 patch → 14x14x3 → 线性投影 → 1536 维     │
│  输出: 256 x 1536 (16x16+1个cls token)           │
└─────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────┐
│               Position Embedding                │
│  可学习的位置编码，加到每个 patch token 上       │
│  输出: 256 x 1536                               │
└─────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────┐
│             ViT Encoder (40 layers)             │
│                                                 │
│  Layer 1-40:                                    │
│  ┌─────────────────────────────────────┐        │
│  │  LayerNorm                           │        │
│  │  Multi-Head Self-Attention (MSA)     │        │
│  │     - 24 heads, 每头 64 维           │        │
│  │     - QKV 投影: 1536x4608            │        │
│  │     - 输出: 256 x 1536               │        │
│  │  Residual Connection                  │        │
│  │  LayerNorm                            │        │
│  │  MLP (GELU activation)               │        │
│  │     - 升维: 1536 → 6144 (4x)         │        │
│  │     - 降维: 6144 → 1536              │        │
│  │  Residual Connection                  │        │
│  └─────────────────────────────────────┘        │
│                                                 │
│  输出: 256 x 1536                               │
└─────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────┐
│               LayerNorm + Linear                │
│  提取 cls token 作为全局特征                      │
│  输出: 1536 维特征向量                           │
└─────────────────────────────────────────────────┘
```

### 4.2 前向过程详解

**Step 1: Patch Embedding**

输入图像 $x \in \mathbb{R}^{3 \times H \times W}$，其中 $H=W=224$。

ViT-g/14 的 patch size $P=14$，所以 patch 数量为：
$$N = \frac{H}{P} \times \frac{W}{P} = 16 \times 16 = 256$$

每个 patch 展平为 $P^2 \times 3 = 14^2 \times 3 = 588$ 维向量，然后通过线性投影映射到 $D=1536$ 维。

同时加一个可学习的 cls token，所以输入序列长度为 $N+1 = 257$。

**Step 2: Position Embedding**

可学习的位置编码 $p \in \mathbb{R}^{257 \times 1536}$，加到 patch token 上。

**Step 3: Transformer Encoder**

40 层 Transformer block，每层包含：
- **Multi-Head Self-Attention (MSA)**：24 个头，每头维度 $d_k = 1536 / 24 = 64$
- **MLP**：两层，隐藏层维度 $D_{ff} = 4 \times 1536 = 6144$

MSA 的计算复杂度为：
$$O(4 \cdot N \cdot D^2 + 2 \cdot N^2 \cdot D)$$

代入 $N=257, D=1536$：
- QKV 投影：$4 \times 257 \times 1536^2 \approx 2.42 \times 10^9$
- 注意力矩阵：$2 \times 257^2 \times 1536 \approx 2.03 \times 10^8$

每层 MSA 约 2.6 GFLOPs。

MLP 的计算复杂度为：
$$O(2 \cdot N \cdot D \cdot D_{ff}) = 2 \times 257 \times 1536 \times 6144 \approx 4.85 \times 10^9$$

每层 MLP 约 4.85 GFLOPs。

40 层总计约：$40 \times (2.6 + 4.85) \approx 298$ GFLOPs。

**Step 4: 特征提取**

取 cls token 的输出作为图像的全局特征表示，维度为 1536。

### 4.3 为什么用 ViT 而不是 CNN？

作者在消融实验（Table 5）中比较了 ViT 和 ConvNeXt，发现：
1. ViT 在大规模预训练后，特征质量提升更明显。
2. ViT 的 patch token 天然适合做密集预测任务（分割、检测）。
3. ViT 的架构更简单，没有归纳偏置，能更好地从海量数据中学习。

## 五、核心创新点

### 创新点一：大规模高质量数据集 LVD-142M

#### (a) 痛点与动机

现有自监督方法的数据集要么太小（ImageNet-1K，128 万张），要么太脏（未经筛选的互联网数据）。作者发现：**数据质量比数据量更重要**。

[Goyal et al., 2021] 的 SEER 模型在 10 亿张 Instagram 图上训练，但特征质量并不比 ImageNet-1K 上训练的 DINO 好多少。这说明原始互联网数据中的噪声（低质量图片、重复内容、不相关图片）会限制特征学习。

#### (b) 方案细节

LVD-142M 的构建流程：

1. **数据源**：从 14 个公开数据集收集原始图片，包括 ImageNet-22K、LAION-400M 的子集、YFCC100M 的子集等。

2. **去重**：使用基于特征的近似去重（Deduplication），计算每张图片的 DINO 特征，用近似最近邻搜索去除相似度 > 0.8 的图片。这一步去掉了约 30% 的数据。

3. **质量筛选**：训练一个二分类器，区分 ImageNet-1K 图片和随机互联网图片。只保留分类器得分 > 0.5 的图片。这一步筛掉了约 40% 的低质量图片。

4. **类别均衡**：使用 k-means 聚类（k=10000），确保每个类别的图片数量不超过 100 万张。避免某些类别（如"猫"）过多。

最终得到 1.42 亿张图片，分布覆盖了自然图像、文本图像、医疗图像、卫星图像等。

#### (c) 为什么有效

从 Table 1 可以看到，在 ImageNet-1K 上训练时，DINOv2 和 DINO 差距不大（82.9% vs 82.6%）。但换到 LVD-142M 后，DINOv2 的 linear probing 准确率提升了 6.5%（从 82.9% 到 89.4%）。

这说明：**数据多样性和质量是特征质量提升的关键**。LVD-142M 比 ImageNet-1K 多了 100 倍的图片，覆盖了更广泛的视觉概念，使模型能学到更通用的特征。

#### (d) 与 Related Work 的关系

- 与 [Goyal et al., 2021] 的 SEER 相比，LVD-142M 多了质量筛选和去重步骤。
- 与 [Mahajan et al., 2018] 的弱监督方法相比，DINOv2 不需要标签，完全自监督。
- 与 CLIP [Radford et al., 2021] 的 4 亿图文对相比，LVD-142M 只有图片没有文本，但通过自监督学到了语义特征。

#### (e) 如果去掉会怎样

Table 1 的消融实验显示：如果只用 ImageNet-1K 训练，DINOv2 的 linear probing 准确率只有 82.9%，比 LVD-142M 训练的结果低 6.5%。这说明数据规模是 DINOv2 成功的关键因素。

### 创新点二：改进的自蒸馏框架

#### (a) 痛点与动机

DINO [Caron et al., 2021] 和 iBOT [Zhou et al., 2022] 是两种互补的自监督方法：
- DINO 做全局特征对比（global-to-global）：让 teacher 和 student 的 cls token 输出一致。
- iBOT 做局部特征预测（local-to-global）：让 student 预测 teacher 对掩码 patch 的输出。

两者单独使用都有局限：DINO 忽略了局部细节，iBOT 忽略了全局语义。作者想结合两者的优势。

#### (b) 方案细节

DINOv2 的训练目标包含三个部分：

**第一部分：DINO 损失（全局-全局对比）**

对于同一张图的两个不同增强视图 $v_1, v_2$，teacher 和 student 分别提取 cls token 特征 $p_t, p_s$。损失函数为交叉熵：

$$\mathcal{L}_{DINO} = - \sum_{i=1}^{K} p_t^{(i)} \log p_s^{(i)}$$

其中 $K$ 是输出维度（默认 65536），$p_t, p_s$ 经过 softmax 归一化。

**第二部分：iBOT 损失（局部-全局预测）**

随机遮住 student 输入中 30% 的 patch，让 student 预测 teacher 对完整图像对应 patch 的输出：

$$\mathcal{L}_{iBOT} = - \frac{1}{M} \sum_{j=1}^{M} \sum_{i=1}^{K} q_t^{(i,j)} \log q_s^{(i,j)}$$

其中 $M$ 是被遮住的 patch 数量，$q_t^{(i,j)}$ 是 teacher 对第 $j$ 个 patch 的第 $i$ 维输出。

**第三部分：Sinkhorn-Knopp 居中**

为了防止 teacher 输出坍缩到均匀分布，使用 Sinkhorn-Knopp 算法对 teacher 输出进行居中（centering）。具体来说，teacher 的输出 $p_t$ 会经过以下变换：

$$\tilde{p}_t = \text{Sinkhorn}(p_t, \tau_t)$$

其中 $\tau_t$ 是 teacher 的温度参数（默认 0.04）。Sinkhorn-Knopp 算法迭代地调整 $p_t$，使其边际分布接近均匀分布。

**第四部分：KoLeo 正则化**

为了增加 batch 内样本的多样性，加入 KoLeo 正则化项：

$$\mathcal{L}_{KoLeo} = - \frac{1}{B} \sum_{i=1}^{B} \log \left( \min_{j \neq i} \|z_i - z_j\|_2 \right)$$

其中 $B$ 是 batch size，$z_i$ 是第 $i$ 个样本的特征向量。

总损失为：
$$\mathcal{L} = \mathcal{L}_{DINO} + \lambda_1 \mathcal{L}_{iBOT} + \lambda_2 \mathcal{L}_{KoLeo}$$

其中 $\lambda_1 = 0.5, \lambda_2 = 0.1$。

#### (c) 为什么有效

从 Table 3 可以看到：
- 单独用 DINO 损失：linear probing 86.3%
- 单独用 iBOT 损失：linear probing 85.7%
- 两者结合：linear probing 88.1%

结合后提升了 1.8-2.4%，说明全局和局部信息是互补的。

KoLeo 正则化进一步提升了 0.5%（从 88.1% 到 88.6%），说明增加 batch 内多样性有助于特征学习。

#### (d) 与 Related Work 的关系

- DINO 损失来自 [Caron et al., 2021] DINO。
- iBOT 损失来自 [Zhou et al., 2022] iBOT。
- Sinkhorn-Knopp 居中来自 [Caron et al., 2020] SwAV。
- KoLeo 正则化来自 [Sablayrolles et al., 2019] Spreading vectors。

DINOv2 的创新在于：**将这些互补的目标整合到一个框架中，并找到了最优的超参数组合**。

#### (e) 如果去掉会怎样

Table 3 的消融实验：
- 去掉 iBOT 损失：linear probing 下降 1.8%（从 88.1% 到 86.3%）
- 去掉 KoLeo 正则化：linear probing 下降 0.5%（从 88.6% 到 88.1%）
- 去掉 Sinkhorn-Knopp 居中：训练发散，无法收敛

### 创新点三：高效训练策略

#### (a) 痛点与动机

ViT-g/14 有 11 亿参数，在 1.42 亿张图上训练，如果使用标准的训练策略，需要数万 GPU 天。作者需要一系列工程优化来降低训练成本。

#### (b) 方案细节

**1. Flash Attention**

使用 [Dao et al., 2022] 的 Flash Attention，将注意力计算的复杂度从 $O(N^2)$ 降低到 $O(N^2 / M)$，其中 $M$ 是 SRAM 大小。实际加速约 2-3 倍。

**2. 混合精度训练**

使用 bfloat16 混合精度训练，在保持精度的同时减少显存占用。所有 linear 和 attention 操作使用 bfloat16，layer norm 和 softmax 使用 float32。

**3. 模型分片（FSDP）**

使用 Fully Sharded Data Parallel（FSDP），将模型参数、梯度和优化器状态分片到 512 个 GPU 上。每个 GPU 只存储 1/512 的模型。

**4. 梯度累积**

使用梯度累积（gradient accumulation）来模拟更大的 batch size。实际 batch size 为 2048，通过 512 个 GPU 每步处理 4 张图实现。

**5. 学习率调度**

使用余弦退火学习率调度，初始学习率 $2 \times 10^{-4}$，warmup 10000 步，最终学习率 $2 \times 10^{-6}$。

#### (c) 为什么有效

从论文的 Appendix D 可以看到：
- 使用 Flash Attention 后，训练速度提升 2.3 倍。
- 使用 FSDP 后，可以在 512 个 GPU 上训练 11 亿参数模型。
- 总训练时间约 5000 GPU 天（512 个 GPU 训练约 10 天）。

#### (d) 与 Related Work 的关系

- Flash Attention 来自 [Dao et al., 2022]。
- FSDP 来自 [Zhao et al., 2023] PyTorch FSDP。
- 混合精度训练来自 [Micikevicius et al., 2018]。

DINOv2 的创新在于：**将这些技术组合成一个高效的训练 pipeline**，使得大规模自监督训练变得可行。

#### (e) 如果去掉会怎样

如果不使用 Flash Attention，训练时间会延长 2.3 倍，从 10 天变成 23 天。如果不使用 FSDP，11 亿参数的模型无法在 512 个 GPU 上训练（需要更多 GPU 或更大的显存）。

## 六、公式详解

### 公式 1：DINO 损失

$$\mathcal{L}_{DINO} = - \sum_{i=1}^{K} p_t^{(i)} \log p_s^{(i)}$$

**(a) 符号定义**
- $K$：输出维度，默认 65536
- $p_t^{(i)}$：teacher 网络输出的第 $i$ 维，经过 softmax 归一化
- $p_s^{(i)}$：student 网络输出的第 $i$ 维，经过 softmax 归一化

**(b) 公式来源**
这是标准的多类交叉熵损失，来自 [Caron et al., 2021] DINO 论文。

**(c) 推导过程**
teacher 和 student 的输出经过 softmax：
$$p_t^{(i)} = \frac{\exp(g_t^{(i)} / \tau_t)}{\sum_{j=1}^{K} \exp(g_t^{(j)} / \tau_t)}$$
$$p_s^{(i)} = \frac{\exp(g_s^{(i)} / \tau_s)}{\sum_{j=1}^{K} \exp(g_s^{(j)} / \tau_s)}$$

其中 $g_t, g_s$ 是 teacher 和 student 的 logits，$\tau_t, \tau_s$ 是温度参数（$\tau_t = 0.04, \tau_s = 0.1$）。

损失函数就是 teacher 输出和 student 输出的交叉熵。

**(d) 直觉理解**
DINO 损失让 student 模仿 teacher 的预测分布。由于 teacher 是 student 的指数移动平均（EMA），这相当于一种自蒸馏：student 不断学习自己的"未来版本"的预测。

关键设计：teacher 的温度更低（$\tau_t=0.04$），所以 teacher 的输出更"尖锐"（接近 one-hot），而 student 的温度更高（$\tau_s=0.1$），输出更"平滑"。这迫使 student 不仅要匹配 teacher 的预测，还要学习预测的置信度。

**(e) 边界情况**
- 当 $\tau_t \to 0$ 时，teacher 的输出退化为 one-hot，损失退化为 hard label 的交叉熵。
- 当 $\tau_t \to \infty$ 时，teacher 的输出退化为均匀分布，损失为常数，无法学习。
- 当 $K=1$ 时，损失退化为 $-\log p_s$，即二分类交叉熵。

### 公式 2：iBOT 损失

$$\mathcal{L}_{iBOT} = - \frac{1}{M} \sum_{j=1}^{M} \sum_{i=1}^{K} q_t^{(i,j)} \log q_s^{(i,j)}$$

**(a) 符号定义**
- $M$：被遮住的 patch 数量，默认 $M = 0.3 \times 256 \approx 77$
- $q_t^{(i,j)}$：teacher 对第 $j$ 个 patch 的第 $i$ 维输出
- $q_s^{(i,j)}$：student 对第 $j$ 个 patch 的第 $i$ 维输出

**(b) 公式来源**
来自 [Zhou et al., 2022] iBOT 论文。

**(c) 推导过程**
与 DINO 损失类似，但作用于 patch token 而不是 cls token。student 的输入是被遮住 30% patch 的图像，teacher 的输入是完整图像。student 需要预测被遮住 patch 的 teacher 输出。

**(d) 直觉理解**
iBOT 损失让 student 从局部信息（可见 patch）推断被遮住的全局信息。这迫使模型学习到图像的局部-全局关系，而不是仅仅依赖全局特征。

例如，看到一只猫的耳朵和尾巴，模型应该能推断出这是一只猫，即使看不到猫的全身。

**(e) 边界情况**
- 当 $M=0$ 时，iBOT 损失退化为 DINO 损失。
- 当 $M=256$ 时，所有 patch 都被遮住，student 无法从输入获得任何信息，只能随机猜测。

### 公式 3：KoLeo 正则化

$$\mathcal{L}_{KoLeo} = - \frac{1}{B} \sum_{i=1}^{B} \log \left( \min_{j \neq i} \|z_i - z_j\|_2 \right)$$

**(a) 符号定义**
- $B$：batch size，默认 2048
- $z_i$：第 $i$ 个样本的特征向量，维度 1536
- $\| \cdot \|_2$：L2 范数

**(b) 公式来源**
来自 [Sablayrolles et al., 2019] Spreading vectors。

**(c) 推导过程**
对于 batch 中的每个样本 $i$，找到与其最接近的样本 $j$（欧氏距离最小），然后最大化这个最小距离的对数。

**(d) 直觉理解**
KoLeo 正则化鼓励 batch 内的样本在特征空间中均匀分布，避免样本坍缩到几个聚类中心。这类似于对比学习中的负样本作用，但更轻量（不需要额外的负样本队列）。

举个例子：如果 batch 中有 10 张猫的图片，KoLeo 会让它们的特征在猫的类别内部也分散开（比如不同品种、不同姿势），而不是全部坍缩到同一个点。

**(e) 边界情况**
- 当 $B=1$ 时，无法计算 $j \neq i$ 的最近邻，KoLeo 损失为 0。
- 当特征空间维度很低时（如 $D=2$），KoLeo 的效果有限，因为空间容量小。

## 十一、小结

DINOv2 的核心贡献是证明了：**当自监督方法遇到足够大、足够干净的数据时，可以超越有监督学习**。

具体来说，DINOv2 做了三件事：
1. **数据工程**：构建了 1.42 亿张图的高质量数据集 LVD-142M。
2. **方法改进**：融合 DINO 和 iBOT 的损失，加入 Sinkhorn-Knopp 居中和 KoLeo 正则化。
3. **工程优化**：使用 Flash Attention、FSDP 等技术，使 11 亿参数模型的高效训练成为可能。

最终，DINOv2 在 16 个视觉任务上达到或超越 SOTA，且不需要任何标注数据。这是自监督视觉学习的一个重要里程碑，证明了"数据 + 工程"可以超越"方法创新"。

**更广泛的影响**：DINOv2 改变了视觉特征学习的范式。在此之前，视觉领域的主流是"有监督预训练 + 下游微调"；在此之后，"自监督预训练 + 零样本评估"成为新的标准。DINOv2 的特征被广泛应用于 SAM、EVA 等后续工作，推动了视觉基础模型的发展。
