# DINOv2: Learning Robust Visual Features without Supervision（2023）

**论文：** [arXiv](https://arxiv.org/abs/2304.07193) · Meta AI Research · 2023

---

## 一、先搞清楚坑在哪

DINO [Caron et al., 2021] 证明了自蒸馏框架在 ViT 上的强大潜力——自监督 ViT 的注意力图能自动分割前景物体。iBOT [Zhou et al., 2022] 进一步引入掩码图像建模（MIM），让自监督 ViT 学到更精细的 patch-level 特征。

但问题很清楚：**这些方法都在小数据上训练**。ImageNet-1K 只有 128 万张图，ImageNet-22K 也只有 1400 万张。而 NLP 领域的大模型（GPT-3、PaLM）都在海量文本上训练——几万亿 token。

所以一个自然的问题浮现：**自监督视觉方法本身已经够好了，限制它们的是数据规模吗？如果把数据从百万级扩展到亿级，特征质量会有多大提升？**

DINOv2 就是冲着这个问题去的。

与此同时，视觉领域有一个明显的分裂：**CLIP 类方法（文本监督）**在图像级任务（分类、检索）上很强，但在像素级任务（分割、深度估计）上不如自监督方法；**自监督方法**在像素级任务上有优势，但整体性能落后于 CLIP。理想情况下，一个"视觉基础模型"应该在所有任务上都能用——不管你是做分类、分割还是检索，都能直接提取特征，零样本搞定。

DINOv2 的目标就是做出这样一个**通用的视觉特征提取器**，不依赖任何文本标注。

## 二、现有方法的真正问题

### 2.1 数据问题：自监督方法用脏数据效果不好

到 2022 年，已经有工作尝试把自监督方法扩展到大规模数据：

- **[Goyal et al., 2021] SEER**：在 Instagram 10 亿张图上训练对比学习模型，使用 SwAV 方法。但特征质量提升有限。
- **[Goyal et al., 2022] SEERv2**：扩展到更大的模型和数据，但仍然不如 CLIP。

作者发现核心问题：**不是数据量不够，是数据质量不行**。互联网数据（Instagram、LAION）噪声很大：低质量图片、重复内容、不相关图片太多。自监督方法对数据质量极其敏感——因为它没有标签来"纠正"错误的学习信号。

这在直觉上说得通：CLIP 有文本作为 weak signal，能在噪声数据中挑出有用的视觉概念；自监督方法只能靠图像之间的相似性，如果数据里大量是噪声图像，学到的特征自然差。

### 2.2 方法问题：大规模自监督训练不稳定

现有自监督方法在小数据上工作良好，但扩展到大模型和数据后，会遇到两个实际问题：

1. **训练不稳定**：DINO/iBOT 的 teacher-student 框架在大模型上容易崩塌。Loss 会突然发散，需要反复调参。
2. **计算成本**：对比学习需要大 batch 和大量负样本，掩码图像建模需要处理掩码 token。随着模型参数和数据量增长，计算成本线性增长。

作者在论文中提到的关键技术痛点：**iBOT 的 DINO head 和 iBOT head 共享参数在小数据上效果好，但在大规模数据上反而不好。** 这说明小数据上的设计决策不能直接迁移到大尺度训练。

### 2.3 评估问题：缺少公平比较

不同论文使用不同的下游任务、不同的微调策略、不同的评估协议，导致无法公平比较。DINOv2 需要在一个统一的基准上评估：包括图像分类（linear probing + finetuning）、密集预测（分割、深度估计）、图像检索、域泛化等 16+ 个任务。

## 三、DINOv2 的核心思路

一句话概括：**用自动构建的高质量数据集 + 改进的自蒸馏框架 + 高效训练策略，训练一个通用的视觉特征提取器，无需任何监督信号。**

具体来说，DINOv2 做了三件事：

1. **数据工程**：构建 LVD-142M——1.42 亿张高质量图像。通过去重、自监督检索、聚类重平衡，从不干净的互联网数据中筛选出干净、多样、平衡的数据集。
2. **方法改进**：融合 DINO（图像级目标）+ iBOT（patch 级目标）+ Sinkhorn-Knopp 居中 + KoLeo 正则化。每个组件都经过消融验证。
3. **高效训练**：使用 Flash Attention、FSDP、混合精度训练，使 ViT-g/14（11 亿参数）能在 512 个 GPU 上高效训练。

最终得到的 ViT-g/14 模型：
- **Linear probing on ImageNet-1K：89.4%**（超过 CLIP ViT-G/14 的 89.1%）
- 在 16 个视觉任务上达到或超越现有 SOTA
- **完全无监督**——不需要任何标注数据就能达到这样的性能

## 四、网络架构详解

DINOv2 使用 Vision Transformer（ViT）作为骨干网络，提供 ViT-S/14、ViT-B/14、ViT-L/14、ViT-g/14 四种规格。

### 4.1 架构规格对比

| 模型 | #Layers | Hidden Dim | #Heads | Head Dim | #Params |
|------|---------|-----------|--------|---------|---------|
| ViT-S/14 | 12 | 384 | 6 | 64 | 21M |
| ViT-B/14 | 12 | 768 | 12 | 64 | 86M |
| ViT-L/14 | 24 | 1024 | 16 | 64 | 304M |
| ViT-g/14 | 40 | 1536 | 24 | 64 | 1.1B |

### 4.2 整体架构图

```
输入图像: 224x224x3
    │
    Patch Embedding (14×14 patches)
    │
    256 patch tokens + 1 cls token (位置编码)
    │
    ├── 16×16 = 256 patches (ViT-g/14: 1536维)
    │
    Transformer Encoder (L layers)
    │
    ├── Layer: 
    │   ├── LayerNorm
    │   ├── Multi-Head Self-Attention
    │   ├── Residual
    │   ├── LayerNorm
    │   └── MLP (GELU, 4× hidden)
    │
    [CLS] token 输出 → 全局特征
    Patch tokens 输出 → 局部特征
    │
    ├── 两种 head:
    │   ├── DINO head (MLP): 处理 [CLS] token → K=65536 维
    │   └── iBOT head (MLP): 处理 patch tokens → K=65536 维
```

### 4.3 为什么用 ViT-g/14？

作者选择了 ViT-g/14（giant 变体，14×14 patch）作为最大模型。这个选择有以下考量：

- **更大的模型容量**：40 层 Transformer、1536 维隐藏层、24 个注意力头，能吸收海量数据中的知识
- **patch size 14 比 16 更小**：224×224 输入下，14×14 产生 256 个 patch（比 16×16 的 196 个多 30%），提供更精细的空间粒度
- **下游蒸馏友好**：大模型训练好后可以蒸馏成小模型（ViT-S/B/L），降低部署成本

### 4.4 投影头的差异

与 DINO 相比，DINOv2 的一个重要改动：

- **DINO**：单投影头，输出维度 K=65536，共享给 cls token 和 patch token
- **DINOv2**：**两个独立的投影头**——DINO head（处理 cls token）+ iBOT head（处理 patch tokens）

这个看起来很小的改动在大规模训练下至关重要。作者发现：**共享头在小数据上有效（iBOT 论文结论），但在大规模数据上反而有害。** 因为 cls token 和 patch token 的学习目标本质上不同，强制共享参数限制了各自的学习空间。

## 五、创新点一：LVD-142M 数据集构建

**这是 DINOv2 最大的贡献——可能是整个项目最重要的部分。**

### 5.1 动机

作者从 NLP 中获取灵感：NLP 的大模型（GPT-3）在海量文本上训练前，通常会对数据做精细的预处理（去重、质量筛选、类别平衡）[Wenzek et al., 2020]。**数据质量比数据量更重要。**

但视觉领域的数据处理一直很粗糙：要么用小量人工标注的干净数据（ImageNet），要么用大量未经筛选的互联网数据（Instagram）。

### 5.2 数据来源

**Curated sources（种子数据集）**：
- ImageNet-22K（1400 万张，22K 类）
- ImageNet-1K train split（128 万张）
- Google Landmarks
- 多个细粒度分类数据集（具体见附录 Table 15）

**Uncurated source（互联网数据）**：
- 从公开的网页抓取数据中提取图像 URL
- 经过安全过滤（NSFW、人脸模糊、PCA hash 去重）
- 最终得到 **12 亿张唯一图片**

### 5.3 处理流程

**Step 1: 去重（Deduplication）**
- 使用 Pizzi et al. [2022] 的 copy detection pipeline
- 移除近似重复图片，提高多样性
- 同时移除与任何下游 benchmark 的 test/validation 集相似的图片（防止数据泄露）

**Step 2: 自监督图像检索**
- 使用预训练的 ViT-H/16（自监督，在 ImageNet-22K 上训练）提取所有图片的 embedding
- 对 12 亿张 uncurated 图片做 k-means 聚类
- 对每张 curated 种子图片，从 uncurated 池中检索 4 个最近邻（cosine similarity）
- 对于小种子数据集，改为从对应聚类中采样 M 张图片

**Step 3: 重平衡**
- k-means 聚类后，限制每个聚类内的图片数量
- 防止某些视觉概念（如"猫"、"人脸"）过度代表

**为什么用 N=4？** 更大的 N 会产生更多碰撞——多张种子图片检索到同一张 uncurated 图片，反而降低了多样性。N=4 提供了最佳平衡。

### 5.4 工程实现

- 使用 Faiss [Johnson et al., 2019] 进行 GPU 加速的近似最近邻搜索
- 20 个节点，每个 8×V100-32GB GPU
- 整个处理流程 **不到 2 天**

### 5.5 效果验证

| 训练数据 | 模型 | Linear Probing |
|----------|------|---------------|
| ImageNet-1K (1.28M) | ViT-g/14 | 82.9% |
| ImageNet-22K (14M) | ViT-g/14 | 85.3% |
| LVD-142M (142M) | ViT-g/14 | **89.4%** |

数据规模从 128 万→1400 万→1.42 亿，性能持续提升，且**没有饱和迹象**。这说明 LVD-142M 的数据质量支持继续扩展模型和数据。

### 5.6 与 Related Work 的关系

- 与 [Goyal et al., 2021] SEER 的区别：SEER 用 uncurated Instagram 数据，DINOv2 做了精细的检索筛选
- 与 CLIP [Radford et al., 2021] 的区别：CLIP 需要图文对，DINOv2 完全不需要文本
- 与 [Schuhmann et al., 2022] LAION 的区别：LAION 使用 CLIP 过滤文本-图像对，DINOv2 完全自监督

### 5.7 如果去掉会怎样

这是 DINOv2 最核心的贡献。从消融表（Table 1）看：
- 去掉 LVD-142M（回到 ImageNet-1K）：linear probing 下降 **6.5%**（89.4% → 82.9%）
- 数据贡献 > 方法改进贡献

## 六、创新点二：改进的自蒸馏框架

### 6.1 整体训练框架

DINOv2 的训练目标 = DINO 损失（全局级） + iBOT 损失（patch 级） + KoLeo 正则化（batch 级），使用 Sinkhorn-Knopp 居中替代原有 centering。

**差异速览（vs DINO/iBOT）**：

| 组件 | DINO | iBOT | DINOv2 |
|------|------|------|--------|
| 自蒸馏（EMA teacher） | ✓ | ✓ | ✓ |
| Multi-crop | ✓ | ✓ | ✓ |
| [CLS] token 对比 | ✓ | ✗ | ✓（无参数共享） |
| Masked patch 预测 | ✗ | ✓ | ✓（无参数共享） |
| 居中策略 | 动量 centering | 动量 centering | **Sinkhorn-Knopp** |
| 正则化 | ✗ | ✗ | **KoLeo** |
| 高分辨率微调 | ✗ | ✗ | **✓** |

### 6.2 DINO 损失（图像级）

与 DINO 完全相同：对同一张图的两个不同增强视图，让 student [CLS] token 的输出匹配 teacher [CLS] token 的输出。

$$\mathcal{L}_{DINO} = - \sum p_t \log p_s$$

其中 $p_t$ 和 $p_s$ 是经过 softmax 归一化的 teacher/student 输出。

### 6.3 iBOT 损失（patch 级）

与 iBOT 相同：随机遮蔽 student 输入中 30% 的 patch，让 student 预测被遮蔽 patch 的 teacher 输出。

$$\mathcal{L}_{iBOT} = - \frac{1}{M} \sum_{i=1}^{M} \sum p_{ti} \log p_{si}$$

其中 $M$ 是被遮蔽的 patch 数量，$p_{ti}$ 和 $p_{si}$ 是第 $i$ 个 patch 的 teacher/student 分布。

### 6.4 两个关键改动

**改动 1：分离 DINO 和 iBOT head 的参数**

在 iBOT 原文中，DINO head 和 iBOT head 共享 MLP 参数。作者在大规模训练中发现：**共享反而降低性能**。因此 DINOv2 使用两个独立的 MLP head，各自学习。

**直觉**：DINO head 要表达"这张图整体是什么"，iBOT head 要表达"这个 patch 在图中是什么"。前者的目标是全局语义，后者是局部结构。在大数据下，这两种目标的差异足够大，共享参数会限制各自的学习。

**改动 2：Sinkhorn-Knopp 居中替代动量 centering**

DINO 的 centering 使用动量更新：$c \leftarrow m c + (1-m) \frac{1}{B} \sum g_{\theta_t}(x_i)$。DINOv2 替换为 SwAV [Caron et al., 2020] 的 Sinkhorn-Knopp 算法：

- 对 teacher 输出做 3 次 Sinkhorn-Knopp 迭代
- 强制 teacher 的 batch 内输出分布接近均匀
- 效果更稳定，防止 teacher 在训练早期崩塌

### 6.5 KoLeo 正则化

KoLeo（Kozachenko-Leonenko）正则化是对比学习中负样本的一种轻量替代：

$$\mathcal{L}_{KoLeo} = - \frac{1}{B} \sum_{i=1}^{B} \log \left( \min_{j \neq i} \|z_i - z_j\|_2 \right)$$

**直觉**：对于 batch 中的每个样本，找到离它最近的另一个样本，然后"推开"它。这鼓励 batch 内的特征在空间上均匀分布，避免坍塌到几个聚类中心。

**为什么有效**：
- 不需要负样本队列（对比学习需要）
- 计算开销极小（O(B²) 但 B=2048 可接受）
- 与自蒸馏损失正交互补

### 6.6 高分辨率微调

作为最后一步，作者在 224×224 预训练后，把分辨率提高到 322×322（或 518×518），微调少量步数。这对密集预测任务（分割、深度估计）特别重要——小 patch 在大输入下能捕捉更精细的结构。

### 6.7 消融证据

| 配置 | Linear Probing | k-NN |
|------|---------------|------|
| 基础（DINO loss only） | 86.3% | 82.4% |
| + iBOT loss | 88.1% | 83.9% |
| + KoLeo | 88.6% | 84.3% |
| + SK centering | **89.4%** | **85.0%** |

每个组件都带来可测量的提升，组件之间互补——说明这不是某一个 trick 在起作用，而是整体设计的成功。

### 6.8 与 Related Work 的关系

- 前驱：DINO [Caron et al., 2021] + iBOT [Zhou et al., 2022] + SwAV centering [Caron et al., 2020] + KoLeo [Sablayrolles et al., 2019]
- DINOv2 的创新在于：**整合 + 大规模验证 + 分离 head 这一反直觉发现**

### 6.9 如果去掉会怎样

- 去掉 iBOT loss：linear probing 降 1.8%（86.3% vs 88.1%）
- 去掉 KoLeo：降 0.5%
- 去掉 SK centering：训练不稳定（作者暗示无法收敛）

## 七、创新点三：高效训练策略

### 7.1 问题

ViT-g/14 有 11 亿参数，在 1.42 亿张图上训练。如果按标准 Transformer 训练方法，需要数万 GPU 天——这在实际中是不可接受的。

### 7.2 解决方案

**Flash Attention [Dao et al., 2022]**

标准 attention 需要计算 $N \times N$ 的注意力矩阵并写入 HBM。Flash Attention 通过 tiling 和重计算，将 IO 复杂度从 $O(N^2 d)$ 降低到 $O(N^2 d / M)$（M=SRAM 大小）。

实践中，作者报告 **2× 加速**，同时对长序列（322×322 分辨率下 token 更多）的加速更明显。

**Fully Sharded Data Parallel (FSDP)**

把模型参数、梯度、优化器状态分片到 512 个 GPU 上。每个 GPU 只存 1/512 的模型，极大降低了单卡显存需求。

没有 FSDP，11B 参数模型需要至少 80GB 显存才能加载单精度——这在 A100 (40GB) 上无法直接训练。

**其他优化**
- 混合精度（bfloat16）：模型精度足够，比 float32 省一半显存
- 梯度累积：模拟大 batch（实际 batch size 2048）
- AdamW 优化器，cosine 学习率调度

### 7.3 效果

- **总训练成本**：约 5000 GPU 天（512×A100 训练约 10 天）
- **比之前同类方法快 2×、省 3× 显存**
- **首次**使大规模自监督 ViT 训练在学术预算内可行

### 7.4 如果去掉会怎样

不使用 Flash Attention，训练时间翻倍（10 天 → 20+ 天）。不使用 FSDP，11B 参数的模型无法在标准 GPU 集群上训练。

## 八、知识蒸馏：大模型教小模型

DINOv2 还做了另一项重要工作——**训练大模型（ViT-g/14），然后蒸馏成小模型（ViT-S/B/L/14）**。

### 8.1 动机

虽然 ViT-g/14 性能最强，但 11 亿参数对于很多实际部署场景来说太大了。理想情况是：大模型负责"学到知识"，小模型负责"部署使用"。

### 8.2 蒸馏方法

简单有效：使用训练好的 ViT-g/14 作为固定 teacher，用 DINOv2 的蒸馏框架训练小模型。不同的是，现在的 teacher 是固定的、高质量的，而不是 EMA 动态更新的。

### 8.3 效果

| 模型 | #Params | Linear Probing | k-NN |
|------|---------|---------------|------|
| DINOv2 ViT-S/14 | 21M | 79.0% | 70.9% |
| DINOv2 ViT-B/14 | 86M | 82.1% | 75.1% |
| DINOv2 ViT-L/14 | 304M | 83.5% | 78.0% |
| DINOv2 ViT-g/14 | 1.1B | **86.5%** | **78.4%** |

ViT-B/14 以 1/13 的参数就达到了 ViT-g/14 的 95% 性能，蒸馏效率很高。

## 九、公式详解

### 公式 1：DINO 损失

$$\mathcal{L}_{DINO} = - \sum_{k=1}^{K} p_t^{(k)} \log p_s^{(k)}$$

- $K=65536$：输出维度
- $p_t$：teacher 的 [CLS] token 输出，经过 softmax + Sinkhorn-Knopp 居中
- $p_s$：student 的 [CLS] token 输出，经过 softmax

**直觉**：让 student 模仿 teacher 的预测分布。由于 teacher 是 student 的 EMA，这是自蒸馏——student 学习追赶自己的过去版本。

**温度设定**：teacher 温度 $\tau_t=0.04$（输出尖锐，近 one-hot），student 温度 $\tau_s=0.1$（输出平滑）。这种不对称为蒸馏提供了学习信号。

### 公式 2：iBOT 损失

$$\mathcal{L}_{iBOT} = - \frac{1}{M} \sum_{i=1}^{M} \sum_{k=1}^{K} q_{ti}^{(k)} \log q_{si}^{(k)}$$

- $M$：被遮蔽的 patch 数量（30% × 256 ≈ 77）
- $q_{ti}, q_{si}$：第 $i$ 个 patch 的 teacher/student 输出

**直觉**：student 只能看到部分 patch（70% 可见），必须从可见 patch 推断被遮蔽 patch 的语义。这迫使模型理解局部-全局对应关系。

**与 DINO 损失的关系**：
- DINO 损失：不同视图间匹配 [CLS] token → 全局-全局
- iBOT 损失：可见 patch 推断遮蔽 patch → 局部-全局
- 两者互补

### 公式 3：KoLeo 正则化

$$\mathcal{L}_{KoLeo} = - \frac{1}{B} \sum_{i=1}^{B} \log \left( \min_{j \neq i} \|z_i - z_j\|_2 \right)$$

- $B=2048$：batch size
- $z_i$：L2 归一化后的特征向量

**直觉理解**：对于 batch 中的每张图 $i$，找到特征空间中离它最近的 $j$（NN），最大化这个最小距离。这相当于"不要让你的邻居太近"。

**数值例子**：
假设 batch 有 4 张图，特征为 2 维单位向量。如果所有图坍缩到 $(0, 1)$，所有 $d_{n,i} \approx 0$，损失接近 $\infty$。如果均匀分布在单位圆上，所有 $d_{n,i} \approx 1$，损失 $\approx 0$。

**与对比学习的关系**：KoLeo 等价于一个 batch 内的隐式对比——没有明确的负样本，但通过"推开最近邻"起到了类似作用。

### 公式 4：Sinkhorn-Knopp 居中

给定 batch teacher 输出矩阵 $P \in \mathbb{R}^{B \times K}$，SK 算法迭代：

1. 行归一化：$P_{ij} \leftarrow P_{ij} / \sum_j P_{ij}$
2. 列归一化：$P_{ij} \leftarrow P_{ij} / \sum_i P_{ij}$
3. 重复步骤 1-2，共 3 次

**效果**：强制所有实例的预测分布接近均匀，防止 teacher 输出坍缩到少数维度。

**为什么比动量 centering 好**：SK 算法在 batch 级别做精确的概率匹配，动量 centering 只是减去一个偏置。在 batch 足够大时，SK 提供更强的正则化。

### 公式 5：总损失

$$\mathcal{L} = \mathcal{L}_{DINO} + \lambda_1 \mathcal{L}_{iBOT} + \lambda_2 \mathcal{L}_{KoLeo}$$

其中 $\lambda_1 = 0.5, \lambda_2 = 0.1$（经验值）。

### 前驱工作

- **[Goyal et al., 2021] SEER**：首次尝试大规模自监督学习，但受限于未筛选数据。

### 同期竞品

- **[OpenCLIP] [Ilharco et al., 2021]**：CLIP 的开源复现，在 LAION-2B 上训练。DINOv2 的主要竞争对手——弱监督方法的最强开源模型。
- **[EVA-CLIP] [Fang et al., 2023]**：基于 EVA 的 CLIP 改进版。DINOv2 ViT-g/14 在其指标上实现了约 +0.1% 的超越。

- **[BEiT v2] [Bao et al., 2022]**：使用 VQ-VAE 编码作为预测目标。DINOv2 使用教师网络的连续特征。
- **[MSN] [Assran et al., 2023]**：Masked Siamese Networks，使用类似自蒸馏的框架。

**核心差异**：DINOv2 是第一篇证明**纯自监督、无文本**方法可以匹配 CLIP 类方法大规模性能的论文。

3. **范式影响**：

   - 推动了"自监督预训练 + 零样本评估"成为新的评估标准。
   - 使 frozen feature 评估（linear probing、k-NN）在社区中广泛采纳。

## 十三、局限

### 局限一：训练成本极高

ViT-g/14 需要 5000 GPU 天（512×A100 × 10 天）。即使蒸馏后的 ViT-S/14 也需要约 100 GPU 天。这远超学术实验室的预算，基本上只有大公司能做。

**论文态度**：承认（在 Appendix D 详细讨论了开销），但开源模型权重让社区可以免训练使用。

**后续缓解**：EVA-02 等改进了训练效率。

### 局限二：数据构建依赖先验模型

LVD-142M 的构建依赖一个预训练的 ViT-H/16 模型来做 embedding 检索。虽然这个模型本身也是自监督的（在 ImageNet-22K 上训练），但**数据 pipeline 对初始模型的选择敏感**。

**论文态度**：未深入讨论。但实践中，即使更换初始化模型，最终数据集质量可能类似。

### 局限三：数据隐私问题

LVD-142M 数据集包含互联网爬取的图片，可能包含个人信息、受版权保护的图片等。论文没有公开 LVD-142M 数据集，只发布了模型权重。

**论文态度**：未讨论数据隐私。

### 局限四：缺少理论解释

DINOv2 的成功主要来自工程优化（更好的数据、更稳定的训练），而非理论突破。为什么自蒸馏在大数据上有效？为什么数据质量比数量更重要？这些问题的理论分析还没有。

**论文态度**：未讨论。

### 局限五：蒸馏后的小模型仍有 gap

虽然蒸馏效果不错，但 ViT-S/14 与 ViT-g/14 之间的性能差距还很大（79.0% vs 86.5%）。小模型的效率-精度 tradeoff 还有改进空间。

```python
class DinoVisionTransformer(nn.Module):

        ])
        self.norm = nn.LayerNorm(embed_dim)
```

关键点：
- `PatchEmbed` 将 14×14 patch 展平并线性投影到 embed_dim
- `257 = 256 patches + 1 cls token`
- 40 层 Transformer block，每层包含 MSA + MLP + residual + LayerNorm

### 14.2 训练框架

代码中的训练循环体现了 DINOv2 的核心设计：

```python
# 伪代码
for images in dataloader:
    # 生成增强视图
    global_views, local_views = multi_crop(images)
    
    # Teacher forward（只对全局视图）
    with torch.no_grad():
        teacher_global = teacher(global_views)
        teacher_cls = teacher_norm(teacher_global[:, 0])  # [CLS]
        teacher_patches = teacher_patch_output  # patch tokens
    
    # Student forward（所有视图）
    masked_views = mask_patches(global_views)  # 遮蔽 30%
    student_masked = student(masked_views)
    student_cls = student_norm(student_masked[:, 0])
    student_patches = student_patch_output
    
    # 损失计算
    dino_loss = cross_entropy(teacher_probs, student_probs)
    ibot_loss = cross_entropy(teacher_patch_probs, student_patch_probs)
    koleo_loss = compute_koleo(student_features)
    
    loss = dino_loss + 0.5 * ibot_loss + 0.1 * koleo_loss
    
    # 反向传播
    loss.backward()
    optimizer.step()
    
    # EMA 更新 teacher
    with torch.no_grad():
        for t_param, s_param in zip(teacher.parameters(), student.parameters()):
            t_param.data = momentum * t_param.data + (1 - momentum) * s_param.data
```

## 十五、小结

DINOv2 的核心贡献可以用一句话概括：**当自监督视觉方法遇到足够大、足够干净的数据时，可以超越弱监督方法（CLIP），且不需要任何标注信号。**

具体来说：

1. **数据工程是最大的贡献** ⟶ LVD-142M 证明了"数据质量 > 数据量"，自动 pipeline 让 1.42 亿张高质量图片自监督筛选成为可能。
2. **方法整合 + 规模化验证** ⟶ DINO + iBOT + SK centering + KoLeo 的组合，以及"分离 head"这一反直觉发现，在大规模训练中被验证有效。
3. **高效训练使大模型可行** ⟶ Flash Attention + FSDP 等技术使 11B 参数模型能在 512 GPU 上 10 天训练完毕。
4. **蒸馏链有效** ⟶ ViT-g/14 → ViT-S/B/L 的蒸馏链让小模型也能受益。

**更广泛的启示**：
- 视觉自监督领域正在重复 NLP 的路径：小数据上开发方法 → 大数据上验证方法的可扩展性 → 数据质量成为核心瓶颈。
- DINOv2 证明了"纯视觉、无文本"的基础模型是可行的，挑战了"必须有 CLIP 式图文数据才能做出好特征"的普遍认知。
- DINOv2 的特征被 SAM、EVA-02 等后续工作广泛采用，推动了视觉基础模型领域的发展。
