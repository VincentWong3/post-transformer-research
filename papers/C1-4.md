# DAB-DETR: Dynamic Anchor Boxes are Better Queries for DETR（2022）

**论文：** [arXiv:2201.12329](https://arxiv.org/abs/2201.12329) · Shilong Liu, Feng Li, Hao Zhang, Xiao Yang, Xianbiao Qi, Hang Su, Jun Zhu, Lei Zhang · Tsinghua University & IDEA · 2022

**技术分支：** C1 — Detection Transformer 化 / C1-4 — set prediction detection 范式

**代码：** [SlongLiu/DAB-DETR](https://github.com/SlongLiu/DAB-DETR)

## 一、先搞清楚"问题在哪里"

2020 年，DETR（Carion et al., 2020）第一次用 Transformer 做端到端目标检测，不需要 anchor 生成、NMS 后处理、proposal 这些手工设计的组件。它把检测建模成 set prediction 问题——100 个 learnable query 通过 cross-attention 从图像中"探取"特征，输出一个无序的物体集合。

这听起来很优雅。但 DETR 有一个致命问题：**训练收敛太慢**。标准 DETR 需要 500 个 epoch 才能在 COCO 上达到 42.0 AP。同期 Faster R-CNN 在 108 个 epoch 就能做到 42.0 AP。DETR 慢了将近 5 倍。

社区很快意识到问题出在 decoder 的 query 设计上。Conditional DETR（Meng et al., 2021）证明，将 learnable query 分解为 content 和 spatial 两部分，再与图像 positional embedding 对齐，可以显著加速收敛。Anchor DETR（Wang et al., 2021）提出直接用 2D anchor points 作为 query。Deformable DETR（Zhu et al., 2021）用 2D reference points + deformable attention 实现了多尺度特征的高效交互。

但这些工作共享一个盲区：**它们都只用了 2D 位置信息（anchor point），忽略了物体的尺度信息（宽和高）**。DAB-DETR 的工作者问了一个更根本的问题：DETR 的 query 到底是什么？能不能不只是查"在哪查"，还知道"查多大的区域"？

## 二、旧路线的真正问题

### 2.1 DETR 的 query 学的是什么？

DETR 的 decoder 中有 100 个 learnable query，每个 query 是一个高维 embedding。论文的学者做了一个关键分析：在 cross-attention 中，query 由两部分组成——content part（decoder embedding）和 positional part（learnable query）。Key 同样由 content part（编码器输出的图像特征）和 positional part（sinusoidal positional embedding）组成。

所以 cross-attention 的权重实际上由两个相似度相乘得到：内容相似度（query content 与 key content 的点积）+ 位置相似度（query position 与 key position 的点积）。

问题在于：**learnable query 的位置信息没有用和图像 position embedding 相同的方式编码**。图像特征的位置编码是 sinusoidal 函数（基于坐标的连续编码），而 learnable query 是随机初始化后硬学出来的离散向量。两者的"语言"不同。

### 2.2 "学不好"还是"看不懂"？

论文做了两个实验来定位瓶颈。

**实验一：固定 query，只训练其他模块。** 如果 query 本身很难学（优化困难），那就用训练好的 DETR query 固定住，只训练 decoder 的其他部分。结果 Figure 3(a) 显示：固定 query 只在非常早期的 epoch（前 25 个 epoch）有轻微改善，之后和原版 DETR 几乎没有区别。这说明"查询学习困难"不是主要原因。

**实验二：可视化 positional attention map。** 论文把 learnable query 和图像 positional encoding 之间的 attention 图可视化，结果 Figure 4(a) 揭示了关键问题：**DETR 的 positional attention 有多峰性（multiple modes）和注意力分散问题**。

具体来说：
- 有些 query 的 attention 有 2 个或更多的聚集中心——同一张图上如果多个物体存在，query 不知道自己应该"盯"哪个
- 有些 query 的 attention 覆盖区域要么太大要么太小——无法注入有效的空间约束
- 没有一个 query 能根据目标的尺度自适应地调整 attention 的"形状"

论文推断：**DETR 的 slow convergence 根因是 query 的多峰性质**。这不是优化困难，而是 query 的位置信号本质上就"说不清自己在哪"。

### 2.3 为什么 2D 不够？

Conditional DETR 用 sinusoidal 编码将 query 的位置信息对齐到图像 positional embedding，解决了"语言不通"的问题，得到了类似高斯核的单峰 attention（Figure 4(b)）。但是它的高斯先验是 isotropic（各向同性）且固定大小的——也就是假设所有物体都是同样大小的圆。

这个假设显然不合理。一只猫（中等大小，椭圆形）和一辆火车（很大，横向很宽）需要的 attention 区域完全不同。2D anchor point（x, y）只能告诉 decoder "在哪里查"，不能告诉它"查多大的区域"。

所以问题归结为：**需要在 query 中引入尺度信息（w, h），让 cross-attention 的位置先验自适应地匹配物体的形状和大小。**

## 三、这篇论文的核心思路

**核心思路是：把 DETR 的 query 从 learned embedding 改成 dynamic anchor box（x, y, w, h），让 decoder 的每一层都知道"在哪儿查"和"查多大的区域"，并且逐层迭代优化这个猜测。**

具体由三个机制组成：

```text
anchor box 作为 query       →  4D (x,y,w,h) 明确的空间先验
       ↓
anchor 逐层动态更新          →  每层预测偏移量，迭代逼近 ground truth
       ↓
宽高调制 attention + 温度调参 →  attention 自适应目标尺度，更精准的 soft ROI pooling
```

这条技术路线的谱系位置是：

```text
Faster R-CNN（proposal + 手工 anchor, 2015）
   ↓
DETR（learned query, 500 epochs 收敛, 2020）
   ↓
Conditional DETR（解耦 content/spatial query, 2021）
   ↓
Anchor DETR（2D anchor point, 忽略尺度, 2021）
   ↓
DAB-DETR（4D anchor box + 尺度调制 attention, 2022） ← 本篇
   ↓
DN-DETR（去噪训练, 2022）
   ↓
DINO（DAB-DETR + DN-DETR, 2022）
```

DAB-DETR 处于一个承上启下的位置。它上面接收了 Conditional DETR 和 Anchor DETR 的 query 位置化思想，下面直接启发了 DINO——DINO 的核心公式就是"DB-DETR 的 anchor box query + DN-DETR 的去噪训练"。

## 四、整体结构与前向流程

### 4.1 整体架构

DAB-DETR 的整体架构遵循 DETR 的设计：CNN backbone + Transformer encoder + Transformer decoder + prediction heads。主要改进在 decoder 部分。

```text
输入图像: H×W×3
    │
    ▼
[CNN Backbone]  (ResNet-50/101)
    │            输出: 空间特征图
    ▼
[Transformer Encoder]  (6 层)
    │            用 self-attention 精炼特征
    ▼
┌────────────────────────────────────────────────────┐
│ Transformer Decoder  (6 层)                        │
│                                                    │
│  每层的输入:                                        │
│     content query = decoder embedding              │
│     positional query = anchor box 的 encoding      │
│                                                    │
│  每层的操作:                                        │
│     1. self-attention:  (C+P) ↔ (C+P) → C        │
│     2. cross-attention: (C, P) ↔ (F, Pos) → C'  │
│         - 宽高调制 positional attention            │
│     3. prediction head: C' → (Δx, Δy, Δw, Δh)    │
│     4. anchor update: A_new = A_old + (Δ...)     │
│                                                    │
│  输出传递给下一层                                    │
└────────────────────┬───────────────────────────────┘
                     │
                     ▼
[Prediction Heads]   最后一层输出 → box + label
    │
    ▼
[Bipartite Matching + Loss]  (匈牙利匹配 + L1 + GIoU + focal loss)
```

### 4.2 Anchor 的编码与使用

给定第 q 个 anchor 框 $A_q = (x_q, y_q, w_q, h_q)$ ，论文将其转换为 positional query：

$$P_q = \text{MLP}(\text{PE}(A_q))$$

其中 PE 是 sinusoidal positional encoding（将浮点数映射到 $D/2$ 维向量）。由于 $A_q$ 是 4 维，将 4 个分量的编码拼接起来得到 $2D$ 维向量，再通过 MLP（两个 Linear+ReLU，第一次降维）投影到 $D$ 维。

### 4.3 Self-Attention 和 Cross-Attention 中的 Query 设计

**Self-attention（公式 3）：**

$$Q_q = C_q + P_q,\quad K_q = C_q + P_q,\quad V_q = C_q$$

query、key 都包含 content + position。注意这里的 position 用的是完整 anchor 的编码 $(x, y, w, h)$ ，所以 self-attention 不仅知道框在哪，还知道框的大小——这帮助 query 之间根据空间位置和大小进行交互。

**Cross-attention（公式 4）：**

借鉴 Conditional DETR 的设计，将 content 和 position 拼接起来（而不是相加），然后用 content 信息通过一个 MLP 学习缩放向量，对 positional embedding 做 element-wise 缩放：

$$Q_q = \text{Cat}(C_q, \text{PE}(x_q, y_q) \cdot \text{MLP}^{(\text{csq})}(C_q))$$

$$K_{x,y} = \text{Cat}(F_{x,y}, \text{PE}(x, y))$$

注意 cross-attention 中只用到了 $(x, y)$ 作为位置查询——尺度信息通过宽高调制机制（第 4.5 节）来影响 attention 权重，而不是直接加在 positional embedding 中。

### 4.4 Anchor 逐层更新

因为 anchor 直接就是可解释的坐标值，所以可以像 Deformable DETR 那样逐层更新。每层 decoder 后接一个 prediction head（所有层共享参数），输出偏移量 $(\Delta x, \Delta y, \Delta w, \Delta h)$ ，加到当前 anchor 上得到下一层的输入。

这在之前的方法中做不到。Conditional DETR 和原版 DETR 的 query 是高维 embedding——就算预测出了 box 偏移，也没法把它转回高维 query 空间。DAB-DETR 直接操作坐标，就绕过了这个"转换困难"。

消融实验（Table 3）显示：去掉 anchor update 后 AP 从 45.7 降到 44.0——下降了 1.7 个点，是单一贡献最大的组件。

### 4.5 宽高调制的高斯核

这是 DAB-DETR 最精巧的设计。

标准 positional attention 计算的是位置编码的点积：

$$\text{Attn}((x,y), (x_{\text{ref}}, y_{\text{ref}})) = (\text{PE}(x) \cdot \text{PE}(x_{\text{ref}}) + \text{PE}(y) \cdot \text{PE}(y_{\text{ref}})) / \sqrt{D}$$

这相当于一个各向同性的高斯先验——对所有物体采样同样的 attention scope。

DAB-DETR 的修改是：**用 anchor 的宽高调制 attention 的 x 部分和 y 部分**（公式 6）：

$$\text{ModulateAttn}((x,y), (x_{\text{ref}}, y_{\text{ref}})) = (\text{PE}(x) \cdot \text{PE}(x_{\text{ref}}) \frac{w_{q,\text{ref}}}{w_q} + \text{PE}(y) \cdot \text{PE}(y_{\text{ref}}) \frac{h_{q,\text{ref}}}{h_q}) / \sqrt{D}$$

其中 $w_q, h_q$ 是当前 anchor 的宽高， $w_{q,\text{ref}}, h_{q,\text{ref}}$ 是通过 content query 学出来的参考宽高： $w_{q,\text{ref}}, h_{q,\text{ref}} = \sigma(\text{MLP}(C_q))$ 。

直觉上：如果 anchor 很宽（ $w_q$ 大）， $x$ 部分的 attention 就会除以一个大的宽值，使得 attention 在水平方向上"摊开"——覆盖更宽的区域。反之如果 anchor 很窄，水平 attention 会收窄。 $y$ 部分同理。

这就把高斯先验从圆变成椭圆——**attention 的形状和方向可以自适应目标形状**。Figure 6(a) 的可视化展示了这种调制效果：宽物体的 attention 在水平方向拉伸，高物体垂直方向拉伸。

消融实验：去掉宽高调制后 AP 从 45.7 降到 45.0。

### 4.6 Temperature Tuning

论文还发现 sinusoidal positional encoding 中的一个隐藏参数——温度 $T$ （公式 8）：

$$\text{PE}(x)_{2i} = \sin(\frac{x}{T^{2i/D}}), \quad \text{PE}(x)_{2i+1} = \cos(\frac{x}{T^{2i/D}})$$

原始 Transformer（Vaswani et al., 2017）将 $T$ 设为 10000，针对 NLP 中整数位置。但 DETR 中坐标 $x$ 是 0 到 1 的浮点数。更大的 $T$ 产生更平坦的 attention 图，更小的 $T$ 产生更尖锐的图。

论文通过经验选择 $T=20$ 用于所有模型。Appendix E 的 Table 6 展示了不同温度的效果： $T=20$ 时 AP 最高（使用 ResNet-50，batch size 64，无 pattern embeddings，无调制 attention 时达到 40.1 AP）。 $T=2$ 对小物体更好（APS=19.3）， $T=10000$ 对大物体更好（APL=58.9），但整体 AP 不如 $T=20$ 。

消融实验：去掉 temperature tuning（使用默认 T=10000）后 AP 从 45.7 降到 44.4。

## 五、核心公式与数学直觉

### 5.1 公式 1：从 anchor 到 positional query

$$P_q = \text{MLP}(\text{PE}(A_q))$$

这是整个架构的起点。4D anchor box → sinusoidal encoding（4×D/2=2D）→ MLP（2D→D）→ positional query。关键洞察：因为 anchor 是实数坐标，可以直接用 sinusoidal 编码——这与图像特征的 positional encoding 使用相同的"语言"。

### 5.2 公式 6：宽高调制

$$\text{ModulateAttn} = (\text{PE}(x)\cdot\text{PE}(x_{\text{ref}})\frac{w_{q,\text{ref}}}{w_q} + \text{PE}(y)\cdot\text{PE}(y_{\text{ref}})\frac{h_{q,\text{ref}}}{h_q})/\sqrt{D}$$

这个公式有一个微妙之处： $w_{q,\text{ref}}$ 和 $h_{q,\text{ref}}$ 不是直接等于 anchor 的宽高，而是通过 content query 学出来的。这样做的原因是：**content query 中包含了从图像特征中提取的语义信息，可以根据实际物体内容决定 attention 应该扩散到多大**，而不是机械地等于 anchor 大小。

举个例子：如果一个 anchor 框得比较大（ $w_q$ 大），但里面其实是一个很小的物体（content query 学出来的 $w_{q,\text{ref}}$ 小），那么 $\frac{w_{q,\text{ref}}}{w_q}$ 就小，attention 就会被"压缩"——相当于用语义信息修正了 anchor 的不准确。

### 5.3 为什么 anchor box 优于 anchor point

Table 3 的消融实验直接回答了这个问题：Row 4（anchor point，2D）AP=44.0 vs Row 3（anchor box，4D）AP=45.0——引入宽高信息带来了 1.0 AP 的提升。

数学上解释：2D anchor point 只能提供各向同性的位置先验。4D anchor box 提供了各向异性的位置先验——x 和 y 方向可以独立调整 attention 的扩散范围。对于目标检测这种物体形状差异极大的任务（一只鞋 vs 一辆车），各向异性的先验显然更合理。

### 7.2 前驱工作：DAB-DETR 继承了谁？

**DETR（Carion et al., 2020）：** 整体框架（CNN backbone + Transformer encoder + decoder + 匈牙利匹配）完全继承。DAB-DETR 本质上是对 DETR decoder query 的重新设计。

**Conditional DETR（Meng et al., 2021）：** 贡献了 cross-attention 中 content-position 解耦的设计（公式 4 中的条件空间查询，即 conditional spatial query）。DAB-DETR 沿用了这个设计，并且指出两者互补——DAB-DETR 的解码器可以与 Conditional DETR 的解码器结合（论文虽未实验，但明确提及）。

**Anchor DETR（Wang et al., 2021）：** 首次提出用 anchor point 作为 query 并通过 pattern embeddings 实现多点预测。DAB-DETR 扩展了 2D → 4D，并引入了尺度调制 attention。

**Deformable DETR（Zhu et al., 2021）：** 迭代框精炼（iterative bounding box refinement）的先驱——虽然其 4D anchor 更新与 deformable attention 紧密耦合，不能通用。DAB-DETR 将其解耦为通用的 anchor box query 设计。

### 7.3 同期竞品

DAB-DETR（2022.01 arXiv）发表时，DETR 家族已经很拥挤了。它与其他方案的核心区别见 Table 1：

| 维度 | DETR | Cond DETR | Anchor DETR | Deform DETR | SMCA | DAB-DETR |
|------|------|-----------|-------------|-------------|------|----------|
| 直接学 anchor? | No | No | 2D point | No | No | **4D box** |
| Reference anchor | — | 2D | 2D | 4D | 4D | **4D** |
| 动态 anchor 更新? | — | — | ✓ | ✓ | — | **✓** |
| 标准 dense attention? | ✓ | ✓ | ✓ | — | ✓ | **✓** |
| 尺度调制 attention? | — | — | — | — | ✓ | **✓** |
| 更新学习到的 spatial query? | — | — | — | — | ✓ | **✓** |

DAB-DETR 是唯一一个同时满足所有 6 个条件的模型。

### 7.4 后续影响：DAB-DETR 直接启发了谁？

**DAB-Deformable-DETR（同篇论文）：** 在同一篇论文中，DAB-DETR 的作者将 dynamic anchor box 移植到 Deformable DETR，用 10 行代码提升了 0.5 AP。这证明 anchor box query 不是标准 cross-attention 的专属设计，可以无缝接入 deformable attention。

**DN-DETR（Li et al., 2022）：** 从训练效率角度加速 DETR，引入去噪训练（query denoising）——给 ground truth box 加噪声作为 decoder 的辅助 query，让 decoder 学习去噪。这条路线与 DAB-DETR 的 anchor box query 路线正交。

**DINO（Zhang et al., 2022）：** 这是 DETR 系列的里程碑——融合 DAB-DETR 的 dynamic anchor box 和 DN-DETR 的去噪训练，在 COCO 上达到 50+ AP。DINO 的 decoder query 设计直接继承了 DAB-DETR 的 anchor box 方案，并加上了 contrastive denoising training（对比去噪训练）。可以说，**DAB-DETR 的 anchor box query 是 DINO 的 decoder 基石。**

**Group DETR（Chen et al., 2022）：** 将 300 个 query 分为多个 group，每组独立做匹配，提升 recall。这个工作也建立在 DAB-DETR 的 anchor box query 之上。

### 7.5 与 Conditional DETR 的关系

论文指出 DAB-DETR 与 Conditional DETR 是互补的。Conditional DETR 的主要贡献是 cross-attention 中的 content-position 解耦（conditional spatial query），DAB-DETR 直接沿用了这个设计。两者的区别是：

- Conditional DETR 用 content embedding 通过 MLP 生成一个"条件化的" spatial query，但它没有显式的 anchor 表示，也没有逐层更新
- DAB-DETR 引入显式的 anchor box，逐层更新，并用宽高调制 attention

论文还提到 DAB-DETR 可以与 Conditional DETR 结合——但可惜没有给出具体的实验结果（这留给了后续工作）。

### 7.6 与后续 DINO 的关系

这是 DAB-DETR 最重要的谱系位置。DINO 的论文标题明确写了 "DETR with Improved DeNoising Anchor Boxes"，其核心是：

1. **Anchor box query**（来自 DAB-DETR）
2. **Denoising training**（来自 DN-DETR）
3. **Contrastive denoising**（新的）
4. **Mixed query selection**（新的，从 encoder 选择 top-K 特征初始化 anchor）

所以 DAB-DETR 直接贡献了 DINO 的 decoder query 设计——DINO 如果不基于 anchor box query，它的混合 query selection 和去噪训练都无法自然落地。

## 八、范式意义

**DAB-DETR 的范式意义：它把 DETR 的 query 从"黑盒 embedding"变成了"可解释的坐标"，建立了一套完整的 anchor 化 query 设计范式。**

具体来说，DAB-DETR 完成了三件事：

### 8.1 把 query 变成了可解释、可操作的东西

DETR 的 100 个 learned embedding 是不可解释的——没有人能说出一个维度为 256 的向量在语义上表示什么。DAB-DETR 直接把 query 定义为 (x, y, w, h)，使得：
- 你能看到每个 query "在查什么位置、多大范围"
- 你能逐层追踪 query 如何向 ground truth 收敛
- 你能初始化 query 时控制位置和大小分布

这种"显式化"是 DETR 系列走向成熟的一步。

### 8.2 统一了 detection Transformer 中的 query 设计

DAB-DETR 的 anchor box query 设计比之前的方案更通用。它是唯一一个同时在 6 个维度上满足所有条件的模型（Table 1）：
- 直接学 anchor（不需要 hand-design anchor 生成）
- 使用 4D reference（不只是 2D point）
- 动态更新（每层 refine）
- 使用标准 dense attention（不是 deformable）
- 尺度调制 attention（自适应目标大小）
- 更新 learning query（不锁死第一层的 query）

这个设计空间定义了后续 DETR-like 模型的设计坐标系——后续的 DINO、Group DETR、H-DETR 基本都在这个框架内做增量改进。

### 8.3 为 DINO 铺平了道路

虽然没有 DN-DETR 的去噪训练就没有 DINO，但如果没有 DAB-DETR 的 anchor box query，DN-DETR 的去噪训练也很难自然集成到 DETR 中。因为去噪训练的核心操作是"给 ground truth box 加噪声作为 query"，这就要求 query 本身就是 box 坐标（或者至少可以解释为 box）。

DAB-DETR 实现了这个前提。所以 DAB-DETR + DN-DETR = DINO 不是一个巧合，而是技术路线上的必然交汇。

### 8.4 分类：bridge node

在技术树中，DAB-DETR 是一个 **bridge node（桥梁节点）**。它连接了两条技术路线：
- **query 显式化路线（Conditional DETR → Anchor DETR → DAB-DETR）**
- **训练优化路线（DN-DETR → DINO）**

没有 DAB-DETR，DINO 的技术实现会更复杂。它是"DETR 从 500 epoch 收敛到 faster CNN 级别"这条路上不可或缺的一步。

## 九、局限性与后续方向

### 9.1 计算开销增加

DAB-DETR-R50 的 GFLOPs 为 94（比 Conditional DETR 的 90 略有增加）。DC5 版本由于高分辨率特征图，GFLOPs 达到 216。在高分辨率输入下，标准 dense attention 的计算量仍然是一个问题。这也解释了为什么后续工作（如 DINO）更经常使用多尺度特征 + deformable attention 的组合。

### 9.2 Anchor 初始化的敏感性

论文 Appendix G 的实验显示，随机固定初始 $(x, y)$ 反而比学习它们更好。这说明 anchor 的初始化方式对最终性能有影响，论文没有深入研究如何初始化 anchor 才能达到最优。这是后续 DINO 的 "query selection"（从 encoder 特征中选择初始化）试图解决的问题。

### 9.3 与 Conditional DETR 的互补关系未充分验证

论文声称 DAB-DETR 与 Conditional DETR 互补，但并没有给出结合的实验结果。这个"互补"的假设是合理的（一个改进 position 编码方式，一个改进了 position 的内容），但缺少实验支撑。

### 9.4 没有涉及多尺度特征

DAB-DETR-DC5-R50 达到 45.7 AP，但仅使用了单尺度特征。如果在多尺度特征上加 DAB-DETR 的 anchor box query，性能可能进一步提高。这个方向在 DINO 中得到了实现。

## 十、小结

DAB-DETR 的核心贡献可以归结为一句话：**把 DETR 的 query 从不可解释的 learned embedding 换成了动态更新的 4D anchor box。**

这个改动看起来简单，但它回答了一个基本问题——"DETR 的 query 到底在查什么？" 之前的答案是"不知道，神经网络自己学的"。DAB-DETR 的答案是"在查一个矩形区域，而且每层都在迭代优化这个矩形的位置和大小"。

这种"显式化"的好处是多方面的：
- 训练收敛更快（50 epoch vs 500 epoch）
- 性能更好（+1.7 AP over Conditional DETR with R50）
- 设计更简单（不用手工调 query embedding 的维度或初始化方式）
- 可解释性更强（你可以直接可视化各层 decoder 的 anchor box）

更重要的是，DAB-DETR 建立的 "anchor box as query" 这个设计范式，直接继承给了 DINO——后者代表了 DETR 系列在 COCO 上的巅峰性能。从这个意义上看，DAB-DETR 是 DETR 从"论文概念验证"走向"实用检测器"的关键一跳。
