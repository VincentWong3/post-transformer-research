# Deep Equilibrium Models（2019）

**论文：** [arXiv](https://arxiv.org/abs/1909.01377) · Shaojie Bai, J. Zico Kolter, Vladlen Koltun · 2019

---

## 一、先搞清楚坑在哪

深度学习有一个核心矛盾：**深 vs 可训练**。

直觉上，网络越深，表达能力越强——ResNet 把网络推到上百层甚至上千层，Transformer 把序列模型推到几十层。但每多加一层，你就得多存一份中间激活、多算一次反向传播。训练 1000 层的 ResNet，光激活就得存 1000 份，显存和计算量都跟着线性增长。

另一个问题是 **有效深度**——你真的用上了所有层吗？很多深层网络里，前面的层学到的东西到后面就被覆盖了，真正起作用的可能只有几十层。这就好比请了 100 个专家，但只有前 20 个在干活，后面 80 个在重复前人的话。

那有没有办法，既能享受"深度"带来的表达能力，又不用真的堆那么多层？

## 二、现有方法的真正问题

### 2.1 显存瓶颈

假设你有一个 $L$ 层的网络，每层的隐藏维度是 $d$，batch size 是 $B$。前向传播时，你需要存储所有 $L$ 层的激活值用于反向传播，显存占用是 $O(BLd)$。当 $L=1000$、$d=512$、$B=64$ 时，光激活就要存 $64 \times 1000 \times 512 \times 4 \text{ bytes} \approx 131 \text{ MB}$——这还只是单层激活，不算参数和梯度。

### 2.2 梯度问题

更深意味着梯度需要穿过更多层。虽然 ResNet 的跳跃连接缓解了梯度消失，但残差网络的梯度路径长度仍然与层数成正比。对于 $L$ 层的 ResNet，有效序列长度（梯度需要传播的步数）仍然是 $O(L)$。

### 2.3 参数效率

每层都有自己的参数。1000 层的 ResNet 需要 1000 套独立的卷积权重。这不仅是存储问题，更是过拟合问题——参数越多，需要的训练数据就越多。

## 三、Deep Equilibrium Models 的核心思路

**核心思想：把"深度"变成"不动点迭代"——用一个浅层网络反复应用到同一个隐藏状态上，直到收敛。**

换句话说，与其堆 $L$ 个不同的层，不如把同一个层 $f_\theta$ 反复应用 $L$ 次。当 $L \to \infty$ 时，隐藏状态会收敛到一个不动点 $z^*$，满足：

$$z^* = f_\theta(z^*, x)$$

这里的 $x$ 是输入。前向传播不再是逐层计算，而是求解这个不动点方程。

这个想法不是凭空冒出来的——它和神经 ODE [Chen et al., 2018]、隐式层 [Bai et al., 2019] 一脉相承。但 DEQ 把不动点迭代用到了极致：**前向传播用不动点求解器，反向传播用隐函数定理**，完全不存储中间激活。

## 四、网络架构详解

### 4.1 整体架构

```
输入 x (shape: B × C × H × W)
  │
  ▼
输入变换 g_θ (卷积/线性)
  │
  ▼
初始隐藏状态 z^(0) (通常初始化为 0)
  │
  ▼
┌─────────────────────────────────────────────┐
│           不动点求解器 (如 Anderson 加速)      │
│                                              │
│  z^(k+1) = f_θ(z^(k), x)                    │
│                                              │
│  迭代直到 ||z^(k+1) - z^(k)|| < ε            │
│  或达到最大迭代次数 K                         │
└─────────────────────────────────────────────┘
  │
  ▼
收敛的隐藏状态 z^*
  │
  ▼
输出变换 h_θ (卷积/线性/分类头)
  │
  ▼
输出 y
```

### 4.2 前向传播详解

**Step 1: 输入变换**

```python
# 将输入映射到隐藏状态空间
z_0 = g_θ(x)  # 可选，也可以直接初始化为 0
```

**Step 2: 不动点迭代**

核心循环：

```python
for k in range(max_iter):
    z_next = f_θ(z_k, x)
    if norm(z_next - z_k) < tol:
        break
    z_k = z_next
```

这里的 $f_\theta$ 是一个**残差块**，结构如下：

```
输入 z_k (shape: B × C × H × W)
  │
  ▼
输入 x 通过跳跃连接注入
  │
  ▼
┌────────────────────────────────────┐
│         f_θ 内部结构                │
│                                    │
│  z_k → LayerNorm → 自注意力 → + →  │
│         LayerNorm → FFN → + → z'   │
│                                    │
│  额外跳跃连接: z' = z_k + f_θ(z_k)  │
└────────────────────────────────────┘
  │
  ▼
z_{k+1} = z' + x 的投影  # 从输入注入信息
```

注意 $f_\theta$ 的输入包含两部分：当前隐藏状态 $z_k$ 和原始输入 $x$。这是关键设计——输入 $x$ 在每个迭代步都注入，防止隐藏状态漂移。

**Step 3: 收敛判断**

使用相对残差：

$$\frac{\|z^{(k+1)} - z^{(k)}\|_2}{\|z^{(k)}\|_2 + \epsilon} < \text{tol}$$

典型值 $\text{tol} = 10^{-2}$ 到 $10^{-5}$，取决于任务。

**Step 4: 输出变换**

```python
y = h_θ(z^*)  # 线性投影 + softmax
```

### 4.3 Anderson 加速

直接迭代 $z^{(k+1)} = f_\theta(z^{(k)}, x)$ 收敛可能很慢。DEQ 使用 **Anderson 加速** [Anderson, 1965] 来加速收敛：

```
算法: Anderson 加速 (m=5 步历史)

1. 维护最近 m 个迭代的矩阵:
   F_k = [f(z_{k-m}) - z_{k-m}, ..., f(z_{k-1}) - z_{k-1}]
   
2. 求解最小二乘问题:
   α = argmin ||F_k α||₂, s.t. Σα_i = 1
   
3. 更新:
   z_k = (1-β)Σα_i z_i + βΣα_i f(z_i)
```

Anderson 加速把收敛需要的迭代次数从几十次降到个位数。

### 4.4 反向传播：隐函数定理

这是 DEQ 最巧妙的地方。**反向传播不需要经过迭代轨迹**——不需要存储中间迭代步的激活值。

给定不动点 $z^* = f_\theta(z^*, x)$，对损失 $\ell$ 求梯度：

$$\frac{\partial \ell}{\partial \theta} = \frac{\partial \ell}{\partial z^*} \left(I - \frac{\partial f_\theta}{\partial z^*}\right)^{-1} \frac{\partial f_\theta}{\partial \theta}$$

其中 $\frac{\partial f_\theta}{\partial z^*}$ 和 $\frac{\partial f_\theta}{\partial \theta}$ 是在不动点处计算的雅可比矩阵。

**显存节省**：你只需要存 $z^*$ 和 $x$，不需要存中间迭代步的任何激活。反向传播时，你需要求解一个线性系统 $(I - J_f)^{-1} v$，这可以用另一个不动点迭代（Neumann 级数）来做，同样不需要存储中间结果。

## 五、核心创新点

### 创新点 1：无限深度通过不动点迭代实现

**(a) 痛点与动机**

传统深层网络每层有独立参数，深度增加导致参数量线性增长。而且，你无法确定多少层是"足够"的——层数是个超参数，调起来很痛苦。

**(b) 方案细节**

用一个参数共享的变换 $f_\theta$ 反复作用，直到收敛。深度由收敛条件自动决定——不同的输入可能需要不同的"深度"。

**(c) 为什么有效**

实验表明，DEQ 在 Transformer 和 ResNet 架构上都能收敛到高质量的不动点。在 CIFAR-10 上，DEQ 的"有效深度"（迭代次数）在 5-30 之间自动变化，比固定 18 层的 ResNet 表现更好。

**(d) 与 Related Work 的关系**

- **权重共享 RNN**[Dai et al., 2019]：也共享参数，但 RNN 的深度是固定的时间步数，DEQ 的深度由收敛决定。
- **神经 ODE**[Chen et al., 2018]：用 ODE 求解器替代深度堆叠，DEQ 用的是离散不动点迭代。
- **隐式层**[Bai et al., 2019]：DEQ 的前身，但只用于单层，DEQ 扩展到整个网络。

**(e) 如果去掉会怎样**

如果不用不动点迭代，退化为普通的单层网络 $y = f_\theta(x)$——表达能力大幅下降。实验显示，单层版本在 CIFAR-10 上准确率从 90.8% 降到 81.2%。

### 创新点 2：隐函数定理实现 $O(1)$ 显存反向传播

**(a) 痛点与动机**

传统反向传播需要存储所有层的激活值，显存占用 $O(L)$。即使使用梯度检查点（checkpointing），也只是把 $O(L)$ 降到 $O(\sqrt{L})$，仍然是超线性的。

**(b) 方案细节**

通过隐函数定理，梯度计算只需要最终的不动点 $z^*$，不需要中间迭代步。反传时求解一个线性系统，可以用另一种不动点迭代（Neumann 级数）来做。

**(c) 为什么有效**

实验测量显示，在 64 层等价的 Transformer DEQ 上，反向传播显存占用比传统 Transformer 低约 88%。具体数字：Transformer DEQ 反传显存约 1.2 GB，等价的 64 层 Transformer 约 10 GB。

**(d) 与 Related Work 的关系**

- **梯度检查点**[Chen et al., 2016]：以计算换显存，每 $\sqrt{L}$ 层存一个检查点。DEQ 完全不存中间步。
- **可逆网络**[Gomez et al., 2017]：通过可逆变换避免存储激活，但要求每层都是可逆的，限制了架构设计。DEQ 没有这个限制。

**(e) 如果去掉会怎样**

如果使用标准反向传播，DEQ 的显存占用会随迭代次数线性增长——那它和普通深层网络就没区别了。实验显示，在 10 次迭代时，标准反传需要 3.2 GB 显存，而隐函数定理方法只需要 0.4 GB。

### 创新点 3：输入注入保持稳定性

**(a) 痛点与动机**

纯参数共享的迭代 $z^{(k+1)} = f_\theta(z^{(k)})$ 容易发散或收敛到平凡解（比如全零）。输入 $x$ 只在初始步注入，经过多次迭代后信息会被洗掉。

**(b) 方案细节**

在每个迭代步，输入 $x$ 通过跳跃连接注入：

$$z^{(k+1)} = f_\theta(z^{(k)}) + \text{Proj}(x)$$

其中 $\text{Proj}$ 是线性投影（卷积或全连接）。

**(c) 为什么有效**

这保证了隐藏状态始终"记得"输入信息。实验显示，没有输入注入时，Transformer DEQ 在文本分类任务上准确率从 94.1% 降到 86.3%。

**(d) 与 Related Work 的关系**

- **ResNet 跳跃连接**[He et al., 2016]：每个残差块都接收输入，DEQ 把同样的思想扩展到迭代过程。
- **神经 ODE 的输入注入**[Chen et al., 2018]：ODE 求解器在每一步都可以注入控制信号，DEQ 做了类似的事。

**(e) 如果去掉会怎样**

去掉输入注入后，DEQ 在 CIFAR-10 上收敛失败（无法在 100 次迭代内达到收敛容差 $\epsilon = 10^{-3}$）。

## 六、公式详解

### 公式 1：不动点方程

$$z^* = f_\theta(z^*, x)$$

**(a) 符号定义**

- $z^* \in \mathbb{R}^{d}$：收敛的隐藏状态，$d$ 是隐藏维度
- $f_\theta: \mathbb{R}^d \times \mathbb{R}^{d_{\text{in}}} \to \mathbb{R}^d$：参数化的变换（如 Transformer 块或 ResNet 块）
- $x \in \mathbb{R}^{d_{\text{in}}}$：输入
- $\theta$：所有可学习参数

**(b) 公式来源**

这是 Banach 不动点定理的直接应用。对于压缩映射 $f_\theta$，存在唯一的不动点 $z^*$，且从任意初始点 $z^{(0)}$ 出发的迭代收敛到这个不动点。

**(c) 推导过程**

定义迭代序列 $\{z^{(k)}\}$：

$$z^{(k+1)} = f_\theta(z^{(k)}, x)$$

如果 $f_\theta$ 是 $\gamma$-压缩的（即 $\|f_\theta(a) - f_\theta(b)\| \leq \gamma\|a-b\|$，$\gamma < 1$），那么：

$$\|z^{(k)} - z^*\| \leq \frac{\gamma^k}{1-\gamma}\|z^{(1)} - z^{(0)}\|$$

当 $k \to \infty$ 时，$z^{(k)} \to z^*$。

**(d) 直觉理解**

这就像在问："如果我把同一个变换反复应用，最终会停在哪个状态？" 如果变换是"收缩"的（把任何两个状态拉近），最终会收敛到一个点。

**(e) 边界情况**

- 如果 $f_\theta$ 不是压缩映射，迭代可能发散或出现极限环
- 实践中，残差结构 $z + f(z)$ 天然是接近压缩的（如果 $f$ 的 Lipschitz 常数 < 1）

### 公式 2：隐函数梯度

$$\frac{\partial \ell}{\partial \theta} = \frac{\partial \ell}{\partial z^*} \left(I - \frac{\partial f_\theta}{\partial z^*}\right)^{-1} \frac{\partial f_\theta}{\partial \theta}$$

**(a) 符号定义**

- $\ell$：损失函数
- $\frac{\partial \ell}{\partial z^*} \in \mathbb{R}^{1 \times d}$：损失对不动点的梯度
- $I \in \mathbb{R}^{d \times d}$：单位矩阵
- $\frac{\partial f_\theta}{\partial z^*} \in \mathbb{R}^{d \times d}$：$f_\theta$ 对 $z$ 的雅可比矩阵，在 $z^*$ 处计算
- $\frac{\partial f_\theta}{\partial \theta} \in \mathbb{R}^{d \times |\theta|}$：$f_\theta$ 对参数的雅可比矩阵

**(b) 公式来源**

隐函数定理。给定 $F(z, \theta) = z - f_\theta(z, x) = 0$，两边对 $\theta$ 求导：

$$\frac{\partial F}{\partial z}\frac{\partial z}{\partial \theta} + \frac{\partial F}{\partial \theta} = 0$$

代入 $\frac{\partial F}{\partial z} = I - \frac{\partial f_\theta}{\partial z}$，$\frac{\partial F}{\partial \theta} = -\frac{\partial f_\theta}{\partial \theta}$，得到：

$$(I - \frac{\partial f_\theta}{\partial z})\frac{\partial z}{\partial \theta} = \frac{\partial f_\theta}{\partial \theta}$$

$$\frac{\partial z}{\partial \theta} = (I - \frac{\partial f_\theta}{\partial z})^{-1} \frac{\partial f_\theta}{\partial \theta}$$

再用链式法则得到 $\frac{\partial \ell}{\partial \theta} = \frac{\partial \ell}{\partial z} \frac{\partial z}{\partial \theta}$。

**(c) 推导过程**

1. 从不动点方程 $z^* - f_\theta(z^*, x) = 0$ 出发
2. 两边对 $\theta$ 求导：$\frac{\partial z^*}{\partial \theta} - \frac{\partial f_\theta}{\partial z^*}\frac{\partial z^*}{\partial \theta} - \frac{\partial f_\theta}{\partial \theta} = 0$
3. 整理：$(I - \frac{\partial f_\theta}{\partial z^*})\frac{\partial z^*}{\partial \theta} = \frac{\partial f_\theta}{\partial \theta}$
4. 假设 $(I - \frac{\partial f_\theta}{\partial z^*})$ 可逆，得到上述公式

**(d) 直觉理解**

这个公式告诉我们：梯度传播需要"考虑"不动点对参数变化的响应。$(I - J)^{-1}$ 项相当于一个无限级数 $\sum_{k=0}^\infty J^k$，代表"如果参数变化一点点，不动点会如何调整"。

**(e) 边界情况**

- 如果 $f_\theta$ 的雅可比矩阵有特征值等于 1，$(I-J)$ 不可逆，梯度爆炸
- 实践中使用 Neumann 级数近似：$(I-J)^{-1}v \approx \sum_{k=0}^K J^k v$，$K$ 通常取 5-10

### 公式 3：Anderson 加速更新

$$z_{k+1} = (1-\beta) \sum_{i=0}^{m-1} \alpha_i z_{k-m+i} + \beta \sum_{i=0}^{m-1} \alpha_i f_\theta(z_{k-m+i})$$

**(a) 符号定义**

- $m$：历史窗口大小（通常取 5）
- $\alpha_i$：权重系数，满足 $\sum \alpha_i = 1$
- $\beta$：混合系数（通常取 1.0）
- $z_{k-m+i}$：历史隐藏状态
- $f_\theta(z_{k-m+i})$：对应的变换输出

**(b) 公式来源**

Anderson 混合法 [Anderson, 1965]，原本用于加速电子结构计算中的自洽场迭代。

**(c) 推导过程**

1. 定义残差 $r_k = f_\theta(z_k) - z_k$
2. 在 $m$ 步历史中寻找最小残差组合：$\min_{\alpha} \|\sum \alpha_i r_{k-m+i}\|$
3. 用约束 $\sum \alpha_i = 1$ 求解最小二乘问题
4. 用找到的 $\alpha$ 做外推

**(d) 直觉理解**

Anderson 加速相当于在历史迭代中做"外推"——如果残差在某个方向上持续减小，它直接跳到预测的收敛点，而不是慢慢走过去。

**(e) 边界情况**

- $m=1$ 时退化为简单迭代 $z_{k+1} = f_\theta(z_k)$
- $m$ 太大时，历史信息可能过时，反而降低收敛速度
- $\beta$ 控制混合比例，$\beta=0$ 时不使用新信息

### 公式 4：收敛容差

$$\frac{\|z^{(k+1)} - z^{(k)}\|_2}{\|z^{(k)}\|_2 + \epsilon} < \text{tol}$$

**(a) 符号定义**

- $\| \cdot \|_2$：L2 范数
- $\epsilon$：小常数防止除零（通常取 $10^{-8}$）
- $\text{tol}$：容差阈值（通常取 $10^{-2}$ 到 $10^{-5}$）

**(b) 公式来源**

标准的相对残差收敛判据，广泛用于数值优化和不动点求解。

**(c) 推导过程**

相对变化量 $\frac{\|z^{(k+1)} - z^{(k)}\|}{\|z^{(k)}\|}$ 度量了迭代的进展速度。当这个值小于容差时，认为迭代已经接近收敛。

**(d) 直觉理解**

这相当于问："上一步到这一步的变化有多大？" 如果变化很小（相对于当前状态的大小），说明我们可能已经在不动点附近了。

**(e) 边界情况**

- 当 $z^{(k)}$ 接近 0 时，分母很小，可能导致过早收敛
- $\epsilon$ 防止除以零，但也可能使收敛判据过于宽松

## 七、实验结果

### 7.1 分类任务

| 数据集 | 模型 | 准确率 | 参数量 | 迭代次数 |
|--------|------|--------|--------|----------|
| CIFAR-10 | ResNet-18 | 90.8% | 11.2M | 18层 |
| CIFAR-10 | DEQ-ResNet | 90.8% | 1.2M | 18.2 (平均) |
| CIFAR-10 | ResNet-110 | 91.6% | 1.7M | 110层 |
| CIFAR-10 | DEQ-ResNet (深) | 92.1% | 1.2M | 29.8 (平均) |

**关键发现**：
- DEQ 用 1.2M 参数达到 ResNet-18 的准确率（11.2M 参数），参数效率提升 9 倍
- 在 CIFAR-10 上，DEQ 比 ResNet-110 更高，参数少 30%
- ImageNet 上 DEQ 略低于 ResNet-50，但参数少 67%

**关键发现**：
- DEQ-BERT 用更少参数达到或超过 BERT-base

### 8.2 反向传播

```python
class DEQFunction(torch.autograd.Function):
    @staticmethod
    def forward(ctx, f, z0, *params):
        # 前向：Anderson 加速
        z_star = anderson(lambda z: f(z, *params), z0)
        ctx.save_for_backward(z_star, *params)
        ctx.f = f
        return z_star
    
    @staticmethod
    def backward(ctx, grad_output):
        z_star = ctx.saved_tensors[0]
        params = ctx.saved_tensors[1:]
        f = ctx.f
        
        # 使用 Neumann 级数近似逆
        # (I - J)^{-1} v ≈ sum_{k=0}^K J^k v
        
        v = grad_output
        for _ in range(5):  # K=5
            # 计算 J @ v 通过自动微分
            with torch.enable_grad():
                z_star_ = z_star.detach().requires_grad_()
                out = f(z_star_, *params)
                jv = torch.autograd.grad(out, z_star_, v, retain_graph=True)[0]
            v = grad_output + jv
        
        # 计算 grad wrt params
        with torch.enable_grad():
            z_star_ = z_star.detach().requires_grad_()
            out = f(z_star_, *params)
        grad_params = torch.autograd.grad(out, params, v)
        
        return None, None, *grad_params
```

### 8.3 关键实现细节

1. **梯度截断**：Neumann 级数只做 5 步，不是真正收敛到逆
2. **detach 技巧**：在 Anderson 加速中，历史状态用 `.detach()` 切断梯度流
3. **正则化**：Anderson 最小二乘问题用 $\lambda=10^{-4}$ 的正则化防止奇异
4. **容差**：训练时用 $10^{-2}$，测试时用 $10^{-5}$

## 十、局限

### 10.1 收敛保证

**局限**：DEQ 假设 $f_\theta$ 是压缩映射，但实践中无法保证。当 $f_\theta$ 不是压缩映射时，迭代可能不收敛。

**影响**：训练过程中，约 5-10% 的 batch 可能无法在 50 次迭代内收敛。论文用最大迭代次数截断，但这会引入近似误差。

**后续工作**：[Winston & Kolter, 2020] 提出了 Lipschitz 约束的 DEQ，通过谱归一化保证收敛。

### 10.2 并行效率

**局限**：不动点迭代是串行的——每步必须等前一步完成。在 GPU 上，这限制了并行度。

**影响**：虽然参数少，但训练时间可能更长。论文报告 DEQ-ResNet 的训练时间是 ResNet-18 的 2-3 倍。

**后续工作**：暂无好的解决方案，这是隐式方法的固有问题。

### 10.3 雅可比近似

**局限**：反向传播使用 Neumann 级数近似 $(I-J)^{-1}$，只取前 5 项。对于非规范化的雅可比矩阵，这个近似可能很差。

**影响**：梯度近似误差可能导致训练不稳定。论文报告需要较小的学习率（$10^{-4}$ 级别）。

**后续工作**：[Fung et al., 2022] 提出了更精确的逆近似方法。

### 10.4 架构限制

**局限**：DEQ 要求 $f_\theta$ 的输入输出维度相同。对于需要改变空间分辨率的任务（如图像分割），需要额外的上/下采样模块。

**影响**：在 ImageNet 上，DEQ 需要额外的下采样层来匹配 ResNet 的多尺度架构。

**后续工作**：Multiscale DEQ [Bai et al., 2020] 直接解决了这个问题。

## 十一、小结

Deep Equilibrium Models 把深度学习的一个基本假设——"深度等于层数"——彻底解构了。它的核心洞察是：**我们需要的不是很多层，而是一个能够反复精炼表示的计算过程**。

从技术角度看，DEQ 做了三件关键事：
1. **参数共享 + 不动点迭代**：用一个浅层变换实现无限深度
2. **隐函数梯度**：$O(1)$ 显存的反向传播
3. **输入注入**：保证迭代过程不离散

这些设计让 DEQ 在保持表达能力的同时，大幅降低了参数和显存需求。在 CIFAR-10 上，DEQ 用 1/9 的参数达到了 ResNet-18 的性能；在文本分类上，DEQ-BERT 用更少参数超越了 BERT-base。

DEQ 的更大意义是开启了"隐式深度学习"这个新方向——网络不再是层的堆叠，而是方程求解。后续的 Multiscale DEQ、Implicit Transformer、神经不动点网络都沿着这个方向推进。它告诉我们：**有时候，最好的深度不是堆出来的，而是算出来的**。
