# # Prefix-Tuning: Optimizing Continuous Prompts for Generation（2021）

**论文：** [arXiv](https://arxiv.org/abs/2101.00190) · Li & Liang · 2021

## 一、先搞清楚坑在哪

自然语言处理领域在 2020-2021 年经历了一场范式转变：从"预训练+微调"转向"预训练+提示（Prompting）"。GPT-3 的惊艳表现让人们意识到，通过精心设计的文本提示（如 "Translate English to French: "），可以在不更新模型参数的情况下诱导预训练语言模型完成特定任务。

但这个范式有个尴尬的问题：**提示的质量高度依赖人工设计**。你需要反复尝试不同的措辞、格式、甚至标点符号，才能找到一个有效的提示。更糟糕的是，对于不同的任务、不同的模型，最优提示完全不同。这就像每次做菜都要重新发明调料配方——费力且不稳定。

AutoPrompt [Shin et al., 2020] 和离散提示搜索 [Jiang et al., 2020] 尝试自动化这个流程，但它们在离散的 token 空间中搜索，效率低且容易陷入次优解。另一种思路——微调（Fine-tuning）——虽然效果稳定，但代价是每个任务都要保存一份完整的模型副本（GPT-2 有 1.5B 参数，GPT-3 有 175B 参数），这在实践中几乎不可行。

## 二、Fine-tuning 的真正问题

先看传统微调范式的问题。给定一个预训练语言模型（如 GPT-2），微调的过程是：

1. 复制整个预训练模型
2. 在所有参数上计算任务损失的梯度
3. 更新所有参数

这带来三个具体问题：

**存储成本爆炸**：假设你有 N 个下游任务，每个任务都需要存储一份完整的模型参数。对于 GPT-2 (1.5B 参数)，一个模型副本约 6GB。100 个任务就是 600GB。对于 GPT-3 (175B 参数)，一个副本约 700GB——这已经不是大多数实验室能承受的了。

**过拟合风险**：微调更新了所有参数，而下游任务的数据量通常很小（几百到几千条）。模型容易"记住"训练集，在分布外数据上泛化能力差。

**跨任务迁移困难**：微调后的模型参数是针对特定任务优化的，不同任务的模型之间没有共享表示。如果你想新增一个任务，必须从头开始微调。

## 三、Prefix-Tuning 的核心思路

**一句话概括**：不更新任何预训练参数，而是在 Transformer 的每一层前面插入一组可学习的"虚拟 token"（prefix），只优化这些 prefix 的参数。

这个思路的直观理解是这样的：想象你有一个训练有素的翻译员（预训练模型）。传统微调相当于给翻译员做手术来改变他的大脑结构（更新所有参数）。Prefix-Tuning 则是在翻译员的耳边悄悄说一些"提示语"（prefix），让他按照特定方向工作——但翻译员本身没有任何改变。

关键洞察是：**语言模型的每一层都在构建上下文表示，而 prefix 可以看作是可学习的、任务特定的"软上下文"**。与离散文本提示不同，prefix 是连续向量，可以直接通过梯度下降优化，不需要在离散 token 空间中搜索。

### 高层次的流程

```
输入文本 → [Prefix tokens] + [真实输入] → 冻结的预训练 LM → 输出
                                        ↑
                             只优化这部分参数
```

## 四、方法详解

### 4.1 问题定义

给定一个预训练的自回归语言模型 $p_{\phi}(y|x)$，其中 $\phi$ 是预训练参数，$x$ 是输入，$y$ 是输出序列。在传统的微调中，我们最大化：

$$\max_{\phi} \sum_{(x,y) \in \mathcal{D}} \sum_{t=1}^{|y|} \log p_{\phi}(y_t | y_{<t}, x)$$

在 Prefix-Tuning 中，我们冻结 $\phi$，只优化 prefix 参数 $P_{\theta}$：

$$\max_{\theta} \sum_{(x,y) \in \mathcal{D}} \sum_{t=1}^{|y|} \log p_{\phi}(y_t | y_{<t}, x, P_{\theta})$$

这里 $P_{\theta}$ 是一组连续向量，被插入到 Transformer 每一层的 key 和 value 前面。

### 4.2 架构设计

如图 1 所示，Prefix-Tuning 的核心是在 Transformer 的每一层都插入 prefix 向量。

![Figure 1: Prefix-Tuning 架构](figures/prefix-tuning/fig1.png)

**图 1 说明**：左侧是标准的 Transformer 层，右侧是插入 prefix 后的结构。红色部分表示可训练的 prefix 参数，灰色部分表示冻结的预训练参数。注意 prefix 被分别插入到 attention 的 key 和 value 路径中。

具体来说，对于 Transformer 的第 $i$ 层，原始的 attention 计算是：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

插入 prefix 后，key 和 value 变为：

$$K' = [P_K^{(i)}; K]$$
$$V' = [P_V^{(i)}; V]$$

其中 $P_K^{(i)} \in \mathbb{R}^{l \times d_k}$ 和 $P_V^{(i)} \in \mathbb{R}^{l \times d_v}$ 是可学习的 prefix 矩阵，$l$ 是 prefix 长度，$[;]$ 表示拼接操作。

### 4.3 为什么只插入 key 和 value？

这是一个重要的设计选择。论文尝试了三种方案：

1. **只插入 key**：效果最差，因为 value 决定了从上下文中"读取"什么信息
2. **插入 key 和 value**：最佳效果，因为 prefix 可以同时影响"关注哪里"和"读取什么"
3. **插入 key、value 和 query**：效果反而下降，因为 query 需要与输入交互，插入人工 query 会干扰这个交互

实验表明，只修改 key 和 value 是最优的。这背后的直觉是：**prefix 应该作为"额外的上下文"被读取，而不是作为"查询"去主动寻找信息**。

### 4.4 Prefix 长度的选择

prefix 长度 $l$ 是一个超参数。太短（如 $l=1$）没有足够的容量来编码任务信息；太长（如 $l=100$）会引入过多参数，可能导致过拟合。

![Figure 2: Prefix 长度的影响](figures/prefix-tuning/fig2.png)

**图 2 说明**：x 轴是 prefix 长度，y 轴是验证集困惑度（越低越好）。可以看到，对于 table-to-text 任务，$l=10$ 左右达到最优；对于 summarization 任务，$l=200$ 左右更好。这说明不同任务需要不同容量的 prefix。

### 4.5 参数重参数化技巧

直接优化 prefix 矩阵 $P_{\theta}$ 会遇到一个问题：prefix 参数的数量是 $2 \times L \times l \times d$（每层两个矩阵，key 和 value），对于 GPT-2 Medium（$L=24, d=1024$）和 $l=10$，这大约是 50 万个参数，训练不稳定。

论文采用了一个巧妙的**重参数化**技巧：不直接优化 prefix 矩阵，而是通过一个小的 MLP 来生成它。

具体来说，定义一个更小的矩阵 $P' \in \mathbb{R}^{l \times d'}$，其中 $d' \ll d$（论文中 $d'=512$），然后通过一个两层的 MLP $g$ 将其映射到实际的 prefix 维度：

$$P_{\theta} = g(P')$$

其中 $g: \mathbb{R}^{l \times d'} \rightarrow \mathbb{R}^{l \times d}$ 是一个前馈网络。训练时只优化 $P'$ 和 $g$ 的参数，推理时可以直接计算 $P_{\theta}$ 并丢弃 $g$。

这个技巧有两个好处：
1. **减少可训练参数**：从 $2Lld$ 降到 $2Lld' + (d' \times d + d \times d)$，大约减少了一半
2. **稳定训练**：更小的参数量使得优化更平滑

## 五、公式详解

### 5.1 自回归语言模型的目标函数

**公式**：

$$\max_{\theta} \log p_{\theta}(y|x) = \max_{\theta} \sum_{t=1}^{|y|} \log p_{\theta}(y_t | y_{<t}, x)$$

**(a) 符号定义**：
- $x$：输入序列（如 "TL;DR:"）
- $y$：输出序列（如摘要文本）
- $y_t$：输出序列的第 $t$ 个 token
- $y_{<t}$：输出序列中位置 $t$ 之前的所有 token
- $\theta$：可训练参数（即 prefix 参数）
- $|y|$：输出序列的长度

**(b) 公式来源**：
这是标准的自回归语言模型目标函数，源自 [Radford et al., 2019] GPT-2 的论文。区别在于，这里的 $\theta$ 只包含 prefix 参数，而不包含预训练参数 $\phi$。

**(c) 推导过程**：
自回归模型将序列概率分解为条件概率的乘积：

$$p(y|x) = \prod_{t=1}^{|y|} p(y_t | y_{<t}, x)$$

取对数后，乘积变为求和，因此最大化对数似然等价于最大化每个位置条件概率的对数和。

**(d) 直觉理解**：
这个公式的意思是：对于输出序列中的每个 token，模型应该能够根据之前的所有 token（包括输入 $x$ 和已生成的输出 $y_{<t}$）来预测它。每个位置的预测误差都会累积到最终损失中。

具体例子：假设输入 $x$="TL;DR: The cat sat on the mat."，输出 $y$="A cat sits on a mat."。那么：
- $t=1$：预测 "A"，条件于 $x$
- $t=2$：预测 "cat"，条件于 $x$ 和 "A"
- $t=3$：预测 "sits"，条件于 $x$ 和 "A cat"
- ...

### 5.2 带 Prefix 的 Attention 计算

**公式**：

$$\text{Attn}(Q, K, V) = \text{softmax}\left(\frac{Q[P_K; K]^T}{\sqrt{d_k}}\right)[P_V; V]$$

**(a) 符号定义**：
- $Q \in \mathbb{R}^{n \times d_k}$：query 矩阵，来自当前层的输入
- $K \in \mathbb{R}^{m \times d_k}$：key 矩阵，来自编码器的输出
- $V \in \mathbb{R}^{m \times d_v}$：value 矩阵，来自编码器的输出
- $P_K \in \mathbb{R}^{l \times d_k}$：可学习的 prefix key 矩阵
- $P_V \in \mathbb{R}^{l \times d_v}$：可学习的 prefix value 矩阵
- $n$：输入序列长度
- $m$：编码器输出序列长度
- $l$：prefix 长度
- $d_k, d_v$：key 和 value 的维度

**(b) 公式来源**：
这是标准 Transformer attention [Vaswani et al., 2017] 的变体。原始公式是：

$$\text{Attn}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

Prefix-Tuning 将 key 和 value 替换为拼接后的版本。

**(c) 推导过程**：

1. 首先，将 prefix key 和原始 key 拼接：$K' = [P_K; K] \in \mathbb{R}^{(l+m) \times d_k}$
2. 计算注意力分数：$S = \frac{QK'^T}{\sqrt{d_k}} \in \mathbb{R}^{n \times (l+m)}$
3. 应用 softmax：$A = \text{softmax}(S) \in \mathbb{R}^{n \times (l+m)}$
4. 将 prefix value 和原始 value 拼接：$V' = [P_V; V] \in \mathbb{R}^{(l+m) \times d_v}$
5. 加权求和：$\text{Attn} = AV' \in \mathbb{R}^{n \times d_v}$

**(d) 直觉理解**：
拼接操作意味着输入序列中的每个 token 都可以选择性地关注 prefix 或原始上下文。prefix 提供了一组"虚拟"的上下文向量，模型可以学习在何时、以多大权重来使用它们。

具体例子：假设 $n=5$（输入 "The cat sat"），$m=3$（编码器输出），$l=2$（prefix 长度）。那么：
- $K'$ 的形状是 $(2+3) \times d_k = 5 \times d_k$
- 注意力分数矩阵 $S$ 的形状是 $5 \times 5$
- 对于输入中的第一个 token "The"，它可以分配注意力权重到两个 prefix token 和三个编码器输出上

### 5.3 重参数化的 MLP

**公式**：

$$P_{\theta} = \text{MLP}_{\theta}(P') = W_2 \cdot \sigma(W_1 \cdot P' + b_1) + b_2$$

**(a) 符号定义**：
- $P' \in \mathbb{R}^{l \times d'}$：低维的 prefix 表示，$d' \ll d$
- $W_1 \in \mathbb{R}^{d \times d'}$：第一层权重矩阵
- $b_1 \in \mathbb{R}^{d}$：第一层偏置
- $W_2 \in \mathbb{R}^{d \times d}$：第二层权重矩阵
- $b_2 \in \mathbb{R}^{d}$：第二层偏置
- $\sigma$：非线性激活函数（论文使用 ReLU）

**(b) 公式来源**：
这是标准的双层 MLP，类似 [Devlin et al., 2019] BERT 中的前馈网络。区别在于输入是低维向量 $P'$，输出是高维的 prefix 矩阵。

**(c) 推导过程**：
1. 输入 $P' \in \mathbb{R}^{l \times d'}$ 通过第一层：$H = W_1 P' + b_1 \in \mathbb{R}^{l \times d}$
2. 应用 ReLU：$H' = \max(0, H) \in \mathbb{R}^{l \times d}$
3. 通过第二层：$P_{\theta} = W_2 H' + b_2 \in \mathbb{R}^{l \times d}$

**(d) 直觉理解**：
这个 MLP 充当了一个"维度扩展器"。低维空间 $d'=512$ 中的每个点对应高维空间 $d=1024$ 中的一个 prefix 矩阵。通过 MLP 的映射，模型可以在低维空间中进行更稳定的优化，同时保持高维空间的表达能力。

## 七、位置

### 7.1 前驱工作

1. **Adapter Tuning [Houlsby et al., 2019]**：在 Transformer 层之间插入小的 adapter 模块。与 Prefix-Tuning 的区别是：adapter 修改了每一层的内部表示，而 prefix 只影响 attention 的计算。

2. **Prompt Engineering [Brown et al., 2020]**：手动设计离散文本提示。Prefix-Tuning 将其扩展到连续空间，实现了端到端优化。

3. **AutoPrompt [Shin et al., 2020]**：在离散空间中搜索最优提示。Prefix-Tuning 证明了连续空间更高效。

### 7.2 同期竞品

1. **LoRA [Hu et al., 2021]**：在 attention 的权重矩阵上添加低秩分解。与 Prefix-Tuning 思路类似，但作用于权重而非输入。

2. **AdapterFusion [Pfeiffer et al., 2021]**：组合多个 adapter 模块。与 Prefix-Tuning 的 multi-task 扩展思路相似。

3. **Diff pruning [Guo et al., 2021]**：学习一个稀疏的"差异向量"来微调。与 Prefix-Tuning 一样追求参数效率。

### 7.3 后续影响

1. **P-Tuning [Liu et al., 2021]**：将 prefix 思想扩展到 encoder-only 模型（如 BERT），用 LSTM 生成 prompt 向量。

2. **Prompt Tuning [Lester et al., 2021]**：简化了 Prefix-Tuning，只在输入层添加 prompt token，效果相近但更简单。

3. **P-Tuning v2 [Liu et al., 2022]**：将 prefix 从 decoder-only 扩展到 encoder-decoder 架构，证明了 deep prompt 比 shallow prompt 更有效。

4. **Multitask Prompt Tuning [Wang et al., 2022]**：共享 prefix 参数实现多任务学习，验证了 prefix 的跨任务迁移能力。

5. **Prefix-Tuning in Vision [Zhou et al., 2022]**：将 prefix 思想引入视觉 Transformer，用于图像生成任务。

## 八、局限

1. **推理速度**：prefix 增加了 attention 计算的序列长度（从 $n$ 到 $n+l$），导致推理延迟增加。对于 $l=100$，延迟增加约 20%。

2. **prefix 长度选择**：不同任务需要不同的 prefix 长度，没有通用的选择准则。论文建议在验证集上搜索，但这增加了调参成本。

3. **跨任务泛化**：虽然 prefix 可以跨任务迁移，但效果不如独立训练。论文实验显示，共享 prefix 在相似任务上效果较好，在差异大的任务上效果下降。

4. **对模型架构的依赖**：当前设计针对自回归模型（GPT）和 encoder-decoder 模型（BART）。对于纯 encoder 模型（BERT），prefix 的作用机制不同，需要重新设计。

5. **可解释性**：与离散提示不同，连续 prefix 难以解释。我们不知道 prefix 中的每个向量编码了什么语义信息。

## 九、小结

Prefix-Tuning 提出了一种优雅的解决方案来应对"大模型微调"的存储和过拟合问题。核心洞察是：**不需要修改模型本身，只需要在每一层添加一组可学习的"软提示"**。

这个工作在三个层面上推动了领域发展：
1. **方法论**：证明了连续空间中的提示优化比离散搜索更有效
2. **实用性**：使大模型在消费级 GPU 上也能进行任务适配
3. **理论**：揭示了 Transformer 的 attention 机制可以被外部向量引导

后续的 Prompt Tuning、P-Tuning v2 等工作都建立在 Prefix-Tuning 的基础上。今天，prefix/prompt tuning 已经成为大模型适配的标准范式之一，与 LoRA、Adapter 等技术并列。
