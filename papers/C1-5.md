# # DN-DETR: Accelerate DETR Training by Introducing Query DeNoising（2022）

**论文：** [arXiv](https://arxiv.org/abs/2203.01305) · Feng Li, Hao Zhang, Shilong Liu, Jian Guo, Lionel M. Ni, Lei Zhang · 2022

---

## 一、先搞清楚坑在哪

目标检测一直是计算机视觉的核心任务。在 DN-DETR 之前，DETR [Carion et al., 2020] 开创性地将 Transformer 引入目标检测，用一组可学习的 object queries 替代了传统的 anchor 和 NMS 后处理。但 DETR 有一个致命问题：**训练收敛极慢**。在 COCO 数据集上，DETR 需要 500 个 epoch 才能收敛，而 Faster R-CNN 只需要 12-36 个 epoch。这个收敛速度使得 DETR 在实际应用中几乎不可用，也成为了后续工作的核心改进方向。

## 二、DETR 的真正问题

DETR 为什么收敛这么慢？直观理解：DETR 使用二分图匹配（bipartite matching）将预测框与 ground truth 一一对应。这个匹配过程是 **不稳定** 的——同一个 ground truth 在不同训练阶段可能匹配到不同的预测 query，导致每个 query 的优化信号不断变化，难以稳定学习。

更具体地说，DETR 的 decoder 在训练初期，object queries 对输出框的预测几乎是随机的。二分图匹配算法（Hungarian algorithm）会将这些随机预测与 ground truth 配对。由于预测是随机的，配对结果也高度不稳定。这导致 decoder 的每个 query 接收到的梯度方向不断变化，每个 query 的监督信号持续漂移，训练难以收敛。

## 三、DN-DETR 的核心思路

DN-DETR 的核心洞察非常简洁：**如果我们能在训练时给 decoder 一些"作弊"信息——即带有已知噪声的 ground truth 框——让 decoder 学会"去噪"，那么 decoder 就能更好地理解 query 和输出框之间的关系，从而加速收敛。**

一句话概括：**将去噪任务（denoising）作为辅助训练任务，与原始的匈牙利匹配损失联合优化，加速 DETR 训练收敛。**

整体思路：
1. 在训练阶段，生成一组"带噪声的 ground truth 框"作为额外的 decoder queries
2. decoder 需要从这些带噪声的输入中恢复出原始的 ground truth 框
3. 这个去噪任务为 decoder 提供了稳定、直接的监督信号，加速了 query 的学习

**关键区别**：在训练时，decoder 的输入 queries 由两部分组成：
1. **可学习的 object queries（原始 DETR queries）**：N 个，用于匈牙利匹配
2. **带噪声的 ground truth queries（denoising queries）**：M 个，用于去噪任务

### 4.2 去噪任务的详细设计

#### （a）痛点与动机

**痛点**：DETR 的 decoder 在训练初期，每个 query 的监督信号高度不稳定。匈牙利匹配的结果随预测质量变化，导致梯度方向不断漂移。

**为什么现有方法失败**：
- DETR [Carion et al., 2020] 完全依赖匈牙利匹配，没有为 decoder 提供直接的监督信号
- Deformable DETR [Zhu et al., 2021] 引入了多尺度特征和可变形注意力，加速了 encoder 训练，但 decoder 的不稳定性问题依然存在
- TSP-FCOS [Sun et al., 2021] 等基于 FCOS 的方法使用密集预测，避免了匹配问题，但无法利用 Transformer 的全局建模能力

**观察**：如果 decoder 能够学会"修复"带噪声的框，那么它自然也能学会从随机 queries 中预测准确框。去噪任务为 decoder 提供了**稳定、直接**的监督信号。

#### （b）方案细节

去噪 queries 的生成过程如下：

1. **对每个 ground truth 框生成多个带噪声的副本**：
   - 对每个 GT 框的 center (cx, cy) 添加高斯噪声：$cx' = cx + \sigma_{pos} \cdot \text{randn}()$
   - 对每个 GT 框的宽高 (w, h) 添加缩放噪声：$w' = w \cdot (1 + \sigma_{size} \cdot \text{randn}())$
   - 对类别标签添加随机扰动：以一定概率替换为其他类别

2. **将带噪声的框作为 decoder 的额外 queries**：
   - 每个带噪声的框经过一个 MLP 映射为 query embedding
   - 与原始 object queries 拼接后一起送入 decoder

3. **损失计算**：
   - 对去噪 queries 的输出，直接计算与对应 GT 框的 L1 损失和 GIoU 损失
   - 对原始 object queries，仍然使用匈牙利匹配后的损失
   - 总损失 = 匈牙利匹配损失 + $\lambda_{dn}$ × 去噪损失

#### （c）为什么有效

去噪任务之所以有效，原因有三：

1. **稳定监督信号**：去噪 queries 与 GT 框有明确的对应关系，每个 query 的梯度方向是固定的，不会因为匹配结果变化而漂移。

2. **直接学习框回归**：去噪任务直接训练 decoder 从带噪声的输入中恢复准确框，这本质上是在学习"框的修正能力"。这种能力可以直接迁移到原始 queries 的预测中。

3. **加速注意力学习**：去噪 queries 在训练初期就能产生合理的注意力图，帮助其他 queries 更快地学习到正确的交叉注意力模式。

从消融实验（论文 Table 2）可以看出：加入去噪训练后，DN-DETR 在 12 epoch 时达到 38.7 AP，而原始 DETR 在相同 epoch 时仅 36.1 AP。

#### （d）与 Related Work 的关系

去噪训练与以下工作有密切关系：

- **DETR [Carion et al., 2020]**：DN-DETR 直接继承 DETR 的架构，增加去噪辅助任务。关键区别是 DETR 完全依赖匈牙利匹配，而 DN-DETR 增加了直接监督。

- **Deformable DETR [Zhu et al., 2021]**：Deformable DETR 通过多尺度可变形注意力加速 encoder，但 decoder 的不稳定性问题依然存在。DN-DETR 的去噪训练与 Deformable DETR 互补，可以叠加使用。

- **Conditional DETR [Meng et al., 2021]**：Conditional DETR 通过条件空间查询（conditional spatial queries）加速 decoder 训练。DN-DETR 与 Conditional DETR 的思路不同——前者通过辅助任务提供稳定监督，后者通过改进 query 设计加速收敛。

- **Mask2Former [Cheng et al., 2022]**：后续工作将去噪训练扩展到实例分割任务，证明了该方法的通用性。

#### （e）如果去掉会怎样

从论文 Table 2 的消融实验可以看出：
- 去掉去噪训练（即退化为原始 DETR）：12 epoch AP 从 38.7 降至 36.1，下降 2.6 AP
- 去掉类别去噪（只保留框去噪）：AP 从 38.7 降至 38.1，下降 0.6 AP
- 去掉框去噪（只保留类别去噪）：AP 从 38.7 降至 37.5，下降 1.2 AP

这表明框去噪比类别去噪更重要，但两者结合效果最好。

### 4.3 注意力掩码

#### （a）痛点与动机

如果让去噪 queries 与原始 queries 自由交互，会带来一个问题：**信息泄露**。去噪 queries 包含 GT 框信息，如果原始 queries 通过自注意力从去噪 queries 中"偷看"到这些信息，会导致模型退化。

**为什么需要注意力掩码**：在训练时，去噪 queries 知道 GT 框的位置。如果不加限制，原始 queries 可以通过自注意力从去噪 queries 获取 GT 信息，使得匈牙利匹配变得 trivial。这会造成 train-test mismatch——测试时没有 GT 信息可用。

#### （b）方案细节

DN-DETR 设计了注意力掩码（attention mask）来防止信息泄露：

1. **自注意力掩码**：在 decoder 的自注意力层中，禁止原始 queries 关注去噪 queries，禁止不同组的去噪 queries 互相关注，但允许去噪 queries 关注原始 queries。

2. **交叉注意力掩码**：交叉注意力不受限制，因为 encoder 的输出不包含 GT 信息。

具体实现：生成一个注意力掩码矩阵 $M \in \{0, -\infty\}^{Q \times Q}$，其中 $Q$ 是所有 queries 的总数。当 query i 和 query j 之间有信息泄露风险时，$M_{ij} = -\infty$，否则 $M_{ij} = 0$。

#### （c）为什么有效

注意力掩码的效果在论文 Table 3 中得到了验证：
- 不使用掩码（信息泄露）：12 epoch AP 从 38.7 降至 37.2，下降 1.5 AP
- 使用掩码：达到 38.7 AP

直白说：没有掩码时，模型学会了"作弊"——从带 GT 信息的去噪 queries 中获取答案，而不是真正学习框回归能力。

#### （d）与 Related Work 的关系

注意力掩码的设计与以下工作相关：

- **BERT [Devlin et al., 2019]**：BERT 的 MLM 任务也使用注意力掩码来防止信息泄露（MASK token 不能看到被掩盖的词）。DN-DETR 的思路与此类似。

- **Masked Autoencoder [He et al., 2022]**：MAE 通过掩码图像块来学习视觉表示。DN-DETR 的去噪 queries 可以被视为一种"掩码"的 GT 框。

- **DAB-DETR [Liu et al., 2022]**：后续工作将去噪训练与 DAB-DETR 的 4D anchor 结合，证明了去噪训练的通用性。

#### （e）如果去掉会怎样

如上所述，去掉注意力掩码后 AP 下降 1.5。更重要的是，如果不使用掩码，模型在测试时会表现更差，因为训练时养成了"偷看"的习惯，测试时没有 GT 信息可用。

### 4.4 去噪组数

#### （a）痛点与动机

每个 GT 框应该生成多少个带噪声的副本？这是一个超参数选择问题。

**为什么需要多组**：如果每个 GT 框只有一个去噪 query，模型可能只学会"微调"（fine-tuning）而非"去噪"（denoising）。多组不同噪声水平的去噪 queries 可以让模型学习到更鲁棒的框修正能力。

#### （b）方案细节

DN-DETR 为每个 GT 框生成 $K$ 个带噪声的副本（论文中使用 $K=5$）。每组使用不同的噪声水平和随机种子。

#### （c）为什么有效

从论文 Table 4 的消融实验可以看出：
- $K=1$：AP 37.8
- $K=3$：AP 38.5
- $K=5$：AP 38.7
- $K=10$：AP 38.6

$K=5$ 达到最优，更多组数没有带来进一步提升。

#### （d）与 Related Work 的关系

多组去噪的设计与数据增强有相似之处——通过增加训练样本的多样性来提高模型鲁棒性。但这里的"多样性"是在 query 空间而非图像空间。

#### （e）如果去掉会怎样

如果 $K=1$，AP 下降约 1.0。模型只能学到一种噪声水平的去噪，泛化能力受限。

## 五、公式详解

### 5.1 去噪 query 生成

**公式 1：带噪声的框生成**

$$cx' = cx + \sigma_{pos} \cdot \epsilon_{pos}, \quad \epsilon_{pos} \sim \mathcal{N}(0, 1)$$
$$cy' = cy + \sigma_{pos} \cdot \epsilon_{pos}, \quad \epsilon_{pos} \sim \mathcal{N}(0, 1)$$
$$w' = w \cdot (1 + \sigma_{size} \cdot \epsilon_{size}), \quad \epsilon_{size} \sim \mathcal{N}(0, 1)$$
$$h' = h \cdot (1 + \sigma_{size} \cdot \epsilon_{size}), \quad \epsilon_{size} \sim \mathcal{N}(0, 1)$$

#### （a）符号定义

- $(cx, cy, w, h)$：ground truth 框的中心坐标和宽高
- $(cx', cy', w', h')$：带噪声的框
- $\sigma_{pos}$：位置噪声的标准差（论文中设为 0.2）
- $\sigma_{size}$：尺寸噪声的标准差（论文中设为 0.4）
- $\epsilon_{pos}, \epsilon_{size}$：从标准正态分布采样的随机噪声

#### （b）公式来源

这是 DN-DETR 原创的噪声生成方案。选择高斯噪声而非均匀噪声，是因为高斯噪声更接近实际预测误差的分布——大多数预测误差较小，少数误差较大。

#### （c）直觉理解

这个公式的直觉是：模拟 decoder 在训练初期的预测误差。训练初期，decoder 的预测几乎随机，中心偏移可能达到 0.2 倍图像尺寸，尺寸误差可能达到 0.4 倍。通过让 decoder 从这种噪声中恢复 GT 框，它学会了"修正"能力。

**具体例子**：假设 GT 框为 (0.5, 0.5, 0.2, 0.3)（归一化坐标），$\sigma_{pos}=0.2, \sigma_{size}=0.4$：
- 假设 $\epsilon_{pos} = 0.5, \epsilon_{size} = -0.3$
- $cx' = 0.5 + 0.2 \times 0.5 = 0.6$
- $cy' = 0.5 + 0.2 \times 0.5 = 0.6$
- $w' = 0.2 \times (1 + 0.4 \times (-0.3)) = 0.2 \times 0.88 = 0.176$
- $h' = 0.3 \times (1 + 0.4 \times (-0.3)) = 0.3 \times 0.88 = 0.264$

decoder 需要从 (0.6, 0.6, 0.176, 0.264) 恢复到 (0.5, 0.5, 0.2, 0.3)。

#### （d）边界情况

- 当 $\sigma_{pos}=0$ 时，去噪 queries 退化为 GT 框本身，去噪任务变为恒等映射
- 当 $\sigma_{size}$ 过大时（如 > 1.0），框可能变为负宽高或超出图像边界，需要裁剪
- 当 $\sigma_{pos}$ 过大时（如 > 0.5），中心可能移出图像，需要限制

### 5.2 总损失函数

**公式 2：DN-DETR 的总损失**

$$\mathcal{L}_{total} = \mathcal{L}_{Hungarian} + \lambda_{dn} \cdot \mathcal{L}_{denoising}$$

其中：
$$\mathcal{L}_{Hungarian} = \sum_{i=1}^{N} \left[ \mathbb{1}_{\{\sigma(i) \neq \emptyset\}} \cdot \left( \mathcal{L}_{cls}(p_i, c_{\sigma(i)}) + \mathcal{L}_{box}(b_i, b_{\sigma(i)}) \right) \right]$$

$$\mathcal{L}_{denoising} = \frac{1}{M} \sum_{j=1}^{M} \left[ \mathcal{L}_{cls}(p_j', c_j) + \mathcal{L}_{box}(b_j', b_j) \right]$$

#### （a）符号定义

- $\mathcal{L}_{Hungarian}$：原始 DETR 的匈牙利匹配损失
- $\mathcal{L}_{denoising}$：去噪损失
- $\lambda_{dn}$：去噪损失的权重（论文中设为 2）
- $N$：原始 object queries 的数量（通常为 300）
- $M$：去噪 queries 的总数（$K \times \text{num\_gt}$）
- $\sigma(i)$：匈牙利匹配的结果，将 query i 匹配到 GT 框
- $p_i, b_i$：query i 的预测类别和框
- $p_j', b_j'$：去噪 query j 的预测类别和框
- $c_j, b_j$：去噪 query j 对应的 GT 类别和框

#### （b）公式来源

匈牙利匹配损失来自 DETR [Carion et al., 2020]。去噪损失是 DN-DETR 新增的，形式上与匈牙利匹配损失中的框回归和分类损失相同。

#### （c）直觉理解

总损失由两项组成：
1. **匈牙利匹配损失**：监督原始 queries 的预测，通过匈牙利匹配建立 query-GT 对应关系
2. **去噪损失**：监督去噪 queries 的预测，直接使用已知的 query-GT 对应关系

关键区别在于：去噪损失不需要匈牙利匹配，因此梯度更稳定。

#### （d）具体例子

假设图像中有 2 个 GT 框，$K=5$，则 $M=10$。原始 queries 有 300 个。

- $\mathcal{L}_{Hungarian}$ 计算 300 个 queries 中匹配到 GT 的 2 个 queries 的损失（其余 298 个被匹配到 ∅）
- $\mathcal{L}_{denoising}$ 计算所有 10 个去噪 queries 的损失

总损失中，去噪损失占主导（因为 $M$ 通常远大于匹配到的 queries 数量），这解释了为什么去噪训练能显著加速收敛。

#### （e）边界情况

- 当 $\lambda_{dn}=0$ 时，退化为原始 DETR
- 当图像中没有 GT 框时，$M=0$，去噪损失为零，总损失退化为匈牙利匹配损失
- 当 $\lambda_{dn}$ 过大时（如 > 10），模型可能过度关注去噪任务而忽略原始匹配任务

### 5.3 注意力掩码

**公式 3：自注意力掩码**

$$Attention(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}} + M\right)V$$

其中 $M_{ij} = \begin{cases} 0, & \text{if } i \text{ can attend to } j \\ -\infty, & \text{otherwise} \end{cases}$

#### （a）符号定义

- $Q, K, V$：query、key、value 矩阵
- $d_k$：缩放因子
- $M$：注意力掩码矩阵

#### （b）公式来源

这是标准 Transformer 注意力机制的扩展，通过添加掩码来控制信息流。DN-DETR 的掩码规则如下：

1. 原始 queries 可以互相关注（$M_{ij}=0$）
2. 去噪 queries 可以互相关注（$M_{ij}=0$）
3. 原始 queries 不能关注去噪 queries（$M_{ij}=-\infty$）
4. 去噪 queries 可以关注原始 queries（$M_{ij}=0$）
5. 不同 GT 框的去噪 queries 不能互相关注（$M_{ij}=-\infty$）

#### （c）直觉理解

规则 3 防止信息泄露：原始 queries 不能从去噪 queries 中"偷看" GT 信息。
规则 5 防止不同 GT 框之间的信息混淆：每个去噪 query 只能关注到对应 GT 框的信息。

#### （d）具体例子

假设有 2 个原始 queries 和 2 个去噪 queries（来自不同 GT 框）：

```
     Q1  Q2  DQ1  DQ2
Q1    0   0   -∞   -∞
Q2    0   0   -∞   -∞
DQ1   0   0    0   -∞
DQ2   0   0   -∞    0
```

- Q1 和 Q2 可以互相关注，但不能关注 DQ1 和 DQ2
- DQ1 可以关注 Q1、Q2 和自己，但不能关注 DQ2
- DQ2 可以关注 Q1、Q2 和自己，但不能关注 DQ1

#### （e）边界情况

- 如果所有 queries 都是去噪 queries（没有原始 queries），掩码退化为仅禁止不同 GT 框之间的交互
- 如果所有 queries 都是原始 queries（没有去噪 queries），掩码退化为全零（标准自注意力）

## 七、位置

### 7.1 前驱工作

1. **DETR [Carion et al., 2020]**：开创性地将 Transformer 用于目标检测。DN-DETR 直接继承 DETR 的架构，解决其收敛慢的问题。

2. **Conditional DETR [Meng et al., 2021]**：通过条件空间查询加速 decoder 训练。DN-DETR 与 Conditional DETR 互补，可以叠加使用。

3. **Deformable DETR [Zhu et al., 2021]**：通过多尺度可变形注意力加速 encoder 训练。DN-DETR 的去噪训练主要针对 decoder，与 Deformable DETR 互补。

4. **SPARSE R-CNN [Sun et al., 2021]**：使用可学习的 proposal 进行迭代优化。DN-DETR 的去噪训练可以视为一种隐式的迭代优化。

### 7.2 同期竞品

1. **DAB-DETR [Liu et al., 2022]**：将 object queries 显式建模为 4D anchor。与 DN-DETR 同期发布，两者可以结合。

2. **Anchor DETR [Wang et al., 2022]**：使用 anchor 点作为 queries。与 DN-DETR 的思路不同——前者改进 query 初始化，后者改进训练策略。

3. **SMCA-DETR [Gao et al., 2021]**：通过空间调制注意力加速收敛。与 DN-DETR 互补——SMCA 改进 attention 机制，DN-DETR 改进训练目标。

### 7.3 后续影响

1. **DINO [Zhang et al., 2022]**：将 DN-DETR 的去噪训练与 DAB-DETR 的 anchor 结合，提出了更完善的去噪方案，成为 DETR 系列的新 SOTA。

2. **Mask DINO [Li et al., 2023]**：将 DINO 扩展到分割任务，去噪训练作为核心组件被保留。

3. **H-DETR [Jia et al., 2023]**：通过混合匹配策略改进去噪训练，进一步加速收敛。

4. **Group DETR [Chen et al., 2022]**：使用多组 queries 进行匹配，与去噪训练有相似思路。

5. **Align-DETR [Cai et al., 2023]**：通过 query 对齐改进去噪训练，解决去噪 queries 与原始 queries 之间的分布差异。

## 八、局限

### 8.1 训练-测试不一致

**局限**：去噪训练只在训练阶段使用，测试时没有 GT 信息可用。这意味着 decoder 在训练和测试时的行为不同。

**影响**：虽然实验证明这种不一致不会导致性能下降，但理论上可能存在优化方向上的偏差——模型可能过度依赖去噪 queries 的特征。

**论文是否承认**：论文没有明确讨论这个问题。

### 8.2 超参数敏感

**局限**：去噪训练引入多个超参数：$\sigma_{pos}$、$\sigma_{size}$、$\lambda_{dn}$、$K$ 等。这些超参数的选择对性能有一定影响。

**影响**：在实际应用中，需要针对不同数据集调参。

**论文是否承认**：论文提供了消融实验，但未讨论超参数的敏感性。

### 8.3 与 GT 数量的耦合

**局限**：去噪 queries 的数量与 GT 框数量直接相关。当图像中 GT 框数量很大时（如密集场景），去噪 queries 的数量会急剧增加，导致计算量上升。

**影响**：在密集场景（如 CrowdHuman）中，计算开销可能成为瓶颈。

**论文是否承认**：论文没有讨论这个问题。

### 8.4 仅适用于训练

**局限**：去噪训练不改变推理过程，因此对推理速度没有提升。

**影响**：DN-DETR 的推理速度与原始 DETR 相同。

## 九、小结

DN-DETR 提出了一个简洁而有效的方案来加速 DETR 训练：通过引入去噪辅助任务，为 decoder 提供稳定、直接的监督信号。核心创新包括：
1. **去噪训练**：将带噪声的 GT 框作为额外 queries，训练 decoder 恢复原始框
2. **注意力掩码**：防止信息泄露，确保去噪训练的有效性

这个方法的优雅之处在于其简洁性——不需要修改 DETR 的架构，只需要在训练时增加一个辅助损失。实验结果证明，DN-DETR 将 DETR 的收敛速度提升了 10 倍（从 500 epoch 到 50 epoch）。

**更广泛的影响**：DN-DETR 的去噪训练思路被后续多个工作继承和改进（DINO、Mask DINO 等），成为 DETR 系列训练的标准组件。这个工作表明，Transformer 模型的训练不稳定性可以通过巧妙的辅助任务设计来解决，而不一定需要修改架构。
