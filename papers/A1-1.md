# BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding（2018）

**论文：** [arXiv](https://arxiv.org/abs/1810.04805) · Devlin et al. · 2018

---

## 一、先搞清楚坑在哪

2018 年以前，NLP 的预训练范式主要是两种：

1. **词向量（Word2Vec, GloVe）**：静态词嵌入，每个词只有一个向量，无法处理一词多义
2. **单向语言模型（ELMo, GPT）**：从左到右（或从右到左）依次预测下一个词，但只能看到单侧上下文

这些方法的共同问题是：**不能同时利用目标词左右两侧的上下文信息**。对于需要完整理解句意的任务（如 QA、NER、情感分析），这个限制是致命的。

直觉上，一个句子中"bank"这个词是"银行"还是"河岸"，需要看整句话才能判断。单向模型只能看到"bank"左边的词——如果关键信息在右边，它就错过了。

## 二、已有方法的真正问题

**ELMo（Peters et al., 2018）**：训练一个从左到右的 LSTM 和一个从右到左的 LSTM，然后把两个方向的表示拼接。这是双向的吗？表面上是，实际上只是两个单向模型的**浅层拼接**——两个方向从未在中间层交互，最后的表示只是左右特征的简单加和。

**GPT（Radford et al., 2018）**：用 Transformer decoder 做从左到右的语言模型预训练。由于 Transformer 的 self-attention 用因果掩码（causal mask），每个 token 只能看到它左边的 token。fine-tune 时再加一个分类头。

核心矛盾：**想理解一句话，你需要看完整上下文；但语言模型预训练的训练目标（预测下一个词）天然是单向的。**

## 三、BERT 的核心思路

一句话：**用 Masked Language Model（MLM）替代从左到右的语言模型，让预训练变成真正的双向。**

具体说：
1. **Masked LM**：随机 mask 15% 的输入 token，让模型预测被 mask 掉的词。这样模型必须同时利用左右两侧的上下文
2. **Next Sentence Prediction（NSP）**：同时预测两句话是否连续，让模型理解句子间关系
3. **Transformer Encoder**：使用双向 self-attention（不像 GPT 用因果掩码），每个 token 可以看到所有其他 token

## 四、网络架构

BERT 使用标准的 **Transformer Encoder**（Vaswani et al., 2017），没有对架构做任何修改。唯一的改动在输入表示和训练目标。

### 输入表示

```
Input:  [CLS] my dog is cute [SEP] he likes playing [SEP]
           │     │   │   │     │     │   │     │       │
           ▼     ▼   ▼   ▼     ▼     ▼   ▼     ▼       ▼
Token Emb + Segment Emb + Position Emb → sum → LayerNorm → Dropout
```

三层嵌入求和：
- **Token Embedding**：WordPiece 分词，30,000 token 词表。特殊 token：[CLS]（分类用）、[SEP]（分隔句子）
- **Segment Embedding**：标记 token 属于句子 A 还是句子 B（0/1）
- **Position Embedding**：可学习的 1D 位置编码，最大 512 位置

### 模型变体

| 模型 | Layers | Hidden | Heads | Params |
|------|--------|--------|-------|--------|
| BERT-Base | 12 | 768 | 12 | 110M |
| BERT-Large | 24 | 1024 | 16 | 340M |

BERT-Large 是 BERT-Base 的 3 倍参数量，但训练数据相同（BookCorpus + Wikipedia, 3.3B words）。

### 架构图

## 五、核心贡献

### 贡献 1: Masked Language Model（MLM）

**动机**：语言模型训练时只看到单侧上下文，但理解语言需要双向信息。

**方案**：随机 mask 15% 的 token，让 Transformer encoder 预测被 mask 的词。因为 encoder 的 self-attention 没有因果掩码，模型可以看到被 mask token 左右两侧的所有 token。

**实现细节（"策略"的一课）**：
- 80%：把选中 token 替换为 `[MASK]`
- 10%：替换为随机 token
- 10%：保持不变

为什么要这样设计？`[MASK]` 只在预训练中出现，fine-tune 时没有。如果 100% 用 `[MASK]`，模型会对 `[MASK]` 过拟合，Fine-tune 时看到真实文本反而表现差。10% 随机 token 强制模型学会"不管输入是什么，只要上下文给够了就能预测正确"；10% 保持原样让模型学会"同一个词也要正常编码，不能完全依赖上下文纠错"。

**和单向语言模型的本质区别**：

```
GPT（从左到右）:  我 → 爱 → [MASK] → (只能看"我""爱")
BERT（双向）:     我 → 爱 → [MASK] ← 你 ← 不 ← (可以看两边)
```

### 贡献 2: Next Sentence Prediction（NSP）

**动机**：很多下游任务（QA、NLI）需要理解两个句子之间的关系。单独的 MLM 只建模 token 级关系。

**方案**：给模型两个句子 A 和 B，50% 概率 B 是 A 的下一句（IsNext），50% 是随机句子（NotNext）。用 [CLS] token 的输出做二分类。

后来 RoBERTa（Liu et al., 2019）证明 NSP 对下游任务提升有限，移除 NSP 后部分任务反而更好。但 NSP 作为 BERT 的设计选择，影响了很多后续工作。

### 贡献 3: Fine-tuning 机制

BERT 的 fine-tune 极其简洁——预训练好的参数保持不变，只需在顶部加一个分类层：
- **单句分类**：取 [CLS] token 输出 → 线性层
- **句对分类**：两个句子拼成 `[CLS] A [SEP] B [SEP]`，[CLS] 输出 → 线性层
- **序列标注**：每个 token 的输出过分类头
- **QA**：预测答案的起始/结束位置

不需要为每个任务设计不同的架构——同一个预训练模型，加不同的头即可。

## 六、公式详解

### MLM 损失

$$\mathcal{L}_{\text{MLM}} = -\sum_{i \in \mathcal{M}} \log P(w_i | \mathbf{h}_i)$$

$$P(w_i | \mathbf{h}_i) = \text{SoftMax}(\mathbf{h}_i\mathbf{W}_{\text{MLM}} + \mathbf{b})_{w_i}$$

- $\mathcal{M}$: 被 mask 的位置集合（15% 的 token）
- $\mathbf{h}_i$: Transformer encoder 在第 i 个位置的输出
- $\mathbf{W}_{\text{MLM}}$: 分类权重矩阵（形状 D × |V|，与 Token Embedding 共享权重）
- $P(w_i|\mathbf{h}_i)$: 对第 i 个位置预测原词 $w_i$ 的概率

**权重共享（Weight Tying）**：$\mathbf{W}_{\text{MLM}}$ 和 Token Embedding 矩阵是同一个——Press & Wolf (2017) 的发现。这减少了参数量，同时强制 MLM 分类头和 embedding 层学到一致的表征。

### NSP 损失

$$\mathcal{L}_{\text{NSP}} = -\log P(\text{IsNext} | \mathbf{h}_{[\text{CLS}]}) + \log P(\text{NotNext} | \mathbf{h}_{[\text{CLS}]})$$

- $\mathbf{h}_{[\text{CLS}]}$: [CLS] token 的输出，形状 D
- $P(\text{IsNext})$: 二分类概率

### 联合训练

$$\mathcal{L} = \mathcal{L}_{\text{MLM}} + \mathcal{L}_{\text{NSP}}$$

两个损失联合优化，没有加权系数（等权）。

### BERT vs GPT 的参数效率

| | BERT-Base | GPT (Radford et al., 2018) |
|--|----------|---------------------------|
| 参数量 | 110M | 117M |
| 预训练数据 | 3.3B words | ~1B words |
| 下游任务提升 | +7.7% (GLUE avg) | +4.2% |
| 训练成本 | 4 TPU × 4 天 | ~1 月 on 8 GPU |

同等参数量下，BERT 的双向预训练比 GPT 的单向预训练数据效率高出近一倍。

## 七、实验结果

### GLUE Benchmark（Table 1）

| 模型 | MNLI | QQP | QNLI | SST-2 | CoLA | STS-B | MRPC | RTE | Avg |
|------|------|-----|------|-------|------|-------|------|-----|-----|

BERT-Large 在 GLUE 上达到 82.1% 平均，比当时 SOTA 高出 7.7 个百分点。这在 NLP 领域是革命性的——一次预训练，横扫 11 个任务。

BERT-Large 在 SQuAD 1.1 上 F1 达到 90.9，比此前 SOTA 高出 5 个百分点。

## 八、代码对照（GitHub: google-research/bert）

官方实现是 TensorFlow。几个关键实现细节：

**1. Mask 策略的实现。** `create_masked_lm_predictions` 函数在预处理阶段就将 15% 的 token 替换为 [MASK]/随机词/原词。这个操作在数据 pipeline 中完成，不是在模型 forward 时——意味着每个 epoch 的 mask 模式不同（先 shuffle 再 mask），增加了训练数据的多样性。

**2. 预训练序列长度。** BERT 使用 512 的最大序列长度，但有 90% 的序列用 128 长度，只有 10% 用 512。这极大地提高了训练效率——128 长度的平方复杂度远低于 512。最后 10% 的训练步骤用 512 长度来修正位置编码。

**3. AdamW 优化器。** 使用带权重衰减修正的 Adam（不是标准 Adam）。BERT 的代码实现了一种叫做 "AdamWeightDecay" 的优化器，对权重矩阵做 L2 惩罚，但对 LayerNorm bias 和 embedding 做权重衰减。

**4. Warmup + Linear Decay。** 10k steps 线性 warmup，然后线性衰减到 0。BERT-Large 训练 1M steps（约 40 epoch）。

## 九、位置

### 前驱工作

- **[Vaswani et al., 2017] Attention Is All You Need**：BERT 的基础架构。注意力机制 + Transformer encoder 直接复用。
- **[Peters et al., 2018] ELMo**：第一个证明深度双向预训练有效的工作，但双向是 LSTM 方向的拼接而非真正交互。
- **[Radford et al., 2018] GPT**：第一个用 Transformer 做预训练的工作，单向语言模型。BERT 在此基础上把 decoder 换成了 encoder，把因果掩码改成了双向 attention。

### 同期竞品

- **[Liu et al., 2019] RoBERTa**：Facebook 对 BERT 的改进，训练更久、数据更多、去掉 NSP、动态 mask。证明 BERT 本身足够好，只是训练不足。
- **[Yang et al., 2019] XLNet**：用排列语言模型（permutation language model）实现双向预测，结合了自回归和自编码的优点。

### 后续影响

BERT 是 NLP 领域最具影响力的论文之一，它的影响远远超出了论文本身：

- **RoBERTa、ALBERT、DistilBERT、ELECTRA、SpanBERT** 等工作直接继承 BERT 的架构和预训练范式。
- BERT 的 [CLS] token + fine-tune 机制被 ViT、DeiT、BEiT 等视觉 Transformer 完全复制。
- BERT 的 MLM 预训练启发了视觉领域的 **BEiT（Masked Image Modeling）** 和 **MAE**。
- BERT 的"预训练 → fine-tune"范式在 GPT-3/4 时代被"预训练 → prompt → in-context learning"部分替代，但 BERT 作为 encoder-only 模型在需要深度理解的场景（分类、NER、搜索）中仍广泛使用。

## 十、局限

1. **MLM 预训练存在 mismatch。** Fine-tune 时没有 [MASK] token，但预训练中模型大量依赖 [MASK] 做预测。虽然 80/10/10 策略缓解了这个问题，但 mismatch 仍然存在。ELECTRA（Clark et al., 2020）用 replaced token detection 彻底解决了这个问题。

2. **NSP 任务太简单。** RoBERTa 证明去掉 NSP 不影响甚至提升性能。NSP 的"句子对"预测任务可以用主题分类完成——两个随机句子通常来自不同的文档，模型只需要判断文档是否相同，不需要真正理解句子关系。

3. **预训练成本高。** BERT-Large 在 4 块 TPU 上训练 4 天（约 16 TPU-days）。虽然当时算力足够，但对学术实验室是门槛。

4. **最大序列长度 512。** 受限于 Transformer 的 $O(N^2)$ 复杂度，BERT 无法处理更长的序列（如整篇论文、完整对话）。后续的 Longformer、BigBird 解决了这个问题。

5. **静态的预训练/微调范式。** 每个下游任务都需要单独 fine-tune，保存单独的模型副本。GPT-3 的 in-context learning 展示了更灵活的方式。

## 十一、小结

BERT 的贡献可以概括为：**把预训练从单向语言模型变成了真正的双向上下文模型。**

它没有发明新的架构（Transformer encoder 是已有的），没有发明新的目标函数（MLM 也是已有的想法），但它做了三个极其关键的工程决策：

1. 双向 Transformer encoder + MLM ← 把"用上下文理解词"这件事做到了极致
2. 80/10/10 mask 策略 ← 用巧妙的设计解决了预训练/微调的不匹配
3. 统一 fine-tune 接口 ← 让预训练模型可以直接应用于任意下游任务

BERT 之后，"预训练 + fine-tune"成了 NLP 的标准范式，直到 GPT-3 出现。即便在 LLM 时代，BERT 仍然是 encoder-only 路线的代表——在需要深度理解（而不只是生成）的场景中，Transformer encoder 仍然是许多实际系统的最佳选择。
