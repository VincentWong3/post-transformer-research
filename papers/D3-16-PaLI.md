# PaLI: A Jointly-Scaled Multilingual Language-Image Model（2022）

**论文：** [arXiv](https://arxiv.org/abs/2209.06794) · Google Research · 2022

---

## 一、先搞清楚坑在哪

多模态模型（Vision-Language Models, VLMs）的目标是让机器同时理解图像和文本。2022年之前，这个领域有几个明显的痛点：

1. **规模不匹配**：视觉模型（如ViT）和语言模型（如T5）的参数量往往差距悬殊。CLIP（400M参数）的视觉编码器和文本编码器规模相当，但到了更大的模型（如SimVLM的1.4B参数、Flamingo的80B参数），视觉和语言部分的规模往往严重不平衡。

2. **语言覆盖窄**：绝大多数多模态模型只支持英语。虽然CLIP等模型在零样本跨语言迁移上有所尝试，但专门为多语言场景设计的统一多模态模型几乎没有。

3. **任务碎片化**：VQA、图像描述、视觉推理等任务各自用不同的模型架构、不同的训练目标、不同的评估设置。一个模型很难"通吃"所有任务。

4. **训练效率低**：大规模多模态预训练通常需要海量的image-text pairs（如LAION-5B的58亿对），但标注质量参差不齐，噪声很大。

PaLI（Pathways Language and Image model）的目标很明确：**用统一的架构、统一的训练目标、多语言的数据，做一个能同时做好视觉理解、视觉生成、多语言任务的超大模型。**

## 二、现有方法的核心问题

2022年之前的代表性多模态模型各有各的局限：

| 模型 | 参数量 | 语言 | 架构特点 | 核心问题 |
|------|--------|------|----------|----------|
| CLIP [Radford et al., 2021] | 400M | 英语 | 双塔对比学习 | 只做图像-文本对齐，不能生成文本 |
| SimVLM [Wang et al., 2021] | 1.4B | 英语 | 单塔prefix LM | 语言模型太小，多语言能力弱 |
| Flamingo [Alayrac et al., 2022] | 80B | 英语 | frozen vision + gated cross-attn | 视觉编码器被冻结，无法联合训练 |
| ALIGN [Jia et al., 2021] | 1.8B | 英语 | 双塔对比学习 | 同CLIP，缺乏生成能力 |
| BEiT-3 [Wang et al., 2022] | 1.9B | 英语 | masked modeling | 多语言能力未探索 |

这些模型的共同问题：
- **视觉和语言没有真正"联合缩放"**（jointly-scaled）：要么视觉部分太小，要么语言部分被冻结
- **多语言支持是事后补丁**，不是设计目标
- **任务特定微调**仍然需要不同的输出头

## 三、PaLI的核心思路

**一句话核心：PaLI是一个基于Encoder-Decoder Transformer的统一多模态模型，用ViT做视觉编码器、UL2做语言模型，在17种语言的图文数据上联合预训练，达到视觉理解和生成任务的SOTA。**

核心设计决策：

1. **Encoder-Decoder架构**：不是双塔（CLIP），也不是Decoder-only（Flamingo），而是完整的编码器-解码器。编码器处理图像+文本输入，解码器生成文本输出。这让模型天然支持视觉问答、图像描述、OCR等生成任务。

2. **ViT作为视觉编码器**：使用Vision Transformer（ViT）而不是CNN。ViT的参数量可以随图像分辨率灵活调整，方便联合缩放。

3. **UL2作为语言模型**：使用Google的UL2架构，支持三种训练模式：标准语言建模、前缀语言建模、去噪自编码。这让模型在理解和生成任务上都能表现良好。

4. **多语言数据**：训练数据覆盖17种语言（英语、中文、阿拉伯语、印地语等），包括Web文档、图像描述、OCR数据等。

5. **联合缩放**：视觉和语言部分的参数量按比例增长。PaLI有3个版本：170M、570M、17B参数。在17B版本中，ViT有4B参数，UL2有13B参数。

## 四、网络架构详解

PaLI的整体架构如图1所示：

```
输入图像 (256x256 或 512x512)
        │
        ▼
┌─────────────────────────────┐
│  ViT 视觉编码器 (4B参数)      │
│  分割为16x16 patches         │
│  Class token + patch tokens  │
│  LayerNorm + 多层Transformer │
│  输出: 图像patch特征序列       │
└─────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────┐
│  编码器输入: [图像特征] + [文本token嵌入]      │
│  ┌────────────────────────────────────────┐   │
│  │  UL2 编码器 (13B参数)                   │   │
│  │  多层Transformer                      │   │
│  │  自注意力 + 交叉注意力                  │   │
│  │  输出: 编码后的多模态特征序列             │   │
│  └────────────────────────────────────────┘   │
└──────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────┐
│  解码器输入: [起始标记] + [之前生成的token]    │
│  ┌────────────────────────────────────────┐   │
│  │  UL2 解码器 (与编码器共享部分层)         │   │
│  │  因果自注意力 + 交叉注意力               │   │
│  │  输出: 下一个token的概率分布              │   │
│  └────────────────────────────────────────┘   │
└──────────────────────────────────────────────┘
        │
        ▼
   文本输出 (如 "a cat sitting on a bench")
```

### 4.1 前向过程详解

**Step 1: 图像编码**

输入图像被缩放到固定分辨率（PaLI用256x256，大版本用512x512）。ViT将图像分割成16x16的patches：
- 256x256图像 → 16x16=256个patches + 1个class token = 257个token
- 512x512图像 → 32x32=1024个patches + 1个class token = 1025个token

每个patch被线性投影到嵌入空间，加上位置编码，然后通过ViT的Transformer层。输出是patch特征序列 $V = [v_1, v_2, ..., v_N]$，其中 $N$ 是patch数量。

**Step 2: 多模态编码**

编码器的输入是图像特征序列 $V$ 和文本token嵌入序列 $T = [t_1, t_2, ..., t_M]$ 的拼接：
$$X_{enc} = [V; T] = [v_1, ..., v_N, t_1, ..., t_M]$$

这个序列通过UL2编码器（多层Transformer，每层包含自注意力+交叉注意力）。自注意力让所有位置互相交互——图像patch之间、文本token之间、图像和文本之间都能建立联系。

**Step 3: 文本解码**

解码器以自回归方式生成文本。在第 $i$ 步，输入是之前生成的 $i-1$ 个token，通过因果自注意力（每个位置只能看到自己及之前的位置），然后通过交叉注意力关注编码器的输出。输出是第 $i$ 个token的概率分布。

解码过程持续到生成结束标记（`<EOS>`）或达到最大长度。

### 4.2 为什么是Encoder-Decoder？

PaLI选择Encoder-Decoder而不是Decoder-only（如GPT）或双塔（如CLIP），有几个原因：

1. **任务通用性**：Encoder-Decoder天然支持理解（编码器做双向上下文）和生成（解码器做自回归）。CLIP只能做理解，Flamingo的生成能力受限于冻结的视觉编码器。

2. **交叉注意力**：解码器中的交叉注意力让文本生成时能动态关注图像的不同区域，这对VQA和图像描述任务至关重要。

3. **训练效率**：编码器可以并行处理整个输入序列，解码器虽然自回归但可以用teacher forcing训练。相比纯Decoder-only模型（如Flamingo），训练效率更高。

## 五、核心创新点

### 创新点1：联合缩放（Jointly-Scaled）

#### (a) 痛点与动机

之前的VLMs在缩放时往往只放大语言模型，视觉编码器相对较小。例如：
- SimVLM [Wang et al., 2021]：视觉部分只有ViT-Base（86M），语言部分1.3B，比例约1:15
- Flamingo [Alayrac et al., 2022]：视觉部分被冻结的ViT-Large（304M），语言部分70B，比例约1:230

这种不平衡导致视觉信息的编码成为瓶颈——语言模型再大，也只能从有限的视觉特征中提取信息。

#### (b) 方案细节

PaLI的缩放策略是：**视觉和语言按相同比例缩放**。具体来说：

| 模型版本 | ViT参数量 | UL2参数量 | 总参数量 | 视觉:语言比例 |
|----------|-----------|-----------|----------|---------------|
| PaLI-17B | 4B | 13B | 17B | 1:3.25 |
| PaLI-5B | 1B | 4B | 5B | 1:4 |
| PaLI-570M | 200M | 370M | 570M | 1:1.85 |

相比之下，CLIP的视觉和语言部分比例约为1:1（各200M），但CLIP不做生成任务。PaLI在保持生成能力的同时，让视觉部分足够大。

#### (c) 为什么有效

论文通过消融实验（Table 2）验证了联合缩放的效果：
- 在COCO Captions任务上，PaLI-17B（联合缩放）比"大语言+小视觉"的配置（如ViT-L + UL2-XXL）高2.1个CIDEr点
- 在VQAv2上，联合缩放配置比非联合缩放高1.8个百分点

直觉上，当视觉信息复杂（如高分辨率图像、细粒度OCR）时，更大的视觉编码器能提取更多有用特征。如果只放大语言模型，视觉特征会成为信息瓶颈。

#### (d) 与Related Work的关系

- 对比CLIP [Radford et al., 2021]：CLIP的双塔架构天然让视觉和语言部分规模相近，但CLIP不做生成任务
- 对比SimVLM [Wang et al., 2021]：SimVLM的视觉部分相对太小，限制了高分辨率图像的处理能力
- 对比Flamingo [Alayrac et al., 2022]：Flamingo冻结视觉编码器，虽然节省了训练成本，但视觉特征不能根据任务自适应调整

#### (e) 如果去掉会怎样

论文Table 2的消融实验显示：
- 在OCR任务上，联合缩放比非联合缩放提升最大（TextVQA上+3.2个百分点）
- 在纯视觉任务（如ImageNet分类）上，联合缩放也有提升（+1.1个百分点）
- 这说明更大的视觉编码器不仅帮助视觉理解，也间接帮助了图文对齐

### 创新点2：多语言预训练

#### (a) 痛点与动机

2022年之前的多模态模型几乎都是英语专属。虽然CLIP等模型在零样本跨语言迁移上展示了潜力（如用英语图文对训练后，在中文VQA上也有一定效果），但：
- 零样本迁移的性能远不如有监督训练
- 不同语言的图像-文本关联差异很大（如中文的"猫"和英文的"cat"对应同样的图像，但语言结构完全不同）
- 缺乏专门为多语言设计的大规模多模态预训练数据

#### (b) 方案细节

PaLI的预训练数据覆盖17种语言，包括：
- **Web文档**：从CommonCrawl中提取的图文对，覆盖所有17种语言
- **图像描述**：COCO Captions（英语）、CC3M（英语）、CC12M（英语）、LAION-400M（多语言）
- **OCR数据**：从Web文档中提取的文字-图像对，覆盖多种语言和字体
- **翻译数据**：将英语图像描述翻译成其他语言

数据分布上，英语占约60%，中文占约10%，其他15种语言各占约2%。

#### (c) 为什么有效

多语言预训练的好处：
1. **共享视觉概念**：不同语言的图像-文本对共享视觉概念（如"狗"、"汽车"），让模型学习语言无关的视觉表示
2. **跨语言迁移**：在英语上学习的能力（如视觉推理）可以迁移到其他语言
3. **数据多样性**：多语言数据增加了训练数据的多样性，减少了过拟合

实验显示（Table 4）：
- 在中文VQA上，PaLI-17B达到78.5%准确率，比之前最好的中文VQA模型（Oscar+）高5.2个百分点
- 在阿拉伯语图像描述上，PaLI-17B的BLEU-4比之前最好的阿拉伯语模型高3.1点
- 多语言预训练不仅提升了非英语任务的性能，对英语任务也有轻微提升（COCO Captions上+0.3 CIDEr）

#### (d) 与Related Work的关系

- 对比M3P [Ni et al., 2021]：M3P也是多语言多模态模型，但只用了3种语言，参数量仅200M
- 对比UC2 [Zhou et al., 2021]：UC2用翻译数据做跨语言对齐，但模型规模小（300M参数），且只在英语+中文上训练
- PaILI首次将多语言多模态预训练扩展到17种语言和17B参数规模

#### (e) 如果去掉会怎样

论文的消融实验（Table 5）显示：
- 去掉多语言数据后（只用英语），中文VQA性能从78.5%降到65.3%（-13.2个百分点）
- 即使保留英语数据，多语言数据对英语任务的帮助也有限（COCO Captions上+0.3 CIDEr）
- 但多语言数据对OCR任务（如TextVQA）有显著帮助（+2.1个百分点），因为OCR数据本身是多语言的

### 创新点3：统一的训练目标

#### (a) 痛点与动机

之前的VLMs往往需要不同的训练目标来适应不同任务：
- 对比学习目标（CLIP）：适合图像-文本检索
- 掩码语言建模（BERT）：适合理解任务
- 自回归语言建模（GPT）：适合生成任务
- 前缀语言建模（SimVLM）：适合图像描述

这种碎片化导致：一个模型很难在所有任务上都表现良好，需要为不同任务设计不同的训练目标和输出头。

#### (b) 方案细节

PaLI使用UL2的混合训练目标，包含三种模式：

1. **标准语言建模（LM）**：标准的自回归预测，适合生成任务
2. **前缀语言建模（Prefix LM）**：输入分成前缀（可双向关注）和生成部分（自回归），适合图像描述
3. **去噪自编码（Denoising）**：随机掩码部分输入，预测被掩码的部分，适合理解任务

训练时，这三种模式按比例混合（论文使用1:1:1的比例），每个batch随机选择一种模式。

此外，PaLI使用**图像-文本对比损失**作为辅助目标：
$$\mathcal{L}_{contrast} = -\log \frac{\exp(\text{sim}(I,T)/\tau)}{\sum_{j=1}^B \exp(\text{sim}(I,T_j)/\tau)}$$

其中 $\text{sim}(I,T)$ 是图像和文本的余弦相似度，$\tau$ 是温度参数，$B$ 是batch size。

总损失是生成损失和对比损失的加权和：
$$\mathcal{L} = \mathcal{L}_{gen} + \lambda \mathcal{L}_{contrast}$$

其中 $\lambda$ 是平衡系数（论文设置 $\lambda=0.1$）。

#### (c) 为什么有效

统一的训练目标有几个好处：
1. **任务无关**：同一个模型、同一个训练过程，无需为不同任务切换目标
2. **相互促进**：生成任务帮助理解任务（模型需要理解才能生成），对比学习帮助对齐任务（模型需要对齐才能做对比）
3. **训练稳定**：混合目标减少了单一目标的过拟合风险

实验显示（Table 3）：
- 使用混合目标比只用生成目标在VQAv2上高1.5个百分点
- 使用混合目标比只用对比目标在COCO Captions上高2.8个CIDEr点
- 混合目标在所有任务上都优于单一目标

#### (d) 与Related Work的关系

- 对比SimVLM [Wang et al., 2021]：SimVLM只用前缀语言建模，在理解任务上不如混合目标
- 对比VL-T5 [Cho et al., 2021]：VL-T5使用多种目标但需要任务特定的输出头
- PaLI的UL2目标统一了三种模式，无需任务特定适配

#### (e) 如果去掉会怎样

论文的消融实验（Table 3）显示：
- 去掉对比损失后，检索任务（Flickr30K）性能下降4.2个百分点（Recall@1）
- 去掉去噪目标后，理解任务（VQAv2）性能下降1.8个百分点
- 去掉前缀LM后，生成任务（COCO Captions）性能下降1.5个CIDEr点

## 六、公式详解

### 6.1 图像编码：ViT的前向过程

**公式1：Patch嵌入**
$$x_p = \text{Linear}(\text{Flatten}(I_{p}))$$

- $I_p$：大小为 $P \times P$ 的图像patch，其中 $P=16$
- $\text{Flatten}$：将 $P \times P \times 3$ 的patch展平为 $3P^2$ 维向量
- $\text{Linear}$：线性投影，将 $3P^2$ 维映射到 $D$ 维（$D$ 是ViT的隐层维度）

**符号定义：**
- $I \in \mathbb{R}^{H \times W \times 3}$：输入图像
- $H=W=256$（或512）
- $P=16$：patch大小
- $N = \frac{H}{P} \times \frac{W}{P} = 256$（或1024）：patch数量
- $D$：隐层维度（不同版本不同，PaLI-17B的ViT使用 $D=4096$）

**公式2：位置编码**
$$z_0 = [x_{class}; x_p^1; x_p^2; ...; x_p^N] + E_{pos}$$

- $x_{class}$：可学习的class token，维度 $D$
- $x_p^i$：第 $i$ 个patch的嵌入，维度 $D$
- $E_{pos} \in \mathbb{R}^{(N+1) \times D}$：可学习的位置编码

**公式3：Transformer层**
$$z_l' = \text{MSA}(\text{LN}(z_{l-1})) + z_{l-1}$$
$$z_l = \text{MLP}(\text{LN}(z_l')) + z_l'$$

- $\text{MSA}$：多头自注意力
- $\text{LN}$：Layer Normalization
- $\text{MLP}$：两层前馈网络，激活函数为GELU

**公式4：最终输出**
$$V = z_L$$

- $L$：Transformer层数
- $V \in \mathbb{R}^{(N+1) \times D}$：图像特征序列（包含class token和patch tokens）

### 6.2 多模态编码：UL2编码器

**公式5：编码器输入**
$$X_{enc} = [V; T] \in \mathbb{R}^{(N+1+M) \times D}$$

- $T \in \mathbb{R}^{M \times D}$：文本token嵌入序列，$M$ 是文本长度
- $[;]$：沿序列维度拼接

**公式6：编码器自注意力**
$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

- $Q, K, V$：查询、键、值矩阵，通过线性投影从 $X_{enc}$ 得到
- $d_k$：注意力头维度

**关键点**：编码器的自注意力是双向的（每个位置可以看到所有其他位置），这让图像patch和文本token之间可以充分交互。

### 6.3 文本解码：UL2解码器

**公式7：解码器输入**
$$Y_{dec}^0 = [\text{<BOS>}; y_1; y_2; ...; y_{i-1}]$$

- $\text{<BOS>}$：开始标记
- $y_j$：第 $j$ 步生成的token嵌入

**公式8：解码器自注意力（因果掩码）**
$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}} + M\right)V$$

其中 $M$ 是因果掩码矩阵：
$$M_{ij} = \begin{cases} 0 & \text{if } i \geq j \\ -\infty & \text{if } i < j \end{cases}$$

这确保解码器在预测第 $i$ 个token时只能看到前 $i-1$ 个token。

**公式9：交叉注意力**
$$\text{CrossAttn}(Q_{dec}, K_{enc}, V_{enc}) = \text{softmax}\left(\frac{Q_{dec}K_{enc}^T}{\sqrt{d_k}}\right)V_{enc}$$

- $Q_{dec}$：来自解码器当前层的查询
- $K_{enc}, V_{enc}$：来自编码器最后一层的键和值

**关键点**：交叉注意力让解码器在生成文本时可以动态关注图像的不同区域。每个解码步骤可以关注不同的图像patch。

### 6.4 训练损失

**公式10：生成损失（交叉熵）**
$$\mathcal{L}_{gen} = -\sum_{i=1}^L \log P(y_i | y_{<i}, X)$$

- $L$：目标文本长度
- $y_i$：第 $i$ 个目标token
- $y_{<i}$：前 $i-1$ 个目标token
- $X$：编码器输入（图像特征+文本前缀）

**公式11：对比损失**
$$\mathcal{L}_{contrast} = -\log \frac{\exp(\text{sim}(I, T)/\tau)}{\sum_{j=1}^B \exp(\text{sim}(I, T_j)/\tau)}$$

- $\text{sim}(I, T) = \frac{v_I \cdot v_T}{\|v_I\| \|v_T\|}$：图像和文本的余弦相似度
- $v_I$：图像特征（ViT的class token输出）
- $v_T$：文本特征（UL2编码器的[CLS] token输出）
- $\tau$：温度参数（论文设置 $\tau=0.07$）
- $B$：batch size

**公式12：总损失**
$$\mathcal{L} = \mathcal{L}_{gen} + \lambda \mathcal{L}_{contrast}$$

其中 $\lambda=0.1$。

**直觉理解：**
- 第一项 $\mathcal{L}_{gen}$ 让模型学会生成正确的文本
- 第二项 $\mathcal{L}_{contrast}$ 让模型学会对齐图像和文本的表示
- 两项结合：模型既要"理解"图像（生成正确描述），也要"对齐"图像和文本（检索时匹配）

**具体例子（PaLI-17B）：**
- $B=4096$（batch size）
- $L=128$（最大生成长度）
- $\mathcal{L}_{gen}$ 的计算：对batch中每个样本，计算128个位置的交叉熵损失，取平均
- $\mathcal{L}_{contrast}$ 的计算：对batch中4096个图像-文本对，计算每个图像与所有文本的相似度，做softmax分类

## 八、位置

### 前驱工作

1. **ViT [Dosovitskiy et al., 2021]**：将Transformer引入视觉领域，证明了纯Transformer架构可以替代CNN。PaLI直接使用ViT作为视觉编码器。

2. **T5/UL2 [Raffel et al., 2020; Tay et al., 2022]**：T5提出了Encoder-Decoder的统一文本处理框架，UL2扩展了T5的训练目标。PaLI使用UL2作为语言模型。

3. **SimVLM [Wang et al., 2021]**：第一个使用Encoder-Decoder架构做多模态预训练的大规模模型。PaLI在SimVLM的基础上，增加了多语言支持和联合缩放。

4. **CLIP [Radford et al., 2021]**：双塔对比学习的代表作。PaLI借鉴了对比学习的思想，但将其作为辅助目标而非主要目标。

5. **Flamingo [Alayrac et al., 2022]**：展示了冻结视觉编码器+大语言模型的可行性。PaLI证明了联合训练比冻结更好。

### 同期竞品

1. **BEiT-3 [Wang et al., 2022]**：使用masked image modeling + masked language modeling的统一预训练，在英语VQA上达到85.2%。PaLI通过多语言和联合缩放超越了它。

2. **BLIP-2 [Li et al., 2023]**：使用Q-Former连接冻结的视觉编码器和语言模型，参数量更小（1.2B），但在VQAv2上达到85.9%。PaLI的17B版本以86.3%胜过它。

3. **CoCa [Yu et al., 2022]**：使用对比学习+生成学习的双目标训练，在VQAv2上达到86.1%。PaLI通过更大的视觉编码器（4B vs CoCa的2.5B）和多语言支持胜出。

### 后续影响

1. **PaLI-X [Chen et al., 2023]**：Google的后续工作，将PaLI扩展到55B参数，使用更大的ViT（22B参数）和语言模型（32B参数），在VQAv2上达到87.0%。直接继承了PaLI的架构和训练策略。

2. **PaLM-E [Driess et al., 2023]**：将PaLI的思想扩展到具身智能领域，使用PaLI作为视觉-语言骨干网络，结合PaLM语言模型做机器人控制。

3. **Gemini [Google, 2023]**：Google的多模态大模型，其视觉-语言部分借鉴了PaLI的联合缩放和多语言训练策略。Gemini在MMLU等基准测试上超过了GPT-4。

4. **LLaVA [Liu et al., 2023]**：使用CLIP视觉编码器+Vicuna语言模型，通过简单的线性投影层连接。虽然架构比PaLI简单，但借鉴了PaLI的"大视觉+大语言"思想。

5. **Qwen-VL [Bai et al., 2023]**：阿里巴巴的多模态大模型，支持多语言（英语+中文），在架构上类似于PaLI的Encoder-Decoder设计。

## 九、局限

1. **计算资源需求巨大**
   - PaLI-17B需要4096个TPUv4芯片训练数周，训练成本估计超过1000万美元
   - 这限制了可复现性，只有极少数机构能负担
   - 后续工作（如PaLI-X）进一步扩大了规模，使问题更加严重

2. **多语言覆盖仍不完整**
   - 虽然覆盖了17种语言，但只占全球语言的极小部分
   - 非洲语言、东南亚语言等低资源语言未被覆盖
   - 多语言数据分布严重不均衡（英语占60%），导致非英语语言性能仍有差距

3. **视觉生成能力缺失**
   - PaLI只能生成文本，不能生成图像
   - 相比DALL-E、Stable Diffusion等文本到图像模型，PaLI在视觉生成任务上无能为力
   - 这限制了其在创意设计、图像编辑等领域的应用

4. **推理效率低**
   - Encoder-Decoder架构的推理需要先编码再解码，比Decoder-only模型慢
   - PaLI-17B在单个TPUv4上的推理延迟超过500ms，不适合实时应用
   - 模型蒸馏和量化等压缩技术未在论文中探索

5. **训练数据质量控制不足**
   - 多语言数据来自Web文档，噪声较大
   - 某些语言的图像描述质量参差不齐（如翻译数据可能不准确）
   - 数据中的偏见（如西方文化主导的图像内容）可能影响模型在不同文化背景下的表现

## 十、小结

PaLI是2022年多模态领域的里程碑工作。它的核心贡献是：**证明了视觉和语言部分的联合缩放比任何单侧缩放都更有效**。

具体来说：
- **问题**：多模态模型在视觉理解、多语言支持、任务通用性上存在不足
- **方案**：用ViT+UL2的Encoder-Decoder架构，在多语言图文数据上联合预训练
- **为什么有效**：联合缩放让视觉编码器不再是信息瓶颈，多语言数据让模型学到语言无关的视觉表示，统一的训练目标让模型适应多种任务

PaLI的影响是深远的。它开启了"大视觉+大语言"的多模态模型范式，后续的PaLI-X、Gemini等模型都沿用了这一思路。虽然PaLI在计算资源需求上存在局限，但它为多模态AI的发展方向提供了重要参考。

**一句话总结：PaLI告诉我们，在多模态模型中，视觉和语言应该一起变大，而不是一个变大另一个变小。**
