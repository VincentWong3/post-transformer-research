# CSWin Transformer: Cross-Shaped Windows for General Vision Backbone（2022）

**论文：** [arXiv:2107.00652](https://arxiv.org/abs/2107.00652) · Xiaoyi Dong, Jianmin Bao, Dongdong Chen, Weiming Zhang, Nenghai Yu, Lu Yuan, Dong Chen, Baining Guo · USTC & Microsoft Research Asia · CVPR 2022

**技术分支：** B2 — 视觉 backbone 层级化 / B2-5 — hierarchical backbone 范式

**代码：** [microsoft/CSWin-Transformer](https://github.com/microsoft/CSWin-Transformer)

---

## 一、问题在哪里

Swin Transformer 在 2021 年用 shifted window attention 证明了层级式局部 attention 可以成为通用视觉 backbone。很成功——COCO 上超 ResNet 一大截，AD E20K 上大幅领先，引用量迅速过万。

但 Swin 有一个结构性的妥协：**每个 Transformer block 的 attention 区域被限制在 7×7 的方形窗口内。** 窗口内的 token 只能和 49 个邻居交互。跨窗口的信息交换靠层间偏移一个一个窗口慢慢传播——下一层的窗口偏移 $(\lfloor7/2\rfloor,\lfloor7/2\rfloor)$ ，把 Layer $l$ 被分割在 4 个不同窗口的区域合并到 Layer $l+1$ 的同一个窗口里。

这个设计的代价是：**感受野的增长非常慢。**

Swin 的一个 block 内的 attention 区域只有 7×7=49 个 token。跨层传播后，每经过一对 W/SW block，等效覆盖区域增加 6 个 token： $7+6L$ ，其中 L 是 W/SW 的对数。Swin-T 的 Stage 3 有 6 个 block（3 对 W/SW），等效覆盖区域约 7+6×3=25 个 token——大约 400 像素。Swin-B 有 18 个 block 在 Stage 3（9 对），覆盖约 7+6×9=61 个 token——大约 976 像素。对于需要全局上下文理解的密集预测任务（全景分割中的实例关系建模、大物体检测的上下文推理），这个传播速度还是太慢。

CSWin 论文明确指出了这一点：

> "However, the receptive field is enlarged quite slowly and it requires stacking a great number of blocks to achieve global self-attention."

所以核心问题变成：**能不能在一个 block 内就给每个 token 更大的 attention 区域，同时保持 $O(N)$ 复杂度？**

## 二、旧路线的真正问题

### 2.1 Swin 的局部窗口 compromise

Swin 的 7×7 窗口 attention 可以在一个 block 内覆盖 49 个 token。这个范围对于建模局部纹理和不大的物体足够了，但对于需要长程依赖的场景不够。

更根本的问题是：**窗口大小 $M=7$ 在浅层和深层都应该一样吗？**

Stage 1 的特征图是 56×56=3136 个 token。一个 7×7 窗口占整个特征图面积的 $49/3136 \approx 1.56\%$ 。在这个深度上，局部优先可能更合理——CNN 的先验表明浅层特征主要提取边缘、纹理等局部模式。

但 Stage 3 的特征图是 14×14=196 个 token。一个 7×7 窗口占 $49/196 = 25\%$ ——窗口几乎覆盖了 1/4 的特征图。到 Stage 4 的 7×7 特征图，窗口覆盖 49/49=100%。Stage 4 的 Swin Block 退化为全局 attention。

这里的观察是：**同一窗口大小在不同深度上的实际覆盖范围是不同的。** 深层特征图分辨率低，7×7 已经接近全局；浅层特征图分辨率高，7×7 仍然是高度局部的。如果能在浅层用小窗口（节约计算），在深层用大窗口（充分建模全局），应该能找到更好的复杂度-效果平衡点。

### 2.2 Axial attention 的串行瓶颈

在 Swin 之前，已经有一些工作尝试用 stripe/axial attention。Axial attention（Ho et al., 2019）把二维 attention 分解为两个一维 attention——先算行方向的 attention，再算列方向的 attention。

这个方法的问题是：**行的 attention 和列的 attention 是串行的。** 必须先做完所有行 attention 才能开始列 attention。串行意味着计算时间翻倍。而且，行 attention 的输出没有和列 attention 的输入同时处理——列 attention 看到的是已经被行 attention 处理过的特征，这种级联可能丢失原始信息。

### 2.3 Criss-cross attention 的重叠浪费

Criss-cross attention（Huang et al., 2019）在一个 block 内同时做水平和垂直的 attention，但它在每个位置上都做密集的高和宽的 attention——实际计算量是 $H \times (W-1) + W \times (H-1)$ 个 query-key 对。问题是 criss-cross 的水平和垂直条带在中间位置重叠（中心点的行和列覆盖同一个地方），这种重叠区域在计算上有浪费。而且 criss-cross 需要对每个位置都做一次单独的 attention——无法 batch 化，硬件难以利用连续内存。

### 2.4 位置编码的迁移性问题

Swin 使用相对位置偏置（RPE），把位置信息注入到 attention 的 logits 中。RPE 是在 attention 内的，所以它只依赖于窗口大小。当输入分辨率变化时（检测任务经常做），窗口大小不变，RPE 可以正常工作。

但 RPE 有一个局限：**它用 per-head bias 来表达位置信息，对 V 没有直接的局部约束。** 这意味着 attention 可以学习到长程连接，但模型没有直接的归纳偏置来鼓励局部性——V 的内容没有收到位置信息的直接影响。

APE（绝对位置编码）的问题更明显：它是为固定分辨率设计的，无法迁移到不同输入尺寸。CPE（条件位置编码）通过卷积生成位置编码，可以适应不同输入尺寸，但它把位置信息加在 attention block 之前，不是加在每个 block 内部。

## 三、核心思路

**这篇论文的核心思路是：把 multi-head attention 的 heads 分成两组，一组做水平条带的 self-attention，一组做垂直条带的 self-attention，并行计算后拼接——这就是 Cross-Shaped Window（十字形窗口）self-attention。**

这个设计的关键洞察是：

1. **并行化条带 attention** — 把 K 个 head 分成两组（各 K/2 个），同时计算水平和垂直条带。和 axial attention 的串行不同，CSWin 是并行的，计算时间不翻倍。

2. **动态条带宽度** — 浅层用小宽度（sw=1,2），深层用大宽度（sw=7,7）。浅层特征图大（56×56），stripe width=1 意味着一行就是一个 stripe，计算量小。深层特征图小（14×14 甚至 7×7），stripe width=7 意味着整个高/宽就是一个 stripe，接近全局 attention。

3. **Multi-head grouping 天然放大 attention 区域** — Swin 的 K 个 head 做的事是一样的（都在 7×7 窗口内做 attention）。CSWin 的 K/2 个 head 做水平条带，K/2 个 head 做垂直条带——同一个 block 内，每个 token 可以同时看到整行和整列的信息。这比 Swin 的 7×7 窗口大得多。

4. **LePE（Locally-enhanced Positional Encoding）** — 直接在 V 上加可学习的局部位置 bias，而不是加在 attention logits 中。这 decouples 了位置编码和 attention 计算，在 V 层面直接增强局部归纳偏置。

整体架构沿用层级式设计（4-stage 金字塔），和 Swin 一样保持和 ResNet/FPN 的兼容性。

## 四、整体结构与 forward 流程

### 4.1 整体架构

CSWin Transformer 的架构和 Swin 类似——4 个 stage，逐级下采样，输出 4 级分辨率特征图。但有两个关键区别：

1. **Patch embedding 用重叠卷积** — 使用 7×7 卷积（stride 4）做 token embedding，而不是 ViT/Swin 的非重叠 patch partition。和 CvT 类似，重叠卷积能保留更多局部连续性信息。

2. **Stage 间下采样用 3×3 卷积** — 替代 Swin 的 Patch Merging 操作。3×3 卷积（stride 2）在每个 stage 之间减少 token 数量并加倍 channel。

```text
输入图像: H×W×3 (224×224×3)
    │
    ▼
[Conv Token Embedding]    7×7 conv, stride 4, C dim
    │                     输出: 56×56×C (C=64 for CSWin-T)
    ▼
┌────────────────────────────────────────────┐
│ Stage 1 (4×下采样, 56×56, C=64)           │
│  1个 CSWin Block (sw=1, heads=2)          │
│  条带: 水平(H) + 垂直(W) 并行              │
└────────────────┬───────────────────────────┘
    │
    ▼
[3×3 Conv, stride 2]   56×56→28×28, 64→128
    │
    ▼
┌────────────────────────────────────────────┐
│ Stage 2 (8×下采样, 28×28, 2C=128)         │
│  2个 CSWin Blocks (sw=2, heads=4)         │
└────────────────┬───────────────────────────┘
    │
    ▼
[3×3 Conv, stride 2]   28×28→14×14, 128→256
    │
    ▼
┌────────────────────────────────────────────┐
│ Stage 3 (16×下采样, 14×14, 4C=256)        │
│  21个 CSWin Blocks (sw=7, heads=8)        │
└────────────────┬───────────────────────────┘
    │
    ▼
[3×3 Conv, stride 2]   14×14→7×7, 256→512
    │
    ▼
┌────────────────────────────────────────────┐
│ Stage 4 (32×下采样, 7×7, 8C=512)          │
│  1个 CSWin Block (sw=7, heads=16)         │
└────────────────────────────────────────────┘
```

注意 Stage 3 有 21 个 block——这是 CSWin 的一个特点。浅层 block 少（Stage 1 只有 1 个），大部分计算集中在 Stage 3。这个设计的直觉：Stage 3（16× 下采样）的 14×14 分辨率平衡了语义信息和空间细节，是密集预测任务中最重要的特征层，值得投入最多的计算资源。

### 4.2 CSWin Block 内部结构

每个 CSWin Block 的结构和标准 Pre-LN Transformer Block 一致，但有两处替换：

```text
输入 x
    │
    ├─→ LN → [CSWin-Attention] ─→ + ─→ LN → MLP → + ─→ 输出
    │        │                              │
    │        └─ LePE (平行分支)              │
    └────────────────────────────────────────┘
```

CSWin-Attention 的内部：

```text
输入 x: [B, N, C] (N=H×W)
    │
    ├─→ QKV 投影: x → Q, K, V
    │
    ├─→ 前 K/2 个 head → 水平条带 attention
    │      将 x 切分为 H/sw 个水平条带
    │      每个条带: sw×W 个 token
    │      在每个条带内做标准 multi-head attention
    │
    ├─→ 后 K/2 个 head → 垂直条带 attention
    │      将 x 切分为 W/sw 个垂直条带
    │      每个条带: H×sw 个 token
    │      在每个条带内做标准 multi-head attention
    │
    └─→ Concat 两组 head 的输出 → 投影 W^O → 输出
```

### 4.3 LePE：位置编码在 V 上

LePE 是 CSWin 的一个独立模块，和 attention 并行计算：

```text
LePE:
    输入 V: [B, N, C] (经过投影的 values)
        │
        ├─→ 还原为 2D 特征图: [B, H, W, C]
        ├─→ 深度可分离卷积 (3×3, padding=1)
        ├─→ 每个 token i 的 LePE 输出 = DWConv(V)[i]
        │
        └─→ 和 attention 输出相加:
             output[i] = Σ_j (α_ij + β_ij) * v_j
             其中 β_ij 由 DWConv 计算，限制在 τ=3 的 Chebyshev 距离内
```

LePE 的关键设计：

1. **在 V 上操作，不是在 QK^T 上** — 这和 RPE 不同。RPE 加在 attention logits 中（`SoftMax(QK^T/√d + B)V`），影响 attention 权重。LePE 加在 V 上（`(α + β)V`），影响 value 本身。这相当于在 value 聚合时加入一个局部邻域的 bias。

2. **通过深度可分离卷积实现** — 3×3 DWConv 提供了一种轻量的局部特征融合方式。因为 DWConv 的卷积核是 per-channel 的，计算量很小。

3. **Chebyshev 距离截断** — 只保留距离 ≤ τ（τ=3）的局部交互。这确保了 LePE 只强化局部偏置，不会引入远处的噪声。

4. **自适应任意输入分辨率** — DWConv 是平移等变的，不会绑定到特定输入尺寸。

## 五、核心模块逐个解释

### 5.1 十字形窗口 Self-Attention（CSWin-Attention）

**问题：** 方形窗口（Swin）限制每个 token 只能和 49 个邻居交互。跨窗口交互靠层间偏移，传播慢。

**解决方案：** 在一个 block 内，让一半的 head 做水平条带 attention（每个 token 看到整行），另一半做垂直条带 attention（每个 token 看到整列）。

**具体工作方式：**

以 CSWin-T 的 Stage 1（56×56, heads=2）为例：
- 前 1 个 head（K/2=1）：做水平条带 attention。sw=1，意味着把 56×56 的特征图切分成 56 个 1×56 的水平条带。每个条带内有 56 个 token，在一个条带内做 self-attention。
- 后 1 个 head（K/2=1）：做垂直条带 attention。sw=1，切分成 56 个 56×1 的垂直条带。每个条带内有 56 个 token，在一个条带内做 self-attention。

每个 token 在 Stage 1 的水平 head 中看到 55 个同行 token，垂直 head 中看到 55 个同列 token。由于 sw=1 时水平和垂直条带只在自身位置重叠，实际覆盖 $56+56-1=111$ 个 token（含自身）。

到了 Stage 3（14×14, heads=8, sw=7）：
- 前 4 个 head：sw=7，把 14×14 切分成 2 个 7×14 的水平条带。每个条带内 98 个 token 之间做 attention。
- 后 4 个 head：sw=7，把 14×14 切分成 2 个 14×7 的垂直条带。每个条带内 98 个 token 之间做 attention。

每个 token 在水平 head 中看到全部 97 个同行 token，在垂直 head 中看到全部 97 个同列 token。水平和垂直条带在中间 $7\times7$ 区域重叠，所以实际覆盖 $98 + 98 - 49 = 147$ 个 token（含自身），占 196 个总 token 的 75%。

Stage 4（7×7, heads=16, sw=7）：
- 前 8 个 head：sw=7（等于高度 7），1 个条带=整张特征图（7×7=49）。本质上就是全局 attention。
- 后 8 个 head：同上。

Stage 4 的 CSWin-Attention 退化为全局 multi-head attention。

### 5.2 Multi-head Grouping：为什么分两组并行

CSWin 的 multi-head grouping 策略是它的核心创新之一。传统 vision transformer 的 multi-head 中，K 个 head 做的是完全相同的 attention 操作——都在同一个 attention 区域上。CSWin 的做法是让不同的 head 做不同类型的 attention。

**这样做的好处：**

1. **不增加计算量。** 每个 head 的 QKV 投影量不变（K 个 head 一共还是 K 个 Q, K, V）。只是水平组和垂直组的 attention 区域不同。总的 FLOPs 等于 K/2 个水平条带 + K/2 个垂直条带的和。

2. **自然覆盖横纵两个方向。** 水平条带让 token 看到同一行上的其他 token，垂直条带让 token 看到同一列上的其他 token。两者一起，每个 token 可以看到一个十字形的区域。

3. **避免了串行处理的延迟。** Axial attention 先做行再做列，总延迟是 time(Horizontal) + time(Vertical)。CSWin 是并行的，总延迟是 max(time(Horizontal), time(Vertical))。

**为什么可以这样做？**

论文的假设是：自然图像没有方向性偏置（natural images do not have directional bias）。猫的耳朵和旁边区域的交互，水平方向和垂直方向应该同样重要。所以把 heads 平均分成水平和垂直两组是合理的。

**并行化后的 attention 拼接：**

$$\text{CSWin-Attention}(X) = \text{Concat}(\text{head}_1, \dots, \text{head}_K)W^O$$

$$\text{head}_k = \begin{cases}
\text{H-Attention}_k(X) & k=1, \dots, K/2 \\
\text{V-Attention}_k(X) & k=K/2+1, \dots, K
\end{cases}$$

### 5.3 动态条带宽度（Dynamic Stripe Width）

这是 CSWin 的第二个关键设计。论文提出了一个直观的观察：**浅层特征图大，适合用小宽度；深层特征图小，适合用大宽度。**

数学解释：CSWin 的复杂度公式是：

$$\Omega(\text{CSWin}) = HWC \times (4C + sw \times H + sw \times W)$$

其中 sw 是条带宽度。当 sw=1 时，第二项很小（ $1 \times H + 1 \times W = H+W$ ）。当 sw=H 时，第二项变为 $H \times H + H \times W = H^2 + HW$ 。

在浅层 Stage 1（H=W=56, C=64），sw=1 时： $4C + 1 \times 56 + 1 \times 56 = 256 + 112 = 368$

在深层 Stage 3（H=W=14, C=256），sw=7 时： $4 \times 256 + 7 \times 14 + 7 \times 14 = 1024 + 98 + 98 = 1220$

注意 Stage 1 的 total FLOPs = $56 \times 56 \times 64 \times 368 \approx 73.8M$ （还包括 QKV 投影），而 sw=1 的 attention 部分 FLOPs 只有 $56 \times 56 \times 64 \times (1 \times 56 + 1 \times 56) \approx 22.5M$ 。

Stage 3 的 total FLOPs = $14 \times 14 \times 256 \times 1220 \approx 61.2M$ ，其中 attention 部分 $14 \times 14 \times 256 \times (7 \times 14 + 7 \times 14) \approx 9.8M$ 。

对比 Swin：Stage 1 的 W-MSA 第二项 $2M^2hwC = 2 \times 49 \times 3136 \times 96 \approx 29.5M$ ，比 CSWin 的 $22.5M$ 多了 31%。

CSWin 在浅层比 Swin 更高效——因为 stripe width=1 比 7×7 窗口更小。在深层，CSWin 用更大的 stripe width（sw=7）来实现更大的 attention 区域——比 Swin 的 7×7 窗口大得多。

### 5.4 LePE：Locally-Enhanced Positional Encoding

**核心想法：** 把位置编码作为 V 上的一个可学习 bias，而不是加在 attention logits 中。

**公式表达：**

标准 attention:
$$z_i = \sum_{j=1}^n \alpha_{ij} v_j, \quad \alpha_{ij} = \exp(q_i^T k_j / \sqrt{d})$$

加了 LePE 后:
$$z_i^k = \sum_{j=1}^n (\alpha_{ij}^k + \beta_{ij}^k) v_j^k$$

其中 $\beta_{ij}^k$ 是通过深度可分离卷积（3×3）从 V 计算得到的局部偏置。

**LePE 的关键特征：**

1. **Per-channel bias，不是 per-head bias** — RPE 对每个 head 有不同的偏置表（ $M^2 \times M^2 \times \text{nH}$ 个参数）。LePE 在每个 channel 上单独计算偏置，不增加 attention 的计算量。

2. **通过 DWConv 实现局部邻域** — 3×3 的深度可分离卷积提供了一种轻量的方式来表达"相邻 token 的 V 应该互相增强"这个先验。DWconv 的参数量极小： $3 \times 3 \times C$ ，和 feature channel 线性相关。

3. **Chebyshev 距离截断（τ=3）** — 只保留距离 ≤ 3 的局部交互。这确保了 LePE 只增强局部性，不会引入 distant noise。

4. **可迁移任意分辨率** — 和 RPE/APE 不同，LePE 不依赖于固定的输入尺寸。DWConv 是平移等变的，在不同的输入分辨率下可以自然工作。

Table 10 的消融验证了 LePE 在密集预测任务上的优势：相比 RPE，LePE 在 COCO 上 +1.2 box AP, +0.9 mask AP，在 ADE20K 上 +1.6 mIoU。

## 六、核心公式与数学直觉

### 6.1 复杂度公式：为什么 CSWin 比 Swin 更高效

**标准 global MSA:**
$$\Omega(\text{MSA}) = 4hwC^2 + 2(hw)^2C$$

**Swin W-MSA:**
$$\Omega(\text{W-MSA}) = 4hwC^2 + 2M^2hwC$$

**CSWin:**
$$\Omega(\text{CSWin}) = HWC \times (4C + sw \times H + sw \times W)$$

把 H=W=h 代入简化:
- Swin: $4h^2C^2 + 2M^2h^2C$
- CSWin: $4h^2C^2 + 2sw \times h^3C$

**当 sw=1 时**（Stage 1/2）:
CSWin: $4h^2C^2 + 2h^3C$
Swin: $4h^2C^2 + 2M^2h^2C$ （M=7, M²=49）

CSWin 的第二项是 $2h^3C$ ，Swin 的是 $2 \times 49 \times h^2C = 98h^2C$ 。当 h > 49 时，CSWin 更贵；当 h < 49 时，Swin 更贵。

Stage 1 h=56: CSWin 第二项 $2 \times 56^3C = 351,232C$ ，Swin 第二项 $98 \times 56^2C = 307,328C$ 。CSWin 略贵（但在实际中 CSWin 的 #heads 更小，所以通道维度也更小）。

**当 sw=7 时**（Stage 3/4）:
CSWin: $4h^2C^2 + 14h^3C$

Stage 3 h=14: CSWin 第二项 $14 \times 14^3C = 38,416C$ ，Swin 第二项 $98 \times 14^2C = 19,208C$

但 CSWin 的 sw=7 在 h=14 上意味着条带宽度等于特征图的整个高度（7×2=14），所以水平条带 attention 覆盖的是整个高度——和全局 attention 一样大。而 Swin 的 7×7 窗口覆盖 49 个 token——在 196 个 token 的特征图中只占 25%。CSWin 用更大的 attention 区域换来了更强的建模能力。

**更重要的对比是 attention 覆盖区域：**

| Stage | 分辨率 | Swin 覆盖 (token) | CSWin 覆盖 (token) | 倍数 |
|-------|--------|------------------|-------------------|------|
| 1 | 56×56 | 49 | 111 (56+56-1) | 2.3× |
| 2 | 28×28 | 49 | 108 (56+56-4) | 2.2× |
| 3 | 14×14 | 49 | 147 (98+98-49) | 3.0× |
| 4 | 7×7 | 49 | 全面 (全局) | — |

这个表格揭示了核心答案：**CSWin 用更简单的操作（按行/列分割而不是方形窗口分割）实现了更大的 attention 区域，尤其是在深层。**

### 6.2 LePE 的数学形式

LePE 的公式：

$$z_i^k = \sum_{j=1}^n (\alpha_{ij}^k + \beta_{ij}^k) v_j^k$$

其中 $\beta_{ij}^k = \theta_k(\Delta x, \Delta y)$ ，由 3×3 DWConv 计算。 $\theta_k$ 表示第 k 个 channel 的局部 kernel 权重。 $\beta_{ij}^k = 0$ 当 token i 和 j 的 Chebyshev 距离 > τ（τ=3）。

这个公式的直觉：
- 标准 attention 中， $v_j$ 只被 $\alpha_{ij}$ （内容相似度权重）缩放。
- LePE 增加了一个**依赖于空间位置**的恒等偏置 $\beta_{ij}$ ——邻近 tokens 的 V 始终有正的偏置，即使它们的 Q 和 K 内容不相似。
- 这给模型注入了很强的局部连续性先验：相邻像素的 value 应该彼此接近。

### 6.3 CSWin Block 计算图

$$\hat{\mathbf{z}}^l = \text{CSWin-Attention}(\text{LN}(\mathbf{z}^{l-1})) + \text{LePE}(V) + \mathbf{z}^{l-1}$$
$$\mathbf{z}^l = \text{MLP}(\text{LN}(\hat{\mathbf{z}}^l)) + \hat{\mathbf{z}}^l$$

和标准 Pre-LN Transformer Block 的区别：
1. MSA 被替换为 CSWin-Attention（水平 + 垂直条带并行）
2. LePE 作为平行分支和 attention 输出相加

## 七、重要图表解释

### 7.1 Figure 1 — 不同 Self-Attention 机制的对比

原图 Figure 1 是全文最重要的概念图。它对比了 6 种 self-attention 机制：

**(a) Full Attention (ViT):** 所有 head 在全部 token 上做 attention。蓝色方框覆盖整张特征图。复杂度 $O(N^2)$ 。

**(b) Window Attention (Swin):** 所有 head 在 7×7 窗口中做 attention。蓝色方框是 M×M 的方形。复杂度 $O(N)$ ，但覆盖区域小。

**(c) Shifted Window (Swin):** 相邻层交替规则划分和偏移划分，跨层实现窗口间交互。但每层每个 head 的 attention 区域仍然是 7×7。

**(d) Axial Attention:** 串行的行和列 attention（先行后列）。每个 head 在行或列上做 attention，但不是并行。

**(e) Axial Attention:** 同上，列方向。

**(f) CSWin Attention（本篇）：** 将 K 个 head 分为两组。前 K/2 个 head 做水平条带 attention（图上的蓝色横条覆盖整行宽度），后 K/2 个 head 做垂直条带 attention（图上的蓝色竖条覆盖整列）。并行计算后拼接。

CSWin 和 (d)(e) 的根本区别：CSWin 是**并行**的（水平组和垂直组同时算），而 axial attention 是**串行**的。CSWin 可以在一个 block 内完成横纵双向的 attention，axial attention 需要两个连续 block。

### 7.2 Figure 2 — 整体架构图

原图 Figure 2 分左右两部分。

**左半部分：整体架构。**
展示了从输入到 4 个 stage 的完整流程：
1. 输入 → 7×7 重叠卷积（stride 4）→ token embedding
2. Stage 1: 1 个 CSWin Block（sw=1, heads=2）
3. 3×3 conv stride 2 下采样
4. Stage 2: 2 个 CSWin Blocks（sw=2, heads=4）
5. 3×3 conv stride 2 下采样
6. Stage 3: 多个 CSWin Blocks（sw=7, heads=8）— 论文中这个图中 N3=21 或 32（根据变体）
7. 3×3 conv stride 2 下采样
8. Stage 4: 1 或 2 个 CSWin Blocks（sw=7, heads=16 或 32）

注意 Stage 3 的 block 数量远多于其他 stage（21 或 32），这是一个设计特点。

**右半部分：CSWin Block 内部结构。**
- 标准 Pre-LN 结构
- CSWin-Attention 替换 MSA
- LePE 作为平行分支和 attention 输出相加
- 残差连接和 MLP

### 7.3 Figure 3 — 位置编码机制对比

原图比较了 4 种位置编码方案。

**APE（Absolute Position Encoding, ViT）:**
在 token embedding 后直接加 learnable 1D position embedding。位置信息在 attention block 之前注入。

**CPE（Conditional Position Encoding, Chu et al., 2021）:**
用 2D convolution 生成 position encoding，然后加到 token embedding 上。可以适应不同输入分辨率。

**RPE（Relative Position Encoding, Swin）:**
不在输入端加位置编码，而是在每个 attention block 的 `SoftMax(QK^T/√d)` 上加偏置 B。偏置表在窗口内按相对位置索引。

**LePE（本篇）:**
也是在每个 attention block 内注入，但不是在 QK^T 上加偏置，而是在 V 上加偏置——`output = Σ(α + β)V`。β 通过深度可分离卷积计算。

### 7.4 Figure 4 — 动态窗口宽度消融图

原图 Figure 4 是一个折线图。横轴是 stripe width（1 到 14），纵轴是 ImageNet Top-1 精度和 FLOPs。两条曲线：

1. 蓝色曲线（精度）：随 stripe width 增大而上升，但在 width≥7 后增速减缓。
2. 红色曲线（FLOPs）：随 stripe width 增大而线性增长。

默认设置 [1,2,7,7] 落在精度-FLOPs 曲线的拐点附近——宽度再增大，精度提升有限但 FLOPs 继续增长。这验证了动态宽度设计的合理性。

## 十、代码阅读

CSWin 的官方实现基于 Swin Transformer 的结构，核心代码约 400-500 行。关键代码路径：

### 10.1 CSWin-Attention 核心实现

```python
class CSWinAttention(nn.Module):
    def __init__(self, dim, num_heads, sw, ...):
        self.num_heads = num_heads
        self.qkv = nn.Linear(dim, dim * 3, bias=qkv_bias)
        self.proj = nn.Linear(dim, dim)
        self.relative_position_bias_table = nn.Parameter(...)
        
        # 预计算相对位置索引（和 Swin 类似但简化）
        self.register_buffer("relative_position_index", ...)
        
    def forward(self, x, mask=None):
        B_, N, C = x.shape
        qkv = self.qkv(x)  # [B_, N, 3C]
        qkv = qkv.reshape(B_, N, 3, self.num_heads, C // self.num_heads)
        qkv = qkv.permute(2, 0, 3, 1, 4)
        q, k, v = qkv[0], qkv[1], qkv[2]  # [B_, nH, N, d]
        
        # 分割 heads
        q_h, q_v = q.chunk(2, dim=1)  # 水平组和垂直组各占一半 heads
        k_h, k_v = k.chunk(2, dim=1)
        v_h, v_v = v.chunk(2, dim=1)
        
        # 水平条带 attention
        # 把 N = H×W 重排为水平条带
        attn_h = ...
        
        # 垂直条带 attention
        attn_v = ...
        
        # 拼接两组输出
        attn = torch.cat([attn_h, attn_v], dim=1)
        
        # LePE 应用在 V 上
        if self.lepe:
            v_lepe = self.dwconv(v.reshape(B_, H, W, C))
            attn = attn + v_lepe
        
        x = attn.transpose(1, 2).reshape(B_, N, C)
        x = self.proj(x)
        return x
```

### 10.2 水平条带 attention 的实现细节

水平条带（Horizontal Stripes）的实现：

1. 将 N×C 的特征还原为 H×W×C
2. 用 `view` 重排为 `[B_, H//sw, sw, W, C]`
3. permute 为每个条带：`[B_ * H//sw, sw*W, C]`
4. 在每个条带内做标准 attention

垂直条带类似，只是维度方向不同。

### 10.3 LePE 实现

LePE 在 CSWin-Attention 中通过 DWConv 实现：

```python
# 在 CSWinAttention.__init__ 中
self.dwconv = nn.Sequential(
    nn.Conv2d(dim, dim, kernel_size=3, stride=1, padding=1, 
              groups=dim, bias=True),  # depthwise conv
    nn.BatchNorm2d(dim),  # 注意：用 BN 不是 LN
)

# 在 forward 中
v = qkv[:, :, :self.num_heads, ...]  # value
v = v.transpose(1, 2).reshape(B, H, W, C)
v_pe = self.dwconv(v.permute(0, 3, 1, 2)).permute(0, 2, 3, 1)
v = v + v_pe  # LePE 直接在 V 上加偏置
```

关键细节：DWConv 的 `groups=dim` 确保每个 channel 独立处理自己的局部邻域，不跨 channel 混合——这正是"深度可分离"的含义。

### 10.4 整体模型结构

```python
class CSWinTransformer(nn.Module):
    def __init__(self, img_size=224, patch_size=4, in_chans=3, 
                 embed_dim=64, depths=[1,2,21,1], num_heads=[2,4,8,16],
                 sw_width=[1,2,7,7], ...):

        self.patch_embed = nn.Sequential(
            nn.Conv2d(3, embed_dim, kernel_size=7, stride=4, padding=3),
            nn.LayerNorm(embed_dim),
            nn.ReLU(inplace=True),

        self.down1 = nn.Conv2d(embed_dim, embed_dim*2, 3, 2, 1)
```

CSWin 在技术树中处于 Swin 之后的"局部 attention 改进"路线。和 Focal、Twins 等同期工作一起，都在设法解决 Swin 的"感受野增长慢"问题，但方案各不相同。

### 11.2 前驱工作

**Swin Transformer（Liu et al., 2021）：** CSWin 继承了层级结构和局部 attention 的基本思想。不同在于 attention 区域的形状（十字形 vs 方形）和跨窗口交互方式（同层并行 vs 层间偏移）。

**Axial Attention（Ho et al., 2019, Stand-Alone Self-Attention 变体）：** 最初提出将 2D attention 分解为行和列 1D attention 以减少复杂度。CSWin 改进了 Axial 的串行问题——用 multi-head grouping 实现了并行。

**Criss-Cross Attention（Huang et al., 2019, CCNet）：** 在语义分割中提出的交叉十字 attention，但在每个位置都做密集的横纵 attention，计算效率低（Table 8 显示 CSWin 比 CCNet 快 2-5 倍）。

**CvT（Wu et al., 2021）：** CSWin 采用了和 CvT 类似的重叠卷积 token embedding（7×7 conv stride 4）——这和 Swin 的 4×4 patch partition 不同。

**CPE（Chu et al., 2021）：** LePE 和 CPE 共享"位置编码由卷积生成"的思想，但 LePE 在每个 attention block 内操作 V 而不是在输入端操作 token embedding。

### 11.3 同期竞品

**Twins（Chu et al., 2021）：** 交替使用局部窗口 attention（Swin 风格）和全局 subsampled attention（PVT 风格）。attention 区域大于 Swin 但小于 CSWin 的十字形覆盖。

**Focal Transformer（Yang et al., 2021）：** 在窗口内做粗细粒度 attention——中心区域做细粒度（每个 pixel 一个 token），外围做粗粒度（pooled token）。用更精细的方式控制 attention 区域。

**MViT（Fan et al., 2021）：** 多尺度 Transformer，在不同分辨率上做 attention，pooling 操作同时降低分辨率并融合信息。

CSWin 和这些工作的最大区别：它不增加 attention 的复杂性（不需要 subsampling、pooling 或粗细粒度 tokenization），只是通过简单的 heads 分组来扩大 single-block attention 区域。

### 11.4 后续影响

CSWin Transformer 展示了"简单的大 attention 区域"思路的有效性。在这个方向上，后续的工作沿着几条路线发展：

1. **更大 attention 区域的研究** — 后来的工作进一步探索了如何更高效地实现近乎全局的 attention，如 MaxViT（Attention in attention block + local-global）和 Unified IO。

2. **条带 attention 的推广** — CSWin 的水平和垂直条带 attention 被后来的高效 attention 方法沿用，如使用轴向/条带的稀疏 attention 和 Mamba（状态空间模型）的方向性设计。

3. **LePE 的启发** — 直接在 V 上做位置编码的思路被后来的工作采用，如一些混合架构中使用卷积 + 自注意力的组合。

## 十三、范式意义

CSWin Transformer 的范式意义可以归纳为三个层面：

### 13.1 挑战了"方形窗口是局部 attention 的最佳形状"的默认假设

Swin 的 7×7 方形窗口在直觉上很合理——邻域通常理解为方形区域。CSWin 证明，**十字形（水平 + 垂直条带）的 attention 区域在同等复杂度下可以实现更大的覆盖范围**，而且在密集预测任务上效果更好。

这个洞察的方法论意义：视觉 Transformer 中很多设计选择（窗口形状、窗口大小、head 的使用方式）都是经验性的，并不一定是最优的。CSWin 通过重新思考"每个 head 应该做什么"这个一般性问题，找到了更好的设计。

### 13.2 "任务分化"的 head 设计哲学

CSWin 的 multi-head grouping 策略隐含了一个更一般的设计哲学：**multi-head 不一定所有 head 做相同的事，可以让不同的 head 关注不同的方面。**

后续的 EfficientFormer、MetaFormer 等架构都沿用了类似思路：让不同 head 负责不同的空间范围或特征类型。这个范式从 CSWin 开始得到了明确的验证。

### 13.3 位置编码的去耦合设计

LePE 把位置编码从 attention 计算中分离出来，直接作用在 V 上。这在数学上更简洁——attention 机制只需要处理内容相似度，局部性由专用的编码模块处理。这种解耦设计后来被很多工作采纳（如 ConvNeXt 中的 DWConv 作为位置编码模块）。

### 13.4 分类：refinement node

在技术树中，CSWin 是一个 **refinement node（精化节点）**。它没有开创一个全新的范式（层级式 Transformer backbone 由 Swin/PVT 建立），但在 Swin 的范式内通过改进 attention 机制和位置编码取得了显著的性能提升。

CSWin 证明了在已有范式内进行深入的技术改进仍然可以获得 1-4 个点的下游任务提升——这在 2022 年的 CV 圈子里是一个重要的结果，说明了视觉 Transformer 架构仍有很大的改进空间。

## 十四、局限

### 14.1 分类精度提升不如下游任务显著

CSWin-B 在 ImageNet-1K 224² 上的 84.2% 比 Swin-B 的 83.3% 高 0.9 个点，但把 CSWin-B 放到 384²（85.4%）才比 Swin-B 384²（84.2%）高 1.2 个点。相比之下，在 COCO 上的提升是 4+ box AP。

这验证了 CSWin 的设计目标：**它主要针对密集预测任务优化，不是分类任务的 SOTA 架构。** 分类只需要全局表示，而 CSWin 的十字形 attention 在大分辨率特征图上（检测/分割需要）的优势更明显。

### 14.2 推理速度略慢

CSWin-B 在 COCO 上的 FPS（9.6）比 Swin-B（11.2）慢了约 15%。虽然精度大幅提升，但速度的差距在某些实时性要求高的场景下会有影响。

原因可能包括：
- Stage 3 的 21 或 32 个 block 的深度导致计算量集中在深层
- 重叠卷积 token embedding（7×7 stride 4）比 Swin 的轻量 patch partition 更慢
- 水平/垂直条带的 reshape + permute 操作

### 14.3 Stage 3 block 数量的不对称性

CSWin-T 在 Stage 3 堆了 21 个 block，在 Stage 4 只有 1 个 block。这个设计选择是经验性的——深层需要更多计算来建模全局关系。但 Stage 3 的 21 个 block 如果换成更少的 block 会怎样？论文没有给出明确的消融。

### 14.4 Stripe width 选择是经验性的

sw=[1,2,7,7] 是在 224×224 输入下为了整除而选择的。如果输入分辨率变化（如 384×384 或 512×512），stripe width 需要调整。论文没有讨论在不同分辨率下如何设置 sw。

### 14.5 训练成本

CSWin 的完整实验（ImageNet + COCO + ADE20K + 消融）需要大量 GPU 时。论文在 ImageNet 上使用默认的 300 epoch 训练，检测任务上使用 1× 和 3× schedule，分割任务上使用 80k 和 160k iterations——所有实验加起来预计需要数百 GPU 天。

## 十五、小结

CSWin Transformer 的核心贡献可以浓缩为一句话：**把 Swin 的方形窗口 attention 替换为十字形（水平 + 垂直条带并行）attention，在相同 FLOPs 下实现了更大的 single-block attention 区域，从而在分类、检测和分割上全面超越 Swin。**

它做了三件具体的事：

1. **Multi-head grouping** — 让一半 head 做水平条带，另一半做垂直条带，并行计算后拼接。这在不增加 FLOPs 的情况下把 attention 区域从一个 7×7 窗口（49 token）扩展到整行 + 整列的十字形（Stage 1 可达 111 token）。

2. **Dynamic stripe width** — 浅层用小宽度（sw=1,2），深层用大宽度（sw=7,7）。在深层实现接近全局的 attention，在浅层保持计算效率。

3. **LePE** — 通过深度可分离卷积在 V 上直接施加局部位置偏置，增强了密集预测任务所需的局部连续性先验。

在 COCO 检测上，CSWin-T 以 Swin-T 80% 的参数量实现了 +3-4.5 box AP 的提升。在 ADE20K 分割上，CSWin-T 在同等参数量下实现了 +4.8 mIoU（UperNet）的提升，而在 Semantic FPN 上用更少的参数（26.1M vs 31.9M）实现了 +6.7 mIoU 的提升。这些差距不是"略好"，而是**系统性的明显的优势**。

CSWin 在技术树中的位置是 refinement node——它没有发明新的范式，而是在 Swin 的层级式 Transformer backbone 范式内，通过更巧妙的 attention 设计实现了显著改进。它为后来的视觉 Transformer 架构提供了重要参考：**有时候，跨越范式不如在范式内做得更好。**
