# CSWin Transformer: A General Vision Transformer Backbone with Cross-Shaped Windows（2022）

**论文：** [arXiv](https://arxiv.org/abs/2107.00652) · Xiaoyi Dong, Jianmin Bao, Dongdong Chen, Weiming Zhang, Nenghai Yu, Lu Yuan, Dong Chen, Baining Guo · 2022

---

## 一、先搞清楚坑在哪

Vision Transformer（ViT）在图像分类上取得了突破，但它的核心问题在于**计算复杂度**。标准的 ViT 使用全局自注意力，复杂度是 $O(N^2)$，其中 $N$ 是 patch 数量。对于一张 $224 \times 224$ 的图像，如果 patch size 是 $16 \times 16$，$N = 196$，还能接受。但如果要处理高分辨率图像或做密集预测任务（检测、分割），$N$ 会急剧增大，$O(N^2)$ 的复杂度就变得不可承受。

Swin Transformer 提出了一个漂亮的解决方案：把注意力限制在局部窗口内，复杂度降到 $O(M^2 \cdot N)$，其中 $M$ 是窗口大小（通常是 7）。同时用 shifted window 实现跨窗口信息交互。这个思路很好，但有个问题：**窗口形状是固定的正方形**。

直白说，Swin 的窗口是 $7 \times 7$ 的正方形。但图像中的物体不都是正方形的——人有高矮胖瘦，建筑有长有宽，文字行是细长的。用固定正方形窗口去捕捉这些结构，效率不高。

## 二、Swin Transformer 的真正问题

Swin 的核心是**局部窗口自注意力 + 移位窗口**。每个 stage 中，图像被划分成不重叠的 $M \times M$ 窗口，在每个窗口内做自注意力。然后通过 shifted window 让不同窗口间的信息流动。

这里有个关键问题：**窗口形状固定为正方形，限制了感受野的形状**。具体来说：

1. **各向同性限制**：正方形窗口在水平和垂直方向上具有相同的感受野范围。但图像中的纹理、边缘、物体往往具有方向性——水平条纹、垂直柱子、斜向边界。用正方形窗口去处理这些结构，要么浪费计算资源（在无关方向上计算），要么信息不够（在关键方向上覆盖不足）。

2. **长距离依赖捕捉困难**：对于细长形状的物体（如一行文字、一条道路），正方形窗口需要多次移位才能覆盖整个结构。每次移位只能移动窗口大小的一半（$\lfloor M/2 \rfloor$），对于 $7 \times 7$ 窗口，一次移位只能移动 3 个像素。要覆盖一个 100 像素宽的水平条纹，需要 14 次移位。

3. **计算效率问题**：Swin 中，每个窗口内的 token 数量是固定的 $M^2$。如果某个区域的特征是水平方向延伸的，用 $7 \times 7$ 窗口去覆盖，垂直方向的 7 个位置中有很多是无关的噪声信息，浪费了注意力计算。

有意思的是，Swin 论文也意识到了这个问题，但他们选择用更大窗口（如 $12 \times 12$）或更深的网络来缓解，而不是从根本上改变窗口形状。

## 三、CSWin Transformer 的核心思路

CSWin 的核心想法非常直接：**把正方形窗口拆成交叉形——一个水平条带和一个垂直条带**。

这不是偶然的。作者观察到，图像中的特征往往具有方向性：水平边缘、垂直边缘、对角线纹理。用条带形窗口去匹配这些方向性特征，比用正方形窗口更高效。

具体来说，CSWin 在每个 Transformer block 中同时计算：
- **水平条带自注意力**：将图像划分成 $1 \times W$ 的细长条带（高度为 1，宽度为整个特征图）
- **垂直条带自注意力**：将图像划分成 $H \times 1$ 的细长条带（高度为整个特征图，宽度为 1）

两个注意力结果拼接后通过一个全连接层融合。这样，每个 token 既能感知水平方向的长距离依赖，又能感知垂直方向的长距离依赖，而且计算复杂度比全局注意力低得多。

## 四、网络架构详解

### 4.1 整体架构

CSWin 遵循 ViT 和 Swin 的分层设计范式，包含 4 个 stage，每个 stage 包含若干 CSWin Transformer Block。整体结构如下：

```
输入图像 (H x W x 3)
    │
    ▼
Patch Embedding (H/4 x W/4 x C)
    │
    ▼
Stage 1: [CSWin Block x N1] ──→ (H/4 x W/4 x C)
    │
    ▼
Patch Merging ──→ (H/8 x W/8 x 2C)
    │
    ▼
Stage 2: [CSWin Block x N2] ──→ (H/8 x W/8 x 2C)
    │
    ▼
Patch Merging ──→ (H/16 x W/16 x 4C)
    │
    ▼
Stage 3: [CSWin Block x N3] ──→ (H/16 x W/16 x 4C)
    │
    ▼
Patch Merging ──→ (H/32 x W/32 x 8C)
    │
    ▼
Stage 4: [CSWin Block x N4] ──→ (H/32 x W/32 x 8C)
    │
    ▼
分类头 / 检测头 / 分割头
```

其中，不同配置的 CSWin 模型参数如下：

| 模型 | C | Stage 1 | Stage 2 | Stage 3 | Stage 4 | 参数量 |
|------|---|---------|---------|---------|---------|--------|
| CSWin-T | 64 | 1 | 2 | 21 | 1 | 23M |
| CSWin-S | 96 | 1 | 2 | 21 | 1 | 35M |
| CSWin-B | 96 | 2 | 4 | 32 | 2 | 78M |
| CSWin-L | 144 | 2 | 4 | 32 | 2 | 173M |

### 4.2 CSWin Transformer Block 详解

每个 CSWin Block 的核心是**交叉形窗口自注意力（Cross-Shaped Window Self-Attention）**。下面是 block 的详细结构：

```
输入特征 (H x W x C)
    │
    ▼
LayerNorm
    │
    ▼
┌─────────────────────────────────────────────┐
│         Cross-Shaped Window Attention       │
│                                             │
│   ┌──────────────┐    ┌──────────────┐      │
│   │ 水平条带注意力  │    │ 垂直条带注意力  │      │
│   │ (1 x W 窗口)  │    │ (H x 1 窗口)  │      │
│   └──────┬───────┘    └──────┬───────┘      │
│          │                   │               │
│          └────────┬──────────┘               │
│                   ▼                          │
│              Concat + Linear                 │
└───────────────────┬─────────────────────────┘
    │
    ▼
残差连接 (+)
    │
    ▼
LayerNorm
    │
    ▼
MLP (两个全连接层 + GELU)
    │
    ▼
残差连接 (+)
    │
    ▼
输出特征 (H x W x C)
```

### 4.3 前向过程逐步骤

假设输入图像尺寸为 $224 \times 224$，使用 CSWin-T 配置。

**步骤 1：Patch Embedding**
- 输入：$224 \times 224 \times 3$
- 用 $4 \times 4$ 的卷积核，stride=4，输出通道 64
- 输出：$56 \times 56 \times 64$（$H/4 = 56$，$W/4 = 56$）

**步骤 2：Stage 1（1 个 CSWin Block）**
- 输入：$56 \times 56 \times 64$
- 水平条带注意力：将特征图分成 56 个 $1 \times 56$ 的条带，每个条带内有 56 个 token
- 垂直条带注意力：将特征图分成 56 个 $56 \times 1$ 的条带，每个条带内有 56 个 token
- 每个条带内做标准的自注意力
- 输出：$56 \times 56 \times 64$

**步骤 3：Patch Merging（下采样）**
- 输入：$56 \times 56 \times 64$
- 用 $2 \times 2$ 的卷积，stride=2，输出通道 128
- 输出：$28 \times 28 \times 128$

**步骤 4：Stage 2（2 个 CSWin Block）**
- 输入：$28 \times 28 \times 128$
- 水平条带：28 个 $1 \times 28$ 条带，每个条带 28 个 token
- 垂直条带：28 个 $28 \times 1$ 条带，每个条带 28 个 token
- 输出：$28 \times 28 \times 128$

**步骤 5：Patch Merging**
- 输入：$28 \times 28 \times 128$
- 输出：$14 \times 14 \times 256$

**步骤 6：Stage 3（21 个 CSWin Block）**
- 输入：$14 \times 14 \times 256$
- 水平条带：14 个 $1 \times 14$ 条带，每个条带 14 个 token
- 垂直条带：14 个 $14 \times 1$ 条带，每个条带 14 个 token
- 输出：$14 \times 14 \times 256$

**步骤 7：Patch Merging**
- 输入：$14 \times 14 \times 256$
- 输出：$7 \times 7 \times 512$

**步骤 8：Stage 4（1 个 CSWin Block）**
- 输入：$7 \times 7 \times 512$
- 水平条带：7 个 $1 \times 7$ 条带，每个条带 7 个 token
- 垂直条带：7 个 $7 \times 1$ 条带，每个条带 7 个 token
- 输出：$7 \times 7 \times 512$

**步骤 9：分类头**
- 全局平均池化：$1 \times 512$
- 全连接层：$512 \rightarrow 1000$
- 输出：1000 类分类 logits

## 五、核心创新点

### 创新点 1：交叉形窗口自注意力

#### (a) 痛点与动机

Swin Transformer 使用 $M \times M$ 的正方形窗口，每个窗口内做自注意力。这个设计的核心假设是：图像中的局部区域是各向同性的——水平和垂直方向的信息同样重要。

但这个假设在自然图像中不成立。考虑一个简单的例子：一张包含水平条纹的图像。在 $7 \times 7$ 的窗口中，水平方向有 7 个像素的信息，但垂直方向只有 7 个像素。如果条纹是水平的，垂直方向的 7 个像素中可能只有 1-2 个包含有用信息（条纹的厚度），其余都是冗余或噪声。

[Liu et al., 2021] 的 Swin Transformer 用 shifted window 来缓解这个问题，但窗口形状仍然是正方形的。这意味着无论怎么移位，每个窗口的感受野始终是各向同性的。

#### (b) 方案细节

CSWin 将正方形窗口拆成两个正交的条带：

- **水平条带**：$1 \times W$，覆盖整个宽度，高度为 1
- **垂直条带**：$H \times 1$，覆盖整个高度，宽度为 1

具体实现时，作者引入了一个参数 $sw$（strip width），控制条带的宽度。但论文中默认使用 $sw=1$，即最细的条带。

对于水平条带：
1. 将特征图 $X \in \mathbb{R}^{H \times W \times C}$ 分成 $H$ 个 $1 \times W$ 的条带
2. 每个条带内有 $W$ 个 token，做自注意力
3. 输出形状：$H \times W \times C$

对于垂直条带：
1. 将特征图 $X \in \mathbb{R}^{H \times W \times C}$ 分成 $W$ 个 $H \times 1$ 的条带
2. 每个条带内有 $H$ 个 token，做自注意力
3. 输出形状：$H \times W \times C$

两个注意力结果拼接后通过一个线性层融合：
$$X_{\text{out}} = \text{Linear}(\text{Concat}(X_{\text{horiz}}, X_{\text{vert}}))$$

#### (c) 为什么有效

交叉形窗口的核心优势在于**方向性感受野**：

1. **水平条带**：每个 token 可以关注整个宽度范围内的所有像素，捕捉水平方向的长距离依赖。对于水平条纹、文字行、地平线等结构，这比正方形窗口高效得多。

2. **垂直条带**：每个 token 可以关注整个高度范围内的所有像素，捕捉垂直方向的长距离依赖。对于垂直柱子、人物轮廓、树木等结构同样高效。

3. **组合效果**：水平和垂直条带的信息通过线性层融合，每个 token 实际上获得了十字形的感受野——水平方向覆盖整个宽度，垂直方向覆盖整个高度。这相当于在 $O(HW)$ 的计算量下获得了 $O(H \times W)$ 的感受野覆盖。

从消融实验来看（论文 Table 4），将 CSWin 中的交叉形注意力替换为 Swin 的正方形窗口注意力，在 ImageNet 上 top-1 准确率下降约 0.5-1.0%。这说明方向性感受野确实带来了性能提升。

#### (d) 与 Related Work 的关系

CSWin 的交叉形注意力与以下工作相关：

- **[Dosovitskiy et al., 2021] ViT**：使用全局注意力，复杂度 $O(N^2)$。CSWin 通过局部条带注意力将复杂度降到 $O(HW)$。
- **[Liu et al., 2021] Swin Transformer**：使用 $M \times M$ 正方形窗口注意力 + shifted window。CSWin 用条带替代正方形，获得方向性感受野。
- **[Wang et al., 2021] PVT**：使用空间缩减注意力（SRA），通过下采样 K 和 V 来降低复杂度。CSWin 采用不同的策略——保持全分辨率但限制注意力范围。
- **[Chu et al., 2021] TNT**：在 patch 内部和 patch 之间做注意力。CSWin 关注的是不同方向上的 patch 间注意力。

如果使用 Swin 的正方形窗口替代 CSWin 的交叉形窗口，每个 token 只能看到 $M \times M$ 的局部区域，需要多次移位才能覆盖长距离依赖。而 CSWin 的一个条带就能覆盖整个宽度或高度。

#### (e) 如果去掉会怎样

论文 Table 4 的消融实验显示，如果去掉交叉形注意力，替换为 Swin 的正方形窗口注意力（保持其他设计不变），CSWin-T 在 ImageNet 上的 top-1 准确率从 82.7% 下降到 82.0%（下降 0.7%）。对于更大的模型 CSWin-B，下降幅度约为 0.5%。

### 创新点 2：条带宽度自适应（LePE）

#### (a) 痛点与动机

标准的自注意力机制是排列不变的——它不关心 token 之间的相对位置。对于图像任务，位置信息至关重要。ViT 使用绝对位置编码，Swin 使用相对位置偏置。

但 CSWin 的条带注意力中，每个条带内的 token 数量可能很大（例如 $1 \times 56$ 的条带中有 56 个 token），而条带本身是细长的。标准的位置编码方式（如相对位置偏置）需要存储 $W^2$ 或 $H^2$ 的偏置矩阵，对于大尺寸特征图，这会导致较大的参数量。

#### (b) 方案细节

CSWin 提出了一种**局部增强位置编码（LePE，Locally-enhanced Positional Encoding）**：

$$ \text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d}}\right)V + \text{PE}(V) $$

其中 $\text{PE}(V)$ 是一个深度可分离卷积（depthwise convolution）操作：
$$\text{PE}(V) = \text{DWConv}(V)$$

这里 DWConv 使用 $3 \times 3$ 的卷积核，padding=1，保持输入输出形状一致。

#### (c) 为什么有效

LePE 的核心思想是：**用卷积隐式编码位置信息，而不是显式地学习位置偏置**。

1. **局部性**：$3 \times 3$ 的深度可分离卷积只在 $3 \times 3$ 的局部邻域内操作，这天然地引入了局部位置信息。每个 token 的特征会被其周围的 8 个邻居的特征所调制。

2. **平移等变性**：卷积是平移等变的——一个物体在图像中移动，其卷积响应也相应移动。这与自注意力的排列不变性形成互补。

3. **计算效率**：深度可分离卷积的计算量远小于全连接层或大矩阵乘法。对于 $H \times W \times C$ 的特征图，$3 \times 3$ 深度可分离卷积的 FLOPs 为 $9 \times H \times W \times C$，而相对位置偏置需要 $H \times W \times H \times W$ 的矩阵。

从消融实验（论文 Table 5）来看，LePE 相比绝对位置编码提升了约 0.3-0.5% 的准确率，相比相对位置偏置提升了约 0.2-0.3%。

#### (d) 与 Related Work 的关系

- **[Vaswani et al., 2017] Transformer**：使用绝对位置编码（正弦/余弦函数）。
- **[Shaw et al., 2018]**：提出相对位置编码，在注意力权重中加入可学习的相对位置偏置。
- **[Liu et al., 2021] Swin Transformer**：使用相对位置偏置，每个 $M \times M$ 窗口内学习一个 $M^2 \times M^2$ 的偏置矩阵。
- **[Chu et al., 2021] CPVT**：使用条件位置编码，通过卷积生成位置编码。
- **[Wu et al., 2021] CvT**：在 Transformer 中引入卷积操作。

CSWin 的 LePE 与 CPVT 和 CvT 的思路类似，都是通过卷积来编码位置信息。但 CSWin 将卷积作为注意力输出的一个附加项（加性），而不是替换注意力机制。

#### (e) 如果去掉会怎样

论文 Table 5 显示，如果去掉 LePE（即只使用标准的自注意力，不添加任何位置编码），CSWin-T 的准确率从 82.7% 下降到 81.8%（下降 0.9%）。如果使用绝对位置编码替代 LePE，准确率为 82.4%（下降 0.3%）。

## 六、公式详解

### 公式 1：交叉形窗口自注意力的计算

**符号定义**：
- $X \in \mathbb{R}^{H \times W \times C}$：输入特征图
- $H, W$：特征图的高度和宽度
- $C$：通道数
- $sw$：条带宽度（strip width），默认 $sw=1$
- $N_{\text{head}}$：注意力头数
- $d = C / N_{\text{head}}$：每个注意力头的维度

**公式来源**：论文公式 (1) 和 (2)，作者原创。

**推导过程**：

水平条带划分：

将特征图 $X$ 沿高度方向划分成 $\lceil H/sw \rceil$ 个条带，每个条带的大小为 $sw \times W \times C$。当 $sw=1$ 时，有 $H$ 个 $1 \times W \times C$ 的条带。

对于第 $i$ 个水平条带 $X_i^{\text{h}} \in \mathbb{R}^{1 \times W \times C}$，将其 reshape 为 $W \times C$ 的矩阵，然后做标准的多头自注意力：

$$ \text{MSA}_{\text{h}}(X_i^{\text{h}}) = \text{Concat}(\text{head}_1, \ldots, \text{head}_{N_{\text{head}}})W^O $$

其中每个 head 的计算为：

$$ \text{head}_j = \text{Softmax}\left(\frac{Q_j K_j^T}{\sqrt{d}}\right)V_j $$

$$ Q_j = X_i^{\text{h}} W_j^Q, \quad K_j = X_i^{\text{h}} W_j^K, \quad V_j = X_i^{\text{h}} W_j^V $$

$W_j^Q, W_j^K, W_j^V \in \mathbb{R}^{C \times d}$，$W^O \in \mathbb{R}^{C \times C}$ 都是可学习的投影矩阵。

垂直条带的计算类似，只是将特征图沿宽度方向划分。

最终输出：

$$ X_{\text{out}} = \text{Linear}(\text{Concat}(X_{\text{h}}, X_{\text{v}})) $$

**直觉理解**：

水平条带注意力中，Q 和 K 都来自同一个 $1 \times W$ 的条带，所以注意力矩阵的大小是 $W \times W$。这意味着每个 token 可以关注同一行中所有其他的 token。垂直条带类似，每个 token 可以关注同一列中所有的 token。

将两者拼接后通过线性层，相当于让网络自己学习如何融合水平和垂直方向的信息。如果某个 token 更依赖水平方向的信息，线性层可以给水平注意力的输出更大的权重。

**具体数值例子**：

对于 CSWin-T 的 Stage 1，输入 $H=56, W=56, C=64$，$N_{\text{head}}=4$，$d=16$。

水平条带注意力：
- 每个条带大小：$1 \times 56 \times 64$
- 每个条带内的 token 数：56
- 注意力矩阵大小：$56 \times 56 = 3136$
- 每个条带的 FLOPs：$2 \times 56 \times 64 \times 16 \times 4 + 56 \times 56 \times 16 \times 4 = 2 \times 229376 + 200704 = 659456$
- 总 FLOPs（56 个条带）：$56 \times 659456 \approx 36.9M$

垂直条带注意力类似，也是 36.9M FLOPs。

总 FLOPs：$36.9M + 36.9M \approx 73.8M$

对比 Swin-T 的 Stage 1（窗口大小 $7 \times 7$）：
- 窗口数：$\lceil 56/7 \rceil \times \lceil 56/7 \rceil = 8 \times 8 = 64$
- 每个窗口内 token 数：$7 \times 7 = 49$
- 每个窗口注意力矩阵大小：$49 \times 49 = 2401$
- 每个窗口 FLOPs：$2 \times 49 \times 64 \times 16 \times 4 + 49 \times 49 \times 16 \times 4 = 2 \times 200704 + 153664 = 555072$
- 总 FLOPs：$64 \times 555072 \approx 35.5M$

CSWin 的 Stage 1 计算量（73.8M）是 Swin（35.5M）的约 2.1 倍。但 CSWin 的每个 token 可以看到整个行或列的信息，而 Swin 的每个 token 只能看到 $7 \times 7$ 的局部区域。

**边界情况**：
- 当 $sw = H = W$ 时，交叉形窗口退化为全局注意力。但实际中 $sw$ 通常远小于 $H$ 和 $W$。
- 当 $H$ 或 $W$ 不是 $sw$ 的整数倍时，需要对特征图进行 padding。

### 公式 2：计算复杂度分析

**符号定义**：
- $H, W$：特征图的高度和宽度
- $C$：通道数
- $N = H \times W$：token 总数
- $sw$：条带宽度

**公式来源**：论文 Section 3.1，作者推导。

**推导过程**：

标准全局自注意力的复杂度：
$$ \Omega(\text{MSA}) = 4HWC^2 + 2(HW)^2C $$

第一项 $4HWC^2$ 来自 QKV 投影（$3HWC^2$）和输出投影（$HWC^2$）。第二项 $2(HW)^2C$ 来自注意力矩阵的计算（$QK^T$ 和 $AV$）。

CSWin 交叉形窗口自注意力的复杂度：

水平条带部分：将特征图分成 $\lceil H/sw \rceil$ 个条带，每个条带内有 $sw \times W$ 个 token。

$$ \Omega(\text{CSWin}_{\text{h}}) = 4HWC^2 + 2 \cdot \lceil H/sw \rceil \cdot (sw \cdot W)^2 \cdot C $$

当 $sw=1$ 时：
$$ \Omega(\text{CSWin}_{\text{h}}) = 4HWC^2 + 2H \cdot W^2 \cdot C = 4HWC^2 + 2HW^2C $$

垂直条带部分类似：
$$ \Omega(\text{CSWin}_{\text{v}}) = 4HWC^2 + 2W \cdot H^2 \cdot C = 4HWC^2 + 2H^2WC $$

总复杂度：
$$ \Omega(\text{CSWin}) = 8HWC^2 + 2HWC(H + W) $$

注意第一项是 $8HWC^2$ 而不是 $4HWC^2$，因为水平和垂直注意力各自有独立的 QKV 投影。

**直觉理解**：

- 第一项 $8HWC^2$ 是 QKV 投影的代价，与 token 数量 $N=HW$ 线性相关。
- 第二项 $2HWC(H+W)$ 是注意力矩阵计算的代价，与 $HW(H+W)$ 相关。

对比 Swin Transformer（窗口大小 $M \times M$）：
$$ \Omega(\text{Swin}) = 4HWC^2 + 2M^2HWC $$

其中 $M$ 是窗口大小（通常为 7）。

**关键区别**：
- Swin 的注意力项是 $O(M^2HW)$，与 $HW$ 线性相关（因为 $M$ 是常数）。
- CSWin 的注意力项是 $O(HW(H+W))$，与 $HW$ 的平方根相关（因为 $H+W \approx 2\sqrt{HW}$ 当 $H=W$ 时）。

对于大特征图，CSWin 的复杂度高于 Swin，但远低于全局注意力（$O(H^2W^2)$）。

**具体数值例子**：

对于 $H=W=56, C=64$：

CSWin-T（$sw=1$）：
$$ \Omega = 8 \times 56 \times 56 \times 64^2 + 2 \times 56 \times 56 \times 64 \times (56 + 56) $$
$$ = 8 \times 56 \times 56 \times 4096 + 2 \times 56 \times 56 \times 64 \times 112 $$
$$ = 102,760,448 + 44,957,696 $$
$$ \approx 147.7M \text{ FLOPs} $$

Swin-T（$M=7$）：
$$ \Omega = 4 \times 56 \times 56 \times 64^2 + 2 \times 7^2 \times 56 \times 56 \times 64 $$
$$ = 4 \times 56 \times 56 \times 4096 + 2 \times 49 \times 56 \times 56 \times 64 $$
$$ = 51,380,224 + 19,660,288 $$
$$ \approx 71.0M \text{ FLOPs} $$

CSWin 的计算量是 Swin 的 2.08 倍。但 CSWin 的每个 token 可以看到 56 个 token（整个行或列），而 Swin 只能看到 49 个 token（$7 \times 7$ 窗口）。从"每 FLOP 能看到多少 token"的角度看：
- CSWin：$56 / (147.7M/56) \approx 0.021$ token/FLOP
- Swin：$49 / (71.0M/56) \approx 0.039$ token/FLOP

Swin 的 FLOP 效率更高，但 CSWin 能看到更远距离的 token。

**边界情况**：
- 当 $H$ 和 $W$ 很大时，CSWin 的 $O(HW(H+W))$ 项会占主导，导致计算量显著增加。但实际中，CSWin 在 Stage 3 和 Stage 4 使用（特征图尺寸较小），在 Stage 1 和 Stage 2 使用（特征图尺寸较大），但通道数 $C$ 较小，所以总计算量可控。
- 当 $H=W=1$ 时，CSWin 退化为单 token 情况，注意力没有意义。

### 8.2 LePE 的实现

LePE 在代码中通过一个 $3 \times 3$ 的深度可分离卷积实现：

```python
class LePE(nn.Module):
    def __init__(self, dim):
        self.conv = nn.Conv2d(dim, dim, 3, 1, 1, groups=dim, bias=False)
        
    def forward(self, x):
        # x: (B, H, W, C)
        x = x.permute(0, 3, 1, 2)  # (B, C, H, W)
        x = self.conv(x)
        x = x.permute(0, 2, 3, 1)  # (B, H, W, C)
        return x
```

注意这里 `groups=dim` 表示是 depthwise 卷积，每个通道独立卷积，没有跨通道的信息混合。

### 8.3 代码中的技巧

1. **条带划分的 reshape 操作**：代码使用 `reshape` 和 `permute` 来实现条带划分，而不是显式地切分张量。这利用了 PyTorch 的内存连续性，效率更高。

2. **注意力掩码**：对于非 $sw$ 整数倍的边缘情况，代码使用注意力掩码来屏蔽无效位置。

3. **初始化**：代码使用了 Truncated Normal 初始化，这是 ViT 和 Swin 中常用的初始化方式。

## 十、局限

### 局限 1：计算效率

**内容**：CSWin 的计算复杂度高于 Swin。对于 $H=W=56$ 的特征图，CSWin 的 FLOPs 是 Swin 的 2.08 倍。虽然 CSWin 在参数量上更优，但实际推理速度可能慢于 Swin。

**影响**：在资源受限的场景（移动端、嵌入式设备）中，CSWin 可能不是最佳选择。

**论文是否承认**：论文在 Section 4.2 中讨论了计算复杂度，但未直接承认效率劣势。

**后续工作**：[Li et al., 2022] 的 EfficientFormer 尝试在保持方向性感受野的同时降低计算量。

### 局限 2：条带宽度固定

**内容**：CSWin 使用固定的条带宽度 $sw=1$，即最细的条带。对于不同尺度的特征，最优的条带宽度可能不同。

**影响**：在 Stage 1（特征图大）中，细条带可能不够高效；在 Stage 4（特征图小）中，细条带可能已经足够覆盖全局。

**论文是否承认**：论文在 Section 3.1 中讨论了 $sw$ 的影响，但未进行系统性分析。

**后续工作**：[Wang et al., 2022] 的 CrossFormer 提出了动态条带宽度，根据特征图尺寸自适应调整 $sw$。

### 局限 3：对非矩形图像的支持

**内容**：CSWin 的交叉形注意力假设图像是矩形的（$H \times W$）。对于非矩形输入（如全景图、不规则图像），需要额外的 padding 或裁剪。

**影响**：限制了 CSWin 在非标准图像任务中的应用。

**论文是否承认**：论文未讨论此问题。

### 局限 4：与卷积的互补性

**内容**：CSWin 的 LePE 使用 $3 \times 3$ 深度可分离卷积，但卷积的操作范围远小于注意力（卷积看 $3 \times 3$，注意力看整个行或列）。两者之间的尺度差异可能导致信息不匹配。

**影响**：在需要精细局部信息的任务（如分割边界）中，卷积可能提供的信息不够。

## 十一、小结

CSWin Transformer 的核心贡献是提出了**交叉形窗口自注意力**，将 Swin 的正方形窗口拆分为水平和垂直条带，使每个 token 能同时捕捉水平方向和垂直方向的长距离依赖。这个设计看似简单，但抓住了图像特征的方向性本质——图像中的结构往往具有方向性，用条带形窗口去匹配这些结构比正方形窗口更高效。

实验证明，CSWin 在 ImageNet 分类、COCO 检测和 ADE20K 分割上均超过 Swin，且参数量更少。这表明方向性感受野确实是一种更优的注意力设计方式。

CSWin 的 broader impact 在于：它推动了视觉 Transformer 中注意力机制的设计从"各向同性"向"各向异性"转变。后续的 CrossFormer、EfficientFormer 等工作都沿用了这一思路。但 CSWin 的计算效率问题也提醒我们：更复杂的注意力机制不一定更好，需要在性能和效率之间找到平衡。
