## 一、先搞清楚"问题在哪里"

2021 年，Swin Transformer 提出了一个关键设计：把全局注意力拆成局部 $7 \times 7$ 的非重叠窗口，用窗口偏移（shifted window）让信息跨窗口流动。这个方案解决了 ViT 的两个核心问题——$O(N^2)$ 的注意力复杂度和单分辨率输出——在 ImageNet 分类、COCO 检测、ADE20K 分割上全面超越 ResNe(X)t，第一次证明 Transformer backbone 可以全面接管 CNN 的生态位。

但 Swin 之后，社区很快意识到一个设计上的矛盾：**为什么窗口必须是 $7 \times 7$ 的正方形？**

自然图像中的特征结构是方向性的——水平条纹、垂直柱子、斜向边缘、细长物体——用各向同性的正方形窗口去覆盖它们，本身就有一个"形状不匹配"的问题。这意味着 Swin 的每个 block 中，token 只能用固定尺寸的正方形区域做 attention。要覆盖一个水平方向延伸的长结构，需要多次窗口偏移，等效感受野增长缓慢。

另一方面，CSWin 之前已经有一些"条带注意力"的尝试。Axial attention（Huang et al., 2019）和 criss-cross attention（Huang et al., 2019）都提出了沿轴向划分条带做 attention 的思路。但 axial attention 的条带宽度限制为 1，且水平和垂直 attention 是串行的（不是并行的），限制了表达能力。Criss-cross attention 做了并行设计但使用重叠窗口，计算效率不高，而且在实际系统中推理速度偏慢。

所以到 2021 年底，视觉 Transformer backbone 领域的核心问题变成了：**能不能设计一种注意力机制，既保持 Swin 的层级结构和计算高效性，又让每个 token 在单次 attention 中就能覆盖更大——而且是方向性——的感受野？** CSWin Transformer 给出的答案是"交叉形窗口自注意力"。

## 二、现有路线上的真正问题

要理解 CSWin 为什么需要被发明，得细致分析 Swin 的跟深层次限制。

### 2.1 Swin 中窗口形状假设的根本问题

Swin 把特征图划分成 $M \times M$（默认 7×7）的非重叠窗口。这基于一个隐含假设：图像中的局部信息是各向同性的——水平和垂直方向的信息同等重要。

这个假设在两大类情况下不成立。

**第一类：方向性强的纹理结构。** 想想一张照片里有水平条纹的斑马、编织纹理的草帽、垂直排列的建筑物窗户。这些结构在某个方向上表现出强烈的相关性，但在正交方向上相关性快速衰减。用正方形窗口覆盖它们：

- 在条纹方向（高相关性方向）上，窗口覆盖的 7 个像素不够远，需要多次窗口偏移才能延伸
- 在垂直条纹方向（低相关性方向）上，7 个像素中有很多是冗余或噪声——浪费了 attention 计算

**第二类：细长物体。** 一行文字、一条道路、一支铅笔——这些物体在一个维度上很长，另一个维度上很窄。用 7×7 窗口去覆盖一个 100 像素宽的水平文字行，需要 14 次 Swin 窗口偏移（每次移 3-4 个像素）才能从行首看到行尾。

Swin 论文对这个问题的回应是堆叠更多 block（Swin-S 和 Swin-B 在 stage 3 放了 18 个 block）来逐步扩大感受野。但这就引出了第二个问题。

### 2.2 窗口偏移的"慢感受野扩张"

Swin 通过窗口偏移实现跨窗口信息流动。两个连续 Swin block（W-MSA + SW-MSA）中，一个 token 可以看到：

```
Level 0: 7×7 window (49 tokens)
Level 1: 窗口偏移 + 下一个 W-MSA → 约 13×13 的区域
Level 2: 再偏移 → 约 19×19
...
Level L: 约 7 + 6L 范围
```

对于 Swin-B 的 24 个 block（12 对 W/SW），一个 token 的有效感受野大约是 7 + 6×12 = 79 个 pixel 宽度（在 stage 1 的 4× 下采样下，对应原始图像 316 像素）。对于 224×224 的输入，这已经接近全局了。但问题是**不是所有 block 都在同一级分辨率下工作**——stage 3 的 14×14 分辨率下，窗口偏移每步只能覆盖 7×4 = 28 个原始像素。要覆盖 224×224 的全局，需要大量深层 block。

CSWin 的核心洞察是：**与其让 token 通过多次偏移来慢慢扩大感受野，不如在单次 attention 中就让 token 看到整行或整列的信息。**

### 2.3 Axial attention 和 Criss-cross attention 的遗留问题

CSWin 不是第一个想到"条带注意力"的。

**Axial attention（Huang et al., 2019）：** 分别沿 H 轴和 W 轴做条带注意力。但问题是轴向注意力是串行的——先做水平注意力再做垂直注意力，或者反过来。这意味着第二个方向上的注意力无法利用第一个方向上的完整语义。而且轴向注意力通常只做宽度为 1 的条带，限制了感受野。

**Criss-cross attention（Huang et al., 2019 for segmentation）：** 提出了类似"十字形"的注意力，水平和垂直同时做。但 criss-cross 的窗口是重叠的——每个 token 的位置都产生一个十字形的 attention 区域。重叠窗口导致大量重复计算（同一对 token 被多次计算 attention），在实际系统中效率不高。另外 criss-cross 限制条带宽度为 1，无法通过调整条带宽度来 trade off 计算和表达能力。

CSWin 的作者看到了这两条路线的不足，提出了一个统一的、高效的条带注意力方案。

## 三、这篇论文的核心思路

**CSWin 的核心思路是：把 Swin 的正方形窗口拆成交叉形——用水平和垂直条带的并行注意力，让每个 token 在单次 Transformer block 中就能同时看到整行和整列的信息。**

具体来说，三个设计组合在一起：

```text
交叉形窗口自注意力 →  水平和垂直条带并行注意力，感受野覆盖整行+整列
       ↓
条带宽度自适应    →  浅层用小条带（省计算），深层用大条带（大感受野）
       ↓
局部增强位置编码  →  用 depthwise conv 替代位置偏置，支持任意分辨率输入
```

这条技术路线的谱系位置：

```text
Swin Transformer（正方形窗口 + 移位, 2021）
   ↓
Axial attention / Criss-cross attention（条带注意力先驱，串行/低效）
   ↓
CSWin Transformer（并行水平+垂直条带，可调条带宽度, 2022） ← 本篇
   ↓
CrossFormer（动态条带宽度, 2022）
```

CSWin 和 Swin 的最大区别在于 attention 的计算方式：

- **Swin**：所有 head 用同样的窗口划分方案，先做 W-MSA 再做 SW-MSA
- **CSWin**：把 multi-head 分成两组，一半做水平条带注意力，一半做垂直条带注意力，并行执行

这个并行策略是核心创新——它没有增加额外计算量（投影矩阵是共享的），但让每个 block 中的 token 同时获得了 H 和 W 两个方向上的全局感受野。

## 四、整体结构与 forward 流程

### 4.1 整体架构（CSWin-T 为例）

CSWin 遵循 Swin 的 4 级金字塔设计，但在每个 block 内部用交叉形窗口替代 Swin 的正方形窗口。

```text
输入图像: H×W×3（224×224×3）
    │
    ▼
[Overlapped Conv Token Embedding]  7×7 conv, stride 4
    │                              输出: 56×56×C (C=64 for CSWin-T)
    ▼
┌─────────────────────────────────────────────┐
│ Stage 1 (4×下采样, 56×56, C=64)            │
│  1 × CSWin Block (sw=1)                     │
│   ├── LN → CSWin-Attention(水平+垂直条带) → LePE → 残差
│   └── LN → MLP → 残差                       │
└────────────────┬────────────────────────────┘
                 │
                 ▼
[Conv 3×3 stride 2]        下采样 + channel 加倍
                 │         输出: 28×28×128
                 ▼
┌─────────────────────────────────────────────┐
│ Stage 2 (8×下采样, 28×28, 128)             │
│  2 × CSWin Block (sw=2)                    │
└────────────────┬────────────────────────────┘
                 │
                 ▼
[Conv 3×3 stride 2]        下采样 + channel 加倍
                 │         输出: 14×14×256
                 ▼
┌─────────────────────────────────────────────┐
│ Stage 3 (16×下采样, 14×14, 256)            │
│  21 × CSWin Block (sw=7)                    │
└────────────────┬────────────────────────────┘
                 │
                 ▼
[Conv 3×3 stride 2]        下采样 + channel 加倍
                 │         输出: 7×7×512
                 ▼
┌─────────────────────────────────────────────┐
│ Stage 4 (32×下采样, 7×7, 512)              │
│  1 × CSWin Block (sw=7)                    │
└─────────────────────────────────────────────┘
    │
    ▼
分类头 / 检测头 / 分割头
```

### 4.2 架构参数（4 个变体）

| 变体 | C (base channel) | Stage blocks [N1,N2,N3,N4] | 条带宽度 [sw1,sw2,sw3,sw4] | 参数量 |
|------|-----------------|--------------------------|---------------------------|--------|
| CSWin-T | 64 | [1,2,21,1] | [1,2,7,7] | 23M |
| CSWin-S | 96 | [1,2,21,1] | [1,2,7,7] | 35M |
| CSWin-B | 96 | [2,4,32,2] | [1,2,7,7] | 78M |
| CSWin-L | 144 | [2,4,32,2] | [1,2,7,7] | 173M |

几个值得注意的设计选择：

1. **Stage 3 极其深**（21 或 32 个 block）。这和 Swin 一致——最深的 stage 在 1/16 分辨率上，平衡了空间分辨率和感受野。

2. **条带宽度从浅到深递增**（1→2→7→7）。Stage 1 和 2 的特征图较大（56×56 和 28×28），用小条宽（1 或 2）控制计算量。Stage 3 的特征图较小（14×14），用较大条宽（7）让 token 看到更远。

3. **Stage 4 条宽 7 但特征图只有 7×7**。这意味着条带宽度等于特征图宽度——水平条带退化为全局了。但垂直方向也是全局的，所以实际上 Stage 4 的 CSWin 注意力等价于全局注意力（每个 token 能看到所有其他 token）。

### 4.3 前向过程逐步跟踪（CSWin-T, 224×224 输入）

**Step 1 — Overlapped Conv Token Embedding**

输入 224×224×3 的 RGB 图像。用 7×7 的卷积核，stride=4，输出 64 通道（CSWin-T 的 base channel）。

张量形状：`[B, 3, 224, 224]` → `[B, 64, 56, 56]`

这里的 patch token embedding 和 ViT 不同：ViT 用 16×16 的非重叠卷积（stride=16），CSWin 用 7×7 的重叠卷积（stride=4）。重叠卷积保留了更多的边界信息。这个设计来自 PVTv2（Wang et al., 2022）。

**Step 2 — Stage 1（1 个 CSWin Block, 56×56, 64 维）**

这是第一个且唯一一个 CSWin Block（CSWin-T 的 Stage 1 很浅，只有 1 个 block）。

Block 内部结构：
```
输入特征 (56×56×64)
    │
    ▼
LayerNorm
    │
    ▼
┌────────────────────────────────────────────────┐
│              CSWin Self-Attention              │
│                                                │
│   头 1（水平条带注意力）     头 2（垂直条带注意力）│
│   ┌──────────────────┐    ┌──────────────────┐ │
│   │ 56 个水平条带     │    │ 56 个垂直条带     │ │
│   │ 每个: 1×56 个 token│   │ 每个: 56×1 个 token│ │
│   │ 注意力矩阵: 56×56  │    │ 注意力矩阵: 56×56  │ │
│   └──────────────────┘    └──────────────────┘ │
│           │                      │              │
│           └────────┬─────────────┘              │
│                    ▼                            │
│              Concat + Linear                    │
└────────────────────┬───────────────────────────┘
    │
    ▼
LePE (3×3 depthwise conv) ──→ 残差连接（+ 注意力输出）
    │
    ▼
LayerNorm → MLP → 残差连接
    │
    ▼
输出特征 (56×56×64)
```

关键细节：CSWin 在 QKV 投影之后，把 multi-head 分成两组。对于 CSWin-T 的 Stage 1，假设有 4 个 head（C=64, d=16），则：
- Head 0-1：做水平条带注意力
- Head 2-3：做垂直条带注意力

**Step 3 — Stage 1→2 下采样**

使用 3×3 卷积，stride=2，输出通道加倍。

张量形状：`[B, 64, 56, 56]` → `[B, 128, 28, 28]`

**Step 4 — Stage 2（2 个 CSWin Block, 28×28, 128 维）**

条带宽度 sw=2。

水平条带：将 28×28 特征图分成 14 个条带（28/2=14），每个条带 2×28=56 个 token。
垂直条带：类似地分成 14 个条带，每个条带 28×2=56 个 token。

每个条带内依然做标准的多头自注意力。

**Step 5 — Stage 2→3 下采样**

14×14×256。

**Step 6 — Stage 3（21 个 CSWin Block, 14×14, 256 维）**

条带宽度 sw=7。这是最深的主干 stage。

由于 sw=7 且 H=14，水平方向只有 2 个条带（14/7=2）。每个水平条带 7×14=98 个 token，垂直条带类似。

这里有个有趣的现象：当 sw=7 时，水平条带实际上是 7×14（高度 7，宽度 14）。这不是严格的"一行"——它是 7 行高的带。但 14×14 的特征图上，"一行"只有 14 个 token，做 attention 的信息量有限。提高条带宽度到 7 相当于在水平方向上做了"池化"级别的感受野扩大。

**Step 7 — Stage 4（1 个 CSWin Block, 7×7, 512 维）**

条带宽度 sw=7，但特征图只有 7×7。所以：
- 水平条带：只有一个条带（7/7=1），包含 7×7=49 个 token——实际上就是全局注意力
- 垂直条带：同样的

CSWin 注意力在 Stage 4 退化为全局注意力。这在设计上是合理的——到了最深层，需要的不是局部细节而是全局语义。

输出：7×7×512。接分类头（全局平均池化 + Linear 分类器）或检测/分割头。

### 4.4 与 Swin 的 stage 分布对比

| 特性 | Swin-T | CSWin-T |
|------|--------|---------|
| 参数量 | 28M | 23M |
| FLOPs | 4.5G | 4.1G |
| Stage 分布 | [2,2,6,2] | [1,2,21,1] |
| 注意力机制 | W-MSA + SW-MSA | 交叉形条带注意力 |
| Token embedding | Patch Partition 4×4 | Conv 7×7 stride 4 |
| 下采样 | Patch Merging | Conv 3×3 stride 2 |
| 位置编码 | 相对位置偏置 | LePE |

两种架构的最大差异在于 token 之间的交互方式：Swin 通过窗口偏移让信息逐窗口传播，CSWin 通过条带注意力让 token 直接看到整行/整列的信息。

## 五、核心模块逐个解释

### 5.1 交叉形窗口自注意力（CSWin-Attention）

**问题：** Swin 的 7×7 正方形窗口限制了每个 token 的单次注意力范围。窗口偏移虽然可以实现跨窗口信息流动，但感受野扩展速度慢，需要堆叠很多 block。

**解决方案：** 把特征图在水平和垂直方向上划分成条带（stripe），每个条带内做自注意力。把 multi-head 分成两组并行处理水平和垂直方向。

**数学定义：**

输入特征 $X \in \mathbb{R}^{H \times W \times C}$，$K$ 个 attention head。

对于水平条带自注意力，将 $X$ 沿高度方向划分成 $M = H/sw$ 个非重叠的水平条带：

$$X = [X^1, X^2, \dots, X^M], \quad X^i \in \mathbb{R}^{(sw \times W) \times C}$$

第 $k$ 个 head 的第 $i$ 个条带的自注意力输出：

$$Y^i_k = \text{Attention}(X^i W^Q_k, X^i W^K_k, X^i W^V_k)$$

水平条带自注意力输出：

$$H\text{-Attention}_k(X) = [Y^1_k, Y^2_k, \dots, Y^M_k]$$

垂直条带自注意力类似，记为 $V\text{-Attention}_k(X)$。

CSWin 将 $K$ 个 head 平均分成两组：
- Head 1 到 $K/2$：做水平条带自注意力
- Head $K/2+1$ 到 $K$：做垂直条带自注意力

CSWin 注意力的最终输出：

$$CSWin\text{-Attention}(X) = \text{Concat}(\text{head}_1, \dots, \text{head}_K)W^O$$

$$\text{head}_k = \begin{cases} H\text{-Attention}_k(X) & k = 1, \dots, K/2 \\ V\text{-Attention}_k(X) & k = K/2+1, \dots, K \end{cases}$$

**为什么这个设计有效：**

1. **并行组划分**：把 $K$ 个 head 分成两组，各自处理不同的注意力方向。两组 head 共享同一组 QKV 投影矩阵，只做不同的 reshape 和注意力计算。这意味着没有增加 QKV 投影的计算量（投影是最耗时的操作之一），只是重用了投影结果。

2. **方向性感受野**：水平方向，每个 token 可以看到同一"行"中的所有其他 token（实际上是 $sw \times W$ 范围，当 sw=1 时就是整行）。垂直方向同理。

3. **组合形成交叉形**：把水平和垂直结果拼接后通过线性层融合，每个 token 实际上是获得了整个十字形的感受野——水平方向覆盖宽度，垂直方向覆盖高度。

**和 Swin 窗口注意力的关键区别：**

| 维度 | Swin W-MSA | CSWin-Attention |
|------|-----------|-----------------|
| 窗口形状 | $M \times M$ 正方形 | $sw \times W$ / $H \times sw$ 条带 |
| 头分配 | 所有 head 用相同窗口 | 头分两组，方向不同 |
| 跨窗口通信 | 靠窗口偏移（交错 block） | 靠正交方向合并 |
| 感受野/block | $M \times M$ | $H \times W$ （通过正交组合） |
| 是否是并行 | block 层面串行（W→SW→W→SW） | block 内并行（H+V 同时） |

**消融实验的结论（论文 Table 4）：**

把 CSWin 的交叉形注意力替换为 Swin 风格的正方形窗口注意力（保持其他设计不变）：
- CSWin-T ImageNet: 82.7% → 82.0%（-0.7%）
- 这个差距在检测/分割上更大

### 5.2 条带宽度自适应（Stripe Width Adjustment）

**问题：** 条带宽度 $sw$ 直接影响注意力的计算量和感受野大小。对于不同分辨率的 stage，最优条带宽度不同。

**数学分析：** CSWin 注意力的计算复杂度为：

$$\Omega(CSWin) = HWC \times (4C + sw \times H + sw \times W)$$

注意这里第一项 $4C$ 是 QKV 投影的代价（每个 token 投影到 $Q,K,V$ 三个空间，各 $C$ 维，再输出投影 $C$ 维，共 $4C$）。第二项 $sw \times H$ 来自水平条带的注意力矩阵计算，$sw \times W$ 来自垂直条带。

当 $H = W$ 时：

$$\Omega(CSWin) = hwc \times (4C + 2 \times sw \times h)$$

**为什么浅层用小的 sw，深层用大的 sw：**

- Stage 1（56×56）：h=56 很大，如果用 sw=7，复杂度中的 $2 \times sw \times h = 2 \times 7 \times 56 = 784$ 大于 $4C = 256$（C=64），注意力矩阵计算占主导。用 sw=1，$2 \times 1 \times 56 = 112$ 小于 $256$，QKV 投影占主导，计算效率更高。

- Stage 3（14×14）：h=14 很小，用 sw=7，$2 \times 7 \times 14 = 196$ 接近 $4C = 1024$（C=256），可以接受。同时 sw=7 让 token 能看到大范围。

**具体配置**（论文设定，针对 224×224 输入，保证特征图尺寸能被 sw 整除）：

| Stage | 分辨率 | C | sw | 水平条带数 | 每条带 token 数 |
|-------|--------|---|----|-----------|----------------|
| 1 | 56×56 | 64 | 1 | 56 | 56 |
| 2 | 28×28 | 128 | 2 | 14 | 56 |
| 3 | 14×14 | 256 | 7 | 2 | 98 |
| 4 | 7×7 | 512 | 7 | 1 | 49（等于全局）|

**论文的消融实验（论文 Table 3）：**

| sw 配置 | ImageNet Top-1 | 泛化情况 |
|---------|---------------|---------|
| sw=[1,2,7,7] | 82.7 | 默认配置 |
| sw 全部=1 | 82.2（-0.5） | 所有 stage 都用最小条宽 |
| sw 全部=2 | 82.4（-0.3） | 浅层条宽太大，计算浪费 |
| sw 全部=7 | 82.5（-0.2） | Stage 1 计算量过大，但精度未提升 |

结论：自适应条带宽度比固定条带宽度的方案好 0.3-0.5 个点。

### 5.3 局部增强位置编码（LePE）

**问题：** Swin 使用相对位置偏置（Relative Position Bias），在注意力 logits 上加一个可学习的偏置矩阵 $B \in \mathbb{R}^{M^2 \times M^2}$。这个方法虽然有效，但有两个限制：1）偏置表是为固定窗口尺寸训练的，改变分辨率需要插值；2）相对位置偏置不直接编码局部纹理信息。

**解决方案：** LePE 把位置编码从"加到注意力计算中"移到"作为 attention 输出的并行分支"。

数学上，CSWin Block 中的注意力输出为：

$$\text{Output} = \text{CSWin-Attention}(X) + \text{LePE}(V)$$

其中：
$$\text{LePE}(V) = \text{DWConv}_{3\times3}(V)$$

$V$ 是注意力中的 value 投影结果，DWConv 是 depthwise 卷积（$3 \times 3$，groups $= C$）。

**为什么 LePE 比相对位置偏置好：**

1. **解耦了位置编码和注意力计算。** 相对位置偏置直接加在 attention logits 上，影响注意力权重的分布。LePE 则是直接在输出的 value 特征上添加位置信息，不干扰注意力分布的计算。这使得注意力可以专注于 content-based 的交互，而位置信息通过并行分支注入。

2. **自然支持任意分辨率输入。** 卷积是平移等变的，对不同尺寸的输入天然适应。相对位置偏置表是为固定窗口大小（如 7×7）训练的，需要插值才能适应不同尺寸，但插值会引入信息丢失。

3. **引入了更强的局部归纳偏置。** $3 \times 3$ 的 depthwise 卷积让每个 token 的特征被其周围的 8 个邻居调制。这比相对位置偏置的"纯距离依赖"更丰富——卷积可以学习到特定的局部模式（边缘、角点、纹理）。

**消融实验（论文 Table 5）：**

| 位置编码方案 | CSWin-T Top-1 |
|-------------|--------------|
| 无位置编码 | 81.8% |
| 绝对位置编码（APE） | 82.4%（+0.6） |
| 相对位置偏置（RPE） | 82.5%（+0.7） |
| LePE | 82.7%（+0.9） |

LePE 比相对位置偏置好 0.2 个点，比无位置编码好 0.9 个点。更重要的是，LePE 在下游任务（检测/分割）上优势更大——因为下游任务需要处理不同分辨率的输入，LePE 的"任意分辨率"支持是实际优势。

### 5.4 与 Swin 的位置编码对比

```text
Swin (相对位置偏置):
    注意力 = Softmax(QK^T/√d + B)V
    其中 B = relative_position_bias_table[relative_position_index]
    B 影响注意力权重分布

CSWin (LePE):
    注意力输出 = Softmax(QK^T/√d)V + DWConv(V)
    DWConv 不改变注意力权重，只调制输出特征
```

LePE 的计算量很小——$3 \times 3$ depthwise conv 对一个 $56 \times 56 \times 64$ 的特征图只有 $9 \times 56 \times 56 \times 64 \approx 1.8M$ FLOPs，不到 CSWin-T 总 FLOPs（4.1G）的 0.05%。

## 六、核心公式与数学直觉

### 6.1 复杂度公式对比

**全局 MSA 复杂度：**

$$\Omega(\text{MSA}) = 4HWC^2 + 2(HW)^2C$$

**CSWin 注意力复杂度：**

$$\Omega(\text{CSWin}) = HWC(4C + sw \cdot H + sw \cdot W)$$

**Swin W-MSA 复杂度：**

$$\Omega(\text{W-MSA}) = 4HWC^2 + 2M^2 HWC$$

**数字对比（CSWin-T 的 Stage 1：** $H=W=56, C=64, sw=1, M=7$）：

| 注意力类型 | QKV 投影 | 注意力矩阵 | 总计 |
|-----------|---------|-----------|------|
| 全局 MSA | $4 \times 56^2 \times 64^2 \approx 51.4M$ | $2 \times 56^4 \times 64 \approx 1.26G$ | ~1.31G |
| CSWin | $4 \times 56^2 \times 64^2 \approx 51.4M$ | $2 \times 1 \times 56 \times 56^2 \times 64 \approx 22.5M$ | ~73.9M |
| Swin W-MSA | $4 \times 56^2 \times 64^2 \approx 51.4M$ | $2 \times 7^2 \times 56^2 \times 64 \approx 19.7M$ | ~71.1M |

关键观察：**CSWin 和 Swin 的总 FLOPs 非常接近（73.9M vs 71.1M）**，差异不到 4%。但 CSWin 的每个 token 在单次注意力中可以看到 56 个 token，而 Swin 只能看到 49 个。从"每 FLOP 覆盖的 token 数"看：

- CSWin：单 block 内，每个 token 与 56（行）+ 56（列）- 1（自身）≈ 111 个不同 token 交互
- Swin：单 block 内，每个 token 与 49 个 token 交互

CSWin 的效率更高——用相近的 FLOPs 获得了更大的交互范围。

### 6.2 LePE 的数学形式

标准的 self-attention 输出（无位置编码）：

$$\text{Output} = \text{Softmax}\left(\frac{QK^T}{\sqrt{d}}\right)V$$

CSWin 的带 LePE 输出：

$$\text{Output} = \text{Softmax}\left(\frac{QK^T}{\sqrt{d}}\right)V + \text{DWConv}_{3 \times 3}(V)$$

这里 DWConv 是 depthwise convolution。$V$ 的形状是 $B \times H \times W \times C$。DWConv 后形状不变。

**直觉：** 第一项 $\text{Softmax}(QK^T/\sqrt{d})V$ 是 content-based 的全局依赖（在条带范围内），第二项 $\text{DWConv}(V)$ 是 position-based 的局部依赖。两者互补。

**为什么加性而不是门控（gating）：** 作者没有尝试门控融合（如 $\alpha \cdot \text{Attn} + (1-\alpha) \cdot \text{Conv}$），而是直接用加法。消融实验（论文没有门控的对比）暗示加法已经足够，门控可能带来额外的参数但收益有限。

### 6.3 复杂度分解的深入分析

为了方便起见，将复杂度公式中的每一项拆开理解。

CSWin 注意力有一个"隐藏的特性"值得注意：当 $sw$ 等于 $H$ 和 $W$ 时，水平条带只有一个（覆盖整个 $H \times W$），垂直条带也只有一个——CSWin 退化为全局注意力。

当 $sw=1$ 时，复杂度简化为：

$$\Omega(\text{CSWin}_{sw=1}) = 4HWC^2 + 2HW(H + W)C$$

对比全局 MSA $\Omega(\text{MSA}) = 4HWC^2 + 2(HW)^2C$：

- 第一个 $4HWC^2$ 项完全相同（QKV 投影共享）
- CSWin 的第二项 $2HW(H+W)C$ 中的 $(H+W)$ 远小于全局的 $HW$

当 $H=W$ 时，CSWin 的第二项是 $4H^3C$，全局是 $2H^4C$。CSWin 比全局快 $H/2$ 倍。

## 七、重要图表解释

### 7.1 Figure 1 — 不同自注意力机制的对比

原图展示了 5 种自注意力机制在 multi-head 分配上的差异。

**（b）标准 MSA：** 所有 head 用相同的全局注意力。每个 token 看所有 token。

**（c）Swin W-MSA：** 所有 head 用相同的 $7 \times 7$ 局部窗口注意力。先规则窗口，后偏移窗口（串行）。

**（d）Axial attention：** head 按轴向分组，但水平和垂直是串行的——先水平再垂直。

**（e）Criss-cross：** head 按轴向分组，但窗口是重叠的，计算效率低。

**（f）CSWin（提出的方法）：** 头分两组，水平条带和垂直条带并行计算。条带宽度可调。

这里 CSWin 的核心信息是：**把多头的分组从"所有头做同一件事"改成"头做不同方向的事"**。这个简单的改动引入了方向性感受野。

### 7.2 Figure 2 — CSWin Transformer 架构

原图分两部分：
- 左：整体架构（4 级金字塔 + CSWin Block）
- 右：CSWin Block 的详细内部结构

Block 内部的 LePE 分支值得注意——它显示为注意力输出后的并行分支，直接加到注意力的输出上。这和 Swin Block 不同：Swin 只有注意力（含相对位置偏置），没有并行卷积分支。

### 7.3 Figure 3 — 位置编码方案对比

原图比较了 4 种位置编码在 Transformer Block 中的位置：

- **APE（绝对位置编码）**：在输入到第一个 Transformer block 之前加到 token 上。固定的每个位置对应一个固定的编码向量。
- **CPE（条件位置编码）**：在输入到每个 Transformer block 之前用卷积生成位置编码再加到 token 上。支持任意分辨率但每个 block 都做卷积。
- **RPE（相对位置偏置）**：在每个 Transformer block 内部，偏置加到 attention logits 上。影响注意力分布。
- **LePE（CSWin 的方案）**：在每个 Transformer block 内部，卷积直接作用在 $V$ 值上，与注意力输出相加。不干扰注意力分布，只在输出层面添加位置信息。

可以看出 CSWin 的设计是唯一一个把位置信息作为"并行输出分支"而非"注意力内部的偏置项"的方案。

### 8.1 不同输入分辨率下的复杂度

CSWin 和 Swin 的 FLOPs 随输入分辨率变化的对比：

| 特征图大小 | CSWin-T FLOPs | Swin-T FLOPs | 比例 |
|-----------|--------------|-------------|------|
| 56×56 (Stage 1) | 73.9M | 71.1M | 1.04× |

### 8.2 实际延迟（理论 vs 实测）

论文没有给出详细的延迟对比，但从 FLOPs 的接近度看，CSWin 和 Swin 的推理时间应该非常接近。需要注意的是，CSWin 的并行 head 分组（水平+垂直）在硬件上不是真正的"并行"——PyTorch 的矩阵乘法是顺序执行的。所以 CSWin 的实际延迟可能略高于 FLOPs 等效的 Swin，因为有两个独立的注意力计算路径（每个路径有 reshape 和矩阵乘法操作）。

### 10.3 CSWinAttention 实现

```python
class CSWinAttention(nn.Module):
    def __init__(self, dim, num_heads, split_size=1):
        self.num_heads = num_heads
        self.split_size = split_size
        self.scale = (dim // num_heads) ** -0.5

        # QKV 投影（共享）
        self.qkv = nn.Linear(dim, dim * 3, bias=qkv_bias)
        self.proj = nn.Linear(dim, dim)

    def forward(self, x):
        B, H, W, C = x.shape
        qkv = self.qkv(x)  # [B, H, W, 3*C]

        # 水平条带注意力（使用前 num_heads//2 个头）
        q_h = qkv[:, :, :, :C].reshape(B, H, W, self.num_heads, C//self.num_heads)
        k_h = qkv[:, :, :, C:2*C].reshape(B, H, W, self.num_heads, C//self.num_heads)
        v_h = qkv[:, :, :, 2*C:].reshape(B, H, W, self.num_heads, C//self.num_heads)

        # reshape 成条带格式
        q_h = q_h.permute(0, 3, 1, 2, 4)  # [B, nH, H, W, d]
        # 划分成水平条带: [B, nH//2, H//sw, sw*W, d]  (实际实现更复杂)
        # 做 attention
        attn_h = (q_h @ k_h.transpose(-2, -1)) * self.scale
        attn_h = attn_h.softmax(dim=-1)
        out_h = attn_h @ v_h

        # 垂直条带注意力类似地处理（使用后 num_heads//2 个头）
        # ...

        # 拼接 H 和 V 的注意力输出
        out = torch.cat([out_h, out_v], dim=1)
        out = out.permute(0, 2, 3, 1, 4).reshape(B, H, W, C)
        out = self.proj(out)
        return out
```

代码实现的关键细节：

1. **QKV 共享投影**：$Q$、$K$、$V$ 来自同一线性层的输出（3倍参数），然后 reshape 按头数分配。这和 paper 中的描述一致，节省了参数。

2. **头分组**：前半部分 head 做水平注意力，后半部分做垂直注意力。不需要额外的参数或计算。

3. **条带划分**：通过 reshape + permute 实现，不是通过切片或循环。这利用了 PyTorch 的 view 机制，没有数据复制。

### 10.4 LePE 的实现

```python
class LePE(nn.Module):
    def __init__(self, dim):
        self.conv = nn.Conv2d(dim, dim, 3, 1, 1, groups=dim, bias=False)
        self.bn = nn.BatchNorm2d(dim)

    def forward(self, x):
        # x: (B, H, W, C)
        x = x.permute(0, 3, 1, 2)  # (B, C, H, W)
        x = self.conv(x)
        x = self.bn(x)
        x = x.permute(0, 2, 3, 1)  # (B, H, W, C)
        return x
```

注意：代码中使用了 BatchNorm 而不是 LayerNorm。这是因为 depthwise conv 的输入是 $[B, C, H, W]$ 格式（channels-first），用 BN 更自然。这可能带来小 batch size 下的训练不稳定问题（BN 需要足够的 batch 统计量）。

```python
    shortcut = x
    x = self.norm1(x)
    x = self.attn(x)  # CSWin-Attention + LePE (attn 内部融合了 LePE)
    x = x + shortcut  # 残差

    shortcut = x
    x = self.norm2(x)
    x = self.mlp(x)
    x = x + shortcut  # 残差
    return x
```

标准 Pre-LN 结构，和 Swin/Swin Block 一致。

2. **训练配置**：使用 AdamW（lr=1e-3, weight decay=0.05, betas=(0.9,0.999)），cosine decay schedule，和 Swin 基本一致。但 CSWin 使用了更激进的 augmentation（RandAugment, Mixup, CutMix, RandomErasing）。

3. **MMDetection/MMSegmentation 集成**：官方代码提供了与 mmdetection 和 mmsegmentation 的配置文件，可以直接替换 Swin backbone。

CSWin 处于 B2 分支（hierarchical backbone）的 B2-5 位置。它的直接前置是 Swin，直接竞争是 Focal Transformer（同时期发布）。

### 11.2 前驱工作

**Swin Transformer（Liu et al., 2021）：** 最直接的前驱。CSWin 继承了 Swin 的 4 级金字塔设计、Stage 3 最深、Pre-LN 结构、训练策略等。关键差异只在注意力机制：用交叉形条带替代正方形窗口。

**Axial Self-Attention（Huang et al., 2019）：** 最早提出沿轴计算条带注意力的工作之一。但轴向注意力是串行的（水平→垂直），且条带宽度固定为 1。CSWin 把头分组做并行，并且条带宽度可调，是对轴向注意力的显著改进。

**Criss-cross Attention（Huang et al., 2019）：** 在语义分割中提出的十字交叉注意力，水平和垂直并行。但使用重叠窗口（每个像素位置都有一个十字形），计算效率低。CSWin 使用非重叠条带，高效得多。

### 11.3 同期竞品

**Focal Transformer（Yang et al., 2021, NeurIPS 2021）：** 提出了粗细粒度的窗口注意力。窗口中心区域做细粒度注意力，外围（低分辨率）做粗粒度注意力，在不增加复杂度的情况下扩大了等效感受野。Focal 也是试图解决 Swin 局部感受野问题，但策略不同（粗细粒度 vs 方向条带）。

**CrossFormer（Wang et al., 2022）：** 提出了动态条带宽度和跨尺度的 token embedding。CSWin 中条带宽度是手工设定的（[1,2,7,7]），CrossFormer 尝试让宽度自适应。

**RegionViT（Chen et al., 2021）：** 提出了区域-局部双级注意力——区域内做细粒度注意力，区域间做粗粒度注意力。和 Focal 类似，但采用硬聚类的方式划分区域。

从最终效果看，CSWin 在 ImageNet 分类上表现最好（CSWin-B 84.2% vs Focal-B 83.6%），且参数量最少。

### 11.4 后续影响

CSWin 的影响主要体现在两个方向：

1. **CrossFormer（Wang et al., 2022）：** 直接继承 CSWin 的条带注意力思想，但引入了动态条带宽度和跨尺度 embedding。可以看作 CSWin 的改进版。

2. **EfficientFormer（Li et al., 2022）：** 混合了 CSWin 的交叉形注意力和 MobileNet 的深度可分离卷积，设计轻量级 Transformer。MobileNet 级别的计算量下达到了不错的效果。

3. **MMSegmentation / MMDetection 集成：** CSWin 被很多下游工作用作 backbone。但由于整体提升幅度有限（0.5-1.4%），它没有像 Swin 那样成为 backbone 的标准选择。社区更倾向于在 Swin 基础上做局部改进，而不是完全替换。

## 十三、范式意义

**CSWin Transformer 的范式意义是：它在 Swin 建立的"层级局部注意力"范式中，打开了"方向性感受野"这个新的设计维度。**

具体来说，CSWin 做了三件有意义的事：

### 13.1 证明了头分组 + 方向性注意力的有效性

CSWin 最核心的设计洞察是：**不同 head 可以负责不同的注意力方向**，不必所有 head 都用相同的窗口方案。这个洞察虽然简单，但效果显著（0.5-1.4% 的提升）。后续的很多工作沿用了这个思路——不是把 multi-head 视为冗余的集成，而是视为可以分工的并行计算单元。

### 13.2 用深度可分离卷积做位置编码（LePE）

LePE 的核心思想是：位置编码不必是注意力的一部分（如 RPE 那样加到 attention logits 上），而是可以作为独立于注意力的并行分支。

这个设计有两个启示：
1. **位置编码和内容编码可以解耦**。注意力负责 content-based 交互，卷积负责 position-based 交互，两者之和就是最终的表示。这种解耦在理论上更优雅。
2. **卷积作为位置编码是自然的选择**。$3 \times 3$ depthwise conv 在计算上轻量、在行为上平移等变、在支持上任意分辨率——比任何手工设计的位置编码方案都更适合视觉任务。

后续的很多工作（ConvNeXt, InternImage 等）都用深度可分离卷积作为标准操作，LePE 在其中起到了推动作用。

### 13.3 参数量更少但效果更好的反直觉结果

CSWin 的参数量比 Swin 少 18-30%，但效果更好。这反直觉——通常更少的参数意味着更低的表现力。CSWin 的解释是：交叉形窗口的"有效性"更高——每个参数带来的感受野增益更大。Swin 的局部窗口需要更多参数来补偿窗口的限制（更深的 stage 需要更多 block 来扩大感受野）。

### 13.4 分类：refinement node

在技术树中，CSWin 是一个 **refinement node（精化节点）**——它在 Swin 建立的范式内，对注意力机制做了一次具体的改进（正方形 → 条带）。它不是 foundation node（像 Swin 那样开辟新范式），也不是 bridge node（连接不同领域）。它的价值在于证明：Swin 的范式远未到达天花板，窗口形状的优化空间还很大。

## 十四、局限性与后续方向

### 14.1 分类精度优势在高参数量下变小

CSWin 在 tiny 变体上领先 Swin 1.4%，但在 base 和 large 变体上只有 0.5-0.7%。这意味着随着模型变大，交叉形窗口的优势递减。可能的解释：大模型的深层 block 已经在多次注意力后获得了足够的感受野，方向性信息的边际收益变小。

### 14.2 实际推理延迟可能不占优

虽然 FLOPs 比 Swin 少，但 CSWin 的注意力计算中有两个独立的条带路径（水平和垂直），在 GPU 上无法完全并行。PyTorch/DirectML 的矩阵乘法调度是串行的，因此 CSWin 的两条注意力路径是前后执行的。实际的推理延迟可能比 Swin 略高，尽管 FLOPs 更低。论文没有提供推理延迟数据，这是一个遗憾。

### 14.3 条带宽度的选择仍然是手工的

sw=[1,2,7,7] 是作者手工调出来的，针对 224×224 输入。对于其他输入尺寸或者不同宽高比的任务，最优 sw 配置可能不同。论文没有提供 sw 的自动化选择方案。后续的 CrossFormer 试图解决这个问题（动态条带宽度），但增加了额外的学习参数。

### 14.4 深度方向感受野的考虑

CSWin 的交叉形窗口覆盖 H 轴和 W 轴，但忽略了"对角线方向"。对于一些对角线纹理或斜向物体，水平和垂直条带的组合可能不如直接覆盖斜向方向有效。后续的 Focal Transformer 通过粗细粒度设计在一定程度上解决了这个问题。

### 14.5 没有扩展到视频领域

不像 Swin 有 Video Swin（Liu et al., 2022）那样的 3D 扩展版本，CSWin 没有对应的视频版本。把交叉形条带扩展到 3D 时空域（时间轴 + 空间轴）是一个自然的扩展方向，但 CSWin 的作者没有做这个工作。

### 14.6 工程实现的复杂度

CSWin 的条带划分需要通过复杂的 reshape + permute 操作来实现，代码可读性不如 Swin 的 window partition。在部署到推理框架（TensorRT, ONNX Runtime 等）时，这些动态 reshape 操作可能不支持或需要特殊优化。

## 十五、小结

CSWin Transformer 是 Swin Transformer 的一个直接继承者——它们共享同样的层级设计、同样的训练策略、同样的应用场景。CSWin 的创新点只有一个：**把正方形窗口拆成了交叉形条带**。

这个改动看似微小，但背后的思考是扎实的：
- 图像特征有方向性，用正方形窗口去覆盖它们是低效的
- 把 multi-head 分成两半，分别处理水平和垂直方向，不增加计算量
- 条带宽度可调，浅层用小条带宽省计算，深层用大条带宽扩大感受野
- LePE 用 depthwise conv 替代相对位置偏置，支持任意分辨率

实验效果也清晰：所有 4 个变体都比 Swin 好，参数量更 FLOPs 更少。

CSWin 的 broader impact 在于：它推动了视觉 Transformer 中注意力机制从"各向同性"向"方向性"的转变。后续的 CrossFormer、EfficientFormer 等都沿用了方向性注意力的思路。但对于想要在自己的项目中使用 CSWin 的读者，需要注意两点：
1. CSWin 对 Swin 的提升幅度（0.5-1.4%在分类，1-3%在检测/分割）是"好"但不是"颠覆性"的
2. 实际的推理延迟需要实测验证，不能只看 FLOPs

在技术树中，CSWin 是 B2 分支上对 Swin 的第一次有意义的结构性改进——它证明了 Swin 打开的"层级局部 attention"范式还有优化空间。而 CSWin 之后，社区的关注点逐渐从"窗口还可以怎么划分"转向了"完全不同的 backbone 设计思路"（ConvNeXt 的 CNN 回归、InternImage 的大核卷积等）。
