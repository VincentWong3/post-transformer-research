# Exploring the Limits of Transfer Learning with a Unified Text-to-Text Transformer（2019）

**论文：** [arXiv](https://arxiv.org/abs/1910.10683) · Colin Raffel, Noam Shazeer, Adam Roberts, Katherine Lee, Sharan Narang, Michael Matena, Yanqi Zhou, Wei Li, Peter J. Liu (Google) · 2019（T5）

---

## 一、先搞清楚坑在哪

2018-2019 年，NLP 的迁移学习进入了一个「百花齐放但难以比较」的阶段。BERT（Devlin et al., 2018）用掩码语言模型（Masked Language Model, MLM）做预训练刷新了 11 项 GLUE（General Language Understanding Evaluation）纪录，GPT（Radford et al., 2018）用单向 LM 做生成，XLNet（Yang et al., 2019）用排列语言模型想兼得两者之长。

但问题来了：**这些方法在太多维度上同时不一样。** 预训练目标不同（MLM vs LM vs 排列语言模型 Permutation Language Model, PLM）、模型架构不同（encoder-only vs decoder-only vs encoder-decoder）、数据不同（BookCorpus vs Wikipedia vs Common Crawl）、微调方法不同——你没法判断某个改进到底是来自目标函数、数据量还是架构选择。

学界需要一张「消融地图」：把所有变量控制住，系统地比较每个设计选择的实际影响。但做这个事的成本极高——光预训练一次大模型就要几万美元。

## 二、现有方法的真正问题

当时的 NLP 迁移学习有三个根本性的困境：

**困境 1：任务格式不统一。** BERT 做分类时要加 `[CLS]` token 取输出，做 QA 时要设计 span 预测头，做生成时要接额外的 decoder。每个任务需要定制模型结构，代码和超参都不一样。这让跨任务对比非常困难。

**困境 2：预训练方法太多，缺少系统对比。** MLM（BERT）、语言模型（GPT）、排列语言模型（XLNet）、替换 token 检测（ELECTRA, Efficiently Learning an Encoder that Classifies Token Replacements Accurately）——每个都说是最好的，但用的数据、模型大小、训练步数都不一致。没人知道到底哪个目标函数最有效，因为从来没有人用相同的条件跑过所有这些方法。

**困境 3：数据质量的定义是黑盒。** 大多数论文用 Wikipedia + Toronto Book Corpus (TBC)（~20GB）做预训练。但也有用 Common Crawl（TB 级）的。但 Common Crawl 原始数据极其嘈杂——大量非自然语言文本、乱码、HTML 残留。怎么清洗？不同清洗策略对下游影响多大？没人系统研究过。

T5 的目标就是一次性回答这三个问题。

## 三、T5 的核心思路

**把所有 NLP 任务统一成「text-to-text」格式——输入文本，输出文本。用同一个 encoder-decoder Transformer，统一 objective，统一训练流程，系统对比所有设计选择。**

```
输入："translate English to German: That is good."
输出："Das ist gut."

输入："cola sentence: The course was interesting."
输出："acceptable"

输入："stsb sentence1: The cat sat on the mat. sentence2: The cat is on the mat."
输出："3.8"
```

这个统一框架的一个关键优势是：**可以用完全相同的模型、损失函数、解码流程跑所有任务。** 无论是翻译、摘要、分类还是回归，输入输出都是文本。这就让跨任务对比变得干净。

## 四、网络架构详解

T5 使用 **encoder-decoder Transformer**，基本结构遵循 Vaswani et al. 2017 的原始设计。

### 整体架构

```
输入序列（tokenized）
    │
    ▼
┌─────────────────────────────────────┐
│           Encoder                    │
│  ┌───┐  ┌───┐       ┌───┐           │
│  │ B1│  │ B2│  ...  │ BL│   L 层     │
│  └───┘  └───┘       └───┘           │
│ 每层: Self-Attention → FFN           │
│     (全可见，双向)                     │
└────────────┬────────────────────────┘
             │ encoder output
             ▼
┌─────────────────────────────────────┐
│           Decoder                    │
│  ┌───┐  ┌───┐       ┌───┐           │
│  │ B1│  │ B2│  ...  │ BL│   L 层     │
│  └───┘  └───┘       └───┘           │
│ 每层: Self-Attention → Cross-Attn → FFN │
│     (causal mask)   (attend to enc) │
└────────────┬────────────────────────┘
             │
             ▼
        Softmax 输出
```

### 与原始 Transformer 的差异

T5 在 Vaswani et al. 的基础上做了 3 个小改动：

1. **Layer Norm 位置**：把 layer normalization 移到残差路径**之前**（Pre-Norm），而不是之后（Post-Norm）。Pre-Norm 的训练更稳定，允许更大的学习率。论文使用了 $10^4$ 步 inverse square root 学习率调度。数学上：$x' = x + Sublayer(LayerNorm(x))$。

2. **Layer Norm 简化**：去掉了 LayerNorm 中的加性偏置（bias），只保留缩放（scale）。即 $\text{LayerNorm}(x) = \frac{x - \mu}{\sigma} \odot \gamma$，没有 $\beta$。

3. **相对位置编码**：使用简化版的相对位置 embedding，每个位置偏置是一个标量（不是向量），直接加到 attention logits 上。共 32 个 embedding buckets，按对数间隔从 0 到 128，超过 128 的偏移映射到同一个 bucket。

### 模型规格

| 模型 | 参数量 | 层数 L | d_model | d_ff | heads |
|------|--------|--------|---------|------|-------|
| T5-Small | 60M | 6 | 512 | 2048 | 8 |
| T5-Base | 220M | 12 | 768 | 3072 | 12 |
| T5-Large | 770M | 24 | 1024 | 4096 | 16 |
| T5-3B | 2.8B | 24 | 1024 | 16384 | 32 |
| T5-11B | 11B | 24 | 1024 | 65536 | 128 |

### 前向传播（以 T5-Base 为例）

1. **输入 Embedding**：序列 $x \in \{1,\ldots,V\}^{n}$ （V=32000），映射到 $E \in \mathbb{R}^{n \times 768}$
2. **Encoder 12 层**：每层做 bidirectional self-attention + FFN，输出 $H_{enc} \in \mathbb{R}^{n \times 768}$
3. **Decoder 12 层**：自回归，每步输入之前生成的 token
   - Causal self-attention（只能看过去）
   - Cross-attention（看 encoder 输出 $H_{enc}$）
   - FFN
4. **输出层**：共享 embedding 权重，Softmax 输出 32000 维分布

## 五、核心创新点

### 创新 1：Text-to-Text 统一框架

**（a）痛点与动机：**
在 T5 之前，每类 NLP 任务需要不同的输出头。分类用 `[CLS]` + 线性层，QA 用 span 预测头，生成用 decoder。这意味着同一个模型架构不能直接套用在不同任务上。这不仅增加了工程复杂度，更致命的是——**你没法在一个统一的框架下公平比较不同预训练策略的效果**。

**（b）方案细节：**
T5 把每个任务都格式化成 text-to-text：
- 分类：输入 "imdb review: This movie was great." → 输出 "positive"
- 蕴含：输入 "mnli premise: ... hypothesis: ..." → 输出 "entailment" / "contradiction"
- 回归（STSB）：输入 "stsb sentence1: ... sentence2: ..." → 输出 "3.8"
- 翻译：输入 "translate English to German: That is good." → 输出 "Das ist gut."

所有任务共享同一个 **encoder-decoder 模型、交叉熵损失、自回归解码**。

**（c）为什么有效：**
统一格式的价值在于**让系统对比成为可能**。因为所有任务都走同一个 pipeline，改变一个变量（比如预训练目标）时，其他所有条件完全不变。这消除了之前所有对比研究中的混淆变量。

**（d）与 Related Work 的关系：**
之前已有将多种任务统一为 QA（McCann et al., 2018, "The Natural Language Decathlon"）或语言模型（Radford et al., 2019, GPT-2）的工作。T5 的不同在于它选择 text-to-text 格式，这和 GPT-2 的自回归生成类似，但用了 encoder-decoder 架构而非 decoder-only。

**（e）消融实验：**
论文没有专门做 text-to-text 与任务特定输出头的详细消融。但 Table 14 的结果表明 text-to-text 格式能够在各个任务上达到 SOTA 级别性能，证明统一格式不损失能力。

### 创新 2：系统对比预训练目标

**（a）痛点与动机：**
到 2019 年为止，已提出的预训练目标包括语言模型、BERT-style MLM（预测 mask token）、XLNet 的排列 LM、ELECTRA 的替换 token 检测等。每个都用不同的设置报告，无法直接比较。一个系统性的 head-to-head 对比如今成了社区最需要的东西。

**（b）方案细节：**
T5 在完全相同条件下对比了 4 类预训练目标（论文 Section 3.2）：
1. **语言模型**（LM）：标准从左到右自回归
2. **BERT-style**：预测 mask 掉的 token（15% 概率）
3. **Deshuffling**：打乱句子顺序，预测原始顺序
4. **MASS-style**（Masked Sequence to Sequence）：mask 连续一段，预测整段

（c）实验结果（来自论文 Table 4，使用基础 transformer + 相同训练设置）：

| 预训练目标 | SQuAD (F1) | GLUE (avg) | CNNDM (ROUGE-2) |
|-----------|-----------|-----------|----------------|
| Prefix language modeling | 77.99 | 80.69 | 18.94 |
| BERT-style (mask) | 80.65 | 82.96 | 19.17 |
| Deshuffling | 67.61 | 73.17 | 18.59 |
| **Replace corrupted spans** | **80.88** | **83.28** | **19.24** |

结论：**Replace corrupted spans 在多数任务上最优。** 注意这些是基础模型的绝对分数——关键不是数值本身，而是不同目标之间的相对排序。

**（e）消融：**
论文进一步测试了不同的 mask 策略（论文 Table 6 & Table 7）：mask 15% / 不同的 mask 比例 / corrupt span length。发现 15% mask 配上 **3 的 corrupt span length** 是最优组合，且 denoising 目标统一后，不需要预测整个序列（只用预测 corrupted 部分），比语言模型每个 token 都预测要高效得多。

### 创新 3：C4 数据集与数据质量研究

**（a）痛点与动机：**
几乎所有预训练语言模型都用 Wikipedia + Toronto Book Corpus (TBC)，但这只有 ~20GB。Common Crawl 有 TB 级但质量极差。没人系统研究过：数据量 vs 数据质量，到底哪个更重要？

**（b）方案细节：**
T5 从 Common Crawl 中清洗出 **Colossal Clean Crawled Corpus（C4）**，745GB 纯英文文本。清洗规则包括：只保留以句号/问号/感叹号结尾的句子、丢弃含脏话的页面、丢弃含 Javascript/花括号的页面（编程语言）、去重等。

**（c）实验结果（论文 Table 8，基础 T5 模型）：**
| 数据 | 大小 | SQuAD (F1) | SuperGLUE |
|------|------|-----------|-----------|
| Wikipedia (only) | 16GB | 81.29 | 68.01 |
| Wikipedia + Toronto Book Corpus (TBC) | 20GB | **82.08** | **73.24** |
| C4 (过滤后) | 745GB | 80.88 | 71.36 |
| C4 unfiltered (原始) | 6.1TB | 78.78 | 68.04 |

结论：**更大的数据不一定更好。** Wikipedia + Toronto Book Corpus（20GB）在 SQuAD 和 SuperGLUE 上反而超过了 C4（745GB）。C4 原始未过滤数据虽然远更大（6.1TB）但效果最差。这说明数据质量和领域相关性有时候比数据量更重要。

## 六、公式详解

### 6.1 Masked Language Modeling 目标

标准 MLM 损失（BERT-style）：

$$L = -\mathbb{E}_{x \sim \mathcal{D}} \sum_{i \in \mathcal{M}} \log P(x_i | x_{\setminus\mathcal{M}})$$

其中 $\mathcal{M}$ 是被 mask 的 token 位置集合，$x_{\setminus\mathcal{M}}$ 是被 mask 后的序列。

**直觉**：模型只能看到上下文中没有被 mask 的 token，需要根据 context 预测被 mask 的词。

### 6.2 T5 的 Denoising 目标

T5 使用的是一种变体——**replace corrupted spans**。随机 mask 15% 的 token，但以**连续的 span**为单位 mask（span length = 3），然后让模型预测完整的 span 序列。

输入：`"Thank you <X> me to your party <Y> week."`
输出：`"<X> for inviting <Y> last <Z>"`

其中 `<X>`、`<Y>`、`<Z>` 是哨兵 token（sentinel tokens），编码器输入中被 mask 的 span 被替换成哨兵 token，解码器需要输出这些哨兵 token 对应的原始内容。

**数学形式**：

令 corrupted 序列为 $x'$，原始序列为 $x$。T5 的 loss 是：

$$L = -\log P(x \mid x') = -\sum_{t} \log P(x_t \mid x_{<t}, x')$$

其中 $x_t$ 是被替换掉的连续 span 的 token。

**为什么用 span 而非单 token：**
论文 Table 3 的消融显示：corrupt span length = 3 在大多数任务上最优。直觉上，预测连续的 3 个 token 比预测单 token 更具挑战性，强迫模型学到更强的语言理解能力。

### 6.3 模型并行

11B 参数模型无法放入单个 TPU。T5 使用 **Mesh TensorFlow** 做模型并行，将每层切分到多个设备。关键公式：

通信量：

$$C_{\text{comm}} = L \times d_{\text{model}} \times \# \text{heads} \times \# \text{attention_splits}$$

计算量：

$$C_{\text{compute}} = L \times (6nd^2 + 2n^2d) \times \frac{1}{\# \text{devices}}$$

### 关键实现细节

**Mesh TensorFlow 切分策略：**
T5 将参数矩阵沿着 `d_model` 维度切分到多个 TPU core。每个 core 持有参数的分片，计算时通过 all-reduce 通信中间结果。11B 模型使用一组 TPU v3 core 训练（TPU Pod 最多包含 1024 个 v3 芯片）。

**Pre-Norm 的代码实现：**
```python
def pre_norm(x, layer_fn):
    """Pre-Norm residual connection."""
    return x + layer_fn(layer_norm(x))
```

**自注意力中的相对位置偏置：**
```python
# relative_position_bucket 将偏移映射到 32 个 bucket
position_bias = relative_position_bucket(
    query_pos - key_pos, num_buckets=32, max_distance=128)
position_embedding = embedding_lookup(position_bias)
attention_logits += position_embedding  # 直接加标量
```

## 九、位置

### 前驱工作

- **BERT（Devlin et al., 2018）**：证明了 MLM 预训练的有效性。T5 的 denoising objective 是对 MLM 的推广——从 mask 单 token 扩展到 mask 连续 span。
- **GPT / GPT-2（Radford et al., 2018, 2019）**：证明了 decoder-only 架构 + 语言模型预训练的强大能力。T5 选择 encoder-decoder 而非 decoder-only，因为 encoder-decoder 的 cross-attention 在翻译和摘要等 seq2seq 任务上自然更合适。
- **ULMFiT（Howard & Ruder, 2018）**：第一个系统化的迁移学习流程（预训练 → 微调）。T5 在 scale 和系统性上做了巨大扩展。

### 同期竞品

- **RoBERTa（Liu et al., 2019）**：研究表明 BERT 的很多设计选择并非最优——动态 masking、更大 batch、更多数据。T5 用更系统的方式做了类似但范围更广的消融。
- **ALBERT（Lan et al., 2019）**：参数共享 + embedding matrix 分解。T5-11B 并没有用这些节省参数的技术，而是直接放大。
- **XLNet（Yang et al., 2019）**：排列语言模型 + Transformer-XL。T5 的消融实验显示它的排列方式并不如 BERT-style mask。

### 后续影响

- **mT5（Xue et al., 2020）**：Google 将 T5 扩展到 101 种语言的多语言版本。
- **T5 v1.1 + Flan-T5**：Google 后续版本在 T5 基础上加了 instruction tuning，大幅提升零样本能力。
- **PaLM / Pathways（Chowdhery et al., 2022）**：Google 的后续大型语言模型继承了 T5 的 encoder-decoder 架构（直到 PaLM 2 转向 decoder-only）。
- **C4 数据集的广泛使用**：C4 成为 NLP 预训练的事实标准数据集之一，被 BART（Bidirectional and Auto-Regressive Transformer）、LongT5、甚至 GPT-3 的数据管道引用。

## 十、局限

1. **纯英文，不覆盖其他语言。** C4 只从 Common Crawl 的英文部分提取。虽然 T5 框架本身不依赖任何语言，但 C4 数据限制了 T5 的语言能力。mT5 后来解决了这个问题。

2. **encoder-decoder 架构的计算效率低于 decoder-only。** T5 的生成需要过完整的 encoder 和自回归的 decoder，推理成本大约是 GPT 的两倍。后来的趋势（GPT-3、Chinchilla、PaLM）证明了 decoder-only 的 scaling law 更强。

3. **text-to-text 格式不是万能的。** 对回归任务（如 STSB，分数 1-5），映射到文本再解析回数字有信息损失。论文报告 STSB 上的 Spearman correlation 比其他方法略低。

4. **计算成本极高。** T5-11B 的训练需要 1024 个 TPU v3 core 运行数月。这个成本大多数实验室无法复现。T5 也因此受限于 Google 的基础设施。

## 十一、小结

T5 的核心贡献不是提出某个惊艳的新算法，而是**提供了一个全方位、系统化的 NLP 迁移学习对照实验**。它把整个领域当时的设计选择——架构、目标函数、数据、微调策略——放到同一个实验框架下，给出了当时最全面的答案。

这个工作的 broader impact 有两方面：
1. **C4 数据集**变成了 NLP 预训练的事实标准
2. **text-to-text 范式**影响了后续几乎所有 seq2seq 模型的设计（BART、ByT5、mT5、Flan-T5）
3. 它证明了**统一的框架可以产生更好的科学理解**——当所有变量都可控时，你才能说清楚某个改进到底好在哪里
