# Training Data-Efficient Image Transformers & Distillation through Attention（DeiT，2020）

**论文：** [arXiv 2012.12877](https://arxiv.org/abs/2012.12877) · Hugo Touvron, Matthieu Cord, Matthijs Douze, Francisco Massa, Alexandre Sablayrolles, Hervé Jégou · Facebook AI, Sorbonne University · 2020

**所属技术分支：** B1. 图像如何 token 化？→ B1-2. patch tokenization 范式


## 一、先搞清楚"问题在哪里"

2020 年底，Vision Transformer（ViT, Dosovitskiy et al., 2020）刚扔下一颗重磅炸弹：Transformer encoder 不用任何卷积，就能在图像分类上达到和 ResNet 打平的效果。这是几十年来卷积网络统治视觉领域的第一次真正挑战。

但 ViT 有一个让所有人都皱眉头的条件——**它必须在 JFT-300M（3 亿张图片）上预训练**。在论文自己的实验里，直接在 ImageNet-1k（128 万张）上训练 ViT，top-1 只有 77.9%，略高于 ResNet-50 的 76.2%（+1.7%），但参数量是其 3.4 倍（86M vs 25M），性价比并不理想（虽然接近，但考虑到参数量差距——86M vs 25M——这个结果并不理想）。

ViT 的作者在论文里明确说了一句话——这句话后来被反复引用：

> transformers "do not generalize well when trained on insufficient amounts of data"

翻译成人话就是：你给 Transformer 的数据不够多，它就学不好。

这就带来了一个非常现实的问题：**不是每个人都有 Google 的算力和数据。**

JFT-300M 是 Google 内部数据集，不公开。Google 可以拿 TPU Pod 集群训练 ViT 几天，但学术界、创业公司、甚至大公司的其他团队，手头只有 ImageNet。如果 Transformer 真的大规模落地必须依赖 JFT 级别的数据，那它本质上还是一个"富人的玩具"。

所以 2020 年底，当 Facebook AI 的 Touvron 等人发表这篇论文时，它回答的是一个非常紧迫的问题：

> 能不能只用 ImageNet，把 ViT 训到和 CNN 打平甚至更好？

如果能，Transformer 作为视觉 backbone 的可及性就会从"Google 专属"变成"人均可及"。这个差距就是 DeiT 的切入点。


## 二、旧路线的真正问题

要理解 DeiT 解决的是什么问题，得先搞清楚：ViT 在 ImageNet-1k 上表现不佳，最根本的原因在哪？

### 2.1 数据量的差距不是线性问题

直观想：JFT-300M 有 3 亿张图，ImageNet 有 128 万张，差了 200 多倍。但问题不仅仅是"数据更多所以更好"这么简单。

卷积网络有强烈的**归纳偏置**——平移不变性、局部连接、权重共享。这意味着 CNN 在数据有限的情况下，仍然能用这些先验知识来约束模型的学习空间。你用 100 张猫图片训练一个 ResNet，它至少能学会"边缘 → 纹理 → 局部形状 → 语义"这个层级结构。

Transformer 没有这些归纳偏置。它的 self-attention 在每一层都去看所有 patch 之间的所有交互。这给了它巨大的表达能力，但也意味着**它没有内置任何视觉先验**。它必须从数据中自己学出所有模式——哪些相邻 patch 有关、哪些远距离 patch 有关、局部纹理长什么样。

对于 JFT-300M 这种规模的数据，Transformer 可以学到足够的模式。但在 ImageNet 这个量级，它就像一个有超高分辨率相机但没有自动对焦能力的摄影师——画质上限很高，但大部分时候都在失焦。

### 2.2 训练策略本身的问题

ViT 的原始训练配置（来自原论文）其实相当粗糙：

- **优化器**：AdamW，lr=0.001，weight decay=0.1（对，0.3——非常激进的 weight decay）
- **数据增强**：几乎没有——没有 MixUp、没有 CutMix、没有 RandAugment
- **正则化**：Dropout（p=0.1），没有 stochastic depth
- **学习率策略**：3.4 epoch 的 warmup 后 cosine decay

这种配置在 JFT-300M 上可能够用——数据足够多，模型不会过拟合。但在 ImageNet 上，这个训练策略直接导致了严重过拟合。

另外还有一个容易被忽略的问题：ViT 的参数量（86M 对于 ViT-B）相对 ImageNet 的 128 万张图片来说太大了。模型容量是够的，但训练策略没有对应的正则化来匹配这个容量。

### 2.3 知识蒸馏的潜力没有被探索

ViT 论文完全没有考虑知识蒸馏。但实际上，知识蒸馏（Knowledge Distillation, KD）对 Transformer 来说可能有独特价值——因为 Transformer 缺乏视觉归纳偏置，而 CNN 恰好有。如果能用一个训练好的 CNN 做 teacher，把它的"视觉直觉"蒸馏给 Transformer，理论上可以补偿 Transformer 的归纳偏置缺失。

但在 DeiT 之前，没有人做过这件事。

### 2.4 效率问题

也是一个现实障碍。ViT 在 384×384 分辨率下需要预先在 JFT 上训练，整个过程耗时数天甚至数周（64 个 TPU v3）。即使只做推理，ViT-B 在 V100 上的吞吐量也只有 292 images/sec。相比之下，ResNet-50 可以达到 1226 images/sec。

效率和数据可及性，这是两个卡住 ViT 落地的瓶颈。


## 三、这篇论文的核心思路

一句话：

> **用更好的训练策略 + 独特的蒸馏机制，让 ViT 在只用 ImageNet 的情况下达到甚至超过 CNN 的精度。**

具体拆开看，DeiT 做了三件事：

### 3.1 训练策略的大幅改进

ViT 的原始训练策略对 ImageNet 来说远远不够。DeiT 引入了：

- **更强的数据增强**：Rand-Augment、MixUp、CutMix、随机擦除
- **更强的正则化**：Stochastic Depth（随机深度）、标签平滑
- **改进的优化**：降低 weight decay（0.3 → 0.05）、更小的学习率（0.003 → 0.0005）、更大的 batch size
- **Repeated Augmentation**：同一张图生成多个不同的增强版本，增加 epoch 内的样本多样性

这些改进本身就让 ViT-B 在 ImageNet 上的 top-1 从 77.9% 跳到 81.8%（+3.9%），**不需要额外数据**。

### 3.2 引入蒸馏 token

这是论文最具独创性的贡献。ViT 有一个 class token（用于分类的向量），DeiT 在此基础上**再添加一个 distillation token**：

```text
输入图片
    ↓
patch embedding → [class] [distillation] [patch1] [patch2] ... [patchN]
    ↓
        Transformer Encoder × L 层
    ↓
[class output]  → 分类 head → 真实标签损失
[distil output] → 分类 head → teacher 标签损失
```text

两个 token 都通过 self-attention 和其他 token 交互，但监督信号不同：
- class token：用真实标签训练
- distillation token：用 teacher 模型的预测训练

关键在于——这两个 token 学到的表征是互补的。实验发现它们的余弦相似度只有约 0.06（非常低），说明它们关注了不同的信息。

### 3.3 Hard-label 蒸馏

DeiT 的实验发现了一个反直觉的结果：**Hard-label 蒸馏（把 teacher 的 argmax 当作硬标签）比传统的 soft 蒸馏（KL 散度）效果更好。**

传统 KD：

```text
Loss = (1-λ) × CE(student, y) + λ × τ² × KL(softmax(student/τ), softmax(teacher/τ))
```text

Hard-label 蒸馏：

```text
Loss = 0.5 × CE(student, y) + 0.5 × CE(student, y_teacher)
```text

其中 y_teacher = argmax(teacher_logits)。这个做法更简单（不需要调 τ 和 λ）、在 Transformer 上效果更好（硬标签对缺乏归纳偏置的 Transformer 来说是一种更明确的约束）。

### 3.4 技术路线位置

```text
CNN (ResNet, EfficientNet)
    ↓
ViT (需要 JFT-300M)
    ↓
┌─ DeiT ───────────────────┐
│  Imagenet-only 训练可行   │ ← 我们是这里
│  + 蒸馏 token             │
└───────────────────────────┘
    ↓
Swin Transformer (层级化视觉 backbone)
    ↓
后续改进...
```text


## 四、整体结构与 forward 流程

DeiT 在架构层面完全沿用了 ViT-B，唯一的修改是加了 distillation token。所以 forward 过程可以简单分为两部分——**无蒸馏模式**和**蒸馏模式**。

### 4.1 无蒸馏模式（Vanilla DeiT）

输入 224×224×3 的 RGB 图片：

```text
输入: 224×224×3
    │
    ▼
[Patch Embedding] ── 16×16 patches
    将图片切成 (224/16)×(224/16) = 14×14 = 196 个 patch
    每个 patch 展开为 16×16×3 = 768 维向量
    线性投影到 D 维（DeiT-B: 768, DeiT-S: 384, DeiT-Ti: 192）
    │
    ▼
Token list:  [class] [patch1] [patch2] ... [patch196]
维度:        1×D + 196×D = 197×D
    │ 加位置编码（可学习，197×D）
    │
    ▼
[Transformer Encoder × L 层]
每层:
    LayerNorm → Multi-Head Self-Attention (全局, 所有 token 两两交互) → 残差连接
    LayerNorm → MLP (D → 4D → D, GELU) → 残差连接
    │
    DeiT-B: L=12, h=12 heads
    DeiT-S: L=12, h=6 heads
    DeiT-Ti: L=12, h=3 heads
    │
    ▼
输出: 197×D
    │
    ▼
拿出 class token 对应的向量 → LayerNorm → Linear head → 1000 类 logits
    │
    ▼
Softmax → 预测概率
```text

### 4.2 蒸馏模式（DeiT ⚗）

在 token list 里增加一个 distillation token：

```text
Token list:  [class] [distillation] [patch1] [patch2] ... [patch196]
维度:        1×D + 1×D + 196×D = 198×D
                          ↑ 新加的
```text

两个分类 head：
- **class head**：接受 class token 输出 → 用真实标签计算交叉熵
- **distillation head**：接受 distillation token 输出 → 用 teacher 的硬标签计算交叉熵

推理时，两个 head 的输出可以：
1. 独立使用（只用 class 或只用 distillation）
2. **联合使用（推荐）**：把两个 logits 加在一起分类

有趣的是，实验显示联合分类器在准确率上超过两个独立分类器（Table 3），说明 class 和 distillation token 学到了互补的特征。

### 4.3 Teacher 选择

DeiT 默认的 teacher 是 RegNetY-16GF（84M 参数），在 ImageNet 上达到 82.9% top-1。为什么选一个表现不算最顶尖的模型做 teacher？因为：

1. **规模匹配**：teacher 的参数和 DeiT-B 相当（84M vs 86M），蒸馏效率高
2. **归纳偏置传递**：RegNetY 是 ConvNet，它的归纳偏置可以通过蒸馏传递给 Transformer
3. **同一个训练集**：只用 ImageNet，不依赖外部数据

论文的一个重要发现是：**用 ConvNet 做 teacher 比用 Transformer 做 teacher 效果更好**（Table 2），即使 ConvNet teacher 的精度低于 Transformer teacher。这直接支持了"蒸馏传递归纳偏置"的假说。


## 五、核心创新模块逐个解释

### 5.1 训练策略改进

#### (a) 问题

ViT 在 ImageNet 上的训练策略有严重不足：数据增强弱、正则化弱、优化器参数不合适。这导致模型在 128 万张图片上过拟合。

#### (b) 解决方案

DeiT 系统性地试验了各种数据增强和正则化方法，最终确定了一套优化配置。

**数据增强层（按应用顺序）：**

```text
原始图片
    ↓
Rand-Augment (强度 9, 幅度 0.5)  ← 随机选择增强操作
    ↓
MixUp (α=0.8)                     ← 两张图线性混合
    ↓
CutMix (概率 1.0)                 ← 剪切粘贴混合
    ↓
随机擦除 (概率 0.25)              ← 随机遮挡区域
    ↓
Repeated Augmentation (×3)        ← 同一图片生成 3 个不同版本
```text

**Repeated Augmentation (RA)** 是其中一个关键技巧。它的做法是：对于 batch 里的每张图片，用不同的随机增强生成多次版本。比如 batch size=1024，RA=3，则实际每 epoch 处理的是 1024×3 个不同增强版本的图片。这大大增加了数据多样性。

**正则化：**
- **Stochastic Depth**（p=0.1）：训练时随机 dropout 某些层（注意不是 dropout，是整层跳过的生存概率）。对深层 Transformer 尤其重要。
- **标签平滑**（ε=0.1）：防止模型对预测过于自信
- **没有 Dropout**：实验发现 Dropout 对 DeiT 有害

**优化：**
- AdamW，lr=0.0005×batchsize/512（线性 scaling），weight_decay=0.05
- Cosine 学习率衰减
- 5 epoch warmup
- 梯度裁剪：无

#### (c) 为什么有效

这套组合拳的核心逻辑是：**Transformer 缺乏归纳偏置，就用更强的数据增强和正则化来"人造"泛化能力。**

没有平移不变性？那就用 Rand-Augment 和 MixUp 让它看过各种变换下的同一样本。没有局部连接？那就用 CutMix 和随机擦除逼它学习局部-全局的关系。

有意思的是，实验证明这些增强方法对 Transformer 的效果增幅**远大于对 CNN 的增幅**——因为 CNN 已经内置了一部分鲁棒性，增强的边际效果小；而 Transformer 从零开始学，每一点增强都是实打实的信息。

### 5.2 蒸馏 token

#### (a) 问题

传统的知识蒸馏方法（将 teacher 的 softmax 输出作为额外监督信号）不是为 Transformer 设计的。它附加在分类 loss 上，但不改变模型内部的表征结构。

#### (b) 解决方案

DeiT 的蒸馏 token 是一个全新设计的机制。它的核心思路是：**把 Teacher 的监督作为一种"身份"嵌入到 Transformer 的 token 结构中，让它和 class token 通过 self-attention 自然交互。**

蒸馏 token 有三个设计要点：

1. **和 class token 同构**：都是可学习的 d 维向量，加入 token 序列，经过所有 transformer 层
2. **独立监督**：最后一层输出的蒸馏 token 经过一个独立的分类 head，用 teacher 的硬标签训练
3. **通过 attention 交互**：在每一层自注意力中，蒸馏 token 和 class token、patch tokens 都能互相看

这样设计的妙处在于：class token 负责拟合真实标签，distillation token 负责模仿 teacher，两者在 attention 过程中互相"帮对方看" patch tokens。class token 从 distillation token 那里获得了 teacher 的"视觉直觉"，但不需要暴露在 teacher 的 soft 输出中。

**为什么两个 token 学到的表征会互补？**

一个可能的解释：class token 接收的监督信号是 one-hot 真实标签（数据分布），distillation token 接收的是 teacher 的 hard 预测（模型分布 + 归纳偏置）。ImageNet 上的 object 类别之间天然有层级关系（如"金毛犬"和"拉布拉多犬"），teacher 的预测会在相似类别上有所区分。class token 从真实标签学到的信息是"这是什么"；distillation token 从 teacher 学到的是"这个物体和什么像"。两者结合自然更强。

实验证据：class token 的决策和 DeiT（无蒸馏）的一致性更高（分歧率 0.109），而 distillation token 的决策和 ConvNet teacher 的一致性更高（分歧率 0.100）。这说明两个 token 确实关注了不同的信息源。

#### (c) 如果去掉蒸馏 token

Table 3 显示：DeiT-B 用传统 hard distillation（只有 class token）在 224 分辨率下是 83.0%，而用蒸馏 token（class + distillation 联合）是 83.4%。加了蒸馏 token 还有 0.4% 的提升。在 384 分辨率下差距更大：83.1%（无蒸馏）vs 84.5%（有蒸馏 token）。1000 epoch 后达到 85.2%。

### 5.3 Hard-label 蒸馏

#### (a) 问题

传统 KD（soft distillation）需要调节两个超参数：温度 τ 和平衡系数 λ。在 Transformer 上，这变得更加敏感——因为 Transformer 的 logits 分布和 CNN 可能有不同的分布特性。

#### (b) 解决方案

Hard-label 蒸馏直接把 teacher 的 argmax 当作硬标签：

```text
y_t = argmax(teacher_logits)
Loss = 0.5 × CE(student_logits, y_true) + 0.5 × CE(student_logits, y_t)
```text

不需要 τ、λ、KL 散度，两条交叉熵损失直接相加。

#### (c) 为什么有效

反直觉的结果是：**Hard 比 Soft 在 Transformer 上更好**。在 CNN 上通常 Soft 更好。

一个解释是：Transformer 的归纳偏置弱，soft 标签的知识"太模糊"，Transformer 不知道该从哪里学起。而硬标签是明确的约束——"老师告诉你答案是猫，你去学"。这和标签平滑带来的效果不同：标签平滑是机器主动约束自己，硬标签蒸馏是 teacher 约束学生。

Table 3 的数据很明确：
- Soft distillation：81.8%（和 no distillation 一样！几乎没有提升）
- Hard distillation（class token only）：83.0%（+1.2%）
- Hard distillation（class + distillation token）：83.4%（+1.6%）


## 六、核心公式与数学直觉

DeiT 涉及的核心公式不多。最重要的两个是蒸馏损失函数。

### 公式 1：Soft Distillation（传统 KD）

$$\mathcal{L}_{\text{global}} = (1-\lambda) \mathcal{L}_{\text{CE}}(\psi(Z_s), y) + \lambda \tau^2 \text{KL}(\psi(Z_s/\tau), \psi(Z_t/\tau))$$

**各项含义：**
- $Z_s$：student 模型的 logits（原始输出，未 softmax）
- $Z_t$：teacher 模型的 logits
- $\psi$：softmax 函数
- $y$：真实标签（one-hot）
- $\tau$：温度参数，控制 softmax 的"软度"。τ 越大，概率分布越平滑
- $\lambda$：平衡蒸馏损失和真实标签损失的比例
- $\tau^2$ 乘在前面是为了让梯度大小和 τ 无关

**直觉解释：**

这个公式想让 student 学两件事：
1. 把真实标签分类对（第一项）
2. 模仿 teacher 的预测分布（第二项）

KL 散度衡量的是两个概率分布的差异。当 $\psi(Z_s/\tau)$ 和 $\psi(Z_t/\tau)$ 完全一致时，KL=0。温度 τ 控制 teacher 信息传递的"粒度"：高 τ → teacher 的预测更平滑 → 类别间的细粒度关系（如"拉布拉多"和"金毛"的相似度）更容易传递。

DeiT 发现这个公式在 Transformer 上效果不佳——推测原因是 Transformer 的 logits 分布特性和 CNN 不同，KL 散度提供的梯度信号不够明确。

### 公式 2：Hard-label Distillation（DeiT 采用的）

$$\mathcal{L}_{\text{global}}^{\text{hardDistill}} = \frac{1}{2} \mathcal{L}_{\text{CE}}(\psi(Z_s), y) + \frac{1}{2} \mathcal{L}_{\text{CE}}(\psi(Z_s), y_t)$$

其中 $y_t = \arg\max_c Z_t(c)$

**各项含义：**
- 第一项：student 对真实标签的交叉熵
- 第二项：student 对 teacher 硬标签的交叉熵
- $y_t$：teacher 预测的最可能类别

**直觉解释：**

这个公式比 Soft Distillation 少了一个超参数（τ），少了一个分布匹配操作（KL），但效果反而更好。它的逻辑是：对 Transformer 来说，明确的目标（"老师说是猫"）比模糊的分布（"老师说有 80% 可能是猫，15% 是狗，5% 是老虎"）更容易学。

一个需要注意的点：$y_t$ 随数据增强变化。如果对图片做了随机裁剪，裁剪后可能看不到猫，teacher 预测变为"草地"。这时第二项变成 student 去拟合"草地"标签。这在 soft distillation 中也会发生，但 hard 更明显。论文将此视为优点——teacher 的预测实际上做了隐式的"标签校正"，考虑到数据增强导致的标签-内容错位。

### 公式 3：DeiT 的整体损失（含蒸馏 token）

当使用蒸馏 token 时，student 有两个分类 head，分别对应 class token 和 distillation token：

$$\mathcal{L} = \frac{1}{2} \underbrace{\mathcal{L}_{\text{CE}}(\psi(Z_s^{\text{class}}), y)}_{\text{class token 损失}} + \frac{1}{2} \underbrace{\mathcal{L}_{\text{CE}}(\psi(Z_s^{\text{distil}}), y_t)}_{\text{distillation token 损失}}$$

其中 $Z_s^{\text{class}}$ 是 class 头输出的 logits，$Z_s^{\text{distil}}$ 是 distillation 头输出的 logits。

训练时两条交叉熵独立计算，通过同一个 student backbone 反向传播。两个 token 的梯度同时更新 backbone 参数，但各自的分类 head 只接收自己的梯度。

## 八、复杂度、效率与可扩展性分析

### 8.1 计算复杂度

DeiT 的架构和 ViT-B 相同，所以理论和 ViT 一样：

- Self-attention 复杂度：$O(N^2 D)$，其中 $N=197$（196 patches + 1 class token）
- MLP 复杂度：$O(N D \cdot 4D) = O(4ND^2)$
- 蒸馏 token 增加 1 个 token：$198 \times 198 / 197 \times 197 \approx 1.01$——复杂度增加约 1%

理论上说，蒸馏 token 几乎不增加计算量。实际上 DeiT-B⚗ 的吞吐量（291 im/s）和 DeiT-B（292 im/s）也几乎完全一致。

### 8.2 训练效率

DeiT 在训练效率上的提升才是真正的亮点：

| 指标 | ViT-B（原始） | DeiT-B |
|------|-------------|--------|
| 预训练数据 | JFT-300M | ImageNet-1k |
| 训练设备 | 64 TPU v3 | 8 GPU V100 |
| 训练时间 | 数日 ~ 数周 | 53 小时（预训练） |
| 微调时间 | - | 20 小时 |
| 单卡训练 | 不可行 | 4 GPU, 3 天 |

从"Google 专属"到"一台 8-GPU 工作站跑两天"，这是 DeIT 在可及性上的最大贡献。

### 8.3 效率瓶颈

DeiT 的效率瓶颈仍然存在：

1. **吞吐量不如轻量级 CNN**：DeiT-B（292 im/s）vs ResNet-50（1226 im/s）——虽然参数多 3 倍，但吞吐量差距超过 4 倍。全局 self-attention 的内存访问模式不如卷积的 sliding window 缓存友好。

2. **推理时的 batch 大小限制**：全局 attention 的内存占用随输入分辨率平方增长。384×384 分辨率（N=576 patches）下，DeiT-B 的吞吐量掉到 85.8 im/s。

3. **小模型效率**：DeiT-Ti（5M 参数）的吞吐量 2536 im/s 很不错，但精度只有 72.2%。Transformer 在极小参数下不如轻量级 CNN。

### 8.4 可扩展性

DeiT 证明了 ViT 在 ImageNet 上训练是可行的，但没有解决 ViT 架构本身的扩展性问题：
- 全局 attention 的 $O(N^2)$ 复杂度限制了高分辨率输入
- 单分辨率特征图不适合检测/分割任务
- 这些限制由后续工作（Swin, PVT）解决

关键的 forward 函数：

```python
def forward_features(self, x):
    # x: (B, 3, H, W)
    
    # Patch embedding
    x = self.patch_embed(x)  # → (B, N, D)
    
    # 添加 class token 和 distillation token
    cls_tokens = self.cls_token.expand(B, -1, -1)
    dist_tokens = self.dist_token.expand(B, -1, -1)
    x = torch.cat((cls_tokens, dist_tokens, x), dim=1)
    # → (B, 1 + 1 + N, D)
    
    # 加位置编码
    x = x + self.pos_embed
    
    # 过 Transformer blocks
    x = self.blocks(x)  # → (B, 2+N, D)
    
    # LayerNorm
    x = self.norm(x)
    
    # 分别取出 class 和 distillation 的输出
    return x[:, 0], x[:, 1]
    # x[:, 0]: class token output (B, D)
    # x[:, 1]: distillation token output (B, D)

def forward(self, x):
    x_cls, x_dist = self.forward_features(x)
    
    # 两个分类头
    x_cls = self.head(x_cls)
    x_dist = self.head_dist(x_dist)
    
    if self.training:
        # 训练时返回两条输出，损失函数分开计算
        return x_cls, x_dist
    else:
        # 推理时，两个 logits 相加再做 softmax
        return (x_cls + x_dist) / 2
```text

### 10.3 蒸馏损失函数（losses.py）

```python
class DistillationLoss:
    def __init__(self, base_criterion, teacher_model, distillation_type, alpha):
        self.base_criterion = base_criterion  # 标准的交叉熵
        self.teacher_model = teacher_model       # 冻结的 teacher 模型
        self.distillation_type = distillation_type  # 'none', 'soft', 'hard'
        self.alpha = alpha  # 蒸馏损失的权重
        
    def __call__(self, inputs, outputs, labels):
        # outputs 是 (student_cls, student_dist)
        student_cls, student_dist = outputs
        
        # 1. class token 对真实标签的交叉熵
        loss_cls = self.base_criterion(student_cls, labels)
        
        if self.distillation_type == 'none':
            return loss_cls
        
        # 2. 蒸馏损失
        with torch.no_grad():
            teacher_logits = self.teacher_model(inputs)
        
        if self.distillation_type == 'soft':
            # Soft distillation: KL 散度
            loss_kd = kl_div(
                log_softmax(student_dist / tau),
                softmax(teacher_logits / tau)
            )
            loss = (1 - self.alpha) * loss_cls + self.alpha * tau**2 * loss_kd
        
        elif self.distillation_type == 'hard':
            # Hard distillation: 直接交叉熵
            teacher_label = teacher_logits.argmax(dim=1)
            loss_dist = self.base_criterion(student_dist, teacher_label)
            loss = 0.5 * loss_cls + 0.5 * loss_dist
        
        return loss
```text

## 十一、Related Work 与技术谱系位置

### 11.1 它继承了谁

- **[Vaswani et al., 2017] Transformer**：DeiT 的底子是标准的 Transformer encoder，保留了 QKV attention、multi-head、FFN、残差连接这些核心组件
- **[Dosovitskiy et al., 2020] ViT**：直接继承了 ViT-B 的架构（patch embedding + class token + 12 层 Transformer），没有动任何架构层面的东西
- **[Hinton et al., 2015] Knowledge Distillation**：继承了知识蒸馏的基本范式（teacher-student），但针对 Transformer 做了关键改进
- **[Wightman, 2019] timm library**：使用了 timm 中 ViT 的 BERT-style 训练技巧（如 stochastic depth、重复增强），timm 的 ViT 训练已经达到 79.35% 的 top-1（原始 ViT 只有 77.9%）

### 11.2 它修正了什么

- **"Transformer 需要大量数据"这个观点的修正**：ViT 论文认为 Transformer 在数据不足时泛化能力差。DeiT 证明了这不是 Transformer 的先天缺陷，而是**训练策略的问题**——用合适的数据增强和正则化，ViT 在 ImageNet-1k 上的训练完全可行
- **蒸馏方法对 Transformer 的适配**：传统 KD 是为 CNN 设计的，直接套用到 Transformer 上效果很差（soft distillation 几乎没用）。DeiT 的 hard-label distillation + distillation token 是针对 Transformer 设计的蒸馏范式

### 11.3 它替代了谁

在"ImageNet 分类 + 不需要 JFT 预训练"这个条件下，DeiT 直接替代了 ViT-B。DeiT-S（22M, 79.8%）替代的是 ResNet-50 级别的模型（25M, 76.2%），DeiT-Ti（5M, 72.2%）替代的是 ResNet-18 级别（12M, 69.8%）。

但注意：DeiT 没有替代 CNN 在**检测/分割**上的地位。它只做了分类。当 ViT 被用作检测/分割的 backbone 时，需要后续的 Swin Transformer 来解决多尺度和效率问题。

1. **ViT 刚发布 2 个月**——领域内正高度关注"Transformer 到底能不能做视觉"这个问题
2. **结果好但门槛高**——所有人都意识到 ViT 的潜力，但 JFT-300M 的门槛让绝大多数团队无法复现
3. **timm 库已经包含改进版 ViT 训练**——Ross Wightman 在 timm 中做了大量优化尝试，DeiT 站在这些工作基础上

## 十三、范式意义

### 分类：训练策略改进 + 结构创新（蒸馏 token）

DeiT 的贡献可以分为两个层面：

**层面 1——训练策略改进（工程价值高，学术影响大）**

证明了 ViT 只需 ImageNet-1k 即可训练到 83.1%（无蒸馏）。这个结论的影响巨大——它告诉社区：
> 不需要等 Google 公开 JFT，也不需要自己花几百万美元去标注数据，直接用 ImageNet 就够了。

这个结论直接推动了视觉 Transformer 研究的"平民化"。

**层面 2——蒸馏 token（结构创新）**

蒸馏 token 是一个简洁但巧妙的设计。它不是"把蒸馏 loss 加到总 loss 上"这种平凡做法，而是把蒸馏的监督信号嵌入到 Transformer 的 token 结构中，通过 self-attention 自然交互。这个设计在后续工作中被广泛采用（如 DINO 对 ViT 的自监督蒸馏）。

### 在技术树中的节点类型

```text
范式节点分类：
主路线（主干创新）：Swin Transformer, DETR, ViT
桥梁节点（可行性验证）：DeiT
增强节点（工程/策略改进）：DeiT
分支节点（旁支路线）：LeViT, T2T-ViT
```text

**DeiT 同时是桥梁节点和增强节点。**

作为桥梁节点，它连接了"ViT 需要巨大数据"和"ViT 在 ImageNet 上可行"这两个阶段。没有 DeiT 的验证，后续的视觉 Transformer 工作（尤其是需要 ImageNet 预训练的 Swin、DINO）的出发点会弱很多。

作为增强节点，它改进了 ViT 的训练策略和蒸馏方法。这些改进中的绝大部分（数据增强组合、Stochastic Depth、Repeated Aug）成为了后来所有视觉 Transformer 训练的默认配置。

### 范式改变：从"数据量决定一切"到"训练策略决定一切"

DeiT 真正重要的启示不是"加了蒸馏 token"，而是它证明了：

> ViT 在 ImageNet-1k 上表现不佳不是架构本身的问题，而是训练策略没有跟上。

这句话听起来简单，但它的影响是深远的。在这之前，领域内倾向于认为 Transformer 需要海量数据是因为它"没有归纳偏置"——这是一个架构的先天限制。DeiT 证明了：只要训练策略足够好（数据增强、正则化、优化），这个限制可以被大幅缓解。

这个思路后来被 ConvNeXt（2022）反向使用——把 ConvNet 往 Transformer 的训练策略上靠，结果 ConvNet 也能达到 Transformer 级别的效果——进一步佐证了"架构差距很大程度上是训练策略差距"这个命题。


## 十四、局限性与后续方向

### 14.1 架构层面的局限

1. **全局 attention 的扩展性问题没有解决**。DeiT 仍然使用 ViT 的全局 self-attention，在 384×384 分辨率下吞吐量从 292 掉到 85.8 im/s。更高分辨率或 dense prediction 任务（检测、分割）无法高效处理。

2. **单分辨率特征图**。DeiT 的输出始终是 14×14 或 24×24（分辨率相关），无法作为 FPN/U-Net 的多尺度 backbone。这限制了它在检测/分割领域的应用。

3. **小模型效率不如 CNN**。DeiT-Ti（5M, 72.2%）比 ResNet-18（12M, 69.8%）略好，但参数量接近 2.5 倍——而且 ResNet-18 的吞吐量更高。在计算资源严重受限的场景下（移动端、嵌入式），DeiT 没有优势。

### 14.2 蒸馏方法的局限

1. **依赖一个好的 ConvNet teacher**。DeiT⚗ 的效果高度依赖于使用的 teacher。如果手头没有训练好的 ConvNet，蒸馏效果会大打折扣。这增加了实际部署的成本。

2. **Teacher 和学生使用相同的训练数据**。Teacher 在 ImageNet 上训练，Student 也在 ImageNet 上受 Teacher 监督——本质上 Student 的知识来源仍然是 ImageNet。如果 teacher 本身有 bias（如特定类别偏好），这些 bias 会通过蒸馏传递给学生。

3. **多教师蒸馏未被探索**。论文只使用单一 teacher。理论上使用多个 teacher（不同架构、不同初始化）可能带来更好的蒸馏效果，但未被验证。

### 14.3 评估的局限

1. **只做了分类和迁移学习**。DeiT 没有在目标检测或语义分割上进行评估（因为架构限制——只有单分辨率输出）。这意味着它的"通用性"只停留在分类层面。

2. **CIFAR 等小数据集上"作弊"嫌疑**。论文报告了在 CIFAR-10/100 上的迁移效果（DeiT-B⚗ 在 CIFAR-100 上达到 93.7%），但 CIFAR 图片尺寸小（32×32），ViT 的 patch size（16×16）意味着每张图只有 4 个 patch——几乎退化为 bag-of-words 模型。

### 14.4 后续方向

DeiT 直接开启了几个后续方向：

- **Swin Transformer（2021）**：解决多尺度和效率问题，让 Transformer 成为通用 backbone
- **DINO（2021）**：把 DeiT 的蒸馏思想扩展到自监督学习——用自蒸馏替代 Teacher-Student 范式
- **DeiT 的 1000 周期训练**：论文发现长周期训练对蒸馏效果提升显著（83.4%→84.2%），暗示可能需要更好的训练策略而非更长的 epoch
- **训练策略的组合优化**：Table 8 显示大部分增强/正则化方法都是累加的，但部分组合可能有相互作用。后续工作可以系统性地分析这些组合的交互效应


## 十五、小结

直白说，DeiT 的价值不只是提出了一个改进训练策略和一个蒸馏 token。

**它真正改变的是"Transformer 做视觉"这件事的门槛。**

在 DeiT 之前：想用 Transformer 做视觉分类，要么有 JFT-300M（基本只有 Google），要么接受 ViT-B 在 ImageNet 上只有 77.9%——和 MobileNet 一样的水平，还不如 ResNet-50。这条路线看起来"有潜力但不可及"。

在 DeiT 之后：任何有一台 8-GPU 机器（甚至 4 卡）的团队，都能在 2-3 天内训练出一个和 EfficientNet-B5 打平的 Transformer 分类器。如果再用 ConvNet 做蒸馏，精度可达 85.2%——超过了带 JFT 预训练的 ViT-B（84.15%）。

**它让"ViT 只能做分类"变成了"ViT 可以做实用的分类"。**

这为后续所有视觉 Transformer 工作扫清了基础障碍。Swin Transformer 之所以能直接使用 ImageNet 预训练，很大程度上是因为 DeIT 的工作证明了这条路可行。DINO 的自监督蒸馏也借鉴了蒸馏 token 的思想。

在技术树中，DeiT 是一块基石——不是最耀眼的，但是没有它，上面的一切都不稳。
