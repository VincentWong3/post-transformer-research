# Training Data-Efficient Image Transformers & Distillation through Attention（2020）

**论文：** [arXiv](https://arxiv.org/abs/2012.12877) · Hugo Touvron, Matthieu Cord, Matthijs Douze, Francisco Massa, Alexandre Sablayrolles, Hervé Jégou · Facebook AI · ICML 2021

## 一、先搞清楚坑在哪

ViT 证明了 Transformer 可以做图像分类，但有一个致命限制：需要 JFT-300M（3 亿图片）级别的预训练数据才能超过 ResNet。在 ImageNet-1K（130 万图片）上从头训练，ViT-B 只有 79.9% top-1，低于同时代的 ResNet-152（78.6%）。对于绝大多数没有 3 亿图片的团队来说，ViT 只是理论成果。

## 二、ViT 小数据训练差的真正原因

ViT 在 ImageNet-1K 上表现差，不是因为架构不好，而是因为训练策略不对。CNN 训练积累了多年经验——数据增强、正则化、学习率调度。ViT 不能直接套用这些经验：

- ViT 的 token 数少（196），过拟合风险高
- 没有卷积的局部归纳偏置，需要更强的正则化
- 位置编码需要更多的数据来学习空间关系

关键洞察：ViT 缺的不是数据，而是正确的训练方法。

## 三、DeiT 的核心思路

一句话：给 ViT 配上现代训练配方，让它在 ImageNet-1K 上从头训练达到 SOTA。

三大贡献：
1. **训练配方**——数据增强（RandAugment、Mixup、CutMix）+ 正则化（Stochastic Depth、Label Smoothing）
2. **知识蒸馏**——用 CNN teacher 蒸馏 Transformer student，配合蒸馏 token 机制
3. **无需 JFT-300M**——只用 ImageNet-1K 训练，DeiT-B 达到 83.8%

## 四、架构详解

DeiT 的架构几乎和 ViT 一样——标准 Transformer encoder + patch embedding + [class] token。唯一的改动是新增了一个 **distillation token**：

```
Input: 224×224×3
    │
    ▼
Patch Embedding (16×16, stride=16) → 196 patches × 768 dim
    │
    ▼
[class token] + [distill token] + 196 patch embeddings + 位置编码 → 198 × 768
    │
    ▼
Transformer Encoder ×12（与 ViT-B 完全一致）
    │
    ┌──────┴──────┐
    │              │
[class] token  [distill] token
    │              │
分类头 (K classes)  蒸馏头 (K classes)
  hard label loss   蒸馏 loss (teacher soft labels)
```

Distillation token 是一个可学习的 token，位置在 [class] token 之后。训练时 [class] 学真实标签，[distill] 学 teacher 的 soft label。两个 token 共享 Transformer encoder 参数。推理时两者任意选一个或取平均。

## 五、核心贡献：训练配方

### 数据增强
- **RandAugment**（Cubuk et al., 2020）：自动数据增强
- **Mixup**（Zhang et al., 2018）：两张图按比例混合
- **CutMix**（Yun et al., 2019）：区域替换式混合
- **Random Erasing**：随机擦除图像区域

### 正则化
- **Stochastic Depth（DropPath）**：以概率 p 跳过整个 Transformer block。p 从 0 线性增长到 0.1（浅层跳过概率低，深层高）。这是 ViT 论文没用的技巧
- **Label Smoothing**：标签平滑
- **Gradient Clip**：梯度裁剪

### 优化器
- AdamW，cosine LR decay，linear warmup 5 epoch
- Weight decay 0.05，batch size 1024，epochs 300

## 六、公式详解

### Hard-label Distillation

$$\mathcal{L}_{\text{global}} = \frac{1}{2}\mathcal{L}_{\text{CE}}(\psi(\mathbf{z}_s), \mathbf{y}) + \frac{1}{2}\mathcal{L}_{\text{CE}}(\psi(\mathbf{z}_t), \mathbf{y}_t)$$

$\mathbf{z}_s$: [class] token 输出，$\mathbf{z}_t$: distill token 输出，$\mathbf{y}_t = \text{argmax}(\mathbf{z}_{\text{teacher}})$ — teacher 的硬标签。

### Soft Distillation

$$\mathcal{L} = (1-\lambda)\mathcal{L}_{\text{CE}}(\psi(\mathbf{z}_s), \mathbf{y}) + \lambda \tau^2 \text{KL}\left(\frac{\psi(\mathbf{z}_t)}{\tau}, \frac{\psi_{\text{teacher}}(\mathbf{x})}{\tau}\right)$$

$\tau$ 是蒸馏温度，温度越高 teacher 的输出越平滑，传递更多信息。$\lambda$ 是蒸馏权重。实验显示 $\tau=1, \lambda=0.5$ 最优。

### Stochastic Depth

$$\mathbf{x}_{l+1} = \mathbf{x}_l + \frac{1}{1-p_l} \cdot \text{Block}_l(\mathbf{x}_l) \cdot \text{Bernoulli}(1-p_l)$$

$p_l$ 从第 0 层到第 $L-1$ 层线性增长：$p_l = \frac{l}{L-1} \cdot p_{\max}$，$p_{\max}=0.1$。深层 block 被 drop 的概率更大。

## 七、实验结果

| 模型 | ImageNet top-1 | 备注 |
|------|---------------|------|
| ViT-B (from scratch) | 79.9 | — |
| DeiT-B (无蒸馏) | 83.6 | 全靠训练配方 |
| **DeiT-B (Conv 蒸馏)** | **83.8** | +0.2% 来自蒸馏 token |

消融实验清晰表明：Stochastic Depth 是最大贡献者（79.9 → 83.1），蒸馏 token 只锦上添花（83.6 → 83.8）。

## 八、代码对照

GitHub: facebookresearch/deit。关键发现：

**1. 蒸馏 token 实现。** `distillation_token` 是 `nn.Parameter`，初始化 zeros，`trunc_normal_(std=.02)`。与 [class] token 拼接。

**2. 数据增强 pipeline。** 使用 timm 库的 RandAugment + CutMix + Mixup。CutMix 和 Mixup 同时修改图片和标签。

**3. 没有使用位置编码插值。** DeiT 只在 224×224 训练和评估，不涉及高分辨率 fine-tune。

## 九、位置

### 前驱

- **[Dosovitskiy et al., 2020] ViT**：基础架构。
- **[Cubuk et al., 2020] RandAugment**：自动数据增强方法。
- **[Huang et al., 2016] Stochastic Depth**：随机深度正则化，DeiT 引入 ViT 的关键技巧。

### 后续影响

- DeiT 的训练配方成了 ViT 训练的"标准配置"——后续几乎所有 ViT 变体（Swin、PVT、CSWin）都在使用 DeiT 风格的数据增强和正则化。
- **[Steiner et al., 2021] How to Train Your ViT?** 系统验证了 DeiT 配方各组件的作用。
- **[Touvron et al., 2022] DeiT III / ResNet Strikes Back**：把 DeiT 配方应用到 ResNet，证明 CNN 也能从中受益。

## 十、局限

1. **依赖 CNN teacher。** DeiT 需要 RegNetY-16GF 作为 teacher，增加了流程复杂度。
2. **Stochastic Depth 需要调参。** $p_{\max}=0.1$ 是经验值，随深度变化需要调整。
3. **没有解决多尺度问题。** 输出单分辨率特征图，不能做检测/分割。
4. **蒸馏 token 的改进有限。** 只贡献 +0.2%，统计显著性边缘。

## 十一、小结

DeiT 证明了 ViT 在 ImageNet-1K 上的失败是训练问题，不是架构问题。它做对了两件事：系统化的训练配方（+3.7%）和蒸馏 token（+0.2%）。DeiT 在 ViT 和 Swin 之间架了一座桥——证明 Transformer 可以在中小规模数据上工作，为后来 Swin、ConvNeXt 的大规模应用铺平了道路。
