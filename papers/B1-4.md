## 一、先搞清楚"问题在哪里"

2017 年 Transformer 在 NLP 领域横空出世之后，视觉领域一直有人在想：能不能把 Transformer 直接拿来处理图像？这条路在 2020 年被 Google 的 ViT 证明是可行的——只要把图像切成 patch，每个 patch 拉平成向量当作 token，扔进标准的 Transformer encoder，就能在图像分类上取得有竞争力的结果。

但 ViT 有一个巨大的问题：数据量需求。

ViT 论文里写得非常清楚——"transformers do not generalize well when trained on insufficient amounts of data"。他们用 ImageNet-1K（120 万张图）直接训练 ViT，效果远不如同量级的 ResNet。只有先在 JFT-300M（3 亿张图）上做大规模预训练，ViT 才能追上甚至超过卷积网络的巅峰水平。

这个结论在当时造成了两个直接影响。

第一个是**门槛问题**。JFT-300M 是 Google 的内部数据集，不公开。学术界和其他公司根本拿不到。如果 ViT 必须依赖这种级别的数据才能工作，那对绝大多数研究者和工程师来说，ViT 就是纸上谈兵——你看得到它的潜力，但你用不了它。

第二个是**范式问题**。ViT 论文的结论似乎在暗示：Transformer 缺乏卷积网络的归纳偏置（translation equivariance、locality），所以必须用海量数据来补偿。如果这个推论是真的，那"用 Transformer 替代 CNN"这条路就需要相当大的数据投入才能走通。

这就引出了一个必须要回答的问题：ViT 到底是真的需要这么多数据，还是只是因为训练方法没做对？

这篇论文就是冲着这个问题去的。

## 二、旧路线的真正问题

ViT 的训练策略基本是直接从 NLP 搬过来的。这样做不是没有道理的——模型结构就是 Transformer，训练方法照搬应该差不到哪去。但实际效果说明，这种"结构照搬 + 训练照搬"的组合在视觉任务上碰到了几个具体的麻烦。

### 2.1 数据增强的缺失

NLP Transformer 的基本训练策略是：大量数据 + 少量增强（甚至不需要增强）。BERT 用的就是 masked language modeling + next sentence prediction，没有任何数据增强。但图像数据和文本数据有个根本区别：图像的语义对几何变换（平移、旋转、缩放、裁剪）不敏感，同一个语义标签对应着无限多种可能的像素排列。如果训练时不引入足够的数据增强，模型就没有动力去学习这种不变性。

ViT 用的是 ImageNet-1K 标准的训练流程，数据增强比较弱（随机裁剪 + 水平翻转）。这对于卷积网络来说够用，因为卷积本身就有 translation equivariance 的归纳偏置。但对 Transformer 来说，没有这种偏置，数据增强就是唯一的补偿手段——但 ViT 没给它。

### 2.2 正则化的缺失

Transformer 本身是一个高容量模型。ViT-B 有 86M 参数，ViT-L 有 307M。用这么高容量的模型在 120 万张图上训练，不做认真的正则化一定会过拟合。ViT 论文里用了 weight decay（0.3）和 dropout（0.1），但效果仍然不够。

这里有一个微妙的矛盾：NLP 里 Transformer 的大规模预训练靠的是数据量而不是正则化，所以开发出来的正则化技术（如 stochastic depth）在 NLP Transformer 中并不是标配。但视觉场景下，数据量不足的问题迫使你必须依赖强大的正则化，而这些技术在 ViT 的原始训练流程里并没有被系统应用。

### 2.3 训练超参数未充分调优

ViT 论文把重点放在了"大规模预训练 + 迁移学习"这个范式上，对于 ImageNet-1K 直接训练的场景，超参数的调优空间其实没有充分探索。学习率、优化器、训练轮数、batch size 这些关键参数，都是根据 JFT-300M 上的经验设定的，直接套用到 ImageNet-1K 上未必最优。

### 2.4 缺乏针对 Transformer 的蒸馏策略

ViT 论文没有讨论蒸馏。但如果我们想用 Transformer 替代 CNN，一个现实问题是：训练 Transformer 需要的数据和计算资源仍然比 CNN 多。如果能用一个大 CNN 或大 Transformer 作为 teacher，蒸馏出一个小学生 Transformer，那么部署阶段的效率问题就能缓解。

但蒸馏本身也有问题：传统蒸馏（soft distillation）把 teacher 的 softmax 输出的完整分布作为监督信号，对于 Transformer 来说是不是最优选择？有没有更贴合 Transformer 结构的蒸馏方式？这些问题在 DeiT 之前没有人系统地研究过。

### 小结

所以 DeiT 要解决的问题可以概括为三点：

1. **ViT 在只有 ImageNet-1K 时效果不好，不是因为结构有问题，而是训练策略不对。**
2. **用正确的训练策略（更强的数据增强 + 正则化 + 更长的训练），ViT 可以在没有外部数据的情况下达到有竞争力的结果。**
3. **Transformer 的蒸馏需要一个结构相关的改进，而不是简单地套用 CNN 时代的蒸馏方法。**

## 三、这篇论文的核心思路

这篇论文的核心思路是：**ViT 不需要外部数据，只需要正确的训练食谱。**

具体来说，作者做了两件事。

第一件事是"食谱工程"。他们把当时已知的各种有效的训练技巧——RandAugment、mixup、cutmix、stochastic depth、repeated augmentation、EMA——组合在一起，配合更小的 batch size、更小的 learning rate、更长的训练 schedule，在仅用 ImageNet-1K 的情况下把 ViT-B 从原来的 77.9% top-1 推到了 81.8%。

第二件事是"结构创新"。他们在 ViT 的基础上加了一个**蒸馏 token（distillation token）**——在输入序列中，除了 class token 之外再放一个额外的 token，这个 token 的目标是学习 teacher 的预测，而不是真实标签。这个 token 通过 self-attention 与 patch tokens 和 class token 交互，然后在输出层通过一个独立的分类头做预测。这个看起来很小的改动，实际上比传统的蒸馏方式（对 logits 做 KL 散度）效果要好得多。

有意思的发现是：**用卷积网络做 teacher 比用另一个 Transformer 做 teacher 效果更好**。这说明蒸馏不只是传递知识，还能传递归纳偏置——卷积网络的 locality 和 translation equivariance 通过蒸馏可以被 Transformer 学生学到。

```text
输入图像
     ↓
    patch embedding (16×16 patches → 768-d vectors)
     ↓
    [CLS token] + [DIST token] + [patch tokens] + [position embedding]
     ↓
    Transformer encoder (12 layers, MSA + FFN)
     ↓
    [CLS output] → linear head → 真实标签预测
    [DIST output] → linear head → teacher 标签预测
```

## 四、整体结构与 forward 流程

DeiT 的架构基本就是 ViT，只是在输入序列里多了一个蒸馏 token。下面按 forward 顺序走一遍。

### Step 1: Patch Embedding

输入是一个 RGB 图像，尺寸为 H×W×3（DeiT 默认用 224×224）。

图像被切成 16×16 的 patch，步长 = 16（即非重叠切分）。对于 224×224 的输入，patch 的数量为：

N = (224/16) × (224/16) = 14 × 14 = 196

每个 patch 是一个 16×16×3 = 768 维的向量，通过一个可训练的线性投影层（其实就是不带偏置的全连接层）映射到 D 维空间（D 取决于模型变体：DeiT-Ti 是 192，DeiT-S 是 384，DeiT-B 是 768）。

shape 变化: (B, 3, 224, 224) → (B, 196, D)

### Step 2: 拼接 token

在 patch tokens 前面拼接两个特殊的 token：

- **[CLS token]**：shape (1, D)，可训练，与 ViT 中的 class token 完全相同。最终用它的输出做真实标签分类。
- **[DIST token]**：shape (1, D)，可训练，是这篇论文的新增内容。最终用它的输出做 teacher 标签分类。

拼接后序列长度为：1 (CLS) + 1 (DIST) + 196 (patches) = 198

shape: (B, 198, D)

### Step 3: 位置编码

加上可训练的位置编码（position embedding），shape 也是 (1, 198, D)。

注意这里比 ViT 多了 1 个位置（因为多了 DIST token），所以 position embedding 的维度从 (1, 197, D) 变成了 (1, 198, D)。

### Step 4: Transformer Encoder

标准的 ViT encoder，由 L=12 层相同的 block 堆叠而成。每个 block 包含：

1. **LayerNorm** → **Multi-head Self-Attention (MSA)** → **残差连接**
2. **LayerNorm** → **MLP (FFN)** → **残差连接**

MSA 的参数：
- 每个 head 的维度 d = 64
- head 数量 h = D/64
- DeiT-Ti: h=3, d=64; DeiT-S: h=6, d=64; DeiT-B: h=12, d=64

MLP 的内层扩展倍数为 4，即 D → 4D → D，激活函数为 GELU。

关键点：**所有 198 个 token 在 attention 层中完全交互**。也就是说，CLS token 和 DIST token 不仅与 patch tokens 交互，它们彼此之间也通过 attention 交互。这是蒸馏 token 能够工作的核心机制——它不是一个独立的蒸馏分支，而是嵌入在 Transformer 的统一 attention 计算中。

shape 变化: (B, 198, D) → 经过 12 层 block → (B, 198, D)

### Step 5: LayerNorm

对 encoder 输出做最后一层 LayerNorm。

### Step 6: 分类头

取 CLS token 和 DIST token 的输分别送入两个独立的线性分类头：

- head：CLS → 1000 类 logits（真实标签预测）
- head_dist：DIST → 1000 类 logits（teacher 标签预测）

### Step 7: 推理时的决策

在训练时，两个分类头分别输出自己的预测结果，分别算 loss（见下文）。

在推理时，两个分类头的 softmax 输出做平均：

```python
return (x + x_dist) / 2
```

这个简单的 late fusion 策略比单独使用任何一个 token 的效果都要好。

## 五、核心模块逐个解释

### 5.1 为什么需要蒸馏 token？

先看一下传统蒸馏（soft distillation）是怎么工作的。

Hinton 2015 年提出的 knowledge distillation，核心思想是让 student 模型不仅学习真实标签，还学习 teacher 模型的"软"输出——也就是 teacher softmax 输出的完整概率分布。这个分布的熵比 one-hot 标签高，包含了类间相似性信息（比如"猫"和"老虎"的概率都比"汽车"高），可以作为额外的监督信号。

传统蒸馏的 loss 为：

ℒ_global = (1-λ) ℒ_CE(ψ(Z_s), y) + λ τ² KL(ψ(Z_s/τ), ψ(Z_t/τ))

其中 Z_s 和 Z_t 分别是 student 和 teacher 的 logits，ψ 是 softmax，τ 是温度参数，λ 是平衡系数。

这种方法的本质是在 **logits 层面**做蒸馏——student 的整个输出向量被约束去接近 teacher 的输出向量。这有两个问题：

1. **学习路径间接**。student 是通过整个输出分布的匹配来学习 teacher 的知识，对于 Transformer 这样一个高度非线性的模型来说，teacher 的监督信号需要在反向传播中穿过整个网络，效率不高。
2. **与 Transformer 结构不耦合**。传统蒸馏的设计是模型无关的，没有利用 Transformer 特有的 token 交互机制。

### 5.2 蒸馏 token 的工作原理

DeiT 的蒸馏 token 的思路完全不一样：**不是在共享分类头上施加蒸馏损失，而是引入一个独立的蒸馏 token，通过 attention 收集信息，由专用分类头输出预测供蒸馏使用。**

具体做法是：
- 在输入序列中加入一个额外的 DIST token，它的初始化方式和 CLS token 一样（truncated normal, std=0.02）。
- DIST token 与所有其他 token（包括 CLS token）通过 self-attention 交互。
- 在输出时，DIST token 的 embedding 经过一个独立的线性分类头 head_dist，预测 teacher 的标签。
- **DIST token 只受 teacher 标签的监督，不受真实标签的监督。**

这样做的好处是：

**第一，学习路径直接。** DIST token 通过 attention 从 patch tokens 中"收集"信息，直接学习模仿 teacher 的预测。这比在 logits 层面做 KL 散度更符合 Transformer 的设计哲学——token 本来就是用来承载信息的。

**第二，DIST token 和 CLS token 学习到的表征是互补的。** 论文中报告了两个 token 在中间层的余弦相似度非常低（平均 0.06），说明它们关注的是不同的特征。即使到了最后一层，相似度也只上升到 0.93——仍然不是 1。这说明两个 token 从相同的 patch tokens 中提取了不同的信息。

作为对比，作者做了一个对照实验：**在 Transformer 中加入两个 CLS token（而不是一个 CLS + 一个 DIST）**。结果两个 CLS token 在训练中完全趋同（cos=0.999），分类性能没有任何提升。这说明 DIST token 的有效性不是因为它是一个"额外的 token"，而是因为它的监督信号与 CLS token 不同——它学的是 teacher 的知识，而不是真实标签。

### 5.3 Hard-label distillation

除了蒸馏 token 这个结构创新，DeiT 还引入了一个蒸馏方法上的改进：**hard-label distillation**。

传统 soft distillation 用 KL 散度匹配 teacher 和 student 的完整 softmax 分布。Hard-label distillation 的做法更简单：直接把 teacher 的 argmax 预测（硬标签）当作真实标签来用。

loss 公式：

ℒ_global^hardDistill = (1/2) ℒ_CE(ψ(Z_s), y) + (1/2) ℒ_CE(ψ(Z_s), y_t)

其中 y_t = argmax_c Z_t(c) 是 teacher 的硬决策。

这个做法的好处是：
- **无参数**：不需要 τ 和 λ 的调优
- **概念简单**：teacher 的预测 y_t 被当作另一个真实标签
- **效果更好**：在 DeiT-B 上，hard distillation（83.0%）明显优于 soft distillation（81.8%）

有趣的是，hard label 会随着数据增强的变化而变化——同一张图经过不同的增强，teacher 的预测可能不同。这实际上给训练增加了额外的多样性。

### 5.4 数据高效训练的关键技术

DeiT 能成功的关键，除了蒸馏 token，还有一整套训练技巧的组合。下面逐个解释。

#### (a) Repeated Augmentation

这是**最重要的单项技巧**之一。传统的 data augmentation 是每张图像在每次 epoch 中被随机增强一次。Repeated augmentation 的做法是：**每张图像在同一 batch 中被增强多次**，即同一个 batch 里包含同一张图的不同增强版本。

这样做的好处是：在 batch size 固定的情况下，repeated augmentation 让模型在同一 batch 中看到更多不同的增强视图，有助于稳定训练。DeiT 用了 3 次重复，这意味着形式上 300 epoch 的训练实际上只看了原始图像 100 个 epoch 的内容（但因为每个 epoch 的 batch 里都是 3 倍重复的增强，训练时间不变）。

#### (b) RandAugment

一个简单的自动增强策略，只有两个参数（N, M）：N 是每张图应用的增强操作数量，M 是增强强度。DeiT 用了 N=2, M=9（基于 RandAugment 原始论文的设置）。

#### (c) Mixup & CutMix

- **Mixup**：以概率 0.8 将两张图按 λ 比例混合，标签也按相同比例混合
- **CutMix**：以概率 1.0 将一张图的 patch 贴到另一张图上（这个 1.0 表示在该 batch 中的所有样本都应用 CutMix，但实际论文中的用法是 CutMix 和 Mixup 轮流使用）

这两者的作用相同：迫使模型学习局部特征，而不是只依赖某些特征（比如背景）。对于缺乏 locality bias 的 Transformer 来说，这种正则化尤其重要。

#### (d) Stochastic Depth (DropPath)

以一定概率随机跳过整个 Transformer block。DeiT 中用了 0.05（DeiT-S）到 0.1（DeiT-B）的 drop rate。这相当于训练了一个深度较浅的网络的 ensemble，能有效缓解过拟合。

对于 Transformer 来说这特别有用——NLP 中 Transformer 通常不（或很少）用 stochastic depth，但视觉 Transformer 因为数据量相对较小，stochastic depth 的正则化效果非常显著。

#### (e) EMA (Exponential Moving Average)

在训练过程中维护模型参数的滑动平均：θ_EMA = 0.99995 × θ_EMA + 0.00005 × θ。

EMA 模型在训练过程中提供了约 0.1 个百分点的精度提升，但 fine-tune 之后差距消失。说明 EMA 的作用主要在训练稳定性上，而不是模型质量本身。

#### (f) 超参数设置

和 ViT 的关键区别：

| 超参数 | ViT-B (原始) | DeiT-B |
|--------|-------------|--------|
| Epochs | 300 | 300（可扩展到 800-1000） |
| Batch size | 4096 | 1024 |
| Learning rate | 0.003 | 0.0005 × batchsize/512 |
| Optimizer | AdamW | AdamW |
| Weight decay | 0.3 | 0.05 |
| Stochastic Depth | 无 | 0.1 |
| Repeated Aug | 无 | 有 |
| RandAugment | 无 | 有 (9/0.5) |
| Mixup | 无 | 0.8 |
| CutMix | 无 | 1.0 |
| Dropout | 0.1 | 无（用 stochastic depth 替代） |
| Label smoothing | 有（ε=0.1） | 有（ε=0.1） |

最关键的差异是 **batch size 和 learning rate**。ViT 用 4096 的 batch size 和 0.003 的 lr，而 DeiT 用 1024 的 batch size 和按比例缩放的 lr（lr = 0.0005 × batchsize/512）。小 batch size + 小 lr 的组合使得训练更稳定，同时降低了对大规模分布式训练基础设施的依赖——DeiT-B 只用 8 张 V100 就可以在 2-3 天内完成训练，而 ViT 需要数十甚至上百张 TPU。

## 六、核心公式与数学直觉

### 6.1 Attention 公式

标准 scaled dot-product attention：

$$\mathrm{Attention}(Q,K,V) = \mathrm{Softmax}(QK^\top / \sqrt{d}) V$$

其中 Q、K、V 来自输入 X 的线性变换：Q = XW_Q, K = XW_K, V = XW_V。

这个公式表面上是做"查询-键"匹配，实际上做的是**信息路由**——每个 token 计算自己对其他 token 的注意力权重，然后从所有 token 的 value 向量中加权聚合信息。

在 DeiT 中，由于有 CLS 和 DIST 两个特殊 token，attention 模式更丰富。两个特殊 token 都会关注 patch tokens（收集视觉信息），也会互相关注（交换各自的"任务导向"信息）。最终 CLS token 学会了从 patch 中提取"这是什么东西"的信息，而 DIST token 学会了提取"teacher 认为这是什么东西"的信息。

### 6.2 蒸馏 loss

Soft distillation：

$$\mathcal{L}_{\mathrm{global}} = (1-\lambda)\mathcal{L}_{\mathrm{CE}}(\psi(Z_s), y) + \lambda \tau^2 \mathrm{KL}(\psi(Z_s/\tau), \psi(Z_t/\tau))$$

这里 KL 散度衡量的是 teacher 和 student 的完整概率分布之间的差异。τ 控制分布的"平滑度"——τ 越大，分布越平滑，类间的细微关系越能被 student 学到。

Hard-label distillation（DeiT 的默认选择）：

$$\mathcal{L}_{\mathrm{global}}^{\mathrm{hardDistill}} = \frac{1}{2}\mathcal{L}_{\mathrm{CE}}(\psi(Z_s), y) + \frac{1}{2}\mathcal{L}_{\mathrm{CE}}(\psi(Z_s), y_t)$$

两个 loss 都是标准的交叉熵，只不过一个是针对真实标签 y，另一个是针对 teacher 的硬预测 y_t。

为什么 hard 比 soft 好？论文没有给出严格的数学证明，但可以从直觉上理解：对于图像分类来说，teacher 的 softmax 分布中除了 top-1 之外的概率往往包含着"第二候选"的信息，但这些信息不一定都是有用的——有时只是反映了 teacher 的不确定性，而非真正的类间关系。直接用 hard label 实际上做了 feature selection，去掉了 teacher 输出中的噪声。

### 6.3 Learning rate scaling

$$\mathrm{lr}_{\mathrm{scaled}} = \frac{\mathrm{lr}}{512} \times \mathrm{batchsize}$$

这是 DeiT 的 lr scaling 公式（Goyal 2017 的 linear scaling rule，但 base 从 256 改成了 512）。物理含义是小 batch size 下梯度估计噪声更大，所以 lr 应该成比例减小。

ViT 用的 batch size 4096，lr 0.003。如果按 DeiT 的公式换算：lr_base = 0.003 × 512/4096 ≈ 0.000375。而 DeiT 实际用的 lr（batch size 1024）是 0.0005 × 1024/512 = 0.001。比线性缩放的理论值稍大，但处于同一个量级。

---

### 6.4 推理阶段分类头融合

推理时，[CLS] token 和 [DIST] token 经过各自的分类头后输出取平均：

$$\text{logit}_{\text{final}} = \frac{1}{2} \left( \text{head}(\mathbf{z}_{[CLS]}) + \text{head}_{\text{dist}}(\mathbf{z}_{[DIST]}) \right)$$

训练时两个分支各自独立优化（一个对真实标签，一个对 teacher 预测），推理时平均结合两者。论文消融显示，平均优于取单个分支的输出。

## 八、复杂度、效率与可扩展性分析

### 8.1 计算复杂度

DeiT 的 Transformer encoder 的计算复杂度完全由 ViT 继承：

- **Self-attention**：O((N+2)² × D)，其中 N=196, D=768（DeiT-B）
- **FFN**：O((N+2) × 8D²)，因为内层 4D，两层共 8D²

相比于 ViT（N+1=197 个 token），DeiT 多了 1 个 DIST token，复杂度增加是线性的：

(N+2)² / (N+1)² = 198² / 197² ≈ 1.01

所以蒸馏 token 的计算开销几乎可以忽略不计——不到 1%。

### 8.2 参数数量

| 模型 | Embed Dim | Heads | 参数 | throughput (im/sec) |
|------|-----------|-------|------|-------------------|
| DeiT-Ti | 192 | 3 | 5M | 2536 |
| DeiT-S | 384 | 6 | 22M | 940 |
| DeiT-B | 768 | 12 | 86M | 292 |
| DeiT-B ↑384 | 768 | 12 | 86M | 85.9 |

如果加上蒸馏 token，模型参数略有增加（多了 DIST token 的 embedding 参数和 head_dist 的参数），但总量增加不到 1M。

### 8.3 训练成本

这是 DeiT 最亮眼的指标之一：

- DeiT-B（300 epoch），8 张 V100：53 小时
- DeiT-S 和 DeiT-Ti（300 epoch），4 张 GPU：不到 3 天
- Fine-tune 到 384 分辨率，8 张 GPU：20 小时（25 epoch）

对比 ViT-B 在 JFT-300M 上的预训练：数十到上百张 TPUv3，训练时间以周为单位。

### 8.4 扩展性分析

DeiT 的训练策略有几个设计上的优势，使得它更容易扩展到更大的模型和数据：

1. **没有 BatchNorm**：不依赖 batch statistics，可以在单卡上用小 batch size 训练，不影响精度。
2. **Stochastic Depth 可调**：随着模型变深，可以增加 drop rate 来控制过拟合。
3. **Repeated Augmentation**：增加了每个 batch 内的数据多样性，对大规模并行训练时的大 batch size 也有帮助（因为每张卡上同一张图的增强版本不同，梯度更新包含了更多的增强信息）。

### 9.2 蒸馏方法对比

| 方法 | 监督方式 | DeiT-Ti 224 | DeiT-S 224 | DeiT-B 224 | DeiT-B ↑384 |
|------|---------|-------------|------------|------------|-------------|
| 无蒸馏 | 仅标签 | 72.2 | 79.8 | 81.8 | 83.1 |
| 传统 soft distillation | soft | 72.2 | 79.8 | 81.8 | 83.2 |
| 传统 hard distillation | hard | 74.3 | 80.9 | 83.0 | 84.0 |
| **蒸馏 token (CLS 头)** | hard | 73.9 | 80.9 | 83.0 | 84.2 |
| **蒸馏 token (DIST 头)** | hard | 74.6 | 81.1 | 83.1 | 84.4 |
| **蒸馏 token (CLS+DIST)** | hard | **74.5** | **81.2** | **83.4** | **84.5** |

可以看到：
- 传统蒸馏（soft 或 hard）加的是 loss 层面的约束，不改变模型结构
- 蒸馏 token 改变了模型结构（多了 DIST token），但不改变 loss（同理是 hard distillation）
- CLS+DIST 的 late fusion 效果最好，说明两个 token 学到的信息是互补的

### 9.3 Teacher 选择的影响

|---------|-------------|------------|-------------|
| DeiT-B（自蒸馏） | 81.8 | 81.9 | 83.1 |

| RegNetY-16GF | 82.9 | 83.1 | 84.2 |

关键结论：**用 convnet 做 teacher 的效果优于用 Transformer 做 teacher**。RegNetY-4GF（80.0%）做 teacher 时，学生已经达到 82.7%，超过了用 DeiT-B（81.8%）自蒸馏的 81.9%。

这意味着弱一点的卷积网络比强一点的 Transformer 更适合做 teacher。这种"跨架构蒸馏"的效果暗示：卷积网络的归纳偏置通过蒸馏被传递给了 Transformer。

### 核心类：DistilledVisionTransformer

`models.py` 中最重要的类是 `DistilledVisionTransformer`，继承自 timm 的 `VisionTransformer`。

```python
class DistilledVisionTransformer(VisionTransformer):
    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        # 新增 DIST token
        self.dist_token = nn.Parameter(torch.zeros(1, 1, self.embed_dim))
        # pos_embed 从 (1, N+1, D) 变为 (1, N+2, D)，多了一个位置给 DIST
        num_patches = self.patch_embed.num_patches
        self.pos_embed = nn.Parameter(torch.zeros(1, num_patches + 2, self.embed_dim))
        # 蒸馏分类头
        self.head_dist = nn.Linear(self.embed_dim, self.num_classes)
```

`forward_features` 方法和 ViT 唯一的区别是多了 DIST token 的 concat 和拆包：

```python
def forward_features(self, x):
    B = x.shape[0]
    x = self.patch_embed(x)
    cls_tokens = self.cls_token.expand(B, -1, -1)
    dist_token = self.dist_token.expand(B, -1, -1)
    # 关键行: 拼接顺序 [CLS, DIST, patch tokens]
    x = torch.cat((cls_tokens, dist_token, x), dim=1)
    x = x + self.pos_embed
    x = self.pos_drop(x)
    for blk in self.blocks:
        x = blk(x)
    x = self.norm(x)
    # 返回 CLS 和 DIST 的输出
    return x[:, 0], x[:, 1]
```

`forward` 方法在训练和推理时行为不同：

```python
def forward(self, x):
    x, x_dist = self.forward_features(x)
    x = self.head(x)            # CLS → 真实标签
    x_dist = self.head_dist(x_dist)  # DIST → teacher 标签
    if self.training:
        return x, x_dist        # 训练时返回两个logits
    else:
        return (x + x_dist) / 2 # 推理时平均
```

### 蒸馏 loss 实现

`losses.py` 中实现了 hard distillation：

```python
# 简单直接：两个交叉熵，各一半权重
loss = 0.5 * ce_loss(student_logits, labels) + 0.5 * ce_loss(student_logits, teacher_labels)
```

### Repeated Augmentation

`samplers.py` 中的 `DistributedSampler` 实现了每个样本在同一个 epoch 中出现多次（每次不同 augmentation）。

核心逻辑：不是在一个 epoch 中遍历所有图像一次，而是遍历 `num_replicas * num_samples * repeated_aug` 次，每次从同一张图像采样不同的 augmentation。

### 模型注册函数

`models.py` 底部有 8 个注册函数，对应 8 个模型变体：

- `deit_tiny_patch16_224`（5M，无蒸馏）
- `deit_small_patch16_224`（22M，无蒸馏）
- `deit_base_patch16_224`（86M，无蒸馏）
- `deit_tiny_distilled_patch16_224`（6M，有蒸馏）
- `deit_small_distilled_patch16_224`（22M，有蒸馏）
- `deit_base_distilled_patch16_224`（87M，有蒸馏）
- `deit_base_patch16_384`（86M，384 分辨率）
- `deit_base_distilled_patch16_384`（87M，384 分辨率，有蒸馏）

非蒸馏版本直接用 `VisionTransformer`，蒸馏版本用 `DistilledVisionTransformer`。

## 十一、Related Work 与技术谱系位置

DeiT 属于"B1. 图像如何 token 化？——范式 B1-2: patch tokenization 范式"这条路线。

```text
ViT (B1-3, 2020)
    │
    ├── DeiT (B1-4, 2021) ← 本篇
    │       │
    │       ├── 解决 ViT 的数据依赖问题
    │       ├── 引入蒸馏 token
    │       └── 证明 Transformer 可在无外部数据下训练
    │
    ├── T2T-ViT (2021)     ← 另一种减少数据需求的方法
    ├── Swin Transformer (2021) ← 分层 Transformer 路线
    └── DINO (2021)        ← 自监督版本 DeiT
```

### 11.1 它继承了谁？

- **ViT (Dosovitskiy et al., 2020)**：直接继承了 ViT 的 patch embedding + Transformer encoder + class token 架构。DeiT-B 和 ViT-B 的架构完全一致。
- **timm (Wightman, 2020)**：训练流程大量借鉴了 timm 库中已有的 ViT 训练改进（stochastic depth、RandAugment 等），而 DeiT 作者之一 Ross Wightman 本身就是 timm 的作者。
- **FixEfficientNet (Touvron et al., 2020)**：fine-tuning 策略来自同一团队之前的 FixEfficientNet 工作（低分辨率预训练 + 高分辨率 fine-tune）。

### 11.2 它修正了谁？

- **ViT 的"数据饥饿"问题**：DeiT 用事实证明 ViT 不需要 JFT-300M 这样的超大规模数据集，只要训练策略正确，ImageNet-1K 就够了。这修正了 ViT 论文中最常被引用的结论。
- **传统知识蒸馏**：DeiT 用蒸馏 token 替代了 logits 层面的 KL 散度，证明结构耦合的蒸馏策略比通用的蒸馏方法更适合 Transformer。

### 11.3 它替代了谁？

DeiT 提供了一类新的"backbone"候选：纯 Transformer 的视觉骨干网络。在 DeiT 之前，如果不用外部数据，视觉 Transformer 的精度远不如卷积网络。DeiT 填补了这个差距——它证明 Transformer 可以在同等数据下和 CNN 竞争。

### 11.4 它启发了谁？

- **DINO (Caron et al., 2021)**：Facebook 同一团队在 DeiT 之后的半年内提出了 DINO——将 DeiT 的自蒸馏思想扩展到自监督学习。DINO 使用 DeiT 的架构，但用蒸馏 token 做自蒸馏（teacher 是 student 自己的 EMA 版本）。
- **DeiT III (Touvron et al., 2022)**：DeiT 作者后续改进了 DeiT 的训练策略，进一步缩小了与 SOTA 的差距。
- **各种 ViT 训练技巧的后续研究**：DeiT 建立的"强数据增强 + 强正则化 + 长训练"范式成为后续所有 ViT 训练的默认方案。

## 十二、时间线位置

```text
2017.06  Transformer (Vaswani et al.) — NLP 领域的标准架构
2020.05  ViT (Dosovitskiy et al.) — 第一个纯 Transformer 图像分类模型，需要 JFT-300M
2020.12  DeiT (Touvron et al.) — 无需外部数据的 ViT 训练 + 蒸馏 token
2021.04  Swin Transformer (Liu et al.) — 分层 Transformer，通用视觉 backbone
2021.06  DINO (Caron et al.) — DeiT 的自监督版本
2021.10  MAE (He et al.) — 掩码图像重建，另一种 ViT 高效训练方法
2022.01  DeiT III (Touvron et al.) — DeiT 训练策略的进一步改进
```

DeiT 出现在 ViT 发布后的第 4 个月。这是一个很特殊的时间窗口：

- ViT 刚刚证明了 Transformer 在视觉上的潜力，但也暴露了对海量数据的依赖
- Google 有 JFT-300M，但学术界和中小公司没有
- Facebook AI 显然意识到了这个问题，并且有动机去解决它

这个时间点的意义在于：**DeiT 的任务不是做"比 ViT 更好的模型"，而是做"在同等条件下和 CNN 竞争的 ViT"**。这个定位决定了它看起来不像 ViT 那样"颠覆性"，但它的实际影响力可能更大——因为它让 Transformer 从"Google 的玩具"变成了"大家都能用的工具"。

从技术路线上看，DeiT 和 ViT 属于同一支（patch tokenization → global self-attention），但 DeiT 的这个改进是决定性的：它让这条路线变得可行——你不必拥有 Google 的算力和数据就能用上 Vision Transformer。

## 十三、范式意义

### 13.1 训练范式层面的贡献

DeiT 最大的范式意义是：**它证明了 ViT 的成功不依赖大规模数据，而依赖正确的训练策略。**

这个结论等价于说：Transformer 的"数据饥饿"不是一个结构缺陷，而是一个训练问题。这意味着所有研究 Transformer 的人可以把注意力从"怎么搞到更多数据"转移到"怎么训练更有效"上。

这是训练策略层面的一次 paradigm shift：ViT 的范式是"大规模预训练 → 下游微调"，而 DeiT 证明"单数据集强训练"也可以达到类似的水平。这降低了 Vision Transformer 的准入门槛。

### 13.2 蒸馏方法的范式意义

蒸馏 token 的提出是 Attention 层面的创新：**不是在共享分类头上施加蒸馏损失，而是引入一个独立的蒸馏 token 和专用分类头。** 这个思路切换很微妙但很重要：

- Logits 层面的蒸馏是"模型无关"的，适用于任何架构
- Token 层面的蒸馏是"模型相关"的，它利用了 Transformer 特有的 token 交互机制

这种"针对架构设计蒸馏方法"的思路是当时没有被充分探索的。它打开了后续研究的一个方向——Token-level distillation，后来在 DINO（自蒸馏）中得到了更完美的体现。

### 13.3 "跨架构蒸馏"的发现

DeiT 发现了卷积网络比 Transformer 更适合做 Transformer 的 teacher。这个发现的意义超出了分类任务本身：

- 它说明归纳偏置是可以"蒸馏传递"的——CNN 的 locality 和 translation equivariance 通过蒸馏可以被 Transformer 学到
- 它为"用更好的 teacher 教更灵活的 student"这一范式提供了有力证据

### 13.4 节点类型判断

DeiT 在技术树中的位置：

- **不是根本性范式变化**（不是第一个 Transformer 视觉模型，不改变 Transformer 的基本结构）
- **不是旁支优化**（不是某个细节的改进）
- 而是 **bridge node（桥接节点）** ——它把 ViT 从"只能在大公司内部跑"的不可实际状态，推进到了"可以作为通用 backbone 使用"的可用状态

如果 ViT 是"这条路走得通"，DeiT 就是"这条路大家都可以走"。

## 十四、局限性与后续方向

### 14.1 架构未变

DeiT 的模型架构与 ViT 完全一致，没有引入任何对视觉任务更友好的归纳偏置。这意味着它和 ViT 共享相同的限制：比如对局部纹理的建模能力弱于 CNN、对小目标的处理不如分层架构、在检测和分割等密集预测任务上的表现需要进一步验证（DeiT 论文只做了分类和简单迁移学习）。

### 14.2 数据增强的依赖性

DeiT 的成功很大程度上依赖于强力数据增强（RandAugment、mixup、cutmix、repeated augmentation）。如果数据量进一步减少（比如只有几万张图像的场景），这些方法的有效性会下降。论文中在 CIFAR-10 上从零训练的结果（98.5%）虽然表明 Transformer 可以从少量数据开始学习，但仍然比不上 ImageNet 预训练后的结果（99.1%）。

### 14.3 蒸馏对 teacher 的依赖

DeiT 的蒸馏方法依赖一个高质量的 teacher（RegNetY-16GF，82.9%）。如果没有这样的 teacher，蒸馏 token 的收益会下降。这是一个实用的限制：不是所有人都有资源和时间先去训练一个好的 RegNetY teacher。

### 14.4 超参数调优成本

DeiT 的训练策略涉及大量超参数（learning rate、weight decay、stochastic depth rate、mixup/cutmix 概率、repeated augmentation 次数等），这些超参数针对 ImageNet-1K 做了调优。迁移到其他数据集时，这些超参数可能需要重新调优。

### 14.5 EMA 的收益有限

论文报告 EMA 在原始训练中的收益只有 0.1 个点，fine-tune 后消失。EMA 虽然在实践中常用，但在 DeiT 的配置下效果并不显著。

### 14.6 后续方向

- **自监督版本的 DeiT**：这就是 DINO 做的工作——把蒸馏 token 的 teacher 从外部模型换成 student 自身的 EMA 版本
- **更强的数据增强**：论文提到"针对 Transformer 设计的数据增强"可能带来进一步提升
- **扩展到更多视觉任务**：DeiT 在检测、分割、视频理解等任务上的表现需要在后续工作中验证
- **更大的学生模型**：DeiT 的蒸馏方法在更大的 Transformer（如 ViT-L/ViT-H）上的表现需要进一步实验
- **训练策略的进一步优化**：DeiT III 后来做了这件事——通过改进训练 recipe，进一步提升了性能

## 十五、小结

DeiT 这篇论文的价值不在于提出了一个全新的架构，而在于做了一件看起来不那么性感但极其重要的事情：它证明了 Vision Transformer 不需要外部数据也能训练到与 CNN 竞争的水平。

它做的两件事——高效训练策略和蒸馏 token——分别对应着"入门门槛的革命"和"蒸馏方法的进化"。

前一个贡献把 ViT 从"需要几百张 TPU + 几亿张图"的奢侈品变成了"4-8 张 GPU + 3 天 + ImageNet-1K"就能用的工具。后一个贡献发现了一个有趣的跨架构蒸馏现象：卷积网络比 Transformer 更适合教 Transformer。

如果把它放进技术树，DeiT 是 ViT 路线上的一个**关键桥接节点**——在它之前，视觉 Transformer 这条路虽然被发现但难以通行；在它之后，这条路被铺平了。DINO、MAE、DeiT III 等一系列后续工作都站在了 DeiT 的肩膀上。

直白说，这篇论文的价值不只是提出了一个更好的 ViT 训练方案，而是把 Vision Transformer 这条路线从"看起来可行"推进到了"可以作为通用范式使用"的阶段。
