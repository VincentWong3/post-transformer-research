# Training Data-Efficient Image Transformers & Distillation through Attention

## 一、先搞清楚坑在哪

2020 年底，ViT 刚证明了纯 Transformer 可以在 ImageNet 上做分类，但有一个致命限制：**ViT 需要 JFT-300M（3 亿图片）级别的预训练数据才能超过 ResNet**。在 ImageNet-1K（130 万图片）上从头训练，ViT-B 只有 79.9% top-1，比同等规模的 ResNet-50 还低。

根本原因：Transformer 缺少 CNN 的归纳偏置（平移等变性、局部连接），需要更多数据才能学到这些结构。对于绝大多数没有 3 亿图片的团队来说，ViT 只是个理论成果，不实用。

DeiT 要解决的问题就是：**如何让 ViT 在 ImageNet-1K 上直接从零训练就能达到 SOTA？**

## 二、ViT 小数据训练差的真正原因

ViT 在 ImageNet-1K 上表现不佳，不是因为架构不好，而是因为**训练策略不对**。

CNN 训练积累了多年经验：数据增强、正则化、学习率调度——ResNet-50 在 ImageNet 上训练 90 epoch 就能收敛。ViT 不能直接套用这些经验，因为：

- **ViT 的 token 数少**（196），过拟合风险高
- **没有卷积的局部归纳偏置**，需要更强的正则化
- **位置编码需要更多的数据来学习空间关系**

Touvron et al. 的关键洞察是：ViT 缺的不是数据，而是**正确的训练方法**。

## 三、DeiT 的核心思路

一句话：**给 ViT 配上完整的现代训练配方（数据增强 + 正则化 + 知识蒸馏），让它在 ImageNet-1K 上从头训练达到 SOTA。**

三大贡献：
1. **训练配方**：一套针对 ViT 优化的训练策略（数据增强、正则化、优化器配置）
2. **知识蒸馏**：用 CNN teacher 蒸馏 Transformer student，结合创新的蒸馏 token 机制
3. **无需 JFT-300M**：只靠 ImageNet-1K，DeiT-B 达到 83.8% top-1，超越 ResNet-152

## 四、网络架构详解

### 架构

DeiT 的架构几乎和 ViT 完全一样——标准的 Transformer encoder + patch embedding + [class] token。唯一的结构性改动是新增了一个 **distillation token**。

```
Input: 224×224×3
    │
    │  Patch Embedding (16×16, stride=16)
    │  196 patches × 768 dim
    │
    ▼
┌────────────────────────────────────────┐
│ 拼接: [class token] + [distill token] + 196 patch embeddings│
│         + 位置编码                      │
│  序列: 198 tokens × 768 dim            │
└──────────────────┬─────────────────────┘
                   │
                   ▼
┌────────────────────────────────────────┐
│ Transformer Encoder ×12 (ViT-B 配置)   │
│ 同 ViT: Pre-LN, MSA, MLP, GELU, Drop  │
└──────────────────┬─────────────────────┘
                   │
        ┌──────────┴──────────┐
        │                     │
    [class] token        [distill] token
        │                     │
        ▼                     ▼
   分类头 (K classes)   蒸馏头 (K classes)
   真实标签 loss         蒸馏 loss (teacher logits)
```

### Distillation Token

DeiT 在 ViT 的输入序列中增加了一个可学习的 distillation token（位置在 [class] token 之后）。经过 Transformer encoder 后：

- **[class] token 的输出**：通过交叉熵损失学习真实标签（hard label）
- **[distill] token 的输出**：通过 KL 散度学习 teacher CNN 的 soft label

两个 token 互不干扰，共享 Transformer encoder 的所有参数。推理时可以选择使用 [class] token 或 [distill] token 的输出，或者两者平均。

**为什么用一个额外的 token 而不是在 [class] token 上同时加蒸馏损失？** 实验证明（Table 5），独立的 distillation token 比共享 [class] token 好——两个 token 可以学到互补的特征。

### 几种蒸馏模式

| 模式 | teacher | student loss | ImageNet top-1 |
|------|---------|-------------|---------------|
| 无蒸馏 | — | 仅 hard label | 79.9 |
| Hard 蒸馏 | RegNetY-16GF | hard label + teacher's argmax | 82.9 |
| Soft 蒸馏 | RegNetY-16GF | hard label + KL(teacher softmax) | 83.0 |
| **Conv 蒸馏** | **RegNetY-16GF** | **hard + distill token soft** | **83.8** |

"Conv 蒸馏"（convolutional teacher with distillation token）是最优组合。

## 五、核心贡献：训练配方

DeiT 的真正贡献不是架构，而是**一套可复现的训练配置**：

### 数据增强
- **RandAugment** (Cubuk et al., 2020)：自动数据增强，随机选择增强操作
- **Mixup** (Zhang et al., 2018)：两张图混合，标签按比例混合
- **CutMix** (Yun et al., 2019)：区域替换式混合
- **Random Erasing**：随机擦除图像区域
- **Color Jitter**：颜色抖动
- 以上增强的组合使得 ViT 在 ImageNet-1K 上不过拟合

### 正则化
- **Stochastic Depth (DropPath)**：以概率 p 随机跳过整个 Transformer block。训练时跳过部分 block，推理时全部使用。从第一层到最后一层，p 线性增长（0→0.1）。这是 ViT 论文中没用的技巧，DeiT 引入后效果显著。
- **Label Smoothing**：标签平滑，防止过自信
- **Repeated Augmentation**：同一个 batch 中每条数据独立增强，增加多样性
- **Gradient Clip**：梯度裁剪，防止训练不稳定

### 优化器
- **AdamW** (Loshchilov & Hutter, 2019)：和 ViT 一样
- **Cosine Learning Rate Decay**：余弦退火学习率
- **Linear Warmup**：前 5 epoch 线性升温
- **Weight Decay**：0.05（比 ViT 的 0.1 小）
- **Batch Size**：1024
- **Epochs**：300（ViT 在 ImageNet-1K 上训练只有 200 epoch）

### ViT 模型尺度缩放（Table 1）

DeiT 还提出了三种不同规模：

| 模型 | #param | FLOPs | 结构 | ImageNet top-1 |
|------|--------|-------|------|---------------|
| DeiT-Ti | 5M | 1.3G | 12层, 192 dim, 3 heads | 74.5 |
| DeiT-S | 22M | 4.6G | 12层, 384 dim, 6 heads | 81.2 |
| DeiT-B | 86M | 17.6G | 12层, 768 dim, 12 heads | 83.8 |

DeiT-S 是效率和精度的最佳平衡点——22M 参数，81.2% top-1。

## 六、公式详解

### 公式 1: Hard-label Distillation

$$\mathcal{L}_{\text{global}} = \frac{1}{2}\mathcal{L}_{\text{CE}}(\psi(\mathbf{z}_s), \mathbf{y}) + \frac{1}{2}\mathcal{L}_{\text{CE}}(\psi(\mathbf{z}_t), \mathbf{y}_t)$$

其中 $\mathbf{z}_s$ 是 [class] token 的输出，$\mathbf{z}_t$ 是 distill token 的输出，$\psi$ 是分类头，$\mathbf{y}_t = \text{argmax}(\mathbf{z}_{\text{teacher}})$ 是 teacher 的硬标签（hard decision）。

直觉：学生不仅要学真实标签，还要学 teacher CNN 的分类决策。如果 teacher 认为某张图"更像狗而不是猫"，学生也会学到这个偏好。

### 公式 2: Soft Distillation with Distillation Token

$$\mathcal{L} = (1-\lambda)\mathcal{L}_{\text{CE}}(\psi(\mathbf{z}_s), \mathbf{y}) + \lambda \tau^2 \text{KL}(\psi(\mathbf{z}_t)/\tau, \psi_{\text{teacher}}(\mathbf{x})/\tau)$$

- $\tau$：蒸馏温度。$\tau=1$ 时和标准 KL 一致，$\tau>1$ 时 teacher 的 softmax 更平滑，传递更多信息
- $\lambda$：蒸馏权重，控制 teacher 和真实标签的相对重要性

KL 散度衡量 teacher 和 student 的 distill token 输出分布之间的差异。这里的关键：distill token 是"专门学 teacher 的"，[class] token 是"专门学真实标签的"，两者分工明确。

### 公式 3: Stochastic Depth (DropPath)

$$\mathbf{x}_{l+1} = \mathbf{x}_l + \frac{1}{1-p_l} \cdot \text{Block}_l(\mathbf{x}_l) \cdot \text{Bernoulli}(1-p_l)$$

训练时，以概率 $p_l$ 跳过整个 Block（Block 的输出为 0，只保留残差连接）。$p_l$ 从第一层的 0 线性增长到最后一层的 $p_{\max}=0.1$：$p_l = \frac{l}{L-1} \cdot p_{\max}$，其中 $L$ 是总层数。

深层 block 被 drop 的概率更大——因为深层的特征更冗余，drop 的风险更低。推理时不 drop（$\times 1$ 恢复正常尺度）。

## 七、实验结果

### ImageNet-1K 分类（从零训练，无外部数据）

| 模型 | 参数量 | FLOPs | Top-1 | Top-5 |
|------|--------|-------|-------|-------|
| ResNet-152 | 60M | 11.6G | 78.6 | 94.2 |
| EfficientNet-B3 | 12M | 1.8G | 81.6 | 95.7 |
| ViT-B/16 (from scratch) | 86M | 17.6G | 79.9 | — |
| **DeiT-B (无蒸馏)** | **86M** | **17.6G** | **83.6** | **96.5** |
| **DeiT-B (Conv 蒸馏)** | **86M** | **17.6G** | **83.8** | **96.6** |
| DeiT-S (无蒸馏) | 22M | 4.6G | 81.2 | 95.4 |
| DeiT-Ti (无蒸馏) | 5M | 1.3G | 74.5 | 91.7 |

关键数据点：
- DeiT-B 无蒸馏 83.6% vs ViT-B 79.9% → **+3.7%**，全靠训练配方
- DeiT-S 22M 用 1/4 参数达到 81.2%，高于 ResNet-152 的 78.6%
- Conv 蒸馏只贡献了 +0.2%（83.6→83.8），真正的大头是训练配方

### 与 ViT 的公平对比（Table 5）

| 训练配置 | ViT-B | DeiT-B |
|---------|-------|--------|
| 基础配置 | 79.9 | 83.1 |
| + Stochastic Depth | — | 83.6 |
| + Distillation Token | — | 83.8 |

实验清晰表明：**Stochastic Depth 是最大的贡献者**，蒸馏 token 只锦上添花。

## 八、代码对照（GitHub: facebookresearch/deit）

官方实现基于 PyTorch，代码结构和 ViT 类似。几个关键发现：

**1. 蒸馏 token 实现。** `distillation_token` 是一个 `nn.Parameter`，初始化为 `torch.zeros(1, 1, embed_dim)`，通过 `trunc_normal_(std=.02)` 初始化。与 [class] token 拼接后喂入 Transformer。

**2. 蒸馏 loss 计算。** 用 `torch.distributions.kl_divergence` 或自定义的 KL，配合 temperature scaling。代码中 teacher 和 student 的输出都除以 $\tau$ 后再 softmax。

**3. 数据增强 pipeline。** `timm` 库的 `RandAugment` + `CutMix` + `Mixup`。CutMix 和 Mixup 会同时修改图片和标签——标签变成混合比例权重。

**4. 训练 300 epoch vs ViT 的 200 epoch。** 相当于多了 50% 的训练迭代。如果 ViT 也训 300 epoch，差距可能会缩小。

**5. 没有使用预训练位置编码插值。** DeiT 在 224×224 上训练并评估，不涉及高分辨率 fine-tune，所以不需要位置编码插值。

## 九、位置

### 前驱工作

- **[Dosovitskiy et al., 2020] ViT**：DeiT 的基础架构。DeiT 证明了 ViT 的训练问题可以通过技巧解决。
- **[Cubuk et al., 2020] RandAugment**：自动数据增强方法，DeiT 的核心增强之一。
- **[Huang et al., 2016] Stochastic Depth**：随机深度正则化，从 ResNet 迁移到 Transformer。
- **[Hinton et al., 2015] Distillation**：知识蒸馏的经典工作，DeiT 的蒸馏机制源自此。

### 同期竞品

- **[Touvron et al., 2021] DeiT III: Revenge of the ViT**：同一团队的后续工作，进一步改进了训练配方。DeiT III 证明不用蒸馏，单靠训练技巧也能让 ViT 达到 85%+ 的精度。
- **[Liu et al., 2021] Swin Transformer**：同期从架构角度解决 ViT 问题（层级化 + 移位窗口）。

### 后续影响

- DeiT 的训练配方几乎成了 ViT 训练的"标准配置"——后续几乎所有 ViT 变体（Swin、PVT、CSWin等）都在使用 DeiT 风格的数据增强和正则化。
- **[Steiner et al., 2021] How to Train Your ViT?**：系统地研究了 DeiT 训练配方各组件的影响，验证了 Stochastic Depth 和数据增强的关键作用。
- **[Touvron et al., 2022] DeiT III / ResNet Strikes Back**：把 DieT 的训练配方进一步改进后应用于 ResNet，发现 ResNet 也能从中获益。
- 知识蒸馏 token 的设计也被后续的蒸馏方法（如 TinyViT、DistilViT）借鉴。

## 十、局限

1. **蒸馏依赖 CNN teacher。** DeiT 需要先在 ImageNet 上训练一个 CNN teacher（RegNetY-16GF）。这增加了流程复杂度——如果 CNN teacher 本身就很强（88%），用它教 ViT（83%）其实没完全发挥 Transformer 的潜力。

2. **Stochastic Depth 需要调参。** $p_{\max}=0.1$ 是经验值，随模型大小和深度变化。DeiT-B 用 0.1，但如果模型更深应该用更大的 $p_{\max}$。

3. **没有解决多尺度问题。** DeiT 和 ViT 一样输出单分辨率特征图，不能直接用于检测/分割。这是架构层面的硬伤，蒸馏解决不了。

4. **DiStillation token 的改进有限。** 实验显示蒸馏 token 只贡献了 +0.2%（83.6→83.8），$p$ 值在统计显著性边缘。单靠数据增强和 Stochastic Depth 就能达到 83.6%。

5. **缺乏强 baseline 的消融。** 论文没有证明"这些训练技巧对 ResNet 不奏效"——实际上后来 ResNet Strikes Back 证明了同样的技巧也能提升 CNN。

## 十一、小结

DeiT 的核心贡献不是发明新技术，而是**证明了 ViT 在 ImageNet-1K 上的失败是训练问题，不是架构问题**。

它做对了两件事：

1. **系统化的训练配方**：Stochastic Depth + RandAugment + Mixup/CutMix，让 ViT 从 79.9% 提升到 83.6%，接近 ResNet 的生态位
2. **Distillation Token**：一种优雅的蒸馏机制，让 CNN 的知识可以迁移到 Transformer

DeiT 在 ViT 和 Swin 之间架了一座桥：它证明了 Transformer 可以在中小规模数据上工作，为后来 Swin、ConvNeXt 等大规模应用铺平了道路。它的训练配方至今仍是 ViT 训练的默认 baseline。
