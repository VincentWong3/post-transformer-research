## 一、先搞清楚"问题在哪里"

2021 年中的视觉自监督学习，大致分了两条路。

一条是 **contrastive learning**：SimCLR、MoCo v3、BYOL，核心是拉近正样本、推开负样本，或者在 BYOL/SimSiam 那种 asymmetric network 结构里隐式避免 collapse。这些方法处理的是"全局视图"——对一张图做两次随机裁剪增强，让 encoder 学会把两张裁剪图映射到相近的 embedding。这是全局的、image-level 的表征学习。

另一条是 **masked image modeling（MIM）**，模仿 NLP 中 BERT 的做法：遮住输入的一部分 patch，让模型去预测被遮住的内容。BEiT（Bao et al., 2021）是这条路的开拓者——它用预训练的 dVAE（来自 DALL-E）把图像 patch 变成离散 token，然后让 ViT 做 masked patch 的分类预测。

这两条路各有侧重。Contrastive 方法擅长全局语义——encoder 的 [CLS] token 可以很好地区分不同物体。MIM 方法擅长局部结构——模型必须理解 patch 之间的空间关系才能预测被遮住的部分。但问题是：**MIM 能否同时获得全局语义和局部结构？**

BEiT 的实验提供了一个值得注意的信号。它的 k-NN 和 linear probing 分数远低于 contrastive 方法——意味着 [CLS] token 的全局语义不够好。而它的 fine-tuning 分数又不错——说明局部结构恢复能力确实帮助了下游任务。这个分裂给人的感觉是：MIM 的方向可能对了，但 BEiT 的具体实现（离线 dVAE tokenizer）拖了后腿。

问题于是变成：**能不能设计一种 MIM 方法，同时获得 DINO 级别的全局语义和 MIM 特有的局部结构？**

## 二、旧路线的真正问题

BEiT 的核心流程是这样的：

1. 准备阶段：用 DALL-E 的 dVAE 在 2.5 亿张图上预训练一个离散 tokenizer（Conceptual Captions + 其他公开数据集）
2. 冻结 tokenizer，把 ImageNet 的每张图的每个 patch 映射成离散 token id（one-hot）
3. 训练 ViT：遮住部分 patch，让 ViT 输出的分类 logits 去匹配 tokenizer 给的 one-hot label

这里面有三个深层问题。

**问题一：离线 tokenizer 只能捕捉低级语义。**

iBOT 论文在自己的消融实验中（Tab. 9）也证实了这一点——dVAE 学习到的主要是 color、texture 等 low-level 特征。用它能重建 patch 的视觉外观，但 token id 里几乎没有"这是一个车轮"、"这是一只鸟的喙"这种高级语义信息。原因也很直接：dVAE 的训练目标是重建像素，不是理解图像内容。

**问题二：两阶段训练复杂且不灵活。**

Tokenier 需要提前在超大数据集上预训练、冻结。如果要换下游数据集（从 ImageNet 到医学图像到卫星图），tokenizer 不能自适应，因为它的参数已经固定了。这在 NLP 里不是大问题（WordPiece 是统计分析，通用性极强），但在视觉里，不同 domain 的视觉特征差异很大。

**问题三：one-hot 离散化对连续图像来说是次优选择。**

NLP 里，"dog"这个词的 token id 是确定无疑的。但图像里，一个 patch 的语义天然具有歧义性——同一个 patch 可以同时属于多个视觉概念（比如"毛茸茸的边缘"和"耳朵的一部分"）。强制 one-hot 离散化会丢失这种模糊语义。

这三个问题的根因是同一个：**BEiT 把 tokenizer 和 target model 分成了两个独立的阶段，tokenizer 本身没有参与主任务的训练，也没有机会在主任务数据上学习到有意义的视觉语义。**

那能不能把 tokenizer 也变成一个 online 的、可训练的模块？

## 三、这篇论文的核心思路

> 这篇论文的核心思路是：**把 MIM 重新形式化为知识蒸馏（knowledge distillation），用 teacher 网络作为 online tokenizer，student 从 masked image 中恢复 patch token，teacher 通过 EMA（Exponential Moving Average，指数移动平均）更新。**

这句话需要拆开来说清楚。

传统知识蒸馏是一个预训练的大模型（teacher）教一个小模型（student）产生相似的输出分布。iBOT 把这个框架借过来做 MIM：

```text
原始图像 → teacher（EMA 更新）→ 输出 patch token 分布（作为监督信号）
         ↓
         student（梯度更新）← 被遮住的图像 ← 遮罩
```

Teacher 对全图输出每个 patch 的 softmax 分布，student 对被遮住的图输出对应 patch 的分布，然后最小化两者的交叉熵。

这样一来，tokenizer 不再是离线预训练好的固定模块——它就是 teacher 网络本身，与 student 网络共享架构，通过 EMA 从 student 的参数逐步移动过来。**Tokenizer 和 target model 同时训练、同时更新。**

但这里有一个问题：如果 teacher 和 student 都从随机初始化开始，teacher 输出的分布一开始就是噪声，MIM loss 没有意义。怎么确保 teacher 的分布具有语义信息？

答案是从 DINO 借来的 [CLS] token 自蒸馏损失 $ \mathcal{L}_{[CLS]} $ ：

```text
L_total = L_CLS([CLS] token 自蒸馏) + L_MIM(patch token 自蒸馏)
```

$ \mathcal{L}_{[CLS]} $ 负责从全局视角让 teacher 学会语义分类——两张增强图应该在同一类上。这个语义信息通过共享的 projection head 传导给 patch token 的预测分支。

**本质上，iBOT 做的事情是：让 [CLS] token 负责"理解这张图里有什么"，让 patch tokens 负责"理解每个局部 patch 属于什么语义类别"。两者共享同一个 head，协同训练。**

## 四、整体结构与 forward 流程

把下面的流程图记住，forward pass 就很清楚了。

```text
输入图像 x
    │
    ├── random augment → 视图 u ──→ ViT(teacher) ──→ [CLS]_t + patch_t
    │                                            │
    │                                            ├── ℒ_{MIM}(patch_t, patch_s)
    │                                            │    (只有被 mask 的位置算 loss)
    │                                            └── ℒ_{[CLS]}([CLS]_t, [CLS]_s)
    │
    ├── random augment → 视图 v ──→ ViT(teacher) ──→ [CLS]_t + patch_t  (对称)
    │
    ├── blockwise mask → u_hat ──→ ViT(student) ──→ [CLS]_s + patch_s
    │
    └── blockwise mask → v_hat ──→ ViT(student) ──→ [CLS]_s + patch_s  (对称)
```

具体的 shape 变化，以 ViT-B/16 + 224x224 输入为例：

```text
输入图像: 3 × 224 × 224
    ↓ patch embed (16×16, stride 16)
patch tokens: 196 × 768   (加 [CLS] token → 197 × 768)
    ↓ ViT-B/16 (12层 Transformer)
encoder output: 197 × 768
    ↓ 取 [CLS] token (第0个)
[CLS]_cls: 1 × 768
    ↓ 取所有 patch tokens (第1~196个)
patch_cls: 196 × 768
    ↓ shared projection head (3层 MLP + L2 bottleneck + 8192 dim)
[CLS]_proj: 1 × 8192  → softmax → 8192 维分布
patch_proj: 196 × 8192 → softmax → 196 × 8192 维分布
```

**关键差异**：DINO 里 student 接收的是全图（未遮罩），iBOT 里 student 接收的是遮罩后的图。Teacher 始终接收全图。

forward 的核心顺序：

1. 对一张图做两次随机增强，得到两个视图 $ \bm{u} $ 和 $ \bm{v} $
2. 对每个视图做 blockwise masking（与 BEiT 相同），得到 $ \hat{\bm{u}} $ 和 $ \hat{\bm{v}} $
3. Teacher 网络（EMA）处理全图 $ \bm{u} $ 和 $ \bm{v} $，输出 [CLS] 和 patch 的分布
4. Student 网络（梯度更新）处理遮罩图 $ \hat{\bm{u}} $ 和 $ \hat{\bm{v}} $，输出对应的分布
5. 计算两个损失：$ \mathcal{L}_{[CLS]} $（[CLS] 上的自蒸馏）+ $ \mathcal{L}_{MIM} $（patch 上的自蒸馏）
6. Student 参数通过梯度下降更新，teacher 参数通过 EMA 从 student 移动
7. Teacher 侧的 centering 也通过 EMA 更新

遮罩策略也有设计：以 0.5 的概率完全不遮（prediction ratio = 0），以 0.5 的概率从 $ [0.1, 0.5] $ 均匀采样一个 ratio。这种 "random MIM" 策略在 multi-crop 场景下可以稳定训练。

## 五、核心模块逐个解释

### 5.1 Online Tokenizer

这是整篇论文最重要的设计。

**它解决了什么？** 离线 tokenizer（dVAE）需要提前在大数据集上训练、冻结，无法适应不同 domain 的数据，且学到的语义层级低。

**它是如何工作的？** Teacher 网络本身就是 tokenizer。对于一个全图 $ \bm{u} $，teacher 输出所有 patch token 的 softmax 分布 $ P_{\bm{\theta}'}^{patch}(\bm{u}) $ 。Student 从遮罩图 $ \hat{\bm{u}} $ 输出 $ P_{\bm{\theta}}^{patch}(\hat{\bm{u}}) $ ，两者计算交叉熵 loss。Teacher 的参数不是通过梯度更新，而是通过 EMA 从 student 的参数移动：

$$ \bm{\theta}' \leftarrow m \cdot \bm{\theta}' + (1-m) \cdot \bm{\theta} $$

其中 $ m $ 是 momentum 系数（接近 1）。这个 EMA 机制来自 MoCo / BYOL / DINO 系列，它使得 teacher 可以看作是 student 的"时间累积版本"——比 student 更平滑、更稳定。

**为什么这个设计合理？** 因为视觉语义本身就是一个渐进涌现（bootstrapping）的过程。初始化时 teacher 的分布是随机的，MIM loss 没有太大意义。但随着 [CLS] token 上的自蒸馏让 teacher 逐步学会语义分类，patch tokens 的分布也会随之变得有语义——两个分支共享 head，语义信息会传递过来。换句话说，**[CLS] 学到的全局语义"教"给了整个 head，而 MIM 分支利用这个语义化的 head 来做更好的局部预测。**

### 5.2 共享 Projection Head

这是论文中一个看似简单但重要的设计。

ViT backbone 的输出包含两类 token：[CLS]（1 个）和 patch tokens（196 个）。iBOT 把它们送入同一个 projection head $ h $ ，得到 8192 维的 logits，然后各自算 loss。

论文对比了几种共享策略：

- 完全不共享：$ h^{[CLS]} $ 和 $ h^{patch} $ 完全独立
- 共享部分层（前两层共享，最后一层分开）
- 完全共享（整篇论文使用的方式）

实验发现完全共享效果最好。直觉上也讲得通：如果头是共享的，那么 [CLS] 上习得的语义分类能力，自然会被 patch tokens 分支使用。Patch tokens 在做 MIM 恢复时，不是靠"像素重建"，而是靠"把 patch 分到正确的语义类别"——这是投影头 8192 维输出的含义。

### 5.3 Softmax 分布 vs One-Hot Token

BEiT 用 dVAE 给每个 patch 分配一个离散 token id，即 one-hot 标签。iBOT 改用 softmax 分布作为监督信号。

**这为什么重要？** 图像的一个 patch 天然是模糊的——一个 16x16 的 patch 可能同时包含天空的一角和树冠的边缘。One-hot 强制模型选择一个类别，这会丢失信息。Softmax 分布允许 teacher 输出"这个 patch 有 60% 的可能性属于类别 A，30% 属于类别 B"这种更丰富的表达。

数值上的差异体现在 Tab. 18 中，但更重要的是设计范式上的差异：**iBOT 不是在"预测一个离散标签"而是在"模仿一个连续分布"。** 这让 MIM 从分类问题变回了知识蒸馏问题——而知识蒸馏天然适合用 soft 目标。

### 5.4 随机遮罩 + Multi-Crop 稳定策略

iBOT 使用 multi-crop augmentation（2 个 global crops + 10 个 local crops），这是 DINO 的做法。但直接把 MIM 和 multi-crop 组合在一起会遇到训练不稳定问题——原因是遮罩后的 global crop 和未遮罩的 local crop 分布不匹配。

解决方法很优雅：随机遮罩。以 0.5 的概率完全不遮罩（prediction ratio = 0），此时整个 iBOT 退化为 DINO；以 0.5 的概率从 [0.1, 0.5] 采样 ratio。这样一来，训练过程中学生既见过全图、也见过遮罩图，分布不再 mismatch，训练曲线稳定得多。

这个设计的价值在于：**它让 iBOT 同时获得了 DINO 的稳定训练和 MIM 的额外信息。**

## 六、核心公式与数学直觉

### 6.1 从 BEiT 到自蒸馏

BEiT 的 MIM loss 可以写作：

$$ -\sum_{i=1}^{N} m_i \cdot P_{\bm{\phi}}(\bm{x}_i)^\mathsf{T} \log P_{\bm{\theta}}(\hat{\bm{x}}_i) \tag{1} $$

其中 $ \bm{\phi} $ 是预训练 dVAE 的参数（冻结），$ \bm{\theta} $ 是 ViT 的参数（训练），$ P_{\bm{\phi}}(\bm{x}_i) $ 是 dVAE 给 patch $ \bm{x}_i $ 分配的 one-hot 标签。

iBOT 把它改成了：

$$ \mathcal{L}_{MIM} = -\sum_{i=1}^{N} m_i \cdot P_{\bm{\theta}'}^{patch}(\bm{u}_i)^\mathsf{T} \log P_{\bm{\theta}}^{patch}(\hat{\bm{u}}_i) \tag{3} $$

区别只有一个参数：**把冻结的 $ \bm{\phi} $ 换成了在线更新的 $ \bm{\theta}' $**。

但这个替换的后果是深远的：
- $ \bm{\phi} $ 是固定的——只能输出 dVAE 预训练时学会的低级特征
- $ \bm{\theta}' $ 是动态的——随着 [CLS] 自蒸馏的进行，逐渐获得语义信息
- $ P_{\bm{\phi}} $ 是 one-hot——丢失了模糊语义
- $ P_{\bm{\theta}'} $ 是 softmax 分布——保留了分布信息

### 6.2 Self-Distillation on [CLS] Token

$$ \mathcal{L}_{[CLS]} = -P_{\bm{\theta}'}^{[CLS]}(\bm{v})^\mathsf{T} \log P_{\bm{\theta}}^{[CLS]}(\bm{u}) \tag{2} $$

这个 loss 来自 DINO。$ \bm{u} $ 和 $ \bm{v} $ 是同一张图的两个增强视图，teacher 处理 $ \bm{v} $ ，student 处理 $ \bm{u} $ 。Loss 是 teacher 分布与 student 分布之间的交叉熵。

**如果去掉 $ \mathcal{L}_{[CLS]} $ ，只保留 MIM 会怎样？** 论文 Tab. 9 给出了数据：k-NN 从 69.1% 掉到 9.5%，linear probing 从 74.2% 掉到 29.8%——几乎完全丧失了语义。这个实验说明 **MIM 本身（局部 patch 重建）并不自动产生全局语义**，需要 [CLS] 自蒸馏来提供。

反过来说，只用 DINO（$ \mathcal{L}_{[CLS]} $ alone）达到 67.9% k-NN、72.5% linear——加上 MIM 后提升到 69.1% 和 74.2%。MIM 带来的增益主要在 patch 级别的表征质量上，这在 downstream dense tasks 中体现得更明显。

### 6.3 总 Loss

$$ \mathcal{L}_{total} = \mathcal{L}_{[CLS]} + \mathcal{L}_{MIM} $$

论文直接在两个 loss 上做了等权求和，没有 scaling factor。这个做法的隐含假设是：两个 loss 的量级在训练过程中大致匹配，不需要额外调参。

### 6.4 Teacher Centering + Student Sharpening

这来自 DINO 的标准做法，不是 iBOT 的原创，但值得提一下，因为它是让 teacher 输出"有意义分布"的关键。

Teacher 的 logits 在做 softmax 之前先减去一个 EMA 积累的 centering 向量 $ C $ ：

$$ \text{teacher output} = \text{softmax}((teacher\_logits - C) / \tau_t) $$

Student 输出直接除以一个较小的温度 $ \tau_s $ （sharpening）：

$$ \text{student output} = \text{softmax}(student\_logits / \tau_s) $$

**效果**：Teacher 因为减去 center，不会坍缩到单一类别（center 会捕获全局类别分布，减掉后迫使 teacher 输出区分度高的信号）。Student 因为温度小（sharp），输出分布更尖锐，鼓励分类更自信。两个合在一起避免了模型崩溃。

有了这些公式基础，下面来看论文中的图表如何直观展示这些 loss 的计算路径——尤其是 cross-matching 在图上如何体现，以及公式背后的实际重构效果。

```text
                  ┌────────────────────────────────────────┐

                  │            /    \                       │
                  │           /      \                      │
                  │    augment     augment                 │
                  │          u       v                     │
                  │         / \     / \                    │
                  │        /   \   /   \                   │
                  │      mask   no  no  mask               │
                  │       u_hat  u   v   v_hat             │
                  │         |   |   |   |                  │
                  │       student  teacher  student        │
                  │         |   |   |   |                  │
                  │    [CLS]_s + patch_s ..  .             │
                  │         |         |                    │

```

我重新组织成更清晰的流：

```text
     u (aug-view1)                           v (aug-view2)
          │                                       │
    blockwise mask                           blockwise mask
          │                                       │
     u_hat (masked)                          v_hat (masked)
          │                                       │
     ┌────┴────┐                            ┌────┴────┐
     │ student │                            │ student │
     └────┬────┘                            └────┬────┘
          │                                       │
         [CLS]_s(u_hat)                         [CLS]_s(v_hat)
          + patch_s(u_hat)                       + patch_s(v_hat)
          │                                       │
          │        ┌───────────┐                   │
          └────────┤ teacher u ├───────────────────┘
                   │ teacher v │
                   └───────────┘
                   teacher outputs:
                   [CLS]_t(u), [CLS]_t(v)
                   patch_t(u), patch_t(v)

Losses:
  L_CLS = H([CLS]_s(u_hat), [CLS]_t(v)) / 2
        + H([CLS]_s(v_hat), [CLS]_t(u)) / 2

  L_MIM = (mask_u * H(patch_s(u_hat), patch_t(u))).sum / mask_u.sum / 2
        + (mask_v * H(patch_s(v_hat), patch_t(v))).sum / mask_v.sum / 2
```

## 八、复杂度、效率与可扩展性

### 8.1 计算开销

iBOT 相比 DINO 增加的计算主要在两点：

1. **额外 forward pass**：Teacher 需要处理全图（$ \bm{u} $ 和 $ \bm{v} $），student 处理遮罩图。而 DINO 里 student 和 teacher 各自处理一个视图。所以 iBOT 实际上比 DINO 多了 teacher 的 forward（因为 DINO 的 teacher 也处理一个视图，两者接近）。严格说，每次迭代有 4 个 forward：teacher 处理 $ \bm{u} $ 和 $ \bm{v} $ ，student 处理 $ \hat{\bm{u}} $ 和 $ \hat{\bm{v}} $。DINO 也是 4 个 forward（teacher 处理 $ \bm{v} $ ，student 处理 $ \bm{u} $ ，对称组再各一次）。所以计算量几乎相同。

2. **Patch token 的 loss 计算**：MIM loss 需要在 196 个 patch tokens 上计算交叉熵，而 DINO 只在 [CLS] token 上算。这增加了一点计算但可忽略。

总体来看，iBOT 在训练时的计算量与 DINO 基本相当。

### 8.2 模型参数量

以 ViT-B/16 为例，ViT backbone 约 85M 参数。Projection head（3 层 MLP，输出 8192 维）约 768x4096 + 4096x8192 + 8192x8192 = 约 100M 参数。所以大部分参数实际上在 projection head 里。

### 8.3 可扩展性

论文注意到一个重要现象（来自 Tab. 2 和 Tab. 3 的对比）：

- 使用 IN-1K 数据，ViT-L/16 上 iBOT 反而不如 BEiT（84.8% vs 85.2%）
- 使用 IN-22K 数据，ViT-L/16 上 iBOT 超过 BEiT（86.6% vs 86.0%）

论文自己的解释是 "iBOT requires more data to train larger model"。这其实暗示了 online tokenizer 的一个局限：因为 tokenizer 也在学，大数据集上才能学到足够好的语义。小数据集上，离线 tokenizer 的固定先验反而可能更好。

这也是 iBOT 的一个内在 trade-off：online tokenizer 更灵活、自适应，但需要更多数据才能达到最佳效果。

## 九、实验结果

### 9.1 ImageNet-1K 分类

**k-NN 和 Linear Probing**：使用 frozen backbone + 分类器。iBOT 以 ViT-B/16 取得 k-NN 77.1%、linear 79.5%。和 DINO（76.1%、78.2%）相比分别提升 1.0% 和 1.3%。使用 ViT-L/16 可以达到 78.0% k-NN 和 81.0% linear。加上 IN-22K 预训练后 ViT-L/16 达到 82.3% linear。

**Fine-tuning**：ViT-B/16 取得 84.0%，比 DINO（83.6%）高 0.4%，比 BEiT（83.4%）高 0.6%。IN-22K 预训练 + ViT-L/16 + 512 分辨率达到 87.8%。

### 9.2 下游密集任务

**COCO 目标检测 + 实例分割**：使用 Cascade Mask R-CNN。ViT-S/16 取得 APb 49.4（b-box AP）、APm 42.6（mask AP）。ViT-B/16 达到 APb 51.2、APm 44.2，超过 BEiT 和 DINO。

**ADE20K 语义分割**：使用 UperNet。ViT-B/16 取得 50.0 mIoU（mean IoU，越高分割质量越好），相比 DINO（46.8）提升 3.2。特别值得注意，使用 linear head（冻结 backbone 只训练分类层）时，iBOT 达到 38.3 mIoU，而 BEiT 只有 27.4——差距是 10.9 个点。这个数据直观地说明了 BEiT 的 patch tokens 缺乏局部语义，而 iBOT 的每块都有用。

### 9.3 鲁棒性

在背景变化、遮挡、分布外样本（IN-A、IN-C）上，iBOT 全面超过 DINO。背后的原因：MIM 迫使模型不依赖特定 patch 的像素信息（因为部分 patch 被遮住了），从而学会了更鲁棒的特征。具体来说，IN-C mCE（mean Corruption Error，越低越好）从 51.7 **下降**到 48.1，IN-A 从 12.3 提升到 13.8。

### 10.3 Loss 实现

`util/losses.py` 中的 loss 函数对应 paper 的公式。大致逻辑：

```python
def loss_fn(student_output, teacher_output, mask=None):
    # student_output / teacher_output: [B, N, D]
    # 注意这里 N = 1 + num_patches, 第一个是 [CLS], 其余是 patch
    
    # 提取 [CLS] token (N 维度上的第一个)
    s_cls = student_output[:, 0]    # [B, D]
    t_cls = teacher_output[:, 0]    # [B, D]
    
    # [CLS] loss (cross-entropy)
    loss_cls = cross_entropy(s_cls, t_cls)
    
    # 提取 patch tokens
    s_patch = student_output[:, 1:]  # [B, N_patch, D]
    t_patch = teacher_output[:, 1:]  # [B, N_patch, D]
    
    # MIM loss - 只在被 mask 的位置计算
    loss_mim = 0
    for b in range(B):
        # mask[b] 是当前 batch 的 mask 标识
        # 只取 mask=1 的位置算 loss
        masked_indices = mask[b] == 1
        loss_mim += cross_entropy(
            s_patch[b, masked_indices],
            t_patch[b, masked_indices]
        )
    
    return loss_cls + loss_mim
```

## 十一、Related Work 与技术谱系位置

iBOT 位于两条技术路线的交汇处。

```text
lineage A: Contrastive / Self-Distillation
├── MoCo (He et al., 2020) — contrastive with momentum encoder
├── BYOL (Grill et al., 2020) — contrastive without negative pairs
├── SimSiam (Chen & He, 2021) — stop-gradient without momentum
├── DINO (Caron et al., 2021) — self-distillation on [CLS] token
└── iBOT (this paper) — self-distillation + MIM

lineage B: Masked Image Modeling
├── Context Encoders (Pathak et al., 2016) — inpainting pixels
├── BEiT (Bao et al., 2021) — masked discrete token prediction
├── iBOT (this paper) — masked soft token prediction via KD
└── MAE (He et al., 2022, concurrent) — asymmetric encoder-decoder
```

iBOT 的独特位置在于：**它把 lineage A 的训练范式（自蒸馏、EMA teacher）应用到了 lineage B 的任务（masked prediction），并在这个过程中解决了 BEiT 的 tokenizer 问题。**

它与相邻论文的关系：

- **DINO**（直接前身）：提供了 $ \mathcal{L}_{[CLS]} $ 和 teacher-student 框架。iBOT 在 DINO 的基础上增加了 patch-level MIM loss 和共享 head
- **BEiT**（同时期工作）：提供了 MIM 的公式化和 blockwise masking。iBOT 用 online tokenizer 替代了 BEiT 的离线 dVAE tokenizer
- **EsViT**（Li et al., 2021a，Swin Transformer 上的自监督方法）：iBOT 在其 Swin-T 实验上与 EsViT 对比，基线用 EsViT 的数据

iBOT 出现的时间点很关键。2021 年 11 月，MAE 和 iBOT 几乎同时出现。两者走的是完全不同的 MIM 路线：

- **MAE**：不对称 encoder-decoder，75% masking ratio，直接预测像素值
- **iBOT**：对称 encoder-decoder（teacher=student），blockwise masking，预测语义分布

## 十三、范式意义

iBOT 的范式意义是 **bridging paradigm**——它连接了两条之前几乎平行的路线。

从"范式改变层级"来看：

- **DINO** = training stabilization paradigm shift（用 self-distillation 替代 contrastive learning）
- **BEiT** = training objective adaptation（把 BERT 的 MLM 从 NLP 搬到 CV）
- **iBOT** = training strategy synthesis（把 self-distillation 和 MIM 合成一个统一框架）

iBOT 并没有发明全新的目标函数或全新的架构，但它做了一件之前没人做到的事：**把 MIM 重新解释为知识蒸馏，从而自然地引入了 EMA teacher、soft target、centering/sharpening 等一系列已经被验证有效的训练技巧。**

这个重新解释的贡献在于统一性。在 iBOT 之前，如果你想在 ViT 预训练中使用 MIM 和 contrastive/self-distillation，你需要多阶段训练或两路并行。iBOT 证明了一个 loss 函数、一个框架就能同时做两件事——而且两者互相增强。

在技术树的节点类型上，iBOT 更像一个 **bridge node**：它本身不是终点，但它打开了一个研究方向的通道——把 mask prediction 和 self-distillation 合在一起训练。

## 十四、局限性与后续方向

### 14.1 大模型 + 小数据的瓶颈

论文自己承认的问题：在 IN-1K 数据上，ViT-L/16 的 iBOT（84.8%）反而不如 BEiT（85.2%）。只有在 IN-22K 数据上才反超。这意味着 **online tokenizer 在小数据上可能不如固定的、在大数据上预训练的 tokenizer 好**。

### 14.2 Multi-Crop 的复杂度

iBOT 依赖 multi-crop augmentation（2 global + 10 local），这是从 DINO 继承的。Multi-crop 增加了训练数据量约 4x（按 effective epoch 计算），而且需要特定的稳定策略（random masking）。这使得 iBOT 的训练 pipeline 较复杂。

### 14.3 与 MAE 的对比局限

iBOT 和 MAE 几乎同时出现，论文没有和 MAE 对比。在后续工作中，MAE 的简单方法在 scalability 上超过了 iBOT。这提示 iBOT 的设计虽然优雅，但 teacher-student 双网络的训练成本（两倍 forward）可能限制了它扩展到更大模型。

### 14.4 理论解释不足

为什么 MIM + self-distillation 能产生局部语义？论文提供了可视化证据，但缺乏理论分析。MIM 带来的鲁棒性改善也有待深入理解：是因为 MIM 类似于 data augmentation 吗？还是因为遮罩训练迫使模型学会了更分布外的特征？

### 14.5 后续方向

DINOv2（2023）在 iBOT 的基础上做了进一步整合：保持 DINO 的 [CLS] 自蒸馏，同时加入 iBOT 风格的 patch-level 蒸馏，并扩展到更大模型和数据。另一个方向是把 online tokenizer 的思想应用到更多模态（视频、3D 点云等）。

## 十五、小结

iBOT 完成了一件反直觉的事：**它证明了 local patch 预测（MIM）和 global semantics 蒸馏（DINO）可以在同一个框架下互相帮助，而不是互相干扰。**

它的核心贡献不是某个模块的发明，而是两个已有范式（MIM 和 self-distillation）的第一次有机融合。在融合过程中，它还顺便解决了 BEiT 的 tokenizer 长尾问题——用 EMA teacher 替代了预训练 dVAE。

这个设计的精妙之处在于：**知识蒸馏的 teacher-student 结构恰好可以映射到 MIM 的 tokenizer-target model 结构。** 把两个范式放在一起看，它们需要的组件（teacher、student、EMA、cross-entropy loss）几乎完全相同。iBOT 只是把 MIM 的"输出端"从 one-hot token 换成了 teacher 的 softmax 分布——这么小的一个改动，带来了一连串好处。

直白说，iBOT 的价值不只是提出了另一个 MIM 方法，而是把 **self-distillation 和 masked prediction 的连接打通了**，为后续 DINOv2 等工作提供了关键的实验证据和设计启发。在技术树里，它属于 bridge node——连接了两条重要的技术路线。
