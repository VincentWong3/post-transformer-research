# iBOT: Image BERT Pre-Training with Online Tokenizer（2021）

**论文：** [arXiv](https://arxiv.org/abs/2111.06377) · 作者：Jinghao Zhou, Chen Wei, Huiyu Wang, Wei Shen, Cihang Xie, Alan Yuille, Tao Kong · 年份：2021

---

## 一、先搞清楚坑在哪

在 iBOT 之前，自监督视觉预训练有两个主流方向：

1. **对比学习**（如 SimCLR、MoCo、SwAV）：通过拉近正样本、推开负样本学习视觉表征。核心问题是**严重依赖数据增强**——正样本对必须是同一张图的不同增强版本。这导致模型学到的特征偏向于"增强不变性"，而不是真正理解图像的语义结构。

2. **掩码图像建模**（Masked Image Modeling, MIM，如 BEiT、MAE）：像 BERT 一样，随机遮住图像的一部分，让模型预测被遮住的内容。核心问题是**需要离散的视觉 token**——图像不像文本有天然的离散词表，必须用预训练的 tokenizer（如 dVAE）将图像 patch 映射到离散 token 空间。

iBOT 要解决的核心矛盾是：**MIM 方法效果好，但它的 tokenizer 是预训练且固定的，这既限制了模型的灵活性，又增加了额外的训练成本。** 能不能让 tokenizer 和视觉编码器一起端到端地训练？

## 二、现有 MIM 方法的真正问题

BEiT [Bao et al., 2021] 的做法是：先用 dVAE 在 ImageNet 上训练一个离散 tokenizer，把 224×224 图像切分成 14×14 个 patch，每个 patch 被映射到 8192 个离散 token 之一。然后固定这个 tokenizer，训练视觉 Transformer 去预测被遮住 patch 的 token ID。

这里有两个关键问题：

1. **Tokenizer 是静态的**：预训练好的 tokenizer 不会随视觉编码器的训练而更新。这意味着 tokenizer 的"语言"是固定的，视觉编码器只能在这个固定的语义空间里学习。如果 tokenizer 的语义空间不够好，视觉编码器的上限就被锁死了。

2. **离散化信息损失**：将连续的图像 patch 映射到离散 token 时，必然丢失细节信息。8192 个 token 看起来很多，但每个 patch 是 16×16×3=768 维的像素空间，压缩到一个 13 位（2^13=8192）的离散 ID，信息量损失了两个数量级。

有意思的是，对比学习那边也有类似的问题——CLIP 和 SimCLR 的"tokenizer"（即数据增强 pipeline）也是固定的，不是端到端学习的。

## 三、iBOT 的核心思路

一句话概括：**让视觉编码器同时扮演两个角色——它既是被训练的"学生"，又是生成训练目标的"老师"（online tokenizer）。**

具体来说，iBOT 的核心设计是：
- 用两个结构相同但参数不同的 ViT：一个在线编码器（online encoder，参数 θ）和一个目标编码器（target encoder，参数 θ'）
- 目标编码器的参数 θ' 是通过对 θ 做指数移动平均（EMA）更新的，不通过梯度下降
- 在线编码器对被遮住的 patch 做预测，目标编码器为被遮住的 patch 提供预测目标
- 两者在**连续的语义空间**中做对比学习，而不是在离散的 token 空间

这就像一个学生在做填空题，但答案不是老师给的固定答案集，而是老师根据当前知识水平动态生成的参考答案——而且老师和学生是同一个人的不同版本。

## 四、方法细节

### 4.1 整体框架

如图 1 所示，iBOT 的框架包含两个 ViT 和一个对比学习目标。

```
                        ┌─────────────────────────────────────┐
                        │          目标编码器 (θ')              │
                        │         (EMA of online)              │
                        │                                     │
                        │  ┌───────────────────────────────┐  │
                        │  │   ViT (teacher)               │  │
                        │  │                               │  │
  ┌──────────────┐      │  │  [CLS] → cls_token'           │  │
  │ 输入图像 x   │      │  │  [MASK] → patch_token'        │  │
  │              │      │  └───────────────────────────────┘  │
  │  ┌────────┐  │      └──────────┬───────────┬──────────────┘
  │  │ 随机   │  │                 │           │
  │  │ 遮罩   │  │                 ▼           ▼
  │  └────────┘  │          cls_token'   patch_token'(i)
  │              │               │              │
  │  ┌────────┐  │               │              │
  │  │ 增强1  │  │               │              │
  │  └────────┘  │               │              │
  │  /  \        │               │  对比学习    │
  │ x1   x2      │               │  (MIM loss)  │
  └──────────────┘               │              │
                                 │              │
                        ┌────────┴──────────────┴──┐
                        │     在线编码器 (θ)         │
                        │                           │
                        │  ┌─────────────────────┐  │
                        │  │   ViT (student)      │  │
                        │  │                      │  │
                        │  │  [CLS] → cls_token   │  │
                        │  │  [MASK] → pred_token │  │
                        │  └─────────────────────┘  │
                        └───────────────────────────┘
```

**图 1：iBOT 框架。** 在线编码器（学生）和目标编码器（老师）共享相同的 ViT 架构。目标编码器的参数通过 EMA 从在线编码器更新。两者对同一图像的不同增强版本做处理，目标编码器为被遮住的 patch 提供预测目标。

### 4.2 前向过程

1. **输入准备**：一张图像 x 经过两次随机增强得到 x1 和 x2（包括随机裁剪、颜色抖动、高斯模糊等）。这类似于对比学习的标准做法。

2. **遮罩**：对 x1 和 x2 分别随机遮罩 40%~60% 的 patch。遮罩策略与 BEiT 相同：用 [MASK] token 替换被选中的 patch。

3. **目标编码器前向**：x1 和 x2 分别送入目标编码器（参数 θ'）。目标编码器输出每个 patch 的表示向量（包括被遮住的 patch 的表示）。这里的关键是**目标编码器也能看到被遮住的 patch**——因为目标编码器的输入是完整的图像（没有遮罩），它只是为在线编码器提供"参考答案"。

4. **在线编码器前向**：x1 和 x2 分别送入在线编码器（参数 θ）。在线编码器同样输出每个 patch 的表示，但对于被遮住的 patch，它看到的只是 [MASK] token，因此需要预测这些 patch 的内容。

5. **损失计算**：两个损失函数：
   - **MIM 损失**：在线编码器预测的被遮住 patch 的表示，与目标编码器对应 patch 的表示做对比学习
   - **CLS 损失**：两个增强版本的 [CLS] token 表示做对比学习

### 4.3 核心创新点

#### 创新点 1：在线 tokenizer（Online Tokenizer）

**（a）痛点与动机**

BEiT 使用预训练的 dVAE 作为 tokenizer，将每个 patch 映射到离散 token。这个 tokenizer 是固定的，不会随着训练过程更新。这意味着：
- 如果 dVAE 的语义空间有偏差（例如对某些纹理过度敏感），视觉编码器会被迫在这个有偏的空间中学习
- 视觉编码器的表征能力受限于 tokenizer 的离散化粒度

**（b）方案细节**

iBOT 抛弃了离散 tokenizer，转而使用**在线编码器自己作为 tokenizer**。具体来说：

对于每个被遮住的 patch i，在线编码器输出一个预测向量 $p_i \in \mathbb{R}^D$，目标编码器输出一个目标向量 $q_i \in \mathbb{R}^D$。损失函数是：

$$\mathcal{L}_{\text{MIM}} = -\sum_{i \in \mathcal{M}} \log \frac{\exp(p_i \cdot q_i / \tau)}{\sum_{j \in \mathcal{V}} \exp(p_i \cdot q_j / \tau)}$$

其中 $\mathcal{M}$ 是被遮住的 patch 集合，$\mathcal{V}$ 是所有 patch 的集合，$\tau$ 是温度参数。

**（c）为什么有效**

这个设计的精妙之处在于：
- **目标编码器提供的是连续空间的表示**，而不是离散 token。这避免了量化带来的信息损失
- **目标编码器通过 EMA 更新**，所以它提供的目标会随着训练逐渐改善。这相当于 tokenizer 在训练过程中不断进化
- **在线编码器和目标编码器共享相同的语义空间**，因为它们有相同的架构，并且目标编码器的参数来自在线编码器的 EMA

从实验结果来看（表 2），iBOT 在 ImageNet 上的 linear probing 准确率达到 79.5%，比 BEiT 的 56.7% 高出 22.8 个百分点。这个巨大差距主要来自在线 tokenizer 的优势。

**（d）与 Related Work 的关系**

iBOT 的在线 tokenizer 设计受到 BYOL [Grill et al., 2020] 和 DINO [Caron et al., 2021] 的启发。BYOL 首次证明不需要负样本的对比学习是可行的，DINO 将类似方法应用到 ViT 上。iBOT 将这种"在线-目标"架构引入掩码图像建模任务。

与 BEiT 相比，iBOT 不需要额外的 dVAE 预训练阶段，简化了训练流程。与 MAE [He et al., 2021] 相比，iBOT 预测的是语义表示而不是原始像素，这使得它学到的特征更具语义性。

**（e）如果去掉会怎样**

表 2 的消融实验显示，如果去掉在线 tokenizer（即使用固定的 dVAE tokenizer），ImageNet linear probing 准确率从 79.5% 下降到约 60%。这证明了在线 tokenizer 的关键作用。

#### 创新点 2：MIM 与对比学习的统一框架

**（a）痛点与动机**

对比学习擅长学习全局语义（通过拉近同一图像的不同增强版本），但缺乏对局部结构的理解。MIM 擅长学习局部结构（通过预测被遮住的 patch），但缺乏全局对比信号。两者互补，但现有工作通常只用一个。

**（b）方案细节**

iBOT 同时使用两个损失：

$$\mathcal{L} = \mathcal{L}_{\text{MIM}} + \lambda \mathcal{L}_{\text{CLS}}$$

其中 $\mathcal{L}_{\text{MIM}}$ 是 patch 级别的对比学习（如上所述），$\mathcal{L}_{\text{CLS}}$ 是图像级别的对比学习：

$$\mathcal{L}_{\text{CLS}} = -\log \frac{\exp(c \cdot c' / \tau)}{\sum_{j} \exp(c \cdot c'_j / \tau)}$$

其中 $c$ 和 $c'$ 是同一图像两个增强版本的 [CLS] token 表示。

**（c）为什么有效**

这个设计让模型同时学习两个层次的表征：
- **patch 级别**：通过 MIM 损失，模型学会理解局部纹理、形状和结构
- **图像级别**：通过 CLS 损失，模型学会理解全局语义和类别信息

两者相互促进：更好的局部理解有助于全局语义的提取，更好的全局语义有助于局部结构的预测。

**（d）与 Related Work 的关系**

对比学习的相关工作（SimCLR、MoCo、SwAV）只使用 CLS 损失，MIM 相关工作（BEiT、MAE）只使用 patch 预测损失。iBOT 是第一个将两者统一到同一个框架中的工作。

**（e）如果去掉会怎样**

表 3 的消融实验显示：
- 只使用 MIM 损失（去掉 $\mathcal{L}_{\text{CLS}}$）：linear probing 准确率从 79.5% 下降到 76.2%
- 只使用 CLS 损失（去掉 $\mathcal{L}_{\text{MIM}}$）：linear probing 准确率从 79.5% 下降到 74.8%
- 两者结合：79.5%，显著优于单独使用任何一个

#### 创新点 3：共享的预测头（Shared Prediction Head）

**（a）痛点与动机**

在标准的对比学习框架中，在线编码器和目标编码器各自有自己的预测头（projection head）。这导致参数数量翻倍，而且两个预测头可能学到不同的映射，使得对比学习的信号变得不稳定。

**（b）方案细节**

iBOT 让在线编码器和目标编码器**共享同一个预测头**。这个预测头是一个 2 层 MLP，将 ViT 的输出映射到对比学习使用的表示空间。

具体实现是：在线编码器的输出经过预测头得到 $p_i$，目标编码器的输出也经过同一个预测头得到 $q_i$。由于预测头是共享的，$p_i$ 和 $q_i$ 在同一个语义空间中。

**（c）为什么有效**

共享预测头有两个好处：
- **参数效率**：不需要为两个编码器各自维护一个预测头
- **语义对齐**：确保在线和目标表示在同一个语义空间中，避免了两个预测头"各说各话"的问题

**（d）与 Related Work 的关系**

BYOL 和 DINO 使用两个独立的预测头（在线编码器有预测头，目标编码器没有）。MoCo v3 使用两个独立的预测头。iBOT 的共享预测头设计是独特的。

**（e）如果去掉会怎样**

表 3 的消融实验显示，如果使用独立的预测头（不共享），linear probing 准确率从 79.5% 下降到 78.1%，下降了 1.4 个百分点。

### 4.4 公式详解

#### 公式 1：MIM 损失

$$\mathcal{L}_{\text{MIM}} = -\sum_{i \in \mathcal{M}} \log \frac{\exp(p_i \cdot q_i / \tau)}{\sum_{j \in \mathcal{V}} \exp(p_i \cdot q_j / \tau)}$$

**（a）符号定义**
- $\mathcal{M}$：被遮住的 patch 集合，大小为 $m$（通常 $m = 0.4 \times N$，其中 $N$ 是总 patch 数）
- $\mathcal{V}$：所有 patch 的集合，大小为 $N$
- $p_i \in \mathbb{R}^D$：在线编码器对 patch i 的预测表示（经过预测头），$D$ 是表示维度（默认 256）
- $q_i \in \mathbb{R}^D$：目标编码器对 patch i 的输出表示（经过同一个预测头）
- $\tau$：温度参数，控制分布的锐度（默认 0.1）

**（b）公式来源**

这个公式是标准的 InfoNCE 损失 [Oord et al., 2018] 的变体。标准的 InfoNCE 是：

$$\mathcal{L}_{\text{InfoNCE}} = -\log \frac{\exp(q \cdot k_+ / \tau)}{\sum_{i=0}^{K} \exp(q \cdot k_i / \tau)}$$

其中 $q$ 是查询，$k_+$ 是正样本，$k_i$ 是负样本。iBOT 将每个被遮住的 patch 作为查询，目标编码器对应 patch 的表示作为正样本，所有 patch 的表示作为负样本。

**（c）推导过程**

对于每个被遮住的 patch i，我们希望在线编码器的预测 $p_i$ 与目标编码器的表示 $q_i$ 尽可能相似，同时与其他 patch 的表示 $q_j (j \neq i)$ 尽可能不同。

这等价于最大化 $p_i$ 和 $q_i$ 的互信息下界。InfoNCE 损失正是这个互信息下界的估计器。

具体推导：假设我们有一个查询 $p_i$ 和一个包含 1 个正样本和 $N-1$ 个负样本的集合。模型需要从 $N$ 个候选中选出正样本。正确选出正样本的概率是：

$$P(\text{correct}) = \frac{\exp(p_i \cdot q_i / \tau)}{\sum_{j=1}^{N} \exp(p_i \cdot q_j / \tau)}$$

最大化这个概率等价于最小化负对数似然，即 InfoNCE 损失。

**（d）直觉理解**

这个损失函数有三个项：

1. **分子** $\exp(p_i \cdot q_i / \tau)$：鼓励预测和目标表示在语义上对齐
2. **分母中的正项** $\exp(p_i \cdot q_i / \tau)$：同上
3. **分母中的负项** $\sum_{j \neq i} \exp(p_i \cdot q_j / \tau)$：鼓励预测与其他 patch 的表示区分开

**具体例子**：
假设有 $N=196$ 个 patch（14×14 网格），遮罩比例 40%，所以 $m=78$ 个被遮住的 patch。表示维度 $D=256$，温度 $\tau=0.1$。

对于某个被遮住的 patch i，假设 $p_i \cdot q_i = 0.8$（正样本相似度），$p_i \cdot q_j$ 的平均值为 0.1（负样本相似度）。

分子：$\exp(0.8 / 0.1) = \exp(8) \approx 2980$
分母：$\exp(8) + \sum_{j \neq i} \exp(0.1 / 0.1) = 2980 + 195 \times \exp(1) \approx 2980 + 195 \times 2.718 \approx 2980 + 530 = 3510$

损失值：$-\log(2980 / 3510) = -\log(0.849) \approx 0.164$

如果正样本相似度降低到 0.5：
分子：$\exp(5) \approx 148$
分母：$148 + 530 = 678$
损失值：$-\log(148 / 678) = -\log(0.218) \approx 1.523$

可以看到，正样本相似度从 0.8 降到 0.5，损失值从 0.164 飙升到 1.523。这说明损失函数对正样本对齐非常敏感。

**（e）边界情况**

- 当 $\tau \to 0$ 时，损失退化为 hard max 形式，只关注最难的负样本
- 当 $\tau \to \infty$ 时，所有 $\exp(\cdot / \tau) \to 1$，损失退化为 $-\log(1/N) = \log N$，不再有学习信号
- 当 $N=1$（只有正样本，没有负样本）时，损失恒为 0，无法学习

#### 公式 2：CLS 损失

$$\mathcal{L}_{\text{CLS}} = -\log \frac{\exp(c \cdot c' / \tau)}{\sum_{j=1}^{B} \exp(c \cdot c'_j / \tau)}$$

**（a）符号定义**
- $c \in \mathbb{R}^D$：在线编码器对当前图像 x1 的 [CLS] token 表示
- $c' \in \mathbb{R}^D$：目标编码器对同一图像 x2 的 [CLS] token 表示
- $c'_j$：目标编码器对 batch 中第 j 张图像的 [CLS] token 表示
- $B$：batch size

**（b）公式来源**

这是 DINO [Caron et al., 2021] 中使用的对比损失。与 SimCLR 不同，这里没有使用显式的负样本，而是将 batch 内其他图像的表示作为隐式负样本。

**（c）推导过程**

与 MIM 损失的推导类似，但这里是在图像级别而不是 patch 级别。查询是当前图像的 [CLS] 表示，正样本是同一图像另一个增强版本的 [CLS] 表示，负样本是 batch 内其他图像的 [CLS] 表示。

**（d）直觉理解**

这个损失鼓励同一图像的不同增强版本在语义空间中靠近，同时鼓励不同图像的表示相互远离。这与 SimCLR 的核心思想一致，但实现方式不同——SimCLR 使用两个编码器分别处理两个增强，而 iBOT 使用在线-目标架构。

**具体例子**：
假设 batch size $B=256$，表示维度 $D=256$，温度 $\tau=0.1$。

对于当前图像，假设 $c \cdot c' = 0.9$（正样本相似度），$c \cdot c'_j$ 的平均值为 0.05（负样本相似度）。

分子：$\exp(0.9 / 0.1) = \exp(9) \approx 8103$
分母：$8103 + 255 \times \exp(0.05 / 0.1) = 8103 + 255 \times \exp(0.5) \approx 8103 + 255 \times 1.649 \approx 8103 + 420 = 8523$

损失值：$-\log(8103 / 8523) = -\log(0.951) \approx 0.050$

如果正样本相似度降低到 0.5：
分子：$\exp(5) \approx 148$
分母：$148 + 420 = 568$
损失值：$-\log(148 / 568) = -\log(0.261) \approx 1.343$

**（e）边界情况**

- 当 $B=1$ 时，分母中只有正样本，损失恒为 0
- 当 $\tau$ 很小时，模型会过度关注负样本，可能导致表示坍塌
- 当 $\tau$ 很大时，所有样本的相似度权重接近，模型无法区分正负样本

#### 公式 3：目标编码器参数更新

$$\theta' \leftarrow m \theta' + (1 - m) \theta$$

**（a）符号定义**
- $\theta$：在线编码器的参数
- $\theta'$：目标编码器的参数
- $m$：动量系数，通常设为 0.996 到 0.999

**（b）公式来源**

这个 EMA 更新公式来自 MoCo [He et al., 2020] 和 BYOL [Grill et al., 2020]。它的核心思想是让目标编码器的参数缓慢追踪在线编码器的参数，而不是直接复制。

**（c）推导过程**

这是一个简单的指数加权平均。在线编码器的参数通过梯度下降更新，目标编码器的参数通过 EMA 更新：

$$\theta_{t+1}' = m \theta_t' + (1 - m) \theta_{t+1}$$

展开：
$$\theta_{t+1}' = m \theta_t' + (1 - m) \theta_{t+1}$$
$$= m(m \theta_{t-1}' + (1 - m) \theta_t) + (1 - m) \theta_{t+1}$$
$$= m^2 \theta_{t-1}' + m(1 - m) \theta_t + (1 - m) \theta_{t+1}$$

可以看到，目标编码器的参数是过去所有在线编码器参数的加权平均，权重指数衰减。

**（d）直觉理解**

EMA 的作用是让目标编码器变成一个"慢速版本"的在线编码器。这样做的原因是：
- 如果目标编码器和在线编码器完全同步（$m=0$），模型会陷入"捷径解"——预测和目标是同一个东西，损失直接为 0
- 如果目标编码器完全不动（$m=1$），它无法适应训练过程中表示空间的变化
- EMA 提供了一个平滑的折中：目标编码器缓慢跟踪在线编码器的变化，同时保持一定的稳定性

**具体例子**：
假设 $m=0.996$，在线编码器每步更新 $\theta$，目标编码器每步做 EMA。

第 1 步：$\theta_1' = 0.996 \theta_0' + 0.004 \theta_1$
第 100 步：$\theta_{100}'$ 包含了过去约 250 步的在线编码器参数（因为 $0.996^{250} \approx 0.368$，即权重衰减到 $1/e$）

这意味着目标编码器比在线编码器"慢"约 250 步。这个时间差足够防止模型走捷径，又不会让目标编码器太落后。

**（e）边界情况**

- $m=0$：目标编码器与在线编码器完全同步，模型崩溃
- $m=1$：目标编码器不更新，永远使用初始参数，无法学习
- $m$ 接近 1（如 0.9995）：目标编码器更新极慢，可能导致目标与在线编码器差距过大
- $m$ 接近 0（如 0.9）：目标编码器更新太快，可能无法提供稳定的训练信号

## 七、位置

### 前驱工作

1. **DINO [Caron et al., 2021]**：iBOT 的直接前驱。DINO 首次将在线-目标架构应用于 ViT，证明了 [CLS] token 的自蒸馏效果。iBOT 在此基础上增加了 MIM 分支。

2. **BEiT [Bao et al., 2021]**：将 BERT 的掩码预训练引入视觉。iBOT 继承了 BEiT 的遮罩策略和 patch 预测思想，但用在线 tokenizer 替代了固定的 dVAE tokenizer。

3. **BYOL [Grill et al., 2020]**：首次证明不需要负样本的对比学习。iBOT 的 EMA 更新机制直接来自 BYOL。

4. **MoCo [He et al., 2020]**：提出动量编码器（即 EMA 更新）的概念。iBOT 的在线-目标架构可以看作 MoCo 的变体。

### 同期竞品

1. **MAE [He et al., 2021]**：与 iBOT 同时期的工作（2021.11 vs 2021.12）。MAE 使用高比例遮罩（75%）和简单的像素重建目标，计算效率更高。但 MAE 学到的特征在语义理解上不如 iBOT。

2. **MaskFeat [Wei et al., 2021]**：使用 HOG 特征作为预测目标，而不是离散 token。与 iBOT 一样，避免了离散化信息损失。

3. **SimMIM [Xie et al., 2021]**：使用像素回归作为预测目标，简单有效。但像素级别的预测可能缺乏语义性。

### 后续影响

1. **EVA [Fang et al., 2022]**：直接扩展：使用 iBOT 的在线 tokenizer 思想，但将 tokenizer 替换为 CLIP 的视觉编码器。证明了更强的 tokenizer 可以带来更好的预训练效果。

2. **BEiT v2 [Peng et al., 2022]**：架构改进：受到 iBOT 的启发，BEiT v2 也使用了在线 tokenizer，但通过向量量化（VQ）将连续表示映射到离散 token，结合了 iBOT 和 BEiT 的优点。

3. **MVP [Wei et al., 2022]**：范式融合：将 iBOT 的在线 tokenizer 与多模态预训练结合，使用语言模型作为 tokenizer 的监督信号。

4. **PeCo [Dong et al., 2022]**：框架采纳：使用 iBOT 的在线 tokenizer 框架，但用感知损失替代对比损失，让 tokenizer 更关注感知质量。

5. **MaskDistill [Zhou et al., 2022]**：直接扩展：将 iBOT 的在线 tokenizer 与知识蒸馏结合，使用教师模型作为 tokenizer。

## 八、局限

### 局限 1：计算开销

**问题**：iBOT 需要同时运行两个 ViT（在线和目标），计算量是单模型的两倍。相比之下，MAE 只用一个编码器和一个轻量解码器，计算效率更高。

**什么时候重要**：在计算资源有限的情况下（如小团队、个人研究者），iBOT 的训练成本可能过高。

**论文是否承认**：论文没有明确讨论计算开销问题。

**后续工作**：EVA 和 BEiT v2 通过使用更高效的 tokenizer 设计来缓解这个问题。

### 局限 2：对数据增强的依赖

**问题**：iBOT 的两个损失都依赖于数据增强。MIM 损失需要不同的增强版本作为正样本，CLS 损失更是直接依赖增强。如果数据增强不够强，模型可能学到的是"增强不变性"而不是真正的语义理解。

**什么时候重要**：在数据量小或数据增强受限的场景（如医学图像、卫星图像），iBOT 的性能可能下降。

**论文是否承认**：论文在消融实验中展示了数据增强的影响，但没有深入讨论这个局限。

**后续工作**：一些工作尝试用扩散模型生成增强数据来缓解这个问题。

### 局限 3：遮罩策略的敏感性

**问题**：iBOT 使用 block-wise 遮罩（相邻 patch 一起遮住），遮罩比例 40%。这个策略是手动设计的，可能不是最优的。不同的遮罩策略（如随机遮罩、grid 遮罩）可能带来不同的效果。

**什么时候重要**：当应用到不同领域（如视频、3D 点云）时，遮罩策略可能需要重新设计。

**论文是否承认**：论文在消融实验中比较了不同遮罩比例，但没有系统研究遮罩策略。

**后续工作**：一些工作（如 Mask3D）探索了针对 3D 数据的遮罩策略。

## 九、小结

iBOT 的核心贡献是提出了**在线 tokenizer**，让掩码图像建模的 tokenizer 可以在训练过程中端到端地学习，而不是使用固定的预训练 tokenizer。这个简单但有效的设计，加上 MIM 和对比学习的统一框架，使得 iBOT 在 ImageNet 线性探测上达到 79.5% 的准确率，比 BEiT 高出 22.8 个百分点。

iBOT 的深层洞见是：**视觉预训练的目标函数和 tokenizer 应该一起学习**。这个思想影响了后续的 BEiT v2、EVA 等工作，推动了自监督视觉预训练从"固定 tokenizer"范式向"可学习 tokenizer"范式的转变。

如果说 MAE 证明了"简单就是美"（用像素重建就能学到好特征），那么 iBOT 证明了"端到端才是真"（让 tokenizer 一起学习才能发挥最大潜力）。这两个工作共同推动了 2021-2022 年自监督视觉预训练的繁荣。
