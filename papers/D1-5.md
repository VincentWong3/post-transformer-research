# ALIGN: Scaling Up Visual and Vision-Language Representation Learning With Noisy Text Supervision（2021）

**论文：** [arXiv:2102.05918](https://arxiv.org/abs/2102.05918) · Chao Jia, Yinfei Yang, Ye Xia, Yi-Ting Chen, Zarana Parekh, Hieu Pham, Quoc V. Le, Yunhsuan Sung, Zhen Li, Tom Duerig · Google · ICML 2021

**技术分支：** D1 — 视觉模型和语言模型是否可以结合？ / D1-2 — 对齐式大规模预训练范式

---

## 一、先搞清楚"问题在哪里"

2021 年初，NLP 已经全面转向无监督预训练——BERT、GPT、T5 都在大规模原始文本上训练，不需要人工标注。但视觉和视觉-语言模型还在"吃精粮"：

- 视觉模型高度依赖 ImageNet、OpenImages、JFT-300M 这种人工标注了类标签的数据集
- 视觉-语言模型好一点，但像 Conceptual Captions（CC-3M）、MSCOCO、Visual Genome 等数据集依然需要大量人工筛选、清洗、语义解析

论文原文的描述很清楚：

> "This costly curation process limits the size of datasets and hence hinders the scaling of trained models."

当时的典型视觉-语言预训练数据集只到 ~10M 级别——比 NLP 的数十亿语料小至少一个数量级。

与此同时，**谷歌和 OpenAI 都想到了同一个方向**：互联网上有巨量的图片替代文本（alt-text），大多数没人管、质量很差，但数量是天文数字。如果能直接用这些"噪声数据"做对比学习，是不是数据规模可以弥补数据质量？

**CLIP（OpenAI）和 ALIGN（Google）在 2021 年 2 月几乎同时发表。** 两者的核心思路和实验结果惊人地相似——这说明"用对比学习做图文对齐"是那个时间点上收敛出来的必然方向。

CLIP 对数据做了更多的清洗（构建高频视觉概念的 allowlist），而 **ALIGN 赌了一把更大的：几乎不做清洗，直接用 1.8B 噪声图文对**。论文的核心论点是：

> "The scale of our corpus can make up for its noise."

**数据规模 > 数据质量**——这是 ALIGN 最大的贡献，也是它和 CLIP 最本质的区别。

## 二、旧路线的真正问题

### 2.1 视觉-语言数据集的天花板

在 ALIGN 之前，视觉-语言预训练数据集的典型大小：

- **Conceptual Captions（CC-3M）：** ~3M 图文对，经过了复杂的过滤和后处理（过滤掉低质量图片、清理文本、平衡类别分布）
- **Visual Genome：** ~5M 密集描述，需要人类标注
- **MSCOCO Captions：** ~1.2M 图文对（120K 图片 × ~5 描述/图）
- **ImageBERT：** 类似规模

这些都在 10M 级别。而当时 NLP 的预训练数据集（如 Common Crawl）已经是 TB 级、数十亿文档级别。

为什么视觉-语言数据这么少？因为**清洗过程太贵了**。Conceptual Captions 的构建流程包括：从 Web 抓取 alt-text → 去重 → 语言检测 → 过滤色情内容 → 过滤低质量/垃圾文本 → 多步后处理。每一步都过滤掉大量数据。

### 2.2 噪声数据的困境

直接放开清洗会引入严重噪声：alt-text 可能完全和图片无关（"1920x1080"、图片文件名、随机字符串）。在噪声数据上训练，传统观点认为模型学不到好特征。

但自监督学习领域的一个趋势正在形成：**对比学习天然对噪声鲁棒。** SimCLR（2020）、MoCo（2020）等工作已经证明，对比学习在大量未标注数据上可以学到比监督学习更好的视觉特征。那么，把图片的 alt-text 看作"自然标注"，用对比学习在噪声图文对上训练，是不是也能 work？

### 2.3 和 CLIP 的关系

CLIP 和 ALIGN 在同一时间做出几乎相同的技术选择，但有一个关键差异：

- **CLIP 的数据构建：** 从 Wikipedia 提取高频视觉概念构成 allowlist，然后从互联网匹配相应图文对——这仍然算某种程度的专家知识干预
- **ALIGN 的数据构建：** 直接复用 Conceptual Captions 的原始数据管道，但几乎去掉所有清洗步骤——**最小化人工干预**

ALIGN 论文原文坦诚地指出：

> "Closely related to our work is CLIP (Radford et al., 2021), which proposes visual representation learning via natural language supervision in a similar contrastive learning setting. Besides using different vision and language encoder architectures, the key difference is on training data: ALIGN follows the natural distribution of image-text pairs from the raw alt-text data, while CLIP collects the dataset by first constructing an allowlist of high-frequency visual concepts from English Wikipedia."

翻译一下：**你们的做法需要专家知识，我们不需要。**

## 三、这篇论文的核心思路

**核心思路异常简单：从 Web 收集 1.8B 噪声图文对，用双塔编码器做对比学习，证明大量噪声数据可以替代精心筛选的数据。**

```text
噪声图文数据（1.8B alt-text 对）
   ↓ 最小频率过滤 → 保留自然分布
双塔编码器（EfficientNet + BERT）
   ↓ 对比学习（normalized softmax loss）
联合嵌入空间
   ↓ 下游任务
零样本分类 / 跨模态检索 / 视觉迁移
```

技术路线的谱系位置：

```text
VSE（视觉语义嵌入，2013/2018） → 小数据上的双塔对比学习
   ↓
ViLBERT / LXMERT / UNITER（2019） → 复杂 cross-attention 架构，10M 级别数据
   ↓
CLIP（2021.01 arXiv）→ 400M 数据，allowlist 筛选
   ↓
ALIGN（2021.02 arXiv）→ 1.8B 数据，最小清洗 ← 本篇
   ↓
SigLIP（2023）→ 用 sigmoid loss 替代 softmax，进一步提升
```

ALIGN 和 CLIP 几乎同期，但 ALIGN 在"数据规模 vs 数据质量"这个维度上走得更极端。

## 四、整体结构与前向流程

### 4.1 模型架构

ALIGN 使用最简单的**双塔架构**：

```text
┌─────────────────┐      ┌───────────────────┐
│   Image Tower    │      │    Text Tower      │
│  EfficientNet    │      │  BERT（[CLS] token）│
│  (global pool)   │      │  + linear projection│
└────────┬────────┘      └────────┬──────────┘
         │                        │
         ▼                        ▼
     L2 normalize             L2 normalize
         │                        │
         └──────────┬─────────────┘
                    │
                    ▼
          cosine similarity
                    │
                    ▼
        normalized softmax loss
        (i2t + t2i, symmetric)
```

**具体配置：**

- **Image encoder：** EfficientNet（有全局池化但不训练分类头的 1×1 conv），输入分辨率 289×289（训练），首先 resize 到 346×346，然后随机 crop
- **Text encoder：** BERT（[CLS] token embedding），100K wordpiece 词汇表（从训练数据生成），最大序列长度 64 tokens（因为输入文本不超过 20 个 unigram）
- **对齐层：** BERT 顶部加全连接层（线性激活），将维度对齐到 image tower 的输出维度
- **两个 encoder 从零开始训练**（不使用预训练权重）

### 4.2 训练配置

| 超参数 | 值 |
|--------|-----|
| Optimizer | LAMB（同时训练 image + text encoder） |
| Learning rate | warmup 到 1e-3（10K steps）→ 线性衰减至 0（1.2M steps, ~12 epochs） |
| Weight decay | 1e-5 |
| Batch size | 16384（1024 TPUv3 cores × 16 pairs/core） |
| Label smoothing | 0.1 |
| Temperature | 可学习（初始化 1.0，收敛到约 1/64） |
| Image resolution | 289×289（训练），346→crop |

LAMB 的选择很有意思——论文说尝试了 SGD with momentum（CNN 用得好）和 ADAM（BERT 用得好），发现 LAMB 是同时训练两个 encoder 的最佳选择。

### 4.3 损失函数

ALIGN 使用对称的 normalized softmax loss（也叫 infoNCE）：

**Image-to-text loss：**

$$L_{i2t} = -\frac{1}{N}\sum_{i}^{N}\log\frac{\exp(x_i^\top y_i / \sigma)}{\sum_{j=1}^{N}\exp(x_i^\top y_j / \sigma)}$$

**Text-to-image loss：**

$$L_{t2i} = -\frac{1}{N}\sum_{i}^{N}\log\frac{\exp(y_i^\top x_i / \sigma)}{\sum_{j=1}^{N}\exp(y_i^\top x_j / \sigma)}$$

总损失： $L = L_{i2t} + L_{t2i}$

其中 $x_i$ 和 $y_j$ 是 L2 归一化的 image/text embedding， $N$ 是 batch size， $\sigma$ 是 temperature（可学习参数）。

核心设计点：
- **对称 loss：** 同时做 image→text 和 text→image 的对比学习
- **In-batch negatives：** 每个 batch 内，所有不匹配的图文对都是负样本
- **All-gather 跨设备：** 将不同 TPU core 上的 embedding 拼接，形成更大的负样本集（batch size 16384 = 1024 cores × 16）
- **可学习 temperature：** 自动调节 logits 的 scale，不需要手动搜索（论文观察到温度在前 100K steps 快速下降到收敛值的约 1.2 倍，然后缓慢降到底）

### 4.4 数据构建

数据来源：复用 Conceptual Captions 构建流程，但几乎去掉所有清洗步骤。

**最终数据集：1.8B 图文对，两个数量级大于 CC-3M。**

具体过滤（非常轻量）：

**Image-based：**
- 移除色情内容
- 短边 > 200 像素，长宽比 < 3
- 丢弃有超过 1000 个 alt-text 的图片
- 移除下游测试集（ILSVRC-2012、Flickr30K、MSCOCO）中的重复/近似重复图片

**Text-based：**
- 移除被超过 10 张图片共享的 alt-text（往往是无关的文本如 "1920x1080"、"alt_img"、"cristina"）
- 移除包含罕见 token 的 alt-text（不在 top 1 亿最频繁 unigram/bigram 中）
- 移除过短（≤3 unigram）或过长（>20 unigram）的文本

结束。就是这么简单。

## 五、实验结果

### 5.1 Zero-shot 图像分类

把类别名作为 text query 喂给 text encoder，用 image-text 检索的方式做分类。使用了 CLIP 定义的 prompt ensemble（每个类别用多个 template 扩展后取平均 embedding，论文发现这带来了 2.9% 的提升）。

**Table 4: Zero-shot 分类（EfficientNet-L2）**

| 数据集 | CLIP | ALIGN |
|--------|------|-------|
| ImageNet | 76.2 | **76.4** |
| ImageNet-R | 88.9 | **92.2** |
| ImageNet-A | **77.2** | 75.8 |
| ImageNet-V2 | 70.1 | 70.1 |

关键观测：
- **ImageNet zero-shot 持平：** ALIGN 的 76.4% 和 CLIP 的 76.2% 几乎一样——但 ALIGN 用了没清洗的噪声数据
- **ImageNet-R 大幅领先（+3.3%）：** ALIGN 在艺术、卡通、素描等非自然图像上更好，可能因为噪声数据带来了更多样的视觉概念分布
- **ImageNet-A 略低：** 在对抗性样本上好模型不一定是好模型

### 5.2 Fine-tuned 图像分类

**Table 5: ImageNet 分类（EfficientNet-L2）**

| 方法 | Backbone | Acc@1（linear probe） | Acc@1（fine-tune） | Acc@5 |
|------|----------|-----------------------|--------------------|-------|
| WSL | ResNeXt-101 32x48d | 83.6 | 85.4 | 97.6 |
| CLIP | ViT-L/14 | 85.4 | - | - |
| BiT | ResNet152x4 | - | 87.54 | 98.46 |
| NoisyStudent | EfficientNet-L2 | - | 88.4 | 98.7 |
| ViT | ViT-H/14 | - | 88.55 | - |
| Meta-Pseudo-Labels | EfficientNet-L2 | - | 90.2 | 98.8 |
| **ALIGN** | **EfficientNet-L2** | **85.5** | **88.64** | **98.67** |

- linear probe（冻结 backbone）：85.5%，超过 CLIP（85.4%）和 WSL（83.6%）
- fine-tune：88.64%，优于 BiT 和 ViT，但不如 Meta-Pseudo-Labels（依赖 ImageNet 和大规模无标签数据的深度交互，且用 800 分辨率测试）
- 论文特别指出 ALIGN 比 NoisyStudent 节省 44% FLOPs（600 分辨率 vs 800）

### 5.3 VTAB（19 个任务）

**Table 6: VTAB 对比**

| 方法 | 全部 | Natural | Specialized | Structured |
|------|------|---------|-------------|------------|
| BiT-L | 78.72 | - | - | - |
| **ALIGN** | **79.99 ± 0.15** | **83.38** | **87.56** | **73.25** |

每个任务 1000 训练样本（800 train + 200 val），50 次超参搜索后选择最优配置。ALIGN 在所有 category 上超过 BiT-L。

### 5.4 细粒度分类迁移

**Table 7**

| 模型 | Oxford Flowers | Oxford Pets | Stanford Cars | Food101 |
|------|---------------|-------------|---------------|---------|
| BiT-L | 99.63 | 96.62 | - | - |
| SAM-baseline | 99.60 | 96.92 | 95.07 | 96.03 |
| SAM-final | 99.65 | 97.10 | 95.96 | 96.18 |
| **ALIGN** | **99.65** | **96.19** | **96.13** | **95.88** |

ALIGN 和 SAM（Sharpness-Aware Minimization）的效果相当，但没有用任何特殊优化器。

### 5.5 跨模态检索（Flickr30K / MSCOCO）

**Table 1: Zero-shot 检索**

| 方法 | Flickr30K i2t R@1 | Flickr30K t2i R@1 | MSCOCO i2t R@1 | MSCOCO t2i R@1 |
|------|-------------------|-------------------|----------------|----------------|
| CLIP | 88.0 | 68.7 | 58.4 | 37.8 |
| **ALIGN** | **88.6** | **75.7** | **58.6** | **45.6** |

Zero-shot t2i 上比 CLIP 高 7%（Flickr）和 7.8%（COCO）。

**Table 1: Fine-tuned 检索**

| 方法 | Flickr30K i2t R@1 | Flickr30K t2i R@1 | MSCOCO i2t R@1 | MSCOCO t2i R@1 |
|------|-------------------|-------------------|----------------|----------------|
| UNITER（cross-attention） | 87.3 | 75.6 | 65.7 | 52.9 |
| GPO | 88.7 | 76.1 | 68.1 | 52.7 |
| **ALIGN** | **95.3** | **84.9** | **77.0** | **59.9** |

论文特别强调：ALIGN 使用最简单的 VSE 形式（dual-encoder + cosine similarity），却超越了所有包含复杂 cross-attention 层的模型（UNITER、ImageBERT、ERNIE-ViL、VILLA 等）。而且 cross-attention 模型在推理时相比双塔架构"orders of magnitudes slower"。

### 5.6 Crisscrossed Captions（CxC）检索

**Table 2**

| 任务 | 最优基线 | ALIGN | 提升 |
|------|---------|-------|------|
| image→text R@1 | 55.9 (DET2T+I2T) | **78.1** | **+22.2** |
| text→image R@1 | 41.7 (DET2T+I2T) | **61.8** | **+20.1** |

跨模态检索的优势巨大，但**模态内检索**（text→text、image→image）提升不明显——论文推测这是因为训练目标只关注跨模态对齐，不关注模态内相似性。

### 5.7 多语言 ALIGN（ALIGN multiling）

论文还做了多语言版本：移除数据管道的语言限制，扩展到 100+ 语言（同样 1.8B 图文对）。修改 BERT 词汇表到 250K。

**Table 11: Multi30K zero-shot**

| 方法 | en | de | fr | cs |
|------|----|----|----|----|
| M3P zero-shot | 57.9 | 36.8 | 27.1 | 20.4 |
| **ALIGN multiling zero-shot** | **90.2** | **84.1** | **84.9** | **63.2** |

零样本结果直接大幅超过 M3P 的零样本，甚至在绝大多数语言上接近 M3P 和 UC2 微调后的水平（en: 90.2 vs M3P 87.7, de: 84.1 vs M3P 82.7, fr: 84.9 vs UC2 83.9）。

## 六、核心洞察与消融实验

### 6.1 数据规模 > 数据质量（最重要的消融）

**Table 9: 不同数据规模**

| 模型 + 数据 | MSCOCO i2t R@1 | MSCOCO t2i R@1 | ImageNet KNN R@1 |
|------------|----------------|----------------|------------------|
| B7+BERT-base + ALIGN full | **55.4** | **41.7** | **69.3** |
| B7+BERT-base + ALIGN 10% | 52.0 | 39.2 | 68.8 |
| B7+BERT-base + CC-3M | 18.9 | 15.5 | 48.7 |
| B3+BERT-mini + ALIGN full | 37.4 | 24.5 | 56.5 |
| B3+BERT-mini + ALIGN 10% | 36.7 | 24.4 | 55.8 |
| B3+BERT-mini + CC-3M | 22.1 | 17.3 | 48.9 |

关键发现：
- **更大的模型需要更大的数据：** B7+BERT-base 在 CC-3M 上过拟合（18.9 R@1，甚至不如 B3+BERT-mini 的 22.1），但在完整 ALIGN 数据上大幅提升（55.4 R@1）
- **小模型数据饱和：** B3+BERT-mini 在 10% ALIGN 数据上已经基本饱和（36.7 vs 37.4）

**Table 10: 数据质量 vs 数据规模的直接对比**

| 数据 | 规模 | MSCOCO i2t R@1 | ImageNet KNN R@1 |
|------|------|----------------|------------------|
| CC-3M（清洗） | 3M | 18.9 | 48.7 |
| ALIGN（噪声） | 3M | 8.1 | 41.3 |
| ALIGN（噪声） | 6M | 15.8 | 47.9 |
| ALIGN（噪声） | 12M | **23.8** | **51.4** |

这是整篇论文中**最关键的数字**：

- 同样 3M 规模，清洗的 CC-3M 大幅优于噪声的 ALIGN 数据（18.9 vs 8.1 R@1）
- 但把噪声数据扩大到 12M（仅 4 倍），ALIGN 的噪声数据已经超越 CC-3M（23.8 vs 18.9）
- 噪声数据 + 4 倍规模 = 更好的模型

这就是"data scale can make up for noise"最直接的证据。

### 6.2 模型架构消融

**Table 8: 关键架构参数**

| 变体 | MSCOCO i2t R@1 | MSCOCO t2i R@1 | ImageNet KNN R@1 |
|------|----------------|----------------|------------------|
| 基线（B5+BERT-base, dim=640, all negs, learnable temp） | 51.7 | 37.5 | 64.6 |
| embedding dim=320 | 50.3 | 34.1 | 64.0 |
| embedding dim=160 | 47.0 | 34.4 | 63.7 |
| embedding dim=80 | 42.0 | 29.3 | 61.9 |
| 50% in-batch negs | 50.2 | 37.0 | 63.8 |
| 25% in-batch negs | 48.7 | 35.8 | 63.3 |
| temp=1/128 | 52.2 | 36.5 | 64.8 |
| temp=1/64 | 52.2 | 37.3 | 64.8 |
| temp=1/32 | 39.6 | 26.9 | 61.2 |

关键发现：
- **Embedding 维度升高有帮助**（640 > 320 > 160 > 80），因此更大的 EfficientNet backbone 使用更高维度（L2 用 1376）
- **更多 in-batch negatives 更好**（100% > 50% > 25%）——印证了 batch size 越大对比学习越有效的观点
- **Temperature 的关键性：** temp=1/32 表现极差（39.6 R@1 vs 52.2）。学习到的 temperature 与最优固定值（1/64 或 1/128）相当——~1/64 是最优区间

### 6.3 Scaling 规律

**Figure 3: 不同 backbone 组合**

论文发现清晰的 scaling 规律：
- Image encoder 容量对视觉任务更重要（L2 + BERT-Mini 优于 B7 + BERT-Large 在 ImageNet KNN 上）
- 图文检索任务上，image 和 text encoder 的重要性大致相当
- ImageNet KNN 从 BERT-Base 开始趋于饱和（BERT-Large 增益有限）

## 七、嵌入空间的分析

### 7.1 细粒度文本检索

论文用 160M CC-BY 图片（不在训练集内）建立了检索系统的演示（Figure 4）。ALIGN 可以：
- 用详细场景描述检索图片
- 细粒度/实例级概念（地标、艺术品）
- 对训练集中不存在的复杂概念泛化

### 7.2 嵌入空间的组合性（Figure 5）

论文展示了 ALIGN 嵌入空间中的线性组合能力——这让人联想到 word2vec 的 "king - man + woman = queen"：

- **Image + text 检索：** 输入熊猫图片 + "Australia" → 输出澳大利亚的类似动物（袋熊）
- **Image + text 检索：** 黑鞋图片 + "beige" → 输出同款米色鞋
- **Image - text 移除：** 图片 - "带饰品" → 输出没饰品但相同场景的图片

这说明 ALIGN 在跨模态嵌入空间中学习到了有意义的线性关系——**不需要显式的组合训练，组合性自动涌现**。

### 8.2 前驱工作

- **VSE / VSE++（2013/2018）：** 首次用双塔+对比学习做视觉语义嵌入。但限制在 Flickr/COCO 的小数据上
- **caption prediction 路线（VirTex, ICMLM, 2020）：** 从图像预测文本描述来学习视觉特征。也用小数据
- **cross-attention V+L 模型（ViLBERT, UNITER, 2019-2020）：** 用 Transformer cross-attention 做更精细的图文交互，效果好但"orders of magnitudes slower"，不适合生产检索系统

## 十、局限性与后续方向

### 10.1 模态内检索表现平平

ALIGN 在跨模态检索上大幅领先，但在 text→text 和 image→image 的模态内检索上提升不大。CxC 数据集上 ALIGN 模态内检索的提升不大——论文指出原因：训练目标只关注跨模态匹配，不关注模态内关系。后续工作中引入多任务学习的方案（如 CxC 论文本身建议的 multi-task）可以有效缓解。

### 10.2 数据偏见

论文明确讨论了社会影响：alt-text 可能包含有害内容（歧视、色情），不做过滤直接训练可能放大这些偏见。此外，Web 数据的分布天然偏向某些人口群体和文化，可能导致模型性能在弱势群体或非主流文化上表现不佳。

### 10.3 模型规模不是极致

ALIGN 最大的配置是 EfficientNet-L2 + BERT-Large，在 2021 年属于大规模模型但远不是极限。后续的 EVA-CLIP、SigLIP 等都在模型规模上继续推进。ALIGN 的多语言版本也用了更简单的 BERT 架构——如果用更大的 text encoder（如 T5）可能进一步提升。

### 10.4 和 ViT 的差距

ALIGN 的 image encoder 用 EfficientNet（CNN），而 CLIP 用了 ViT。后来 ViT 在多模态模型中逐渐占据主导（如 CLIP 自身的 ViT-L/14 版本在 zero-shot 上表现更好）。ALIGN 没有探索 Transformer 作为 image encoder，这是架构选择上的局限。

## 十一、范式意义

**ALIGN 的范式意义：它用最极端的方式证明了"数据规模可以弥补数据质量"——噪声 + 对比学习 = 好特征。**

这个发现的深远影响不止于一篇论文：

1. **降低了视觉-语言预训练的数据门槛。** 不需要复杂的清洗、不需要人工标注、不需要专家构建 allowlist——只需要 Web 上现成的 alt-text。这意味着任何有足够算力的团队都能复现类似的工作。

2. **为后续数据工程提供了新方向。** ALIGN 之后，社区不再追求"更干净的数据"，而是转向"更大、更全、更多样化的数据"。BLIP 的 caption bootstrapping、LiT 的 text encoder 锁定、SigLIP 的 loss 改进——都是在 ALIGN 的"大量噪声数据"基线上做增量。

3. **同时期的技术收敛。** ALIGN 和 CLIP 在 2021 年 1-2 月几乎同时出现，核心方法几乎相同（双塔 + 对比学习 + 大规模数据）。这不是巧合——而是"自监督 + 对比学习 + Web 数据"这条技术路线成熟的标志。

4. **双塔架构的胜利。** ALIGN 用最简单的双塔 + cosine similarity 超越了所有包含复杂 cross-attention 的模型。这证明了对于检索任务，**embedding 质量比交互复杂度更重要**。相比于 ViLBERT/UNITER 等需要 O(n²) 交叉注意力计算的模型，双塔可以提前计算和索引 embedding，推理时只需快速最近邻搜索——这才是生产级检索系统的正确设计。

**在技术树中，ALIGN 和 CLIP 共同构成了 D1-2 "对齐式大规模预训练"范式的 foundation node。** 它们的共同信念是：一个简单到极致的对比学习目标函数 + 足够大规模的数据 = 可迁移的跨模态表示。这个信念深刻影响了后续几乎所有图文预训练工作。
