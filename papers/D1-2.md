# LXMERT: Learning Cross-Modality Encoder Representations from Transformers（2019）

**论文：** [arXiv](https://arxiv.org/abs/1908.01495) · Tan & Bansal · 2019

---

## 一、先搞清楚坑在哪

视觉-语言（Vision-and-Language, V&L）任务要解决的核心问题是：**如何让模型同时理解图像和文本，并捕捉它们之间的对应关系**。典型的任务包括：

- **视觉问答（VQA）**：给一张图 + 一个问题，输出答案
- **图像描述（Captioning）**：给一张图，生成一句描述
- **视觉推理（NLVR2）**：给两张图 + 一句描述，判断描述是否正确
- **指代表达理解（RefCOCO+）**：给一张图 + 一句描述，定位对应的物体

这些任务的关键在于，模型必须建立语言中的词（如"红裙子"）与图像中的区域（如某个 bounding box）之间的跨模态对应关系。

在 LXMERT 之前，主流做法分为两类：

1. **单流模型（Single-stream）**：如 ViLBERT 的同期工作 VisualBERT，将图像区域特征和文本 token 拼接后送入一个 Transformer 编码器。缺点是：同一层 self-attention 既做模态内也做跨模态交互，难以区分。
2. **两阶段方法**：先用目标检测器提取图像区域特征，再用 LSTM/注意力机制做跨模态融合。缺点是：跨模态交互不够充分，且没有大规模预训练。

**核心矛盾**：现有方法要么跨模态交互不够强（两阶段），要么模态内和跨模态交互混在一起（单流），导致模型难以同时学好模态内表示和跨模态对齐。

## 二、现有方法的真正问题

具体来说，单流模型（如 VisualBERT）把图像特征和文本 token 拼成一个序列：

```
[CLS] w1 w2 ... wn [SEP] obj1 obj2 ... objk [SEP]
```

然后送入一个 Transformer。这里的问题在于：

1. **注意力混叠**：同一个 self-attention 层既要处理"猫"和"狗"的文本关系（模态内），又要处理"猫"和图像中某个区域的关系（跨模态）。模型需要同时学到两种不同的注意力模式，训练难度大。
2. **无法区分交互类型**：单流模型没有显式的模块来区分"文本内部交互"、"图像内部交互"和"文本-图像交互"。
3. **缺乏针对性的预训练任务**：很多方法直接拿 BERT 的 MLM（Masked Language Modeling）做预训练，但没有设计针对跨模态对齐的任务。

## 三、LXMERT 的核心思路

**一句话总结**：LXMERT 设计了一个三编码器架构——一个文本编码器、一个图像编码器、一个跨模态编码器——分别负责语言内部、视觉内部和语言-视觉之间的交互，并通过五个预训练任务联合训练。

![图 1：LXMERT 整体架构](figures/lxmert/fig1_architecture.png)

*图 1：LXMERT 的三编码器架构。左侧是文本编码器（蓝色），右侧是图像编码器（绿色），中间是跨模态编码器（橙色）。文本和图像特征先独立编码，然后通过 cross-attention 层进行跨模态交互。*

核心设计思路是**分治**：先把模态内的关系建模好，再让它们交互。这就像让两个专家先各自熟悉自己的领域，然后一起讨论跨领域的问题。

## 四、网络架构详解

### 4.1 输入表示

**文本输入**：
- 使用 BERT tokenizer 将句子分割成 subword tokens
- 每个 token 的表示 = token embedding + segment embedding + position embedding
- 输入形状：`[batch_size, seq_len, d_model]`，其中 `seq_len ≤ 20`（LXMERT 的默认设置）

**图像输入**：
- 使用 Faster R-CNN（预训练在 Visual Genome 上）提取图像中的物体区域
- 每个区域输出一个 2048 维的 RoI 特征 + 一个 5 维的 bounding box 坐标 `[x1, y1, x2, y2, w*h]`
- 通过线性投影将 RoI 特征降到 `d_model` 维
- 通过另一个线性层将坐标编码到 `d_model` 维
- 最终图像特征 = RoI 特征 + 坐标编码
- 输入形状：`[batch_size, num_objects, d_model]`，其中 `num_objects = 36`（默认保留 36 个检测框）

### 4.2 前向传播过程

```
文本输入 (batch, 20, 768)    图像输入 (batch, 36, 768)
        │                           │
        ▼                           ▼
┌─────────────────┐         ┌─────────────────┐
│  文本编码器 (×9)  │         │  图像编码器 (×5)  │
│  Self-Attention  │         │  Self-Attention  │
│  FFN             │         │  FFN             │
└────────┬─────────┘         └────────┬─────────┘
         │                            │
         ▼                            ▼
  文本特征 (batch, 20, 768)    图像特征 (batch, 36, 768)
         │                            │
         └────────────┬───────────────┘
                      │
                      ▼
         ┌─────────────────────────┐
         │     跨模态编码器 (×5)     │
         │                         │
         │  ┌─────────────────┐   │
         │  │   Cross-Attention│   │
         │  │   文本→图像      │   │
         │  └─────────────────┘   │
         │  ┌─────────────────┐   │
         │  │   Cross-Attention│   │
         │  │   图像→文本      │   │
         │  └─────────────────┘   │
         │  ┌─────────────────┐   │
         │  │   Bi-Attention   │   │
         │  └─────────────────┘   │
         │  ┌─────────────────┐   │
         │  │      FFN        │   │
         │  └─────────────────┘   │
         └────────┬───────────────┘
                  │
                  ▼
     跨模态特征 (batch, 20+36, 768)
```

**关键模块详解**：

#### 文本编码器（Language Encoder）
- 9 层 Transformer，每层包含 self-attention + FFN
- 只做文本内部的 self-attention，不接触图像信息
- **为什么 9 层？** 论文实验发现，文本编码器需要足够深才能学到好的语言表示。9 层是经验最优值（见消融实验 Table 6）

#### 图像编码器（Visual Encoder）
- 5 层 Transformer，每层包含 self-attention + FFN
- 只做图像区域之间的 self-attention，捕捉物体之间的关系
- **为什么 5 层？** 图像区域的交互相对简单（主要是空间关系和语义关系），5 层足够

#### 跨模态编码器（Cross-Modality Encoder）
- 5 层，每层包含三个关键组件：
  1. **Cross-Attention（文本→图像）**：文本 token 作为 query，图像区域作为 key/value
  2. **Cross-Attention（图像→文本）**：图像区域作为 query，文本 token 作为 key/value
  3. **Bi-Attention（双向注意力）**：文本和图像 token 拼接后做 self-attention

**为什么需要三个注意力机制？**
- 文本→图像注意力：让语言中的词关注到相关的图像区域（如"dog"关注狗的区域）
- 图像→文本注意力：让图像区域关注到相关的文本 token（如狗的区域关注"dog"这个词）
- 双向注意力：让所有 token 之间充分交互，捕捉更复杂的跨模态关系

### 4.3 输出处理

- 跨模态编码器输出形状：`[batch_size, 56, 768]`（20 个文本 token + 36 个图像区域）
- 取 `[CLS]` token 的输出作为全局表示，用于分类任务
- 取所有文本 token 的输出用于语言相关任务（如 MLM）
- 取所有图像区域的输出用于视觉相关任务（如 MRM）

## 五、核心创新点

### 5.1 创新点一：三编码器架构

#### (a) 痛点与动机

单流模型（如 VisualBERT）把所有 token 拼在一起做 self-attention，导致：
1. 模型需要同时学习三种注意力模式（文本-文本、图像-图像、文本-图像），训练难度大
2. 文本和图像的 embedding 空间不同，直接拼接可能导致表示不匹配
3. 没有显式的模块来专门处理跨模态交互

#### (b) 方案细节

LXMERT 将编码器分为三个独立的部分：
- 文本编码器（9 层）：纯语言 self-attention
- 图像编码器（5 层）：纯视觉 self-attention
- 跨模态编码器（5 层）：三种注意力机制组合

#### (c) 为什么有效

消融实验（Table 6）显示：
- 去掉跨模态编码器（只用文本+图像编码器）：VQA 准确率下降 2.3%
- 去掉图像编码器（只用文本+跨模态编码器）：VQA 准确率下降 1.8%
- 去掉文本编码器（只用图像+跨模态编码器）：VQA 准确率下降 1.5%

这说明每个编码器都贡献了独特的价值。

#### (d) 与 Related Work 的关系

- **ViLBERT**（Lu et al., 2019）是同期工作，也用了类似的双流架构，但 LXMERT 多了图像编码器（ViLBERT 没有独立的视觉 self-attention）
- **VisualBERT**（Li et al., 2019）是单流模型，把所有 token 拼在一起
- LXMERT 的三编码器设计是对 ViLBERT 的改进，增加了视觉内部的交互

#### (e) 如果去掉会怎样

从消融实验（Table 6）看：
- 去掉跨模态编码器：VQA 从 72.4% 降到 70.1%（↓2.3%）
- 去掉图像编码器：VQA 从 72.4% 降到 70.6%（↓1.8%）
- 去掉文本编码器：VQA 从 72.4% 降到 70.9%（↓1.5%）

三者都不可或缺。

### 5.2 创新点二：五个预训练任务

#### (a) 痛点与动机

之前的 V&L 预训练方法通常只用 1-2 个任务（如 MLM + ITM），无法充分学习跨模态对齐。具体问题：
1. MLM 只学习语言，不涉及视觉
2. ITM（Image-Text Matching）是全局匹配，不学习细粒度的区域-词对齐
3. 没有任务让模型学习图像内部的关系

#### (b) 方案细节

LXMERT 设计了五个预训练任务：

**1. Masked Language Modeling (MLM)**
- 随机 mask 15% 的文本 token
- 模型需要根据图像和其他文本 token 预测被 mask 的词
- 损失函数：交叉熵

**2. Masked Object Prediction (视觉版 MLM)**
- 随机 mask 15% 的图像区域（特征置零）
- 模型需要预测被 mask 区域的语义类别（用 Faster R-CNN 的 1600 类标签）
- 损失函数：交叉熵

**3. Masked Region Feature Regression (MRFR)**
- 随机 mask 15% 的图像区域
- 模型需要回归被 mask 区域的特征向量
- 损失函数：L2 距离

**4. Masked Region Classification (MRC)**
- 随机 mask 15% 的图像区域
- 模型需要预测被 mask 区域的物体类别
- 损失函数：KL 散度（与 Faster R-CNN 的检测置信度分布对齐）

**5. Image-Text Matching (ITM)**
- 给定图像-文本对，判断是否匹配
- 50% 正样本 + 50% 负样本（随机替换图像或文本）
- 使用 [CLS] token 的输出做二分类
- 损失函数：交叉熵

#### (c) 为什么有效

消融实验（Table 5）显示：
- 去掉所有视觉任务（只用 MLM + ITM）：VQA 从 72.4% 降到 70.8%（↓1.6%）
- 去掉 MRFR 和 MRC（只用 MLM + MRM + ITM）：VQA 从 72.4% 降到 71.5%（↓0.9%）
- 去掉 ITM：VQA 从 72.4% 降到 71.0%（↓1.4%）

每个任务都贡献了不同的信息：
- MLM 学习语言理解
- MRM（MRFR + MRC）学习视觉理解
- ITM 学习全局跨模态对齐

#### (d) 与 Related Work 的关系

- **BERT**（Devlin et al., 2019）的 MLM 启发了 LXMERT
- **VideoBERT**（Sun et al., 2019）用了类似的 masked 预测思路
- **ViLBERT**（Lu et al., 2019）用了 MLM + ITM，但没有视觉 masked 任务
- LXMERT 的创新在于：不仅 mask 文本，还 mask 图像区域，并且设计了两种不同的视觉预测任务（特征回归 + 类别分类）

#### (e) 如果去掉会怎样

从 Table 5 看，五个任务的效果是累加的：
- 只用 MLM + ITM：70.8%
- 加 MRM：71.5%（+0.7%）
- 加 MRFR + MRC + ITM：72.4%（+0.9% 相比只用 MLM+ITM）

## 六、公式详解

### 6.1 Cross-Attention 公式

**文本→图像注意力**：

$$ \text{Attention}(Q_l, K_v, V_v) = \text{softmax}\left(\frac{Q_l K_v^T}{\sqrt{d_k}}\right) V_v $$

其中：
- $Q_l = X_l W_q^l$：文本 query，$X_l \in \mathbb{R}^{20 \times 768}$，$W_q^l \in \mathbb{R}^{768 \times 64}$
- $K_v = X_v W_k^v$：图像 key，$X_v \in \mathbb{R}^{36 \times 768}$，$W_k^v \in \mathbb{R}^{768 \times 64}$
- $V_v = X_v W_v^v$：图像 value，$X_v \in \mathbb{R}^{36 \times 768}$，$W_v^v \in \mathbb{R}^{768 \times 64}$
- $d_k = 64$：每个注意力头的维度

**直觉理解**：
- 每个文本 token 会计算与所有 36 个图像区域的注意力权重
- 如果文本是"dog"，它应该对图像中狗的区域分配高注意力
- 输出形状：$\mathbb{R}^{20 \times 64}$，每个头输出 20 个 token 的加权图像特征

**图像→文本注意力** 同理，只是 query 来自图像，key/value 来自文本。

### 6.2 总损失函数

$$ \mathcal{L} = \mathcal{L}_{\text{MLM}} + \mathcal{L}_{\text{MRM}} + \mathcal{L}_{\text{ITM}} + \mathcal{L}_{\text{MRFR}} + \mathcal{L}_{\text{MRC}} $$

所有损失等权重相加（没有调权重超参数）。

**各损失的计算**：

1. **MLM 损失**：
$$ \mathcal{L}_{\text{MLM}} = -\sum_{i \in \mathcal{M}_l} \log P(w_i | X_l^{\text{masked}}, X_v) $$
其中 $\mathcal{M}_l$ 是被 mask 的文本 token 集合，$P(w_i)$ 是预测正确词的概率。

2. **MRM 损失**（包括 MRFR 和 MRC）：
$$ \mathcal{L}_{\text{MRFR}} = \sum_{j \in \mathcal{M}_v} ||f_j - \hat{f}_j||_2^2 $$
$$ \mathcal{L}_{\text{MRC}} = -\sum_{j \in \mathcal{M}_v} \sum_{c=1}^{C} p_{\text{det}}(c|o_j) \log p_{\text{model}}(c|o_j) $$
其中 $\mathcal{M}_v$ 是被 mask 的图像区域集合，$f_j$ 是原始 RoI 特征，$\hat{f}_j$ 是预测特征，$C=1600$ 是物体类别数，$p_{\text{det}}$ 是 Faster R-CNN 的检测置信度。

3. **ITM 损失**：
$$ \mathcal{L}_{\text{ITM}} = -\log P(\text{match} | X_l, X_v) - \log(1 - P(\text{match} | X_l', X_v)) $$
其中 $X_l'$ 是负样本的文本。

**具体数值例子**（Batch size = 256，序列长度 20，图像区域 36）：
- MLM：mask 15% = 256 × 20 × 0.15 = 768 个 token 需要预测
- MRM：mask 15% = 256 × 36 × 0.15 = 1382 个区域需要预测
- ITM：256 个正样本 + 256 个负样本

|------|-----------|--------|
| Bottom-Up (Anderson et al., 2018) | - | 65.3% |

**关键发现**：
- LXMERT 比单流模型 VisualBERT 高 1.6%

- 预训练数据量很重要（LXMERT 用了更多数据）

| 文本层数 | 图像层数 | 跨模态层数 | VQA |
|---------|---------|-----------|-----|

## 八、代码对照

LXMERT 的官方代码在 GitHub 上（https://github.com/airsplay/lxmert）。关键实现细节：

### 8.1 模型定义（lxmert.py）

```python
class LXMERT(nn.Module):
    def __init__(self):
        # 文本编码器：9 层 Transformer
        self.lang_encoder = nn.ModuleList([
            TransformerLayer() for _ in range(9)
        ])
        # 图像编码器：5 层 Transformer
        self.visn_encoder = nn.ModuleList([
            TransformerLayer() for _ in range(5)
        ])
        # 跨模态编码器：5 层，每层包含 cross-attention
        self.cross_encoder = nn.ModuleList([
            CrossModalityLayer() for _ in range(5)
        ])
```

### 8.2 跨模态层的实现

```python
class CrossModalityLayer(nn.Module):
    def forward(self, lang_feats, visn_feats):
        # 文本→图像 cross-attention
        lang_feats = self.cross_att_lang2visn(lang_feats, visn_feats)
        # 图像→文本 cross-attention  
        visn_feats = self.cross_att_visn2lang(visn_feats, lang_feats)
        # 双向 self-attention（拼接后）
        combined = torch.cat([lang_feats, visn_feats], dim=1)
        combined = self.bi_attention(combined)
        # 分离回两个分支
        lang_feats = combined[:, :20, :]
        visn_feats = combined[:, 20:, :]
        return lang_feats, visn_feats
```

**代码中的关键技巧**：
1. **LayerNorm 位置**：每个注意力层和 FFN 层后都有 LayerNorm
2. **Dropout**：所有注意力层使用 0.1 的 dropout
3. **GELU 激活**：FFN 中使用 GELU 而不是 ReLU
4. **权重初始化**：使用正态分布初始化，标准差 0.02

### 8.3 训练细节（代码中但论文未详细说明）

- 学习率：5e-5，使用 warmup（前 10% 步数线性增长到 5e-5，然后线性衰减）
- Batch size：256（分布式训练）
- 训练步数：200k 步（约 20 个 epoch）
- 优化器：Adam，β1=0.9, β2=0.999
- 梯度裁剪：最大梯度范数 1.0

## 九、位置

### 前驱工作
- **BERT** (Devlin et al., 2019)：提供了 Transformer 编码器和 MLM 预训练范式
- **Bottom-Up Attention** (Anderson et al., 2018)：使用 Faster R-CNN 提取图像区域特征
- **Visual Genome** (Krishna et al., 2017)：提供了大规模图像-文本对数据，Faster R-CNN 在此数据集上预训练

### 同期竞品
- **ViLBERT** (Lu et al., 2019)：双流架构，但缺少独立的图像编码器
- **VisualBERT** (Li et al., 2019)：单流架构，跨模态交互不够充分
- **Unicoder-VL** (Li et al., 2020)：类似的双流设计，但预训练任务不同

## 十、局限

### 局限一：对目标检测器的依赖
- **具体问题**：LXMERT 依赖 Faster R-CNN 提取图像区域特征，这引入了额外的计算开销和误差
- **影响场景**：对于小物体或罕见物体，检测器可能无法正确提取特征，导致下游任务性能下降
- **论文是否承认**：论文在实验部分提到使用预训练的 Faster R-CNN，但没有深入讨论其局限性
- **后续改进**：CLIP (Radford et al., 2021) 和 ViT (Dosovitskiy et al., 2021) 的出现让端到端训练成为可能，后续工作如 ALBEF 不再依赖目标检测器

### 局限二：序列长度受限
- **具体问题**：LXMERT 固定使用 20 个文本 token 和 36 个图像区域，对于长文本或密集场景不够灵活
- **影响场景**：对于需要细粒度描述的图像（如密集场景描述），36 个区域可能不够
- **论文是否承认**：论文没有明确讨论这个限制
- **后续改进**：后续工作（如 OFA）使用更大的序列长度和更灵活的输入处理

### 局限三：预训练数据规模
- **具体问题**：LXMERT 的预训练数据（COCO + Visual Genome + VQA）相对较小，约 18 万张图像
- **影响场景**：相比后来使用 4 亿图像-文本对的 CLIP，LXMERT 的泛化能力有限

- **后续改进**：后续工作使用更大规模的数据集，如 Conceptual Captions 和 LAION-400M

## 十一、小结

LXMERT 的核心贡献是提出了一个**三编码器架构**（文本编码器 + 图像编码器 + 跨模态编码器），通过**分治策略**分别建模模态内和跨模态交互。配合**五个预训练任务**（MLM、MRFR、MRC、MRM、ITM），模型在 VQA、GQA、NLVR2 等多个 V&L 任务上达到了当时的最佳性能。

**更广泛的影响**：LXMERT 确立了 V&L 预训练的基本范式——模态分离 + 多任务预训练。后续的 ALBEF、BLIP、VLMo 等工作都继承了这个思路。虽然现在已经被更大的模型（如 OFA、BEiT-3）超越，但 LXMERT 的设计思想仍然影响着多模态预训练领域的发展方向。
