## 一、先搞清楚"问题在哪里"

2018-2019 年，NLP 领域迎来了 BERT 的爆发。Devlin et al. 证明了大规模的 masked language model 预训练 + 下游 fine-tuning 这个范式，可以在 GLUE、SQuAD 等任务上取得显著的提升。自然而然地，社区开始问一个问题：**这个范式能不能扩展到视觉-语言（V&L）任务？**

当时的 V&L 领域，典型的做法是这样的：用 Faster R-CNN 从图像中检测出物体区域（RoI features），再用一个 LSTM/GRU 编码问题/描述，最后通过 attention 机制让文本去"看"图像的对应区域。代表作是 Anderson et al. (2018) 的 Bottom-Up and Top-Down Attention（BUTD）。这个架构在 VQA 上取得了当时最好的结果，但有一个根本性的局限：

**单模态 encoder 之间的交互太浅了。** BUTD 只在最后一层做了一次文本-图像的 attention 交互——语言侧的 GRU 独立编码完整的问题，图像侧的 Faster R-CNN 提取区域特征，最后用 attention 把两者连接起来。跨模态理解依赖于这一次"握手"，没有深层的模态间信息交换。

但更根本的问题是另一回事：**在 LXMERT 之前，没有一个统一的、可迁移的视觉-语言预训练模型。** NLP 有 BERT 做 backbone，CV 有 ResNet 做 backbone，但 V&L 领域——每个任务从头训练自己的模型，每个架构只在特定 benchmark 上有效，缺少通用的预训练+微调框架。

所以 2019 年夏天的竞争格局是这样的：**把 BERT 式的预训练范式搬到视觉-语言多模态场景**——用海量的图文对（image-caption pairs / image-QA pairs）训练一个可以理解两模态之间关系的模型，然后在下游任务上微调。ViLBERT（Lu et al.）、VisualBERT（Li et al.）和 LXMERT（Tan & Bansal）几乎同时（arXiv 时间都在 2019 年 8 月）提交了各自的工作，成了这个赛道的第一批探路者。

## 二、旧路线的真正问题

要理解 LXMERT 的设计，得先搞清楚"为什么 BUTD 不够好"——不是它做错了什么，而是它的架构有结构性限制。

### 2.1 交互深度：一次 attention 不够

BUTD 的架构可以简化为：

```text
图像 → Faster R-CNN → 物体 RoI features
   ↓
   ↓  (attention 只在最后一层)
   ↑
文本 → GRU → 句子 encoding
   ↓
答案预测
```

语言 encoder（GRU）从头到尾独立编码句子，从来没有在编码过程中"看"过图像。图像特征也是独立的，从来没有在编码中"看"过文本。直到最后做 prediction 的时候，才通过 attention 把两者对齐。

问题在于：**人类在回答"图片里有几个气球？"这个问题时，读到"几个"这个词的时候就已经在看图像了。** 但 BUTD 的 GRU 在读到"几个"时，对图像一无所知——它在整个编码过程中都没有获取过视觉信息。交叉模态的交互被压缩到了最后一层，这是不够的。

### 2.2 视觉特征：位置在哪里？

BUTD 使用 Faster R-CNN 的 RoI features（2048 维）并编码了 bbox 空间特征（归一化 x₁, y₁, x₂, y₂, area），但跨模态交互限于简单的 attention 融合，缺少深层跨模态编码。"左边的狗"和"右边的猫"的区别完全靠位置区分。忽略位置信息，模型就无法理解空间关系。

你可以想象一个简单的场景：问"左边的物体是什么？"如果模型不知道哪个物体在左边，它就无法回答这个问题。

### 2.3 物体关系：独立的物体还是相关联的场景？

BUTD 把每个检测到的物体当成独立的特征向量，没有建模物体之间的关系。但真实场景中，物体之间有丰富的交互——"人拿着手机"、"猫坐在椅子下面"、"杯子在桌子上"。这些关系对 V&L 任务至关重要，但 BUTD 的架构无法显式建模它们。

### 2.4 预训练的缺失：每项任务从零开始

这可能是最大的问题。在 NLP 领域，BERT 的预训练使得下游任务只需要很少的数据就能达到很好的效果。但在 V&L 领域，每个模型都是从头训练的——没有大范围的无监督/自监督预训练来帮助模型建立通用的视觉-语言对应理解。

综合这四个问题，一个更好的 V&L 模型应该：
1. **深层跨模态交互** — 语言编码过程中要能"看到"图像，图像编码过程中要能"看到"文本
2. **位置信息嵌入** — 物体的空间位置不能丢弃
3. **物体关系建模** — 物体之间需要交互，不只是独立特征
4. **大规模多任务预训练** — 用海量图文对预训练，学习通用的跨模态表征

LXMERT 就是针对这四个问题设计的。

## 三、这篇论文的核心思路

**核心思路：设计一个三编码器架构——语言 encoder + 物体关系 encoder + 跨模态 encoder，然后在一个大规模（9.18M 图文对）数据集上用五种预训练任务联合训练，一次性解决跨模态交互深度不够、位置信息缺失、物体关系缺失和预训练缺失四个问题。**

具体来说，架构分三层：

```text
[语言 encoder]             [物体关系 encoder]
  (9层 self-attention)       (5层 self-attention)
       ↓                           ↓
            [跨模态 encoder]
       (5层 cross-attention + self-attention)
       ↓              ↓
  语言输出          视觉输出        跨模态输出（[CLS]）
```

这五个预训练任务形成了几个维度的覆盖：

```text
语言任务:  masked cross-modality LM       ← 从视觉推断被遮的词
视觉任务:  RoI-feature regression +       ← 从文本推断被遮的物体
            detected-label classification
跨模态任务: cross-modality matching +      ← 图文是否匹配 + 回答图像问题
            image question answering
```

这条技术路线的谱系位置是：

```text
BUTD（LSTM + attention，2018）——单层交互
   ↓
BERT（语言预训练，2019）——NLP 灵感来源
   ↓
┌──────────────────────────────────────┐
│ ViLBERT（双流 co-attention, 2019.08）│——同期竞品
│ VisualBERT（单流拼接, 2019.08）      │——同期竞品
│ LXMERT（三编码器, 2019.08）           │——本篇
└──────────────────────────────────────┘
   ↓
UNITER（单流统一预训练, 2020）
   ↓
OSCAR / VILLA / ERNIE-ViL（2020-2021）
```

LXMERT 和同期工作 ViLBERT、VisualBERT 的核心区别：
- ViLBERT 是**双流**架构——两个独立的 Transformer（语言 + 视觉），只在特定层通过 co-attention 交互
- VisualBERT 是**单流**架构——图像 token + 文本 token 拼接到一个 Transformer 里
- LXMERT 是**三编码器**架构——比 ViLBERT 多了物体关系 encoder（专门建模物体间关系），比 VisualBERT 有更精细的跨模态交互控制

## 四、整体结构与前向流程

### 4.1 整体架构

LXMERT 可以理解为一个三阶段的 pipeline：

```text
输入:  图像                          + 句子/问题
        ↓                                ↓
[输入嵌入]                          [输入嵌入]
   Faster R-CNN 检测物体              WordPiece tokenize
   RoI 特征 (2048-d) + 位置编码       Word embedding + 位置编码
   每个图像取 36 个物体 (固定)         句子长度 n
        ↓                                ↓
[Object-Relationship Encoder]       [Language Encoder]
   5 层 Transformer                    9 层 Transformer
   只做 self-attention (视觉侧)        只做 self-attention (语言侧)
        ↓                                ↓
┌───────────────────────────────────────────────┐
│           [Cross-Modality Encoder]              │
│            5 层 Transformer                     │
│    每层:  Cross-Attention (双向)                  │
│         → Self-Attention (语言侧+视觉侧)          │
│         → Feed-Forward                          │
└───────────────────────────────────────────────┘
        ↓           ↓               ↓
   语言输出     视觉输出        跨模态输出 ([CLS])
   (向量序列)   (向量序列)        (单个向量)
```

### 4.2 前向过程逐步跟踪

**Step 1 — 输入嵌入**

*语言侧*：句子通过 WordPiece tokenizer 分词，每个词 wi 通过 word embedding 得到向量，位置通过 index embedding 编码，两者相加后 LayerNorm。

```python
# 语言嵌入
w_hat_i = WordEmbed(w_i)       # 词嵌入
u_hat_i = IdxEmbed(i)          # 位置嵌入
h_i = LayerNorm(w_hat_i + u_hat_i)  # LayerNorm 后加和
```

*视觉侧*：首先用 Faster R-CNN（在 Visual Genome 上预训练，冻结参数作为特征提取器）检测图像中的物体。每张图像取固定 36 个物体（而不是像 Anderson et al. 那样用可变数量），目的是最大化 batch 计算的利用率，减少 padding 开销。

每个物体 oj 有两个特征：
- RoI 特征 fj：2048 维，来自 Faster R-CNN 的区域特征
- 位置特征 pj：bounding box 坐标（归一化后形成的位置向量）

**关键设计：位置感知嵌入**

LXMERT 没有像 BUTD 那样直接使用 RoI 特征，而是显式地把 RoI 特征和位置特征融合：

```python
f_hat_j = LayerNorm(W_f @ f_j + b_f)   # RoI 特征投影 + LayerNorm
p_hat_j = LayerNorm(W_p @ p_j + b_p)   # 位置特征投影 + LayerNorm
v_j = (f_hat_j + p_hat_j) / 2          # 平均融合
```

这里有两个重要的设计选择：
1. **LayerNorm 在融合之前**分别对两种特征做归一化，再求和平均。目的是平衡两种不同来源特征的"能量"，避免一方主导另一方。
2. **平均融合**而不是直接拼接。这是一个更轻量的特征融合方式。

**Step 2 — 单模态 Encoder**

语言和视觉特征分别进入各自的单模态 encoder。

**Language Encoder（9 层）**：标准的 Transformer encoder，每层包含 self-attention sub-layer + feed-forward sub-layer（2 个全连接层），加残差连接和 LayerNorm。和 BERT 的结构完全一致。

为什么语言 encoder 比视觉 encoder 层数多（9 vs 5）？论文的解释是："More layers are used in the language encoder to balance the visual features extracted from 101-layer Faster R-CNN."——Faster R-CNN 本身已经是非常深的网络（101 层的 ResNet backbone），它的输出已经是高度抽象的视觉特征。语言侧如果层数太少，表征能力会不平衡。

**Object-Relationship Encoder（5 层）**：这也是一个标准的 Transformer encoder，但作用于视觉特征。它的作用是**建模物体之间的关系**——"人生在椅子上"、"杯子在桌子上"这类关系不通过外部知识，而是通过自注意力让物体之间互相"看到"。

**Step 3 — 跨模态 Encoder**

这是 LXMERT 最核心的部分。跨模态 encoder 有 5 层，每层结构如下：

```text
输入: 语言特征 {h_i^{k-1}} + 视觉特征 {v_j^{k-1}}
        │
        ▼
  [Bi-directional Cross-Attention]
        ├── Language→Vision: h_i^k_hat = CrossAtt_L→R(h_i^{k-1}, {v_j^{k-1}})
        └── Vision→Language: v_j^k_hat = CrossAtt_R→L(v_j^{k-1}, {h_i^{k-1}})
        │
        ▼
  [Self-Attention]（语言侧和视觉侧分别做）
        ├── Language: h_i_tilde = SelfAtt_L→L(h_i^k_hat, {h_i^k_hat})
        └── Vision:   v_j_tilde = SelfAtt_R→R(v_j^k_hat, {v_j^k_hat})
        │
        ▼
  [Feed-Forward]
        ├── Language FF → {h_i^k}
        └── Vision FF → {v_j^k}
        │
        ▼
  输出: 语言特征 {h_i^k} + 视觉特征 {v_j^k}
```

**关键：双向 Cross-Attention 的设计**

在每一层，cross-attention 是双向的：
- **Language→Vision**：每个语言 token 以视觉 token 作为 context，去"看"图像中哪些区域和当前词相关
- **Vision→Language**：每个视觉 token 以语言 token 作为 context，去"看"文本中哪些词和当前物体相关

这和 ViLBERT 的 co-attention 非常相似。区别在于 ViLBERT 是双流架构（语言和视觉分别有独立的 Transformer 层，只在特定层通过 co-attention 交互），而 LXMERT 把 cross-attention 放到了同一个层内，然后紧接着做 self-attention 和 FF。

为什么 cross-attention 之后还要做 self-attention？因为 cross-attention 让语言和视觉互相"看了对方一眼"，但各自模态内部的 token 之间还需要再整合这些新信息。self-attention 做的就是这件事——让语言侧的 token 之间互相交换"我看到了什么"，视觉侧同理。

**Step 4 — 输出表示**

有三个输出：
- **语言输出**：跨模态 encoder 最后输出的语言特征序列 {h_i}
- **视觉输出**：跨模态 encoder 最后输出的视觉特征序列 {v_j}
- **跨模态输出**：语言序列中 [CLS] token 对应的特征向量。和 BERT 一样，[CLS] 被用来表示整个输入对的联合表征

### 4.3 配置参数

| 参数 | 值 | 说明 |
|------|-----|------|
| NL | 9 | 语言 encoder 层数 |
| NR | 5 | 物体关系 encoder 层数 |
| NX | 5 | 跨模态 encoder 层数 |
| Hidden size | 768 | 和 BERT-BASE 一致 |
| 等效总层数 | 12 | (9+5)/2+5=12，对齐 BERT-BASE |
| Objects/图像 | 36 | 固定数量，避免 padding |
| Head 数 | 12 | multi-head attention |

| 预训练 epochs | 20 | 含 10 epoch w/o QA + 10 epoch w/ QA |

| 预训练硬件 | 4× Titan Xp | 10 天 |

## 五、五个预训练任务详解

LXMERT 共使用了五个预训练任务，分为三类。

### 5.1 语言任务：Masked Cross-Modality LM

**设置：** 以 0.15 的概率随机 mask 句子中的词，要求模型预测被遮的词。

**和 BERT 的区别：**
- BERT 只能从语言上下文中推断被遮的词（"I went to the [MASK] to buy groceries." → 从"went"、"buy"、"groceries"推断是"store"）
- LXMERT 除了语言上下文，还能从**视觉信息**推断——如果图像显示某人手持胡萝卜，那么被遮的词很可能是"carrot"

**论文里的例子：** "The man is holding a [MASK]." 从语言上下文很难确定是胡萝卜还是手机，但从图像内容可以明确判断。

**设计意图：** 强制模型建立**从视觉到语言**的对齐关系——语言中的某个词，在视觉上对应的区域是什么。

### 5.2 视觉任务：Masked Object Prediction

**设置：** 以 0.15 的概率用零向量随机 mask 物体的 RoI 特征，要求模型预测被遮物体的属性。分为两个子任务：

**子任务 1 — RoI-Feature Regression（回归）：** 用 L2 loss 回归被遮物体的原始 RoI 特征。这个任务让模型学习从可见物体或语言线索推断被遮物体的视觉特征。

**子任务 2 — Detected-Label Classification（分类）：** 用 cross-entropy loss 预测被遮物体的检测标签（Faster R-CNN 输出的类别标签）。虽然检测标签有噪声（不同数据集的标签空间不一致），但实验表明这个分类任务对预训练有贡献。

**设计意图：** 建立**从语言到视觉**的对齐关系——语言描述中的内容，对应视觉上的哪些物体。

### 5.3 跨模态任务

**任务 1 — Cross-Modality Matching：** 以 0.5 的概率用一个不匹配的句子替换（从另一张图像的句子中随机取），要求模型预测图像-句子是否匹配。类似于 BERT 的 Next Sentence Prediction，但这里是跨模态版本的匹配判断。

**任务 2 — Image Question Answering：** 预训练数据中大约 1/3 的句子是对图像的问题。当图像-问题匹配时（未被替换），要求模型预测问题的答案。预训练中构建了一个包含 9500 个答案候选的共享答案表，覆盖了三个图像 QA 数据集中约 90% 的问题。

**为什么需要 QA 预训练任务？**

论文通过消融实验（Table 4）证明了 QA 任务的重要性。没有 QA 预训练时，NLVR2 的准确率从 74.9% 掉到 72.4%。QA 任务让模型学会对图像中的内容做更细粒度的理解——"回答关于图像的问题"这个能力，比单纯的 mask-and-predict 更接近下游任务的需求。

### 5.4 预训练数据

LXMERT 聚合了五个图文数据集：

| 数据来源 | 图像来源 | 图文对数量 |
|---------|---------|-----------|
| MS COCO Caption | 不共享 | 0.75M |
| MS COCO ∩ VG | 共享 | 4.30M |
| VG - MS COCO | Visual Genome 独有 | 4.13M |
| **合计** | **180K 张不同图像** | **9.18M** |

具体包含的源数据集：
- COCO Caption（描述）
- Visual Genome Caption（描述）
- VQA v2.0（问答）
- GQA（问答）
- VG-QA（问答）

总语料约为 10 亿词和 650 万个图像物体。所有 train/dev 数据被使用，但 test 数据被排除以确保公平评估。

## 六、核心公式与数学直觉

### 6.1 位置感知嵌入

$$v_j = \frac{1}{2}\left(\text{LayerNorm}(W_f f_j + b_f) + \text{LayerNorm}(W_p p_j + b_p)\right)$$

**直觉：** RoI 特征 fj（2048d）编码了"这是什么物体"的信息；位置特征 pj 编码了物体在图像中的边界框坐标。两者单独 LayerNorm 后取平均，确保视觉内容和空间位置被同等对待。

### 6.2 双向 Cross-Attention

$$\hat{h}^k_i = \text{CrossAtt}_{L \rightarrow R}\left(h^{k-1}_i, \{v^{k-1}_1, \ldots, v^{k-1}_m\}\right)$$
$$\hat{v}^k_j = \text{CrossAtt}_{R \rightarrow L}\left(v^{k-1}_j, \{h^{k-1}_1, \ldots, h^{k-1}_n\}\right)$$

**直觉：** 在每一层，语言 token 去"看"所有视觉 token，视觉 token 也去"看"所有语言 token。这是一个双向的、逐层的信息交换。关键点：cross-attention 是跨模态的"检索"——每个语言 token 从视觉池中检索相关上下文，反之亦然。

### 6.3 跨模态 Layer 内的 Self-Attention

$$\tilde{h}^k_i = \text{SelfAtt}_{L \rightarrow L}\left(\hat{h}^k_i, \{\hat{h}^k_1, \ldots, \hat{h}^k_n\}\right)$$
$$\tilde{v}^k_j = \text{SelfAtt}_{R \rightarrow R}\left(\hat{v}^k_j, \{\hat{v}^k_1, \ldots, \hat{v}^k_m\}\right)$$

**直觉：** cross-attention 之后，每个模态的 token 获取了跨模态信息。self-attention 的作用是让同模态的 token 之间共享这些新信息——"我看到了什么，你也看看。"

### 6.4 总体预训练损失

$$\mathcal{L} = \mathcal{L}_{\text{MLM}} + \mathcal{L}_{\text{FeatReg}} + \mathcal{L}_{\text{LabelCls}} + \mathcal{L}_{\text{Match}} + \mathcal{L}_{\text{QA}}$$

所有损失使用相等的权重。注意 QA 预训练任务只在最后的 10 个 epoch 中使用——因为论文发现 QA 任务收敛更快，且需要更小的学习率。

### 7.1 Figure 1 — LXMERT 模型架构

这张图展示了 LXMERT 的完整架构。最左侧是输入嵌入层（语言 WordPiece + 位置嵌入，视觉 RoI + 位置编码），中间是三个 encoder（语言 encoder 在底部、物体关系 encoder 在顶部、跨模态 encoder 在中间），最右侧是三个输出。

- **左侧虚线框**（单模态 layer）：Self-Attention → FF，标准 Transformer block
- **右侧虚线框**（跨模态 layer）：Cross-Attention（双向）→ Self-Attention（语言侧+视觉侧）→ FF

图的中间展示了跨模态的 cross-modality matching 和 image QA 任务。特别值得注意的是，QA 任务只在图像和问题匹配时进行（即 cross-modality matching 分类器预测为"匹配"时）。

### 7.4 Table 2 — 主要结果

LXMERT 在三个数据集上的结果：

**VQA（test-standard）：** 72.5%（超过当时 SOTA 的 70.63%，+2.1%）
- Binary 问题：88.2%（SOTA 85.8%）
- Number 问题：54.2%（SOTA 53.7%）
- Other 问题：63.1%（SOTA 60.7%）

**GQA（test-standard）：** 60.3%（超过 SOTA 的 57.1%，+3.2%）
- Binary 问题：77.8%（SOTA 76.0%）
- Open 问题：45.0%（SOTA 40.4%）

**NLVR2（Test-U）：** 76.2%（超过 SOTA 的 53.5%，+22.7% absolute）
- Consistency：42.1%（SOTA 12.0%）

NLVR2 上的巨大提升（22% absolute accuracy，48% relative error reduction）是最有说服力的结果——因为 LXMERT 的预训练数据全部来自 COCO / Visual Genome，而 NLVR2 的图像和描述完全不在此范围内。这说明 LXMERT 学到了**可迁移的跨模态理解**，而不是简单记忆了特定数据集的特征。

## 八、消融与分析

### 8.1 BERT 对比实验（Table 3）

论文做了系统的对比实验，试图把 BERT 用在 V&L 任务上：

1. **BERT+BUTD**：用 BERT 替换 BUTD 的 GRU。结果和 GRU 版本差不多（VQA 63.1% vs 62.8%）。说明光换语言 encoder 不够。

2. **BERT+CrossAtt**：在 BERT 基础上加 LXMERT 的 cross-attention 层。结果随层数增加提升（1 层 → 5 层：64.6% → 66.5%），但最好的 66.5% 仍然远低于 LXMERT 预训练后的 69.9%。**差距 3.4%**。

3. **BERT+LXMERT 结构 + 从头训练**：用 LXMERT 架构但不做跨模态预训练。结果只有 65.1%（VQA），远低于预训练后的 69.9%。

4. **BERT+LXMERT + BERT 参数初始化**：把 BERT 的顶层 9 层参数加载到 LXMERT 的语言 encoder。预训练后效果反而不如从头训练（VQA 68.8% vs 69.9%）——论文给出的解释是 BERT 已经在纯语言 MLM 上预训练过了，它在预训练中倾向于依赖语言信息而不学习跨模态连接。

**结论：** 光有好的架构不够，光有 BERT 的预训练参数也不够。必须进行专门的跨模态预训练。

### 8.2 QA 预训练的重要性（Table 4）

| 设置 | VQA | GQA | NLVR2 |
|------|-----|-----|-------|
| P20（无 QA）+ DA | 68.0 | 58.1 | — |
| P20（无 QA） | 68.9 | 58.2 | 72.4 |
| P10+QA10 + DA | 69.1 | 59.2 | — |
| **P10+QA10（完整）** | **69.9** | **60.0** | **74.9** |

两个发现：
1. QA 预训练在所有数据集上带来提升，NLVR2 上 +2.5%
2. Data Augmentation（DA）在微调阶段反而降低效果——说明在预训练阶段引入 QA 数据比在微调阶段做数据增强更有效

### 8.3 视觉预训练任务的贡献（Table 5）

| 设置 | VQA | GQA | NLVR2 |
|------|-----|-----|-------|
| 无视觉任务 | 66.3 | 57.1 | 50.9 |
| 仅 FeatReg | 69.2 | 59.5 | 72.9 |
| 仅 LabelCls | 69.5 | 59.3 | 73.5 |
| **FeatReg + LabelCls** | **69.9** | **60.0** | **74.9** |

没有视觉预训练任务时，NLVR2 的效果从 74.9% 暴跌到 50.9%（接近随机）。这说明**跨模态理解严重依赖于视觉侧的任务**——只做语言 masking 不会让模型真正理解图像的内容。特征回归和标签分类联合使用效果最好。

## 九、复杂度与效率

### 9.1 模型参数量

LXMERT 的参数量与 BERT-BASE 相当（hidden size 768，12 head，等效 12 层）。具体来说：

- 语言 encoder（9 层）：~9/12 × BERT 参数量
- 物体关系 encoder（5 层）：~5/12 × BERT 参数量
- 跨模态 encoder（5 层）：含额外的 cross-attention 参数，比标准 Transformer 层多 2 套 QKV 投影

### 9.2 训练代价

- 预训练：4× Titan Xp GPU，10 天
- 优化步数：约 670K 步（20 epochs，batch size 256）
- 相比之下：BERT 在相同 hidden size 下用约 1M 步、16 TPUs

### 9.3 固定物体数量的设计选择

LXMERT 每张图像固定取 36 个物体检测框（而非可变数量）。这个选择的优势是避免了 batch 内的 padding 开销，最大化 GPU 利用。代价是可能丢失一些对特定任务重要的物体（如果图像中物体多于 36 个）。但在实践中，36 个框对大多数 V&L 任务已经足够。


LXMERT 处于 D1-1 "早期跨模态 encoder 范式"的节点，和 ViLBERT、VisualBERT 同期。

### 10.2 前驱工作

**BUTD（Anderson et al., 2018）：** 提供了 object-level 的图像表示范式——用 Faster R-CNN 检测物体并提取 RoI 特征。LXMERT 直接继承了这一点，但加了位置编码和物体关系 encoder。

**BERT（Devlin et al., 2019）：** 提供了预训练 + 微调的范式。LXMERT 把 MLM 扩展到跨模态版本，把 NSP 扩展到跨模态匹配。

**Transformer（Vaswani et al., 2017）：** 提供了 self-attention 和 cross-attention 的基础架构。

### 10.3 同期竞品

**ViLBERT（Lu et al., arXiv 2019.08）：** 双流架构——两个独立的 Transformer（12 层语言 + 6 层视觉），在 6、9、12 层之间插入 co-attention。ViLBERT 和 LXMERT 都是在探索"如何让 Transformer 做跨模态交互"，但走了不同的路线：
- ViLBERT 是两路并行 + 中间交叉
- LXMERT 是单路独立编码 + 之后做深层交叉

**VisualBERT（Li et al., arXiv 2019.08）：** 最简洁的方案——直接把视觉 token 和文本 token 拼接到一个 BERT 里。缺点是没有显式的跨模态 attention 控制。

LXMERT 在提交时（EMNLP 2019 截止日 2019.05.21）是最早的一批工作之一。论文第 6 节对比了后续的 ViLBERT 和 VisualBERT，LXMERT 在重叠的 benchmark 上领先：VQA 高 1.5%，NLVR2 高 9%。

### 10.4 后续影响

**UNITER（Chen et al., 2020）：** 统一单流架构，但引入了更精细的条件 masking 策略（conditional masking），成为后续工作的新基线。

**OSCAR（Li et al., 2020）：** 引入检测标签（detection tags）作为"锚点"，把单流的性能推到新的高度。

**ERNIE-ViL（Yu et al., 2020）：** 在预训练中引入场景图（scene graph），显式建模物体-属性-关系三元组。

**整体影响：**
- 引用量超过 2,300 次（截至 2026 年）
- 奠定了"跨模态预训练"这个范式的早期基础
- 其五任务预训练设计（MLM + MRM + Matching + QA）成为后续工作的标准模板
- 和 ViLBERT、VisualBERT 一起，共同定义了 2019-2020 年 V&L 预训练的研究格局

## 十二、范式意义

**LXMERT 的范式意义：它是 2019 年"视觉-语言预训练三巨头"之一，共同开创了跨模态预训练这个方向。**

具体来说，LXMERT 做了三件在范式层面有重要意义的事：

### 12.1 证明了 BERT 式预训练在 V&L 上有效

在 LXMERT 之前，社区不确定把 BERT 的 MLM 范式搬到多模态场景会不会有效。LXMERT 用 VQA +2.1%、NLVR2 +22% 的明确证据证明了：**在大量图文对上进行自监督/弱监督预训练，可以学到可迁移的跨模态理解**。这个结论今天看来像是常识，但在 2019 年 5 月（截稿时）不是。

### 12.2 建立了"三编码器 + 多任务预训练"的设计范式

这个范式包含三个设计决策，被后续大量工作沿用：
1. **物体级视觉特征**（Faster R-CNN feature）作为图像输入，而非 CNN feature map
2. **显式的位置编码**，融合到视觉特征中
3. **多任务联合预训练**（MLM + MRM + Matching + QA），覆盖语言理解、视觉理解和跨模态对齐

### 12.3 跨模态预训练 + 零数据迁移的验证

NLVR2 的实验是最有说服力的——预训练完全不使用 NLVR2 的任何数据，微调后仍能取得 22% 的提升。这验证了跨模态预训练学到了**通用的、可迁移的**视觉-语言对齐能力，而不是对预训练数据集的过拟合。

### 12.4 分类：foundation node（在 V&L 子领域内）

在技术树的 V&L 子分支中，LXMERT 是一个 foundation node。它和 ViLBERT、VisualBERT 一起建立了"跨模态预训练"这个子领域，后来被 UNITER、OSCAR、ERNIE-ViL 等论文继承和超越。

## 十三、局限性与后续方向

### 13.1 三编码器的复杂度

LXMERT 有三组独立的 encoder 参数（语言 + 视觉 + 跨模态），总参数量大于等效 12 层的 BERT。后续工作证明，更简单的单流架构（UNITER、OSCAR）可以达到甚至超过 LXMERT 的效果。这意味着"复杂架构"不是必要的——更好的预训练策略可能比更复杂的架构更重要。

### 13.2 Faster R-CNN 的瓶颈

LXMERT 依赖 Faster R-CNN 提取物体特征，且检测器在预训练和微调中都被冻结（不参与训练）。这带来了两个问题：
1. **检测器本身的 bias**：Visual Genome 上训练的 Faster R-CNN 只能检测 1,600 类物体，很多罕见的视觉概念被过滤掉了
2. **不可端到端训练**：视觉特征提取器无法根据下游任务的需求做调整

### 13.3 预训练数据局限

9.18M 图文对比后来的标准（如 CLIP 的 400M 图文对）小了两个数量级。且所有数据来自 COCO 和 Visual Genome，同属自然图像领域，在域外（如医学图像、卫星图像、手绘图）上的效果未知。

### 13.4 下游任务局限

LXMERT 只在三种任务上评估（VQA、GQA、NLVR2），任务类型相对单一。没有在指代理解（Referring Expression）、图像描述（Image Captioning）、图像检索（Image Retrieval）等任务上评估。后来 UNITER 在更多任务上做了统一评估，暴露了 LXMERT 的覆盖盲区。

### 13.5 结论窗口期短暂

作为 2019 年的工作，LXMERT 的领先窗口非常短暂。不到一年后，单流架构（UNITER, 2020.02）在几乎所有任务上超过了 LXMERT。再过一年，对比学习范式（CLIP, 2021.01）彻底改变了 V&L 领域的预训练方式——从"encoder 式对齐"转向了"对齐式预训练"，不再需要物体检测器，直接用图像-文本对比学习。

### 13.6 与 ViLBERT 的对比：事后看更有趣

从 2026 年回头看，LXMERT 和 ViLBERT 的竞争很有意思。LXMERT 在当时取得了一点点领先（VQA 高 1.5%），但 ViLBERT 的**双流架构后来被更广泛地沿用**——在 ViLBERT 的架构上，语言流和视觉流可以分别独立升级（比如把语言流换成更强的 LLM），而 LXMERT 的三流设计在这个方向上不够灵活。这也从侧面说明：跨模态架构的分流设计（语言和视觉保持相对独立）在后来的工作中更受欢迎。

## 十四、小结

LXMERT 是 2019 年 V&L 跨模态预训练的开拓者之一。它和 ViLBERT、VisualBERT 几乎同时（2019 年 8 月）独立提出了同一个核心想法：**用 BERT 式的遮蔽预测 + 匹配任务 + QA 任务，在海量图文对上预训练一个 Transformer 模型，学到通用的跨模态表征，然后再微调到下游任务。**

LXMERT 的独特贡献在于它的**三编码器架构**：一个语言 encoder、一个物体关系 encoder（视觉侧 self-attention）、一个跨模态 encoder（双向 cross-attention）。和 ViLBERT 的双流架构相比，它多了物体关系编码这一层设计；和 VisualBERT 的单流架构相比，它的跨模态交互更有结构性。

从历史角度看，LXMERT 的最大价值可能不是它的架构——因为后来单流架构证明了更简单的设计也能做得更好——而是它**用实验明确验证了"跨模态预训练有效"这个命题**。NLVR2 上从 54% 到 76% 的提升（22% absolute）是一个不可忽视的信号：大规模多任务预训练在 V&L 领域同样有效，而且不需要目标域的图像。

V&L 领域的后续发展（UNITER、OSCAR、CLIP、BLIP）都站在了这个起点上。LXMERT 为后来的工作铺好了路——证明了方向对了，剩下的就是优化路径。
