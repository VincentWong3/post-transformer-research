# LeViT: a Vision Transformer in ConvNet's Clothing for Faster Inference（2021）

**论文：** [arXiv](https://arxiv.org/abs/2104.01136) · 作者：Ben Graham, Alaaeldin El-Nouby, Hugo Touvron, Pierre Stock, Armand Joulin, Hervé Jégou, Matthijs Douze · 年份：2021

## 一、先搞清楚坑在哪

Transformer 架构在 NLP 领域的成功（Vaswani et al., 2017）催生了将其应用于视觉任务的尝试。ViT（Dosovitskiy et al., 2021）首次证明，纯 Transformer 架构在图像分类任务上可以达到甚至超越 CNN 的精度。然而，ViT 有一个致命伤：它太重了。

ViT 的问题具体体现在：

1. **高分辨率下计算爆炸**：ViT 将图像切分为 16×16 的 patch，对于 224×224 的输入，序列长度仅为 196（14×14），这还能接受。但一旦输入分辨率提升到 384×384，序列长度变成 576（24×24），自注意力 $O(N^2)$ 的复杂度立刻让推理速度骤降。

2. **缺乏视觉先验**：ViT 需要在大规模数据集（如 ImageNet-21k 或 JFT-300M）上预训练才能达到与 CNN 相当的性能。这是因为 Transformer 没有卷积的局部性和平移等变性这两个归纳偏置，导致数据效率低。

3. **推理速度慢**：ViT 的 FLOPs 虽然可能低于某些 CNN，但实际硬件上的推理速度却更慢。这是因为 Transformer 中的密集矩阵乘法（特别是多头注意力中的 QKV 投影和输出投影）在现代 GPU 上虽然高效，但 patch 嵌入层的卷积操作和自注意力的 softmax 计算却成为瓶颈。

这就引出了 LeViT 的核心问题：**能不能设计一个既保留 Transformer 的表达能力，又继承 CNN 的推理效率的混合架构？**

## 二、CNN 的真正问题

CNN 经过数十年的发展，已经有非常成熟的优化方案。MobileNetV2（Sandler et al., 2018）通过深度可分离卷积大幅降低了计算量，EfficientNet（Tan & Le, 2019）通过神经架构搜索找到了精度-效率的 Pareto 最优解。

但 CNN 也有自己的天花板：

1. **感受野受限**：标准卷积的核大小通常是 3×3，要获得全局感受野需要堆叠大量层。虽然可以通过空洞卷积（Dilated Convolution）或大核卷积（如 ConvNeXt 的 7×7）来扩大感受野，但这通常以增加参数量为代价。

2. **动态权重缺失**：卷积核是固定的（训练完成后），对所有输入使用相同的权重。而 Transformer 的自注意力是内容相关的（content-dependent），每个位置的注意力权重根据输入动态计算。这种动态性被认为是对表达能力的重要提升。

3. **通道交互效率低**：标准卷积同时处理空间和通道维度，这在计算上是高效的（单个矩阵乘法），但通道间的信息混合需要通过 1×1 卷积（Pointwise Convolution）来完成，这增加了额外的计算。

有意思的是，CNN 的这些"问题"恰恰是 Transformer 的优势所在，而 Transformer 的问题又恰好是 CNN 的优势。LeViT 的直觉很简单：**把两者的优点结合起来，而不是在两者之间二选一。**

## 三、LeViT 的核心思路

LeViT 的核心思路可以用一句话概括：**用 CNN 的架构（多尺度特征金字塔 + 下采样）来承载 Transformer 的计算（自注意力 + MLP），从而在推理速度上接近 CNN，在表达力上接近 Transformer。**

乍看之下，这似乎只是简单的"CNN + Transformer"混合，但 LeViT 的设计远比这精巧。它不是在 CNN 骨架后面接一个 Transformer 头（如 ViT 的做法），而是从网络的最底层就开始融合两种架构。

高层次的架构概览如下：

1. **输入处理**：使用一个类似 CNN 的 stem 模块（多个卷积层的堆叠）将输入图像快速下采样到 1/16 分辨率，同时将通道数扩展到合理的维度（如 128 或 256）。

2. **主体网络**：由多个 LeViT 块堆叠而成，每个块包含一个注意力子层和一个 MLP 子层。与 ViT 不同的是，LeViT 在注意力子层中使用了**下采样注意力**（downsampling attention），即在注意力计算的同时降低空间分辨率。

3. **分类头**：与 ViT 类似，使用一个额外的 class token 或全局平均池化后接一个线性分类层。

这里有个关键：LeViT 没有像 ViT 那样保持固定的序列长度，而是像 CNN 一样逐步降低空间分辨率、增加通道数。这种设计直接借鉴了 CNN 的多尺度特征金字塔思想，但用 Transformer 的自注意力来替代卷积进行特征提取。

## 四、网络架构详解

### 4.1 整体架构图

```
输入图像 (224×224×3)
     │
     ▼
┌─────────────────────────────────────┐
│            Stem 模块                  │
│   Conv 3×3 (stride 2) → BatchNorm   │
│   Conv 3×3 (stride 2) → BatchNorm   │
│   Conv 3×3 (stride 2) → BatchNorm   │
│   输出: 28×28×128                    │
└─────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────┐
│       LeViT 阶段 1 (2个块)           │
│   注意力头数: 4                       │
│   通道数: 128 → 128                  │
│   分辨率: 28×28                      │
└─────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────┐
│       下采样 + LeViT 阶段 2 (3个块)   │
│   注意力头数: 6                       │
│   通道数: 128 → 256                  │
│   分辨率: 28×28 → 14×14              │
└─────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────┐
│       下采样 + LeViT 阶段 3 (4个块)   │
│   注意力头数: 8                       │
│   通道数: 256 → 384                  │
│   分辨率: 14×14 → 7×7                │
└─────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────┐
│          分类头                       │
│   全局平均池化                        │
│   线性层 (384 → 1000)                │
└─────────────────────────────────────┘
     │
     ▼
    输出 (1000维类别概率)
```

这是 LeViT-256 的架构，不同变体（LeViT-128/192/256/384）的具体参数有所不同，但整体结构一致。

### 4.2 前向过程详解

让我们以 LeViT-256 为例，走一遍完整的前向过程。输入图像为 $224 \times 224 \times 3$。

**Step 1: Stem 模块**

输入: $224 \times 224 \times 3$

Stem 由 3 个卷积层组成，每个卷积层后接 BatchNorm 和 Hardswish 激活函数：
- Conv 3×3, stride 2, 输出通道 32: $112 \times 112 \times 32$
- Conv 3×3, stride 2, 输出通道 64: $56 \times 56 \times 64$
- Conv 3×3, stride 2, 输出通道 128: $28 \times 28 \times 128$

输出: $28 \times 28 \times 128$

注意这里没有使用 ViT 的 patch 嵌入（一个大的 16×16 卷积，stride 16），而是用 3 个小卷积逐步下采样。这样做的好处是：
- 在早期阶段保持较高的空间分辨率，保留更多细节信息
- 使用 BatchNorm（而不是 LayerNorm），因为 BatchNorm 在推理时可以融合到卷积中，加速计算
- Hardswish 激活函数比 GELU 更容易量化（LeViT 考虑了 INT8 量化）

**Step 2: 阶段 1（分辨率保持不变）**

输入: $28 \times 28 \times 128$，序列化后为 $784 \times 128$（784 个 token，每个 token 128 维）

阶段 1 包含 2 个 LeViT 块。每个块的结构如下：

```
输入 (H×W×C)
     │
     ▼
┌─────────────────────────────┐
│     注意力子层 (Attention)    │
│   ├─ BatchNorm (预归一化)     │
│   ├─ 多头注意力 (4头)         │
│   └─ 残差连接                 │
└─────────────────────────────┘
     │
     ▼
┌─────────────────────────────┐
│       MLP 子层               │
│   ├─ BatchNorm (预归一化)     │
│   ├─ 线性层 (C → 2C)         │
│   ├─ Hardswish               │
│   ├─ 线性层 (2C → C)         │
│   └─ 残差连接                 │
└─────────────────────────────┘
     │
     ▼
输出 (H×W×C)
```

**注意力子层的详细计算：**

与 ViT 使用 LayerNorm 不同，LeViT 使用 BatchNorm。这里有个关键：BatchNorm 在训练时计算 batch 均值和方差，但在推理时使用固定的 running mean 和 running variance，可以融合到前一个卷积或线性层中，不增加额外的推理开销。

注意力计算本身与标准多头注意力一致，但有一个重要的区别：LeViT 使用了**下采样注意力**（仅在阶段之间使用，阶段内部保持分辨率不变）。

**MLP 子层的详细计算：**

与 ViT 的 MLP 不同，LeViT 使用 Hardswish 替代 GELU，使用 2× 扩展比（而非标准的 4×）。更窄的 MLP 意味着更少的参数和更快的推理。

**Step 3: 下采样 + 阶段 2**

从阶段 1 到阶段 2 之间有一个下采样操作。LeViT 的下采样通过**下采样注意力**（Downsampling Attention）实现，这是 LeViT 的核心创新之一。

下采样注意力模块的工作方式：

1. **输入**: $28 \times 28 \times 128$（784 个 token）
2. **空间下采样**: 将 784 个 token 通过 2×2 的平均池化（stride 2）下采样到 $14 \times 14$（196 个 token）
3. **通道扩展**: 将通道数从 128 扩展到 256
4. **注意力计算**: 下采样后的 196 个 token 作为 query，原始 784 个 token 作为 key 和 value

```
Query: 196 × 256 (来自下采样后的 token)
Key:   784 × 128 (来自原始 token)
Value: 784 × 128 (来自原始 token)

注意力权重: 196 × 784 (query 和 key 的点积)
输出: 196 × 256 (注意力权重与 value 的加权和)
```

这种设计的直觉是：下采样后的 token（query）需要从原始 token（key/value）中聚合信息，而不是从自身聚合。这类似于 CNN 中的池化操作，但通过注意力机制实现了内容相关的下采样。

阶段 2 包含 3 个 LeViT 块，分辨率保持在 $14 \times 14$，通道数为 256。

**Step 4: 下采样 + 阶段 3**

类似地，从阶段 2 到阶段 3 再次进行下采样注意力：
- 输入: $14 \times 14 \times 256$（196 个 token）
- 下采样到 $7 \times 7$（49 个 token）
- 通道扩展到 384

阶段 3 包含 4 个 LeViT 块，分辨率保持在 $7 \times 7$，通道数为 384。

**Step 5: 分类头**

输入: $7 \times 7 \times 384$（49 个 token）

LeViT 没有使用 ViT 的 class token，而是使用全局平均池化（Global Average Pooling, GAP）将 49 个 token 聚合成一个 384 维的向量，然后通过一个线性层映射到 1000 维的类别概率。

使用 GAP 而不是 class token 的好处是：
- 减少了一个额外的可学习参数
- 对输入分辨率的变化更鲁棒（GAP 对空间维度是置换不变的）
- 更接近 CNN 的设计，便于后续的迁移学习

### 4.3 每个设计的"为什么"

**为什么用 BatchNorm 而不是 LayerNorm？**

BatchNorm 在推理时可以融合到前一个卷积或线性层中，不增加额外的计算开销。LayerNorm 则不行，它需要单独计算均值和方差。在推理速度敏感的部署场景中，这个差异很关键。

**为什么用 Hardswish 而不是 GELU？**

Hardswish 是 Swish 的分段线性近似：
$$
\text{Hardswish}(x) = x \cdot \frac{\text{ReLU6}(x+3)}{6}
$$

GELU 需要计算高斯误差函数（erf），计算量较大。Hardswish 只需要简单的乘法和加法操作，且对 INT8 量化更友好。

**为什么用 2× MLP 扩展比而不是 4×？**

ViT 使用 4× 扩展比（MLP 的隐藏层维度是输入维度的 4 倍），这占用了 Transformer 中约 2/3 的参数和计算量。LeViT 将扩展比降低到 2×，显著减少了计算量，同时通过注意力机制来弥补 MLP 表达能力的损失。

**为什么用下采样注意力而不是简单的池化？**

简单的池化（如最大池化或平均池化）是固定的、与内容无关的操作。下采样注意力允许模型根据输入内容动态决定如何聚合信息，保留了 Transformer 的动态权重优势。

## 五、核心创新点

### 创新点 1：下采样注意力（Downsampling Attention）

#### (a) 痛点与动机

在标准 ViT 中，所有层都保持相同的序列长度。这意味着即使到了网络的深层（应该关注全局语义信息），仍然需要处理与浅层相同数量的 token。这导致了两个问题：

1. **计算浪费**：深层的高分辨率 token 大部分是冗余的（因为语义信息通常是低分辨率的）
2. **缺乏多尺度特征**：CNN 通过不同分辨率的特征图构建了多尺度金字塔，这对下游任务（如检测、分割）至关重要

[He et al., 2016] 的 ResNet 证明了多尺度特征金字塔的有效性。[Tan & Le, 2019] 的 EfficientNet 进一步表明，在网络的深度、宽度和分辨率之间进行平衡可以显著提升效率。

#### (b) 方案细节

下采样注意力在降低空间分辨率的同时增加通道数。具体来说，假设输入有 $N$ 个 token，每个 token 的维度为 $C_{\text{in}}$，输出有 $M$ 个 token（$M < N$），每个 token 的维度为 $C_{\text{out}}$（$C_{\text{out}} > C_{\text{in}}$）。

下采样注意力的计算过程如下：

1. **Query 生成**：对输入 token 进行空间下采样（通过平均池化或步长卷积），得到 $M$ 个 query token，维度为 $C_{\text{out}}$
2. **Key 和 Value**：保持原始 $N$ 个 token 作为 key 和 value，维度为 $C_{\text{in}}$
3. **注意力计算**：
   $$
   \text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{C_{\text{in}}}}\right)V
   $$
   其中 $Q \in \mathbb{R}^{M \times C_{\text{out}}}$，$K \in \mathbb{R}^{N \times C_{\text{in}}}$，$V \in \mathbb{R}^{N \times C_{\text{in}}}$

4. **输出**: $M$ 个 token，每个维度 $C_{\text{out}}$

#### (c) 为什么有效

下采样注意力相当于让每个下采样后的 token 去"关注"原始 token 中的信息，然后聚合到自身。这比简单的池化更灵活，因为注意力权重是内容相关的。

从实验结果来看（论文 Table 2），使用下采样注意力比使用平均池化 + 标准注意力的组合在 ImageNet top-1 准确率上高出约 0.5-1.0 个百分点，同时推理速度相当。

#### (d) 与 Related Work 的关系

[Wang et al., 2021] 的 PVT（Pyramid Vision Transformer）也提出了类似的金字塔 Transformer 架构，但 PVT 使用标准注意力加空间下采样，而 LeViT 使用下采样注意力。两者的区别在于：PVT 先下采样再计算注意力，而 LeViT 在下采样的 token（query）和原始 token（key/value）之间计算注意力。

[Wu et al., 2021] 的 CvT（Convolutional Vision Transformer）使用卷积进行下采样，然后使用标准 Transformer 块。LeViT 的不同之处在于将下采样与注意力融合为一个操作。

#### (e) 如果去掉会怎样

从论文 Table 3 的消融实验可以看出：
- 去掉下采样注意力（改为标准注意力 + 平均池化）会导致 top-1 准确率下降约 0.8%
- 进一步去掉下采样（保持所有层相同分辨率）会导致 FLOPs 增加约 40%，但准确率仅提升 0.2%

这说明下采样注意力在效率-精度权衡中起到了关键作用。

### 创新点 2：注意力偏置（Attention Bias）

#### (a) 痛点与动机

标准 Transformer 的自注意力是全局的——每个 token 可以关注所有其他 token。这在视觉任务中并非最优，因为图像中的局部相关性远强于全局相关性。一个像素通常只与其相邻的像素有强相关性。

[Ramachandran et al., 2019] 的 Stand-Alone Self-Attention 已经证明了在视觉任务中引入局部注意力偏置的有效性。

#### (b) 方案细节

LeViT 在注意力计算中加入了可学习的位置偏置，具体来说，对于 query 位置 $i$ 和 key 位置 $j$，注意力得分计算为：

$$
\text{AttentionScore}(i, j) = q_i^T k_j + b_{\Delta x, \Delta y}
$$

其中 $\Delta x = x_i - x_j$，$\Delta y = y_i - y_j$，$b_{\Delta x, \Delta y}$ 是一个可学习的标量偏置，只依赖于两个 token 之间的相对位置。

这种偏置的维度是 $(2H-1) \times (2W-1)$，其中 $H$ 和 $W$ 是特征图的高度和宽度。对于 $14 \times 14$ 的特征图，偏置矩阵的大小为 $27 \times 27$（729 个参数），远小于全局注意力矩阵的 $196 \times 196$（38,416 个参数）。

#### (c) 为什么有效

注意力偏置相当于给模型提供了一个先验：相邻位置的 token 应该具有更高的注意力权重。这类似于卷积的局部感受野，但更加灵活——模型可以通过学习调整这个偏置，使其适应不同的任务和数据。

与绝对位置编码（如 ViT 使用的可学习位置嵌入）相比，注意力偏置有两个优势：
1. **平移等变性**：相对位置偏置对平移是等变的，而绝对位置编码不是
2. **参数效率**：偏置矩阵的大小与特征图尺寸无关，只与相对位置范围有关

#### (d) 与 Related Work 的关系

注意力偏置并非 LeViT 首创。[Shaw et al., 2018] 在机器翻译任务中首次提出了相对位置偏置。[Hu et al., 2019] 的 Squeeze-and-Excitation 网络使用了类似的思想，但作用于通道维度而非空间维度。

LeViT 的贡献在于将注意力偏置与下采样注意力结合，并证明了在视觉 Transformer 中，即使是简单的相对位置偏置也足以替代复杂的位置编码方案。

#### (e) 如果去掉会怎样

从论文 Table 3 的消融实验可以看出：
- 去掉注意力偏置（改为绝对位置编码或没有位置编码）会导致 top-1 准确率下降约 0.5-1.0%
- 将注意力偏置替换为绝对位置编码会导致参数增加约 10%，但准确率反而更低

### 创新点 3：激活函数和归一化层的选择

#### (a) 痛点与动机

ViT 使用 GELU 激活函数和 LayerNorm 归一化。GELU 需要计算高斯误差函数（erf），在推理时比简单的 ReLU 或 Hardswish 慢得多。LayerNorm 需要计算每个 token 的均值和方差，无法像 BatchNorm 那样融合到前一个线性层中。

[Howard et al., 2019] 的 MobileNetV3 证明了 Hardswish 在移动端 CNN 中的有效性。[Ioffe & Szegedy, 2015] 的 BatchNorm 是 CNN 中加速训练和推理的标准方法。

#### (b) 方案细节

LeViT 使用以下组合：
- **激活函数**: Hardswish（MLP 中），替代 GELU
- **归一化**: BatchNorm（注意力子层和 MLP 子层中），替代 LayerNorm

Hardswish 的计算公式：
$$
\text{Hardswish}(x) = x \cdot \frac{\text{ReLU6}(x+3)}{6}
$$

其中 $\text{ReLU6}(x) = \min(\max(0, x), 6)$。

#### (c) 为什么有效

Hardswish 相比于 GELU 的计算优势：
- GELU: 需要计算 $\text{erf}(x)$，这是一个超越函数，计算复杂度高
- Hardswish: 只需要 $\text{ReLU6}$ 和简单的乘除操作，计算复杂度低

BatchNorm 相比于 LayerNorm 的推理优势：
- BatchNorm: 推理时 $\hat{x} = \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} \cdot \gamma + \beta$ 可以融合为 $x \cdot w + b$ 的形式，与前面的卷积或线性层合并
- LayerNorm: 需要为每个 token 单独计算均值和方差，无法融合

从论文 Table 4 可以看出，将 GELU 替换为 Hardswish 后，推理速度提升了约 15%，而准确率仅下降 0.1%。

#### (d) 与 Related Work 的关系

[Ramachandran et al., 2017] 提出的 Swish 激活函数展示了自门控（self-gating）机制的有效性。Hardswish 是 Swish 的分段线性近似，由 [Howard et al., 2019] 在 MobileNetV3 中提出。

[Xiong et al., 2020] 的论文证明了在 Transformer 中使用 LayerNorm 预归一化的有效性。LeViT 保留了预归一化的结构（在注意力/MLP 之前进行归一化），但将 LayerNorm 替换为 BatchNorm。

#### (e) 如果去掉会怎样

从论文 Table 4 的消融实验可以看出：
- 将 Hardswish 替换为 GELU：推理速度下降约 15%，准确率提升 0.1%
- 将 BatchNorm 替换为 LayerNorm：推理速度下降约 20%，准确率下降 0.3%（因为 LayerNorm 与 CNN stem 的兼容性较差）

### 创新点 4：CNN 风格的 Stem 模块

#### (a) 痛点与动机

ViT 使用一个大的 16×16 卷积（stride 16）将图像切分为 patch。这种设计有两个问题：
1. **信息损失**：在早期阶段就大幅降低空间分辨率，丢失了细节信息
2. **计算效率低**：大的卷积核（16×16）在 GPU 上的计算效率不如多个小卷积核（3×3）

[Xie et al., 2017] 的 Aggregated Residual Transformations 和 [Szegedy et al., 2015] 的 Inception 网络都证明了使用多个小卷积核替代大卷积核的有效性。

#### (b) 方案细节

LeViT 的 stem 模块由 3 个卷积层组成，每个卷积层使用 3×3 核、stride 2，后接 BatchNorm 和 Hardswish：

```
Conv 3×3, stride 2, C=32 → BN → Hardswish
Conv 3×3, stride 2, C=64 → BN → Hardswish
Conv 3×3, stride 2, C=128 → BN → Hardswish
```

对于 224×224 的输入，输出为 28×28×128。

#### (c) 为什么有效

这种设计相当于将 ViT 的 patch 嵌入层（一个 16×16 卷积）分解为 3 个 3×3 卷积的堆叠。这样做的好处是：

1. **增加非线性**：3 个卷积层之间有 2 个激活函数，而单个 16×16 卷积只有 1 个激活函数
2. **参数效率**：3 个 3×3 卷积的参数为 $3 \times 3 \times 3 \times 32 + 3 \times 3 \times 32 \times 64 + 3 \times 3 \times 64 \times 128 = 864 + 18,432 + 73,728 = 93,024$，而单个 16×16 卷积的参数为 $16 \times 16 \times 3 \times 128 = 98,304$，两者相近但前者计算量更小
3. **保留细节**：逐步下采样保留了更多中间特征图

#### (d) 与 Related Work 的关系

[He et al., 2016] 的 ResNet 使用 7×7 卷积（stride 2）作为 stem，然后接最大池化（stride 2）。LeViT 使用 3 个 3×3 卷积替代，这与 [Sandler et al., 2018] 的 MobileNetV2 的 stem 设计类似。

[Wu et al., 2021] 的 CvT 也使用了类似的卷积 stem，但 CvT 的 stem 输出分辨率是 14×14（下采样 16 倍），而 LeViT 是 28×28（下采样 8 倍），保留了更高的分辨率。

#### (e) 如果去掉会怎样

从论文 Table 5 的消融实验可以看出：
- 将 3 个 3×3 卷积替换为 1 个 16×16 卷积（ViT 风格）：推理速度提升约 5%，但准确率下降约 0.8%
- 将 3 个 3×3 卷积替换为 2 个 3×3 卷积（下采样 8 倍 → 下采样 4 倍）：推理速度下降约 10%，准确率提升 0.2%

这说明卷积 stem 在精度-效率权衡中起到了重要作用。

## 六、公式详解

### 公式 1：注意力偏置的计算

$$
\text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T + B}{\sqrt{d_k}}\right)V
$$

**符号定义：**
- $Q \in \mathbb{R}^{N \times d_k}$：query 矩阵
- $K \in \mathbb{R}^{N \times d_k}$：key 矩阵
- $V \in \mathbb{R}^{N \times d_v}$：value 矩阵
- $B \in \mathbb{R}^{N \times N}$：注意力偏置矩阵，其中 $B_{ij} = b_{\Delta x, \Delta y}$，$\Delta x = x_i - x_j$，$\Delta y = y_i - y_j$
- $d_k$：key 的维度（等于 query 的维度）
- $N$：序列长度

**公式来源：**

这是标准缩放点积注意力（Vaswani et al., 2017）的变体，加入了相对位置偏置（Shaw et al., 2018）。标准注意力公式为：
$$
\text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

LeViT 在 $QK^T$ 的基础上加上了 $B$，相当于给每个 query-key 对添加了一个可学习的偏置。

**推导过程：**

1. 计算 query 和 key 的点积：$S = QK^T \in \mathbb{R}^{N \times N}$
2. 加入偏置：$S' = S + B$
3. 缩放：$S'' = S' / \sqrt{d_k}$
4. Softmax：$A = \text{Softmax}(S'') \in \mathbb{R}^{N \times N}$
5. 加权聚合：$O = AV \in \mathbb{R}^{N \times d_v}$

**直觉理解：**

注意力偏置相当于给注意力矩阵加了一个"先验模板"。例如，对于图像中的相邻像素，偏置值较大（鼓励关注），对于相距较远的像素，偏置值较小（抑制关注）。模型通过学习调整这个模板，使其适应具体的任务。

**具体例子：**

假设 $14 \times 14$ 的特征图，$d_k = 64$。

对于位置 $(5, 5)$ 和 $(7, 8)$：
- $\Delta x = 5 - 7 = -2$
- $\Delta y = 5 - 8 = -3$
- $b_{\Delta x, \Delta y} = b_{-2, -3}$ 是一个可学习的标量

假设 $b_{-2, -3} = 0.3$，$q_{(5,5)}^T k_{(7,8)} = 12.5$，则注意力得分（缩放前）为 $12.5 + 0.3 = 12.8$。

**边界情况：**

- 当 $B = 0$（所有偏置为 0）时，退化为标准注意力
- 当 $d_k$ 很大时，缩放因子 $\sqrt{d_k}$ 防止 softmax 进入饱和区
- 对于特征图边缘的位置，某些相对位置可能不存在（例如，左上角位置没有左上方的邻居），此时对应的偏置参数不会被使用

### 公式 2：下采样注意力的复杂度分析

$$
\text{Complexity} = O\left(M N d_k + N d_k d_v\right)
$$

**符号定义：**
- $M$：下采样后的序列长度（query 的数量）
- $N$：原始序列长度（key/value 的数量）
- $d_k$：query 和 key 的维度
- $d_v$：value 的维度

**公式来源：**

标准注意力的复杂度为 $O(N^2 d_k + N d_k d_v)$（Dosovitskiy et al., 2021）。下采样注意力将第一项从 $O(N^2 d_k)$ 降低为 $O(M N d_k)$，因为 query 的数量从 $N$ 降低到了 $M$。

**推导过程：**

下采样注意力的计算包括两部分：
1. **注意力权重计算**：$QK^T$，其中 $Q \in \mathbb{R}^{M \times d_k}$，$K \in \mathbb{R}^{N \times d_k}$，计算量为 $M \times N \times d_k$ 次乘法
2. **加权聚合**：$AV$，其中 $A \in \mathbb{R}^{M \times N}$，$V \in \mathbb{R}^{N \times d_v}$，计算量为 $M \times N \times d_v$ 次乘法

总复杂度为 $O(M N (d_k + d_v))$，通常简写为 $O(M N d_k)$（假设 $d_k \approx d_v$）。

**直觉理解：**

下采样注意力相当于将标准注意力中的 $N^2$ 项替换为 $MN$ 项。因为 $M < N$（通常 $M = N/4$ 或 $M = N/16$），所以复杂度显著降低。

**具体例子：**

考虑 LeViT-256 中从阶段 1 到阶段 2 的下采样注意力：
- $N = 28 \times 28 = 784$
- $M = 14 \times 14 = 196$
- $d_k = 128$（key 的维度）
- $d_v = 256$（value 的维度，因为输出通道扩展了）

标准注意力复杂度：$O(784^2 \times 128 + 784 \times 128 \times 256) = O(78,643,200 + 25,690,112) \approx O(104M)$

下采样注意力复杂度：$O(196 \times 784 \times 128 + 784 \times 128 \times 256) = O(19,660,800 + 25,690,112) \approx O(45M)$

降低约 57% 的计算量。

**边界情况：**

- 当 $M = N$（不下采样）时，退化为标准注意力
- 当 $M = 1$（下采样到单个 token）时，复杂度为 $O(N d_k)$，相当于全局平均池化
- 当 $d_k$ 远大于 $N$ 时，第二项 $N d_k d_v$ 占主导

### 公式 3：Hardswish 激活函数

$$
\text{Hardswish}(x) = x \cdot \frac{\text{ReLU6}(x+3)}{6}
$$

**符号定义：**
- $x$：输入标量或张量
- $\text{ReLU6}(x) = \min(\max(0, x), 6)$：截断的 ReLU

**公式来源：**

Hardswish 是 Swish 激活函数（Ramachandran et al., 2017）的分段线性近似。Swish 定义为 $x \cdot \sigma(x)$，其中 $\sigma$ 是 sigmoid 函数。Hardswish 用 $\text{ReLU6}(x+3)/6$ 近似 sigmoid。

**推导过程：**

Swish 的精确形式为 $x \cdot \frac{1}{1+e^{-x}}$。Hardswish 的近似为：

1. 将 sigmoid 近似为 $\frac{\text{ReLU6}(x+3)}{6}$
2. 当 $x \le -3$ 时，$\text{ReLU6}(x+3) = 0$，所以 $\text{Hardswish}(x) = 0$
3. 当 $-3 < x < 3$ 时，$\text{ReLU6}(x+3) = x+3$，所以 $\text{Hardswish}(x) = x \cdot \frac{x+3}{6}$
4. 当 $x \ge 3$ 时，$\text{ReLU6}(x+3) = 6$，所以 $\text{Hardswish}(x) = x$

**直觉理解：**

Hardswish 在 $x \le -3$ 时输出 0（类似 ReLU），在 $x \ge 3$ 时输出 $x$（线性），在中间区域输出一个二次函数。这种"软饱和"特性与 Swish 类似，但计算更简单。

**具体例子：**

- $x = 0$：$\text{Hardswish}(0) = 0 \cdot \frac{\text{ReLU6}(3)}{6} = 0 \cdot \frac{3}{6} = 0$
- $x = 2$：$\text{Hardswish}(2) = 2 \cdot \frac{\text{ReLU6}(5)}{6} = 2 \cdot \frac{5}{6} = 1.67$
- $x = 4$：$\text{Hardswish}(4) = 4 \cdot \frac{\text{ReLU6}(7)}{6} = 4 \cdot \frac{6}{6} = 4$
- $x = -2$：$\text{Hardswish}(-2) = -2 \cdot \frac{\text{ReLU6}(1)}{6} = -2 \cdot \frac{1}{6} = -0.33$

**边界情况：**

- 当 $x \to -\infty$ 时，Hardswish 趋近于 0（饱和）
- 当 $x \to +\infty$ 时，Hardswish 趋近于 $x$（线性）
- 在 $x = -3$ 和 $x = 3$ 处，Hardswish 是连续的但不可导（分段点）

## 八、位置

### 前驱工作

1. **ViT (Dosovitskiy et al., 2021)**：首次证明纯 Transformer 可以在图像分类上达到 SOTA，但需要大规模预训练且推理速度慢。LeViT 直接继承了 Transformer 的自注意力机制，但通过引入 CNN 的架构设计来解决推理速度问题。

2. **DeiT (Touvron et al., 2021)**：通过知识蒸馏和数据增强技术，使 ViT 能够在 ImageNet 上从头训练。LeViT 借鉴了 DeiT 的训练策略，但进一步优化了架构设计。

3. **MobileNetV3 (Howard et al., 2019)**：证明了 Hardswish 激活函数和深度可分离卷积在移动端 CNN 中的有效性。LeViT 直接采用了 Hardswish 和类似 CNN 的 stem 设计。

4. **EfficientNet (Tan & Le, 2019)**：通过神经架构搜索找到了精度-效率的 Pareto 最优解。LeViT 的架构设计也追求类似的目标，但使用 Transformer 块替代了部分卷积层。

5. **Relative Position Bias (Shaw et al., 2018)**：在机器翻译中首次提出相对位置偏置。LeViT 将其引入视觉 Transformer，替代了 ViT 的绝对位置编码。

### 同期竞品

1. **PVT (Wang et al., 2021)**：也提出了金字塔视觉 Transformer 架构，但使用标准注意力加空间下采样，而非 LeViT 的下采样注意力。PVT 更关注下游任务（检测、分割），而 LeViT 更关注推理速度。

2. **CvT (Wu et al., 2021)**：在 Transformer 中引入卷积操作，包括卷积的 patch 嵌入和卷积的注意力。CvT 和 LeViT 都使用了卷积 stem，但 CvT 在注意力中也使用了卷积（深度可分离卷积），而 LeViT 保持了纯注意力的计算。

3. **Swin Transformer (Liu et al., 2021)**：通过移动窗口注意力实现了高效的层次化 Transformer。Swin 和 LeViT 都采用了多尺度设计，但 Swin 使用局部窗口注意力，而 LeViT 使用全局注意力加下采样。

4. **T2T-ViT (Yuan et al., 2021)**：通过递归地将相邻 token 拼接来逐步降低序列长度。T2T-ViT 和 LeViT 都使用了渐进的 token 减少策略，但 T2T-ViT 使用拼接操作，而 LeViT 使用下采样注意力。

### 后续影响

1. **EfficientViT (Cai et al., 2022)**：进一步优化了 LeViT 的设计，提出了更高效的视觉 Transformer 架构，在推理速度和精度之间取得了更好的权衡。

2. **MobileViT (Mehta & Rastegari, 2022)**：将 CNN 和 Transformer 的融合推向了移动端，使用了类似 LeViT 的混合架构设计，但更注重模型大小和内存效率。

3. **FastViT (Vasu et al., 2023)**：在 LeViT 的基础上引入了重参数化技术，进一步提升了推理速度，同时保持了高精度。

4. **EdgeViT (Pan et al., 2022)**：专门为边缘设备设计的视觉 Transformer，借鉴了 LeViT 的下采样注意力和 CNN stem 设计。

5. **LightViT (Huang et al., 2022)**：通过引入轻量级的注意力机制和 MLP 设计，进一步降低了视觉 Transformer 的计算复杂度。

## 九、局限

### 局限 1：对下游任务的适应性有限

LeViT 的设计主要针对图像分类任务，其下采样注意力机制使得特征图的分辨率变化与分类任务的需求高度契合。但对于需要高分辨率特征图的下游任务（如目标检测、语义分割），LeViT 的架构可能需要较大的修改。

**具体表现：** LeViT 在阶段 3 输出的特征图分辨率为 7×7，对于检测任务来说太低，需要额外的上采样或特征金字塔网络。

**论文是否承认：** 论文主要关注分类任务，没有讨论下游任务的迁移。

**后续工作：** PVT（Wang et al., 2021）和 Swin Transformer（Liu et al., 2021）更注重下游任务的适应性，提供了更好的多尺度特征表示。

### 局限 2：训练策略的依赖

LeViT 的训练策略比较复杂，需要数据增强、知识蒸馏和 EMA（指数移动平均）等多种技术才能达到最佳性能。

## 十、小结

LeViT 是一篇在视觉 Transformer 领域具有重要影响的论文。它的核心贡献在于：**通过将 CNN 的多尺度金字塔设计和高效的归一化/激活函数引入 Transformer，实现了在推理速度上接近 CNN、在表达能力上接近 Transformer 的混合架构。**

具体来说，LeViT 做了以下几件事：
1. **下采样注意力**：在降低空间分辨率的同时进行注意力计算，既减少了计算量，又保留了 Transformer 的动态权重优势
2. **注意力偏置**：用可学习的相对位置偏置替代 ViT 的绝对位置编码，引入了局部先验，提高了参数效率
3. **CNN 风格的架构设计**：使用卷积 stem、BatchNorm、Hardswish 等 CNN 中验证有效的技术，显著提升了推理速度

LeViT 的 broader impact 在于：它证明了 Transformer 和 CNN 不是非此即彼的选择，而是可以取长补短、深度融合的。这种"混合架构"的思路对后续的 EfficientViT、MobileViT、FastViT 等工作产生了深远影响。

从更宏观的角度看，LeViT 代表了视觉 Transformer 发展的一个重要趋势：**从纯 Transformer（ViT）到混合架构（LeViT、CvT），再到 CNN 化的 Transformer（ConvNeXt），最终走向了架构的融合与统一。** LeViT 在这个趋势中扮演了承前启后的角色，为后续的轻量级视觉 Transformer 设计提供了重要的参考。
