# Flamingo: a Visual Language Model for Few-Shot Learning（2022）

**论文：** [arXiv:2204.14198](https://arxiv.org/abs/2204.14198) · Jean-Baptiste Alayrac, Jeff Donahue, Pauline Luc, Antoine Miech et al. · DeepMind · NeurIPS 2022

**技术分支：** D2 — 视觉模型和语言模型是否可以结合？ / D2-8 — 冻结 LLM + 视觉适配器

---

## 一、先搞清楚"问题在哪里"

2022 年之前的视觉-语言模型（VLM）大致分两条路。

**第一条路是双塔模型**，比如 CLIP、ALIGN。图像和文本各走各的 encoder，最后在 embedding space 里做对比学习。这条路的核心假设是：只要把视觉和语言的表示空间对齐，下游任务自然能通过某种方式利用这个对齐。这个假设在 zero-shot 分类上成立——给定一张图，CLIP 能选出最匹配的文本——但一旦遇到需要**生成式**的任务，比如视觉问答（VQA）或图像描述（captioning），CLIP 就无能为力了。为什么？因为 CLIP 只输出一个全局的图文相似度分数，它没有能力做"跨模态的信息组合"，更不具备**语言生成能力**。

**第二条路是端到端多模态 Transformer**，比如 UNITER、ViLBERT。这些模型把图像区域特征（来自 object detector）和文本 token 拼在一起，用 Transformer 做统一建模。这条路的问题是：**成本太高，扩展性差**。训练一个 UNITER 需要先跑一个 Faster R-CNN 提取图像区域特征，然后训练一个 12 层 Transformer。更致命的是，每次换任务或换数据集，都要重新训练整个模型或做 domain-specific fine-tuning。这不是一个可以"一次训练、到处使用"的范式。

**第三条路正在兴起：大规模语言模型（LLM）的 in-context learning 能力。** 2020 年的 GPT-3 展现了一个惊人的事实：给一个冻结的 LLM 几个输入/输出样例，它就能"学会"做新任务，不需要任何权重更新。研究者自然想问：**能不能把视觉信息也以同样灵活的方式"喂"进一个冻结的 LLM，让它学会看图说话？**

所以到了 2022 年，研究者面临的核心矛盾是：

> 我们有了 CLIP 这种强大的预训练视觉模型，但只有检索能力；我们有了 GPT-3 这种强大的 in-context learner，但只能理解文本。能不能把两者桥接起来，让一个冻结的 LLM 也具备视觉理解能力，而且不需要重新训练？

这就是 Flamingo 要解决的问题。

## 二、旧路线的真正问题

在 Flamingo 之前，想让 LLM 处理视觉信息，最直接的做法是：**把视觉特征当作文本 token 的 prefix，然后让 LLM 继续生成文本**。比如，用 CLIP 提取图像特征，拼在文本 prompt 前面，然后输入给一个冻结的 LLM。但这条路有三个深层问题。

### 2.1 表示空间不匹配（Representation Gap）

CLIP 的 visual encoder 输出的是 patch-level 特征，这些特征是在对比学习目标下训练的。它们擅长区分"图里是不是有猫"，但不擅长回答"猫和沙发的空间关系是什么"。LLM 期望的输入是 token 序列，每个 token 有明确的语义角色（比如主语、谓语、宾语）。直接把 CLIP 特征拼进去当 prefix token，LLM 不知道这些向量代表什么。

更准确地说，**视觉特征的"语义粒度"和 LLM 词 token 的"语义粒度"是不对齐的**。一个图像 patch 可能包含多个物体的一部分，而一个文本 token 通常对应一个完整的词或子词单元。这种粒度不对齐导致 LLM 无法有效地"解读"视觉特征。

### 2.2 冻结 LLM 的注意力机制不知道看哪里

LLM 的自注意力是在 token 之间计算的。如果视觉特征被当作 prefix token 放在文本 token 之前，那么 LLM 的注意力头必须学会"什么时候该看视觉 token，什么时候该看文本 token"。

问题是：**冻结的 LLM 的注意力模式已经被纯文本训练固化了**。它在纯文本上训练了几万亿个 token，所有的 attention head 都已经学会了"第一个 token 关注第二个 token，第三个关注第五个"这类文本特定的模式。突然塞进视觉 token，这些 attention head 不知道如何给它们分配注意力权重。

### 2.3 因果掩码导致视觉信息稀释

LLM 是 causal decoder（因果解码器），它只能从左到右生成，不能回头看。如果视觉信息只作为 prefix 输入（放在文本的最前面），那么 LLM 在生成后面的文本时，只能通过 prefix 的 hidden state 来**间接**访问视觉信息。随着生成步数增长，视觉信息在 hidden state 中的信号会被逐渐稀释——self-attention 的残差路径会慢慢"冲刷"掉 prefix 的影响。

对于需要持续引用视觉细节的任务（比如图像描述，模型需要在生成过程中反复"查看"图像），这很致命。比如描述"一只橙色的猫坐在蓝色的沙发上"，生成到"蓝色的沙发"时，模型早就把"猫的颜色"相关的视觉信息稀释掉了。

## 三、这篇论文的核心思路

Flamingo 的核心思路可以用一句话概括：

> **不是把视觉信号塞进 LLM 的输入端，而是在 LLM 的每一层之间插入视觉感知的"入口"——让 LLM 在生成每个 token 时，都能主动"看到"视觉信息。**

具体来说，三个设计组合在一起：

```
冻结的 Vision Encoder（NFNet-F6） → 感知视觉
        ↓
Perceiver Resampler → 变长输入 → 固定数量的视觉 token
        ↓
Gated Cross-Attention → 在冻结 LLM 的每一对 frozen layer 之间插入
        → 让 LLM 在生成每个 token 时都能访问视觉信息
        ↓
冻结的 LLM（Chinchilla） → 基于文本和视觉信息做自回归生成
```

```text
这条技术路线的谱系位置是：

双塔模型（CLIP/ALIGN, 2021）: 图文对齐但无生成能力
   ↓
端到端多模态 Transformer（UNITER/ViLBERT, 2019）: 有生成能力但成本高
   ↓
Flamingo（2022） ← 本篇：冻结 LLM + 桥接视觉 + in-context few-shot
   ↓
OpenFlamingo（2023）: 开源复现
BLIP-2（2023）: 用 Q-Former 替代 Perceiver Resampler
LLaVA（2023）: 用简单线性层替代复杂桥接模块
```

Flamingo 在技术路线上的关键转折是：**不再把视觉建模作为多模态模型的"主体"**，而是把视觉作为一个**外挂能力**，桥接到已经训练好的、冻结的 LLM 上。这种方法最大的好处是：LLM 的语言能力完全不受影响（因为权重是冻结的），只需要训练一个轻量的桥接模块。这也是为什么 Flamingo 被归类为"冻结 LLM + 视觉适配器"范式（D2-8）的开创者。

## 四、整体架构与 forward 流程

### 4.1 整体架构

Flamingo 是一个视觉条件化的自回归文本生成模型（visually-conditioned autoregressive text generation model）。它的输入是一系列文本 token 和图像/视频的**交错序列**，输出是自由形式的文本。

```text
输入: [<image>] <文本A> [<image>] <文本B> ...
    │
    ▼
┌─────────────────────────────────────────┐
│ Vision Encoder (NFNet-F6, 冻结)         │
│  输入: 图像/视频帧                       │
│  输出: 变长的时空视觉特征 X_f            │
│  注: 视频 → 每帧独立通过 vision encoder │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│ Perceiver Resampler                     │
│  输入: 变长时空特征 X_f（位置编码后）    │
│  输出: 固定数量（64个）视觉 token        │
│  机制: learned latent queries 交叉注意   │
│        到 X_f + learned latents          │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│ Frozen LM (Chinchilla, 冻结)            │
│  条件: 文本 token 序列 +                  │
│        来自 Perceiver 的视觉 token        │
│        通过 GATED CROSS-ATTENTION 注入   │
│  输出: 自回归文本生成                     │
│                                          │
│  ┌─────────────────────────────────┐    │
│  │ Frozen LM Block l               │    │
│  │   ↓                              │    │
│  │ self-attention (冻结)             │    │
│  │   ↓                              │    │
│  │ ┌──────────────────────────┐    │    │
│  │ │ GATED CROSS-ATTENTION    │ ←  │    │← 视觉 token
│  │ │ (新训练, 2层)             │    │    │
│  │ │ Query: 来自 LLM hidden    │    │    │
│  │ │ Key/Value: 视觉 token     │    │    │
│  │ │ 输出: 加 tanh 门控再残差  │    │    │
│  │ └──────────────────────────┘    │    │
│  │   ↓                              │    │
│  │ FFN (冻结)                        │    │
│  └─────────────────────────────────┘    │
│        重复 L 层 (每2层插入1个 xattn)   │
└─────────────────────────────────────────┘
    │
    ▼
输出: 文本 token（自回归生成）
```

### 4.2 模型规格（3 个变体）

| 变体 | LLM 参数量 | Vision Encoder | Total 参数量 | LM Layers | 视觉 token 数 |
|------|-----------|---------------|-------------|-----------|-------------|
| Flamingo-3B | 1.4B | 430M (NFNet-F0) | 3B | 24 | 64 |
| Flamingo-9B | 7.1B | 430M (NFNet-F0) | 9B | 40 | 64 |
| Flamingo-80B | 70B (Chinchilla) | 430M (NFNet-F6) | 80B | 80 | 64 |

Vision Encoder 在三个变体中都是冻结的。LLM 也是冻结的。**只有新增的桥接组件——Perceiver Resampler 和 Gated Cross-Attention 层——是训练的。** 这意味着整个 80B 模型的训练参数只有约 10-20B 参数（主要是 cross-attention 和 perceiver 的权重）。

### 4.3 前向过程逐步跟踪

**Step 1 — Vision Encoding**

输入图像或视频帧。使用 NFNet（Normalizer-Free ResNet）作为 vision encoder。NFNet-F6（最大版本）在大规模对比学习（ALIGN-style）上预训练。对视频，先采样 n 帧，每帧独立通过 vision encoder，输出多帧特征。

NFNet 的输出是二维特征图（对图像）或三维时空特征（对视频）。由于 CNN 隐式编码了空间信息，Flamingo 只在时间维度上加位置编码（temporal position encoding），不需要空间位置编码。

**Step 2 — Perceiver Resampler**

输入：变长的时空视觉特征（来自 NFNet 的输出）。
输出：**固定数量的视觉 token**（64 个）。

具体做法：
- 对每帧特征加 learnt temporal position encoding
- 所有帧特征展平并拼接 → 变长的视觉特征序列 X_f
- 用一组 **learned latent queries**（固定数量，64 个）对 X_f 做 cross-attention
- 同时将 learned latents 自身也作为 key/value 拼接到 X_f 上（论文发现这样效果好稍好一些）

关键意义：**Perceiver Resampler 把任意分辨率、任意帧数的视觉输入统一映射到固定数量的视觉 token。** 无论输入是 224×224 的图片还是 16 帧的视频，输出总是 64 个 token。这为后续的 cross-attention 提供了统一接口。

**Step 3 — Frozen LM + Gated Cross-Attention**

LLM（冻结的 Chinchilla）接收文本 token。在每对 frozen LM layers 之间（相邻冻结层之间）插入新训练的 **Gated Cross-Attention (xattn-dense)** 层。论文具体做法是**每两个冻结层之间插一个**：第 1 层后、第 3 层后、第 5 层后……以此类推。对于 80 层的 Chinchilla，插入11 个 cross-attention 层（每 7 层插一个）。

Gated Cross-Attention 的结构：

```python
# 伪代码：Gated Cross-Attention
def gated_xattn(x, visual_tokens):
    """
    x: 来自冻结 LLM 层的 hidden state [B, L, d]
    visual_tokens: 来自 Perceiver Resampler [B, 64, d]
    """
    # Cross-attention: query 来自 LLM, key/value 来自视觉 token
    x_attn = cross_attention(query=x, key=visual_tokens, value=visual_tokens)

    # 两层 MLP（FFN）处理 cross-attention 输出
    x_attn = dense_layers(x_attn)

    # tanh 门控 + 残差连接
    gate = tanh(alpha)  # alpha 是可学习的标量，初始化=0
    return x + gate * x_attn  # 初始时 gate≈0，直接跳过 → 不影响预训练 LLM 的初始化
```

**tanh 门控的作用至关重要：**

1. **初始化为 0**：训练开始时 `tanh(0) = 0`，cross-attention 的输出完全被阻断。此时 Flamingo 的行为等同于纯文本的 Chinchilla——语言能力不受破坏。

2. **随着训练梯度调整**：`alpha` 逐渐增长到非零值，视觉信息逐渐注入。图 6（论文附录）展示了这个演化过程——不同层的门控激活值有不同的大小，门控值的绝对值随深度增加而增大（高层大于底层）。

3. **每层独立学习**：不同层的 `tanh` 门控学习到不同的强度，说明不同层以不同方式利用视觉信息。

## 五、核心模块逐个解释

### 5.1 Perceiver Resampler：变长→定长的桥接

**问题：** Vision encoder 的输出随输入分辨率和帧数变化。一张 224×224 图片输出约 50×50=2500 个特征（NFNet 下采样倍率），一段 16 帧视频输出 2500×16=40000 个特征。但 LLM 需要固定的计算模式——不能每来一个视频都重新调整注意力结构。

**解决方案：** 用一组 learned latent queries（64 个）通过 cross-attention 从变长视觉特征中提取固定数量的视觉 token。具体结构类似于 Perceiver（Jaegle et al., 2021）：

```
Latent Queries (64, d) → Cross-Attention → 64 个视觉 token
    Q                     K, V (来自视觉特征 + latent queries 拼接)
```

**为什么是 Perceiver-style 而不是简单池化？**

- 简单池化（Global Average Pooling）丢失了所有空间细节——"猫在沙发左边"变成了无法编码的信息
- 注意力池化（learned queries）可以让不同 query 关注不同空间位置，保留了空间信息的结构化表示
- 每个 query 相当于"一个视觉问题"：query 1 关注前景物体，query 2 关注背景，query 3 关注颜色，等等

### 5.2 Gated Cross-Attention（核心创新）

**问题：** 视觉信息需要以方法注入 LLM，既能被 LLM 访问，又不会干扰 LLM 的预训练表示。

**解决方案：** 在冻结的 LLM layers 之间插入可训练的 cross-attention 层，用 tanh 门控初始化为零。这是 Flamingo 最关键的架构创新。

**为什么不在最前面拼接（prefix-style）？**

论文用实验（附录 B.3.1 的消融实验）证明，prefix-style 的视觉注入方式远不如 cross-attention 方式。原因正是 2.3 节讨论的因果掩码导致的信息稀释问题：prefix 视觉 token 在长时间生成中逐渐被遗忘。

**为什么是层间插入而不是层内替换？**

层间插入保留了 LLM 的完整结构——self-attention 和 FFN 都不需要修改。cross-attention 作为附加模块，只负责"注入视觉信息"，不影响 LLM 自身的"推理路径"。这意味着：

- LLM 的权重完全不变
- 新增层的梯度只在训练时反向传播到 cross-attention 参数，不影响冻结层的任何参数
- 训练后，cross-attention 层"寄生"在冻结 LLM 上，随时可以附加/分离

**插入密度：** 论文选择每两层插入一个 cross-attention 层。不是每层插一层（节省计算），也不是只在顶层插（视觉信息太靠后）。这个选择在实验中验证是合理的权衡。

### 5.3 多图/视频 attention 机制

**问题：** 输入可能有多个图像或一段视频，每个图像/视频生成 64 个 token。当 prompt 包含多张图（比如 4-shot 场景：4 个样例图 + 1 个查询图 = 5 张图），总共有 5×64=320 个视觉 token。cross-attention 需要让当前文本 token 能"看到"所有过去的视觉信息。

**解决方案：** 在 cross-attention 中，每个 LLM hidden state 的 query 不仅能注意"当前"的视觉 token，还能注意**所有之前出现过的**视觉 token。

具体来说，Flamingo 维护一个**视觉 token 的缓存**，cross-attention 的 key/value 来自所有历史视觉 token（包括之前样例图的）。生成第 5 句描述时，模型仍然可以查看第 1 张图的视觉 token。

**掩码规则：** 文本 token 不能看到"未来"的视觉 token（符合因果性），但可以看到所有"过去"的视觉 token。对于视频帧，同一视频的不同帧之间互相可见。

### 5.4 M3W 数据 + 数据混合训练

Flamingo 的训练数据来自三个来源：

**M3W（Multi-Modal Massive Web，多模态海量网页）**

- 从百万级网页中爬取的图文交错序列
- 每条样本：随机子序列 L=256 token，取前 N=5 张图像
- 图像位置用 `<;image>;` 标记
- 额外插入 `<;end-of-chunk>;` token 标记段落结束
- 后续图像被丢弃以节省算力

**ALIGN（1.8B 图文对）**

- 1.8B 图像 + alt-text 对
- 来自 Google 的网页数据
- 噪声大但量级巨大

**LTIP（Long Text & Image Pairs，312M 图文对）**

- Flamingo 团队自建数据集，312M 图像-文本对
- 目标是更好的质量和更长的描述文本

**VTP（Video & Text Pairs，27M 视频-文本对）**

- 27M 短视频，平均时长约 22 秒
- 配有人工筛选的句子级描述

**训练策略：** 多目标训练（multi-objective training）。对每个数据集的损失加权求和：

$$ \sum_{m=1}^{M} \lambda_m \cdot \mathbb{E}_{(x,y) \sim \mathcal{D}_m} \left[ -\sum_{\ell=1}^{L} \log p(y_\ell | y_{<\ell}, x) \right] $$

其中 $\lambda_m$ 是每个数据集的权重。M3W 的权重最高（因为它包含最自然的图文交错信号），LTIP 和 VTP 次之，ALIGN 再次之。

## 六、核心公式与数学直觉

### 6.1 条件概率建模

Flamingo 建模文本 $y$ 在视觉输入 $x$ 条件下的条件概率：

$$ p(y|x) = \prod_{\ell=1}^{L} p(y_\ell | y_{<\ell}, x) $$

这是一个标准的自回归语言模型公式，但关键区别是：** $x$ 是视觉输入（图像或视频）， $y$ 是文本输出**。LLM 的标准自回归公式 $p(y) = \prod p(y_\ell | y_{<\ell})$ 变成了**以视觉信息为条件的自回归生成**。

这个公式的一个隐含假设是：**视觉条件 $x$ 对每个生成步骤都是可访问的**——这正是通过 gated cross-attention 实现的效果。每个 token 生成时，cross-attention 层让模型能重新"查看"视觉 token。

### 6.2 Gated Cross-Attention 的数学形式

$$ \text{GA}(\mathbf{h}, \mathbf{v}) = \mathbf{h} + \tanh(\alpha) \cdot \text{FFN}\left( \text{CrossAttn}(\mathbf{h}, \mathbf{v}, \mathbf{v}) \right) $$

其中：
- $\mathbf{h}$ 是来自冻结 LLM 层的 hidden state（query）
- $\mathbf{v}$ 是来自 Perceiver Resampler 的视觉 token（key/value）
- $\text{CrossAttn}$ 是标准的 cross-attention： $\text{Softmax}\left(\frac{QK^T}{\sqrt{d}}\right)V$
- $\text{FFN}$ 是两层 MLP（dense layer + activation + dense layer）
- $\alpha$ 是可学习的标量，初始化均为 0
- $\tanh(\alpha)$ 限制了门控值始终在 $[-1, 1]$ 范围，训练开始时为 0

**残差结构的意义：** 如果去掉残差（直接把 cross-attention 输出加到主路径上），训练初期 LLM 的输出分布会受到破坏（ $\tanh$ 输出的值即使很小，累加也会扰动分布）。残差让模型先通过"干净路径"（原始 LLM）输出正确的文本分布，cross-attention 的输出只在视觉信息确实有用时才贡献信号。

### 6.3 Perceiver Resampler 的 attention

$$ \mathbf{v}_{\text{out}} = \text{CrossAttn}(\mathbf{q}_{\text{latent}}, [\mathbf{X}_f; \mathbf{q}_{\text{latent}}], [\mathbf{X}_f; \mathbf{q}_{\text{latent}}]) $$

其中 $\mathbf{q}_{\text{latent}}$ 是 64 个 learned latent queries， $\mathbf{X}_f$ 是展平的时空视觉特征。

将 learnt latents 拼接到 key/value 中（而非仅作为 query）是 Flamingo 的一个小设计选择，论文实验发现比标准 Perceiver（key/value 只用 X_f）效果稍好。直观解释：latent 自身在 cross-attention 中可以"看到"其他 latent，这一拼接实际上引入了 latent 之间的交互通道。

## 八、消融与对比

论文主要在 Appendix B.3 中报告消融实验。以下是核心发现（来自论文原文）：

### 8.1 Cross-attention vs Prefix-style

Cross-attention 层的视觉注入方式优于 prefix-style 的视觉拼接方式。原因如前所述：因果掩码导致 prefix 视觉 token 在生成过程中被稀释。

### 8.2 Gating（tanh 门控）的必要性

没有 tanh 门控的 cross-attention 在训练初期会严重破坏冻结 LLM 的输出分布。门控让 cross-attention 在学习视觉适配的同时，不干扰 LLM 已有的语言能力。

Figure 6 显示了不同层的 gating 绝对值随训练步数的演化：**所有层都逐渐学会使用视觉信息**，且门控值的绝对值随深度增加而增大（高层大于底层）。这说明高层特征对视觉信息的利用更积极，或者高层 hidden state 的抽象程度与视觉 token 的语义对应关系更强。

### 8.3 Perceiver latent 数量的影响

64 个 latent query 的设定是精度和计算开销之间的平衡。更少的 latent（如 32）损失信息，更多的 latent（如 128）增加计算开销但收益递减。

### 9.1 技术树中的位置

```text
│

       │
       └── [LLaVA-NeXT, 2024] ← 多尺度视觉 + 更高分辨率
```

Flamingo 处于 D2-8（冻结 LLM + 视觉适配器）的**根节点**位置。后续的 BLIP-2、LLaVA、MiniGPT-4 都是在这个范式下的变体。

### 9.2 Flamingo 继承了谁？

**Chinchilla（Hoffmann et al., 2022）：** Flamingo 使用的冻结 LLM 就是 DeepMind 同期发布的 Chinchilla（70B）。Chinchilla 在 compute-optimal 的条件下训练（更多 token、更少参数），语言能力和推理能力优于同规模的 GPT-3。

**ALIGN（Jia et al., 2021）：** Flamingo 使用了 ALIGN 的 1.8B 图文对数据集，以及 ALIGN-style 的对比学习来预训练 vision encoder。ALIGN 的噪声容忍训练策略让 Flamingo 可以从大规模 web 数据中学习。

**Perceiver（Jaegle et al., 2021）：** Perceiver Resampler 的设计直接继承自 Perceiver 架构——learned latent queries 做 cross-attention，把变长输入映射到固定输出。

**NFNet（Brock et al., 2021）：** Normalizer-Free ResNet，Flamingo 的 vision encoder。NFNet 去掉了 Batch Normalization，在大 batch 训练下更稳定。

**In-context Learning（GPT-3, Brown et al., 2020）：** Flamingo 的 few-shot 能力继承自 LLM 的 in-context learning 能力。Flamingo 证明了这种能力可以无缝扩展到视觉模态。

### 9.3 Flamingo 的先发优势与局限

Flamingo 是这篇论文中训练的和评估的最大模型版本，称为 Flamingo（或 Flamingo-80B）。以下是它的部分核心局限：

1. **专有性：** 代码和数据都未开源。模型权重也未发布。这直接催生了 OpenFlamingo 等开源复现工作。

2. **训练成本高：** 1536 TPU × 15 天，非大型机构不可复现。

3. **Perceiver Resampler 的复杂度：** 后续工作发现，一个更简单的桥接模块（如 BLIP-2 的 Q-Former 或 LLaVA 的简单线性层）也能达到类似的性能，说明 Perceiver 的复杂度可能不是必需的。

4. **视觉分辨率受限：** NFNet 的输入分辨率固定（通常 224×224 或 384×384），对于需要高分辨率细节的任务（如 OCR、细粒度分类）效果不好。

### 9.4 后续影响

Flamingo 的影响是范式级别的：

**OpenFlamingo（2023）：** 由 Stanford CRFM、UW、CMU 等联合发布的开源复现。使用了 OpenCLIP 的 vision encoder 和 LLaMA/MPT 作为 LLM。证明 Flamingo 的架构可以在开源组件上复现。

**BLIP-2（2023）：** Salesforce 提出 Q-Former（Querying Transformer），本质上和 Flamingo 的 Perceiver Resampler 异曲同工。区别在于 Q-Former 直接和冻结的 vision encoder + 冻结的 LLM 做两阶段训练，不训练 cross-attention 层。BLIP-2 证明了"冻结 LLM + 轻量桥接模块"范式的有效性。

**LLaVA（2023）：** 更进一步，用简单的线性投影层替代 cross-attention + perceiver，将视觉 token 直接映射到 LLM 的 embedding space。证明即使是极简单的适配器，配合 instruction tuning 也能取得很好的效果。

## 十、范式意义

Flamingo 的范式意义在于：**它首次证明了一个冻结的、纯文本预训练的大语言模型可以通过轻量的桥接模块获得视觉理解能力，而且 in-context learning 的能力可以自然地从文本扩展到多模态。**

具体来说，Flamingo 完成了三件事，每一件放在当时来看都是重要突破：

### 10.1 打破了"多模态模型必须从头训练"的认知

在 Flamingo 之前，主流的假设是：要做多模态理解，必须从零训练一个多模态模型（或者至少部分训练）。Flamingo 证明：**视觉和语言可以分开训练，再通过轻量桥接模块连接**。Vision encoder 和 LLM 都保持冻结，真正训练的只有桥接模块——参数量和训练成本大幅降低。

这不是（只是）工程上的优化。它说明了一个本质问题：**视觉理解和语言理解的核心能力可能是在训练中独立获得的，不需要多模态对齐信号也能学到。** 多模态信号只需要教模型"如何桥接"这两个独立的表示空间。

### 10.2 将 in-context learning 从文本扩展到多模态

GPT-3 的 in-context learning 能力是纯文本的：给几个文本样例，模型就能做新任务。Flamingo 证明了这种能力可以无缝扩展到图文交错输入的场景。

这意味着：**不需要为每个多模态任务训练一个单独的模型。** 一个 Flamingo 模型，通过不同的 few-shot prompt，可以做 captioning、VQA、visual dialogue、video QA 等完全不同的任务。每一次都不需要微调，只需要换 prompt 中的图文样例。

这个启示深刻影响了后续的所有开放域多模态模型——从 GPT-4V 到 Gemini，都采用了类似的"用一个通用模型做所有任务"的范式。

### 10.3 定义了"冻结 LLM + 适配器"的接口规范

Flamingo 确立了后续工作的设计空间：

1. **Vision encoder**：冻结还是微调？（Flamingo 冻结，后续工作有变体）
2. **桥接模块**：Perceiver Resampler / Q-Former / 简单线性层 / MLP？
3. **注入方式**：Cross-attention（Flamingo style）vs prefix（BLIP-2 Q-Former style）
4. **LLM**：冻结还是微调？（Flamingo 冻结，LLaVA 半冻结）
5. **训练数据**：图文交错 vs 图文对 vs 视觉 instruction

几乎所有后续的 MLLM（Multi-modal Large Language Model）工作都在这个设计空间内取不同的点。

### 10.4 分类：foundation node

在技术树中，Flamingo 是一个 **foundation node（基础节点）**——它是"冻结 LLM + 视觉适配器"范式的开创者。

它建立在 CLIP（双塔对齐）、GPT-3（in-context learning）、Chinchilla（compute-optimal LLM）等前驱工作上，但又开辟了一个全新的技术方向。后续的 BLIP-2、LLaVA、MiniGPT-4、IDEFICS 等都是在这个方向上的探索和改进，没有离开 Flamingo 定义的坐标系。

## 十一、局限与讨论

### 11.1 视觉分辨率受限

NFNet 的固定输入分辨率限制了 Flamingo 在高分辨率场景下的表现。后续的 LLaVA-NeXT 通过多尺度视觉特征解决这个问题，而 GPT-4V 则使用更先进的分辨率自适应机制。

### 11.2 桥接模块的复杂度冗余

Perceiver Resampler + Gated Cross-Attention 的两段式桥接结构被证明不是必需的。BLIP-2 用一个 Q-Former 完成两段桥接，LLaVA 甚至只用简单线性层。这引出一个问题：Flamingo 的桥接模块是否过度设计？或者，Perceiver 的复杂度在需要处理视频这种高度变长输入时才有价值？

### 11.3 专有性导致的复现困难

论文没有开源代码和模型权重，数据也是专有的（M3W、LTIP、VTP 均未发布）。这在一定程度上限制了 Flamingo 的社区影响力。相比之下，后续的 LLaVA 和 BLIP-2 的开源程度更高，因此在社区中的实际影响力更大。

### 11.4 训练成本

最大模型在 1536 TPU 上训练 15 天，按照 Google Cloud TPU v4 的定价，训练成本约在数百万美元级别（保守估计）。这远超大多数研究机构的预算，也解释了为什么 Flamingo 没有产生广泛的直接复现。

### 11.5 测试集泄露的风险

论文在 Appendix A.3.4 中描述了去重策略，确保训练数据和评估 benchmark 没有重叠。但这种基于 URL 和句子级别去重的方法可能不完美——文本描述虽然不同，但语义上重叠的场景仍然可能存在。

## 十三、小结

直白说，Flamingo 的价值不只是提出了一个能打的视觉-语言模型，而是开创了一个全新的技术范式：**冻结的 LLM + 轻量视觉适配器**。

在它之前，做多模态模型的思路是"视觉和语言一起训练"。这个思路的问题是成本太高、灵活性太低——每换一个任务就要重新训练或做昂贵的 fine-tuning。

在它之后，思路变成了"视觉适配器桥接到冻结的 LLM"。语言能力由预训练 LLM 提供（不需要多模态数据），视觉能力由预训练 vision encoder 提供（不需要多模态数据），桥接模块用多模态数据训练。**三层解耦，各司其职。**

这个转变靠的是两个核心洞察：

1. **LLM 的 in-context learning 能力可以扩展到多模态。** 只要视觉信息能按照 LLM 能理解的方式注入（cross-attention），LLM 就能像理解纯文本 prompt 一样理解多模态 prompt。

2. **视觉和语言的核心能力可以分开获取，再通过桥接组装。** 这打破了过去"多模态模型必须从头训练"的假设，极大降低了多模态模型的门槛。

在技术树中，Flamingo 是一个 foundation node。它所定义的"冻结 LLM + 视觉适配器"范式，直接催生了 BLIP-2、LLaVA、MiniGPT-4 等一系列后续工作，也深刻影响了 GPT-4V 和 Gemini 等闭源模型的设计思路。今天几乎所有的主流多模态大模型（MLLM），无论是开源还是闭源，都能在 Flamingo 的设计空间中找到自己的位置。
